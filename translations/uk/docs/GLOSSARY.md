## Основні поняття

**AI Agent** - Система, яка використовує ШІ для автономного мислення та дій. [Модуль 04](../04-tools/README.md)

**Chain** - Послідовність операцій, де вихід використовується для наступного кроку.

**Chunking** - Розбиття документів на менші частини. Типово: 300-500 токенів з перекриттям. [Модуль 03](../03-rag/README.md)

**Context Window** - Максимальна кількість токенів, яку модель може обробити. GPT-5.2: 400К токенів.

**Embeddings** - Числові вектори, що представляють значення тексту. [Модуль 03](../03-rag/README.md)

**Function Calling** - Модель генерує структуровані запити для виклику зовнішніх функцій. [Модуль 04](../04-tools/README.md)

**Hallucination** - Коли моделі генерують неправильну, але правдоподібну інформацію.

**Prompt** - Текстовий вхід у мовну модель. [Модуль 02](../02-prompt-engineering/README.md)

**Semantic Search** - Пошук за змістом з використанням ембеддингів, а не ключових слів. [Модуль 03](../03-rag/README.md)

**Stateful vs Stateless** - Stateless: без пам'яті. Stateful: зберігає історію розмови. [Модуль 01](../01-introduction/README.md)

**Tokens** - Основні текстові одиниці, які обробляють моделі. Впливає на вартість і обмеження. [Модуль 01](../01-introduction/README.md)

**Tool Chaining** - Послідовне виконання інструментів, де вихід впливає на наступний виклик. [Модуль 04](../04-tools/README.md)

## Компоненти LangChain4j

**AiServices** - Створює типобезпечні інтерфейси AI сервісів.

**OpenAiOfficialChatModel** - Уніфікований клієнт для моделей OpenAI та Azure OpenAI.

**OpenAiOfficialEmbeddingModel** - Створює ембеддинги за допомогою клієнта OpenAI Official (підтримує OpenAI та Azure OpenAI).

**ChatModel** - Основний інтерфейс для мовних моделей.

**ChatMemory** - Підтримує історію розмови.

**ContentRetriever** - Знаходить релевантні частини документів для RAG.

**DocumentSplitter** - Розбиває документи на фрагменти.

**EmbeddingModel** - Перетворює текст у числові вектори.

**EmbeddingStore** - Зберігає і отримує ембеддинги.

**MessageWindowChatMemory** - Підтримує рухоме вікно останніх повідомлень.

**PromptTemplate** - Створює повторно використовувані підказки з `{{variable}}` змінними.

**TextSegment** - Текстовий фрагмент з метаданими. Використовується в RAG.

**ToolExecutionRequest** - Представляє запит на виконання інструменту.

**UserMessage / AiMessage / SystemMessage** - Типи повідомлень у розмові.

## Концепції ШІ/МЛ

**Few-Shot Learning** - Надання прикладів у підказках. [Модуль 02](../02-prompt-engineering/README.md)

**Large Language Model (LLM)** - ШІ моделі, навчені на великих обсягах тексту.

**Reasoning Effort** - Параметр GPT-5.2, який контролює глибину мислення. [Модуль 02](../02-prompt-engineering/README.md)

**Temperature** - Контролює випадковість результату. Низьке = детерміноване, високе = креативне.

**Vector Database** - Спеціалізована база даних для ембеддингів. [Модуль 03](../03-rag/README.md)

**Zero-Shot Learning** - Виконання завдань без прикладів. [Модуль 02](../02-prompt-engineering/README.md)

## Guardrails - [Модуль 00](../00-quick-start/README.md)

**Defense in Depth** - Багаторівневий підхід безпеки, що поєднує guardrails на рівні додатку та фільтри безпеки провайдера.

**Hard Block** - Провайдер видає HTTP 400 помилку за серйозні порушення контенту.

**InputGuardrail** - Інтерфейс LangChain4j для перевірки користувацького вводу до досягнення LLM. Заощаджує кошти та час, блокуючи шкідливі підказки раніше.

**InputGuardrailResult** - Тип повернення перевірки guardrail: `success()` або `fatal("reason")`.

**OutputGuardrail** - Інтерфейс для перевірки відповідей ШІ перед поверненням користувачам.

**Provider Safety Filters** - Вбудовані фільтри контенту від AI провайдерів (наприклад, GitHub Models), що ловлять порушення на рівні API.

**Soft Refusal** - Модель ввічливо відмовляється відповідати без видачі помилки.

## Агентний модуль - [Модуль 05](../05-mcp/README.md)

**@Agent Annotation** - Позначає інтерфейси як AI агенти з декларативним визначенням поведінки.

**Agent Listener** - Хук для моніторингу виконання агента через `beforeAgentInvocation()` та `afterAgentInvocation()`.

**Agentic Scope** - Спільна пам'ять, де агенти зберігають результати за ключем `outputKey` для споживання іншими агентами.

**AgenticServices** - Фабрика для створення агентів через `agentBuilder()` та `supervisorBuilder()`.

**Conditional Workflow** - Маршрутизація за умовами до різних спеціалістів-агентів.

**Human-in-the-Loop** - Шаблон робочого процесу з додаванням людських контрольних точок для схвалення або перегляду контенту.

**langchain4j-agentic** - Maven залежність для декларативного створення агентів (експериментальна).

**Loop Workflow** - Ітеративне виконання агента до виконання умови (наприклад, оцінка якості ≥ 0.8).

**outputKey** - Параметр анотації агента, що вказує, куди зберігати результати в Agentic Scope.

**Parallel Workflow** - Одночасний запуск кількох агентів для незалежних завдань.

**Response Strategy** - Як супервайзер формує фінальну відповідь: LAST, SUMMARY або SCORED.

**Sequential Workflow** - Виконання агентів послідовно, де вихід переходить на наступний крок.

**Supervisor Agent Pattern** - Складний агентний шаблон, де супервайзер LLM динамічно вирішує, яких підагентів викликати.

## Протокол контексту моделі (MCP) - [Модуль 05](../05-mcp/README.md)

**langchain4j-mcp** - Maven залежність для інтеграції MCP у LangChain4j.

**MCP** - Протокол контексту моделі: стандарт для підключення AI додатків до зовнішніх інструментів. Створив один раз — користуйся скрізь.

**MCP Client** - Додаток, що підключається до MCP серверів для пошуку та використання інструментів.

**MCP Server** - Сервіс, що надає інструменти через MCP з чіткими описами та схемами параметрів.

**McpToolProvider** - Компонент LangChain4j, який обгортає MCP інструменти для використання в AI сервісах і агентах.

**McpTransport** - Інтерфейс для MCP комунікації. Імплементації включають Stdio та HTTP.

**Stdio Transport** - Транспорт локального процесу через stdin/stdout. Корисний для доступу до файлової системи чи CLI інструментів.

**StdioMcpTransport** - LangChain4j імплементація, що запускає MCP сервер як дочірній процес.

**Tool Discovery** - Клієнт запитує сервер про доступні інструменти з описами та схемами.
