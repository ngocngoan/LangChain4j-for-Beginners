## तुम्ही काय शिकाल

जर तुम्ही क्विक स्टार्ट पूर्ण केले असेल, तर तुम्हाला प्रॉम्प्ट पाठवणे आणि प्रतिसाद मिळवणे कसे आहे ते पाहिले असेल. हे त्याची पायाभूत गोष्ट आहे, पण वास्तविक अँप्लिकेशन्सना अधिक काही हवे असते. हा मॉड्यूल तुम्हाला कसे संभाषणात्मक AI तयार करायचे ते शिकवेल जे संदर्भ लक्षात ठेवते आणि स्थिती कायम ठेवते — ज्यामुळे एकदा वापरल्या गेलेल्या डेमो आणि उत्पादनासाठी तयार अँप्लिकेशन यामध्ये फरक होतो.

आपण संपूर्ण मार्गदर्शनात Azure OpenAI चे GPT-5.2 वापरणार आहोत कारण त्याच्या प्रगत तर्कशक्तीमुळे वेगवेगळ्या पॅटर्न्सचे वर्तन अधिक स्पष्ट दिसून येते. जेव्हा तुम्ही मेमरी जोडता, तेव्हा फरक स्पष्ट दिसून येतो. यामुळे प्रत्येक घटक तुमच्या अँप्लिकेशनला काय देतो हे समजणे सोपे होते.

तुम्ही एकच अँप्लिकेशन तयार कराल जे दोन्ही पॅटर्न दाखवेल:

**स्टेटलेस चॅट** - प्रत्येक विनंती स्वतंत्र असते. मॉडेलला मागील संदेशांची कोणतीही आठवण नसते. हा पॅटर्न तुम्ही क्विक स्टार्टमध्ये वापरला होता.

**स्टेटफुल संभाषण** - प्रत्येक विनंतीमध्ये संभाषणाचा इतिहास असतो. मॉडेल अनेक टर्नसाठी संदर्भ राखते. हाच उत्पादनाच्या अँप्लिकेशन्ससाठी आवश्यक आहे.

## पूर्वअटी

- Azure सदस्यता ज्यात Azure OpenAI प्रवेश आहे
- Java 21, Maven 3.9+
- Azure CLI (https://learn.microsoft.com/en-us/cli/azure/install-azure-cli)
- Azure Developer CLI (azd) (https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/install-azd)

> **नोट:** Java, Maven, Azure CLI आणि Azure Developer CLI (azd) आधीच devcontainer मध्ये स्थापित आहेत.

> **नोट:** हा मॉड्यूल Azure OpenAI वरील GPT-5.2 वापरतो. तैनाती `azd up` द्वारे स्वयंचलितरित्या कॉन्फिगर केली आहे - कोडमध्ये मॉडेल नावात बदल करू नका.

## मूलभूत समस्या समजून घेणे

भाषा मॉडेल्स स्टेटलेस असतात. प्रत्येक API कॉल स्वतंत्र असतो. जर तुम्ही "माझं नाव जॉन आहे" म्हटलंत आणि नंतर विचारलं "माझं नाव काय आहे?", तर मॉडेलला कळत नाही की तुम्ही स्वतःची ओळख करून दिली होती. ते प्रत्येक विनंती पहिल्या संभाषणासारखी समजते.

हे सोप्या Q&A साठी ठीक आहे, पण वास्तविक अँप्लिकेशन्ससाठी निरुपयोगी आहे. ग्राहक सेवा बॉट्सना ते तुम्ही काय सांगितलं त्याची आठवण ठेवावी लागते. वैयक्तिक सहाय्यकांना संदर्भ हवे असतात. कोणत्याही बहु-टर्न संभाषणासाठी मेमरी आवश्यक आहे.

*स्टेटलेस (स्वतंत्र कॉल्स) आणि स्टेटफुल (संदर्भ जाणून) संभाषणांमधील फरक*

## टोकन्स समजून घेणे

संभाषणात उडी मारण्याआधी, टोकन्स समजून घेणे महत्त्वाचे आहे - मुलतत्वातील मजकूर एकके ज्यावर भाषा मॉडेल्स प्रक्रिया करतात:

<img src="../../../translated_images/mr/token-explanation.c39760d8ec650181.webp" alt="टोकन स्पष्टीकरण" width="800"/>

*मजकूर कसा टोकन्समध्ये तूटतो याचा उदाहरण - "I love AI!" हे 4 स्वतंत्र प्रक्रिया युनिट्स होते*

टोकन्स AI मॉडेल्सना मजकूर मोजण्याचा आणि प्रक्रिया करण्याचा मार्ग आहे. शब्द, विरामचिन्हे, आणि अगदी जागाही टोकन्स असू शकतात. तुमच्या मॉडेलकडे एक टोकन मर्यादा असते (GPT-5.2 साठी 400,000, ज्यामध्ये 272,000 इनपुट टोकन्स आणि 128,000 आऊटपुट टोकन्स असू शकतात). टोकन्स समजून घेणे तुम्हाला संभाषणाची लांबी आणि खर्च व्यवस्थापित करण्यात मदत करते.

## मेमरी कशी कार्य करते

चॅट मेमरी स्टेटलेस समस्या सोडवते कारण ती संभाषणाचा इतिहास राखते. तुमची विनंती मॉडेलकडे पाठवण्यापूर्वी, फ्रेमवर्क संबंधित मागील संदेश ऍड करते. जेव्हा तुम्ही विचारता "माझं नाव काय आहे?", तेव्हा सिस्टम संपूर्ण संभाषणाचा इतिहास पाठवते, ज्यामुळे मॉडेलला कळते की तुम्ही आधी "माझं नाव जॉन आहे" म्हणालात.

LangChain4j मेमरी अंमलबजावण्या देते ज्या हे स्वयंचलितपणे हाताळतात. तुम्ही किती संदेश ठेवायचे ते निवडता आणि फ्रेमवर्क संदर्भ विंडो व्यवस्थापित करते.

<img src="../../../translated_images/mr/memory-window.bbe67f597eadabb3.webp" alt="मेमरी विंडो संकल्पना" width="800"/>

*MessageWindowChatMemory अलीकडील संदेशांसाठी स्लायडिंग विंडो ठेवते, जुनाट संदेश आपोआप ड्रॉप करते*

## हे LangChain4j कसे वापरते

हा मॉड्यूल क्विक स्टार्ट विस्तारतो Spring Boot एकत्र करून आणि संभाषणात्मक मेमरी जोडून. भाग कसे जुळतात ते खालीलप्रमाणे:

**डिपेंडन्सीज** - दोन LangChain4j लायब्ररी जोडाः

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```

**चॅट मॉडेल** - Azure OpenAI ला Spring bean म्हणून कॉन्फिगर करा ([LangChainConfig.java](../../../01-introduction/src/main/java/com/example/langchain4j/config/LangChainConfig.java)):

```java
@Bean
public OpenAiOfficialChatModel openAiOfficialChatModel() {
    return OpenAiOfficialChatModel.builder()
            .baseUrl(azureEndpoint)
            .apiKey(azureApiKey)
            .modelName(deploymentName)
            .timeout(Duration.ofMinutes(5))
            .maxRetries(3)
            .build();
}
```

बिल्डर `azd up` ने सेट केलेल्या पर्यावरण चलांमधून क्रेडेन्शियल वाचतो. `baseUrl` तुमच्या Azure endpoint वर सेट केल्याने OpenAI क्लायंट Azure OpenAI सोबत काम करतो.

**संभाषण मेमरी** - MessageWindowChatMemory वापरून चॅट इतिहास ट्रॅक करा ([ConversationService.java](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java)):

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("My name is John"));
memory.add(AiMessage.from("Nice to meet you, John!"));

memory.add(UserMessage.from("What's my name?"));
AiMessage aiMessage = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage);
```

`withMaxMessages(10)` वापरून शेवटचे 10 संदेश ठेवण्यासाठी मेमरी तयार करा. वापरकर्ता आणि AI संदेश `UserMessage.from(text)` आणि `AiMessage.from(text)` सारख्या टाइप केलेल्या रॅपरने जोडा. `memory.messages()` वापरून इतिहास मिळवा आणि तो मॉडेलला पाठवा. सेवा संभाषण ID नुसार वेगवेगळ्या मेमरी इंस्टन्सेस साठवते, ज्यामुळे एकापेक्षा अधिक वापरकर्ते एकाच वेळी चॅट करू शकतात.

> **🤖 [GitHub Copilot](https://github.com/features/copilot) चॅटसह प्रयत्न करा:** [`ConversationService.java`](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java) उघडा आणि विचारा:
> - "विंडो भरल्यावर MessageWindowChatMemory कोणते संदेश काढते कसा ठरवतो?"
> - "मी इन-मेमरीऐवजी डेटाबेस वापरून कस्टम मेमरी साठवणूक तयार करू शकतो का?"
> - "जुन्या संभाषणाचा इतिहास संक्षेप करण्यासाठी मी सहारा (सुमरी) कसा जोडू शकतो?"

स्टेटलेस चॅट एंडपॉइंट मेमरी पूर्णपणे वगळतो - फक्त `chatModel.chat(prompt)` क्विक स्टार्ट प्रमाणे. स्टेटफुल एंडपॉइंट मेमरीमध्ये संदेश जोडतो, इतिहास मिळवतो, आणि त्याचा संदर्भ प्रत्येक विनंतीसोबत समाविष्ट करतो. मॉडेल कॉन्फिगरेशन सारखेच, पॅटर्न वेगळे.

## Azure OpenAI इन्फ्रास्ट्रक्चर तैनात करा

**Bash:**
```bash
cd 01-introduction
azd up  # सदस्यत्व आणि स्थान निवडा (eastus2 शिफारस केली आहे)
```

**PowerShell:**
```powershell
cd 01-introduction
azd up  # सदस्यता आणि स्थान निवडा (पूर्व युएस 2 शिफारस केली आहे)
```

> **नोट:** जर तुम्हाला टाइमआउट त्रुटी आली (`RequestConflict: Cannot modify resource ... provisioning state is not terminal`), तर फक्त पुन्हा `azd up` चालवा. Azure संसाधने अजून बॅकग्राउंड मध्ये तैनात होत असू शकतात, आणि पुन्हा प्रयत्नाने संसाधने अंतिम स्थिती गाठल्यावर तैनाती पूर्ण होते.

यामुळे:
1. Azure OpenAI संसाधन GPT-5.2 आणि text-embedding-3-small मॉडेल्ससह तैनात होईल
2. प्रोजेक्टच्या रूट मध्ये क्रेडेन्शियलसह `.env` फाइल स्वयंचलित तयार होईल
3. सर्व आवश्यक पर्यावरण चल सेट होतील

**तैनाती समस्यांबाबत?** [इन्फ्रास्ट्रक्चर README](infra/README.md) पहा ज्यात सबडोमेन नावातील संघर्ष, मॅन्युअल Azure पोर्टल तैनाती पावले, आणि मॉडेल कॉन्फिगरेशन मार्गदर्शन यांचा तपशील आहे.

> ```

**अँप्लिकेशन्स सुरू करा:**

**पर्याय 1: Spring Boot डॅशबोर्ड वापरून (VS Code वापरकर्त्यांसाठी शिफारस)**

डेव्ह कंटेनरमध्ये Spring Boot डॅशबोर्ड विस्तार समाविष्ट आहे, जो सर्व Spring Boot अँप्लिकेशन्स व्यवस्थापित करण्यासाठी दृश्यमान इंटरफेस पुरवतो. VS Code च्या डाव्या बाजूला असलेल्या Activity Bar मध्ये (Spring Boot चिन्ह पहा) हा सापडतो.

Spring Boot डॅशबोर्डमधून तुम्ही:
- कार्यस्थानातील सर्व उपलब्ध Spring Boot अँप्लिकेशन्स पाहू शकता
- एक क्लिकने अँप्लिकेशन्स चालू/बंद करू शकता
- रियल-टाइममध्ये लॉग्स पाहू शकता
- अँप्लिकेशन स्थिती पाहू शकता

फक्त "introduction" जवळील प्ले बटणावर क्लिक करा आणि हा मॉड्यूल सुरू करा, किंवा सर्व मॉड्यूल एकत्र सुरू करा.

*डॅशबोर्ड जेथे दोन्ही Simple Chat (स्टेटलेस) आणि Conversational Chat (स्टेटफुल) पर्याय आहेत*

### स्टेटलेस चॅट (डाव्या पॅनेल)

हे पहिले प्रयत्न करा. "माझं नाव जॉन आहे" असे विचारा आणि लगेच "माझं नाव काय आहे?" विचारा. मॉडेलला आठवण राहणार नाही कारण प्रत्येक संदेश स्वतंत्र आहे. हे बेसिक भाषा मॉडेल इंटीग्रेशनमधील मूलभूत समस्या दर्शवते - संभाषणाचा संदर्भ नाही.

<img src="../../../translated_images/mr/simple-chat-stateless-demo.13aeb3978eab3234.webp" alt="स्टेटलेस चॅट डेमो" width="800"/>

*AI पूर्वीच्या संदेशातून तुमचं नाव आठवत नाही*

### स्टेटफुल चॅट (उजव्या पॅनेल)

आता तेच सिरीअल येथे प्रयत्न करा. "माझं नाव जॉन आहे" आणि मग "माझं नाव काय आहे?" विचारल्यावर ते लक्षात ठेवते. फरक म्हणजे MessageWindowChatMemory — ते संभाषणाचा इतिहास राखते आणि प्रत्येक विनंतीसोबत तो संदर्भ पाठवते. हे उत्पादन स्तरावरले संभाषणात्मक AI कसे कार्य करते याचे उदाहरण आहे.

<img src="../../../translated_images/mr/conversational-chat-stateful-demo.e5be9822eb23ff59.webp" alt="स्टेटफुल चॅट डेमो" width="800"/>

*AI पूर्वीच्या संभाषणातून तुमचं नाव आठवत आहे*
