## व्हिडिओ वॉकथ्रू

हा थेट सत्र पाहा जो तुम्हाला या मॉड्यूलसह प्रारंभ करण्यास मदत करतो: [LangChain4j सह प्रारंभ करणे - थेट सत्र](https://www.youtube.com/live/nl_troDm8rQ?si=6b85S8xGjWnT2fX9)

## तुम्ही काय शिकणार आहात

जर तुम्ही क्विक स्टार्ट पूर्ण केले असेल तर, तुम्हाला कसे प्रांप्ट पाठवायचे आणि प्रतिसाद कसा मिळवायचा हे बघितले असेल. तो पाया आहे, पण वास्तविक अ‍ॅप्लिकेशन्सना अधिक गोष्टींची गरज असते. हा मॉड्यूल तुम्हाला कसे संवादात्मक AI तयार करायचे हे शिकवतो जे संदर्भ लक्षात ठेवते आणि स्टेट राखते - ज्यामुळे एक वेगळा डेमो आणि उत्पादनासाठी तयार अ‍ॅप्लिकेशन यामध्ये फरक पडतो.

या मार्गदर्शिकेत आम्ही Azure OpenAI चे GPT-5.2 वापरणार आहोत कारण त्याच्या प्रगत तर्कशक्तीमुळे वेगवेगळ्या पॅटर्न्सचे वर्तन अधिक स्पष्ट होते. जेव्हा तुम्ही मेमरी जोडता, तेव्हा तुम्हाला फरक स्पष्ट दिसतो. हे प्रत्येक घटक तुमच्या अ‍ॅप्लिकेशनमध्ये काय आणतो हे समजण्यास सुलभ करते.

तुम्ही एकच अ‍ॅप्लिकेशन तयार कराल जे दोन्ही पॅटर्न्स दाखवते:

**स्टेटलेस चॅट** - प्रत्येक विनंती स्वतंत्र असते. मॉडेलला मागील संदेशांची आठवण नसते. हा तोच पॅटर्न आहे ज्याचा तुम्ही क्विक स्टार्टमध्ये वापर केला होता.

**स्टेटफुल संभाषण** - प्रत्येक विनंतीत संवाद इतिहास समाविष्ट असतो. मॉडेल अनेक टर्न्स मधून संदर्भ राखते. हेच उत्पादन अ‍ॅप्लिकेशन्सना आवश्यक आहे.

## पूर्वअट

- Azure सब्सक्रिप्शन ज्यामध्ये Azure OpenAI प्रवेश आहे
- Java 21, Maven 3.9+
- Azure CLI (https://learn.microsoft.com/en-us/cli/azure/install-azure-cli)
- Azure Developer CLI (azd) (https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/install-azd)

> **नोट:** Java, Maven, Azure CLI आणि Azure Developer CLI (azd) दिलेल्या devcontainer मध्ये आधीच स्थापित आहेत.

> **नोट:** हा मॉड्यूल Azure OpenAI वर GPT-5.2 वापरतो. `azd up` द्वारे डिप्लॉयमेंट आपोआप कॉन्फिगर होते - कोडमधील मॉडेल नाव बदलू नका.

## मूलभूत समस्या समजून घेणे

भाषा मॉडेल्स स्टेटलेस असतात. प्रत्येक API कॉल स्वतंत्र असतो. जर तुम्ही "माझं नाव जॉन आहे" पाठवलं आणि नंतर "माझं नाव काय आहे?" विचारलं, तर मॉडेलला माहिती नसते की तुम्ही आधी काय बोललं होतं. तो प्रत्येक विनंतीला जणू काही ती पहिलीच संभाषण आहे असं समजतो.

हे सोप्या प्रश्न-उत्तरांसाठी ठीक आहे पण वास्तविक अ‍ॅप्लिकेशन्ससाठी निरुपयोगी आहे. ग्राहक सेवा बॉटला काय तुम्ही सांगितले ते लक्षात ठेवावे लागते. वैयक्तिक सहाय्यकांना संदर्भ आवश्यक आहे. कोणतेही बहु-टर्न संभाषण स्मृतीची गरज असते.

*स्टेटलेस (स्वतंत्र कॉल्स) आणि स्टेटफुल (संदर्भ-जाणकार) संभाषणांमधील फरक*

## टोकन्स समजून घेणे

संवादांकडे जाण्यापूर्वी, टोकन्स समजून घेणे महत्त्वाचे आहे - भाषा मॉडेल्स जे मूलभूत मजकूराच्या घटकांवर प्रक्रिया करतात:

<img src="../../../translated_images/mr/token-explanation.c39760d8ec650181.webp" alt="टोकन स्पष्टीकरण" width="800"/>

*"I love AI!" हा मजकूर चार स्वतंत्र टोकन्समध्ये कसा तुटतो याचा उदाहरण*

टोकन्स हे AI मॉडेल्स ज्यामुळे मजकूर मोजतात आणि प्रक्रिया करतात. शब्द, विरामचिन्हे, आणि अगदी रिकाम्या जागाही टोकन्स असू शकतात. तुमच्या मॉडेलला एकावेळी किती टोकन्स प्रक्रिया करता येतात याची मर्यादा आहे (GPT-5.2 साठी 400,000, ज्यामध्ये 272,000 इनपुट आणि 128,000 आउटपुट टोकन्स आहेत). टोकन्स समजण्याने संभाषणाची लांबी आणि किंमत व्यवस्थापित करणे सोपे होते.

## मेमरी कशी कार्य करते

चॅट मेमरी स्टेटलेस समस्येचे समाधान करते जेव्हा संभाषण इतिहास राखला जातो. मॉडेलला तुमची विनंती पाठवण्यापूर्वी, फ्रेमवर्क संबंधित मागील संदेश जोडतो. जेव्हा तुम्ही "माझं नाव काय आहे?" विचारता, तेव्हा संपूर्ण संभाषण इतिहास मॉडेलला पाठवला जातो, ज्यामुळे त्याला लक्षात राहील की तुम्ही आधी "माझं नाव जॉन आहे" असं सांगितलं होतं.

LangChain4j मेमरीच्या अंमलबजावणी प्रदान करते जी हे स्वयंचलितपणे हाताळते. तुम्ही किती संदेश ठेवायचे ते ठरवता आणि फ्रेमवर्क संदर्भ विंडो व्यवस्थापित करतो.

<img src="../../../translated_images/mr/memory-window.bbe67f597eadabb3.webp" alt="मेमरी विंडो संकल्पना" width="800"/>

*MessageWindowChatMemory अलीकडील संदेशांचा एक स्लायडिंग विंडो राखते, जुने आपोआप काढून टाकते*

## हे LangChain4j कसे वापरते

हा मॉड्यूल क्विक स्टार्टमध्ये स्प्रिंग बूट समाकलित करून संभाषण मेमरी जोडतो. घटक कसे जुळतात ते खालीलप्रमाणे:

**आदाता (Dependencies)** - दोन LangChain4j लायब्ररी जोडाव्यात:

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```

**चॅट मॉडेल** - Azure OpenAI ला स्प्रिंग बीन म्हणून कॉन्फिगर करा ([LangChainConfig.java](../../../01-introduction/src/main/java/com/example/langchain4j/config/LangChainConfig.java)):

```java
@Bean
public OpenAiOfficialChatModel openAiOfficialChatModel() {
    return OpenAiOfficialChatModel.builder()
            .baseUrl(azureEndpoint)
            .apiKey(azureApiKey)
            .modelName(deploymentName)
            .timeout(Duration.ofMinutes(5))
            .maxRetries(3)
            .build();
}
```

बिल्डर `azd up` द्वारे सेट केलेल्या वातावरण चलांमधून क्रेडेन्शियल्स वाचतो. `baseUrl` मध्ये तुमचा Azure endpoint सेट केल्याने OpenAI क्लायंट Azure OpenAI सह कार्य करतो.

**संभाषण मेमरी** - MessageWindowChatMemory वापरून चॅट इतिहास ट्रॅक करा ([ConversationService.java](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java)):

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("My name is John"));
memory.add(AiMessage.from("Nice to meet you, John!"));

memory.add(UserMessage.from("What's my name?"));
AiMessage aiMessage = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage);
```

`withMaxMessages(10)` वापरून मेमरी तयार करा ज्यामुळे मागील 10 संदेश राहतील. वापरकर्ता आणि AI संदेश टाइप केलेल्या रॅपर वापरून जोडा: `UserMessage.from(text)` आणि `AiMessage.from(text)`. `memory.messages()` वापरून इतिहास मिळवा आणि मॉडेलला पाठवा. प्रत्येक संभाषण ID साठी स्वतंत्र मेमरी साठवली जाते, ज्यामुळे अनेक वापरकर्ते एकाच वेळी चॅट करू शकतात.

> **🤖 [GitHub Copilot](https://github.com/features/copilot) Chat सह प्रयत्न करा:** [`ConversationService.java`](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java) उघडा आणि विचारा:
> - "MessageWindowChatMemory विंडो भरल्यावर कोणते संदेश काढते कसे ठरवते?"
> - "मी इन-मेमरीऐवजी डेटाबेस वापरून कस्टम मेमरी स्टोरेज कशी अमलात आणू शकतो?"
> - "जुना संभाषण इतिहास संक्षेपित करण्यासाठी सारांश कसा जोडता येईल?"

स्टेटलेस चॅट एन्डपॉइंट मध्ये मेमरी वापरली जात नाही - फक्त `chatModel.chat(prompt)` क्विक स्टार्ट प्रमाणे. स्टेटफुल एन्डपॉइंटमध्ये मेमरी कडून संदेश जोडले जातात, इतिहास मिळवला जातो आणि प्रत्येक विनंतीसह संदर्भ समाविष्ट केला जातो. मॉडेल कॉन्फिगरेशन सारखेच, पण वेगळ्या पॅटर्न्स.

## Azure OpenAI इन्फ्रास्ट्रक्चर डिप्लॉय करा

**Bash:**
```bash
cd 01-introduction
azd up  # सदस्यता आणि स्थान निवडा (eastus2 शिफारस केलेले)
```

**PowerShell:**
```powershell
cd 01-introduction
azd up  # सदस्यता आणि स्थान निवडा (eastus2 शिफारस केली आहे)
```

> **नोट:** जर तुम्हाला टाइमआउट एरर (`RequestConflict: Cannot modify resource ... provisioning state is not terminal`) येत असेल, तर फक्त पुन्हा `azd up` चालवा. Azure संसाधने अजूनही मागोमाग प्रोव्हिजनिंग होते, पुन्हा प्रयत्न केल्याने डिप्लॉयमेंट पूर्ण होईल जेव्हा संसाधने टर्मिनल स्थितीत येतील.

हे खालील गोष्टी करेल:
1. GPT-5.2 आणि text-embedding-3-small मॉडेल्ससह Azure OpenAI रिसोर्स डिप्लॉय करेल
2. प्रोजेक्ट रूटमध्ये क्रेडेन्शियल्ससह `.env` फाइल आपोआप तयार करेल
3. आवश्यक असलेल्या सर्व वातावरण चल सेट करेल

**डिप्लॉयमेंटमध्ये अडचण येत आहे?** अधिक तपशीलांसाठी [Infrastructure README](infra/README.md) पहा, ज्यात सबडोमेन नावाच्या संघर्षांचे निराकरण, मॅन्युअल Azure पोर्टल डिप्लॉयमेंट पावले, आणि मॉडेल कॉन्फिगरेशन मार्गदर्शन आहे.

**डिप्लॉयमेंट यशस्वी आहे का ते तपासा:**

**Bash:**
```bash
cat ../.env  # AZURE_OPENAI_ENDPOINT, API_KEY इत्यादी दाखवले पाहिजे.
```

**PowerShell:**
```powershell

```

> **नोट:** `azd up` आदेश आपोआप `.env` फाइल तयार करतो. जर नंतर अपडेट करायचा असेल, तर तुम्ही `.env` फाइल मॅन्युअली संपादित करू शकता किंवा खालील आदेश वापरून पुन्हा तयार करू शकता:
>
> **Bash:**
> ```bash
> cd ..
> bash .azd-env.sh
> ```
>
> **PowerShell:**
> ```powershell
> cd ..
> .\.azd-env.ps1
> ```

**अ‍ॅप्लिकेशन्स सुरु करा:**

**पर्याय 1: Spring Boot डॅशबोर्ड वापरून (VS Code वापरकर्त्यांसाठी शिफारस)**

devcontainer मध्ये Spring Boot डॅशबोर्ड विस्तार समाविष्ट आहे, जो सर्व Spring Boot अ‍ॅप्लिकेशन्स व्यवस्थापित करण्यासाठी दृश्य इंटरफेस प्रदान करतो. तुम्हाला हा VS Code च्या डावे भागातील Activity Bar मध्ये Spring Boot आयकॉन दिसेल.

Spring Boot डॅशबोर्डमधून तुम्ही:
- कार्यक्षेत्रातील सर्व Spring Boot अ‍ॅप्लिकेशन्स पाहू शकता
- एक क्लिक करून अ‍ॅप्लिकेशन्स सुरू किंवा थांबवू शकता
- अ‍ॅप्लिकेशन लॉग्स रिअल-टाइममध्ये पाहू शकता
- अ‍ॅप्लिकेशन स्थिती पाहू शकता

"introduction" जवळील प्ले बटण क्लिक करा हा मॉड्यूल सुरू करण्यासाठी अथवा सर्व मॉड्यूल एकत्र सुरू करा.


दोन्ही स्क्रिप्ट्स आपोआप रूट `.env` फाइलमधून पर्यावरण चल लोड करतात आणि जर JARs अस्तित्वात नसतील तर तयार करतात.

> **नोट:** जर तुम्हाला सर्व मॉड्यूल्स मॅन्युअली बिल्ड करायचे असल्यास आधी:
>
> **Bash:**
> ```bash
> cd ..  # Go to root directory
> mvn clean package -DskipTests
> ```
>
> **PowerShell:**
> ```powershell
> cd ..  # Go to root directory
> mvn clean package -DskipTests
> ```

तुमच्या ब्राऊझरमध्ये http://localhost:8080 उघडा.

**थांबवण्यासाठी:**

**Bash:**
```bash
./stop.sh  # हा फक्त मॉड्यूल आहे
# किंवा
cd .. && ./stop-all.sh  # सर्व मॉड्यूल्स
```

**PowerShell:**
```powershell
.\stop.ps1  # फक्त हा मॉड्यूल
# किंवा
cd ..; .\stop-all.ps1  # सर्व मॉड्यूल
```


## अ‍ॅप्लिकेशन वापरणे

हे अ‍ॅप्लिकेशन दोन वेगळ्या चॅट अंमलबजावण्या बाजूने बाजूला देतो.

<img src="../../../translated_images/mr/home-screen.121a03206ab910c0.webp" alt="अ‍ॅप्लिकेशन होम स्क्रीन" width="800"/>

*डॅशबोर्ड ज्यात सोपी चॅट (स्टेटलेस) आणि संवादात्मक चॅट (स्टेटफुल) पर्याय दिसतात*

### स्टेटलेस चॅट (डावा पॅनेल)

हे आधी प्रयत्न करा. "माझं नाव जॉन आहे" विचारून नंतर लगेच "माझं नाव काय आहे?" विचारा. मॉडेलला आठवण राहणार नाही कारण प्रत्येक संदेश स्वतंत्र आहे. हा मूलभूत समस्या आहे भाषा मॉडेल्सच्या साध्या एकत्रीकरणाची - संभाषणाचा संदर्भ नाही.

<img src="../../../translated_images/mr/simple-chat-stateless-demo.13aeb3978eab3234.webp" alt="स्टेटलेस चॅट डेमो" width="800"/>

*AIला मागील संदेशामधून तुमचं नाव आठवत नाही*

### स्टेटफुल चॅट (उजवा पॅनेल)

आता इथे तेच क्रम वापरून पाहा. "माझं नाव जॉन आहे" मग "माझं नाव काय आहे?" विचारल्यावर लक्षात राहते. फरक म्हणजे MessageWindowChatMemory - तो संभाषण इतिहास राखतो आणि प्रत्येक विनंतीसोबत तो संदर्भ देतो. असाच उत्पादन संघ संवादात्मक AI कसा काम करतो.

<img src="../../../translated_images/mr/conversational-chat-stateful-demo.e5be9822eb23ff59.webp" alt="स्टेटफुल चॅट डेमो" width="800"/>

*AIला संभाषणात पूर्वी सांगितलेलं तुमचं नाव लक्षात आहे*

दोन्ही पॅनेल GPT-5.2 मॉडेल वापरतात. फरक फक्त मेमरीचा आहे. त्यामुळे मेमरी तुमच्या अ‍ॅप्लिकेशनसाठी काय आणते ते स्पष्ट होते आणि ते खऱ्या वापरासाठी का महत्त्वाचे आहे हे कळते.
