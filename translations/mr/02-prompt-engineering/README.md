## आपण काय शिकाल

<img src="../../../translated_images/mr/what-youll-learn.c68269ac048503b2.webp" alt="आपण काय शिकाल" width="800"/>

मागील मॉड्यूलमध्ये, आपण पाहिले कसे मेमरी संवादात्मक AI सक्षम करते आणि GitHub मॉडेल वापरून मूलभूत संवाद साधला. आता आपण प्रॉम्प्ट्सवर लक्ष केंद्रित करू — म्हणजे विचारलेले प्रश्न — Azure OpenAI च्या GPT-5.2 वापरून. आपण आपले प्रॉम्प्ट कसे रचता यामुळे मिळणाऱ्या प्रतिसादांच्या गुणवत्तेवर मोठा परिणाम होतो. आम्ही प्रॉम्प्टिंग तंत्रांची पुनरावलोकन करून सुरुवात करू, नंतर GPT-5.2च्या क्षमता पूर्णपणे वापरणाऱ्या आठ प्रगत पद्धतींकडे जाऊ.

आम्ही GPT-5.2 वापरणार आहोत कारण यात तर्कशक्ती नियंत्रण आहे - तुम्ही मॉडेलला उत्तर देण्यापूर्वी किती विचार करायचा ते सांगू शकता. यामुळे विविध प्रॉम्प्टिंग धोरणे अधिक स्पष्ट होतात आणि तुम्हाला कोणती पद्धत कधी वापरायची हे समजायला मदत होते. GitHub मॉडेलच्या तुलनेत Azure ची कमी रेट लिमिट्स देखील फायदेशीर ठरतील.

## पूर्वतयारी

- मॉड्यूल 01 पूर्ण केलेले असणे (Azure OpenAI संसाधने तैनात केलेली असणे)
- `.env` फाइल मूळ निर्देशिकेत Azure क्रेडेन्शियल्ससह (मॉड्यूल 01 मध्ये `azd up` वापरून तयार केलेली)

## प्रॉम्प्ट अभियांत्रण समजून घेणे

<img src="../../../translated_images/mr/what-is-prompt-engineering.5c392a228a1f5823.webp" alt="प्रॉम्प्ट अभियांत्रण म्हणजे काय?" width="800"/>

प्रॉम्प्ट अभियांत्रण म्हणजे अशा इनपुट मजकुराचे डिझाइन करणे जे तुम्हाला आवश्यक परिणाम सातत्याने देते. हे फक्त प्रश्न विचारणे नाही — हे विनंत्या अशा प्रकारे रचण्याबाबत आहे की मॉडेलला अचूक काय हवे आहे आणि ते कसे देय याची समज येईल.

याला सहकाऱ्यास सूचना देण्यासारखे समजा. "बग दुरुस्त करा" हा अस्पष्ट आहे. "UserService.java च्या 45व्या ओळीत null pointer exception दुरुस्त करा, null तपासणी जोडून" हे विशिष्ट आहे. भाषा मॉडेल देखील याच प्रकारे काम करतात - तपशीलवारपणा आणि रचना महत्त्वाची आहे.

<img src="../../../translated_images/mr/how-langchain4j-fits.dfff4b0aa5f7812d.webp" alt="LangChain4j कसा सुसंगत होतो" width="800"/>

LangChain4j हा पायाभूत सुविधा पुरवतो — मॉडेल कनेक्शन, मेमरी, आणि संदेश प्रकार — तर प्रॉम्प्ट पॅटर्न म्हणजे तुम्ही त्या पायाभूत सुविधेतून पाठवलेला काळजीपूर्वक रचलेला मजकूर. मुख्य बांधकाम खंड आहेत `SystemMessage` (जो AI चा वागणूक व भूमिका सेट करतो) आणि `UserMessage` (जो तुमची प्रत्यक्ष विनंती नेतो).

## प्रॉम्प्ट अभियांत्रण मूलतत्त्वे

<img src="../../../translated_images/mr/five-patterns-overview.160f35045ffd2a94.webp" alt="पाच प्रॉम्प्ट अभियांत्रण पॅटर्न्स चे सारांश" width="800"/>

या मॉड्यूलमधील प्रगत पद्धतींमध्ये उतरण्यापूर्वी, चला पाच मूलभूत प्रॉम्प्टिंग तंत्रे पुनरावलोकन करू. ही अशी मुलभूत पायाभूत रचना आहे जी प्रत्येक प्रॉम्प्ट अभियंत्यास माहित असावी. जर तुम्ही आधीच [क्विक स्टार्ट मॉड्यूल](../00-quick-start/README.md#2-prompt-patterns) पूर्ण केले असेल, तर तुम्ही हे प्रत्यक्षात पाहिलेले आहे — येथे त्यांचा सैद्धांतिक चौकट आहे.

### झिरो-शॉट प्रॉम्प्टिंग

सर्वात सोपी पद्धत: मॉडेलला कोणताही उदाहरण न देता थेट सूचना द्या. मॉडेल पूर्णपणे त्याच्या प्रशिक्षणावर अवलंबून राहते कार्य समजून चालवणे. जेथे अपेक्षित वर्तन स्पष्ट असेल अशा सरळ विनंत्यांसाठी हे चांगले कार्य करते.

<img src="../../../translated_images/mr/zero-shot-prompting.7abc24228be84e6c.webp" alt="झिरो-शॉट प्रॉम्प्टिंग" width="800"/>

*उदाहरणांशिवाय थेट सुचना — मॉडेल फक्त सुचनेवरून कार्य गृहीत धरते*

```java
String prompt = "Classify this sentiment: 'I absolutely loved the movie!'";
String response = model.chat(prompt);
// प्रतिसाद: "सकारात्मक"
```

**कधी वापरावे:** सोप्या वर्गीकरणासाठी, थेट प्रश्नांसाठी, भाषांतरासाठी किंवा कोणत्याही कार्यासाठी ज्यामध्ये मॉडेलला अतिरिक्त मार्गदर्शनाची गरज नाही.

### फ्यू-शॉट प्रॉम्प्टिंग

तुम्ही मॉडेलला अनुसरण करण्याची पद्धत दाखवणारी उदाहरणे द्या. मॉडेल तुमच्या उदाहरणांमधून अपेक्षित इनपुट-आउटपुट स्वरुप शिकते आणि नवीन इनपुटवर ते लागू करते. ज्या कार्यांसाठी अपेक्षित स्वरूप किंवा वर्तन स्पष्ट नसते अशा ठिकाणी यामुळे सातत्य फार सुधारते.

<img src="../../../translated_images/mr/few-shot-prompting.9d9eace1da88989a.webp" alt="फ्यू-शॉट प्रॉम्प्टिंग" width="800"/>

*उदाहरणांमधून शिकणे — मॉडेल पॅटर्न ओळखते आणि नवीन इनपुटवर लागू करते*

```java
String prompt = """
    Classify the sentiment as positive, negative, or neutral.
    
    Examples:
    Text: "This product exceeded my expectations!" → Positive
    Text: "It's okay, nothing special." → Neutral
    Text: "Waste of money, very disappointed." → Negative
    
    Now classify this:
    Text: "Best purchase I've made all year!"
    """;
String response = model.chat(prompt);
```

**कधी वापरावे:** सानुकूल वर्गीकरण, सातत्यपूर्ण स्वरूपन, विशिष्ट क्षेत्रातील कार्य, किंवा जेव्हा झिरो-शॉट निकाल अस्थिर असतात.

### चेन ऑफ थॉट

मॉडेलला त्याचा तर्क पायरी-दर-पायरी दाखवायला सांगा. उत्तरावर थेट कूद न करता, मॉडेल समस्येचे विभाजन करते आणि प्रत्येक भाग स्पष्टपणे हाताळते. यामुळे गणित, तर्क, आणि बहु-धाप रंगाच्या कार्यांमध्ये अचूकता सुधारते.

<img src="../../../translated_images/mr/chain-of-thought.5cff6630e2657e2a.webp" alt="चेन ऑफ थॉट प्रॉम्प्टिंग" width="800"/>

*पायरी-दर-पायरी तर्कशक्ती — गुंतागुंतीच्या समस्यांचे स्पष्ट तार्किक टप्प्यात विघटन*

```java
String prompt = """
    Problem: A store has 15 apples. They sell 8 apples and then 
    receive a shipment of 12 more apples. How many apples do they have now?
    
    Let's solve this step-by-step:
    """;
String response = model.chat(prompt);
// मॉडेल असा दर्शवतो: 15 - 8 = 7, नंतर 7 + 12 = 19 सफरचंदी
```

**कधी वापरावे:** गणित समस्यांसाठी, तर्क पझलसाठी, डिबगिंगसाठी, किंवा जेथे तर्क प्रक्रिया दर्शविले तर अचूकता आणि विश्वास सुधारतो.

### भूमिका-आधारित प्रॉम्प्टिंग

तुमचा प्रश्न विचारण्यापूर्वी AI साठी एखादी भूमिका किंवा व्यक्तिरेखा सेट करा. हे प्रतिसादाच्या टोन, खोली, आणि लक्ष केंद्रित करण्याचा संदर्भ देते. "सॉफ्टवेअर आर्किटेक्ट" हे "जूनियर डेव्हलपर" किंवा "सिक्युरिटी ऑडिटर" पेक्षा वेगळे सल्ले देते.

<img src="../../../translated_images/mr/role-based-prompting.a806e1a73de6e3a4.webp" alt="भूमिका-आधारित प्रॉम्प्टिंग" width="800"/>

*संदर्भ आणि व्यक्तिरेखा सेट करताना — दिलेल्या भूमिकेनुसार त्याच प्रश्नाला वेगळा प्रतिसाद मिळतो*

```java
String prompt = """
    You are an experienced software architect reviewing code.
    Provide a brief code review for this function:
    
    def calculate_total(items):
        total = 0
        for item in items:
            total = total + item['price']
        return total
    """;
String response = model.chat(prompt);
```

**कधी वापरावे:** कोड समीक्षा, ट्यूटोरिंग, क्षेत्रविशिष्ट विश्लेषण, किंवा जेव्हा तुम्हाला विशिष्ट तज्ञाची पातळी किंवा दृष्टीकोन पाहिजे असेल.

### प्रॉम्प्ट टेम्पलेट्स

चल (व्हेरिएबल) प्लेसहोल्डर्ससह पुनर्वापर करण्यायोग्य प्रॉम्प्ट तयार करा. प्रत्येक वेळी नवीन प्रॉम्प्ट लिहिण्यापेक्षा, एकदा टेम्पलेट ठरवा आणि वेगवेगळ्या मूल्ये भरा. LangChain4j चा `PromptTemplate` वर्ग हे `{{variable}}` सिन्टॅक्ससह सोपे करतो.

<img src="../../../translated_images/mr/prompt-templates.14bfc37d45f1a933.webp" alt="प्रॉम्प्ट टेम्पलेट्स" width="800"/>

*चल प्लेसहोल्डर्ससह पुनर्वापरयोग्य प्रॉम्प्ट — एक टेम्पलेट, अनेक वापर*

```java
PromptTemplate template = PromptTemplate.from(
    "What's the best time to visit {{destination}} for {{activity}}?"
);

Prompt prompt = template.apply(Map.of(
    "destination", "Paris",
    "activity", "sightseeing"
));

String response = model.chat(prompt.text());
```

**कधी वापरावे:** वेगवेगळ्या इनपुटसह पुनरावृत्त चौकशा, बॅच प्रक्रिया, पुनर्वापरयोग्य AI वर्कफ्लोज, किंवा जेथे प्रॉम्प्ट संरचना सारखी राहते पण डेटा बदलतो.

---

ही पाच मूलतत्त्वे तुम्हाला बहुतांश प्रॉम्प्टिंग कार्यांसाठी मजबूत साधनसामग्री देतात. या मॉड्यूलच्या उरलेल्या भागात **आठ प्रगत पद्धती** आहेत ज्या GPT-5.2 च्या तर्कशक्ती नियंत्रण, स्वयं-मूल्यांकन, आणि संरचित आउटपुट क्षमतांचा पूर्ण वापर करतात.

## प्रगत पद्धती

मूलभूत तत्त्वे पूर्ण केल्यावर, चला या मॉड्यूलला अनोखा बनवणाऱ्या आठ प्रगत पद्धतींकडे जाऊ. सर्व समस्या समान दृष्टिकोन नको असतो. काही प्रश्नांना वेगाचे थेट उत्तर हवे असते, काहींना खोल विचार करणे आवश्यक असते. काहींना दृश्यमान तर्क हवा असतो, काहींना फक्त निकाल. खालील प्रत्येक पॅटर्न वेगळ्या परिस्थितीसाठी सुधारित केलेला आहे — आणि GPT-5.2 चे तर्कशक्ती नियंत्रण फरक अधिक स्पष्ट करतात.

<img src="../../../translated_images/mr/eight-patterns.fa1ebfdf16f71e9a.webp" alt="आठ प्रॉम्प्टिंग पॅटर्न्स" width="800"/>

*आठ प्रॉम्प्ट अभियांत्रण पॅटर्न्स आणि त्यांचे वापर केसेसचे अवलोकन*

<img src="../../../translated_images/mr/reasoning-control.5cf85f0fc1d0c1f3.webp" alt="GPT-5.2 सह तर्कशक्ती नियंत्रण" width="800"/>

*GPT-5.2 चे तर्कशक्ती नियंत्रण तुम्हाला मॉडेलने किती विचार करायचा ते ठरवण्याची मुभा देते — जलद थेट उत्तरांपासून खोल संशोधनापर्यंत*

<img src="../../../translated_images/mr/reasoning-effort.db4a3ba5b8e392c1.webp" alt="तर्कशक्ती प्रयत्नांची तुलना" width="800"/>

*कमी विरक्तता (जलद, थेट) विरुद्ध जास्त विरक्तता (सखोल, अन्वेषणात्मक) तर्कशक्ती दृष्टिकोन*

**कमी विरक्तता (जलद आणि लक्ष केंद्रीत)** - सोप्या प्रश्नांसाठी जिथे तुम्हाला जलद, थेट उत्तर हवे आहे. मॉडेल थोडेसेच तर्क करते - जास्तीत जास्त 2 पायऱ्या. गणना, शोध किंवा सरळ प्रश्नांसाठी वापरा.

```java
String prompt = """
    <reasoning_effort>low</reasoning_effort>
    <instruction>maximum 2 reasoning steps</instruction>
    
    What is 15% of 200?
    """;

String response = chatModel.chat(prompt);
```

> - "कमी विरक्तता आणि जास्त विरक्तता प्रॉम्प्टिंग पॅटर्न्समधील फरक काय आहे?"
> - "प्रॉम्प्ट्समध्ये XML टॅग्स AI च्या प्रतिसादाच्या रचनेला कसे मदत करतात?"
> - "स्वयं-प्रतिबिंबित पॅटर्न्स आणि थेट सूचनेचा वापर कधी करावा?"

**जास्त विरक्तता (सखोल आणि पूर्ण)** - गुंतागुंतीच्या समस्यांसाठी ज्यांना सखोल विश्लेषण हवे. मॉडेल सखोल शोध करतो आणि सविस्तर तर्क दाखवतो. सिस्टम डिझाइन, आर्किटेक्चर निर्णय, किंवा गुंतागुंतीच्या संशोधनासाठी वापरा.

```java
String prompt = """
    <reasoning_effort>high</reasoning_effort>
    <instruction>explore thoroughly, show detailed reasoning</instruction>
    
    Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```

**कार्य अंमलबजावणी (पायरी-त-by-पायरी प्रगती)** - अनेक चरणांच्या वर्कफ्लोसाठी. मॉडेल पूर्वी योजना देतो, प्रत्येक टप्पा करताना वर्णन करतो, मग सारांश देतो. मिग्रेशन, अंमलबजावणी, किंवा कोणत्याही अनेक-चरण प्रक्रियेसाठी वापरा.

```java
String prompt = """
    <task>Create a REST endpoint for user registration</task>
    <preamble>Provide an upfront plan</preamble>
    <narration>Narrate each step as you work</narration>
    <summary>Summarize what was accomplished</summary>
    """;

String response = chatModel.chat(prompt);
```

चेन-ऑफ-थॉट प्रॉम्प्टिंग मॉडेलला त्याचा तर्क प्रक्रिया दाखवायला सांगते, ज्यामुळे गुंतागुंतीच्या कार्यांची अचूकता वाढते. पायरी-दर-पायरी विघटन मानवी आणि AI दोघांनाही लॉजिक समजण्यास मदत करते.

> - "लांब चालणाऱ्या ऑपरेशन्ससाठी कार्य अंमलबजावणी पॅटर्न कसा रूपांतरित कराल?"
> - "प्रॉडक्शन अ‍ॅप्लिकेशन्समध्ये टूल प्रीएंबल्स स्ट्रक्चर करण्याचे सर्वोत्तम आचारसंहिता काय आहेत?"
> - "UI मध्ये मधली प्रगती अपडेट्स कशी कॅप्चर आणि दाखवू शकतो?"

*पुढाकार → अंमलबजावणी → सारांश कार्यप्रवाह अनेक-चरण कार्यांसाठी*

**स्वयं-प्रतिबिंबित करणारा कोड** - उत्पादन-गुणवत्तेचा कोड तयार करण्यासाठी. मॉडेल कोड तयार करतो, गुणवत्ता निकषांशी तपासतो, व सुधारणा करत राहतो. नवीन फिचर्स किंवा सेवा तयार करताना वापरा.

```java
String prompt = """
    <task>Create an email validation service</task>
    <quality_criteria>
    - Correct logic and error handling
    - Best practices (clean code, proper naming)
    - Performance optimization
    - Security considerations
    </quality_criteria>
    <instruction>Generate code, evaluate against criteria, improve iteratively</instruction>
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/mr/self-reflection-cycle.6f71101ca0bd28cc.webp" alt="स्वयं-प्रतिबिंब चक्र" width="800"/>

*पायरी-दर-पायरी सुधारणा लूप - तयार करा, मूल्यांकन करा, समस्या ओळखा, सुधारणा करा, पुनरावृत्ती करा*

**संरचित विश्लेषण** - सातत्यपूर्ण मूल्यमापनासाठी. मॉडेल निश्चित चौकटीत कोड परीक्षण करतो (योग्यता, प्रॅक्टिसेस, कार्यक्षमता, सुरक्षा). कोड रिव्ह्यू किंवा गुणवत्ता मूल्यमापनासाठी वापरा.

```java
String prompt = """
    <code>
    public List getUsers() {
        return database.query("SELECT * FROM users");
    }
    </code>
    
    <framework>
    Evaluate using these categories:
    1. Correctness - Logic and functionality
    2. Best Practices - Code quality
    3. Performance - Efficiency concerns
    4. Security - Vulnerabilities
    </framework>
    """;

String response = chatModel.chat(prompt);
```

> - "विभिन्न प्रकारच्या कोड रिव्ह्यूसाठी विश्लेषण चौकट कशी सानुकूलित करावी?"
> - "संरचित आउटपुट प्रोग्रामॅटिकली कसे पार्स आणि वापरावे?"
> - "विविध रिव्ह्यू सेशन्समध्ये सतत गंभीरता स्तर कसा राखावा?"

<img src="../../../translated_images/mr/structured-analysis-pattern.0af3b690b60cf2d6.webp" alt="संरचित विश्लेषण पॅटर्न" width="800"/>

*सातत्यपूर्ण कोड रिव्ह्यूसाठी चार-श्रेणी चौकट आणि गंभीरता स्तरांसह*

**मल्टी-टर्न चॅट** - संदर्भाची आवश्यकता असलेल्या संवादांसाठी. मॉडेल जुने संदेश लक्षात ठेवतो आणि त्यावर आधारित नवीन प्रतिक्रिया तयार करतो. इंटरैक्टिव हेल्प सेशन्स किंवा गुंतागुंतीच्या प्रश्नोत्तरांसाठी वापरा.

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("What is Spring Boot?"));
AiMessage aiMessage1 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage1);

memory.add(UserMessage.from("Show me an example"));
AiMessage aiMessage2 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage2);
```

<img src="../../../translated_images/mr/context-memory.dff30ad9fa78832a.webp" alt="संदर्भ मेमरी" width="800"/>

*संवाद संदर्भ अनेक टर्न्समध्ये संचयित होतो जोपर्यंत टोकन मर्यादा पोहोचत नाही*

**पायरी-पायरीने तर्क** - दृश्यमान लॉजिक आवश्यक असलेल्या समस्यांसाठी. मॉडेल प्रत्येक टप्प्याचा स्पष्ट तर्क दाखवतो. गणित विषयक समस्यांसाठी, तर्क पझल्ससाठी, किंवा विचार प्रक्रिया समजून घेण्यासाठी वापरा.

```java
String prompt = """
    <instruction>Show your reasoning step-by-step</instruction>
    
    If a train travels 120 km in 2 hours, then stops for 30 minutes,
    then travels another 90 km in 1.5 hours, what is the average speed
    for the entire journey including the stop?
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/mr/step-by-step-pattern.a99ea4ca1c48578c.webp" alt="पायरी-पायरी पॅटर्न" width="800"/>

*समस्यांचे स्पष्ट तार्किक टप्प्यांमध्ये विघटन*

**बांधिल स्वरुप** - विशिष्ट स्वरूप आवश्यक असलेल्या प्रतिसादांसाठी. मॉडेल कडकपणे स्वरूप आणि लांबी नियमांचे पालन करतो. सारांशासाठी किंवा नेमकं आउटपुट संरचना हवी असताना वापरा.

```java
String prompt = """
    <constraints>
    - Exactly 100 words
    - Bullet point format
    - Technical terms only
    </constraints>
    
    Summarize the key concepts of machine learning.
    """;

String response = chatModel.chat(prompt);
```

## पॅटर्न्सची तपासणी

वेब इंटरफेस तुम्हाला वेगवेगळ्या प्रॉम्प्टिंग धोरणांसह प्रयोग करण्याची मुभा देते. प्रत्येक पॅटर्न भिन्न समस्या सोडवतो - त्यांनी पहा की कोणता दृष्टिकोन कधी उत्कृष्ट काम करतो.

### कमी आणि जास्त उत्सुकता

"15% of 200 काय आहे?" असा सोपा प्रश्न कमी उत्सुकतेचा वापर करून विचारा. तुम्हाला ताबडतोब थेट उत्तर मिळेल. आता "उच्च-ट्रॅफिक API साठी कॅशिंग धोरण डिझाईन करा" असा जटिल प्रश्न जास्त उत्सुकतेसह विचारा. पहा कसा मॉडेल हळूहळू उत्तर देतो आणि सविस्तर तर्क तयार करीत आहे. तोच मॉडेल, तोच प्रश्न रचना - पण प्रॉम्प्ट त्याला किती विचार करायचा हे सांगतो.

## तुम्ही खरीखुरी काय शिकत आहात

**कारण करण्याचा प्रयत्न सर्व काही बदलतो**

GPT-5.2 तुम्हाला तुमच्या प्रॉम्प्टद्वारे संगणकीय प्रयत्न नियंत्रित करण्याची मुभा देते. कमी प्रयत्न म्हणजे जलद प्रतिसाद कमी शोधासह. जास्त प्रयत्न म्हणजे मॉडेल खोलवर विचार करण्यासाठी वेळ घालवते. तुम्ही शिकत आहात कार्य जटिलतेनुसार प्रयत्न जुळवायचा - सोप्या प्रश्नांवर वेळ वाया घालवू नका, पण जटिल निर्णयांवरही घाई करू नका.

**रचना वर्तनाला मार्गदर्शन करते**

प्रॉम्प्टमधील XML टॅग्स तुम्हाला दिसताहेत का? ते फक्त सजावटीसाठी नाहीत. मॉडेल्स मुक्त लेखनापेक्षा रचनेच्या सूचना अधिक विश्वासार्हपणे पाळतात. बहु-चरण प्रक्रिया किंवा जटिल लॉजिकसाठी रचना मॉडेलच्या स्थितीचा आणि पुढे काय टाकायचे आहे हे ट्रॅक करण्यात मदत करते.

<img src="../../../translated_images/mr/prompt-structure.a77763d63f4e2f89.webp" alt="Prompt Structure" width="800"/>

*स्पष्ट विभाग आणि XML-शैलीची संघटना असलेले चांगल्या रचनेचे प्रॉम्प्टचे शरीररचना*

**स्वत:च्या मूल्यांकनाद्वारे गुणवत्ता**

स्वतःचे प्रतिबिंब करणारे पॅटर्न गुणवत्ता निकष स्पष्ट करून कार्य करतात. "मॉडेल बरोबर करते" अशी आशा करण्याऐवजी, तुम्ही त्याला नक्की काय "योग्य" आहे ते सांगता: बरोबर लॉजिक, त्रुटी हाताळणी, कामगिरी, सुरक्षा. मग मॉडेल आपले उत्पादन स्वतःच मूल्यांकन करू शकते आणि सुधारणा करते. हे कोड निर्मितीला लॉटरीपासून प्रक्रियेत रूपांतरित करते.

**संदर्भ मर्यादित आहे**

बहु-टर्न संभाषणे प्रत्येक विनंतीसह संदेश इतिहास समाविष्ट करून कार्य करतात. पण मर्यादा आहे - प्रत्येक मॉडेलमध्ये जास्तीत जास्त टोकन संख्या असते. संभाषणे वाढल्यावर, तुम्हाला संदर्भ योग्य राखण्यासाठी धोरणे लागू करावी लागतात जेणेकरून तो मर्यादा ओलांडणार नाही. हा मॉड्यूल तुम्हाला स्मृती कशी कार्य करते ते दाखवतो; नंतर तुम्ही शिकाल की कधी सारांश द्यायचा, कधी विसरायचे, आणि कधी पुन्हा आणायचे.
