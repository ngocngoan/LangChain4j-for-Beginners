## वीडियो वॉकथ्रू

इस लाइव सेशन को देखें जो इस मॉड्यूल के साथ शुरुआत करने का तरीका समझाता है: [LangChain4j के साथ शुरुआत करना - लाइव सेशन](https://www.youtube.com/live/nl_troDm8rQ?si=6b85S8xGjWnT2fX9)

## आप क्या सीखेंगे

यदि आपने क्विक स्टार्ट पूरा किया है, तो आपने देखा होगा कि प्रॉम्प्ट कैसे भेजे जाते हैं और प्रतिक्रियाएँ प्राप्त होती हैं। यही आधार है, लेकिन वास्तविक ऐप्लिकेशन को इससे अधिक चाहिए। यह मॉड्यूल आपको संवादात्मक AI बनाने का तरीका सिखाता है जो संदर्भ को याद रखता है और स्थिति बनाए रखता है - जो एक एकबारगी डेमो और प्रोडक्शन-तैयार ऐप्लिकेशन के बीच का अंतर है।

हम इस गाइड में Azure OpenAI के GPT-5.2 का उपयोग करेंगे क्योंकि इसकी उन्नत तर्क क्षमता विभिन्न पैटर्न के व्यवहार को स्पष्ट रूप से दर्शाती है। जब आप मेमोरी जोड़ेंगे, तो आप स्पष्ट रूप से फर्क देखेंगे। इससे यह समझना आसान हो जाता है कि प्रत्येक घटक आपके ऐप्लिकेशन को क्या लेकर आता है।

आप एक ऐसा ऐप्लिकेशन बनाएंगे जो दोनों पैटर्न दिखाएगा:

**स्टेटलेस चैट** - प्रत्येक अनुरोध स्वतंत्र होता है। मॉडल को पिछले संदेशों की कोई स्मृति नहीं होती। यह वही पैटर्न है जिसका आपने क्विक स्टार्ट में उपयोग किया था।

**स्टेटफुल संवाद** - प्रत्येक अनुरोध में वार्तालाप इतिहास शामिल होता है। मॉडल कई चरणों में संदर्भ बनाए रखता है। यह वही है जो प्रोडक्शन ऐप्लिकेशन को चाहिए।

## पूर्वापेक्षाएँ

- Azure सब्सक्रिप्शन जिसमें Azure OpenAI एक्सेस हो
- Java 21, Maven 3.9+
- Azure CLI (https://learn.microsoft.com/en-us/cli/azure/install-azure-cli)
- Azure Developer CLI (azd) (https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/install-azd)

> **ध्यान दें:** Java, Maven, Azure CLI और Azure Developer CLI (azd) पहले से ही प्रदान किए गए devcontainer में स्थापित हैं।

> **ध्यान दें:** यह मॉड्यूल Azure OpenAI पर GPT-5.2 का उपयोग करता है। तैनाती `azd up` के माध्यम से स्वचालित रूप से कॉन्फ़िगर होती है - कोड में मॉडल नाम को संशोधित न करें।

## मूल समस्या को समझना

भाषा मॉडल स्टेटलेस होते हैं। प्रत्येक API कॉल स्वतंत्र होती है। यदि आप कहते हैं "मेरा नाम जॉन है" और फिर पूछते हैं "मेरा नाम क्या है?" तो मॉडल को पता नहीं होता कि आपने अभी खुद का परिचय दिया है। यह हर अनुरोध को ऐसा मानता है जैसे यह आपकी पहली बातचीत हो।

यह सरल सवाल-जबाव के लिए ठीक है लेकिन वास्तविक ऐप्लिकेशन के लिए उपयोगी नहीं है। ग्राहक सेवा बॉट को याद रखना पड़ता है कि आपने उनसे क्या कहा। व्यक्तिगत सहायक को संदर्भ चाहिए। किसी भी बहु-चरणीय वार्तालाप के लिए स्मृति आवश्यक है।

*स्टेटलेस (स्वतंत्र कॉल) और स्टेटफुल (संदर्भ-संवेदनशील) वार्तालाप के बीच का अंतर*

## टोकन्स को समझना

वार्तालाप में डुबकी लगाने से पहले, टोकन्स को समझना महत्वपूर्ण है - ये वह मूल इकाइयाँ हैं जिन्हें भाषा मॉडल संसाधित करते हैं:

<img src="../../../translated_images/hi/token-explanation.c39760d8ec650181.webp" alt="टोकन व्याख्या" width="800"/>

*कैसे टेक्स्ट को टोकन्स में तोड़ा जाता है - "I love AI!" चार अलग-अलग प्रोसेसिंग यूनिट्स बनते हैं*

टोकन्स AI मॉडल टेक्स्ट को मापने और संसाधित करने का तरीका हैं। शब्द, विराम चिह्न, और यहाँ तक कि रिक्त स्थान भी टोकन हो सकते हैं। आपके मॉडल के पास यह सीमित होता है कि वह कितने टोकन्स को एक बार में प्रोसेस कर सकता है (GPT-5.2 के लिए 400,000 टोकन्स, जिसमें 272,000 इनपुट टोकन्स और 128,000 आउटपुट टोकन्स शामिल हैं)। टोकन्स को समझना आपको वार्तालाप की लंबाई और लागत प्रबंधन में मदद करता है।

## मेमोरी कैसे काम करती है

चैट मेमोरी स्टेटलेस समस्या का समाधान है, यह वार्तालाप इतिहास बनाए रखती है। मॉडल को अनुरोध भेजने से पहले, फ्रेमवर्क प्रासंगिक पिछले संदेशों को पहले जोड़ देता है। जब आप पूछते हैं "मेरा नाम क्या है?" तो सिस्टम वास्तव में पूरी वार्तालाप इतिहास भेजता है, जिससे मॉडल को यह पता चलता है कि आपने पहले कहा था "मेरा नाम जॉन है।"

LangChain4j मेमोरी इम्प्लीमेंटेशन प्रदान करता है जो इसे स्वचालित रूप से संभालते हैं। आप चुनते हैं कि आप कितने संदेश रखने चाहते हैं और फ्रेमवर्क संदर्भ विंडो नियंत्रित करता है।

<img src="../../../translated_images/hi/memory-window.bbe67f597eadabb3.webp" alt="मेमोरी विंडो अवधारणा" width="800"/>

*MessageWindowChatMemory हाल के संदेशों की एक स्लाइडिंग विंडो बनाए रखता है, पुराने संदेश स्वतः हटा देता है*

## यह LangChain4j का उपयोग कैसे करता है

यह मॉड्यूल क्विक स्टार्ट का विस्तार करके Spring Boot को एकीकृत करता है और वार्तालाप मेमोरी जोड़ता है। यहाँ घटक कैसे मेल खाते हैं:

**निर्भरता** - दो LangChain4j लाइब्रेरीज़ जोड़ें:

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```
  
**चैट मॉडल** - Azure OpenAI को Spring bean के रूप में कॉन्फ़िगर करें ([LangChainConfig.java](../../../01-introduction/src/main/java/com/example/langchain4j/config/LangChainConfig.java)):

```java
@Bean
public OpenAiOfficialChatModel openAiOfficialChatModel() {
    return OpenAiOfficialChatModel.builder()
            .baseUrl(azureEndpoint)
            .apiKey(azureApiKey)
            .modelName(deploymentName)
            .timeout(Duration.ofMinutes(5))
            .maxRetries(3)
            .build();
}
```
  
बिल्डर `azd up` द्वारा सेट किए गए पर्यावरण चर से क्रेडेंशियल पढ़ता है। `baseUrl` को आपके Azure Endpoint पर सेट करने से OpenAI क्लाइंट Azure OpenAI के साथ काम करता है।

**वार्तालाप मेमोरी** - MessageWindowChatMemory के साथ चैट इतिहास ट्रैक करें ([ConversationService.java](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java)):

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("My name is John"));
memory.add(AiMessage.from("Nice to meet you, John!"));

memory.add(UserMessage.from("What's my name?"));
AiMessage aiMessage = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage);
```
  
`withMaxMessages(10)` के साथ मेमोरी बनाएं ताकि अंतिम 10 संदेश रखे जाएं। यूजर और AI संदेशों को टाइपेड रैपर के साथ जोड़ें: `UserMessage.from(text)` और `AiMessage.from(text)`। इतिहास को `memory.messages()` से प्राप्त करें और इसे मॉडल को भेजें। यह सेवा प्रत्येक वार्तालाप ID के लिए अलग मेमोरी इंस्टेंस स्टोर करती है, जिससे अनेक उपयोगकर्ता एक साथ चैट कर सकते हैं।

> **🤖 [GitHub Copilot](https://github.com/features/copilot) चैट के साथ आज़माएं:** [`ConversationService.java`](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java) खोलें और पूछें:
> - "MessageWindowChatMemory विंडो भर जाने पर किन संदेशों को छोड़ने का निर्णय कैसे लेता है?"
> - "क्या मैं इन-मेमोरी के बजाय डेटाबेस का उपयोग करके कस्टम मेमोरी स्टोरेज लागू कर सकता हूँ?"
> - "मैं पुरानी वार्तालाप इतिहास को संक्षेपित करने के लिए सारांश कैसे जोड़ूंगा?"

स्टेटलेस चैट एंडपॉइंट मेमोरी पूरी तरह से छोड़ देता है - सीधे `chatModel.chat(prompt)` जैसा कि क्विक स्टार्ट में था। स्टेटफुल एंडपॉइंट मेमोरी में संदेश जोड़ता है, इतिहास प्राप्त करता है, और उस संदर्भ को प्रत्येक अनुरोध के साथ शामिल करता है। मॉडल कॉन्फ़िगरेशन समान है, पैटर्न अलग।

## Azure OpenAI इन्फ्रास्ट्रक्चर तैनात करें

**Bash:**
```bash
cd 01-introduction
azd up  # सदस्यता और स्थान चुनें (eastus2 अनुशंसित)
```
  
**PowerShell:**
```powershell
cd 01-introduction
azd up  # सदस्यता और स्थान चुनें (eastus2 अनुशंसित)
```
  
> **ध्यान दें:** यदि आपको टाइमआउट त्रुटि मिलती है (`RequestConflict: Cannot modify resource ... provisioning state is not terminal`), तो बस `azd up` फिर चलाएं। Azure संसाधन पृष्ठभूमि में अभी भी तैयार हो सकते हैं, और पुनः प्रयास से तैनाती तब पूरी हो जाएगी जब संसाधन टर्मिनल स्थिति में पहुँचेंगे।

यह तैनाती करेगा:  
1. GPT-5.2 और text-embedding-3-small मॉडल के साथ Azure OpenAI संसाधन तैनात करेगा  
2. प्रोजेक्ट रूट में क्रेडेंशियल के साथ `.env` फ़ाइल स्वचालित रूप से बनाएगा  
3. सभी आवश्यक पर्यावरण चर सेट करेगा  

**तैनाती में समस्या हो रही है?** विस्तृत समस्या निवारण के लिए [इन्फ्रास्ट्रक्चर README](infra/README.md) देखें जिसमें सबडोमेन नाम संघर्ष, मैन्युअल Azure पोर्टल तैनाती के चरण, और मॉडल कॉन्फ़िगरेशन मार्गदर्शन शामिल हैं।

**तैनाती सफल हुई है इसका सत्यापन करें:**

**Bash:**
```bash
cat ../.env  # AZURE_OPENAI_ENDPOINT, API_KEY, आदि दिखाना चाहिए।
```
  
**PowerShell:**
```powershell
Get-Content ..\.env  # AZURE_OPENAI_ENDPOINT, API_KEY, आदि दिखाना चाहिए।
```
  
> **ध्यान दें:** `azd up` कमांड स्वचालित रूप से `.env` फाइल बनाता है। यदि बाद में अपडेट करना हो, तो आप `.env` फ़ाइल मैन्युअली संपादित कर सकते हैं या फिर इसे पुनः जनरेट करने के लिए चलाएं:  
>  
> **Bash:**  
> ```bash
> cd ..
> bash .azd-env.sh
> ```
  
> **PowerShell:**  
> ```powershell
> cd ..
> .\.azd-env.ps1
> ```

**PowerShell:**

```
  
**एप्लिकेशन शुरू करें:**

**विकल्प 1: Spring Boot डैशबोर्ड का उपयोग करना (VS Code उपयोगकर्ताओं के लिए अनुशंसित)**

devcontainer में Spring Boot Dashboard एक्सटेंशन शामिल है, जो सभी Spring Boot ऐप्लिकेशन को प्रबंधित करने के लिए एक दृश्य इंटरफ़ेस प्रदान करता है। इसे VS Code के Activity Bar में बायीं ओर Spring Boot आइकन के रूप में खोजें।

Spring Boot Dashboard से आप:  
- कार्यस्थान में सभी उपलब्ध Spring Boot ऐप्लिकेशन देख सकते हैं  
- एक क्लिक में ऐप्लिकेशन शुरू/रोक सकते हैं  
- वास्तविक समय में लॉग देख सकते हैं  
- ऐप्लिकेशन की स्थिति मॉनिटर कर सकते हैं  

बस "introduction" के बगल में प्ले बटन पर क्लिक करें ताकि यह मॉड्यूल शुरू हो, या सभी मॉड्यूल एक साथ शुरू करें।

```
  
दोनों स्क्रिप्ट रूट `.env` फ़ाइल से पर्यावरण चर लोड करती हैं और यदि JAR फाइलें मौजूद नहीं हैं तो वे उन्हें बनाएंगी।

> **ध्यान दें:** यदि आप सभी मॉड्यूल मैन्युअली बनाना पसंद करते हैं तो शुरू करने से पहले:  
>  
> **Bash:**  
> ```bash
> cd ..  # Go to root directory
> mvn clean package -DskipTests
> ```
  
> **PowerShell:**  
> ```powershell
> cd ..  # Go to root directory
> mvn clean package -DskipTests
> ```
  

ब्राउज़र में http://localhost:8080 खोलें।

**रोकने के लिए:**

## एप्लिकेशन का उपयोग करना

एप्लिकेशन एक वेब इंटरफ़ेस प्रदान करता है जिसमें दो चैट कार्यान्वयन एक साथ स्थित हैं।

<img src="../../../translated_images/hi/home-screen.121a03206ab910c0.webp" alt="एप्लिकेशन होम स्क्रीन" width="800"/>

*डैशबोर्ड जो सरल चैट (स्टेटलेस) और संवादात्मक चैट (स्टेटफुल) विकल्प दिखाता है*

### स्टेटलेस चैट (बाईं पैनल)

पहले इसे आज़माएं। "मेरा नाम जॉन है" कहें और तुरंत पूछें "मेरा नाम क्या है?" मॉडल याद नहीं रखेगा क्योंकि प्रत्येक संदेश स्वतंत्र है। यह मूल भाषा मॉडल इंटीग्रेशन की मूल समस्या दिखाता है - कोई संवाद संदर्भ नहीं।

<img src="../../../translated_images/hi/simple-chat-stateless-demo.13aeb3978eab3234.webp" alt="स्टेटलेस चैट डेमो" width="800"/>

*एआई पिछला संदेश से आपका नाम याद नहीं रखता*

### स्टेटफुल चैट (दाईं पैनल)

अब यही क्रम यहां आज़माएं। "मेरा नाम जॉन है" कहें और फिर "मेरा नाम क्या है?" यह याद रखेगा। अंतर MessageWindowChatMemory है - यह वार्तालाप इतिहास बनाए रखता है और प्रत्येक अनुरोध के साथ इसे शामिल करता है। यही वह तरीका है जिससे प्रोडक्शन संवादात्मक AI काम करता है।

<img src="../../../translated_images/hi/conversational-chat-stateful-demo.e5be9822eb23ff59.webp" alt="स्टेटफुल चैट डेमो" width="800"/>

*एआई बातचीत में पहले बताया गया आपका नाम याद रखता है*

दोनों पैनल एक ही GPT-5.2 मॉडल का उपयोग करते हैं। केवल अंतर मेमोरी है। इससे स्पष्ट होता है मेमोरी आपके ऐप्लिकेशन को क्या देती है और क्यों यह वास्तविक उपयोग मामलों के लिए आवश्यक है।
