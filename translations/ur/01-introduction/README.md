## جو آپ سیکھیں گے

اگر آپ نے فوری آغاز مکمل کیا، تو آپ نے دیکھا کہ پرامپٹس کیسے بھیجیں اور جوابات کیسے حاصل کریں۔ یہ بنیاد ہے، مگر حقیقی ایپلیکیشنز کو مزید چیزوں کی ضرورت ہوتی ہے۔ یہ ماڈیول آپ کو سکھائے گا کہ آپ کس طرح ایسی محاوری AI بنا سکتے ہیں جو سیاق و سباق یاد رکھتی ہو اور حالت برقرار رکھتی ہو - یہ فرق ہے ایک عارضی ڈیمو اور ایک پیداواری ایپلیکیشن کے درمیان۔

ہم اس رہنمائی میں پورے وقت Azure OpenAI کے GPT-5.2 کا استعمال کریں گے کیونکہ اس کی اعلی استدلالی صلاحیتیں مختلف پیٹرنز کے رویے کو زیادہ واضح بناتی ہیں۔ جب آپ یادداشت شامل کریں گے، تو آپ فرق واضح طور پر دیکھ سکیں گے۔ اس سے یہ سمجھنا آسان ہوتا ہے کہ ہر جزو آپ کی ایپلیکیشن میں کیا لاتا ہے۔

آپ ایک ایسی ایپلیکیشن بنائیں گے جو دونوں پیٹرنز کو ظاہر کرتی ہے:

**بغیر یادداشت چیٹ** - ہر درخواست آزاد ہوتی ہے۔ ماڈل کو پچھلے پیغامات کی کوئی یادداشت نہیں ہوتی۔ یہ وہی پیٹرن ہے جو آپ نے فوری آغاز میں استعمال کیا تھا۔

**یادداشت کے ساتھ محاورت** - ہر درخواست میں گفتگو کی تاریخ شامل ہوتی ہے۔ ماڈل متعدد چکر کے دوران سیاق و سباق کو برقرار رکھتا ہے۔ یہ وہ چیز ہے جس کی پیداواری ایپلیکیشنز کو ضرورت ہوتی ہے۔

## بنیادی مسئلے کو سمجھنا

لینگویج ماڈلز بغیر یادداشت کے ہوتے ہیں۔ ہر API کال آزاد ہوتی ہے۔ اگر آپ کہیں "میرا نام جان ہے" اور پھر پوچھیں "میرا نام کیا ہے؟"، تو ماڈل کو معلوم نہیں ہوتا کہ آپ نے ابھی اپنا تعارف کرایا ہے۔ یہ ہر درخواست کو پہلی گفتگو کی طرح سمجھتا ہے۔

یہ سادہ سوال و جواب کے لیے ٹھیک ہے لیکن حقیقی ایپلیکیشنز کے لیے بے کار ہے۔ کسٹمر سروس بوٹس کو یہ یاد رکھنا پڑتا ہے کہ آپ نے کیا بتایا تھا۔ ذاتی معاونین کو سیاق و سباق کی ضرورت ہوتی ہے۔ کوئی بھی کثیر چکری گفتگو یادداشت کی متقاضی ہے۔

<img src="../../../translated_images/ur/token-explanation.c39760d8ec650181.webp" alt="Token Explanation" width="800"/>

*ٹیکسٹ کو ٹوکینز میں کس طرح توڑا جاتا ہے کا ایک مثال - "I love AI!" چار علیحدہ پروسیسنگ یونٹس بن جاتے ہیں*

ٹوکینز وہ اندازہ اور پیمائش کے واحد ہیں جو AI ماڈلز متن کے لیے کرتے ہیں۔ الفاظ، رموزِ اوقاف، یہاں تک کہ اسپیس بھی ٹوکینز ہوسکتی ہیں۔ آپ کے ماڈل کی ایک حد ہوتی ہے کہ وہ کتنے ٹوکینز کو ایک ساتھ پروسیس کر سکتا ہے (GPT-5.2 کے لیے 400,000، جن میں تک 272,000 ان پٹ ٹوکینز اور 128,000 آؤٹ پٹ ٹوکینز شامل ہیں)۔ ٹوکینز کو سمجھنا آپ کو گفتگو کی لمبائی اور قیمتوں کا انتظام کرنے میں مدد دیتا ہے۔

## میموری کیسے کام کرتی ہے

چیٹ میموری بغیر یادداشت کا مسئلہ حل کرتی ہے کیونکہ یہ گفتگو کی تاریخ کو برقرار رکھتی ہے۔ آپ کی درخواست ماڈل کو بھیجنے سے پہلے، فریم ورک متعلقہ پچھلے پیغامات کو شامل کرتا ہے۔ جب آپ پوچھتے ہیں "میرا نام کیا ہے؟"، تو سسٹم اصل میں پوری گفتگو کی تاریخ بھیجتا ہے، جس سے ماڈل دیکھ سکتا ہے کہ آپ نے پہلے کہا تھا "میرا نام جان ہے۔"

LangChain4j میموری کی ایسی امپلیمنٹیشنز فراہم کرتا ہے جو یہ خودکار طور پر سنبھالتے ہیں۔ آپ چن سکتے ہیں کہ کتنے پیغامات محفوظ رکھے جائیں اور فریم ورک سیاق و سباق کی ونڈو کا انتظام کرتا ہے۔

<img src="../../../translated_images/ur/memory-window.bbe67f597eadabb3.webp" alt="Memory Window Concept" width="800"/>

*MessageWindowChatMemory تازہ ترین پیغامات کی سلائیڈنگ ونڈو رکھتا ہے، خود بخود پرانے پیغامات کو حذف کرتا ہے*

## یہ LangChain4j کیسے استعمال کرتا ہے

یہ ماڈیول فوری آغاز کو بڑھاتا ہے Spring Boot کے انضمام سے اور گفتگو کی میموری شامل کرتا ہے۔ یہاں مختلف حصے کس طرح جڑتے ہیں:

**انحصارات** - دو LangChain4j لائبریریز شامل کریں:

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```

**چیٹ ماڈل** - Azure OpenAI کو Spring bean کے طور پر کنفیگر کریں ([LangChainConfig.java](../../../01-introduction/src/main/java/com/example/langchain4j/config/LangChainConfig.java)):

```java
@Bean
public OpenAiOfficialChatModel openAiOfficialChatModel() {
    return OpenAiOfficialChatModel.builder()
            .baseUrl(azureEndpoint)
            .apiKey(azureApiKey)
            .modelName(deploymentName)
            .timeout(Duration.ofMinutes(5))
            .maxRetries(3)
            .build();
}
```

بلڈر پروفائلز کو `azd up` کے ذریعے سیٹ کردہ انوائرمنٹ ویریبلز سے پڑھتا ہے۔ `baseUrl` کو آپ کے Azure اینڈپوائنٹ پر سیٹ کرنے سے اوپن اے آئی کلائنٹ Azure OpenAI کے ساتھ کام کرتا ہے۔

**گفتگو کی میموری** - MessageWindowChatMemory کے ساتھ چیٹ کی ہسٹری کو ٹریک کریں ([ConversationService.java](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java)):

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("My name is John"));
memory.add(AiMessage.from("Nice to meet you, John!"));

memory.add(UserMessage.from("What's my name?"));
AiMessage aiMessage = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage);
```

میموری بنائیں `withMaxMessages(10)` کے ساتھ تاکہ آخری 10 پیغامات رکھے جائیں۔ صارف اور AI پیغامات شامل کریں ٹائپڈ ریپرز کی مدد سے: `UserMessage.from(text)` اور `AiMessage.from(text)`۔ ہسٹری حاصل کریں `memory.messages()` سے اور ماڈل کو بھیجیں۔ سروس ہر گفتگو کے ID کے لیے الگ الگ میموری انstances رکھتی ہے، جس سے متعدد صارفین ایک ساتھ چیٹ کر سکتے ہیں۔

> **🤖 GitHub Copilot Chat کے ساتھ آزمائیں:** [`ConversationService.java`](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java) کھولیں اور پوچھیں:
> - "MessageWindowChatMemory ونڈو بھرنے پر کون سے پیغامات حذف کرتا ہے؟"
> - "کیا میں ان-میموری کی بجائے ڈیٹابیس کا استعمال کرتے ہوئے کسٹم میموری اسٹوریج بنا سکتا ہوں؟"
> - "میں قدیم گفتگو کی تاریخ کو تلخیص کے ذریعے سکیڑنے کا اضافہ کیسے کر سکتا ہوں؟"

بغیر یادداشت چیٹ اینڈ پوائنٹ میموری کو بالکل چھوڑ دیتا ہے - بالکل `chatModel.chat(prompt)` جیسے فوری آغاز میں۔ یادداشت والا اینڈپوائنٹ پیغامات کو میموری میں شامل کرتا ہے، ہسٹری حاصل کرتا ہے، اور ہر درخواست کے ساتھ سیاق و سباق بھیجتا ہے۔ ایک ہی ماڈل کی ترتیب ہے، مختلف پیٹرنز۔

## Azure OpenAI انفراسٹرکچر کی تعیناتی

**بیش:**
```bash
cd 01-introduction
azd up  # سبسکرپشن اور مقام منتخب کریں (مشورہ دیا گیا ہے eastus2)
```

**پاور شیل:**
```powershell
cd 01-introduction
azd up  # سبسکرپشن اور مقام منتخب کریں (eastus2 تجویز کیا گیا)
```

> **نوٹ:** اگر آپ کو ٹائم آؤٹ ایرر ( `RequestConflict: Cannot modify resource ... provisioning state is not terminal`) ملے تو بس `azd up` دوبارہ چلائیں۔ Azure وسائل ابھی بھی پس منظر میں تعینات ہو سکتے ہیں، اور دوبارہ کوشش کرنے سے ان کی تعیناتی مکمل ہو جاتی ہے جب وسائل آخری حالت میں پہنچ جائیں۔

یہ کرے گا:
1. Azure OpenAI وسیلہ GPT-5.2 اور text-embedding-3-small ماڈلز کے ساتھ تعینات کرے گا
2. خودکار طریقے سے `.env` فائل پروجیکٹ کے روٹ میں کریڈینشلز کے ساتھ بنائے گا
3. تمام ضروری انوائرمنٹ ویریبلز ترتیب دے گا

**تعیناتی میں دشواری ہے؟** [Infrastructure README](infra/README.md) میں تفصیلی مسئلہ حل دیکھیں، جیسے سب ڈومین نام کے تصادم، Azure پورٹل سے دستی تعیناتی کے اقدامات، اور ماڈل کنفیگریشن کی رہنمائی۔

**تعیناتی کی کامیابی کی تصدیق کریں:**

**بیش:**
```bash
cat ../.env  # دکھانا چاہیے AZURE_OPENAI_ENDPOINT، API_KEY، وغیرہ۔
```

**پاور شیل:**
```powershell
Get-Content ..\.env  # AZURE_OPENAI_ENDPOINT, API_KEY وغیرہ دکھانا چاہیے۔
```

> **نوٹ:** `azd up` خود بخود `.env` فائل بنا دیتا ہے۔ اگر آپ کو بعد میں اسے اپ ڈیٹ کرنا ہو تو آپ یا تو `.env` فائل کو دستی طور پر ایڈٹ کریں یا اسے دوبارہ بنانے کے لیے یہ کمانڈ چلائیں:
>
> **بیش:**
> ```bash
> cd ..
> bash .azd-env.sh
> ```
>
> **پاور شیل:**
> ```powershell
> cd ..
> .\.azd-env.ps1
> ```

**ایپلیکیشنز شروع کریں:**

**آپشن 1: Spring Boot ڈیش بورڈ استعمال کرتے ہوئے (VS Code صارفین کے لیے تجویز کردہ)**

dev-container میں Spring Boot Dashboard ایکسٹینشن شامل ہے، جو تمام Spring Boot ایپلیکیشنز کو منظم کرنے کے لیے بصری انٹرفیس فراہم کرتا ہے۔ آپ اسے VS Code کے بائیں جانب ایکٹیویٹی بار میں (Spring Boot آئیکون دیکھیں) تلاش کر سکتے ہیں۔

Spring Boot Dashboard سے آپ یہ کر سکتے ہیں:
- ورک اسپیس میں موجود تمام Spring Boot ایپلیکیشنز دیکھیں
- ایک کلک سے ایپلیکیشنز کو شروع/روکیں
- ایپلیکیشن لاگز کو حقیقی وقت میں دیکھیں
- ایپلیکیشن کی حالت مانیٹر کریں

بس "introduction" کے سامنے پلے بٹن پر کلک کریں تاکہ یہ ماڈیول شروع ہو، یا تمام ماڈیولز کو ایک ساتھ شروع کریں۔

### بغیر یادداشت چیٹ (بائیں پینل)

پہلے یہ آزمائیں۔ پوچھیں "میرا نام جان ہے" اور پھر فوراً پوچھیں "میرا نام کیا ہے؟" ماڈل یاد نہیں رکھے گا کیونکہ ہر پیغام آزاد ہے۔ یہ بنیادی لینگویج ماڈل انٹیگریشن کا مسئلہ دکھاتا ہے - گفتگو کا کوئی سیاق و سباق نہیں۔

<img src="../../../translated_images/ur/simple-chat-stateless-demo.13aeb3978eab3234.webp" alt="Stateless Chat Demo" width="800"/>

*AI آپ کے پچھلے پیغام سے آپ کا نام یاد نہیں رکھتا*

### یادداشت کے ساتھ چیٹ (دائیں پینل)

اب یہی ترتیب یہاں آزمائیں۔ پوچھیں "میرا نام جان ہے" اور پھر "میرا نام کیا ہے؟" اس بار یاد رکھتا ہے۔ فرق MessageWindowChatMemory ہے - یہ گفتگو کی ہسٹری کو برقرار رکھتا ہے اور ہر درخواست کے ساتھ شامل کرتا ہے۔ یہ وہ طریقہ ہے جس سے پیداواری محاوری AI کام کرتا ہے۔

<img src="../../../translated_images/ur/conversational-chat-stateful-demo.e5be9822eb23ff59.webp" alt="Stateful Chat Demo" width="800"/>

*AI گفتگو کے ابتدا میں آپ کا نام یاد رکھتا ہے*

دونوں پینلز ایک ہی GPT-5.2 ماڈل استعمال کرتے ہیں۔ واحد فرق یادداشت ہے۔ یہ واضح کرتا ہے کہ یادداشت آپ کی ایپلیکیشن کے لیے کیا معنی رکھتی ہے اور کیوں یہ حقیقی استعمال کے لیے ناگزیر ہے۔
