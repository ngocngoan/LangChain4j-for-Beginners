## ویڈیو واک تھرو

یہ لائیو سیشن دیکھیں جو اس ماڈیول کے ساتھ شروع ہونے کا طریقہ بتاتا ہے: [LangChain4j کے ساتھ شروع کرنا - لائیو سیشن](https://www.youtube.com/live/nl_troDm8rQ?si=6b85S8xGjWnT2fX9)

## آپ کیا سیکھیں گے

اگر آپ نے فوری آغاز مکمل کیا ہے، تو آپ نے دیکھا کہ پرامپٹس بھیجنا اور جوابات حاصل کرنا کیسے ہوتا ہے۔ یہ بنیاد ہے، لیکن حقیقی ایپلیکیشنز کو زیادہ ضرورت ہوتی ہے۔ یہ ماڈیول آپ کو وہ مکالماتی AI بنانے کا طریقہ سکھاتا ہے جو سیاق و سباق کو یاد رکھتا ہے اور حالت برقرار رکھتا ہے — جو ایک ایک بار کا ڈیمو اور پروڈکشن کے قابل ایپلیکیشن کے درمیان فرق ہے۔

ہم اس گائیڈ کے دوران Azure OpenAI کا GPT-5.2 استعمال کریں گے کیونکہ اس کی جدید استدلال کی صلاحیتیں مختلف پیٹرنز کے رویے کو زیادہ واضح بناتی ہیں۔ جب آپ میموری شامل کریں گے، تو فرق واضح طور پر نظر آئے گا۔ اس سے سمجھنا آسان ہو جاتا ہے کہ ہر جزو آپ کی ایپلیکیشن میں کیا لاتا ہے۔

آپ ایک ایسی ایپلیکیشن بنائیں گے جو دونوں پیٹرنز کو ظاہر کرتی ہے:

**اسٹیٹ لیس چیٹ** - ہر درخواست خود مختار ہے۔ ماڈل کو پچھلے پیغامات کی کوئی یادداشت نہیں ہوتی۔ یہ وہی پیٹرن ہے جو آپ نے فوری آغاز میں استعمال کیا تھا۔

**اسٹیٹ فل گفتگو** - ہر درخواست مکالمے کی تاریخ شامل کرتی ہے۔ ماڈل متعدد دورانیوں میں سیاق و سباق کو برقرار رکھتا ہے۔ یہ وہی چیز ہے جو پروڈکشن ایپلیکیشنز کو چاہیے۔

## پیشگی شرائط

- Azure سبسکرپشن جس میں Azure OpenAI تک رسائی ہو
- Java 21، Maven 3.9+
- Azure CLI (https://learn.microsoft.com/en-us/cli/azure/install-azure-cli)
- Azure Developer CLI (azd) (https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/install-azd)

> **نوٹ:** Java، Maven، Azure CLI اور Azure Developer CLI (azd) پہلے سے فراہم کردہ devcontainer میں انسٹال کیے ہوئے ہیں۔

> **نوٹ:** یہ ماڈیول Azure OpenAI پر GPT-5.2 استعمال کرتا ہے۔ تعیناتی خود بخود `azd up` کے ذریعے ترتیب دی گئی ہے — کوڈ میں ماڈل کا نام تبدیل نہ کریں۔

## بنیادی مسئلے کو سمجھنا

لینگویج ماڈلز اسٹیٹ لیس ہوتے ہیں۔ ہر API کال خود مختار ہوتی ہے۔ اگر آپ "میرا نام جان ہے" بھیجیں اور پھر پوچھیں "میرا نام کیا ہے؟"، ماڈل کو اندازہ نہیں ہوتا کہ آپ نے اپنا تعارف ابھی کیا ہے۔ یہ ہر درخواست کو ایسے سمجھتا ہے جیسے یہ آپ کی پہلی گفتگو ہو۔

یہ سادہ سوال و جواب کے لیے ٹھیک ہے لیکن حقیقی ایپلیکیشنز کے لیے بے کار ہے۔ کسٹمر سروس بوٹس کو یاد رکھنا پڑتا ہے کہ آپ نے کیا کہا تھا۔ ذاتی معاونین کو سیاق و سباق کی ضرورت ہوتی ہے۔ کوئی بھی کثیر دورانیہ گفتگو میموری کی متقاضی ہوتی ہے۔

*اسٹیٹ لیس (آزاد کالز) اور اسٹیٹ فل (سیاق و سباق سے آگاہ) گفتگو کے درمیان فرق*

## ٹوکنز کو سمجھنا

گفتگو میں غوطہ لگانے سے پہلے، ٹوکنز کو سمجھنا ضروری ہے — وہ بنیادی اکائیاں جو زبان کے ماڈل پروسیس کرتے ہیں:

<img src="../../../translated_images/ur/token-explanation.c39760d8ec650181.webp" alt="ٹوکن کی وضاحت" width="800"/>

*متن کو ٹوکنز میں کیسے توڑا جاتا ہے کی مثال — "I love AI!" کو 4 الگ الگ پروسیسنگ یونٹس میں تبدیل کیا جاتا ہے*

ٹوکنز وہ طریقہ ہیں جن سے AI ماڈلز متن کو ماپتے اور پروسیس کرتے ہیں۔ الفاظ، رموز اوقاف، اور حتیٰ کہ خانے بھی ٹوکن ہو سکتے ہیں۔ آپ کے ماڈل کی پروسیسنگ کی حد ہے کہ وہ کتنے ٹوکنز کو ایک بار میں سنبھال سکتا ہے (GPT-5.2 کے لیے 400,000، جس میں 272,000 ان پٹ ٹوکنز اور 128,000 آؤٹ پٹ ٹوکنز شامل ہیں)۔ ٹوکنز کو سمجھنے سے آپ گفتگو کی لمبائی اور لاگتوں کا انتظام کر سکتے ہیں۔

## میموری کیسے کام کرتی ہے

چیٹ میموری اسٹیٹ لیس مسئلہ حل کرتی ہے اور گفتگو کی تاریخ کو برقرار رکھتی ہے۔ ماڈل کو آپ کی درخواست بھیجنے سے پہلے، فریم ورک متعلقہ پچھلے پیغامات کو شامل کرتا ہے۔ جب آپ پوچھیں "میرا نام کیا ہے؟"، تو سسٹم درحقیقت پوری گفتگو کی تاریخ بھیجتا ہے، جس سے ماڈل دیکھ پاتا ہے کہ آپ نے پہلے کہا تھا "میرا نام جان ہے"۔

LangChain4j میموری کی ایسی امپلیمینٹیشنز فراہم کرتا ہے جو یہ خود بخود سنبھالتی ہیں۔ آپ یہ منتخب کرتے ہیں کہ کتنے پیغامات رکھے جائیں اور فریم ورک سیاق و سباق کی ونڈو کا انتظام کرتا ہے۔

<img src="../../../translated_images/ur/memory-window.bbe67f597eadabb3.webp" alt="میموری ونڈو کا تصور" width="800"/>

*MessageWindowChatMemory حالیہ پیغامات کی ایک سلائیڈنگ ونڈو برقرار رکھتا ہے، پرانے پیغامات کو خود بخود خارج کرتا ہے*

## یہ کیسے LangChain4j استعمال کرتا ہے

یہ ماڈیول فوری آغاز کو Spring Boot کے ساتھ انٹیگریٹ کر کے اور گفتگو کی میموری شامل کر کے اس میں اضافہ کرتا ہے۔ یہ ہے کہ ٹکڑے کیسے جڑتے ہیں:

**انحصارات** — دو LangChain4j لائبریریاں شامل کریں:

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```
  
**چیٹ ماڈل** — Azure OpenAI کو Spring bean کے طور پر ترتیب دیں ([LangChainConfig.java](../../../01-introduction/src/main/java/com/example/langchain4j/config/LangChainConfig.java)):

```java
@Bean
public OpenAiOfficialChatModel openAiOfficialChatModel() {
    return OpenAiOfficialChatModel.builder()
            .baseUrl(azureEndpoint)
            .apiKey(azureApiKey)
            .modelName(deploymentName)
            .timeout(Duration.ofMinutes(5))
            .maxRetries(3)
            .build();
}
```
  
بلڈر `azd up` کے ذریعے متعین کردہ ماحولیات متغیرات سے اسناد پڑھتا ہے۔ `baseUrl` کو آپ کے Azure اینڈپوائنٹ پر سیٹ کرنا Azure OpenAI کلائنٹ کو Azure OpenAI کے ساتھ کام کرنے دیتا ہے۔

**گفتگو کی میموری** — MessageWindowChatMemory کے ذریعے چیٹ تاریخ کا پتہ رکھیں ([ConversationService.java](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java)):

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("My name is John"));
memory.add(AiMessage.from("Nice to meet you, John!"));

memory.add(UserMessage.from("What's my name?"));
AiMessage aiMessage = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage);
```
  
میموری `withMaxMessages(10)` کے ساتھ بنائیں تاکہ آخری 10 پیغامات محفوظ ہوں۔ صارف اور AI پیغامات کو ٹائپڈ ریپرسنٹیشنز کے ساتھ شامل کریں: `UserMessage.from(text)` اور `AiMessage.from(text)`۔ تاریخ کو `memory.messages()` کے ذریعے حاصل کریں اور ماڈل کو بھیجیں۔ یہ سروس ہر گفتگو کے ID کے لیے علیحدہ میموری اقسام رکھتی ہے، جس سے کئی صارف بیک وقت چیٹ کر سکتے ہیں۔

> **🤖 [GitHub Copilot](https://github.com/features/copilot) چیٹ کے ساتھ کوشش کریں:** [`ConversationService.java`](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java) کھولیں اور پوچھیں:  
> - "MessageWindowChatMemory ونڈو مکمل ہونے پر کون سے پیغامات خارج کرتا ہے؟"  
> - "کیا میں میموری اسٹوریج کو ڈیٹا بیس میں ان میموری کی جگہ خود عملدرآمد کر سکتا ہوں؟"  
> - "میں پرانی گفتگو کی تاریخ کو سمیٹنے کے لیے خلاصہ کاری کیسے شامل کروں؟"

اسٹیٹ لیس چیٹ اینڈپوائنٹ میں میموری شامل نہیں کی جاتی — بس `chatModel.chat(prompt)` جیسا کہ فوری آغاز میں تھا۔ اسٹیٹ فل اینڈپوائنٹ پیغامات میموری میں شامل کرتا ہے، تاریخ حاصل کرتا ہے، اور ہر درخواست کے ساتھ سیاق و سباق شامل کرتا ہے۔ وہی ماڈل ترتیب، مختلف پیٹرنز۔

## Azure OpenAI انفراسٹرکچر تعینات کریں

**Bash:**
```bash
cd 01-introduction
azd up  # سبسکرپشن اور مقام منتخب کریں (ایسٹ یو ایس 2 تجویز کردہ)
```
  
**PowerShell:**
```powershell
cd 01-introduction
azd up  # رکنیت اور مقام منتخب کریں (eastus2 کی سفارش کی گئی)
```
  
> **نوٹ:** اگر آپ کو timeout ایرر (`RequestConflict: Cannot modify resource ... provisioning state is not terminal`) ملے، بس `azd up` دوبارہ چلائیں۔ Azure وسائل ابھی پس منظر میں تعینات ہو رہے ہو سکتے ہیں، اور دوبارہ کوشش کرنے سے تعیناتی مکمل ہونے دیتی ہے جب وسائل ایک حتمی حالت میں پہنچ جائیں۔

یہ کرے گا:  
1. GPT-5.2 اور text-embedding-3-small ماڈلز کے ساتھ Azure OpenAI وسیلہ تعینات کرے گا  
2. خود بخود پروجیکٹ روٹ میں `.env` فائل بنائے گا جس میں اسناد ہوں گی  
3. تمام مطلوبہ ماحولیاتی متغیرات سیٹ کرے گا

**تعیناتی مسائل؟** تفصیلی مسئلہ حل کے لیے [Infrastructure README](infra/README.md) دیکھیں جس میں ذیلی ڈومین نام کے تنازعات، دستی Azure Portal تعیناتی کے مراحل، اور ماڈل کنفیگریشن کی رہنمائی شامل ہے۔

**تعیناتی کی کامیابی کی تصدیق کریں:**

**Bash:**
```bash
cat ../.env  # AZURE_OPENAI_ENDPOINT, API_KEY, وغیرہ دکھانا چاہیے۔
```
  
**PowerShell:**
```powershell
Get-Content ..\.env  # ظاہر کرنا چاہیے AZURE_OPENAI_ENDPOINT، API_KEY، وغیرہ۔
```
  
> **نوٹ:** `azd up` خود بخود `.env` فائل بناتا ہے۔ اگر آپ کو اسے بعد میں اپڈیٹ کرنا ہو، تو آپ یا تو `.env` فائل کو دستی طور پر ترمیم کر سکتے ہیں یا اسے دوبارہ بنانے کے لیے یہ چلائیں:  
>  
> **Bash:**
> ```bash
> cd ..
> bash .azd-env.sh
> ```
>  
> **PowerShell:**
> ```powershell
> cd ..
> .\.azd-env.ps1
> ```


## ایپلیکیشن کو مقامی طور پر چلائیں

**تعیناتی کی تصدیق:**

یقین دہانی کریں کہ روٹ ڈائریکٹری میں Azure اسناد کے ساتھ `.env` فائل موجود ہو:

**Bash:**
```bash
cat ../.env  # دکھانا چاہیے AZURE_OPENAI_ENDPOINT، API_KEY، DEPLOYMENT
```
  
**PowerShell:**
```powershell
Get-Content ..\.env  # AZURE_OPENAI_ENDPOINT، API_KEY، DEPLOYMENT دکھانا چاہیے
```
  
**ایپلیکیشنز شروع کریں:**

**آپشن 1: Spring Boot ڈیش بورڈ استعمال کریں (VS Code صارفین کے لیے سفارش کردہ)**

devcontainer میں Spring Boot Dashboard ایکسٹینشن شامل ہے جو تمام Spring Boot ایپلیکیشنز کا انتظام کرنے کے لیے ایک بصری انٹرفیس فراہم کرتا ہے۔ آپ اسے VS Code کی بائیں طرف ایکٹیویٹی بار میں تلاش کر سکتے ہیں (Spring Boot آئیکن ڈھونڈیں)۔

Spring Boot Dashboard سے، آپ:  
- ورک اسپیس میں دستیاب تمام Spring Boot ایپلیکیشنز دیکھ سکتے ہیں  
- ایک کلک سے ایپلیکیشنز شروع/روک سکتے ہیں  
- ایپلیکیشن لاگز حقیقی وقت میں دیکھ سکتے ہیں  
- ایپلیکیشن کی حالت کی نگرانی کر سکتے ہیں  

بس "introduction" کے ساتھ پلے بٹن پر کلک کریں تاکہ یہ ماڈیول شروع ہو، یا ایک ساتھ تمام ماڈیولز شروع کریں۔

<img src="../../../translated_images/ur/dashboard.69c7479aef09ff6b.webp" alt="Spring Boot ڈیش بورڈ" width="400"/>

**آپشن 2: شیل اسکرپٹس استعمال کریں**

تمام ویب ایپلیکیشنز شروع کریں (ماڈیول 01-04):

**Bash:**
```bash
cd ..  # روٹ ڈائریکٹری سے
./start-all.sh
```
  
**PowerShell:**
```powershell
cd ..  # روٹ ڈائریکٹری سے
.\start-all.ps1
```
  
یا صرف یہ ماڈیول شروع کریں:

**Bash:**
```bash
cd 01-introduction
./start.sh
```
  
**PowerShell:**
```powershell
cd 01-introduction
.\start.ps1
```
  
دونوں اسکرپٹس روٹ `.env` فائل سے خود بخود ماحولیاتی متغیرات لوڈ کرتے ہیں اور اگر JARs موجود نہ ہوں تو انہیں بنائیں گے۔

> **نوٹ:** اگر آپ تمام ماڈیولز کو شروع کرنے سے پہلے دستی طور پر بنانا پسند کرتے ہیں:  
>  
> **Bash:**
> ```bash
> cd ..  # Go to root directory
> mvn clean package -DskipTests
> ```
>  
> **PowerShell:**
> ```powershell
> cd ..  # Go to root directory
> mvn clean package -DskipTests
> ```
  
براوزر میں http://localhost:8080 کھولیں۔

**روکنے کے لیے:**

**Bash:**
```bash
./stop.sh  # صرف یہ ماڈیول
# یا
cd .. && ./stop-all.sh  # تمام ماڈیولز
```
  
**PowerShell:**
```powershell
.\stop.ps1  # صرف یہ ماڈیول
# یا
cd ..; .\stop-all.ps1  # تمام ماڈیولز
```

*ڈیش بورڈ جو سادہ چیٹ (اسٹیٹ لیس) اور مکالماتی چیٹ (اسٹیٹ فل) آپشنز دکھاتا ہے*

### اسٹیٹ لیس چیٹ (بائیں پینل)

سب سے پہلے یہ آزمائیں۔ "میرا نام جان ہے" پوچھیں اور پھر فوراً "میرا نام کیا ہے؟" ماڈل یاد نہیں رکھے گا کیونکہ ہر پیغام خود مختار ہے۔ یہ بنیادی زبان ماڈل انٹیگریشن کے ساتھ بنیادی مسئلہ ظاہر کرتا ہے — کوئی گفتگو کا سیاق و سباق نہیں۔

<img src="../../../translated_images/ur/simple-chat-stateless-demo.13aeb3978eab3234.webp" alt="اسٹیٹ لیس چیٹ ڈیمو" width="800"/>

*AI پچھلے پیغام سے آپ کا نام یاد نہیں رکھتا*

### اسٹیٹ فل چیٹ (دائیں پینل)

اب یہی تسلسل یہاں آزمائیں۔ "میرا نام جان ہے" اور پھر "میرا نام کیا ہے؟" اس بار ماڈل یاد رکھتا ہے۔ فرق MessageWindowChatMemory ہے — یہ گفتگو کی تاریخ برقرار رکھتا ہے اور ہر درخواست کے ساتھ سیاق و سباق شامل کرتا ہے۔ یہ پروڈکشن مکالماتی AI کا طریقہ ہے۔

<img src="../../../translated_images/ur/conversational-chat-stateful-demo.e5be9822eb23ff59.webp" alt="اسٹیٹ فل چیٹ ڈیمو" width="800"/>

*AI گفتگو کے شروع سے آپ کا نام یاد رکھتا ہے*

دونوں پینلز ایک ہی GPT-5.2 ماڈل استعمال کرتے ہیں۔ فرق صرف میموری کا ہے۔ یہ واضح کرتا ہے کہ میموری آپ کی ایپلیکیشن میں کیا لاتی ہے اور یہ حقیقی استعمال کے کیسز کے لیے کیوں ضروری ہے۔
