## آپ کیا سیکھیں گے

<img src="../../../translated_images/ur/what-youll-learn.c68269ac048503b2.webp" alt="آپ کیا سیکھیں گے" width="800"/>

پچھلے ماڈیول میں، آپ نے دیکھا کہ میموری کس طرح کنورسیشنل اے آئی کو ممکن بناتی ہے اور GitHub ماڈلز کو بنیادی بات چیت کے لیے استعمال کیا۔ اب ہم اس بات پر توجہ دیں گے کہ آپ سوالات کیسے پوچھتے ہیں — یعنی پرامپٹس خود — Azure OpenAI کے GPT-5.2 کے ذریعے۔ آپ کے پرامپٹس کی ساخت آپ کو ملنے والے جوابات کے معیار کو نمایاں طور پر متاثر کرتی ہے۔ ہم بنیادی پرامپٹنگ تکنیکوں کا جائزہ لے کر شروع کرتے ہیں، اور پھر آٹھ جدید پیٹرنز میں جاتے ہیں جو GPT-5.2 کی صلاحیتوں کو مکمل فائدہ دیتے ہیں۔

ہم GPT-5.2 کا استعمال کریں گے کیونکہ یہ استدلال کو کنٹرول کرنے کا موقع فراہم کرتا ہے - آپ ماڈل کو بتا سکتے ہیں کہ جواب دینے سے پہلے کتنا سوچنا ہے۔ یہ مختلف پرامپٹنگ حکمت عملیوں کو زیادہ واضح بناتا ہے اور آپ کو سمجھنے میں مدد دیتا ہے کہ کون سا طریقہ کب استعمال کرنا ہے۔ اس کے علاوہ Azure کے GPT-5.2 کے لیے کم ریٹ لمٹس GitHub ماڈلز کے مقابلے میں فائدہ مند ہیں۔

## پرامپٹ انجینئرنگ کو سمجھنا

<img src="../../../translated_images/ur/what-is-prompt-engineering.5c392a228a1f5823.webp" alt="پرامپٹ انجینئرنگ کیا ہے؟" width="800"/>

پرامپٹ انجینئرنگ سے مراد ایسا ان پٹ متن ڈیزائن کرنا ہے جو مستقل آپ کو مطلوبہ نتائج حاصل کرے۔ یہ صرف سوالات پوچھنے کے بارے میں نہیں ہے - بلکہ درخواستوں کی اس طرح ساخت کرنا ہے کہ ماڈل بالکل سمجھ جائے کہ آپ کیا چاہتے ہیں اور اسے کیسے فراہم کرنا ہے۔

اسے ایسے سمجھیں جیسے آپ کسی ساتھی کو ہدایات دے رہے ہوں۔ "بگ کو ٹھیک کرو" مبہم ہے۔ "UserService.java کی لائن 45 میں null pointer استثناء کو null چیک شامل کرکے ٹھیک کریں" مخصوص ہے۔ زبان کے ماڈلز بھی اسی طرح کام کرتے ہیں — وضاحت اور ساخت اہم ہے۔

<img src="../../../translated_images/ur/how-langchain4j-fits.dfff4b0aa5f7812d.webp" alt="لانگ چین4جے کیسے فٹ ہوتا ہے" width="800"/>

LangChain4j بنیادی ڈھانچہ مہیا کرتا ہے — ماڈل کنکشنز، میموری، اور پیغام کی اقسام — جبکہ پرامپٹ پیٹرنز صرف وہ محتاط ساخت شدہ متن ہیں جو آپ اس بنیادی ڈھانچے کے ذریعے بھیجتے ہیں۔ کلیدی بنیادی اراکین `SystemMessage` (جو AI کے رویے اور کردار کو سیٹ کرتا ہے) اور `UserMessage` (جو آپ کی اصل درخواست لے کر آتا ہے) ہیں۔

## پرامپٹ انجینئرنگ کے بنیادی اصول

<img src="../../../translated_images/ur/five-patterns-overview.160f35045ffd2a94.webp" alt="پرامپٹ انجینئرنگ کے پانچ پیٹرنز کا جائزہ" width="800"/>

اس ماڈیول میں جدید پیٹرنز میں جانے سے پہلے، آئیے پانچ بنیادی پرامپٹنگ تکنیکوں کا جائزہ لیں۔ یہ وہ بنیادی بلاکس ہیں جنہیں ہر پرامپٹ انجینئر کو جاننا چاہیے۔ اگر آپ نے پہلے [Quick Start ماڈیول](../00-quick-start/README.md#2-prompt-patterns) کیا ہے تو آپ نے انہیں عمل میں دیکھا ہوگا — یہ ان کے پیچھے کا نظریاتی فریم ورک ہے۔

### زیرو شاٹ پرامپٹنگ

سب سے آسان طریقہ: ماڈل کو براہ راست ہدایت دیں بغیر کسی مثال کے۔ ماڈل تربیت پر مکمل انحصار کرتا ہے کہ وہ کام کو سمجھے اور انجام دے۔ یہ سادہ درخواستوں کے لیے بہتر کام کرتا ہے جہاں متوقع رویہ واضح ہو۔

<img src="../../../translated_images/ur/zero-shot-prompting.7abc24228be84e6c.webp" alt="زیرو شاٹ پرامپٹنگ" width="800"/>

*بغیر مثالوں کے براہ راست ہدایت — ماڈل صرف ہدایت سے کام سمجھتا ہے*

```java
String prompt = "Classify this sentiment: 'I absolutely loved the movie!'";
String response = model.chat(prompt);
// جواب: "مثبت"
```
  
**کب استعمال کریں:** سادہ درجہ بندی، سیدھے سوالات، ترجمے، یا کوئی بھی کام جسے ماڈل بغیر اضافی رہنمائی کے ہینڈل کر سکتا ہو۔

### فیو شاٹ پرامپٹنگ

مثالیں فراہم کریں جو ماڈل کو جس پیٹرن کی پیروی کرنی ہے دکھائیں۔ ماڈل آپ کی مثالوں سے ان پٹ-آؤٹ پٹ فارمیٹ سیکھتا ہے اور اسے نئے ان پٹ پر لاگو کرتا ہے۔ اس سے ایسے کاموں میں مستقل مزاجی نمایاں طور پر بہتر ہوتی ہے جہاں مطلوبہ فارمیٹ یا رویہ واضح نہیں ہوتا۔

<img src="../../../translated_images/ur/few-shot-prompting.9d9eace1da88989a.webp" alt="فیو شاٹ پرامپٹنگ" width="800"/>

*مثالوں سے سیکھنا — ماڈل پیٹرن شناخت کرتا ہے اور نئے ان پٹ پر لاگو کرتا ہے*

```java
String prompt = """
    Classify the sentiment as positive, negative, or neutral.
    
    Examples:
    Text: "This product exceeded my expectations!" → Positive
    Text: "It's okay, nothing special." → Neutral
    Text: "Waste of money, very disappointed." → Negative
    
    Now classify this:
    Text: "Best purchase I've made all year!"
    """;
String response = model.chat(prompt);
```
  
**کب استعمال کریں:** حسبِ ضرورت درجہ بندی، مستقل فارمیٹنگ، مخصوص دائرہ کے کام، یا جب زیرو شاٹ نتائج غیر مستقل ہوں۔

### چین آف تھوٹ

ماڈل سے کہیں کہ وہ اپنا استدلال قدم بہ قدم دکھائے۔ جواب پر فوری چھلانگ لگانے کی بجائے، ماڈل مسئلے کو توڑ کر ہر حصہ واضح طور پر حل کرتا ہے۔ یہ ریاضی، منطق، اور کئی قدموں والے استدلال کے کاموں میں درستگی کو بہتر بناتا ہے۔

<img src="../../../translated_images/ur/chain-of-thought.5cff6630e2657e2a.webp" alt="چین آف تھوٹ پرامپٹنگ" width="800"/>

*قدم بہ قدم استدلال — پیچیدہ مسائل کو واضح منطقی مراحل میں توڑنا*

```java
String prompt = """
    Problem: A store has 15 apples. They sell 8 apples and then 
    receive a shipment of 12 more apples. How many apples do they have now?
    
    Let's solve this step-by-step:
    """;
String response = model.chat(prompt);
// ماڈل دکھاتا ہے: 15 - 8 = 7، پھر 7 + 12 = 19 سیب
```
  
**کب استعمال کریں:** ریاضی کے مسائل، منطقی پہیلیاں، ڈی بگنگ، یا کوئی بھی کام جہاں استدلالی عمل دکھانا درستگی اور اعتماد کو بڑھاتا ہے۔

### رول بیسڈ پرامپٹنگ

سوال پوچھنے سے پہلے AI کے لیے کوئی کردار یا شخصیت سیٹ کریں۔ یہ ایسا سیاق و سباق فراہم کرتا ہے جو جواب کے لہجے، گہرائی، اور توجہ کو شکل دیتا ہے۔ "سوفٹ ویئر آرکیٹیکٹ" ایک "جونئیر ڈیویلپر" یا "سیکیورٹی آڈیٹر" کے مقابلے میں مختلف مشورہ دیتا ہے۔

<img src="../../../translated_images/ur/role-based-prompting.a806e1a73de6e3a4.webp" alt="رول بیسڈ پرامپٹنگ" width="800"/>

*سیاق و سباق اور کردار سیٹ کرنا — ایک ہی سوال کو مختلف کردار میں مختلف جواب ملتا ہے*

```java
String prompt = """
    You are an experienced software architect reviewing code.
    Provide a brief code review for this function:
    
    def calculate_total(items):
        total = 0
        for item in items:
            total = total + item['price']
        return total
    """;
String response = model.chat(prompt);
```
  
**کب استعمال کریں:** کوڈ ریویوز، تدریس، مخصوص دائرہ کار کا تجزیہ، یا جب آپ کو کسی خاص مہارت کی سطح یا نقطہ نظر کے مطابق جوابات چاہیے ہوں۔

### پرامپٹ ٹیمپلیٹس

متغیر جگہ داروں کے ساتھ دوبارہ استعمال ہونے والے پرامپٹس بنائیں۔ ہر بار نیا پرامپٹ لکھنے کی بجائے، ایک ٹیمپلیٹ مرتب کریں اور مختلف قدروں سے اسے بھریں۔ LangChain4j کی `PromptTemplate` کلاس `{{variable}}` نحو کے ساتھ یہ آسان بناتی ہے۔

<img src="../../../translated_images/ur/prompt-templates.14bfc37d45f1a933.webp" alt="پرامپٹ ٹیمپلیٹس" width="800"/>

*متغیر جگہ داروں کے ساتھ دوبارہ استعمال کے قابل پرامپٹس — ایک ٹیمپلیٹ، کئی استعمال*

```java
PromptTemplate template = PromptTemplate.from(
    "What's the best time to visit {{destination}} for {{activity}}?"
);

Prompt prompt = template.apply(Map.of(
    "destination", "Paris",
    "activity", "sightseeing"
));

String response = model.chat(prompt.text());
```
  
**کب استعمال کریں:** مختلف ان پٹ کے ساتھ بار بار سوالات، بیچ پروسیسنگ، دوبارہ استعمال کے قابل AI ورک فلو بنانے کے لیے، یا ایسی کسی بھی صورت جہاں پرامپٹ کی ساخت ایک جیسی رہے پر ڈیٹا مختلف ہو۔

---

یہ پانچ بنیادی اصول آپ کو زیادہ تر پرامپٹنگ کاموں کے لیے مضبوط ٹول کٹ دیتے ہیں۔ باقی ماڈیول ان پر مبنی ہے، اور **آٹھ جدید پیٹرنز** شامل کرتا ہے جو GPT-5.2 کی استدلالی کنٹرول، خود جائزہ، اور منظم آؤٹ پٹ صلاحیتوں کا فائدہ اٹھاتے ہیں۔

## جدید پیٹرنز

بنیادی اصول سمجھنے کے بعد، آئیے آٹھ جدید پیٹرنز کی طرف بڑھیں جو اس ماڈیول کو منفرد بناتے ہیں۔ تمام مسائل کے لیے ایک ہی طریقہ کار ضروری نہیں۔ کچھ سوالات کے فوری جواب چاہیے، کچھ کو گہرائی میں سوچنا پڑتا ہے۔ کچھ کو دکھائی دینے والا استدلال چاہیے، بعض صرف نتائج۔ ہر پیٹرن مختلف منظرنامے کے لیے بہتر بنایا گیا ہے — اور GPT-5.2 کا استدلال کنٹرول فرق کو اور واضح بناتا ہے۔

<img src="../../../translated_images/ur/eight-patterns.fa1ebfdf16f71e9a.webp" alt="آٹھ پرامپٹنگ پیٹرنز" width="800"/>

*آٹھ پرامپٹ انجینئرنگ پیٹرنز اور ان کے استعمال کا جائزہ*

<img src="../../../translated_images/ur/reasoning-control.5cf85f0fc1d0c1f3.webp" alt="GPT-5.2 کے ساتھ استدلال کا کنٹرول" width="800"/>

*GPT-5.2 کا استدلال کنٹرول آپ کو بتانے دیتا ہے کہ ماڈل کو کتنا سوچنا چاہیے — جلدی سیدھے جواب سے لے کر گہرائی میں تحقیق تک*

<img src="../../../translated_images/ur/reasoning-effort.db4a3ba5b8e392c1.webp" alt="استدلال کی کوشش کی موازنہ" width="800"/>

*کم جوش (تیز، سیدھا) بمقابلہ زیادہ جوش (مفصل، تلاش کرنے والا) استدلال کے طریقے*

**کم جوش (تیز اور مرکوز)** - آسان سوالات کے لیے جہاں آپ تیز، سیدھے جواب چاہتے ہیں۔ ماڈل کم سے کم استدلال کرتا ہے — زیادہ سے زیادہ 2 قدم۔ اس کا استعمال حساب، تلاش، یا سیدھے سادے سوالات کے لیے کریں۔

```java
String prompt = """
    <reasoning_effort>low</reasoning_effort>
    <instruction>maximum 2 reasoning steps</instruction>
    
    What is 15% of 200?
    """;

String response = chatModel.chat(prompt);
```
  
> 💡 **GitHub کوپائلٹ کے ساتھ دریافت کریں:** [`Gpt5PromptService.java`](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java) کھولیں اور پوچھیں:  
> - "کم جوش اور زیادہ جوش پرامپٹنگ پیٹرنز میں کیا فرق ہے؟"  
> - "XML ٹیگز پرامپٹس میں AI کے جواب کی ساخت میں کیسے مدد دیتے ہیں؟"  
> - "میں سیلف ریفلیکشن پیٹرنز کب اور سیدھی ہدایت کب استعمال کروں؟"

**زیادہ جوش (گہرائی اور مفصل)** - پیچیدہ مسائل کے لیے جہاں آپ جامع تجزیہ چاہتے ہیں۔ ماڈل بھرپور طریقے سے تحقیق کرتا ہے اور تفصیلی استدلال دکھاتا ہے۔ اس کا استعمال سسٹم ڈیزائن، آرکیٹیکچر کے فیصلے، یا پیچیدہ تحقیق کے لیے کریں۔

```java
String prompt = """
    <reasoning_effort>high</reasoning_effort>
    <instruction>explore thoroughly, show detailed reasoning</instruction>
    
    Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```
  
**ٹاسک ایکزیکیوشن (قدم بہ قدم پیش رفت)** - کئی مراحل پر مبنی ورک فلو کے لیے۔ ماڈل پہلے ایک منصوبہ فراہم کرتا ہے، ہر قدم کام کرتے ہوئے بیان کرتا ہے، پھر خلاصہ دیتا ہے۔ اس کا استعمال مائیگریشنز، نفاذات، یا کسی بھی کئی مراحل پر مشتمل عمل کے لیے کریں۔

```java
String prompt = """
    <task>Create a REST endpoint for user registration</task>
    <preamble>Provide an upfront plan</preamble>
    <narration>Narrate each step as you work</narration>
    <summary>Summarize what was accomplished</summary>
    """;

String response = chatModel.chat(prompt);
```
  
چین آف تھوٹ پرامپٹنگ واضح طور پر ماڈل سے کہتی ہے کہ وہ اپنے استدلالی عمل کو ظاہر کرے، جو پیچیدہ کاموں کی درستگی کو بہتر بناتا ہے۔ قدم بہ قدم تجزیہ انسان اور AI دونوں کو منطق سمجھنے میں مدد دیتا ہے۔

> **🤖 [GitHub کوپائلٹ](https://github.com/features/copilot) چیٹ کے ساتھ کوشش کریں:** اس پیٹرن کے بارے میں پوچھیں:  
> - "طویل چلنے والے آپریشنز کے لیے ٹاسک ایکزیکیوشن پیٹرن کو کیسے اپنایا جائے؟"  
> - "پروڈکشن ایپلیکیشنز میں ٹول پری ایمبلز کی ساخت کے بہترین طریقے کیا ہیں؟"  
> - "UI میں عبوری پیش رفت کی تازہ کاریوں کو کیسے گرفت اور دکھایا جا سکتا ہے؟"

<img src="../../../translated_images/ur/task-execution-pattern.9da3967750ab5c1e.webp" alt="ٹاسک ایکزیکیوشن پیٹرن" width="800"/>

*منصوبہ بنائیں → عمل کریں → خلاصہ کریں چند مراحل کے کاموں کے لیے*

**سیلف ریفلیکٹنگ کوڈ** - پروڈکشن معیار کا کوڈ تیار کرنے کے لیے۔ ماڈل کوڈ تیار کرتا ہے، معیار کی پیمائش سے چیک کرتا ہے، اور اسے بار بار بہتر بناتا ہے۔ جب نئے فیچرز یا سروسز بنائیں تو اس کا استعمال کریں۔

```java
String prompt = """
    <task>Create an email validation service</task>
    <quality_criteria>
    - Correct logic and error handling
    - Best practices (clean code, proper naming)
    - Performance optimization
    - Security considerations
    </quality_criteria>
    <instruction>Generate code, evaluate against criteria, improve iteratively</instruction>
    """;

String response = chatModel.chat(prompt);
```
  
<img src="../../../translated_images/ur/self-reflection-cycle.6f71101ca0bd28cc.webp" alt="سیلف ریفلیکشن سائیکل" width="800"/>

*تکراری بہتری کا چکر - تخلیق کریں، جائزہ لیں، مسائل شناخت کریں، بہتر کریں، دہرائیں*

**منظم تجزیہ** - مستقل جائزے کے لیے۔ ماڈل ایک مقررہ فریم ورک (صحیحیت، عمل، کارکردگی، سیکیورٹی) کے تحت کوڈ کا جائزہ لیتا ہے۔ کوڈ ریویوز یا معیار کی جانچ کے لیے استعمال کریں۔

```java
String prompt = """
    <code>
    public List getUsers() {
        return database.query("SELECT * FROM users");
    }
    </code>
    
    <framework>
    Evaluate using these categories:
    1. Correctness - Logic and functionality
    2. Best Practices - Code quality
    3. Performance - Efficiency concerns
    4. Security - Vulnerabilities
    </framework>
    """;

String response = chatModel.chat(prompt);
```

> - "مختلف قسم کے کوڈ ریویوز کے لیے تجزیاتی فریم ورک کو کیسے حسبِ ضرورت بنایا جا سکتا ہے؟"  
> - "پروگراماتی طور پر منظم آؤٹ پٹ کو پارس اور عمل کرنے کا بہترین طریقہ کیا ہے؟"  
> - "مستقل سنگینی کی سطحوں کو مختلف ریویو اجلاسوں میں کیسے یقینی بنایا جائے؟"

**کئی چکر کی بات چیت** - ایسی گفتگو کے لیے جسے سیاق و سباق کی ضرورت ہو۔ ماڈل پچھلے پیغامات یاد رکھتا ہے اور ان پر مزید تعمیر کرتا ہے۔ انٹرایکٹو مدد سیشنز یا پیچیدہ سوال و جواب کے لیے استعمال کریں۔

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("What is Spring Boot?"));
AiMessage aiMessage1 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage1);

memory.add(UserMessage.from("Show me an example"));
AiMessage aiMessage2 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage2);
```

*بات چیت کا سیاق و سباق کئی چکروں میں جمع ہوتا ہے جب تک کہ ٹوکن کی حد نہ پہنچ جائے*

**قدم بہ قدم استدلال** - ایسے مسائل کے لیے جہاں واضح منطق کی ضرورت ہو۔ ماڈل ہر قدم کے لیے واضح استدلال دکھاتا ہے۔ ریاضی کے مسائل، منطقی پہیلیوں، یا سمجھنے کے لیے کہ سوچنے کا عمل کیسے ہے، اس کے لیے استعمال کریں۔

```java
String prompt = """
    <instruction>Show your reasoning step-by-step</instruction>
    
    If a train travels 120 km in 2 hours, then stops for 30 minutes,
    then travels another 90 km in 1.5 hours, what is the average speed
    for the entire journey including the stop?
    """;

String response = chatModel.chat(prompt);
```

*مسائل کو واضح منطقی مراحل میں تقسیم کرنا*

**محدود شدہ آؤٹ پٹ** - ایسے جوابات کے لیے جنہیں مخصوص فارمٹ کے تقاضے پورے کرنا ہوں۔ ماڈل فارمٹ اور لمبائی کے قواعد کی سختی سے پیروی کرتا ہے۔ خلاصوں یا جہاں آپ کو بالکل درست آؤٹ پٹ کی ساخت چاہیے، وہاں استعمال کریں۔

```java
String prompt = """

    - Exactly 100 words
    - Bullet point format
    - Technical terms only
    </constraints>
    
    Summarize the key concepts of machine learning.
    """;

```

اسپرنگ بوٹ ڈیش بورڈ سے، آپ کر سکتے ہیں:  
- ورک اسپیس میں دستیاب تمام اسپرنگ بوٹ ایپلیکیشنز دیکھیں  
- ایک کلک میں ایپلیکیشنز شروع/بند کریں  
- ایپلیکیشن لاگز کو حقیقی وقت میں دیکھیں  
- ایپلیکیشن کی حالت مانیٹر کریں  

بس "prompt-engineering" کے پاس پلے بٹن پر کلک کریں تاکہ یہ ماڈیول شروع ہو جائے، یا تمام ماڈیولز کو ایک ساتھ شروع کریں۔

## آپ کیا واقعی سیکھ رہے ہیں  

**دلیل کی کوشش ہر چیز بدل دیتی ہے**  

GPT-5.2 آپ کو اپنے پرامپٹس کے ذریعے حسابی کوشش کو کنٹرول کرنے دیتا ہے۔ کم کوشش کا مطلب ہے فوری جوابات کم تحقیق کے ساتھ۔ زیادہ کوشش کا مطلب ہے کہ ماڈل عمیق سوچ میں وقت لگاتا ہے۔ آپ سیکھ رہے ہیں کہ کوشش کو کام کی پیچیدگی کے مطابق استعمال کریں - آسان سوالات پر وقت ضائع نہ کریں، اور پیچیدہ فیصلوں میں جلد بازی نہ کریں۔  

**ساخت رویے کی رہنمائی کرتی ہے**  

کیا آپ نے پرامپٹس میں XML ٹیگز دیکھے؟ یہ صرف سجاوٹ نہیں ہیں۔ ماڈلز منظم ہدایات پر آزادانہ متن سے زیادہ بھروسہ کرتے ہیں۔ جب آپ کو کثیر مراحل والے عمل یا پیچیدہ منطق چاہیے، تو ساخت ماڈل کی مدد کرتی ہے کہ وہ معلوم رکھے کہ وہ کہاں ہے اور اگلا کیا ہوگا۔

*مضبوط سیکشنز اور XML طرز کی تنظیم کے ساتھ اچھی ساخت کا پرامپٹ*  

**خود جائزہ کے ذریعے معیار**  

خود تجزیہ کرنے والے پیٹرنز معیار کے معیارات کو واضح کرتے ہیں۔ اس کے بجائے کہ ماڈل سے امید کی جائے کہ "صحیح کرے"، آپ اسے بتاتے ہیں کہ "صحیح" کا مطلب کیا ہے: درست منطق، غلطی کا ہینڈلنگ، کارکردگی، سیکورٹی۔ پھر ماڈل اپنی آؤٹ پٹ کا جائزہ لے کر بہتری لا سکتا ہے۔ یہ کوڈ جنریشن کو قرعہ اندازی سے ایک مستحکم عمل میں بدل دیتا ہے۔  

**سیاق و سباق محدود ہے**  

ملٹی ٹرن بات چیت ہر درخواست کے ساتھ پیغام کی تاریخ شامل کر کے کام کرتی ہے۔ مگر حد ہوتی ہے - ہر ماڈل کے ٹوکنز کی ایک زیادہ سے زیادہ تعداد ہوتی ہے۔ جب بات چیت بڑھتی ہے، تو آپ کو مناسب سیاق و سباق کو محفوظ رکھنے کے طریقے تلاش کرنے ہوں گے بغیر حد پہنچے۔ یہ ماڈیول دکھاتا ہے کہ یادداشت کیسے کام کرتی ہے؛ بعد میں آپ سیکھیں گے کب خلاصہ کرنا ہے، کب بھولنا ہے، اور کب معلومات دوبارہ لانی ہے۔
