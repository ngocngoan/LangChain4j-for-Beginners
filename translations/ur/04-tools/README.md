## آپ کیا سیکھیں گے

اب تک، آپ نے سیکھا ہے کہ AI کے ساتھ بات چیت کیسے کی جاتی ہے، مؤثر پرامپٹس کیسے بنائے جاتے ہیں، اور جوابات کو اپنے دستاویزات میں کس طرح گراؤنڈ کیا جاتا ہے۔ لیکن ابھی بھی ایک بنیادی حد بندی ہے: زبان کے ماڈل صرف متن تخلیق کر سکتے ہیں۔ وہ موسم چیک نہیں کر سکتے، حساب کتاب نہیں کر سکتے، ڈیٹا بیس کو کوئری نہیں کر سکتے، یا بیرونی نظاموں کے ساتھ تعامل نہیں کر سکتے۔

ٹولز اسے بدل دیتے ہیں۔ ماڈل کو ایسی فنکشنز تک رسائی دے کر جو وہ کال کر سکتا ہے، آپ اسے ایک متن جنریٹر سے ایک ایجنٹ میں تبدیل کر دیتے ہیں جو عمل کر سکتا ہے۔ ماڈل فیصلہ کرتا ہے کہ اسے کب ٹول کی ضرورت ہے، کون سا ٹول استعمال کرنا ہے، اور کون سے پیرامیٹرز پاس کرنے ہیں۔ آپ کا کوڈ فنکشن کو چلاتا ہے اور نتیجہ واپس کرتا ہے۔ ماڈل اس نتیجے کو اپنے جواب میں شامل کرتا ہے۔

## پہلے سے درکار چیزیں

- ماڈیول ۰۱ مکمل کر لیا ہے (Azure OpenAI وسائل تعینات کیے ہوئے ہیں)
- روٹ ڈائریکٹری میں `.env` فائل Azure اسناد کے ساتھ موجود ہے (ماڈیول ۰۱ میں `azd up` کے ذریعے بنائی گئی)

> **نوٹ:** اگر آپ نے ماڈیول ۰۱ مکمل نہیں کیا ہے، تو پہلے وہاں دی گئی تعیناتی کی ہدایات پر عمل کریں۔

## ٹولز کے ساتھ AI ایجنٹس کو سمجھنا

> **📝 نوٹ:** اس ماڈیول میں "ایجنٹس" سے مراد AI اسسٹنٹس ہیں جن کو ٹول کالنگ کی صلاحیتوں سے بہتر بنایا گیا ہے۔ یہ مختلف ہے **ایجنٹک AI** پیٹرنز سے (خود مختار ایجنٹس جو منصوبہ بندی، یادداشت، اور متعدد مراحل کی استدلال کرتے ہیں) جنہیں ہم [ماڈیول ۰۵: MCP](../05-mcp/README.md) میں کور کریں گے۔

بغیر ٹولز کے، زبان کا ماڈل صرف اپنے تربیتی ڈیٹا سے متن پیدا کر سکتا ہے۔ اگر آپ اس سے موجودہ موسم دریافت کریں، تو اسے اندازہ لگانا پڑے گا۔ ٹولز دیں، اور یہ موسم کی API کال کر سکتا ہے، حساب کتاب کر سکتا ہے، یا ڈیٹا بیس کو کوئری کر سکتا ہے — پھر ان حقیقی نتائج کو اپنی جواب میں شامل کر لیتا ہے۔

<img src="../../../translated_images/ur/what-are-tools.724e468fc4de64da.webp" alt="Wبغیر ٹولز کے بمقابلہ ٹولز کے ساتھ" width="800"/>

*بغیر ٹولز ماڈل صرف اندازہ لگا سکتا ہے — ٹولز کے ساتھ یہ APIs کال کر سکتا ہے، حساب لگا سکتا ہے، اور حقیقی وقت کا ڈیٹا واپس کر سکتا ہے۔*

ٹولز کے ساتھ AI ایجنٹ ایک **ریٰزننگ اور عمل (ReAct)** پیٹرن پر عمل کرتا ہے۔ ماڈل صرف جواب نہیں دیتا — یہ سوچتا ہے کہ اسے کیا چاہیے، عمل کرتا ہے ٹول کال کر کے، نتیجہ دیکھتا ہے، اور پھر فیصلہ کرتا ہے کہ دوبارہ عمل کرے یا آخری جواب دے:

1. **توجہ کریں** — ایجنٹ صارف کے سوال کا تجزیہ کرتا ہے اور فیصلہ کرتا ہے کہ اسے کس معلومات کی ضرورت ہے
2. **عمل کریں** — ایجنٹ صحیح ٹول منتخب کرتا ہے، درست پیرامیٹرز تیار کرتا ہے، اور اسے کال کرتا ہے
3. **مشاہدہ کریں** — ایجنٹ ٹول کی آؤٹ پٹ وصول کرتا ہے اور نتیجہ کا جائزہ لیتا ہے
4. **دہرائیں یا جواب دیں** — اگر مزید ڈیٹا چاہیے، تو ایجنٹ لوپ میں واپس آتا ہے؛ ورنہ قدرتی زبان میں جواب بناتا ہے

<img src="../../../translated_images/ur/react-pattern-detail.96a5efeeb6dd2f61.webp" alt="ری ایکٹ پیٹرن" width="800"/>

*ری ایکٹ چکر — ایجنٹ سوچتا ہے کہ کیا کرنا ہے، ٹول کال کر کے عمل کرتا ہے، نتیجہ دیکھتا ہے، اور جب تک آخری جواب نہ دے لوپ چلتا ہے۔*

یہ خودکار طریقے سے ہوتا ہے۔ آپ ٹولز اور ان کی تفصیلات تعریف کرتے ہیں۔ ماڈل فیصلہ کرتا ہے کہ کب اور کیسے استعمال کرنا ہے۔

## ٹول کالنگ کیسے کام کرتی ہے

### ٹول کی تعریفات

[WeatherTool.java](../../../04-tools/src/main/java/com/example/langchain4j/agents/tools/WeatherTool.java) | [TemperatureTool.java](../../../04-tools/src/main/java/com/example/langchain4j/agents/tools/TemperatureTool.java)

آپ فنکشنز کو واضح تفصیلات اور پیرامیٹر وضاحتوں کے ساتھ تعریف کرتے ہیں۔ ماڈل ان تفصیلات کو اپنے سسٹم پرامپٹ میں دیکھتا ہے اور سمجھتا ہے کہ ہر ٹول کیا کرتا ہے۔

```java
@Component
public class WeatherTool {
    
    @Tool("Get the current weather for a location")
    public String getCurrentWeather(@P("Location name") String location) {
        // آپ کی موسم تلاش کی منطق
        return "Weather in " + location + ": 22°C, cloudy";
    }
}

@AiService
public interface Assistant {
    String chat(@MemoryId String sessionId, @UserMessage String message);
}

// اسسٹنٹ خود بخود Spring Boot کے ذریعے منسلک ہوتا ہے:
// - ChatModel بین
// - @Component کلاسز سے تمام @Tool طریقے
// - سیشن مینجمنٹ کے لیے ChatMemoryProvider
```
  
ذیل میں تصویر ہر تشریح کی وضاحت کرتی ہے اور دکھاتی ہے کہ AI کو کب ٹول کال کرنا ہے اور کون سے دلائل پاس کرنے ہیں:

<img src="../../../translated_images/ur/tool-definitions-anatomy.f6468546037cf28b.webp" alt="ٹول کی تعریفات کی ساخت" width="800"/>

*ٹول کی تعریف کی ساخت — @Tool AI کو بتاتا ہے کب اسے استعمال کرنا ہے، @P ہر پیرامیٹر کی وضاحت کرتا ہے، اور @AiService سب کچھ اسٹارٹ اپ پر آپس میں جوڑ دیتا ہے۔*

> - "میں جعلی ڈیٹا کی بجائے حقیقی موسم کی API جیسے OpenWeatherMap کو کیسے شامل کروں؟"  
> - "ایک اچھی ٹول تفصیل کیا ہوتی ہے جو AI کو صحیح استعمال میں مدد دیتی ہے؟"  
> - "ٹولز کی عمل آوری میں API غلطیوں اور ریٹ لیمٹس کو کیسے سنبھالوں؟"

### فیصلہ سازی

جب صارف پوچھتا ہے "سیئٹل میں موسم کیسا ہے؟"، تو ماڈل تصادفی طور پر ٹول منتخب نہیں کرتا۔ یہ صارف کے ارادے کا موازنہ ہر دستیاب ٹول کی تفصیل سے کرتا ہے، اہمیت کے لحاظ سے سکور دیتا ہے، اور بہترین میچ منتخب کرتا ہے۔ پھر ایک منظم فنکشن کال بناتا ہے جس میں درست پیرامیٹر شامل ہوتے ہیں — اس معاملے میں `location` کو `"Seattle"` سیٹ کرتا ہے۔

اگر کسی ٹول کا صارف کی درخواست سے میل نہ کھائے، تو ماڈل اپنے علم سے جواب دیتا ہے۔ اگر کئی ٹولز میچ کرتے ہیں، تو سب سے مخصوص ٹول منتخب کرتا ہے۔

<img src="../../../translated_images/ur/decision-making.409cd562e5cecc49.webp" alt="AI فیصلہ کرتی ہے کون سا ٹول استعمال کرنا ہے" width="800"/>

*ماڈل ہر دستیاب ٹول کو صارف کے ارادے کے مقابلے میں جانچتا ہے اور بہترین میچ منتخب کرتا ہے — اسی لیے واضح، مخصوص ٹول کی تفصیلات لکھنا ضروری ہے۔*

### جواب کی تخلیق

ماڈل موسم کا ڈیٹا وصول کرتا ہے اور اسے صارف کے لیے قدرتی زبان میں جواب میں ڈھالتا ہے۔

### آرکیٹیکچر: اسپرنگ بوٹ آٹو وائرنگ

یہ ماڈیول LangChain4j کے اسپرنگ بوٹ انضمام کو استعمال کرتا ہے جس میں ڈی کلیرٹو `@AiService` انٹرفیس ہوتے ہیں۔ اسٹارٹ اپ پر اسپرنگ بوٹ ہر `@Component` کو دریافت کرتا ہے جس میں `@Tool` طریقے ہوتے ہیں، آپ کا `ChatModel` بین، اور `ChatMemoryProvider` — پھر ان سب کو ایک واحد `Assistant` انٹرفیس میں پہلے سے بنائے بغیر جوڑ دیتا ہے۔

<img src="../../../translated_images/ur/spring-boot-wiring.151321795988b04e.webp" alt="اسپرنگ بوٹ آٹو وائرنگ آرکیٹیکچر" width="800"/>

*@AiService انٹرفیس ChatModel، ٹول کمپونینٹس، اور میموری پرووائیڈر کو آپس میں باندھتا ہے — اسپرنگ بوٹ یہ سب آٹو میٹک طریقے سے سنبھالتا ہے۔*

اس طریقے کے کلیدی فوائد:

- **اسپرنگ بوٹ آٹو وائرنگ** — ChatModel اور ٹولز خود بخود انجیکٹ ہوتے ہیں  
- **@MemoryId پیٹرن** — خودکار سیشن بیسڈ میموری مینجمنٹ  
- **سنگل انسٹینس** — اسسٹنٹ ایک بار بنایا جاتا ہے اور بہتر کارکردگی کے لیے دوبارہ استعمال ہوتا ہے  
- **ٹائپ سیف ایکزیکیوشن** — جاوا طریقے براہ راست ٹائپ کنورژن کے ساتھ کال ہوتے ہیں  
- **ملٹی ٹرن آرکیسٹریشن** — ٹول چیننگ کو خودکار ہینڈل کرتا ہے  
- **زیرو بوائلر پلیٹ** — کوئی دستی `AiServices.builder()` کالز یا میموری ہیش میپ نہیں

متبادل طریقے (دستی `AiServices.builder()`) زیادہ کوڈ کی ضرورت رکھتے ہیں اور اسپرنگ بوٹ انضمام کے فوائد نہیں دیتے۔

## ٹول چیننگ

**ٹول چیننگ** — ٹول بیسڈ ایجنٹس کی اصل طاقت اس وقت دکھائی دیتی ہے جب ایک سوال کے لیے متعدد ٹولز کی ضرورت ہو۔ پوچھیں "سیئٹل میں فارن ہائیٹ میں موسم کیسا ہے؟" اور ایجنٹ خود بخود دو ٹولز کو چین کرتا ہے: پہلے `getCurrentWeather` کال کرتا ہے تاکہ سیلسئیس میں درجہ حرارت حاصل کرے، پھر اس ویلیو کو `celsiusToFahrenheit` کو بھیج کر کنورژن کرتا ہے — یہ سب ایک گفتگو کے ٹرن میں۔

<img src="../../../translated_images/ur/tool-chaining-example.538203e73d09dd82.webp" alt="ٹول چیننگ کی مثال" width="800"/>

*ٹول چیننگ عمل میں — ایجنٹ پہلے getCurrentWeather کال کرتا ہے، پھر سیلسئیس کا نتیجہ celsiusToFahrenheit میں بھیجتا ہے، اور مشترکہ جواب دیتا ہے۔*

یہ چلتی ہوئی ایپلیکیشن میں یوں دکھائی دیتا ہے — ایجنٹ ایک ہی بات چیت کے ٹرن میں دو ٹول کالز کو چین کرتا ہے:

<a href="images/tool-chaining.png"><img src="../../../translated_images/ur/tool-chaining.3b25af01967d6f7b.webp" alt="ٹول چیننگ" width="800" style="border: 1px solid #ddd; box-shadow: 0 2px 8px rgba(0,0,0,0.1);"/></a>

*حقیقی ایپلیکیشن کا آؤٹ پٹ — ایجنٹ خود بخود getCurrentWeather → celsiusToFahrenheit کو ایک ٹرن میں چین کرتا ہے۔*

**خوبصورتی سے ناکام ہونا** — ایسے شہر کا موسم پوچھیں جو جعلی ڈیٹا میں نہ ہو۔ ٹول ایک ایرر میسج واپس کرتا ہے، اور AI وضاحت کرتا ہے کہ وہ مدد نہیں کر سکتا، بجائے اس کے کہ ایپ کریش ہو جائے۔ ٹولز محفوظ طریقے سے ناکام ہوتے ہیں۔

<img src="../../../translated_images/ur/error-handling-flow.9a330ffc8ee0475c.webp" alt="غلطیوں کے انتظام کا بہاؤ" width="800"/>

*جب کوئی ٹول ناکام ہوتا ہے، تو ایجنٹ ایرر پکڑتا ہے اور کریش ہونے کے بجائے مددگار وضاحت کے ساتھ جواب دیتا ہے۔*

یہ سب ایک ہی گفتگو کے ٹرن میں ہوتا ہے۔ ایجنٹ خودکار طور پر متعدد ٹول کالز کو سنبھالتا ہے۔

## ایپلیکیشن چلائیں

**تعیناتی کی تصدیق کریں:**

روٹ ڈائریکٹری میں `.env` فائل Azure اسناد کے ساتھ موجود ہو (ماڈیول ۰۱ کے دوران بنائی گئی):  
```bash
cat ../.env  # انہیں دکھانا چاہیے AZURE_OPENAI_ENDPOINT، API_KEY، DEPLOYMENT
```


**ایپلیکیشن شروع کریں:**

> **نوٹ:** اگر آپ نے ماڈیول ۰۱ سے `./start-all.sh` کے ذریعے تمام ایپلیکیشنز شروع کر دی ہیں، تو یہ ماڈیول پہلے ہی پورٹ ۸۰۸۴ پر چل رہا ہے۔ آپ نیچے کے آغاز کے کمانڈز چھوڑ کر سیدھا http://localhost:8084 پر جا سکتے ہیں۔

**اختیار ۱: اسپرنگ بوٹ ڈیش بورڈ کا استعمال (VS Code صارفین کے لیے تجویز کردہ)**

ڈیولپمنٹ کنٹینر میں اسپرنگ بوٹ ڈیش بورڈ توسیع شامل ہے جو تمام اسپرنگ بوٹ ایپلیکیشنز کو منیج کرنے کے لیے بصری انٹرفیس فراہم کرتا ہے۔ یہ آپ کو VS Code کی بائیں طرف ایکٹیوٹی بار میں (اسپرنگ بوٹ آئیکن دیکھیں) مل جائے گا۔

اسپرنگ بوٹ ڈیش بورڈ سے آپ:  
- ورک اسپیس میں تمام دستیاب اسپرنگ بوٹ ایپلیکیشنز دیکھ سکتے ہیں  
- ایک کلک سے ایپلیکیشنز کو شروع/روک سکتے ہیں  
- ایپلیکیشن لاگز کو حقیقی وقت میں دیکھ سکتے ہیں  
- ایپلیکیشن کی صورتحال مانیٹر کر سکتے ہیں

بس "tools" کے پاس پلے بٹن پر کلک کریں تاکہ یہ ماڈیول شروع ہو، یا ایک ساتھ تمام ماڈیولز شروع کریں۔

<img src="../../../translated_images/ur/dashboard.9b519b1a1bc1b30a.webp" alt="اسپرنگ بوٹ ڈیش بورڈ" width="400"/>

**اختیار ۲: شیل اسکرپٹس کا استعمال**

تمام ویب ایپلیکیشنز (ماڈیول ۰۱-۰۴) شروع کریں:

**بش:**  
```bash
cd ..  # روٹ ڈائریکٹری سے
./start-all.sh
```
  
**پاور شیل:**  
```powershell
cd ..  # جڑ ڈائریکٹری سے
.\start-all.ps1
```
  
یا صرف یہ ماڈیول شروع کریں:

**بش:**

**پاور شیل:**  
```powershell
cd 04-tools
.\start.ps1
```
  
یہ دونوں اسکرپٹس خودکار طور پر روٹ `.env` فائل سے ماحول کی متغیرات لوڈ کرتے ہیں اور اگر JAR فائلز موجود نہیں تو بنا دیتے ہیں۔

> **نوٹ:** اگر آپ تمام ماڈیولز کو دستی طور پر بنا کر چلانا چاہتے ہیں:  
>  
> **بش:**  
> ```bash
> cd ..  # Go to root directory
> mvn clean package -DskipTests
> ```
  
> **پاور شیل:**  
> ```powershell
> cd ..  # Go to root directory
> mvn clean package -DskipTests
> ```
  

اپنے براؤزر میں http://localhost:8084 کھولیں۔

**بش:**

./stop.sh  # یہ ماڈیول صرف
# یا
cd .. && ./stop-all.sh  # تمام ماڈیولز

**پاور شیل:**  
```powershell
.\stop.ps1  # صرف یہ ماڈیول

```
  

## ایپلیکیشن کا استعمال

ایپلیکیشن ایک ویب انٹرفیس فراہم کرتی ہے جہاں آپ ایک AI ایجنٹ سے بات کر سکتے ہیں جسے موسم اور درجہ حرارت کنورژن ٹولز تک رسائی حاصل ہے۔

<a href="images/tools-homepage.png"><img src="../../../translated_images/ur/tools-homepage.4b4cd8b2717f9621.webp" alt="AI Agent Tools Interface" width="800" style="border: 1px solid #ddd; box-shadow: 0 2px 8px rgba(0,0,0,0.1);"/></a>

*AI ایجنٹ ٹولز انٹرفیس - ٹولز کے ساتھ بات چیت کے لئے فوری مثالیں اور چیٹ انٹرفیس*

### سادہ ٹول استعمال کریں امتحان کریں
آسان درخواست سے شروع کریں: "100 ڈگری فارن ہائیٹ کو سیلسیس میں تبدیل کریں"۔ ایجنٹ پہچان لیتا ہے کہ اسے درجہ حرارت کی تبدیلی کے ٹول کی ضرورت ہے، مناسب پیرامیٹرز کے ساتھ اسے کال کرتا ہے، اور نتیجہ واپس کرتا ہے۔ دھیان دیں کہ یہ کتنا قدرتی محسوس ہوتا ہے - آپ نے یہ مخصوص نہیں کیا کہ کون سا ٹول استعمال کرنا ہے یا اسے کیسے کال کرنا ہے۔

### ٹول چیننگ کا تجربہ کریں

اب کچھ زیادہ پیچیدہ کوشش کریں: "سیئیٹل کا موسم کیا ہے اور اسے فارن ہائیٹ میں تبدیل کریں؟" دیکھیں کہ ایجنٹ اسے مرحلہ وار کیسے انجام دیتا ہے۔ پہلے موسم معلوم کرتا ہے (جو سیلسیس میں واپس آتا ہے)، پہچانتا ہے کہ اسے فارن ہائیٹ میں تبدیل کرنا ہے، تبدیلی کے ٹول کو کال کرتا ہے، اور دونوں نتائج کو ایک جواب میں ملاتا ہے۔

### گفتگو کے بہاؤ کو دیکھیں

چیٹ انٹرفیس گفتگو کی تاریخ کو برقرار رکھتا ہے، جس سے آپ کثیر مرحلہ بات چیت کر سکتے ہیں۔ آپ تمام پچھلے سوالات اور جوابات دیکھ سکتے ہیں، جس سے گفتگو کو ٹریک کرنا اور سمجھنا آسان ہو جاتا ہے کہ ایجنٹ کس طرح متعدد تبادلے میں سیاق و سباق بناتا ہے۔

<a href="images/tools-conversation-demo.png"><img src="../../../translated_images/ur/tools-conversation-demo.89f2ce9676080f59.webp" alt="کثیر ٹول کالز کے ساتھ گفتگو" width="800" style="border: 1px solid #ddd; box-shadow: 0 2px 8px rgba(0,0,0,0.1);"/></a>

*کثیر مرحلہ گفتگو میں آسان تبدیلیاں، موسم کی تلاشیں، اور ٹول چیننگ دکھائی گئی ہے*

### مختلف درخواستوں کے ساتھ تجربہ کریں

مختلف امتزاج آزما کر دیکھیں:
- موسم کی تلاشیں: "ٹوکیو کا موسم کیسا ہے؟"
- درجہ حرارت کی تبدیلیاں: "25°C کیا کیلون میں ہے؟"
- مشترکہ سوالات: "پیرس کا موسم چیک کریں اور بتائیں کہ کیا یہ 20°C سے زیادہ ہے"

دھیان دیں کہ ایجنٹ کس طرح قدرتی زبان کی تعبیر کرتا ہے اور مناسب ٹول کالز کے ساتھ میپ کرتا ہے۔

## اہم تصورات

### ReAct پیٹرن (منطق اور عمل)

ایجنٹ سوچنے (یہ فیصلہ کرنا کہ کیا کرنا ہے) اور عمل کرنے (ٹولز استعمال کرنا) کے درمیان باری باری چلتا ہے۔ یہ پیٹرن خودمختار مسئلہ حل کرنے کی اجازت دیتا ہے بجائے اس کے کہ صرف ہدایت پر جواب دے۔

### ٹول کی وضاحتیں اہم ہیں

آپ کے ٹول وضاحتی معیار کا براہ راست اثر ہوتا ہے کہ ایجنٹ انہیں کتنی اچھائی سے استعمال کرتا ہے۔ واضح، مخصوص وضاحتیں ماڈل کو سمجھنے میں مدد دیتی ہیں کہ کب اور کیسے ہر ٹول کو کال کرنا ہے۔

### سیشن مینجمنٹ

`@MemoryId` تشریح خودکار سیشن بیسڈ میموری مینجمنٹ کو فعال کرتی ہے۔ ہر سیشن آئی ڈی کو اس کا اپنا `ChatMemory` انسٹینس ملتا ہے جو `ChatMemoryProvider` بین کے ذریعہ منظم ہوتا ہے، اس طرح متعدد صارفین بیک وقت ایجنٹ سے بات چیت کر سکتے ہیں بغیر ان کی گفتگو ایک دوسرے میں مکس ہوئے۔

<img src="../../../translated_images/ur/session-management.91ad819c6c89c400.webp" alt="@MemoryId کے ساتھ سیشن مینجمنٹ" width="800"/>

*ہر سیشن آئی ڈی کو الگ تھلگ گفتگو کی تاریخ سے میپ کیا جاتا ہے — صارفین کبھی ایک دوسرے کے پیغامات نہیں دیکھتے۔*

### ایرر ہینڈلنگ

ٹولز ناکام ہو سکتے ہیں — APIs کا ٹائم آؤٹ ہو سکتا ہے، پیرامیٹرز غلط ہو سکتے ہیں، بیرونی خدمات بند ہو سکتی ہیں۔ پروڈکشن ایجنٹس کو ایرر ہینڈلنگ کی ضرورت ہوتی ہے تاکہ ماڈل مسائل کی وضاحت کر سکے یا متبادل کوششیں کر سکے بجائے اس کے کہ پوری ایپلیکیشن کریش ہو جائے۔ جب کوئی ٹول ایک استثناء پھینکتا ہے، LangChain4j اسے پکڑتا ہے اور ایرر میسج کو ماڈل کو واپس دیتا ہے، جو پھر مسئلہ کو قدرتی زبان میں بیان کر سکتا ہے۔

## دستیاب ٹولز

نیچے دیا گیا خاکہ آپ کو بتاتا ہے کہ آپ کتنے وسیع پیمانے پر ٹولز بنا سکتے ہیں۔ یہ ماڈیول موسم اور درجہ حرارت کے ٹولز دکھاتا ہے، لیکن وہی `@Tool` پیٹرن کسی بھی جاوا طریقے پر کام کرتا ہے — ڈیٹا بیس کی تلاشوں سے لے کر ادائیگی کے عمل تک۔

<img src="../../../translated_images/ur/tool-ecosystem.aad3d74eaa14a44f.webp" alt="ٹول ماحولیاتی نظام" width="800"/>

*کوئی بھی جاوا طریقہ جس پر @Tool لگایا گیا ہو AI کے لیے دستیاب ہو جاتا ہے — پیٹرن ڈیٹا بیس، APIs، ای میل، فائل آپریشنز، اور مزید تک پھیلتا ہے۔*

## ٹول بیسڈ ایجنٹس کب استعمال کریں

<img src="../../../translated_images/ur/when-to-use-tools.51d1592d9cbdae9c.webp" alt="ٹولز کب استعمال کریں" width="800"/>

*ایک فوری فیصلہ گائیڈ — ٹولز حقیقی وقت کے ڈیٹا، حسابات، اور اعمال کے لیے ہیں؛ عمومی علم اور تخلیقی کاموں کو ان کی ضرورت نہیں۔*

**ٹولز استعمال کریں جب:**
- جواب دینے کے لیے حقیقی وقت کا ڈیٹا چاہیے (موسم، اسٹاک کی قیمتیں، انوینٹری)
- آپ کو سادہ ریاضی سے ماوراے حسابات کی ضرورت ہو
- ڈیٹا بیس یا APIs تک رسائی
- اعمال کرنا (ای میل بھیجنا، ٹکٹ بنانا، ریکارڈز اپڈیٹ کرنا)
- متعدد ڈیٹا ذرائع کو یکجا کرنا

**ٹولز استعمال نہ کریں جب:**
- سوالات عمومی علم سے جواب دیے جا سکتے ہوں
- جواب صرف بات چیت جیسا ہو
- ٹول کی تاخیر تجربہ بہت سست بنا دے

## ٹولز بمقابلہ RAG

ماڈیولز 03 اور 04 دونوں AI کی صلاحیتیں بڑھاتے ہیں، لیکن بنیادی طور پر مختلف طریقوں سے۔ RAG ماڈل کو **علم** تک رسائی دیتا ہے دستاویزات بازیافت کر کے۔ ٹولز ماڈل کو **عمل کرنے** کی صلاحیت دیتے ہیں فنکشن کال کر کے۔

<img src="../../../translated_images/ur/tools-vs-rag.ad55ce10d7e4da87.webp" alt="ٹولز بمقابلہ RAG موازنہ" width="800"/>

*RAG جامد دستاویزات سے معلومات بازیافت کرتا ہے — ٹولز اعمال انجام دیتے ہیں اور متحرک، حقیقی وقت کا ڈیٹا حاصل کرتے ہیں۔ بہت سے پروڈکشن سسٹمز دونوں کو ملاتے ہیں۔*

عملی طور پر، بہت سے پروڈکشن سسٹمز دونوں کو ملاتے ہیں: RAG جوابات کو آپ کی دستاویزات میں بنیاد فراہم کرنے کے لیے، اور ٹولز زندہ ڈیٹا حاصل کرنے یا عملیات کرنے کے لیے۔
