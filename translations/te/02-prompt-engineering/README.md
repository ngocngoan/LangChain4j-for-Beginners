## మీరు నేర్చుకోగల విషయాలు

<img src="../../../translated_images/te/what-youll-learn.c68269ac048503b2.webp" alt="What You'll Learn" width="800"/>

మునుపటి మాడ్యూల్లో, మీరు మెమోరీ ఎలా సంభాషణాత్మక AI కి సహాయపడుతుందో చూశారు మరియు మౌలిక పరస్పర చర్యలకు GitHub మోడల్స్ ఉపయోగించారు. ఇప్పుడు మనం మీరు అడిగే ప్రశ్నలకు — ప్రాంప్ట్స్ స్వయంగా — Azure OpenAI యొక్క GPT-5.2 ఉపయోగించి ఎలా అడగాలో దృష్టిపెడతాము. మీరు మీ ప్రాంప్ట్స్ ను ఎలా నిర్మిస్తారో మీకు వస్తున్న ప్రతిస్పందనల నాణ్యతను గణనీయంగా ప్రభావితం చేస్తుంది. మొదట ప్రాథమిక ప్రాంప్టింగ్ సాంకేతికతలను సమీక్షించి, తరువాత GPT-5.2 యొక్క పూర్తి సామర్థ్యాలను ఉపయోగించే ఎనిమిది ఆధునిక నమూనాలకు వెళ్తాము.

GPT-5.2 ను ఉపయోగిస్తాము ఎందుకంటే ఇది తర్క నియంత్రణను పరిచయం చేస్తుంది - మీరు మోడల్ ఎంత ఆలోచన చేయాలో చెప్పవచ్చు. ఇది విభిన్న ప్రాంప్టింగ్ వ్యూహాలను మరింత స్పష్టంగా చేస్తుంది మరియు ఎప్పుడు ఏ విధానాన్ని ఉపయోగించాలో అర్థం చేసుకోవడంలో సహాయపడుతుంది. GitHub మోడల్స్ తో పోల్చుకుంటే Azure లో GPT-5.2 కి తక్కువ రేట్ పరిమితులు కలిగిన ఉపయోగం కూడా ఉంటుంది.

## ప్రాంప్ట్ ఇంజినీరింగ్ అవగాహన

<img src="../../../translated_images/te/what-is-prompt-engineering.5c392a228a1f5823.webp" alt="What is Prompt Engineering?" width="800"/>

ప్రాంప్ట్ ఇంజినీరింగ్ అంటే మీరు కావలసిన ఫలితాలను నిరంతరం అందించే ఇన్పుట్ టెక్స్ట్ డిజైన్ చేయడం. ఇది కేవలం ప్రశ్నలు అడగడం కాదు — మీరు చేయలిచే అభ్యర్థనలను మోడల్ బాగా అర్థం చేసుకునేటట్లు నిర్మించడం.

దీనిని సహచరుడికి సూచనలు ఇవ్వడంలా పరిగణించండి. "బగ్ ఫిక్స్ చేయి" అనేది అస్పష్టమైంది. "UserService.java లైన్ 45 లో నల్ పాయింటర్ ఎక్సెప్షన్ కి నల్ చెక్ జోడించి సరిచేయి" అనేది స్పష్టమే. భాషా మోడల్స్ కూడా ఇదే విధంగా పనిచేస్తాయి — స్పష్టత మరియు నిర్మాణం ముఖ్యం.

<img src="../../../translated_images/te/how-langchain4j-fits.dfff4b0aa5f7812d.webp" alt="How LangChain4j Fits" width="800"/>

LangChain4j మౌలిక సదుపాయాలు — మోడల్ కనెక్షన్లు, మెమోరీ, సందేశ రకాలు — అందిస్తుంది, అలాగే ప్రాంప్ట్ నమూనాలు ఆ మౌలిక సదుపాయాలలో జాగ్రత్తగా నిర్మించబడిన టెక్స్ట్ మాత్రమె. కీలక నిర్మాణ భాగాలు `SystemMessage` (AI ప్రవర్తన మరియు పాత్రను సెట్ చేస్తుంది) మరియు `UserMessage` (మీ నిజమైన అభ్యర్థనతో ఉంటుంది).

## ప్రాంప్ట్ ఇంజినీరింగ్ మూలాలు

<img src="../../../translated_images/te/five-patterns-overview.160f35045ffd2a94.webp" alt="Five Prompt Engineering Patterns Overview" width="800"/>

ఈ మాడ్యూల్లోని ఆధునిక నమూనాలకు ముందుగా, ఐదు మూల ప్రాంప్టింగ్ సాంకేతికతలను సమీక్షిద్దాం. అవి ప్రతి ప్రాంప్ట్ ఇంజినీరింగ్ అందరికీ తెలిసి ఉండాల్సిన ప్రాథమిక నిర్మాణ భాగాలు. మీరు ఇప్పటికే [Quick Start module](../00-quick-start/README.md#2-prompt-patterns) లో ఈయి చూసి ఉంటే — వీటి ఆలోచనాత్మక రూపరం ఇక్కడ ఉంది.

### జీరో-షాట్ ప్రాంప్టింగ్

సడలిన విధానం: మోడల్ కు ఎలాంటి డెమో లేకుండా నేరుగా ఆదేశం ఇవ్వడం. మోడల్ తన శిక్షణపై ఆధారపడి పనితీరు అర్థం చేసుకుని అమలు చేస్తుంది. స్పష్టమైన అభ్యర్థనలకు ఇది బాగుంటుంది.

<img src="../../../translated_images/te/zero-shot-prompting.7abc24228be84e6c.webp" alt="Zero-Shot Prompting" width="800"/>

*ఎలాంటి ఉదాహరణలు లేకుండా నేరుగా ఆదేశం — మోడల్ ఆదేశమే task అనుకుంటుంది*

```java
String prompt = "Classify this sentiment: 'I absolutely loved the movie!'";
String response = model.chat(prompt);
// ప్రతిస్పందన: "ధనాత్మకము"
```

**ఎప్పుడు ఉపయోగించాలి:** సులభ వర్గీకరణలు, నేరుగా ప్రశ్నలు, అనువాదాలు, లేదా అదనపు మార్గదర్శకం అవసరంలేని ఏ పని.

### ఫ్యూ-షాట్ ప్రాంప్టింగ్

మోడల్ అనుసరించాల్సిన నమూనా చూపించే ఉదాహరణలు ఇవ్వడం. ఉదాహరణల నుంచి ఇన్పుట్-అవుట్పుట్ ఫార్మాట్ నేర్చుకుని కొత్త ఇన్పుట్ కు వర్తింపజేస్తుంది. అవాంఛిత ఫార్మాట్ లేదా ప్రవర్తన పొరపాట్లలో మెరుగైన సుస్పష్టత ఇస్తుంది.

<img src="../../../translated_images/te/few-shot-prompting.9d9eace1da88989a.webp" alt="Few-Shot Prompting" width="800"/>

*ఉదాహరణల నుంచి నేర్చుకోడం — మోడల్ నమూనాను గుర్తించి కొత్త ఇన్పుట్‌కి వర్తింపచేస్తుంది*

```java
String prompt = """
    Classify the sentiment as positive, negative, or neutral.
    
    Examples:
    Text: "This product exceeded my expectations!" → Positive
    Text: "It's okay, nothing special." → Neutral
    Text: "Waste of money, very disappointed." → Negative
    
    Now classify this:
    Text: "Best purchase I've made all year!"
    """;
String response = model.chat(prompt);
```

**ఎప్పుడు ఉపయోగించాలి:** కస్టమ్ వర్గీకరణలు, సుస్పష్ట ఫార్మాటింగ్, డొమైన్-స్పెసిఫిక్ పనులు, లేదా జీరో-షాట్ అనితరస్పందన.

### చెయిన్ ఆఫ్ థాట్

మోడల్ తన తర్కాన్ని దశలవారీగా చూపించమని అడగడం. ప్రత్యక్ష సమాధానం ఇచ్చేందుకు బదులుగా సమస్యను భాగాలవారీగా పగుళ్ళతో పరిష్కరిస్తుంది. ఇది గణితం, తర్కం, బహుళ-దశ తర్కంలో యథార్థతను పెంచుతుంది.

<img src="../../../translated_images/te/chain-of-thought.5cff6630e2657e2a.webp" alt="Chain of Thought Prompting" width="800"/>

*దశలవారీ తర్కం — క్లిష్ట సమస్యలను స్పష్టమైన లాజికల్ దశలుగా వేరుచేసడం*

```java
String prompt = """
    Problem: A store has 15 apples. They sell 8 apples and then 
    receive a shipment of 12 more apples. How many apples do they have now?
    
    Let's solve this step-by-step:
    """;
String response = model.chat(prompt);
// మోడల్ చూపిస్తుంది: 15 - 8 = 7, తర్వాత 7 + 12 = 19 సేపుళ్లు
```

**ఎప్పుడు ఉపయోగించాలి:** గణితం సమస్యలు, తర్క పజిల్స్, డీబగ్గింగ్, లేదా తర్క ప్రక్రియ చూపడం వల్ల నమ్మకం పెరగడం అనుకున్నప్పడు.

### రోల్-ఆధారిత ప్రాంప్టింగ్

ప్రశ్న అడగడానికి ముందు AI కు పాత్ర లేదా వ్యక్తిత్వం సెట్ చేయడం. ఇది స్పందన స్వరము, లోతు, ఫోకస్ ను ప్రభావితం చేస్తుంది. "సాఫ్ట్‌వేర్ ఆర్కిటెక్ట్" గలది ఒక "జూనియర్ డెవలపర్" గలదానికంటే వేరే సలహా ఇస్తుంది.

<img src="../../../translated_images/te/role-based-prompting.a806e1a73de6e3a4.webp" alt="Role-Based Prompting" width="800"/>

*సందర్భం మరియు పాత్ర సెట్ చేయడం — అదే ప్రశ్నకు విలక్షణ ప్రతిస్పందన వస్తుంది పాత్ర ఆధారంగా*

```java
String prompt = """
    You are an experienced software architect reviewing code.
    Provide a brief code review for this function:
    
    def calculate_total(items):
        total = 0
        for item in items:
            total = total + item['price']
        return total
    """;
String response = model.chat(prompt);
```

**ఎప్పుడు ఉపయోగించాలి:** కోడ్ సమీక్షలు, ట్యూయిట‌రింగ్, డొమైన్-నిర్దిష్ట విశ్లేషణ, లేదా ప్రత్యేక నైపుణ్య స్థాయికి అనుగుణమైన తిప్పులు కావలసినప్పుడు.

### ప్రాంప్ట్ టెంప్లేట్స్

వేరియబుల్ స్థానాధారాలతోకి పునర్వినియోగయోగ్య ప్రాంప్ట్స్ సృష్టించడం. ప్రతి సారీ కొత్త ప్రాంప్ట్ రాయడానికి బదులు ఒక సారి టెంప్లేట్ను నిర్వచించి, వేర్వేరు విలువలతో దాన్ని భర్తీ చేయండి. LangChain4j యొక్క `PromptTemplate` తరగతి ఈ పనిని `{{variable}}` సింటాక్స్ తో సులభతరం చేస్తుంది.

<img src="../../../translated_images/te/prompt-templates.14bfc37d45f1a933.webp" alt="Prompt Templates" width="800"/>

*వేరియబుల్స్ ఉన్న పునర్వినియోగ ప్రాంప్ట్స్ — ఒక టెంప్లేట్, అనేక ఉపయోగాలు*

```java
PromptTemplate template = PromptTemplate.from(
    "What's the best time to visit {{destination}} for {{activity}}?"
);

Prompt prompt = template.apply(Map.of(
    "destination", "Paris",
    "activity", "sightseeing"
));

String response = model.chat(prompt.text());
```

**ఎప్పుడు ఉపయోగించాలి:** వేర్వేరు ఇన్పుట్లతో పునరావృత ప్రశ్నలు, బ్యాచ్ ప్రాసెసింగ్, పునర్వినియోగ AI వర్క్‌ఫ్లోలు, లేదా ప్రాంప్ట్ నిర్మాణం ఒకేగా ఉండి డేటానే మారే సందర్భాలు.

---

ఈ ఐదు మూలాలు ఎక్కువ శాతం ప్రాంప్ట్ అవసరాలకు బలమైన సాధనం ఇస్తాయి. ఈ మాడ్యూల్ మిగిలిన భాగం GPT-5.2 యొక్క తర్క నియంత్రణ, స్వీయ-మూల్యాంకన, మరియు నిర్మిత అవుట్పుట్ సామర్థ్యాలను ఉపయోగించే **ఎనిమిది అధునాతన నమూనాలతో** కొనసాగుతుంది.

## అత్యాధునిక నమూనాలు

మూలాలు కవర్ చేసిన తర్వాత ఈ మాడ్యూల్ ప్రత్యేకత చేసే ఎనిమిది అధునాతన నమూనాలకి బయలుదేరుదాం. అన్ని సమస్యలకు ఒకే విధానం అవసరం కాదు. కొన్ని ప్రశ్నలకు వేగవంతమైన సమాధానాలు కావాలి, మరికొన్నింటికి లోతైన ఆలోచన అవసరం. కొన్నింటికి కనిపించే తర్కం అవసరం, మరికొన్నింటికి కేవలం ఫలితాలు కావాలి. కింది ప్రతి నమూనా వేరే వేరే సందర్భానికి సరిపోతుంది — మరియు GPT-5.2 యొక్క తర్క నియంత్రణ తేడాలను మరింత స్పష్టంగా చేస్తుంది.

<img src="../../../translated_images/te/eight-patterns.fa1ebfdf16f71e9a.webp" alt="Eight Prompting Patterns" width="800"/>

*ఎనిమిది ప్రాంప్ట్ ఇంజినీరింగ్ నమూనాల అవలోకనం మరియు వాటి వాడకాలు*

<img src="../../../translated_images/te/reasoning-control.5cf85f0fc1d0c1f3.webp" alt="Reasoning Control with GPT-5.2" width="800"/>

*GPT-5.2 తర్క నియంత్రణ మోడల్ ఎంత ఆలోచించాలో మీరు నిర్ణయించవచ్చు — వేగవంతమైన ప్రత్యక్ష సమాధానాల నుండి లోతైన అన్వేషణ వరకు*

**తక్కువ ఆసక్తి (త్వరకు & లక్ష్యమై)** - సులభ ప్రశ్నల కోసం వేగవంతమైన ప్రత్యక్ష సమాధానాలు కావలసినప్పుడు. మోడల్ తర్కం చాలా తక్కువగా చేస్తుంది - గరిష్టం 2 దశలు. గణనలు, లుక్‌అప్పులు, లేదా సూటిగా ప్రశ్నలకు దీనిని ఉపయోగించండి.

```java
String prompt = """
    <context_gathering>
    - Search depth: very low
    - Bias strongly towards providing a correct answer as quickly as possible
    - Usually, this means an absolute maximum of 2 reasoning steps
    - If you think you need more time, state what you know and what's uncertain
    </context_gathering>
    
    Problem: What is 15% of 200?
    
    Provide your answer:
    """;

String response = chatModel.chat(prompt);
```

> 💡 **GitHub Copilot తో అన్వేషించండి:** [`Gpt5PromptService.java`](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java) తెరవండి మరియు అడగండి:
> - "తక్కువ ఆసక్తి మరియు ఎక్కువ ఆసక్తి ప్రాంప్టింగ్ నమూనాల మధ్య తేడా ఏమిటి?"
> - "ప్రాంప్ట్స్ లో XML ట్యాగులు AI ప్రతిస్పందనను ఎలా నిర్మాణం చేస్తాయి?"
> - "స్వీయ-పరిశీలన నమూనాలు vs నేరినిర్దేశన ఎప్పుడు ఉపయోగించాలి?"

**ఎక్కువ ఆసక్తి (లోతైన & సమగ్ర)** - క్లిష్ట సమస్యలకు సమగ్ర విశ్లేషణ కావలసినప్పుడు. మోడల్ లోతుగా అన్వేషించి, వివరమైన తర్కాన్ని చూపిస్తుంది. సిస్టం డిజైన్, ఆర్కిటెక్చర్ నిర్ణయాలు, లేదా క్లిష్ట పరిశోధనకు ఇది ఉపయోగపడుతుంది.

```java
String prompt = """
    Analyze this problem thoroughly and provide a comprehensive solution.
    Consider multiple approaches, trade-offs, and important details.
    Show your analysis and reasoning in your response.
    
    Problem: Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```

**టాస్క్ ఎగ్జిక్యూషన్ (దశలవారీ పురోగతి)** - బహుళ దశ వర్క్‌ఫ్లోలకు. మోడల్ ముందుగా ప్రణాళికను ఇస్తుంది, దశలవారీగా పని వివరించి, చివరిగా సారాంశం ఇస్తుంది. మైగ్రేషన్లు, అమలు కార్యక్రమాలు, లేదా ఏదైనా బహుళ దశ ప్రక్రియలకు ఇది సరిపోతుంది.

```java
String prompt = """
    <task_execution>
    1. First, briefly restate the user's goal in a friendly way
    
    2. Create a step-by-step plan:
       - List all steps needed
       - Identify potential challenges
       - Outline success criteria
    
    3. Execute each step:
       - Narrate what you're doing
       - Show progress clearly
       - Handle any issues that arise
    
    4. Summarize:
       - What was completed
       - Any important notes
       - Next steps if applicable
    </task_execution>
    
    <tool_preambles>
    - Always begin by rephrasing the user's goal clearly
    - Outline your plan before executing
    - Narrate each step as you go
    - Finish with a distinct summary
    </tool_preambles>
    
    Task: Create a REST endpoint for user registration
    
    Begin execution:
    """;

String response = chatModel.chat(prompt);
```

చెయిన్ ఆఫ్ థాట్ ప్రాంప్టింగ్ మోడల్ కి దశలవారీగా తర్కం చూపమని స్పష్టంగా అడగడం. ఇది క్లిష్ట పనులకు ఖచ్చితత్వం పెంచుతుంది. దశల వారీ విభజన మానవులు మరియు AI తర్కం సులభంగా అర్థం చేసుకునేలా చేస్తుంది.

> **🤖 [GitHub Copilot](https://github.com/features/copilot) చాట్ తో ప్రయత్నించండి:** ఈ నమూనా గురించి అడగండి:
> - "విస్మృత నిర్వహణాల కోసం టాస్క్ ఎగ్జిక్యూషన్ నమూనా ఎలా అనుకూలీకరించాలి?"
> - "ప్రొడక్షన్ అప్లికేషన్లలో టూల్ ప్రీంబుల్స్ నిర్మాణానికి ఉత్తమ పద్ధతులు ఏమిటి?"
> - "UI లో మధ్యవర్తి పురోగతి అప్‌డేట్లను ఎలా పట్టుకోగలము మరియు చూపించగలము?"

<img src="../../../translated_images/te/task-execution-pattern.9da3967750ab5c1e.webp" alt="Task Execution Pattern" width="800"/>

*బహుళ దశ పనులకు ప్రణాళిక → అమలు → సారాంశం వర్క్‌ఫ్లో*

**స్వీయ-పరిశీలన కోడ్** - ప్రొడక్షన్-నాణ్యత కోడ్ సృష్టించడానికి. మోడల్ ప్రొడక్షన్ ప్రమాణాలను అనుసరించి తప్పిద నిర్వహణతో కూడిన కోడ్ ఉత్పత్తి చేస్తుంది. కొత్త ఫీచర్లు లేదా సర్వీసులు నిర్మించే సమయాల్లో ఉపయోగించండి.

```java
String prompt = """
    Generate Java code with production-quality standards: Create an email validation service
    Keep it simple and include basic error handling.
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/te/self-reflection-cycle.6f71101ca0bd28cc.webp" alt="Self-Reflection Cycle" width="800"/>

*పునరావృత అభివృద్ధి చక్రం - సృష్టి, మూల్యాంకనం, సమస్యల గుర్తింపు, అభివృద్ధి, పునరావృతం*

**స్ట్రక్చర్డ్ విశ్లేషణ** - సుస్పష్ట మూల్యాంకన కొరకు. మోడల్ కోడ్ ను స్థిరమైన ఫ్రేమ్‌వర్క్ (ఖచ్చితత్వం, ఆచారాలు, పనితీరు, భద్రత, నిర్వహించదగినది) ఉపయోగించి సమీక్షిస్తుంది. కోడ్ సమీక్షల లేదా నాణ్యత అంచనాలకు ఉపయోగించండి.

```java
String prompt = """
    <analysis_framework>
    You are an expert code reviewer. Analyze the code for:
    
    1. Correctness
       - Does it work as intended?
       - Are there logical errors?
    
    2. Best Practices
       - Follows language conventions?
       - Appropriate design patterns?
    
    3. Performance
       - Any inefficiencies?
       - Scalability concerns?
    
    4. Security
       - Potential vulnerabilities?
       - Input validation?
    
    5. Maintainability
       - Code clarity?
       - Documentation?
    
    <output_format>
    Provide your analysis in this structure:
    - Summary: One-sentence overall assessment
    - Strengths: 2-3 positive points
    - Issues: List any problems found with severity (High/Medium/Low)
    - Recommendations: Specific improvements
    </output_format>
    </analysis_framework>
    
    Code to analyze:
    ```
    public List getUsers() {
        return database.query("SELECT * FROM users");
    }
    ```
    Provide your structured analysis:
    """;

String response = chatModel.chat(prompt);
```

> **🤖 [GitHub Copilot](https://github.com/features/copilot) చాట్ తో ప్రయత్నించండి:** స్ట్రక్చర్డ్ విశ్లేషణ గురించి అడగండి:
> - "వివిధ రకాల కోడ్ సమీక్షలకు విశ్లేషణ ఫ్రేమ్‌వర్క్‌ను ఎలా అనుకూలీకరించవచ్చు?"
> - "స్ట్రక్చర్డ్ అవుట్పుట్‌ను ప్రోగ్రామాటిక్గా పార్స్ చేసి కార్యకలాపాలు చేయడం ఉత్తమ పద్ధతి?"
> - "వివిధ సమీక్ష సెషన్లలో స్థిరతా తీవ్రతా స్థాయులను ఎలా నిర్ధారించాలి?"

<img src="../../../translated_images/te/structured-analysis-pattern.0af3b690b60cf2d6.webp" alt="Structured Analysis Pattern" width="800"/>

*స్థిరమైన కోడ్ సమీక్షలకు తీవ్రతా స్థాయి ఆధారిత ఫ్రేమ్‌వర్క్*

**మల్టీ-టర్న్ చాట్** - సందర్భం అవసరమైన సంభాషణలకు. మోడల్ గత సందేశాలను గుర్తుంచుకుంటూ వాటిని ఆధారంగా కాంప్లీిషన్లను తయారు చేస్తుంది. ఇంటరాక్టివ్ సహాయం సెషన్లు లేదా క్లిష్ట Q&Aకి ఉపయోగపడుతుంది.

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("What is Spring Boot?"));
AiMessage aiMessage1 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage1);

memory.add(UserMessage.from("Show me an example"));
AiMessage aiMessage2 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage2);
```

<img src="../../../translated_images/te/context-memory.dff30ad9fa78832a.webp" alt="Context Memory" width="800"/>

*చర్చా సందర్భం అనేక టర్న్‌లలో జమ కావడం టోకెన్ పరిమితి వరకు*

**విధినిబద్ధమైన తర్కం** - ప్రత్యక్షమైన లాజిక్కు అవసరమైన సమస్యలకు. మోడల్ ప్రతీ దశకు స్పష్టమైన తర్కాన్ని చూపిస్తుంది. గణిత, తర్క పజిల్స్ లేదా ఆలోచనా ప్రక్రియా అర్థం చేసుకోవాలనుకునే సందర్భాల్లో ఉపయోగించండి.

```java
String prompt = """
    <instruction>Show your reasoning step-by-step</instruction>
    
    If a train travels 120 km in 2 hours, then stops for 30 minutes,
    then travels another 90 km in 1.5 hours, what is the average speed
    for the entire journey including the stop?
    """;

String response = chatModel.chat(prompt);
```

*సమస్యలను స్పష్టమైన లాజికల్ దశలుగా విభజించటం*

**పరిమిత అవుట్పుట్** - నిర్దిష్ట ఫార్మాట్ అవసరాలు ఉన్న స్పందనలకు. మోడల్ ఫార్మాట్, పొడవు నియమాలను కఠినంగా అనుసరిస్తుంది. సారాంశాలు లేకపోతే ఖచ్చితమైన అవుట్పుట్ నిర్మాణం కావాలసిన సందర్భాల్లో ఉపయోగించండి.

```java
String prompt = """
    <constraints>
    - Exactly 100 words
    - Bullet point format
    - Technical terms only
    </constraints>
    
    Summarize the key concepts of machine learning.
    """;

String response = chatModel.chat(prompt);
```

అజూర్ క్రెడెన్షియల్స్ తో రూట్ డైరెక్టరీలో `.env` ఫైల్ ఉందని ధృవీకరించండి (మాడ్యూల్ 01 సమయంలో సృష్టించబడింది):
```bash
cat ../.env  # AZURE_OPENAI_ENDPOINT, API_KEY, DEPLOYMENT చూపించాలి
```

**అప్లికేషన్ ప్రారంభించండి:**

> **గమనిక:** మీరు మాడ్యూల్ 01 నుండి `./start-all.sh` ద్వారా ఇప్పటికే అన్ని అప్లికేషన్లను ప్రారంభించినట్లయితే, ఈ మాడ్యూల్ ఇప్పటికే పోర్ట్ 8083 లో నడుస్తోంది. కాబట్టి క్రింద ఇచ్చిన ప్రారంభ ఆదేశాలను తప్పవేయండి మరియు నేరుగా http://localhost:8083 కి వెళ్లండి.

**ఆప్షన్ 1: స్ప్రింగ్ బూట్ డాష్‌బోర్డ్ ఉపయోగించడం (VS కోడ్ వినియోగదారులకు సిఫారసు)**
డెవ్ కంటైనర్‌లో స్ప్రింగ్ బూట్ డాష్‌బోర్డ్ ఎక్స్‌టెన్షన్ ఉంది, ఇది అన్ని స్ప్రింగ్ బూట్ అప్లికేషన్లు నిర్వహించడానికి ఒక విజువల్ ఇంటర్‌ఫేస్‌ను అందిస్తుంది. మీరు దీన్ని VS కోడ్‌లో ఎడమవైపు యాక్టివిటీ బార్‌లో (స్ప్రింగ్ బూట్ ఐకాన్ కోసం చూడండి) పొందవచ్చు.

స్ప్రింగ్ బూట్ డాష్‌బోర్డ్ నుండి మీరు:
- వర్క్‌స్పేస్‌లో అందుబాటులో ఉన్న అన్ని స్ప్రింగ్ బూట్ అప్లికేషన్లను చూడవచ్చు
- సింగిల్ క్లిక్‌తో అప్లికేషన్లను ప్రారంభించవచ్చు/ఆపవచ్చు
- అప్లికేషన్ లాగులు రియల్-టైమ్‌లో చూడవచ్చు
- అప్లికేషన్ స్థితిని పరిక్షించవచ్చు

"prompt-engineering" పక్కన ఉన్న ప్లే బటన్‌పై సింప్లీ క్లిక్ చేయడం ద్వారా ఈ మాడ్యూల్‌ను ప్రారంభించండి, లేదా అన్ని మాడ్యూల్స్‌ను ఒకేసారి ప్రారంభించండి.

## నమూనాల అన్వేషణ

వెబ్ ఇంటర్‌ఫేస్ మీకు వివిధ ప్రాంప్టింగ్ వ్యూహాలతో ప్రయోగం చేసుకోవడానికి అనుమతిస్తుంది. ప్రతి నమూనా వేరే సమస్యలను పరిష్కరిస్తుంది - ప్రతి విధానం ఎప్పుడు మెరుగ్గా పనిచేస్తేది అనేది చూడటానికి ప్రయత్నించండి.

> **గమనిక: స్ట్రీమింగ్ వర్సెస్ నాన్-స్ట్రీమింగ్** — ప్రతి నమూనా పేజీలో రెండు బటన్లు ఉంటాయి: **🔴 స్ట్రీమ్ రెస్పాన్స్ (లైవ్)** మరియు ఒక **నాన్-స్ట్రీమింగ్** ఎంపిక. స్ట్రీమింగ్ మోడల్ రూపొందించిన ప్రతీ టోకెన్‌ను రియల్-టైమ్‌లో ప్రదర్శించడానికి సర్వర్-సెంట్ ఈవెంట్స్ (SSE) ను ఉపయోగిస్తుంది, కాబట్టి మీరు తక్షణమే పురోగతిని చూడవచ్చు. నాన్-స్ట్రీమింగ్ ఎంపిక మొత్తం ప్రతిస్పందన ముగిసేవరకు వేచి ఉంటుంది. లోతైన తర్కం అవసరం కలిగించే ప్రాంప్ట్‌లకు (ఉదాహరణకు, హై ఈగర్నెస్, సెల్ఫ్-రిఫ్లెక్టింగ్ కోడ్), నాన్-స్ట్రీమింగ్ కాల్ చాలా గంటలపాటు లేదా నిమిషాలపాటు తేలికగా పనిచేయకపోవచ్చు - దాదాపు ఎలాంటి ప్రత్యక్ష ఫీడ్‌బ్యాక్ లేనిదే. **సంక్లిష్టమైన ప్రాంప్ట్‌లపై ప్రయోగం చేసే సమయంలో స్ట్రీమింగ్ ఉపయోగించండి** తద్వారా మీరు మోడల్ పని చేస్తున్నట్లుగా చూడగలుగుతారు మరియు అభ్యర్థన టైమ్-అవుట్ అయ్యిందనే భావన నుండి తప్పించుకోగలుగుతారు.
>
> **గమనిక: బ్రౌజర్ అవసరం** — స్ట్రీమింగ్ ఫీచర్ Fetch Streams API (`response.body.getReader()`) ను ఉపయోగిస్తుంది, ఇది పూర్తి బ్రౌజర్ (క్రోమ్, ఎడ్జ్, ఫైర్‌ఫాక్స్, సఫారి) అవసరం. ఇది VS కోడ్‌లోని బిల్టిన్ సింపుల్ బ్రౌజర్‌లో పనిచేయదు, ఎందుకంటే అతని వెబ్‌వ్యూ ReadableStream API ను మద్దతు ఇవ్వదు. మీరు సింపుల్ బ్రౌజర్‌ను ఉపయోగిస్తే, నాన్-స్ట్రీమింగ్ బటన్లు సాధారణంగా పనిచేస్తాయి — కేవలం స్ట్రీమింగ్ బటన్లే ప్రభావితం అవుతాయి. పూర్తి అనుభవం కోసం `http://localhost:8083` ను బాహ్య బ్రౌజర్ లో తెరవండి.

### లో ఈగర్నెస్ వర్సెస్ హై ఈగర్నెస్

"200 యొక్క 15% ఎంత?" వంటి సరళమైన ప్రశ్నను లో ఈగర్నెస్‌తో అడగండి. మీరు తక్షణ, ప్రత్యక్ష సమాధానాన్ని పొందుతారు. ఇప్పుడు "హై ట్రాఫిక్ API కోసం క్యాషింగ్ వ్యూహాన్ని రూపకల్పన చేయండి" అనే సంక్లిష్టమైన ప్రశ్నను హై ఈగర్నెస్‌తో అడగండి. **🔴 స్ట్రీమ్ రెస్పాన్స్ (లైవ్)** నొక్కి మోడల్ వివరణాత్మక తర్కాన్ని టోకెన్-బయట ఒకటో ఒక్కటి కనబడుతున్నదిగా చూడండి. అదే మోడల్, అదే ప్రశ్నా నిర్మాణం - కానీ ప్రాంప్ట్ మోడల్ ఎంత ఆలోచించాలో చెపుతుంది.

### కార్యనిర్వాహణ (టూల్ ప్రీంబుల్స్)

బహుళ-దశ పనితీరు ముందస్తు ప్లానింగ్ మరియు పురోగతి వివరణ ద్వారా లాభపడుతుంది. మోడల్ దేనిని చేయబోతుందో తెలియజేస్తుంది, ప్రతి దశను వివరిస్తుంది, తర్వాత ఫలితాల‌ను సారాంశం చేస్తుంది.

### సెల్ఫ్-రిఫ్లెక్టింగ్ కోడ్

"ఇమెయిల్ పరీక్షా సేవను సృష్టించండి" అని ప్రయత్నించండి. కేవలం కోడ్ ఉత్పత్తి చేసి ఆగకుండా, మోడల్ ఉత్పత్తి చేసి, నాణ్యత ప్రమాణాల పట్ల అంచనా వేసి, దోషాలను గుర్తించి, మెరుగుపరుస్తుంది. మీరు మోడల్ పునరావృతం అయ్యే ప్రక్రియను చూడవచ్చు, కోడ్ ఉత్పత్తి ప్రామాణికాలను చేరేదాకా.

### నిర్మిత విశ్లేషణ

కోడ్ సమీక్షలకు స్థిరమైన మూల్యాంకన ఫ్రేమ్‌వర్క్‌లు అవసరం. మోడల్ కోడ్‌ను ఖచ్చితత్వం, అభ్యాసాలు, పనితీరు, భద్రత వంటి స్థాయిలతో విశ్లేషిస్తుంది.

### బహుళ-చర్యానం చాట్

"స్ప్రింగ్ బూట్ అంటే ఏమిటి?" అని అడగండి. వెంటనే "ఉదాహరణ చూపించు" అని అడగండి. మోడల్ మీ ప్రథమ ప్రశ్నను స్మరిస్తుంది మరియు స్ప్రింగ్ బూట్ ఉదాహరణను ప్రత్యేకంగా అందిస్తుంది. జ్ఞాపకశక్తి లేకుండా రెండవ ప్రశ్న అంటే చాలా అస్పష్టంగా ఉంటుంది.

### దశపతన తర్కం

గణిత సమస్యలను ఎంచుకుని దాని మీద దశపతన తర్కం మరియు లో ఈగర్నెస్‌తో ప్రయత్నించండి. లో ఈగర్నెస్ తక్షణమే సమాధానం ఇస్తుంది - వేగంగా కానీ స్పష్ట نہ గా. దశపతన మీరు ప్రక్కోక్కి లెక్కనలను మరియు నిర్ణయాలను చూపిస్తుంది.

### పరిమిత అవుట్పుట్

మీకు ఖచ్చితమైన ఫార్మాట్లు లేదా పద సంఖ్య అవసరమైతే ఈ నమూనా కఠినంగా ఆ పాటిస్తుందని నిర్ధారిస్తుంది. ఖచ్చితంగా 100 పదాలు కలిగిన సారాంశాన్ని బుల్లెట్ పాయింట్ ఫార్మాట్‌లో ఉత్పత్తి చేయడానికి ప్రయత్నించండి.

## మీరు నిజంగా ఏమి నేర్చుకుంటున్నారు

**తర్కం చేసే శ్రమ ప్రతిదీ మారుస్తుంది**

GPT-5.2 మీ ప్రాంప్ట్‌ల ద్వారా కంప్యూటేషనల్ శ్రమను నియంత్రించడానికి అనుమతిస్తుంది. తక్కువ శ్రమ అంటే త్వరిత ప్రతిస్పందనలు కనీస అన్వేషణతో. ఎక్కువ శ్రమ అంటే మోడల్ లోతుగా ఆలోచించడానికి సమయం తీసుకుంటుంది. మీరు పాఠం తీసుకుంటున్నారు ఆ పనిని ఎంత కష్టపడాలో సరిపెట్టుకోవడం - సరళమైన ప్రశ్నలపై సమయం వృధా చేయవద్దు, కానీ సంక్లిష్ట నిర్ణయాలపై కూడా తొందరపడకూడదు.

**సంస్కరణ ప్రవర్తనను మార్గనిర్దేశం చేస్తుంది**

ప్రాంప్ట్‌లలో XML ట్యాగ్‌లు గమనించారా? అవి అలంకరణ కావు. మోడల్స్ నిర్మిత సూచనలకు స్వేచ్ఛా పాఠ్యంతో పోలిస్తే వ్య‌వ‌హార ఉత్సాహంతో పాటిస్తారు. మీరు బహుళ-దశా ప్రాసెస్‌లు లేదా సంక్లిష్ట తర్కాలు అవసరమైతే, నిర్మాణం మోడల్ ఎక్కడ ఉన్నదో మరియు తదుపరి ఏమిటో ట్రాక్ చేయడానికి సహాయపడుతుంది.

<img src="../../../translated_images/te/prompt-structure.a77763d63f4e2f89.webp" alt="Prompt Structure" width="800"/>

*స్పష్టమైన సెక్షన్లు మరియు XML-శైలి వ్యవస్థాపనతో ఒక బాగా నిర్మిత ప్రాంప్ట్ శరీరం*

**నాణ్యత తాని-మూల్యాంకన ద్వారా**

సెల్ఫ్-రిఫ్లెక్టింగ్ నమూనాలు నాణ్యత ప్రమాణాలను స్పష్టంగా చేస్తాయి. మోడల్ "సరైన పని చేస్తుందని" ఆశించడానికి బదులుగా, మీరు దీన్ని "సరైనది" అంటే ఏమిటో ఖచ్చితంగా అంటారు: సరైన తర్కం, లోపాలను నిర్వహణ, పనితీరు, భద్రత. తద్వారా మోడల్ తన స్వంత ఉత్పత్తిని అంచనా వేసి, మెరుగుపరచగలుగుతుంది. ఇది కోడ్ సృష్టిని లాటరీ నుంచి ప్రక్రియగా మార్చుతుంది.

**సందర్భం పరిమిత**

బహుళ-చర్య సంభాషణలు ప్రతి అభ్యర్థనతో మెసేజ్ చరిత్రను కలిగి నడుస్తాయి. కానీ ఒక పరిమితి ఉంది - ప్రతి మోడల్‌కు గరిహిత టోకెన్ లిమిట్ ఉంటుంది. సంభాషణలు పెరిగినపుడు, మీరు సంబంధిత సందర్భాన్ని ఉంచడానికి వ్యూహాలు అవసరం అవుతాయి కానీ ఆ మితిని తాకకుండా ఉండాలి. ఈ మాడ్యూల్‌ని మీరు గుర్తించవచ్చు ఎలా జ్ఞాపకం పని చేస్తుందో; తర్వాత మీరు తెలుసుకుంటారు ఎప్పుడు సారాంశం చేయాలి, ఎప్పుడు మరచిపోవాలి, ఎప్పుడు తిరిగి తెచ్చుకోవాలి.
