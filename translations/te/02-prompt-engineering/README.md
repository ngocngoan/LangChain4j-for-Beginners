## మీరు నేర్చుకునేవి

<img src="../../../translated_images/te/what-youll-learn.c68269ac048503b2.webp" alt="మీరు నేర్చుకునేవి" width="800"/>

మునుపటి మాడ్యూల్‌లో, మీరు మెమొరీయూ సంభాషణాత్మక AIని ఎలా సాధ్యం చేస్తుందో చూశారు మరియు GitHub మోడళ్ళను ప్రాథమిక ఇంటరాక్షన్ల కోసం ఉపయోగించారు. ఇప్పుడు మనం ప్రశ్నలను ఎలా అడుగుతారో చూసేద్దాం — ప్రాంప్ట్స్ అని పిలువబడేలా — Azure OpenAI యొక్క GPT-5.2 ఉపయోగించి. మీరు మీ ప్రాంప్ట్‌లను ఎలా నిర్మిస్తారో, మీరు పొందే సమాధానాల నాణ్యతపై ఇది ప్రభావం చూపుతుంది. మేము ప్రాథమిక ప్రాంప్టింగ్ సాంకేతికతల సమీక్షతో మొదలుపెట్టి, తరువాత GPT-5.2 సామర్థ్యాలను పూర్తిగా ఉపయోగించే ఎనిమిది ఆధునిక నమూనాలనూ చూద్దాం.

మనం GPT-5.2 ఉపయోగిస్తాం ఎందుకంటే ఇది వాదన నియంత్రణను పరిచయం చేస్తుంది — మీరు మోడల్ ఎంతగా ఆలోచించాలో చెప్పవచ్చు. ఇది వివిధ ప్రాంప్టింగ్ వ్యూహాలను స్పష్టంగా చేస్తుంది మరియు ప్రతి దాన్ని ఎప్పుడు ఉపయోగించాలో అర్థం చేసుకోవడంలో సహాయపడుతుంది. Azure యొక్క తక్కువ రేటు పరిమితులు GPT-5.2 కోసం GitHub మోడళ్లతో పోల్చితే మనకు లాభదాయకం.

## ప్రాంప్ట్ ఇంజనీర్‌ఇంగ్ అర్థం చేసుకోవడం

<img src="../../../translated_images/te/what-is-prompt-engineering.5c392a228a1f5823.webp" alt="ప్రాంప్ట్ ఇంజనీర్‌ఇంగ్ అంటే ఏమిటి?" width="800"/>

ప్రాంప్ట్ ఇంజనీర్‌ఇంగ్ అనేది మీరు కోరుకున్న ఫలితాలు సులభంగా వచ్చేలా ఇన్‌పుట్ టెక్స్ట్ రూపకల్పన చేయడమని అర్థం. ఇది కేవలం ప్రశ్నలను అడుగటం కాదు — ఇది ఆ రీతిలో అభ్యర్థనలు నిర్మించడం, తద్వారా మోడల్ మీకు కావాల్సినది స్పష్టంగా అర్థం చేసుకుని ఎలా అందించాలని తెలిసేలా చేయడం.

ఇది సహోద్యోగిని హدایతలివ్వటంలాగా భావించండి. "బగ్ సరిచేయి" అనటం అర్థం మిగిలదు. "UserService.java లైన్ 45లోని నల్ పాయింటర్ ఎక్సెప్షన్‌ను నల్ చెక్ చేరుస్తూ సరిచేయి" అనటం స్పష్టంగా ఉంటుంది. భాషా నమూనాలు కూడా ఇదే విధంగా పనిచేస్తాయి — స్పష్టత మరియు నిర్మాణం ముఖ్యం.

<img src="../../../translated_images/te/how-langchain4j-fits.dfff4b0aa5f7812d.webp" alt="LangChain4j ఎలా సరిపోతుంది" width="800"/>

LangChain4j మౌలిక సదుపాయాలు అందిస్తుంది—మోడల్ కనెక్షన్లు, మెమొరీ మరియు సందేశ రకాలు—మరియు ప్రాంప్ట్ నమూనాలు కాని వాటిని ఉపయోగించే జాగ్రత్తగా నిర్మించిన టెక్స్ట్ మాత్రమే. ముఖ్యమైన నిర్మాణ భాగాలు `SystemMessage` (AI యొక్క ప్రవర్తన మరియు పాత్రను సెట్ చేస్తుంది) మరియు `UserMessage` (మీ నిజమైన అభ్యర్థనను తీసుకుంటుంది).

## ప్రాంప్ట్ ఇంజనీర్‌ఇంగ్ ప్రాథమికాలు

<img src="../../../translated_images/te/five-patterns-overview.160f35045ffd2a94.webp" alt="ప్రాంప్ట్ ఇంజనీర్‌ఇంగ్ ఐదు నమూనాల అవలోకనం" width="800"/>

ఈ మాడ్యూల్‌లో ఆధునిక నమూనాలు ప్రారంభించడానికి ముందు, ఐదు ప్రాథమిక ప్రాంప్టింగ్ సాంకేతికతలను సమీక్షిద్దాం. ఇవి ప్రతి ప్రాంప్ట్ ఇంజనీర్ తెలుసుకోవాల్సిన నిర్మాణ బెల్లాలు. మీరు ఇప్పటికే [క్విక్ స్టార్ట్ మాడ్యూల్](../00-quick-start/README.md#2-prompt-patterns) చూసి ఉంటే, ఈ దృశ్యపటాన్ని చూస్తారు — వీటి వెనుక సిద్ధాంతం ఇక్కడ ఉంది.

### Zero-Shot ప్రాంప్టింగ్

సహజమైన సులభ విధానం: మోడల్‌కు ఎటువంటి ఉదాహరణలు ఇవ్వకుండా ప్రత్యక్ష సూచన ఇవ్వడం. మోడల్ పూర్తిగా తన శిక్షణపై ఆధారపడుతుంది పనిని అర్థం చేసుకుని నెరవేర్చటానికి. ఇది స్పష్టమైన పని కోసం బాగా పనిచేస్తుంది.

<img src="../../../translated_images/te/zero-shot-prompting.7abc24228be84e6c.webp" alt="Zero-Shot ప్రాంప్టింగ్" width="800"/>

*ఉదాహరణలు లేకుండా ప్రత్యక్ష సూచన — మోడల్ సూచనలే ఆధారంగా పని అర్థం చేసుకుంటుంది*

```java
String prompt = "Classify this sentiment: 'I absolutely loved the movie!'";
String response = model.chat(prompt);
// స్పందన: "ధనాత్మకం"
```

**ఎప్పుడు ఉపయోగించాలి:** సులభ వర్గీకరణలు, ప్రత్యక్ష ప్రశ్నలు, అనువాదాలు లేదా అదనపు మార్గనిర్దేశం అవసరం లేని పని.

### Few-Shot ప్రాంప్టింగ్

మోడల్ పాటించాల్సిన నమూనాను చూపించే ఉదాహరణలను ఇవ్వండి. మోడల్ మీ ఉదాహరణల నుండి ఇన్పుట్-అవుట్పుట్ ఫార్మాట్ను నేర్చుకుని, కొత్త ఇనుపుట్లపై వర్తింపజేస్తుంది. ఇది గమ్యస్థానం లేదా ప్రవర్తన స్పష్టంగా లేకపోవడంతో సమర్ధతను పెంచుతుంది.

<img src="../../../translated_images/te/few-shot-prompting.9d9eace1da88989a.webp" alt="Few-Shot ప్రాంప్టింగ్" width="800"/>

*ఉదాహరణల నుండి నేర్చుకోవడం — మోడల్ నమూనా గుర్తించి కొత్త ఇనుపుట్లపై వర్తింపజేస్తుంది*

```java
String prompt = """
    Classify the sentiment as positive, negative, or neutral.
    
    Examples:
    Text: "This product exceeded my expectations!" → Positive
    Text: "It's okay, nothing special." → Neutral
    Text: "Waste of money, very disappointed." → Negative
    
    Now classify this:
    Text: "Best purchase I've made all year!"
    """;
String response = model.chat(prompt);
```

**ఎప్పుడు ఉపయోగించాలి:** కస్టమ్ వర్గీకరణలు, సారూప్యమైన ఫార్మాటింగ్, విభాగ ప్రత్యేక పనులు, లేదా zero-shot ఫలితాలు అస్పష్టమైనప్పుడు.

### చెయిన్ ఆఫ్ థాట్

మోడల్ దశబద్ధంగా తర్కాన్ని చూపించాలని అడగండి. ప్రత్యక్ష సమాధానానికి కాకుండా, మోడల్ సమస్యను విడదీసి ప్రతి భాగాన్ని స్పష్టంగా ప్రేక్షకులకు చూపిస్తుంది. ఇది గణితం, తర్కం మరియు బహుళ దశల వాదన పనులకు ఖచ్చితత్వాన్ని పెంచుతుంది.

<img src="../../../translated_images/te/chain-of-thought.5cff6630e2657e2a.webp" alt="చెయిన్ ఆఫ్ థాట్ ప్రాంప్టింగ్" width="800"/>

*దశల వారీగా తర్కం — సంక్లిష్ట సమస్యలను స్పష్ట తార్కీక దశలుగా విభజించడం*

```java
String prompt = """
    Problem: A store has 15 apples. They sell 8 apples and then 
    receive a shipment of 12 more apples. How many apples do they have now?
    
    Let's solve this step-by-step:
    """;
String response = model.chat(prompt);
// మోడల్ చూపిస్తుంది: 15 - 8 = 7, తరువాత 7 + 12 = 19 సేబులు
```

**ఎప్పుడు ఉపయోగించాలి:** గణిత సమస్యలు, తర్క పజిల్స్, డీబగ్గింగ్ లేదా తర్క ప్రవేశాన్ని చూపటం ఖచ్చితత్వం మరియు నమ్మకం పెంచే సందర్భాలు.

### పాత్ర ఆధారిత ప్రాంప్టింగ్

మీ ప్రశ్న అడగటం క్రితం AIకి పాత్ర లేదా వ్యక్తిత్వాన్ని సెట్ చేయండి. ఇది సమాధానంలో టోన్, వివరాలకు మరియు దృష్టికోణం ప్రభావం చూపే సందర్భాన్ని అందిస్తుంది. "సాఫ్ట్‌వేర్ ఆర్కిటెక్ట్" "జూనియర్ డెవలపర్" లేదా "సెక్యూరిటీ ఆడిటర్" కంటే వేరే సలహాలను ఇస్తుంది.

<img src="../../../translated_images/te/role-based-prompting.a806e1a73de6e3a4.webp" alt="పాత్ర ఆధారిత ప్రాంప్టింగ్" width="800"/>

*సందర్భం మరియు పాత్ర సెట్ చేయడం — అదే ప్రశ్నకు ఎటు పాత్ర ఉన్నదీ ఆధారంగా వేరే సమాధానం వస్తుంది*

```java
String prompt = """
    You are an experienced software architect reviewing code.
    Provide a brief code review for this function:
    
    def calculate_total(items):
        total = 0
        for item in items:
            total = total + item['price']
        return total
    """;
String response = model.chat(prompt);
```

**ఎప్పుడు ఉపయోగించాలి:** కోడ్ సమీక్షలు, బోధన, విభాగ ప్రత్యేక విశ్లేషణ లేదా విభిన్న నైపుణ్య స్థాయిలకు అనుగుణంగా సమాధానాలు కావాలన్నప్పుడు.

### ప్రాంప్ట్ టెంప్లేట్స్

మార్పిడ 가능한 స్థానాలు కలిగిన పునర్వినియోగ ప్రాంప్ట్‌లు సృష్టించండి. ప్రతి సారి కొత్త ప్రాంప్ట్ వ్రాయకుండా, ఒకసారి టెంప్లేట్ నిర్వచించి వేరువేరు విలువలు నింపండి. LangChain4j యొక్క `PromptTemplate` క్లాస్ `{{variable}}` సింటాక్స్ తో దీనిని సులభతరం చేస్తుంది.

<img src="../../../translated_images/te/prompt-templates.14bfc37d45f1a933.webp" alt="ప్రాంప్ట్ టెంప్లేట్స్" width="800"/>

*వెలియునిచ్చే స్థానాలతో పునర్వినియోగ ప్రాంప్ట్‌లు — ఒక టెంప్లేట్ అనేక ఉపయోగాలు*

```java
PromptTemplate template = PromptTemplate.from(
    "What's the best time to visit {{destination}} for {{activity}}?"
);

Prompt prompt = template.apply(Map.of(
    "destination", "Paris",
    "activity", "sightseeing"
));

String response = model.chat(prompt.text());
```

**ఎప్పుడు ఉపయోగించాలి:** విభిన్న ఇన్‌పుట్స్ తో పునరావృత ప్రశ్నలు, బ్యాచ్ ప్రాసెసింగ్, పునర్వినియోగ AI కార్యకలాపాలు, లేదా ప్రాంప్ట్ నిర్మాణం అలానే ఉండి డేటానే మారితే సరిపోతుంది.

---

ఈ ఐదు ప్రాథమికాలు చాలా ప్రాంప్టింగ్ పనులకు ప్రాథమిక టూల్‌కిట్ ఇస్తాయి. మిగతా ఈ మాడ్యూల్ వాటిపై ఆధారపడి GPT-5.2 యొక్క వాదన నియంత్రణ, స్వయం-మూల్యాంకనం మరియు వ్యవస్థీకృత అవుట్పుట్ సామర్థ్యాలను చేర్చే ఎనిమిది ఆధునిక నమూనాలను అర్థం చేసుకుంటుంది.

## అధునాతన నమూనాలు

ప్రాథమికాలు పూర్తయ్యాక, ఈ మాడ్యూల్ ప్రత్యేకతకు ఎనిమిది ఆధునిక నమూనాల వైపు పయనిద్దాం. అన్ని సమస్యలకు ఒకటే విధానం అవసరం కాదు. కొన్ని ప్రశ్నలకు తక్షణ సమాధానాలు కావాలి, మరికొన్ని లోతైన ఆలోచన అవసరం. కొన్ని తర్కాన్ని చూపించాలి, మరికొన్ని ఫలితాలు కనీసం కావాలి. క్రింద ప్రతి నమూనా ప్రత్యేక పరిస్థితులకు సరిపోయేలా రూపొందించబడింది — మరియు GPT-5.2 వాదన నియంత్రణ ఈ తేడాలను మరింత సూక్ష్మంగా చేస్తుంది.

<img src="../../../translated_images/te/eight-patterns.fa1ebfdf16f71e9a.webp" alt="ఎనిమిది ప్రాంప్టింగ్ నమూనాలు" width="800"/>

*ఎనిమిది ప్రాంప్ట్ ఇంజనీర్‌ఇంగ్ నమూనాల అవలోకనం మరియు వాటి ఉపయోగాల*

<img src="../../../translated_images/te/reasoning-control.5cf85f0fc1d0c1f3.webp" alt="GPT-5.2 తో వాదన నియంత్రణ" width="800"/>

*GPT-5.2 వాదన నియంత్రణ ద్వారా మోడల్ ఎంత ఆలోచించాలో మీరు నిర్ణయించవచ్చు — వేగంగా ప్రత్యక్ష సమాధానాల నుండి లోతైన పరిశీలన వరకు*

<img src="../../../translated_images/te/reasoning-effort.db4a3ba5b8e392c1.webp" alt="వాదన శ్రమా సరిపోలిక" width="800"/>

*తక్కువ ఆసక్తి (వేగంగా, ప్రత్యక్షంగా) vs అధిక ఆసక్తి (సమగ్రంగా, అన్వేషణాత్మకంగా) వాదన విధానాలు*

**తక్కువ ఆసక్తి (త్వరిత & లక్ష్యసాధక)** - మీకు సూటిగా మరియు వేగంగా సమాధానాలు కావాలన్న ప్రశ్నల కోసం. మోడల్ కనీసమైన వాదన చేస్తుంది - గరిష్టం 2 దశలు. లెక్కింపులు, సూచనలు లేదా సరళ ప్రశ్నలకు ఈ విధానం ఉపయోగించండి.

```java
String prompt = """
    <reasoning_effort>low</reasoning_effort>
    <instruction>maximum 2 reasoning steps</instruction>
    
    What is 15% of 200?
    """;

String response = chatModel.chat(prompt);
```

> 💡 **GitHub Copilot తో అన్వేషించండి:** [`Gpt5PromptService.java`](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java) ఓపెన్ చేసి అడగండి:
> - "తక్కువ ఆసక్తి మరియు అధిక ఆసక్తి ప్రాంప్టింగ్ నమూనాల మధ్య వ్యత్యాసం ఏమిటి?"
> - "ప్రాంప్ట్‌లలో XML ట్యాగ్లు AI యొక్క సమాధాన నిర్మాణానికి ఎలా సహాయపడతాయి?"
> - "స్వీయ-ప్రతిబింబ నమూనాలు vs ప్రత్యక్ష సూచనలను ఎప్పుడు ఉపయోగించాలి?"

**అధిక ఆసక్తి (లోతైన & సమగ్ర)** - మీరు సమగ్ర విశ్లేషణ కావాలనుకునే సంక్లిష్ట సమస్యలకు. మోడల్ పూర్తిగా అన్వేషించి, వివరమైన వాదనను చూపుతుంది. సిస్టమ్ డిజైన్, శిల్ప నిర్ణయాలు లేదా సంక్లిష్ట పరిశోధనలకు ఇది ఉపయోగించండి.

```java
String prompt = """
    <reasoning_effort>high</reasoning_effort>
    <instruction>explore thoroughly, show detailed reasoning</instruction>
    
    Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```

**పని అమలు (దశల వారీ ప్రగతిపథం)** - బహుళ దశల పనాల కోసం. మోడల్ ముందుగూనె ప్రణాళిక ఇస్తుంది, ప్రతి దశను వివరిస్తూ పయనిస్తుంది, తరువాత సమీక్ష ఇస్తుంది. మైగ్రేషన్లు, అమలు, లేదా ఏ బహుళ దశల ప్రక్రియ కోసం ఉపయోగించండి.

```java
String prompt = """
    <task>Create a REST endpoint for user registration</task>
    <preamble>Provide an upfront plan</preamble>
    <narration>Narrate each step as you work</narration>
    <summary>Summarize what was accomplished</summary>
    """;

String response = chatModel.chat(prompt);
```

చెయిన్-ఆఫ్-థాట్ ప్రాంప్టింగ్ మోడల్ ను దాని వాదన ప్రక్రియను చూపించాలని స్పష్టంగా అడిగేది, సంక్లిష్టమైన పనుల ఖచ్చితత్వం పెంచుతుంది. దశల వారీ విభజన మానవులకు మరియు AIకి తర్కాన్ని అర్థమవ్వడానికి సహాయపడుతుంది.

> - "దీర్ఘకాలిక ఆపరేషన్ల కోసం పని అమలు నమూనాను ఎలా అనుకూలపరచగలను?"
> - "ఉత్పత్తి అప్లికేషన్లలో టూల్ ప్రీాంబుల్స్ నిర్మాణానికి ఉత్తమ ప్రాక్టీసులు ఏమిటి?"
> - "UIలో మధ్యవర్తి ప్రగతిని ఎలా పట్టుకోవచ్చు మరియు ప్రదర్శించగలము?"

<img src="../../../translated_images/te/task-execution-pattern.9da3967750ab5c1e.webp" alt="పని అమలు నమూనా" width="800"/>

*ప్లాన్ → అమలు → సమీక్ష బహుళ దశల పనుల వర్క్‌ఫ్లో*

**స్వీయ-ప్రతిబింబించే కోడ్** - ఉత్పత్తి-నాణ్యత కోడ్ నిర్మాణం కోసం. మోడల్ కోడ్‌ను రూపొందించి, నాణ్యత ప్రమాణాలతో తనిఖీ చేసి, దాన్ని మెరుగుపరుస్తుంది. కొత్త ఫీచర్స్ లేదా సర్వీసులు నిర్మిస్తున్నప్పుడు ఉపయోగించండి.

```java
String prompt = """
    <task>Create an email validation service</task>
    <quality_criteria>
    - Correct logic and error handling
    - Best practices (clean code, proper naming)
    - Performance optimization
    - Security considerations
    </quality_criteria>
    <instruction>Generate code, evaluate against criteria, improve iteratively</instruction>
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/te/self-reflection-cycle.6f71101ca0bd28cc.webp" alt="స్వీయ-ప్రతిబింబ సైకిల్" width="800"/>

*పునరావృత మెరుగుదల సైకిల్ - రూపొందించు, మూల్యాంకన చేయు, సమస్యలు గుర్తించు, మెరుగాచేయు, పునరావృతం చేస్తున్నది*

**సంఖ్యాత్మక విశ్లేషణ** - సమానమైన మూల్యాంకన కోసం. మోడల్ ఒక స్థిరమైన ఫ్రేమ్‌వర్క్ (సరైనదని, ఆచరణలు, పనితనం, భద్రత) ఉపయోగించి కోడ్‌ను సమీక్షిస్తుంది. కోడ్ సమీక్షలు లేదా నాణ్యత అంచనాలకు ఉపయోగించండి.

```java
String prompt = """
    <code>
    public List getUsers() {
        return database.query("SELECT * FROM users");
    }
    </code>
    
    <framework>
    Evaluate using these categories:
    1. Correctness - Logic and functionality
    2. Best Practices - Code quality
    3. Performance - Efficiency concerns
    4. Security - Vulnerabilities
    </framework>
    """;

String response = chatModel.chat(prompt);
```

> - "వివిధ కోడ్ సమీక్షలకు ఫ్రేమ్‌వర్క్‌ను ఎలా అనుకూలీకరించగలను?"
> - "సంఖ్యాత్మక అవుట్పుట్‌ను ప్రోగ్రామాటిక్‌గా పార్స్ చేసి ఎలా ఉపయోగించాలి?"
> - "సమానమైన తీవ్రత స్థాయిలను వివిధ సమీక్షల సౌభాగ్యాల్లో ఎలా ఖాతరపరచాలి?"

<img src="../../../translated_images/te/structured-analysis-pattern.0af3b690b60cf2d6.webp" alt="సంఖ్యాత్మక విశ్లేషణ నమూనా" width="800"/>

*నాలుగు వర్గాల ఫ్రేమ్‌వర్క్ సహా, తీవ్రత స్థాయిలతో సమానమైన కోడ్ సమీక్షలు*

**బహు-తదుపరి చాట్** - సందర్భం అవసరమైన సంభాషణలకు. మోడల్ ముందటి సందేశాలను గుర్తుంచుకుని వాటిపై ఆధారపడి సమాధానాలు ఇస్తుంది. అంతర్రాష్ట్ర సహాయం సమావేశాలు లేదా సంక్లిష్ట ప్రశ్నల సమాధానాలకు ఉపయోగించండి.

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("What is Spring Boot?"));
AiMessage aiMessage1 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage1);

memory.add(UserMessage.from("Show me an example"));
AiMessage aiMessage2 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage2);
```

<img src="../../../translated_images/te/context-memory.dff30ad9fa78832a.webp" alt="సందర్భ మెమొరీ" width="800"/>

*చర్చా సందర్భం అనేక తుతుత్తులతో కూడి టోకెన్ పరిమితి నిండేవరకు నిల్వ చేయబడుతుంది*

**దశబద్ధమైన వాదన** - దృశ్యమైన తార్కిక ధోరణి అవసరమైన సమస్యలకు. మోడల్ ప్రతి దశ కోసం స్పష్టమైన వాదనను చూపిస్తుంది. గణిత సమస్యలు, తర్క పజిల్స్ లేదా ఆలోచనా ప్రక్రియ అర్థం చేసుకోవాల్సినప్పుడు ఉపయోగించండి.

```java
String prompt = """
    <instruction>Show your reasoning step-by-step</instruction>
    
    If a train travels 120 km in 2 hours, then stops for 30 minutes,
    then travels another 90 km in 1.5 hours, what is the average speed
    for the entire journey including the stop?
    """;

String response = chatModel.chat(prompt);
```

*సమస్యలని స్పష్టమైన తార్కిక దశలుగా విభజించడం*


డెవ్ కంటైనర్‌లో Spring Boot డ్యాష్‌బోర్డ్ ఎక్స్‌టెన్షన్ ఉంది, ఇది అన్ని Spring Boot అప్లికేషన్లను నిర్వహించేందుకు గرافికల్ ఇంటర్‌ఫేస్ ఇస్తుంది. దీన్ని VS Code ఎడమ వైపు ఉన్న Activity Bar లో (Spring Boot ఐకాన్) చూడవచ్చు.
Spring Boot డాష్‌బోర్డ్ నుండి, మీరు చేయవచ్చు:
- వర్క్‌స్టేషన్లో అందుబాటులో ఉన్న అన్ని Spring Boot అప్లికేషన్‌లను చూడండి
- ఒక క్లిక్‌తో అప్లికేషన్‌లను స్టార్ట్/స్టాప్ చేయండి
- అప్లికేషన్ లాగ్‌లను రియల్ టైమ్‌లో వీక్షించండి
- అప్లికేషన్ స్థితిని పర్యవేక్షించండి

"prompt-engineering" పక్కన ఉన్న ప్లే బటన్ క్లిక్ చేయడం ద్వారా ఈ మాడ్యూల్‌ను ప్రారంభించండి, లేదా అన్ని మాడ్యూల్‌లను ఒకేసారిగా ప్రారంభించండి.

GPT-5.2 ద్వారా మీరు ప్రాంప్ట్‌ల ద్వారా గణనాత్మక ప్రయత్నాన్ని నియంత్రించవచ్చు. తక్కువ ప్రయత్నం అంటే వేగంగా, తక్కువ అన్వేషణతో జవాబులు. ఎక్కువ ప్రయత్నం అంటే మోడల్ లోతైన ఆలోచనకు సమయం తీసుకుంటుంది. మీరు టాస్క్ కష్టం మరియు ప్రయత్నం సరిపోయే విధంగా సరిపోల్చ learn చేస్తున్నారు - సులభ ప్రశ్నల్లో సమయం వృథా చేయవద్దు, కానీ సంక్లిష్ట నిర్ణయాలలో తొందరపడకండి.

**స్థూపం ప్రవర్తనను సూచిస్తుంది**

ప్రాంప్ట్‌ల XML ట్యాగ్‌లు గమనించారా? అవి అలంకరణ మాత్రమే కావు. మోడల్‌లు నిర్మిత ఆదేశాలను స్వేచ్ఛాస్థాయిలో ఉన్న టెక్స్ట్ కంటే విశ్వసనీయంగా అనుసరిస్తాయి. బహుళ దశా ప్రక్రియలు లేదా సంక్లిష్ట తార్కాలు ఉన్నప్పుడు, నిర్మాణం మోడల్ ఎలా ఉన్నదో, తర్వాత ఏమి చేయాలో ట్రాక్ చేయడంలో సహాయపడుతుంది.

<img src="../../../translated_images/te/prompt-structure.a77763d63f4e2f89.webp" alt="Prompt Structure" width="800"/>

*స్పష్ట విభాగాలు మరియు XML-శైలీ వ్యవస్థతో బాగా నిర్మిత ప్రాంప్ట్ యొక్క శరీరం*

**నాణ్యత స్వయం-పరిశీలన ద్వారా**

స్వయం-పరిశీలన నమూనాలు నాణ్యత ప్రమాణాలను స్పష్టంగా చేస్తాయి. మోడల్ "సరైనది చేస్తుందనుకునే" సహాయకం కాకుండా, మీరు "సరైనది" అంటే ఏమిటి అంటే: సరిగా తార్కికత, తప్పుల నిర్వహణ, పనితీరు, భద్రత. మోడల్ ఆ ఉత్పత్తిని స్వయంగా మూల్యాంకనం చేసి మెరుగుపరుచుకోవచ్చు. ఇది కోడ్ జనరేషన్‌ను లాటరీ నుండి ఒక ప్రాసెస్‌గా మార్చుతుంది.

**సందర్భం పరిమితి కలిగిఉంది**

బహుట్ర సంభాషణలు ప్రతీ అభ్యర్థనతో సందేశ చరిత్రను కలుపుకోవడం ద్వారా పనిచేస్తాయి. అయితే పరిమితి ఉంది - ప్రతి మోడల్ లో గరిష్ట టోకెన్ కౌంట్ ఉంటుంది. సంభాషణలు పెరిగే కొద్దీ, సంబంధిత సందర్భాన్ని నిలుపుకోవడానికి వ్యూహాలు అవసరం ఉంటాయి, ఆ సరిహద్దును తాకకుండా. ఈ మాడ్యూల్ మీరు జ్ఞాపకం ఎలా పనిచేస్తుందో చూపిస్తుంది; తర్వాత మీరు ఎప్పుడు సారాంశం చేయాలి, ఎప్పుడు మరచిపోవాలి, ఎప్పుడు తిరిగి తీసుకురావాలి అంటే నేర్చుకుంటారు.
