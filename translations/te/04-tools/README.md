## మీకు నేర్చుకోవాల్సింది ఏమిటి

ఇప్పటివరకు, మీరు AIతో సంభాషణలు ఎలా నిర్వహించాలో, సూత్రాలు సమర్ధవంతంగా ఎలా నిర్మించాలో, మరియు మీ డాక్యుమెంట్లలోను సమాధానాలను ఎలా ఆధారపెట్టాలో నేర్చుకున్నారు. కానీ ఇంకా ఒక ప్రాథమిక పరిమితి ఉంది: భాషా నమూనాలు టెక్స్ట్ మాత్రమే ఉత్పత్తి చేయగలవు. అవి వాతావరణం చూసుకోలేవు, లెక్కింపులు చేయలేవు, డేటాబేస్‌లను ప్రశ్నించలేవు లేదా బయటి వ్యవస్థలతో సంబంధం కలిగి ఉండలేవు.

టూల్స్ దీన్ని మార్చేస్తాయి. నమూనాకు వాడగల ఫంక్షన్లను అందించడం ద్వారా, మీరు దాన్ని టెక్స్ట్ తయారీదారుగా కాకుండా చర్యలు తీసుకునే ఏజెంటుగా మార్చుతారు. నమూనా ఎప్పుడెప్పుడు టూల్ అవసరమో, ఏ టూల్ వాడాలో, మరియు ఏమి ఇచ్చాలో నిర్ణయిస్తుంది. మీ కోడ్ ఆ ఫంక్షన్ నడిపించి ఫలితాన్ని తిరిగి ఇస్తుంది. నమూనా ఆ ఫలితాన్ని తన సమాధానంలో చేర్చుకుంటుంది.

## టూల్స్ ఉన్న AI ఏజెంట్‌ల అర్థం చేసుకోవడం

> **📝 గమనిక:** ఈ మాడ్యూల్‌లో "ఏజెంట్స్" అనే పదం టూల్-కాల్ చేసే సామర్థ్యాలతో పెంపొందించిన AI సహాయకులను సూచిస్తుంది. ఇది [మాడ్యూల్ 05: MCP](../05-mcp/README.md) లో పరిగణించబోయే **Agentic AI** ప్యాటర్న్‌లు (ఆటోనమస్ ఏజెంట్లు, ప్లానింగ్, మెమరీ, మరియు మల్టీ-స్టెప్ తర్కం) నుండి వేరుగా ఉంటుంది.

టూల్స్ లేకుండా, భాషా నమూనా తన శిక్షణ డేటాల నుండి మాత్రమే టెక్స్ట్ ఉత్పత్తి చేయగలదు. మీకు సమకాలీన వాతావరణం తెలుసుకోవాలంటే, అది ఊహించాల్సి వస్తుంది. కానీ టూల్స్ ఇచ్చితే, అది వాతావరణ API, లెక్కింపులు, లేదా డేటాబేస్‌కు కాల్ చేయగలదు — ఆ వాస్తవ ఫలితాలను తన సమాధానంలో జోడిస్తుంది.

<img src="../../../translated_images/te/what-are-tools.724e468fc4de64da.webp" alt="Without Tools vs With Tools" width="800"/>

*టూల్స్ లేకుండా నమూనా అంచనా మాత్రమే వేస్తుంది — టూల్స్ ఉన్నప్పుడు అది APIsను పిలవగలదు, లెక్కింపులు నడిపించగలదు, మరియు వాస్తవ-సమయ డేటాను అందించగలదు.*

టూల్స్ ఉన్న AI ఏజెంట్ ఒక **తర్కం మరియు చరిస్తేర్చడం (ReAct)** ప్యాటర్న్ ని అనుసరిస్తుంది. నమూనా కేవలం స్పందించదు — అది అవసరమైనది ఏమిటో ఆలోచించేది, టూల్ పిలిచేది, ఫలితాన్ని పరిశీలిస్తది, మరియు తిరిగి చర్య తీసుకోవాలా లేదా చివరి సమాధానం ఇవ్వాలా నిర్ణయిస్తది:

1. **తర్కించు** — వాడుకరి ప్రశ్నను విశ్లేషించి ఏ సమాచారము కావాలో నిర్ధారిస్తుంది
2. **చరిస్తేర్చు** — సరైన టూల్ ను ఎంచుకొని, సరైన పారామితులు సృష్టించి పిలుస్తుంది
3. **పరిశీలించు** — టూల్ అవుట్పుట్ ని పొందుతుంది మరియు ఫలితాన్ని అంచనా వేస్తుంది
4. **మళ్ళీ లేదా స్పందించు** — ఎక్కువ డేటా అవసరం అయితే తిరిగి ప్రయత్నిస్తుంది; లేని పక్షంలో సహజ భాష సమాధానం తయారు చేస్తుంది

<img src="../../../translated_images/te/react-pattern-detail.96a5efeeb6dd2f61.webp" alt="ReAct Pattern" width="800"/>

*ReAct చక్రం — ఏజెంట్ తర్కం చేసి, టూల్ కాల్ చేస్తుంది, ఫలితాన్ని చూస్తుంది, మరియు చివరి సమాధానం ఎప్పుడు ఇవ్వాలో నిర్ణయిస్తుంది.*

ఈ ప్రక్రియ ఆటోమేటిక్‌గా జరుగుతుంది. మీరు టూల్స్ మరియు వాటి వివరణలను నిర్వచిస్తారు. నమూనా వాటిని ఎప్పుడు ఎలా వాడాలో నిర్ణయిస్తుంది.

## టూల్ కాలింగ్ ఎలా పనిచేస్తుంది

### టూల్ నిర్వచనలు

[WeatherTool.java](../../../04-tools/src/main/java/com/example/langchain4j/agents/tools/WeatherTool.java) | [TemperatureTool.java](../../../04-tools/src/main/java/com/example/langchain4j/agents/tools/TemperatureTool.java)

మీరు క్లియర్ వివరణలు మరియు పారామితుల స్పెసిఫికేషన్తో ఫంక్షన్లను నిర్వచిస్తారు. నమూనా ఈ వివరణలను తన సిస్టమ్ ప్రాంప్ట్‌లో చూస్తుంది మరియు ప్రతి టూల్ ఏమి చేస్తుందో అర్థం చేసుకుంటుంది.

```java
@Component
public class WeatherTool {
    
    @Tool("Get the current weather for a location")
    public String getCurrentWeather(@P("Location name") String location) {
        // మీ వాతావరణం శోధన లాజిక్
        return "Weather in " + location + ": 22°C, cloudy";
    }
}

@AiService
public interface Assistant {
    String chat(@MemoryId String sessionId, @UserMessage String message);
}

// అసిస్టెంట్ Spring Boot చేత ఆటోమేటిక్గా వైర్డ్ చేయబడింది:
// - ChatModel బీన్
// - @Component తరగతులలోని అన్ని @Tool విధానాలు
// - సెషన్ నిర్వహణ కోసం ChatMemoryProvider
```

క్రింది డయాగ్రామ్ ప్రతి అనోటేషన్‌ని విరమించి చూపిస్తుంది మరియు ఏ విధంగా అవి AIకి టూల్ పిలవాల్సిన సమయం మరియు ఆర్జుమెంట్స్ ఎలా ఇవ్వాలో అర్థం చేసుకోవడంలో సహాయపడతాయో వివరిస్తుంది:

<img src="../../../translated_images/te/tool-definitions-anatomy.f6468546037cf28b.webp" alt="Anatomy of Tool Definitions" width="800"/>

*టూల్ నిర్వచన నిర్మాణం — @Tool ఐтим్ టూల్ ఎప్పుడు వాడాలో AIకి చెప్తుంది, @P ప్రతి పారామితిని వివరిస్తుంది, మరియు @AiService స్టార్ట్ అప్ సమయంలో సంధానం చేస్తుంది.*

> - "టూల్ అమలులో API లోపాలు మరియు రేట్ లిమిట్‌లను ఎలా హ్యాండిల్ చేయాలి?"

### నిర్ణయించడం

వాడుకరి "సియాటిల్‌లో వాతావరణం ఏంటి?" అడుగుతున్నప్పుడు, నమూనా యాదృచ్ఛికంగా టూల్ ఎంచుకోదు. ఇది ప్రతి టూల్ వివరణకు వ్యతిరేకంగా వాడుకరి ఉద్దేశాన్ని సిమించింది చేసి, వాటికి స్కోరు ఇస్తుంది మరియు అత్యుత్తమ సరిపోలును ఎంచుకుంటుంది. తర్వాత సరైన పారామితులతో నిర్మిత ఫంక్షన్ కాల్ ఉత్పత్తి చేస్తుంది — ఈ సందర్భంలో `location` ను `"Seattle"` గా సెట్ చేస్తుంది.

టూల్ సరిపోకపోతే, నమూనా తన జ్ఞానానికి ఆధారంగా సమాధానం ఇస్తుంది. ఒకাধিক టూల్స్ సరిపోతే, అత్యంత స్పష్టమైనది ఎంచుకుంటుంది.

<img src="../../../translated_images/te/decision-making.409cd562e5cecc49.webp" alt="How the AI Decides Which Tool to Use" width="800"/>

*నమూనా ప్రతి అందుబాటులో ఉన్న టూల్‌ని వాడుకరి ఉద్దేశంతో పోల్చి చూసి ఉత్తమ సరిపోలును ఎంచుకుంటుంది — అందుకే క్లియర్, స్పష్టమైన టూల్ వివరణలు ముఖ్యం.*

నమూనా వాతావరణ డేటాను స్వీకరించి, వాడుకరి కోసం సహజ భాషా సమాధానంగా ఫార్మాట్ చేస్తుంది.

### ఆర్కిటెక్చర్: స్ప్రింగ్ బూట్ ఆటో-వైరింగ్

ఈ మాడ్యూల్ LangChain4j యొక్క స్ప్రింగ్ బూట్ ఇంటిగ్రేషన్‌ని డిక్లరేటివ్ `@AiService` ఇంటర్‌ఫేసులతో ఉపయోగిస్తుంది. స్టార్ట్ అప్ సమయంలో స్ప్రింగ్ బూట్ ప్రతి `@Component`ను కనుగొంటుంది, అందులో ఉన్న `@Tool` పద్ధతులతో సహా, మీరు ఇచ్చిన `ChatModel` బీన్ మరియు `ChatMemoryProvider`ను సింగిల్ `Assistant` ఇంటర్‌ఫేస్‌గా జత చేస్తుంది, బొత్తికెక్కని బోయిలర్‌ప్లేట్ లేకుండా.

<img src="../../../translated_images/te/spring-boot-wiring.151321795988b04e.webp" alt="Spring Boot Auto-Wiring Architecture" width="800"/>

*@AiService ఇంటర్‌ఫేస్ ChatModel, టూల్ కంపోనెంట్లు, మెమరీ ప్రొవైడర్ ను కలిపి - స్ప్రింగ్ బూట్ ఆటోమేటిక్‌గా అన్ని వైర్లను నిర్వహిస్తుంది.*

ఈ విధానం కీలక లాభాలు:

- **స్ప్రింగ్ బూట్ ఆటో-వైరింగ్** — ChatModel మరియు టూల్స్ ఆటోమేటిక్ ఇంజెక్ట్ చేయబడతాయి
- **@MemoryId ప్యాటర్న్** — ఆటోమేటిక్ సెషన్ ఆధారిత మెమరీ నిర్వహణ
- **ఒక్క సారి పనిని చేస్తూ** — Assistant ఒక్కసారి సృష్టించి మెరుగైన పనితీరు కోసం రీయూజ్ చేస్తుంది
- **టైప్-సేఫ్ అమలు** — జావా పద్ధతులు టైప్ మార్పిడి తో నేరుగా పిలవబడతాయి
- **మల్టీ-టర్న్ ఆర్కెస్ట్రేషన్** — టూల్ చైనింగ్ ఆటోమేటిక్ గా నిర్వహిస్తుంది
- **జీరో బోయిలర్‌ప్లేట్** — మానవీయంగా `AiServices.builder()` కాల్స్ లేకుండా లేదా మెమరీ హ్యాష్‌మ్యాప్ అవసరం లేదు

ఇతర పద్ధతులు (మాన్యువల్ `AiServices.builder()`) ఎక్కువ కోడ్ అవసరం మరియు స్ప్రింగ్ బూట్ ఇంటిగ్రేషన్ లాభాలు కోల్పోతాయి.

## టూల్ చైనింగ్

**టూల్ చైనింగ్** — ఒకే ప్రశ్నకు అనేక టూల్స్ అవసరం పడినప్పుడు టూల్-ఆధారిత ఏజెంట్ యొక్క అసలైన శక్తి బయటపడుతుంది. "సియాటిల్‌లో వాతావరణం ఫారన్హైట్లో ఏంటి?" అడిగితే ఏజెంట్ ఆటోమేటిక్‌గా రెండు టూల్స్ చైన్ చేస్తుంది: మొదట `getCurrentWeather` కాల్ చేసి సెల్సియస్ లో ఉష్ణోగ్రత తెలుసుకుంటుంది, దానిని `celsiusToFahrenheit`కి పంపించి మార్పిడి చేస్తుంది — ఇది ఒకే సంభాషణ టర్న్‌లో జరుగుతుంది.

<img src="../../../translated_images/te/tool-chaining-example.538203e73d09dd82.webp" alt="Tool Chaining Example" width="800"/>

*టూల్ చైనింగ్లో చర్య — ఏజెంట్ ముందుగా getCurrentWeatherని పిలుస్తుంది, తర్వాత సెల్సియస్ ఫలితాన్ని celsiusToFahrenheitకి పంపుతుంది, మరియు కలిపిన సమాధానం ఇస్తుంది.*

**గ్రేస్‌ఫుల్ ఫెయిల్యూర్లు** — మాక్ డేటాలో లేనిది నగరంలో వాతావరణం అడిగితే టూల్ లోప సందేశాన్ని తిరిగి ఇస్తుంది, AI సహాయం చేయలేనంటుంది, క్రాష్ కాకుండా. టూల్స్ సురక్షితంగా ఫెయిల్ అవుతాయి. క్రింది డయాగ్రామ్ ఇద్దరు పద్ధతులను కాంపేర్లు చేస్తుంది — సరైన లోప నిర్వహణతో ఏజెంట్ ఎర్రర్‌ను పట్టుకుని సహాయక వ్యాఖ్యతో ప్రతిస్పందిస్తుంది, లేదంటే యాప్ మొత్తం క్రాష్ అవుతుంది:

<img src="../../../translated_images/te/error-handling-flow.9a330ffc8ee0475c.webp" alt="Error Handling Flow" width="800"/>

*టూల్ విఫలమైనప్పుడు ఏజెంట్ లోపాన్ని పట్టుకుని సహాయక వివరణతో ప్రతిచర్య చూపిస్తుంది, క్రాష్ కాకుండా.*

ఇది ఒకే సంభాషణ టర్న్‌లో జరుగుతుంది. ఏజెంట్ స్వతంత్రంగా అనేక టూల్ కాల్స్ ని నిర్వహిస్తుంది.

స్ప్రింగ్ బూట్ డాష్‌బోర్డ్ నుండి మీరు:
- వర్క్‌స్పేస్‌లో అందుబాటులో ఉన్న అన్ని స్ప్రింగ్ బూట్ అప్లికేషన్స్ చూడవచ్చు

- అప్లికేషన్ లాగ్‌లను రియల్టైంలో వీక్షించడం
- అప్లికేషన్ స్థితి పర్యవేక్షణ

"tools" పక్కన ఉన్న ప్లే బటన్ నొక్కి ఈ మాడ్యూల్‌ను ప్రారంభించండి, లేదా ఒకేసారి అన్ని మాడ్యూల్‌లను ప్రారంభించండి.

*AI ఏజెంట్ టూల్స్ ఇంటర్ఫేస్ - టూల్స్ తో పరస్పరం జరపడానికి త్వరిత ఉదాహరణలు మరియు చాట్ ఇంటర్ఫేస్*

### సింపుల్ టూల్ వినియోగం ప్రయత్నించండి

"100 డిగ్రీల ఫారెన్ హీట్ ను సెల్సియస్ గా మారుస్తుంది" అనే సరళమైన అభ్యర్థనతో ప్రారంభించండి. ఏజెంట్ ఎలాంటి టూల్ అవసరమో గుర్తించి, సరైన పారామీటర్లతో టూల్ ను పిలుస్తుంది మరియు ఫలితాన్ని ఇవ్వుతుంది. ఇది ఎంత సహజంగా అనిపిస్తుందో గమనించండి - మీరు ఏ టూల్ ఉపయోగించాలో లేదా ఎలా పిలవాలో చెప్పలేదు.

### టూల్ చైన్ పరీక్షించండి

ఇప్పుడు మరింత క్లిష్టమైనదాన్ని ప్రయత్నించండి: "సియాటిల్ లో వాతావరణం ఏమిటి మరియు దాన్ని ఫారెన్ హీట్ కి మార్చండి?" ఏజెంట్ దాన్ని దశల వారీగా ఎలా పరిష్కరిస్తుందో చూడండి. ముందుగా వాతావరణ సమాచారం (సెల్సియస్ లో ఉంటుంది) తాగుతుంది, ఫారెన్ హీట్ కి మార్పు చేయాల్సిన అవసరం గుర్తించి, మార్పు టూల్ ను పిలుస్తుంది మరియు రెండు ఫలితాలను ఒక సమాధానంగా కలిపి ఇచ్చుతుంది.

### సంభాషణ ప్రవాహం చూడండి

చాట్ ఇంటర్ఫేస్ సంభాషణ చరిత్రను కొనసాగిస్తుంది, ఇది మీరు బహుళ-తిరుగుల పరస్పర చర్యలు చేయడానికి అనుమతిస్తుంది. మీరు అన్ని గత అభ్యర్థనలు మరియు ప్రతిస్పందనలను చూడవచ్చు, ఇది సంభాషణను ట్రాక్ చేయడాన్ని మరియు ఏజెంట్ ఎలా పలు మార్పులలో సందర్భాన్ని నిర్మిస్తున్నదో అర్థం చేసుకోవడాన్ని సులభం చేస్తుంది.

<a href="images/tools-conversation-demo.png"><img src="../../../translated_images/te/tools-conversation-demo.89f2ce9676080f59.webp" alt="Conversation with Multiple Tool Calls" width="800" style="border: 1px solid #ddd; box-shadow: 0 2px 8px rgba(0,0,0,0.1);"/></a>

*సరళమైన మార్పులు, వాతావరణ సమాచారం, మరియు టూల్ చేనింగ్ ను చూపే బహుళ-తిరుగు సంభాషణ*

### వేర్వేరు అభ్యర్థనలతో ప్రయోగం చేయండి

వివిధ కలయికలను ప్రయత్నించండి:
- వాతావరణ సమాచారాలు: "టోక్యోలో వాతావరణం ఏమిటి?"
- ఉష్ణోగ్రత మార్పులు: "25°C ని కెల్విన్ లో ఎంత?"
- సంయుక్త ప్రశ్నలు: "పారిస్ లో వాతావరణం చూస్తే అది 20°C కంటే ఎక్కువగా ఉందా చెప్తారా?"

ఏజెంట్ సహజ భాషను ఎలా అర్థం చేసుకుంటుందో మరియు సరైన టూల్ పిలుపులకు ఎలా మ్యాపు చేస్తుందో గమనించండి.

## ముఖ్య భావనలు

### ReAct నమూనా (తర్కం మరియు చర్య)

ఏజెంట్ తర్కం (ఏమి చేయాలో నిర్ణయించడం) మరియు చర్య (టూల్స్ ఉపయోగించడం) మధ్య మారుతూ ఉంటుంది. ఈ నమూనా, కేవలం సూచనలకు ప్రతిస్పందించడం కాకుండా స్వతంత్ర సమస్య పరిష్కారం సాధ్యమవుతుంది.

### టూల్ వివరణలు ముఖ్యం

మీ టూల్ వివరణల నాణ్యత ఏజెంట్ వాటిని ఎంత బాగా ఉపయోగించగలదో మీరుగానే ప్రభావితం చేస్తుంది. స్పష్టమైన, ప్రత్యేక వివరణలు మోడల్ కు ఏ టూల్ ను ఎప్పుడు మరియు ఎలా పిలవాలో అర్థం చేసుకోవడంలో సహాయపడతాయి.

### సెషన్ నిర్వహణ

`@MemoryId` అనోటేషన్ స్వయంచాలక సెషన్-ఆధారిత మేమొరీ నిర్వహణను అనుమతిస్తుంది. ప్రతి సెషన్ ID కు `ChatMemory` అంశం కలిగి ఉంటుంది, దీన్ని `ChatMemoryProvider` బీన్ నిర్వహిస్తుంది, కాబట్టి బహుళ మంది ఉపయోగకర్కులు ఏజెంట్ తో పరస్పర చర్యలు జరపడానికి వారి సంభాషణలు కలవకుండా ఉంటాయి. దిగువ చిత్రణ బహుళ మంది యూజర్లను వారి సెషన్ ID ల ఆధారంగా వేరుగా నిల్వ చేసిన సంభాషణ చరిత్రలకు ఎలా దారితీస్తుందో చూపిస్తుంది:

<img src="../../../translated_images/te/session-management.91ad819c6c89c400.webp" alt="Session Management with @MemoryId" width="800"/>

*ప్రతి సెషన్ ID వేరే సంభాషణ చరిత్రను సూచిస్తుంది — యూజర్లు ఒకరి సందేశాలు చూడరు.*

### లోపాలు నిర్వహణ

టూల్స్ తప్పులు ఇవ్వవచ్చు — APIs టైమ్ అవుట్ అవ్వచ్చు, పారామీటర్లు అందుబాటులో లేకపోవచ్చు, బయటి సేవలు డౌన్ కావచ్చు. ప్రొడక్షన్ ఏజెంట్లు లోపాలు నిర్వహణ అవసరం, దాంతో మోడల్ సమస్యల గురించి వివరించగలదు లేదా ప్రత్యామ్నాయాలు ప్రయత్నించగలదు, మొత్తం అప్లికేషన్ క్రాష్ కాకుండా. టూల్ లోపం చూపినపుడు, LangChain4j దానిని పట్టుకొని లోప సందేశాన్ని మోడల్ కు తిరిగి ఇస్తుంది, తద్వారా మోడల్ సహజ భాషలో సమస్య వివరించగలదు.

## అందుబాటులో ఉన్న టూల్స్

క్రింది చిత్రణ మీరు నిర్మించగల విస్తృత టూల్స్ పరిరక్షణను చూపిస్తుంది. ఈ మాడ్యూలు వాతావరణ మరియు ఉష్ణోగ్రత టూల్స్ ని చూపిస్తుంది, కానీ అదే `@Tool` నమూనా ఏ Java మెథడ్ కు వర్తిస్తుంది — డేటాబేస్ ప్రశ్నల నుండి చెల్లింపుల ప్రాసెసింగ్ వరకు.

<img src="../../../translated_images/te/tool-ecosystem.aad3d74eaa14a44f.webp" alt="Tool Ecosystem" width="800"/>

*@Tool తో అనోటేట్ చేసిన ఏ Java మెథడ్ ను AI కి అందుబాటులో ఉంచవచ్చు — ఈ నమూనా డేటాబేసులు, APIs, ఇమెయిల్, ఫైల్ ఆపరేషన్స్ మరియు మరిన్ని మరో విస్తరిస్తుంది.*

## టూల్ ఆధారిత ఏజెంట్లను ఎప్పుడు ఉపయోగించాలి

ప్రతి అభ్యర్థనకు టూల్స్ అవసరం ఉండదు. నిర్ణయం ఏ ఐ ఆధారంగా బయటి వ్యవస్థలతో పరస్పరం చేయాలా లేదా తన సొంత జ్ఞానం ఆధారంగా సమాధానం ఇవ్వగలదా అనేక ఒప్పందంపై ఆధారపడి ఉంటుంది. క్రింది మార్గదర్శకం ఎప్పుడు టూల్స్ విలువను చేర్పిస్తాయో ఎప్పుడు అవసరం లేదో సంక్షిప్తంగా చూపిస్తుంది:

<img src="../../../translated_images/te/when-to-use-tools.51d1592d9cbdae9c.webp" alt="When to Use Tools" width="800"/>

*త్వరిత నిర్ణయ మార్గదర్శకం — టూల్స్ ప్రత్యక్ష డేటా, గణనలు, చర్యలకు; సాధారణ జ్ఞానం మరియు సృజనాత్మక పనులు టూల్స్ అవసరం ఉండవు.*

## టూల్స్ vs RAG

మాడ్యూల్స్ 03 మరియు 04 రెండూ AI చేయగలదాన్ని విస్తరించే విధానం కానీ ప్రాథమికంగా వేరుగా ఉంటాయి. RAG డాక్యుమెంట్లను తెప్పించే ద్వారా **జ్ఞానం** అందిస్తుంది. టూల్స్ ఫంక్షన్లను పిలవడం ద్వారా **చర్యలు** చేపట్టగల సామర్థ్యం ఇస్తాయి. క్రింది చిత్రణ ఈ రెండు దృక్ఫణాలను పక్కకు పక్క ఉన్నట్లుగా చూపిస్తది — ప్రతి సమయంలో ఎలా వర్క్ ఫ్లో పనిచేస్తుందో, వాటి మధ్య వ్యత్యాసాలు ఏవో:

<img src="../../../translated_images/te/tools-vs-rag.ad55ce10d7e4da87.webp" alt="Tools vs RAG Comparison" width="800"/>

*RAG స్థిర డాక్యుమెంట్ల నుండి సమాచారాన్ని వెతుకుతుంది — టూల్స్ చర్యలను కార్యనిర్వహణ చేస్తాయి మరియు ప్రస్తుత, ప్రత్యక్ష డేటా తీసుకొస్తాయి. చాలామంది ప్రొడక్షన్ వ్యవస్థలు రెండింటినీ కలిపి ఉపయోగిస్తాయి.*

ప్రాక్టీస్లో, చాలామంది ప్రొడక్షన్ వ్యవస్థలు రెండు విధానాలను కలిపి ఉపయోగిస్తాయి: మీ డాక్యుమెంటేషన్ లో సమాధానాలను స్థాపించడానికి RAG, ప్రత్యక్ష డేటా తీసుకోవడం లేదా ఆపరేషన్స్ నిర్వహించడానికి టూల్స్.
