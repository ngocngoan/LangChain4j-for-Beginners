## మీరు నేర్చుకోనుందేమనేది

మీరు క్విక్ స్టార్ట్ పూర్తిచేశాక, మీరు ప్రకటనలు పంపటం మరియు స్పందనలు పొందటం ఎలా అనేది చూశారు. ఇది ఆధారం, కానీ నిజమైన అప్లికేషన్లు మరిన్ని అవసరం. ఈ మాడ్యూల్ మీరు కాంటెక్స్ట్ గుర్తుంచుకునే మరియు స్టేట్‌ను నిర్వహించే సంభాషణాత్మక AI ని ఎలా నిర్మించాలో నేర్పుతుంది - ఒకసారి చూపించే డెమో మరియు ప్రొడక్షన్-గత అప్లికేషన్ మధ్య తేడా.

మేము ఈ గైడ్ లో Azure OpenAI యొక్క GPT-5.2 ను ఉపయోగిస్తాము ఎందుకంటే దీని అధునాతన తర్క సామర్ధ్యాలు వివిధ నమూనాల ప్రవర్తనను స్పష్టంగా చూపిస్తాయి. మీరు మెమరీని జతచేసినప్పుడు, మీరు స్పష్టంగా తేడాను చూస్కుంటారు. ఇది ప్రతి భాగం మీ అప్లికేషన్‌కు ఏమి తీసుకువస్తుందో అర్థం చేసుకోవడాన్ని సులభం చేస్తుంది.

మీరు రెండు నమూనాలను ప్రదర్శించే ఒక అప్లికేషన్ నిర్మిస్తారు:

**స్టేట్లెస్ చాట్** - ప్రతి అభ్యర్థన స్వతంత్రం. మోడల్ కు గత సందేశాల గురించి మెమరీ లేదు. ఇది మీరు క్విక్ స్టార్ట్ లో ఉపయోగించిన నమూనా.

**స్టేట్‌ఫుల్ సంభాషణ** - ప్రతి అభ్యర్థనలో సంభాషణ చరిత్ర ఉంటుంది. మోడల్ పలు మార్లు కాంటెక్స్ట్‌ను నిర్వహిస్తుంది. ఇది ప్రొడక్షన్ అప్లికేషన్లకు అవసరం.

## ముందస్తు అర్హతలు

- Azure సబ్‌స్క్రిప్షన్ తో Azure OpenAI erişimi
- Java 21, Maven 3.9+ 
- Azure CLI (https://learn.microsoft.com/en-us/cli/azure/install-azure-cli)
- Azure Developer CLI (azd) (https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/install-azd)

> **గమనిక:** Java, Maven, Azure CLI మరియు Azure Developer CLI (azd) ప్రీ-ఇన్స్టాల్ చేయబడ్డాయి ప్రొవైడెడ్ డెవ్‌కంటైనర్‌లో.

> **గమనిక:** ఈ మాడ్యూల్ Azure OpenAI పై GPT-5.2 ను ఉపయోగిస్తుంది. డిప్లాయ్‌మెంట్ `azd up` ద్వారా ఆటోమేటిక్‌గా కాన్ఫిగర్ చేయబడింది - కోడ్‌లో మోడల్ పేరు మార్చవద్దు.

## మూల సమస్య అవగాహన

భాషా నమూనాలు స్టేట్లెస్. ప్రతి API కాల్ స్వతంత్రం. మీరు "నా పేరు జాన్" అని పంపితే మరియు "నా పేరు ఏమిటి?" అని అడగగానే, మోడల్ మీరు కొత్త పరిచయం చేసుకున్నట్టు తెలుసుకోదు. ఇది ప్రతి అభ్యర్థనను మీరు ఎప్పుడూ చేసిన మొదటి సంభాషణగా పరిగణిస్తుంది.

ఇది సాదా ప్రశ్నోత్తరాలకే సరైనది కానీ నిజమైన అప్లికేషన్లకు ఉపయోగపడదు. కస్టమర్ సర్వీస్ బాట్లు మీరు చెప్పిన విషయాన్ని గుర్తుంచుకోవాలి. వ్యక్తిగత సహాయకులు కాంటెక్స్ట్ అవసరం. ఎలాంటి బహుళ-దశ సంభాషణకు మెమరీ అవసరం.

<img src="../../../translated_images/te/stateless-vs-stateful.cc4a4765e649c41a.webp" alt="Stateless vs Stateful Conversations" width="800"/>

*స్టేట్లెస్ (స్వతంత్ర కాల్స్) మరియు స్టేట్‌ఫుల్ (కాంటెక్స్ట్-అవగాహనా) సంభాషణల మధ్య తేడా*

## టోక్సెన్ల అర్థం

సంభాషణలలోకి దిగేముందు, టోక్సెన్లను అర్థం చేసుకోవడం ముఖ్యం - భాషా నమూనాలు ప్రాసెస్ చేసే టెక్స్‌ట్ యొక్క మూలభాగాలు:

<img src="../../../translated_images/te/token-explanation.c39760d8ec650181.webp" alt="Token Explanation" width="800"/>

*ఎలా టెక్స్‌ట్ టోక్సెన్లుగా విభజించబడుతుందో ఉదాహరణ - "I love AI!" నాలుగు వేర్వేరు ప్రాసెసింగ్ యూనిట్లుగా మారుతుంది*

టోక్సెన్లు AI నమూనాలు టెక్స్‌ట్‌ను కొలవటానికి మరియు ప్రాసెస్ చేయటానికి ఉపయోగిస్తాయి. పదాలు, వాక్యచిహ్నాలు, మరియు ఖాళీలు కూడా టోక్సెన్లు కావచ్చు. మీ మోడల్ కి ఒకేసారి ప్రాసెస్ చేయగల టోక్సెన్ల పరిమితి ఉంటుంది (GPT-5.2 కి 400,000, అందులో 272,000 ఇన్‌పుట్ టోక్సెన్లు మరియు 128,000 అవుట్‌పుట్ టోక్సెన్లు). టోక్సెన్ల అర్థం చేసుకోవడం ద్వారా సంభాషణ పొడవు మరియు ఖర్చులు నియంత్రించవచ్చు.

## మెమరీ ఎలా పనిచేస్తుంది

చాట్ మెమరీ స్టేట్లెస్ సమస్యను పరిష్కరిస్తుంది సంభాషణ చరిత్రను నిర్వహించడం ద్వారా. మీరు అభ్యర్థనని మోడల్‌కు పంపించే ముందు, ఫ్రేమ్‌వర్క్ సంబంధిత గత సందేశాలను ముందుముట్టిస్తుంది. మీరు "నా పేరు ఏమిటి?" అని అడిగితే, వ్యవస్థ అసలు మొత్తం సంభాషణ చరితరాన్ని పంపుతుంది, అందుచేత మోడల్ మీరింత ముందుగా "నా పేరు జాన్" అని చెప్పారని చూస్తుంది.

LangChain4j మెమరీ అమలులను ఆపోమేటిగ్గా నిర్వహిస్తుంది. మీరు ఎంతమంది సందేశాల్ని నిల్వ చేసుకోవాలో ఎంచుకోవచ్చు మరియు ఫ్రేమ్‌వర్క్ కాంటెక్స్ట్ విండోను నిర్వహిస్తుంది.

<img src="../../../translated_images/te/memory-window.bbe67f597eadabb3.webp" alt="Memory Window Concept" width="800"/>

*MessageWindowChatMemory తాజా సందేశాల యొక్క స్లైడింగ్ విండోని నిర్వహిస్తుంది, పాతవాటిని ఆటోమేటిగ్గా తొలగిస్తుంది*

## ఇది LangChain4j ను ఎలా ఉపయోగిస్తుందో

ఈ మాడ్యూల్ క్విక్ స్టార్ట్ ను Spring Boot తో కలిపి సంబంధిత సంభాషణ మెమరీని జతచేస్తుంది. భాగాలు ఇలా చేరతాయి:

**అవలంబనలు** - రెండు LangChain4j లైబ్రరీలు జోడించండి:

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```

**చాట్ మోడల్** - Azure OpenAI ని Spring బీన్ గా కాన్ఫిగర్ చేయండి ([LangChainConfig.java](../../../01-introduction/src/main/java/com/example/langchain4j/config/LangChainConfig.java)):

```java
@Bean
public OpenAiOfficialChatModel openAiOfficialChatModel() {
    return OpenAiOfficialChatModel.builder()
            .baseUrl(azureEndpoint)
            .apiKey(azureApiKey)
            .modelName(deploymentName)
            .timeout(Duration.ofMinutes(5))
            .maxRetries(3)
            .build();
}
```

బిల్డర్ క్రెడెంటియల్స్ ను `azd up` ద్వారా సెట్ చేసిన పర్యావరణ చరాల నుండి చదువుతుంది. `baseUrl` ను మీ Azure ఎండ్‌పాయింట్‌గా సెట్ చేయడం Azure OpenAI క్లయింట్ ని Azure OpenAIతో పని చేయడానికి చేస్తుంది.

**సంభాషణ మెమరీ** - MessageWindowChatMemory తో చాట్ చరిత్రను ట్రాక్ చేయండి ([ConversationService.java](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java)):

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("My name is John"));
memory.add(AiMessage.from("Nice to meet you, John!"));

memory.add(UserMessage.from("What's my name?"));
AiMessage aiMessage = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage);
```

`withMaxMessages(10)` తో మెమరీ సృష్టించి చివరి 10 సందేశాలను నిల్వ చేయండి. Typed wrappers తో యూజర్ మరియు AI సందేశాలను జోడించండి: `UserMessage.from(text)` మరియు `AiMessage.from(text)`. చరిత్రను `memory.messages()` తో పునఃప్రాప్తి చేసి మోడల్‌కు పంపండి. సర్వీస్ సంభాషణ ID ప్రకారం వేరు మెమరీ ఇన్స్టెన్సులు నిల్వ చేస్తుంది, ఇది బహుళ యూజర్లు ఒకేసారి చాట్ చేసుకోవడానికి అనుమతిస్తుంది.

> **🤖 [GitHub Copilot](https://github.com/features/copilot) Chat తో ప్రయత్నించండి:** [`ConversationService.java`](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java) తెరిచి అడగండి:
> - "MessageWindowChatMemory విండో పూర్తి అయినప్పుడు messages ను ఎలాంటి ముసిగించి మోసుకుంటుంది?"
> - "నేను in-memory కాకుండా డేటాబేస్ ఉపయోగించి కస్టమ్ మెమరీ స్టోరేజ్ ఎలా అమలు చేయగలను?"
> - "పాత సంభాషణ చరిత్రని సంక్షిప్తం చేయటానికి సారాంశం ఎలా జత చేస్తాను?"

స్టేట్లెస్ చాట్ ఎండ్‌పాయింట్ మెమరీని పూర్తిగా మినహాయిస్తుంది - కేవలం `chatModel.chat(prompt)` క్విక్ స్టార్ట్ లాంటి విధంగా ఉంటుంది. స్టేట్‌ఫుల్ ఎండ్‌పాయింట్ సందేశాలను మెమరీలో చేరుస్తుంది, చరిత్రను పునఃప్రాప్తి చేస్తుంది, మరియు ప్రతీ అభ్యర్థనతో ఆ కాంటెక్స్ట్‌ను సహా పంపుతుంది. సమాన మోడల్ కాన్ఫిగరేషన్, వేరు నమూనాలు.

```powershell
azd up  # సభ్యత్వం మరియు స్థానం (eastus2 సిఫారసు చేయబడింది) ఎంచుకోండి
```

> **గమనిక:** మీరు టైమౌట్ లోపం ఎదుర్కుంటే (`RequestConflict: Cannot modify resource ... provisioning state is not terminal`), సింప్లీ `azd up` ను మళ్లీ నడపండి. Azure వనరులు ఇప్పటికీ బ్యాక్గ్రౌండ్ లో ప్రొవైజనింగ్ జరుగుతుంటాయి, రెండవ సారి ప్రయత్నించడం వనరులు టర్మినల్ స్థితిలోకి వచ్చాక డిప్లాయ్‌మెంట్ పూర్తి అవుతుంది.

ఇది:
1. GPT-5.2 మరియు text-embedding-3-small మోడల్స్‌తో Azure OpenAI వనరును డిప్లాయ్ చేస్తుంది
2. ప్రాజెక్ట్ రూట్‌లో క్రెడెన్షియల్స్‌తో ఆటోమేటిక్‌గా `.env` ఫైల్ ఉత్పత్తి చేస్తుంది
3. అన్ని అవసరమైన పర్యావరణ చరాలను సెట్ చేస్తుంది

**డిప్లాయ్‌మెంట్ సమస్యలున్నా?** [Infrastructure README](infra/README.md) చూసి సబ్‌డొమైన్ పేరు ఘర్షణలు, Azure పోర్టల్ లో మాన్యువల్ డిప్లాయ్‌మెంట్ దశలు, మోడల్ కాన్ఫిగరేషన్ మార్గదర్శకాలను తెలుసుకోండి.

**డిప్లాయ్‌మెంట్ విజయవంతమైందని ధృవీకరించండి:**

**Bash:**
```bash
cat ../.env  # AZURE_OPENAI_ENDPOINT, API_KEY, మరియు ఇతర వివరాలు చూపించాలి.
```

**PowerShell:**
```powershell
Get-Content ..\.env  # AZURE_OPENAI_ENDPOINT, API_KEY, మొదలగునవి చూపించాలి.
```

> **గమనిక:** `azd up` ఆదేశం ఆటోమేటిక్‌గా `.env` ఫైల్‌ని సృష్టిస్తుంది. మీరు దానిని తర్వాత అప్‌డేట్ చేయాలనుకుంటే, జాగ్రత్తగా `.env` ఫైల్‌ను ఎడిట్ చేయండి లేదా తిరిగి ఉత్పత్తి చేయండి:
>
> **Bash:**
> ```bash
> cd ..
> bash .azd-env.sh
> ```
>
> **PowerShell:**
> ```powershell
> cd ..
> .\.azd-env.ps1
> ```

## అప్లికేషన్‌ను స్థానికంగా నడపడం

**డిప్లాయ్‌మెంట్ ధృవీకరణ:**

Azure క్రెడెన్షియల్స్‌తో `.env` ఫైల్ రూట్ డైరెక్టరీలో ఉందో తనిఖీ చేయండి:

**Bash:**
```bash
cat ../.env  # AZURE_OPENAI_ENDPOINT, API_KEY, DEPLOYMENT చూపించాలి
```

**PowerShell:**
```powershell
Get-Content ..\.env  # AZURE_OPENAI_ENDPOINT, API_KEY, DEPLOYMENT చూపించాలి
```

**అప్లికేషన్లను ప్రారంభించండి:**

**ఆప్షన్ 1: Spring Boot డాష్‌బోర్డ్ ఉపయోగించడం (VS కోడ్ వినియోగదారులకు సిఫార్సు)**

డెవ్ కంటైనర్ Spring Boot డాష్‌బోర్డ్ విస్తరణతో వచ్చింది, ఇది అన్ని Spring Boot అప్లికేషన్లను నిర్వహించేందుకు విజువల్ ఇన్టర్ఫేస్ ఇస్తుంది. మీరు VS కోడ్ లో ఎడమవైపు యాక్టివిటీ బార్ లో Spring Boot ఐకాన్ చూడగలరు.

Spring Boot డాష్‌బోర్డ్ నుంచి మీరు:
- వర్క్‌స్పేస్ లో అందుబాటులో ఉన్న Spring Boot అప్లికేషన్లను చూడవచ్చు
- ఒక క్లిక్తో అప్లికేషన్లు ప్రారంభించండి/ఆగనివ్వండి
- అప్లికేషన్ లాగ్స్ ప్రత్యక్షంగా చూడండి
- అప్లికేషన్ స్థితిని పర్యవేక్షించండి

కేవలం "introduction" పక్కన ఉన్న ప్లే బటన్‌ను క్లిక్ చేసి ఈ మాడ్యూల్‌ను ప్రారంభించండి లేకపోతే అన్నీ ఒకేసారి ప్రారంభించండి.

<img src="../../../translated_images/te/dashboard.69c7479aef09ff6b.webp" alt="Spring Boot Dashboard" width="400"/>

**Bash:**

## అప్లికేషన్ ఉపయోగించడం

ఈ అప్లికేషన్ ఈ రెండు చాట్ అమలుచేసే వెబ్ ఇంటర్ఫేస్‌ను పక్కపక్కనే అందిస్తుంది.

<img src="../../../translated_images/te/home-screen.121a03206ab910c0.webp" alt="Application Home Screen" width="800"/>

*స్టేట్లెస్ సింపుల్ చాట్ మరియు స్టేట్‌ఫుల్ సంభాషణ చాట్ ఎంపికలతో డాష్‌బోర్డ్ ని చూపిస్తుంది*

### స్టేట్లెస్ చాట్ (ఎడమ ప్యానెల్)

ముందుగా దీన్ని ప్రయత్నించండి. "నా పేరు జాన్" అని అడగండి మరియు వెంటనే "నా పేరు ఏమిటి?" అని అడగండి. మోడల్ గుర్తుంచుకోదు ఎందుకంటే ప్రతీ సందేశం స్వతంత్రం. ఇది ప్రాథమిక భాషా నమూనా ఇంటిగ్రేషన్ లో మూల సమస్యను చూపిస్తుంది - సంభాషణ కాంటెక్స్ట్ లేదు.

<img src="../../../translated_images/te/simple-chat-stateless-demo.13aeb3978eab3234.webp" alt="Stateless Chat Demo" width="800"/>

*AI మీ పేరును గత సందేశం నుండి గుర్తుంచుకోదు*

### స్టేట్‌ఫుల్ చాట్ (కుడి ప్యానెల్)

ఇప్పటి అదే వరసను ఇక్కడ ప్రయత్నించండి. "నా పేరు జాన్" అని అడగండి మరియు తరువాత "నా పేరు ఏమిటి?" ఇది ఈసారి గుర్తుంచుకుంటుంది. తేడా MessageWindowChatMemory అందులో ఉంది - ఇది సంభాషణ చరిత్రను నిర్వహించి ప్రతీ అభ్యర్థనతో అది సహా పంపుతుంది. ఇదే ప్రొడక్షన్ సంభాషణాత్మక AI పనిచేసే విధానం.

<img src="../../../translated_images/te/conversational-chat-stateful-demo.e5be9822eb23ff59.webp" alt="Stateful Chat Demo" width="800"/>

*AI మీరు ముందుగా చెప్పిన పేరును గుర్తు చేసుకుంటుంది*

రెండు ప్యానెల్స్ ఒకే GPT-5.2 మోడల్ ఉపయోగిస్తాయి. ఒక్కటిమాత్రం మెమరీ తేడా. ఇది మెమరీ మీ అప్లికేషన్‌కు ఏమి తెస్తుందో, మరియు అసలు వాడుకలకు అది ఎందుకు అవసరమో స్పష్టంగా చూపిస్తుంది.
