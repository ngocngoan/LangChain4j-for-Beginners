## మీరు నేర్చుకునే విషయాలు

మీరు వెంటనే ప్రారంభం పూర్తి చేసినట్లయితే, మీరు ప్రాంప్ట్‌లు పంపడం మరియు ప్రతిస్పందనలు పొందడం ఎలా ఉందో చూశారు. అదే చట్టం, కానీ నిజమైన అప్లికేషన్లు ఎక్కువ కావాలి. ఈ మాడ్యూల్ మీకు సంభాషణ AIని ఎలా నిర్మించాలో నేర్పుతుంది, ఇది సందర్భాన్ని గుర్తుంచుకుని స్టేట్‌ను నిర్వహిస్తుంది - ఒకసారి ప్రదర్శించేది మరియు ఉత్పత్తి-సిద్ధమైన అప్లికేషన్ మధ్య వ్యత్యాసం.

మేము ఈ గైడ్ లో Azure OpenAI యొక్క GPT-5.2 ని ఉపయోగిస్తాం ఎందుకంటే దాని అభివృద్ధిగల తర్క సామర్థ్యాలు వివిధ నమూనాల ప్రవర్తనని స్పష్టంగా చూపిస్తాయి. మీరు మెమోరీని చేర్చినప్పుడు, మీరు తేడా ని స్పష్టంగా చూస్తారు. ఇది ప్రతీ భాగం మీ అప్లికేషన్‌కు ఏమి తీసుకువస్తుందో అర్థం చేసుకోవడాన్ని సులభం చేస్తుంది.

మీరు రెండు నమూనాలను చూపించే ఒక అప్లికేషన్‌ను నిర్మిస్తారు:

**స్టేట్‌లెస్ చాట్** - ప్రతి అనుకోకుండా వేరుగా ఉంటుంది. మోడల్ పూర్వ సందేశాలపై ఎలాంటి జ్ఞాపకం పెట్టుకోదు. ఇది మీరు వెంటనే ప్రారంభంలో ఉపయోగించిన నమూనా.

**స్టేట్‌ఫుల్ సంభాషణ** - ప్రతి అభ్యర్థన సంభాషణ చరిత్రను కలిగి ఉంటుంది. మోడల్ అనేక మార్లు సందర్భాన్ని నిర్వహిస్తుంది. ఇది ఉత్పత్తి అప్లికేషన్లకు అవసరం.

## ముందస్తు అవసరాలు

- Azure OpenAI యాక్సెస్‌తో Azure సబ్‌స్క్రిప్షన్
- Java 21, Maven 3.9+
- Azure CLI (https://learn.microsoft.com/en-us/cli/azure/install-azure-cli)
- Azure Developer CLI (azd) (https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/install-azd)

> **గమనిక:** Java, Maven, Azure CLI మరియు Azure Developer CLI (azd) ప్రోగ్రామ్ డెవ్ కంటైనర్‌లో ముందుగా ఇన్‌స్టాల్ అవ్వబడ్డాయి.

> **గమనిక:** ఈ మాడ్యూల్ Azure OpenAI పై GPT-5.2 ను ఉపయోగిస్తుంది. `azd up` ద్వారా డిప్లాయ్‌మెంట్ ఆటోమేటిక్‌గా ఉంటుంది - కోడ్‌లో మోడల్ పేరుని మార్చవద్దు.

## మూల సమస్యకు అవగాహన

భాషా నమూనాలు స్టేట్‌లెస్. ప్రతి API పిలుపు స్వతంత్రం. మీరు "నా పేరు జాన్" అని పంపించి "నా పేరు ఏమిటి?" అని అడగగా, మోడల్ మీరు స్వయంగా పరిచయం చేసుకున్నారని తెలియదు. ఇది ప్రతి అభ్యర్థనను మొదటి సంభాషణగా చూస్తుంది.

సరళమైన ప్రశ్న-జవాబు కోసం ఇది సరైనది కానీ నిజమైన అప్లికేషన్లకు పనిష్ఠ కాదు. కస్టమర్ సర్వీస్ బాట్లు మీరు చెప్పినదాన్ని గుర్తు పెట్టుకోవాలి. వ్యక్తిగత సహాయం స్థితిని అర్థం చేసుకోవాలి. ఏ multi-turn సంభాషణకు మెమోరీ అవసరం.

<img src="../../../translated_images/te/stateless-vs-stateful.cc4a4765e649c41a.webp" alt="Stateless vs Stateful Conversations" width="800"/>

*స్టేట్‌లెస్ (స్వతంత్ర పిలుపులు) మరియు స్టేట్‌ఫుల్ (సందర్భం తెలుసుకునే) సంభాషణల మధ్య తేడా*

## టోకెన్ల అర్థం

సంభాషణలలోకి లోతుగా వెళ్ళేముందు, భాషా నమూనాలు ప్రక్రియించే టెక్స్ట్ యొక్క ప్రాథమిక యూనిట్లు అయిన టోకెన్లను అర్థం చేసుకోవడం ముఖ్యం:

<img src="../../../translated_images/te/token-explanation.c39760d8ec650181.webp" alt="Token Explanation" width="800"/>

*టెక్స్ట్ ఎలా టోకెన్లలో విడిపిస్తుంది - "I love AI!" 4 వేర్వేరు ప్రాసెసింగ్ యూనిట్లుగా మారుతుంది*

టోకెన్లు AI మోడళ్లు టెక్స్ట్‌ను ఎలా కొలుస్తాయో మరియు ప్రాసెస్ చేస్తాయో సూచిస్తాయి. మాటలు, పంక్చువేషన్, స్పేసులు కూడా టోకెన్లు కావచ్చు. మీ మోడల్ ఒకసారిగా ఎంత టోకెన్లు ప్రాసెస్ చేయగలదో పరిమితి (GPT-5.2 కోసం 400,000 టోకెన్లు, 272,000 ఇన్‌పుట్ టోకెన్లు మరియు 128,000 అవుట్‌పుట్ టోకెన్లు). టోకెన్ల అర్థం సంభాషణ పొడవు మరియు ఖర్చులను నిర్వహించడంలో సహాయపడుతుంది.

## మెమోరీ ఎలా పనిచేస్తుంది

చాట్ మెమోరీ స్టేట్‌లెస్ సమస్యని సంభాషణ చరిత్రను నిర్వహించడం ద్వారా పరిష్కరిస్తుంది. మీరు అభ్యర్థనను మోడల్‌కు పంపేముందు, ఫ్రేమ్‌వర్క్ సంబంధిత పూర్వ సందేశాలను ముందుగా జతచేస్తుంది. మీరు "నా పేరు ఏమిటి?" అని అడిగినప్పుడు, వ్యవస్థ మొత్తం సంభాషణ చరిత్రను పంపుతుంది, మోడల్ మీరు ముందు "నా పేరు జాన్" అన్నారని అర్థం చేసుకుంటుంది.

LangChain4j మెమోరీ అమలు విధానాలను ఆటోమేటిక్‌గా అందిస్తుంది. మీరు ఎంత సంకెళ్లు నిర్ధారించాలో ఎంచుకోవచ్చు మరియు ఫ్రేమ్‌వర్క్ సరిగా సందర్భం విండోను నిర్వహిస్తుంది.

<img src="../../../translated_images/te/memory-window.bbe67f597eadabb3.webp" alt="Memory Window Concept" width="800"/>

*MessageWindowChatMemory సమీప సందేశాల స్లైడింగ్ విండోను నిర్వహించి పాత సందేశాలను ఆటోమేటిక్‌గా తొలగిస్తుంది*

## ఇది LangChain4j ను ఎలా ఉపయోగిస్తుంది

ఈ మాడ్యూల్ వెంటనే ప్రారంభంకి Spring Bootను ఎంబెడ్డ్ చేసి సంభాషణ మెమోరీని చేర్చి ఉంటుంది. భాగాల అనుసంధానం ఇలా ఉంటుంది:

**ఆధారాలు** - రెండు LangChain4j లైబ్రరీలను జతచేయండి:

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```

**చాట్ మోడల్** - Azure OpenAI ని Spring బీన్‌గా కాన్ఫిగర్ చేయండి ([LangChainConfig.java](../../../01-introduction/src/main/java/com/example/langchain4j/config/LangChainConfig.java)):

```java
@Bean
public OpenAiOfficialChatModel openAiOfficialChatModel() {
    return OpenAiOfficialChatModel.builder()
            .baseUrl(azureEndpoint)
            .apiKey(azureApiKey)
            .modelName(deploymentName)
            .timeout(Duration.ofMinutes(5))
            .maxRetries(3)
            .build();
}
```

బిల్డర్ `azd up` ద్వారా సెట్ చేయబడిన వాతావరణ వేరియబుల్స్ నుండి క్రెడెన్షియల్స్ చదువుతుంది. `baseUrl` ను మీ Azure ఎండ్‌పాయింట్‌గా సెట్ చేస్తే OpenAI క్లయింట్ Azure OpenAI తో పనిచేస్తుంది.

**సంభాషణ మెమోరీ** - MessageWindowChatMemoryతో చాట్ చరిత్రను ట్రాక్ చేయండి ([ConversationService.java](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java)):

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("My name is John"));
memory.add(AiMessage.from("Nice to meet you, John!"));

memory.add(UserMessage.from("What's my name?"));
AiMessage aiMessage = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage);
```

క్రొత్త మెమోరీని `withMaxMessages(10)` తో సృష్టించండి, ఇది చివరి 10 సందేశాలను ఉంచుతుంది. Typed రాప్‌ర్లతో యూజర్ మరియు AI సందేశాలను జత చేయండి: `UserMessage.from(text)`, `AiMessage.from(text)`. చరిత్రను `memory.messages()` తో తీసుకుని మోడల్‌కు పంపండి. సర్వీస్ ప్రతీ సంభాషణ IDకి వేరే మెమోరీ ఇన్స్టెన్స్‌లను నిల్వ చేయడం వల్ల అనేక యూజర్లు సమాంతరంగా చాట్ చేయవచ్చు.

> **🤖 [GitHub Copilot](https://github.com/features/copilot) చాట్‌తో పరీక్షించండి:** [`ConversationService.java`](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java) తెరవండి మరియు అడగండి:
> - "MessageWindowChatMemory విండో నిండినప్పుడు ఏ సందేశాలను తొలగించాలో ఎలా నిర్ణయిస్తుంది?"
> - "నేను మెమోరీ స్టోరేజ్‌ని ఇన్-మెమోరీ కాకుండా డేటాబేస్ ఉపయోగించి ఎలా అమలు చేయవచ్చు?"
> - "పాత సంభాషణ చరిత్రను సంకీర్ణం చేయడానికి సారాంశం (summarization) ఎలా చేర్చాలి?"

స్టేట్‌లెస్ చాట్ ఎండ్‌పాయింట్ మెమోరీని పూర్తిగా వదిలివేస్తుంది - కేవలం `chatModel.chat(prompt)` వంటిదే, వెంటనే ప్రారంభంలో ఉపయోగించినట్లుగా. స్టేట్‌ఫుల్ ఎండ్‌పాయింట్ సందేశాలను మెమోరీకి చేర్చుతుంది, చరిత్ర తీసుకుంటుంది మరియు ఆ సందర్భాన్ని ప్రతి అభ్యర్థనతో జత చేయును. అదే మోడల్ కాన్ఫిగరేషన్, వేరువేరు నమూనాలు.

```powershell
azd up  # సభ్యత్వం మరియు స్థానం ఎంచుకోండి (eastus2 సిఫార్సు చేయబడింది)
```

> **గమనిక:** మీరు టైమ్‌ఔట్ లోపం ఎదురైతే (`RequestConflict: Cannot modify resource ... provisioning state is not terminal`), `azd up`ని మళ్లీ రన్ చేయండి. Azure వనరులు బ్యాక్‌గ్రౌండ్‌లో ఇంకా అమర్చబడుతున్నాయి కనుక, వనరులు టర్మినల్ స్థితికి చేరినప్పుడు పునఃచేసుకోవడం డిప్లాయ్‌మెంట్‌ను పూర్తిచేస్తుంది.

ఇది:
1. GPT-5.2 మరియు text-embedding-3-small మోడళ్లతో Azure OpenAI వనరును డిప్లాయ్ చేస్తుంది
2. ప్రాజెక్టు రూట్‌లో క్రెడెన్షియల్స్‌తో `.env` ఫైల్ ఆటోమేటిక్‌గా సృష్టిస్తుంది
3. అవసరమైన వాతావరణ వేరియబుల్స్ అందిస్తుంది

**డిప్లాయ్‌మెంట్ సమస్యలు ఉంటే?** [Infrastructure README](infra/README.md) లో సబ్‌డొమైన్ పేరు సంభందించిన సమస్యలు, Azure పోర్టల్ ద్వారా మాన్యువల్ డిప్లాయ్‌మెంట్ దశలూ, మోడల్ కాన్ఫిగరేషన్ సూచనలు చదవండి.

**డిప్లాయ్‌మెంట్ విజయవంతమైంది కాబట్టి నిర్ధారించండి:**

**Bash:**
```bash
cat ../.env  # AZURE_OPENAI_ENDPOINT, API_KEY, మొదలైనవి చూపించాలి.
```

**PowerShell:**
```powershell
Get-Content ..\.env  # AZURE_OPENAI_ENDPOINT, API_KEY మొదలైనవి చూపించాలి.
```

> **గమనిక:** `azd up` ఆదేశం ఆటోమేటిక్‌గా `.env` ఫైల్‌ను సృష్టిస్తుంది. తరువాత దీన్ని అప్‌డేట్ చేయాలంటే, మీరు `.env` ఫైల్‌ని మానువల్‌గా సవరించవచ్చు లేదా క్రింది విధంగా తిరిగి ఉత్పత్తి చేయవచ్చు:
>
> **Bash:**
> ```bash

> ```
>
> **PowerShell:**
> ```powershell

## అప్లికేషన్‌ను లోకల్‌గా నడపడం

**డిప్లాయ్‌మెంట్ నిర్ధారించండి:**

Azure క్రెడెన్షియల్స్ తో రూట్ డైరెక్టరీలో `.env` ఫైల్ ఉందో చూసుకోండి:

**Bash:**
```bash

```

**PowerShell:**
```powershell

```

**అప్లికేషన్లను మొదలు పెట్టండి:**

**వికల్పం 1: Spring Boot Dashboard ఉపయోగించడం (VS Code వినియోగదారులకు సిఫార్సు)**

డెవ్ కంటైనర్‌లో Spring Boot డాష్‌బోర్డు విస్తరణ ప్రస్తుతం ఉంటుంది, ఇది అన్ని Spring Boot అప్లికేషన్లను ఒక దృశ్య ఇంటర్‌ఫేస్‌లో నిర్వహిస్తుంది. మీరు దీనిని VS Code ఎడమ వైపు Activity Barలో Spring Boot ఐకాన్ ద్వారా కనుగొనవచ్చు.

Spring Boot డాష్‌బోర్డులో, మీరు:
- వర్క్‌స్పేస్‌లో ప్రతి Spring Boot అప్లికేషన్‌ను చూడవచ్చు
- ఒక క్లిక్‌తో అప్లికేషన్లను ప్రారంభించండి/నిలిపివేయండి
- అప్లికేషన్ లాగ్‌లను రియల్ఫైంలో చూడండి
- అప్లికేషన్ స్థితిని పర్యవేక్షించండి

సంప్రదింపుగా "introduction" పక్కన ఉన్న ప్లే బటన్ నొక్కి ఈ మాడ్యూల్‌ను ఆరంభించండి, లేదా అన్ని మాడ్యూల్స్‌ను ఒక్కేసారి ప్రారంభించండి.

<img src="../../../translated_images/te/dashboard.69c7479aef09ff6b.webp" alt="Spring Boot Dashboard" width="400"/>

## అప్లికేషన్ ఉపయోగించడం

అప్లికేషన్ రెండు చాట్ అమలులను పార్శ్వం పక్కన చూపించే వెబ్ ఇంటర్‌ఫేస్ అందిస్తుంది.

<img src="../../../translated_images/te/home-screen.121a03206ab910c0.webp" alt="Application Home Screen" width="800"/>

*సాధారణ చాట్ (స్టేట్‌లెస్) మరియు సంభాషణ చాట్ (స్టేట్‌ఫుల్) ఎంపికలను చూపించే డాష్‌బోర్డు*

### స్టేట్‌లెస్ చాట్ (ఎడమ ప్యానెల్)

ముందుగా దీన్ని ప్రయత్నించండి. మీరు "నా పేరు జాన్" అని అడిగి వెంటనే "నా పేరు ఏమిటి?" అని అడగండి. మోడల్ జ్ఞాపకాన్ని ఉంచదు ఎందుకంటే ప్రతి సందేశం స్వతంత్రం. ఇది ప్రాథమిక భాషా మోడల్ ఇంటిగ్రేషన్‌కి ప్రధాన సమస్యను చూపిస్తుంది - సంభాషణ సందర్భం లేదు.

<img src="../../../translated_images/te/simple-chat-stateless-demo.13aeb3978eab3234.webp" alt="Stateless Chat Demo" width="800"/>

*AI ముందు సందేశం నుండి మీ పేరును గుర్తించదు*

### స్టేట్‌ఫుల్ చాట్ (కుడి ప్యానెల్)

ఇప్పుడీ అదే సీక్వెన్స్ ఇక్కడ ప్రయత్నించండి. మీరు "నా పేరు జాన్" అని అడిగి "నా పేరు ఏమిటి?" అని అడిగినప్పుడు ఇది గుర్తుపెట్టుకుంటుంది. తేడా MessageWindowChatMemory - ఇది సంభాషణ చరిత్రను నిర్వహించి ప్రతి అభ్యర్థనతో ఆ సమాచారాన్ని జతచేస్తుంది. ఇది ఉత్పత్తి సంభాషణ AI ఎలా పనిచేస్తుందో చూపిస్తుంది.

<img src="../../../translated_images/te/conversational-chat-stateful-demo.e5be9822eb23ff59.webp" alt="Stateful Chat Demo" width="800"/>

*AI సంభాషణ ప్రారంభంలో మీ పేరును గుర్తించుకుంటుంది*

రెండు ప్యానెల్స్ ఒకే GPT-5.2 మోడల్‌ను ఉపయోగిస్తాయి. ఒకటిన్నర తేడా మాత్రమే మెమోరీ. ఇది మెమోరీ మీ అప్లికేషన్‌కు ఏమి తీసుకువస్తుందో, నిజ జీవిత వినియోగాల్లో ఎందుకు అవసరమో స్పష్టం చేస్తుంది.
