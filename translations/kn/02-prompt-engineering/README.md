## ನೀವು ಏನು ಕಲಿಯುವಿರಿ

ಹಿಂದಿನ ಮಡ್ಯೂಲ್‌ನಲ್ಲಿ, ನೀವು ಮೆಮೊರಿ ಸಂಭಾಷಣಾತ್ಮಕ AIಗೆ ಹೇಗೆ ಸಹಾಯಮಾಡುತ್ತದೆ ಎಂಬುದನ್ನು ಕಂಡಿದ್ದೀರಿ ಮತ್ತು GitHub ಮಾದರಿಗಳನ್ನು ಮೂಲ ಸಂವಹನಗಳಿಗಾಗಿ ಬಳಸಿದ್ದೀರಿ. ಈಗ ನಾವು ಪ್ರಶ್ನೆಗಳನ್ನು ಹೇಗೆ ಕೇಳುತ್ತೀರಿ ಎಂಬುದರ ಮೇಲೆ ಗಮನಹರಿಸುತ್ತೇವೆ - ಪ್ರಾಂಪ್ಟ್‌ಗಳೇ - Azure OpenAI ಯ GPT-5.2 ಉಪಯೋಗಿಸುವ ಮೂಲಕ. ನೀವು ನಿಮ್ಮ ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ಹೇಗೆ ರಚಿಸುತ್ತೀರಿ ಎಂಬುದು ನೀವು ಪಡೆಯುವ ಉತ್ತರಗಳ ಗುಣಮಟ್ಟವನ್ನು ಮಹತ್ವಪೂರ್ಣವಾಗಿ ಪ್ರಭಾವಿಸುತ್ತದೆ.

ನಾವು GPT-5.2 ಅನ್ನು ಬಳಸುತ್ತೇವೆ ಏಕೆಂದರೆ ಇದು ತರ್ಕ ನಿಯಂತ್ರಣವನ್ನು ಪರಿಚಯಿಸುತ್ತದೆ - ನೀವು ಮಾದರಿಯನ್ನು ಉತ್ತರಿಸುವ ಮೊದಲು ಎಷ್ಟು ಚಿಂತಿಸುವುದೆಂದು ಹೇಳಬಹುದು. ಇದು ವಿಭಿನ್ನ ಪ್ರಾಂಪ್ಟ್ ತಂತ್ರಗಳನ್ನು ಸ್ಪಷ್ಟವಾಗಿಸುತ್ತದೆ ಮತ್ತು ನೀವು ಯಾವ ವಿಧಾನವನ್ನು ಯಾವಾಗ ಉಪಯೋಗಿಸುವುದು ಎಂಬುದನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು ಸಹಾಯಮಾಡುತ್ತದೆ. GitHub ಮಾದರಿಗಳಿಗಿಂತ Azure ಗೆ GPT-5.2 ನಲ್ಲಿ ಕಡಿಮೆ ದರ ನಿರ್ಬಂಧಗಳು ಸಹ ನಮಗೆ ಲಭಿಸುತ್ತವೆ.

## ಪ್ರಾಂಪ್ಟ್ ಇಂಜಿನಿಯರಿಂಗ್ ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು

ಪ್ರಾಂಪ್ಟ್ ಇಂಜಿನಿಯರಿಂಗ್ ಎಂದರೆ ನೀವು ಬೇಕಾದ ಫಲಿತಾಂಶಗಳನ್ನು ಸ್ಥಿರವಾಗಿ ಪಡೆಯಲು ಇನ್‌ಪುಟ್ ಪಠ್ಯವನ್ನು ವಿನ್ಯಾಸಗೊಳಿಸುವುದು. ಇದು ಕೇವಲ ಪ್ರಶ್ನೆಗಳನ್ನು ಕೇಳುವುದಲ್ಲ - ಇದು ನಿಮ್ಮ ಬೇಡಿಕೆಯನ್ನು ರಚಿಸುವುದು, ಆದ್ದರಿಂದ ಮಾದರಿ ನೀವು ಏನು ಬೇಕು ಮತ್ತು ಅದನ್ನು ಹೇಗೆ ಒದಗಿಸಲು ಎಂಬುದನ್ನು ಸ್ಪಷ್ಟವಾಗಿ ಅರ್ಥಮಾಡಿಕೊಳ್ಳುತ್ತದೆ.

ನೀವು ಸಹೋದ್ಯೋಗಿಗೆ ಸೂಚನೆಗಳನ್ನು ನೀಡುವುದು ಹಾಗೆ ಇದನ್ನು ಚಿಂತಿಸಿ. "ಬಗ್ ಸರಿಪಡಿಸಿ" ಎಂಬುವದು ಅಪಾಯ ಅರ್ಥವನ್ನು ಕೊಡುತ್ತದೆ. "UserService.java ರ 45ನೇ ಸಾಲಿನಲ್ಲಿ ನಲ್ ಪಾಯಿಂಟರ್ ಎಕ್ಸೆಪ್ಷನ್‌ಗೆ ನಲ್ ಚೆಕ್ ಸೇರಿಸಿ" ಎಂದು ಹೇಳುವುದು ವಿಶೇಷವಾಗಿದೆ. ಭಾಷಾ ಮಾದರಿಗಳು ಕೂಡ ಇದೇ ರೀತಿಯವರಾಗಿವೆ - ನಿರ್ದಿಷ್ಟತೆ ಮತ್ತು ರಚನೆ ಮುಖ್ಯ.

## ಇದು ಹೇಗೆ LangChain4j ಬಳಕೆಮಾಡುತ್ತದೆ

ಈ ಮಡ್ಯೂಲ್ ಹಿಂದಿನ ಮಡ್ಯೂಲ್‌ಗಳಿಂದ ಬಳಸುತಿರುವ LangChain4j ಮೂಲಭೂತದೊಂದಿಗೆ ಸುಧಾರಿತ ಪ್ರಾಂಪ್ಟ್ ಮಾದರಿಗಳನ್ನು ತೋರಿಸುತ್ತದೆ, ಪ್ರಾಂಪ್ಟ್ ರಚನೆ ಮತ್ತು ತರ್ಕ ನಿಯಂತ್ರಣದ ಮೇಲೆ ಕೇಂದ್ರೀಕೃತವಾಗಿದೆ.

<img src="../../../translated_images/kn/langchain4j-flow.48e534666213010b.webp" alt="LangChain4j Flow" width="800"/>

*LangChain4j ನಿಮ್ಮ ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು Azure OpenAI GPT-5.2 ಗೆ ಸಂಪರ್ಕಿಸುವುದು*

**ಅನಿವಾರ್ಯತೆಗಳು** - Module 02 ಕೆಳಗಿನ langchain4j ಅನಿವಾರ್ಯತೆಗಳನ್ನು `pom.xml` ನಲ್ಲಿ ಉಪಯೋಗಿಸುತ್ತದೆ:
```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```

**OpenAiOfficialChatModel ವ್ಯಾಖ್ಯಾನ** - [LangChainConfig.java](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/config/LangChainConfig.java)

ಚಾಟ್ ಮಾದರಿಯನ್ನು Spring ಬೀನ್ ಮೂಲಕ OpenAI ಅಧಿಕೃತ ಕ್ಲೈಂಟ್ ಬಳಸಿ ಕೈಯಿಂದ ಸಂರಚಿಸಲಾಗಿದೆ, ಅದು Azure OpenAI ಅಂತಿಮಸ್ಥಾನಗಳನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ. ಮಡ್ಯೂಲ್ 01 ನಿಂದ ಪ್ರಮುಖ ವ್ಯತ್ಯಾಸವು ಅದು `chatModel.chat()` ಕ್ಕೆ ಕಳುಹಿಸುವ ಪ್ರಾಂಪ್ಟ್ ರಚನೆಗೆ ಸಂಬಂಧಿಸಿದೆ, ಮಾದರಿಯನ್ನು ಸ್ವತಃ ರಚಿಸುವುದಕ್ಕೆ ಅಲ್ಲ.

**ಸಿಸ್ಟಮ್ ಮತ್ತು ಬಳಕೆದಾರ ಸಂದೇಶಗಳು** - [Gpt5PromptService.java](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java)

LangChain4j ಸಂದೇಶ ಪ್ರಕಾರಗಳನ್ನು ಸ್ಪಷ್ಟತೆಯಿಗಾಗಿ பிரித்து ರಕ್ಷಿಸುತ್ತದೆ. `SystemMessage` ಯಂತೆ AI ನ ವರ್ತನೆ ಮತ್ತು ಸುತ್ತು (ಉದಾಹರಣೆಗೆ "ನೀವು ಒಂದು ಕೋಡ್ ವಿಮರ್ಶಕರು"), ಮತ್ತು `UserMessage` ವಾಸ್ತವ ಬೇಡಿಕೆಯನ್ನು ಹೊಂದಿದೆ. ಇದರಿಂದ ವಿಭಿನ್ನ ಬಳಕೆದಾರ ಪ್ರಶ್ನೆಗಳ ವೇಳೆ ಶಾಶ್ವತ AI ವರ್ತನೆ ಉಳಿಯುತ್ತದೆ.

```java
SystemMessage systemMsg = SystemMessage.from(
    "You are a helpful Java programming expert."
);

UserMessage userMsg = UserMessage.from(
    "Explain what a List is in Java"
);

String response = chatModel.chat(systemMsg, userMsg);
```

<img src="../../../translated_images/kn/message-types.93e0779798a17c9d.webp" alt="Message Types Architecture" width="800"/>

*SystemMessage ಶಾಶ್ವತ ಸುತ್ತು ಕೊಡುತ್ತದೆ ಮತ್ತು UserMessages ವೈಯಕ್ತಿಕ ಬೇಡಿಕೆಗಳನ್ನು ಹೊಂದಿವೆ*

**MessageWindowChatMemory ಬಹು-ಘಟ್ಟಕ್ಕಾಗಿ** - ಬಹು-ಘಟ್ಟ ಸಂಭಾಷಣಾ ಮಾದರಿಗಾಗಿ, ನಾವು Module 01 ನ MessageWindowChatMemory ಅನ್ನು ಮರುಬಳಕೆಮಾಡುತ್ತೇವೆ. ಪ್ರತಿ ಸೆಷನ್ ತನ್ನದೇ ಮೆಮರಿ ಪ್ರತಿಯನ್ನು `Map<String, ChatMemory>` ನಲ್ಲಿ ಸಂಗ್ರಹಿಸುತ್ತದೆ, ಇದರಿಂದ ಮಿಶ್ರಣವಿಲ್ಲದೆ ಒಂದು ಸಮಯದಲ್ಲಿ ಅನೇಕ ಸಂಭಾಷಣೆಗಳು ಸಾಧ್ಯವಾಗುತ್ತವೆ.

**ಪ್ರಾಂಪ್ಟ್ ಟೆಂಪ್ಲೇಟ್ಸ್** - ಮುಖ್ಯ ಒತ್ತು ಪ್ರಾಂಪ್ಟ್ ಇಂಜಿನಿಯರಿಂಗ್ ಮೇಲೆ ಇದೆ, ಹೊಸ LangChain4j API ಗಳ ಮೇಲೆ ಅಲ್ಲ. ಪ್ರತಿ ಮಾದರಿ (ಕಡಿಮೆ ಉತ್ಸಾಹ, ಹೆಚ್ಚಿನ ಉತ್ಸಾಹ, ಕಾರ್ಯ ನಿರ್ವಹಣೆ ಇತ್ಯಾದಿ) ಅದೇ `chatModel.chat(prompt)` ವಿಧಾನವನ್ನು ಬಳಸುತ್ತದೆ ಆದರೆ ಜಾಗರೂಕತೆಯಿಂದ ರಚಿಸಲಾದ ಪ್ರಾಂಪ್ಟ್ ಸ್ಟ್ರಿಂಗ್‌ಗಳೊಂದಿಗೆ. XML ಟ್ಯಾಗ್‌ಗಳು, ಸೂಚನೆಗಳು ಮತ್ತು ಫಾರ್ಮ್ಯಾಟಿಂಗ್ ಎಲ್ಲವೂ ಪ್ರಾಂಪ್ಟ್ ಪಠ್ಯದ ಭಾಗವಾಗಿವೆ, LangChain4j ವೈಶಿಷ್ಟ್ಯಗಳು ಅಲ್ಲ.

**ತರ್ಕ ನಿಯಂತ್ರಣ** - GPT-5.2 ನ ತರ್ಕ ಪ್ರಯತ್ನವನ್ನು "ಗರಿಷ್ಠ 2 ತರ್ಕ ಹಂತಗಳು" ಅಥವಾ "ವಿಸ್ತೃತವಾಗಿ ಪರಿಶೀಲಿಸಿ" ಎಂಬಂತೆ ಪ್ರಾಂಪ್ಟ್ ಸೂಚನೆಗಳಿಂದ ನಿಯಂತ್ರಿಸಲಾಗುತ್ತದೆ. ಇವು ಪ್ರಾಂಪ್ಟ್ ಇಂಜಿನಿಯರಿಂಗ್ ತಂತ್ರಗಳು, LangChain4j ಸಂರಚನೆಗಳು ಅಲ್ಲ. ಲೈಬ್ರರಿ ನಿಮ್ಮ ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ಮಾದರಿಗೆ ಹಂಚಿಕೊಳ್ಳುತ್ತದೆ.

ಮುಖ್ಯ ಸಾರಾಂಶ: LangChain4j ಮೂಲಭೂತವನ್ನು ಒದಗಿಸುತ್ತದೆ ([LangChainConfig.java](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/config/LangChainConfig.java) ಮೂಲಕ ಮಾದರಿ ಸಂಪರ್ಕ, ಮೆಮೊರಿ ಮತ್ತು ಸಂದೇಶ ನಿರ್ವಹಣೆ [Gpt5PromptService.java](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java) ಮೂಲಕ), ಮತ್ತು ಈ ಮಡ್ಯೂಲ್ ಆ ಮೂಲಭೂತದಲ್ಲಿ ಪರಿಣಾಮಕಾರಿಯಾದ ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ರಚಿಸುವುದನ್ನು ಕಲಿಸುತ್ತದೆ.

## ಮೂಲ ಮಾದರಿಗಳು

ಎಲ್ಲಾ ಸಮಸ್ಯೆಗಳು ಒಂದೇ ವಿಧಾನವನ್ನು ಬೇಡುವುದಿಲ್ಲ. ಕೆಲವು ಪ್ರಶ್ನೆಗಳು ತುರ್ತು ಉತ್ತರಗಳನ್ನು ಬೇಕಾಗಿವೆ, ಇತರರು ಆಳವಾದ ವಿಚಾರಪಡುವಿಕೆಯನ್ನು. ಕೆಲವು ದೃಶ್ಯ ತರ್ಕವನ್ನು ಬೇಡುತ್ತವೆ, ಇತರರು ಕೇವಲ ಫಲಿತಾಂಶಗಳನ್ನು. ಈ ಮಡ್ಯೂಲ್ ಎಂಟು ಪ್ರಾಂಪ್ಟ್ ಮಾದರಿಗಳನ್ನು ತೋರುತ್ತದೆ - ಪ್ರತಿ ವಿಭಿನ್ನ ಪರಿಸ್ಥಿತಿಗಳಿಗೆ ಸೂಕ್ತವಾಗಿವೆ. ನೀವು ಎಲ್ಲವನ್ನೂ ಪ್ರಯೋಗಿಸಿ ಯಾವಾಗ ಯಾವ ವಿಧಾನ ಉತ್ತಮವೆಂದು ಕಲಿಯುತ್ತೀರಿ.

<img src="../../../translated_images/kn/eight-patterns.fa1ebfdf16f71e9a.webp" alt="Eight Prompting Patterns" width="800"/>

*ಎಂಟು ಪ್ರಾಂಪ್ಟ್ ಇಂಜಿನಿಯರಿಂಗ್ ಮಾದರಿಗಳ ಅವಲೋಕನ ಹಾಗೂ ಅವುಗಳ ಉಪಯೋಗ ಪ್ರಕರಣಗಳು*

<img src="../../../translated_images/kn/reasoning-effort.db4a3ba5b8e392c1.webp" alt="Reasoning Effort Comparison" width="800"/>

*ಕಡಿಮೆ ಉತ್ಸಾಹ (ವೇಗವಾಗಿ, ನೇರವಾಗಿ) ಮತ್ತು ಹೆಚ್ಚು ಉತ್ಸಾಹ (ವಿಸ್ತೃತ, ಪರಿಶೀಲನಾತ್ಮಕ) ತರ್ಕ ವಿಧಾನಗಳ ಹೋಲಿಕೆ*

**ಕಡಿಮೆ ಉತ್ಸಾಹ (ಶೀಘ್ರ ಮತ್ತು ಕೇಂದ್ರೀಕೃತ)** - ಸರಳ ಪ್ರಶ್ನೆಗಳಿಗಾಗಿ, ಮತ್ತು ನಿಮಗೆ ವೇಗವಾದ ನೇರ ಉತ್ತರ ಬೇಕಾದಾಗ.Use this for calculations, lookups, or straightforward questions.

```java
String prompt = """
    <reasoning_effort>low</reasoning_effort>
    <instruction>maximum 2 reasoning steps</instruction>
    
    What is 15% of 200?
    """;

String response = chatModel.chat(prompt);
```

> 💡 **GitHub Copilot ಜೊತೆ ಅನ್ವೇಷಿಸಿ:** [`Gpt5PromptService.java`](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java) ತೆರೆಯಿರಿ ಮತ್ತು ಕೇಳಿ:
> - "ಕಡಿಮೆ ಉತ್ಸಾಹ ಮತ್ತು ಹೆಚ್ಚಿನ ಉತ್ಸಾಹ ಪ್ರಾಂಪ್ಟ್ ಮಾದರಿಗಳ ನಡುವಿನ ವ್ಯತ್ಯಾಸವೇನು?"
> - "ಪ್ರಾಂಪ್ಟ್‌ಗಳಲ್ಲಿ XML ಟ್ಯಾಗ್‌ಗಳು AI ದ ಉತ್ತರ ರಚನೆಯಲ್ಲಿ ಹೇಗೆ ಸಹಾಯಮಾಡುತ್ತವೆ?"
> - "ಸ್ವಯಂ ಪರಿಶೀಲನೆ ಮಾದರಿಗಳನ್ನು ನೇರ ಸೂಚನೆಯೊಂದಿಗೆ ಯಾವಾಗ ಬಳಸಬೇಕು?"

**ಹೆಚ್ಚು ಉತ್ಸಾಹ (ಆಳವಾದ ಮತ್ತು ವಿಸ್ತೃತ)** - ಸಂಕೀರ್ಣ ಸಮಸ್ಯೆಗಳಿಗಾಗಿ, ನೀವು ಸಮಗ್ರ ವಿಶ್ಲೇಷಣೆಯನ್ನು ಬೇಕಾದಾಗ. ಮಾದರಿ ಗೌರವಯುತವಾಗಿ ಪರಿಶೀಲಿಸುತ್ತದೆ ಮತ್ತು ವಿಸ್ತೃತ ತರ್ಕವನ್ನು ತೋರಿಸುತ್ತದೆ. ವ್ಯವಸ್ಥೆ ವಿನ್ಯಾಸ, ವಾಸ್ತುಶಿಲ್ಪ ನಿರ್ಧಾರಗಳು ಅಥವಾ ಗಂಭೀರ ಸಂಶೋಧನೆಗಾಗಿ ಈ ರೀತಿ ಉಪಯೋಗಿಸಿ.

```java
String prompt = """
    <reasoning_effort>high</reasoning_effort>
    <instruction>explore thoroughly, show detailed reasoning</instruction>
    
    Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```

**ಕಾರ್ಯ ನಿರ್ವಹಣೆ (ಹಂತ ಹಂತವಾಗಿ ಪ್ರಗತಿ)** - ಬಹು ಹಂತದ ಕೆಲಸಗಳಿಗಾಗಿ. ಮಾದರಿ ಮೊದಲು ಯೋಜನೆ ನೀಡುತ್ತದೆ, ಪ್ರತಿ ಹಂತವನ್ನು ವರ್ಣಿಸುತ್ತದೆ, ನಂತರ ಸಾರಾಂಶ ನೀಡುತ್ತದೆ. ಮೈಸಿಗುಮಾರಿಗೆ, ಅನುಷ್ಠಾನಗಳಿಗೆ ಅಥವಾ ಯಾವುದೇ ಬಹು ಹಂತ ಕಾರ್ಯಗಳಿಗೆ ಇದು ಸೂಕ್ತ.

```java
String prompt = """
    <task>Create a REST endpoint for user registration</task>
    <preamble>Provide an upfront plan</preamble>
    <narration>Narrate each step as you work</narration>
    <summary>Summarize what was accomplished</summary>
    """;

String response = chatModel.chat(prompt);
```

ಚೈನ್-ಆಫ್-ಥಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್ ಮಾದರಿಗೆ ತನ್ನ ತರ್ಕ ಕ್ರಮವನ್ನು ತೋರಿಸಲು ಸ್ಪಷ್ಟವಾಗಿ ಕೇಳುತ್ತದೆ, ಸಂಕೀರ್ಣ ಕಾರ್ಯಗಳ ಸಮರ್ಪಕತೆಯನ್ನು ಸುಧಾರಿಸುತ್ತದೆ. ಹಂತ-ಹಂತ ವಿಭಾಗವು ಮಾನವರು ಮತ್ತು AI ಎರಡರಿಗೂ ತರ್ಕವನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.

> **🤖 [GitHub Copilot](https://github.com/features/copilot) ಚಾಟ್ ಮೂಲಕ ಪ್ರಯತ್ನಿಸಿ:** ಈ ಮಾದರಿಯನ್ನು ಕುರಿತು ಕೇಳಿ:
> - "ದೀರ್ಘಕಾಲಿಕ ಕಾರ್ಯಗಳಿಗೆ ಕಾರ್ಯ ನಿರ್ವಹಣೆ ಮಾದರಿಯನ್ನು ಹೇಗೆ ಹೊಂದಿಸಬಹುದು?"
> - "ಉತ್ಪಾದನಾ ಅಪ್ಲಿಕೇಶನ್‌ಗಳಲ್ಲಿ ಉಪಕರಣ ಪೂರ್ವಪ್ರಸಂಗಗಳನ್ನು ರಚಿಸುವ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು ಯಾವುವು?"
> - "UI ಯಲ್ಲಿ ಮಧ್ಯಂತರ ಪ್ರಗತಿ ನವೀಕರಣಗಳನ್ನು ಹೇಗೆ ಸೆರೆಹಿಡಿದು ಪ್ರದರ್ಶಿಸಬಹುದು?"

<img src="../../../translated_images/kn/task-execution-pattern.9da3967750ab5c1e.webp" alt="Task Execution Pattern" width="800"/>

*ಯೋಜನೆ → ಕಾರ್ಯಗತಗೊಳಿಕೆ → ಸಾರಾಂಶ ಹಂತ-ಹಂತ ಕಾರ್ಯಗಳಿಗಾಗಿ*

**ಸ್ವಯಂ ಪರಿಶೀಲನೆ ಕೋಡ್** - ಉತ್ಪಾದನಾ ಗುಣಮಟ್ಟದ ಕೋಡ್ ರಚನೆಗಾಗಿ. ಮಾದರಿ ಕೋಡ್ ಸೃಷ್ಟಿಸುತ್ತದೆ, ಗುಣಮಟ್ಟ ಮಾನದಂಡಗಳಿಗೆ ಪರಿಶೀಲಿಸುತ್ತದೆ ಮತ್ತು ಕ್ರಮೇಣ ಸುಧಾರಣೆ ಮಾಡುತ್ತದೆ. ಹೊಸ ವೈಶಿಷ್ಟ್ಯಗಳು ಅಥವಾ ಸೇವೆಗಳು ನಿರ್ಮಿಸುವಾಗ ಇದನ್ನು ಉಪಯೋಗಿಸಿ.

```java
String prompt = """
    <task>Create an email validation service</task>
    <quality_criteria>
    - Correct logic and error handling
    - Best practices (clean code, proper naming)
    - Performance optimization
    - Security considerations
    </quality_criteria>
    <instruction>Generate code, evaluate against criteria, improve iteratively</instruction>
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/kn/self-reflection-cycle.6f71101ca0bd28cc.webp" alt="Self-Reflection Cycle" width="800"/>

*ಅವರ್ತಿತ ಸುಧಾರಣೆ ಚಕೃತಿ - sೃಷ್ಟಿ, ಮೌಲ್ಯಮಾಪನ, ಸಮಸ್ಯೆ ಗುರುತುಹಾಕುವುದು, ಸುಧಾರಣೆ, ಪುನರಾವೃತ್ತಿ*

**ಸಂರಚಿತ ವಿಶ್ಲೇಷಣೆ** - ಸ್ಥಿರ ಮೌಲ್ಯಮಾಪನಕ್ಕಾಗಿ. ಕೋಡ್ ನಿಗ್ರಹದಲ್ಲಿ ಸ್ಥಿರ ವೀಕ್ಷಣೆ (ಸರಿಯಾದತೆ, ಅಭ್ಯಾಸಗಳು, ಕಾರ್ಯಕ್ಷಮತೆ, ಭದ್ರತೆ). ಕೋಡ್ ವಿಮರ್ಶೆಗಳು ಅಥವಾ ಗುಣಮಟ್ಟ ಮೌಲ್ಯಮಾಪನಗಳಿಗೆ ಉಪಯೋಗಿಸಿ.

```java
String prompt = """
    <code>
    public List getUsers() {
        return database.query("SELECT * FROM users");
    }
    </code>
    
    <framework>
    Evaluate using these categories:
    1. Correctness - Logic and functionality
    2. Best Practices - Code quality
    3. Performance - Efficiency concerns
    4. Security - Vulnerabilities
    </framework>
    """;

String response = chatModel.chat(prompt);
```

> - "ಬೇಬಿಧ ಪ್ರಕಾರದ ಕೋಡ್ ವಿಮರ್ಶೆಗಳಿಗೆ ವಿಶ್ಲೇಷಣೆ ಫ್ರೇಮ್ವರ್ಕ್ ಅನ್ನು ಹೇಗೆ ಕಸ್ಟಮೈಸ್ ಮಾಡಬಹುದು?"
> - "ಸಂರಚಿತ ಔಟ್ಪುಟ್ ಅನ್ನು ಪ್ರೋಗ್ರಾಮ್ಯಾಟಿಕವಾಗಿ ವಿಶ್ಲೇಷಿಸಲು ಮತ್ತು ಕ್ರಮ ಕೈಗೊಳ್ಳಲು ಉತ್ತಮ ವಿಧಾನವೇನು?"
> - "ವಿಭিন্ন ವಿಮರ್ಶೆ ಸೆಷನ್ಸ್ ನಡುವಿನ ಗಂಭೀರತೆಯನ್ನು ಸ್ಥಿರವಾಗಿಡಲು ಹೇಗೆ ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಬಹುದು?"

<img src="../../../translated_images/kn/structured-analysis-pattern.0af3b690b60cf2d6.webp" alt="Structured Analysis Pattern" width="800"/>

*ನಿತ್ಯ ಕೋಡ್ ವಿಮರ್ಶೆಗಾಗಿ ನಾಲ್ಕು ವರ್ಗಗಳ ಫ್ರೆ임್ವರ್ಕ್ ಮತ್ತು ಗಂಭೀರತಾ ಮಟ್ಟಗಳು*

**ಬಹು-ಘಟ್ಟ ಚಾಟ್** - ಸಂಭಾಷಣೆಗಳಿಗೆ ಸುತ್ತು ಅಗತ್ಯವಿದ್ದಾಗ. ಮಾದರಿ ಹಿಂದಿನ ಸಂದೇಶಗಳನ್ನು ನೆನಪುಮಾಡಿಕೊಂಡು ಅದರ ಮೇಲೆ ನಿರ್ಮಿಸುತ್ತದೆ. ಇಂಟರಕ್ಟಿವ್ ಸಹಾಯ ಅಥವಾ ಸಂಕೀರ್ಣ ಪ್ರಶ್ನೋತ್ತರಗಳಿಗೆ ಇದನ್ನು ಉಪಯೋಗಿಸಿ.

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("What is Spring Boot?"));
AiMessage aiMessage1 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage1);

memory.add(UserMessage.from("Show me an example"));
AiMessage aiMessage2 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage2);
```

<img src="../../../translated_images/kn/context-memory.dff30ad9fa78832a.webp" alt="Context Memory" width="800"/>

*ಸಂಭಾಷಣಾ ಸುತ್ತುಗಳ ಸ್ಮೃತಿ πολλentligibility ಹೆಚ್ಚು ಮಿತಿ ತಲುಪುವವರೆಗೆ ಸ೦ಗ್ರಹಿಸುತ್ತದೆ*

**ಹಂತ ಹಂತವಾದ ತರ್ಕ** - ದೃಶ್ಯಮಾನ ಲಾಜಿಕ್ ಬೇಕಾದ ಸಮಸ್ಯೆಗಳಿಗೆ. ಮಾದರಿ ಪ್ರತಿ ಹಂತದ ತರ್ಕವನ್ನು ಸ್ಪಷ್ಟವಾಗಿ ತೋರುತ್ತದೆ. ಗಣಿತ ಸಮಸ್ಯೆಗಳು, ಲಾಜಿಕ್ ಪಜಲ್ಗಳು ಅಥವಾ ವಿಚೇತನ ಪ್ರಕ್ರಿಯೆಯನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಬೇಕಾದಾಗ ಉಪಯೋಗಿಸಿ.

```java
String prompt = """
    <instruction>Show your reasoning step-by-step</instruction>
    
    If a train travels 120 km in 2 hours, then stops for 30 minutes,
    then travels another 90 km in 1.5 hours, what is the average speed
    for the entire journey including the stop?
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/kn/step-by-step-pattern.a99ea4ca1c48578c.webp" alt="Step-by-Step Pattern" width="800"/>

*ಸಂಕೀರ್ಣ ಸಮಸ್ಯೆಗಳನ್ನು ನಿರ್ದಿಷ್ಟ ತರ್ಕ ಹಂತಗಳಾಗಿ ವಿಭಜಿಸುವುದು*

**ದೇರಿಕೆಯ ಉತ್ಪಾದನೆ** - ನಿರ್ದಿಷ್ಟ ಸ್ವರೂಪದ ನಿಬಂಧನೆಗಳಿಗೆ. ಮಾದರಿ ಸ್ವರೂಪ ಮತ್ತು ಉದ್ದ ನಿಯಮಗಳನ್ನು ಕಟ್ಟುನಿಟ್ಟಾಗಿ ಅನುಸರಿಸುತ್ತದೆ. ಸಾರಾಂಶಗಳು ಅಥವಾ ನಿಖರ ಸ್ವರೂಪದ ಔಟ್ಪುಟ್ ಬೇಕಾದಾಗ ಉಪಯೋಗಿಸಿ.

```java
String prompt = """
    <constraints>
    - Exactly 100 words
    - Bullet point format
    - Technical terms only
    </constraints>
    
    Summarize the key concepts of machine learning.
    """;

String response = chatModel.chat(prompt);
```

*ನಿರ್ದಿಷ್ಟ ಸ್ವರೂಪ, ಉದ್ದ ಮತ್ತು ರಚನೆ ನಿಯಮಗಳನ್ನು ಜಾರಿಗೊಳಿಸುವುದು*

**ಆಯ್ಕೆ 1: Spring Boot ಡಾಶ್‌ಬೋರ್ಡ್ ಬಳಕೆ (VS Code ಬಳಕೆದಾರರಿಗೆ ಶಿಫಾರಸು)**

ಡೆವ್ ಕಂಟೈನರ್‌ನಲ್ಲಿ Spring Boot ಡಾಶ್‌ಬೋರ್ಡ್ ವಿಸ್ತರಣೆ ಸೇರಿಸಲಾಗಿದೆ, ಇದು ಎಲ್ಲ Spring Boot ಅಪ್ಲಿಕೇಶನ್‌ಗಳ ನಿರ್ವಹಣೆಗಾಗಿ ದೃಷ್ಯಾತ್ಮಕ ಇಂಟರ್ಫೇಸ್ ನ್ನು ಒದಗಿಸುತ್ತದೆ. VS Code ನಲ್ಲಿ ಎಡಭಾಗದ ಕ್ರಿಯಾಶೀಲತೆ ಪಟ್ಟಿಯಲ್ಲಿ (Spring Boot ಐಕಾನ್ ಹುಡುಕండి) ಲಭ್ಯವಿದೆ.

Spring Boot ಡಾಶ್‌ಬೋರ್ಡ್ ನಿಂದ ನೀವು:
- ಕಾರ್ಯಕ್ಷೇತ್ರದಲ್ಲಿರುವ ಎಲ್ಲಾ Spring Boot ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನೋಡಬಹುದು
- ಒಂದು ಕ್ಲಿಕ್‌ನಿಂದ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಆರಂಭ/ನಿಲ್ಲಿಸಬಹುದು
- ಅಪ್ಲಿಕೇಶನ್ ಲಾಗ್‌ಗಳನ್ನು ಯಥಾರ್ಥಕಾಲದಲ್ಲಿ ನೋಡಬಹುದು
- ಅಪ್ಲಿಕೇಶನ್ ಸ್ಥಿತಿಯನ್ನು ತುರ್ತುಪಾಲಿಸಬಹುದು

"prompt-engineering" ಬದಿಯಲ್ಲಿ ಇದ್ದ ಪ್ಲೇ ಬಟನ್ ಕ್ಲಿಕ್ ಮಾಡಿ ಈ ಮಡ್ಯೂಲ್ ಪ್ರಾರಂಭಿಸಿ, ಅಥವಾ ಎಲ್ಲಾ ಮಡ್ಯೂಲ್‌ಗಳನ್ನು ಒಂದೇ ಸಮಯದಲ್ಲಿ ಪ್ರಾರಂಭಿಸಿ.

<img src="../../../translated_images/kn/dashboard.da2c2130c904aaf0.webp" alt="Spring Boot Dashboard" width="400"/>

**ಆಯ್ಕೆ 2: ಶೆಲ್ ಸ್ಕ್ರಿಪ್ಟ್‌ಗಳನ್ನು ಬಳಸು:**

ಎಲ್ಲಾ ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು (ಮಡ್ಯೂಲ್ 01-04) ಪ್ರಾರಂಭಿಸಿ:

**ಬ್ಯಾಶ್:**

```

ಇಲ್ಲವೇ ಈ ಮಡ್ಯೂಲ್ ಮಾತ್ರ ಪ್ರಾರಂಭಿಸಿ:

**ಬ್ಯಾಶ್:**

```

**ಪವರ್‌ಶೆಲ್:**
```powershell
cd 02-prompt-engineering
.\start.ps1
```

ಎರಡು ಸ್ಕ್ರಿಪ್ಟ್‌ಗಳು ರೂಟ್ `.env` ಫೈಲ್ ನಿಂದ ಪರಿಸರ ವ್ಯತ್ಯಾಸಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಲೋಡ್ ಮಾಡುತ್ತವೆ ಮತ್ತು JAR ಫೈಲ್‌ಗಳನ್ನು ಇಲ್ಲದಿದ್ದರೆ ರಚಿಸುತ್ತವೆ.

> **ಸೂಚನೆ:** ನಿಮ್ಮ ಬಯಸಿದರೆ ಮಡ್ಯೂಲ್‌ಗಳನ್ನು ಕೈಯಿಂದ ನಿರ್ಮಿಸಿ ನಂತರ ಪ್ರಾರಂಭಿಸಬಹುದು:
>
> **ಬ್ಯಾಶ್:**
> ```bash

> ```
>
> **ಪವರ್‌ಶೆಲ್:**
> ```powershell
> cd ..  # Go to root directory
> mvn clean package -DskipTests
> ```

ನಿಮ್ಮ ಬ್ರೌಸರ್‌ನಲ್ಲಿ http://localhost:8083 ತೆರೆಯಿರಿ.

**ನಿಲ್ಲಿಸಲು:**

**ಬ್ಯಾಶ್:**

cd .. && ./stop-all.sh  # ಎಲ್ಲಾ ಮಾಳಿಕೆಗಳು

```powershell

cd ..; .\stop-all.ps1  # ಎಲ್ಲಾ ಮೋಡ್ಯೂಲುಗಳು
```

## ಮಾದರಿಗಳನ್ನು ಅನ್ವೇಷಿಸುವುದು

ವೆಬ್ ಇಂಟರ್ಫೇಸ್ ನಿಮಗೆ ವಿಭಿನ್ನ ಪ್ರಾಂಪ್ಟಿಂಗ್ ತಂತ್ರಗಳನ್ನು ಪ್ರಯೋಗಿಸಲು ಅವಕಾಶ ಮಾಡಿಕೊಡುತ್ತದೆ. ಪ್ರತಿಯೊಂದು ಮಾದರಿ ವಿಭಿನ್ನ ಸಮಸ್ಯೆಗಳನ್ನು ಪರಿಹರಿಸುತ್ತದೆ - ಯಾವಾಗ ಯಾವ ವಿಧಾನ ಮಿಗಿಲಾಗಿ ಕೆಲಸ ಮಾಡುತ್ತದೆ ಎಂಬುದನ್ನು ನೋಡಲು ಪ್ರಯತ್ನಿಸಿ.

### ಕಡಿಮೆ ವಿರುದ್ಧ ಉದ್ದೇಶ

ಸರಳವಾದ ಪ್ರಶ್ನೆಯನ್ನು ಕೇಳಿ "200 ರ 15% ಎಷ್ಟು?" ಕಡಿಮೆ ಉತ್ಸಾಹ ಬಳಸಿ. ನಿಮಗೆ ತಕ್ಷಣದ, ನೇರ ಉತ್ತರ ಸಿಗುತ್ತದೆ. ಈಗ "ಹಾಯ್ ಟ್ರಾಫಿಕ್ API ಗಾಗಿ ಕ್ಯಾಶಿಂಗ್ ತಂತ್ರವನ್ನು ವಿನ್ಯಾಸಮಾಡಿ" ಎಂದು ಕೇಳಿ, ಹೆಚ್ಚಿನ ಉತ್ಸಾಹ ಉಪಯೋಗಿಸಿ. ಮಾದರಿ ಹೇಗೆ ನಿಧಾನವಾಗಿ ಚಿಂತಿಸಿ ವಿಸ್ತೃತ ತರ್ಕವನ್ನು ನೀಡುತ್ತದೆ ಎಂದು ಗಮನಿಸಿ. ಅದೇ ಮಾದರಿ, ಅದೇ ಪ್ರಶ್ನೆಯ ರಚನೆ - ಆದರೆ ಪ್ರಾಂಪ್ಟ್ ಅದಕ್ಕೆ ಎಷ್ಟು ಚಿಂತನೆ ಮಾಡಲು ಹೇಳುತ್ತದೆ.
<img src="../../../translated_images/kn/low-eagerness-demo.898894591fb23aa0.webp" alt="ಕಡಿಮೆ ಆಸಕ್ತಿ ಡೆಮೋ" width="800"/>

*ಕನಿಷ್ಟ ವಿಚಾರಸಮ್ಮಿಶ್ರಣದೊಂದಿಗೆ ಬೇಗನೆ ಲೆಕ್ಕ ಹಾಕುವುದು*

<img src="../../../translated_images/kn/high-eagerness-demo.4ac93e7786c5a376.webp" alt="ಹೆಚ್ಚು ಆಸಕ್ತಿ ಡೆಮೋ" width="800"/>

*ಸಂಗ್ರಹಣ ನೀತಿ ಸಂಪೂರ್ಣ (2.8MB)*

### ಕೆಲಸದ ನಿರ್ವಹಣೆ (ಟೂಲ್ ಪೂರ್ವ ಲೇಖನಗಳು)

ಬಹು ಹಂತದ ಕೆಲಸಗಳು ಮುಂಚಿತ ಯೋಜನೆ ಮತ್ತು ಪ್ರಗತಿ ವಿವರಣೆಯಿಂದ ಲಾಭ ಹೊಂದುತ್ತವೆ. ಮಾದರಿ ಅದು ಮಾಡುವುದೇನು ಎಂಬುದನ್ನು ವಿವರಿಸುತ್ತದೆ, ಪ್ರತಿ ಹಂತವನ್ನು ವಿವರಿಸುತ್ತದೆ, ನಂತರ ಫಲಿತಾಂಶಗಳನ್ನು ಸಾರಾಂಶಗೊಳಿಸುತ್ತದೆ.

*ಹಂತದಿಂದ ಹಂತಕ್ಕೆ ವಿವರಣೆ ಸಹಿತ REST ಎಂಡ್ಪಾಯಿಂಟ್ ರಚನೆ (3.9MB)*

### ಸ್ವಯಂ-ಪರಿಶೀಲಿಸುವ ಕೋಡ್

"ಇಮೇಲ್ ಮಾನ್ಯತೆ ಸೇವೆ ರಚಿಸಿ" ಎಂದು ಪ್ರಯತ್ನಿಸಿ. ಕೇವಲ ಕೋಡ್ ರಚಿಸಿ ನಿಲ್ಲಿಸುವ ಬದಲು, ಮಾದರಿ ರಚಿಸಿ, ಗುಣಮಟ್ಟ ಮಾನದಂಡಗಳ ವಿರುದ್ಧ ಮೌಲ್ಯಮಾಪನ ಮಾಡಿ, ದುರ್ಬಲತೆಯನ್ನು ಗುರುತಿಸಿ, ಮತ್ತು ಸುಧಾರಿಸುತ್ತದೆ. ನೀವು ಅದನ್ನು ತತ್ವಾಗತದಂತೆ ಪುನರಾವೃತ್ತಿ ಮಾಡುವುದನ್ನು ನೋಡುತ್ತೀರಿ.

ಕೋಡ್ ಪರಿಶೀಲನೆಗಳು ನಿರಂತರ ಮೌಲ್ಯಮಾಪನ ರೂಪರೇಖೆಗಳನ್ನು ಅವಲಂಬಿಸಬೆಕು. ಮಾದರಿ ನಿಶ್ಚಿತ ವರ್ಗಗಳ (ಸರಿಯಾದತೆ, ಅಭ್ಯಾಸಗಳು, ಕಾರ್ಯಕ್ಷಮತೆ, ಭದ್ರತೆ) ವ್ಯಾಪ್ತಿಯಲ್ಲಿ ಕೋಡ್ ವಿಶ್ಲೇಷಣೆ ಮಾಡುತ್ತದೆ ಗಂಭೀರತೆಯ ಮಟ್ಟಗಳಿಂದ.

*ರೂಪರೇಖೆಯ ಆಧಾರಿತ ಕೋಡ್ ವಿಮರ್ಶೆ*

### ಬಹು-ತಿರುವು ಚಾಟ್

"Spring Boot ಎಂದರೆ ಏನು?" ಎಂಬುದನ್ನು ಕೇಳಿ, ಬಳಿಕ ತಕ್ಷಣ "ನಮೂನೆ ತೋರಿಸಿ" ಎಂದು ಕೇಳಿ. ಮಾದರಿ ನಿಮ್ಮ ಮೊದಲ ಪ್ರಶ್ನೆಯನ್ನು ನೆನಸಿಕೊಳ್ಳುತ್ತದೆ ಮತ್ತು ಪ್ರತ್ಯೇಕವಾಗಿ Spring Boot ಉದಾಹರಣೆಯನ್ನು ನಿಮಗೆ ನೀಡುತ್ತದೆ. ನೆನಪಿಲ್ಲದೆ, ಎರಡನೇ ಪ್ರಶ್ನೆ ಬಹಳ ಅಸ್ಪಷ್ಟವಾಗಬಹುದಾಗಿತ್ತು.

### ಹಂತದಿಂದ ಹಂತದ ವಿವರಣೆ

ಗಣಿತ ಸಮಸ್ಯೆಯನ್ನು ಆಯ್ಕೆ ಮಾಡಿ ಮತ್ತು ಅದನ್ನು ಹಂತದಿಂದ ಹಂತದ ವಿವರಣೆ ಮತ್ತು ಕಡಿಮೆ ಆಸಕ್ತಿಯೊಂದಿಗೆ ಪ್ರಯತ್ನಿಸಿ. ಕಡಿಮೆ ಆಸಕ್ತಿ ಕೇವಲ ಉತ್ತರವನ್ನು ನೀಡುತ್ತದೆ - ತ್ವರಿತ ಆದರೆ अस्पಷ್ಟ. ಹಂತದಿಂದ ಹಂತ ವಿವರಣೆ ಪ್ರತಿಯೊಬ್ಬ ಲೆಕ್ಕಾಚಾರ ಮತ್ತು ನಿರ್ಧಾರವನ್ನು ತೋರಿಸುತ್ತದೆ.

## ನೀವು ನಿಜವಾಗಿಯೂ ಕಲಿತಿರುವುದು

**ವಿವೇಚನೆ ಪ್ರಯತ್ನದಿಂದ ಎಲ್ಲಾ ಬದಲಾಗುತ್ತದೆ**

GPT-5.2 ನಿಮ್ಮ ಸೂಚನೆಗಳ ಮೂಲಕ ಲೆಕ್ಕಾಚಾರದ ಪ್ರಯತ್ನವನ್ನು ನಿಯಂತ್ರಿಸಲು ಅನುಮತಿಸುತ್ತದೆ. ಕಡಿಮೆ ಪ್ರಯತ್ನ ಅಂದ್ರೆ ವಿನಿಶ್ಚಿತ ಅನ್ವೇಷಣೆ ಸಹಿತ ದ್ರುತ ಪ್ರತಿಕ್ರಿಯೆಗಳು. ಹೆಚ್ಚಿನ ಪ್ರಯತ್ನ ಅಂದರೆ ಮಾದರಿ ಗಟ್ಟಿಯಾಗಿ ಆಲೋಚಿಸಲು ಸಮಯ ತೆಗೆದುಕೊಳ್ಳುತ್ತದೆ. ನೀವು ಕಾರ್ಯದ ಸಂಕೀರ್ಣತೆಯೊಡನೆ ಪ್ರಯತ್ನವನ್ನು ಹೊಂದಿಸಲು ಕಲಿಯುತ್ತೀರಿ — ಸುಲಭ ಪ್ರಶ್ನೆಗಳಿಗೆ ಸಮಯ ವ್ಯರ್ಥ ಮಾಡದೆ, ಆದರೆ ಸಂಕೀರ್ಣ ನಿರ್ಧಾರಗಳನ್ನು ಕೂಡ ತುಪ್ಪು ಹೋಗದೆ ನಡೆಸಿರಿ.

**ಸರಚನೆ ವರ್ತನೆಗೆ ಮಾರ್ಗದರ್ಶನ ಮಾಡುತ್ತದೆ**

ಸೂಚನೆಗಳಲ್ಲಿ XML ಟ್ಯಾಗ್‌ಗಳನ್ನು ಗಮನಿಸಿ? ಅವು ಶೃಂಗಾರಿಕವಲ್ಲ. ಮಾದರಿಗಳು ರಚನಾತ್ಮಕ ಸೂಚನೆಗಳನ್ನು ಮುಕ್ತಾದರ್ಶ ಪಠ್ಯದ ಹೋಲಿಕೆಯಲ್ಲಿ ಹೆಚ್ಚು ನಂಬಿಕೆರಹಿತವಾಗಿ ಅನುಸರಿಸುತ್ತವೆ. ನೀವು ಬಹು ಹಂತದ ಪ್ರಕ್ರಿಯೆಗಳು ಅಥವಾ ಸಂಕೀರ್ಣ ತರ್ಕ ಬೇಕಾದಾಗ, ರಚನೆ ಮಾದರಿಯನ್ನು ಅದರ ಪ್ರಸ್ತಿತಿನಲ್ಲಿನ ಸ್ಥಾನ ಮತ್ತು ಮುಂದೇನಿದೆ ಎಂದು ಅನುಸರಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.

<img src="../../../translated_images/kn/prompt-structure.a77763d63f4e2f89.webp" alt="ಸೂಚನೆಯ ರಚನೆ" width="800"/>

*ಸ್ಪಷ್ಟ ವಿಭಾಗಗಳು ಮತ್ತು XML ಶೈಲಿಯ ಸಂಯೋಜನೆಯೊಂದಿಗೆ ಒಳ್ಳೆಯ ರಚನೆಯ ಸೂಚನೆಯ ರೂಪರೇಖೆ*

**ಗುಣಮಟ್ಟ ಸ್ವಯಂ-ಮೌಲ್ಯಮಾಪನದ ಮೂಲಕ**

ಸ್ವಯಂ-ಪರಿಶೀಲನೆ ಮಾದರಿಗಳು ಗುಣಮಟ್ಟ ಮಾನದಂಡಗಳನ್ನು ಸ್ಪಷ್ಟಪಡಿಸುವ ಮೂಲಕ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತವೆ. “ಮಾದರಿ ಸರಿಯಾಗಿ ಮಾಡುತ್ತದೆ” ಎಂದು ಭಾವಿಸುವ ಬದಲು, ನೀವು ಅದಕ್ಕೆ ಸ್ಪಷ್ಟವಾಗಿ “ಸರಿಯಾದ” ಅರ್ಥ: ಸರಿಯಾದ ತರ್ಕ, ದೋಷ ನಿರ್ವಹಣೆ, ಕಾರ್ಯಕ್ಷಮತೆ, ಭದ್ರತೆ ಇತ್ಯಾದಿಗಳನ್ನು ಹೇಳುತ್ತೀರಿ. ಆಗ ಮಾದರಿ ತನ್ನ ಸ್ವಂತ ಉತ್ಪನ್ನವನ್ನು ಮೌಲ್ಯಮಾಪನ ಮಾಡಿ ಸುಧಾರಿಸಬಹುದು. ಇದು ಕೋಡ್ ರಚನೆಯನ್ನು ಲಾಟರಿ ಹಣೆತುಂಬಿಸಿದ ಪ್ರಕ್ರಿಯೆಯನ್ನಾಗಿ ಮಾಡುತ್ತದೆ.

**ಸಂದರ್ಭ ಸೀಮಿತವಾಗಿದೆ**

ಬಹು-ತಿರುವು ಸಂವಾದಗಳು ಪ್ರತಿ ವಿನಂತಿಗೆ ಸಂದೇಶ ಇತಿಹಾಸವನ್ನು ಸೇರಿಸುವ ಮೂಲಕ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತವೆ. ಆದರೆ ಗಣನೆ ಮಿತಿ ಇದೆ - ಪ್ರತಿಯೊಬ್ಬ ಮಾದರಿಗೂ ಗರಿಷ್ಟ ಟೋಕನ್ ಎಣಿಕೆ ಇರುತ್ತದೆ. ಸಂವಾದಗಳು ಹೆಚ್ಚಾದಂತೆ, ನೀವು ಸಂಬಂಧಿತ ಸಂದರ್ಭವನ್ನು ಉಳಿಸಲು ತಂತ್ರಗಳನ್ನು ಬೇಕಾಗುತ್ತದೆ ಅದರ ಮಿತಿಯನ್ನು ತಾಳದೆ. ಈ ಘಟಕವು ನಿಮಗೆ ಮೆಮೊರಿ ಹೇಗೆ ಕೆಲಸ ಮಾಡುತ್ತದೆ ಎಂಬುದನ್ನು ತೋರಿಸುತ್ತದೆ; ನಂತರ ನೀವು ಯಾವಾಗ ಸಾರಾಂಶ ಮಾಡಬೇಕು, ಯಾವಾಗ ಮರೆತುಕೊಳ್ಳಬೇಕು ಮತ್ತು ಯಾವಾಗ ಪುನರುಪ್ರಾಪ್ತಿ ಮಾಡಬೇಕು ಎಂದು ಕಲಿಯುವಿರಿ.
