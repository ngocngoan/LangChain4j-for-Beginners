## ನೀವು ಕಲಿಯುವುದೇನು

<img src="../../../translated_images/kn/what-youll-learn.c68269ac048503b2.webp" alt="ನೀವು ಕಲಿಯುವುದೇನು" width="800"/>

ಹಿಂದಿನ ಮಾಲಿಕೆಯಲ್ಲಿ, ನೀವು ಸ್ಮರಣೆ ಹೇಗೆ ಸಂವಾದಾತ್ಮಕ AI ಅನ್ನು ಸಕ್ರಿಯಗೊಳಿಸುತ್ತದೆ ಎಂದು ನೋಡಿದ್ದೀರಿ ಮತ್ತು ಮೂಲ ಸಂವಹನಗಳಿಗೆ GitHub ಮಾದರಿಗಳು ಬಳಸಿದ್ದೀರಿ. ಈಗ ನಾವು ಹೇಗೆ ಪ್ರಶ್ನೆಗಳನ್ನು ಕೇಳ್ತೇವೆ — ಅಂದರೆ ಪ್ರಾಂಪ್ಟ್ ಗಳೇ — Azure OpenAI ನ GPT-5.2 ಬಳಸಿ. ನೀವು ಪ್ರಾಂಪ್ಟ್ ಗಳನ್ನು ರಚಿಸುವ ವಿಧಾನವು ನೀವು ಪಡೆಯುವ ಪ್ರತಿಕ್ರಿಯೆಗಳ ಗುಣಮಟ್ಟವನ್ನು ಹೆಚ್ಚಾಗಿ ಪ್ರಭಾವಿತ ಮಾಡುತ್ತದೆ. ನಾವು ಮೂಲಭೂತ ಪ್ರಾಂಪ್ಟ್ ತಂತ್ರಗಳನ್ನು ಪರಿಗಣಿಸುವುದರಿಂದ ಪ್ರಾರಂಭಿಸಿ, ನಂತರ GPT-5.2 ಅವರ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಸಂಪೂರ್ಣ ಉಪಯೋಗಿಸುವ ಎಂಟು ಅಗ್ರಗಣ್ಯ ಮಾದರಿಗಳತ್ತ ಸಾಗುತ್ತೇವೆ.

ನಾವು GPT-5.2 ಬಳಸುತ್ತೇವೆ ಏಕೆಂದರೆ ಇದು ಯುಕ್ತಿವಾದ ನಿಯಂತ್ರಣವನ್ನು ಪರಿಚಯಿಸುತ್ತದೆ - ನೀವು ಮಾದರಿಗೆ ಉತ್ತರಿಸುವ ಮೊದಲು ಎಷ್ಟು ಯೋಚನೆ ಮಾಡುವುದೆಂದು ಹೇಳಬಹುದು. ಇದರಿಂದ ವಿವಿಧ ಪ್ರಾಂಪ್ಟಿಂಗ್ ತಂತ್ರಗಳು ಸ್ಪಷ್ಟವಾಗುತ್ತವೆ ಮತ್ತು ಯಾವWHEN ಮಾಹಿತಿ ಹುಡುಕಬೇಕು ಎಂಬುದನ್ನು ನಿಖರವಾಗಿ ತಿಳಿದುಕೊಳ್ಳಬಹುದು. GitHub ಮಾದರಿಗಳಿಗಿಂತ Azure ನ ಕಡಿಮೆ ದರ ನಿರ್ಬಂಧಗಳಿಂದಲೂ ನಾವು ಲಾಭ ಪಡೆಯುತ್ತೇವೆ.

## ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಅನ್ನು ತಿಳಿದುಕೊಳ್ಳುವುದು

<img src="../../../translated_images/kn/what-is-prompt-engineering.5c392a228a1f5823.webp" alt="ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಎಂದರೆ ಏನು?" width="800"/>

ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಎಂದರೆ ನಿಮಗೆ ಬೇಕಾದ ಫಲಿತಾಂಶಗಳನ್ನು ಸದೃಢವಾಗಿ ಪಡೆಯಲು ಇನ್‌ಪುಟ್ ಪಠ್ಯವನ್ನು ವಿನ್ಯಾಸಗೊಳಿಸುವುದು. ಇದು ಕೇವಲ ಪ್ರಶ್ನೆಗಳನ್ನು ಕೇಳುವುದಲ್ಲ — ಇದು ಮಾದರಿಯು ನಿಮಗೆ ಏನು ಬೇಕೆಂದು ಬಲವಾಗಿ ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು ಮತ್ತು ಅದನ್ನು ಹೇಗೆ ನೀಡಬೇಕು ಎಂಬುದನ್ನು ಸರಿಯಾಗಿ ವಿನ್ಯಾಸಗೊಳಿಸುವುದರ ಬಗ್ಗೆ.

ನೀವು ಒಂದು ಸಹೋದ್ಯೋಗಿಗೆ ಸೂಚನೆ ನೀಡುತ್ತಿರುವಂತೆ ಏನು ಎಂದುಕೊಳ್ಳಿ. "ಬಗ್ ಸರಿಪಡಿಸಿ" ಎಂಬುದು ಸ್ಪಷ್ಟವಲ್ಲದ ಸೂಚನೆ. "UserService.java ಲೈನ್ 45 ರಲ್ಲಿ ನೋಲ್ ಪತ್ರಿಕೆಯ ತಪ್ಪನ್ನು ನೋಲ್ ಪರಿಶೀಲನೆಯನ್ನು ಸೇರ್ಪಡೆ ಮಾಡುವ ಮೂಲಕ ಸರಿಪಡಿಸಿ" ಎಂಬುದು ವಿಶೇಷವಾಗಿ ವಿವರಣೆ ನೀಡುತ್ತದೆ. ಭಾಷಾ ಮಾದರಿಗಳು ಇಷ್ಟು ವೈಶಿಷ್ಟ್ಯಪೂರ್ಣ ಮತ್ತು ರಚನಾತ್ಮಕವಾಗಿರಬೇಕು.

<img src="../../../translated_images/kn/how-langchain4j-fits.dfff4b0aa5f7812d.webp" alt="ಲ್ಯಾಂಗ್ಚೇನ್4ಜೆ ಹೇಗೆ ಹೊಂದಿಕೊಳುವುದೆ" width="800"/>

LangChain4j ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ — ಮಾದರಿ ಸಂಪರ್ಕಗಳು, ಸ್ಮರಣೆ ಮತ್ತು ಸಂದೇಶ ಪ್ರಕಾರಗಳು — ಒದಗಿಸುತ್ತದೆ, ಪ್ರಾಂಪ್ಟ್ ಮಾದರಿಗಳು ಆ ಇನ್ಫ್ರಾಸ್ಟ್ರಕ್ಚರ್ ಮೂಲಕ ನೀವು ಕಳುಹಿಸುವ ನಿಖರವಾಗಿ ರಚಿಸಲಾದ ಪಠ್ಯ ಮಾತ್ರ. ಮುಖ್ಯ ಅಂಶಗಳು `SystemMessage` (AI ನ ವರ್ತನೆ ಮತ್ತು ಪಾತ್ರವನ್ನು ಸેટುಮಾಡುವುದು) ಮತ್ತು `UserMessage` (ನಿಮ್ಮ ನಿಜವಾದ ವಿನಂತಿಯನ್ನು ಒಗೆಯುವುದು) ಆಗಿವೆ.

## ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಮೂಲಭೂತಾಂಶಗಳು

<img src="../../../translated_images/kn/five-patterns-overview.160f35045ffd2a94.webp" alt="ಐದು ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಮಾದರಿಗಳ ವೀಕ್ಷಣೆ" width="800"/>

ಈ ಮಾಲಿಕೆಯಲ್ಲಿ ಅಗ್ರಗಣ್ಯ ಮಾದರಿಗಳಲ್ಲಿ ಹೋಗುವ ಮೊದಲು, ಐದು ಮೂಲಭೂತ ಪ್ರಾಂಪ್ಟ್ ತಂತ್ರಗಳನ್ನು ಪರಿಶೀಲಿಸೋಣ. ಇವು ಪ್ರತಿ ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರ್ ತಿಳಿದುಕೊಳ್ಳಬೇಕಾದ ನಿರ್ಮಾಣ ಅವಯವಗಳು. ನೀವು ಈಗಾಗಲೇ [ಕ್ವಿಕ್ ಸ್ಟಾರ್ಟ್ ಮಾಲಿಕೆ](../00-quick-start/README.md#2-prompt-patterns) ಅಡಿಯಲ್ಲಿ ಇದನ್ನು ಅನುಭವಿಸಿದ್ದೀರಾ — ಇವುಗಳ ಪেছಿಮುಖದಲ್ಲಿ ತತ್ವಚಿಂತನೆ ಇದೆ.

### ಝೆರೋ-ಶಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್

ಸರಳ ವಿಧಾನ: ಮಾದರಿಗೆ ಯಾವುದೇ ಉದಾಹರಣೆಗಳಿಲ್ಲದೆ ನೇರ ಸೂಚನೆ ನೀಡಿ. ಮಾದರಿ ಸಂಪೂರ್ಣವಾಗಿ ತನ್ನ ತರಬೇತಿ ಮೇಲೆ ನಿರ್ಭರಿಸಿ ಕಾರ್ಯವನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುತ್ತದೆ ಮತ್ತು ನಿಭಾಯಿಸುತ್ತದೆ. ನಿರೀಕ್ಷಿತ ವರ್ತನೆ ಸ್ಪಷ್ಟವಾಗುವ ಸರಳ ವಿನಂತಿಗಳಿಗೆ ಇದು ಚೆನ್ನಾಗಿರುತ್ತದೆ.

<img src="../../../translated_images/kn/zero-shot-prompting.7abc24228be84e6c.webp" alt="ಝೆರೋ-ಶಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್" width="800"/>

*ಉದಾಹರಣೆಗಳಿಲ್ಲದ ನೇರ ಸೂಚನೆ — ಮಾದರಿ ಸೂಚನೆಯಿಂದಲೇ ಕಾರ್ಯವನ್ನು ಗ್ರಹಿಸುತ್ತದೆ*

```java
String prompt = "Classify this sentiment: 'I absolutely loved the movie!'";
String response = model.chat(prompt);
// ಪ್ರತಿಕ್ರಿಯೆ: "ಮೊತ್ತದ"
```

**ಬಳಸಬೇಕಾಗುವಾಗ:** ಸರಳ ವರ್ಗೀಕರಣಗಳು, ನೇರ ಪ್ರಶ್ನೆಗಳು, ಅನುವಾದಗಳು ಅಥವಾ ಯಾವುದೇ ರೀತಿಯ ಮಾರ್ಗದರ್ಶನವಿಲ್ಲದ ಕಾರ್ಯಗಳಿಗೆ.

### ಫ್ಯೂ-ಶಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್

ನೀವು ಮಾದರಿ ಅನುಸರಿಸಬೇಕಾದ ಮಾದರಿಯನ್ನು ತೋರಿಸುವ ಉದಾಹರಣೆಗಳನ್ನು ಒದಗಿಸಿ. ಮಾದರಿ ನಿಮ್ಮ ಉದಾಹರಣೆಗಳಿಂದ ನಿರೀಕ್ಷಿತ ಇನ್‌ಪುಟ್-ಔಟ್‌ಪುಟ್ ಸ್ವರೂಪವನ್ನು ಕಲಿಯುತ್ತದೆ ಮತ್ತು ಹೊಸ ಇನ್‌ಪುಟ್‌ಗಳಿಗೆ ಅದನ್ನು ಅನ್ವಯಿಸುತ್ತದೆ. ಇದು ಫಾರ್ಮ್ಯಾಟ್ ಅಥವಾ ವರ್ತನೆಯು ಸ್ಪಷ್ಟವಾಗದ ಕಾರ್ಯಗಳಿಗೆ ಸಾಮರಸ್ಯವನ್ನು劇ವಾಗಿ ಸುಧಾರಿಸುತ್ತದೆ.

<img src="../../../translated_images/kn/few-shot-prompting.9d9eace1da88989a.webp" alt="ಫ್ಯೂ-ಶಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್" width="800"/>

*ಉದಾಹರಣೆಗಳಿಂದ ಕಲಿಯುವುದು — ಮಾದರಿ ಮಾದರಿಯನ್ನು ಗುರುತಿಸಿ ಹೊಸ ಇನ್‌ಪುಟ್‌ಗಳಿಗೆ ಅನ್ವಯಿಸುತ್ತದೆ*

```java
String prompt = """
    Classify the sentiment as positive, negative, or neutral.
    
    Examples:
    Text: "This product exceeded my expectations!" → Positive
    Text: "It's okay, nothing special." → Neutral
    Text: "Waste of money, very disappointed." → Negative
    
    Now classify this:
    Text: "Best purchase I've made all year!"
    """;
String response = model.chat(prompt);
```

**ಬಳಸಬೇಕಾಗುವಾಗ:** ಕಸ್ಟಮ್ ವರ್ಗೀಕರಣಗಳು, ಸಮಾನ ಫಾರ್ಮ್ಯಾಟಿಂಗ್, ಕ್ಷೇತ್ರ-ನಿರ್ದಿಷ್ಟ ಕಾರ್ಯಗಳು, ಅಥವಾ ಝೆರೋ-ಶಾಟ್ ಫಲಿತಾಂಶಗಳು ಅಸಮರ್ಪಕವಾದ ಸಮಯದಲ್ಲಿ.

### ಚೈನ್ ಆಫ್ ಥಾಟ್

ಮಾದರಿಯಿಂದ ತನ್ನ ಯುಕ್ತಿವಾದವನ್ನು ಹಂತದ ಮೂಲಕ ತೋರಿಸಲು ಕೇಳಿ. ನೇರ ಉತ್ತರಕ್ಕೆ ಕೊಡೆ ಬಂದುತ್ತೊಳದ ಬದಲಾಗಿ, ಮಾದರಿ ಸಮಸ್ಯೆಯನ್ನು ಭಾಗಭಾಗವಾಗಿ ಭೇದಿಸಿ ಸ್ಪಷ್ಟವಾಗಿ ಕೆಲಸಮಾಡುತ್ತದೆ. ಇದರ ಮೂಲಕ ಗಣಿತ, ತರ್ಕ ಮತ್ತು ಬಹು ಹಂತ ಯುಕ್ತಿವಾದ ಕಾರ್ಯಗಳಲ್ಲಿ ಖಚಿತತೆ ಸಾಧಿಸಲಾಗುತ್ತದೆ.

<img src="../../../translated_images/kn/chain-of-thought.5cff6630e2657e2a.webp" alt="ಚೈನ್ ಆಫ್ ಥಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್" width="800"/>

*ಹಂತದ ಮೂಲಕ ಯುಕ್ತಿವಾದ — ಸಂಕೀರ್ಣ ಸಮಸ್ಯೆಗಳ ಬಗ್ಗೆ ಸ್ಪಷ್ಟ ತರ್ಕ ಹಂತಗಳನ್ನು ಒದಗಿಸುತ್ತಿದೆ*

```java
String prompt = """
    Problem: A store has 15 apples. They sell 8 apples and then 
    receive a shipment of 12 more apples. How many apples do they have now?
    
    Let's solve this step-by-step:
    """;
String response = model.chat(prompt);
// ಮಾದರಿ ತೋರಿಸುತ್ತದೆ: 15 - 8 = 7, ನಂತರ 7 + 12 = 19 ಸೇಬು
```

**ಬಳಸಬೇಕಾಗುವಾಗ:** ಗಣಿತ ಸಮಸ್ಯೆಗಳು, ತರ್ಕ ಪಜಲ್‌ಗಳು, ಡಿಬಗ್ಗಿಂಗ್, ಅಥವಾ ಯುಕ್ತಿವಾದ ಪ್ರಕ್ರಿಯೆ ತೋರಿಸುವಾಗ ವಿಶ್ವಾಸಾರ್ಹತೆ ಮತ್ತು ಖಚಿತತೆ ಸುಧಾರಿಸುತ್ತದೆ.

### ಪಾತ್ರಾಧಾರಿತ ಪ್ರಾಂಪ್ಟಿಂಗ್

ಪ್ರಶ್ನೆ ಕೇಳುವ ಮೊದಲು AI ಗೆ ಒಂದು ವ್ಯಕ್ತಿತ್ವ ಅಥವಾ ಪಾತ್ರವನ್ನು ನಿಗದಿಪಡಿಸಿ. ಇದು ಪ್ರತಿಕ್ರಿಯೆಯ ಧ್ವನಿ, ಆಳ ಹಾಗೂ ಕೇಂದ್ರೀಕರಣವನ್ನು ರೂಪಿಸುವ ಸಂದರ್ಭ ನೀಡುತ್ತದೆ. "ಸಾಫ್ಟ್‌ವೇರ್ ಸ್ಥಾಪಕ" "ಜೂನಿಯರ್ ಡೆವಲಪರ್" ಅಥವಾ "ಭದ್ರತಾ ವಿಮರ್ಶಕ" ಗಳಿಗಿಂತ ವಿಭಿನ್ನ ಸಲಹೆ ನೀಡಬಹುದು.

<img src="../../../translated_images/kn/role-based-prompting.a806e1a73de6e3a4.webp" alt="ಪಾತ್ರಾಧಾರಿತ ಪ್ರಾಂಪ್ಟಿಂಗ್" width="800"/>

*ಸಂದರ್ಭ ಮತ್ತು ಪಾತ್ರ ನಿಗದಿಪಡಿಸುವುದು — ನೀಡಿದ ಪಾತ್ರ ಆಧಾರಿತವಾಗಿ ಒಂದೇ ಪ್ರಶ್ನೆಗೆ ವಿಭಿನ್ನ ಪ್ರತಿಕ್ರಿಯೆಗಳು*

```java
String prompt = """
    You are an experienced software architect reviewing code.
    Provide a brief code review for this function:
    
    def calculate_total(items):
        total = 0
        for item in items:
            total = total + item['price']
        return total
    """;
String response = model.chat(prompt);
```

**ಬಳಸಬೇಕಾದಾಗ:** ಕೋಡ್ ವಿಮರ್ಶೆಗಳು, ಟ್ಯೂಟೋರಿಂಗ್, ಕ್ಷೇತ್ರ-ನಿರ್ದಿಷ್ಠ ವಿಶ್ಲೇಷಣೆ, ಅಥವಾ ನಿರ್ದಿಷ್ಟ ಪರಿಣಿತಿಗೆ ಹೊಂದಿಕೊಂಡ ಪ್ರತಿಕ್ರಿಯೆಗಳ ಅಗತ್ಯವಿರುವಾಗ.

### ಪ್ರಾಂಪ್ಟ್ ಟೆಂಪ್ಲೇಟುಗಳು

ಚಾರಣೀಯ ಪ್ರಾಂಪ್ಟ್‌ಗಳಿಗಾಗಿ Placeholder variable ಗಳನ್ನು ಹೊಂದಿರುವ ಮರುಬಳಕೆ ಮಾಡಬಹುದಾದ ಟೆಂಪ್ಲೇಟುಗಳನ್ನು ರಚಿಸಿ. ಪ್ರತಿ ಬಾರಿ ಹೊಸ ಪ್ರಾಂಪ್ಟ್ ಬರೆಯುವುದು ಬದಲು ಒಂದೇ ಟೆಂಪ್ಲೇಟು ಅನ್ನು ಮೇಲ್ಮನವಿ ಮೌಲ್ಯಗಳೊಂದಿಗೆ ಬಳಕೆಮಾಡಿ. LangChain4j ನ `PromptTemplate` ಕ್ಲಾಸ್ ಇದನ್ನು `{{variable}}` ಸಿಂಟ್ಯಾಕ್ಸ್ ಮೂಲಕ ಸುಲಭ ಮಾಡುತ್ತದೆ.

<img src="../../../translated_images/kn/prompt-templates.14bfc37d45f1a933.webp" alt="ಪ್ರಾಂಪ್ಟ್ ಟೆಂಪ್ಲೇಟುಗಳು" width="800"/>

*Placeholder variable ಗಳೊಂದಿಗೆ ಮರುಬಳಕೆ ಪ್ರಾಂಪ್ಟ್‌ಗಳು — ಒಂದು ಟೆಂಪ್ಲೇಟು, ಅನೇಕ ಉಪಯೋಗಗಳು*

```java
PromptTemplate template = PromptTemplate.from(
    "What's the best time to visit {{destination}} for {{activity}}?"
);

Prompt prompt = template.apply(Map.of(
    "destination", "Paris",
    "activity", "sightseeing"
));

String response = model.chat(prompt.text());
```

**ಬಳಸಬೇಕಾಗುವಾಗ:** ವಿಭಿನ್ನ ಇನ್‌ಪುಟ್ಗಳ್ಳೊಂದಿಗಿನ ಪುನರಾವರ್ತಿತ ಪ್ರಶ್ನೆಗಳು, ಬ್ಯಾಚ್ ಪ್ರಾಸೆಸಿಂಗ್, ಮರುಬಳಕೆ ಅವಕಾಶವಿರುವ AI ವರ್ಕ್‌ಫ್ಲೋಗಳು, ಅಥವಾ ಪ್ರಾಂಪ್ಟ್ ರಚನೆ ಸಮಾನವಾಗಿದ್ದಾಗ ಮಾತ್ರ ಡೇಟಾಗಳು ಬದಲಾಗುವುದು.

---

ಈ ಐದು ಮೂಲಭೂತಾಂಶಗಳು ಬಹುತೇಕ ಪ್ರಾಂಪ್ಟಿಂಗ್ ಕಾರ್ಯಗಳಿಗೆ ಭದ್ರ ಸಾಧನ ಸಂಪತ್ತು ಒದಗಿಸುತ್ತವೆ. ಈ ಮಾಲಿಕೆಯ ಉಳಿದ ಭಾಗವು GPT-5.2 ನ ಯುಕ್ತಿವಾದ ನಿಯಂತ್ರಣ, ಸ್ವಯಂ ಮೌಲ್ಯಮಾಪನ ಮತ್ತು ಸಂರಚಿತ ಔಟ್‌ಪುಟ್ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಪೂರ್ತಿಯಾಗಿ ಉಪಯೋಗಿಸುವ **ಎಂಟು ಅಗ್ರಗಣ್ಯ ಮಾದರಿಗಳ**ತ್ತ ಸಾಗುತ್ತದೆ.

## ಅಗ್ರಗಣ್ಯ ಮಾದರಿಗಳು

ಮೂಲಭೂತಾಂಶಗಳನ್ನು ಮುಗಿಸಿದ ಮೇಲೆ, ಈ ಮಾಲಿಕೆಯನ್ನು ವಿಶಿಷ್ಟಗೊಳಿಸುವ ಎಂಟು ಅಗ್ರಗಣ್ಯ ಮಾದರಿಗಳತ್ತ ಹೋಗೋಣ. ಎಲ್ಲ ಪದ್ದತಿಗಳು ಒಂದೇ ರೀತಿಯಾದ ತಂತ್ರವನ್ನು ಬೇಕಾಗುವುದಿಲ್ಲ. ಕೆಲವು ಪ್ರಶ್ನೆಗಳು ತಕ್ಷಣದ ಉತ್ತರ ಬೇಕಾಗುತ್ತದೆ, ಇನ್ನೆರಡು ಆಳವಾದ ಯೋಚನೆಗೆ. ಕೆಲವಿಷ್ಟು ದೃಶ್ಯಯುಕ್ತ ಯುಕ್ತಿವಾದ ಬೇಕಾಗುತ್ತದೆ, ಬೇರೆಯೊಂದು ಫಲಿತಾಂಶ ಬೇಕಾಗುತ್ತದೆ. ಕೆಳಗಿನ ಪ್ರತಿ ಮಾದರಿ ವಿಭಿನ್ನ ಸಂದರ್ಭಕ್ಕಾಗಿ ಸೂತ್ರಗೊಳ್ಳಲಾಯಿತು — ಮತ್ತು GPT-5.2 ಯುಕಿತ್ವ ನಿಯಂತ್ರಣವು ವ್ಯತ್ಯಾಸಗಳನ್ನು ಇನ್ನಷ್ಟು ಸ್ಪಷ್ಟಗೊಳಿಸುತ್ತದೆ.

<img src="../../../translated_images/kn/eight-patterns.fa1ebfdf16f71e9a.webp" alt="ಎಂಟು ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಮಾದರಿಗಳು" width="800"/>

*ಎಂಟು ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಮಾದರಿಗಳು ಮತ್ತು ಅವುಗಳ ಉಪಯೋಗಗಳು ವೀಕ್ಷಣೆ*

<img src="../../../translated_images/kn/reasoning-control.5cf85f0fc1d0c1f3.webp" alt="GPT-5.2 ನೊಂದಿಗೆ ಯುಕ್ತಿವಾದ ನಿಯಂತ್ರಣ" width="800"/>

*GPT-5.2 ಯುಕ್ತಿವಾದ ನಿಯಂತ್ರಣ ನಿಮಗೆ ಮಾದರಿ ಎಷ್ಟು ಯೋಚನೆ ಮಾಡಬೇಕೆಂದು ಕಳುಹಿಸುವ ಅವಕಾಶವನ್ನು ನೀಡುತ್ತದೆ — ಸ್ತಂಭವಾಗಿ ನೇರ ಉತ್ತರಗಳಿಂದ ಆಳವಾದ ಪರಿಶೀಲನೆಗಳವರೆಗೆ*

**ಕಡಿಮೆ ಆಸಕ್ತಿ (ವೇಗವಾಗಿ ಮತ್ತು ಧ್ಯಾನದಿಂದ)** - ಸರಳ ಪ್ರಶ್ನೆಗಳಿಗಾಗಿ ನೀವು ವೇಗವಾಗಿ ಮತ್ತು ನೇರ ಉತ್ತರಗಳನ್ನು ಬೇಕಾದಾಗ. ಮಾದರಿ ಕನಿಷ್ಠ ಯುಕ್ತಿವಾದ ಮಾಡುತ್ತದೆ - ಗರಿಷ್ಟ 2 ಹಂತಗಳು. ಗಣನೆಗಳು, ಲುಕ್ಅಪ್ಗಳ ಅಥವಾ ಸರಳ ಪ್ರಶ್ನೆಗಳಿಗೆ ಬಳಸಿರಿ.

```java
String prompt = """
    <context_gathering>
    - Search depth: very low
    - Bias strongly towards providing a correct answer as quickly as possible
    - Usually, this means an absolute maximum of 2 reasoning steps
    - If you think you need more time, state what you know and what's uncertain
    </context_gathering>
    
    Problem: What is 15% of 200?
    
    Provide your answer:
    """;

String response = chatModel.chat(prompt);
```

> 💡 **GitHub Copilot ನೊಂದಿಗೆ ಅನ್ವೇಷಿಸಿ:** [`Gpt5PromptService.java`](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java) ತೆರೆಯಿರಿ ಮತ್ತು ಕೇಳಿ:
> - "ಕಡಿಮೆ ಆಸಕ್ತಿ ಮತ್ತು ಹೆಚ್ಚು ಆಸಕ್ತಿ ಪ್ರಾಂಪ್ಟ್ ಮಾದರಿಗಳ有什么 ವ್ಯತ್ಯಾಸ?"
> - "ಪ್ರಾಂಪ್ಟಿನಲ್ಲಿನ XML ಟ್ಯಾಗ್ಗಳು AI ಪ್ರತಿಕ್ರಿಯೆಯನ್ನು ಹೇಗೆ ವಿನ್ಯಾಸ ಮಾಡುತ್ತವೆ?"
> - "ಸ್ವಯಂ-ಪರಿಶೀಲನೆ ಮಾದರಿಗಳನ್ನು ಮತ್ತು ನೇರ ಸೂಚನೆಯನ್ನು ಯಾವಾಗ ಬಳಸಬೇಕು?"

**ಹೆಚ್ಚು ಆಸಕ್ತಿ (ಆಳವಾದ ಮತ್ತು ಸಂಪೂರ್ಣ)** - ಸಂಕೀರ್ಣ ಸಮಸ್ಯೆಗಳಿಗಾಗಿ, ನೀವು ಸಮಗ್ರ ವಿಶ್ಲೇಷಣೆ ಬಯಸುವಾಗ. ಮಾದರಿ ಸಂಪೂರ್ಣವಾಗಿ ಅನ್ವೇಷಿಸಿ ವಿಶದ ಯುಕ್ತಿವಾದ ತೋರಿಸುತ್ತದೆ. ವ್ಯವಸ್ಥಾ ವಿನ್ಯಾಸ, ಸ್ಥಾಪನ ನಿರ್ಣಯಗಳು ಅಥವಾ ಸಂಕೀರ್ಣ ಸಂಶೋಧನೆಗೆ ಇದನ್ನು ಬಳಸಿ.

```java
String prompt = """
    Analyze this problem thoroughly and provide a comprehensive solution.
    Consider multiple approaches, trade-offs, and important details.
    Show your analysis and reasoning in your response.
    
    Problem: Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```

**ಕಾರ್ಯ ನಿರ್ವಹಣೆ (ಹಂತ ಹಂತದ ಪ್ರಗತಿ)** - ಬಹು ಹಂತದ ಕಾರ್ಯಪ್ರವಾಹಗಳಿಗಾಗಿ. ಮಾದರಿ ಪೂರ್ವ ಯೋಜನೆಯನ್ನು ಒದಗಿಸುತ್ತದೆ, ಪ್ರತಿ ಹಂತವನ್ನು ವಿವರವಾಗಿ ವಿವರಿಸುತ್ತದೆ, ನಂತರ ಸಾರಾಂಶ ಒದಗಿಸುತ್ತದೆ. ವಲಸೆ, ಅನುಷ್ಠಾನಗಳು ಅಥವಾ ಯಾವುದೇ ಬಹು ಹಂತ ಪ್ರಕ್ರಿಯೆಗೆ ಇದನ್ನು ಬಳಸಿ.

```java
String prompt = """
    <task_execution>
    1. First, briefly restate the user's goal in a friendly way
    
    2. Create a step-by-step plan:
       - List all steps needed
       - Identify potential challenges
       - Outline success criteria
    
    3. Execute each step:
       - Narrate what you're doing
       - Show progress clearly
       - Handle any issues that arise
    
    4. Summarize:
       - What was completed
       - Any important notes
       - Next steps if applicable
    </task_execution>
    
    <tool_preambles>
    - Always begin by rephrasing the user's goal clearly
    - Outline your plan before executing
    - Narrate each step as you go
    - Finish with a distinct summary
    </tool_preambles>
    
    Task: Create a REST endpoint for user registration
    
    Begin execution:
    """;

String response = chatModel.chat(prompt);
```

ಚೈನ್ ಆಫ್ ಥಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್ ಸ್ಪಷ್ಟವಾಗಿ ಮಾದರಿಯಿಂದ ತನ್ನ ಯುಕ್ತಿವಾದ ಪ್ರಕ್ರಿಯೆಯನ್ನು ತೋರಿಸಲು ಕೇಳುತ್ತದೆ, ಸಂಕೀರ್ಣ ಕಾರ್ಯಗಳ ಖಚಿತತೆಯನ್ನು ಸುಧಾರಿಸುತ್ತದೆ. ಹಂತ-ಹಂತದ ವಿಭಜನೆ ಮಾನವರೂ ಮತ್ತು AI ರೂಯುಕ್ತರನ್ನು ತರ್ಕವನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.

> **🤖 GitHub Copilot Chat ನೊಂದಿಗೆ ಪ್ರಯತ್ನಿಸಿ:** ಈ ಮಾದರಿಯನ್ನು ಕುರಿತು ಕೇಳಿ:
> - "ದೀರ್ಘ ಅವಧಿಯ ಕಾರ್ಯಗಳುಗಾಗಿ ಕಾರ್ಯ ನಿರ್ವಹಣೆ ಮಾದರಿಯನ್ನು ನಾನು ಹೇಗೆ ಹೊಂದಿಸಬೇಕು?"
> - "ಉತ್ಫಾದನಾ ಅಪ್ಲಿಕೇಶನ್‌ಗಳಲ್ಲಿ ಉಪಕರಣ ಪ್ರೀಾಂಬಲ್ಸ್ ರಚನೆಗೆ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು ಯಾವುವು?"
> - "UI ನಲ್ಲಿ ಮಧ್ಯಂತರ ಪ್ರಗತಿ ನವೀಕರಣಗಳನ್ನು ನಾನು ಹೇಗೆ ಸೆರೆಹಿಡಿದು ಪ್ರದರ್ಶಿಸಬಹುದು?"

<img src="../../../translated_images/kn/task-execution-pattern.9da3967750ab5c1e.webp" alt="ಕಾರ್ಯ ನಿರ್ವಹಣೆ ಮಾದರಿ" width="800"/>

*ಯೋಚನೆ → ಕೈಗೊಳ್ಳು → ಸಾರಾಂಶ ಬಹು- ಹಂತದ ಕಾರ್ಯಪ್ರವಾಹ*

**ಸ್ವಯಂ-ಪರಿಶೀಲನಾ ಕೋಡ್** - ಉತ್ಪಾದನಾ ಗುಣಮಟ್ಟದ ಕೋಡ್ ರಚನೆಗೆ. ಮಾದರಿ ಉತ್ಪಾದನಾ ಮಾನದಂಡಗಳ ಪ್ರಕಾರ ಕೋಡ್ ರಚನೆ ಮಾಡುತ್ತದೆ, ಸೂಕ್ತ ದೋಷ ನಿರ್ವಹಣೆ ಸಹಿತ. ಹೊಸ ವೈಶಿಷ್ಟ್ಯಗಳು ಅಥವಾ ಸೇವೆಗಳಿಗೆ ಇದನ್ನು ಬಳಸಿ.

```java
String prompt = """
    Generate Java code with production-quality standards: Create an email validation service
    Keep it simple and include basic error handling.
    """;

String response = chatModel.chat(prompt);
```

*ಪರಿವೀಕ್ಷಣೆಯ ಮರುಪದವಿ - ರಚಿಸಿ, ಮೌಲ್ಯಮಾಪನ ಮಾಡು, ಸಮಸ್ಯೆ ಗುರುತುಮಾಡು, ಸುಧಾರಣೆ ಮಾಡಿ, ಪುನರಾವರ್ತನೆ*

**ಸಂರಚಿತ ವಿಶ್ಲೇಷಣೆ** - ಸ್ಥಿರ ಮೌಲ್ಯಮಾಪನಕ್ಕಾಗಿ. ಮಾದರಿ ಕೋಡ್ ಅನ್ನು ಸ್ಥಿರ ರೂಪರೇಖೆಯೊಂದಿಗೆ ವಿಮರ್ಶಿಸುತ್ತದೆ (ನಿಖರತೆ, ಅಭ್ಯಾಸಗಳು, ಕಾರ್ಯಕ್ಷಮತೆ, ಭದ್ರತೆ, ನಿರ್ವಹಣಾ ಸಾಮರ್ಥ್ಯ). ಕೋಡ್ ವಿಮರ್ಶೆ ಅಥವಾ ಗುಣಮಟ್ಟ ಮೌಲ್ಯಮಾಪನಗಳಿಗೆ ಇದನ್ನು ಬೆಯಲ್ಲಿ.

```java
String prompt = """

    You are an expert code reviewer. Analyze the code for:
    ```

> - "ವಿಭಿನ್ನ ರೀತಿಯ ಕೋಡ್ ವಿಮರ್ಶೆಗಳಿಗಾಗಿ ವಿಶ್ಲೇಷಣಾ ಫ್ರೇಮ್‌ವೇರ್ಮ್ ಅನ್ನು ನಾನು ಹೇಗೆ ಕಸ್ಟಮೈಸ್ ಮಾಡಬಹುದು?"
> - "ಸಂರಚಿತ ಔಟ್‌ಪುಟ್ ಅನ್ನು ಪ್ರೋಗ್ರಾಮ್ಮ್ಯಾಟಿಕಾಗಿ ವಿಶ್ಲೇಷಿಸಿ ಕ್ರಿಯೆಗೊಳಿಸುವ ಉತ್ತಮ ಮಾರ್ಗ ಯಾವುದು?"
> - "ಕೊನೆಯ ವಿಮರ್ಶಾ ಸತ್ರಗಳಾದ್ಯಂತ ಸತತ ಗಂಭೀರತೆ ಮಟ್ಟಗಳಾಗಿ ನಾನು ಹೇಗೆ ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಬಹುದು?"

*ಗಂಭೀರತೆ ಮಟ್ಟಗಳನ್ನು ಹೊಂದಿಸಿ ಸಮಾನ ಕೋಡ್ ವಿಮರ್ಶೆಗಳ ಫ್ರೇಮ್‌ವೇರ್ಮ್*

**ಬಹು-ತಿರುವು ಚಾಲನ್** - ಸಂದರ್ಶನೆಗಳಿಗೆ ನಿರಂತರ ಸನ್ನಿವೇಶ ಬೇಕಾದಾಗ. ಮಾದರಿ ಹಿಂದಿನ ಸಂದೇಶಗಳನ್ನು ನೆನಪಿಡುತ್ತದೆ ಮತ್ತು ಅದಕ್ಕನುಗುಣವಾಗಿ ಜೋಡಿಸುತ್ತದೆ. ಪಾಠ ಮಾದರಿ ಸಹಾಯ ಕಾರ್ಯಗಳು ಅಥವಾ ಸಂಕೀರ್ಣ ಪ್ರಶ್ನೋತ್ತರಗಳಿಗೆ ಇದನ್ನು ಬಳಸಿ.

**ಹಂತದ ಮೂಲಕ ಯುಕ್ತಿವಾದ** - ದೃಶ್ಯ ತರ್ಕದ ಅಗತ್ಯವಿರುವ ಸಮಸ್ಯೆಗಳಿಗೆ. ಮಾದರಿ ಪ್ರತಿ ಹಂತದ ತರ್ಕವನ್ನು ಸ್ಪಷ್ಟವಾಗಿ ತೋರಿಸುತ್ತದೆ. ಗಣಿತ ಸಮಸ್ಯೆಗಳು, ತರ್ಕ ಪಜಲ್‌ಗಳು, ಅಥವಾ ಯೋಚನೆ ಪ್ರಕ್ರಿಯೆಯನ್ನು ಅರಿತುಕೊಳ್ಳಬೇಕಾಗಿರುವಾಗ ಇದನ್ನು ಬಳಸಿ.

**ಮર્યಾದಿತ ಔಟ್‌ಪುಟ್** - ನಿರ್ದಿಷ್ಟ ಸ್ವರೂಪ್ ನಿಯಮಗಳೊಂದಿಗೆ ಪ್ರತಿಕ್ರಿಯೆಗಳಿಗೆ. ಮಾದರಿ ಸ್ವರೂಪ ಮತ್ತು ಉದ್ದದ ನಿಯಮಗಳನ್ನು ಕಟ್ಟಿಕೊಂಡು ಅನುಸರಿಸುತ್ತದೆ. ಸಾರಾಂಶಗಳಿಗೆ ಅಥವಾ ನಿಖರ ಔಟ್‌ಪುಟ್ ರಚನೆಯಾಗುಬೇಕಾದಾಗ ಇದನ್ನು ಬಳಸಿ.

ಡೇವ್ ಕಂಟೇನರ್ ಸ್ಪ್ರಿಂಗ್ ಬೂಟ್ ಡ್ಯಾಶ್‌ಬೋರ್ಡ್ ವಿಸ್ತರಣೆ ಒಳಗೊಂಡಿದೆ, ಇದು ಎಲ್ಲಾ ಸ್ಪ್ರಿಂಗ್ ಬೂಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿರ್ವಹಿಸಲು ದೃಶ್ಯಾತ್ಮಕ ಇಂಟರ್‌ಫೇಸನ್ನು ಒದಗಿಸುತ್ತದೆ. ನೀವು ಇದನ್ನು VS Code ನ ಎಡಬದಿ<Activity Bar> ನಲ್ಲಿ ಸ್ಪ್ರಿಂಗ್ ಬೂಟ್ ಐಕಾನ್ ನೋಡಬಹುದಾಗಿದೆ.

ಸ್ಪ್ರಿಂಗ್ ಬೂಟ್ ಡ್ಯಾಶ್‌ಬೋರ್ಡ್ ಮೂಲಕ ನೀವು:
- ವರ್ಕ್‌ಸ್ಪೇಸ್‌ನಲ್ಲಿ ಲಭ್ಯವಿರುವ ಎಲ್ಲಾ ಸ್ಪ್ರಿಂಗ್ ಬೂಟ್ ಅಪ್ಲಿಕೇಶನ್ಗಳು ನೋಡಬಹುದು
- ಒಮ್ಮೆ ಕ್ಲಿಕ್ ಮಾಡಿ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಪ್ರಾರಂಭ/ನಿರ್ವಹಣೆ ಮಾಡಬಹುದು
- ಅಪ್ಲಿಕೇಶನ್ ಡೈರೆಕ್ಟ್ ಲಾಗ್‌ಗಳನ್ನು ರಿಯಲ್-ಟೈಂನಲ್ಲಿ ನೋಡಬಹುದು
- ಅಪ್ಲಿಕೇಶನ್ ಸ್ಥಿತಿಯನ್ನು ಗಮನಿಸಬಹುದು
"prompt-engineering" ಪಕ್ಕದಲ್ಲಿ ಇರುವ ಪ್ಲೇ ಬಟನ್‌ನನ್ನು ಕ್ಲಿಕ್ ಮಾಡಿ ಈ ಮೋಡ್ಯೂಲ್ ಅನ್ನು ಪ್ರಾರಂಭಿಸಿ, ಅಥವಾ ಎಲ್ಲ ಮೋಡ್ಯೂಲ್‌ಗಳನ್ನು ಒಂದೇ ಸಮಯದಲ್ಲಿ ಪ್ರಾರಂಭಿಸಿ.

## ಪ್ಯಾಟರ್ನ್‌ಗಳನ್ನು ಅನ್ವೇಷಿಸುವುದು

ವೆಬ್ ಇಂಟರ್ಫೇಸ್ ವಿವಿಧ ಪ್ರಾಂಪ್ಟಿಂಗ್ ತಂತ್ರಗಳನ್ನು ಪ್ರಯೋಗಿಸಲು ನಿಮಗೆ ಅವಕಾಶ ಕೊಡುತ್ತದೆ. ಪ್ರತಿ ಪ್ಯಾಟರ್ನ್ ವಿವಿಧ ಸಮಸ್ಯೆಗಳನ್ನು ಪರಿಹರಿಸುತ್ತದೆ - ಯಾವುದೇ ಸಮಯದಲ್ಲಿ ಯಾವ ವಿಧಾನ ಚೆನ್ನಾಗಿ ಕೆಲಸ ಮಾಡುತ್ತದೆ ಎಂದು ಪರಿಶೀಲಿಸಿ.

> **ಟಿಪ್ಪಣಿ: ಸ್ಟ್ರೀಮಿಂಗ್ ವಿರುದ್ಧ ನಾನ್-ಸ್ಟ್ರೀಮಿಂಗ್** — ಪ್ರತಿಯೊಂದು ಪ್ಯಾಟರ್ನ್ ಪುಟದಲ್ಲಿ ಎರಡು ಬಟನ್ಗಳು ಇರುತ್ತವೆ: **🔴 ಸ್ಟ್ರೀಮ್ ಪ್ರತಿಕ್ರಿಯೆ (ライブ)** ಮತ್ತು **ನಾನ್-ಸ್ಟ್ರೀಮಿಂಗ್** ಆಯ್ಕೆ. ಸ್ಟ್ರೀಮಿಂಗ್ ಸರ್ವರ್-ಸೆಂಟ್ ಇವೆಂಟ್‌ಗಳು (SSE) ಬಳಸುತ್ತದೆ, ಅದು ಮಾದರಿ ಟೋಕೆನ್‌ಗಳನ್ನು ರಿಯಲ್-ಟೈಮ್‌ನಲ್ಲಿ ತೋರಿಸುತ್ತದೆ, ಆದ್ದರಿಂದ ನೀವು ತಕ್ಷಣ ಪ್ರಗತಿಯನ್ನು ಕಾಣಬಹುದು. ನಾನ್-ಸ್ಟ್ರೀಮಿಂಗ್ ಆಯ್ಕೆಯು ಸಂಪೂರ್ಣ ಪ್ರತಿಕ್ರಿಯೆಗಾಗಿ ಕಾಯುತ್ತದೆ. ಆಳವಾದ ತರ್ಕವನ್ನುಂಟು ಮಾಡುವ ಪ್ರಾಂಪ್ಟ್‌ಗಳಿಗೆ (ಉದಾ: ಹೈ ಈಗರ್‌ನೆಸ್, ಸೆಲ್ಫ್-ರಿಫ್ಲೆಕ್ಟಿಂಗ್ ಕೋಡ್), ನಾನ್-ಸ್ಟ್ರೀಮಿಂಗ್ ಕಾಲ್ ಬಹಳ ಹೆಚ್ಚು ಸಮಯ ತೆಗೆದುಕೊಳ್ಳಬಹುದು — ಕೆಲವೊಮ್ಮೆ ನಿಮಿಷಗಳು — ಯಾವುದೇ ದೃಶ್ಯ ಪ್ರತಿಕ್ರಿಯೆಯಿಲ್ಲದೆ. **ಕಠಿಣ ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ಪ್ರಯೋಗಿಸುವಾಗ ಸ್ಟ್ರೀಮಿಂಗ್ ಬಳಸಿ** ಅದರಿಂದ ನೀವು ಮಾದರಿಯ ಕೆಲಸ ಮಾಡುವುದನ್ನು ನೋಡಬಹುದು ಮತ್ತು ವಿನಂತಿಯ ಸಮಯ ಮುಗಿಯಿತು ಎಂಬ ಅನುಭವ ತಪ್ಪಿಸಿಕೊಳ್ಳಬಹುದು.
>
> **ಟಿಪ್ಪಣಿ: ಬ್ರೌಸರ್ ಅಗತ್ಯತೆ** — ಸ್ಟ್ರೀಮಿಂಗ್ ವೈಶಿಷ್ಟ್ಯ Fetch Streams API (`response.body.getReader()`) ಬಳಕೆ ಮಾಡುತ್ತದೆ, ಇದು ಪೂರ್ಣ ಬ್ರೌಸರ್‌ಗಳ (ಕ್ರೋಮ್, ಎಡ್ಜ್, ಫೈರ್‌ಫಾಕ್ಸ್, ಸಫಾರಿ) ಜೊತೆ ಲಭ್ಯವಿದೆ. ಇದು VS ಕೋಡ್‌ನ ರೂಢಿಯಾಗಿ ಇರುವ ಸರಳ ಬ್ರೌಸರ್‌ನಲ್ಲಿ పనిచೆಯುವುದಿಲ್ಲ, ಏಕೆಂದರೆ ಅದರ ವೆಬ್‌ವ್ಯೂ ReadableStream API ಅನ್ನು ಬೆಂಬಲಿಸುವುದಿಲ್ಲ. ನೀವು ಸರಳ ಬ್ರೌಸರ್ ಅನ್ನು ಬಳಸಿದರೆ, ನಾನ್-ಸ್ಟ್ರೀಮಿಂಗ್ ಬಟನ್ಗಳು ಸಾಮಾನ್ಯವಾಗಿ ಕೆಲಸಮಾಡುತ್ತವೆ — ಕೇವಲ ಸ್ಟ್ರೀಮಿಂಗ್ ಬಟನ್ಗಳು ಪ್ರಭಾವಿತವಾಗುತ್ತವೆ. ಪೂರ್ಣ ಅನುಭವಕ್ಕಾಗಿ `http://localhost:8083` ಅನ್ನು ಯಾವುದೇ ಹೊರಗಿನ ಬ್ರೌಸರ್‌ನಲ್ಲಿ ತೆರೆಯಿರಿ.

### ಕಡಿಮೆ ಮತ್ತು ಹೆಚ್ಚಾದ ಈಗರ್‌ನೇಸ್

ಕಡಿಮೆ ಈಗರ್‌ನೇಸ್ ಬಳಸಿ "200 ರ 15% ಏನು?" ಎಂಬ ಸರಳ ಪ್ರಶ್ನೆಯನ್ನು ಕೇಳಿ. ನಿಮಗೆ ತಕ್ಷಣ, ನೇರ ಉತ್ತರ ಸಿಗುತ್ತದೆ. ಈಗ ಬಹಳ ಸಂಕೀರ್ಣವಾದ "ಹೆಚ್ಚು ಸಂಚಾರ ಇರುವ API ಗೆ ಕ್ಯಾಷಿಂಗ್ ತಂತ್ರವನ್ನು ವಿನ್ಯಾಸಗೊಳಿಸಿ" ಎಂದು ಹೆಚ್ಚಾದ ಈಗರ್‌ನೇಸ್ ಬಳಸಿ ಕೇಳಿ. **🔴 ಸ್ಟ್ರೀಮ್ ಪ್ರತಿಕ್ರಿಯೆ (ライブ)** ಕ್ಲಿಕ್ ಮಾಡಿ ಮತ್ತು ಮಾದರಿ ವಿವರವಾದ ತರ್ಕವನ್ನು ಟೋಕನ್ ಪ್ರತಿ ಟೋಕನ್ ಆಗಿ ನೋಡಿರಿ. ಒಂದೇ ಮಾದರಿ, ಒಂದೇ ಪ್ರಶ್ನೆ ರಚನೆ - ಆದರೆ ಪ್ರಾಂಪ್ಟ್ ಅದಕ್ಕೆ ಎಷ್ಟು ಹೆಚ್ಚು ಯೋಚನೆ ಮಾಡಬೇಕು ಎಂಬುದನ್ನು ಹೇಳುತ್ತದೆ.

### ಕාර්ಯ ನಿರ್ವಹಣೆ (ಟೂಲ್ಸ್ ಪೂರ್ವಭಾವಿ ಸಂದೇಶಗಳು)

ಬಹು ಹಂತದ ಕೆಲಸದ ಸಾಗಣೆಗಳು ಮುಂಚಿತ ಯೋಜನೆ ಮತ್ತು ಪ್ರಗತಿ ವಿವರಣೆಯಿಂದ ಲಾಭ ಪಡೆಯುತ್ತವೆ. ಮಾದರಿ ಅದು ಏನು ಮಾಡುವುದು ಎಂದು ವಿವರಿಸುತ್ತದೆ, ಪ್ರತಿ ಹಂತವನ್ನು ವಿವರವಾಗಿ ಹೇಳುತ್ತದೆ, ನಂತರ ಫಲಿತಾಂಶಗಳನ್ನು ಸಮಾರೋಹಿಸುತ್ತವೆ.

### ಸ್ವಯಂ-ಪರಿಶೀಲನಾ ಕೋಡ್

"ಇಮೇಲ್ ಪರಿಶೀಲನೆ ಸೇವೆಯನ್ನು ರಚಿಸಿ" ಎಂದು ಪ್ರಯತ್ನಿಸಿ. ಕೇವಲ ಕೋಡ್ ರಚಿಸಿ ನಿಲ್ಲಿಸುವ ಬದಲು, ಮಾದರಿ ಕೋಡ್ ರಚಿಸಿ, ಗುಣಮಟ್ಟ ಮಾನದಂಡಗಳ ವಿರುದ್ಧ ಮೌಲ್ಯಮಾಪನ ಮಾಡುತ್ತದೆ, ದುರ್ಬಲತೆಗಳನ್ನು ಗುರುತಿಸಿ ಸುಧಾರಣೆ ಮಾಡುತ್ತದೆ. ನೀವು ಪೂರ್ಣ ಉತ್ಪಾದನಾ ಪ್ರಮಾಣಗಳಿಗೆ ತಲುಪಿದವರೆಗೆ ಅದನ್ನು ಪುನರಾವರ್ತಿಸುತ್ತಿದೆ ಎಂದು ನೋಡಲಿದ್ದೀರಿ.

### ಸಂರಚಿತ ವಿಶ್ಲೇಷಣೆ

ಕೋಡ್ ವಿಮರ್ಶೆಗಳಲ್ಲಿ ಸ್ಥಿರ ಮೌಲ್ಯಮಾಪನ ರೂಪರೇಖೆಗಳು ಬೇಕಾಗುತ್ತವೆ. ಮಾದರಿ ಕೋಡ್ ಅನ್ನು ನಿಶ್ಚಿತ ವರ್ಗಗಳ (ಸರಿಯಾದತೆ, ಅಭ್ಯಾಸಗಳು, ಕಾರ್ಯಕ್ಷಮತೆ, ಭದ್ರತೆ)ೊಂದಿಗೆ ಗಂಭೀರತಾ ಮಟ್ಟಗಳನ್ನು ಬಳಸಿ ವಿಶ್ಲೇಷಿಸುತ್ತದೆ.

### ಬಹು-ತಿರುಗುವ ಸೈಟ್

"Spring Boot ಏನು?" ಎಂದು ಕೇಳಿ ನಂತರ ತಕ್ಷಣ "ಒಂದು ಉದಾಹರಣೆಯನ್ನು ತೋರಿಸಿ" ಎಂಬುದನ್ನು ಕೇಳಿ. ಮಾದರಿ ನಿಮ್ಮ ಮೊದಲ ಪ್ರಶ್ನೆಯನ್ನು ನೆನಸಿಕೊಂಡು, ಸ್ಪ್ರೆಿಂಗ್ ಬೂಟ್ ಉದಾಹರಣೆಯನ್ನು ವಿಶೇಷವಾಗಿ ನೀಡುತ್ತದೆ. ಸ್ಮರಣೆ ಇಲ್ಲದೆ, ಆಗ ಎರಡನೇ ಪ್ರಶ್ನೆ ಬಹಳ ಅಸ್ಪಷ್ಟವಾಗುತ್ತದೆ.

### ಹಂತ ಹಂತವಾಗಿ ತರ್ಕ

ಒಂದು ಗಣಿತ ಸಮಸ್ಯೆಯನ್ನು ಆಯ್ಕೆ ಮಾಡಿ ಮತ್ತು ಹಂತ-ಹಂತದ ತರ್ಕ ಮತ್ತು ಕಡಿಮೆ ಈಗರ್ನೆಸ್ ಎರಡೂ ಬಳಸಿ ಪ್ರಯತ್ನಿಸಿ. ಕಡಿಮೆ ಈಗರ್ನೆಸ್ ಈಗ ಉತ್ತರವನ್ನು ಮಾತ್ರ ನೀಡುತ್ತದೆ — ವೇಗವಾದ ಆದರೆ ಅಸ್ಪಷ್ಟ. ಹಂತ ಹಂತದ ತರ್ಕಗಳು ನಿಮಗೆ ಪ್ರತಿಯೊಂದು ಲೆಕ್ಕ ಮತ್ತು ತೀರ್ಮಾನವನ್ನು ತೋರಿಸುತ್ತವೆ.

### ನಿರ್ಬಂಧಿತ ಫಲಿತಾಂಶ

ನೀವು ನಿಖರವಾಗಿ ನಿರ್ದಿಷ್ಟ ಸ್ವರೂಪಗಳು ಅಥವಾ ಪದಗಳ ಸಂಖ್ಯೆಯನ್ನು ಬೇಕಾದಾಗ, ಈ ಪ್ಯಾಟರ್ನ್ ಕಟ್ಟುನಿಟ್ಟಾಗಿ ಅನುಸರಿಸುವಂತೆ ಮಾಡುತ್ತದೆ. 100 ಪದಗಳ ಬುಲೆಟ್ ಪಾಯಿಂಟ್ ಸ್ವರೂಪದಲ್ಲಿ ಸಮ್ಮೇಳನವನ್ನು ರಚಿಸಲು ಪ್ರಯತ್ನಿಸಿ.

## ನೀವು ವಾಸ್ತವವಾಗಿ ಏನು ಕಲಿಯುತ್ತಿದ್ದೀರಿ

**ತರ್ಕದ ಪ್ರಯತ್ನ ಎಲ್ಲವನ್ನೂ ಬದಲಿಸುತ್ತವೆ**

GPT-5.2 ನಿಮ್ಮ ಪ್ರಾಂಪ್ಟ್‌ಗಳ ಮೂಲಕ ಗಣನಾತ್ಮಕ ಪ್ರಯತ್ನವನ್ನು ನಿಯಂತ್ರಿಸಲು ಅವಕಾಶ ಮಾಡಿಕೊಡುತ್ತದೆ. ಕಡಿಮೆ ಪ್ರಯತ್ನ ವೇಗವಾದ ಪ್ರತಿಕ್ರಿಯೆಗಳು ಮತ್ತು ಕನಿಷ್ಠ ಅನ್ವೇಷಣೆಯನ್ನು ಕನಸಿನಂತೆ ಮಾಡುತ್ತದೆ. ಹೆಚ್ಚು ಪ್ರಯತ್ನ ಎಂದರೆ ಮಾದರಿ ಆಳವಾಗಿ ಯೋಚಿಸುವುದಕ್ಕೆ ಸಮಯ ತೆಗೆದುಕೊಳ್ಳುತ್ತದೆ. ನೀವು ಕಾರ್ಯದ ಸಂಕೀರ್ಣತೆಗೆ ಪ್ರಯತ್ನವನ್ನು ಹೊಂದಿಸಲು ಕಲಿಯುತ್ತಿದ್ದೀರಾ - ಸರಳ ಪ್ರಶ್ನೆಗಳಿಗೆ ಸಮಯ ವ್ಯರ್ಥ ಮಾಡಬೇಡಿ, ಆದರೆ ಸಂಕೀರ್ಣ ನಿರ್ಣಯಗಳನ್ನು ಕೂಡಾ ಶೀಘ್ರಗತಿಯಲ್ಲಿ ಮಾಡಬೇಡಿ.

**ರಚನೆ ವರ್ತನೆ ಮಾರ್ಗದರ್ಶಿಸುತ್ತದೆ**

ಪ್ರಾಂಪ್ಟ್‌ಗಳಲ್ಲಿ XML ಟ್ಯಾಗ್ಗಳನ್ನು ಗಮನಿಸುತ್ತೀರಾ? ಅವು ಅಲಂಕಾರಿಕವಲ್ಲ. ಮಾದರಿಗಳು ರಚಿಸಿಕೊಂಡ ಸೂಚನೆಗಳನ್ನು ಮುಕ್ತರೂಪದ ಪಠ್ಯಕ್ಕಿಂತ ಹೆಚ್ಚು ನಂಬಿಕೆಯುತವಾಗಿ ಅನುಸರಿಸುತ್ತವೆ. ಬಹು ಹಂತದ ಪ್ರಕ್ರಿಯೆಗಳು ಅಥವಾ ಸಂಕೀರ್ಣ ತರ್ಕ ಬೇಕಾಗುವಾಗ, ರಚನೆ ಮಾದರಿಗೆ ಅದು ಎಲ್ಲಿ ಇದೆ ಮತ್ತು ಮುಂದೇನು ಎಂಬುದನ್ನು ಅನುಸರಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.

*ಸ್ಪಷ್ಟ ವಿಭಾಗಗಳು ಮತ್ತು XML ಶೈಲಿಯ ಸಂಘಟನೆಯೊಂದಿಗೆ ಚೆನ್ನಾಗಿ ರಚಿಸಲಾದ ಪ್ರಾಂಪ್ಟ್‌ನ ಶರೀರವಿಜ್ಞಾನ*

**ಸ್ವಯಂ-ಮೌಲ್ಯಮಾಪನದ ಮೂಲಕ ಗುಣಮಟ್ಟ**

ಸ್ವಯಂ-ಪರಿಶೀಲನ ಪ್ಯಾಟರ್ನ್‌ಗಳು ಗುಣಮಟ್ಟ ಮಾನದಂಡಗಳನ್ನು ಸ್ಪಷ್ಟವಾಗಿ ಮಾಡುತ್ತವೆ. ಮಾದರಿ "ಸರಿಯಾಗಿ ಮಾಡುತ್ತದೆ" ಎಂದು ನಿರೀಕ್ಷಿಸುವ ಬದಲು, ನೀವು "ಸರಿಯೆಂದರೆ" ಎಂದರೆ ಏನು ಅಂತ ಹೇಳುತ್ತೀರಿ: ಸರಿಯಾದ ತರ್ಕ, ದೋಷ ನಿರ್ವಹಣೆ, ಕಾರ್ಯಕ್ಷಮತೆ, ಭದ್ರತೆ. ನಂತರ ಮಾದರಿ ತನ್ನದ್ದನ್ನು ಮೌಲ್ಯಮಾಪನ ಮಾಡಿ ಸುಧಾರಿಸಬಹುದು. ಇದರಿಂದ ಕೋಡ್ ನಿರ್ಮಾಣವು ಲಾಟರಿ ಆಗಿ ಬದಲು ಪ್ರಕ್ರಿಯೆಯಾಗುತ್ತದೆ.

**ಪರಿಸರ ಸೀಮಿತವಾಗಿದೆ**

ಬಹು-ತಿರುಗುವ ಸಂಭಾಷಣೆಗಳು ಪ್ರತಿ ವಿನಂತಿಯೊಂದಿಗೆ ಸಂದೇಶ ಇತಿಹಾಸವನ್ನು ಹೊಂದಿವೆ. ಆದರೆ ಗಡಿಯಿದೆ - ಪ್ರತಿಯೊಂದು ಮಾದರಿಯು ಗರಿಷ್ಟ ಟೋಕನ್ ಸಂಖ್ಯೆಯನ್ನು ಹೊಂದಿದೆ. ಸಂಭಾಷಣೆಗಳು ಹೆಚ್ಚಿದಂತೆ, ನೀವು ಸಂಬಂಧಿತ ಪರಿಸರವನ್ನು ಉಳಿಸಿಕೊಳ್ಳಲು ತಂತ್ರಗಳನ್ನು ಬಳಸಬೇಕಾಗುತ್ತದೆ, ಅದು ಗಡಿಯನ್ನು ಮೀರದಂತೆ ಮಾಡಬಹುದು. ಈ ಮೋಡ್ಯೂಲ್ ನೀವು ಸ್ಮರಣೆ ಹೇಗೆ ಕೆಲಸ ಮಾಡುತ್ತದೆಯೋ ತೋರಿಸುತ್ತದೆ; ನಂತರ ನೀವು ಯಾವಾಗ ಸಾರಾಂಶ ಮಾಡಬೇಕು, ಯಾವಾಗ ಮರೆಮಾಡಬೇಕು ಮತ್ತು ಯಾವಾಗ ಮರುಪಡೆಯಬೇಕು ಎಂದು ಕಲಿತೀರಿ.
