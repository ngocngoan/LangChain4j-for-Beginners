## ನೀವು ಕಲಿಯುತ್ತಿರುವುದು

<img src="../../../translated_images/kn/what-youll-learn.c68269ac048503b2.webp" alt="ನೀವು ಕಲಿಯುತ್ತಿರುವುದು" width="800"/>

ಹಿಂದಿನ ಮಾಡ್ಯೂಲ್‌ನಲ್ಲಿ, ನೀವು ಮೆಮೊರಿ ಸಂವಾದಾತ್ಮಕ AI ಯನ್ನು ಹೇಗೆ ಸಾಧ್ಯವಾಗಿಸುತ್ತದೆ ಎಂಬುದನ್ನು ನೋಡಿದಿರಿ ಮತ್ತು ಮಾರ್ಗಸೂಚಿ ಕ್ರಿಯೆಗಳು ಮಾಡಲು GitHub ಮಾದರಿಗಳನ್ನು ಬಳಕೆಯೆನ್ನಿಸಿದರು. ಈಗ ನಾವು ಪ್ರಶ್ನೆಗಳನ್ನು ಹೇಗೆ ಕೇಳಬೇಕು — ಅಂದರೆ ಪ್ರಾಂಪ್ಟ್ಗಳು — ಎಂದು Azure OpenAI ನ GPT-5.2 ಬಳಸಿ ಗಮನಹರಿಸುವಾಗಿದ್ದೇವೆ. ನಿಮ್ಮ ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ನೀವು ಹೇಗೆ ರಚಿಸುವಿರೋ ಅದು ನೀವು ಪಡೆಯುವ ಪ್ರತಿಕ್ರಿಯೆಗಳ ಗುಣಮಟ್ಟವನ್ನು ಗಮನಾರ್ಹವಾಗಿ ಪ್ರಭಾವಿಸುತ್ತದೆ. ನಾವು ಮೂಲಪ್ರಾಂಪ್ಟ್ ತಂತ್ರಗಳನ್ನು ಪರಿಶೀಲನೆ ಮಾಡುತ್ತೀವೆ, ನಂತರ GPT-5.2 ಯ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಸಂಪೂರ್ಣವಾಗಿ ಪ್ರಯೋಜನಪಡಿಸುವ ಎಂಟು ಅತ್ಯಾಧುನಿಕ ಪ್ಯಾಟರ್ನ್ಗಳತ್ತ ಭ್ರಮನ ಮಾಡುತ್ತೀವೆ.

ನಾವು GPT-5.2 ಯನ್ನು ಬಳಸುತ್ತೇವೆ ಏಕೆಂದರೆ ಇದು ಕಾರಣೀಕರಣ ನಿಯಂತ್ರಣವನ್ನು ಪರಿಚಯಿಸುತ್ತದೆ - ನೀವು ಮಾದರಿಗೆ ಉತ್ತರಿಸುವ ಮುನ್ನ ಎಷ್ಟು ಚಿಂತನೆ ಮಾಡಬೇಕೇ ಎಂಬುದನ್ನು ಹೇಳಬಹುದು. ಇದರಿಂದ ವಿಭಿನ್ನ ಪ್ರಾಂಪ್ಟಿಂಗ್ ತಂತ್ರಗಳು ಸ್ಪಷ್ಟವಾಗುತ್ತವೆ ಮತ್ತು ಯಾವಾಗ ಯಾವ ರೀತಿಯನ್ನು ಬಳಸಬೇಕು ಎಂಬುದನ್ನು ನೀವು ಸಮರ್ಥವಾಗಿ ಅರ್ಥಮಾಡಿಕೊಳ್ಳಬಹುದು. GitHub ಮಾದರಿಗಳ ಜೊತೆಗೋಲಿಸಿಕೊಂಡಾಗ Azureನಲ್ಲಿನ GPT-5.2 ಗೆ ಕಡಿಮೆ ದರ ಮಿತಿಗಳೂ ಪ್ರಯೋಜನಕಾರಿಯಾಗಿದೆ.

## ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು

<img src="../../../translated_images/kn/what-is-prompt-engineering.5c392a228a1f5823.webp" alt="ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಎಂದರೇನು?" width="800"/>

ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಎಂದರೆ ನೀವು ಬೇಕಾದ ಫಲಿತಾಂಶಗಳನ್ನು ನಿರಂತರವಾಗಿ ಪಡೆಯುವಂತೆ ಇನ್‌ಪುಟ್ ಪಠ್ಯವನ್ನು ವಿನ್ಯಾಸಗೊಳಿಸುವುದು. ಇದು ಕೇವಲ ಪ್ರಶ್ನೆ ಕೇಳುವುದಲ್ಲ - ಮಾದರಿಗೆ ನೀವು ಏನನ್ನೂ ಮಾಡಬೇಕೆಂದು ಸ್ಪಷ್ಟವಾಗಿ ತಿಳಿದಂತೆ ವಿನಂತಿಗಳನ್ನು ರಚಿಸುವುದು.

ಸರಳವಾಗಿ ಹೇಳಬೇಕಾದರೆ ನೀವು ಸಹೋದ್ಯೋಗಿಗೆ ಆದೇಶ ನೀಡುತ್ತಿರುವಂತೆ. "ಕೊಪ್ಪವನ್ನು ಸರಿಪಡಿಸಿ" ಅಬ್ಬರಿದಿದೆ. "UserService.java ರೇಖೆ 45 ಯಲ್ಲಿರುವ ನಲ್ ಪಾಯಿಂಟರ್ ಎಕ್ಸೆಪ್ಶನ್‍ಗೆ ನಲ್ ಪರಿಶೀಲನೆ ಸೇರಿಸಿ" ಎಂದರೆ ಸ್ಪಷ್ಟವಾಗಿದೆ. ಭಾಷಾ ಮಾದರಿಗಳು ಸಹ ಹಾಗೆಯೇ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತವೆ - ಸ್ಪಷ್ಟತೆ ಮತ್ತು ರಚನೆಯು ಮುಖ್ಯ.

<img src="../../../translated_images/kn/how-langchain4j-fits.dfff4b0aa5f7812d.webp" alt="ಲಾಂಗ್ ಚೈನ್ 4ಜಿ ಹೇಗೆ ಹೊಂದಿಕೊಳ್ಳುತ್ತದೆ" width="800"/>

LangChain4j ಮೂಲಸೌಕರ್ಯವನ್ನು ಒದಗಿಸುತ್ತದೆ — ಮಾದರಿ ಸಂಪರ್ಕಗಳು, ಮೆಮೊರಿ, ಮತ್ತು ಸಂದೇಶ ಪ್ರಕಾರಗಳು — ಮತ್ತು ಪ್ರಾಂಪ್ಟ್ ಪ್ಯಾಟರ್ನ್ಗಳು ಆ ಮೂಲಸೌಕರ್ಯದ ಮೂಲಕ ಕಾಳಜಿ ವಹಿಸಿಕೊಂಡಿರುವ ವಿವರಣಾತ್ಮಕ ಪಠ್ಯ ಮಾತ್ರ. ಪ್ರಮುಖ ಅವಯವಗಳು `SystemMessage` (ಅದರ ಮೂಲಕ AI ನ ವರ್ತನೆ ಮತ್ತು ಪಾತ್ರ ನಿಗದಿಪಡಿಸಲಾಗುತ್ತದೆ) ಮತ್ತು `UserMessage` (ನಿಮ್ಮ ನಿಜವಾದ ವಿನಂತಿಯನ್ನು ಹೊತ್ತುಕೊಳ್ಳುತ್ತದೆ) ಆಗಿವೆ.

## ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಮೂಲತತ್ವಗಳು

<img src="../../../translated_images/kn/five-patterns-overview.160f35045ffd2a94.webp" alt="ಐದು ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಮಾದರಿಗಳ ಅವಲೋಕನ" width="800"/>

ಈ ಮಾಡ್ಯೂಲ್‌ನ ಅತ್ಯಾಧುನಿಕ ಮಾದರಿಗಳತ್ತ ತೆರಳುವ ಮೊದಲು, ಐದು ಮೂಲತತ್ವದ ಪ್ರಾಂಪ್ಟ್ ತಂತ್ರಗಳನ್ನು ಪರಿಶೀಲಿಸೋಣ. ಇವು ಪ್ರತಿ ಪ್ರಾಂಪ್ಟ್ ಇಂಜಿನಿಯರ್ ತಿಳಿದುಕೊಳ್ಳಬೇಕಾದ ಮೂಲಭೂತ ಪದಾರ್ಥಗಳು. ನೀವು ಈಗಾಗಲೇ [ಕ್ವಿಕ್ ಸ್ಟಾರ್ಟ್ ಮಾಡ್ಯೂಲ್](../00-quick-start/README.md#2-prompt-patterns) ಮೂಲಕ ಈ ತಂತ್ರಗಳನ್ನು ಅನುಭವಿಸಿದ್ದೀರಿ — ಇಲ್ಲಿ ಅದರ ಹಿನ್ನಲೆ ಸಂಪ್ರದಾಯ ವಿವರಿಸಲಾಗಿದೆ.

### ಶೂನ್ಯ-ಶಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್

ಸರಳವಾದ ವಿಧಾನ: ಮಾದರಿಗೆ ಉದಾಹರಣೆ ಇಲ್ಲದೆ ನೇರ ಸೂಚನೆ ನೀಡುವುದು. ಮಾದರಿ ಅದರ ತರಬೇತಿಯನ್ನು ಸಂಪೂರ್ಣವಾಗಿ ಅವಲಂಬಿಸಿ ಕಾರ್ಯವನ್ನು ಅರ್ಥಮಾಡಿಕೊಂಡು ನಿರ್ವಹಿಸುತ್ತದೆ. ಇದು ನಿರೀಕ್ಷಿತ ವರ್ತನೆ ಸ್ಪಷ್ಟವಾಗಿರುವ ಸರಳ ವಿನಂತಿಗಳಿಗೆ ಸೂಕ್ತವಾಗಿದೆ.

<img src="../../../translated_images/kn/zero-shot-prompting.7abc24228be84e6c.webp" alt="ಶೂನ್ಯ-ಶಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್" width="800"/>

*ಉದಾಹರಣೆ ಇಲ್ಲದೆ ನೇರ ಸೂಚನೆ — ಮಾದರಿ ಸೂಚನೆಯಿಂದಲೇ ಕಾರ್ಯವನ್ನು ಊಹಿಸುತ್ತದೆ*

```java
String prompt = "Classify this sentiment: 'I absolutely loved the movie!'";
String response = model.chat(prompt);
// ಪ್ರತಿಕ್ರಿಯೆ: "ಧನಾತ್ಮಕ"
```

**ಬಳಸದ ಸಮಯ:** ಸರಳ ವರ್ಗೀಕರಣಗಳು, ನೇರ ಪ್ರಶ್ನೆಗಳು, ಭಾಷಾಂತರಗಳು, ಅಥವಾ ನೆರವಿನ ಅಗತ್ಯವಿಲ್ಲದ ಯಾವುದೇ ಕಾರ್ಯ.

### ಕಡಿಮೆ-ಶಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್

ನೀವು ಮಾದರಿಯಿಂದ ಅನುಸರಿಸಲು ಬಯಸುವ ಮಾದರಿಯನ್ನು ತೋರಿಸುವ ಉದಾಹರಣೆಗಳನ್ನು ಪೂರೈಸಿ. ಮಾದರಿ ನಿಮ್ಮ ಉದಾಹರಣೆಗಳಿಂದ ನಿರೀಕ್ಷಿತ ಇನ್‌ಪುಟ್-ಔಟ್‌ಪುಟ್ ತಳಿಕೆ ಕಲಿಯುತ್ತದೆ ಮತ್ತು ಹೊಸ ಇನ್‌ಪುಟ್‌ಗಳಿಗೆ ಅದನ್ನು ಅನ್ವಯಿಸುತ್ತದೆ. ಈ ತಂತ್ರದಿಂದ ನಿರೀಕ್ಷಿತ ಸ್ವರೂಪ ಅಥವಾ ವರ್ತನೆ ತಿಳಿಹೀನ ಆಗಿರುವ ಕಾರ್ಯಗಳಿಗೆ ಸ್ಥಿರತೆ ಹೆಚ್ಚಾಗುತ್ತದೆ.

<img src="../../../translated_images/kn/few-shot-prompting.9d9eace1da88989a.webp" alt="ಕಡಿಮೆ-ಶಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್" width="800"/>

*ಉದಾಹರಣೆಗಳಿಂದ ಕಲಿಕೆಯುವಿಕೆ — ಮಾದರಿ ಮಾದರಿಯನ್ನು ಗುರುತಿಸಿ ಹೊಸ ಇನ್‌ಪುಟ್‌ಗಳಿಗೆ ಅನ್ವಯಿಸುತ್ತದೆ*

```java
String prompt = """
    Classify the sentiment as positive, negative, or neutral.
    
    Examples:
    Text: "This product exceeded my expectations!" → Positive
    Text: "It's okay, nothing special." → Neutral
    Text: "Waste of money, very disappointed." → Negative
    
    Now classify this:
    Text: "Best purchase I've made all year!"
    """;
String response = model.chat(prompt);
```

**ಬಳಸದ ಸಮಯ:** ಕಸ್ಟಮ್ ವರ್ಗೀಕರಣಗಳು, ಸಮಾನ ಸ್ವರೂಪ ನಿರ್ವಹಣೆ, ಡೋಮೇನ್ ವಿಶೇಷ ಕೆಲಸಗಳು, ಅಥವಾ ಶೂನ್ಯ-ಶಾಟ್ ನ ಫಲಿತಾಂಶಗಳು ಅಸ್ಥಿರವಾಗಿರುವಾಗ.

### ಚೇನ್ ಆಫ್ ಥಾಟ್

ಮಾದರಿಯನ್ನು ತನ್ನ ಕಾರಣೀಕರಣವನ್ನು ಹಂತ-ಹಂತವಾಗಿ ತೋರಿಸಲು ಕೇಳಿ. ನೇರ ಉತ್ತರಕ್ಕೆ ಬದಲು, ಮಾದರಿ ಸಮಸ್ಯೆಯನ್ನು ವಿಭಜಿಸಿ ಪ್ರತಿ ಭಾಗವನ್ನು ಸ್ಪಷ್ಟವಾಗಿ ನಿಭಾಯಿಸುತ್ತದೆ. ಇದು ಗಣಿತ, ತರ್ಕ, ಮತ್ತು ಬಹು-ಹಂತದ ಕಾರಣೀಕರಣ ಕಾರ್ಯಗಳಲ್ಲಿ ಸರಿಯಾದ ಉತ್ತರಗಳನ್ನು ಹೆಚ್ಚಿಸುತ್ತದೆ.

<img src="../../../translated_images/kn/chain-of-thought.5cff6630e2657e2a.webp" alt="ಚೇನ್ ಆಫ್ ಥಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್" width="800"/>

*ಹಂತ-ಹಂತವಾಗಿ ಕಾರಣೀಕರಣ — ಸಂಕೀರ್ಣ ಸಮಸ್ಯೆಗಳನ್ನು ಸ್ಪಷ್ಟವಾದ ತರ್ಕ ಹಂತಗಳಿಗೆ ವಿಭಜಿಸುವುದು*

```java
String prompt = """
    Problem: A store has 15 apples. They sell 8 apples and then 
    receive a shipment of 12 more apples. How many apples do they have now?
    
    Let's solve this step-by-step:
    """;
String response = model.chat(prompt);
// ಮಾದರಿ ತೋರಿಸುತ್ತದೆ: 15 - 8 = 7, ನಂತರ 7 + 12 = 19 ಸೇಬುಗಳು
```

**ಬಳಸದ ಸಮಯ:** ಗಣಿತದ ಸಮಸ್ಯೆಗಳು, ತರ್ಕ ಪಜಲ್‌ಗಳು, ಡಿಬಗ್ಗಿಂಗ್, ಅಥವಾ ಕಾರಣೀಕರಣ ಪ್ರಕ್ರಿಯೆಯನ್ನು ತೋರಿಸುವುದರಿಂದ ನಿಖರತೆ ಮತ್ತು ನಂಬಿಕೆ ಆರುಮುತ್ತದೆ.

### ಪಾತ್ರಾಧಾರಿತ ಪ್ರಾಂಪ್ಟಿಂಗ್

ನಿಮ್ಮ ಪ್ರಶ್ನೆಗೆ ಯಾವ AI ಪಾತ್ರ ಅಥವಾ ವ್ಯಕ್ತಿತ್ವವನ್ನು ಸಂಸ್ಥಾಪಿಸಲು ಕೇಳಿ. ಇದು ಉತ್ತರದ ಶೈಲಿ, ಆಳ, ಮತ್ತು ಗಮನಕ್ಕೆ ಪ್ರಾಸಂಗಿಕತೆ ನೀಡುತ್ತದೆ. "ಸಾಫ್ಟ್ವೇರ್ ಆರ್ಕೆಟೆಕ್ಟ್" ಆಗಿರುವವನು "ಜೂನಿಯರ್ ಡೆವಲಪರ್" ಅಥವಾ "ಸೆಕ್ಯುರಿಟಿ ഓಡಿಟರ್"ಯಿಂದ ವಿಭಿನ್ನ ಸಲಹೆಗಳನ್ನು ನೀಡುತ್ತಾನೆ.

<img src="../../../translated_images/kn/role-based-prompting.a806e1a73de6e3a4.webp" alt="ಪಾತ್ರಾಧಾರಿತ ಪ್ರಾಂಪ್ಟಿಂಗ್" width="800"/>

*ಸಂದರ್ಭ ಮತ್ತು ಪಾತ್ರ ವ್ಯವಸ್ಥೆ — ಅದೇ ಪ್ರಶ್ನೆಗೆ ನಿಯೋಜಿತ ಪಾತ್ರದ ಪ್ರಕಾರ ವಿಭಿನ್ನ ಪ್ರತಿಕ್ರಿಯೆ*

```java
String prompt = """
    You are an experienced software architect reviewing code.
    Provide a brief code review for this function:
    
    def calculate_total(items):
        total = 0
        for item in items:
            total = total + item['price']
        return total
    """;
String response = model.chat(prompt);
```

**ಬಳಸದ ಸಮಯ:** ಕೋಡ್ ವಿಮರ್ಶೆಗಳು, ಟ್ಯುಟೋರಿಂಗ್, ಡೋಮೇನ್ ಸ್ಪೆಶಲೈಸ್ಡ್ ವಿಶ್ಲೇಷಣೆಗಳು, ಅಥವಾ ನಿರ್ದಿಷ್ಟ ಪರಿಣಿತ ಮಟ್ಟ ಅಥವಾ ದೃಷ್ಟಿಕೋನಕ್ಕೆ ಹೊಂದಿಕೃತಿ ಬೆರವಣಿಗೆಯ ಅಗತ್ಯವಿದ್ದಾಗ.

### ಪ್ರಾಂಪ್ಟ್ ಟೆಂಪ್ಲೇಟ್ಗಳು

ಬದಲಾಗುವ ಮೌಲ್ಯಗಳನ್ನು ಹೊಂದಿರುವ ಪುನರಾಯೋಗಾರ್ಹ ಪ್ರಾಂಪ್ಟ್ ಗಳನ್ನು ರಚಿಸು. ಪ್ರತಿವಾರ ಪ್ರಾಂಪ್ಟ್ ಬರೆಯುವ ಬದಲು, ಒಂದು ಟೆಂಪ್ಲೇಟ್ನ್ನು ಒಮ್ಮೆ ಹೇರಿದು, ವಿಭಿನ್ನ ಮೌಲ್ಯಗಳನ್ನು ತುಂಬಬಹುದು. LangChain4j ಯ `PromptTemplate` ವರ್ಗವು ಇದನ್ನು `{{variable}}` ಸಂಕೇತದ ಮೂಲಕ ಸುಲಭಗೊಳಿಸುತ್ತದೆ.

<img src="../../../translated_images/kn/prompt-templates.14bfc37d45f1a933.webp" alt="ಪ್ರಾಂಪ್ಟ್ ಟೆಂಪ್ಲೇಟ್ಗಳು" width="800"/>

*ವೇರಿಯಬಲ್ ಪ್ಲೇಸ್‌ಹೋಲ್ಡರ್ಗಳೊಂದಿಗೆ ಪುನರಾಯೋಗಾರ್ಹ ಪ್ರಾಂಪ್ಟ್‌ಗಳು — ಒಂದು ಟೆಂಪ್ಲೇಟು, ನಾನಾ ಬಳಕೆಗಳು*

```java
PromptTemplate template = PromptTemplate.from(
    "What's the best time to visit {{destination}} for {{activity}}?"
);

Prompt prompt = template.apply(Map.of(
    "destination", "Paris",
    "activity", "sightseeing"
));

String response = model.chat(prompt.text());
```

**ಬಳಸದ ಸಮಯ:** ವಿಭಿನ್ನ ಇನ್‌ಪುಟ್‌ಗಳೊಂದಿಗೆ ಪುನರಾವರ್ತಿತ ಪ್ರಶ್ನೆಗಳು, ಬ್ಯಾಚ್ ಪ್ರಕ್ರಿಯೆ, ಪುನರಾಯೋಗಾರ್ಹ AI ವರ್ಕ್‌ಫ್ಲೋ ನಿರ್ಮಾಣ, ಅಥವಾ ಪ್ರಾಂಪ್ಟ್ ರಚನೆ ಸ್ಥಿರವಾಗಿದ್ದರೂ ಡೇಟಾ ಬದಲಾಗುವ ಸಂದರ್ಭ.

---

ಈ ಐದು ಮೂಲತತ್ವಗಳು ಬಹುತೇಕ ಪ್ರಾಂಪ್ಟ್ ಕಾರ್ಯಗಳಿಗೆ ಆಧಾರ ನೀಡುತ್ತವೆ. ಈ ಮಾಡ್ಯೂಲ್‌ನ ಉಳಿದ ಭಾಗವು GPT-5.2 ಯ ಕಾರಣೀಕರಣ ನಿಯಂತ್ರಣ, ಸ್ವ-ಮುಳಿವು, ಮತ್ತು ಸಂರಚಿತ ಔಟ್‌ಪುಟ್ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಉಪಯೋಗಿಸುವ **ಎಂಟು ಅತ್ಯಾಧುನಿಕ ಮಾದರಿಗಳನ್ನು** ಒಳಗೊಂಡಿದೆ.

## ಅತ್ಯಾಧುನಿಕ ಮಾದರಿಗಳು

ಮೂಲತತ್ವಗಳನ್ನು ಆವರಿಸಿದ ನಂತರ, ಇಲ್ಲಿವೆ ಈ ಮಾಡ್ಯೂಲ್ ವೈಶಿಷ್ಟ್ಯಪೂರ್ಣವಾಗಿಸುವ ಎಂಟು ಅತ್ಯಾಧುನಿಕ ಮಾದರಿಗಳು. ಎಲ್ಲಾ ಸಮಸ್ಯೆಗಳಿಗೂ ಏಕೇ ವಿನ್ಯಾಸ ಆಯ್ಕೆ ಮಾಡಬೇಕಾಗಿಲ್ಲ. ಕೆಲವು ಪ್ರಶ್ನೆಗಳಿಗೆ ತ್ವರಿತ ಉತ್ತರ ಬೇಕಾಗುತ್ತದೆ, ಇತರರಿಗೆ ಆಳವಾದ ಚಿಂತನೆ ಅಗತ್ಯ. ಕೆಲವವರಿಗೆ ದೃಶ್ಯತ ಕಾರಣೀಕರಣ ಬೇಕಾಗುತ್ತದೆ, ಕೆಲವರಿಗೆ ಬರೀ ಫಲಿತಾಂಶ. ಕೆಳಗಿನ ಪ್ರತಿ ಮಾದರಿಯನ್ನೂ ವಿಭಿನ್ನ ಪರಿಸ್ಥಿತಿಗೆ ಹೊಂದಿದ್ದು — GPT-5.2 ಯ ಕಾರಣೀಕರಣ ನಿಯಂತ್ರಣದಿಂದ ಇವುಗಳ ವ್ಯತ್ಯಾಸಗಳು ಇನ್ನಷ್ಟು ಸ್ಪಷ್ಟವಾಗಿವೆ.

<img src="../../../translated_images/kn/eight-patterns.fa1ebfdf16f71e9a.webp" alt="ಎಂಟು ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಮಾದರಿಗಳು" width="800"/>

*ಎಂಟು ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಮಾದರಿಗಳ ಅವಲೋಕನ ಮತ್ತು ಅವುಗಳ ಉಪಯೋಗಗಳು*

<img src="../../../translated_images/kn/reasoning-control.5cf85f0fc1d0c1f3.webp" alt="GPT-5.2 ಮೂಲಕ ಕಾರಣೀಕರಣ ನಿಯಂತ್ರಣ" width="800"/>

*GPT-5.2 ಯ ಕಾರಣೀಕರಣ ನಿಯಂತ್ರಣ ನಿಮಗೆ ಎಷ್ಟು ಅರಿವು ಮತ್ತು ಮೆರಗು ಮಾಡಬೇಕು ಎಂದು ಸೂಚಿಸಲು ಅವಕಾಶ ನೀಡುತ್ತದೆ — ತ್ವರಿತ ನೇರ ಉತ್ತರದಿಂದ ಆಳವಾದ ಸಂಶೋಧನೆ ತನಕ*

**ಕಡಿಮೆ ಆಸಕ್ತಿ (ತ್ವರಿತ ಮತ್ತು ನಿರ್ದಿಷ್ಟ)** - ಸರಳ ಪ್ರಶ್ನೆಗಳಿಗೆ ವೇಗವಾಗಿ ನೇರ ಉತ್ತರ ಬೇಕಾದ ಸಂದರ್ಭದಲ್ಲಿ. ಮಾದರಿ ಕನಿಷ್ಟ ಕಾರಣೀಕರಣವನ್ನು ಮಾಡುತ್ತದೆ - ಗರಿಷ್ಠ 2 ಹಂತಗಳು. ಗಣನೆಗಳು, ಪೂರೈಸಿಕೆ, ಅಥವಾ ಸರಳ ಪ್ರಶ್ನೆಗಾಗಿ ಇದನ್ನು ಬಳಸಿ.

```java
String prompt = """
    <context_gathering>
    - Search depth: very low
    - Bias strongly towards providing a correct answer as quickly as possible
    - Usually, this means an absolute maximum of 2 reasoning steps
    - If you think you need more time, state what you know and what's uncertain
    </context_gathering>
    
    Problem: What is 15% of 200?
    
    Provide your answer:
    """;

String response = chatModel.chat(prompt);
```

> 💡 **GitHub Copilot ಜೊತೆಗೆ ಅನ್ವೇಷಿಸಿ:** [`Gpt5PromptService.java`](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java) ತೆಗೆಯಿರಿ ಮತ್ತು ಕೇಳಿ:
> - "ಕಡಿಮೆ ಆಸಕ್ತಿ ಮತ್ತು ಹೆಚ್ಚಿನ ಆಸಕ್ತಿ ಪ್ರಾಂಪ್ಟಿಂಗ್ ಮಾದರಿಗಳ ನಡುವಿನ ವ್ಯತ್ಯಾಸವೇನು?"
> - "ಪ್ರಾಂಪ್ಟ್‌ಗಳಲ್ಲಿ XML ಟ್ಯಾಗ್ಸ್ AI ಪ್ರತಿಕ್ರಿಯೆ ರಚನೆಗೆ ಹೇಗೆ ಸಹಾಯ ಮಾಡುತ್ತವೆ?"
> - "ಸ್ವ-ಪರಿಶೀಲನೆ ಮಾದರಿಗಳು ಮತ್ತು ನೇರ ಸೂಚನೆಗಳ rivalry ಯಾವಾಗ ಬಳಸಬೇಕು?"

**ಹೈ ಆಸಕ್ತಿ (ಆಳವಾದ ಮತ್ತು ಸಮಗ್ರ)** - ಜಟಿಲ ಸಮಸ್ಯೆಗಳಿಗೆ ಸಮಗ್ರ ವಿಶ್ಲೇಷಣೆ ಬೇಕಾಗಿರುವಾಗ. ಮಾದರಿ ಆಳವಾಗಿ ಪರಿಶೀಲನೆ ಮಾಡಿ ವಿವರವಾದ ಕಾರಣೀಕರಣ ತೋರಿಸುತ್ತದೆ. ಸಿಸ್ಟಮ್ ವಿನ್ಯಾಸ, ವಾಸ್ತುಶಿಲ್ಪ ನಿರ್ಣಯಗಳು, ಅಥವಾ ಜಟಿಲ ಸಂಶೋಧನೆಗೆ ಇದನ್ನು ಬಳಸಿ.

```java
String prompt = """
    Analyze this problem thoroughly and provide a comprehensive solution.
    Consider multiple approaches, trade-offs, and important details.
    Show your analysis and reasoning in your response.
    
    Problem: Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```

**ಕಾರ್ಯ ನಿರ್ವಹಣೆ (ಹಂತ-ಹಂತ ಪ್ರಗತಿ)** - ಬಹು-ಹಂತದ ಕಾರ್ಯಪ್ರವಾಹಗಳಿಗೆ. ಮಾದರಿ ಮುಂಭಾಗದ ಯೋಜನೆಯನ್ನು ನೀಡುತ್ತದೆ, ಪ್ರತಿ ಹಂತವನ್ನು ವಿವರಿಸಿ ಹೇಳುತ್ತದೆ, ನಂತರ ಸಾರಾಂಶ ನೀಡುತ್ತದೆ. ವಲಸೆ, ಜಾರಿಗೆ, ಅಥವಾ ಯಾವುದೇ ಬಹು-ಹಂತ ಪ್ರಕ್ರಿಯೆಗೆ ಉಪಯುಕ್ತ.

```java
String prompt = """
    <task_execution>
    1. First, briefly restate the user's goal in a friendly way
    
    2. Create a step-by-step plan:
       - List all steps needed
       - Identify potential challenges
       - Outline success criteria
    
    3. Execute each step:
       - Narrate what you're doing
       - Show progress clearly
       - Handle any issues that arise
    
    4. Summarize:
       - What was completed
       - Any important notes
       - Next steps if applicable
    </task_execution>
    
    <tool_preambles>
    - Always begin by rephrasing the user's goal clearly
    - Outline your plan before executing
    - Narrate each step as you go
    - Finish with a distinct summary
    </tool_preambles>
    
    Task: Create a REST endpoint for user registration
    
    Begin execution:
    """;

String response = chatModel.chat(prompt);
```

ಚೇನ್-ಆಫ್-ಥಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್ ಸ್ಪಷ್ಟವಾಗಿ ಪರಿಣಾಮಕಾರಿ ಕೆಲಸಗಳಿಗೆ ತನ್ನ ಕಾರಣೀಕರಣ ಪ್ರಕ್ರಿಯೆಯನ್ನು ತೋರಲು ಮಾದರಿಯನ್ನು ಕೇಳುತ್ತದೆ, ಇದರಿಂದ ನಿಖರತೆ ಹೆಚ್ಚಾಗುತ್ತದೆ. ಹಂತ-ಹಂತ ಪ್ರದೇಶಿಸುವಿಕೆ ಮಾನವರಿಗೂ AI ಗಾಗಿ ತರ್ಕವನ್ನು ಹೇರಳವಾಗಿ ಅರ್ಥಮಾಡಿಕೊಡುವುದು.

> **🤖 GitHub Copilot ಚಾಟ್ ನಲ್ಲಿ ಪ್ರಯತ್ನಿಸಿ:** ಈ ಮಾದರಿಯ ಬಗ್ಗೆ ಕೇಳಿ:
> - "ದೀರ್ಘಕಾಲದ ಕಾರ್ಯಗಳಿಗೆ ಕಾರ್ಯ ನಿರ್ವಹಣೆ ಮಾದರಿಯನ್ನು ನಾನು ಹೇಗೆ ಹೊಂದಿಸಬಹುದು?"
> - "ಉತ್ಪಾದನಾ ಅಪ್ಲಿಕೇಶನ್‌ಗಳಲ್ಲಿ ಟೂಲ್ ಪ್ರೀಂಬಲ್ಸ್ ರಚನೆಗೆ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳು ಏನು?"
> - "UI ನಲ್ಲಿ ಮಧ್ಯಂತರ ಪ್ರಗತಿ ನವೀಕರಣಗಳನ್ನು ನಾನು ಹೇಗೆ ಹಿಡಿದು ಪ್ರದರ್ಶಿಸಬಹುದು?"

<img src="../../../translated_images/kn/task-execution-pattern.9da3967750ab5c1e.webp" alt="ಕಾರ್ಯ ನಿರ್ವಹಣೆ ಮಾದರಿ" width="800"/>

*ಯೋಜನೆ → ನಿರ್ವಹಣೆ → ಸಾರಾಂಶ ಕಾರ್ಯಪ್ರವಾಹ ಬಹು-ಹಂತ ಕೆಲಸಗಳಿಗೆ*

**ಸ್ವ-ಪರಿಶೀಲನೆ ಕೋಡ್** - ಉತ್ಪಾದನಾ ಗುಣಮಟ್ಟದ ಕೋಡ್ ರಚನೆಗೆ. ಮಾದರಿ ಸರಿಯಾದ ದೋಷ ನಿರ್ವಹಣೆಯೊಂದಿಗೆ ಉತ್ಪಾದನಾ ಮಾನದಂಡಗಳನ್ನು ಅನುಸರಿಸುವ ಕೋಡ್ ರಚಿಸುತ್ತದೆ. ಹೊಸ ವೈಶಿಷ್ಟ್ಯಗಳು ಅಥವಾ ಸೇವೆಗಳನ್ನು ನಿರ್ಮಿಸುವಾಗ ಇದನ್ನು ಬಳಸಿ.

```java
String prompt = """
    Generate Java code with production-quality standards: Create an email validation service
    Keep it simple and include basic error handling.
    """;

String response = chatModel.chat(prompt);
```

*ಪುನರಾವರ್ತಿತ ಸುಧಾರಣಾ ಲೂಪ್ - ರಚಿಸಿ, ಮೌಲ್ಯಮಾಪನ ಮಾಡಿ, ಸಮಸ್ಯೆಗಳನ್ನು ಗುರುತಿಸಿ, ಸುಧಾರಿಸಿ, ಪುನರಾವರ್ತಿಸಿರಿ*

**ಸಂರಚಿತ ವಿಶ್ಲೇಷಣೆ** - ಸ್ಥಿರ ಮೌಲ್ಯಮಾಪನಕ್ಕೆ. ಮಾದರಿ ಕೋಡ್ ಪರಿಶೀಲನೆ ಸ್ಥಿರರೂಪದಲ್ಲಿ (ತಪ್ಪು, ಕಾರ್ಯಪಟುತೆ, ಕೌಶಲ್ಯಗಳು, ಭದ್ರತೆ, ನಿರ್ವಹಿಸಬಹುದಾದೀತೆ) ಮಾಡುತ್ತದೆ. ಕೋಡ್ ವಿಮರ್ಶೆಗಳು ಅಥವಾ ಗುಣಮಟ್ಟ ಮೌಲ್ಯಮಾಪನಗಳಿಗೆ ಉಪಯುಕ್ತ.

String prompt = """

**ಬಹು-ತಿರುವು ಚಾಟ್** - ಸಂವಾದಗಳಿಗೆ ಸಂದರ್ಭ ಅಗತ್ಯವಿದ್ದರೆ. ಮಾದರಿ ಹಿಂದಿನ ಸಂದೇಶಗಳನ್ನು ನೆನಪಿನಲ್ಲಿ ಇಟ್ಟುಕೊಂಡು ಅವುಗಳ ಮೇಲೆ ನಿರ್ಮಿಸುತ್ತಾನೆ. ಇಂಟರ್ಯಾಕ್ಟಿವ್ ಸಹಾಯ ಅಧಿವೇಶನಗಳು ಅಥವಾ ಜಟಿಲ ಪ್ರಶ್ನೋತ್ತರಗಳಿಗೆ ಇದನ್ನು ಬಳಸಿ.

**ಹಂತ-ಬದಲೆನ್ ಕಾರಣೀಕರಣ** - ದೃಶ್ಯತಾರಿತ ತರ್ಕ ಬೇಕಾದ ಸಮಸ್ಯೆಗಳಿಗೆ. ಮಾದರಿ ಪ್ರತಿ ಹಂತಕ್ಕೆ ಸ್ಪಷ್ಟ ಕಾರಣೀಕರಣ ತೋರಿಸುತ್ತದೆ. ಗಣಿತ ಸಮಸ್ಯೆಗಳು, ತರ್ಕ ಪಜಲ್ಸ್, ಅಥವಾ ಯೋಚನೆ ಪ್ರಕ್ರಿಯೆಯನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಬೇಕಾದಾಗ ಬಳಸಿ.

**ನಿಗದಿತ ಔಟ್‌ಪುಟ್** - ನಿಖರ ಸ್ವರೂಪದ ಅವಶ್ಯಕತೆಗಳೊಂದಿಗೆ ಪ್ರತಿಕ್ರಿಯೆಗಳಿಗೆ. ಮಾದರಿ ಸ್ವರೂಪ ಮತ್ತು ಉದ್ದ ನಿಯಮಗಳನ್ನು ಕಟ್ಟುನಿಟ್ಟಾಗಿ ಅನುಸರಿಸುತ್ತದೆ. ಸಾರಾಂಶಗಳು ಅಥವಾ ನಿಖರ ಔಟ್‌ಪುಟ್ ರಚನೆ ಬೇಕಾದಾಗ ಬಳಸಿ.

## ಇತರ ಅರ್ಹತೆ ಹೊಂದಿದ ಆಜುರ್ ಸಂಪನ್ಮೂಲಗಳನ್ನು ಬಳಸುವುದು

**ನಿಯೋಜನೆಯನ್ನು ಪರಿಶೀಲಿಸಿ:**

ರೂಟ್ ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ `.env` ಕಡತವು Azure ದೃಢೀಕರಣಗಳೊಂದಿಗೆ (ಮಾಡ್ಯೂಲ್ 01 ಸಮಯದಲ್ಲಿ ರಚಿಸಲಾಗಿದೆ) ಅಸ್ತಿತ್ವದಲ್ಲಿದೆಯೇ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ:
```bash
cat ../.env  # AZURE_OPENAI_ENDPOINT, API_KEY, DEPLOYMENT ಅನ್ನು ತೋರಿಸಬೇಕು
```

**ಅಪ್ಲಿಕೇಶನ್ ಪ್ರಾರಂಭಿಸಿ:**

> **ಗುರುತು:** ನೀವು ಈಗಾಗಲೇ ಮಾಡುವ ಎಲ್ಲಾ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು `./start-all.sh` ಮೂಲಕ ಮಾಡ್ಯೂಲ್ 01 ರಲ್ಲಿ ಪ್ರಾರಂಭಿಸಿಕೊಂಡಿದ್ದರೆ, ಈ ಮಾಡ್ಯೂಲ್ ಈಗಾಗಲೇ ಪೋರ್ಟ್ 8083 ಮೇಲೆ ಚಲಿಸುತ್ತಿದೆ. ಕೆಳಗಿನ ಪ್ರಾರಂಭ ಆದೆಶಗಳನ್ನು ಮೀರಿ ನೇರವಾಗಿ http://localhost:8083 ಗೆ ಹೋಗಬಹುದು.
**ಆಯ್ಕೆ 1: ಸ್ಪ್ರಿಂಗ್ ಬೂಟ್ ಡ್ಯಾಶ್‌ಬೋರ್ಡ್ ಬಳಸಿ (VS ಕೋಡ್ ಬಳಕೆದಾರರಿಗೆ ಶಿಫಾರಸು ಮಾಡಲಾಗಿದೆ)**

ಡೆವ್ ಕಂಟೈನರ್ ಸ್ಪ್ರಿಂಗ್ ಬೂಟ್ ಡ್ಯಾಶ್‌ಬೋರ್ಡ್ ವಿಸ್ತರಣೆ ಒಳಗೊಂಡಿದೆ, ಇದು ಎಲ್ಲಾ ಸ್ಪ್ರಿಂಗ್ ಬೂಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿರ್ವಹಿಸಲು ದೃಶ್ಯಾತ್ಮಕ ಇಂಟರ್ಫೇಸ್ ಅನ್ನು ಒದಗಿಸುತ್ತದೆ. ನೀವು ಇದನ್ನು VS ಕೋಡ್‌ನ ಎಡಬಕ್ಕೆ ಇರುವ<Activity Bar> ನಲ್ಲಿ ಕಂಡುಹಿಡಿಯಬಹುದು (स्प್ರಿಂಗ್ ಬೂಟ್ ಐಕಾನ್ ಅನ್ನು ಹುಡುಕಿ).

ಸ್ಪ್ರಿಂಗ್ ಬೂಟ್ ಡ್ಯಾಶ್‌ಬೋರ್ಡ್‌ನಿಂದ, ನೀವು:
- ವರ್ಕ್‌ಸ್ಪೇಸ್‌ನಲ್ಲಿನ ಎಲ್ಲ ಲಭ್ಯವಿರುವ ಸ್ಪ್ರಿಂಗ್ ಬೂಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನೋಡಬಹುದು
- ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಒಂದು ಕ್ಲಿಕ್‌ನಲ್ಲಿ ಪ್ರಾರಂಭ/ನಿಲ್ಲಿಸಬಹುದು
- ಅಪ್ಲಿಕೇಶನ್ ಲಾಗ್‌ಗಳನ್ನು ರಿಯಲ್-ಟೈಮ್‌ನಲ್ಲಿ ವೀಕ್ಷಿಸಬಹುದು
- ಅಪ್ಲಿಕೇಶನ್ ಸ್ಥಿತಿಯನ್ನು ಮೇಲ್ವಿಚಾರಣೆ ಮಾಡಬಹುದು

"prompt-engineering" ಗೆ ಪಕ್ಕದ ಪ್ಲೇ ಬಟ್‌ನ್ ಕ್ಲಿಕ್ ಮಾಡಿ ಈ ಮೋಡ್ಯೂಲ್ ಅನ್ನು ಪ್ರಾರಂಭಿಸಿ, ಅಥವಾ ಎಲ್ಲ ಮೋಡ್ಯೂಲ್‌ಗಳನ್ನು ಒಂದೇ ಬಾರಿ ಪ್ರಾರಂಭಿಸಿ.

## ಮಾದರಿಗಳನ್ನು ಅನ್ವೇಷಿಸುವುದು

ವೆಬ್ ಇಂಟರ್ಫೇಸ್ ನಿಮಗೆ ವಿಭಿನ್ನ ಪ್ರಾಂಪ್ಟಿಂಗ್ ತಂತ್ರಗಳನ್ನು ಪ್ರಯೋಗಿಸಲು ಅವಕಾಶ ನೀಡುತ್ತದೆ. ಪ್ರತಿ ಮಾದರಿ ವಿಭಿನ್ನ ಸಮಸ್ಯೆಗಳನ್ನು ಪರಿಹರಿಸುತ್ತದೆ - ಪ್ರತಿಯೊಂದು ವಿಧಾನವು ಯಾವಾಗ ಪ್ರಭಾವ ಹೆಚ್ಚಿಸುತ್ತದೆ ಅನ್ನೋದನ್ನು ನೋಡಿ ಪ್ರಯತ್ನಿಸಿ.

> **ಗಮನಿಸಿ: ಸ್ಟ್ರೀಮಿಂಗ್ ವಿರುದ್ಧ ನಾನ್-ಸ್ಟ್ರೀಮಿಂಗ್** — ಪ್ರತಿಯೊಂದು ಮಾದರಿ ಪುಟದಲ್ಲಿ ಎರಡು ಬಟನ್‌ಗಳು ಇವೆ: **🔴 ಸ್ಟ್ರೀಮ್ ಪ್ರತಿಕ್ರಿಯೆ (ಲೈವ್)** ಮತ್ತು **ನಾನ್-ಸ್ಟ್ರೀಮಿಂಗ್** ಆಯ್ಕೆ. ಸ್ಟ್ರೀಮಿಂಗ್ Server-Sent Events (SSE) ಬಳಸುತ್ತದೆ, ಇದರಿಂದ ಮಾದರಿ ತಯಾರಿಸುವ ಟೋಕನ್‌ಗಳನ್ನು ತಕ್ಷಣವೇ ವೀಕ್ಷಿಸಬಹುದು. ನಾನ್-ಸ್ಟ್ರೀಮಿಂಗ್ ಆಯ್ಕೆ ಸಂಪೂರ್ಣ ಪ್ರತಿಕ್ರಿಯೆಯನ್ನು ಕಾಯುತ್ತದೆ ಮತ್ತು ನಂತರ ತೋರಿಸುತ್ತದೆ. ಆಳವಾದ ತರ್ಕವನ್ನು ಪ್ರಚೋದಿಸುವ ಪ್ರಾಂಪ್ಟ್‌ಗಳು (ಉದಾಹರಣೆಗೆ, ಹೈ ಇAGERನೆಸ್, ಸ್ವ-ಪರಿಶೀಲನ ಸಂಪಾದನೆ)ಗಾಗಿ, ನಾನ್-ಸ್ಟ್ರೀಮಿಂಗ್ ಕಾಲ್ ಬಹಳ ಸಮಯ ಸೆಳೆಯಬಹುದು — ಕೆಲವೊಮ್ಮೆ ನಿಮಿಷಗಳ ಕಾಲ — ಯಾವುದೇ ದೃಶ್ಯ ಪ್ರತಿಕ್ರಿಯೆಯಿಲ್ಲದೆ. **ಸಂಕೀರ್ಣ ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ಪ್ರಯೋಗಿಸುವಾಗ ಸ್ಟ್ರೀಮಿಂಗ್ ಬಳಸಿ** ಇದು ಮಾದರಿ ಹೇಗೆ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತಿದೆ ಎಂಬುದನ್ನು ತೋರಿಸುತ್ತದೆ ಮತ್ತು ವಿನಂತಿ ಅವಧಿ ಮೀರಿದಂತೆ ತೋರುವ ಭ್ರಮೆಯನ್ನು ದೂರಮಾಡುತ್ತದೆ.
>
> **ಗಮನಿಸಿ: ಬ್ರೌಸರ್ ಅಗತ್ಯ** — ಸ್ಟ್ರೀಮಿಂಗ್ ವೈಶಿಷ್ಟ್ಯವು Fetch Streams API (`response.body.getReader()`) ನ್ನು ಬಳಸುತ್ತದೆ, ಇದು ಪೂರ್ಣ ಬ್ರೌಸರ್ (ಕ್ರೋಮ್, ಏಡ್ಜ್, ಫೈರ್ಫಾಕ್ಸ್, ಸಫಾರಿ) ಅಗತ್ಯವಿದೆ. VS ಕೋಡ್‌ನ ಸ್ಥಾಪಿತ ಸಿಂಪಲ್ ಬ್ರೌಸರ್‌ನಲ್ಲಿ ಇದು ಕಾರ್ಯನಿರ್ವಹಿಸುವುದಿಲ್ಲ, ಏಕೆಂದರಿಂದ ಅದರ ವೆಬ್ ವೀಕ್ಷಣೆ ReadableStream API ಗೆ ಬೆಂಬಲ ನೀಡುವುದಿಲ್ಲ. ಸಿಂಪಲ್ ಬ್ರೌಸರ್ ಬಳಸಿದರೆ ನಾನ್-ಸ್ಟ್ರೀಮಿಂಗ್ ಬಟನ್‌ಗಳು ಸರ್ವಸಾಮಾನ್ಯವಾಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸುವುದಲ್ಲದೆ ಕ್ರಿಯಾಶೀಲವಾಗಿವೆ — ಕೇವಲ ಸ್ಟ್ರೀಮಿಂಗ್ ಬಟನ್‌ಗಳು ಮಾತ್ರ ಪ್ರಭಾವಿತವಾಗಿವೆ. ಪೂರ್ಣ ಅನುಭವಕ್ಕಾಗಿ http://localhost:8083 ಅನ್ನು ಬಾಹ್ಯ ಬ್ರೌಸರ್‌ನಲ್ಲಿ ತೆರೆಯಿರಿ.

### ಕಡಿಮೆ ವಿರುದ್ಧ ಹೆಚ್ಚಿನ ಇAGERನೆಸ್

"200 ರ 15% ಎಷ್ಟು?" ಎಂಬ ಸರಳ ಪ್ರಶ್ನೆಯನ್ನು ಕಡಿಮೆ ಇAGERನೆಸ್ನೊಂದಿಗೆ ಕೇಳಿ. ನಿಮಗೆ ತಕ್ಷಣ ಮತ್ತು ನೇರ ಉತ್ತರ ಸಿಗುತ್ತದೆ. ಈಗ "ಹೈ ಟ್ರಾಫಿಕ್ API ಗೆ ಕ್ಯಾಚಿಂಗ್ ತಂತ್ರವೊಂದನ್ನು ವಿನ್ಯಾಸ ಮಾಡಿ" ಎಂಬ ಸಂಕೀರ್ಣ ಪ್ರಶ್ನೆಯನ್ನು ಹೆಚ್ಚಿನ ಇAGERನೆಸ್ನೊಂದಿಗೆ ಕೇಳಿ. **🔴 ಸ್ಟ್ರೀಮ್ ಪ್ರತಿಕ್ರಿಯೆ (ಲೈವ್)** ಕ್ಲಿಕ್ ಮಾಡಿ ಮತ್ತು ಮಾದರಿಯ ವಿವರವಾದ ತರ್ಕವನ್ನು ಟೋಕನ್-ಬೈ-ಟೋಕನ್ ವೀಕ್ಷಿಸಿ. ಅದೇ ಮಾದರಿ, ಅದೇ ಪ್ರಶ್ನೆ ರಚನೆ — ಆದರೆ ಪ್ರಾಂಪ್ಟ್ sellele ತರದಷ್ಟು ಚಿಂತನೆಮಾಡುತ್ತೆ ಎಂದು ಸೂಚಿಸುತ್ತದೆ.

### ಟಾಸ್‌ಕ್ ಕಾರ್ಯತತ್ಪಾದನೆ (ಟೂಲ್ ಪ್ರೀ ಅಂಬಲ್ಸ್)

ಬಹು-ಹಂತ ಕಾರ್ಯಪ್ರವಾಹಗಳು ಮುನ್ನೋಟ ಯೋಜನೆ ಮತ್ತು ಪ್ರಗತಿ ವರದಿಗೊಳಿಸುವಿಕೆಯಿಂದ ಲಾಭ ಪಡೆಯುತ್ತವೆ. ಮಾದರಿ ಏನು ಮಾಡಲಿದೆ ಎಂದು ವಿವರಿಸುತ್ತದೆ, ಪ್ರತಿ ಹಂತವನ್ನು ವಿವರಿಸುತ್ತದೆ, ನಂತರ ಫಲಿತಾಂಶಗಳನ್ನು ಸಾರಾಂಶಗೊಳಿಸುತ್ತದೆ.

### ಸ್ವ-ಪರಿಶೀಲನ ಕೋಡ್

"ಇಮೇಲ್ ಪರಿಶೀಲನೆ ಸೇವೆಯನ್ನು ರಚಿಸಿ" ಎಂದು ಪ್ರಯತ್ನಿಸಿ. ಸಿದ್ಧಾಂತವಾಗಿ ಕೋಡ್ ತಯಾರಿಸುವುದಲ್ಲದೆ, ಮಾದರಿ ಗುಣಮಟ್ಟದ ಮಾನದಂಡಗಳಿಗೆ ವಿರುದ್ಧವಾಗಿ ಮೌಲ್ಯಮಾಪನ ಮಾಡಿ, ದುರ್ಜನ್ತುಗಳನ್ನು ಪತ್ತೆಹಚ್ಚಿ, ಮತ್ತು ಸುಧಾರಣೆ ಮಾಡುತ್ತದೆ. ನೀವು ಅದನ್ನು ತಯಾರಾಗುವವರೆಗೆ ಪುನರಾವರ್ತಿಸುತ್ತೀರಿ.

### ಸಂರಚಿತ ವಿಶ್ಲೇಷಣೆ

ಕೋಡ್ ವಿಮರ್ಶೆಗಳಿಗೆ ಸ್ಥಿರ ಮೌಲ್ಯಮಾಪನ ಫ್ರೇಮ್ವರ್ಕ್‌ಗಳು ಅಗತ್ಯ. ಮಾದರಿ ವಿವರಿಸಿದ ವರ್ಗಗಳ (ತಪ್ಪಿಲ್ಲದಿಕೆ, ಅಭ್ಯಾಸಗಳು, ಕಾರ್ಯಕ್ಷಮತೆ, ಭದ್ರತೆ) ಮೇಲೆ ವಿಶ್ಲೇಷಣೆ ಮಾಡುತ್ತದೆ.

### ಬಹು-ತಿರುವಿನ ಚಾಟ್

"স্প್رಿಂಗ್ ಬೂಟ್ ಏನು?" ಎಂದು ಕೇಳಿ ನಂತರ ತಕ್ಷಣ "ಉದಾಹರಣೆಯೊಂದನ್ನು ತೋರಿಸಿ" ಎನ್ನಿ. ಮಾದರಿ ನಿಮ್ಮ ಮೊದಲ ಪ್ರಶ್ನೆಯನ್ನು ನೆನಸಿಕೊಂಡು ಸ್ಪ್ರಿಂಗ್ ಬೂಟ್ ಉದಾಹರಣೆ ನೀಡುತ್ತದೆ. ಸ್ಮೃತಿ ಇಲ್ಲದೆ, ಅದರ ಎರಡನೇ ಪ್ರಶ್ನೆ ಅಸ್ಪಷ್ಟವಾಗುತ್ತಿತ್ತು.

### ಹಂತ-ಹಂತವಾಗಿ ತರ್ಕ

ಗಣಿತದ ಸಮಸ್ಯೆಯನ್ನು ಆರಿಸಿ ಮತ್ತು ಹಂತ-ಹಂತ ತರ್ಕ ಮತ್ತು ಕಡಿಮೆ ಇAGERನೆಸ್ ಎರಡೂ ಪ್ರಯತ್ನಿಸಿ. ಕಡಿಮೆ ಇAGERನೆಸ್ ಮೈತ್ರಿತ ರೀತಿಯಲ್ಲಿ ಉತ್ತರ ನೀಡುತ್ತದೆ - ವೇಗವಾಗಿ ಆದರೆ ಸ್ಪಷ್ಟವಲ್ಲದೆ. ಹಂತ-ಹಂತ ತರ್ಕ ಪ್ರತಿ ಲೆಕ್ಕಾಚಾರ ಮತ್ತು ನಿರ್ಧಾರವನ್ನು ತೋರಿಸುತ್ತದೆ.

### ನಿಯಂತ್ರಿತ ನೆರಳು

ನೀವು ಖಚಿತ ರೂಪಾಂತರಗಳು ಅಥವಾ ಪದಗಳ ಸಂಖ್ಯೆಯನ್ನು ಬೇಕಾದಾಗ, ಈ ಮಾದರಿ ಕಟ್ಟುನಿಟ್ಟಿನ ಅನುಸರಣೆ ಕಾಪಾಡುತ್ತದೆ. 100 ಪದಗಳನ್ನು ತೂಕದ ಪಟ್ಟಿಯಾಗಿ ಉತ್ಪಾದಿಸುವುದನ್ನು ಪ್ರಯತ್ನಿಸಿ.

## ನೀವು ನಿಜವಾಗಿಯೂ ಕಲಿಯುತ್ತಿರುವುದು

**ತರ್ಕದ ಪ್ರಯತ್ನ ಎಲ್ಲವನ್ನೂ ಬದಲಾಯಿಸುತ್ತದೆ**

GPT-5.2 ನಿಮ್ಮ ಪ್ರಾಂಪ್ಟ್‌ಗಳ ಮೂಲಕ ಲೆಕ್ಕಾಚಾರ ಪ್ರಯತ್ನವನ್ನು ನಿಯಂತ್ರಿಸಲು ಅನುಮತಿಸುತ್ತದೆ. ಕಡಿಮೆ ಪ್ರಯತ್ನವು ವೇಗದ ಪ್ರತಿಕ್ರಿಯೆಗಳೊಂದಿಗೆ ಕನಿಷ್ಟ ಅನ್ವೇಷಣೆ ಇರುತ್ತದೆ. ಹೆಚ್ಚಿನ ಪ್ರಯತ್ನವು ಮಾದರಿಯು ಆಳವಾಗಿ ಚಿಂತಿಸಲು ಸಮಯ ತೆಗೆದುಕೊಳ್ಳುತ್ತದೆ. ನೀವು ಕಾರ್ಯ ಸಂಕೀರ್ಣತೆಗೆ ಪ್ರಯತ್ನವನ್ನು ಹೊಂದಿಸುವುದನ್ನು ಕಲಿಯುತ್ತಿದ್ದೀರಿ - ಸರಳ ಪ್ರಶ್ನೆಗಳಿಗೆ ಸಮಯ ವ್ಯಯಿಸಬೇಡಿ, ಆದರೆ ಸಂಕೀರ್ಣ ನಿರ್ಧಾರಗಳನ್ನು ಕೂಡ ಶೀಘ್ರವಾಗಿ ಮಾಡಬೇಡಿ.

**ರಚನೆ ನಡೆದುಕೊಳ್ಳುವಿಕೆಯನ್ನು ಮಾರ್ಗನಿರ್ಧರಿಸುತ್ತದೆ**

ಪ್ರಾಂಪ್ಟ್‌ಗಳಲ್ಲಿರುವ XML ಟ್ಯಾಗ್ಗಳನ್ನು ಗಮನಿಸಿ? ಅವು ನೆಗಡಾಗಿರುವುದಲ್ಲ. ಮಾದರಿಗಳು ರಚಿತ ಸೂಚನೆಗಳನ್ನು ಸ್ವತಂತ್ರ ಪಠ್ಯಕ್ಕಿಂತ ಹೆಚ್ಚು ನಂಬಿಕೆಯೊಂದಿಗೆ ಅನುಸರಿಸುತ್ತವೆ. ನೀವು ಬಹು-ಹಂತ ಪ್ರಕ್ರಿಯೆಗಳ ಅಥವಾ ಜಟಿಲ ತರ್ಕ ಬೇಕಿದ್ದಾಗ, ರಚನೆ ಮೂಲಕ ಮಾದರಿ ತನ್ನ ಸ್ಥಾನ ಹಾಗೂ ಮುಂದೇನು ಬರುವುದೆಂಬುದನ್ನು ಹಾದುಹೋಗಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.

**ಗುಣಮಟ್ಟ ಸ್ವ-ಮೌಲ್ಯಮಾಪನದ ಮೂಲಕ**

ಸ್ವ-ಪರಿಶೀಲನಾ ಮಾದರಿಗಳು ಗುಣಮಟ್ಟದ ಮಾನದಂಡಗಳನ್ನು ಸ್ಪಷ್ಟಪಡಿಸುವ ಮೂಲಕ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತವೆ. ಮಾದರಿ "ಸರಿಯಾಗಿ ಮಾಡುತ್ತದೆ" ಎಂದು ನಿರೀಕ್ಷಿಸುವ ಬದಲು, ನೀವು "ಸರಿಯಾದದ್ದು" ಎಂದರೇನು ಎಂಬುದನ್ನು ನಿಖರವಾಗಿ ಹೇಳುತ್ತೀರಿ: ಸರಿಯಾದ ತಟಸ್ಥತೆ, ದೋಷ ನಿರ್ವಹಣೆ, ಕಾರ್ಯಕ್ಷಮತೆ, ಭದ್ರತೆ. ಬಳಿಕ ಮಾದರಿ ತನ್ನ ಸ್ವಂತ ಔಟ್ಪುಟ್ ಅನ್ನು ಮೌಲ್ಯಮಾಪನ ಮಾಡಿ ಸುಧಾರಣೆ ಮಾಡಬಹುದು. ಇದು ಕೋಡ್ ತಯಾರಿಕೆಯನ್ನು ಲಾಟರಿ ಇರಬಲ್ಲುದನ್ನೂ ಪ್ರಕ್ರಿಯೆಯಾಗಿಸಲು ಮಾರ್ಪಡಿಸುತ್ತದೆ.

**ಸಂದರ್ಭವು ನಿರ್ಧಿಷ್ಟ**

ಬಹು-ತಿರುವಿನ ಸಂಭಾಷಣೆಗಳು ಪ್ರತಿ ವಿನಂತಿಯೊಂದಿಗೆ ಸಂದೇಶ ಇತಿಹಾಸವನ್ನು ಒಳಗೊಂಡಿರುವ ಮೂಲಕ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತವೆ. ಆದರೆ ಮಿತಿ ಇದೆ - ಪ್ರತಿ ಮಾದರಿಯು ಗರಿಷ್ಟ ಟೋಕನ್ ಎಣಿಕೆಯಿಂದ ಕೂಡಿದೆ. ಸಂಭಾಷಣೆಗಳು ಹೆಚ್ಚು ಆಗಿದ್ದಂತೆ, ಪ್ರಸ್ತುತ ಸಂಬಂಧಿತ ಸੰਦਰಭವನ್ನು ಉಳಿಸಲು ತಂತ್ರಗಳನ್ನು ನೀವು ಅಗತ್ಯಪಡುವಿರಿ. ಈ ಮೋಡ್ಯೂಲ್ ನಿಮ್ಮಿಗೆ ಸ್ಮೃತಿ ಹೇಗೆ ಕೆಲಸ ಮಾಡುತ್ತದೆ ಎಂಬುದನ್ನು ತೋರಿಸುತ್ತದೆ; ನಂತರ ನೀವು ಯಾವಾಗ ಸಾರಾಂಶ ಮಾಡಬೇಕು, ಯಾವಾಗ ಮರೆತು ಬಿಡಬೇಕು ಮತ್ತು ಯಾವಾಗ ಪತ್ತೆ ಮಾಡಲು ಬೇಕೆಂದು ಕಲಿಯುತ್ತೀರಿ.
