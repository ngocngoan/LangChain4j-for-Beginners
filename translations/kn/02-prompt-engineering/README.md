## ನೀವು ಕಲಿಯುವವುಗಳು

<img src="../../../translated_images/kn/what-youll-learn.c68269ac048503b2.webp" alt="ನೀವು ಕಲಿಯುವವುಗಳು" width="800"/>

ಹಿಂದಿನ ಮód್ಯೂಳಿನಲ್ಲಿ, ನೀವು ಮೆಮೊರಿ ಸಂಭಾಷಣಾಶೀಲ AI ಅನ್ನು ಹೇಗೆ ಸ Journalism ಸಹಾಯಕರಾಗಿ ಉಪಯೋಗಿಸಬಹುದು ಎಂದು ನೋಡಿದ್ದೀರಿ ಮತ್ತು GitHub ಮಾದರಿಗಳನ್ನು ಮೂಲಭೂತ ಸಂವಾದಗಳಿಗೆ ಬಳಸದಿದ್ದಿರಿ. ಈಗ ನಾವು ಪರಿಶೀಲಿಸುವುದು ನೀವು ಹೇಗೆ ಪ್ರಶ್ನೆಗಳನ್ನು ಕೇಳುತ್ತೀರಿ — ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು — Azure OpenAIನ GPT-5.2 ಬಳಸಿ. ನೀವು ನಿಮ್ಮ ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ರಚಿಸುವ ವಿಧಾನವು ಪ್ರತಿಕ್ರಿಯೆಗಳ ಗುಣಮಟ್ಟವನ್ನು ಬಹಳ ಹೆಚ್ಚಿಸುತ್ತದೆ. ನಾವು ಮೂಲ ಪ್ರಾಂಪ್ಟ್ ತಂತ್ರಗಳನ್ನು ಪರಿಶೀಲಿಸುವುದರಿಂದ ಪ್ರಾರಂಭಿಸುತ್ತೇವೆ; ನಂತರ GPT-5.2 ಯ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಸಂಪೂರ್ಣವಾಗಿ ಉಪಯೋಗಿಸುವ ಎಂಟು ಉನ್ನತ ವಿನ್ಯಾಸಗಳನ್ನು ಅನುಷ್ಠಾನಗೊಳಿಸುತ್ತೇವೆ.

GPT-5.2 ಯನ್ನು ನಾವು ಬಳಸುವುದೇಕೆಂದರೆ ಅದು ಯುಕ್ತಿಯ ನಿಯಂತ್ರಣವನ್ನು ಪರಿಚಯಿಸುತ್ತದೆ — ನೀವು ಮಾದರಿಯನ್ನು ಉತ್ತರಿಸುವ ಮುನ್ನ ಎಷ್ಟು ಯೋಚನೆಯು ಮಾಡಬೇಕು ಎಂದು ಹೇಳಬಹುದು. ಇದು ವಿಭಿನ್ನ ಪ್ರಾಂಪ್ಟ್ ರಣನೀತಿಗಳನ್ನು ಹೆಚ್ಚಿನ ಸ್ಪಷ್ಟತೆಯಿಂದ ತೋರಿಸುತ್ತದೆ ಮತ್ತು ಯಾವಾಗ ಯಾವ ವಿಧಾನವನ್ನು ಬಳಸಬೇಕೆಂದು ತಿಳಿಯಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ. GitHub ಮಾದರಿಗಳಿಗಿಂತ Azure GPT-5.2 ಗೆ ಕಡಿಮೆ ರೇಟು ಮಿತಿಗಳು ಇರಬೇಕಾಗಿರುವುದರಿಂದ ನಾವು ಅದರಿಂದ ಪ್ರಯೋಜನ ಪಡೆಯುತ್ತೇವೆ.

## ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಅನ್ನು ಅರ್ಥೈಸಿಕೊಳ್ಳುವುದು

<img src="../../../translated_images/kn/what-is-prompt-engineering.5c392a228a1f5823.webp" alt="ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಎಂದರೇನು?" width="800"/>

ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಎಂದರೆ ನೀವು ಬೇಕಾದ ಫಲಿತಾಂಶಗಳನ್ನು ಏಕಾಗ್ರತೆಯಾಗಿಸಲು ಇನ್‌ಪುಟ್ ಪಠ್ಯವನ್ನು ವಿನ್ಯಾಸಗೊಳಿಸುವುದಾಗಿದೆ. ಇದು ಕೇವಲ ಪ್ರಶ್ನೆಗಳನ್ನು ಕೇಳುವುದಲ್ಲ — ಇದು ಮಾದರಿಯು ನಿಖರವಾಗಿ ನೀವು ಏನು ಬಯಸುತ್ತೀರಿ ಮತ್ತು ಅದನ್ನು ಹೇಗೆ ಒದಗಿಸಬೇಕೆಂಬುದನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವಂತೆ ವಿನಂತಿಗಳನ್ನು ರಚಿಸುವುದಾಗಿದೆ.

ಇದನ್ನು ಸಹೋದ್ಯೋಗಿಗೆ ಸೂಚನೆ ಒದಗಿಸುವಂತೆ ಬಿಂಬಿಸಿ. "ತೊಳೆಯಿರಿ" ಎನ್ನುವುದು ಅಸ್ಪಷ್ಟ. "UserService.java ಫೈಲ್‌ನಲ್ಲಿ 45ನೇ ಸಾಲಿನ ನಲ್ ಪಾಯಿಂಟರ್ ಎಕ್ಸೆಪ್ಷನ್ ಅನ್ನು ನಲ್ ಚೆಕ್ ಸೇರಿಸುವ ಮೂಲಕ ಸರಿಪಡಿಸಿ" ಎಂದರೆ ವಿಶಿಷ್ಟವಾಗಿದೆ. ಭಾಷಾ ಮಾದರಿಗಳು ಕೂಡ ಇದೇ ರೀತಿಯಲ್ಲಿ ಕೆಲಸ ಮಾಡುತ್ತವೆ — ನಿಖರತೆ ಮತ್ತು ರಚನೆ ಮುಖ್ಯ.

<img src="../../../translated_images/kn/how-langchain4j-fits.dfff4b0aa5f7812d.webp" alt="ಲ್ಯಾಂಗ್‌ಚೈನ್4ಜೆ ಹೇಗೆ ಹೊಂದಿಕೊಳ್ಳುತ್ತದೆ" width="800"/>

ಲ್ಯಾಂಗ್‌ಚೈನ್4ಜೆ ಮೂಲಸೌಕರ್ಯವನ್ನು ಒದಗಿಸುತ್ತದೆ — ಮಾದರಿ ಸಂಪರ್ಕಗಳು, ಮೆಮೊರಿ, ಮತ್ತು ಸಂದೇಶ ವಿಧಗಳು — ಆದರೆ ಪ್ರಾಂಪ್ಟ್ ವಿನ್ಯಾಸಗಳು ಆ ಮೂಲಸೌಕರ್ಯದಲ್ಲಿ ನೀವು ಕಳುಹಿಸುವ ಸೂಕ್ಷ್ಮವಾಗಿ ವಿನ್ಯಾಸಗೊಳಿಸಿದ ಪಠ್ಯ ಮಾತ್ರ. ಪ್ರಮುಖ ಕಟ್ಟಡ ಬ್ಲಾಕ್ಗಳಾಗಿ `SystemMessage` (ಯಾವುದರಲ್ಲಿ AI ನ ವರ್ತನೆ ಮತ್ತು ಪಾತ್ರವನ್ನು ಹೊಂದಿಸಲಾಗುತ್ತದೆ) ಮತ್ತು `UserMessage` (ನಿಮ್ಮ ನಿಜವಾದ ವಿನಂತಿಯನ್ನು ಹೊರುತ್ತದೆ) ಇರುತ್ತವೆ.

## ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಮೂಲತತ್ವಗಳು

<img src="../../../translated_images/kn/five-patterns-overview.160f35045ffd2a94.webp" alt="ಐದು ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ವಿನ್ಯಾಸಗಳ ಅವಲೋಕನ" width="800"/>

ಈ ಅಧ್ಯಾಯದ ಉನ್ನತ ವಿನ್ಯಾಸಗಳಿಗೆ ಮುನ್ನಡೆಯುವುದಕ್ಕೆ ಮುನ್ನ, ಐದು ಆಧಾರಭೂತ ಪ್ರಾಂಪ್ಟ್ ತಂತ್ರಗಳನ್ನು ಪರಿಶೀಲಿಸೋಣ. ಪ್ರತಿ ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರ್ ತಿಳಿದಿರುವುದು ಮುಖ್ಯ. ನೀವು ಈಗಾಗಲೇ [ಕ್ವಿಕ್ ಸ್ಟಾರ್ಟ್ ಮಾಡ್ಯೂಳಿನ](../00-quick-start/README.md#2-prompt-patterns) ಮೂಲಕ ಈ ತಂತ್ರಗಳನ್ನು ನೋಡಿದ್ದೀರಾ — ಇವುಗಳ ಹಿಂದೆ ಕಲ್ಪನಾ ಚಟುವಟಿಕೆ ಇದೆ.

### ಶೂನ್ಯ-ಶಾಟ್ ಪ್ರಾಂಪ್ಟ್

ಸರಳ ವಿಧಾನ: ಮಾದರಿಗೆ ಉದಾಹರಣೆ ಇಲ್ಲದೆ ನೇರ ಸೂಚನೆ ನೀಡುವುದು. ಮಾದರಿ ಸಂಪೂರ್ಣವಾಗಿ ಅದರ ತರಬೇತಿಯ ಮೇರೆಗೆ ಕಾರ್ಯವನ್ನು ತಿಳಿಯುತ್ತ ಮೈಮರೆಸುತ್ತದೆ. ನಿರೀಕ್ಷಿತ ವರ್ತನೆ ಸ್ಪಷ್ಟವಾಗಿರುವ ನೇರ ವಿನಂತಿಗಳಿಗೆ ಇದು ಸೂಕ್ತ.

<img src="../../../translated_images/kn/zero-shot-prompting.7abc24228be84e6c.webp" alt="ಶೂನ್ಯ-ಶಾಟ್ ಪ್ರಾಂಪ್ಟ್" width="800"/>

*ಉದಾಹರಣೆಗಳಿಲ್ಲದ ನೇರ ಸೂಚನೆ — ಮಾದರಿ ಸೂಚನೆಯಲ್ಲಿ ನಿರೀಕ್ಷಿತ ಕಾರ್ಯವನ್ನು ಊಹಿಸುತ್ತದೆ*

```java
String prompt = "Classify this sentiment: 'I absolutely loved the movie!'";
String response = model.chat(prompt);
// ಪ್ರತಿಕ್ರಿಯೆ: "ಧನಾತ್ಮಕ"
```

**ಯಾವಾಗ ಬಳಸುವುದು:** ಸರಳ ವರ್ಗೀಕರಣಗಳು, ನೇರ ಪ್ರಶ್ನೆಗಳು, ಭಾಷಾಂತರಗಳು ಅಥವಾ ಹೆಚ್ಚುವರಿ ಮಾರ್ಗದರ್ಶನವಿಲ್ಲದ ಯಾವುದೇ ಕಾರ್ಯ.

### ಕೆಲವು-ಶಾಟ್ ಪ್ರಾಂಪ್ಟ್

ನೀವು ಬಯಸುವ ಮಾದರಿ ಅನುಸರಿಸಬೇಕಾದ ವಿನ್ಯಾಸವನ್ನು ತೋರಿಸುವ ಉದಾಹರಣೆಗಳನ್ನು ನೀಡಿರಿ. ಮಾದರಿ ನಿಮ್ಮ ಉದಾಹರಣೆಗಳಿಂದ ನಿರೀಕ್ಷಿತ ಇನ್‌ಪುಟ್-ಔಟ್‌ಪುಟ್ ವಿನ್ಯಾಸವನ್ನು ಕಲಿಯುತ್ತದೆ ಮತ್ತು ಹೊಸ ಇನ್‌ಪುಟ್‌ಗಳಿಗೆ ಇದು ಅನ್ವಯಿಸುತ್ತದೆ. ಇದು ನಿರೀಕ್ಷಿತ ಮಾದರಿ ಅಥವಾ ವರ್ತನೆ ಸ್ಪಷ್ಟವಿಲ್ಲದ ಕೆಲಸಗಳಿಗೆ ಒಂದು ಸಮರ್ಥತೆಯನ್ನು ಹೆಚ್ಚಿಸುತ್ತದೆ.

<img src="../../../translated_images/kn/few-shot-prompting.9d9eace1da88989a.webp" alt="ಕೆಲವು-ಶಾಟ್ ಪ್ರಾಂಪ್ಟ್" width="800"/>

*ಉದಾಹರಣೆಗಳಿಂದ ಕಲಿಯುವುದು — ಮಾದರಿ ವಿನ್ಯಾಸವನ್ನು ಗುರುತಿಸಿ ಹೊಸ ಇನ್‌ಪುಟ್‌ಗಳ ಮೇಲೆ ಅನ್ವಯಿಸುತ್ತದೆ*

```java
String prompt = """
    Classify the sentiment as positive, negative, or neutral.
    
    Examples:
    Text: "This product exceeded my expectations!" → Positive
    Text: "It's okay, nothing special." → Neutral
    Text: "Waste of money, very disappointed." → Negative
    
    Now classify this:
    Text: "Best purchase I've made all year!"
    """;
String response = model.chat(prompt);
```

**ಯಾವಾಗ ಬಳಸುವುದು:** ಕ್ರಮಬದ್ಧ ವರ್ಗೀಕರಣಗಳು, ಸರಳ ಸ್ವರೂಪಣೆ, ವಿಶಿಷ್ಟ ಕ್ಷೇತ್ರ-ನಿರ್ದಿಷ್ಟ ಕಾರ್ಯಗಳು ಅಥವಾ ಶೂನ್ಯ-ಶಾಟ್ ಫಲಿತಾಂಶ ಅಸಮರ್ಪಕವಾದಾಗ.

### ಚೇನ್ ಆಫ್ ಥಾಟ್

ಮಾದರಿಯನ್ನು ಹಂತ ಹಂತವಾಗಿ ತನ್ನ ಯುಕ್ತಿಯನ್ನು ತೋರಿಸಲು ಕೇಳಿ. ನೇರ ಉತ್ತರಕ್ಕೆ ಹಾರಿಡುವ ಬದಲು, ಮಾದರಿ ಸಮಸ್ಯೆಯನ್ನು ವಿಭಾಗಿಸಿ ಪ್ರತಿಯೊಂದು ಭಾಗವನ್ನು ಸ್ಪಷ್ಟವಾಗಿ ವಿವರಿಸುತ್ತದೆ. ಇದು ಗಣಿತ, ಲಾಜಿಕ್ ಮತ್ತು ಬಹು ಹಂತ ಯುಕ್ತಿಗಳ ಮೇಲೆ ಶುದ್ಧತೆಯನ್ನು ಹೆಚ್ಚಿಸುತ್ತದೆ.

<img src="../../../translated_images/kn/chain-of-thought.5cff6630e2657e2a.webp" alt="ಚೇನ್ ಆಫ್ ಥಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್" width="800"/>

*ಹಂತ ಹಂತದ ಯುಕ್ತಿ — ಸಂಕೀರ್ಣ ಸಮಸ್ಯೆಗಳನ್ನು ಸ್ಪಷ್ಟವಾದ ಲಾಜಿಕ್ ಹಂತಗಳಲ್ಲಿ ವಿಭಜಿಸುವುದು*

```java
String prompt = """
    Problem: A store has 15 apples. They sell 8 apples and then 
    receive a shipment of 12 more apples. How many apples do they have now?
    
    Let's solve this step-by-step:
    """;
String response = model.chat(prompt);
// ಮಾದರಿ ತೋರಿಸುತ್ತದೆ: 15 - 8 = 7, ನಂತರ 7 + 12 = 19 ಸೇಬುಗಳು
```

**ಯಾವಾಗ ಬಳಸುವುದು:** ಗಣಿತ ಸಮಸ್ಯೆಗಳು, ಲಾಜಿಕ್ ಪಜಲ್ಸ್, ಡೀಬಗ್ಗಿಂಗ್ ಅಥವಾ ಯುಕ್ತಿ ಪ್ರಕ್ರಿಯೆಯನ್ನು ತೋರಿಸುವುದರಿಂದ ಶುದ್ಧತೆ ಮತ್ತು ನಂಬಿಕೆಯು ಹೆಚ್ಚಾಗುವ ಯಾವುದೇ ಕಾರ್ಯ.

### ಪಾತ್ರ ಆಧಾರಿತ ಪ್ರಾಂಪ್ಟ್

ನಿಮ್ಮ ಪ್ರಶ್ನೆ ಕೇಳುವ ಮುನ್ನ AI ಗಾಗಿ ವ್ಯಕ್ತಿತ್ವ ಅಥವಾ ಪಾತ್ರವನ್ನು ನಿಶ್ಚಿತಗೊಳಿಸಿ. ಇದು ಉತ್ತರದ ಶೈಲಿ, ಆಳ ಮತ್ತು ಕೇಂದ್ರೀಕರಣವನ್ನು ರೂಪಿಸುವ ಸಾಂದರ್ಭಿಕತೆಯನ್ನು ಒದಗಿಸುತ್ತದೆ. "ಸಾಫ್ಟ್ವೇರ್ ಆರ್ಕಿಟೆಕ್ಟ್" "ಜೂನಿಯರ್ ಡೆವಲಪರ್" ಅಥವಾ "ಸುರಕ್ಷತಾ ತಪಾಸಕ" ಕ್ಕಿಂತ ವಿಭಿನ್ನ ಸಲಹೆ ನೀಡುತ್ತದೆ.

<img src="../../../translated_images/kn/role-based-prompting.a806e1a73de6e3a4.webp" alt="ಪಾತ್ರ ಆಧಾರಿತ ಪ್ರಾಂಪ್ಟ್" width="800"/>

*ಸಾಂದರ್ಭಿಕತೆ ಮತ್ತು ಪಾತ್ರವನ್ನು ಹೊಂದಿಸುವುದು — ನಿಗದಿತ ಪಾತ್ರ ಆಧರಿಸಿ ಒಂದೇ ಪ್ರಶ್ನೆಗೆ ವಿಭಿನ್ನ ಉತ್ತರಗಳೆರಡು*

```java
String prompt = """
    You are an experienced software architect reviewing code.
    Provide a brief code review for this function:
    
    def calculate_total(items):
        total = 0
        for item in items:
            total = total + item['price']
        return total
    """;
String response = model.chat(prompt);
```

**ಯಾವಾಗ ಬಳಸುವುದು:** ಕೋಡ್ ಸಮೀಕ್ಷೆಗಳು, ಟ್ಯುಟೋರಿಂಗ್, ಕ್ಷೇತ್ರ-ನಿರ್ದಿಷ್ಟ ವಿಶ್ಲೇಷಣೆ ಅಥವಾ ನಿರ್ದಿಷ್ಟ ಪರಿಣಿತ ಮಟ್ಟ ಅಥವಾ ದೃಷ್ಟಿಕೋನಕ್ಕೆ ಹೊಂದುತ್ತಿರುವ ಪ್ರತಿಕ್ರಿಯೆಗಳ ಬೇಕಾದಾಗ.

### ಪ್ರಾಂಪ್ಟ್ ಟೆಂಪ್ಲೇಟುಗಳು

ಚಾರಣಮಾಡಬಹುದಾದ ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ರಚಿಸಿ. ಪ್ರತಿಸಾರಿ ಹೊಸ ಪ್ರಾಂಪ್ಟ್ ಬರೆಯುವುದಕ್ಕೆ ಬದಲಾಗಿ, ಒಂದು ಟೆಂಪ್ಲೇಟನ್ನು ಒಂದೇ ಬಾರಿ ನಿರ್ಧರಿಸಿ ಭಿನ್ನ ಮೌಲ್ಯಗಳನ್ನು ತುಂಬಿ ಬಳಸಿಕೊಳ್ಳಿ. ಲ್ಯಾಂಗ್‌ಚೈನ್4ಜೆನ `PromptTemplate` ತರಗತಿ `{{variable}}` ವ್ಯಾಕರಣದೊಂದಿಗೆ ಇದನ್ನು ಸುಲಭಗೊಳಿಸುತ್ತದೆ.

<img src="../../../translated_images/kn/prompt-templates.14bfc37d45f1a933.webp" alt="ಪ್ರಾಂಪ್ಟ್ ಟೆಂಪ್ಲೇಟುಗಳು" width="800"/>

*ಚಾರಣಮಾಡಬಹುದಾದ ಪ್ರಾಂಪ್ಟ್‌ಗಳು ಮತ್ತು ವ್ಯತ್ಯಾಸದ ನಿಯತಾಂಕಗಳು — ಒಂದು ಟೆಂಪ್ಲೇಟು, ಅನೇಕ ಬಳಕೆ*

```java
PromptTemplate template = PromptTemplate.from(
    "What's the best time to visit {{destination}} for {{activity}}?"
);

Prompt prompt = template.apply(Map.of(
    "destination", "Paris",
    "activity", "sightseeing"
));

String response = model.chat(prompt.text());
```

**ಯಾವಾಗ ಬಳಸುವುದು:** ವಿಭಿನ್ನ ಇನ್‌ಪುಟ್ಗಳೊಂದಿಗೆ ಪುನರಾವರ್ತಿತ ಪ್ರಶ್ನೆಗಳು, ಬ್ಯಾಚ್ ಪ್ರಕ್ರಿಯೆ, ಚಾರಣಮಾಡಬಹುದಾದ AI ಕಾರ್ಯಪ್ರವಾಹ ನಿರ್ಮಾಣ, ಅಥವಾ ಪ್ರಾಂಪ್ಟ್ ರಚನೆ ಇದೇ ಇದ್ದು ಡೇಟಾ ಬದಲಾಗುವ ಸನ್ನಿವೇಶಗಳು.

---

ಈ ಐದು ಮೂಲತತ್ವಗಳು ಬಹುತೇಕ ಪ್ರಾಂಪ್ಟ್ ಕಾರ್ಯಗಳಿಗೆ ಉಚಿತವಾದ ಉಪಕರಣಸಾಮಗ್ರಿಯನ್ನು ಒದಗಿಸುತ್ತವೆ. ಈ ಅಧ್ಯಾಯದ დარჩಿದ ಭಾಗವು GPT-5.2 ಯ ಯುಕ್ತಿಯ ನಿಯಂತ್ರಣ, ಸ್ವ-ಮೌಲ್ಯಮಾಪನ ಮತ್ತು ಸಂರಚಿತ ಔಟ್‌ಪುಟ್ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಬಳಸುವ **ಎಂಟು ಉನ್ನತ ವಿನ್ಯಾಸಗಳು** ಒಳಗೊಂಡಿದೆ.

## ಅಧಿಕವಾದ ವಿನ್ಯಾಸಗಳು

ಮೂಲತತ್ವಗಳಾದ ನಂತರ, ಈ ಅಧ್ಯಾಯವನ್ನು ವಿಶಿಷ್ಟಗೊಳಿಸುವ ಎಂಟು ಉನ್ನತ ವಿನ್ಯಾಸಗಳಿಗೆ ಹೋಗೋಣ. ಎಲ್ಲಾ ಸಮಸ್ಯೆಗಳು ಸಮಾನ ವಿಧಾನವನ್ನು ಅಗತ್ಯವಿಲ್ಲ. ಕೆಲವು ಪ್ರಶ್ನೆಗಳು ವೇಗದ ಉತ್ತರಗಳನ್ನು ಬೇಕಾಗಿರುತೆ, ಮತ್ತಷ್ಟು ಆಳವಾದ ಯೋಚನೆಯನ್ನು ಬೇಡಿಕೊಳ್ಳುತ್ತವೆ. ಕೆಲವು ಪ್ರದರ್ಶಿತ ಯುಕ್ತಿಯನ್ನು ಅಗತ್ಯವಿರುತ್ತದೆ, ಇನ್ನೆರಡು ಕೇವಲ ಫಲಿತಾಂಶ ಬೇಕಾಗಿರುತ್ತದೆ. ಕೆಳಗಿನ ಪ್ರತಿಯೊಂದು ವಿನ್ಯಾಸವು ವಿಭಿನ್ನ ಸ್ಥಿತಿಗತಿಯಿಗಾಗಿ ಉಚಿತಗೊಳಿಸಲಾಗಿದೆ — ಮತ್ತು GPT-5.2 ಯ ಯುಕ್ತಿಯ ನಿಯಂತ್ರಣ ಈ ವ್ಯತ್ಯಾಸಗಳನ್ನು ಇನ್ನೂ ಸ್ಪಷ್ಟ ಮಾಡುತ್ತದೆ.

<img src="../../../translated_images/kn/eight-patterns.fa1ebfdf16f71e9a.webp" alt="ಎಂಟು ಪ್ರಾಂಪ್ಟ್ ವಿನ್ಯಾಸಗಳು" width="800"/>

*ಎಂಟು ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ವಿನ್ಯಾಸಗಳ ಅವಲೋಕನ ಮತ್ತು ಅವುಗಳ ಬಳಕೆ ಪ್ರಕರಣಗಳು*

<img src="../../../translated_images/kn/reasoning-control.5cf85f0fc1d0c1f3.webp" alt="GPT-5.2 ಯೊಂದಿಗೆ ಯುಕ್ತಿಯ ನಿಯಂತ್ರಣ" width="800"/>

*GPT-5.2 ಯ ಯುಕ್ತಿಯ ನಿಯಂತ್ರಣವು ನೀವು ಮಾದರಿ ಎಷ್ಟು ಯೋಚಿಸಿ ಉತ್ತರಿಸಬೇಕು ಎಂದು ವಿವರಿಸಲು ಅವಕಾಶ ನೀಡುತ್ತದೆ — ವೇಗದ ನೇರ ಉತ್ತರಗಳಿಂದ ಆಳವಾದ ಅನ್ವೇಷಣೆಯವರೆಗೆ*

<img src="../../../translated_images/kn/reasoning-effort.db4a3ba5b8e392c1.webp" alt="ಯುಕ್ತಿಯ ಪ್ರಯತ್ನದ ಹೋಲಿಕೆ" width="800"/>

*ಕಡಿಮೆ ಆಸಕ್ತಿಯ (ವೇಗವಾಗಿರುವ, ನೇರ) ಮತ್ತು ಹೆಚ್ಚು ಆಸಕ್ತಿಯ (ವಿಸ್ತೃತ, ಅನ್ವೇಷಣಾತ್ಮಕ) ಯುಕ್ತಿ ವಿಧಾನಗಳ ಹೋಲಿಕೆ*

**ಕಡಿಮೆ ಆಸಕ್ತಿ (ವೇಗವಾಗಿ ಮತ್ತು ಕೇಂದ್ರೀಕೃತ)** - ಸರಳ ಪ್ರಶ್ನೆಗಳಿಗೆ ವೇಗದ ನೇರ ಉತ್ತರ ಬೇಕಾದಾಗ. ಮಾದರಿ ಕನಿಷ್ಠ ಯುಕ್ತಿಯೊಂದಿಗೆ ಕಾರ್ಯಗತಗೊಳ್ಳುತ್ತದೆ — ಗರಿಷ್ಠ 2 ಹಂತಗಳು. ಗಣನೆ, ಹುಡುಕಾಟ ಅಥವಾ ನೇರ ಪ್ರಶ್ನೆಗಳಿಗೆ ಇದನ್ನು ಬಳಸಿ.

```java
String prompt = """
    <context_gathering>
    - Search depth: very low
    - Bias strongly towards providing a correct answer as quickly as possible
    - Usually, this means an absolute maximum of 2 reasoning steps
    - If you think you need more time, state what you know and what's uncertain
    </context_gathering>
    
    Problem: What is 15% of 200?
    
    Provide your answer:
    """;

String response = chatModel.chat(prompt);
```

> 💡 **GitHub Copilot ನೊಂದಿಗೆ ಅನ್ವೇಷಿಸಿ:** [`Gpt5PromptService.java`](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java) ತೆರೆಯಿರಿ ಮತ್ತು ಕೇಳಿ:
> - "ಕಡಿಮೆ ಆಸಕ್ತಿ ಮತ್ತು ಹೆಚ್ಚು ಆಸಕ್ತಿ ಪ್ರಾಂಪ್ಟ್ ವಿನ್ಯಾಸಗಳ ನಡುವಿನ ವ್ಯತ್ಯಾಸವೇನು?"
> - "ಪ್ರಾಂಪ್ಟ್‌‌ಗಳಲ್ಲಿ XML ಟ್ಯಾಗ್ಗಳಿಂದ AI ಉತ್ತರದ ರಚಿಸಿ ಹೇಗೆ ಸಹಾಯ ಮಾಡುತ್ತದೆ?"
> - "ಸ್ವ-ಪರಿಶೀಲನೆ ವಿನ್ಯಾಸಗಳನ್ನು ಮತ್ತು ನೇರ ಸೂಚನೆಗಳನ್ನು ಯಾವಾಗ ಬಳಸಬೇಕು?"

**ಹೆಚ್ಚು ಆಸಕ್ತಿ (ಆಳವಾದ ಮತ್ತು ವಿಸ್ತೃತ)** - ಸಂಕೀರ್ಣ ಸಮಸ್ಯೆಗಳಿಗೆ ಸಮಗ್ರ ವಿಶ್ಲೇಷಣೆ ಬೇಕಾದಾಗ. ಮಾದರಿ ವಿಶ್ಲೇಷಣೆಯನ್ನು ಸೂಕ್ಷ್ಮವಾಗಿ ಮಾಡಿ ವಿವರವಾದ ಯುಕ್ತಿಯನ್ನು ತೋರ್ಪಡಿಸುತ್ತದೆ. ವ್ಯವಸ್ಥೆಯ ವಿನ್ಯಾಸ, ವಾಸ್ತುಶಿಲ್ಪ ನಿರ್ಧಾರಗಳು, ಅಥವಾ ಸಂಕೀರ್ಣ ಸಂಶೋಧನೆಗೆ ಇದನ್ನು ಉಪಯೋಗಿಸಿ.

```java
String prompt = """
    Analyze this problem thoroughly and provide a comprehensive solution.
    Consider multiple approaches, trade-offs, and important details.
    Show your analysis and reasoning in your response.
    
    Problem: Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```

**ಕಾರ್ಯ ನೆರವೇರಿಕೆ (ಹಂತ ಹಂತವಾಗಿ ಪ್ರಗತಿ)** - ಬಹು ಹಂತ ಕಾರ್ಯಪದ್ಧತಿಗಳಿಗೆ. ಮಾದರಿ ಮೊದಲೇ ಒಂದು ಯೋಜನೆಯನ್ನು ನೀಡುತ್ತದೆ, ಕೆಲಸ ಮಾಡುವ ಪ್ರತಿಯೊಂದು ಹಂತವನ್ನು ವಿವರಿಸುತ್ತದೆ, ನಂತರ ಸಂಕ್ಷಿಪ್ತ ಸಾರಾಂಶ ನೀಡುತ್ತದೆ. ನಿರ್ವಹಣೆ, ಜಾರಿಗೊಳಿಸುವಿಕೆ ಅಥವಾ ಯಾವುದೇ ಬಹು ಹಂತ ಪ್ರಕ್ರಿಯೆಗಳಿಗೆ ಇದನ್ನು ಬಳಸಿ.

```java
String prompt = """
    <task_execution>
    1. First, briefly restate the user's goal in a friendly way
    
    2. Create a step-by-step plan:
       - List all steps needed
       - Identify potential challenges
       - Outline success criteria
    
    3. Execute each step:
       - Narrate what you're doing
       - Show progress clearly
       - Handle any issues that arise
    
    4. Summarize:
       - What was completed
       - Any important notes
       - Next steps if applicable
    </task_execution>
    
    <tool_preambles>
    - Always begin by rephrasing the user's goal clearly
    - Outline your plan before executing
    - Narrate each step as you go
    - Finish with a distinct summary
    </tool_preambles>
    
    Task: Create a REST endpoint for user registration
    
    Begin execution:
    """;

String response = chatModel.chat(prompt);
```

ಚೇನ್-ಆಫ್-ಥಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್ ಮಾದರಿಯಿಂದ ಯುಕ್ತಿ ಪ್ರಕ್ರಿಯೆಯನ್ನು ತೋರಿಸುವಂತೆ ಹಚ್ಚುತ್ತದೆ, ಇದು ಸಂಕೀರ್ಣ ಕಾರ್ಯಗಳಿಗೆ ಶುದ್ಧತೆಯನ್ನು ಸುಧಾರಿಸುತ್ತದೆ. ಹಂತ ಹಂತದ ವಿಭಜನೆ ಮಾನವರಿಗೂ ಮತ್ತು AIಗೂ ಲಾಜಿಕ್ ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು ಸಹಾಯಕ.

*ಪದವಿ: ಯೋಜಿಸಿ → ನೇರವಾಗಿರಿ → ಸಾರಾಂಶ ರಚಿಸಿ ಬಹು ಹಂತ ಕಾರ್ಯಗಳಿಗಾಗಿ*

**ಸ್ವ-ಪರಿಶೀಲನೆಯ ಕೋಡ್** - ಉತ್ಪಾದನಾ-ಗುಣಮಟ್ಟದ ಕೋಡ್ ತಯಾರಿಸಲು. ಮಾದರಿ ಉತ್ಪಾದನಾ ಮಾನದಂಡಗಳ ಅನುಸರಿಸಿಕೋಡ್ ರಚಿಸುತ್ತದೆ ಮತ್ತು ಸರಿಯಾದ ದೋಷ ನಿರ್ವಹಣೆ ಒದಗಿಸುತ್ತದೆ. ಹೊಸ ಲಕ್ಷಣಗಳು ಅಥವಾ ಸೇವೆಗಳು ನಿರ್ಮಿಸುವಾಗ ಇದನ್ನು ಬಳಸಿ.

*ಪುನರಾವರ್ತಿತ ಸುಧಾರಣಾ ಚಕ್ರ - ರಚನೆ, ಮೌಲ್ಯಮಾಪನ, ಸಮಸ್ಯೆ ಗುರುತಿಸುವಿಕೆ, ಸುಧಾರಣೆ, ಮರುಕಳುಹಿಸುವಿಕೆ*

**ಸಂರಚಿತ ವಿಶ್ಲೇಷಣೆ** - ಸಮರ್ಥನೀಯ ಮೌಲ್ಯಮಾಪನಕ್ಕಾಗಿ. ಮಾದರಿ ಒಂದು ನಿಗದಿತ_FRAMEWORK (ಸರಿಯಾದುದು, ಪ್ರಾಕ್ಟಿಸಸ್, ಕಾರ್ಯಕ್ಷಮತೆ, ಭದ್ರತೆ, ನಿರ್ವಹಣೆ ಸಾಧ್ಯತೆ) ಗೆ ತಕ್ಕಂತೆ ಕೋಡ್ ಪರಿಶೀಲನೆ ಮಾಡುತ್ತದೆ. ಕೋಡ್ ವಿಮರ್ಶೆ ಅಥವಾ ಗುಣಮಟ್ಟ ಮೌಲ್ಯಮಾಪನಗಳಿಗೆ ಇದನ್ನು ಬಳಸಿರಿ.

- ರಿಯಲ್-ಟೈಮ್‌ನಲ್ಲಿ ಅಪ್ಲಿಕೇಶನ್ ಲಾಗ್‌ಗಳನ್ನು ವೀಕ್ಷಿಸಲು

ವೆಬ್ ಇಂಟರ್ಫೇಸ್ ನಿಮಗೆ ವಿಭಿನ್ನ ಪ್ರಾಂಪ್ಟಿಂಗ್ ತಂತ್ರಗಳನ್ನು ಪ್ರಯೋಗಿಸಲು ಅವಕಾಶನೀಡುತ್ತದೆ. ಪ್ರತಿ ಮಾದರಿ ವಿಭಿನ್ನ ಸಮಸ್ಯೆಗಳಿಗೆ ಪರಿಹಾರ ನೀಡುತ್ತದೆ - ಪ್ರತಿ ವಿಧಾನವು ಯಾವಾಗ ಬೆಳಕಾಗುತ್ತದೆ ಎಂದು ನೋಡಿ ಪ್ರಯತ್ನಿಸಿ.

### ಕಡಿಮೆ ಮತ್ತು ಅಧಿಕ ಆಸಕ್ತಿ

"200 ರ 15% ಏನು?" ಎಂದು ಕೇವಲ ಸರಳ ಪ್ರಶ್ನೆಯನ್ನು ಕಡಿಮೆ ಆಸಕ್ತಿಯಿಂದ ಕೇಳಿ. ನಿಮಗೆ ಕ್ಷಣಿಕ, ನೇರ ಉತ್ತರ ಸಿಗುತ್ತದೆ. ಈಗ "ಹೈ-ಟ್ರಾಫಿಕ್ API ಗೆ ಕ್ಯಾಚ್ stratagy ವಿನ್ಯಾಸ ಮಾಡಿ" ಎಂಬ ಕಂದुकದ ಪ್ರಶ್ನೆಯನ್ನು ಅಧಿಕ ಆಸಕ್ತಿಯಿಂದ ಕೇಳಿ. ಮಾದರಿ ನಿಧಾನವಾಗಿ ಚಿಂತನೆ ಮಾಡಿ ವಿವರವಾದ ತರ್ಕವನ್ನು ಒದಗಿಸುತ್ತದೆ. ಅದೇ ಮಾದರಿ, ಅದೇ ಪ್ರಶ್ನಾ ರಚನೆ - ಆದರೆ ಪ್ರಾಂಪ್ಟ್ ಅದಕ್ಕೆ ಎಷ್ಟು ಚಿಂತನೆ ಮಾಡಬೇಕೆಂದು ಹೇಳುತ್ತದೆ.

<img src="../../../translated_images/kn/low-eagerness-demo.898894591fb23aa0.webp" alt="Low Eagerness Demo" width="800"/>

*ಕನಿಷ್ಠ ತರ್ಕದೊಂದಿಗೆ ತ್ವರಿತ ಲೆಕ್ಕಾಚಾರ*

<img src="../../../translated_images/kn/high-eagerness-demo.4ac93e7786c5a376.webp" alt="High Eagerness Demo" width="800"/>

*ಸಂಕ್ಷಿಪ್ತ ಕ್ಯಾಸಿಂಗ್ stratagy (2.8MB)*

### ಕಾರ್ಯ ನಿರ್ವಹಣೆ (ಟೂಲ್ ಪ್ರೀಂಬಲ್ಸ್)

ಬಹು-ಹಂತ ಕಾರ್ಯಪ್ರವಾಹಗಳು ಮುಂಚಿತ ಯೋಜನೆ ಮತ್ತು ಪ್ರಗತಿ ವರ್ಣನೆದಿಂದ ಲಾಭಗಳಿಸುತ್ತವೆ. ಮಾದರಿ ಅದು ಏನು ಮಾಡಲಿದೆ, ಪ್ರತಿ ಹಂತವನ್ನು ವಿವರಿಸಿ, ನಂತರ ಫಲಿತಾಂಶಗಳನ್ನು ಸಾರಾಂಶಗೊಳಿಸುತ್ತದೆ.

<img src="../../../translated_images/kn/tool-preambles-demo.3ca4881e417f2e28.webp" alt="Task Execution Demo" width="800"/>

*ಹಂತ-ಹಂತವಾಗಿ ವರ್ಣನೆಯೊಂದಿಗೆ REST endpoint ಸೃಷ್ಟಿ (3.9MB)*

### ಸ್ವಯಂ-ಪರಿಶೀಲನೆ ಕೋಡ್

"ಇಮೇಲ್ ಮಾನ್ಯತೆ ಸೇವೆ ಸೃಷ್ಟಿಸಿ" ಎಂದು ಪ್ರಯತ್ನಿಸಿ. ಕೇವಲ ಕೋಡ್ ರಚಿಸಿ ನಿಲ್ಲಿಸುವ ಬದಲು, ಮಾದರಿ ರಚಿಸಿ, ಗುಣಮಟ್ಟದ ಮಾನದಂಡಗಳ ವಿರುದ್ಧ ಮೌಲ್ಯಮಾಪನ ಮಾಡಿ, ದುರ್ಬಲತೆಗಳನ್ನು ಗುರುತಿಸಿ ಮತ್ತು ಸುಧಾರಿಸುತ್ತದೆ. ನೀವು ಇದು ಉತ್ಪಾದನಾ ಮಾನದಂಡಗಳನ್ನು ತಲುಪುವವರೆಗೆ ಪುನರಾವರ್ತನೆ ಮಾಡತಾನೆಂದು ನೋಡುತ್ತೀರಿ.

<img src="../../../translated_images/kn/self-reflecting-code-demo.851ee05c988e743f.webp" alt="Self-Reflecting Code Demo" width="800"/>

*ಪೂರ್ಣ ಇಮೇಲ್ ಮಾನ್ಯತೆ ಸೇವೆ (5.2MB)*

### ರಚನಾತ್ಮಕ ವಿಶ್ಲೇಷಣೆ

ಕೋಡ್ ವಿಮರ್ಶೆಗೆ ಸ್ಥಿರ ಮೌಲ್ಯಮಾಪನ ಚಟುವಟಿಕೆಗಳ ಅವಶ್ಯಕತೆ ಇದೆ. ಮಾದರಿ ಸರಿಯಾದತೆಯ (correctness), ಅಭ್ಯಾಸಗಳು (practices), ಕಾರ್ಯಕ್ಷಮತೆ (performance), ಭದ್ರತೆ (security) ಮುಂತಾದ ಸ್ಥಿತಿಗಳಲ್ಲಿ ಅವಲೋಕನ ಮಾಡುತ್ತದೆ.

<img src="../../../translated_images/kn/structured-analysis-demo.9ef892194cd23bc8.webp" alt="Structured Analysis Demo" width="800"/>

*ಫ್ರೇಮ್ವರ್ಕ್ನ ಆಧಾರದ ಕೋಡ್ ವಿಮರ್ಶೆ*

### ಬಹು-ತಿರುವು ಸಂಭಾಷಣೆ

"Spring Boot ಎಂದರೆ ಏನು?" ಎಂದು ಕೇಳಿ, ಆಮೇಲೆ ಕೂಡಲೇ "ಉದಾಹರಣೆಯನ್ನು ತೋರಿಸಿ" ಎಂದು ಕೇಳಿ. ಮೊದಲು ಕೇಳಿದ ಪ್ರಶ್ನೆಯನ್ನು ಮಾದರಿ ನೆನಸಿಕೊಳ್ಳುತ್ತದೆ ಮತ್ತು ನಿಮಗೆ ನಿಖರವಾಗಿ Spring Bootದ ಉದಾಹರಣೆ ನೀಡುತ್ತದೆ. ನೆನಪಿಲ್ಲದೆ, ಎರಡನೇ ಪ್ರಶ್ನೆ ಬಹಳ अस्पಷ್ಟವಾಗಬಹುದು.

<img src="../../../translated_images/kn/multi-turn-chat-demo.0d2d9b9a86a12b4b.webp" alt="Multi-Turn Chat Demo" width="800"/>

*ಪ್ರಶ್ನೆಗಳ ಬಣ್ಣ ತಾಲೀಪು ರಕ್ಷಣೆ*

### ಹಂತ ಹಂತದ ತರ್ಕ

ಒಂದು ಗಣಿತದ ಸಮಸ್ಯೆಯನ್ನು ಆಯ್ಕೆ ಮಾಡಿ ಮತ್ತು ಹಂತ-ಹಂತದ ತರ್ಕ ಹಾಗೂ ಕಡಿಮೆ ಆಸಕ್ತಿಯಿಂದ ಪ್ರಯತ್ನಿಸಿ. ಕಡಿಮೆ ಆಸಕ್ತಿ ನಿಮಗೆ ಉತ್ತರವನ್ನು ಮಾತ್ರ ನೀಡುತ್ತದೆ - ವೇಗವಾಗಿ ಆದರೆ ಅಸ್ಪಷ್ಟವಾಗಿ. ಹಂತ-ಹಂತ ತರ್ಕ ಎಲ್ಲ ಲೆಕ್ಕಾಚಾರ ಮತ್ತು ನಿರ್ಣಯವನ್ನು ತೋರಿಸುತ್ತದೆ.

### ನಿರ್ಬಂಧಿತ ಔಟ್‌പುಟ್

ನೀವು ನಿರ್ದಿಷ್ಟ ಸ್ವರೂಪಗಳು ಅಥವಾ ಪದಗಳ ಸಂಖ್ಯೆಯನ್ನು ಬೇಕಾದರೆ, ಈ ಮಾದರಿ ಕಟ್ಟುನಿಟ್ಟಾದ ಅನುಸರಣೆಯನ್ನು ಒತ್ತಾಯಿಸುತ್ತದೆ. ಬಿಂದುವಿನ ಸ್ವರೂಪದಲ್ಲಿ ನಿಖರವಾಗಿ 100 ಪದಗಳ ಸಾರಾಂಶವನ್ನು ರಚಿಸಿ.

<img src="../../../translated_images/kn/constrained-output-demo.567cc45b75da1633.webp" alt="Constrained Output Demo" width="800"/>

*ಸ್ವರೂಪ ನಿಯಂತ್ರಣದೊಂದಿಗೆ ಯಂತ್ರ ಅಧ್ಯಯನ ಸಾರಾಂಶ*

## ನೀವು ನಿಜವಾಗಿಯೂ ಕಲಿಯುತ್ತಿರುವುದು

**ತರ್ಕ ಪ್ರಯತ್ನ ಎಲ್ಲವನ್ನು ಬದಲಿಸುತ್ತದೆ**

GPT-5.2 ನಿಮ್ಮ ಪ್ರಾಂಪ್ಟ್‌ಗಳ ಮೂಲಕ ಗಣನೆ ಪ್ರಯತ್ನವನ್ನು ನಿಯಂತ್ರಿಸಲು ಅವಕಾಶ ನೀಡುತ್ತದೆ. ಕಡಿಮೆ ಪ್ರಯತ್ನವು ತ್ವರಿತ ಪ್ರತಿಕ್ರಿಯೆ ಮತ್ತು ಕನಿಷ್ಠ ಹುಡುಕಾಟವನ್ನು ಅರ್ಥ ಮಾಡಿಕೊಳ್ಳುತ್ತದೆ. ಹೆಚ್ಚಿನ ಪ್ರಯತ್ನವು ಆಳವಾಗಿ ಯೋಚನೆ ಮಾಡಲು ಸಮಯ ತೆಗೆದುಕೊಳ್ಳುತ್ತದೆ. ನೀವು ಕಾರ್ಯದ сложностью ಗೆ ಅನುಗುಣವಾಗಿ ಪ್ರಯತ್ನವನ್ನು ಹೊಂದಿಸುವುದನ್ನು ಕಲಿದುಕೊಳ್ಳುತ್ತಿದ್ದಾರೆ - ಸರಳ ಪ್ರಶ್ನೆಗಳಲ್ಲಿ ಸಮಯ ವ್ಯರ್ಥಮಾಡಬಹುದಿಲ್ಲ, ಆದರೆ ಸಂಕೀರ್ಣ ನಿರ್ಧಾರಗಳಲ್ಲಿ ಜಾಸ್ತಿ ಬೇಗಬಾರದಿರಿ.

**ರಚನೆ ನಡೆ ನಮೂದಿಸು ತಾಳು**

ಪ್ರಾಂಪ್ಟ್‌ಗಳಲ್ಲಿನ XML ಟ್ಯಾಗ್‌ಗಳನ್ನು ಗಮನಿಸಿರಿ? ಅವು ಅಲಂಕಾರಿಕವಲ್ಲ. ಮಾದರಿಗಳು ಸುಗಮವಾಗಿ ಸರಳ ಮತ್ತು XMLಹೀಗಿರಲು ಸ್ಟ್ರಕ್ಚರ್ಡ್ ಸೂಚನೆಗಳನ್ನು ಅನುಸರಿಸುತ್ತವೆ. ಬಹು ಹಂತ ಪ್ರಕ್ರಿಯೆಗಳು ಅಥವಾ ಸಂಕೀರ್ಣ ತರ್ಕ ಬೇಕಾದಾಗ, ರಚನೆ ಮಾದರಿಯನ್ನು ಹಿಂಬಾಲಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.

<img src="../../../translated_images/kn/prompt-structure.a77763d63f4e2f89.webp" alt="Prompt Structure" width="800"/>

*ಸ್ಪಷ್ಟ ವಿಭಾಗ ಮತ್ತು XML ಶೈಲಿಯ ಸಂಘಟನೆ ಸಹಿತ ಉತ್ತಮ ರಚನೆಯ ಪ್ರಾಂಪ್ಟ್ ವಿಶ್ಲೇಷಣೆ*

**ಗುಣಮಟ್ಟ ಸ್ವಯಂ ಮೌಲ್ಯಮಾಪನದಿಂದ**

ಸ್ವಯಂ ಪರಿಶೀಲನೆ ಮಾದರಿಗಳು ಗುಣಮಟ್ಟದ ಮಾನದಂಡಗಳನ್ನು ಸ್ಪಷ್ಟಪಡಿಸುವ ಮೂಲಕ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತವೆ. ಮಾದರಿ "ಸರಿಯಾಗಿ" ಎಂದು ಆಶಿಸುವ ಬದಲು, ನೀವು ಅದಕ್ಕೆ "ಸರಿಯಾಗಿ" ಎಂದರೆ ಏನು ಎಂಬುದನ್ನು ನಿಖರವಾಗಿ ಹೇಳುತ್ತೀರಿ: ಸರಿಯಾದ ಲಾಜಿಕ್, ದೋಷ ನಿರ್ವಹಣೆ, ಕಾರ್ಯಕ್ಷಮತೆ, ಭದ್ರತೆ. ಆಗ ಮಾದರಿ ಅದರ ಹೊರತನ್ನು ಮೌಲ್ಯಮಾಪನ ಮಾಡಿ ಸುಧಾರಿಸಬಹುದು. ಇದು ಕೇಸ್ ರಚನೆಯನ್ನು ಲಾಟರಿ ಅಲ್ಲ, ಪ್ರಕ್ರಿಯೆಯಾಗಿಸುವುದು.

**ಸಂದರ್ಭವು ಸೀಮಿತವಾಗಿದೆ**

ಬಹು-ತಿರುವು ಸಂಭಾಷಣೆಗಳು ಪ್ರತಿ ವಿನಂತಿಯೊಂದಿಗೆ ಸಂದೇಶ ಇತಿಹಾಸವನ್ನು ಒಳಗೊಂಡಿರುತ್ತವೆ. ಆದರೆ ಗರಿಷ್ಠ ಟೋಕನ್ ಮಿತಿ ಇದೆ. ಸಂಭಾಷಣೆ ವೃದ್ಧಿಸುವಂತೆ, ನೀವು ಸಂಬಂಧಿತ ಸಂದರ್ಭವನ್ನು ಕಾಪಾಡುವ ತಂತ್ರಗಳು ಬೇಕಾಗುತ್ತವೆ ಬೇಡ ಟೋಕನ್ ಮಿತಿ ತಲುಪದಂತೆ. ಈ ಮಾಡ್ಯೂಲ್ ಮೆಮೋರಿಯ ಕಾರ್ಯವನ್ನು ತೋರಿಸುತ್ತದೆ; ನಂತರ ನೀವು 언제 ಸಾರಾಂಶ ಮಾಡಬೇಕು, 언제 ಮರೆತುಹೋಗಬೇಕು, 언제 ವಿಮರ್ಶಿಸಬೇಕು ಎಂದು ಕಲಿಯುತ್ತೀರಿ.
