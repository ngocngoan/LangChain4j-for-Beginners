## ನೀವು ಏನು ಕಲಿಯುತ್ತೀರಿ

<img src="../../../translated_images/kn/what-youll-learn.c68269ac048503b2.webp" alt="ನೀವು ಏನು ಕಲಿಯುತ್ತೀರಿ" width="800"/>

ಹಿಂದಿನ ವಿಷಯದಲ್ಲಿ ನೀವು ಸಂವಹನಾತ್ಮಕ AIಗೆ ಮೆಮೊರಿಯು ಹೇಗೆ ಸಹಾಯಮಾಡುತ್ತದೆ ಎಂದು ನೋಡಿದ್ದಿರಿ ಮತ್ತು ಮೂಲಭೂತ ಸಂವಹನಗಳಿಗೆ GitHub ಮಾದರಿಗಳನ್ನು ಬಳಸಿದ್ದೀರಿ. ಈಗ ನಾವು ನಿಮ್ಮ ಪ್ರಶ್ನೆಗಳನ್ನು ಕೇಳುವ ವಿಧಾನ — ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು — Azure OpenAI ನ GPT-5.2 ಮೂಲಕ ಹೇಗೆ ವಿನ್ಯಾಸಗೊಳಿಸಬೇಕೆಂದು ಗಮನ ನೀಡುತ್ತಿದ್ದೇವೆ. ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ರಚಿಸುವ ವಿಧಾನವು ನೀವು ಪಡೆಯುವ ಪ್ರತಿಕ್ರಿಯೆಗಳ ಗುಣಮಟ್ಟವನ್ನು ಬಹಳ ಪ್ರಭಾವಿಸುತ್ತದೆ. ನಾವು ಪ್ರಾಂಪ್ಟ್ ಇಂಜಿನಿಯರಿಂಗ್ ಮೂಲತತ್ವಗಳ ವಿಮರ್ಶೆಯಿಂದ ಪ್ರಾರಂಭಿಸಿ, ನಂತರ GPT-5.2 ಅವರ ಸಂಪೂರ್ಣ ಸಾಮರ್ಥ್ಯವನ್ನು ಬಳಸುವ ಎಂಟು ಪ್ರಗತಿಪರ ಮಾದರಿಗಳಿಗೆ ಸಾಗುತ್ತೇವೆ.

ನೀವು GPT-5.2 ಬಳಸದಿರುವುದು ಕಾರಣ ಘನ: ಇದರಿಂದ ಯುಕ್ತಿವಾದಕ್ಕೆ ನಿಯಂತ್ರಣ ನೀಡಲ್ಪಟ್ಟಿದೆ - ಎಲ್ಲಾ ಉತ್ತರ ನೀಡುವುದಕ್ಕೂ ಮೊದಲು ಎಷ್ಟು ಆಲೋಚನೆ ಮಾಡಬೇಕೆಂದು ಮಾದರಿಗೆ ನೀವು ತಿಳಿಸಬಹುದು. ಇದು ವಿವಿಧ ಪ್ರಾಂಪ್ಟ್ ತಂತ್ರಗಳನ್ನು ಸ್ಪಷ್ಟಗೊಳಿಸುತ್ತದೆ ಮತ್ತು ಯಾವ ಸಂದರ್ಭಗಳಲ್ಲಿ ಯಾವ ವಿಧಾನವನ್ನು ಬಳಸಬೇಕೆಂದು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ. GitHub ಮಾದರಿಗಳಿಗಿಂತ GPT-5.2 ಗೆ ಅಜೂರ್ ನ ಕಡಿಮೆ ದರ ಮಿತಿ ಲಾಭವಾಗುತ್ತದೆ.

## ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಅರ್ಥಮಾಡಿಕೊಳ್ಳಿ

<img src="../../../translated_images/kn/what-is-prompt-engineering.5c392a228a1f5823.webp" alt="ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಎಂದರೇನು?" width="800"/>

ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಎಂದರೆ ನೀವು ಬೇಕಾದ ಫಲಿತಾಂಶಗಳನ್ನು ಸತತವಾಗಿ ನೀಡುವ ಇನ್‌ಪುಟ್ ಪಠ್ಯವನ್ನು ವಿನ್ಯಾಸಗೊಳಿಸುವುದಾಗಿದೆ. ಇದು ಕೇವಲ ಪ್ರಶ್ನೆಗಳನ್ನು ಕೇಳುವುದಲ್ಲ - ಆದರೆ ಮಾದರಿ ನೀವು ಏನು ಬಯಸುತ್ತೀರಿ ಮತ್ತು ಅದನ್ನು ಹೇಗೆ ಒದಗಿಸಲು ಎಂಬುದನ್ನು ನಿಖರವಾಗಿ ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು ವಿನಂತಿಗಳನ್ನು ರಚಿಸುವುದಾಗಿದೆ.

ಇದನ್ನು ಸಹೋದ್ಯೋಗಿಗೆ ಸೂಚನೆ ನೀಡುವುದಂತೆ ಅಲಂಕಾರಿತವಾಗಿ ನೋಡುವಿರಿ. "ಬಗ್ ಸರಿಪಡಿಸಿ" ಅಂದರೆ ಸಂಶಯಾಸ್ಪದವಾದುದೇ. "UserService.java ನ ಲೈನ್ 45 ರಲ್ಲಿ ನಲ್ ಪಾಯಿಂಟರ್ ಎಕ್ಸೆಪ್ಷನ್ ಸರಿಪಡಿಸಲು ನಲ್ ಚೆಕ್ಅನ್ನು ಸೇರಿಸಿ" ಎಂದರೆ ಸ್ಪಷ್ಟವಾಗಿದೆ. ಭಾಷಾ ಮಾದರಿಗಳು ಕೂಡ ಹೀಗೆಯೇ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತವೆ - ನಿಖರತೆ ಮತ್ತು ರಚನೆ ಮುಖ್ಯ.

<img src="../../../translated_images/kn/how-langchain4j-fits.dfff4b0aa5f7812d.webp" alt="LangChain4j ಹೇಗೆ ಹೊಂದಿಕೊಳ್ಳುತ್ತದೆ" width="800"/>

LangChain4j ಮೂಲಸೌಕರ್ಯವನ್ನು ಒದಗಿಸುತ್ತದೆ — ಮಾದರಿ ಸಂಪರ್ಕಗಳು, ಮೆಮೊರಿ ಮತ್ತು ಸಂದೇಶ ಪ್ರಕಾರಗಳು — ಮತ್ತು ಪ್ರಾಂಪ್ಟ್ ಮಾದರಿಗಳು ಆ ಮೂಲಸೌಕರ್ಯದಿಂದ ನೀವು ಕಳುಹಿಸುವ ಜಾಗರೂಕವಾಗಿ ರಚಿಸಲಾದ ಪಠ್ಯವೇ ಆಗಿವೆ. ಮುಖ್ಯ ಕಟ್ಟಲುಕಾ ನಿಟ್ಟಿನಲ್ಲಿ `SystemMessage` (ಯಾವುದು AI ನ ವರ್ತನೆ ಮತ್ತು ಪಾತ್ರವನ್ನು ಸ್ಥಾಪಿಸುತ್ತದೆ) ಮತ್ತು `UserMessage` (ನಿಮ್ಮ ನಿಜವಾದ ವಿನಂತಿಗಳನ್ನು ಸಾಗಿಸುತ್ತದೆ) ಸಿರುವರು.

## ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಮೂಲತತ್ವಗಳು

<img src="../../../translated_images/kn/five-patterns-overview.160f35045ffd2a94.webp" alt="ಐದು ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಮಾದರಿಗಳ ಆವಲೋಕನ" width="800"/>

ಈ ವಿಷಯದ ಪ್ರಗತಿಪರ ಮಾದರಿಗಳಲ್ಲಿ ಮುಳುಗುವ ಮೊದಲು, ಐದು ಆಧಾರಭೂತ ಪ್ರಾಂಪ್ಟ್ ತಂತ್ರಗಳನ್ನು ಪರಿಶೀಲಿಸೋಣ. ಇವು ಪ್ರತಿ ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರ್ ತಿಳಿದುಕೊಳ್ಳಬೇಕಾದ ಕಟ್ಟಡುಗಳು. ನೀವು ಈಗಾಗಲೇ [ಕ್ವಿಕ್ ಸ್ಟಾರ್ಟ್ ಮೋಡ್ಯೂಲ್](../00-quick-start/README.md#2-prompt-patterns) ಮೂಲಕ ಕೆಲಸಮಾಡಿದ್ದೀರಾ, ಇಲ್ಲಿ ಅದಕ್ಕೆ ಹಿಂದಿನ ಸಿದ್ಧಾಂತ ಹೆಸರಾಗಿವೆ.

### ಝೀರೋ-ಶಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್

ಸರಳ ವಿಧಾನ: ಮಾದರಿಗೆ ಯಾವುದೇ ಉದಾಹರಣೆ ಇಲ್ಲದೆ ನೇರ ಸೂಚನೆ ನೀಡಿ. ಮಾದರಿ ಪೂರ್ವಪ್ರಶಿಕ್ಷಣದ ಆಧಾರದಲ್ಲಿ ಕಾರ್ಯವನ್ನು ತಿಳಿದು ಮತ್ತು ನಿರ್ವಹಿಸುತ್ತದೆ. ನಿರೀಕ್ಷಿತ ವರ್ತನೆ ಸ್ಪಷ್ಟವಾದ ಸರಳ ವಿನಂತಿಗಳಿಗಾಗಿ ಇದು ಉತ್ತಮ.

<img src="../../../translated_images/kn/zero-shot-prompting.7abc24228be84e6c.webp" alt="ಝೀರೋ-ಶಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್" width="800"/>

*ಉದಾಹರಣೆಗಳಿಲ್ಲದೆ ನೇರ ಸೂಚನೆ — ಮಾದರಿ ಸೂಚನೆಯಿಂದಲೇ ಕಾರ್ಯವನ್ನು ಊಹಿಸುತ್ತದೆ*

```java
String prompt = "Classify this sentiment: 'I absolutely loved the movie!'";
String response = model.chat(prompt);
// ಪ್ರತ್ಯುತ್ತರ: "ಧನಾತ್ಮಕ"
```

**ಬಳಸಬೇಕಾಗಿದ್ದಾಗ:** ಸರಳ ವರ್ಗೀಕರಣಗಳು, ನೇರ ಪ್ರಶ್ನೆಗಳು, ಅನುವಾದಗಳು ಅಥವಾ ಹೆಚ್ಚಿನ ಮಾರ್ಗದರ್ಶನವಿಲ್ಲದೆ ಮಾದರಿ ನಡೆಸಬಹುದಾದ ಯಾವುದೇ ಕೆಲಸ.

### ಫ್ಯೂ-ಶಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್

ನೀವು ಬಯಸುವ ಮಾದರಿಯನ್ನು ತೋರಿಸುವ ಉದಾಹರಣೆಗಳನ್ನು ಒದಗಿಸಿ. ಮಾದರಿ ನಿಮ್ಮ ಉದಾಹರಣೆಗಳಿಂದ ಇನ್‌ಪುಟ್-ಔಟ್‌ಪುಟ್ ಸ್ವರೂಪವನ್ನು ಕಲಿತು ಹೊಸ ಇನ್‌ಪುಟ್‌ಗಳಿಗೆ ಅನ್ವಯಿಸುತ್ತದೆ. ಇದು ವಿನ್ಯಾಸ ಅಥವಾ ವರ್ತನೆ ಸ್ಪಷ್ಟವಲ್ಲದ ಕೆಲಸಗಳಿಗೆ ಬಹುಮಾನವನ್ನು ಸುಧಾರಿಸುತ್ತದೆ.

<img src="../../../translated_images/kn/few-shot-prompting.9d9eace1da88989a.webp" alt="ಫ್ಯೂ-ಶಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್" width="800"/>

*ಉದಾಹರಣೆಗಳಿಂದ ಕಲಿಕೆ — ಮಾದರಿ ಮಾದರಿಯನ್ನು ಗುರುತಿಸಿ ಮತ್ತು ಹೊಸ ಇನ್‌ಪುಟ್‌ಗಳಿಗೆ ಅನ್ವಯಿಸುತ್ತದೆ*

```java
String prompt = """
    Classify the sentiment as positive, negative, or neutral.
    
    Examples:
    Text: "This product exceeded my expectations!" → Positive
    Text: "It's okay, nothing special." → Neutral
    Text: "Waste of money, very disappointed." → Negative
    
    Now classify this:
    Text: "Best purchase I've made all year!"
    """;
String response = model.chat(prompt);
```

**ಬಳಸಬೇಕಾಗಿದ್ದಾಗ:** ಕಸ್ಟಮ್ ವರ್ಗೀಕರಣಗಳು, ಸ್ಥಿರ ಸ್ವರೂಪ, ಕ್ಷೇತ್ರ ವಿಶೇಷ ಕೆಲಸಗಳು ಅಥವಾ ಝೀರೋ-ಶಾಟ್ ಫಲಿತಾಂಶಗಳು ಅಸ್ಥಿರವಾಗಿರುವಾಗ.

### ಚೇನ್ ಆಫ್ ಥಾಟ್

ಮಾದರಿಯನ್ನು ತನ್ನ ಯುಕ್ತಿವಾದವನ್ನು ಹಂತ ಹಂತವಾಗಿ ತೋರಿಸಲು ಕೇಳಿ. ತಕ್ಷಣ ಉತ್ತರ ನೀಡದ ಬದಲು, ಮಾದರಿ ಸಮಸ್ಯೆಯನ್ನು ವಿಭಜಿಸಿ ಪ್ರತಿ ಅಂಶವನ್ನು ಸ್ಪಷ್ಟವಾಗಿಯಾಗಿ ಕೆಲಸಮಾಡುತ್ತದೆ. ಗಣಿತ, ತರ್ಕ, ಮತ್ತು ಬಹು ಹಂತದ ಯುಕ್ತಿವಾದ ಕೆಲಸಗಳಲ್ಲಿ ಖಚಿತತೆ ಹೆಚ್ಚುತ್ತದೆ.

<img src="../../../translated_images/kn/chain-of-thought.5cff6630e2657e2a.webp" alt="ಚೇನ್ ಆಫ್ ಥಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್" width="800"/>

*ಹಂತದರ್ಜೆಯ ಯುಕ್ತಿವಾದ — ಸಂಕೀರ್ಣ ಸಮಸ್ಯೆಗಳನ್ನು ಸ್ಪಷ್ಟ ಲಾಜಿಕಲ್ ಹಂತಗಳಾಗಿ ವಿಭಜಿಸುವುದು*

```java
String prompt = """
    Problem: A store has 15 apples. They sell 8 apples and then 
    receive a shipment of 12 more apples. How many apples do they have now?
    
    Let's solve this step-by-step:
    """;
String response = model.chat(prompt);
// ಮಾದರಿ ತೋರಿಸುತ್ತದೆ: 15 - 8 = 7, ನಂತರ 7 + 12 = 19 ಸೇಬುಗಳು
```

**ಬಳಸಬೇಕಾಗಿದ್ದಾಗ:** ಗಣಿತ ಸಮಸ್ಯೆಗಳು, ತರ್ಕ ಮಾಲಿಕೆಗಳು, ಡಿಬಗಿಂಗ್ ಅಥವಾ ಯುಕ್ತಿವಾದ ಪ್ರಕ್ರಿಯೆಯನ್ನು ತೋರಿಸುವುದು ಖಚಿತತೆ ಮತ್ತು ನಂಬಿಕೆ ಹೆಚ್ಚಿಸುವಾಗ.

### ಭೂಮಿಕೆ ಆಧಾರಿತ ಪ್ರಾಂಪ್ಟಿಂಗ್

ನಿಮ್ಮ ಪ್ರಶ್ನೆಗಾಗಿ AI ಗೆ ಒಂದು ಪಾತ್ರ ಅಥವಾ ವ್ಯಕ್ತಿತ್ವವನ್ನು ಸವಾಲಾಗಿ ಮಾಡಿ. ಇದರಿಂದ ಪ್ರತಿಕ್ರಿಯೆಯ ಶೈಲಿ, ಆಳತೆ ಮತ್ತು ધ્યાનವು ರೂಪುಗೊಳ್ಳುತ್ತದೆ. "ಸಾಫ್ಟ್ವೇರ್ ವಾಸ್ತುಶಿಲ್ಪಿ" "ಜೂನಿಯರ್ ಡೆವಲಪರ್" ಅಥವಾ "ಸುರಕ್ಷತಾ ఆటಿಟರ್"ಗಿಂತ ವಿಭಿನ್ನ ಸಲಹೆಗಳನ್ನು ನೀಡುತ್ತದೆ.

<img src="../../../translated_images/kn/role-based-prompting.a806e1a73de6e3a4.webp" alt="ಭೂಮಿಕೆ ಆಧಾರಿತ ಪ್ರಾಂಪ್ಟಿಂಗ್" width="800"/>

*ಸಂದರ್ಭ ಮತ್ತು ಪಾತ್ರವನ್ನು ಹೊಂದಿಸುವುದು — ಅದೇ ಪ್ರಶ್ನೆಗೆ ನಿವಾಸಿತ ಪಾತ್ರದ ಆಧಾರದ ಮೇಲೆ ವಿಭಿನ್ನ ಉತ್ತರಗಳು ಸಿಗುತ್ತವೆ*

```java
String prompt = """
    You are an experienced software architect reviewing code.
    Provide a brief code review for this function:
    
    def calculate_total(items):
        total = 0
        for item in items:
            total = total + item['price']
        return total
    """;
String response = model.chat(prompt);
```

**ಬಳಸಬೇಕಾಗಿದ್ದಾಗ:** ಕೋಡ್ ಪರಿಶೀಲನೆಗಳು, ತರಬೇತಿ, ಕ್ಷೇತ್ರ-ವಿಶೇಷ ವಿಶ್ಲೇಷಣೆಗಳು ಅಥವಾ ನಿಶ್ಚಿತ ಪರಿಣತಿಯ ಮಟ್ಟ ಅಥವಾ ದೃಷ್ಟಿಕೋನಕ್ಕೆ ಹೊಂದಿಕೆ ಬೇಕಾಗುವಾಗ.

### ಪ್ರಾಂಪ್ಟ್ ಟೆಂಪ್ಲೇಟ್ಗಳು

ಬದಲಾಗುವ ಹಸ್ತಾಂಗಗಳೊಂದಿಗೆ ಮರುಬಳಕೆಮಾಡಬಹುದಾದ ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ರಚಿಸಿ. ಪ್ರತಿ ಬಾರಿ ಹೊಸ ಪ್ರಾಂಪ್ಟ್ ಬರೆದ ಬದಲು, ಒಂದೇ ಟೆಂಪ್ಲೇಟ್ನಲ್ಲಿ ವಿವಿಧ ಮೌಲ್ಯಗಳನ್ನಿರಿಸಿ. LangChain4j ನ `PromptTemplate` ಕ್ಲಾಸ್ `{{variable}}` ರಚನೆ ಮೂಲಕ ಇದು ಸುಲಭವಾಗಿಸುತ್ತದೆ.

<img src="../../../translated_images/kn/prompt-templates.14bfc37d45f1a933.webp" alt="ಪ್ರಾಂಪ್ಟ್ ಟೆಂಪ್ಲೇಟ್ಗಳು" width="800"/>

*ಬದಲಾಗುವ ಹಸ್ತಾಂಗಗಳ ಜೊತೆ ಮರುಬಳಕೆ ಮಾಡಬಹುದಾದ ಪ್ರಾಂಪ್ಟ್‌ಗಳು — ಒಂದು ಟೆಂಪ್ಲೇಟ್, ಅನೇಕ ಬಳಕೆಗಳು*

```java
PromptTemplate template = PromptTemplate.from(
    "What's the best time to visit {{destination}} for {{activity}}?"
);

Prompt prompt = template.apply(Map.of(
    "destination", "Paris",
    "activity", "sightseeing"
));

String response = model.chat(prompt.text());
```

**ಬಳಸಬೇಕಾಗಿದ್ದಾಗ:** ವಿಭಿನ್ನ ಇನ್‌ಪುಟ್‌ಗಳೊಂದಿಗೆ ಪುನರಾವರ್ತಿತ ಪ್ರಶ್ನೆಗಳು, ಬ್ಯಾಚ್ ಪ್ರಕ್ರಿಯೆ, ಮರುಬಳಕೆಮಾಡಬಹುದಾದ AI ವರ್ಕ್‌ಫ್ಲೋ ನಿರ್ಮಾಣ, ಅಥವಾ ಪ್ರಾಂಪ್ಟ್ ರಚನೆ ಸ್ಥಿರವಾಗಿದ್ದು ದತ್ತಾಂಶ ಬದಲಾಗುವ ಯಾವುದೇ ಸಂದರ್ಭ.

---

ಈ ಐದು ಮೂಲತತ್ವಗಳು ಬಹುತೇಕ ಪ್ರಾಂಪ್ಟ್ ಕಾರ್ಯಗಳಿಗೆ ನಿಮ್ಮಿಗೆ ಧೃಡ ಉಪಕರಣ ಸಂಗ್ರಹ ನೀಡುತ್ತವೆ. ಮಿಕ್ಕಂತು ಈ ವಿಷಯದಲ್ಲಿ GPT-5.2 ಯ ಯುಕ್ತಿವಾದ ನಿಯಂತ್ರಣ, ಸ್ವ-ಮೌಲ್ಯಮಾಪನ ಮತ್ತು ಸಂರಚಿತ output ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಬಳಸಿಕೊಂಡ **ಎಂಟು ಪ್ರಗತಿಪರ ಮಾದರಿಗಳನ್ನು** ಒಳಗೊಂಡಿದೆ.

## ಅಧಿಕ ಪ್ರಗತಿಪರ ಮಾದರಿಗಳು

ಮೂಲತತ್ವಗಳನ್ನು ಮುಕ್ತಾಯಮಾಡಿ, ಈ ವಿಷಯವನ್ನು ವಿಶಿಷ್ಟವಾಗಿಸುವ ಎಂಟು ಪ್ರಗತಿಪರ ಮಾದರಿಗಳಿಗೆ ಹೋಗೋಣ. ಎಲ್ಲಾ ಸಮಸ್ಯೆಗಳಿಗೂ ಒಂದೇ ವಿಧಾನ ಬೇಕಾಗುವದು ಅಲ್ಲ. ಕೆಲ ಪ್ರಶ್ನೆಗಳಿಗೆ ತ್ವರಿತ ಉತ್ತರಗಳು ಬೇಕಾಗುತ್ತವೆ, ಕೆಲವರಿಗೆ ಆಳವಾದ ಆಲೋಚನೆ. ಕೆಲವರಿಗೆ ದೃಶ್ಯಮಾನ ಯುಕ್ತಿವಾದ, ಮತ್ತವರಿಗೆ ಪರಿಣಾಮ. ಕೆಳಗಿನ ಪ್ರತಿಯೊಂದು ಮಾದರಿಯು ವಿಭಿನ್ನ ಪರಿಸ್ಥಿತಿಗೆ ಅನುಗುಣವಾಗಿದ್ದು — ಮತ್ತು GPT-5.2 ಯ ಯುಕ್ತಿವಾದ ನಿಯಂತ್ರಣ ವ್ಯತ್ಯಾಸಗಳನ್ನು ಇನ್ನಷ್ಟು ಸ್ಪಷ್ಟಗೊಳಿಸುತ್ತದೆ.

<img src="../../../translated_images/kn/eight-patterns.fa1ebfdf16f71e9a.webp" alt="ಎಂಟು ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಮಾದರಿಗಳು" width="800"/>

*ಎಂಟು ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಮಾದರಿಗಳ ಆವಲೋಕನ ಮತ್ತು ಅವುಗಳ ಬಳಕೆ ಘಟನೆಗಳು*

<img src="../../../translated_images/kn/reasoning-control.5cf85f0fc1d0c1f3.webp" alt="GPT-5.2 ಯೊಂದಿಗೆ ಯುಕ್ತಿವಾದ ನಿಯಂತ್ರಣ" width="800"/>

*GPT-5.2 ಯ ಯುಕ್ತಿವಾದ ನಿಯಂತ್ರಣದೊಂದಿಗೆ ಮಾದರಿಯು ಎಷ್ಟು ಆಲೋಚನೆ ಮಾಡಬೇಕೆಂದು ಹೇಳಬಹುದು — ತ್ವರಿತ ನೇರ ಉತ್ತರಗಳಿಂದ ಆಳವಾದ ಅನ್ವೇಷಣೆವರೆಗೂ*

<img src="../../../translated_images/kn/reasoning-effort.db4a3ba5b8e392c1.webp" alt="ಯುಕ್ತಿವಾದ ಪ್ರಯತ್ನದ ಹೋಲಿಕೆ" width="800"/>

*ಕಡಿಮೆ ಉತ್ಸಾಹ (ತ್ವರಿತ, ನೇರ) ಮತ್ತು ಹೆಚ್ಚು ಉತ್ಸಾಹ ( ಆಳವಾದ, ಅನ್ವೇಷಣಾತ್ಮಕ ) ಯುಕ್ತಿವಾದ ವಿಧಾನಗಳು*

**ಕಡಿಮೆ ಉತ್ಸಾಹ (ತ್ವರಿತ ಮತ್ತು ಕೇಂದ್ರಿತ)** - ಸರಳ ಪ್ರಶ್ನೆಗಳಿಗೆ ಇಷ್ಟಪಡುವ ಉತ್ತರಗಳು. ಮಾದರಿ ಕನಿಷ್ಟ ಯುಕ್ತಿವಾದ ಮಾಡುತ್ತದೆ - ಗರಿಷ್ಠ 2 ಹಂತ. ಇದನ್ನು ಗಣನೆ, ಹುಡುಕಾಟ ಅಥವಾ ಸರಳ ಪ್ರಶ್ನೆಗಳಿಗೆ ಬಳಸಿ.

```java
String prompt = """
    <reasoning_effort>low</reasoning_effort>
    <instruction>maximum 2 reasoning steps</instruction>
    
    What is 15% of 200?
    """;

String response = chatModel.chat(prompt);
```

> 💡 **GitHub Copilot ಜೊತೆ ಅನ್ವೇಷಿಸಿ:** [`Gpt5PromptService.java`](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java) ತೆರೆದು ಕೇಳಿ:
> - "ಕಡಿಮೆ ಉತ್ಸಾಹ ಮತ್ತು ಹೆಚ್ಚು ಉತ್ಸಾಹ ಪ್ರಾಂಪ್ಟ್ ಮಾದರಿಗಳ ವ್ಯತ್ಯಾಸವೇನು?"
> - "ಪ್ರಾಂಪ್ಟ್‌ಗಳಲ್ಲಿ XML ಟ್ಯಾಗ್‌ಗಳು AI ಪ್ರತಿಕ್ರಿಯೆಯನ್ನು ರಚಿಸಲು ಹೇಗೆ ಸಹಾಯಕವಾಗಿವೆ?"
> - "ಸ್ವ-ಪರಿಶೀಲನೆ ಮಾದರಿಗಳನ್ನು ನೇರ ಸೂಚನೆಗಳಿಗಿಂತ ಯಾವಾಗ ಬಳಸಬೇಕು?"

**ಹೆಚ್ಚು ಉತ್ಸಾಹ (ಆಳವಾದ ಮತ್ತು ಸಂಪೂರ್ಣ)** - ಜಟಿಲ ಸಮಸ್ಯೆಗಳಿಗೆ ಸಮಗ್ರ ವಿಶ್ಲೇಷಣೆಯು ಬೇಕಾದಾಗ. ಮಾದರಿ ಆಳವಾಗಿ ಅನ್ವೇಷಿಸಿ ವಿವರವಾದ ಯುಕ್ತಿವಾದವನ್ನು ತೋರಿಸುತ್ತದೆ. ಇದನ್ನು ಸಿಸ್ಟಮ್ ವಿನ್ಯಾಸ, ವಾಸ್ತುಶಿಲ್ಪದ ನಿರ್ಧಾರಗಳು ಅಥವಾ ಜಟಿಲ ಸಂಶೋಧನೆಗಳಿಗೆ ಬಳಸಿ.

```java
String prompt = """
    <reasoning_effort>high</reasoning_effort>
    <instruction>explore thoroughly, show detailed reasoning</instruction>
    
    Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```

**ಕಾರ್ಯ ನಿರ್ವಹಣೆ (ಹಂತ ಹಂತದ ಪ್ರಗತಿ)** - ಬಹು ಹಂತದ ಕೆಲಸಗಳಿಗೆ. ಮಾದರಿ ಮೊದಲು ಯೋಜನೆಯನ್ನು ನೀಡುತ್ತದೆ, ಪ್ರತಿ ಹಂತವನ್ನು ತಿಳಿಸುವ ಮೂಲಕ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ ಮತ್ತು ನಂತರ ಸಾರಾಂಶ ನೀಡುತ್ತದೆ. ಇದನ್ನು ವಲಸೆ, ಅನ್ವಯಿಕೆಗಳು ಅಥವಾ ಯಾವುದೇ ಬಹು ಹಂತದ ಪ್ರಕ್ರಿಯಗೆ ಬಳಸಿ.

```java
String prompt = """
    <task>Create a REST endpoint for user registration</task>
    <preamble>Provide an upfront plan</preamble>
    <narration>Narrate each step as you work</narration>
    <summary>Summarize what was accomplished</summary>
    """;

String response = chatModel.chat(prompt);
```

ಚೇನ್-ಆಫ್-ಥಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್ ಮಾದರಿಯ ಯುಕ್ತಿವಾದ ಪ್ರಕ್ರಿಯೆಯನ್ನು ಸ್ಪಷ್ಟವಾಗಿ ತೋರಿಸುವಂತೆ ಕೇಳುತ್ತದೆ, ಜಟಿಲ ಕೆಲಸಗಳ ಖಚಿತತೆ ಹೆಚ್ಚಿಸುತ್ತದೆ. ಹಂತ ಹಂತದ ವ್ಯಾಖ್ಯಾನವು ಮಾನವರಿಗೂ AI ಗೆಲುವಿನ ಲಾಜಿಕ್ ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.

> **🤖 [GitHub Copilot](https://github.com/features/copilot) ಚಾಟ್‌ನಲ್ಲಿ ಪ್ರಯತ್ನಿಸಿ:** ಈ ಮಾದರಿಯನ್ನು ಕೇಳಿ:
> - "ದೀರ್ಘಕಾಲಿಕ ಕಾರ್ಯಾಚರಣೆಗಳಿಗೆ ಕಾರ್ಯ ನಿರ್ವಹಣಾ ಮಾದರಿಯನ್ನು ಹೇಗೆ ತಕ್ಕಮ ಮಾಡಬಹುದು?"
> - "ಉತ್ಪಾದನಾ ಅಪ್ಲಿಕೇಶನ್‌ಗಳಲ್ಲಿ ಉಪಕರಣ ಪೂರ್ವಕಥನಗಳನ್ನು ರೂಪಿಸುವ ಉತ್ತಮ ಅಭ್ಯಾಸವೇನು?"
> - "UI ನಲ್ಲಿ ಮಧ್ಯಂತರ ಪ್ರಗತಿ ನವೀಕೃತಿಗಳನ್ನು ಹಿಡಿದು ತೋರಿಸುವ ವಿಧಾನವೇನು?"

<img src="../../../translated_images/kn/task-execution-pattern.9da3967750ab5c1e.webp" alt="ಕಾರ್ಯ ನಿರ್ವಹಣೆ ಮಾದರಿ" width="800"/>

*ಯೋಜನೆ → ನಿರ್ವಹಣೆ → ಸಾರಾಂಶ ಕಾರ್ಯಪ್ರವಾಹ ಬಹು ಹಂತದ ಕೆಲಸಗಳಿಗೆ*

**ಸ್ವ-ಪರಿಶೀಲನೆಯ ಕೋಡ್** - ಉತ್ಪಾದನಾ ಗುಣಮಟ್ಟದ ಕೋಡ್ ರಚನೆಗಾಗಿ. ಮಾದರಿ ಕೋಡ್ ರಚಿಸಿ, ಗುಣಮಟ್ಟದ ಮಾನದಂಡಗಳಿಗೆ ಲೆಕ್ಕಿಸು, ಮತ್ತು ಕ್ರಮೇಣ ಸುಧಾರಣೆ ಮಾಡುತ್ತದೆ. ಹೊಸ ವೈಶಿಷ್ಟ್ಯಗಳು ಅಥವಾ ಸೇವೆಗಳನ್ನು ನಿರ್ಮಿಸುವಾಗ ಇದನ್ನು ಬಳಸಿ.

```java
String prompt = """
    <task>Create an email validation service</task>
    <quality_criteria>
    - Correct logic and error handling
    - Best practices (clean code, proper naming)
    - Performance optimization
    - Security considerations
    </quality_criteria>
    <instruction>Generate code, evaluate against criteria, improve iteratively</instruction>
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/kn/self-reflection-cycle.6f71101ca0bd28cc.webp" alt="ಸ್ವ-ಪರಿಶೀಲನೆಯ ಚಕ್ರ" width="800"/>

*ಕ್ರಮೇಣ ಸುಧಾರಣೆ ಲೂಪ್ - ರಚನೆ, ಮೌಲ್ಯಮಾಪನ, ಸಮಸ್ಯೆ ಗುರುತಿಸುವುದು, ಸುಧಾರಣೆ, ಪುನರಾವರ್ತನೆ*

**ಸಂರಚಿತ ವಿಶ್ಲೇಷಣೆ** - ಸ್ಥಿರ ಮೌಲ್ಯಮಾಪನಕ್ಕಾಗಿ. ಮಾದರಿ ನಿರ್ದಿಷ್ಟ ಚಟುವಟಿಕೆ (ಸರಿಯಾದತೆ, ಅಭ್ಯಾಸಗಳು, ಪ್ರದರ್ಶನ, ಭದ್ರತೆ)ಗಳಿಗೆ ಕೋಡ್ ಪರಿಶೀಲಿಸುತ್ತದೆ. ಇದನ್ನು ಕೋಡ್ ವಿಮರ್ಶೆಗಳು ಅಥವಾ ಗುಣಮಟ್ಟ ಮೌಲ್ಯಮಾಪನಕ್ಕೆ ಬಳಸಿ.

```java
String prompt = """
    <code>
    public List getUsers() {
        return database.query("SELECT * FROM users");
    }
    </code>
    
    <framework>
    Evaluate using these categories:
    1. Correctness - Logic and functionality
    2. Best Practices - Code quality
    3. Performance - Efficiency concerns
    4. Security - Vulnerabilities
    </framework>
    """;

String response = chatModel.chat(prompt);
```

> **🤖 [GitHub Copilot](https://github.com/features/copilot) ಚಾಟ್‌ನಲ್ಲಿ ಪ್ರಯತ್ನಿಸಿ:** ಸಂರಚಿತ ವಿಶ್ಲೇಷಣೆಯ ಬಗ್ಗೆ ಕೇಳಿ:
> - "ವಿಭಿನ್ನ ಕೋಡ್ ವಿಮರ್ಶೆಗಳಿಗೆ ವಿಶ್ಲೇಷಣಾ ರೂಪರೇಖೆಯನ್ನು ತಕ್ಕಮಾಡುವುದು ಹೇಗೆ?"
> - "ಸಂರಚಿತ output ಅನ್ನು ಪ್ರೋಗ್ರಾಮ್ಯಾಟಿಕ ಆಗಿ ಪರ್ಸ್ ಮಾಡುವುದು ಮತ್ತು ಉಪಯೋಗಿಸುವ ಉತ್ತಮ ಮಾರ್ಗವೇನು?"
> - "ಬೇರೆ ಬೇರೆ ವಿಮರ್ಶೆ ಅಧಿವೇಶನಗಳ ನಡುವೆ ಸ್ಥಿರ ಗಂಭೀರತೆ ಮಟ್ಟವನ್ನು ಹೇಗೆ ಖಚಿತಪಡಿಸಿಕೊಳ್ಳುವುದು?"

<img src="../../../translated_images/kn/structured-analysis-pattern.0af3b690b60cf2d6.webp" alt="ಸಂರಚಿತ ವಿಶ್ಲೇಷಣೆ ಮಾದರಿ" width="800"/>

*ಗಂಭೀರತೆ ಮಟ್ಟಗಳೊಂದಿಗೆ ಸ್ಥಿರ ಕೋಡ್ ವಿಮರ್ಶೆಗಳಿಗಾಗಿ ನಾಲ್ಕು ವರ್ಗಗಳ ರೂಪರೇಖೆ*

**ಬಹು-ತಃ ಚಾಟ್** - ಸಂಭಾಷಣೆಗೆ ಸಾಂದರ್ಭಿಕ ಅವಶ್ಯಕತೆ ಇರುವಾಗ. ಮಾದರಿ ಹಿಂದಿನ ಸಂದೇಶಗಳನ್ನು ನೆನಸಿಕೊಳ್ಳುತ್ತದೆ ಮತ್ತು ಅವುಗಳ ಮೇಲೆ ನಿರ್ಮಿಸುತ್ತವೆ. ಇದನ್ನು ಕ್ರಿಯಾತ್ಮಕ ಸಹಾಯ ಸೆಷನ್ಗಳು ಅಥವಾ ಸಂಕೀರ್ಣ Q&A ಗಾಗಿ ಬಳಸಿ.

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("What is Spring Boot?"));
AiMessage aiMessage1 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage1);

memory.add(UserMessage.from("Show me an example"));
AiMessage aiMessage2 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage2);
```

<img src="../../../translated_images/kn/context-memory.dff30ad9fa78832a.webp" alt="ಸಂದರ್ಭ ಮೆಮೊರಿ" width="800"/>

*ಬಹು ಹಂತಗಳಲ್ಲಿ ಸಂಭಾಷಣೆಯ ಸಂದರ್ಭದ ಸಂಕಲನವು ಟೋಕನ್ ಮಿತಿಯನ್ನು ತಲುಪುವವರೆಗೆ*

**ಅಡ್ಡಹಾದಿ ನಿರ್ಧಾರ** - ದೃಷ್ಟಿಗೋಚರ ಲಾಜಿಕಿನ ಅಗತ್ಯವಿರುವ ಸಮಸ್ಯೆಗಳಿಗೆ. ಮಾದರಿ ಪ್ರತಿ ಹಂತಕ್ಕೆ ಸ್ಪಷ್ಟ ಯುಕ್ತಿವಾದ ತೋರಿಸುತ್ತದೆ. ಇದನ್ನು ಗಣಿತ ಸಮಸ್ಯೆಗಳು, ತರ್ಕ ಪಜಲ್ಗಳು ಅಥವಾ ಯುಕ್ತಿವಾದ ಪ್ರಕ್ರಿಯೆಯನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಬೇಕಾದಾಗ ಬಳಸಿ.

```java
String prompt = """
    <instruction>Show your reasoning step-by-step</instruction>
    
    If a train travels 120 km in 2 hours, then stops for 30 minutes,
    then travels another 90 km in 1.5 hours, what is the average speed
    for the entire journey including the stop?
    """;

String response = chatModel.chat(prompt);
```

*ಸಮಸ್ಯೆಗಳನ್ನು ಸ್ಪಷ್ಟ ಲಾಜಿಕಲ್ ಹಂತಗಳಿಗೆ ವಿಭಜಿಸುವುದು*

**ನಿಯಂತ್ರಿತ output** - ನಿರ್ದಿಷ್ಟ ಸ್ವರೂಪ ಅವಶ್ಯಕತೆಗಳೊಂದಿಗೆ ಉತ್ತರಗಳಿಗೆ. ಮಾದರಿ ಸ್ವರೂಪ ಮತ್ತು ಕತ್ತರಾವಣೆಯ ನಿಯಮಗಳನ್ನು ಕಟ್ಟುನಿಟ್ಟಾಗಿ ಅನುಸರಿಸುತ್ತದೆ. ಅನುಶೀಲನೆಗಳಿಗಾಗಿ ಅಥವಾ ನಿಖರವಾದ output ರಚನೆ ಬೇಕಾದಾಗ ಬಳಸಿರಿ.

```java
String prompt = """

    - Exactly 100 words
    - Bullet point format
    - Technical terms only
    </constraints>
    
    Summarize the key concepts of machine learning.
    """;

```

*ನಿರ್ದಿಷ್ಟ ಪ್ರಕಾರ, ಉದ್ದ ಮತ್ತು ರಚನೆಯ ನಿಯಮಗಳನ್ನು ಜರುಗಿಸುವುದು*

ಡೆವ್ ಕಂಟೈನರ್ ಸ್ಪ್ರಿಂಗ್ ಬುಟ್ ಡ್ಯಾಶ್‌ಬೋರ್ಡ್ ವಿಸ್ತರಣೆವನ್ನು ಹೊಂದಿದೆ, ಇದು ಎಲ್ಲಾ ಸ್ಪ್ರಿಂಗ್ ಬುಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿರ್ವಹಿಸಲು ವೀಕ್ಷಣೀಯ ಇಂಟರ್ಫೇಸ್ ನೀಡುತ್ತದೆ. ನೀವು ಅದನ್ನು VS Code ಎಡಭಾಗದ ಕ್ರಿಯಾಶೀಲತೆ ಪಟ್ಟಿಯಲ್ಲಿ (Spring Boot ಐಕಾನ್ ನೋಡಿ) ಕಾಣಬಹುದು.

### ಕನಿಷ್ಠ ಮತ್ತು ಹೆಚ್ಚಿನ ಆಸಕ್ತಿ

"200 ರ 15% ಎಷ್ಟು?" ಎಂಬ ಸರಳ ಪ್ರಶ್ನೆಯನ್ನು ಕನಿಷ್ಠ ಆಸಕ್ತಿಯೊಂದಿಗೆ ಕೇಳಿ. ನಿಮಗೆ ಕ್ಷಣಾರ್ಧದಲ್ಲಿ ನೇರ ಉತ್ತರ ಸಿಗುತ್ತದೆ. ಈಗ "ಹೆಚ್ಚಿನ ಟ್ರಾಫಿಕ್ APIಗಾಗಿ ಕ್ಯಾಶಿಂಗ್ ತಂತ್ರವನ್ನು ವಿನ್ಯಾಸಗೊಳಿಸಿ" ಎಂದು ಹೆಚ್ಚಿನ ಆಸಕ್ತಿಯೊಂದಿಗೆ ಏನಾದರೂ ಸಂಕೀರ್ಣವನ್ನು ಕೇಳಿ. ಮಾದರಿ ನಿಧಾನವಾಗಿ ಚಿಂತಿಸಿ ವಿವರವಾದ ಕಾರಣೀಕರಣ ನೀಡುತ್ತದೆ. ಒಂದೇ ಮಾದರಿ, ಒಂದೇ ಪ್ರಶ್ನೆಯ ರಚನೆ - ಆದರೆ ಪ್ರಾಂಪ್ಟ್ ಎಷ್ಟು ಆಲೋಚನೆ ಮಾಡಬೇಕೆಂದು ಹೇಳುತ್ತದೆ.

*ಹೆಜ್ಜೆ ಹೆಜ್ಜೆ ಮೂಲಕ REST ಎಂಡ್ಪಾಯಿಂಟ್ ಸೃಷ್ಟಿಸುವುದು (3.9MB)*

### ಸ್ವ-ಪರಿಶೀಲನಾ ಕೋಡ್

"ಇಮೇಲ್ ಮಾನ್ಯತೆ ಸೇವೆ ಸೃಷ್ಟಿಸಿ" ಎಂದು ಪ್ರಯತ್ನಿಸಿ. ಕೇವಲ ಕೋಡ್ ರಚಿಸುವುದರ ಬದಲು, ಮಾದರಿ ರಚಿಸುತ್ತದೆ, ಗುಣಮಟ್ಟದ ಮಾನದಂಡಗಳ ವಿರುದ್ಧ ತಿಶೀರ್ಷಿಕೆ ಮಾಡುತ್ತದೆ, ದುರ್ಬಲತೆಗಳನ್ನು ಗುರುತಿಸಿ ಸುಧಾರಿಸುತ್ತದೆ. ನೀವು ಕೋಡ್ ಉತ್ಪಾದನಾ ಮಟ್ಟವನ್ನು ತಲುಪುವವರೆಗೆ ಪುನರಾವರ್ತನೆ ನಡೆಯುತ್ತದೆ.

<img src="../../../translated_images/kn/self-reflecting-code-demo.851ee05c988e743f.webp" alt="Self-Reflecting Code Demo" width="800"/>

*ಸಂಪೂರ್ಣ ಇಮೇಲ್ ಮಾನ್ಯತೆ ಸೇವೆ (5.2MB)*

### ರಚನೆಗೊಳಿಸಿದ ವಿಶ್ಲೇಷಣೆ

ಕೋಡ್ ವಿಮರ್ಶೆಗಳಿಗೆ ಸुसಂಸ್ಥಿತ ಗುಣಮಟ್ಟ ಮೌಲ್ಯಮಾಪನ ಅಗತ್ಯ. ಮಾದರಿ ನಿರ್ದಿಷ್ಟ ವರ್ಗಗಳ (ಸರಿಯಾದತೆ, ಅಭ್ಯಾಸಗಳು, ಕಾರ್ಯಕ್ಷಮತೆ, ಭದ್ರತೆ) ಜೊತೆಗೆ ತೀವ್ರತೆ ಮಟ್ಟಗಳಲ್ಲಿ ಕೋಡ್ ವಿಶ್ಲೇಷಣೆ ಮಾಡುತ್ತದೆ.

<img src="../../../translated_images/kn/structured-analysis-demo.9ef892194cd23bc8.webp" alt="Structured Analysis Demo" width="800"/>

*ಗುಣಮಟ್ಟ ಆಧಾರಿತ ಕೋಡ್ ವಿಮರ್ಶೆ*

### ಬಹು-ಸುತ್ತಿನ ಚಾಟ್

"ಸ್ಪ್ರಿಂಗ್ ಬೂಟ್ ಎಂದರೆ ಏನು?" ಎಂದು ಕೇಳಿ ನಂತರ ತಕ್ಷಣ "ಉದಾಹರಣೆಯನ್ನು ತೋರಿಸಿ" ಎಂದು ಹಿಜ್ಜೆ ಹಿಂಬಾಲಿಸಿ. ಮಾದರಿ ನಿಮ್ಮ ಮೊದಲ ಪ್ರಶ್ನೆಯನ್ನು ನೆನಪಿಡುತ್ತದೆ ಮತ್ತು ವಿಶೇಷವಾಗಿ ಸ್ಪ್ರಿಂಗ್ ಬೂಟ್ ಉದಾಹರಣೆಯನ್ನು ನೀಡುತ್ತದೆ. ನೆನಪಿಲ್ಲದಿದ್ದರೆ, ಎರಡನೇ ಪ್ರಶ್ನೆ ಅಸ್ಪಷ್ಟವಾಗುತ್ತದೆ.

<img src="../../../translated_images/kn/multi-turn-chat-demo.0d2d9b9a86a12b4b.webp" alt="Multi-Turn Chat Demo" width="800"/>

*ಪ್ರಶ್ನೆಗಳ ನಡುವಣ ಸಂಧಿಭಾವ ಉಳಿಸುವಿಕೆ*

### ಹೆಜ್ಜೆ ಹೆಜ್ಜೆ ಕಾರಣೀಕರಣ

ಗಣಿತದ ಸಮಸ್ಯೆಯನ್ನು ಆಯ್ಕೆ ಮಾಡಿ ಮತ್ತು ಅದನ್ನು ಹೆಜ್ಜೆ ಹೆಜ್ಜೆ ಕಾರಣೀಕರಣ ಮತ್ತು ಕನಿಷ್ಠ ಆಸಕ್ತಿಯೊಂದಿಗೆ ಪ್ರಯತ್ನಿಸಿ. ಕನಿಷ್ಠ ಆಸಕ್ತಿ ಕೇವಲ ಉತ್ತರವನ್ನು ನೀಡುತ್ತದೆ - ವೇಗವಾಗಿ ಆದರೆ ಅಸ್ಪಷ್ಟವಾಗಿ. ಹೆಜ್ಜೆ ಹೆಜ್ಜೆ ಪ್ರತಿಯೊಂದು ಲೆಕ್ಕಾಚಾರ ಮತ್ತು ನಿರ್ಧಾರವನ್ನು ತೋರಿಸುತ್ತದೆ.

<img src="../../../translated_images/kn/step-by-step-reasoning-demo.12139513356faecd.webp" alt="Step-by-Step Reasoning Demo" width="800"/>

*ಸ್ಪಷ್ಟ ಹಂತಗಳೊಂದಿಗೆ ಗಣಿತ ಸಮಸ್ಯೆ*

### ನಿಯಂತ್ರಿತ ಹೊರಹುಡುಕು

ನಿಮಗೆ ವಿಶಿಷ್ಟ ಫಾರ್ಮಾಟ್ ಅಥವಾ ಶಬ್ದ ಸಂಖ್ಯೆಗಳ ಅಗತ್ಯವಿದ್ದಾಗ, ಈ ಮಾದರಿ ಕಠಿಣ ನಿಯಮಾವಳಿಗಳನ್ನು ಜಾರಿಗೊಳಿಸುತ್ತದೆ. 100 ಪದಗಳ ನಿಖರ ಸಮಾರೋಪವನ್ನು ಬುಲೆಟ್ ಪಾಯಿಂಟ್ ಫಾರ್ಮಾಟ್‌ನಲ್ಲಿ ಸೃಷ್ಟಿಸಿ ಪ್ರಯತ್ನಿಸಿ.

<img src="../../../translated_images/kn/constrained-output-demo.567cc45b75da1633.webp" alt="Constrained Output Demo" width="800"/>

*ಫಾರ್ಮಾಟ್ ನಿಯಂತ್ರಣದೊಂದಿಗೆ ಯಂತ್ರ ಅಧ್ಯಯನ ಸಮಾರೋಪ*

## ನೀವು ನಿಜವಾಗಿಯೂ ಕಲಿಯುತ್ತಿರುವುದು

**ಕಾರಣೀಕರಣ ಪ್ರಯತ್ನ ಎಲ್ಲವನ್ನೂ ಬದಲಾಯಿಸುತ್ತದೆ**

GPT-5.2 ನಿಮಗೆ ನಿಮ್ಮ ಪ್ರಾಂಪ್ಟ್‌ಗಳಿಂದ ಗಣನಾತ್ಮಕ ಪ್ರಯತ್ನದ ನಿಯಂತ್ರಣವನ್ನು ನೀಡುತ್ತದೆ. ಕಡಿಮೆ ಪ್ರಯತ್ನವೂ ಅಲ್ಪ ತನಿಖೆಯೊಂದಿಗೆ ವೇಗವಾದ ಪ್ರತಿಕ್ರಿಯೆ ನೀಡುತ್ತದೆ. ಹೆಚ್ಚಿನ ಪ್ರಯತ್ನಗಿಂತ ಮಾದರಿ ಆಳವಾಗಿ ಯೋಚಿಸಲು ಸಮಯ ತೆಗೆದುಕೊಳ್ಳುತ್ತದೆ. ನೀವು ಕಾರ್ಯ ಸಾಂಕುಲ್ಯಕ್ಕನುಗುಣವಾಗಿ ಪ್ರಯತ್ನವನ್ನು ಹೊಂದಿಸುವುದನ್ನು ಕಲಿಯುತ್ತಿದ್ದೀರಾ - ಸರಳ ಪ್ರಶ್ನೆಗಳಿಗೆ ಸಮಯವನ್ನು ವ್ಯರ್ಥ ಮಾಡಬೇಡಿ, ಸಂಘಟಿತ ನಿರ್ಧಾರಗಳತ್ತ ಬೇಗ ಬಂದುಕೊಳ್ಳಬೇಡಿ.

**ರಚನೆ ವರ್ತನೆಯನ್ನು ಮಾರ್ಗದರ್ಶಿಸುತ್ತದೆ**

ಪ್ರಾಂಪ್ಟ್‌ಗಳಲ್ಲಿ XML ಟ್ಯಾಗ್‌ಗಳನ್ನು ಗಮನಿಸಿದ್ದೀರಾ? ಅವು ಅಲಂಕರಣವಲ್ಲ. ಮಾದರಿಗಳು ರಚನೆಯ ಸೂಚನೆಗಳನ್ನು ಮುಕ್ತರೂಪದ ಪಠ್ಯದಿಗಿಂತ ಹೆಚ್ಚು ನಂಬಿಕೆಯಿಂದ ಅನುಸರಿಸುತ್ತವೆ. ನೀವು ಬಹು ಹಂತದ ಪ್ರಕ್ರಿಯೆಗಳ ಅಥವಾ ಸಂಕೀರ್ಣ ನೈತಿಕತೆಯ ಅಗತ್ಯವಿದ್ದಾಗ, ರಚನೆ ಮಾದರಿಗೆ ಸ್ವಯಂಸ್ಥಿತಿಯನ್ನು ಕಾಪಾಡಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.

<img src="../../../translated_images/kn/prompt-structure.a77763d63f4e2f89.webp" alt="Prompt Structure" width="800"/>

*ಸ್ಪಷ್ಟ ವಿಭಾಗಗಳು ಮತ್ತು XML ಶೈಲಿಯ ಸಂಘಟನೆ ಹೊಂದಿದ ಚೆನ್ನಾಗಿ ರಚಿಸಲಾಗಿರುವ ಪ್ರಾಂಪ್ಟ್‌ನ ರಚನೆ*

**ಗುಣಮಟ್ಟ ಸ್ವ-ಮೌಲ್ಯಮಾಪನದ ಮೂಲಕ**

ಸ್ವ-ಪರಿಶೀಲನಾ ಮಾದರಿಗಳು ಗುಣಮಟ್ಟದ ಮಾನದಂಡಗಳನ್ನು ಸ್ಪಷ್ಟಪಡಿಸುವ ಮೂಲಕ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತವೆ. ಮಾದರಿ "ಸರಿಯಾಗಿ ಮಾಡುತ್ತದೆ" ಎಂದು ಆಸೆ ಹಿಡಿಯಿರಿ ಬದಲು, ನೀವು "ಸರಿಯೆಂದರೆ ಏನು" ಎಂದೂ ನಿಜವಾಗಿಯೂ ಹೇಳುತ್ತೀರಿ: ಸರಿಯಾದ ಲಾಜಿಕ್, ದೋಷ ನಿರ್ವಹಣೆ, ಕಾರ್ಯಕ್ಷಮತೆ, ಭದ್ರತೆ. ಆಮೇಲೆ ಮಾದರಿ ತನ್ನ ಉತ್ಪನ್ನವನ್ನು ಮೌಲ್ಯಮಾಪನ ಮಾಡಿ ಸುಧಾರಣೆ ಮಾಡಬಹುದು. ಇದು ಕೋಡ್ ತಯಾರಿಕೆಯನ್ನು ಲಾಟರಿ ಅಲ್ಲದೆ ಪ್ರಕ್ರಿಯೆ ಆಗಿಸುತ್ತದೆ.

**ಸಂದರ್ಭವು ಸೀಮಿತ**

ಬಹು-ಸುತ್ತಿನ ಸಂಭಾಷಣೆಗಳು ಪ್ರತಿ ವಿನಂತಿಯೊಂದಿಗೆ ಸಂದೇಶ ಇತಿಹಾಸವನ್ನು ಒಳಗೊಂಡಿವೆ. ಆದರೆ ಇದಕ್ಕೆ ಮಿತಿಯಿದೆ - ಪ್ರತಿಯೊಂದು ಮಾದರಿಗೂ ಗರಿಷ್ಠ ಟೋಕನ್ ಲೆಕ್ಕವಿದೆ. ಸಂಭಾಷಣೆಗಳು ವೃದ್ಧಿಯಾಗುವುದಾದರೆ, ಪ್ರಸ್ತುತ ಸಂಗತಿಯನ್ನು ಕಾಪಾಡಲು ತಂತ್ರಗಳನ್ನು ಅಗತ್ಯವಿರುತ್ತದೆ ಅದು ಮಿತಿಯನ್ನು ತುಂಡುಮಾಡದೆ. ಈ ಮಾಡ್ಯೂಲ್ ಅನ್ನು ನೀವು ನೆನಪಿನ ಕಾರ್ಯತಂತ್ರಗಳನ್ನು ಹೇಗೆ ಬಳಸುವಿರಿ ಎಂದು ತೋರಿಸುತ್ತದೆ; ಮುಂದೇನು ಸಂಕ್ಷಿಪ್ತಗೊಳಿಸಬೇಕು, ಯಾವಾಗ ಮರೆತುಹೋದರಬೇಕು ಮತ್ತು ಯಾವಾಗ തിരిగి ಪಡೆದೊಡಲಬೇಕೆಂದು ನೀವು ತಿಳಿಯುತ್ತೀರಿ.
