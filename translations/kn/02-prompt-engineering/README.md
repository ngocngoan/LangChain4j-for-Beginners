## ನೀವು ಕಲಿಯೋದು ಏನು

<img src="../../../translated_images/kn/what-youll-learn.c68269ac048503b2.webp" alt="ನೀವು ಕಲಿಯೋದು ಏನು" width="800"/>

ಹಿಂದಿನ Module ನಲ್ಲಿ ನೀವು ಮೆಮರಿಯು ಸಂವಾದಾತ್ಮಕ AI ಅನ್ನು ಹೇಗೆ ಸක්್ರೀಯಗೊಳಿಸುತ್ತದೆ ಎಂದು ನೋಡಿದ್ದೀರಾ ಮತ್ತು GitHub ಮಾದರಿಗಳನ್ನು ಮೂಲ ಸಂವಹನಕ್ಕಾಗಿ ಬಳಸಿದ್ದೀರಿ. ಪ್ರಸ್ತುತ, ನಾವು ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು—ಹೊಂದಿಸುವ ಪ್ರಶ್ನೆಗಳ ಸ್ವರೂಪವನ್ನು—ಜೊತೆ Azure OpenAI ಯ GPT-5.2 ಬಳಸಿ ಗಮನ ನೀಡುತ್ತೇವೆ. ನೀವು ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ರಚಿಸುವ ರೀತಿಯೇ ಪಡೆದ ಉತ್ತರಗಳ ಗುಣಮಟ್ಟವನ್ನು ಬಹುಮಟ್ಟಿಗೆ ಪ್ರಭಾವಿತಗೊಳಿಸುತ್ತದೆ. ನಾವು ಮೂಲಭೂತ ಪ್ರಾಂಪ್ಟ್ ತಂತ್ರಗಳನ್ನು ಸಮೀಕ್ಷಿಸಿದ ಮೇಲೆ, GPT-5.2 ಯ ಪೂರ್ಣ ಸಾಮರ್ಥ್ಯವನ್ನು ಬಳಸಿಕೊಂಡು ಆರು ಅತ್ಯಾಧುನಿಕ ಮಾದರಿಗಳತ್ತ ಸಾಗುತ್ತೇವೆ.

ನಾವು GPT-5.2 ಯ ಬಳಕೆಯನ್ನು ಆಯ್ಕೆ ಮಾಡುತ್ತೇವೆ ಏಕೆಂದರೆ ಇದು ತರ್ಕ ನಿಯಂತ್ರಣವನ್ನು ಪರಿಚಯಿಸುತ್ತದೆ - ನೀವು ಮಾದರಿಗೆ ಉತ್ತರಿಸಲು ಮೊದಲು ಎಷ್ಟು ಚಿಂತನೆ ಮಾಡಬೇಕೆಂದು ತಿಳಿಸಬಹುದು. ಇದರಿಂದ ವಿಭಿನ್ನ ಪ್ರಾಂಪ್ಟ್ ತಂತ್ರಗಳ ಬಹಳ ಸ್ಪಷ್ಟತೆ ಮೂಡುತ್ತದೆ ಮತ್ತು ಯಾವಾಗ ಯಾವ ವಿಧಾನ ಬಳಸಬೇಕೆಂಬುದನ್ನು ಅರಿತುಕೊಳ್ಳಲು ಸಹಾಯವಾಗುತ್ತದೆ. GitHub ಮಾದರಿಗಳಿಗಿಂತ ಕಡಿಮೆ ದರಮಿತಿಗಳೊಂದಿಗೆ Azure ನಿಂದ ಲಾಭವೂ ಇದೆ.

## ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಅರ್ಥ ಮಾಡಿಕೊಳ್ಳುವುದು

<img src="../../../translated_images/kn/what-is-prompt-engineering.5c392a228a1f5823.webp" alt="ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಎಂದರೇನು?" width="800"/>

ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಎಂದರೆ ನೀವು ಬೇಕಾದ ಫಲಿತಾಂಶಗಳನ್ನು ನಿರಂತರವಾಗಿ ಪಡೆಯಲು ಇನ್ಪುಟ್ ಪಠ್ಯವನ್ನು ವಿನ್ಯಾಸಗೊಳಿಸುವ ಕೆಲಸ. ಪ್ರಶ್ನೆಗಳು ಕೇಳುವುದಕ್ಕಷ್ಟೇ ಅಲ್ಲ, ಮಾದರಿ ನಿಮ್ಮ ಬಯಕೆಯನ್ನು ಸರಿಯಾಗಿ ಅರ್ಥಮಾಡಿ ಹೇಗೆ ಸಲ್ಲಿಸಬೇಕೆಂದು ವಿನಂತಿಗಳನ್ನು ರಚಿಸುವುದು ಇದು.

ನೀವು ಸಹೋದ್ಯೋಗಿಗೆ ಸೂಚನೆ ನೀಡುವುದು ಎಂದು ಭಾವಿಸಿ. "ತಪ್ಪು ಸರಿಪಡಿಸಿ" ಎಂದರೆ ಅಸ್ಪಷ್ಟವಾಗಿದೆ. "UserService.java ಡೈನ 45 ರಲ್ಲಿ ನಲ್ ಪಾಯಿಂಟರ್ ತಪ್ಪು ತಿದ್ದಿ, ನಲ್ ಚೆಕ್ ಸೇರಿಸಿ" ಎಂದರೆ ಈಗ ಸ್ಪಷ್ಟವಾಗಿದೆ. ಭಾಷಾ ಮಾದರಿಗಳು ಕೂಡ ಇದೇ ರೀತಿಯಲ್ಲಿ ಕೆಲಸ ಮಾಡುತ್ತವೆ - ವಿಶೇಷತೆ ಮತ್ತು ರಚನೆ ಮುಖ್ಯ.

<img src="../../../translated_images/kn/how-langchain4j-fits.dfff4b0aa5f7812d.webp" alt="LangChain4j ಹೇಗೆ ಹೊಂದಿಕೆಯಾಗುತ್ತದೆ" width="800"/>

LangChain4j ಮೂಲಸೌಕರ್ಯವನ್ನು ಒದಗಿಸುತ್ತದೆ — ಮಾದರಿ ಸಂಪರ್ಕಗಳು, ಮೆಮರಿ, ಮತ್ತು ಸಂದೇಶ ಪ್ರಕಾರಗಳು — ಆದರೆ ಪ್ರಾಂಪ್ಟ್ ಮಾದರಿಗಳು ಆ ಮೂಲಸೌಕರ್ಯದ ಮೂಲಕ ನೀವು ಕಳುಹಿಸುವ ಜಾಗರೂಕವಾಗಿ ರಚಿಸಲಾದ ಪಠ್ಯ ಮಾತ್ರ. ಮುಖ್ಯ ನಿರ್ಮಾಣ ಮೂಲಗಳು `SystemMessage` (AI ನ ವರ್ತನೆ ಮತ್ತು ಪಾತ್ರವನ್ನು ನಿಶ್ಚಯಿಸುತ್ತದೆ) ಮತ್ತು `UserMessage` (ನಿಮ್ಮ ನಿಜವಾದ ವಿನಂತಿಯನ್ನು ಹೊತ್ತುಕೊಳ್ಳುತ್ತದೆ).

## ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಮೂಲಭೂತಾಂಶಗಳು

<img src="../../../translated_images/kn/five-patterns-overview.160f35045ffd2a94.webp" alt="ಐదు ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಮಾದರಿಗಳ ಅವಲೋಕನ" width="800"/>

ಈ Module ನ ಅತ್ಯಾಧುನಿಕ ಮಾದರಿಗಳಲ್ಲಿ ಮುದುಕೊಳ್ಳುವ ಮುನ್ನ, ಐದು ಮೂಲಭೂತ ಪ್ರಾಂಪ್ಟ್ ತಂತ್ರಗಳನ್ನು ವಿಮರ್ಶಿಸೋಣ. ಪ್ರತೀ ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರ್ ತಿಳಿದುಕೊಳ್ಳಬೇಕಾದ these ಬಂಡಲುಗಳಾಗಿವೆ. ನೀವು ಈಗಾಗಲೇ [Quick Start module](../00-quick-start/README.md#2-prompt-patterns) ಮುಖಾಂತರ ಇವು ನೋಡಿದ್ದೀರಾ — ಇಲ್ಲಿ ಅವುಗಳ ಹಿನ್ನೆಲೆ ಕಲ್ಪನೆ ಇದೆ.

### ಶೂನ್ಯ-ಶಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್

ಸರಳ ವಿಧಾನ: ಮಾದರಿಗೆ ಯಾವುದೇ ಉದಾಹರಣೆಗಳಿಲ್ಲದೆ ನೇರ ಸೂಚನೆ ನೀಡಿ. ಮಾದರಿ ತನ್ನ ತರಬೇತಿಯ ಮೇಲೆ ಸಂಪೂರ್ಣವಾಗಿ ಅವಲಂಬಿಸಿ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ. ನಿರೀಕ್ಷಿತ ವರ್ತನೆ ಸ್ಪಷ್ಟವಾಗಿರುವ ಸೂಕ್ಷ್ತ ವಿನಂತಿಗಳಿಗೆ ಇದೊಂದು ಉತ್ತಮ ವಿಧಾನ.

<img src="../../../translated_images/kn/zero-shot-prompting.7abc24228be84e6c.webp" alt="ಶೂನ್ಯ-ಶಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್" width="800"/>

*ಉದಾಹರಣೆಗಳಿಲ್ಲದೆ ನೇರ ಸೂಚನೆ — ಮಾದರಿ ಸೂಚನೆಯಿಂದಲೇ ಕಾರ್ಯವನ್ನು ಊಹಿಸುತ್ತದೆ*

```java
String prompt = "Classify this sentiment: 'I absolutely loved the movie!'";
String response = model.chat(prompt);
// ಪ್ರತಿಕ್ರಿಯೆ: "ಧನಾತ್ಮಕ"
```

**ಬಳಿಸುವ ಸಂದರ್ಭಗಳು:** ಸರಳ ವರ್ಗೀಕರಣಗಳು, ನೇರ ಪ್ರಶ್ನೆಗಳು, ಭಾಷಾಂತರಗಳು ಅಥವಾ ಯಾವುದೇ ಟಾಸ್ಕ್ ಮಾದರಿ ಹೆಚ್ಚಿನ ಮಾರ್ಗದರ್ಶನವಿಲ್ಲದೆ ನಿರ್ವಹಿಸಬಹುದಾದಾಗ.

### ಕಿಮ್ಮತ್ತು-ಶಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್

ನೀವು ಬಯಸುವ ಮಾದರಿಯನ್ನು ಪ್ರಸ್ತುತಪಡಿಸುವ ಉದಾಹರಣೆಗಳನ್ನು ಒದಗಿಸಿ. ಮಾದರಿ ನಿಮಗೆ ನೀಡಿದ ಉದಾಹರಣೆಗಳಿಂದ ನಿರೀಕ್ಷಿತ ಇನ್ಪುಟ್-ಔಟ್‌ಪುಟ್ ಸ್ವರೂಪವನ್ನು ಕಲಿಯುತ್ತದೆ ಮತ್ತು ಹೊಸ ಇನ್ಪುಟ್‌ಗಳಿಗೆ ಅನ್ವಯಿಸುತ್ತದೆ. ಇದು ಅನಿರೀಕ್ಷಿತ ಸ್ವರೂಪ ಅಥವಾ ವರ್ತನೆ ಇರುವ ಕಾರ್ಯಗಳಿಗೆ ಸ್ಥಿರತೆಗೆ ಸಹಾಯ ಮಾಡುತ್ತದೆ.

<img src="../../../translated_images/kn/few-shot-prompting.9d9eace1da88989a.webp" alt="ಕಿಮ್ಮತ್ತು-ಶಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್" width="800"/>

*ಉದಾಹರಣೆಗಳಿಂದ ಕಲಿಯುವುದು — ಮಾದರಿ ಮಾದರಿಯನ್ನು ಗುರುತಿಸಿ ಹೊಸ ಇನ್ಪುಟ್ಗಳಿಗೆ ಅನ್ವಯಿಸುತ್ತದೆ*

```java
String prompt = """
    Classify the sentiment as positive, negative, or neutral.
    
    Examples:
    Text: "This product exceeded my expectations!" → Positive
    Text: "It's okay, nothing special." → Neutral
    Text: "Waste of money, very disappointed." → Negative
    
    Now classify this:
    Text: "Best purchase I've made all year!"
    """;
String response = model.chat(prompt);
```

**ಬಳಿಸುವ ಸಂದರ್ಭಗಳು:** ಕಸ್ಟಮ್ ವರ್ಗೀಕರಣಗಳು, ಸತತ ಸ್ವರೂಪ, ಕ್ಷೇತ್ರ-ನಿರ್ಧಿಷ್ಟ ಕಾರ್ಯಗಳು ಅಥವಾ ಶೂನ್ಯ-ಶಾಟ್ ಫಲಿತಾಂಶಗಳು ಅಸ್ಥಿರವಾಗಿದ್ದಾಗ.

### ಚೇನ್ ಆಫ್ ಥಾಟ್

ಮಾದರಿಯನ್ನು ಹಂತ ಹಂತದ ತರ್ಕವನ್ನು ತೋರಿಸಲು ಕೇಳಿ. ನೇರ ಉತ್ತರಕ್ಕೆ ಹತ್ತಿರವಾಗದೆ, ಸಮಸ್ಯೆಯನ್ನು ವಿಭಜಿಸಿ ಪ್ರತಿ ಭಾಗವನ್ನು ಸ್ಪಷ್ಟವಾಗಿ ವಿಶ್ಲೇಷಿಸುತ್ತದೆ. ಗಣಿತ, ತರ್ಕ ಮತ್ತು ಬಹು-ಹಂತ ತರ್ಕ ಕಾರ್ಯಗಳಲ್ಲಿ ನಿಖರತೆ ಹೆಚ್ಚಿಸುತ್ತದೆ.

<img src="../../../translated_images/kn/chain-of-thought.5cff6630e2657e2a.webp" alt="ಚೇನ್ ಆಫ್ ಥಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್" width="800"/>

*ಹಂತ ಹಂತದ ತರ್ಕ — ಸಂಕೀರ್ಣ ಸಮಸ್ಯೆಗಳನ್ನು ಸ್ಪಷ್ಟ ತಾರ್ಕಿಕ ಹಂತಗಳಿಗೆ ಮುರಿದುಹಾಕುವುದು*

```java
String prompt = """
    Problem: A store has 15 apples. They sell 8 apples and then 
    receive a shipment of 12 more apples. How many apples do they have now?
    
    Let's solve this step-by-step:
    """;
String response = model.chat(prompt);
// ಮಾದರಿ ತೋರಿಸುತ್ತದೆ: 15 - 8 = 7, ಬಳಿಕ 7 + 12 = 19 ಸೇಬುಗಳು
```

**ಬಳಿಸುವ ಸಂದರ್ಭಗಳು:** ಗಣಿತದ ಸಮಸ್ಯೆಗಳು, ತರ್ಕ ಪಜಲ್ಗಳು, ಡಿಬಗಿಂಗ್ ಅಥವಾ ತರ್ಕ ಪ್ರಕ್ರಿಯೆಯನ್ನು ತೋರಿಸುವುದರಿಂದ ನಿಖರತೆ ಮತ್ತು ನಂಬಿಕೆಗೆ ಉತ್ತೇಜನವುಂಟಾಗುವ ಯಾವುದೇ ಕಾರ್ಯ.

### ಪಾತ್ರಾಧಾರಿತ ಪ್ರಾಂಪ್ಟಿಂಗ್

ನಿಮ್ಮ ಪ್ರಶ್ನೆ ಕೇಳುವ ಮೊದಲು AI ಗೆ ಒಂದು ಪಾತ್ರ ಅಥವಾ ವ್ಯಕ್ತಿತ್ವವನ್ನು ನಿಶ್ಚಯಿಸಿ. ಇದರಿಂದ ಉತ್ತರದ ಧ್ವನಿ, ಆಳ ಮತ್ತು ಕೇಂದ್ರಿತ ವಿಷಯ ರೂಪುಗೊಳ್ಳುತ್ತದೆ. "ಸಾಫ್ಟ್‌ವೇರ್ ಆರ್ಕಿಟೆಕ್ಟ್" ಯೊಂದಿಗೆ "ಜూనಿಯರ್ ಡೆವಲಪರ್" ಅಥವಾ "ಸುರಕ್ಷತಾ ವರಿದರ್ಶಕ" ಉದ್ದೇಶ ವ್ಯತ್ಯಾಸವಾಗಿರುತ್ತದೆ.

<img src="../../../translated_images/kn/role-based-prompting.a806e1a73de6e3a4.webp" alt="ಪಾತ್ರಾಧಾರಿತ ಪ್ರಾಂಪ್ಟಿಂಗ್" width="800"/>

*ಸಂದರ್ಭ ಮತ್ತು ಪಾತ್ರ ನಿಗದಿಪಡಿಸುವುದು — ನೀಡಲಾದ ಪಾತ್ರದ ಆಧಾರದ ಮೇಲೆ ಒಂದೇ ಪ್ರಶ್ನೆಗೆ ವಿಭಿನ್ನ ಉತ್ತರ*

```java
String prompt = """
    You are an experienced software architect reviewing code.
    Provide a brief code review for this function:
    
    def calculate_total(items):
        total = 0
        for item in items:
            total = total + item['price']
        return total
    """;
String response = model.chat(prompt);
```

**ಬಳಿಸುವ ಸಂದರ್ಭಗಳು:** ಕೋಡ್ ವಿಮರ್ಶೆಗಳು, ಪಾಠೋಪಯೋಗ, ಕ್ಷೇತ್ರ-ನಿರ್ಧಿಷ್ಟ ವಿಶ್ಲೇಷಣೆ, ಅಥವಾ ವಿಶೇಷ ತಜ್ಞತೆಯ ಮಟ್ಟಕ್ಕೆ ಹೊಂದಿಕೆಯಾಗುವ ಉತ್ತರಗಳು ಬೇಕಾದಾಗ.

### ಪ್ರಾಂಪ್ಟ್ ಟೆಂಪ್ಲೇಟುಗಳು

ಪರಿವರ್ತನಶೀಲ ಸ್ಥಾನಗ್ರಾಹಿಗಳೊಂದಿಗೆ ಪುನರ್ವ ಬಳಕೆ ಮಾಡಬಹುದಾದ ಪ್ರಾಂಪ್ಟ್ ಗಳು ರಚಿಸಿ. ಪ್ರತಿ ಸಾರಿ ಹೊಸ ಪ್ರಾಂಪ್ಟ್ ಬರೆದ ಬದಲು, ಒಂದು 템್ಪ್ಲೇಟ್ ಒಮ್ಮೆ ನಿರ್ಧರಿಸಿ ವಿಭಿನ್ನ ಮೌಲ್ಯಗಳನ್ನು ತುಂಬಿ. LangChain4j ನ `PromptTemplate` ವರ್ಗವನ್ನು `{{variable}}` ವ್ಯಾಕ್ಯರಚನೆಯೊಂದಿಗೆ ಇದನ್ನು ಸುಲಭಗೊಳಿಸುತ್ತದೆ.

<img src="../../../translated_images/kn/prompt-templates.14bfc37d45f1a933.webp" alt="ಪ್ರಾಂಪ್ಟ್ ಟೆಂಪ್ಲೇಟುಗಳು" width="800"/>

*ಸ್ಥಾನಗ್ರಾಹಿಗಳೊಂದಿಗೆ ಪುನರ್ವ ಬಳಕೆ ಮಾಡಬಹುದಾದ ಪ್ರಾಂಪ್ಟ್ ಗಳು — ಒಂದು 템್ಪ್ಲೇಟ್, ಹಲವಾರು ಉಪಯೋಗಗಳು*

```java
PromptTemplate template = PromptTemplate.from(
    "What's the best time to visit {{destination}} for {{activity}}?"
);

Prompt prompt = template.apply(Map.of(
    "destination", "Paris",
    "activity", "sightseeing"
));

String response = model.chat(prompt.text());
```

**ಬಳಿಸುವ ಸಂದರ್ಭಗಳು:** ವಿಭಿನ್ನ ಇನ್ಪುಟ್‌ಗಳೊಂದಿಗೆ ಪುನರಾವೃತ್ತಿ ಪ್ರಶ್ನೆಗಳು, ಬ್ಯಾಚ್ ಪ್ರಕ್ರಿಯೆ, ಪುನರ್ವ ಬಳಕೆಯ AI ಕಾರ್ಯಪ್ರವಾಹ ನಿರ್ಮಾಣ, ಅಥವಾ ಪ್ರಾಂಪ್ಟ್ ರಚನೆ ಒಂದೇ ಇದ್ದು ದತ್ತಾಂಶ ಬದಲಾಗುವ ಯಾವುದೇ ಪರಿಸ್ಥಿತಿ.

---

ಇವು ಐದು ಮೂಲಭೂತಗಳು ಹೆಚ್ಚಿನ ಪ್ರಾಂಪ್ಟ್ ಕಾರ್ಯಗಳಿಗೆ ನಿಮಗೆ ಭದ್ರ ಟೂಲ್ ಕಿಟ್ ನೀಡುತ್ತವೆ. ಈ Module ಉಳಿದ ಪಾಠಗಳು GPT-5.2 ಯ ತರ್ಕ ನಿಯಂತ್ರಣ, ಸ್ವ-मೌಲ್ಯಮಾಪನ ಮತ್ತು ಸಂರಚಿತ ಔಟ್‌ಪುಟ್ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಉಪಯೋಗಿಸುವ **ಎಂಟು ಅತ್ಯಾಧುನಿಕ ಮಾದರಿಗಳ** ಮೇಲೆ ಆಧರಿಸಿದೆ.

## ಅತ್ಯಾಧುನಿಕ ಮಾದರಿಗಳು

ಮೂಲಭೂತಾಂಶಗಳನ್ನು ಸಂಪೂರ್ಣವಾಗಿ ಕಲಿತ ನಂತರ, ಈ Module ನ ವಿಶಿಷ್ಟತೆಯನ್ನು ನೀಡುವ ಎಂಟು ಅತ್ಯಾಧುನಿಕ ಮಾದರಿಗಳತ್ತ ಸಾಗೋಣ. ಎಲ್ಲಾ ಸಮಸ್ಯೆಗಳು ಒಂದೇ ರೀತಿಯ ಮಾರ್ಗವನ್ನು ಅವಲಂಬಿಸುವುದಿಲ್ಲ. ಕೆಲವು ಪ್ರಶ್ನೆಗಳು ತಕ್ಷಣದ ಉತ್ತರಗಳನ್ನು ಬೇಕಾಗಿರುತ್ತವೆ, ಇತರವು ಆಳವಾದ ಚಿಂತನೆ ಅಗತ್ಯವಿರುತ್ತದೆ. ಕೆಲವು ದೃಶ್ಯಮಾನದ ತರ್ಕ ಬೇಕಾಗುತ್ತದೆ, ಇನ್ನ ಕೆಲವು ಕೇವಲ ಫಲಿತಾಂಶ ಬೇಕು. ಕೆಳಗಿನ ಪ್ರತಿ ಮಾದರಿ ವಿಭಿನ್ನ ಪರಿಸ್ಥಿತಿಗೆ ತಕ್ಕಂತೆ ಅತ್ಯುತ್ತಮಗೊಳಿಸಲಾಗಿದೆ — ಮತ್ತು GPT-5.2 ಯ ತರ್ಕ ನಿಯಂತ್ರಣವು ವಿಭಿನ್ನತೆಗಳನ್ನು ಇನ್ನಷ್ಟು ಸ್ಪಷ್ಟಗೊಳಿಸುತ್ತದೆ.

<img src="../../../translated_images/kn/eight-patterns.fa1ebfdf16f71e9a.webp" alt="ಎಂಟು ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಮಾದರಿಗಳ ಅವಲೋಕನ" width="800"/>

*ಎಂಟು ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ ಮಾದರಿಗಳ ಮತ್ತು ಅವುಗಳ ಉಪಯೋಗ ಪ್ರಕರಣಗಳ ಅವಲೋಕನ*

<img src="../../../translated_images/kn/reasoning-control.5cf85f0fc1d0c1f3.webp" alt="GPT-5.2 ಒಡನೆ ತರ್ಕ ನಿಯಂತ್ರಣ" width="800"/>

*GPT-5.2 ಯ ತರ್ಕ ನಿಯಂತ್ರಣ ನಿಮಗೆ ಮಾದರಿ ಎಷ್ಟು ಚಿಂತನೆ ಮಾಡಬೇಕೆಂಬುದನ್ನು ನಿಗದಿಪಡಿಸಲು ಅನುಕೂಲ ಕಲ್ಪಿಸುತ್ತದೆ — ವೇಗದ ನೇರ ಉತ್ತರದಿಂದ ಆಳವಾದ ಅನ್ವೇಷಣೆಗೆ*

**ಕಡಿಮೆ ಆಸಕ್ತಿ (ಶೀಘ್ರ ಮತ್ತು ಕೇಂದ್ರೀಕೃತ)** - ಸರಳ ಪ್ರಶ್ನೆಗಳಿಗೆ ವೇಗವಾಗಿ ನೇರ ಉತ್ತರ ಬೇಕಾದಾಗ. ಮಾದರಿಯು ಕನಿಷ್ಠ ತರ್ಕ ಮಾಡುತ್ತದೆ - ಗರಿಷ್ಠ 2 ಹಂತಗಳು. ಲೆಕ್ಕಾಚಾರ, ಹುಡುಕಾಟ ಅಥವಾ ಸರಳ ಪ್ರಶ್ನೆಗಳಿಗೆ ಬಳಸಿ.

```java
String prompt = """
    <context_gathering>
    - Search depth: very low
    - Bias strongly towards providing a correct answer as quickly as possible
    - Usually, this means an absolute maximum of 2 reasoning steps
    - If you think you need more time, state what you know and what's uncertain
    </context_gathering>
    
    Problem: What is 15% of 200?
    
    Provide your answer:
    """;

String response = chatModel.chat(prompt);
```

> 💡 **GitHub Copilot ಸಹಾಯದಿಂದ ಅನ್ವೇಷಿಸಿ:** [`Gpt5PromptService.java`](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java) ನಲ್ಲಿ ವಿಚಾರಿಸಿ:
> - "ಕಡಿಮೆ ಆಸಕ್ತಿ ಮತ್ತು ಹೆಚ್ಚಿನ ಆಸಕ್ತಿಯ ಪ್ರಾಂಪ್ಟಿಂಗ್ ಮಾದರಿಗಳ ವ್ಯತ್ಯಾಸ ಏನು?"
> - "XML ಟ್ಯಾಗ್ಗಳು ಪ್ರಾಂಪ್ಟ್‌ಗಳಲ್ಲಿ AI ಉತ್ತರವನ್ನು ಹೇಗೆ ವಿನ್ಯಾಸಗೊಳಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತವೆ?"
> - "ಸ್ವ-ಪರಿವೀಕ್ಷಣಾ ಮಾದರಿಗಳು ಮತ್ತು ನೇರ ಸೂಚನೆಗಳನ್ನು ಯಾವಾಗ ಬಳಸಬೇಕು?"

**ಹೆಚ್ಚಿನ ಆಸಕ್ತಿ (ಆಳವಾದ ಮತ್ತು ಸಂಪೂರ್ಣ)** - ಸಂಕೀರ್ಣ ಸಮಸ್ಯೆಗಳಿಗೆ ಸಮಗ್ರ ವಿಶ್ಲೇಷಣೆ ಬೇಕಾದಾಗ. ಮಾದರಿ ಸಂಪೂರ್ಣ ಅನ್ವೇಷಿಸಿ ವಿವರವಾದ ತರ್ಕವನ್ನು ತೋರಿಸುತ್ತದೆ. ವ್ಯವಸ್ಥೆ ವಿನ್ಯಾಸ, ವಾಸ್ತುಶಿಲ್ಪ ನಿರ್ಣಯಗಳು ಅಥವಾ ಸಂಕೀರ್ಣ ಸಂಶೋಧನೆಗೆ ಬಳಸಿ.

```java
String prompt = """
    Analyze this problem thoroughly and provide a comprehensive solution.
    Consider multiple approaches, trade-offs, and important details.
    Show your analysis and reasoning in your response.
    
    Problem: Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```

**ಕಾರ್ಯ ನಿಯೋಜನೆ (ಹಂತ ಹಂತದ ಪ್ರಗತಿ)** - ಬಹು-ಹಂತ ಕಾರ್ಯಸ್ವರೂಪಗಳಿಗೆ. ಮಾದರಿ ಮೊದಲಿಗೆ ಯೋಜನೆಯನ್ನು ಒದಗಿಸುತ್ತದೆ, ಪ್ರತಿ ಹಂತವನ್ನು ಕೆಲಸಿಸುತ್ತಾ ವರ್ಣಿಸುತ್ತದೆ, ನಂತರ ಸಂಕ್ಷಿಪ್ತು ನೀಡುತ್ತದೆ. ಸ್ಥಳಾಂತರಗಳು, ಜಾರಿಗೊಳ್ಳುವಿಕೆಗಳು ಅಥವಾ ಯಾವುದೇ ಬಹು ಹಂತ ಪ್ರಕ್ರಿಯೆಗೆ ಉಪಯುಕ್ತ.

```java
String prompt = """
    <task_execution>
    1. First, briefly restate the user's goal in a friendly way
    
    2. Create a step-by-step plan:
       - List all steps needed
       - Identify potential challenges
       - Outline success criteria
    
    3. Execute each step:
       - Narrate what you're doing
       - Show progress clearly
       - Handle any issues that arise
    
    4. Summarize:
       - What was completed
       - Any important notes
       - Next steps if applicable
    </task_execution>
    
    <tool_preambles>
    - Always begin by rephrasing the user's goal clearly
    - Outline your plan before executing
    - Narrate each step as you go
    - Finish with a distinct summary
    </tool_preambles>
    
    Task: Create a REST endpoint for user registration
    
    Begin execution:
    """;

String response = chatModel.chat(prompt);
```

ಚೇನ್-ಆಫ್-ಥಾಟ್ ಪ್ರಾಂಪ್ಟಿಂಗ್ ಸ್ಪಷ್ಟವಾಗಿ ಮಾದರಿಗೆ ತನ್ನ ತರ್ಕ ಪ್ರಕ್ರಿಯೆಯನ್ನು ತೋರಿಸಲು ಕೇಳುತ್ತದೆ, ಇದು ಸಂಕೀರ್ಣ ಕಾರ್ಯಗಳಲ್ಲಿ ನಿಖರತೆಯನ್ನು ಹೆಚ್ಚಿಸುತ್ತದೆ. ಹಂತ ಹಂತದ ವಿಭಾಗವು ಮಾನವರಿಗೂ AI ಗೆಲೂ ತರ್ಕವನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು ಸಹಾಯಕ.

> **🤖 GitHub Copilot ಚಾಟ್ ಮೂಲಕ ಪ್ರಯತ್ನಿಸಿ:** ಈ ಮಾದರಿಯನ್ನು ಕೇಳಿ:
> - "ದೀರ್ಘಾವಧಿ ಕಾರ್ಯಗಳಿಗಾಗಿ ಕಾರ್ಯ ಕಾರ್ಯಗತಿ ಮಾದರಿಯನ್ನು ಹೇಗೆ ಹೊಂದಿಸಬಹುದು?"
> - "ಉತ್ಪಾದನಾ ಅಪ್ಲಿಕೇಶನ್‌ಗಳಲ್ಲಿ ಟೂಲ್ ಪ್ರೀಅಂಬಲ್ಸ್ ಅನ್ನು ರಚಿಸುವ ಉತ್ತಮ ವಿಧಾನಗಳೇನು?"
> - "ಐವನಲ್ಲಿ ಮಧ್ಯಸ್ಥ ಪ್ರಗತಿ ನವೀಕರಣಗಳನ್ನು ಹೇಗೆ ಸೆರೆ ಹಿಡಿದು ಪ್ರದರ್ಶಿಸಬಹುದು?"

<img src="../../../translated_images/kn/task-execution-pattern.9da3967750ab5c1e.webp" alt="ಕಾರ್ಯ ನಿಯೋಜನೆ ಮಾದರಿ" width="800"/>

*ಯೋಜನೆ → ಕಾರ್ಯಗತಿ → ಸಂಕ್ಷೇಪಣಾ ಕಾರ್ಯಪಥ*

**ಸ್ವ-ಪ್ರತಿಬಿಂಬಿಸುವ ಕೋಡ್** - ಉತ್ಪಾದನಾ ಗುಣಮಟ್ಟದ ಕೋಡ್ ತಯಾರಿಸಲು. ಮಾದರಿ ಉತ್ಪಾದನಾ ಧೋರಣೆಯನ್ವಯ ಕೋಡ್ ರಚಿಸುತ್ತದೆ ಹಾಗೂ ಸೂಕ್ತ ದೋಷ ನಿರ್ವಹಣೆಯನ್ನೂ ಒಳಗೊಂಡಿರುತ್ತದೆ. ಹೊಸ ವೈಶಿಷ್ಟ್ಯಗಳು ಅಥವಾ ಸರ್ವೀಸರು ನಿರ್ಮಿಸುವಾಗ ಇದನ್ನು ಬಳಸಿ.

```java
String prompt = """
    Generate Java code with production-quality standards: Create an email validation service
    Keep it simple and include basic error handling.
    """;

String response = chatModel.chat(prompt);
```

*ಪುನರಾವೃತ್ತಿ ಸುಧಾರಣಾ ಲూప್ - ರಚಿಸಿ, ಮೌಲ್ಯಮಾಪನ ಮಾಡಿ, ಸಮಸ್ಯೆ ಗುರುತು ಮಾಡು, ಸುಧಾರಿಸು, ಪುನರಾವರ್ತಿಸಿ*

**ಸಂರಚಿತ ವಿಶ್ಲೇಷಣೆ** - ಸ್ಥಿರ ಮೌಲ್ಯಮಾಪನಕ್ಕಾಗಿ. ಮಾದರಿ ಕód್ ಅನ್ನು ನಿಗದಿತ ಚೌಕಟ್ಟಿನಲ್ಲಿ ವಿಮರ್ಶಿಸುತ್ತದೆ (ಸರಿಯಾದತೆ, ಅಭ್ಯಾಸಗಳು, ಕಾರ್ಯಕ್ಷಮತೆ, ಭದ್ರತೆ, ನಿರ್ವಹಣಾ ಸಾಧ್ಯತೆ). ಕೋಡ್ ವಿಮರ್ಶೆಗಳು ಅಥವಾ ಗುಣಮಟ್ಟ ಮೌಲ್ಯಮಾಪನಗಳಿಗೆ ಬಳಸಿ.

**ಹಂತ ಹಂತದ ತರ್ಕ** - ದೃಶ್ಯ ತಂದುಕೊಳ್ಳುವ ತರ್ಕ ಅಗತ್ಯವಿರುವ ಸಮಸ್ಯೆಗಳಿಗೆ. ಮಾದರಿ ಪ್ರತಿ ಹಂತಕ್ಕೆ ಸ್ಪಷ್ಟ ತರ್ಕವನ್ನು ತೋರಿಸುತ್ತದೆ. ಗಣಿತ ಸಮಸ್ಯೆಗಳು, ತರ್ಕ ಪಜಲ್ಗಳು ಅಥವಾ ಚಿಂತನೆ ಪ್ರಕ್ರಿಯೆಯನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು ಬೇಕಾದಾಗ ಉಪಯುಕ್ತ.

ಡೆವ್ ಕಂಟೈನರ್ ಸ್ಪ್ರಿಂಗ್ ಬೂಟ್ ಡ್ಯಾಶ್‌ಬೋರ್ಡ್ ವಿಸ್ತರಣೆ ಅನ್ನು ಒಳಗೊಂಡಿದೆ, ಇದು ಎಲ್ಲಾ ಸ್ಪ್ರಿಂಗ್ ಬೂಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿರ್ವಹಿಸಲು ದೃಶ್ಯಾತ್ಮಕ ಬಳಕೆಮುಖವನ್ನು ಒದಗಿಸುತ್ತದೆ. VS Code ನ ಎಡಭಾಗದಲ್ಲಿ ಇರುವ ಕ್ರಿಯಾಶೀಲತಾ ಬಾರ್‌ನಲ್ಲಿ (Spring Boot ಚಿಹ್ನೆಯನ್ನು ಹುಡುಕಿ) ಇದನ್ನು ಕಾಣಬಹುದು.

- ಒತ್ತುವಿಕೆ ಮೂಲಕ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಪ್ರಾರಂಭ/ನಿರ್ವಹಿಸಬಹುದು
- ಅಪ್ಲಿಕೇಶನ್ ಲಾಗ್‌ಗಳನ್ನು ನೇರವಾಗಿ ವೀಕ್ಷಿಸಬಹುದು

ವೆಬ್ ಇಂಟರ್ಫೇಸ್ ನಿಮಗೆ ವಿಭಿನ್ನ ಪ್ರಾಂಪ್ಟ್ ತಂತ್ರಗಳನ್ನು ಪ್ರಯೋಗಿಸಲು ಅನುಮತಿಸುತ್ತದೆ. ಪ್ರತಿ ಮಾದರಿ ವಿಭಿನ್ನ ಸಮಸ್ಯೆಗಳನ್ನು ಪರಿಹರಿಸುತ್ತದೆ - ಪ್ರತಿ ವಿಧಾನವು ಯಾವಾಗ ಬೆಳಗುತ್ತದೆ ಎನ್ನುವುದನ್ನು ನೋಡಲು ಅವುಗಳನ್ನು ಪ್ರಯತ್ನಿಸಿ.

### ಕಡಿಮೆ ಮತ್ತು ಹೆಚ್ಚಿನ ಆಸಕ್ತಿ

"200 ರ 15% ఏమಿದೆ?" ಎಂಬ ಸರಳ ಪ್ರಶ್ನೆಯನ್ನು ಕಡಿಮೆ ಆಸಕ್ತಿಯಿಂದ ಕೇಳಿ. ನಿಮಗೆ ತಕ್ಷಣ, ನೇರ ಉತ್ತರ ದೊರೆತುಬರುತ್ತದೆ. ಈಗ "ಹೆಚ್ಚು ಟ್ರಾಫಿಕ್ ಇರುವ API ಗಾಗಿ ಕ್ಯಾಶಿಂಗ್ ತಂತ್ರವನ್ನು ವಿನ್ಯಾಸಮಾಡಿ" ಎಂಬ ಸಂಕೀರ್ಣ ಪ್ರಶ್ನೆಯನ್ನು ಹೆಚ್ಚಿನ ಆಸಕ್ತಿಯಿಂದ ಕೇಳಿ. ಮಾದರಿ ನಿಧಾನಗೊಳ್ಳುತ್ತದೆ ಮತ್ತು ವಿವರವಾದ ಕಾರಣವನ್ನು ಒದಗಿಸುತ್ತದೆ. ಅದೇ ಮಾದರಿ, ಅದೇ ಪ್ರಶ್ನೆಯ ರಚನೆ - ಆದರೆ ಪ್ರಾಂಪ್ಟ್ ಎಷ್ಟು ಆಲೋಚನೆ ಮಾಡು ಎಂಬುದನ್ನು ಸೂಚಿಸುತ್ತದೆ.

### ಕಾರ್ಯ ನಿರ್ವಹಣೆ (ಟೂಲ್ ಪೀರೆಂಬಲ್ಸ್)

ಬಹು-ಹಂತ ಕಾರ್ಯಪಥಗಳು ಮುಂಚಿತ ಯೋಜನೆ ಮತ್ತು ಪ್ರಗತಿ ವಿವರಣೆಗಳಿಂದ ಲಾಭ ಪಡೆಯುತ್ತವೆ. ಮಾದರಿ ಅದು ಏನು ಮಾಡಲು ಹೋಗುತ್ತಿದೆ ಎಂದು ರೂಪರೋಪಿಸಿ, ಪ್ರತಿ ಹಂತವನ್ನು ವಿವರಿಸಿ ನಂತರ ಫಲಿತಾಂಶಗಳನ್ನು ಸಾರಾಂಶಗೊಳಿಸುತ್ತದೆ.

<img src="../../../translated_images/kn/tool-preambles-demo.3ca4881e417f2e28.webp" alt="Task Execution Demo" width="800"/>

*ಹಂತ ಹಂತದ ವಿವರಣೆ ಜೊತೆಗೆ REST ಅಂತರ್ಜಾಲಕ್ಕೆ ಭೇಟಿ ಸೃಷ್ಟಿಸುವುದು (3.9MB)*

### ಸ್ವ-ಅವಲೋಕನ ಕೋಡ್

"ಇಮೇಲ್ ಮಾನ್ಯತೆ ಸೇವೆಯನ್ನು ಸೃಷ್ಟಿಸಿ" ಎಂದು ಪ್ರಯತ್ನಿಸಿ. ಕೇವಲ ಕೋಡ್ ಉತ್ಪಾದನೆ ಮಾಡಿ ನಿಲ್ಲಿಸುವ ಬದಲು, ಮಾದರಿ ಉತ್ಪಾದಿಸಿ, ಗುಣಮಟ್ಟ ಪ್ರಮಾಣಗಳಿಗೆ ವಿರುದ್ಧವಾಗಿ ಮೌಲ್ಯಮಾಪನ ಮಾಡಿ, ದುರ್ಬಲತೆಗಳನ್ನು ಗುರುತಿಸಿ ಮತ್ತು ಸುಧಾರಣೆ ಮಾಡುತ್ತದೆ. ನೀವು ಕೋಡ್ ಉತ್ಪಾದನೆ ಉತ್ಪಾದನೆಗಿಂತ ವೃತ್ತಿಪರ ಮಾನದಂಡಗಳಿಗೆ ತಲುಪುವ ತನಕ ಅದನ್ನು ಪುನರಾವರ್ತಿಸುವುದನ್ನು ನೋಡುತ್ತೀರಿ.

<img src="../../../translated_images/kn/self-reflecting-code-demo.851ee05c988e743f.webp" alt="Self-Reflecting Code Demo" width="800"/>

*ಪೂರ್ಣ ಇಮೇಲ್ ಮಾನ್ಯತೆ ಸೇವೆ (5.2MB)*

### ರಚನೆಗೊಳಿಸಲಾದ ವಿಶ್ಲೇಷಣೆ

ಕೋಡ್ ವಿಮರ್ಶೆಗಳು ಸ್ಥಿರ ಮೌಲ್ಯಮಾಪನ ಚಟುವಟಿಕೆಗಳನ್ನು ಅಗತ್ಯವಿರುತ್ತದೆ. ಮಾದರಿ ನಿರ್ದಿಷ್ಟ ವರ್ಗಗಳಿಂದ (ಸರಿಯಾದತೆ, ಅಭ್ಯಾಸಗಳು, ಕಾರ್ಯಕ್ಷಮತೆ, ಭದ್ರತೆ) ಮತ್ತು ತೀವ್ರತೆ ಮಟ್ಟಗಳಿಂದ ಕೋಡ್ ವಿಶ್ಲೇಷಿಸುತ್ತದೆ.

<img src="../../../translated_images/kn/structured-analysis-demo.9ef892194cd23bc8.webp" alt="Structured Analysis Demo" width="800"/>

*ಫ್ರೇಮ್‌ವರ್ಕ್ ಆಧಾರದ ಕೋಡ್ ವಿಮರ್ಶೆ*

### ಬಹು-ತೆರವು ಸಂಭಾಷಣೆ

"Spring Boot ಎಂದರೆ ಏನು?" ಎಂದು ಕೇಳಿ ನಂತರ ತಕ್ಷಣ "ತೋರು ಒಂದು ಉದಾಹರಣೆ" ಎಂದು ಅನುವೃಷ್ಟಿ ನೀಡಿ. ಮಾದರಿ ನಿಮ್ಮ ಮೊದಲ ಪ್ರಶ್ನೆಯನ್ನು ನೆನಪಿಡುತ್ತದೆ ಮತ್ತು ನಿಮಗೆ ಸ್ಪ್ರಿಂಗ್ ಬೂಟ್ ಉದಾಹರಣೆ ನೀಡುತ್ತದೆ. ನೆನಪಿಲ್ಲದೆ, ಆ ಎರಡನೇ ಪ್ರಶ್ನೆ ಅಸ್ಪಷ್ಟವಾಗುತ್ತಿತ್ತು.

<img src="../../../translated_images/kn/multi-turn-chat-demo.0d2d9b9a86a12b4b.webp" alt="Multi-Turn Chat Demo" width="800"/>

*ಪ್ರಶ್ನೆಗಳ ನಡುವಿನ ಸನ್ನಿವೇಶ ಸಂರಕ್ಷಣೆ*

### ಹಂತ ಹಂತದ ಕಾರಣ

ಒಂದು ಗಣಿತ ಸಮಸ್ಯೆಯನ್ನು ಆಯ್ಕೆಮಾಡಿ ಮತ್ತು ಅದನ್ನು ಹಂತ ಹಂತದ ಕಾರಣ ಮತ್ತು ಕಡಿಮೆ ಆಸಕ್ತಿಯಿಂದ ಪ್ರಯತ್ನಿಸಿ. ಕಡಿಮೆ ಆಸಕ್ತಿ ನಿಮಗೆ ತ್ವರಿತ ಉತ್ತರವನ್ನು ಮಾತ್ರ ನೀಡುತ್ತದೆ - ತ್ವರಿತ ಆದರೆ ಅಸ್ಪಷ್ಟ. ಹಂತ ಹಂತದ ಕ್ರಮವೂ ಎಲ್ಲಾ ಲೆಕ್ಕಾಚಾರ ಮತ್ತು ತೀರ್ಮಾನವನ್ನು ತೋರಿಸುತ್ತದೆ.

<img src="../../../translated_images/kn/step-by-step-reasoning-demo.12139513356faecd.webp" alt="Step-by-Step Reasoning Demo" width="800"/>

*ಸ್ಪಷ್ಟ ಹಂತಗಳೊಂದಿಗೆ ಗಣಿತ ಸಮಸ್ಯೆ*

### ನಿಯಂತ್ರಿತ ಔಟ್‌ಪುಟ್

ನೀವು ನಿಖರವಾಗಿ ನಿರ್ದಿಷ್ಟ ಬಗೆಯ ಆಕಾರಗಳು ಅಥವಾ ಪದಗಳ ಸಂಖ್ಯೆಯನ್ನು ಬೇಕಾದಾಗ, ಈ ಮಾದರಿ ಕಟ್ಟುನಿಟ್ಟಾದ ಅನುಷ್ಠಾನವನ್ನು ಜಾರಿಗೆ ತರುತ್ತದೆ. 100 ಪದಗಳ ಸಮಗ್ರ ಸಾರಾಂಶವನ್ನು ಬುಲೆಟ್ ಪಾಯಿಂಟ್ ಫಾರ್ಮ್ಯಾಟ್‌ನಲ್ಲಿ ರಚಿಸುವುದನ್ನು ಪ್ರಯತ್ನಿಸಿ.

<img src="../../../translated_images/kn/constrained-output-demo.567cc45b75da1633.webp" alt="Constrained Output Demo" width="800"/>

*ರೂಪ ನಿಯಂತ್ರಣದೊಂದಿಗೆ ಯಂತ್ರ ಕಲಿಕೆಯ ಸಾರಾಂಶ*

## ನೀವು ನಿಜವಾಗಿಯೂ ಕಲಿಯುತ್ತಿರುವುದು

**ಕಾರಣದ ಪ್ರಯತ್ನ ಎಲ್ಲವನ್ನೂ ಬದಲಾಯಿಸುತ್ತದೆ**

GPT-5.2 ನಿಮ್ಮ ಪ್ರಾಂಪ್ಟ್‌ಗಳ ಮೂಲಕ ಗಣಿತ ಪ್ರಯತ್ನವನ್ನು ನಿಯಂತ್ರಿಸಲು ಅನುಮತಿಸುತ್ತದೆ. ಕಡಿಮೆ ಪ್ರಯತ್ನ ಅರ್ಥವೇ ಬೇಗ ಪ್ರತಿಕ್ರಿಯೆಯೊಂದಿಗೆ ಕನಿಷ್ಟ ಅನ್ವೇಷಣೆಯನ್ನು ಸೂಚಿಸುತ್ತದೆ. ಹೆಚ್ಚಿನ ಪ್ರಯತ್ನ ಅರ್ಥವೇ ಮಾದರಿ ಆಳವಾಗಿ ಯೋಚಿಸಲು ಸಮಯ ತೆಗೆದುಕೊಳ್ಳುತ್ತದೆ. ನೀವು ಕಾರ್ಯ ಸಂಕೀರ್ಣತೆಗೆ ಪ್ರಯತ್ನವನ್ನು ಹೊಂದಿಸಲು ಕಲಿಯುತ್ತೀರಿ - ಸರಳ ಪ್ರಶ್ನೆಗಳಿಗೆ ಸಮಯ ನಿಷ್ಟವಿಲ್ಲದೆ ಮೀಸಲಿಡಬೇಡಿ, ಆದರೆ ಸಂಕೀರ್ಣ ತೀರ್ಮಾನಗಳಿಗೆ ಕೂಡ ತ್ವರಿತ ಮಾಡಬೇಡಿ.

**ರಚನೆಯು ದಿಕ್ಕೆ ನೀಡುತ್ತದೆ**

ಪ್ರಾಂಪ್ಟ್‌ಗಳಲ್ಲಿ XML ಟ್ಯಾಗ್ಗಳನ್ನು ಗಮನಿಸಿದ್ದೀರಾ? ಅವು ಭೂಷಣ ಮುಖ್ಯವಲ್ಲ. ಮಾದರಿಗಳು ಸ್ವತಃ ನಿರ್ಮಿತ ನಿರ್ದೇಶನಗಳನ್ನು ಮುಕ್ತಾವಿನ್ಯಾಸ ಪಠ್ಯದಿಗಿಂತ ಹೆಚ್ಚು ವಿಶ್ವಾಸಾರ್ಹವಾಗಿ ಅನುಸರಿಸುತ್ತವೆ. ನೀವು ಬಹು-ಹಂತ ಪ್ರಕ್ರಿಯೆಗಳು ಅಥವಾ ಸಂಕೀರ್ಣ ಲಾಜಿಕ್ ಬೇಕಾದಾಗ, ರಚನೆ ಮಾದರಿಯನ್ನು ಅದರ ಇದ್ದ ಸ್ಥಳ ಮತ್ತು ಮುಂದೇನು ಬರುವುದನ್ನು ಹಾದುಹೋಗಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.

<img src="../../../translated_images/kn/prompt-structure.a77763d63f4e2f89.webp" alt="Prompt Structure" width="800"/>

*ಸ್ಪಷ್ಟ ವಿಭಾಗಗಳ ಮತ್ತು XML ಶೈಲಿಯ ವ್ಯವಸ್ಥೆಯೊಂದಿಗೆ ಚೆನ್ನಾಗಿ ರಚಿಸಲಾದ ಪ್ರಾಂಪ್ಟ್ ರಚನೆಗೊಳಿಸುವಿಕೆ*

**ಗುಣಮಟ್ಟ ಸ್ವ-ಮೌಲ್ಯಮಾಪನದ ಮೂಲಕ**

ಸ್ವ-ಅವಲೋಕನ ಮಾದರಿಗಳು ಗುಣಮಟ್ಟದ ಮಾನದಂಡಗಳನ್ನು ಸ್ಪಷ್ಟವಾಗಿ ಮಾಡುತ್ತಿದೆ. ಮಾದರಿ "ಸರಿಯಾಗಿ ಮಾಡುತ್ತಿದೆ" ಎಂದು ನಿರೀಕ್ಷಿಸುವ ಬದಲು ನೀವು "ಸರಿಯಾಗಿದೆ" ಎಂದರ್ಥವೇನು ಎಂಬುದನ್ನು ನಿಖರವಾಗಿ ತಿಳಿಸುತ್ತೀರಿ: ಸರಿಯಾದ ಲಾಜಿಕ್, ದೋಷ ನಿರ್ವಹಣೆ, ಕಾರ್ಯಕ್ಷಮತೆ, ಭದ್ರತೆ. ನಂತರ ಮಾದರಿ ತನ್ನದೇ ಉತ್ಪನ್ನವನ್ನು ಅರ್ಹವಾಗಿ ಮೌಲ್ಯಮಾಪನ ಮಾಡಿ ಸುಧಾರಿಸುತ್ತದೆ. ಇದು ಕೋಡ್ ಉತ್ಪಾದನೆಯನ್ನು ಲಾಟರಿ ಅಲ್ಲದೆ ಪ್ರಕ್ರಿಯೆಯಾಗಿ ಪರಿವರ್ತಿಸುತ್ತದೆ.

**ಸನ್ನಿವೇಶವು ಸೀಮಿತವಾಗಿದೆ**

ಬಹು-ತೆರವು ಸಂಭಾಷಣೆಗಳು ಪ್ರತ್ಯೇಕ ವಿನಂತಿಗಳೊಂದಿಗೆ ಸಂದೇಶ ಹಿಸ್ಟೆರಿ ಸೇರಿಸುವ ಮೂಲಕ ಕೆಲಸ ಮಾಡುತ್ತವೆ. ಆದರೆ ಮಿತಿಯಿದೆ - ಪ್ರತಿ ಮಾದರಿಯು ಗರಿಷ್ಠ ಟೋಕನ್ ಮಿತಿ ಹೊಂದಿದೆ. ಸಂಭಾಷಣೆಗಳು ವೃದ್ಧಿಯಾಗುವಂತೆ ನೀವು ಸಂಬಂಧಿತ ಸನ್ನಿವೇಶವನ್ನು ಕಾಪಾಡಲು ತಂತ್ರಗಳನ್ನು ತಿಳಿಯಬೇಕಾಗುತ್ತದೆ ಮತ್ತು ಅದರ ಮಿತಿ ಹರಿಯದಂತೆ ನೋಡಿಕೊಳ್ಳಬೇಕು. ಈ ಮೊಡ್ಯೂಲ್ ನಿಮಗೆ ನೆನಪಿನ ಪ್ರಕ್ರಿಯೆಯನ್ನು ತೋರಿಸುತ್ತದೆ; ನಂತರ ನೀವು ಸಂಕ್ಷೇಪಣೆ, ಮರೆತುಹೋಗುವಿಕೆ ಮತ್ತು ಹಿಂಪಡೆಯುವ ಸಮಯಗಳನ್ನು ತಿಳಿಯುತ್ತೀರಿ.
