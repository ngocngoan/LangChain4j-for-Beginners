## ಮೂಲಗರcstring್ಯಂಗಳು

**AI ಏಜೆಂಟ್** - ಸ್ವತಂತ್ರವಾಗಿ ತರ್ಕ್ಮಾಡಿ ಮತ್ತು ಕಾರ್ಯಾನುಷ್ಠಾನ ಮಾಡುವ AI ಉಪಾಯಕ ವ್ಯವಸ್ಥೆ. [ಮಾಡ್ಯೂಲ್ 04](../04-tools/README.md)

**ಚೈನ್** - ನಿರಂತರ ಕಾರ್ಯಾಲಯಗಳ ಸರಪಳಿ, ಒಂದು ಹೊರತಂಡವು ಮುಂದಿನ ಹಂತಕ್ಕೆ ಆಹಾರ ನೀಡುತ್ತದೆ.

**ಚಂಕಿಂಗ್** - ದಾಖಲೆಗಳನ್ನು ಚಿಕ್ಕ ತುಂಡುಗಳಾಗಿ ವಿಭಜಿಸುವುದು. ಸಾಮಾನ್ಯ: 300-500 ಟೋಕನ್‌ಗಳು ನಿಧಾನವಾಗಿ ಒಪ್ಪಂದ. [ಮಾಡ್ಯೂಲ್ 03](../03-rag/README.md)

**ಕಾಂಟೆಕ್ಸ್ಟ್ ವಿಂಡೋ** - ಮಾದರಿಯು ಪ್ರಕ್ರಿಯೆ ಮಾಡಬಹುದಾದ ಗರಿಷ್ಠ ಟೋಕನ್‌ಗಳು. GPT-5.2: 400,000 ಟೋಕನ್‌ಗಳು (272,000 ಇನ್‌ಪುಟ್, 128,000 ಔಟ್‌ಪುಟ್ ವರೆಗೆ).

**ಎಂಬೆಡ್ಡಿಂಗ್ಸ್** - ಪಠ್ಯದ ಅರ್ಥವನ್ನು ಪ್ರತಿನಿಧಿಸುವ ಸಂಖ್ಯಾತ್ಮಕ ವೆಕ್ಟರ್‌ಗಳು. [ಮಾಡ್ಯೂಲ್ 03](../03-rag/README.md)

**ಫಂಕ್ಷನ್ ಕಾಲಿಂಗ್** - ಮಾದರಿ ಹೊರಗಿನ ಕಾರ್ಯಗಳನ್ನು ಕರೆ ಮಾಡಲು ರಚನೆಯ ವಿನಂತಿಗಳನ್ನು ಉತ್ಪಾದಿಸುತ್ತದೆ. [ಮಾಡ್ಯೂಲ್ 04](../04-tools/README.md)

**ಹ್ಯಾಲೂಸಿನೇಷನ್** - ಮಾದರಿಗಳು ತಪ್ಪು ಆದರೆ ಸಾಧ್ಯತೆಯ ಮಾಹಿತಿಯನ್ನು ರಚಿಸುವಾಗ.

**ಪ್ರಾಂಪ್ಟ್** - ಭಾಷಾ ಮಾದರಿಗೆ ಪಠ್ಯದ ಇನ್‌ಪುಟ್. [ಮಾಡ್ಯೂಲ್ 02](../02-prompt-engineering/README.md)

**ಸೆಮ್ಯಾಂಟಿಕ್ ಶೋಧ** - ಕೀವರ್ಡ್ಗಳಿಲ್ಲದೆ ಅರ್ಥ ಬಳಸಿ ಹುಡುಕು. [ಮಾಡ್ಯೂಲ್ 03](../03-rag/README.md)

**ಸ್ಟೇಟ್‌ಫುಲ್ ವಿರುದ್ಧ ಸ್ಟೇಟ್‌ಲೆಸ್** - ಸ್ಟೇಟ್‌ಲೆಸ್: ಜ್ಞಾಪಕವಿಲ್ಲ. ಸ್ಟೇಟ್‌ಫುಲ್: ಸಂಭಾಷಣಾ ಇತಿಹಾಸವನ್ನು ಕಾಯ್ದು ಕೊಳ್ಳುತ್ತದೆ. [ಮಾಡ್ಯೂಲ್ 01](../01-introduction/README.md)

**ಟೋಕನ್ಗಳು** - ಮಾದರಿಗಳು ಪ್ರಕ್ರಿಯೆ ಮಾಡುವ ಮೂಲ ಪಠ್ಯ ಘಟಕಗಳು. ವೆಚ್ಚ ಮತ್ತು ಮಿತಿಗಳನ್ನು ಪ್ರಭಾವಿಸುತ್ತದೆ. [ಮಾಡ್ಯೂಲ್ 01](../01-introduction/README.md)

**ಟೂಲ್ ಚೈನಿಂಗ್** - ನಿರಂತರ ಟೂಲ್ ಕಾರ್ಯಾಚರಣೆ, ಹೊರತಂಡ ಮುಂದಿನ ಕರೆಗಾಗಿ ಮಾಹಿತಿ ನೀಡುತ್ತದೆ. [ಮಾಡ್ಯೂಲ್ 04](../04-tools/README.md)

## LangChain4j ಘಟಕಗಳು

**AiServices** - ಪ್ರಕಾರ-ಸುರಕ್ಷಿತ AI ಸೇವಾ ಇಂಟರ್ಫೇಸ್‌ಗಳನ್ನು ರಚಿಸುತ್ತದೆ.

**OpenAiOfficialChatModel** - OpenAI ಮತ್ತು ಅಜುರ್ OpenAI ಮಾದರಿಗಳಿಗೆ ಏಕೀಕೃತ ಕ್ಲೈಂಟ್.

**OpenAiOfficialEmbeddingModel** - OpenAI ಅಧಿಕೃತ ಕ್ಲೈಂಟ್ ಬಳಸಿ ಎಂಬೆಡ್ಡಿಂಗ್ಸ್ ರಚಿಸುತ್ತದೆ (OpenAI ಮತ್ತು ಅಜುರ್ OpenAI ಎರಡನ್ನೂ ಬೆಂಬಲಿಸುತ್ತದೆ).

**ChatModel** - ಭಾಷಾ ಮಾದರಿಗಳಿಗೆ ಕೇಂದ್ರ ಇಂಟರ್ಫೇಸ್.

**ChatMemory** - ಸಂಭಾಷಣಾ ಇತಿಹಾಸವನ್ನು ಕಾಯ್ದುಕೊಳ್ಳುತ್ತದೆ.

**ContentRetriever** - RAG ಗಾಗಿ ಸಂಬಂಧಿತ ಡಾಕ್ಯುಮೆಂಟ್ ಚಂಕ್‌ಗಳನ್ನು ಕಂಡುಹಿಡಿಯುತ್ತದೆ.

**DocumentSplitter** - ಡಾಕ್ಯುಮೆಂಟ್‌ಗಳನ್ನು ಚಂಕ್‌ಗಳಲ್ಲಿ ವಿಭಜಿಸುತ್ತದೆ.

**EmbeddingModel** - ಪಠ್ಯವನ್ನು ಸಂಖ್ಯಾತ್ಮಕ ವೆಕ್ಟರ್‌ಗಳಾಗಿ ಪರಿವರ್ತಿಸುತ್ತದೆ.

**EmbeddingStore** - ಎಂಬೆಡ್ಡಿಂಗ್ಗಳನ್ನು ಸಂಗ್ರಹಿಸಿ ಮತ್ತು ತಲುಪುತ್ತದೆ.

**MessageWindowChatMemory** - ಇತ್ತೀಚಿನ ಸಂದೇಶಗಳ ಸ್ಲೈಡಿಂಗ್ ವಿಂಡೋವನ್ನು ಕಾಯ್ದುಕೊಳ್ಳುತ್ತದೆ.

**PromptTemplate** - `{{variable}}` ಪ್ಲೇಸ್‌ಹೋಲ್ಡರ್‌ಗಳೊಂದಿಗೆ ಪುನಃಬಳಕೆ ಸಾಧ್ಯವಾದ ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ರಚಿಸುತ್ತದೆ.

**TextSegment** - ಮೆಟಾಡೇಟಾವಿರುವ ಪಠ್ಯದ ತುಂಡು. RAGನಲ್ಲಿ ಬಳಕೆ.

**ToolExecutionRequest** - ಟೂಲಿನ ಕಾರ್ಯಾಚರಣೆಯ ವಿನಂತಿ ಪ್ರತಿನಿಧಿಸುತ್ತದೆ.

**UserMessage / AiMessage / SystemMessage** - ಸಂಭಾಷಣಾ ಸಂದೇಶ ಮಾದರಿಗಳು.

## AI/ML ಸಂಯೋಜನೆಗಳು

**ಫ್ಯೂ-ಶಾಟ್ ಲರ್ನಿಂಗ್** - ಪ್ರಾಂಪ್ಟ್‌ನಲ್ಲಿ ಉದಾಹರಣೆಗಳನ್ನು ನೀಡುವುದು. [ಮಾಡ್ಯೂಲ್ 02](../02-prompt-engineering/README.md)

**ಲಾರ್ಜ್ ಲ್ಯಾಂಗ್ವೇಜ್ ಮಾದರಿ (LLM)** - ವಿಶಾಲವಾದ ಪಠ್ಯ ಡೇಟಾದ ಮೇಲೆ ತರಬೇತಿ ಪಡೆದ AI ಮಾದರಿಗಳು.

**ತರ್ಕ ಪ್ರಯತ್ನ** - ಅನ್ವಯವನ್ನು ಆಳವಾಗಿ ನಿಯಂತ್ರಿಸುವ GPT-5.2 ಪ್ಯಾರಾಮೀಟರ್. [ಮಾಡ್ಯೂಲ್ 02](../02-prompt-engineering/README.md)

**ಟೆಂಪರೆಚರ್** - ಔಟ್‌ಪುಟ್ ಯಾದೃಚ್ಛಿಕತೆಯನ್ನು ನಿಯಂತ್ರಿಸುತ್ತದೆ. ಕಡಿಮೆ=ನಿರ್ಧಿಷ್ಟ, ಹೆಚ್ಚು=ರಚನಾತ್ಮಕ.

**ವೆಕ್ಟರ್ ಡೇಟಾಬೇಸ್** - ಎಂಬೆಡ್ಡಿಂಗ್ಸ್‌ಗಾಗಿ ಪರಿಣತ ಡೇಟಾಬೇಸ್. [ಮಾಡ್ಯೂಲ್ 03](../03-rag/README.md)

**ಶೂನ್ಯ-ಶಾಟ್ ಲರ್ನಿಂಗ್** - ಉದಾಹರಣೆಗಳಿಲ್ಲದೆ ಕಾರ್ಯ ಮಾಡುವಿಕೆ. [ಮಾಡ್ಯೂಲ್ 02](../02-prompt-engineering/README.md)

## ಗಾರ್ಡರೈಲ್ಸ್ - [ಮಾಡ್ಯೂಲ್ 00](../00-quick-start/README.md)

**ಡಿಫೆನ್ಸ್ ಇನ್ ಡೆಪ್ತ್** - ಅಪ್ಲಿಕೇಶನ್-ಮಟ್ಟದ ಗಾರ್ಡರೈಲ್ಸ್ ಮತ್ತು ಪೂರೈಕೆದಾರರ ಸುರಕ್ಷತಾ ಫಿಲ್ಟರ್‌ಗಳನ್ನು ಸಂಯೋಜಿಸುವ ಬಹು-ಮಟ್ಟದ ಭದ್ರತಾ ದೃಷ್ಟಿಕೋನ.

**ಹಾರ್ಡ್ ಬ್ಲಾಕ್** - ಗಂಭೀರ ವಿಷಯ ಉಲ್ಲಂಘನೆಗಳಿಗಾಗಿ ಪೂರೈಕೆದಾರ HTTP 400 ದೋಷವನ್ನು ಹೊಡುತ್ತದೆ.

**ಇನ್‌ಪುಟ್ ಗಾರ್ಡರೈಲ್** - ಲ್ಯಾಂಗ್ಚೈನ್4j ಇಂಟರ್ಫೇಸ್, LLM ಮುಂಚೆ ಬಳಕೆದಾರ ಇನ್‌ಪುಟ್ ಅನ್ನು ತಪಾಸಣೆ ಮಾಡುತ್ತದೆ. ಹಾನಿಕರ ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ಆರಂಭದಲ್ಲಿ ತಡೆಯುವುದರಿಂದ ವೆಚ್ಚ ಮತ್ತು ವಿಳಂಬ ಉಳಿಸುತ್ತದೆ.

**ಇನ್‌ಪುಟ್ ಗಾರ್ಡರೈಲ್‌ರಿಸಲ್ಟ್** - ಗಾರ್ಡರೈಲ್ ಪರಿಶೋಧನೆಗೆ ಮರುಮಾಡುವ ಪ್ರಕಾರ: `success()` ಅಥವಾ `fatal("reason")`.

**ಔಟ್‌ಪುಟ್ ಗಾರ್ಡರೈಲ್** - ಬಳಕೆದಾರರಿಗೆ ಪೋಸ್ಟ್ ಆಗುವ AI ಪ್ರತಿಕ್ರಿಯೆಗಳನ್ನು ಪರಿಶೀಲಿಸುವ ಇಂಟರ್ಫೇಸ್.

**ಪೂರೈಕೆದಾರ ಸುರಕ್ಷತಾ ಫಿಲ್ಟರ್‌ಗಳು** - AI ಪೂರೈಕೆದಾರರಿಂದ (ಉದಾ. GitHub ಮಾದರಿಗಳು) API ಮಟ್ಟದಲ್ಲಿ ಉಲ್ಲಂಘನೆಗಳನ್ನು ಹಿಡಿಯುವ ಒಳನಾಡು ವಿಷಯ ಫಿಲ್ಟರ್‌ಗಳು.

**ಸಾಫ್ಟ್ ರಿಫ್ಯೂಸಲ್** - ಮಾದರಿ ದೋಷವಿಲ್ಲದೆ ವಿನಮ್ರವಾಗಿ ಉತ್ತರ ನೀಡಲು ನಿರಾಕರಿಸುವುದು.

## ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ - [ಮಾಡ್ಯೂಲ್ 02](../02-prompt-engineering/README.md)

**ಚೈನ್-ಆಫ್-ಥಾಟ್** - ಉತ್ತಮ ನಿಖರತೆಗೆ ಹಂತ ಹಂತreasoning.

**ಶ್ರುತಿದೋಷಿತ ಔಟ್‌ಪುಟ್** - ನಿರ್ದಿಷ್ಟ ರೂಪ ಅಥವಾ ರಚನೆ ಜಾರಿಮಾಡುವುದು.

**ಹೈ ಈಗರ್ನೆಸ್** - ಆಳವಾದ ತರ್ಕಕ್ಕಾಗಿ GPT-5.2 ಮಾದರಿ.

**ಲೋ ಈಗರ್ನೆಸ್** - ವೇಗದ ಉತ್ತರಗಳಿಗಾಗಿ GPT-5.2 ಮಾದರಿ.

**ಮಲ್ಟಿ-ಟರ್ನ್ ಸಂಭಾಷಣೆ** - ವಾಣಿಜ್ಯ ವಿನಿಮಯದ ಸಂದರ್ಭವನ್ನು ಕಾಯ್ದುಕೊಳ್ಳುವುದು.

**ಪಾತ್ರಾಧಾರಿತ ಪ್ರಾಂಪ್ಟಿಂಗ್** - ಸಿಸ್ಟಮ್ ಸಂದೇಶಗಳ ಮೂಲಕ ಮಾದರಿಯ ವ್ಯಕ್ತಿತ್ವವನ್ನು ಸೆಟ್ ಮಾಡುವುದು.

**ಸ್ವ-ಪರಿಶೀಲನೆ** - ಮಾದರಿ ತನ್ನ ಔಟ್‌ಪುಟ್ ಅನ್ನು ಮೌಲ್ಯಮಾಪನ ಮಾಡಿ ಸುಧಾರಿಸುತ್ತದೆ.

**ರಚನಾತ್ಮಕ ವಿಶ್ಲೇಷಣೆ** - ಸ್ಥಿರ ಮೌಲ್ಯಮಾಪನ ಚೌಕಟ್ಟಿನೊಂದಿಗೆ.

**ಕಾರ್ಯನಿರ್ವಾಹಣಾ ಮಾದರಿ** - ಯೋಜನೆ → ಅನುಷ್ಠಾನ → ಸಾರಾಂಶ.

## RAG (ಪುನಃಯೋಜಿತ ಉತ್ಪಾದನೆ) - [ಮಾಡ್ಯೂಲ್ 03](../03-rag/README.md)

**ಡಾಕ್ಯುಮೆಂಟ್ ಪ್ರಕ್ರಿಯೆ ಪೈಪ್‌ಲೈನ್** - ಲೋಡ್ → ಚಂಕ್ → ಎಂಬೆಡ್ → ಸಂಗ್ರಹಿಸಿ.

**ಇನ್-ಮೆಮರಿ ಎಂಬೆಡ್ಡಿಂಗ್ ಸ್ಟೋರ್** - ಪರೀಕ್ಷೆಯಿಗಾಗಿ ಅಸ್ಥಿರ ಸಂಗ್ರಹಣೆ.

**RAG** - ಪ್ರತಿಕ್ರಿಯೆಗಳಿಗೆ ಜಾಗೃತತೆಯನ್ನು ಒದಗಿಸಲು ರಿಟ್ರೈವಲ್ ಮತ್ತು ಉತ್ಪಾದನೆಯನ್ನು ಸಂಯೋಜಿಸುತ್ತದೆ.

**ಸಮಾನತೆ ಅಂಕ** - ಅರ್ಥಪೂರ್ಣ ಸಮಾನತೆಯ ಪ್ರಮಾಣ (0-1).

**ಮೂಲ ಉಲ್ಲೇಖ** - ಪಡೆದ ವಿಷಯದ ಮೆಟಾಡೇಟಾ.

## ಏಜೆಂಟುಗಳು ಮತ್ತು ಟೂಲ್ಸ್ - [ಮಾಡ್ಯೂಲ್ 04](../04-tools/README.md)

**@ಟೂಲ್ ಅನೋಟೇಷನ್** - ಜಾವಾ ವಿಧಾನಗಳನ್ನು AI-ಕೆಲೆಯ ಸಾಧನಗಳಾಗಿ ಗುರುತಿಸುವುದು.

**ರಿಏಕ್ಟ್ ಮಾದರಿ** - ತರ್ಕ್ಮಾಡಿ → ಕಾರ್ಯ ನೀಡಿ → ತಿಳಿದುಕೊಳ್ಳಿ → ಪುನರಾವರ್ತನೆ.

**ಸೆಷನ್ ನಿರ್ವಹಣೆ** - ವಿಭಿನ್ನ ಬಳಕೆದಾರರಿಗೆ ವಿಭಿನ್ನ ಸಂದರ್ಭಗಳನ್ನು ನಿರ್ವಹಣೆ.

**ಟೂಲ್** - AI ಏಜೆಂಟ್ ಕರೆ ಮಾಡಬಹುದಾದ ಕಾರ್ಯ.

**ಟೂಲ್ ವಿವರಣೆ** - ಟೂಲಿನ ಉದ್ದೇಶ ಮತ್ತು ಪ್ಯಾರಾಮೀಟರ್‌ಗಳ ದಾಖಲಾತಿ.

## ಏಜೆಂಟಿಕ್ ಮODULE - [ಮಾಡ್ಯೂಲ್ 05](../05-mcp/README.md)

**@ಏಜೆಂಟ್ ಅನೋಟೇಷನ್** - ಘೋಷಣಾತ್ಮಕ ವರ್ತನೆ ವಿವರಣೆಯೊಂದಿಗೆ ಏಜೆಂಟ್‌ಗಳನ್ನು ಗುರುತಿಸುವ ಇಂಟರ್ಫೇಸ್‌ಗಳು.

**ಏಜೆಂಟ್ ಲಿಸನರ್** - `beforeAgentInvocation()` ಮತ್ತು `afterAgentInvocation()` ಮೂಲಕ ಏಜೆಂಟ್ ಕಾರ್ಯ ನಿರೀಕ್ಷಣೆಗೆ ಹೂಕ್.

**ಏಜೆಂಟಿಕ್ ವ್ಯಾಪ್ತಿ** - `outputKey` ಬಳಸಿ ಏಜೆಂಟ್‌ಗಳು ಫಲಿತಾಂಶಗಳನ್ನು ಸಂಗ್ರಹಿಸುವ ಹಂಚಿಕೆಯ ಆತನಕೃತಿ ಜ್ಞಾಪಕ.

**ಏಜೆಂಟಿಕ್ ಸೇವೆಗಳು** - `agentBuilder()` ಮತ್ತು `supervisorBuilder()` ಬಳಸಿ ಏಜೆಂಟ್‌ಗಳ ರಚನೆಗಾಗಿ ಕಾರ್ಖಾನೆ.

**ಶರತ್ತ್ workflow** - ವಿಭಿನ್ನ ವಿಶೇಷಜ್ಞ ಏಜೆಂಟುಗಳಿಗೆ ನಿಯಮಗಳ ಆಧಾರದ ಮೇಲೆ ಮಾರ್ಗ.

**ಮಾನವ-ನಿಮಿತ್ತ ಲೂಪ್** - ಮನುಷ್ಯ სპეციಫಿಕ್ ನ್ಯಾಯಮೂಲ್ಯ ಅಥವಾ ವಿಷಯ ವಿಮర్శನಿಗಾಗಿ ಕಾರ್ಯಪ್ರವಾಹ ಮಾದರಿ.

**langchain4j-agentic** - ಘೋಷಣಾತ್ಮಕ ಏಜೆಂಟ್ ನಿರ್ಮಾಣಕ್ಕೆ ಮೇವೆನ್ ಅವಲಂಬನೆ (ಪ್ರಯೋಗಾತ್ಮಕ).

**ಲೂಪ್ ವರ್ಕ್‌ಫ್ಲೋ** - ಗುಣಮಟ್ಟ ಅಂಕ ≥ 0.8 ಮುಂತಾದ ನಿಯಮ ಪೂರೈಸುವ ತನಕ ಏಜೆಂಟ್ ಕಾರ್ಯ ನಿರ್ವಹಣೆ ಪುನರಾವರ್ತಿಸುವುದು.

**outputKey** - ಏಜೆಂಟ್ ಅನೋಟೇಷನ್ ಪ್ಯಾರಾಮೀಟರ್, ಏಜೆಂಟಿಕ್ ವ್ಯಾಪ್ತಿಯಲ್ಲಿ ಫಲಿತಾಂಶ ಹುರಿಯುವ ಸ್ಥಳ.

**ಪ್ಯಾರೆಲ್ಲಲ್ ವರ್ಕ್‌ಫ್ಲೋ** - ಸ್ವತಂತ್ರ ಕಾರ್ಯಗಳಿಗೆ ಹಲವಾರು ಏಜೆಂಟ್‌ಗಳನ್ನು ಒಂದೇ ಅವಧಿಯಲ್ಲಿ ಚಲಾಯಿಸುವುದು.

**ಪ್ರತಿಕ್ರಿಯೆ ತಂತ್ರ** - ಮೇಲ್ವಿಚಾರಕ ಅಂತಿಮ ಉತ್ತರವನ್ನು ರೂಪಿಸುವ ವಿಧಾನ: ಅಂತಿಮ, ಸಾರಾಂಶ ಅಥವಾ ಅಂಕಿತ.

**ಕ್ರಮಬದ್ಧ ವರ್ಕ್‌ಫ್ಲೋ** - ಕಾರ್ಯ ಕ್ರಮ ಸರಿಯಲ್ಲಿದ್ಯವಾಗಿ ನಿರ್ವಹಣೆ, ಹೊರತಂಡ ಮುಂದಿನ ಹಂತಕ್ಕೆ ಸಾಗುತ್ತದೆ.

**ಮೇಲ್ವಿಚಾರಕ ಏಜೆಂಟ್ ಮಾದರಿ** - ಮೇಲ್ವಿಚಾರಕ LLM ಡೈನಾಮಿಕ್ ಉಪ ಏಜೆಂಟ್‌ಗಳನ್ನು ಕರೆಮಾಡಲು ತೀರ್ಮಾನಿಸುವ ಪ್ರಗತಿಯಾದ ಏಜೆಂಟಿಕ್ ಮಾದರಿ.

## ಮಾಡೆಲ್ ಕಾಂಟೆಕ್ಸ್ಟ್ ಪ್ರೋಟೋಕಾಲ್ (MCP) - [ಮಾಡ್ಯೂಲ್ 05](../05-mcp/README.md)

**langchain4j-mcp** - LangChain4jನಲ್ಲಿ MCP ಏಕೀಕರಣಕ್ಕಾಗಿ ಮೇವೆನ್ ಅವಲಂಬನೆ.

**MCP** - ಮಾಡೆಲ್ ಕಾಂಟೆಕ್ಸ್ಟ್ ಪ್ರೋಟೋಕಾಲ್: AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಹೊರಗಿನ ಟೂಲ್ಗಳಿಗೆ ಸಂಪರ್ಕಿಸುವ ಮಾನದಂಡ. ಒಮ್ಮೆ ರಚಿಸಿ ಎಲ್ಲೆಡೆ ಬಳಸಿ.

**MCP ಕ್ಲೈಂಟ್** - MCP ಸರ್ವರ್‌ಗಳಿಗೆ ಸಂಪರ್ಕ ಹೊಂದಿ ಟೂಲ್ಗಳನ್ನು ಪತ್ತೆಮಾಡಿ ಬಳಕೆಗೆ ಸುಧಾರಿಸುವ ಅಪ್ಲಿಕೇಶನ್.

**MCP ಸರ್ವರ್** - ಸ್ಪಷ್ಟ ವಿವರಣೆ ಮತ್ತು ಪ್ಯಾರಾಮೀಟರ್ schemas ಮೂಲಕ ಟೂಲ್ಗಳನ್ನು MCP ಮೂಲಕ ಅನಾವರಣ ಮಾಡುವ ಸೇವೆ.

**McpToolProvider** - LangChain4j ಘಟಕ, MCP ಟೂಲ್ಗಳನ್ನು AI ಸೇವೆಗಳು ಮತ್ತು ಏಜೆಂಟ್‌ಗಳಲ್ಲಿ ಬಳಸಿ ಮುಡುಪುಗೊಳಿಸುತ್ತದೆ.

**McpTransport** - MCP ಸಂಪರ್ಕಕ್ಕಾಗಿ ಇಂಟರ್ಫೇಸ್. ಅನುಷ್ಠಾನಗಳು Stdio ಮತ್ತು HTTP ಒಳಗೊಂಡಿವೆ.

**Stdio ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್** - ಸ್ಥಳೀಯ ಪ್ರಕ್ರಿಯೆಯ ಸಂಪರ್ಕ, stdin/stdout ಮೂಲಕ. ಫೈಲ್‌ಸಿಸ್ಟಮ್ ಪ್ರವೇಶ ಅಥವಾ ಕಮಾಂಡ್‌ ಲೈನ್ ಟೂಲ್ಗಳಿಗಾಗಿ ಲಾಭಕಾರಿ.

**StdioMcpTransport** - LangChain4j ಅನುಷ್ಠಾನ, subprocess ಆಗಿ MCP ಸರ್ವರ್ ಅನ್ನು ಪ್ರಾರಂಭಿಸುವುದು.

**ಟೂಲ್ ಡಿಸ್ಕವರಿ** - ಕ್ಲೈಂಟ್ ಸರ್ವರ್‌ಗೆ ಟೂಲ್ಗಳಿಗಾಗಿ ವಿವರಣೆ ಮತ್ತು schemas ಜೊತೆ ಪ್ರಶ್ನೆಗಳು.
