## ಮೂಲ ಕಲ್ಪನೆಗಳು

**AI ಏಜೆಂಟ್** - ಸ್ವಾಯತ್ತವಾಗಿ ಯುಕ್ತಿ ಮಾಡಿ ಕ್ರಮ ಕೈಗೊಳ್ಳುವ AI ಬಳಸಿ ವ್ಯವಸ್ಥೆ. [ಘಟಕ 04](../04-tools/README.md)

**ಚೈನ್** - ಹೊರತುಪಡಿಸಿದ ಪ್ರವೃತ್ತಿಯನ್ನು ಮುಂದಿನ ಹಂತಕ್ಕೆ ಜೋಡಿಸುವ ಕಾರ್ಯಗಳ ಸರಣ knife.

**ಚಂಕಿಂಗ್** - ದಾಖಲೆಗಳನ್ನು ಸಣ್ಣ ತುಂಡುಗಳಾಗಿ ಭಾಗಿಸುವುದು. ಸಾಮಾನ್ಯ: 300-500 ಸಂಕೇತಗಳು ಮುಂಭಾಗದ ಅಂಶಗಳೊಂದಿಗೆ. [ಘಟಕ 03](../03-rag/README.md)

**ಸಂದರ್ಭ ವಿಂಡೊ** - ಮಾದರಿ ಪ್ರಕ್ರಿಯೆ ಮಾಡಬಹುದಾದ ಗರಿಷ್ಠ ಸಂಕೇತಗಳು. GPT-5.2: 400K ಸಂಕೇತಗಳು.

**ಎಂಬೆಡ್ಡಿಂಗ್‍ಗಳು** - ಪಠ್ಯದ ಅರ್ಥವನ್ನು ಸೂಚಿಸುವ ಸಂಖ್ಯೆಮೂಲಕ ವೇಕ್ಟರ್ ಗಳು. [ಘಟಕ 03](../03-rag/README.md)

**ಫಂಕ್ಶನ್ ಕರೆ** - ಮಾದರಿ ಕಡೆಯು ಹೊರಗಿನ ಕಾರ್ಯಗಳನ್ನು ಕರೆ ಮಾಡಲು ರಚನೆಗೊಳಿಸಿದ ವಿನಂತಿಗಳನ್ನು ರಚಿಸುತ್ತದೆ. [ಘಟಕ 04](../04-tools/README.md)

**ಹ್ಯಾಲ್ಯೂಸಿನೇಷನ್** - ಮಾದರಿಗಳು ತಪ್ಪಾದ ಆದರೆ ಸಾಧ್ಯವಾದ ಮಾಹಿತಿ ರಚಿಸುವಾಗ.

**ಪ್ರಾಂಪ್ಟ್** - ಭಾಷಾ ಮಾದರಿಗೆ ಪಠ್ಯ ಇನ್ಪುಟ್. [ಘಟಕ 02](../02-prompt-engineering/README.md)

**ಅರ್ಥಪೂರ್ಣ ಹುಡುಕು** - ಕೀವರ್ಡ್‌ಗಳು ಅಲ್ಲದೆ ಎಂಬೆಡ್ಡಿಂಗ್‍ಗಳ ಮೂಲಕ ಅರ್ಥದಿಂದ ಹುಡುಕುವುದು. [ಘಟಕ 03](../03-rag/README.md)

**ಸ್ಥಿತಿಸಂಪನ್ನ ಹಾಗೂ ಸ್ಥಿತಿಲೇಷ** - ಸ್ಥಿತಿಲೇಷ: ನೆನಪಿಲ್ಲದೆ. ಸ್ಥಿತಿಸಂಪನ್ನ: ಸಂಭಾಷಣಾ ಇತಿಹಾಸವನ್ನು ಕಾಯ್ದಿರಿಸುತ್ತದೆ. [ಘಟಕ 01](../01-introduction/README.md)

**ಸಂಕೇತಗಳು** - ಮಾದರಿಗಳು ಪ್ರಕ್ರಿಯೆ ಮಾಡುವ ಮೂಲ ಪಠ್ಯ ಘಟಕಗಳು. ವೆಚ್ಚ ಮತ್ತು ಮಿತಿ ಮೇಲೆ ಪ್ರಭಾವ ಬೀರುತ್ತವೆ. [ಘಟಕ 01](../01-introduction/README.md)

**ಸಾಧನದ ಸರಣಿ** - ಹೊರತುಪಡಿಸಿದ ನಿರ್ಗಮನ ಮುಂದಿನ ಕರೆಗೆ ಮಾಹಿತಿ ನೀಡುವ ಕ್ರಮಬದ್ದ ಸಾಧನ ಕಾರ್ಯಾಚರಣೆ. [ಘಟಕ 04](../04-tools/README.md)

## LangChain4j ಘಟಕಗಳು

**AiServices** - ಪ್ರಕಾರ-ಸೇಫ್ AI ಸೇವಾ ಇಂಟರ್ಫೇ್ಸ್ಗಳನ್ನು ಸೃಷ್ಟಿಸುತ್ತದೆ.

**OpenAiOfficialChatModel** - OpenAI ಮತ್ತು Azure OpenAI ಮಾದರಿಗಳಿಗಾಗಿ ಏಕೀಕೃತ ಗ್ರಾಹಕ.

**OpenAiOfficialEmbeddingModel** - OpenAI ಅಧಿಕೃತ ಗ್ರಾಹಕ ಬಳಕೆಮಾಡಿ ಎಂಬೆಡ್ಡಿಂಗ್‍ಗಳನ್ನು ಸೃಷ್ಟಿಸುತ್ತದೆ (OpenAI ಮತ್ತು Azure OpenAI ಎರಡಿಗಾಗಿಯೂ ಬೆಂಬಲ).

**ChatModel** - ಭಾಷಾ ಮಾದರಿಗಳ ಮಾಲಿಕ ಇಂಟರ್ಫೇಸ್.

**ChatMemory** - ಸಂಭಾಷಣಾ ಇತಿಹಾಸವನ್ನು ಕಾಯ್ದಿರಿಸುತ್ತದೆ.

**ContentRetriever** - RAG ಗೆ ಸಂಬಂಧಿಸಿದ ದಾಖಲೆಯ ತುಂಡುಗಳನ್ನು ಹುಡುಕುತ್ತದೆ.

**DocumentSplitter** - ದಾಖಲೆಯನ್ನು ತುಂಡುಗಳಾಗಿ ವಿಭಾಗಿಸುತ್ತದೆ.

**EmbeddingModel** - ಪಠ್ಯವನ್ನು ಸಂಖ್ಯೆಮೂಲಕ ವೇಕ್ಟರ್‌ಗಳಾಗಿ ಪರಿವರ್ತಿಸುತ್ತದೆ.

**EmbeddingStore** - ಎಂಬೆಡ್ಡಿಂಗ್‍ಗಳನ್ನು ಸಂಗ್ರಹಿಸಿ ದಾಖಲಿಸುವುದು.

**MessageWindowChatMemory** - ಇತ್ತೀಚಿನ ಸಂದೇಶಗಳ ಸ್ಲೈಡಿಂಗ್ ವಿಂಡೋವನ್ನು ಕಾಯ್ದಿರಿಸುತ್ತದೆ.

**PromptTemplate** - `{{variable}}` ಪ್ಲೇಸ್‌ಹೋಲ್ಡರ್ಗಳನ್ನು ಹೊಂದಿರುವ ಪುನಃಪ್ರಯೋಜನೀಯ ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ರಚಿಸುವುದು.

**TextSegment** - ಮೆಟಾಡೇಟಾ ಜೊತೆಗೆ ಪಠ್ಯ ತುಣುಕು. RAG ನಲ್ಲಿ ಬಳಸಲಾಗುತ್ತದೆ.

**ToolExecutionRequest** - ಸಾಧನ ಕಾರ್ಯಾಚರಣಾ ವಿನಂತಿಯನ್ನು ಪ್ರತಿನಿಧಿಸುತ್ತದೆ.

**UserMessage / AiMessage / SystemMessage** - ಸಂಭಾಷಣೆ ಸಂದೇಶ ಪ್ರಕಾರಗಳು.

## AI/ML ಕಲ್ಪನೆಗಳು

**ನೀವು-ಶಾಟ್ ಕಲಿಕೆ** - ಪ್ರಾಂಪ್ಟ್‌ನಲ್ಲಿ ಉದಾಹರಣೆಗಳನ್ನು ನೀಡುವುದು. [ಘಟಕ 02](../02-prompt-engineering/README.md)

**ದೊಡ್ಡ ಭಾಷಾ ಮಾದರಿ (LLM)** - ವಿವಿದ ಪಠ್ಯ ಡೇಟಾದ ಮೇಲೆ ತರಬೇತುಗೊಂಡ AI ಮಾದರಿಗಳು.

**ಯುಕ್ತಿವಾದ ಶಿಷ್ಟತೆ** - GPT-5.2 ನಿಯಂತ್ರಣ ಬಸಿಸಿ ಯೋಚನಾ ಆಳ.

**ತಾಪಮಾನ** - ನಿರ್ಗಮನದ ಯಾದೃಚ್ಛಿಕತೆಯನ್ನು ನಿಯಂತ್ರಿಸುತ್ತದೆ. ಕಡಿಮೆ=ನಿರ್ಣಾಯಕ, ಹೆಚ್ಚು=ಸೃಜನಾತ್ಮಕ.

**ವೇಕ್ಟರ್ ಡೇಟಾಬೇಸ್** - ಎಂಬೆಡ್ಡಿಂಗ್‍ಗಳಿಗಾಗಿ ವಿಶೇಷ ಡೇಟಾಬೇಸ್. [ಘಟಕ 03](../03-rag/README.md)

**ಶೂನ್ಯ-ಶಾಟ್ ಕಲಿಕೆ** - ಉದಾಹರಣೆಗಳಿಲ್ಲದೆ ಕಾರ್ಯಗಳನ್ನು ನಿರ್ವಹಿಸುವುದು. [ಘಟಕ 02](../02-prompt-engineering/README.md)

## ರಕ್ಷಕ ಬಾಗಿಲುಗಳು - [ಘಟಕ 00](../00-quick-start/README.md)

**ಆಳದಲ್ಲಿ ರಕ್ಷಣಾ ಕ್ರಮ** - ಅಪ್ಲಿಕೇಶನ್-ಮಟ್ಟದ ರಕ್ಷಕ ಬಾಗಿಲುಗಳು ಮತ್ತು ಒದಗಿಸುವವರ ಭದ್ರತಾ ಫಿಲ್ಟರ್‌ಗಳ ಸಂಯೋಜಿತ ಬಹು-ಪರ್ತ ಭದ್ರತಾ ವಿಧಾನ.

**ಹಾರ್ಡ್ ಬ್ಲಾಕ್** - ಗಂಭೀರ ವಿಷಯ ಉಲ್ಲಂಘನೆಗಳಿಗೆ ಒದಗಿಸುವವರು HTTP 400 ದೋಷ ತಳ್ಳುತ್ತಾರೆ.

**InputGuardrail** - ಬಳಸುವವರ ಇನ್ಪುಟ್ ಮಾನ್ಯಗೊಳಿಸುವ LangChain4j ಇಂಟರ್ಫೇಸ್. ದೋಷಕರ ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ಬೇಗನೆ ತಡೆಯುವುದರಿಂದ ವೆಚ್ಚ ಮತ್ತು ವಿಳಂಬ ಉಳಿಸುತ್ತದೆ.

**InputGuardrailResult** - ಗಾರ್ಡ್ರೈಲ್ ಮಾನ್ಯತೆಗಾಗಿ ಮರಳುವ ಪ್ರಕಾರ: `success()` ಅಥವಾ `fatal("reason")`.

**OutputGuardrail** - ಬಳಕೆದಾರರಿಗೆ ಮರಳಿಸುವ ಮುನ್ನ AI ಉತ್ತರಗಳನ್ನು ಪರಿಶೀಲಿಸುವ ಇಂಟರ್ಫೇಸ್.

**ಒದಗಿಸುವವರು ಭದ್ರತಾ ಫಿಲ್ಟರ್‌ಗಳು** - AI ಒದಗಿಸುವವರಿಂದ ಮೂಲಭೂತ ವಿಷಯ ಫಿಲ್ಟರ್‌ಗಳು (ಉದಾ: GitHub ಮಾದರಿಗಳು) API ಹಂತದಲ್ಲಿ ಉಲ್ಲಂಘನೆಯನ್ನು ಹಿಡಿಯುತ್ತವೆ.

**ಸೂಮಾರಾಜಿತ ನಿರಾಕರಣೆ** - ಮಾದರಿ ದಯವಿಟ್ಟು ಉತ್ತರಿಸುವುದನ್ನು ತಿರಸ್ಕರಿಸುತ್ತದೆ, ದೋಷವನ್ನಿಲ್ಲದೆ.

## ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್ - [ಘಟಕ 02](../02-prompt-engineering/README.md)

**ಚೈನ್-ಆಫ್-ಥಾಟ್** - ಉತ್ತಮ ಸರಿಯಾದ ಉತ್ತರಿಗಾಗಿ ಹಂತ-ಹಂತ ನಿಯುಕ್ತಿ.

**ಬಂಧಿತ_output** - ನಿರ್ದಿಷ್ಟ ರೂಪರೇಖೆ ಅಥವಾ ರಚನೆಯನ್ನು ಬಲವಂತಪಡಿಸುವುದು.

**ಹೆಚ್ಚು ಆಸಕ್ತಿ** - ಸಮಗ್ರ ಯುಕ್ತಿವಾದಕ್ಕೆ GPT-5.2 ಮಾದರಿ ಪರಿಕಲ್ಪನೆ.

**ಕಡಿಮೆ ಆಸಕ್ತಿ** - ಬೇಗನೆ ಉತ್ತರಗಳಿಗೆ GPT-5.2 ಮಾದರಿ ಪರಿಕಲ್ಪನೆ.

**ಬಹು-ತೊರ್ನ್ ಸಂಭಾಷಣೆ** - ವಿನಿಮಯಗಳ ಮೂಲಕ ಸಂಧರ್ಭವನ್ನು ಕಾಯ್ದಿರಿಸುವುದು.

**ಪಾತ್ರ-ಆಧಾರಿತ ಪ್ರಾಂಪ್ಟಿಂಗ್** - ವ್ಯವಸ್ಥಾ ಸಂದೇಶಗಳ ಮೂಲಕ ಮಾದರಿ ವ್ಯಕ್ತಿತ್ವವನ್ನು ನಿರ್ಧರಿಸುವುದು.

**ಸ್ವ-ಪರಿಶೀಲನೆ** - ಮಾದರಿ ತನ್ನ ನಿರ್ಗಮನವನ್ನು ಮೌಲ್ಯಮಾಪನ ಮಾಡುತ್ತಾ ಸುಧಾರಿಸುತ್ತದೆ.

**ರಚನೆಯ ವಿಶ್ಲೇಷಣೆ** - ಸ್ಥಿರ ಮೌಲ್ಯಮಾಪನ ಚೌಕಟ್ಟನ್ನು ಅನುಸರಿಸುವುದು.

**ಕಾರ್ಯಾಚರಣಾ ಮಾದರಿ** - ಯೋಜನೆ → ಕಾರ್ಯಾಚರಣೆ → ಸಾರಾಂಶ.

## RAG (ಪುನಃಪ್ರಾಪ್ತಿದ ನೆರವಿನ ಉತ್ಪಾದನೆ) - [ಘಟಕ 03](../03-rag/README.md)

**ದಾಖಲೆ ಪ್ರಕ್ರಿಯೆ ಪೈಪ್ಲೈನ್** - ಲೋಡ್ → ಚಂಕ್ → ಎಂಬೆಡ್ → ಸಂಗ್ರಹ.

**ಸ್ಮರಣಲ್ಲಿ నిలದಿರುವ ಎಂಬೆಡ್ಡಿಂಗ್ ಅಂಗಡಿ** - ತಾತ್ಕಾಲಿಕ ಸಂಗ್ರಹಣೆ ಪರೀಕ್ಷೆಗಾಗಿ.

**RAG** - ಉತ್ತರಗಳನ್ನು ಭೂಮಿಗೂಡಿಸುವಂತೆ ಪುನಃಪ್ರಾಪ್ತಿಯೊಂದಿಗೆ ಉತ್ಪಾದನೆಯನ್ನು ಸಂಯೋಜಿಸುತ್ತದೆ.

**ಸಮಾನತೆ ಅಂಕೆ** - ಅರ್ಥಪೂರ್ಣ ಸಮಾನತೆಯ ಅಳತೆ (0-1).

**ಮೂಲ ಉಲ್ಲೇಖ** - ಪುನಃಪ್ರಾಪ್ತಿಗೊಂಡ ವಿಷಯದ ಮೆಟಾಡೇಟಾ.

## ಏಜೆಂಟ್ಗಳು ಮತ್ತು ಸಾಧನಗಳು - [ಘಟಕ 04](../04-tools/README.md)

**@Tool ಅಂಕನ** - ಜಾವಾ ಮಾಡೆತೊಡಬುಗಳನ್ನು AI ಕರೆಹಾಕಬಹುದಾದ ಸಾಧನಗಳೆಂದು ಗುರುತಿಸುವುದು.

**ReAct ಮಾದರಿ** - ಯುಕ್ತಿ → ಕ್ರಮ → ಅವಲೋಕನೆ → ಪುನರಾವೃತ್ತಿ.

**ಸೆಶನ್ ನಿರ್ವಹಣೆ** - ವಿಭಿನ್ನ ಬಳಕೆದಾರರಿಗಾಗಿ ಪ್ರತ್ಯೇಕ ಸಂದರ್ಭಗಳು.

**ಸಾಧನ** - AI ಏಜೆಂಟ್ ಕರೆ ಮಾಡಬಹುದಾದ ಕಾರ್ಯ.

**ಸಾಧನ ವರ್ಣನೆ** - ಸಾಧನ ಉದ್ದೇಶ ಮತ್ತು ನಿಯತಾಂಕಗಳ ಡಾಕ್ಯುಮೆಂಟೇಷನ್.

## ಏಜೆಂಟಿಕ್ ಘಟಕ - [ಘಟಕ 05](../05-mcp/README.md)

**@Agent ಅಂಕನ** - ಡಿಕ್ಲಾರೆಟಿವ್ ನಡಚಳಿಕೆ ವ್ಯಾಖ್ಯಾನದಲ್ಲಿ ಇಂಟರ್ಫೇಸ್‌ಗಳನ್ನು AI ಏಜೆಂಟ್‌ಗಳಾಗಿ ಗುರುತಿಸುವುದು.

**ಏಜೆಂಟ್ ಶ್ರೋತೆ** - `beforeAgentInvocation()` ಮತ್ತು `afterAgentInvocation()` ಮೂಲಕ ಏಜೆಂಟ್ ಕಾರ್ಯಾಚರಣೆಯನ್ನು ಗಮನಿಸುವ ಹೂಕ್.

**ಏಜೆಂಟಿಕ್ ವ್ಯಾಪ್ತಿ** - ಏಜೆಂಟ್ಗಳು `outputKey` ಬಳಸಿ ಫಲಿತಾಂಶಗಳನ್ನು ಸಂಗ್ರಹಿಸಿ ನಂತರದ ಏಜೆಂಟ್ ಗಳು ಉಪಯೋಗಿಸುವ ಸಹಬಳಕೆ ನೆನಪು.

**AgenticServices** - `agentBuilder()` ಮತ್ತು `supervisorBuilder()` ಬಳಸಿಕೊಂಡು ಏಜೆಂಟ್ಗಳನ್ನು ಸೃಷ್ಟಿಸುವ ಕಾರ್ಖಾನೆ.

**ಶರತು ಆಧಾರಿತ ಕಾರ್ಯಪ್ರವಾಹ** - ವಿಭಿನ್ನ ತಜ್ಞ ಏಜೆಂಟ್‌ಗಳಿಗೆ ಶರತ್ತು ಆಧರಿಸಿ ಮಾರ್ಗಸೂಚಿ.

**ಮಾನವ-ಮಧ್ಯಸ್ಥ** - ಅನುಮತಿ ಅಥವಾ ವಿಷಯ ಪರಿಶೀಲನೆಗಾಗಿ ಮಾನವ ತಪಾಸಣಾ ಬિંદುಗಳನ್ನು ಸೇರಿಸುವ ಕಾರ್ಯಪ್ರವಾಹ ಮಾದರಿ.

**langchain4j-agentic** - ಡಿಕ್ಲಾರೆಟಿವ್ ಏಜೆಂಟ್ ನಿರ್ಮಾಣಕ್ಕಾಗಿ ಮೆವೆನ್ ಅವಲಂಬನೆ (ಪ್ರಯೋಗಾತ್ಮಕ).

**ಲೂಪ್ ಕಾರ್ಯಪ್ರವಾಹ** - ಗುಣಮಟ್ಟ ಅಂಕ ≥ 0.8 ಶರತು ಪೂರೈಸುವವರೆಗೆ ಏಜೆಂಟ್ ಕಾರ್ಯಾಚರಣೆ ಪುನರಾವೃತ್ತಿ.

**outputKey** - ಏಜೆಂಟ್ ಅಂಕನ ನಿಯತಾಂಕ, Agentic ವ್ಯಾಪ್ತಿಯಲ್ಲಿ ಫಲಿತಾಂಶ ಸಂಗ್ರಹಿಸುವ ಸ್ಥಳವನ್ನು ಸೂಚಿಸುವುದು.

**ಸಮಾನಾಂತರ ಕಾರ್ಯಪ್ರವಾಹ** - ಸ್ವತಂತ್ರ ಕಾರ್ಯಗಳಿಗೆ ಬಹು ಏಜೆಂಟ್‌ಗಳನ್ನು ಒಂದೇ ಸಮಯದಲ್ಲಿ ಚಲಾಯಿಸುವುದು.

**ಪ್ರತಿಕ್ರಿಯೆ ತಂತ್ರ** - ಮೇಲ್ವಿಚಾರಕ ಅಂತಿಮ ಉತ್ತರವನ್ನು ರೂಪಿಸುವ ವಿಧಾನ: ಕೊನೆ, ಸಾರಾಂಶ, ಅಥವಾ ಅಂಕಿತ.

**ಕ್ರಮಬದ್ಧ ಕಾರ್ಯಪ್ರವಾಹ** - ಹಂತ ಕ್ರಮದಲ್ಲಿರುವ ಏಜೆಂಟ್‌ಗಳನ್ನು ಕಾರ್ಯಗತಗೊಳಿಸುವುದು, ಹೊರತುಪಡಿಸಿದ ನಿರ್ಗಮವು ಮುಂದಿನ ಹಂತಕ್ಕೆ ನೇರವಾಗಿ ಸಾಗುತ್ತದೆ.

**ಮೇಲ್ವಿಚಾರಕ ಏಜೆಂಟ್ ಮಾದರಿ** - ಮೇಲ್ವಿಚಾರಕ LLM ಜ dinamically ಉಪ-ಏಜೆಂಟ್‌ಗಳನ್ನು ಕರೆ ಮಾಡಲು ನಿರ್ಧರಿಸುವ ಸುಧಾರಿತ ಏಜೆಂಟಿಕ್ ಮಾದರಿ.

## ಮಾದರಿ ಸಂದರ್ಭ ಪ್ರೋಟೋಕಾಲ್ (MCP) - [ಘಟಕ 05](../05-mcp/README.md)

**langchain4j-mcp** - LangChain4j ನಲ್ಲಿ MCP ಏಕರೂಪತೆಗಾಗಿ ಮೆವೆನ್ ಅವಲಂಬನೆ.

**MCP** - ಮಾದರಿ ಸಂದರ್ಭ ಪ್ರೋಟೋಕಾಲ್: AI ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಹೊರಲೆ ಸಾಧನಗಳಿಗೆ ಸಂಪರ್ಕಿಸುವ ಮಾನದಂಡ. ಒಮ್ಮೆ ನಿರ್ಮಿಸಿ ಎಲ್ಲೆಡೆ ಬಳಸಿ.

**MCP ಗ್ರಾಹಕ** - MCP ಸರ್ವರ್‌ಗಳಿಗೆ ಸಂಪರ್ಕಿಸಿ ಸಾಧನಗಳನ್ನು ಹುಡುಕಲು ಮತ್ತು ಬಳಸಲು ಅಪ್ಲಿಕೇಶನ್.

**MCP ಸರ್ವರ್** - ಸ್ಪಷ್ಟ ವರ್ಣನೆಗಳು ಮತ್ತು ನಿಯತಾಂಕ ಸ್ಖೇಮಗಳೊಂದಿಗೆ MCP ಮೂಲಕ ಸಾಧನಗಳನ್ನು ಹೊರಹಾಕುವ ಸೇವೆ.

**McpToolProvider** - MCP ಸಾಧನಗಳನ್ನು AI ಸೇವೆಗಳು ಮತ್ತು ಏಜೆಂಟ್‌ಗಳಲ್ಲಿ ಬಳಸುವಂತೆ ಹೊಡೆತ ಮಾಡುವ LangChain4j ಘಟಕ.

**McpTransport** - MCP ಸಂವಹನಿಗಾಗಿ ಇಂಟರ್ಫೇಸ್. ಪರಿಣಾಮಕಾರಿ ಕಾರ್ಯಕ್ಷಮತೆ ಗಳಿಗೆ Stdio ಮತ್ತು HTTP ಸೇರಿವೆ.

**Stdio ಸಾರಿಗೆ** - ಲೋಕಲ್ ಪ್ರಕ್ರಿಯೆ ಸಾರಿಗೆ, stdin/stdout ಮೂಲಕ. ಕಡತ ಸಂಪಾದನೆ ಮತ್ತು ಕಮಾಂಡ್-ಲೈನ್ ಸಾಧನಗಳಿಗೆ ಉಪಯುಕ್ತ.

**StdioMcpTransport** - LangChain4j ಅಳವಡಿಕೆ, MCP ಸರ್ವರ್ ಅನ್ನು ಉಪಪ್ರಕ್ರಿಯೆಯಾಗಿ ಪ್ರಾರಂಭಿಸುವುದು.

**ಸಾಧನ ಹುಡುಕಾಟ** - ಗ್ರಾಹಕವು ಲಭ್ಯ ಸಾಧನಗಳಿಗೆ ವಿವರಣೆ ಮತ್ತು ಸ್ಖೇಮಗಳೊಂದಿಗೆ ಕೇಳಿಕೆ ಮಾಡುವುದು.
