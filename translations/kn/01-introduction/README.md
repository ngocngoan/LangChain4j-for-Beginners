## ನೀವು ಕಲಿಯಲಿರುವುದು

ನೀವು ತ್ವರಿತ ಪ್ರಾರಂಭವನ್ನು ಪೂರ್ಣಗೊಳಿಸಿದರೆ, ನೀವು ಸೂಚನೆಗಳನ್ನು ಕಳುಹಿಸುವುದು ಮತ್ತು ಪ್ರತಿಕ್ರಿಯೆಗಳು ಪಡೆಯುವುದು ಹೇಗೆ ಎಂಬುದನ್ನು ನೋಡಿದ್ದೀರಿ. ಅದು ಆಧಾರವಾಗಿದ್ದು, ವಾಸ್ತವಿಕ ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಗೆ ಇನ್ನೂ ಹೆಚ್ಚುವರಿ ಅಗತ್ಯವಿದೆ. ಈ ಮೊಡ್ಯೂಲ್ ನೀವು ಸಂಭಾಷಣೆಯ AI ಅನ್ನು ನಿರ್ಮಿಸುವ ವಿಧಾನವನ್ನು ಕಲಿಸುತ್ತದೆ, ಇದು ಸಂದರ್ಭವನ್ನು ನೆನಸಿದೆ ಮತ್ತು ಸ್ಥಿತಿಯನ್ನು ಕಾಪಾಡುತ್ತದೆ -ಒಂದು ಟನೇ ಡೆಮೊ ಮತ್ತು ಉತ್ಪಾದನಾ ಸಿದ್ಧ ಅಪ್ಲಿಕೇಶನ್ ನಡುವೆ ಇರುವ ವ್ಯತ್ಯಾಸ.

ನಾವು ಈ ಮಾರ್ಗದರ್ಶನದಲ್ಲಿ Azure OpenAI ಯ GPT-5.2 ಅನ್ನು ಬಳಸುತ್ತೇವೆ ಏಕೆಂದರೆ ಅದರ ಉನ್ನತ ತರ್ಕ ಸಾಮರ್ಥ್ಯಗಳು ವಿವಿಧ ಮಾದರಿಗಳ ವರ್ತನೆಗಳನ್ನು ಹೆಚ್ಚು ಸ್ಪಷ್ಟವಾಗಿ ತೋರಿಸುತ್ತವೆ. ನೀವು ಸ್ಮರಣೆಯನ್ನು ಸೇರಿಸಿದಾಗ, ವ್ಯತ್ಯಾಸ ಸ್ಪಷ್ಟವಾಗುತ್ತದೆ. ಇದು ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಗೆ ಪ್ರತಿಯೊಂದು ಘಟಕ ಏನು ತರಬಹುದು ಎಂಬುದನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು ಸುಲಭಗೊಳಿಸುತ್ತದೆ.

ನೀವು ಎರಡು ಮಾದರಿಗಳನ್ನು ಪ್ರದರ್ಶಿಸುವ ಅಪ್ಲಿಕೇಶನ್ ಒಂದನ್ನು ನಿರ್ಮಿಸುವಿರಿ:

**ಸ್ಟೇಟ್‌ಲೆಸ್ ಚಾಟ್** - ಪ್ರತಿ ವಿನಂತಿ ಸ್ವತಂತ್ರವಾಗಿರುತ್ತದೆ. ಮಾದರಿ ಹಿಂದಿನ ಸಂದೇಶಗಳನ್ನು ನೆನಸುವುದಿಲ್ಲ. ಇದು ನೀವು ತ್ವರಿತ ಪ್ರಾರಂಭದಲ್ಲಿ ಬಳಸಿದ ಮಾದರಿಯಾಗಿದೆ.

**ಸ್ಟೇಟ್‌ಫುಲ್ ಸಂಭಾಷಣೆ** - ಪ್ರತಿ ವಿನಂತಿಯಲ್ಲಿ ಸಂಭಾಷಣಾ ಇತಿಹಾಸ ಸೇರಿದೆ. ಮಾದರಿ ಅನೇಕ ತಿರುಗಾಟಗಳ ನಡುವೆಯೇ ಸಂದರ್ಭವನ್ನು ಕಾಪಾಡುತ್ತದೆ. ಇದನ್ನು ಉತ್ಪಾದನಾ ಅಪ್ಲಿಕೇಶನ್‌ಗಳು ಅಗತ್ಯವಾಗಿರುತ್ತವೆ.

##Prerequisites

- Azure OpenAI ಪ್ರವೇಶವಿರುವ Azure ಸಬ್ಸ್ಕ್ರಿಪ್ಷನ್
- ಜಾವಾ 21, ಮೇವೆನ್ 3.9+
- Azure CLI (https://learn.microsoft.com/en-us/cli/azure/install-azure-cli)
- Azure ಅಭಿವೃದ್ಧಿ CLI (azd) (https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/install-azd)

> **ಸೂಚನೆ:** ಜೆಎಸ್, ಮೇವೆನ್, Azure CLI ಮತ್ತು Azure ಅಭಿವೃದ್ಧಿ CLI (azd) ಸಾಕಿಂಚಿತ ಡೆಪ್ಲಳ್ಳಮ್ಯಾನ್ಯಲ್ಲಿವೆ.

> **ಸೂಚನೆ:** ಈ ಮೊಡ್ಯೂಲ್ GPT-5.2 ಅನ್ನು Azure OpenAI ನಲ್ಲಿ ಬಳಸುತ್ತದೆ. ನಿಯೋಜನೆ ಸ್ವಯಂಚಾಲಿತವಾಗಿ `azd up` ಮೂಲಕ ಸಂರಚಿತಗೊಂಡಿದೆ - ಕೋಡ್‌ನಲ್ಲಿ ಮಾದರಿ ಹೆಸರನ್ನು ಬದಲಿಸಬೇಡಿ.

## ಮೂಲ ಸಮಸ್ಯೆಯ ಅರ್ಥ

ಭಾಷಾ ಮಾದರಿಗಳು ಸ್ಟೇಟ್‌ಲೆಸ್ ಆಗಿವೆ. ಪ್ರತಿ API ಕರೆ ಸ್ವತಂತ್ರವಾಗಿದೆ. ನೀವು "My name is John" ಎಂದು ಕಳುಹಿಸಿದ ನಂತರ "What's my name?" ಎಂದು ಕೇಳಿದರೆ, ಮಾದರಿಗೆ ನೀವು ಸ್ವತಃ ಪರಿಚಯಿಸಿಕೊಂಡುದನ್ನು ತಿಳಿದಿಲ್ಲ. ಮುದ್ರಣಾತ್ಮಕವಾಗಿ ಪ್ರತಿ ವಿನಂತಿಯನ್ನು ಮೊದಲ ಸಂಭಾಷಣೆ ಎಂದು ಕಾಣುತ್ತದೆ.

ಇದು ಸರಳ ಪ್ರಶ್ನೋತ್ತರಕ್ಕೆ ಸೂಕ್ತವಾದರೂ, ವಾಸ್ತವಿಕ ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಗೆ ಬಳಕೆಗೆ ಅಸಾಧ್ಯ. ಗ್ರಾಹಕರ ಸೇವೆ ಬಾಟ್‌ಗಳು ನೀವು ಹೇಳಿದುದನ್ನು ನೆನಸಬೇಕು. ವೈಯಕ್ತಿಕ ಸಹಾಯಕರು ಸಂದರ್ಭ ಅಗತ್ಯವಿದೆ. ಯಾವುದೇ ಬಹು-ತಿರುಗಾಟ ಸಂಭಾಷಣೆಗೆ ಸ್ಮರಣೆಯ ಅಗತ್ಯವಿದೆ.

<img src="../../../translated_images/kn/stateless-vs-stateful.cc4a4765e649c41a.webp" alt="ಸ್ಟೇಟ್‌ಲೆಸ್ ವಿರುದ್ಧ ಸ್ಟೇಟ್‌ಫುಲ್ ಸಂಭಾಷಣೆಗಳು" width="800"/>

*ಸ್ಟೇಟ್‌ಲೆಸ್ (ಸ್ವತಂತ್ರ ಕರೆಗಳು) ಮತ್ತು ಸ್ಟೇಟ್‌ಫುಲ್ (ಸಂದರ್ಭ ಅರಿವು ಹೊಂದಿರುವ) ಸಂಭಾಷಣೆಗಳ ನಡುವಿನ ವ್ಯತ್ಯಾಸ*

## ಟೋಕನ್‌ಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು

ಸಂಭಾಷಣೆಯೊಳಗೆ ಸಂಚరಿಸಲು ಮೊದಲು, ಟೋಕನ್ಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು ಅಗತ್ಯ - ಭಾಷಾ ಮಾದರಿಗಳು ಪ್ರಕ್ರಿಯೆ ಮಾಡುವ ಪಠ್ಯದ ಮೂಲ ಘಟಕಗಳು:

<img src="../../../translated_images/kn/token-explanation.c39760d8ec650181.webp" alt="ಟೋಕನ್ ವಿವರಣೆ" width="800"/>

*ಪಠ್ಯವು ಹೇಗೆ ಟೋಕನ್ಗಳಾಗಿ ವಿಭಾಗಿಸಲಾಗುತ್ತದೆ ಎಂಬ ಉದಾಹರಣೆ - "I love AI!" 4 ಪ್ರತ್ಯೇಕ ಪ್ರಕ್ರಿಯೆ ಘಟಕಗಳಾಗಿ*

ಟೋಕನ್ಗಳು AI ಮಾದರಿಗಳು ಪಠ್ಯವನ್ನು ಅಳತೆಮಾಡುವ ಮತ್ತು ಪ್ರಕ್ರಿಯೆ ಮಾಡುವ ವಿಧಾನ. ಪದಗಳು, ವ್ಯಾಕರಣ ಗುರುತುಗಳು ಮತ್ತು ನಿರಾಕರಿತ ಸ್ಥಳಗಳು ಕೂಡ ಟೋಕನಾಗಬಹುದು. ನಿಮ್ಮ ಮಾದರಿಗೆ ಪ್ರತಿ ಸಾರಿ ಯಾವಷ್ಟು ಟೋಕನ್ಗಳನ್ನು ಪ್ರಕ್ರಿಯೆ ಮಾಡಬಹುದು ಎಂಬ ಸೀಮಿತವಿದೆ (GPT-5.2 ಗಾಗಿ 400,000, 272,000 ಇನ್‌ಪುಟ್ ಟೋಕನ್ಗಳು ಮತ್ತು 128,000 ಔಟ್‌ಪುಟ್ ಟೋಕನ್ಗಳವರೆಗೆ). ಟೋಕನ್ಗಳ ಅರ್ಥಮಾಡಿಕೊಳಲು ಸಂಭಾಷಣೆಯ ವಿಸ್ತಾರ ಮತ್ತು ವೆಚ್ಚಗಳನ್ನು ನಿರ್ವಹಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.

## ಸ್ಮರಣೆ ಹೇಗೆ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ

ಚಾಟ್ ಸ್ಮರಣೆ ಸ್ಟೇಟ್‌ಲೆಸ್ ಸಮಸ್ಯೆಯನ್ನು ಪರಿಹರಿಸಲು ಸಂಭಾಷಣಾ ಇತಿಹಾಸವನ್ನು ಕಾಪಾಡುತ್ತದೆ. ವಿನಂತಿಯನ್ನು ಮಾದರಿಗೆ ಕಳುಹಿಸುವ ಮೊದಲು, ವೇದಿಕೆ ಸಂಬಂಧಿಸಿದ ಹಿಂದಿನ ಸಂದೇಶಗಳನ್ನು ಮುಂಚೆಯೇ ಸೇರಿಸುತ್ತದೆ. ನೀವು "What's my name?" ಎಂದು ಕೇಳಿದಾಗ, ವ್ಯವಸ್ಥೆ ಸಂಪೂರ್ಣ ಸಂಭಾಷಣಾ ಇತಿಹಾಸವನ್ನು ಕಳುಹಿಸುತ್ತದೆ, ಇದರಿಂದ ಮಾದರಿ ನೀವು قب್ಲೆ "My name is John" ಎಂದು ಹೇಳಿದುದನ್ನು ನೋಡಿ ತಿಳಿದುಕೊಳ್ಳುತ್ತದೆ.

LangChain4j ಸ್ಮರಣೆ ಅನುಷ್ಠಾನಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ನಿರ್ವಹಿಸುತ್ತದೆ. ನೀವು ಕಾಯ್ದುಕೊಳ್ಳಬೇಕಾದ ಸಂದೇಶಗಳ ಸಂಖ್ಯೆಯನ್ನು ಆಯ್ಕೆ ಮಾಡಬಹುದು ಮತ್ತು ವೇದಿಕೆ ಸಂದರ್ಭ ಗ_WINDOWರನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ.

<img src="../../../translated_images/kn/memory-window.bbe67f597eadabb3.webp" alt="ಸ್ಮರಣೆ ಕಿಟಕಿ ಕಲ್ಪನೆ" width="800"/>

*MessageWindowChatMemory ಇತ್ತೀಚಿನ ಸಂದೇಶಗಳ ಸ್ಲೈಡಿಂಗ್ ವಿಂಡೋವನ್ನು ಕಾಪಾಡುತ್ತದೆ, ಹಳೆಯವುಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ತೆಗೆಯುತ್ತದೆ*

## ಇದು LangChain4j ಯನ್ನು ಹೇಗೆ ಬಳಸುತ್ತದೆ

ಈ ಮೊಡ್ಯೂಲ್ ತ್ವರಿತ ಪ್ರಾರಂಭವನ್ನು ವಿಸ್ತರಿಸಿ Spring Boot ಅನ್ನು ಸಂಯೋಜಿಸಿ ಮತ್ತು ಸಂಭಾಷಣಾ ಸ್ಮರಣೆ ಸೇರಿಸುತ್ತದೆ. ಭಾಗಗಳು ಹೇಗೆ ಹೊಂದಿಕೆಯಾಗಿವೆ ಎಂದು ಇಲ್ಲಿದೆ:

**ಆವಶ್ಯಕತೆಗಳು** - LangChain4j ಪುಸ್ತಕಾಲಯದ ಎರಡು ಅನ್ನು ಸೇರಿಸಿ:

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```

**ಚಾಟ್ ಮಾದರಿ** - Azure OpenAI ಯನ್ನು Spring ಬೀನ್ ಆಗಿ ಸಂರಚಿಸಿ ([LangChainConfig.java](../../../01-introduction/src/main/java/com/example/langchain4j/config/LangChainConfig.java)):

```java
@Bean
public OpenAiOfficialChatModel openAiOfficialChatModel() {
    return OpenAiOfficialChatModel.builder()
            .baseUrl(azureEndpoint)
            .apiKey(azureApiKey)
            .modelName(deploymentName)
            .timeout(Duration.ofMinutes(5))
            .maxRetries(3)
            .build();
}
```

ಬಿಲ್ಡರ್ `azd up` ಮೂಲಕ ಸೆಟ್ ಮಾಡಿದ ಪರಿಸರ ಚರಗಳನ್ನು ಓದುತ್ತದೆ. ನಿಮ್ಮ Azure ಎಂಡ್‌ಪಾಯಿಂಟ್‌ಗೆ `baseUrl`ನ್ನು ಸೆಟ್ ಮಾಡುವ ಮೂಲಕ OpenAI ಕ್ಲೈಂಟ್ Azure OpenAI ಜೊತೆಗೆ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ.

**ಸಂಭಾಷಣಾ ಸ್ಮರಣೆ** - MessageWindowChatMemory ಯೊಂದಿಗೆ ಚಾಟ್ ಇತಿಹಾಸವನ್ನು ಟ್ರ್ಯಾಕ್ ಮಾಡಿ ([ConversationService.java](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java)):

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("My name is John"));
memory.add(AiMessage.from("Nice to meet you, John!"));

memory.add(UserMessage.from("What's my name?"));
AiMessage aiMessage = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage);
```

ಕೊನೆಯ 10 ಸಂದೇಶಗಳನ್ನು ಕಾಯ್ದುಕೊಳ್ಳಲು `withMaxMessages(10)` ಬಳಸಿ ಸ್ಮರಣೆ ರಚಿಸಿ. Typed wrappers ಮೂಲಕ ಯೂಸರ್ ಮತ್ತು AI ಸಂದೇಶಗಳನ್ನು ಸೇರಿಸಿ: `UserMessage.from(text)` ಮತ್ತು `AiMessage.from(text)`. ಇತಿಹಾಸವನ್ನು `memory.messages()` ಮೂಲಕ ಪಡೆದು, ಅದನ್ನು ಮಾದರಿಗೆ ಕಳುಹಿಸಿ. ಸೇವೆ ಸಂಭಾಷಣಾ ಐಡಿಯ ಪ್ರತಿ ಪ್ರತ್ಯೇಕ ಸ್ಮರಣೆ ಅನುಕೂಲಗಳನ್ನು ಸಂಗ್ರಹಿಸುತ್ತದೆ, ಅನೇಕ ಬಳಕೆದಾರರು ಒಂದೇ ಸಮಯದಲ್ಲಿ ಸಂಭಾಷಣೆಗೆ ಆಗಬಹುದು.

> **🤖 GitHub Copilot ಚಾಟ್ ಜೊತೆಗೆ ಪ್ರಯತ್ನಿಸಿ:** [`ConversationService.java`](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java) ತೆರೆಯಿರಿ ಮತ್ತು ಕೇಳಿ:
> - "MessageWindowChatMemory ವಿಂಡೋ ತುಂಬಿದಾಗ ಯಾವ ಸಂದೇಶಗಳನ್ನು ತেম್ಮಬೇಕೆಂದು ಹೇಗಾಗಿದೆ?"
> - "ನಾನು ಸಾಮಾನ್ಯ ಸ್ಮರಣೆ ಸಂಗ್ರಹಣೆಯನ್ನುಸಾಮಾನ್ಯ ಡೇಟಾಬೇಸ್ ಬಳಸಿ ಹೇಗೆ ಮಾಡಬಹುದು?"
> - "ಹಳೆಯ ಸಂಭಾಷಣಾ ಇತಿಹಾಸವನ್ನು ಸಂಗ್ರಹಿಸಲು ಸಾರಾಂಶವನ್ನು ನಾನು ಹೇಗೆ ಸೇರಿಸಬಹುದು?"

ಸ್ಟೇಟ್‌ಲೆಸ್ ಚಾಟ್ ಎಂಡ್‌ಪಾಯಿಂಟ್ ಸಂಪೂರ್ಣವಾಗಿ ಸ್ಮರಣೆಯನ್ನು ಬಿಟ್ಟು ಬಿಡುತ್ತದೆ - `chatModel.chat(prompt)` ಹೀಗೂ ತ್ವರಿತ ಪ್ರಾರಂಭದಲ್ಲಿದ್ದಂತೆ. ಸ್ಟೇಟ್‌ಫುಲ್ ಎಂಡ್‌ಪಾಯಿಂಟ್ ಸಂದೇಶಗಳನ್ನು ಸ್ಮರಣೆಯಲ್ಲಿ ಸೇರಿಸಿ, ಇತಿಹಾಸವನ್ನು ಪಡೆಯುತ್ತಾ, ಪ್ರತಿ ವಿನಂತಿಯೊಂದಿಗೆ ಆ ಸಂದರ್ಭವನ್ನು ಸೇರಿಸುತ್ತದೆ. ಅದೇ ಮಾದರಿ ಸಂರಚನೆ, ವಿಭಿನ್ನ ಮಾದರಿಗಳು.
