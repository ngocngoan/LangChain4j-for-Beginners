## ನೀವು ಕಲಿಯುವದು

ನೀವು ದ್ರುತ ಪ್ರಾರಂಭ ಮುಗಿಸಿದ್ದರೆ, ಪ್ರಾಂಪ್ಟ್‌ಗಳನ್ನು ಕಳುಹಿಸುವುದು ಮತ್ತು ಪ್ರತಿಕ್ರಿಯೆಗಳನ್ನು ಪಡೆಯುವುದು ಹೇಗೆ ಎಂಬುದನ್ನು ನೋಡಿದ್ದೀರಾ. ಅದು ಮೂಲಾಧಾರ ಆದರೆ ನಿಜವಾದ ಅಪ್ಲಿಕೇಶನ್ಗಳು ಹೆಚ್ಚು ಅಗತ್ಯವೇ. ಈ ಮಾಯಾಜಾಲವು ಸಂವಹನಾತ್ಮಕ AI ಅನ್ನು ನಿರ್ಮಿಸುವುದು, ಅದು ಸಾಂದರ್ಭಿಕಾರಣೆಯನ್ನು ನೆನಪಿಡುತ್ತಾ ಸ್ಥಿತಿಯನ್ನು ಕಾಯ್ದುಕೊಳ್ಳುತ್ತದೆ ಎಂದು ಬೋಧಿಸುತ್ತದೆ - ಇದು ಒಂದು-ಸಮಯದ ಪ್ರದರ್ಶನ ಮತ್ತು ಉತ್ಪಾದನೆ- ಸಿದ್ಧ ಅಪ್ಲಿಕೇಶನ್ ನಡುವಿನ ಭೇದ.

ನಾವು ಈ ಮಾರ್ಗದರ್ಶಿಯಲ್ಲಿ Azure OpenAI ಯ GPT-5.2 ಅನ್ನು ಬಳಸುತ್ತೇವೆ ಏಕೆಂದರೆ ಇದರ ಸುಧಾರಿತ ವಿವೇಚನಾ ಸಾಮರ್ಥ್ಯಗಳು ವಿಭಿನ್ನ ಮಾದರಿಗಳ ವರ್ತನೆ ಗೋಚರಿಸುತ್ತವೆ. ನೀವು ಸ್ಮೃತಿಯನ್ನು ಸೇರಿಸಿದಾಗ, ನೀವು ಸ್ಪಷ್ಟವಾಗಿ ಭೇದವನ್ನು ನೋಡುತ್ತೀರಿ. ಇದು ಪ್ರತಿಯೊಂದು ಘಟಕದ ಅಪ್ಲಿಕೇಶನ್‌ಗೆ ಏನು ತರುತ್ತದೆ ಎಂದು ಬಲವಾಗಿ ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.

ನೀವು ಎರಡೂ ಮಾದರಿಗಳನ್ನು ತೋರಿಸುವ ಒಂದು ಅಪ್ಲಿಕೇಶನ್ ನಿರ್ಮಿಸುತ್ತೀರಿ:

**ಸ್ಟೇಟ್‌ಲೇಸ್ ಚಾಟ್** - ಪ್ರತಿಯೊಂದು ವಿನಂತಿ ಸ್ವತಂತ್ರವಾಗಿದೆ. ಮಾದರಿಗೆ ಹಿಂದಿನ ಸಂದೇಶಗಳ ಸ್ಮೃತಿ ಇಲ್ಲ. ನೀವು ದ್ರುತ ಪ್ರಾರಂಭದಲ್ಲಿ ಬಳಸಿದ್ದು ಇದೇ ಮಾದರಿ.

**ಸ್ಟೇಟ್ಫುಲ್ ಸಂವಾದ** - ಪ್ರತಿಯೊಂದು ವಿನಂತಿಯಲ್ಲಿ ಸಂವಾದ ಇತಿಹಾಸ ಸೇರಿದೆ. ಮಾದರಿ ಹಲವು ಬಾರತಿನೊಳಗಿನ ಸಾಂದರ್ಭಿಕ ತತ್ವವನ್ನು ಕಾಯ್ದುಕೊಳ್ಳುತ್ತದೆ. ಇದು ಉತ್ಪಾದನಾ ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಗೆ ಅಗತ್ಯವಾಗಿರುತ್ತದೆ.

## ಪೂರ್ವಾವಶ್ಯಕತೆಗಳು

- Azure OpenAI ಪ್ರವೇಶ ಹೊಂದಿರುವ Azure ಚಂದಾ ಖಾತೆ
- Java 21, Maven 3.9+
- Azure CLI (https://learn.microsoft.com/en-us/cli/azure/install-azure-cli)
- Azure Developer CLI (azd) (https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/install-azd)

> **ಗಮನಿಸಿ:** Java, Maven, Azure CLI ಮತ್ತು Azure Developer CLI (azd) Devcontainer ನಲ್ಲಿ ಪೂರ್ವ-ಸ್ಥಾಪಿತವಾಗಿವೆ.

> **ಗಮನಿಸಿ:** ಈ ಮಾಯಾಜಾಲವು Azure OpenAI ನ GPT-5.2 ಉಪಯೋಗಿಸುತ್ತದೆ. ನಿಯೋಜನೆ ಸ್ವಯಂಚಾಲಿತವಾಗಿ `azd up` ಮೂಲಕ ಸಂರಚಿಸಲಾಗಿದೆ - ಕೋಡಿನಲ್ಲಿ ಮಾದರಿ ಹೆಸರನ್ನು ಬದಲಾಯಿಸಬೇಡಿ.

## ಮೂಲ ಸಮಸ್ಯೆಯನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು

ಭಾಷಾ ಮಾದರಿಗಳು ಸ್ಟೇಟ್‌ಲೇಸ್ ಆಗಿವೆ. ಪ್ರತಿ API ಕರೆ ಸ್ವತಂತ್ರವಾಗಿದೆ. ನೀವು "ನನ್ನ ಹೆಸರು ಜಾನ್" ಎಂದು ಕಳುಹಿಸಿದಾಗ ಮತ್ತು ನಂತರ "ನನ್ನ ಹೆಸರು ಏನು?" ಎಂದು ಕೇಳಿದಾಗ, ಮಾದರಿಗೆ ನೀವು ಈಗಲೇ ಪರಿಚಯ ಮಾಡಿಸಿಕೊಂಡಿದ್ದೀರಿ ಎಂಬುದರ ತಿಳಿವಳಿಕೆ ಇಲ್ಲ. ಇದು ಪ್ರತಿಯೊಂದು ವಿನಂತಿಯನ್ನು ನೀವು ಮೊದಲ ಸಂವಾದ ಮಾಡುತ್ತಿರುವಂತೆ ಕಾಣುತ್ತದೆ.

ಸರಳ ಪ್ರಶ್ನೋತ್ತರಗಳಿಗೆ ಇದು ಚೆನ್ನಾಗಿದ್ದು, ನಿಜವಾದ ಅಪ್ಲಿಕೇಶನ್ಗಳಿಗೆ ಬೆರೆವುದಿಲ್ಲ. ಗ್ರಾಹಕ ಸೇವಾ ಬೋಟ್ಗಳು ನೀವು ಏನು ಹೇಳಿದ್ದೀರೋ ಅದನ್ನು ನೆನಪಿರಿಸಲು ಬೇಕು. ವೈಯಕ್ತಿಕ ಸಹಾಯಕರು ಸಾಂದರ್ಭಿಕಾರಣೆಯ ಅಗತ್ಯವಿದೆ. ಯಾವುದೇ ಬಹುಬಾರ್ತಿ ಸಂವಾದಕ್ಕೆ ಸ್ಮೃತಿ ಬೇಕು.

<img src="../../../translated_images/kn/stateless-vs-stateful.cc4a4765e649c41a.webp" alt="ಸ್ಟೇಟ್‌ಲೇಸ್ ಮತ್ತು ಸ್ಟೇಟ್ಫುಲ್ ಸಂವಾದಗಳು" width="800"/>

*ಸ್ವತಂತ್ರ ಕರೆಗಳು (ಸ್ಟೇಟ್‌ಲೇಸ್) ಮತ್ತು ಸಾಂದರ್ಭಿಕ ಜಾಗೃತಿ ಹೊಂದಿದ ಸಂವಾದಗಳು (ಸ್ಟೇಟ್ಫುಲ್) ನಡುವಿನ ವ್ಯತ್ಯಾಸ*

## ಟೋಕನ್ಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು

ಸಂವಾದಗಳಿಗೆ ಮುಚ್ಚಳಾಗುವ ಮೊದಲು, ಟೋಕನ್ಗಳನ್ನು ಅರ್ಥ ಮಾಡಿಕೊಳ್ಳುವುದು ಮಹತ್ವಕ್ಕಾಗಿ - ಭಾಷಾ ಮಾದರಿಗಳು ಪ್ರಕ್ರಿಯೆ ಮಾಡುವ ಮೂಲ ಪಠ್ಯದ ಘಟಕಗಳು:

<img src="../../../translated_images/kn/token-explanation.c39760d8ec650181.webp" alt="ಟೋಕನ್ ವಿವರಣೆ" width="800"/>

*ಪಠ್ಯವು ಟೋಕನ್ಗಳಾಗಿ ವಿಭಜನೆಯ ಉದಾಹರಣೆ - "ನಾನು AI ನನ್ನು ಪ್ರೀತಿಸುತ್ತೇನೆ!" 4 ಪ್ರತ್ಯೇಕ ಪ್ರಕ್ರಿಯೆ ಘಟಕಗಳಾಗುತ್ತದೆ*

ಟೋಕನ್ಗಳು AI ಮಾದರಿಗಳು ಪಠ್ಯವನ್ನು ಅಳೆಯುವ ಮತ್ತು ಪ್ರಕ್ರಿಯೆ ಮಾಡುವ ವಿಧಾನ. ಪದಗಳು, विरಾಮಚಿತ್ರಗಳು, ಮತ್ತು ಖಾಲಿ ಜಾಗಗಳೂ ಕೂಡ ಟೋಕನ್ಗಳು ಇರಬಹುದು. ನಿಮ್ಮ ಮಾದರಿಗೆ ಒಟ್ಟು ಸ್ವೀಕರಿಸುವ ಟೋಕನ್ಗಳ ಮಿತಿ ಇದೆ (GPT-5.2 ಗೆ 400,000, ಅದರಲ್ಲಿ 272,000 ಇನ್‌ಪುಟ್ ಟೋಕನ್ಗಳು ಮತ್ತು 128,000 ಔಟ್‌ಪುಟ್ ಟೋಕನ್ಗಳು). ಟೋಕನ್ಗಳ ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು ಸಂವಾದದ ಉದ್ದ ಮತ್ತು ವೆಚ್ಚಗಳನ್ನು ನಿಯಂತ್ರಿಸಲು ಸಹಕಾರಿಯಾಗುತ್ತದೆ.

## ಸ್ಮೃತಿ ಹೇಗೆ ಕೆಲಸ ಮಾಡುತ್ತದೆ

ಚಾಟ್ ಸ್ಮೃತಿ ಸ್ಟೇಟ್‌ಲೇಸ್ ಸಮಸ್ಯೆಯನ್ನು ಸಂವಾದ ಇತಿಹಾಸವನ್ನು ಕಾಯ್ದುಕೊಳ್ಳುವ ಮೂಲಕ ಪರಿಹರಿಸುತ್ತದೆ. ನಿಮ್ಮ ವಿನಂತಿಯನ್ನು ಮಾದರಿಗೆ ಕಳುಹಿಸುವ ಮೊದಲು, ಪ್ರಯುಕ್ತ ಸಂಬಂಧಿಸಿದ ಹಿಂದಿನ ಸಂದೇಶಗಳನ್ನು ಜೋಡಿಸಲಾಗುತ್ತದೆ. ನೀವು "ನನ್ನ ಹೆಸರು ಏನು?" ಎಂದು ಕೇಳಿದಾಗ, ವ್ಯವಸ್ಥೆ ಪೂರ್ತಿ ಸಂವಾದ ಇತಿಹಾಸವನ್ನು ಕಳುಹಿಸುತ್ತದೆ, ಹೀಗೆ ಮಾದರಿ ನೀವು ಹಿಂದಿನಲ್ಲಿಯೇ "ನನ್ನ ಹೆಸರು ಜಾನ್" ಎಂದಿದ್ದೀರಿ ಎಂದು ನೋಡುವುದಕ್ಕೆ ಸಾಧ್ಯವಾಗುತ್ತದೆ.

LangChain4j ಸ್ಮೃತಿ ಅನುಷ್ಠಾನಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ನಿರ್ವಹಿಸುತ್ತದೆ. ನೀವು ಎಷ್ಟು ಹೆಚ್ಚು ಸಂದೇಶಗಳನ್ನು ಕಾಯ್ದುಕೊಳ್ಳಬೇಕೆಂದು ಆರಿಸಿಕೊಂಡು, ಫ್ರೇಂವರ್ಕ್ ಸಾಂದರ್ಭಿಕ ವಿಂಡೋವನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ.

<img src="../../../translated_images/kn/memory-window.bbe67f597eadabb3.webp" alt="ಸ್ಮೃತಿ ವಿಂಡೋ ತತ್ವ" width="800"/>

*MessageWindowChatMemory ಇತ್ತೀಚಿನ ಸಂದೇಶಗಳ ಸ್ಲೈಡಿಂಗ್ ವಿಂಡೋವನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ, ಹಳೆಯ ಸಂದೇಶಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಮಾಯಮಾಡಿ ಬಿಡುತ್ತದೆ*

## ಇದು LangChain4j ಅನ್ನು ಹೇಗೆ ಉಪಯೋಗಿಸುತ್ತದೆ

ಈ ಮಾಯಾಜಾಲವು ದ್ರುತ ಪ್ರಾರಂಭವನ್ನು ಮತ್ತಷ್ಟು ವಿಸ್ತರಿಸಿ Spring Boot ಅನ್ನು ಏರ್ಪಡಿಸುವುದು ಮತ್ತು ಸಂವಾದ ಸ್ಮೃತಿಯನ್ನು ಸೇರಿಸುವುದನ್ನು ಒಳಗೊಂಡಿದೆ. ಜೋತೆಯಾಗಿ ಶೇರುವಿಕೆಗಳು ಈ ರೀತಿ:

**ಅವಶ್ಯಕತೆಗಳು** - ಎರಡು LangChain4j ಲೈಬ್ರರಿಗಳನ್ನು ಸೇರಿಸಿ:

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```

**ಚಾಟ್ ಮಾದರಿ** - Azure OpenAI ಅನ್ನು Spring ಬಿನ್ ಆಗಿ ಸಂರಚಿಸಿ ([LangChainConfig.java](../../../01-introduction/src/main/java/com/example/langchain4j/config/LangChainConfig.java)):

```java
@Bean
public OpenAiOfficialChatModel openAiOfficialChatModel() {
    return OpenAiOfficialChatModel.builder()
            .baseUrl(azureEndpoint)
            .apiKey(azureApiKey)
            .modelName(deploymentName)
            .timeout(Duration.ofMinutes(5))
            .maxRetries(3)
            .build();
}
```

ಬಿಲ್ಡರ್ ಯಾವುದೇ `azd up` ಮೂಲಕ ಸೆಟ್ ಮಾಡಲಾದ ಪರಿಸರ ಚರಗಳನ್ನು ಓದುತ್ತದೆ. `baseUrl` ಅನ್ನು ನಿಮ್ಮ Azure ಅಂತಿಮ ಬಿಂದುವಿಗೆ ಸೆಟ್ ಮಾಡುವುದು OpenAI ಕ್ಲೈಂಟ್ ಅನ್ನು Azure OpenAI ಜೊತೆಗೆ ಕೆಲಸ ಮಾಡಲು ಅನುಮತಿಸುತ್ತದೆ.

**ಸಂವಾದ ಸ್ಮೃತಿ** - MessageWindowChatMemory ಬಳಸಿಕೊಂಡು ಚಾಟ್ ಇತಿಹಾಸವನ್ನು ಟ್ರ್ಯಾಕ್ ಮಾಡುತ್ತದೆ ([ConversationService.java](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java)):

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("My name is John"));
memory.add(AiMessage.from("Nice to meet you, John!"));

memory.add(UserMessage.from("What's my name?"));
AiMessage aiMessage = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage);
```

ಇದರಲ್ಲಿ `withMaxMessages(10)` ಮೂಲಕ ಕೊನೆಯ 10 ಸಂದೇಶಗಳನ್ನು ಕಾಯ್ದುಕೊಳ್ಳುತ್ತದೆ. ಬಳಕೆದಾರ ಮತ್ತು AI ಸಂದೇಶಗಳನ್ನು ಟೈಪ್ ಮಾಡಿದ ಅವರಣಗಳೊಂದಿಗೆ ಸೇರಿಸಿ: `UserMessage.from(text)` ಮತ್ತು `AiMessage.from(text)`. ಇತಿಹಾಸವನ್ನು `memory.messages()` ಮೂಲಕ ಪಡೆದು ಮಾದರಿಗೆ ಕಳುಹಿಸಿ. ಸೇವೆವು ಪ್ರತಿ ಸಂವಾದ ID ಗೆ ಪ್ರತ್ಯೇಕ ಸ್ಮೃತಿ ಉದಾಹರಣೆಗಳನ್ನು ಸಂಗ್ರಹಿಸಿ, ಅನೇಕ ಬಳಕೆದಾರರು ಒಂದೇ ಸಮಯದಲ್ಲಿ ಚಾಟ್ ಮಾಡಬಹುದು.

> **🤖 GitHub Copilot [ಚಾಟ್](https://github.com/features/copilot) ಬಳಸಿ ಪ್ರಯತ್ನಿಸಿ:** [`ConversationService.java`](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java) ತೆರೆದು ಕೇಳಿ:
> - "MessageWindowChatMemory ವಿಂಡೋ ತುಂಬಿದಾಗ ಯಾವ ಸಂದೇಶಗಳನ್ನು ಬಿಡುವುದನ್ನು ಹೇಗೆ ನಿರ್ಧರಿಸುತ್ತದೆ?"
> - "ನಾನು ಇನ್-ಮೆಮೊರಿ ಬದಲಾಗಿ ಡೇಟಾಬೇಸ್ ಬಳಸಿ ಕಸ್ಟಮ್ ಸ್ಮೃತಿ ಸಂಗ್ರಹಣೆ ಮಾಡಬಹುದುವೇ?"
> - "ಹಳೆಯ ಸಂವಾದ ಇತಿಹಾಸವನ್ನು ಸರುಳಿಸುವ ಸಂಗ್ರಹಣೆಯನ್ನು ಹೇಗೆ ಸೇರಿಸಬಹುದು?"

ಸ್ಟೇಟ್‌ಲೇಸ್ ಚಾಟ್ ಎন্ড್‌ಪಾಯಿಂಟ್ ಸ್ಮೃತಿಯನ್ನು ಸಂಪೂರ್ಣವಾಗಿ ಹೋಗಲಾಡಿಸುತ್ತದೆ - ಕೇವಲ `chatModel.chat(prompt)` ಅನ್ನು ಬಳಸುತ್ತದೆ. ಸ್ಟೇಟ್ಫುಲ್ ಎಂಡ್ಪಾಯಿಂಟ್ ಸಂದೇಶಗಳನ್ನು ಸ್ಮೃತಿಗೆ ಸೇರಿಸಿ, ಇತಿಹಾಸವನ್ನು ಪಡೆದು, ಪ್ರತಿಯೊಂದು ವಿನಂತಿಯೊಂದಿಗೆ ಆ ಸಾಂದರ್ಭಿಕತೆಯನ್ನು ಸೇರಿಸುತ್ತದೆ. ಅದೇ ಮಾದರಿ ಸಂರಚನೆ, ವಿಭಿನ್ನ ಮಾದರಿಗಳು.

ಎರಡು ಪ್ಯಾನೆಲ್ಗಳು ಕೂಡ ಅದೇ GPT-5.2 ಮಾದರಿಯನ್ನು ಉಪಯೋಗಿಸುತ್ತವೆ. ಮಾತ್ರ ಭೇದವೇ ಸ್ಮೃತಿ. ಇದು ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ನಿಗೆ ಸ್ಮೃತಿ ಏನು ತರುತ್ತದೆ ಮತ್ತು ನಿಜವಾದ ಬಳಕೆಗೆ ಇದು ಎಷ್ಟು ಅವಶ್ಯಕ ಎಂಬುದನ್ನು ಸ್ಪಷ್ಟಪಡಿಸುತ್ತದೆ.
