## ನೀವು ಏನು ಕಲಿಯುತ್ತೀರಿ

ನೀವು ತ್ವರಿತ ಪ್ರಾರಂಭವನ್ನು ಪೂರ್ಣಗೊಳಿಸಿದರೆ, ನೀವು ಪ್ರಾಂಪ್ಟ್‌ಗಳು ಹೇಗೆ ಕಳುಹಿಸಬೇಕು ಮತ್ತು ಪ್ರತಿಕ್ರಿಯೆಗಳು ಹೇಗೆ ಪಡೆಯಬೇಕು ಎಂದು ನೋಡಿದ್ದೀರಿ. ಅದು ಆಧಾರವಾಗಿದೆ, ಆದರೆ ನಿಜವಾದ ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಗೆ ಇನ್ನೂ ಹೆಚ್ಚು ಅಗತ್ಯವಿದೆ. ಈ ಮೋಡ್ಯೂಲ್ ಸಂಭಾಷಣಾತ್ಮಕ AI ಅನ್ನು ನಿರ್ಮಿಸುವುದನ್ನು ಕಲಿಸುತ್ತದೆ, ಅದು ಸಂದರ್ಭವನ್ನು ನೆನಸಿಕೊಳ್ಳುತ್ತದೆ ಮತ್ತು ಸ್ಥಿತಿಯನ್ನು ಪಾಲಿಸುತ್ತದೆ - ಒಮ್ಮೆ ಮಾಡುವ ಡೆಮೋ ಮತ್ತು ಉತ್ಪಾದನಾ ತಯಾರಿರುವ ಅಪ್ಲಿಕೇಶನ್ ನಡುವಿನ ವ್ಯತ್ಯಾಸ.

ನಾವು ಈ ಮಾರ್ಗದರ್ಶಿಯಲ್ಲಿ Azure OpenAI ರ GPT-5.2 ಅನ್ನು ಬಳಸದಿರುವೆವು ಏಕೆಂದರೆ ಅದರ ಪರಿಣತ ಯುಕ್ತಿ ಸಾಮರ್ಥ್ಯಗಳಿಂದ ವಿವಿಧ ಮಾದರಿಗಳ ವರ್ತನೆ ಬಹಳ ಸ್ಪಷ್ಟವಾಗುತ್ತದೆ. ನೀವು ಮೆಮರಿಯನ್ನು ಸೇರಿಸಿದಾಗ, ವ್ಯತ್ಯಾಸ ಸ್ಪಷ್ಟವಾಗಿ ಕಾಣುತ್ತದೆ. ಇದು ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್‌ಗೆ ಪ್ರತಿ ಘಟಕವು ಏನು ತರುತ್ತದೆ ಎಂದು ಗ್ರಹಿಸಲು ಸುಲಭವಿರುತ್ತದೆ.

ನೀವು ಎರಡು ಮಾದರಿಗಳನ್ನು ಪ್ರದರ್ಶಿಸುವ ಒಂದು ಅಪ್ಲಿಕೇಶನ್ ನಿರ್ಮಿಸಲಾಗುವುದು:

**ಸ್ಟೇಟ್‌ಲೆಸ್ ಚಾಟ್** - ಪ್ರತಿ ವಿನಂತಿ ಸ್ವತಂತ್ರವಾಗಿದೆ. ಮಾದರಿಯು ಹಿಂದಿನ ಸಂದೇಶಗಳ ಮೆಮರಿ ಹೊಂದಿಲ್ಲ. ಇದು ನೀವು ತ್ವರಿತ ಪ್ರಾರಂಭದಲ್ಲಿ ಬಳಸಿದ ಮಾದರಿಯಾಗಿದೆ.

**ಸ್ಟೇಟ್ಫುಲ್ ಸಂಭಾಷಣೆ** - ಪ್ರತಿ ವಿನಂತಿಯಲ್ಲಿ ಸಂಭಾಷಣಾ ಇತಿಹಾಸವು ಸೇರಿಸಲಾಗಿದೆ. ಮಾದರಿ ಅನೇಕ ತಿರುವುಗಳಾದರೋ_Context_ ಅನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ. ಇದು ಉತ್ಪಾದನಾ ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಗೆ ಅಗತ್ಯ.

## ಪ್ರೇರಿತ ಅಗತ್ಯಗಳು

- Azure OpenAI ಪ್ರವೇಶ ಹೊಂದಿರುವ Azure ಸಬ್ಸ್ಕ್ರಿಪ್ಷನ್
- ಜಾವಾ 21, ಮೂವನ್ 3.9+
- Azure CLI (https://learn.microsoft.com/en-us/cli/azure/install-azure-cli)
- Azure Developer CLI (azd) (https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/install-azd)

> **ಟಿಪ್ಪಣಿ:** ಜಾವಾ, ಮೂವನ್, Azure CLI ಮತ್ತು Azure Developer CLI (azd) ಈಗಾಗಲೇ devcontainer ನಲ್ಲಿ ಸ್ಥಾಪಿತವಾಗಿದೆ.

> **ಟಿಪ್ಪಣಿ:** ಈ ಮೋಡ್ಯೂಲ್ Azure OpenAI ನಲ್ಲಿ GPT-5.2 ನ್ನು ಬಳಸುತ್ತದೆ. ನಿಯೋಜನೆ ಸ್ವಯಂಚಾಲಿತವಾಗಿಯೇ `azd up` ಮೂಲಕ ಮಾಡಿದಂತೆ ಇದೆ - ಕೋಡ್‌ನಲ್ಲಿನ ಮಾದರಿ ಹೆಸರನ್ನು ಬದಲಾಯಿಸಬೇಡಿ.

## ಮೂಲ ಸಮಸ್ಯೆಯನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು

ಭಾಷಾ ಮಾದರಿಗಳು ಸ್ಟೇಟ್‌ಲೆಸ್ ಆಗಿವೆ. ಪ್ರತಿ API ಕರೆ ಸ್ವತಂತ್ರ. ನೀವು "ನನ್ನ ಹೆಸರೇ ಜಾನ್" ಎಂದು ಕಳುಹಿಸಿದ ನಂತರ "ನನ್ನ ಹೆಸರು ಏನು?" ಎಂದರೆ, ಮಾದರಿಗೆ ನೀವು ಈಗಲೇ ಪರಿಚಯಿಸಿಕೊಂಡಿದ್ದೀರಾ ಎಂದು ಗೊತ್ತಿಲ್ಲ. ಇದು ಪ್ರತಿ ವಿನಂತಿಯನ್ನು ಮೊದಲ ಸಂಭಾಷಣೆಯಂತೆ ಪರಿಗಣಿಸುತ್ತದೆ.

ಇದು ಸರಳ ಪ್ರಶ್ನೋತ್ತರಗಳಿಗೆ ಒಳ್ಳೆಯದು ಆದರೆ ನಿಜವಾದ ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಗೆ ಅಪ್ರಯೋಜನಕಾರಿಯಾಗಿದೆ. ಗ್ರಾಹಕ ಸೇವಾ ಬಾಟ್‌ಗಳು ನಿಮಗೆ ಹೇಳಿದುದನ್ನು ನೆನಸಿಕೊಳ್ಳಬೇಕು. ವೈಯಕ್ತಿಕ ಸಹಾಯಕರಿಗೆ_Context_ ಅಗತ್ಯವಿದೆ. ಯಾವುದೇ ಬಾಹ್ಯ ಸಂಭಾಷಣೆಗೆ ಮೆಮರಿ ಅಗತ್ಯ.

<img src="../../../translated_images/kn/stateless-vs-stateful.cc4a4765e649c41a.webp" alt=" ಸ್ಟೇಟ್‌ಲೆಸ್ ವಿರುದ್ಧ ಸ್ಟೇಟ್ಫುಲ್ ಸಂಭಾಷಣೆಗಳು" width="800"/>

*ಸ್ಟೇಟ್‌ಲೆಸ್ (ಸ್ವತಂತ್ರ ಕರೆಗಳು) ಮತ್ತು ಸ್ಟೇಟ್ಫುಲ್ (ಸಂದರ್ಭ-ಜಾಗೃತಿ) ಸಂಭಾಷಣೆಗಳ ನಡುವಿನ ವ್ಯತ್ಯಾಸ*

## ಟೊಕೆನ್ಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು

ಸಂಭಾಷಣೆಗಳೊಳಗೆ ಮುಳಿದು ಹೋಗುವ ಮೊದಲು, ಟೊಕೆನ್ಗಳ ಬಗ್ಗೆ ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು ಮಹತ್ವದಿದ್ದು - ಭಾಷಾ ಮಾದರಿಗಳು ಪ್ರಕ್ರಿಯೆ ಮಾಡುವ ಮೂಲ ಘಟಕಗಳು:

<img src="../../../translated_images/kn/token-explanation.c39760d8ec650181.webp" alt="ಟೊಕೆನ್ ವಿವರಣೆ" width="800"/>

*ಪಠ್ಯವು ಟೊಕೆನ್ಗಳಾಗಿ ಹೋಯ್ದಿರುವ ಉದಾಹರಣೆ - "I love AI!"  ಅರ್ಧ 4 ಪ್ರತ್ಯೇಕ ಪ್ರಕ್ರಿಯೆ ಘಟಕಗಳಾಗಿ*

ಟೊಕೆನ್ಗಳೇ AI ಮಾದರಿಗಳು ಪಠ್ಯವನ್ನು ಅಳೆಯಲು ಮತ್ತು ಪ್ರಕ್ರಿಯೆ ಮಾಡಲು ಬಳಸುವ ವಿಧಾನ. ಪದಗಳು, ಚಿಹ್ನೆಗಳು ಮತ್ತು ಇಡಿಮುಡಿಗಳು ಸಹ ಟೊಕೆನ್ಗಳಾಗಬಹುದು. ನಿಮ್ಮ ಮಾದರಿಗೆ ಒಟ್ಟಾರೆ ಎಷ್ಟು ಟೊಕೆನ್ಗಳನ್ನು ಒ一次ದಾಗಿ ಪ್ರಕ್ರಿಯೆ ಮಾಡಬಹುದು ಎಂಬುದಕ್ಕೆ ಗಡಿ ಇದೆ (GPT-5.2 ಗಾಗಿ 400,000, 272,000 ಇನ್ಪುಟ್ ಟೊಕೆನ್ಸ್ ಮತ್ತು 128,000 ಔಟ್ಪುಟ್ ಟೊಕೆನ್ಸ್). ಟೊಕೆನ್ಗಳು ಸಂಭಾಷಣೆಯ ಉದ್ದ ಮತ್ತು ವೆಚ್ಚಗಳನ್ನು ನಿರ್ವಹಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತವೆ.

## ಮೆಮರಿ ಹೇಗೆ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ

ಚಾಟ್ ಮೆಮರಿ ಸ್ಟೇಟ್‌ಲೆಸ್ ಸಮಸ್ಯೆಯನ್ನು ಸಂಭಾಷಣಾ ಇತಿಹಾಸವನ್ನು ನಿರ್ವಹಿಸುವ ಮೂಲಕ ಪರಿಹರಿಸುತ್ತದೆ. ನಿಮ್ಮ ವಿನಂತಿಯನ್ನು ಮಾದರಿಗೆ ಕಳುಹಿಸುವ ಮೊದಲು, ಫ್ರೇಮ್‌ವರ್ಕ್ ಸಂಬಂಧಿಸಿದ ಹಿಂದಿನ ಸಂದೇಶಗಳನ್ನು ಸೇರಿಸುತ್ತದೆ. ನೀವು "ನನ್ನ ಹೆಸರು ಏನು?" ಎಂದು ಕೇಳಿದಾಗ, ಸಿಸ್ತಮ್ ಆದರೆ ಸಂಪೂರ್ಣ ಸಂಭಾಷಣಾ ಇತಿಹಾಸವನ್ನು ಕಳುಹಿಸುತ್ತದೆ, ಹೀಗೆ ಮಾದರಿ ನೀವು "ನನ್ನ ಹೆಸರು ಜಾನ್" ಎಂದು ಹೇಳಿದಿರಿ ಎಂಬುದನ್ನು ನೋಡಬಹುದು.

LangChain4j 自己自动 ವ್ಯವಸ್ಥಾಪನೆಯಾದ ಮೆಮರಿ ಅನುಷ್ಠಾನಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ. ನೀವು ಎಷ್ಟು ಸಂದೇಶಗಳನ್ನು ಉಳಿಸಬೇಕೆಂದು ಆಯ್ಕೆ ಮಾಡಬಹುದು ಮತ್ತು ಫ್ರೇಮ್‌ವರ್ಕ್_Context_ ವಿಂಡೋವನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ.

<img src="../../../translated_images/kn/memory-window.bbe67f597eadabb3.webp" alt="ಮೆಮರಿ ವಿಂಡೋ ಕಲ್ಪನೆ" width="800"/>

*MessageWindowChatMemory ಇತ್ತೀಚಿನ ಸಂದೇಶಗಳ ಸರಕು ವಿಂಡೋವನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ, ಹಳೆಯವುಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಪಳೆಯುತ್ತದೆ*

## ಇದು LangChain4j ಅನ್ನು ಹೇಗೆ ಬಳಸುತ್ತದೆ

ಈ ಮೋಡ್ಯೂಲ್ ತ್ವರಿತ ಪ್ರಾರಂಭವನ್ನು ವಿಸ್ತರಿಸುತ್ತದೆ Spring Boot ಜೊತೆಗೆ ಮತ್ತು ಸಂಭಾಷಣಾ ಮೆಮರಿಯನ್ನು ಸೇರಿಸುತ್ತದೆ. ಭಾಗಗಳು ಹೀಗೆ ಜೋಡಾಗಿ ಕೆಲಸ ಮಾಡುತ್ತವೆ:

**ಅವಲಂಬನೆಗಳು** - ಎರಡು LangChain4j ಗ್ರಂಥಾಲಯಗಳನ್ನು ಸೇರಿಸಿ:

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```

**ಚಾಟ್ ಮಾದರಿ** - Azure OpenAI ನ್ನು ಒಂದು Spring ಬೀನ್ ಆಗಿ ಹೊಂದಿಸಿ ([LangChainConfig.java](../../../01-introduction/src/main/java/com/example/langchain4j/config/LangChainConfig.java)):

```java
@Bean
public OpenAiOfficialChatModel openAiOfficialChatModel() {
    return OpenAiOfficialChatModel.builder()
            .baseUrl(azureEndpoint)
            .apiKey(azureApiKey)
            .modelName(deploymentName)
            .timeout(Duration.ofMinutes(5))
            .maxRetries(3)
            .build();
}
```

ಬಿಲ್ಡರ್ `azd up` ಮೂಲಕ ಸಿದ್ಧಪಡಿಸಲಾದ ಪರಿಸರ ಚರಗಳನ್ನು ಓದುತ್ತದೆ. `baseUrl` ಅನ್ನು ನಿಮ್ಮ Azure ಎಂಡ್ಪಾಯಿಂಟ್ ಗೆ ಹೊಂದಿಸುವುದು OpenAI ಕ್ಲೈಂಟ್ ಅನ್ನು Azure OpenAI ಜೊತೆಗೆ ಕೆಲಸಮಾಡುವಂತೆ ಮಾಡುತ್ತದೆ.

**ಸಂಭಾಷಣಾ ಮೆಮರಿ** - MessageWindowChatMemory ಜೊತೆ ಚಾಟ್ ಇತಿಹಾಸವನ್ನು ಟ್ರ್ಯಾಕ್ ಮಾಡಿ ([ConversationService.java](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java)):

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("My name is John"));
memory.add(AiMessage.from("Nice to meet you, John!"));

memory.add(UserMessage.from("What's my name?"));
AiMessage aiMessage = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage);
```

`withMaxMessages(10)` ನೊಂದಿಗೆ ಮೆಮರಿಯನ್ನು ರಚಿಸಿ, ಕೊನೆಯ 10 ಸಂದೇಶಗಳನ್ನು ಉಳಿತಾಯ ಮಾಡಲು. ಬಳಕೆದಾರ ಮತ್ತು AI ಸಂದೇಶಗಳನ್ನು ಟೈಪ್ ಮಾಡಲಾದ ವ್ರಾಪರ್ಸ್ `UserMessage.from(text)` ಮತ್ತು `AiMessage.from(text)` ಜೊತೆಗೆ ಸೇರಿಸಿ. ಇತಿಹಾಸವನ್ನು `memory.messages()` ನಿಂದ ಪಡೆದು ಮಾದರಿಗೆ ಕಳುಹಿಸಿ. ಸರ್ವೀಸ್ ಪ್ರತಿ ಸಂಭಾಷಣೆ ID ಗೆ ವಿಭಿನ್ನ ಮೆಮರಿ ಇನ್ಸ್ಟನ್ಸ್‌ಗಳನ್ನು ಉಳಿಸುತ್ತದೆ, ಇದರಿಂದ ಅನೇಕ ಬಳಕೆದಾರರು ಸಮಕಾಲಿಕವಾಗಿ ಚಾಟ್ ಮಾಡಬಹುದು.

> **🤖 GitHub Copilot ಜೊತೆ ಪ್ರಯತ್ನಿಸಿ:** [`ConversationService.java`](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java) ತೆರೆದು ಕೇಳಿ:
> - "MessageWindowChatMemory ವಿಂಡೋ ತುಂಬಿ ಹೋದಾಗ ಯಾವ ಸಂದೇಶಗಳನ್ನು ಬಿಟ್ಟೀಡುತ್ತದೆ ಎಂದು ಹೇಗೆ ನಿರ್ಧರಿಸುತ್ತದೆ?"
> - "ನಾನು ಡೇಟಾಬೇಸ್ ಬಳಸಿ ಕಸ್ಟಮ್ ಮೆಮರಿ ಭಂಡಾರವನ್ನು ಅನುಷ್ಠಾನ ಮಾಡಬಹುದೇ?"
> - "ಹಳೆಯ ಸಂಭಾಷಣಾ ಇತಿಹಾಸವನ್ನು ಸಂಕ್ಷಿಪ್ತ ಮಾಡುವುದಕ್ಕೆ ವಿವರಾಂಶವನ್ನೇನು ಸೇರಿಸಬಹುದು?"

ಸ್ಟೇಟ್‌ಲೆಸ್ ಚಾಟ್ ಎಂಡಪಾಯಿಂಟ್ ಮೆಮರಿಯನ್ನು ಸಂಪೂರ್ಣವಾಗಿ ಬಿಟ್ಟುಬಿಡುತ್ತದೆ - ಸರಳವಾಗಿ `chatModel.chat(prompt)` ಹಾಗೆ ತ್ವರಿತ ಪ್ರಾರಂಭ. ಸ್ಟೇಟ್ಫುಲ್ ಎಂಡಪಾಯಿಂಟ್ ಮೆಸೆಜ್‌ಗಳನ್ನು ಮೆಮರಿಯಲ್ಲಿ ಸೇರಿಸಿ, ಇತಿಹಾಸವನ್ನು ಪಡೆದು Context ಜೊತೆಗೆ ಪ್ರತಿ ವಿನಂತಿಗೆ ಸೇರಿಸುತ್ತದೆ. ಅದೇ ಮಾದರಿ ಹೊಂದಾಣಿಕೆ, ಬೇರೆಯಾದ ಮಾದರಿಗಳು.

### ಸ್ಟೇಟ್‌ಲೆಸ್ ಚಾಟ್ (ಎಡ ಫಲಕ)

ಮೊದಲಿಗೆ ಇದನ್ನು ಪ್ರಯತ್ನಿಸಿ. "ನನ್ನ ಹೆಸರು ಜಾನ್" ಎಂದು ಕೇಳಿ ನಂತರ ತಕ್ಷಣವೇ "ನನ್ನ ಹೆಸರು ಏನು?" ಎಂದು ಕೇಳಿ. ಮಾದರಿ ನೆನಸಿಕೊಳ್ಳುವುದಿಲ್ಲ ಏಕೆಂದರೆ ಪ್ರತಿ ಸಂದೇಶ ಸ್ವತಂತ್ರವಾಗಿದೆ. ಇದು ಮೂಲ ಭಾಷಾ ಮಾದರಿ ಅನ್ನು ಸೇರಿಸುವಲ್ಲಿನ ಮುಖ್ಯ ಸಮಸ್ಯೆಯನ್ನು ಪ್ರದರ್ಶಿಸುತ್ತದೆ - ಸಂಭಾಷಣಾ_Context_ ಇಲ್ಲ.

<img src="../../../translated_images/kn/simple-chat-stateless-demo.13aeb3978eab3234.webp" alt="ಸ್ಟೇಟ್‌ಲೆಸ್ ಚಾಟ್ ಡೆಮೋ" width="800"/>

*AI ಹಿಂದಿನ ಸಂದೇಶದಿಂದ ನಿಮ್ಮ ಹೆಸರು ನೆನಸಿಕೊಳ್ಳದು*

### ಸ್ಟೇಟ್ಫುಲ್ ಚಾಟ್ (ಬಲ ಫಲಕ)

ಈಗ ಇದೇ ಕ್ರಮವನ್ನು ಇಲ್ಲಿ ಪ್ರಯತ್ನಿಸಿ. "ನನ್ನ ಹೆಸರು ಜಾನ್" ಮತ್ತು ನಂತರ "ನನ್ನ ಹೆಸರು ಏನು?" ಈ ಬಾರಿ ಅದು ನೆನಸಿಕೊಳ್ಳುತ್ತದೆ. ವ್ಯತ್ಯಾಸವೇ MessageWindowChatMemory - ಇದು ಸಂಭಾಷಣಾ ಇತಿಹಾಸವನ್ನು ನಿರ್ವಹಿಸುತ್ತದೆ ಮತ್ತು ಪ್ರತಿ ವಿನಂತಿಗೆ_Context_ ಸೇರಿಸುತ್ತದೆ. ಇದಾಗಿಯೇ ಉತ್ಪಾದನಾ ಸಂಭಾಷಣಾತ್ಮಕ AI ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ.

<img src="../../../translated_images/kn/conversational-chat-stateful-demo.e5be9822eb23ff59.webp" alt="ಸ್ಟೇಟ್ಫುಲ್ ಚಾಟ್ ಡೆಮೋ" width="800"/>

*AI ಮುಂಚಿನ ಸಂಭಾಷಣೆಯಿಂದ ನಿಮ್ಮ ಹೆಸರನ್ನು ನೆನಸಿಕೊಳ್ಳುತ್ತದೆ*

ಎರಡು ಫಲಕಗಳೂ ಇದೇ GPT-5.2 ಮಾದರಿಯನ್ನು ಬಳಕೆ ಮಾಡುತ್ತವೆ. ಒಂದೇ ವ್ಯತ್ಯಾಸವೆಂದರೆ ಮೆಮರಿ. ಇದು ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್‌ಗೆ ಮೆಮರಿ ಏನು ತರುತ್ತದೆ ಹಾಗೂ ನಿಜವಾದ ಉಪಯೋಗಕ್ಕೆ ಏಕೆ ಅವಶ್ಯಕ ಎಂಬುದನ್ನು ಸ್ಪಷ್ಟಗೊಳಿಸುತ್ತದೆ.
