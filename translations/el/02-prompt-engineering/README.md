## Τι θα μάθετε

Στην προηγούμενη μονάδα, είδατε πώς η μνήμη επιτρέπει τη συνομιλιακή τεχνητή νοημοσύνη και χρησιμοποιήσατε Μοντέλα GitHub για βασικές αλληλεπιδράσεις. Τώρα θα επικεντρωθούμε στον τρόπο που κάνετε ερωτήσεις - τις ίδιες τις προτροπές - χρησιμοποιώντας το GPT-5.2 της Azure OpenAI. Ο τρόπος που δομείτε τις προτροπές σας επηρεάζει δραματικά την ποιότητα των απαντήσεων που λαμβάνετε.

Θα χρησιμοποιήσουμε το GPT-5.2 επειδή εισάγει τον έλεγχο συλλογισμού - μπορείτε να πείτε στο μοντέλο πόσο σκέψη να κάνει πριν απαντήσει. Αυτό καθιστά τις διαφορετικές στρατηγικές προτροπής πιο εμφανείς και σας βοηθά να κατανοήσετε πότε να χρησιμοποιείτε κάθε προσέγγιση. Επιπλέον, θα επωφεληθούμε από τους λιγότερους περιορισμούς ρυθμού στην Azure για το GPT-5.2 σε σύγκριση με τα Μοντέλα GitHub.

## Κατανόηση της Μηχανικής Προτροπών

Η μηχανική προτροπών αφορά στον σχεδιασμό του κειμένου εισόδου που σας δίνει σταθερά τα αποτελέσματα που χρειάζεστε. Δεν είναι απλά το να κάνετε ερωτήσεις - είναι το να δομείτε τα αιτήματα ώστε το μοντέλο να καταλαβαίνει ακριβώς τι θέλετε και πώς να το παραδώσει.

Σκεφτείτε το σαν να δίνετε εντολές σε έναν συνάδελφο. Το "Διόρθωσε το σφάλμα" είναι αόριστο. Το "Διόρθωσε την εξαίρεση null pointer στην UserService.java στη γραμμή 45 προσθέτοντας έλεγχο null" είναι συγκεκριμένο. Τα γλωσσικά μοντέλα λειτουργούν με τον ίδιο τρόπο - η συγκεκριμενοποίηση και η δομή έχουν σημασία.

## Πώς Χρησιμοποιεί το LangChain4j

Αυτή η μονάδα παρουσιάζει προηγμένα πρότυπα προτροπών χρησιμοποιώντας το ίδιο θεμέλιο LangChain4j από τις προηγούμενες μονάδες, με έμφαση στη δομή προτροπής και τον έλεγχο συλλογισμού.

<img src="../../../translated_images/el/langchain4j-flow.48e534666213010b.webp" alt="Ροή LangChain4j" width="800"/>

*Πώς το LangChain4j συνδέει τις προτροπές σας με το Azure OpenAI GPT-5.2*

**Εξαρτήσεις** - Η Μονάδα 02 χρησιμοποιεί τις ακόλουθες εξαρτήσεις langchain4j που ορίζονται στο `pom.xml`:
```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```

**Διαμόρφωση OpenAiOfficialChatModel** - [LangChainConfig.java](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/config/LangChainConfig.java)

Το μοντέλο συνομιλίας ρυθμίζεται χειροκίνητα ως bean Spring χρησιμοποιώντας τον επίσημο πελάτη OpenAI, που υποστηρίζει τα τελικά σημεία Azure OpenAI. Η βασική διαφορά από τη Μονάδα 01 είναι πώς διαμορφώνουμε τις προτροπές που στέλνονται στο `chatModel.chat()`, όχι η ρύθμιση του μοντέλου αυτή καθαυτή.

**Συστήματα και Μηνύματα Χρήστη** - [Gpt5PromptService.java](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java)

Το LangChain4j διαχωρίζει τους τύπους μηνυμάτων για σαφήνεια. Το `SystemMessage` ορίζει τη συμπεριφορά και το πλαίσιο του AI (όπως "Είσαι αναλυτής κώδικα"), ενώ το `UserMessage` περιέχει το πραγματικό αίτημα. Αυτός ο διαχωρισμός σας επιτρέπει να διατηρείτε συνεπή συμπεριφορά AI σε διαφορετικά αιτήματα χρηστών.

```java
SystemMessage systemMsg = SystemMessage.from(
    "You are a helpful Java programming expert."
);

UserMessage userMsg = UserMessage.from(
    "Explain what a List is in Java"
);

String response = chatModel.chat(systemMsg, userMsg);
```

<img src="../../../translated_images/el/message-types.93e0779798a17c9d.webp" alt="Αρχιτεκτονική Τύπων Μηνυμάτων" width="800"/>

*Το SystemMessage παρέχει επίμονο πλαίσιο ενώ τα UserMessages περιέχουν μεμονωμένα αιτήματα*

**MessageWindowChatMemory για Πολύ-γύρους** - Για το πρότυπο πολύ-γύρων συνομιλίας, επαναχρησιμοποιούμε το `MessageWindowChatMemory` από τη Μονάδα 01. Κάθε συνεδρία έχει τη δική της μνήμη αποθηκευμένη σε `Map<String, ChatMemory>`, επιτρέποντας πολλαπλές ταυτόχρονες συνομιλίες χωρίς ανάμειξη πλαισίου.

**Πρότυπα Προτροπών** - Το πραγματικό επίκεντρο εδώ είναι η μηχανική προτροπών, όχι νέα API του LangChain4j. Κάθε πρότυπο (χαμηλός ενθουσιασμός, υψηλός ενθουσιασμός, εκτέλεση εργασιών, κτλ.) χρησιμοποιεί την ίδια μέθοδο `chatModel.chat(prompt)` αλλά με προσεκτικά δομημένα κείμενα προτροπών. Οι ετικέτες XML, οι οδηγίες και η μορφοποίηση είναι όλα μέρος του κειμένου της προτροπής, όχι χαρακτηριστικά του LangChain4j.

**Έλεγχος Συλλογισμού** - Ο συλλογισμός του GPT-5.2 ελέγχεται μέσω οδηγιών προτροπής όπως "μέγιστο 2 βήματα συλλογισμού" ή "εξερεύνησε διεξοδικά". Αυτές είναι τεχνικές μηχανικής προτροπών, όχι ρυθμίσεις LangChain4j. Η βιβλιοθήκη απλώς μεταφέρει τις προτροπές σας στο μοντέλο.

Το βασικό συμπέρασμα: Το LangChain4j παρέχει την υποδομή (σύνδεση μοντέλου μέσω [LangChainConfig.java](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/config/LangChainConfig.java), μνήμη, διαχείριση μηνυμάτων μέσω [Gpt5PromptService.java](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java)), ενώ αυτή η μονάδα διδάσκει πώς να δημιουργήσετε αποτελεσματικές προτροπές μέσα σε αυτή την υποδομή.

## Οι Βασικά Πρότυπα

Δεν χρειάζονται όλα τα προβλήματα την ίδια προσέγγιση. Κάποιες ερωτήσεις χρειάζονται γρήγορες απαντήσεις, άλλες βαθιά σκέψη. Κάποιες χρειάζονται ορατό συλλογισμό, άλλες απλά αποτελέσματα. Αυτή η μονάδα καλύπτει οκτώ πρότυπα προτροπών - κάθε ένα βελτιστοποιημένο για διαφορετικά σενάρια. Θα πειραματιστείτε με όλα για να μάθετε πότε λειτουργεί καλύτερα κάθε προσέγγιση.

<img src="../../../translated_images/el/eight-patterns.fa1ebfdf16f71e9a.webp" alt="Οκτώ Πρότυπα Μηχανικής Προτροπών" width="800"/>

*Επισκόπηση των οκτώ προτύπων μηχανικής προτροπών και των περιπτώσεων χρήσης τους*

<img src="../../../translated_images/el/reasoning-effort.db4a3ba5b8e392c1.webp" alt="Σύγκριση Προσπάθειας Συλλογισμού" width="800"/>

*Χαμηλός ενθουσιασμός (γρήγορος, άμεσος) vs Υψηλός ενθουσιασμός (διεξοδικός, διερευνητικός) προσεγγίσεις συλλογισμού*

**Χαμηλός Ενθουσιασμός (Γρήγορο & Εστιασμένο)** - Για απλές ερωτήσεις που θέλετε γρήγορες, άμεσες απαντήσεις. Το μοντέλο κάνει ελάχιστο συλλογισμό - μέγιστο 2 βήματα. Χρησιμοποιήστε το για υπολογισμούς, αναζητήσεις ή απλές ερωτήσεις.

```java
String prompt = """
    <reasoning_effort>low</reasoning_effort>
    <instruction>maximum 2 reasoning steps</instruction>
    
    What is 15% of 200?
    """;

String response = chatModel.chat(prompt);
```

> 💡 **Εξερευνήστε με το GitHub Copilot:** Ανοίξτε το [`Gpt5PromptService.java`](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java) και ρωτήστε:
> - "Ποια είναι η διαφορά μεταξύ των προτύπων προτροπών χαμηλού και υψηλού ενθουσιασμού;"
> - "Πώς βοηθούν οι ετικέτες XML στη δομή της απάντησης του AI;"
> - "Πότε πρέπει να χρησιμοποιώ πρότυπα αυτο-αντανάκλασης αντί άμεσων οδηγιών;"

**Υψηλός Ενθουσιασμός (Βαθύς & Διεξοδικός)** - Για σύνθετα προβλήματα όπου θέλετε εκτενή ανάλυση. Το μοντέλο εξερευνά διεξοδικά και δείχνει λεπτομερή συλλογισμό. Χρησιμοποιήστε το για σχεδιασμό συστημάτων, αρχιτεκτονικές αποφάσεις ή σύνθετη έρευνα.

```java
String prompt = """
    <reasoning_effort>high</reasoning_effort>
    <instruction>explore thoroughly, show detailed reasoning</instruction>
    
    Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```

**Εκτέλεση Εργασιών (Πρόοδος Βήμα-βήμα)** - Για ροές εργασίας πολλαπλών βημάτων. Το μοντέλο παρέχει ένα αρχικό σχέδιο, αφηγείται κάθε βήμα καθώς δουλεύει, και μετά δίνει περίληψη. Χρησιμοποιήστε το για μεταναστεύσεις, υλοποιήσεις ή οποιαδήποτε διαδικασία πολλαπλών βημάτων.

```java
String prompt = """
    <task>Create a REST endpoint for user registration</task>
    <preamble>Provide an upfront plan</preamble>
    <narration>Narrate each step as you work</narration>
    <summary>Summarize what was accomplished</summary>
    """;

String response = chatModel.chat(prompt);
```

Η προτροπή αλυσίδας σκέψης ζητά ρητά από το μοντέλο να δείξει τη διαδικασία συλλογισμού του, βελτιώνοντας την ακρίβεια σε σύνθετα καθήκοντα. Η ανάλυση βήμα-βήμα βοηθά τόσο τους ανθρώπους όσο και το AI να κατανοήσουν τη λογική.

> **🤖 Δοκιμάστε με [GitHub Copilot](https://github.com/features/copilot) Chat:** Ρωτήστε σχετικά με αυτό το πρότυπο:
> - "Πώς θα προσαρμόζατε το πρότυπο εκτέλεσης εργασιών για μακροχρόνιες λειτουργίες;"
> - "Ποιες είναι οι βέλτιστες πρακτικές για τη δομή προοιμίων εργαλείων σε παραγωγικές εφαρμογές;"
> - "Πώς μπορώ να καταγράψω και να εμφανίσω ενδιάμεσα ενημερώσεις προόδου σε UI;"

<img src="../../../translated_images/el/task-execution-pattern.9da3967750ab5c1e.webp" alt="Πρότυπο Εκτέλεσης Εργασιών" width="800"/>

*Σχεδιασμός → Εκτέλεση → Περίληψη ροής εργασίας για εργασίες πολλαπλών βημάτων*

**Αυτοανακλαστικός Κώδικας** - Για παραγωγικού επιπέδου γεννήτρια κώδικα. Το μοντέλο παράγει κώδικα, τον ελέγχει με βάση κριτήρια ποιότητας, και τον βελτιώνει επαναληπτικά. Χρησιμοποιήστε το όταν χτίζετε νέα χαρακτηριστικά ή υπηρεσίες.

```java
String prompt = """
    <task>Create an email validation service</task>
    <quality_criteria>
    - Correct logic and error handling
    - Best practices (clean code, proper naming)
    - Performance optimization
    - Security considerations
    </quality_criteria>
    <instruction>Generate code, evaluate against criteria, improve iteratively</instruction>
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/el/self-reflection-cycle.6f71101ca0bd28cc.webp" alt="Κύκλος Αυτο-Αντανάκλασης" width="800"/>

*Επαναληπτικός βελτιωτικός κύκλος - παραγωγή, αξιολόγηση, εντοπισμός ζητημάτων, βελτίωση, επανάληψη*

**Δομημένη Ανάλυση** - Για συνεπή αξιολόγηση. Το μοντέλο αναθεωρεί κώδικα χρησιμοποιώντας ένα σταθερό πλαίσιο (ορθότητα, πρακτικές, απόδοση, ασφάλεια). Χρησιμοποιήστε το για ανασκοπήσεις κώδικα ή αξιολογήσεις ποιότητας.

```java
String prompt = """
    <code>
    public List getUsers() {
        return database.query("SELECT * FROM users");
    }
    </code>
    
    <framework>
    Evaluate using these categories:
    1. Correctness - Logic and functionality
    2. Best Practices - Code quality
    3. Performance - Efficiency concerns
    4. Security - Vulnerabilities
    </framework>
    """;

String response = chatModel.chat(prompt);
```

> - "Ποιος είναι ο καλύτερος τρόπος για να αναλύσω και να ενεργήσω με βάση δομημένη έξοδο προγραμματιστικά;"
> - "Πώς διασφαλίζω συνεπή επίπεδα σοβαρότητας σε διαφορετικές συνεδρίες ανασκόπησης;"

<img src="../../../translated_images/el/structured-analysis-pattern.0af3b690b60cf2d6.webp" alt="Πρότυπο Δομημένης Ανάλυσης" width="800"/>

*Πλαίσιο τεσσάρων κατηγοριών για συνεπείς ανασκοπήσεις κώδικα με επίπεδα σοβαρότητας*

**Πολύ-γύρος Συνομιλία** - Για συνομιλίες που χρειάζονται πλαίσιο. Το μοντέλο θυμάται προηγούμενα μηνύματα και χτίζει πάνω σε αυτά. Χρησιμοποιήστε το για διαδραστικές συνεδρίες βοήθειας ή σύνθετο Q&A.

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("What is Spring Boot?"));
AiMessage aiMessage1 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage1);

memory.add(UserMessage.from("Show me an example"));
AiMessage aiMessage2 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage2);
```

<img src="../../../translated_images/el/context-memory.dff30ad9fa78832a.webp" alt="Μνήμη Πλαισίου" width="800"/>

*Πώς συσσωρεύεται το πλαίσιο συνομιλίας σε πολλαπλούς γύρους μέχρι να φτάσει το όριο tokens*

**Βήμα-βήμα Συλλογισμός** - Για προβλήματα που απαιτούν ορατή λογική. Το μοντέλο δείχνει ρητό συλλογισμό για κάθε βήμα. Χρησιμοποιήστε το για μαθηματικά προβλήματα, γρίφους λογικής ή όταν θέλετε να κατανοήσετε τη διαδικασία σκέψης.

```java
String prompt = """
    <instruction>Show your reasoning step-by-step</instruction>
    
    If a train travels 120 km in 2 hours, then stops for 30 minutes,
    then travels another 90 km in 1.5 hours, what is the average speed
    for the entire journey including the stop?
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/el/step-by-step-pattern.a99ea4ca1c48578c.webp" alt="Πρότυπο Βήμα-βήμα" width="800"/>

*Διαίρεση προβλημάτων σε ρητά λογικά βήματα*

**Περιορισμένη Έξοδος** - Για απαντήσεις με συγκεκριμένες απαιτήσεις μορφής. Το μοντέλο ακολουθεί αυστηρά κανόνες μορφής και μήκους. Χρησιμοποιήστε το για περιλήψεις ή όταν χρειάζεστε ακριβή δομή εξόδου.

```java
String prompt = """
    <constraints>
    - Exactly 100 words
    - Bullet point format
    - Technical terms only
    </constraints>
    
    Summarize the key concepts of machine learning.
    """;

String response = chatModel.chat(prompt);
```

*Επιβολή συγκεκριμένων απαιτήσεων μορφής, μήκους και δομής*

## Χρήση Υφιστάμενων Πόρων Azure

**Επαλήθευση ανάπτυξης:**

Βεβαιωθείτε ότι το αρχείο `.env` υπάρχει στον ριζικό φάκελο με διαπιστευτήρια Azure (δημιουργήθηκε κατά τη Μονάδα 01):
```bash
cat ../.env  # Πρέπει να εμφανίσει AZURE_OPENAI_ENDPOINT, API_KEY, DEPLOYMENT
```

**Εκκίνηση εφαρμογής:**

> **Σημείωση:** Αν έχετε ήδη εκκινήσει όλες τις εφαρμογές χρησιμοποιώντας το `./start-all.sh` από τη Μονάδα 01, αυτή η μονάδα τρέχει ήδη στη θύρα 8083. Μπορείτε να παραλείψετε τις εντολές εκκίνησης παρακάτω και να πάτε απευθείας στο http://localhost:8083.

- Εκκινήσετε/σταματήσετε εφαρμογές με ένα κλικ
- Δείτε τα αρχεία καταγραφής εφαρμογών σε πραγματικό χρόνο
- Παρακολουθήσετε την κατάσταση εφαρμογής

Απλά πατήστε το κουμπί αναπαραγωγής δίπλα στο "prompt-engineering" για να εκκινήσετε αυτή τη μονάδα, ή εκκινήστε όλες τις μονάδες ταυτόχρονα.

## Τι Μαθαίνεις Πραγματικά

**Η Προσπάθεια Λογισμού Αλλάζει Τα Πάντα**

Το GPT-5.2 σου επιτρέπει να ελέγχεις την υπολογιστική προσπάθεια μέσω των ερωτημάτων σου. Η χαμηλή προσπάθεια σημαίνει γρήγορες απαντήσεις με ελάχιστη εξερεύνηση. Η υψηλή προσπάθεια σημαίνει ότι το μοντέλο παίρνει χρόνο να σκεφτεί βαθειά. Μαθαίνεις να ταιριάζεις την προσπάθεια με την πολυπλοκότητα της εργασίας – μη χάνεις χρόνο σε απλές ερωτήσεις, αλλά μη βιάζεσαι σε σύνθετες αποφάσεις.

**Η Δομή Καθοδηγεί τη Συμπεριφορά**

Παρατήρησες τις ετικέτες XML στα ερωτήματα; Δεν είναι διακοσμητικές. Τα μοντέλα ακολουθούν δομημένες οδηγίες πιο αξιόπιστα από ελεύθερο κείμενο. Όταν χρειάζεσαι πολύβηκες διαδικασίες ή σύνθετη λογική, η δομή βοηθά το μοντέλο να παρακολουθεί που βρίσκεται και τι ακολουθεί.

<img src="../../../translated_images/el/prompt-structure.a77763d63f4e2f89.webp" alt="Δομή Ερωτήματος" width="800"/>

*Ανατομία ενός καλά δομημένου ερωτήματος με σαφείς ενότητες και οργάνωση τύπου XML*

**Ποιότητα Μέσω Αυτοαξιολόγησης**

Τα πρότυπα αυτοαξιολόγησης λειτουργούν κάνοντας ρητά τα κριτήρια ποιότητας. Αντί να ελπίζεις ότι το μοντέλο "θα το κάνει σωστά", τού λες ακριβώς τι σημαίνει "σωστό": σωστή λογική, χειρισμός σφαλμάτων, απόδοση, ασφάλεια. Το μοντέλο μπορεί μετά να αξιολογήσει τη δική του έξοδο και να βελτιωθεί. Αυτό μετατρέπει τη δημιουργία κώδικα από λαχείο σε διαδικασία.

**Το Συμφραζόμενο Είναι Πεπερασμένο**

Οι συνομιλίες πολλών γύρων λειτουργούν περιλαμβάνοντας το ιστορικό μηνυμάτων σε κάθε αίτημα. Αλλά υπάρχει όριο – κάθε μοντέλο έχει μέγιστο αριθμό tokens. Καθώς οι συνομιλίες μεγαλώνουν, χρειάζεσαι στρατηγικές για να κρατάς τα σχετικά συμφραζόμενα χωρίς να φτάνεις στο όριο. Αυτό το μάθημα δείχνει πώς δουλεύει η μνήμη· αργότερα θα μάθεις πότε να συνοψίζεις, πότε να ξεχνάς, και πότε να ανακτάς.
