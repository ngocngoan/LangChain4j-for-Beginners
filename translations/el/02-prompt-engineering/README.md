## Τι θα Μάθετε

<img src="../../../translated_images/el/what-youll-learn.c68269ac048503b2.webp" alt="Τι θα Μάθετε" width="800"/>

Στο προηγούμενο module, είδατε πώς η μνήμη επιτρέπει τη συνομιλιακή AI και χρησιμοποιήσατε τα Μοντέλα GitHub για βασικές αλληλεπιδράσεις. Τώρα θα εστιάσουμε στο πώς κάνετε ερωτήσεις — στις ίδιες τις προτροπές — χρησιμοποιώντας το GPT-5.2 του Azure OpenAI. Ο τρόπος που διαμορφώνετε τις προτροπές σας επηρεάζει δραστικά την ποιότητα των απαντήσεων που λαμβάνετε. Ξεκινάμε με μια ανασκόπηση των βασικών τεχνικών προτροπών και μετά προχωράμε σε οκτώ προηγμένα πρότυπα που εκμεταλλεύονται πλήρως τις δυνατότητες του GPT-5.2.

Θα χρησιμοποιήσουμε το GPT-5.2 γιατί εισάγει τον έλεγχο της σκέψης - μπορείτε να πείτε στο μοντέλο πόση σκέψη να κάνει πριν απαντήσει. Αυτό κάνει τις διαφορετικές στρατηγικές προτροπών πιο εμφανείς και βοηθά να κατανοήσετε πότε να χρησιμοποιείτε κάθε προσέγγιση. Θα επωφεληθούμε επίσης από τους λιγότερους περιορισμούς ρυθμού στο Azure για το GPT-5.2 σε σύγκριση με τα Μοντέλα GitHub.

## Κατανόηση Μηχανικής Προτροπής

<img src="../../../translated_images/el/what-is-prompt-engineering.5c392a228a1f5823.webp" alt="Τι είναι η Μηχανική Προτροπής;" width="800"/>

Η μηχανική προτροπής αφορά τον σχεδιασμό του κειμένου εισόδου που διασφαλίζει σταθερά τα επιθυμητά αποτελέσματα. Δεν πρόκειται μόνο για το να κάνετε ερωτήσεις - αλλά για τη δομή των αιτημάτων έτσι ώστε το μοντέλο να κατανοεί ακριβώς τι θέλετε και πώς να το παραδώσει.

Σκεφτείτε το σαν να δίνετε οδηγίες σε έναν συνάδελφο. Το "Διόρθωσε το σφάλμα" είναι ασαφές. Το "Διόρθωσε την εξαίρεση null pointer στη γραμμή 45 του UserService.java προσθέτοντας έλεγχο null" είναι συγκεκριμένο. Τα γλωσσικά μοντέλα λειτουργούν με τον ίδιο τρόπο - η συγκεκριμενοποίηση και η δομή έχουν σημασία.

<img src="../../../translated_images/el/how-langchain4j-fits.dfff4b0aa5f7812d.webp" alt="Πώς ταιριάζει το LangChain4j" width="800"/>

Το LangChain4j παρέχει την υποδομή — συνδέσεις μοντέλου, μνήμη και τύπους μηνυμάτων — ενώ τα πρότυπα προτροπών είναι απλώς προσεκτικά δομημένο κείμενο που στέλνετε μέσω αυτής της υποδομής. Τα βασικά δομικά στοιχεία είναι το `SystemMessage` (που ορίζει τη συμπεριφορά και τον ρόλο της AI) και το `UserMessage` (που μεταφέρει το πραγματικό σας αίτημα).

## Βασικές Αρχές Μηχανικής Προτροπής

<img src="../../../translated_images/el/five-patterns-overview.160f35045ffd2a94.webp" alt="Επισκόπηση Πέντε Προτύπων Μηχανικής Προτροπής" width="800"/>

Πριν εμβαθύνουμε στα προηγμένα πρότυπα αυτού του module, ας αναθεωρήσουμε πέντε θεμελιώδεις τεχνικές προτροπής. Αυτά είναι τα δομικά στοιχεία που κάθε μηχανικός προτροπών θα πρέπει να γνωρίζει. Αν έχετε ήδη περάσει από το [Quick Start module](../00-quick-start/README.md#2-prompt-patterns), τα έχετε δει σε δράση — εδώ είναι το εννοιολογικό πλαίσιο πίσω από αυτά.

### Προτροπή Μηδενικού Παραδείγματος

Η απλούστερη προσέγγιση: δίνετε στο μοντέλο μια άμεση οδηγία χωρίς παραδείγματα. Το μοντέλο βασίζεται αποκλειστικά στην εκπαίδευσή του για να καταλάβει και να εκτελέσει την εργασία. Αυτό λειτουργεί καλά για απλά αιτήματα όπου η αναμενόμενη συμπεριφορά είναι προφανής.

<img src="../../../translated_images/el/zero-shot-prompting.7abc24228be84e6c.webp" alt="Προτροπή Μηδενικού Παραδείγματος" width="800"/>

*Άμεση οδηγία χωρίς παραδείγματα — το μοντέλο συμπεραίνει την εργασία μόνο από την οδηγία*

```java
String prompt = "Classify this sentiment: 'I absolutely loved the movie!'";
String response = model.chat(prompt);
// Απάντηση: "Θετικό"
```

**Πότε να τη χρησιμοποιήσετε:** Απλές κατηγοριοποιήσεις, άμεσες ερωτήσεις, μεταφράσεις ή οποιαδήποτε εργασία που το μοντέλο μπορεί να χειριστεί χωρίς επιπλέον καθοδήγηση.

### Προτροπή Λίγων Παραδειγμάτων

Παρέχετε παραδείγματα που δείχνουν το μοτίβο που θέλετε να ακολουθήσει το μοντέλο. Το μοντέλο μαθαίνει τη ζητούμενη μορφή εισόδου-εξόδου από τα παραδείγματα σας και την εφαρμόζει σε νέα δεδομένα. Αυτό βελτιώνει δραστικά τη συνέπεια για εργασίες όπου το επιθυμητό φορμάτ ή η συμπεριφορά δεν είναι προφανή.

<img src="../../../translated_images/el/few-shot-prompting.9d9eace1da88989a.webp" alt="Προτροπή Λίγων Παραδειγμάτων" width="800"/>

*Μάθηση από παραδείγματα — το μοντέλο αναγνωρίζει το μοτίβο και το εφαρμόζει σε νέα εισαγωγικά*

```java
String prompt = """
    Classify the sentiment as positive, negative, or neutral.
    
    Examples:
    Text: "This product exceeded my expectations!" → Positive
    Text: "It's okay, nothing special." → Neutral
    Text: "Waste of money, very disappointed." → Negative
    
    Now classify this:
    Text: "Best purchase I've made all year!"
    """;
String response = model.chat(prompt);
```

**Πότε να τη χρησιμοποιήσετε:** Προσαρμοσμένες κατηγοριοποιήσεις, συνεπής μορφοποίηση, εργασίες ειδικού τομέα ή όταν τα αποτελέσματα μηδενικού παραδείγματος είναι ασυνεπή.

### Αλυσίδα Σκέψης

Ζητήστε από το μοντέλο να δείξει τη λογική του βήμα προς βήμα. Αντί να προχωρά απευθείας σε απάντηση, το μοντέλο αναλύει το πρόβλημα και δουλεύει κάθε κομμάτι ξεχωριστά. Αυτό βελτιώνει την ακρίβεια σε μαθηματικά, λογική και πολύπλοκες διεργασίες σκέψης.

<img src="../../../translated_images/el/chain-of-thought.5cff6630e2657e2a.webp" alt="Προτροπή Αλυσίδας Σκέψης" width="800"/>

*Λογισμός βήμα προς βήμα — ανάλυση σύνθετων προβλημάτων σε ρητά λογικά βήματα*

```java
String prompt = """
    Problem: A store has 15 apples. They sell 8 apples and then 
    receive a shipment of 12 more apples. How many apples do they have now?
    
    Let's solve this step-by-step:
    """;
String response = model.chat(prompt);
// Το μοντέλο δείχνει: 15 - 8 = 7, μετά 7 + 12 = 19 μήλα
```

**Πότε να τη χρησιμοποιήσετε:** Μαθηματικά προβλήματα, γρίφους λογικής, αποσφαλμάτωση ή οποιαδήποτε εργασία όπου η παρουσίαση της διαδικασίας σκέψης βελτιώνει την ακρίβεια και την εμπιστοσύνη.

### Προτροπή με Βάση Ρόλους

Ορίστε μια περσόνα ή ρόλο για την AI πριν κάνετε την ερώτηση. Αυτό παρέχει το πλαίσιο που διαμορφώνει τον τόνο, το βάθος και την εστίαση της απάντησης. Ένας "αρχιτέκτονας λογισμικού" δίνει διαφορετικές συμβουλές από έναν "αναπτυξιακό junior" ή έναν "ελεγκτή ασφαλείας".

<img src="../../../translated_images/el/role-based-prompting.a806e1a73de6e3a4.webp" alt="Προτροπή με Βάση Ρόλους" width="800"/>

*Ορισμός πλαισίου και περσόνας — η ίδια ερώτηση λαμβάνει διαφορετική απάντηση ανάλογα με τον ρόλο*

```java
String prompt = """
    You are an experienced software architect reviewing code.
    Provide a brief code review for this function:
    
    def calculate_total(items):
        total = 0
        for item in items:
            total = total + item['price']
        return total
    """;
String response = model.chat(prompt);
```

**Πότε να τη χρησιμοποιήσετε:** Ανασκοπήσεις κώδικα, διδασκαλία, ανάλυση ειδικού τομέα ή όταν χρειάζεστε απαντήσεις προσαρμοσμένες σε συγκεκριμένο επίπεδο εμπειρίας ή οπτική.

### Πρότυπα Προτροπών

Δημιουργήστε επαναχρησιμοποιήσιμες προτροπές με μεταβλητές θέσεις. Αντί να γράφετε νέα προτροπή κάθε φορά, ορίστε ένα πρότυπο μία φορά και συμπληρώστε διαφορετικές τιμές. Η κλάση `PromptTemplate` του LangChain4j το κάνει εύκολο με τη σύνταξη `{{variable}}`.

<img src="../../../translated_images/el/prompt-templates.14bfc37d45f1a933.webp" alt="Πρότυπα Προτροπών" width="800"/>

*Επαναχρησιμοποιήσιμες προτροπές με μεταβλητές θέσεις — ένα πρότυπο, πολλές χρήσεις*

```java
PromptTemplate template = PromptTemplate.from(
    "What's the best time to visit {{destination}} for {{activity}}?"
);

Prompt prompt = template.apply(Map.of(
    "destination", "Paris",
    "activity", "sightseeing"
));

String response = model.chat(prompt.text());
```

**Πότε να τη χρησιμοποιήσετε:** Επαναλαμβανόμενα ερωτήματα με διαφορετικά εισαγωγικά, παρτίδες επεξεργασίας, κατασκευή επαναχρησιμοποιήσιμων AI ροών εργασίας ή κάθε σενάριο όπου η δομή προτροπής παραμένει σταθερή αλλά τα δεδομένα αλλάζουν.

---

Αυτές οι πέντε βασικές τεχνικές σας παρέχουν ένα στέρεο εργαλείο για τις περισσότερες εργασίες προτροπών. Το υπόλοιπο αυτού του module βασίζεται σε αυτές με **οκτώ προηγμένα πρότυπα** που αξιοποιούν τον έλεγχο σκέψης, την αυτοαξιολόγηση και τη δομημένη έξοδο του GPT-5.2.

## Προηγμένα Πρότυπα

Αφού καλύψουμε τα βασικά, ας προχωρήσουμε στα οκτώ προηγμένα πρότυπα που κάνουν αυτό το module μοναδικό. Δεν χρειάζονται όλα τα προβλήματα την ίδια προσέγγιση. Μερικές ερωτήσεις χρειάζονται γρήγορες απαντήσεις, άλλες βαθιά σκέψη. Μερικές χρειάζονται ορατή λογική, άλλες απλώς αποτελέσματα. Κάθε πρότυπο παρακάτω είναι βελτιστοποιημένο για διαφορετικό σενάριο — και ο έλεγχος σκέψης του GPT-5.2 κάνει τις διαφορές ακόμη πιο έντονες.

<img src="../../../translated_images/el/eight-patterns.fa1ebfdf16f71e9a.webp" alt="Οκτώ Πρότυπα Προτροπής" width="800"/>

*Επισκόπηση των οκτώ προτύπων μηχανικής προτροπής και των περιπτώσεων χρήσης τους*

<img src="../../../translated_images/el/reasoning-control.5cf85f0fc1d0c1f3.webp" alt="Έλεγχος Σκέψης με το GPT-5.2" width="800"/>

*Ο έλεγχος σκέψης του GPT-5.2 σας επιτρέπει να καθορίσετε πόση σκέψη πρέπει να κάνει το μοντέλο — από γρήγορες άμεσες απαντήσεις έως βαθιά εξερεύνηση*

**Χαμηλός Ενθουσιασμός (Γρήγορο & Εστιασμένο)** - Για απλές ερωτήσεις όπου θέλετε γρήγορες, άμεσες απαντήσεις. Το μοντέλο κάνει ελάχιστη σκέψη - μέχρι 2 βήματα. Χρησιμοποιήστε το αυτό για υπολογισμούς, αναζητήσεις ή απλές ερωτήσεις.

```java
String prompt = """
    <context_gathering>
    - Search depth: very low
    - Bias strongly towards providing a correct answer as quickly as possible
    - Usually, this means an absolute maximum of 2 reasoning steps
    - If you think you need more time, state what you know and what's uncertain
    </context_gathering>
    
    Problem: What is 15% of 200?
    
    Provide your answer:
    """;

String response = chatModel.chat(prompt);
```

**Υψηλός Ενθουσιασμός (Βαθιά & Εξονυχιστική)** - Για πολύπλοκα προβλήματα όπου θέλετε αναλυτική αξιολόγηση. Το μοντέλο εξερευνά εξονυχιστικά και δείχνει λεπτομερή λογισμό. Χρησιμοποιήστε το για σχεδιασμό συστημάτων, αποφάσεις αρχιτεκτονικής ή σύνθετη έρευνα.

```java
String prompt = """
    Analyze this problem thoroughly and provide a comprehensive solution.
    Consider multiple approaches, trade-offs, and important details.
    Show your analysis and reasoning in your response.
    
    Problem: Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```

**Εκτέλεση Εργασιών (Πρόοδος Βήμα-Βήμα)** - Για πολλαπλά βήματα εργασιών. Το μοντέλο παρέχει αρχικό σχέδιο, περιγράφει κάθε βήμα καθώς προχωρά και μετά δίνει περίληψη. Χρησιμοποιήστε το για μετεγκαταστάσεις, υλοποιήσεις ή οποιαδήποτε διαδικασία πολλών βημάτων.

```java
String prompt = """
    <task_execution>
    1. First, briefly restate the user's goal in a friendly way
    
    2. Create a step-by-step plan:
       - List all steps needed
       - Identify potential challenges
       - Outline success criteria
    
    3. Execute each step:
       - Narrate what you're doing
       - Show progress clearly
       - Handle any issues that arise
    
    4. Summarize:
       - What was completed
       - Any important notes
       - Next steps if applicable
    </task_execution>
    
    <tool_preambles>
    - Always begin by rephrasing the user's goal clearly
    - Outline your plan before executing
    - Narrate each step as you go
    - Finish with a distinct summary
    </tool_preambles>
    
    Task: Create a REST endpoint for user registration
    
    Begin execution:
    """;

String response = chatModel.chat(prompt);
```

Η προτροπή αλυσίδας σκέψης ζητά ρητά από το μοντέλο να δείξει τη διαδικασία λογισμού του, βελτιώνοντας την ακρίβεια σε σύνθετες εργασίες. Η ανάλυση βήμα προς βήμα βοηθά τόσο τους ανθρώπους όσο και την AI να κατανοήσουν τη λογική.

<img src="../../../translated_images/el/task-execution-pattern.9da3967750ab5c1e.webp" alt="Πρότυπο Εκτέλεσης Εργασιών" width="800"/>

*Σχέδιο → Εκτέλεση → Περίληψη ροής εργασίας για εργασίες πολλών βημάτων*

**Κώδικας Αυτοανάλυσης** - Για παραγωγικό κώδικα υψηλής ποιότητας. Το μοντέλο δημιουργεί κώδικα σύμφωνα με τα στάνταρ παραγωγής με σωστή διαχείριση σφαλμάτων. Χρησιμοποιήστε το αυτό όταν χτίζετε νέες λειτουργίες ή υπηρεσίες.

```java
String prompt = """
    Generate Java code with production-quality standards: Create an email validation service
    Keep it simple and include basic error handling.
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/el/self-reflection-cycle.6f71101ca0bd28cc.webp" alt="Κύκλος Αυτοανάλυσης" width="800"/>

*Επαναληπτικός βελτιωτικός κύκλος - δημιουργία, αξιολόγηση, εντοπισμός προβλημάτων, βελτίωση, επανάληψη*

**Δομημένη Ανάλυση** - Για συνεπή αξιολόγηση. Το μοντέλο εξετάζει τον κώδικα χρησιμοποιώντας ένα σταθερό πλαίσιο (ορθότητα, πρακτικές, απόδοση, ασφάλεια, συντηρησιμότητα). Χρησιμοποιείστε το για ανασκοπήσεις κώδικα ή αξιολογήσεις ποιότητας.

```java
String prompt = """
    <analysis_framework>
    You are an expert code reviewer. Analyze the code for:
    
    1. Correctness
       - Does it work as intended?
       - Are there logical errors?
    
    2. Best Practices
       - Follows language conventions?
       - Appropriate design patterns?
    
    3. Performance
       - Any inefficiencies?
       - Scalability concerns?
    
    4. Security
       - Potential vulnerabilities?
       - Input validation?
    
    5. Maintainability
       - Code clarity?
       - Documentation?
    
    <output_format>
    Provide your analysis in this structure:
    - Summary: One-sentence overall assessment
    - Strengths: 2-3 positive points
    - Issues: List any problems found with severity (High/Medium/Low)
    - Recommendations: Specific improvements
    </output_format>
    </analysis_framework>
    
    Code to analyze:
    ```
    public List getUsers() {
        return database.query("SELECT * FROM users");
    }
    ```
    Provide your structured analysis:
    """;

String response = chatModel.chat(prompt);
```

## Εξερεύνηση των Προτύπων

Η διαδικτυακή διεπαφή σας επιτρέπει να πειραματιστείτε με διαφορετικές στρατηγικές προτροπής. Κάθε πρότυπο λύνει διαφορετικά προβλήματα – δοκιμάστε τα για να δείτε πότε λάμπει κάθε προσέγγιση.

> **Σημείωση: Streaming vs Non-Streaming** — Κάθε σελίδα προτύπου προσφέρει δύο κουμπιά: **🔴 Stream Response (Live)** και μια επιλογή **Μη ροής (Non-streaming)**. Το streaming χρησιμοποιεί Server-Sent Events (SSE) για να εμφανίζει τα tokens σε πραγματικό χρόνο καθώς το μοντέλο τα παράγει, ώστε να βλέπετε άμεσα την πρόοδο. Η μη-ροή επιλογή περιμένει ολόκληρη την απάντηση πριν την εμφανίσει. Για προτροπές που πυροδοτούν βαθύ συλλογισμό (π.χ. High Eagerness, Self-Reflecting Code), η μη-ροή κλήση μπορεί να διαρκέσει πολύ — μερικές φορές λεπτά — χωρίς ορατική ανατροφοδότηση. **Χρησιμοποιήστε το streaming όταν πειραματίζεστε με σύνθετες προτροπές** ώστε να βλέπετε το μοντέλο να λειτουργεί και να αποφεύγετε την εντύπωση ότι το αίτημα έχει λήξει.
>
> **Σημείωση: Απαιτήσεις Περιηγητή** — Η λειτουργία streaming χρησιμοποιεί το Fetch Streams API (`response.body.getReader()`) που απαιτεί πλήρη περιηγητή (Chrome, Edge, Firefox, Safari). ΔΕΝ λειτουργεί στον ενσωματωμένο Simple Browser του VS Code, καθώς η webview του δεν υποστηρίζει το ReadableStream API. Αν χρησιμοποιείτε τον Simple Browser, τα κουμπιά μη-ροής θα λειτουργούν κανονικά — μόνο τα κουμπιά streaming επηρεάζονται. Ανοίξτε το `http://localhost:8083` σε εξωτερικό περιηγητή για πλήρη εμπειρία.

### Χαμηλός vs Υψηλός Ενθουσιασμός (Low vs High Eagerness)

Κάντε μια απλή ερώτηση όπως «Πόσο είναι το 15% του 200;» χρησιμοποιώντας Low Eagerness. Θα λάβετε άμεση, απευθείας απάντηση. Τώρα κάντε κάτι πολύπλοκο όπως «Σχεδίασε μια στρατηγική caching για API με μεγάλη επισκεψιμότητα» χρησιμοποιώντας High Eagerness. Κάντε κλικ στο **🔴 Stream Response (Live)** και παρακολουθήστε τον αναλυτικό συλλογισμό του μοντέλου να εμφανίζεται token-προς-token. Ίδιο μοντέλο, ίδια δομή ερώτησης – αλλά η προτροπή του λέει πόσο πρέπει να σκεφτεί.

### Εκτέλεση Εργασιών (Tool Preambles)

Οι πολύ-βήμα διαδικασίες επωφελούνται από τον αρχικό σχεδιασμό και την αφήγηση προόδου. Το μοντέλο περιγράφει τι θα κάνει, αφηγείται κάθε βήμα και μετά συνοψίζει τα αποτελέσματα.

### Αυτοαναστοχαστικός Κώδικας

Δοκιμάστε το «Δημιούργησε μια υπηρεσία επικύρωσης email». Αντί να παράγει απλώς κώδικα και να σταματά, το μοντέλο παράγει, αξιολογεί βάσει κριτηρίων ποιότητας, εντοπίζει αδυναμίες και βελτιώνει. Θα δείτε να επαναλαμβάνει αυτή τη διαδικασία μέχρι ο κώδικας να πληροί τα πρότυπα παραγωγής.

### Δομημένη Ανάλυση

Οι αναθεωρήσεις κώδικα χρειάζονται σταθερά πλαίσια αξιολόγησης. Το μοντέλο αναλύει τον κώδικα με σταθερές κατηγορίες (ορθότητα, πρακτικές, απόδοση, ασφάλεια) με επίπεδα σοβαρότητας.

### Πολλαπλοί Γύροι Συνομιλίας (Multi-Turn Chat)

Ρωτήστε «Τι είναι το Spring Boot;» και ακολουθήστε αμέσως με «Δείξε μου ένα παράδειγμα». Το μοντέλο θυμάται την πρώτη ερώτηση και σας δίνει συγκεκριμένο παράδειγμα Spring Boot. Χωρίς μνήμη, η δεύτερη ερώτηση θα ήταν πολύ ασαφής.

### Βήμα-βήμα Σκέψη

Επιλέξτε ένα μαθηματικό πρόβλημα και δοκιμάστε το με Βήμα-βήμα Σκέψη και Low Eagerness. Ο χαμηλός ενθουσιασμός δίνει απλά την απάντηση – γρήγορα αλλά αδιαφανώς. Το βήμα-βήμα εμφανίζει κάθε υπολογισμό και απόφαση.

### Περιορισμένη Έξοδος

Όταν χρειάζεστε συγκεκριμένες μορφές ή αριθμό λέξεων, αυτό το πρότυπο επιβάλλει αυστηρή τήρηση. Δοκιμάστε να δημιουργήσετε μια περίληψη με ακριβώς 100 λέξεις σε μορφή κουκκίδων.

## Τι Μάθετε Πραγματικά

**Η Προσπάθεια Σκέψης Τα Αλλάζει Όλα**

Το GPT-5.2 σας επιτρέπει να ελέγχετε την υπολογιστική προσπάθεια μέσα από τις προτροπές σας. Η χαμηλή προσπάθεια σημαίνει γρήγορες απαντήσεις με ελάχιστη εξερεύνηση. Η υψηλή προσπάθεια σημαίνει ότι το μοντέλο αφιερώνει χρόνο για βαθύ στοχασμό. Μαθαίνετε να προσαρμόζετε την προσπάθεια στην πολυπλοκότητα της εργασίας – μην σπαταλάτε χρόνο σε απλές ερωτήσεις, αλλά μην βιάζεστε ούτε στις σύνθετες αποφάσεις.

**Η Δομή Καθοδηγεί τη Συμπεριφορά**

Παρατηρήσατε τις ετικέτες XML στις προτροπές; Δεν είναι απλώς διακοσμητικές. Τα μοντέλα ακολουθούν πιο αξιόπιστα τις δομημένες οδηγίες από το ελεύθερο κείμενο. Όταν χρειάζεστε πολύ-βήμα διαδικασίες ή σύνθετη λογική, η δομή βοηθά το μοντέλο να παρακολουθεί που βρίσκεται και τι ακολουθεί.

<img src="../../../translated_images/el/prompt-structure.a77763d63f4e2f89.webp" alt="Prompt Structure" width="800"/>

*Ανατομία μιας καλά δομημένης προτροπής με σαφείς ενότητες και οργάνωση τύπου XML*

**Ποιότητα Μέσα από Αυτοαξιολόγηση**

Τα πρότυπα αυτοαναστοχασμού λειτουργούν κάνοντας τα κριτήρια ποιότητας ρητά. Αντί να ελπίζετε ότι το μοντέλο «το κάνει σωστά», του λέτε τι σημαίνει ακριβώς «σωστά»: ορθή λογική, χειρισμός σφαλμάτων, απόδοση, ασφάλεια. Το μοντέλο τότε μπορεί να αξιολογήσει το αποτέλεσμα του και να βελτιωθεί. Μετατρέπει τη δημιουργία κώδικα από λοταρία σε μία διαδικασία.

**Το Πλαίσιο είναι Πεπερασμένο**

Οι πολλαπλοί γύροι συνομιλίας λειτουργούν συμπεριλαμβάνοντας το ιστορικό μηνυμάτων με κάθε αίτημα. Αλλά υπάρχει όριο – κάθε μοντέλο έχει μέγιστο αριθμό tokens. Καθώς οι συνομιλίες μεγαλώνουν, θα χρειαστείτε στρατηγικές ώστε να κρατάτε σχετικό το πλαίσιο χωρίς να το ξεπεράσετε. Αυτό το module σας δείχνει πώς λειτουργεί η μνήμη· αργότερα θα μάθετε πότε να συνοψίζετε, πότε να ξεχνάτε και πότε να ανακαλείτε.
