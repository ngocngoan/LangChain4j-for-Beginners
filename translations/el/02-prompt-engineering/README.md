## Τι θα μάθετε

<img src="../../../translated_images/el/what-youll-learn.c68269ac048503b2.webp" alt="Τι θα μάθετε" width="800"/>

Στο προηγούμενο module, είδατε πώς η μνήμη επιτρέπει τις συνομιλιακές AI και χρησιμοποιήσατε τα GitHub Models για βασικές αλληλεπιδράσεις. Τώρα θα εστιάσουμε στο πώς κάνετε ερωτήσεις — τα ίδια τα prompts — χρησιμοποιώντας το GPT-5.2 της Azure OpenAI. Ο τρόπος που δομείτε τα prompts επηρεάζει δραματικά την ποιότητα των απαντήσεων που λαμβάνετε. Ξεκινάμε με μια ανασκόπηση των βασικών τεχνικών ερωτημάτων, και στη συνέχεια προχωράμε σε οκτώ προηγμένα μοτίβα που αξιοποιούν πλήρως τις δυνατότητες του GPT-5.2.

Θα χρησιμοποιήσουμε το GPT-5.2 γιατί εισάγει τον έλεγχο της συλλογιστικής - μπορείτε να πείτε στο μοντέλο πόση σκέψη να κάνει πριν απαντήσει. Αυτό καθιστά τις διαφορετικές στρατηγικές ερωτημάτων πιο εμφανείς και βοηθά να κατανοήσετε πότε να χρησιμοποιείτε κάθε προσέγγιση. Θα επωφεληθούμε επίσης από τους λιγότερους περιορισμούς ρυθμού της Azure για το GPT-5.2 σε σύγκριση με τα GitHub Models.

## Κατανόηση της Μηχανικής Ερωτημάτων

<img src="../../../translated_images/el/what-is-prompt-engineering.5c392a228a1f5823.webp" alt="Τι είναι η Μηχανική Ερωτημάτων;" width="800"/>

Η μηχανική ερωτημάτων αφορά στη σχεδίαση κειμένου εισόδου που σας δίνει σταθερά τα αποτελέσματα που χρειάζεστε. Δεν πρόκειται απλά για το να κάνετε ερωτήσεις - είναι για το πώς διαμορφώνετε τα αιτήματα ώστε το μοντέλο να καταλαβαίνει ακριβώς τι θέλετε και πώς να το παραδώσει.

Σκεφτείτε το σαν να δίνετε εντολές σε έναν συνάδελφο. "Διόρθωσε το σφάλμα" είναι αόριστο. "Διόρθωσε το null pointer exception στο UserService.java γραμμή 45 προσθέτοντας έλεγχο για null" είναι συγκεκριμένο. Τα μοντέλα γλώσσας λειτουργούν με τον ίδιο τρόπο - η εξειδίκευση και η δομή είναι σημαντικές.

<img src="../../../translated_images/el/how-langchain4j-fits.dfff4b0aa5f7812d.webp" alt="Πώς ταιριάζει το LangChain4j" width="800"/>

Το LangChain4j παρέχει την υποδομή — συνδέσεις με μοντέλα, μνήμη, και τύπους μηνυμάτων — ενώ τα μοτίβα ερωτημάτων είναι απλά προσεκτικά δομημένο κείμενο που στέλνετε μέσω αυτής της υποδομής. Τα βασικά δομικά στοιχεία είναι το `SystemMessage` (που ορίζει τη συμπεριφορά και τον ρόλο της AI) και το `UserMessage` (που μεταφέρει το δικό σας αίτημα).

## Βασικά στοιχεία Μηχανικής Ερωτημάτων

<img src="../../../translated_images/el/five-patterns-overview.160f35045ffd2a94.webp" alt="Επισκόπηση Πέντε Μοτίβων Μηχανικής Ερωτημάτων" width="800"/>

Πριν βουτήξουμε στα προηγμένα μοτίβα αυτού του module, ας ανασκοπήσουμε πέντε θεμελιώδεις τεχνικές ερωτημάτων. Αυτά είναι τα βασικά δομικά στοιχεία που κάθε μηχανικός ερωτημάτων θα πρέπει να γνωρίζει. Αν έχετε ήδη δουλέψει στο [Quick Start module](../00-quick-start/README.md#2-prompt-patterns), τα έχετε δει στην πράξη — εδώ είναι το εννοιολογικό πλαίσιο πίσω από αυτά.

### Zero-Shot Prompting

Η απλούστερη προσέγγιση: δώστε στο μοντέλο μια άμεση εντολή χωρίς παραδείγματα. Το μοντέλο βασίζεται αποκλειστικά στην εκπαίδευσή του για να κατανοήσει και να εκτελέσει την εργασία. Αυτό λειτουργεί καλά για απλά αιτήματα όπου η αναμενόμενη συμπεριφορά είναι προφανής.

<img src="../../../translated_images/el/zero-shot-prompting.7abc24228be84e6c.webp" alt="Zero-Shot Prompting" width="800"/>

*Άμεση εντολή χωρίς παραδείγματα — το μοντέλο συμπεραίνει την εργασία μόνο από την εντολή*

```java
String prompt = "Classify this sentiment: 'I absolutely loved the movie!'";
String response = model.chat(prompt);
// Απάντηση: "Θετικό"
```

**Πότε να το χρησιμοποιήσετε:** Απλές κατηγοριοποιήσεις, άμεσες ερωτήσεις, μεταφράσεις ή οποιαδήποτε εργασία το μοντέλο μπορεί να χειριστεί χωρίς πρόσθετη καθοδήγηση.

### Few-Shot Prompting

Παρέχετε παραδείγματα που δείχνουν το μοτίβο που θέλετε να ακολουθήσει το μοντέλο. Το μοντέλο μαθαίνει τη μορφή εισόδου-εξόδου από τα παραδείγματα και την εφαρμόζει σε νέες εισόδους. Αυτό βελτιώνει δραματικά τη συνέπεια σε εργασίες όπου η επιθυμητή μορφή ή συμπεριφορά δεν είναι προφανής.

<img src="../../../translated_images/el/few-shot-prompting.9d9eace1da88989a.webp" alt="Few-Shot Prompting" width="800"/>

*Μάθηση από παραδείγματα — το μοντέλο αναγνωρίζει το μοτίβο και το εφαρμόζει σε νέες εισόδους*

```java
String prompt = """
    Classify the sentiment as positive, negative, or neutral.
    
    Examples:
    Text: "This product exceeded my expectations!" → Positive
    Text: "It's okay, nothing special." → Neutral
    Text: "Waste of money, very disappointed." → Negative
    
    Now classify this:
    Text: "Best purchase I've made all year!"
    """;
String response = model.chat(prompt);
```

**Πότε να το χρησιμοποιήσετε:** Προσαρμοσμένες κατηγοριοποιήσεις, συνεπής μορφοποίηση, εργασίες ειδικού τομέα, ή όταν τα αποτελέσματα zero-shot είναι ασυνεπή.

### Chain of Thought

Ζητήστε από το μοντέλο να δείξει τη σκέψη του βήμα προς βήμα. Αντί να δώσει απευθείας απάντηση, το μοντέλο αναλύει το πρόβλημα και δουλεύει κάθε μέρος με σαφήνεια. Αυτό βελτιώνει την ακρίβεια σε μαθηματικά, λογική και πολύ-βηματικές εργασίες.

<img src="../../../translated_images/el/chain-of-thought.5cff6630e2657e2a.webp" alt="Chain of Thought Prompting" width="800"/>

*Σταδιακή συλλογιστική — διάσπαση πολύπλοκων προβλημάτων σε σαφή λογικά βήματα*

```java
String prompt = """
    Problem: A store has 15 apples. They sell 8 apples and then 
    receive a shipment of 12 more apples. How many apples do they have now?
    
    Let's solve this step-by-step:
    """;
String response = model.chat(prompt);
// Το μοντέλο δείχνει: 15 - 8 = 7, μετά 7 + 12 = 19 μήλα
```

**Πότε να το χρησιμοποιήσετε:** Μαθηματικά προβλήματα, γρίφοι λογικής, αποσφαλμάτωση, ή οποιαδήποτε εργασία όπου η εμφάνιση της διαδικασίας συλλογισμού βελτιώνει την ακρίβεια και την αξιοπιστία.

### Role-Based Prompting

Ορίστε μια προσωπικότητα ή ρόλο για την AI πριν κάνετε την ερώτησή σας. Αυτό παρέχει πλαίσιο που διαμορφώνει τον τόνο, το βάθος και το επίκεντρο της απάντησης. Ένας "software architect" δίνει διαφορετικές συμβουλές από έναν "junior developer" ή έναν "security auditor".

<img src="../../../translated_images/el/role-based-prompting.a806e1a73de6e3a4.webp" alt="Role-Based Prompting" width="800"/>

*Ορισμός πλαισίου και προσωπικότητας — η ίδια ερώτηση παίρνει διαφορετική απάντηση ανάλογα με τον ανατεθειμένο ρόλο*

```java
String prompt = """
    You are an experienced software architect reviewing code.
    Provide a brief code review for this function:
    
    def calculate_total(items):
        total = 0
        for item in items:
            total = total + item['price']
        return total
    """;
String response = model.chat(prompt);
```

**Πότε να το χρησιμοποιήσετε:** Κριτικές κώδικα, διδασκαλία, αναλύσεις ειδικού τομέα, ή όταν χρειάζεστε απαντήσεις προσαρμοσμένες σε συγκεκριμένο επίπεδο ειδίκευσης ή οπτική.

### Prompt Templates

Δημιουργήστε επαναχρησιμοποιήσιμα prompts με μεταβλητές θέσεις. Αντί να γράφετε νέο prompt κάθε φορά, ορίστε ένα πρότυπο μία φορά και συμπληρώστε διαφορετικές τιμές. Η κλάση `PromptTemplate` του LangChain4j το κάνει εύκολο με τη σύνταξη `{{variable}}`.

<img src="../../../translated_images/el/prompt-templates.14bfc37d45f1a933.webp" alt="Prompt Templates" width="800"/>

*Επαναχρησιμοποιήσιμα prompts με μεταβλητές θέσεις — ένα πρότυπο, πολλές χρήσεις*

```java
PromptTemplate template = PromptTemplate.from(
    "What's the best time to visit {{destination}} for {{activity}}?"
);

Prompt prompt = template.apply(Map.of(
    "destination", "Paris",
    "activity", "sightseeing"
));

String response = model.chat(prompt.text());
```

**Πότε να το χρησιμοποιήσετε:** Επαναλαμβανόμενα ερωτήματα με διαφορετικές εισόδους, επεξεργασία παρτίδας, δημιουργία επαναχρησιμοποιήσιμων ροών εργασίας AI, ή κάθε σενάριο όπου η δομή του prompt παραμένει η ίδια αλλά αλλάζουν τα δεδομένα.

---

Αυτά τα πέντε βασικά σας δίνουν ένα ισχυρό εργαλείο για τις περισσότερες εργασίες ερωτημάτων. Η υπόλοιπη ενότητα χτίζει πάνω τους με **οκτώ προηγμένα μοτίβα** που αξιοποιούν τον έλεγχο συλλογιστικής, την αυτοαξιολόγηση και τις δομημένες δυνατότητες εξόδου του GPT-5.2.

## Προηγμένα Μοτίβα

Με τα βασικά καλυμμένα, ας προχωρήσουμε στα οκτώ προηγμένα μοτίβα που κάνουν μοναδικό αυτό το module. Όχι όλα τα προβλήματα απαιτούν την ίδια προσέγγιση. Κάποιες ερωτήσεις χρειάζονται γρήγορες απαντήσεις, άλλες βαθιά σκέψη. Κάποιες χρειάζονται ορατή συλλογιστική, άλλες μόνο αποτελέσματα. Κάθε μοτίβο παρακάτω είναι βελτιστοποιημένο για διαφορετικό σενάριο — και ο έλεγχος συλλογιστικής του GPT-5.2 κάνει τις διαφορές ακόμα πιο έντονες.

<img src="../../../translated_images/el/eight-patterns.fa1ebfdf16f71e9a.webp" alt="Οκτώ Μοτίβα Prompting" width="800"/>

*Επισκόπηση των οκτώ μοτίβων μηχανικής ερωτημάτων και των χρήσεων τους*

<img src="../../../translated_images/el/reasoning-control.5cf85f0fc1d0c1f3.webp" alt="Έλεγχος Συλλογιστικής με GPT-5.2" width="800"/>

*Ο έλεγχος συλλογιστικής του GPT-5.2 σας επιτρέπει να καθορίσετε πόση σκέψη πρέπει να κάνει το μοντέλο — από γρήγορες άμεσες απαντήσεις έως βαθιά εξερεύνηση*

<img src="../../../translated_images/el/reasoning-effort.db4a3ba5b8e392c1.webp" alt="Σύγκριση Προσπάθειας Συλλογιστικής" width="800"/>

*Χαμηλός ζήλος (γρήγορη, άμεση) vs Υψηλός ζήλος (εξονυχιστική, διερευνητική) προσεγγίσεις συλλογιστικής*

**Χαμηλός Ζήλος (Γρήγορο & Εστιασμένο)** - Για απλές ερωτήσεις όπου θέλετε γρήγορες, άμεσες απαντήσεις. Το μοντέλο κάνει ελάχιστη συλλογιστική - μέγιστο 2 βήματα. Χρησιμοποιήστε το για υπολογισμούς, αναζητήσεις ή απλές ερωτήσεις.

```java
String prompt = """
    <reasoning_effort>low</reasoning_effort>
    <instruction>maximum 2 reasoning steps</instruction>
    
    What is 15% of 200?
    """;

String response = chatModel.chat(prompt);
```

> - "Ποια είναι η διαφορά μεταξύ των μοτίβων ερωτημάτων χαμηλού και υψηλού ζήλου;"
> - "Πώς βοηθούν τα XML tags στα prompts στη δομή της απάντησης της AI;"
> - "Πότε πρέπει να χρησιμοποιώ τα μοτίβα αυτο-αντανάκλασης vs άμεσης εντολής;"

**Υψηλός Ζήλος (Βαθύ & Εξονυχιστικό)** - Για σύνθετα προβλήματα όπου θέλετε ολοκληρωμένη ανάλυση. Το μοντέλο ερευνά διεξοδικά και δείχνει λεπτομερή συλλογιστική. Χρησιμοποιήστε το για σχεδιασμό συστημάτων, αποφάσεις αρχιτεκτονικής ή σύνθετη έρευνα.

```java
String prompt = """
    <reasoning_effort>high</reasoning_effort>
    <instruction>explore thoroughly, show detailed reasoning</instruction>
    
    Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```

**Εκτέλεση Εργασιών (Πρόοδος Βήμα προς Βήμα)** - Για ροές εργασίας με πολλά βήματα. Το μοντέλο παρέχει ένα αρχικό πλάνο, αφηγείται κάθε βήμα που εκτελεί, και μετά δίνει μια περίληψη. Χρησιμοποιήστε το για μετεγκαταστάσεις, υλοποιήσεις ή οποιαδήποτε πολύ-βηματική διαδικασία.

```java
String prompt = """
    <task>Create a REST endpoint for user registration</task>
    <preamble>Provide an upfront plan</preamble>
    <narration>Narrate each step as you work</narration>
    <summary>Summarize what was accomplished</summary>
    """;

String response = chatModel.chat(prompt);
```

Το Chain-of-Thought prompting ζητά ρητά από το μοντέλο να δείξει τη συλλογιστική του διαδικασία, βελτιώνοντας την ακρίβεια σύνθετων εργασιών. Η ανάλυση βήμα προς βήμα βοηθά τόσο τους ανθρώπους όσο και την AI να κατανοήσουν τη λογική.

<img src="../../../translated_images/el/task-execution-pattern.9da3967750ab5c1e.webp" alt="Μοτίβο Εκτέλεσης Εργασίας" width="800"/>

*Πλάνο → Εκτέλεση → Περίληψη ροής εργασίας για πολύ-βηματικές εργασίες*

**Αυτο-ανακλαστικός Κώδικας** - Για δημιουργία κώδικα παραγωγικής ποιότητας. Το μοντέλο παράγει κώδικα, ελέγχει αν πληροί ποιοτικά κριτήρια, και τον βελτιώνει επαναληπτικά. Χρησιμοποιήστε το για δημιουργία νέων λειτουργιών ή υπηρεσιών.

```java
String prompt = """
    <task>Create an email validation service</task>
    <quality_criteria>
    - Correct logic and error handling
    - Best practices (clean code, proper naming)
    - Performance optimization
    - Security considerations
    </quality_criteria>
    <instruction>Generate code, evaluate against criteria, improve iteratively</instruction>
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/el/self-reflection-cycle.6f71101ca0bd28cc.webp" alt="Κύκλος Αυτο-Αντανάκλασης" width="800"/>

*Επαναληπτικός κύκλος βελτίωσης - παραγωγή, αξιολόγηση, ανίχνευση προβλημάτων, βελτίωση, επανάληψη*

**Δομημένη Ανάλυση** - Για συνεπή αξιολόγηση. Το μοντέλο αξιολογεί κώδικα με σταθερό πλαίσιο (ορθότητα, πρακτικές, απόδοση, ασφάλεια). Χρησιμοποιήστε το για κριτικές κώδικα ή αξιολογήσεις ποιότητας.

```java
String prompt = """
    <code>
    public List getUsers() {
        return database.query("SELECT * FROM users");
    }
    </code>
    
    <framework>
    Evaluate using these categories:
    1. Correctness - Logic and functionality
    2. Best Practices - Code quality
    3. Performance - Efficiency concerns
    4. Security - Vulnerabilities
    </framework>
    """;

String response = chatModel.chat(prompt);
```

> **🤖 Δοκιμάστε με [GitHub Copilot](https://github.com/features/copilot) Chat:** Ρωτήστε για δομημένη ανάλυση:
> - "Πώς μπορώ να προσαρμόσω το πλαίσιο ανάλυσης για διαφορετικούς τύπους κριτικών κώδικα;"
> - "Ποιος είναι ο καλύτερος τρόπος να αναλύσω και να ενεργήσω πάνω σε δομημένη έξοδο προγραμματιστικά;"
> - "Πώς διασφαλίζω συνεπείς βαθμίδες σοβαρότητας σε διαφορετικές συνεδρίες ανασκόπησης;"

<img src="../../../translated_images/el/structured-analysis-pattern.0af3b690b60cf2d6.webp" alt="Μοτίβο Δομημένης Ανάλυσης" width="800"/>

*Πλαίσιο τεσσάρων κατηγοριών για συνεπή κριτική κώδικα με επίπεδα σοβαρότητας*

**Πολύστροφες Συνομιλίες** - Για συζητήσεις που χρειάζονται πλαίσιο. Το μοντέλο θυμάται προηγούμενα μηνύματα και χτίζει πάνω τους. Χρησιμοποιήστε το για διαδραστικές συνεδρίες βοήθειας ή σύνθετο Q&A.

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("What is Spring Boot?"));
AiMessage aiMessage1 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage1);

memory.add(UserMessage.from("Show me an example"));
AiMessage aiMessage2 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage2);
```

<img src="../../../translated_images/el/context-memory.dff30ad9fa78832a.webp" alt="Μνήμη Πλαισίου" width="800"/>

*Πώς συσσωρεύεται το πλαίσιο της συνομιλίας σε πολλές στροφές μέχρι να φτάσει το όριο token*

## Εξερεύνηση των Προτύπων

Η διαδικτυακή διεπαφή σάς επιτρέπει να πειραματιστείτε με διαφορετικές στρατηγικές προτροπής. Κάθε πρότυπο λύνει διαφορετικά προβλήματα - δοκιμάστε τα για να δείτε πότε λάμπει κάθε προσέγγιση.

Κάντε μια απλή ερώτηση όπως "Πόσο είναι το 15% του 200;" χρησιμοποιώντας Χαμηλή Επιθυμία. Θα λάβετε άμεση, απευθείας απάντηση. Τώρα ρωτήστε κάτι πιο σύνθετο όπως "Σχεδιάστε μια στρατηγική caching για ένα API με υψηλή κίνηση" χρησιμοποιώντας Υψηλή Επιθυμία. Παρακολουθήστε πώς το μοντέλο επιβραδύνει και παρέχει λεπτομερή επιχειρηματολογία. Ίδιο μοντέλο, ίδια δομή ερώτησης - αλλά η προτροπή του λέει πόσο να σκεφτεί.

## Τι Μαθαίνετε Πραγματικά

**Η Προσπάθεια Λογικής Αλλάζει τα Πάντα**

Το GPT-5.2 σας επιτρέπει να ελέγχετε την υπολογιστική προσπάθεια μέσω των prompts σας. Χαμηλή προσπάθεια σημαίνει γρήγορες απαντήσεις με ελάχιστη εξερεύνηση. Υψηλή προσπάθεια σημαίνει ότι το μοντέλο αφιερώνει χρόνο για βαθύ στοχασμό. Μαθαίνετε να ταιριάζετε την προσπάθεια με την πολυπλοκότητα της εργασίας – μην ξοδεύετε χρόνο σε απλές ερωτήσεις, αλλά μην βιάζεστε και σε πολύπλοκες αποφάσεις.

**Η Δομή Καθοδηγεί τη Συμπεριφορά**

Παρατηρήσατε τα XML tags στα prompts; Δεν είναι διακοσμητικά. Τα μοντέλα ακολουθούν τις δομημένες οδηγίες πιο αξιόπιστα από τον ελεύθερο κείμενο. Όταν χρειάζεστε διαδικασίες πολλαπλών βημάτων ή πολύπλοκη λογική, η δομή βοηθά το μοντέλο να παρακολουθεί πού βρίσκεται και τι ακολουθεί.

<img src="../../../translated_images/el/prompt-structure.a77763d63f4e2f89.webp" alt="Prompt Structure" width="800"/>

*Ανατομία ενός καλά δομημένου prompt με σαφείς ενότητες και οργάνωση τύπου XML*

**Ποιότητα Μέσω Αυτοαξιολόγησης**

Τα μοτίβα αυτοαποτίμησης λειτουργούν κάνοντας ρητά τα κριτήρια ποιότητας. Αντί να ελπίζετε ότι το μοντέλο "θα το κάνει σωστά", το λέτε ακριβώς τι σημαίνει "σωστά": σωστή λογική, χειρισμός σφαλμάτων, απόδοση, ασφάλεια. Το μοντέλο μπορεί τότε να αξιολογήσει την παραγωγή του και να βελτιωθεί. Αυτό μετατρέπει τη δημιουργία κώδικα από λοταρία σε μια διαδικασία.

**Το Πλαίσιο Είναι Πεπερασμένο**

Οι συνομιλίες πολλαπλών γύρων λειτουργούν με την προσθήκη ιστορικού μηνυμάτων σε κάθε αίτημα. Αλλά υπάρχει όριο – κάθε μοντέλο έχει μέγιστο αριθμό tokens. Καθώς οι συνομιλίες μεγαλώνουν, θα χρειαστείτε στρατηγικές για να κρατήσετε το σχετικό πλαίσιο χωρίς να φτάσετε σε όριο. Αυτή η μονάδα σας δείχνει πώς λειτουργεί η μνήμη· αργότερα θα μάθετε πότε να συνοψίζετε, πότε να ξεχνάτε και πότε να ανακτάτε.
