## Τι θα Μάθετε

<img src="../../../translated_images/el/what-youll-learn.c68269ac048503b2.webp" alt="Τι θα Μάθετε" width="800"/>

Στην προηγούμενη ενότητα, είδατε πώς η μνήμη επιτρέπει την συνομιλιακή τεχνητή νοημοσύνη και χρησιμοποιήσατε Μοντέλα GitHub για βασικές αλληλεπιδράσεις. Τώρα θα εστιάσουμε στο πώς κάνετε ερωτήσεις — τις προτροπές αυτές καθαυτές — χρησιμοποιώντας το GPT-5.2 του Azure OpenAI. Ο τρόπος με τον οποίο δομείτε τις προτροπές σας επηρεάζει δραματικά την ποιότητα των απαντήσεων που λαμβάνετε. Ξεκινάμε με μια ανασκόπηση των θεμελιωδών τεχνικών προτροπής, και μετά προχωράμε σε οκτώ προηγμένα πρότυπα που εκμεταλλεύονται πλήρως τις δυνατότητες του GPT-5.2.

Θα χρησιμοποιήσουμε το GPT-5.2 επειδή εισάγει τον έλεγχο της σκέψης — μπορείτε να πείτε στο μοντέλο πόση σκέψη να κάνει πριν απαντήσει. Αυτό καθιστά τις διαφορετικές στρατηγικές προτροπής πιο εμφανείς και βοηθά να καταλάβετε πότε να χρησιμοποιήσετε κάθε προσέγγιση. ## Κατανόηση της Μηχανικής Προτροπών

<img src="../../../translated_images/el/what-is-prompt-engineering.5c392a228a1f5823.webp" alt="Τι είναι η Μηχανική Προτροπών;" width="800"/>

Η μηχανική προτροπών αφορά το σχεδιασμό του κειμένου εισόδου που σας δίνει σταθερά τα αποτελέσματα που χρειάζεστε. Δεν αφορά απλώς το να κάνετε ερωτήσεις — αφορά τη δομολόγηση αιτημάτων ώστε το μοντέλο να καταλαβαίνει ακριβώς τι θέλετε και πώς να το παραδώσει.

Σκεφτείτε το σαν να δίνετε οδηγίες σε έναν συνάδελφο. Το «Διόρθωσε το σφάλμα» είναι ασαφές. Το «Διόρθωσε το σφάλμα null pointer στην UserService.java στη γραμμή 45 προσθέτοντας έναν έλεγχο null» είναι συγκεκριμένο. Τα γλωσσικά μοντέλα λειτουργούν με τον ίδιο τρόπο — η ακρίβεια και η δομή έχουν σημασία.

<img src="../../../translated_images/el/how-langchain4j-fits.dfff4b0aa5f7812d.webp" alt="Πώς ταιριάζει το LangChain4j" width="800"/>

Το LangChain4j παρέχει τη δομή — συνδέσεις μοντέλων, μνήμη, και τύπους μηνυμάτων — ενώ τα πρότυπα προτροπών είναι απλώς προσεκτικά δομημένο κείμενο που στέλνετε μέσω αυτής της δομής. Τα βασικά δομικά στοιχεία είναι το `SystemMessage` (που ορίζει τη συμπεριφορά και τον ρόλο της AI) και το `UserMessage` (που φέρει το πραγματικό σας αίτημα).

## Βασικά της Μηχανικής Προτροπών

<img src="../../../translated_images/el/five-patterns-overview.160f35045ffd2a94.webp" alt="Επισκόπηση Πέντε Προτύπων Μηχανικής Προτροπών" width="800"/>

Πριν βουτήξουμε στα προηγμένα πρότυπα αυτής της ενότητας, ας ανασκοπήσουμε πέντε θεμελιώδεις τεχνικές προτροπής. Αυτές είναι τα δομικά στοιχεία που κάθε μηχανικός προτροπών πρέπει να γνωρίζει. Αν έχετε ήδη περάσει από την [Ενότητα Γρήγορης Εκκίνησης](../00-quick-start/README.md#2-prompt-patterns), τα έχετε δει σε δράση — εδώ είναι το εννοιολογικό πλαίσιο πίσω από αυτά.

### Προτροπή Μηδενικού Παραδείγματος

Η απλούστερη προσέγγιση: δίνετε στο μοντέλο μια άμεση οδηγία χωρίς παραδείγματα. Το μοντέλο στηρίζεται εξ ολοκλήρου στην εκπαίδευσή του για να καταλάβει και να εκτελέσει την εργασία. Αυτό λειτουργεί καλά για απλά αιτήματα όπου η αναμενόμενη συμπεριφορά είναι προφανής.

<img src="../../../translated_images/el/zero-shot-prompting.7abc24228be84e6c.webp" alt="Προτροπή Μηδενικού Παραδείγματος" width="800"/>

*Άμεση οδηγία χωρίς παραδείγματα — το μοντέλο συμπεραίνει την εργασία μόνο από την οδηγία*

```java
String prompt = "Classify this sentiment: 'I absolutely loved the movie!'";
String response = model.chat(prompt);
// Απάντηση: "Θετικό"
```

**Πότε να χρησιμοποιηθεί:** Απλές κατηγοριοποιήσεις, άμεσες ερωτήσεις, μεταφράσεις ή οποιαδήποτε εργασία το μοντέλο μπορεί να χειριστεί χωρίς επιπλέον καθοδήγηση.

### Προτροπή Λίγων Παραδειγμάτων

Παρέχετε παραδείγματα που δείχνουν το μοτίβο που θέλετε να ακολουθήσει το μοντέλο. Το μοντέλο μαθαίνει τη μορφή εισόδου-εξόδου από τα παραδείγματά σας και την εφαρμόζει σε νέες εισόδους. Αυτό βελτιώνει δραματικά τη συνέπεια σε εργασίες όπου η επιθυμητή μορφή ή συμπεριφορά δεν είναι προφανής.

<img src="../../../translated_images/el/few-shot-prompting.9d9eace1da88989a.webp" alt="Προτροπή Λίγων Παραδειγμάτων" width="800"/>

*Μάθηση από παραδείγματα — το μοντέλο αναγνωρίζει το μοτίβο και το εφαρμόζει σε νέες εισόδους*

```java
String prompt = """
    Classify the sentiment as positive, negative, or neutral.
    
    Examples:
    Text: "This product exceeded my expectations!" → Positive
    Text: "It's okay, nothing special." → Neutral
    Text: "Waste of money, very disappointed." → Negative
    
    Now classify this:
    Text: "Best purchase I've made all year!"
    """;
String response = model.chat(prompt);
```

**Πότε να χρησιμοποιηθεί:** Προσαρμοσμένες κατηγοριοποιήσεις, συνεπής μορφοποίηση, εργασίες ειδικού τομέα, ή όταν τα αποτελέσματα μηδενικού παραδείγματος είναι ασυνεπή.

### Αλυσίδα Σκέψης

Ζητήστε από το μοντέλο να δείξει τη λογική του βήμα-βήμα. Αντί να μεταβαίνει απευθείας σε μια απάντηση, το μοντέλο αναλύει το πρόβλημα και δουλεύει κάθε μέρος ρητά. Αυτό βελτιώνει την ακρίβεια σε μαθηματικά, λογική και εργασίες πολλαπλών βημάτων.

<img src="../../../translated_images/el/chain-of-thought.5cff6630e2657e2a.webp" alt="Προτροπή Αλυσίδας Σκέψης" width="800"/>

*Λογική βήμα-βήμα — ανάλυση σύνθετων προβλημάτων σε ρητά λογικά βήματα*

```java
String prompt = """
    Problem: A store has 15 apples. They sell 8 apples and then 
    receive a shipment of 12 more apples. How many apples do they have now?
    
    Let's solve this step-by-step:
    """;
String response = model.chat(prompt);
// Το μοντέλο δείχνει: 15 - 8 = 7, μετά 7 + 12 = 19 μήλα
```

**Πότε να χρησιμοποιηθεί:** Μαθηματικά προβλήματα, γρίφοι λογικής, debugging, ή οποιαδήποτε εργασία όπου η εμφάνιση της διαδικασίας σκέψης βελτιώνει την ακρίβεια και την εμπιστοσύνη.

### Προτροπή με Βάση Ρόλο

Ορίστε μια προσωποποίηση ή ρόλο για την AI πριν κάνετε την ερώτησή σας. Αυτό παρέχει το πλαίσιο που διαμορφώνει τον τόνο, το βάθος, και την εστίαση της απάντησης. Ένας "αρχιτέκτονας λογισμικού" δίνει διαφορετικές συμβουλές από έναν "νεότερο προγραμματιστή" ή έναν "ελεγκτή ασφαλείας".

<img src="../../../translated_images/el/role-based-prompting.a806e1a73de6e3a4.webp" alt="Προτροπή με Βάση Ρόλο" width="800"/>

*Ορισμός πλαισίου και προσωπικότητας — η ίδια ερώτηση παίρνει διαφορετική απάντηση ανάλογα με τον ανατεθειμένο ρόλο*

```java
String prompt = """
    You are an experienced software architect reviewing code.
    Provide a brief code review for this function:
    
    def calculate_total(items):
        total = 0
        for item in items:
            total = total + item['price']
        return total
    """;
String response = model.chat(prompt);
```

**Πότε να χρησιμοποιηθεί:** Επισκοπήσεις κώδικα, διδασκαλία, ανάλυση ειδικού τομέα, ή όταν χρειάζεστε απαντήσεις προσαρμοσμένες σε συγκεκριμένο επίπεδο εμπειρογνωμοσύνης ή προοπτική.

### Πρότυπα Προτροπών

Δημιουργήστε επαναχρησιμοποιήσιμες προτροπές με μεταβλητές θέσεις. Αντί να γράφετε νέα προτροπή κάθε φορά, ορίστε ένα πρότυπο μία φορά και συμπληρώστε διαφορετικές τιμές. Η κλάση `PromptTemplate` του LangChain4j το κάνει εύκολο με σύνταξη `{{variable}}`.

<img src="../../../translated_images/el/prompt-templates.14bfc37d45f1a933.webp" alt="Πρότυπα Προτροπών" width="800"/>

*Επαναχρησιμοποιήσιμες προτροπές με μεταβλητές θέσεις — ένα πρότυπο, πολλές χρήσεις*

```java
PromptTemplate template = PromptTemplate.from(
    "What's the best time to visit {{destination}} for {{activity}}?"
);

Prompt prompt = template.apply(Map.of(
    "destination", "Paris",
    "activity", "sightseeing"
));

String response = model.chat(prompt.text());
```

**Πότε να χρησιμοποιηθεί:** Επαναλαμβανόμενα ερωτήματα με διαφορετικές εισόδους, μαζική επεξεργασία, δημιουργία επαναχρησιμοποιήσιμων ροών εργασίας AI, ή οποιοδήποτε σενάριο όπου η δομή της προτροπής παραμένει ίδια αλλά τα δεδομένα αλλάζουν.

---

Αυτά τα πέντε βασικά σας δίνουν ένα στιβαρό εργαλείο για τις περισσότερες εργασίες προτροπής. Το υπόλοιπο αυτής της ενότητας βασίζεται σε αυτά με **οκτώ προηγμένα πρότυπα** που αξιοποιούν τον έλεγχο σκέψης, την αυτοαξιολόγηση και τις δομημένες εξόδους του GPT-5.2.

## Προηγμένα Πρότυπα

Με τα βασικά καλυμμένα, ας περάσουμε στα οκτώ προηγμένα πρότυπα που κάνουν αυτήν την ενότητα μοναδική. Όχι όλα τα προβλήματα χρειάζονται την ίδια προσέγγιση. Μερικές ερωτήσεις χρειάζονται γρήγορες απαντήσεις, άλλες βαθιά σκέψη. Μερικές χρειάζονται ορατή σκέψη, άλλες απλώς αποτελέσματα. Κάθε πρότυπο παρακάτω είναι βελτιστοποιημένο για διαφορετικό σενάριο — και ο έλεγχος σκέψης του GPT-5.2 κάνει τις διαφορές ακόμη πιο έντονες.

<img src="../../../translated_images/el/eight-patterns.fa1ebfdf16f71e9a.webp" alt="Οκτώ Πρότυπα Προτροπής" width="800"/>

*Επισκόπηση των οκτώ προτύπων μηχανικής προτροπών και των περιπτώσεων χρήσης τους*

<img src="../../../translated_images/el/reasoning-control.5cf85f0fc1d0c1f3.webp" alt="Έλεγχος Σκέψης με GPT-5.2" width="800"/>

*Ο έλεγχος σκέψης του GPT-5.2 σας επιτρέπει να προσδιορίσετε πόση σκέψη πρέπει να κάνει το μοντέλο — από γρήγορες άμεσες απαντήσεις έως βαθιά εξερεύνηση*

**Χαμηλός Ενθουσιασμός (ΓρήΓανο & Εστιασμένο)** - Για απλές ερωτήσεις όπου θέλετε γρήγορες, άμεσες απαντήσεις. Το μοντέλο κάνει ελάχιστη σκέψη - μέγιστο 2 βήματα. Χρησιμοποιήστε το για υπολογισμούς, αναζητήσεις, ή απλές ερωτήσεις.

```java
String prompt = """
    <context_gathering>
    - Search depth: very low
    - Bias strongly towards providing a correct answer as quickly as possible
    - Usually, this means an absolute maximum of 2 reasoning steps
    - If you think you need more time, state what you know and what's uncertain
    </context_gathering>
    
    Problem: What is 15% of 200?
    
    Provide your answer:
    """;

String response = chatModel.chat(prompt);
```

> 💡 **Εξερευνήστε με GitHub Copilot:** Ανοίξτε το [`Gpt5PromptService.java`](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java) και ρωτήστε:
> - "Ποια η διαφορά ανάμεσα σε πρότυπα προτροπής χαμηλού και υψηλού ενθουσιασμού;"
> - "Πώς βοηθούν οι ετικέτες XML στις προτροπές στη δομή της απάντησης της AI;"
> - "Πότε πρέπει να χρησιμοποιώ πρότυπα αυτοαποτίμησης έναντι άμεσης οδηγίας;"

**Υψηλός Ενθουσιασμός (Βαθύς & Εμπεριστατωμένος)** - Για σύνθετα προβλήματα όπου θέλετε ολοκληρωμένη ανάλυση. Το μοντέλο εξερευνά εις βάθος και δείχνει λεπτομερή επιχειρηματολογία. Χρησιμοποιήστε το για σχεδιασμό συστήματος, αποφάσεις αρχιτεκτονικής, ή σύνθετη έρευνα.

```java
String prompt = """
    Analyze this problem thoroughly and provide a comprehensive solution.
    Consider multiple approaches, trade-offs, and important details.
    Show your analysis and reasoning in your response.
    
    Problem: Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```

**Εκτέλεση Εργασίας (Πρόοδος Βήμα-Βήμα)** - Για ροές εργασίας πολλαπλών βημάτων. Το μοντέλο δίνει ένα σχέδιο εκ των προτέρων, περιγράφει κάθε βήμα κατά την εκτέλεση, και μετά παρέχει μια περίληψη. Χρησιμοποιήστε το για μεταφορές, υλοποιήσεις, ή οποιαδήποτε διαδικασία πολλαπλών βημάτων.

```java
String prompt = """
    <task_execution>
    1. First, briefly restate the user's goal in a friendly way
    
    2. Create a step-by-step plan:
       - List all steps needed
       - Identify potential challenges
       - Outline success criteria
    
    3. Execute each step:
       - Narrate what you're doing
       - Show progress clearly
       - Handle any issues that arise
    
    4. Summarize:
       - What was completed
       - Any important notes
       - Next steps if applicable
    </task_execution>
    
    <tool_preambles>
    - Always begin by rephrasing the user's goal clearly
    - Outline your plan before executing
    - Narrate each step as you go
    - Finish with a distinct summary
    </tool_preambles>
    
    Task: Create a REST endpoint for user registration
    
    Begin execution:
    """;

String response = chatModel.chat(prompt);
```

Η προτροπή Αλυσίδας Σκέψης ζητά ρητά από το μοντέλο να δείξει τη διαδικασία λογικής του, βελτιώνοντας την ακρίβεια σε σύνθετες εργασίες. Η ανάλυση βήμα-βήμα βοηθά τόσο τους ανθρώπους όσο και την AI να κατανοήσουν τη λογική.

> **🤖 Δοκιμάστε με το [GitHub Copilot](https://github.com/features/copilot) Chat:** Ρωτήστε για αυτό το πρότυπο:
> - "Πώς θα προσαρμόσω το πρότυπο εκτέλεσης εργασίας για διεργασίες μεγάλης διάρκειας;"
> - "Ποιες είναι οι βέλτιστες πρακτικές για τη δομή των προειδοποιητικών μηνυμάτων εργαλείων σε εφαρμογές παραγωγής;"
> - "Πώς μπορώ να αιχμαλωτίσω και να εμφανίσω ενημερώσεις προόδου σε μια διεπαφή χρήστη;"

<img src="../../../translated_images/el/task-execution-pattern.9da3967750ab5c1e.webp" alt="Πρότυπο Εκτέλεσης Εργασίας" width="800"/>

*Ροή εργασίας Σχεδίαση → Εκτέλεση → Περίληψη για εργασίες πολλαπλών βημάτων*

**Κώδικας Αυτοαποτίμησης** - Για παραγωγικό κώδικα ποιότητας. Το μοντέλο δημιουργεί κώδικα σύμφωνα με τα πρότυπα παραγωγής με κατάλληλο χειρισμό σφαλμάτων. Χρησιμοποιήστε το όταν χτίζετε νέες λειτουργίες ή υπηρεσίες.

```java
String prompt = """
    Generate Java code with production-quality standards: Create an email validation service
    Keep it simple and include basic error handling.
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/el/self-reflection-cycle.6f71101ca0bd28cc.webp" alt="Κύκλος Αυτοαποτίμησης" width="800"/>

*Επαναληπτικός βρόχος βελτίωσης - δημιουργία, αξιολόγηση, εντοπισμός προβλημάτων, βελτίωση, επανάληψη*

**Δομημένη Ανάλυση** - Για συνεπή αξιολόγηση. Το μοντέλο αναθεωρεί κώδικα χρησιμοποιώντας ένα σταθερό πλαίσιο (ορθότητα, πρακτικές, απόδοση, ασφάλεια, διαχειρισιμότητα). Χρησιμοποιήστε το για επισκοπήσεις κώδικα ή αξιολογήσεις ποιότητας.

```java
String prompt = """
    <analysis_framework>
    You are an expert code reviewer. Analyze the code for:
    
    1. Correctness
       - Does it work as intended?
       - Are there logical errors?
    
    2. Best Practices
       - Follows language conventions?
       - Appropriate design patterns?
    
    3. Performance
       - Any inefficiencies?
       - Scalability concerns?
    
    4. Security
       - Potential vulnerabilities?
       - Input validation?
    
    5. Maintainability
       - Code clarity?
       - Documentation?
    
    <output_format>
    Provide your analysis in this structure:
    - Summary: One-sentence overall assessment
    - Strengths: 2-3 positive points
    - Issues: List any problems found with severity (High/Medium/Low)
    - Recommendations: Specific improvements
    </output_format>
    </analysis_framework>
    
    Code to analyze:
    ```
    public List getUsers() {
        return database.query("SELECT * FROM users");
    }

## Εξερεύνηση των Προτύπων

Η διαδικτυακή διεπαφή σας επιτρέπει να πειραματιστείτε με διαφορετικές στρατηγικές prompt. Κάθε πρότυπο λύνει διαφορετικά προβλήματα - δοκιμάστε τα για να δείτε πότε λάμπει η κάθε προσέγγιση.

> **Σημείωση: Streaming vs Non-Streaming** — Κάθε σελίδα προτύπου προσφέρει δύο κουμπιά: **🔴 Stream Response (Live)** και μια επιλογή **Non-streaming**. Το streaming χρησιμοποιεί Server-Sent Events (SSE) για να εμφανίζει τα tokens σε πραγματικό χρόνο καθώς το μοντέλο τα παράγει, οπότε βλέπετε την πρόοδο άμεσα. Η επιλογή non-streaming περιμένει ολόκληρη την απάντηση πριν την εμφανίσει. Για prompts που προκαλούν βαθύ συλλογισμό (π.χ., High Eagerness, Self-Reflecting Code), η κλήση non-streaming μπορεί να διαρκέσει πολύ — μερικές φορές λεπτά — χωρίς ορατή ανταπόκριση. **Χρησιμοποιήστε streaming όταν πειραματίζεστε με περίπλοκα prompts** ώστε να δείτε το μοντέλο σε δράση και να αποφύγετε την εντύπωση ότι η αίτηση έχει λήξει.
>
> **Σημείωση: Απαίτηση Browser** — Η λειτουργία streaming χρησιμοποιεί το Fetch Streams API (`response.body.getReader()`) που απαιτεί πλήρη browser (Chrome, Edge, Firefox, Safari). Δεν λειτουργεί στον ενσωματωμένο Simple Browser του VS Code, καθώς το webview του δεν υποστηρίζει το ReadableStream API. Αν χρησιμοποιείτε τον Simple Browser, τα κουμπιά non-streaming λειτουργούν κανονικά — μόνο τα κουμπιά streaming επηρεάζονται. Ανοίξτε το `http://localhost:8083` σε εξωτερικό browser για την πλήρη εμπειρία.

### Χαμηλή vs Υψηλή Προθυμία (Low vs High Eagerness)

Κάντε μια απλή ερώτηση όπως "Ποιο είναι το 15% του 200;" χρησιμοποιώντας Low Eagerness. Θα λάβετε μια άμεση, απευθείας απάντηση. Τώρα ρωτήστε κάτι σύνθετο όπως "Σχεδίασε στρατηγική caching για ένα API με υψηλή επισκεψιμότητα" χρησιμοποιώντας High Eagerness. Κάντε κλικ στο **🔴 Stream Response (Live)** και παρακολουθήστε τη λεπτομερή σκέψη του μοντέλου να εμφανίζεται token-κατά-token. Ίδιο μοντέλο, ίδια δομή ερώτησης - αλλά το prompt του λέει πόση σκέψη να κάνει.

### Εκτέλεση Εργασιών (Προλόγοι Εργαλείων)

Οι πολυβάθμιες ροές εργασίας ωφελούνται από προγραμματισμό και αφήγηση της προόδου. Το μοντέλο περιγράφει τι θα κάνει, αφηγείται κάθε βήμα και μετά συνοψίζει τα αποτελέσματα.

### Αυτο-Αναστοχαζόμενος Κώδικας

Δοκιμάστε "Δημιουργία υπηρεσίας επικύρωσης email". Αντί να παράγει απλά κώδικα και να σταματά, το μοντέλο παράγει, αξιολογεί με βάση κριτήρια ποιότητας, εντοπίζει αδυναμίες και βελτιώνει. Θα δείτε να επαναλαμβάνει μέχρι ο κώδικας να πληροί τα πρότυπα παραγωγής.

### Δομημένη Ανάλυση

Οι ανασκοπήσεις κώδικα απαιτούν σταθερά πλαίσια αξιολόγησης. Το μοντέλο αναλύει τον κώδικα με καθορισμένες κατηγορίες (ορθότητα, πρακτικές, απόδοση, ασφάλεια) με επίπεδα σοβαρότητας.

### Συνομιλία Πολλαπλών Γύρων

Ρωτήστε "Τι είναι το Spring Boot;" και αμέσως μετά "Δείξε μου ένα παράδειγμα". Το μοντέλο θυμάται την πρώτη σας ερώτηση και σας δίνει συγκεκριμένο παράδειγμα Spring Boot. Χωρίς μνήμη, η δεύτερη ερώτηση θα ήταν πολύ ασαφής.

### Λογική Βήμα-βήμα

Επιλέξτε ένα μαθηματικό πρόβλημα και δοκιμάστε το με το Step-by-Step Reasoning και Low Eagerness. Η χαμηλή προθυμία δίνει απλώς την απάντηση - γρήγορη αλλά αδιαφανή. Το βήμα-βήμα σας δείχνει κάθε υπολογισμό και απόφαση.

### Περιορισμένη Έξοδος

Όταν χρειάζεστε συγκεκριμένες μορφές ή αριθμό λέξεων, αυτό το πρότυπο επιβάλλει αυστηρή τήρηση. Δοκιμάστε να δημιουργήσετε περίληψη με ακριβώς 100 λέξεις σε μορφή κουκίδων.

## Τι Μαθαίνετε Πραγματικά

**Η Δυσκολία της Σκέψης Αλλάζει τα Πάντα**

Το GPT-5.2 σας επιτρέπει να ελέγχετε την υπολογιστική προσπάθεια μέσω των prompts σας. Η χαμηλή προσπάθεια σημαίνει γρήγορες απαντήσεις με ελάχιστη εξερεύνηση. Η υψηλή προσπάθεια σημαίνει ότι το μοντέλο αφιερώνει χρόνο για βαθιά σκέψη. Μαθαίνετε να ταιριάζετε την προσπάθεια με την πολυπλοκότητα της εργασίας - μην χάνετε χρόνο σε απλές ερωτήσεις, αλλά μην βιάζεστε στις σύνθετες αποφάσεις.

**Η Δομή Καθοδηγεί τη Συμπεριφορά**

Παρατηρήσατε τις ετικέτες XML στα prompts; Δεν είναι διακοσμητικές. Τα μοντέλα ακολουθούν τις δομημένες οδηγίες πιο αξιόπιστα από το ελεύθερο κείμενο. Όταν χρειάζεστε πολυβάθμιες διαδικασίες ή σύνθετη λογική, η δομή βοηθά το μοντέλο να καταλαβαίνει πού βρίσκεται και τι έρχεται μετά.

<img src="../../../translated_images/el/prompt-structure.a77763d63f4e2f89.webp" alt="Prompt Structure" width="800"/>

*Ανατομία ενός καλά δομημένου prompt με σαφείς ενότητες και οργάνωση σε στυλ XML*

**Ποιότητα Μέσω Αυτο-Αξιολόγησης**

Τα πρότυπα αυτο-αναστοχασμού λειτουργούν κάνοντας τα κριτήρια ποιότητας σαφή. Αντί να ελπίζετε ότι το μοντέλο "το κάνει σωστά", του λέτε ακριβώς τι σημαίνει "σωστά": σωστή λογική, διαχείριση σφαλμάτων, απόδοση, ασφάλεια. Το μοντέλο μπορεί τότε να αξιολογήσει την ίδια του την έξοδο και να βελτιωθεί. Αυτό μετατρέπει τη δημιουργία κώδικα από λαχείο σε διαδικασία.

**Το Πλαίσιο Είναι Πεπερασμένο**

Οι συνομιλίες πολλαπλών γύρων λειτουργούν με την προσθήκη ιστορικού μηνυμάτων σε κάθε αίτημα. Αλλά υπάρχει όριο - κάθε μοντέλο έχει μέγιστο αριθμό tokens. Καθώς οι συνομιλίες μεγαλώνουν, θα χρειαστείτε στρατηγικές για να κρατάτε το σχετικό πλαίσιο χωρίς να φτάνετε στο όριο. Αυτό το module σας δείχνει πώς λειτουργεί η μνήμη. Αργότερα θα μάθετε πότε να συνοψίζετε, πότε να ξεχνάτε και πότε να ανακτάτε.
