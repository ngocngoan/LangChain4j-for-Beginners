## Τι θα Μάθετε

Αν ολοκληρώσατε το γρήγορο ξεκίνημα, είδατε πώς να στέλνετε προτροπές και να λαμβάνετε απαντήσεις. Αυτή είναι η βάση, αλλά οι πραγματικές εφαρμογές χρειάζονται περισσότερα. Αυτό το module διδάσκει πώς να δημιουργείτε συνομιλητική AI που θυμάται το πλαίσιο και διατηρεί κατάσταση - η διαφορά μεταξύ μιας επίδειξης μιας φοράς και μιας εφαρμογής έτοιμης για παραγωγή.

Θα χρησιμοποιήσουμε το Azure OpenAI GPT-5.2 καθ’ όλη τη διάρκεια αυτού του οδηγού επειδή οι προηγμένες ικανότητες συλλογισμού του καθιστούν πιο εμφανείς τις συμπεριφορές των διαφόρων προτύπων. Όταν προσθέσετε μνήμη, θα δείτε καθαρά τη διαφορά. Αυτό καθιστά πιο εύκολο να καταλάβετε τι φέρνει το κάθε στοιχείο στην εφαρμογή σας.

Θα δημιουργήσετε μια εφαρμογή που επιδεικνύει και τα δύο πρότυπα:

**Συζήτηση χωρίς Κατάσταση** - Κάθε αίτημα είναι ανεξάρτητο. Το μοντέλο δεν έχει μνήμη προηγούμενων μηνυμάτων. Αυτό είναι το πρότυπο που χρησιμοποιήσατε στο γρήγορο ξεκίνημα.

**Συνομιλία με Κατάσταση** - Κάθε αίτημα περιλαμβάνει το ιστορικό της συνομιλίας. Το μοντέλο διατηρεί το πλαίσιο μέσα από πολλές εναλλαγές. Αυτό απαιτούν οι εφαρμογές παραγωγής.

## Προαπαιτούμενα

- Συνδρομή Azure με πρόσβαση σε Azure OpenAI
- Java 21, Maven 3.9+
- Azure CLI (https://learn.microsoft.com/en-us/cli/azure/install-azure-cli)
- Azure Developer CLI (azd) (https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/install-azd)

> **Note:** Η Java, το Maven, το Azure CLI και το Azure Developer CLI (azd) είναι προεγκατεστημένα στο δοσμένο devcontainer.

> **Note:** Αυτό το module χρησιμοποιεί το GPT-5.2 στο Azure OpenAI. Η ανάπτυξη ρυθμίζεται αυτόματα μέσω `azd up` - μην τροποποιείτε το όνομα του μοντέλου στον κώδικα.

## Κατανόηση του Πυρηνικού Προβλήματος

Τα μοντέλα γλώσσας είναι χωρίς κατάσταση. Κάθε κλήση API είναι ανεξάρτητη. Αν στείλετε "Το όνομά μου είναι Ιωάννης" και μετά ρωτήσετε "Ποιο είναι το όνομά μου;", το μοντέλο δεν έχει ιδέα ότι μόλις παρουσιαστήκατε. Θεωρεί κάθε αίτημα σαν να είναι η πρώτη συνομιλία που είχατε ποτέ.

Αυτό είναι εντάξει για απλές ερωταπαντήσεις, αλλά άχρηστο για πραγματικές εφαρμογές. Τα bots εξυπηρέτησης πελατών πρέπει να θυμούνται τι τους λέτε. Οι προσωπικοί βοηθοί χρειάζονται πλαίσιο. Κάθε συνομιλία με πολλές εναλλαγές απαιτεί μνήμη.

<img src="../../../translated_images/el/stateless-vs-stateful.cc4a4765e649c41a.webp" alt="Stateless vs Stateful Conversations" width="800"/>

*Η διαφορά ανάμεσα σε συζητήσεις χωρίς κατάσταση (ανεξάρτητες κλήσεις) και με κατάσταση (με επίγνωση πλαισίου)*

## Κατανόηση των Tokens

Πριν εμβαθύνετε στις συνομιλίες, είναι σημαντικό να κατανοήσετε τα tokens - τις βασικές μονάδες κειμένου που επεξεργάζονται τα μοντέλα γλώσσας:

<img src="../../../translated_images/el/token-explanation.c39760d8ec650181.webp" alt="Token Explanation" width="800"/>

*Παράδειγμα του πώς το κείμενο χωρίζεται σε tokens - "I love AI!" γίνεται 4 ξεχωριστές μονάδες επεξεργασίας*

Τα tokens είναι ο τρόπος που τα μοντέλα AI μετρούν και επεξεργάζονται κείμενο. Λέξεις, στίξη και ακόμα και κενά μπορεί να είναι tokens. Το μοντέλο σας έχει όριο πόσα tokens μπορεί να επεξεργαστεί ταυτόχρονα (400.000 για GPT-5.2, με έως 272.000 input tokens και 128.000 output tokens). Η κατανόηση των tokens βοηθά στη διαχείριση του μήκους της συνομιλίας και των εξόδων.

## Πώς Λειτουργεί η Μνήμη

Η μνήμη συνομιλίας λύνει το πρόβλημα της απουσίας κατάστασης διατηρώντας το ιστορικό συνομιλίας. Πριν στείλετε το αίτημά σας στο μοντέλο, το πλαίσιο προσθέτει τα σχετικά προηγούμενα μηνύματα. Όταν ρωτάτε "Ποιο είναι το όνομά μου;", το σύστημα στέλνει ολόκληρο το ιστορικό της συνομιλίας, επιτρέποντας στο μοντέλο να δει ότι είπατε νωρίτερα "Το όνομά μου είναι Ιωάννης."

Το LangChain4j παρέχει υλοποιήσεις μνήμης που διαχειρίζονται αυτόματα αυτό το ζήτημα. Επιλέγετε πόσα μηνύματα θέλετε να κρατηθούν και το πλαίσιο διαχειρίζεται το παράθυρο πλαισίου.

<img src="../../../translated_images/el/memory-window.bbe67f597eadabb3.webp" alt="Memory Window Concept" width="800"/>

*Το MessageWindowChatMemory διατηρεί ένα συρόμενο παράθυρο με τα πρόσφατα μηνύματα, απορρίπτοντας αυτόματα τα παλιά*

## Πώς Χρησιμοποιεί Αυτό το LangChain4j

Αυτό το module επεκτείνει το γρήγορο ξεκίνημα ενσωματώνοντας το Spring Boot και προσθέτοντας μνήμη συνομιλίας. Δείτε πώς ενώνονται τα κομμάτια:

**Εξαρτήσεις** - Προσθέστε δύο βιβλιοθήκες LangChain4j:

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```

**Μοντέλο Συνομιλίας** - Διαμορφώστε το Azure OpenAI ως Spring bean ([LangChainConfig.java](../../../01-introduction/src/main/java/com/example/langchain4j/config/LangChainConfig.java)):

```java
@Bean
public OpenAiOfficialChatModel openAiOfficialChatModel() {
    return OpenAiOfficialChatModel.builder()
            .baseUrl(azureEndpoint)
            .apiKey(azureApiKey)
            .modelName(deploymentName)
            .timeout(Duration.ofMinutes(5))
            .maxRetries(3)
            .build();
}
```

Ο builder διαβάζει τα διαπιστευτήρια από μεταβλητές περιβάλλοντος που ορίζονται από το `azd up`. Η ρύθμιση του `baseUrl` στη διεύθυνση του Azure endpoint σας κάνει τον πελάτη OpenAI να δουλεύει με το Azure OpenAI.

**Μνήμη Συνομιλίας** - Παρακολουθήστε το ιστορικό συνομιλίας με το MessageWindowChatMemory ([ConversationService.java](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java)):

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("My name is John"));
memory.add(AiMessage.from("Nice to meet you, John!"));

memory.add(UserMessage.from("What's my name?"));
AiMessage aiMessage = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage);
```

Δημιουργήστε μνήμη με `withMaxMessages(10)` για να κρατήσετε τα τελευταία 10 μηνύματα. Προσθέστε μηνύματα χρήστη και AI με τυποποιημένα wrappers: `UserMessage.from(text)` και `AiMessage.from(text)`. Ανάκτηση ιστορικού με `memory.messages()` και στείλτε το στο μοντέλο. Η υπηρεσία αποθηκεύει ξεχωριστές στιγμές μνήμης ανά πακέτο συνομιλίας, επιτρέποντας πολλούς χρήστες να συνομιλούν ταυτόχρονα.

> **🤖 Δοκιμάστε με [GitHub Copilot](https://github.com/features/copilot) Chat:** Ανοίξτε το [`ConversationService.java`](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java) και ρωτήστε:
> - "Πώς αποφασίζει το MessageWindowChatMemory ποια μηνύματα να απορρίψει όταν το παράθυρο γεμίσει;"
> - "Μπορώ να υλοποιήσω αποθήκευση μνήμης με βάση δεδομένων αντί για μνήμη RAM;"
> - "Πώς θα πρόσθετα σύνοψη για να συμπιέσω το παλιό ιστορικό συνομιλίας;"

Το σημείο επικοινωνίας chat χωρίς κατάσταση παραλείπει εντελώς τη μνήμη - απλώς `chatModel.chat(prompt)` όπως στο γρήγορο ξεκίνημα. Το σημείο επικοινωνίας με κατάσταση προσθέτει μηνύματα στη μνήμη, ανακτά το ιστορικό και συμπεριλαμβάνει το πλαίσιο με κάθε αίτημα. Ίδια διαμόρφωση μοντέλου, διαφορετικά πρότυπα.

## Ανάπτυξη Υποδομής Azure OpenAI

**Bash:**
```bash
cd 01-introduction
azd up  # Επιλέξτε συνδρομή και τοποθεσία (συνιστάται eastus2)
```

**PowerShell:**
```powershell
cd 01-introduction
azd up  # Επιλέξτε συνδρομή και τοποθεσία (προτείνεται eastus2)
```

> **Note:** Αν συναντήσετε σφάλμα χρονικού ορίου (`RequestConflict: Cannot modify resource ... provisioning state is not terminal`), απλά τρέξτε ξανά `azd up`. Οι πόροι Azure μπορεί να είναι ακόμα σε διαδικασία δημιουργίας στο παρασκήνιο και η προσπάθεια επανάληψης επιτρέπει στην ανάπτυξη να ολοκληρωθεί μόλις οι πόροι φτάσουν σε τελική κατάσταση.

Αυτό θα:
1. Αναπτύξει Azure OpenAI resource με μοντέλα GPT-5.2 και text-embedding-3-small
2. Δημιουργήσει αυτόματα αρχείο `.env` στη ριζική διαδρομή του έργου με διαπιστευτήρια
3. Ρυθμίσει όλες τις απαιτούμενες μεταβλητές περιβάλλοντος

**Αν έχετε προβλήματα ανάπτυξης:** Δείτε το [Infrastructure README](infra/README.md) για λεπτομερείς οδηγίες αντιμετώπισης προβλημάτων, όπως συγκρούσεις ονομάτων υποτομέων, βήματα χειροκίνητης ανάπτυξης στο Azure Portal και οδηγίες διαμόρφωσης μοντέλου.

**Επαληθεύστε ότι η ανάπτυξη πέτυχε:**

**Bash:**
```bash
cat ../.env  # Θα πρέπει να εμφανίζει το AZURE_OPENAI_ENDPOINT, API_KEY, κ.λπ.
```

**PowerShell:**
```powershell
Get-Content ..\.env  # Πρέπει να εμφανίζει το AZURE_OPENAI_ENDPOINT, το API_KEY, κ.λπ.
```

> **Note:** Η εντολή `azd up` δημιουργεί αυτόματα το αρχείο `.env`. Αν χρειαστεί να το ενημερώσετε αργότερα, μπορείτε είτε να επεξεργαστείτε το αρχείο `.env` χειροκίνητα είτε να το αναδημιουργήσετε τρέχοντας:
>
> **Bash:**
> ```bash
> cd ..
> bash .azd-env.sh
> ```
>
> **PowerShell:**
> ```powershell
> cd ..
> .\.azd-env.ps1
> ```

Η εφαρμογή παρέχει μια διαδικτυακή διεπαφή με δύο υλοποιήσεις chat δίπλα δίπλα.

*Πίνακας ελέγχου που δείχνει και τις δύο επιλογές: Απλή Συζήτηση (χωρίς κατάσταση) και Συνδυαστική Συζήτηση (με κατάσταση)*

### Συζήτηση χωρίς Κατάσταση (Αριστερό Πάνελ)

Δοκιμάστε πρώτα αυτό. Ρωτήστε "Το όνομά μου είναι Ιωάννης" και αμέσως μετά ρωτήστε "Ποιο είναι το όνομά μου;" Το μοντέλο δεν θα θυμάται γιατί κάθε μήνυμα είναι ανεξάρτητο. Αυτό επιδεικνύει το βασικό πρόβλημα με την απλή ενσωμάτωση μοντέλων γλώσσας - απουσία πλαισίου συνομιλίας.

<img src="../../../translated_images/el/simple-chat-stateless-demo.13aeb3978eab3234.webp" alt="Stateless Chat Demo" width="800"/>

*Η AI δεν θυμάται το όνομά σας από το προηγούμενο μήνυμα*

### Συζήτηση με Κατάσταση (Δεξί Πάνελ)

Τώρα δοκιμάστε την ίδια ακολουθία εδώ. Ρωτήστε "Το όνομά μου είναι Ιωάννης" και μετά "Ποιο είναι το όνομά μου;" Αυτή τη φορά θυμάται. Η διαφορά είναι το MessageWindowChatMemory - διατηρεί το ιστορικό συνομιλίας και το περιλαμβάνει με κάθε αίτημα. Έτσι λειτουργεί η συνομιλητική AI παραγωγής.

<img src="../../../translated_images/el/conversational-chat-stateful-demo.e5be9822eb23ff59.webp" alt="Stateful Chat Demo" width="800"/>

*Η AI θυμάται το όνομά σας από την προηγούμενη συνομιλία*

Και τα δύο πάνελ χρησιμοποιούν το ίδιο μοντέλο GPT-5.2. Η μόνη διαφορά είναι η μνήμη. Αυτό καθιστά σαφές τι προσφέρει η μνήμη στην εφαρμογή σας και γιατί είναι απαραίτητη για πραγματικές χρήσεις.
