## Τι θα μάθετε

Αν ολοκληρώσατε το γρήγορο ξεκίνημα, είδατε πώς να στέλνετε ερωτήματα και να λαμβάνετε απαντήσεις. Αυτή είναι η βάση, αλλά οι πραγματικές εφαρμογές χρειάζονται περισσότερα. Αυτό το module σας διδάσκει πώς να δημιουργείτε συστήματα συνομιλίας τεχνητής νοημοσύνης που θυμούνται το πλαίσιο και διατηρούν κατάσταση - η διαφορά μεταξύ μιας επίδειξης μίας φοράς και μιας εφαρμογής παραγωγής.

Θα χρησιμοποιήσουμε το Azure OpenAI GPT-5.2 καθ’ όλη τη διάρκεια αυτού του οδηγού επειδή οι προηγμένες ικανότητές του στην επιχειρηματολογία καθιστούν πιο εμφανείς τις συμπεριφορές των διαφορετικών προτύπων. Όταν προσθέσετε μνήμη, θα δείτε καθαρά τη διαφορά. Αυτό διευκολύνει την κατανόηση του τι φέρνει κάθε στοιχείο στην εφαρμογή σας.

Θα δημιουργήσετε μία εφαρμογή που δείχνει και τα δύο πρότυπα:

**Συνομιλία χωρίς Κατάσταση** - Κάθε αίτημα είναι ανεξάρτητο. Το μοντέλο δεν έχει μνήμη προηγούμενων μηνυμάτων. Αυτό είναι το πρότυπο που χρησιμοποιήσατε στο γρήγορο ξεκίνημα.

**Συνομιλία με Κατάσταση** - Κάθε αίτημα περιλαμβάνει το ιστορικό συνομιλίας. Το μοντέλο διατηρεί το πλαίσιο σε πολλούς γύρους. Αυτό απαιτούν οι εφαρμογές παραγωγής.

## Απαιτήσεις

- Συνδρομή Azure με πρόσβαση στο Azure OpenAI
- Java 21, Maven 3.9+
- Azure CLI (https://learn.microsoft.com/en-us/cli/azure/install-azure-cli)
- Azure Developer CLI (azd) (https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/install-azd)

> **Σημείωση:** Η Java, το Maven, το Azure CLI και το Azure Developer CLI (azd) είναι προεγκατεστημένα στον παρεχόμενο devcontainer.

> **Σημείωση:** Αυτό το module χρησιμοποιεί το GPT-5.2 στο Azure OpenAI. Η ανάπτυξη ρυθμίζεται αυτόματα μέσω `azd up` - μην τροποποιήσετε το όνομα του μοντέλου στον κώδικα.

## Κατανόηση του Κεντρικού Προβλήματος

Τα γλωσσικά μοντέλα δεν διατηρούν κατάσταση. Κάθε κλήση API είναι ανεξάρτητη. Αν στείλετε "Ονομάζομαι Γιάννης" και μετά ρωτήσετε "Πώς με λένε;", το μοντέλο δεν έχει ιδέα ότι μόλις παραθέσατε το όνομά σας. Θεωρεί κάθε αίτημα σαν να είναι η πρώτη συνομιλία που έχετε ποτέ.

Αυτό είναι εντάξει για απλές ερωτήσεις και απαντήσεις, αλλά άχρηστο για πραγματικές εφαρμογές. Τα bots εξυπηρέτησης πελατών πρέπει να θυμούνται τι τους λέτε. Οι προσωπικοί βοηθοί χρειάζονται πλαίσιο. Κάθε συνομιλία με πολλούς γύρους απαιτεί μνήμη.

<img src="../../../translated_images/el/stateless-vs-stateful.cc4a4765e649c41a.webp" alt="Συνομιλίες χωρίς Κατάσταση εναντίον με Κατάσταση" width="800"/>

*Η διαφορά μεταξύ συνομιλιών χωρίς κατάσταση (ανεξάρτητες κλήσεις) και με κατάσταση (ενημερωμένες με πλαίσιο)*

## Κατανόηση των Tokens

Πριν εμβαθύνετε στις συνομιλίες, είναι σημαντικό να κατανοήσετε τα tokens - τις βασικές μονάδες κειμένου που επεξεργάζονται τα γλωσσικά μοντέλα:

<img src="../../../translated_images/el/token-explanation.c39760d8ec650181.webp" alt="Εξήγηση Token" width="800"/>

*Παράδειγμα για το πώς το κείμενο χωρίζεται σε tokens - το "I love AI!" γίνεται 4 ξεχωριστές μονάδες επεξεργασίας*

Τα tokens είναι ο τρόπος με τον οποίο τα μοντέλα τεχνητής νοημοσύνης μετρούν και επεξεργάζονται το κείμενο. Λέξεις, στίξη και ακόμα και κενά μπορούν να είναι tokens. Το μοντέλο σας έχει όριο πόσα tokens μπορεί να επεξεργαστεί ταυτόχρονα (400.000 για το GPT-5.2, με έως 272.000 tokens εισόδου και 128.000 tokens εξόδου). Η κατανόηση των tokens βοηθά στη διαχείριση του μήκους της συνομιλίας και του κόστους.

## Πώς Λειτουργεί η Μνήμη

Η μνήμη συνομιλίας λύνει το πρόβλημα της έλλειψης κατάστασης διατηρώντας το ιστορικό συνομιλίας. Πριν στείλετε το αίτημά σας στο μοντέλο, το πλαίσιο προσαρτά σχετικά προηγούμενα μηνύματα. Όταν ρωτάτε "Πώς με λένε;", το σύστημα στην πραγματικότητα στέλνει ολόκληρο το ιστορικό συνομιλίας, επιτρέποντας στο μοντέλο να δει ότι προηγουμένως είπατε "Ονομάζομαι Γιάννης."

Το LangChain4j παρέχει υλοποιήσεις μνήμης που διαχειρίζονται αυτό αυτόματα. Επιλέγετε πόσα μηνύματα να διατηρείτε και το πλαίσιο διαχειρίζεται το παράθυρο πλαισίου.

<img src="../../../translated_images/el/memory-window.bbe67f597eadabb3.webp" alt="Έννοια Παραθύρου Μνήμης" width="800"/>

*Το MessageWindowChatMemory διατηρεί ένα κυλιόμενο παράθυρο πρόσφατων μηνυμάτων, απορρίπτοντας αυτόματα τα παλιά*

## Πώς Χρησιμοποιεί το LangChain4j

Αυτό το module επεκτείνει το γρήγορο ξεκίνημα ενσωματώνοντας το Spring Boot και προσθέτοντας μνήμη συνομιλίας. Δείτε πώς συνδέονται τα κομμάτια:

**Εξαρτήσεις** - Προσθέστε δύο βιβλιοθήκες LangChain4j:

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```

**Μοντέλο Συνομιλίας** - Ρυθμίστε το Azure OpenAI ως Spring bean ([LangChainConfig.java](../../../01-introduction/src/main/java/com/example/langchain4j/config/LangChainConfig.java)):

```java
@Bean
public OpenAiOfficialChatModel openAiOfficialChatModel() {
    return OpenAiOfficialChatModel.builder()
            .baseUrl(azureEndpoint)
            .apiKey(azureApiKey)
            .modelName(deploymentName)
            .timeout(Duration.ofMinutes(5))
            .maxRetries(3)
            .build();
}
```

Ο builder διαβάζει τα διαπιστευτήρια από τις μεταβλητές περιβάλλοντος που ορίζονται από το `azd up`. Η ρύθμιση `baseUrl` στην τελική σας διεύθυνση Azure κάνει τον client OpenAI να λειτουργεί με το Azure OpenAI.

**Μνήμη Συνομιλίας** - Παρακολουθείστε το ιστορικό συνομιλίας με το MessageWindowChatMemory ([ConversationService.java](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java)):

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("My name is John"));
memory.add(AiMessage.from("Nice to meet you, John!"));

memory.add(UserMessage.from("What's my name?"));
AiMessage aiMessage = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage);
```

Δημιουργήστε τη μνήμη με `withMaxMessages(10)` για να κρατάτε τα τελευταία 10 μηνύματα. Προσθέστε μηνύματα χρήστη και AI με τύπους wrapper: `UserMessage.from(text)` και `AiMessage.from(text)`. Ανακτήστε το ιστορικό με `memory.messages()` και στείλτε το στο μοντέλο. Η υπηρεσία αποθηκεύει ξεχωριστές στιγμές μνήμης ανά ID συνομιλίας, επιτρέποντας σε πολλούς χρήστες να συνομιλούν ταυτόχρονα.

> **🤖 Δοκιμάστε με τη συνομιλία του [GitHub Copilot](https://github.com/features/copilot):** Ανοίξτε το [`ConversationService.java`](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java) και ρωτήστε:
> - "Πώς αποφασίζει το MessageWindowChatMemory ποια μηνύματα να απορρίψει όταν το παράθυρο είναι γεμάτο;"
> - "Μπορώ να υλοποιήσω προσαρμοσμένη αποθήκευση μνήμης χρησιμοποιώντας βάση δεδομένων αντί για μνήμη RAM;"
> - "Πώς θα πρόσθετα περίληψη για συμπίεση του παλαιού ιστορικού συνομιλίας;"

Το endpoint για συνομιλία χωρίς κατάσταση παραλείπει εντελώς τη μνήμη - απλά `chatModel.chat(prompt)` όπως στο γρήγορο ξεκίνημα. Το endpoint με κατάσταση προσθέτει μηνύματα στη μνήμη, ανακτά το ιστορικό και περιλαμβάνει το πλαίσιο σε κάθε αίτημα. Ίδια ρύθμιση μοντέλου, διαφορετικά πρότυπα.

## Ανάπτυξη Υποδομής Azure OpenAI

**Bash:**
```bash
cd 01-introduction
azd up  # Επιλέξτε συνδρομή και τοποθεσία (συνιστάται eastus2)
```

**PowerShell:**
```powershell
cd 01-introduction
azd up  # Επιλέξτε συνδρομή και τοποθεσία (προτείνεται eastus2)
```

> **Σημείωση:** Αν αντιμετωπίσετε σφάλμα χρονικού ορίου (`RequestConflict: Cannot modify resource ... provisioning state is not terminal`), απλά τρέξτε ξανά `azd up`. Οι πηγές Azure πιθανόν να εξακολουθούν να αναπτύσσονται στο παρασκήνιο και η επανάληψη επιτρέπει την ολοκλήρωση μόλις αυτές φτάσουν σε τελικό στάδιο.

Αυτό θα:
1. Αναπτύξει το πόρο Azure OpenAI με τα μοντέλα GPT-5.2 και text-embedding-3-small
2. Δημιουργήσει αυτόματα το αρχείο `.env` στη ρίζα του έργου με τα διαπιστευτήρια
3. Ρυθμίσει όλες τις απαιτούμενες μεταβλητές περιβάλλοντος

**Έχετε προβλήματα με την ανάπτυξη;** Δείτε το [Infrastructure README](infra/README.md) για λεπτομερή αντιμετώπιση προβλημάτων, συμπεριλαμβανομένων συγκρούσεων ονομάτων υποτομέων, χειροκίνητων βημάτων ανάπτυξης στο Azure Portal και οδηγιών ρύθμισης μοντέλου.

**Επαληθεύστε ότι η ανάπτυξη πέτυχε:**

**Bash:**
```bash
cat ../.env  # Θα πρέπει να εμφανίζει AZURE_OPENAI_ENDPOINT, API_KEY, κ.λπ.
```

**PowerShell:**
```powershell
Get-Content ..\.env  # Πρέπει να εμφανίζει AZURE_OPENAI_ENDPOINT, API_KEY, κ.λπ.
```

> **Σημείωση:** Η εντολή `azd up` δημιουργεί αυτόματα το αρχείο `.env`. Αν χρειαστεί να το ενημερώσετε αργότερα, μπορείτε είτε να το επεξεργαστείτε χειροκίνητα είτε να το αναδημιουργήσετε τρέχοντας:
>
> **Bash:**
> ```bash
> cd ..
> bash .azd-env.sh
> ```
>
> **PowerShell:**
> ```powershell
> cd ..
> .\.azd-env.ps1
> ```
