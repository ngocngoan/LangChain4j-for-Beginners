## Τι θα μάθετε

Μέχρι τώρα, έχετε μάθει πώς να έχετε συνομιλίες με την ΤΝ, να δομείτε τα αιτήματα με αποτελεσματικότητα και να θεμελιώνετε τις απαντήσεις στα έγγραφά σας. Αλλά υπάρχει ακόμη ένας θεμελιώδης περιορισμός: τα γλωσσικά μοντέλα μπορούν μόνο να παράγουν κείμενο. Δεν μπορούν να ελέγξουν τον καιρό, να κάνουν υπολογισμούς, να ερωτήσουν βάσεις δεδομένων ή να αλληλεπιδράσουν με εξωτερικά συστήματα.

Τα εργαλεία αλλάζουν αυτό. Δίνοντας στο μοντέλο πρόσβαση σε λειτουργίες που μπορεί να καλέσει, το μετατρέπετε από γεννήτρια κειμένου σε πράκτορα που μπορεί να αναλάβει δράσεις. Το μοντέλο αποφασίζει πότε χρειάζεται ένα εργαλείο, ποιο εργαλείο να χρησιμοποιήσει, και ποιες παραμέτρους να περάσει. Ο κώδικάς σας εκτελεί τη λειτουργία και επιστρέφει το αποτέλεσμα. Το μοντέλο ενσωματώνει αυτό το αποτέλεσμα στην απάντησή του.

## Κατανόηση των Πρακτόρων ΤΝ με Εργαλεία

> **📝 Σημείωση:** Ο όρος "πράκτορες" σε αυτήν την ενότητα αναφέρεται σε βοηθούς ΤΝ ενισχυμένους με δυνατότητες κλήσης εργαλείων. Αυτό διαφέρει από τα **Agentic AI** πρότυπα (αυτόνομοι πράκτορες με σχεδιασμό, μνήμη και πολυσταδιακή λογική) που θα καλύψουμε στην [Ενότητα 05: MCP](../05-mcp/README.md).

Χωρίς εργαλεία, ένα γλωσσικό μοντέλο μπορεί μόνο να παράγει κείμενο από τα εκπαιδευτικά του δεδομένα. Ρωτήστε το για τον τρέχοντα καιρό, και πρέπει να μαντέψει. Δώστε του εργαλεία, και μπορεί να καλέσει API καιρού, να εκτελέσει υπολογισμούς ή να ρωτήσει μια βάση δεδομένων — και μετά να ενσωματώσει αυτά τα πραγματικά αποτελέσματα στην απάντησή του.

<img src="../../../translated_images/el/what-are-tools.724e468fc4de64da.webp" alt="Without Tools vs With Tools" width="800"/>

*Χωρίς εργαλεία το μοντέλο μόνο μαντεύει — με εργαλεία μπορεί να καλεί API, να εκτελεί υπολογισμούς και να επιστρέφει δεδομένα σε πραγματικό χρόνο.*

Ένας πράκτορας ΤΝ με εργαλεία ακολουθεί ένα πρότυπο **Λογικής και Δράσης (ReAct)**. Το μοντέλο δεν απαντά απλά — σκέφτεται τι χρειάζεται, ενεργεί καλώντας ένα εργαλείο, παρατηρεί το αποτέλεσμα και μετά αποφασίζει αν θα ενεργήσει ξανά ή θα δώσει την τελική απάντηση:

1. **Λογική** — Ο πράκτορας αναλύει την ερώτηση του χρήστη και καθορίζει τι πληροφορίες χρειάζεται
2. **Δράση** — Ο πράκτορας επιλέγει το σωστό εργαλείο, παράγει τις σωστές παραμέτρους και το καλεί
3. **Παρατήρηση** — Ο πράκτορας λαμβάνει την έξοδο του εργαλείου και αξιολογεί το αποτέλεσμα
4. **Επανάληψη ή Απάντηση** — Αν χρειάζονται περισσότερα δεδομένα, ο πράκτορας επαναλαμβάνει· αλλιώς, συνθέτει μια απάντηση σε φυσική γλώσσα

<img src="../../../translated_images/el/react-pattern-detail.96a5efeeb6dd2f61.webp" alt="ReAct Pattern" width="800"/>

*Ο κύκλος ReAct — ο πράκτορας λογικά αποφασίζει τι να κάνει, ενεργεί καλώντας ένα εργαλείο, παρατηρεί το αποτέλεσμα και συνεχίζει μέχρι να δώσει την τελική απάντηση.*

Αυτό συμβαίνει αυτόματα. Εσείς ορίζετε τα εργαλεία και τις περιγραφές τους. Το μοντέλο αναλαμβάνει τη λήψη αποφάσεων σχετικά με το πότε και πώς να τα χρησιμοποιεί.

## Πώς λειτουργεί η Κλήση Εργαλείων

### Ορισμοί Εργαλείων

[WeatherTool.java](../../../04-tools/src/main/java/com/example/langchain4j/agents/tools/WeatherTool.java) | [TemperatureTool.java](../../../04-tools/src/main/java/com/example/langchain4j/agents/tools/TemperatureTool.java)

Ορίζετε λειτουργίες με σαφείς περιγραφές και προδιαγραφές παραμέτρων. Το μοντέλο βλέπει αυτές τις περιγραφές στο σύστημα προτροπής του και καταλαβαίνει τι κάνει το κάθε εργαλείο.

```java
@Component
public class WeatherTool {
    
    @Tool("Get the current weather for a location")
    public String getCurrentWeather(@P("Location name") String location) {
        // Η λογική αναζήτησης καιρού σας
        return "Weather in " + location + ": 22°C, cloudy";
    }
}

@AiService
public interface Assistant {
    String chat(@MemoryId String sessionId, @UserMessage String message);
}

// Ο βοηθός συσχετίζεται αυτόματα από το Spring Boot με:
// - Bean ChatModel
// - Όλες οι μέθοδοι @Tool από κλάσεις @Component
// - ChatMemoryProvider για διαχείριση συνεδριών
```
  
Το παρακάτω διάγραμμα αναλύει κάθε σχολιασμό και δείχνει πώς κάθε κομμάτι βοηθά την ΤΝ να κατανοήσει πότε να καλέσει το εργαλείο και ποια ορίσματα να περάσει:

<img src="../../../translated_images/el/tool-definitions-anatomy.f6468546037cf28b.webp" alt="Anatomy of Tool Definitions" width="800"/>

*Ανατομία ενός ορισμού εργαλείου — το @Tool λέει στην ΤΝ πότε να το χρησιμοποιήσει, το @P περιγράφει κάθε παράμετρο, και το @AiService συνδέει τα πάντα κατά την εκκίνηση.*

### Λήψη Αποφάσεων

Όταν ένας χρήστης ρωτάει «Ποιος είναι ο καιρός στο Σιάτλ;», το μοντέλο δεν επιλέγει τυχαία ένα εργαλείο. Συγκρίνει τον σκοπό του χρήστη με κάθε περιγραφή εργαλείου που έχει στη διάθεσή του, βαθμολογεί κάθε μία ως προς τη σχετικότητα, και επιλέγει την καλύτερη. Στη συνέχεια παράγει ένα δομημένο κάλεσμα λειτουργίας με τις σωστές παραμέτρους — σε αυτή την περίπτωση, ορίζοντας το `location` στην τιμή `"Seattle"`.

Αν κανένα εργαλείο δεν ταιριάζει στο αίτημα του χρήστη, το μοντέλο επιστρέφει απάντηση από τις γνώσεις του. Αν ταιριάξουν πολλαπλά εργαλεία, επιλέγει το πιο συγκεκριμένο.

<img src="../../../translated_images/el/decision-making.409cd562e5cecc49.webp" alt="How the AI Decides Which Tool to Use" width="800"/>

*Το μοντέλο αξιολογεί κάθε διαθέσιμο εργαλείο έναντι του σκοπού του χρήστη και επιλέγει το καλύτερο — γι’ αυτό η συγγραφή σαφών, συγκεκριμένων περιγραφών εργαλείων έχει σημασία.*

### Δημιουργία Απάντησης

Το μοντέλο λαμβάνει τα δεδομένα καιρού και τα μορφοποιεί σε μια απάντηση φυσικής γλώσσας προς τον χρήστη.

### Αρχιτεκτονική: Αυτόματη Σύνδεση στο Spring Boot

Αυτή η ενότητα χρησιμοποιεί την ενσωμάτωση LangChain4j με το Spring Boot μέσω των δηλωτικών interfaces `@AiService`. Κατά την εκκίνηση, το Spring Boot ανακαλύπτει κάθε `@Component` που περιέχει μεθόδους με `@Tool`, το bean `ChatModel`, και τον `ChatMemoryProvider` — και τα συνδέει όλα σε ένα ενιαίο interface `Assistant` με μηδενική επανάληψη κώδικα.

<img src="../../../translated_images/el/spring-boot-wiring.151321795988b04e.webp" alt="Spring Boot Auto-Wiring Architecture" width="800"/>

*Το interface @AiService συνδέει το ChatModel, τα συστατικά εργαλείων και τον πάροχο μνήμης — το Spring Boot αναλαμβάνει αυτόματα όλες τις συνδέσεις.*

Κύρια οφέλη αυτής της προσέγγισης:

- **Αυτόματη σύνδεση Spring Boot** — ChatModel και εργαλεία εγχέονται αυτόματα
- **Πρότυπο @MemoryId** — Αυτόματη διαχείριση μνήμης με βάση τη συνεδρία
- **Ενιαία περίπτωση** — Ο βοηθός δημιουργείται μία φορά και επαναχρησιμοποιείται για καλύτερη απόδοση
- **Εκτέλεση με ασφάλεια τύπων** — Κλήσεις Java με απευθείας μετατροπή τύπων
- **Ορχήστρα πολυσταδιακής συνομιλίας** — Αντιμετωπίζει αυτόματα την αλυσιδωτή χρήση εργαλείων
- **Μηδενική επανάληψη κώδικα** — Χωρίς χειροκίνητες κλήσεις `AiServices.builder()` ή αποθήκευση σε HashMap

Εναλλακτικές προσεγγίσεις (χειροκίνητο `AiServices.builder()`) απαιτούν περισσότερο κώδικα και στερούνται τα οφέλη ενσωμάτωσης με Spring Boot.

## Αλυσιδωτή Χρήση Εργαλείων

**Αλυσιδωτή Χρήση Εργαλείων** — Η πραγματική δύναμη των πρακτόρων με εργαλεία εμφανίζεται όταν μια μόνο ερώτηση απαιτεί πολλά εργαλεία. Ρωτήστε «Ποιος είναι ο καιρός στο Σιάτλ σε Φαρενάιτ;» και ο πράκτορας αλυσιδώνει αυτόματα δύο εργαλεία: πρώτα καλεί τη `getCurrentWeather` για να πάρει τη θερμοκρασία σε Κελσίου, μετά περνά αυτήν την τιμή στη `celsiusToFahrenheit` για μετατροπή — όλα σε μία μόνο συνομιλία.

<img src="../../../translated_images/el/tool-chaining-example.538203e73d09dd82.webp" alt="Tool Chaining Example" width="800"/>

*Αλυσιδωτή χρήση εργαλείων σε δράση — ο πράκτορας καλεί πρώτα το getCurrentWeather, μετά περάει το αποτέλεσμα σε celsiusToFahrenheit, και παρέχει συνδυασμένη απάντηση.*

Έτσι φαίνεται αυτό στην τρέχουσα εφαρμογή — ο πράκτορας αλυσιδώνει δύο κλήσεις εργαλείων σε μία μόνο συνομιλία:

<a href="images/tool-chaining.png"><img src="../../../translated_images/el/tool-chaining.3b25af01967d6f7b.webp" alt="Tool Chaining" width="800" style="border: 1px solid #ddd; box-shadow: 0 2px 8px rgba(0,0,0,0.1);"/></a>

*Πραγματικό αποτέλεσμα της εφαρμογής — ο πράκτορας αλυσιδώνει αυτόματα getCurrentWeather → celsiusToFahrenheit σε μία συνομιλία.*

**Ομαλή Διαχείριση Αποτυχιών** — Ζητήστε τον καιρό για μία πόλη που δεν υπάρχει στα ψεύτικα δεδομένα. Το εργαλείο επιστρέφει μήνυμα σφάλματος, και η ΤΝ εξηγεί ότι δεν μπορεί να βοηθήσει αντί να καταρρεύσει. Τα εργαλεία αποτυγχάνουν με ασφάλεια.

<img src="../../../translated_images/el/error-handling-flow.9a330ffc8ee0475c.webp" alt="Error Handling Flow" width="800"/>

*Όταν ένα εργαλείο αποτυγχάνει, ο πράκτορας πιάνει το σφάλμα και απαντά με μια χρήσιμη εξήγηση αντί να καταρρεύσει.*

Αυτό συμβαίνει σε μία μόνο συνομιλία. Ο πράκτορας ορχηστρώνει αυτόνομα πολλές κλήσεις εργαλείων.

### Δοκιμή Απλής Χρήσης Εργαλείων
Ξεκινήστε με ένα απλό αίτημα: "Μετατρέψτε 100 βαθμούς Φαρενάιτ σε Κελσίου". Ο πράκτορας αναγνωρίζει ότι χρειάζεται το εργαλείο μετατροπής θερμοκρασίας, το καλεί με τις σωστές παραμέτρους και επιστρέφει το αποτέλεσμα. Παρατηρήστε πόσο φυσικό φαίνεται αυτό - δεν καθορίσατε ποιο εργαλείο να χρησιμοποιηθεί ή πώς να το καλέσει.

### Δοκιμή Αλυσιδωτής Χρήσης Εργαλείων

Τώρα δοκιμάστε κάτι πιο σύνθετο: "Ποιος είναι ο καιρός στο Σιατλ και μετατρέψτε τον σε Φαρενάιτ;" Παρακολουθήστε τον πράκτορα να εκτελεί τα βήματα. Πρώτα παίρνει τον καιρό (που επιστρέφει σε Κελσίου), αναγνωρίζει ότι πρέπει να μετατρέψει σε Φαρενάιτ, καλεί το εργαλείο μετατροπής και συνδυάζει και τα δύο αποτελέσματα σε μία απάντηση.

### Δείτε τη Ροή της Συνομιλίας

Η διεπαφή συνομιλίας διατηρεί το ιστορικό συνομιλιών, επιτρέποντάς σας να έχετε συνομιλίες πολλαπλών γύρων. Μπορείτε να δείτε όλα τα προηγούμενα ερωτήματα και απαντήσεις, καθιστώντας εύκολη την παρακολούθηση της συνομιλίας και την κατανόηση του τρόπου με τον οποίο ο πράκτορας οικοδομεί το πλαίσιο μέσα από πολλαπλές ανταλλαγές.

<a href="images/tools-conversation-demo.png"><img src="../../../translated_images/el/tools-conversation-demo.89f2ce9676080f59.webp" alt="Συνομιλία με Πολλαπλές Κλήσεις Εργαλείων" width="800" style="border: 1px solid #ddd; box-shadow: 0 2px 8px rgba(0,0,0,0.1);"/></a>

*Πολύ-γύρος συνομιλία που δείχνει απλές μετατροπές, αναζητήσεις καιρού και αλυσιδωτή χρήση εργαλείων*

### Πειραματιστείτε με Διάφορα Αιτήματα

Δοκιμάστε διαφορετικούς συνδυασμούς:
- Αναζητήσεις καιρού: "Ποιος είναι ο καιρός στο Τόκιο;"
- Μετατροπές θερμοκρασίας: "Πόσο είναι 25°C σε Κέλβιν;"
- Συνδυασμένα ερωτήματα: "Ελέγξτε τον καιρό στο Παρίσι και πείτε μου αν είναι πάνω από 20°C"

Παρατηρήστε πώς ο πράκτορας ερμηνεύει τη φυσική γλώσσα και τη χαρτογραφεί σε κατάλληλες κλήσεις εργαλείων.

## Κύριες Έννοιες

### Πρότυπο ReAct (Σκέψη και Δράση)

Ο πράκτορας εναλλάσσεται μεταξύ σκέψης (αποφασίζοντας τι να κάνει) και δράσης (χρησιμοποιώντας εργαλεία). Αυτό το πρότυπο επιτρέπει αυτόνομη επίλυση προβλημάτων αντί απλώς ανταπόκρισης σε οδηγίες.

### Σημασία Περιγραφής Εργαλείων

Η ποιότητα των περιγραφών των εργαλείων επηρεάζει άμεσα το πώς τα χρησιμοποιεί ο πράκτορας. Σαφείς, συγκεκριμένες περιγραφές βοηθούν το μοντέλο να καταλάβει πότε και πώς να καλέσει κάθε εργαλείο.

### Διαχείριση Συνεδρίας

Η σημείωση `@MemoryId` ενεργοποιεί αυτόματη διαχείριση μνήμης βασισμένη σε συνεδρίες. Κάθε αναγνωριστικό συνεδρίας αποκτά τη δική του περίπτωση `ChatMemory` που διαχειρίζεται το bean `ChatMemoryProvider`, έτσι ώστε πολλοί χρήστες να μπορούν να αλληλεπιδρούν με τον πράκτορα ταυτόχρονα χωρίς να αναμειγνύονται οι συνομιλίες τους.

<img src="../../../translated_images/el/session-management.91ad819c6c89c400.webp" alt="Διαχείριση Συνεδρίας με @MemoryId" width="800"/>

*Κάθε αναγνωριστικό συνεδρίας αντιστοιχεί σε απομονωμένο ιστορικό συνομιλίας — οι χρήστες δεν βλέπουν ποτέ τα μηνύματα των άλλων.*

### Διαχείριση Σφαλμάτων

Τα εργαλεία μπορεί να αποτύχουν — οι API μπορεί να έχουν λήξη χρόνου, παράμετροι ίσως είναι άκυροι, εξωτερικές υπηρεσίες μπορεί να πέσουν. Οι πράκτορες παραγωγής χρειάζονται διαχείριση σφαλμάτων ώστε το μοντέλο να μπορεί να εξηγεί τα προβλήματα ή να δοκιμάζει εναλλακτικές αντί να καταρρέει ολόκληρη η εφαρμογή. Όταν ένα εργαλείο ρίχνει εξαίρεση, το LangChain4j την πιάνει και τροφοδοτεί το μήνυμα σφάλματος πίσω στο μοντέλο, που στη συνέχεια μπορεί να εξηγήσει το πρόβλημα με φυσική γλώσσα.

## Διαθέσιμα Εργαλεία

Το διάγραμμα παρακάτω παρουσιάζει το ευρύ οικοσύστημα εργαλείων που μπορείτε να κατασκευάσετε. Αυτό το module επιδεικνύει εργαλεία καιρού και θερμοκρασίας, αλλά το ίδιο πρότυπο `@Tool` δουλεύει για οποιαδήποτε μέθοδο Java — από ερωτήματα βάσεων δεδομένων μέχρι επεξεργασία πληρωμών.

<img src="../../../translated_images/el/tool-ecosystem.aad3d74eaa14a44f.webp" alt="Οικοσύστημα Εργαλείων" width="800"/>

*Οποιαδήποτε μέθοδος Java σημειωμένη με @Tool γίνεται διαθέσιμη στην AI — το πρότυπο επεκτείνεται σε βάσεις δεδομένων, APIs, email, λειτουργίες αρχείων, και άλλα.*

## Πότε να Χρησιμοποιείτε Πράκτορες με Εργαλεία

<img src="../../../translated_images/el/when-to-use-tools.51d1592d9cbdae9c.webp" alt="Πότε να Χρησιμοποιείτε Εργαλεία" width="800"/>

*Ένας γρήγορος οδηγός αποφάσεων — τα εργαλεία είναι για δεδομένα σε πραγματικό χρόνο, υπολογισμούς και ενέργειες· η γενική γνώση και οι δημιουργικές εργασίες δεν τα χρειάζονται.*

**Χρησιμοποιήστε εργαλεία όταν:**
- Η απάντηση απαιτεί δεδομένα σε πραγματικό χρόνο (καιρός, τιμές μετοχών, αποθέματα)
- Πρέπει να κάνετε υπολογισμούς πέραν απλής αριθμητικής
- Πρόσβαση σε βάσεις δεδομένων ή APIs
- Εκτέλεση ενεργειών (αποστολή email, δημιουργία αιτημάτων, ενημέρωση εγγραφών)
- Συνδυασμός πολλαπλών πηγών δεδομένων

**Μην χρησιμοποιείτε εργαλεία όταν:**
- Οι ερωτήσεις απαντώνται από γενική γνώση
- Η απάντηση είναι καθαρά συνομιλιακή
- Η καθυστέρηση του εργαλείου θα έκανε την εμπειρία πολύ αργή

## Εργαλεία vs RAG

Τα modules 03 και 04 επεκτείνουν τι μπορεί να κάνει η AI, αλλά με θεμελιωδώς διαφορετικούς τρόπους. Το RAG δίνει στο μοντέλο πρόσβαση σε **γνώση** μέσω ανάκτησης εγγράφων. Τα εργαλεία δίνουν στο μοντέλο την ικανότητα να εκτελεί **ενέργειες** μέσω κλήσεων συναρτήσεων.

<img src="../../../translated_images/el/tools-vs-rag.ad55ce10d7e4da87.webp" alt="Σύγκριση Εργαλείων με RAG" width="800"/>

*Το RAG ανακτά πληροφορίες από στατικά έγγραφα — τα εργαλεία εκτελούν ενέργειες και φέρνουν δυναμικά, σε πραγματικό χρόνο δεδομένα. Πολλά συστήματα παραγωγής συνδυάζουν και τα δύο.*

Στην πράξη, πολλά συστήματα παραγωγής συνδυάζουν και τις δύο προσεγγίσεις: RAG για θεμελίωση απαντήσεων στη τεκμηρίωσή σας, και Εργαλεία για αναζήτηση ζωντανών δεδομένων ή εκτέλεση λειτουργιών.
