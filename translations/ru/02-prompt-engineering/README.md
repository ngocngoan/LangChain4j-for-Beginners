## Чему вы научитесь

<img src="../../../translated_images/ru/what-youll-learn.c68269ac048503b2.webp" alt="Что вы изучите" width="800"/>

В предыдущем модуле вы увидели, как память позволяет создавать разговорный ИИ, и использовали модели GitHub для базовых взаимодействий. Теперь мы сосредоточимся на том, как вы задаёте вопросы — то есть сами запросы — с использованием GPT-5.2 от Azure OpenAI. То, как вы структурируете запросы, существенно влияет на качество ответов. Мы начнем с обзора основных техник создания запросов, а затем перейдём к восьми продвинутым шаблонам, которые полностью используют возможности GPT-5.2.

Мы используем GPT-5.2, потому что он вводит управление рассуждениями — вы можете указывать модели, сколько размышлять перед ответом. Это делает стратегии создания запросов более очевидными и помогает понять, когда применять тот или иной подход. Также мы получаем преимущество в виде меньших ограничений по лимитам Azure для GPT-5.2 по сравнению с моделями GitHub.

## Понимание создания запросов

<img src="../../../translated_images/ru/what-is-prompt-engineering.5c392a228a1f5823.webp" alt="Что такое создание запросов?" width="800"/>

Создание запросов — это разработка входного текста, который стабильно даёт необходимые вам результаты. Это не просто постановка вопросов — это структурирование запросов так, чтобы модель точно понимала, чего вы хотите и как это предоставить.

Подумайте об этом как об инструкциях для коллеги. «Исправь баг» — слишком общее. «Исправь ошибку NullPointerException в UserService.java на строке 45, добавив проверку на null» — конкретно. С языковыми моделями то же — важна конкретика и структура.

<img src="../../../translated_images/ru/how-langchain4j-fits.dfff4b0aa5f7812d.webp" alt="Как работает LangChain4j" width="800"/>

LangChain4j предоставляет инфраструктуру — подключение к моделям, память и типы сообщений — а шаблоны запросов — это просто тщательно структурированный текст, который вы передаете через эту инфраструктуру. Ключевые строительные блоки — это `SystemMessage` (задаёт поведение и роль ИИ) и `UserMessage` (содержит ваш фактический запрос).

## Основы создания запросов

<img src="../../../translated_images/ru/five-patterns-overview.160f35045ffd2a94.webp" alt="Обзор пяти шаблонов создания запросов" width="800"/>

Прежде чем перейти к продвинутым шаблонам этого модуля, давайте вспомним пять базовых техник создания запросов. Это строительные блоки, которые должен знать каждый создатель запросов. Если вы уже проходили [Модуль Быстрого старта](../00-quick-start/README.md#2-prompt-patterns), вы уже видели их на практике — здесь показана концептуальная основа.

### Нулевая подготовка (Zero-Shot)

Самый простой подход: дать модели прямую инструкцию без примеров. Модель целиком опирается на своё обучение, чтобы понять и выполнить задачу. Такой подход хорошо работает для очевидных заданий.

<img src="../../../translated_images/ru/zero-shot-prompting.7abc24228be84e6c.webp" alt="Нулевая подготовка" width="800"/>

*Прямая инструкция без примеров — модель выводит задачу только из инструкции*

```java
String prompt = "Classify this sentiment: 'I absolutely loved the movie!'";
String response = model.chat(prompt);
// Ответ: "Позитивный"
```

**Когда использовать:** Простая классификация, прямые вопросы, переводы или любые задачи, которые модель может решить без дополнительного руководства.

### Подготовка с примерами (Few-Shot)

Предоставьте примеры, показывающие шаблон, которому модель должна следовать. Модель узнает ожидаемый формат входа и выхода из ваших примеров и применяет его к новым данным. Это существенно повышает последовательность там, где формат или поведение неочевидны.

<img src="../../../translated_images/ru/few-shot-prompting.9d9eace1da88989a.webp" alt="Подготовка с примерами" width="800"/>

*Обучение на примерах — модель распознаёт шаблон и применяет его для новых данных*

```java
String prompt = """
    Classify the sentiment as positive, negative, or neutral.
    
    Examples:
    Text: "This product exceeded my expectations!" → Positive
    Text: "It's okay, nothing special." → Neutral
    Text: "Waste of money, very disappointed." → Negative
    
    Now classify this:
    Text: "Best purchase I've made all year!"
    """;
String response = model.chat(prompt);
```

**Когда использовать:** Кастомная классификация, единообразное форматирование, задачи из специфичных областей или когда результаты нулевой подготовки нестабильны.

### Последовательное рассуждение

Попросите модель показывать свои рассуждения шаг за шагом. Вместо того чтобы сразу выдать ответ, модель разбирает проблему и подробно объясняет каждый этап. Это повышает точность для задач в математике, логике и многоступенчатом рассуждении.

<img src="../../../translated_images/ru/chain-of-thought.5cff6630e2657e2a.webp" alt="Последовательное рассуждение" width="800"/>

*Пошаговое рассуждение — разбиение сложных задач на явные логические этапы*

```java
String prompt = """
    Problem: A store has 15 apples. They sell 8 apples and then 
    receive a shipment of 12 more apples. How many apples do they have now?
    
    Let's solve this step-by-step:
    """;
String response = model.chat(prompt);
// Модель показывает: 15 - 8 = 7, затем 7 + 12 = 19 яблок
```

**Когда использовать:** Математические задачи, логические головоломки, отладка или любые задачи, где демонстрация процесса рассуждений улучшает точность и доверие.

### Создание запросов на основе роли

Задайте ИИ персону или роль перед вопросом. Это задаёт контекст, формирующий тон, глубину и фокус ответа. "Архитектор ПО" даст один совет, а "младший разработчик" или "аудитор безопасности" — другой.

<img src="../../../translated_images/ru/role-based-prompting.a806e1a73de6e3a4.webp" alt="Создание запросов на основе роли" width="800"/>

*Определение контекста и персоны — один и тот же вопрос получает разные ответы в зависимости от роли*

```java
String prompt = """
    You are an experienced software architect reviewing code.
    Provide a brief code review for this function:
    
    def calculate_total(items):
        total = 0
        for item in items:
            total = total + item['price']
        return total
    """;
String response = model.chat(prompt);
```

**Когда использовать:** Ревью кода, обучение, аналитика по предметной области или когда нужны ответы, адаптированные под уровень экспертизы или точку зрения.

### Шаблоны запросов

Создавайте повторно используемые запросы с переменными заполнителями. Вместо того чтобы писать новый запрос каждый раз, определите шаблон один раз и подставляйте разные значения. Класс `PromptTemplate` в LangChain4j облегчает это с помощью синтаксиса `{{variable}}`.

<img src="../../../translated_images/ru/prompt-templates.14bfc37d45f1a933.webp" alt="Шаблоны запросов" width="800"/>

*Повторно используемые запросы с переменными заполнителями — один шаблон, много применений*

```java
PromptTemplate template = PromptTemplate.from(
    "What's the best time to visit {{destination}} for {{activity}}?"
);

Prompt prompt = template.apply(Map.of(
    "destination", "Paris",
    "activity", "sightseeing"
));

String response = model.chat(prompt.text());
```

**Когда использовать:** Повторяющиеся запросы с разными входными данными, пакетная обработка, создание многократно используемых AI-воркфлоу или любые сценарии, где структура запроса остаётся неизменной, а данные меняются.

---

Эти пять основ дают вам надёжный набор инструментов для большинства задач создания запросов. Остальная часть этого модуля строится на них с помощью **восьми продвинутых шаблонов**, которые используют управление рассуждениями GPT-5.2, самооценку и структурированный вывод.

## Продвинутые шаблоны

После освоения основ перейдём к восьми продвинутым шаблонам, делающим этот модуль уникальным. Не все задачи решаются одинаково. Одни требуют быстрых ответов, другие — глубоких размышлений. Кому-то важны видимые рассуждения, кому-то — только результаты. Каждый из приведённых ниже шаблонов оптимизирован для разной ситуации — и управление рассуждениями в GPT-5.2 ярко проявляет эти различия.

<img src="../../../translated_images/ru/eight-patterns.fa1ebfdf16f71e9a.webp" alt="Восемь шаблонов создания запросов" width="800"/>

*Обзор восьми шаблонов создания запросов и их сферы применения*

<img src="../../../translated_images/ru/reasoning-control.5cf85f0fc1d0c1f3.webp" alt="Управление рассуждениями с GPT-5.2" width="800"/>

*Управление рассуждениями GPT-5.2 позволяет указывать, сколько модель должна думать — от быстрых прямых ответов до глубокого анализа*

**Низкая вовлечённость (быстро и сфокусировано)** — для простых вопросов, где нужны быстрые, прямые ответы. Модель выполняет минимальное рассуждение — максимум 2 шага. Используйте для вычислений, запросов и простых вопросов.

```java
String prompt = """
    <context_gathering>
    - Search depth: very low
    - Bias strongly towards providing a correct answer as quickly as possible
    - Usually, this means an absolute maximum of 2 reasoning steps
    - If you think you need more time, state what you know and what's uncertain
    </context_gathering>
    
    Problem: What is 15% of 200?
    
    Provide your answer:
    """;

String response = chatModel.chat(prompt);
```

> 💡 **Изучите с GitHub Copilot:** Откройте [`Gpt5PromptService.java`](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java) и спросите:
> - "В чём разница между шаблонами с низкой и высокой вовлечённостью?"
> - "Как XML теги в запросах помогают структурировать ответ ИИ?"
> - "Когда следует использовать шаблоны с самоанализом вместо прямых инструкций?"

**Высокая вовлечённость (глубоко и тщательно)** — для сложных задач, где нужна полная аналитика. Модель внимательно изучает и показывает детальные рассуждения. Используйте для системного проектирования, архитектурных решений и сложных исследований.

```java
String prompt = """
    Analyze this problem thoroughly and provide a comprehensive solution.
    Consider multiple approaches, trade-offs, and important details.
    Show your analysis and reasoning in your response.
    
    Problem: Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```

**Выполнение задач (пошаговый прогресс)** — для многоэтапных процессов. Модель заранее строит план, комментирует каждый шаг по мере выполнения и даёт итог. Используйте для миграций, внедрений или сложных многошаговых процедур.

```java
String prompt = """
    <task_execution>
    1. First, briefly restate the user's goal in a friendly way
    
    2. Create a step-by-step plan:
       - List all steps needed
       - Identify potential challenges
       - Outline success criteria
    
    3. Execute each step:
       - Narrate what you're doing
       - Show progress clearly
       - Handle any issues that arise
    
    4. Summarize:
       - What was completed
       - Any important notes
       - Next steps if applicable
    </task_execution>
    
    <tool_preambles>
    - Always begin by rephrasing the user's goal clearly
    - Outline your plan before executing
    - Narrate each step as you go
    - Finish with a distinct summary
    </tool_preambles>
    
    Task: Create a REST endpoint for user registration
    
    Begin execution:
    """;

String response = chatModel.chat(prompt);
```

Prompting chain-of-thought (последовательное рассуждение) явно просит модель показать процесс рассуждений, что повышает точность для сложных задач. Пошаговое разбиение помогает понимать логику и людям, и ИИ.

> **🤖 Попробуйте с [GitHub Copilot](https://github.com/features/copilot) Chat:** Спросите о шаблоне:
> - "Как адаптировать шаблон выполнения задач для длительных операций?"
> - "Какие лучшие практики для структурирования вступительных блоков инструментов в производственных приложениях?"
> - "Как можно захватывать и отображать промежуточный прогресс в UI?"

<img src="../../../translated_images/ru/task-execution-pattern.9da3967750ab5c1e.webp" alt="Шаблон выполнения задач" width="800"/>

*План → Выполнение → Итог для многошаговых задач*

**Самоанализ кода** — для генерации кода производственного качества. Модель создаёт код по стандартам, с правильной обработкой ошибок. Используйте для создания новых функций и сервисов.

```java
String prompt = """
    Generate Java code with production-quality standards: Create an email validation service
    Keep it simple and include basic error handling.
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/ru/self-reflection-cycle.6f71101ca0bd28cc.webp" alt="Цикл самоанализа" width="800"/>

*Итеративный цикл улучшений — сгенерировать, оценить, выявить проблемы, улучшить, повторить*

**Структурированный анализ** — для последовательной оценки. Модель анализирует код по фиксированной схеме (корректность, практики, производительность, безопасность, поддерживаемость). Используйте для ревью кода или оценки качества.

```java
String prompt = """
    <analysis_framework>
    You are an expert code reviewer. Analyze the code for:
    
    1. Correctness
       - Does it work as intended?
       - Are there logical errors?
    
    2. Best Practices
       - Follows language conventions?
       - Appropriate design patterns?
    
    3. Performance
       - Any inefficiencies?
       - Scalability concerns?
    
    4. Security
       - Potential vulnerabilities?
       - Input validation?
    
    5. Maintainability
       - Code clarity?
       - Documentation?
    
    <output_format>
    Provide your analysis in this structure:
    - Summary: One-sentence overall assessment
    - Strengths: 2-3 positive points
    - Issues: List any problems found with severity (High/Medium/Low)
    - Recommendations: Specific improvements
    </output_format>
    </analysis_framework>
    
    Code to analyze:
    ```
    public List getUsers() {
        return database.query("SELECT * FROM users");
    }
    ```
    Provide your structured analysis:
    """;

String response = chatModel.chat(prompt);
```

> - "Как настроить схему анализа для разных видов ревью кода?"
> - "Как лучше всего разбирать и программно обрабатывать структурированный вывод?"
> - "Как обеспечить одинаковую степень серьёзности во всех сессиях ревью?"

<img src="../../../translated_images/ru/structured-analysis-pattern.0af3b690b60cf2d6.webp" alt="Шаблон структурированного анализа" width="800"/>

*Фреймворк для последовательного ревью кода с уровнями серьёзности*

**Многоходовой чат** — для диалогов, где важен контекст. Модель запоминает предыдущие сообщения и строит ответы на их основе. Используйте для интерактивной поддержки и сложных вопросов-ответов.

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("What is Spring Boot?"));
AiMessage aiMessage1 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage1);

memory.add(UserMessage.from("Show me an example"));
AiMessage aiMessage2 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage2);
```

<img src="../../../translated_images/ru/context-memory.dff30ad9fa78832a.webp" alt="Память контекста" width="800"/>

*Как накопление контекста в диалогах растёт по мере нескольких ходов, пока не достигнется лимит токенов*

**Пошаговое рассуждение** — для задач, требующих видимой логики. Модель показывает рассуждения на каждом шаге. Используйте для математических задач, логических головоломок или когда надо понять процесс мышления.

```java
String prompt = """
    <instruction>Show your reasoning step-by-step</instruction>
    
    If a train travels 120 km in 2 hours, then stops for 30 minutes,
    then travels another 90 km in 1.5 hours, what is the average speed
    for the entire journey including the stop?
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/ru/step-by-step-pattern.a99ea4ca1c48578c.webp" alt="Шаблон пошагового рассуждения" width="800"/>

*Разбиение задач на явные логические шаги*

**Ограниченный вывод** — для ответов с определёнными требованиями к формату. Модель строго соблюдает правила формата и длины. Используйте для резюме или когда нужна точная структура вывода.

```java
String prompt = """
    <constraints>
    - Exactly 100 words
    - Bullet point format
    - Technical terms only
    </constraints>
    
    Summarize the key concepts of machine learning.
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/ru/constrained-output-pattern.0ce39a682a6795c2.webp" alt="Шаблон ограниченного вывода" width="800"/>

*Принудительное соблюдение требований по формату, длине и структуре*

## Изучение шаблонов

Веб-интерфейс позволяет экспериментировать с разными стратегиями prompt-ов. Каждый шаблон решает разные задачи — попробуйте их, чтобы понять, когда каждый подход работает лучше.

> **Примечание: Потоковый vs Непотоковый режим** — На каждой странице шаблона есть две кнопки: **🔴 Stream Response (Live)** и **Непотоковый** вариант. Потоковый режим использует SSE (Server-Sent Events), чтобы показывать токены в реальном времени по мере их генерации моделью — вы видите ход выполнения сразу. Непотоковый режим ждёт полный ответ перед отображением. Для запросов с глубоким рассуждением (например, High Eagerness, Self-Reflecting Code) непотоковый вызов может занимать очень много времени — иногда минуты — без видимой обратной связи. **Используйте потоковый режим при экспериментах с сложными prompt-ами**, чтобы видеть работу модели и не думать, что запрос завис.
>
> **Примечание: Требования к браузеру** — Потоковая функция использует Fetch Streams API (`response.body.getReader()`), который поддерживается полноценными браузерами (Chrome, Edge, Firefox, Safari). Она **не работает** в встроенном в VS Code Simple Browser, так как его веб-просмотрщик не поддерживает API ReadableStream. Если вы используете Simple Browser, непотоковые кнопки будут работать нормально — только потоковые кнопки недоступны. Для полного опыта откройте `http://localhost:8083` во внешнем браузере.

### Низкая vs Высокая интенсивность мышления

Задайте простой вопрос, например «Что такое 15% от 200?» с Низкой интенсивностью. Вы получите мгновенный, прямой ответ. Теперь задайте что-то сложное вроде «Разработай стратегию кеширования для высоконагруженного API» с Высокой интенсивностью. Нажмите **🔴 Stream Response (Live)** и наблюдайте, как детальное рассуждение модели появляется по токенам. Та же модель, та же структура вопроса — но prompt сообщает, сколько думать.

### Исполнение задач (Tool Preambles)

Многоэтапные рабочие процессы выигрывают от предварительного планирования и озвучивания прогресса. Модель описывает, что будет делать, рассказывает о каждом шаге, а затем подводит итоги.

### Саморефлексирующий код

Попробуйте «Создать сервис проверки электронной почты». Вместо простого генератора кода модель создаёт, оценивает по критериям качества, выявляет слабые места и улучшает. Вы увидите, как она итерирует до тех пор, пока код не соответствует производственным стандартам.

### Структурированный анализ

Код-ревью требует последовательных критериев оценки. Модель анализирует код с использованием фиксированных категорий (корректность, практики, производительность, безопасность) с уровнями серьёзности.

### Многоходовой чат

Спросите «Что такое Spring Boot?», затем сразу же добавьте «Покажи пример». Модель помнит ваш первый вопрос и даёт пример именно по Spring Boot. Без памяти второй вопрос был бы слишком расплывчатым.

### Пошаговое рассуждение

Выберите математическую задачу и попробуйте решить её с помощью Пошагового рассуждения и Низкой интенсивности. Низкая интенсивность даёт быстрый, но поверхностный ответ. Пошаговое показывает все вычисления и решения.

### Ограниченный вывод

Когда нужен конкретный формат или точное количество слов, этот шаблон заставляет строго соблюдать требования. Попробуйте сгенерировать резюме ровно из 100 слов в формате маркированного списка.

## Что вы действительно изучаете

**Интенсивность мышления меняет всё**

GPT-5.2 позволяет контролировать вычислительные усилия через ваши prompt-ы. Низкая интенсивность = быстрые ответы с минимумом перебора. Высокая интенсивность = модель тратит время на глубокое размышление. Вы учитесь подбирать усилия под сложность задачи — не тратьте время на простые вопросы, но и не спешите с важными решениями.

**Структура направляет поведение**

Видите XML-теги в prompt-ах? Это не просто украшение. Модели надёжнее следуют структурированным инструкциям, чем свободному тексту. Когда нужна многошаговая логика или сложные процессы, структура помогает модели отслеживать текущий этап и следующий.

<img src="../../../translated_images/ru/prompt-structure.a77763d63f4e2f89.webp" alt="Prompt Structure" width="800"/>

*Анатомия хорошо структурированного prompt-а с чёткими секциями и XML-подобной организацией*

**Качество через самооценку**

Саморефлексивные шаблоны работают, делая критерии качества явными. Вместо того чтобы надеяться, что модель «сделает правильно», вы чётко указываете, что значит «правильно»: корректная логика, обработка ошибок, производительность, безопасность. Модель может оценить собственный вывод и улучшить его. Это превращает генерацию кода из лотереи в процесс.

**Контекст ограничен**

Многоходовые разговоры работают, включая историю сообщений в каждый запрос. Но есть лимит — у каждой модели максимальное количество токенов. По мере роста разговоров нужно уметь удерживать релевантный контекст, не превышая этот лимит. Этот модуль показывает, как работает память; позже вы узнаете, когда суммировать, когда забывать и когда извлекать.
