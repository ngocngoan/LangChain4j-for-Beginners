## Основные понятия

**AI Agent** - Система, использующая ИИ для автономного рассуждения и действий. [Модуль 04](../04-tools/README.md)

**Chain** - Последовательность операций, где вывод является входом для следующего шага.

**Chunking** - Разбиение документов на меньшие части. Обычно: 300-500 токенов с перекрытием. [Модуль 03](../03-rag/README.md)

**Context Window** - Максимальное количество токенов, которые модель может обработать. GPT-5.2: 400К токенов.

**Embeddings** - Числовые векторы, представляющие смысл текста. [Модуль 03](../03-rag/README.md)

**Function Calling** - Модель генерирует структурированные запросы для вызова внешних функций. [Модуль 04](../04-tools/README.md)

**Hallucination** - Когда модели генерируют неверную, но правдоподобную информацию.

**Prompt** - Текстовый ввод в языковую модель. [Модуль 02](../02-prompt-engineering/README.md)

**Semantic Search** - Поиск по смыслу с использованием эмбеддингов, а не ключевых слов. [Модуль 03](../03-rag/README.md)

**Stateful vs Stateless** - Stateless: без памяти. Stateful: сохраняет историю диалога. [Модуль 01](../01-introduction/README.md)

**Tokens** - Основные текстовые единицы, которые моделям нужно обработать. Влияют на цены и лимиты. [Модуль 01](../01-introduction/README.md)

**Tool Chaining** - Последовательное выполнение инструментов, где вывод влияет на следующий вызов. [Модуль 04](../04-tools/README.md)

## Компоненты LangChain4j

**AiServices** - Создает типобезопасные интерфейсы AI-сервисов.

**OpenAiOfficialChatModel** - Унифицированный клиент для моделей OpenAI и Azure OpenAI.

**OpenAiOfficialEmbeddingModel** - Создает эмбеддинги с помощью официального клиента OpenAI (поддерживает оба: OpenAI и Azure OpenAI).

**ChatModel** - Основной интерфейс языковых моделей.

**ChatMemory** - Поддерживает историю диалога.

**ContentRetriever** - Ищет релевантные части документов для RAG.

**DocumentSplitter** - Разбивает документы на части.

**EmbeddingModel** - Преобразует текст в числовые векторы.

**EmbeddingStore** - Хранит и извлекает эмбеддинги.

**MessageWindowChatMemory** - Поддерживает «скользящее окно» последних сообщений.

**PromptTemplate** - Создает повторно используемые подсказки с заполнителями `{{variable}}`.

**TextSegment** - Текстовый сегмент с метаданными. Используется в RAG.

**ToolExecutionRequest** - Представляет запрос на выполнение инструмента.

**UserMessage / AiMessage / SystemMessage** - Типы сообщений в диалоге.

## Понятия ИИ/МО

**Few-Shot Learning** - Предоставление примеров в подсказках. [Модуль 02](../02-prompt-engineering/README.md)

**Large Language Model (LLM)** - ИИ-модели, обученные на огромных объемах текстовых данных.

**Reasoning Effort** - Параметр GPT-5.2, контролирующий глубину рассуждений. [Модуль 02](../02-prompt-engineering/README.md)

**Temperature** - Контролирует случайность вывода. Низкая = детерминированный, высокая = креативный.

**Vector Database** - Специализированная база данных для эмбеддингов. [Модуль 03](../03-rag/README.md)

**Zero-Shot Learning** - Выполнение задач без примеров. [Модуль 02](../02-prompt-engineering/README.md)

## Guardrails - [Модуль 00](../00-quick-start/README.md)

**Defense in Depth** - Многоуровневый подход к безопасности, объединяющий «ограждения» на уровне приложения с фильтрами безопасности провайдера.

**Hard Block** - Провайдер возвращает ошибку HTTP 400 при серьезных нарушениях контента.

**InputGuardrail** - Интерфейс LangChain4j для валидации пользовательского ввода до передачи в LLM. Экономит расходы и время, блокируя вредоносные подсказки на раннем этапе.

**InputGuardrailResult** - Тип результата валидации guardrail: `success()` или `fatal("reason")`.

**OutputGuardrail** - Интерфейс для проверки ответов ИИ перед возвратом пользователю.

**Provider Safety Filters** - Встроенные фильтры контента у провайдеров ИИ (например, GitHub Models), которые перехватывают нарушения на уровне API.

**Soft Refusal** - Модель вежливо отказывается отвечать без генерации ошибки.

## Agentic Module - [Модуль 05](../05-mcp/README.md)

**@Agent Annotation** - Помечает интерфейсы как ИИ-агентов с декларативным определением поведения.

**Agent Listener** - Хук для мониторинга выполнения агента через `beforeAgentInvocation()` и `afterAgentInvocation()`.

**Agentic Scope** - Общая память, где агенты сохраняют результаты с помощью `outputKey` для последующих агентов.

**AgenticServices** - Фабрика для создания агентов через `agentBuilder()` и `supervisorBuilder()`.

**Conditional Workflow** - Маршрутизация на различных специалистов в зависимости от условий.

**Human-in-the-Loop** - Паттерн, добавляющий контрольные точки с участием человека для утверждения или проверки контента.

**langchain4j-agentic** - Зависимость Maven для декларативного создания агентов (экспериментально).

**Loop Workflow** - Повторное выполнение агента до выполнения условия (например, оценка качества ≥ 0.8).

**outputKey** - Параметр аннотации агента, указывающий, где сохраняются результаты в Agentic Scope.

**Parallel Workflow** - Одновременный запуск нескольких агентов для независимых задач.

**Response Strategy** - Как супервайзер формулирует окончательный ответ: LAST, SUMMARY или SCORED.

**Sequential Workflow** - Выполнение агентов по порядку, где вывод передается на следующий шаг.

**Supervisor Agent Pattern** - Продвинутый агентный паттерн, где супервайзер LLM динамически решает, каких подагентов вызывать.

## Протокол контекста модели (MCP) - [Модуль 05](../05-mcp/README.md)

**langchain4j-mcp** - Зависимость Maven для интеграции MCP в LangChain4j.

**MCP** - Model Context Protocol: стандарт для подключения AI-приложений к внешним инструментам. Один раз создать, использовать везде.

**MCP Client** - Приложение, подключающееся к MCP-серверам для обнаружения и использования инструментов.

**MCP Server** - Сервис, предоставляющий инструменты через MCP с понятным описанием и схемами параметров.

**McpToolProvider** - Компонент LangChain4j, оборачивающий MCP-инструменты для использования в AI-сервисах и агентах.

**McpTransport** - Интерфейс для MCP-коммуникации. Реализации включают Stdio и HTTP.

**Stdio Transport** - Локальный транспорт процесса через stdin/stdout. Полезен для доступа к файловой системе или командной строке.

**StdioMcpTransport** - Реализация LangChain4j, запускающая MCP-сервер как подпроцесс.

**Tool Discovery** - Клиент запрашивает у сервера доступные инструменты с описаниями и схемами.
