## तपाईंले के सिक्नुहुनेछ

यदि तपाईंले छिटो सुरु गरेको खण्ड पूरा गर्नुभयो भने, तपाईंले कसरी प्रॉम्प्ट पठाउने र जवाफ प्राप्त गर्ने भनेर देख्नुभयो। त्यो आधार हो, तर वास्तविक अनुप्रयोगहरूलाई बढी चाहिन्छ। यो मोड्युलले तपाईंलाई यस्तो वार्तालापात्मक AI निर्माण गर्न सिकाउँछ जुन सन्दर्भ सम्झन्छ र अवस्था कायम राख्छ - जुन एक पटकको डेमो र उत्पादन-तयार अनुप्रयोग बीचको फरक हो।

यो मार्गदर्शनमा हामी Azure OpenAI को GPT-5.2 प्रयोग गर्नेछौं किनकि यसको उन्नत तर्क क्षमताहरूले विभिन्न बानीहरू स्पष्ट बनाउँछ। जब तपाईंले स्मृति थप्नुहुन्छ, तपाईंले फरक स्पष्ट देख्नुहुनेछ। यसले तपाईंको अनुप्रयोगलाई प्रत्येक घटकले के ल्याउँछ बुझ्न सजिलो बनाउँछ।

तपाईंले एक अनुप्रयोग बनाउनुहुनेछ जसले दुवै बानीहरू प्रदर्शन गर्छ:

**स्टेटलेस च्याट** - प्रत्येक अनुरोध स्वतन्त्र हुन्छ। मोडलले अघिल्लो सन्देशहरूको सम्झना गर्दैन। यो पछिल्लो छिटो सुरुमा तपाईंले प्रयोग गर्नुभएको बानी हो।

**स्टेटफुल वार्तालाप** - प्रत्येक अनुरोधमा वार्तालाप इतिहास समावेश हुन्छ। मोडलले धेरै पटकको संवादमा सन्दर्भ कायम राख्छ। उत्पादन अनुप्रयोगहरूले यसलाई चाहिन्छ।

## पूर्वआवश्यकताहरू

- Azure सदस्यता जसमा Azure OpenAI पहुँच छ
- Java 21, Maven 3.9+
- Azure CLI (https://learn.microsoft.com/en-us/cli/azure/install-azure-cli)
- Azure Developer CLI (azd) (https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/install-azd)

> **सूचना:** Java, Maven, Azure CLI र Azure Developer CLI (azd) प्रदान गरिएको devcontainer मा पहिल्यै इन्स्टल गरिएको छ।

> **सूचना:** यो मोड्युल Azure OpenAI मा GPT-5.2 प्रयोग गर्दछ। परिनियोजन `azd up` मार्फत स्वचालित रूपमा कन्फिगर गरिएको छ - कोडमा मोडल नाम परिवर्तन नगर्नुहोस्।

## कोर समस्याको बुझाइ

भाषा मोडलहरू स्टेटलेस हुन्छन्। प्रत्येक API कल स्वतन्त्र हुन्छ। यदि तपाईंले "मेरो नाम जोन हो" भन्नुभयो र त्यसपछि "मेरो नाम के हो?" सोध्नुभयो भने मोडललाई थाहा हुँदैन तपाईंले आफूलाई भर्खर चिनाउनुभयो। यो प्रत्येक अनुरोधलाई पहिलो वार्तालापजस्तो व्यवहार गर्छ।

यो सिधा प्रश्नोत्तरको लागि ठीक छ तर वास्तविक अनुप्रयोगहरूको लागि बेकार छ। ग्राहक सेवा बोटहरूले तपाईंले के भनेको सम्झनुपर्छ। व्यक्तिगत सहायकहरूले सन्दर्भ चाहिन्छ। कुनै पनि बहु-पटकको वार्तालापलाई स्मृतिको आवश्यक पर्दछ।

*स्टेटलेस (स्वतन्त्र कलहरू) र स्टेटफुल (सन्दर्भ-ज्ञानी) वार्तालाप बीचको भिन्नता*

## टोकन्स बुझ्ने

वार्तालापमा झर्ने पहिले, टोकन्स बुझ्न महत्वपूर्ण छ - भाषा मोडलहरूले प्रक्रिया गर्ने पाठका आधारभूत एकाइहरू:

<img src="../../../translated_images/ne/token-explanation.c39760d8ec650181.webp" alt="टोकन व्याख्या" width="800"/>

*कसरी पाठलाई टोकन्समा तोडिन्छ - "I love AI!" ४ अलग एकाइहरूमा परिणत हुन्छ*

टोकन्स नै AI मोडलहरूले मापन र प्रक्रिया गर्ने तरिका हो। शब्द, विराम चिह्न र यहाँसम्म कि खाली स्थान पनि टोकन्स हुन सक्छन्। तपाईंको मोडलले एकपटकमा कति टोकन्स प्रक्रिया गर्न सक्छ भन्ने सीमा छ (GPT-5.2 का लागि ४००,०००, जसमा २७२,००० इनपुट र १२८,००० आउटपुट टोकन्स सम्म)। टोकन्स बुझ्नुले वार्तालापको लामो समय र लागत व्यवस्थापनमा मद्दत गर्छ।

## स्मृति कसरी काम गर्छ

च्याट स्मृतिले स्टेटलेस समस्यालाई वार्तालाप इतिहास कायम राखेर समाधान गर्छ। मोडलमा अनुरोध पठाउनु अघि, फ्रेमवर्कले सम्बन्धित अघिल्लो सन्देशहरू थप्छ। जब तपाईं "मेरो नाम के हो?" भन्छौं, प्रणाली वास्तवमा पुरा वार्तालाप इतिहास पठाउँछ, जसले मोडललाई देखाउँछ कि तपाईंले पहिले "मेरो नाम जोन हो" भन्नुभयो।

LangChain4jले यस्तो स्मृति कार्यान्वयनहरू प्रदान गर्छ जुन यो स्वचालित रूपमा व्यवस्थापन गर्छ। तपाईंले कति सन्देशहरू राख्ने चयन गर्नुहुन्छ र फ्रेमवर्कले सन्दर्भ विन्डो चलाउँछ।

<img src="../../../translated_images/ne/memory-window.bbe67f597eadabb3.webp" alt="स्मृति विन्डो अवधारणा" width="800"/>

*MessageWindowChatMemory ले हालसालैका सन्देशहरूको एक स्लाइडिङ विन्डो कायम राख्छ, पुराना स्वतः हटाउँछ*

## LangChain4j कसरी प्रयोग गर्छ

यो मोड्युलले छिटो सुरु भएकोलाई विस्तार गर्दै Spring Boot समन्वय गर्दछ र वार्तालाप स्मृति थप्दछ। यहाँ तत्त्वहरू कसरी जोडिन्छन्:

**निर्भरता** - दुई LangChain4j लाइब्रेरीहरू थप्नुहोस्:

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```

**च्याट मोडल** - Azure OpenAI लाई Spring bean को रूपमा कन्फिगर गर्नुहोस् ([LangChainConfig.java](../../../01-introduction/src/main/java/com/example/langchain4j/config/LangChainConfig.java)):

```java
@Bean
public OpenAiOfficialChatModel openAiOfficialChatModel() {
    return OpenAiOfficialChatModel.builder()
            .baseUrl(azureEndpoint)
            .apiKey(azureApiKey)
            .modelName(deploymentName)
            .timeout(Duration.ofMinutes(5))
            .maxRetries(3)
            .build();
}
```

बिल्डरले `azd up` द्वारा सेट गरिएको वातावरण चरहरूबाट प्रमाणपत्र पढ्छ। `baseUrl` तपाईंको Azure अन्त बिन्दुमा सेट गर्दा OpenAI क्लाइन्ट Azure OpenAI सँग काम गर्छ।

**वार्तालाप स्मृति** - MessageWindowChatMemory सँग च्याट इतिहास ट्र्याक गर्नुहोस् ([ConversationService.java](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java)):

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("My name is John"));
memory.add(AiMessage.from("Nice to meet you, John!"));

memory.add(UserMessage.from("What's my name?"));
AiMessage aiMessage = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage);
```

`withMaxMessages(10)` प्रयोग गरेर स्मृतिमा अन्तिम १० सन्देशहरू राख्नुहोस्। प्रयोगकर्ता र AI सन्देशहरू `UserMessage.from(text)` र `AiMessage.from(text)` मार्फत थप्नुहोस्। इतिहास ल्याउन `memory.messages()` प्रयोग गर्नुहोस् र मोडललाई पठाउनुहोस्। सेवा प्रत्येक वार्तालाप ID को लागि अलग स्मृति बनाउँछ जसले धेरै प्रयोगकर्ताहरूलाई एकै समयमा कुरा गर्न अनुमति दिन्छ।

> **🤖 [GitHub Copilot](https://github.com/features/copilot) च्याटसँग प्रयास गर्नुहोस्:** [`ConversationService.java`](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java) खोल्नुहोस् र सोध्नुहोस्:
> - "MessageWindowChatMemory ले विन्डो भरिएपछि कुन सन्देशहरू हटाउने निर्णय कसरी गर्छ?"
> - "के म कस्टम स्मृति भण्डारण इन-मेमोरीको सट्टा डेटाबेस प्रयोग गरेर लागू गर्न सक्छु?"
> - "पुराना वार्तालाप इतिहासलाई संक्षेप गर्न म कसरी सारांश थप्न सक्छु?"

स्टेटलेस च्याट अन्त बिन्दुले स्मृति पूरै छोड्छ - केवल `chatModel.chat(prompt)` जसरी छिटो सुरुमा थिय। स्टेटफुल अन्तबिन्दु सन्देशहरू स्मृतिमा थप्छ, इतिहास ल्याउँछ, र प्रत्येक अनुरोधसँग त्यो सन्दर्भ समावेश गर्दछ। एउटै मोडल कन्फिगरेसन, फरक बानीहरू।

## Azure OpenAI पूर्वाधार परिनियोजन गर्नुहोस्

**Bash:**
```bash
cd 01-introduction
azd up  # सदस्यता र स्थान चयन गर्नुहोस् (eastus2 सिफारिस गरिएको)
```

**PowerShell:**
```powershell
cd 01-introduction
azd up  # सदस्यता र स्थान चयन गर्नुहोस् (eastus2 सिफारिस गरिएको)
```

> **सूचना:** यदि तपाईंले टाइमआउट त्रुटि ( `RequestConflict: Cannot modify resource ... provisioning state is not terminal`) पाउनुभयो भने, `azd up` फेरि चलाउनुहोस्। Azure स्रोतहरू पृष्ठभूमिमा अझै परिनियोजन हुँदै हुन सक्छन्, र पुन: प्रयास गर्दा स्रोतहरू टर्मिनल अवस्थामा पुग्दा परिनियोजन पूरा हुन्छ।

यसले गर्नेछ:
1. GPT-5.2 र text-embedding-3-small मोडलसहित Azure OpenAI स्रोत परिनियोजन गर्ने
2. प्रमाणपत्रहरूसहित स्वत: `.env` फाइल परियोजना रुटमा सिर्जना गर्ने
3. सबै आवश्यक वातावरण चरहरू सेट गर्ने

**परिनियोजन समस्या छ?** विस्तारमा समस्या समाधानका लागि [Infrastructure README](infra/README.md) हेर्नुहोस् जसमा उपडोमेन नाम द्वन्द्व, Azure Portal मा म्यानुअल परिनियोजन चरणहरू, र मोडल कन्फिगरेसन मार्गदर्शन समावेश छ।

**परिनियोजन सफल भयो कि भनेर पुष्टि गर्नुहोस्:**

**Bash:**
```bash
cat ../.env  # AZURE_OPENAI_ENDPOINT, API_KEY, आदि देखाउनुपर्छ।
```

**PowerShell:**
```powershell
Get-Content ..\.env  # AZURE_OPENAI_ENDPOINT, API_KEY, आदिहरू देखाउनु पर्दछ।
```

> **सूचना:** `azd up` कमाण्डले स्वत: `.env` फाइल बनाउँछ। तपाईंलाई पछि यसलाई अपडेट गर्न परे, `.env` फाइल म्यानुअली सम्पादन गर्न सक्नुहुन्छ वा पुनः उत्पन्न गर्न तलको कमाण्ड चलाउन सक्नुहुन्छ:
>
> **Bash:**
> ```bash
> cd ..
> bash .azd-env.sh
> ```

> **PowerShell:**
> ```powershell
> cd ..
> .\.azd-env.ps1
> ```

दुबै स्क्रिप्टहरूले स्वतः रूपमा रुट `.env` फाइलबाट वातावरण चरहरू लोड गर्छन् र यदि JAR हरू छैनन् भने बिल्ड पनि गर्छन्।

> **सूचना:** यदि तपाईंले सुरु गर्नु अघि सबै मोड्युलहरू म्यानुअली बिल्ड गर्न चाहनुभयो भने:
>
> **Bash:**
> ```bash
> cd ..  # Go to root directory
> mvn clean package -DskipTests
> ```

> **PowerShell:**
> ```powershell
> cd ..  # Go to root directory
> mvn clean package -DskipTests
> ```

http://localhost:8080 आफ्नो ब्राउजरमा खोल्नुहोस्।

**रोक्नका लागि:**

## अनुप्रयोग प्रयोग गर्दै

अनुप्रयोगले दुई च्याट कार्यान्वयनहरू सँगसँगै वेब इन्टरफेस प्रदान गर्दछ।

<img src="../../../translated_images/ne/home-screen.121a03206ab910c0.webp" alt="अनुप्रयोग गृह स्क्रीन" width="800"/>

*ड्यासबोर्डले सरल च्याट (स्टेटलेस) र वार्तालापात्मक च्याट (स्टेटफुल) विकल्पहरू दुवै देखाउँछ*

### स्टेटलेस च्याट (बायाँ प्यानल)

पहिले यो प्रयास गर्नुहोस्। "मेरो नाम जोन हो" भनी सोध्नुहोस् र तुरुन्त "मेरो नाम के हो?" सोध्नुहोस्। मोडलले सम्झन सक्दैन किनभने प्रत्येक सन्देश स्वतन्त्र छ। यो साधारण भाषा मोडल समाकलनको मूल समस्या हो - कुनै वार्तालाप सन्दर्भ छैन।

<img src="../../../translated_images/ne/simple-chat-stateless-demo.13aeb3978eab3234.webp" alt="स्टेटलेस च्याट डेमो" width="800"/>

*AI ले अघिल्लो सन्देशबाट तपाईंको नाम सम्झँदैन*

### स्टेटफुल च्याट (दायाँ प्यानल)

अब उस्तै अनुक्रम यहाँ प्रयास गर्नुहोस्। "मेरो नाम जोन हो" र त्यसपछि "मेरो नाम के हो?" यस पटक याद गर्छ। भिन्नता MessageWindowChatMemory हो - यसले वार्तालाप इतिहास कायम राख्छ र प्रत्येक अनुरोधसँग समावेश गर्छ। उत्पादन वार्तालाप AI यसरी काम गर्छ।

<img src="../../../translated_images/ne/conversational-chat-stateful-demo.e5be9822eb23ff59.webp" alt="स्टेटफुल च्याट डेमो" width="800"/>

*AI ले पुरानो वार्तालापबाट तपाईंको नाम सम्झन्छ*

दुवै प्यानलले एउटै GPT-5.2 मोडल प्रयोग गर्छ। फरक मात्र स्मृति हो। यसले स्पष्ट पार्छ स्मृतिले तपाईंको अनुप्रयोगमा के ल्याउँछ र किन यो वास्तविक प्रयोगका लागि आवश्यक छ।
