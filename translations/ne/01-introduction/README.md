## तपाईंले के सिक्नुहुनेछ

यदि तपाईंले छिटो सुरु गर्नुभयो भने, तपाईंले कसरी प्रॉम्प्टहरू पठाउने र प्रतिक्रिया प्राप्त गर्ने देख्नुभयो। त्यो आधारभूत हो, तर वास्तविक अनुप्रयोगहरूलाई थप आवश्यक छ। यो मोड्युलले तपाइँलाई कसरी संवादात्मक AI बनाउने सिकाउँछ जसले सन्दर्भ सम्झन्छ र अवस्था कायम राख्छ - जुन एक पटकको डेमो र उत्पादन-तयार अनुप्रयोग बीचको फरक हो।

हामी यस गाइडमा Azure OpenAI को GPT-5.2 प्रयोग गर्नेछौं किनकि यसको उन्नत तर्कक्षमता विभिन्न ढाँचाहरूको व्यवहारलाई स्पष्ट बनाउँछ। जब तपाइँ मेमोरी थप्नुहुनेछ, तपाइँले स्पष्ट रूपमा भिन्नता देख्नुहुनेछ। यसले तपाइँको अनुप्रयोगमा प्रत्येक कम्पोनेंटले के ल्याउँछ भनेर बुझ्न सजिलो बनाउँछ।

तपाईंले एक अनुप्रयोग निर्माण गर्नुहुनेछ जुन दुवै ढाँचाहरू प्रदर्शन गर्दछ:

**स्टेटलेस च्याट** - हरेक अनुरोध स्वतन्त्र हुन्छ। मोडेलले अघिल्लो सन्देशहरूको कुनै मेमोरी राख्दैन। यो हो तपाईंले छिटो सुरु गर्दा प्रयोग गरेको ढाँचा।

**स्टेटफुल संवाद** - हरेक अनुरोधमा संवाद इतिहास समावेश हुन्छ। मोडेलले धेरै चरणको सन्दर्भ कायम राख्छ। यो उत्पादन अनुप्रयोगहरूमा आवश्यक हुन्छ।

भाषा मोडेलहरू स्टेटलेस हुन्छन्। प्रत्येक API कल स्वतन्त्र हुन्छ। यदि तपाईं "मेरो नाम जोन हो" पठाउनु भयो र त्यसपछि "मेरो नाम के हो?" सोध्नु भयो भने, मोडेलले थाहा पाउँदैन कि तपाईंले पहिल्यै आफ्नो परिचय दिनुभएको थियो। यो हरेक अनुरोधलाई मान्छ एकदमै पहिलो संवाद हो।

यो सरल Q&A का लागि ठीक हुन्छ तर वास्तविक अनुप्रयोगहरूको लागि बेकार छ। ग्राहक सेवा बोटहरूले तपाईंले के भनेको कुरा सम्झन आवश्यक हुन्छ। व्यक्तिगत सहायकहरूले सन्दर्भ चाहिन्छ। कुनै पनि बहु-चरणको संवादलाई सम्झना आवश्यक पर्छ।

संवादहरूमा पसे अघि, टोकनहरू बुझ्नु आवश्यक छ - पाठका आधारभूत एकाइहरू जुन भाषा मोडेलहरूले प्रक्रिया गर्छन्:

<img src="../../../translated_images/ne/token-explanation.c39760d8ec650181.webp" alt="Token Explanation" width="800"/>

*पाठ कसरी टोकनहरूमा विभाजित हुन्छ उदाहरण - "म AI लाई माया गर्छु!" चार अलग अलग प्रसंस्करण इकाइहरू हुन्छन्*

टोकनहरूले AI मोडेलहरूले पाठ मापन र प्रक्रिया गर्छन्। शब्दहरू, विराम चिह्नहरू, र यहाँसम्म कि ठाउँहरू पनि टोकन हुन सक्छन्। तपाइँको मोडेलसँग यस क्रममा प्रक्रियाकरण गर्न सक्ने टोकन सीमित छ (GPT-5.2 का लागि ४००,००० टोकन, जसमा २७२,००० इनपुट टोकन र १२८,००० आउटपुट टोकन सम्म)। टोकन बुझ्नाले तपाइँलाई संवादको लम्बाइ र लागत व्यवस्थापन गर्न मद्दत गर्दछ।

## मेमोरी कसरी काम गर्छ

च्याट मेमोरीले स्टेटलेस समस्यालाई संवाद इतिहास कायम गरेर समाधान गर्छ। मोडेललाई अनुरोध पठाउनु अघि, फ्रेमवर्कले सम्बन्धित अघिल्लो सन्देशहरू अग्रपंक्तिमा राख्दछ। जब तपाईं "मेरो नाम के हो?" सोध्नु हुन्छ, प्रणालीले वास्तविकमा सम्पूर्ण संवाद इतिहास पठाउँछ, जसले मोडेललाई थाहा हुन्छ कि तपाईंले पहिल्यै "मेरो नाम जोन हो" भनेको थियो।

LangChain4j ले मेमोरी इम्प्लिमेन्टेसनहरू प्रदान गर्दछ जुन यो स्वचालित रूपमा ह्यान्डल गर्दछ। तपाईंले कति सन्देशहरू राख्ने निर्णय गर्नुहुन्छ र फ्रेमवर्क सन्दर्भ विन्डो व्यवस्थापन गर्छ।

<img src="../../../translated_images/ne/memory-window.bbe67f597eadabb3.webp" alt="Memory Window Concept" width="800"/>

*MessageWindowChatMemory ले हालैका सन्देशहरूको स्लाइडिङ विन्डो कायम गर्छ, पुराना सन्देशहरू स्वचालित रूपमा हटाउँछ*

## यसले LangChain4j कसरी प्रयोग गर्छ

यो मोड्युलले छिटो सुरुवातलाई विस्तार गर्दछ स्प्रिंग बुटसँग एकीकृत गरेर र संवाद मेमोरी थपेर। यी भागहरू कसरी मिल्छन्:

**आश्रितता** - दुई LangChain4j पुस्तकालयहरू थप्नुहोस्:

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```

**च्याट मोडेल** - Azure OpenAI लाई स्प्रिंग बीनको रूपमा कन्फिगर गर्नुहोस् ([LangChainConfig.java](../../../01-introduction/src/main/java/com/example/langchain4j/config/LangChainConfig.java)):

```java
@Bean
public OpenAiOfficialChatModel openAiOfficialChatModel() {
    return OpenAiOfficialChatModel.builder()
            .baseUrl(azureEndpoint)
            .apiKey(azureApiKey)
            .modelName(deploymentName)
            .timeout(Duration.ofMinutes(5))
            .maxRetries(3)
            .build();
}
```

बिल्डरले क्रेडेन्शियलहरू पढ्छ जुन `azd up` द्वारा सेट गरिएका वातावरण चरहरूबाट। `baseUrl` तपाईँको Azure endpoint मा सेट गर्दा OpenAI क्लाइँट Azure OpenAI सँग काम गर्दछ।

**संवाद मेमोरी** - MessageWindowChatMemory मार्फत च्याट इतिहास ट्र्याक गर्नुहोस् ([ConversationService.java](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java)):

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("My name is John"));
memory.add(AiMessage.from("Nice to meet you, John!"));

memory.add(UserMessage.from("What's my name?"));
AiMessage aiMessage = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage);
```

`withMaxMessages(10)` प्रयोग गरेर पछिल्ला १० सन्देशहरू राख्न मेमोरी बनाउनुहोस्। प्रयोगकर्ता र AI सन्देशहरूलाई टाइप गरिएका र्यापरहरू `UserMessage.from(text)` र `AiMessage.from(text)` मार्फत थप्नुहोस्। इतिहास `memory.messages()` बाट प्राप्त गरी मोडेलमा पठाउनुहोस्। सेवा प्रत्येक संवाद ID का लागि अलग मेमोरी उदाहरणहरू संग्रह गर्छ, जसले एकै समयमा धेरै प्रयोगकर्ताहरूलाई च्याट गर्न अनुमति दिन्छ।

> **🤖 [GitHub Copilot](https://github.com/features/copilot) संग प्रयास गर्नुहोस्:** [`ConversationService.java`](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java) खोल्नुहोस् र सोध्नुहोस्:
> - "जब विन्डो भरिन्छ, MessageWindowChatMemoryले कुन सन्देशहरू फाल्छ भनेर कसरी निर्णय गर्छ?"
> - "के म इन-मेमोरीको सट्टा डेटाबेस प्रयोग गरेर अनुकूलित मेमोरी भण्डारण लागू गर्न सक्छु?"
> - "पुरानो संवाद इतिहास संक्षेप गर्न मैले कसरी सर्गमिकीकरण थप्न सक्छु?"

स्टेटलेस च्याट अन्तबिन्दुले मेमोरी पूर्ण रूपमा छोड्छ - जस्तै छिटो सुरुवातमा `chatModel.chat(prompt)` मात्र। स्टेटफुल अन्तबिन्दुले मेमोरीमा सन्देशहरू थप्छ, इतिहास फिर्ता ल्याउँछ, र प्रत्येक अनुरोधसँग त्यो सन्दर्भ समावेश गर्छ। उही मोडेल कन्फिगरेसन, फरक ढाँचाहरू।

## Azure OpenAI पूर्वाधार तैनाथ गर्नुहोस्

**Bash:**
```bash
cd 01-introduction
azd up  # सदस्यता र स्थान चयन गर्नुहोस् (eastus2 सिफारिस गरिएको)
```

**PowerShell:**
```powershell
cd 01-introduction
azd up  # सदस्यता र स्थान छान्नुहोस् (eastus2 सिफारिस गरिएको)
```

> **टिप्पणी:** यदि तपाईंलाई समय सीमा त्रुटि (`RequestConflict: Cannot modify resource ... provisioning state is not terminal`) आउँछ भने, सरलत: फेरि `azd up` चलाउनुहोस्। Azure स्रोतहरू पृष्ठभूमिमा अझै तैनाथ हुँदै हुन सक्छन्, र पुन: प्रयास गर्दा स्रोतहरू टर्मिनल अवस्थामा पुग्दा तैनाथीकरण पूरा हुन्छ।

यसले तल गर्नेछ:
1. GPT-5.2 र text-embedding-3-small मोडेलहरू सहित Azure OpenAI स्रोत तैनाथ गर्ने
2. प्रोजेक्ट रूटमा स्वतः `.env` फाइल सिर्जना गर्ने क्रेडेन्शियलहरूसहित
3. आवश्यक सम्पूर्ण वातावरण चरहरू सेटअप गर्ने

**तैनाथीकरणमा समस्या छ?** विस्तृत समस्या समाधानका लागि [पूर्वाधार README](infra/README.md) हेर्नुहोस् जसमा सबडोमेन नाम द्वन्द्वहरू, Azure पोर्टलमा म्यानुअल तैनाथीकरण चरणहरू, र मोडल कन्फिगरेसन सल्लाहहरू समावेश छन्।

**तैनाथीकरण सफल भयो भनेर पुष्टि गर्नुहोस्:**

**Bash:**
```bash
cat ../.env  # AZURE_OPENAI_ENDPOINT, API_KEY, आदि देखाउनुपर्छ।
```

**PowerShell:**
```powershell
Get-Content ..\.env  # AZURE_OPENAI_ENDPOINT, API_KEY, आदि देखाउनुपर्छ।
```

> **टिप्पणी:** `azd up` आदेशले स्वत: `.env` फाइल सिर्जना गर्दछ। यदि पछि यसलाई अद्यावधिक गर्न आवश्यक भएमा, तपाईं `.env` फाइललाई म्यानुअली सम्पादन गर्न सक्नुहुन्छ वा निम्न आदेश चलाएर पुन: सिर्जना गर्न सक्नुहुन्छ:
>
> **Bash:**
> ```bash
> cd ..
> bash .azd-env.sh
> ```
>
> **PowerShell:**
> ```powershell
> cd ..
> .\.azd-env.ps1
> ```

**विकल्प २: shell स्क्रिप्टहरू प्रयोग गर्दै**

सबै वेब एप्लिकेशनहरू (मोड्युल ०१-०४) सुरु गर्नुहोस्:

**Bash:**
```bash
cd ..  # रुट डिरेक्टरीबाट
./start-all.sh
```

**PowerShell:**
```powershell
cd ..  # मूल निर्देशिका देखि
.\start-all.ps1
```

वा केवल यो मोड्युल सुरु गर्नुहोस्:

**Bash:**
```bash
cd 01-introduction
./start.sh
```

**PowerShell:**
```powershell
cd 01-introduction
.\start.ps1
```

दुवै स्क्रिप्टहरूले स्वतः रूट `.env` फाइलबाट वातावरण चरहरू लोड गर्छन् र यदि JAR फाइलहरू छैनन् भने निर्माण गर्छन्।

> **टिप्पणी:** यदि तपाईं सुरू गर्नु अघि सबै मोड्युलहरू म्यानुअली निर्माण गर्न चाहनुहुन्छ भने:
>
> **Bash:**
> ```bash
> cd ..  # Go to root directory
> mvn clean package -DskipTests
> ```
>
> **PowerShell:**
> ```powershell
> cd ..  # Go to root directory
> mvn clean package -DskipTests
> ```

तपाईंको ब्राउजरमा http://localhost:8080 खोल्नुहोस्।

**रोक्न:**

**Bash:**
```bash
./stop.sh  # केवल यो मोड्युल
# वा
cd .. && ./stop-all.sh  # सबै मोड्युलहरू
```

**PowerShell:**
```powershell
.\stop.ps1  # यो मोड्युल मात्रै
# वा
cd ..; .\stop-all.ps1  # सबै मोड्युलहरू
```

एप्लिकेशनले दुई च्याट कार्यान्वयनहरू साइड-बाइ-साइड वेब इन्टरफेसमा उपलब्ध गराउँछ।

<img src="../../../translated_images/ne/home-screen.121a03206ab910c0.webp" alt="Application Home Screen" width="800"/>

*ड्यासबोर्डले दुवै सिम्पल च्याट (स्टेटलेस) र संवादात्मक च्याट (स्टेटफुल) विकल्पहरू देखाउँछ*

### स्टेटलेस च्याट (बायाँ प्यानल)

पहिले यसलाई प्रयास गर्नुहोस्। "मेरो नाम जोन हो" भन्नुहोस् र तुरुन्तै "मेरो नाम के हो?" सोध्नुहोस्। मोडेलले सम्झँदैन किनकि प्रत्येक सन्देश स्वतन्त्र छ। यसले आधारभूत भाषा मोडेल एकीकरणको मुख्य समस्या देखाउँछ - कुनै संवाद सन्दर्भ छैन।

<img src="../../../translated_images/ne/simple-chat-stateless-demo.13aeb3978eab3234.webp" alt="Stateless Chat Demo" width="800"/>

*AI ले अघिल्लो सन्देशबाट तपाईंको नाम सम्झँदैन*

### स्टेटफुल च्याट (दायाँ प्यानल)

अब यहाँ सोही क्रम प्रयास गर्नुहोस्। "मेरो नाम जोन हो" र त्यसपछि "मेरो नाम के हो?" यो पटक मोडेलले सम्झन्छ। भिन्नता MessageWindowChatMemory हो - यसले संवाद इतिहास कायम गर्छ र प्रत्येक अनुरोधसँग सन्दर्भ समावेश गर्छ। यसरी उत्पादन संवादात्मक AI काम गर्छ।

<img src="../../../translated_images/ne/conversational-chat-stateful-demo.e5be9822eb23ff59.webp" alt="Stateful Chat Demo" width="800"/>

*AI ले पहिलेको संवादबाट तपाईंको नाम सम्झन्छ*

दुबै प्यानलहरूले उही GPT-5.2 मोडेल प्रयोग गर्छन्। एक मात्र भिन्नता मेमोरी हो। यसले स्पष्ट पार्दछ कि मेमोरीले तपाइँको अनुप्रयोगमा के ल्याउँछ र किन यो वास्तविक प्रयोगका लागि आवश्यक छ।
