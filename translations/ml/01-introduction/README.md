## നിങ്ങൾക്കു പഠിക്കാനുള്ളത്

നിങ്ങൾ ക്വിക് സ്റ്റാർട്ട് പൂർത്തിയാക്കിയെങ്കിൽ, നിങ്ങൾ പ്ര്രൊംപ്റ്റുകൾ അയച്ച് പ്രതികരണങ്ങൾ എങ്ങനെ കിട്ടിയെന്നതും കാണാം. അത് അടിസ്ഥാനമാണ്, പക്ഷേ യഥാർത്ഥ അപ്ലിക്കേഷനുകൾക്ക് കൂടുതൽ വേണ്ടിവരും. ഈ മൊഡ്യൂൾ കോൺടെക്സ്റ്റ് ഓർമ്മിക്കുന്നയും സ്റ്റേറ്റ് നിലനിർത്തുന്നതുമായ സംഭാഷണ AI നിർമ്മാണം എങ്ങനെ ചെയ്യാമെന്ന് പഠിപ്പിക്കുന്നു — ഒറ്റ തവണ ഡെമോയും പ്രൊഡക്ഷൻ തയാറായ അപ്ലിക്കേഷനും തമ്മിലുള്ള വ്യത്യാസ്‌നം.

ഈ ഗൈഡിൽ മുഴുവൻ Azure OpenAI യുടെ GPT-5.2 ഉപയോഗിക്കും, കാരണം അതിന്റെ ആധുനിക നിരീക്ഷണ ശേഷികൾ വിവിധ മാതൃകകളുടെ പെരുമാറ്റങ്ങൾ വ്യക്തമാക്കുന്നു. നിങ്ങൾ മെമ്മറി ചേർക്കുമ്പോൾ, വ്യത്യാസം വ്യക്തമായി കാണാം. ഇത് ഓരോ ഘടകവും നിങ്ങളുടെ അപ്ലിക്കേഷനിൽ എന്തുകൊണ്ട് പ്രധാനമാണെന്ന് മനസ്സിലാക്കാൻ സഹായിക്കുന്നു.

നിങ്ങൾ താഴെ പറയുന്ന രണ്ടു മാതൃകകളും പ്രദർശിപ്പിക്കുന്ന ഒരു അപ്ലിക്കേഷൻ നിർമ്മിക്കും:

**സ്റ്റേറ്റ്‌ലസ് ചാറ്റ്** - ഓരോ അഭ്യർത്ഥനയുമും സ്വതന്ത്രമാണ്. മാതൃക മുൻപ് അയച്ച സന്ദർശനങ്ങളെ ഓർമ്മിക്കുന്നില്ല. ഇത് നിങ്ങൾ ക്വിക് സ്റ്റാർട്ടിൽ ഉപയോഗിച്ചത് തന്നെയാണ്.

**സ്റ്റേറ്റ്ഫുൾ സംഭാഷണം** - ഓരോ അഭ്യർത്ഥനയിലും സംഭാഷണ ചരിത്രം ഉൾപ്പെടുത്തുന്നു. മാതൃക പല തവണകളിലും കോൺടെക്സ്റ്റ് നിലനിർത്തുന്നു. പ്രൊഡക്ഷൻ അപ്ലിക്കേഷനുകൾക്ക് ഇത് ആവശ്യമാണ്.

## ആവശ്യമായ വ്യവസ്ഥകള്‍

- Azure സബ്‌സ്‌ക്രിപ്ഷൻ Azure OpenAI ആക്സസ് ഉള്ളത്
- Java 21, Maven 3.9+
- Azure CLI (https://learn.microsoft.com/en-us/cli/azure/install-azure-cli)
- Azure Developer CLI (azd) (https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/install-azd)

> **കുറിപ്പ്:** Java, Maven, Azure CLI, Azure Developer CLI (azd) എന്നിവ പ്രൊവൈഡുചെയ്ത devcontainer-ൽ മുൻകൂട്ടി ഇൻസ്റ്റാൾ ചെയ്തിട്ടുണ്ട്.

> **കുറിപ്പ്:** ഈ മൊഡ്യൂൾ Azure OpenAI യിൽ GPT-5.2 ഉപയോഗിക്കുന്നു. `azd up` വഴി ഡിപ്ലോയ്മെന്റ് സ്വയം ക്രമീകരിച്ചിട്ടുണ്ടാകുന്നതിനാൽ കോഡിൽ മോഡൽ പേര് മാറ്റരുത്.

## പ്രധാന പ്രശ്‌നം മനസ്സിലാക്കല്‍

ഭാഷാ മോഡലുകൾ സ്റ്റേറ്റ്‌ലസ് ആണ്. ഓരോ API കോളും സ്വന്തമാണ്. നിങ്ങൾ "എന്റെ പേര് ജോൺ ആണ്" എന്നു അയച്ച ശേഷം "എന്റെ പേര് എന്താണ്?" എന്ന് ചോദിച്ചാൽ, മാതൃകയ്ക്ക് നിങ്ങൾ പുറത്തിട്ട തൽസമയ വിവരം അറിയില്ല. എല്ലാ അഭ്യർദ്ധനകളും ആദ്യ സംഭാഷണം എന്നു കരുതുന്നു.

തികച്ചും ലളിതമായ Q&A-കൾക്ക് ഇത് പ്രയോജനം, പക്ഷേ യഥാർത്ഥ അപ്ലിക്കേഷനുകളിൽ അത്ഭുതമില്ല. ഉപഭോക്തൃ സേവന ബോട്ടുകൾ നിങ്ങൾ പറഞ്ഞു തന്നതു ഓർമ്മിക്കണം. വ്യക്തിഗത അസിസ്റ്റന്റുകൾക്ക് കോൺടെക്സ്റ്റ് വേണം. മൾട്ടി-ടേൺ സംഭാഷണം ഒരു മെമ്മറി ആവശ്യമാണ്.

<img src="../../../translated_images/ml/stateless-vs-stateful.cc4a4765e649c41a.webp" alt="സ്റ്റേറ്റ്‌ലസ് വാഴ് സ്റ്റേറ്റ്ഫുൾ സംഭാഷണങ്ങൾ" width="800"/>

*സ്റ്റേറ്റ്‌ലസ് (സ്വതന്ത്ര കോളുകൾ) വാഴ് സ്റ്റേറ്റ്ഫുൾ (കോൺടെക്സ്റ്റ് അറിവുള്ള) സംഭാഷണങ്ങളുടെ തമ്മിലുള്ള വ്യത്യാസം*

## ടോക്കണുകൾ മനസ്സിലാക്കല്‍

സംഭാഷണത്തിൽ പോകുന്നതിന് മുമ്പ്, ടോക്കണുകൾ — ഭാഷാ മോഡലുകൾ പ്രോസസ് ചെയ്യുന്നതിനുള്ള ബേസിക് യൂണിറ്റുകൾ — മനസ്സിലാക്കേണ്ടതാണ്:

<img src="../../../translated_images/ml/token-explanation.c39760d8ec650181.webp" alt="ടോക്കൺ വിശദീകരണം" width="800"/>

*"I love AI!" എന്ന വാചകം നാലു പൊരുത്തപ്പെടുന്ന പ്രോസസ്സിംഗ് യൂണിറ്റുകളായി ചേരുന്നു എന്ന ഉദാഹരണം*

ടോക്കണുകൾ ആണ് AI മോഡലുകൾ വാചകങ്ങൾ അളക്കുകയും പ്രോസസ്സ് ചെയ്യുകയും ചെയ്യുന്നത്. വാക്കുകൾ, പുണ്ക്തി ചിഹ്നങ്ങൾ, സന്ധികളും ടോക്കണുകൾ ആകാം. നിങ്ങളുടെ മോഡലിന് ഒരിക്കലും ഒരേസമയം എത്ര ടോക്കണുകൾ പ്രോസസ് ചെയ്യാമെന്നു നിശ്ചിത പരിധി ഉണ്ട് (GPT-5.2 ന് 400,000, 272,000 ഇൻപുട്ട് ടോക്കണുകളും 128,000 ഔട്ട്‌പുട്ട് ടോക്കണുകളും). ടോക്കണുകൾ മനസ്സിലാക്കുന്നത് സംഭാഷണത്തിന്റെ ദൈർഘ്യവും ചെലവും നിയന്ത്രിക്കാൻ സഹായിക്കുന്നു.

## മെമ്മറി എങ്ങനെ പ്രവര്ത്തിക്കുന്നു

ചാറ്റ് മെമ്മറി സ്റ്റേറ്റ്‌ലസ് പ്രശ്‌നം പരിഹരിക്കുന്നു സംഭാഷണ ചരിത്രം നിലനിർത്തി. നിങ്ങളുടെ അഭ്യർത്ഥന മാതൃകക്ക് അയക്കുന്നതിന് മുമ്പ്, ഫ്രെയിംവർക്ക് ബന്ധപ്പെട്ട മുമ്പ് അയച്ച സന്ദേശങ്ങൾ ചേർക്കുന്നു. "എന്റെ പേര് എന്താണെന്നു" ചോദിക്കുമ്പോൾ, സിസ്റ്റം യഥാർത്ഥത്തിൽ മുഴുവൻ സംഭാഷണ ചരിത്രം അയക്കുന്നു, അതുകൊണ്ട് മാതൃകയെ നിങ്ങൾ മുമ്പ് "എന്റെ പേര് ജോൺ ആണ്" എന്നു പറഞ്ഞതായി കാണാൻ കഴിയുന്നു.

LangChain4j ഓട്ടോമാറ്റിക് ആയി ഇത് കൈകാര്യം ചെയ്യുന്ന മെമ്മറി പുതുക്കലുകൾ നൽകുന്നു. നിങ്ങൾ എത്ര സന്ദേശങ്ങൾ നിലനിർത്തണമെന്ന് തിരഞ്ഞെടുക്കും, ഫ്രെയിംവർക്ക് കോൺടെക്സ്റ്റ് വിൻഡോ കൈകാര്യം ചെയ്യും.

<img src="../../../translated_images/ml/memory-window.bbe67f597eadabb3.webp" alt="മെമ്മറി വിൻഡോ ആശയം" width="800"/>

*MessageWindowChatMemory ഏറ്റവും പുതിയ സന്ദേശങ്ങളുടെ സ്ലൈഡിംഗ് വിൻഡോ നിലനിർത്തുന്നു, പഴയവ സ്വയം ഒഴിവാക്കുന്നു*

## ഇത് LangChain4j എങ്ങനെ ഉപയോഗിക്കുന്നു

ഈ മൊഡ്യൂൾ ക്വിക് സ്റ്റാർട്ട് വിപുലമാക്കുന്നു സ്‌പ്രിംഗ് ബൂട്ട് ഉൾപ്പെടുത്തി സംഭാഷണ മെമ്മറി ചേർത്ത്. ഘടകങ്ങൾ ഇതുപോലെ ചേർന്നിരിക്കുന്നു:

**ഡിപ്പെൻഡൻസികൾ** - രണ്ട് LangChain4j ലൈബ്രറികൾ ചേർക്കുക:

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```

**ചാറ്റ് മോഡൽ** - Azure OpenAI സ്പ്രിംഗ് ബീൻ ആയി കോൺഫിഗർ ചെയ്യുക ([LangChainConfig.java](../../../01-introduction/src/main/java/com/example/langchain4j/config/LangChainConfig.java)):

```java
@Bean
public OpenAiOfficialChatModel openAiOfficialChatModel() {
    return OpenAiOfficialChatModel.builder()
            .baseUrl(azureEndpoint)
            .apiKey(azureApiKey)
            .modelName(deploymentName)
            .timeout(Duration.ofMinutes(5))
            .maxRetries(3)
            .build();
}
```

`azd up` വഴി സജ്ജമാക്കിയ പരിസ്ഥിതി വ്യത്യാസങ്ങളായ സന്ദർഭങ്ങളിൽ നിന്നും ക്രെഡൻഷ്യലുകൾ വായിക്കുന്നതാണു ബിൽഡർ. നിങ്ങളുടെ Azure endpoint-ന് `baseUrl` സജ്ജീകരിക്കുന്നത് Azure OpenAI കസ്റ്റമൈസേഷൻ ചെയ്യാൻ സഹായിക്കുന്നു.

**സംഭാഷണ മെമ്മറി** - MessageWindowChatMemory ഉപയോഗിച്ചു സംഭാഷണ ചരിത്രം പിന്തുടരുക ([ConversationService.java](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java)):

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("My name is John"));
memory.add(AiMessage.from("Nice to meet you, John!"));

memory.add(UserMessage.from("What's my name?"));
AiMessage aiMessage = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage);
```

`withMaxMessages(10)` ഉപയോഗിച്ച് അവസാന 10 സന്ദേശങ്ങൾ മാത്രം സൂക്ഷിക്കാൻ മെമ്മറി സൃഷ്‌ടിക്കുക. Typed wrapperഉപയോഗിച്ച് user & AI സന്ദേശങ്ങൾ ചേർക്കുക: `UserMessage.from(text)` & `AiMessage.from(text)`. `memory.messages()` കൊണ്ട് ചരിത്രം ലഭിച്ച് മാതൃകക്ക് അയയ്ക്കുക. ഓരോ സംഭാഷണ ഐഡിക്കും പ്രത്യേകം മെമ്മറി സംഭരിച്ചു, വിവിധ ഉപയോക്താക്കൾക്ക് സമാന്തരമായി സംവദിക്കാനാകും.

> **🤖 [GitHub Copilot](https://github.com/features/copilot) Chat ഉപയോഗിച്ച് പരീക്ഷിക്കുക:** [`ConversationService.java`](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java) തുറക്കുക, ചോദിക്കാം:
> - MessageWindowChatMemory വിൻഡോ പൂർണ്ണമായപ്പോൾ സന്ദേശങ്ങൾ എങ്ങനെ ഒഴിവാക്കും?
> - ഇൻ-മെമ്മറിയിൽ അല്ലാതെയുള്ള ഡാറ്റാബേസ് ഉപയോഗിച്ച് കസ്റ്റം മെമ്മറി സംഭരണം എങ്ങനെ നടപ്പിലാക്കാം?
> - പഴയ സംഭാഷണ ചരിത്രം ചുരുക്കാൻസംഗ്രഹിക്കൽ എങ്ങനെ ചേർക്കാം?

സ്റ്റേറ്റ്‌ലസ്സ് ചാറ്റ് എന്റ്‌പോയിന്റ് മെമ്മറി ഉപയോഗിക്കുന്നില്ല - ക്വിക് സ്റ്റാർട്ടിലുപയോഗിച്ച പോലെ `chatModel.chat(prompt)` മാത്രം. സ്റ്റേറ്റ്ഫുൾ എന്റ്‌പോയിന്റ് സന്ദേശങ്ങൾ മെമ്മറിയിൽ ചേർക്കുന്നു, ചരിത്രം തിരിച്ച്, നിന്നു കോണ്ടക്സ്റ്റും ഉൾപ്പെടുത്തുന്നു. മോഡൽ കോൺഫിഗറേഷൻ സമാനം, മാതൃക വ്യത്യസ്‌തം.

ഇതുവഴി:
1. GPT-5.2 & text-embedding-3-small മോഡലുകളോടെ Azure OpenAI റിസോഴ്‌സ് വിന്യസിക്കുന്നു
2. പ്രോജക്റ്റ് റൂട്ട് ഡയറക്ടറിയിൽ ക്രെഡൻഷ്യൽസോടെ `.env` ഫയൽ സ്വയം സൃഷ്‌ടിക്കും
3. ആവശ്യമായ പരിസ്ഥിതി വ്യത്യാസങ്ങൾ സജ്ജമാക്കും

**ഡിപ്ലോയ്മെന്റ് വിജയിച്ചെന്ന് പരിശോധിക്കുക:**

### സ്റ്റേറ്റ്‌ലസ് ചാറ്റ് (ഇടത് പാനൽ)

### സ്റ്റേറ്റ്‌ഫുൾ ചാറ്റ് (വലത് പാനൽ)

സമാനമായ പരമ്പര ഇത് ഇവിടെ പരീക്ഷിക്കുക. "എന്റെ పేరు ജോൺ ആണ്" എന്നു ചോദിച്ച് ശേഷം "എന്റെ പേര് എന്താണു?" ചോദിച്ചാൽ ഓർമ്മിക്കുന്നു. വ്യത്യാസം MessageWindowChatMemory ആണ് - ഇത് സംഭാഷണ ചരിത്രം നിലനിർത്തുന്നു, ഓരോ അഭ്യർ‍ത്ഥനയോടും അത് ഉൾപ്പെടുന്നു. ഇതാണ് പ്രൊഡക്ഷൻ സംഭാഷണ AI പ്രവര്‍ത്തിക്കുന്നത്.
