## നിങ്ങൾ പഠിക്കാനുള്ളത്

ക്വിക്ക് സ്റ്റാർട്ട് പൂർത്തിയാക്കിയാൽ, നിങ്ങൾ പ്രോംപ്റ്റുകൾ അയച്ച് പ്രതികരണങ്ങൾ സ്വീകരിക്കുന്നതെങ്ങനെ എന്ന് കണ്ടു. അത് അടിസ്ഥാനമാണ്, എന്നാൽ യാഥാർത്ഥ്യപ്രയോഗങ്ങൾക്ക് കൂടുതൽ ആവശ്യമാണ്. ഈ മോഡ്യൂൾ, കൺടെക്സ്റ്റും സ്റ്റേറ്റും ഓർമിക്കുന്ന സംഭാഷണ AI എങ്ങനെ നിർമ്മിക്കാമെന്ന് പഠിപ്പിക്കുന്നു - ഒറ്റ പ്രദർശനത്തിനും ഉത്പാദന-നുള്ള പ്രയോഗത്തിനും ഉള്ള വ്യത്യാസം.

ഈ ഗൈഡ് മുഴുവൻ Azure OpenAI-യുടെ GPT-5.2 ഉപയോഗിക്കും, കാരണം അതിന്റെ പുരോഗമിത തർക്കശേഷികൾ വിവിധ മാതൃകകളുടെ പെരുമാറ്റം ഇത് വ്യക്തമാക്കുന്നു. മെമ്മറി ചേർക്കുമ്പോൾ നിങ്ങൾക്ക് വ്യത്യാസം വ്യക്തമായി കാണാം. ഇത് നിങ്ങളുടെ പ്രയോഗത്തിലെ ഓരോ ഘടകവും എന്താണ് നൽകുന്നത് എന്ന് എളുപ്പത്തിൽ മനസിലാക്കാൻ സഹായിക്കും.

നിങ്ങൾ രണ്ടു മാതൃകകളും പ്രദർശിപ്പിക്കുന്ന ഒരു പ്രയോഗം നിർമ്മിക്കും:

**സ്റ്റേറ്റ്ലസ് ചാറ്റ്** - ഓരോ അഭ്യർത്ഥനയും സ്വതന്ത്രമാണ്. മോഡലിന് മുന്‍ സന്ദേശങ്ങളുടെ മെമ്മറി ഇല്ല. ക്വിക്ക് സ്റ്റാർട്ടിൽ നിങ്ങൾ ഉപയോഗിച്ച മാതൃക ഇതാണ്.

**സ്റ്റേറ്റ്‌ഫുൾ സംഭാഷണം** - ഓരോ അഭ്യർത്ഥനയും സംഭാഷണ ചരിത്രം ഉൾക്കൊള്ളുന്നു. മോഡൽ പല തവണയും ഉള്ള സംഭാഷണത്തിന്റെ കൺടെക്സ്റ്റ് നിലനിർത്തുന്നു. ഉത്പാദന പ്രയോഗങ്ങൾക്ക് ഇതാണ് ആവശ്യം.

## മുൻഅവശ്യങ്ങൾ

- Azure സബ്സ്ക്രിപ്ഷൻ Azure OpenAI ആക്‌സസ് സഹിതം
- Java 21, Maven 3.9+ 
- Azure CLI (https://learn.microsoft.com/en-us/cli/azure/install-azure-cli)
- Azure Developer CLI (azd) (https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/install-azd)

> **ഗമിക്കുക:** Java, Maven, Azure CLI, Azure Developer CLI (azd) നൽകിയ devcontainer-ൽ സെറ്റ് ചെയ്തിട്ടുണ്ട്.

> **ഗമിക്കുക:** ഈ മോഡ്യൂൾ Azure OpenAI-യിൽ GPT-5.2 ഉപയോഗിക്കുന്നു. `azd up` വഴി സ്വയം കോൺഫിഗർ ചെയ്യുന്ന ഡിപ്പ്ലോയ്മെന്റ് ആണ് - കോഡിൽ മോഡൽ നാമം മാറ്റരുത്.

## മൂലപ്രശ്നം മനസിലാക്കൽ

ഭാഷാ മോഡലുകൾ സ്റ്റേറ്റ്ലസാണ്. ഓരോ API കോളും സ്വതന്ത്രമാണ്. നിങ്ങൾ "എന്റെ പേര് ജോൺ ആണ്" എന്ന് അയച്ച്, "എന്റെ പേര് എന്താണ്?" എന്ന് ചോദിയ്ക്കുമ്പോൾ, മോഡലിന് നിങ്ങളെ പരിശീലിപ്പിച്ചതായി അറിവില്ല. അത് ഓരോ അഭ്യർത്ഥനയും ആദ്യത്തേ സംഭാഷണം പോലെ കരുതുന്നു.

സിമ്പിൾ Q&A-യ്ക്ക് ഇത് മതി, എന്നാൽ യഥാർത്ഥ പ്രയോഗങ്ങൾക്ക് ഇത് അപ്രയോജനം. കസ്റ്റമർ സർവീസ് ബോട്ടുകൾ നിങ്ങൾ പറഞ്ഞത MCST ആയിരിക്കും ഓർക്കണം. വ്യക്തിഗത സഹായികൾക്കു context ആവശ്യമുണ്ട്. ഏത് മൾട്ടി-ടേൺ സംഭാഷണത്തിനും മെമ്മറി വേണം.

<img src="../../../translated_images/ml/stateless-vs-stateful.cc4a4765e649c41a.webp" alt="Stateless vs Stateful Conversations" width="800"/>

*സ്ഥിരം (സ്വതന്ത്ര കോൾസ്) - സ്റ്റേറ്റ്‌ഫുൾ (context-aware) സംഭാഷണങ്ങൾ തമ്മിലുള്ള വ്യത്യാസം*

## ടോക്കണുകൾ മനസിലാക്കൽ

സംഭാഷണങ്ങളിൽ പോകുന്നതിനു മുൻപ്, ടോക്കണുകൾ എന്തൊക്കെയാണ് എന്നു മനസിലാക്കുന്നത് പ്രധാനമാണ് - ഭാഷാ മോഡലുകൾ പ്രോസസ്സ് ചെയ്യുന്ന അടിസ്ഥാന എഴുത്ത് ഘടകങ്ങൾ:

<img src="../../../translated_images/ml/token-explanation.c39760d8ec650181.webp" alt="Token Explanation" width="800"/>

*ടെക്സ്റ്റ് എങ്ങനെ ടോക്കണുകളായി വിഭജിക്കപ്പെടുന്നു എന്ന ഉദാഹരണം - "I love AI!" നാലു വ്യത്യസ്ത പ്രോസസിംഗ് യൂണിറ്റുകൾ ആകുന്നു*

ടോക്കണുകൾ AI മോഡലുകൾക്ക് ടെക്സ്റ്റ് അളക്കാനും പ്രോസസ്സ് ചെയ്യാനുമുള്ള മാർഗമാണ്. വാക്കുകൾ, പുണ്ക്തിയും, ഇടവേളകളും ടോക്കണുകൾ ആകാം. നിങ്ങളുടെ മോഡലിന് ഒരേസമയം എത്ര ടോക്കണുകൾ പ്രോസസ് ചെയ്യാമെന്നതിന് ഒരു പരിധിയുണ്ട് (GPT-5.2-ക്ക് 400,000, അതിൽക്ക് 272,000 ഇൻപുട്ട് ടോക്കണുകളും 128,000 ഔട്പുട്ട് ടോക്കണുകളും). ടോക്കണുകൾ മനസ്സിലാക്കുന്നത് സംഭാഷണ നീളംയും ചെലവും നിയന്ത്രിക്കാൻ സഹായിക്കുന്നു.

## മെമ്മറി എങ്ങനെ പ്രവർത്തിക്കുന്നു

ചാറ്റ് മെമ്മറി സ്റ്റേറ്റ്ലസ് പ്രശ്നം പരിഹരിക്കുന്നു സംഭാഷണ ചരിത്രം നിലനിർത്തുമ്പോൾ. മോഡലിൽ അഭ്യർത്ഥന അയയ്‌ക്കുന്നതിനു മുൻപ്, ഫ്രെയിംവർക്ക് ബന്ധപ്പെട്ട മുൻപ് സന്ദേശങ്ങൾ ചേർക്കുന്നു. നിങ്ങൾ "എന്റെ പേര് എന്താണ്?" എന്ന് ചോദിക്കുമ്പോൾ, സിസ്റ്റം മുഴുവൻ ചരിത്രവും അയച്ചു, മോഡൽ മുമ്പ് "എന്റെ പേര് ജോൺ ആണ്" എന്നു പറഞ്ഞത് കാണാൻ കഴിയും.

LangChain4j മെമ്മറി ഇംപ്ലിമെന്റേഷനുകൾ നൽകുന്നു, ഇത് സ്വയം കൈകാര്യം ചെയ്യുന്നു. നിങ്ങൾ എത്ര സന്ദേശങ്ങൾ സൂക്ഷിക്കണമെന്ന് തിരഞ്ഞെടുക്കുന്നു, ഫ്രെയിംവർക്ക് കൺടെക്സ്റ്റ് വിൻഡോ നിയന്ത്രിക്കുന്നു.

<img src="../../../translated_images/ml/memory-window.bbe67f597eadabb3.webp" alt="Memory Window Concept" width="800"/>

*MessageWindowChatMemory പുതിയ സന്ദേശങ്ങളുടെ സ്ലൈഡിംഗ്ഗ് വിൻഡോ നിലനിർത്തുന്നു, പഴയ സന്ദേശങ്ങൾ സ്വയം ഒഴിവാക്കുന്നു*

## LangChain4j എങ്ങനെ ഉപയോഗിക്കുന്നു

ക്വിക്ക് സ്റ്റാർട്ട് വിപുലീകരിച്ച് Spring Boot ഇന്റഗ്രേറ്റ് ചെയ്ത് സംഭാഷണ മെമ്മറി ചേർക്കുന്നു. ഘടകങ്ങൾ ഇങ്ങനെ ചേർത്ത് തരുമെന്ന് കാണാം:

**അനുബന്ധങ്ങൾ** - രണ്ട് LangChain4j ലൈബ്രറികൾ ചേർക്കുക:

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```

**ചാറ്റ് മോഡൽ** - Azure OpenAI എങ്ങനെ Spring ബീൻ ആയി കോൺഫിഗർ ചെയ്യാം ([LangChainConfig.java](../../../01-introduction/src/main/java/com/example/langchain4j/config/LangChainConfig.java)):

```java
@Bean
public OpenAiOfficialChatModel openAiOfficialChatModel() {
    return OpenAiOfficialChatModel.builder()
            .baseUrl(azureEndpoint)
            .apiKey(azureApiKey)
            .modelName(deploymentName)
            .timeout(Duration.ofMinutes(5))
            .maxRetries(3)
            .build();
}
```

ബിൽഡർ environment variables-ലെ ക്രെഡൻഷ്യൽസ് വായിക്കുന്നു, അത് `azd up` സജ്ജമാക്കുന്നു. `baseUrl` നിങ്ങളുടെ Azure എൻഡ്പോയിന്റ് ആക്കി സജ്ജീകരിക്കുന്നത് OpenAI ക്ലയന്റിന് Azure OpenAI-യിൽ പ്രവർത്തിക്കാൻ സഹായിക്കുന്നു.

**സംഭാഷണ മെമ്മറി** - MessageWindowChatMemory ഉപയോഗിച്ച് ചാറ്റ് ചരിത്രം നിരീക്ഷിക്കുക ([ConversationService.java](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java)):

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("My name is John"));
memory.add(AiMessage.from("Nice to meet you, John!"));

memory.add(UserMessage.from("What's my name?"));
AiMessage aiMessage = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage);
```

`withMaxMessages(10)` ഉപയോഗിച്ച് അവസാന 10 സന്ദേശങ്ങൾ സൂക്ഷിക്കും. Typed wrapper-കൾ: `UserMessage.from(text)` , `AiMessage.from(text)` ഉപയോഗിച്ച് യൂസർ, AI സന്ദേശങ്ങൾ ചേർക്കുക. ചരിത്രം `memory.messages()` വഴി ആക്സസ് ചെയ്ത് മോഡലിലേക്ക് അയയ്ക്കുന്നു. സർവീസ് ഓരോ സംഭാഷണ IDക്ക് പ്രത്യേകം മെമ്മറി സംരക്ഷിക്കുന്നു, ഇത് ഒരേസമയം പല യൂസർമാർക്കും ചാറ്റ് ചെയ്യാൻ ഇടവരുത്തുന്നു.

> **🤖 [GitHub Copilot](https://github.com/features/copilot) ചാറ്റോടെ പരീക്ഷിക്കുക:** [`ConversationService.java`](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java) തുറന്ന് ചോദിക്കുക:  
> - "MessageWindowChatMemory വിൻഡോ പൂരിപ്പിക്കുമ്പോൾ ഏതെന്ത് സന്ദേശങ്ങൾ ഒഴിവാക്കണമെന്ന് എങ്ങനെ തീരുമാനിക്കുന്നു?"  
> - "In-memory പകരം ഡാറ്റാബേസ് ഉപയോഗിച്ച് കസ്റ്റം മെമ്മറി സ്റ്റോറേജ് എങ്ങനെ എങ്ങനെ നടപ്പിലാക്കാം?"  
> - "പഴയ സംഭാഷണ ചരിത്രം സംഗ്രഹിക്കാൻ കൂട്ടിച്ചേർക്കൽ എങ്ങനെ ചേർക്കാം?"

സ്റ്റേറ്റ്ലസ് ചാറ്റ് എൻഡ്‌പോയിന്റ് മെമ്മറി മുഴുവൻ ഒഴിവാക്കുന്നു - ക്വിക് സ്റ്റാർട്ടിലെപ്പോലെ `chatModel.chat(prompt)` മാത്രം. സ്റ്റേറ്റ്‌ഫുൾ എൻഡ്‌പോയിന്റ് സന്ദേശങ്ങൾ മെമ്മറിയിൽ ചേർക്കുന്നു, ചരിത്രം തിരികെ കൊണ്ടുവരുന്നു, ഓരോ അഭ്യർത്ഥനയോടും context ഉൾക്കൊള്ളുന്നു. മോഡൽ കോൺഫിഗറേഷൻ പതിവ്, മാതൃകകൾ വ്യത്യസ്തം.

## Azure OpenAI ഇൻഫ്രാസ്ട്രക്‌ചർ ഡിപ്പ്ലോയ് ചെയ്യുക

**Bash:**
```bash
cd 01-introduction
azd up  # സബ്സ്ക്രിപ്ഷനും സ്ഥലം (eastus2 ശിപാർശ ചെയ്യുന്നു) തിരഞ്ഞെടുക്കുക
```

**PowerShell:**
```powershell
cd 01-introduction
azd up  # സബ്സ്ക്രിപ്ഷനും സ്ഥിതിയും തിരഞ്ഞെടുക്കുക (eastus2 ശുപാർശ ചെയ്യുന്നു)
```

> **ഗമിക്കുക:** `RequestConflict: Cannot modify resource ... provisioning state is not terminal` പോലുള്ള timeout error വന്നാൽ, മടങ്ങി `azd up` വീണ്ടും ഓടിക്കുക. Azure റിസോഴ്സുകൾ പിന്‍വിധിയില്‍ പ്രോസസ്സിംഗ് നടത്തുന്നതുകൊണ്ടാണ് ഇങ്ങനെ സംഭവിക്കുന്നത്, റിസോഴ്സുകൾ ഫൈനൽ നിലയിൽ എത്തിയാൽ ഡിപ്പ്ലോയ്മെന്റ് പൂർത്തിയാകും.

ഇത് ചെയ്യും:  
1. GPT-5.2, text-embedding-3-small മോഡലുകളോടെ Azure OpenAI റിസോഴ്സ് ഡിപ്പ്ലോയ്മെന്റ്  
2. പ്രോജക്റ്റ് റൂട്ട്-ല `.env` ഫയൽ ഓട്ടോമാറ്റിക്കായി ജനറേറ്റ് ചെയ്തു ക്രെഡൻഷ്യൽസുകളോടെ  
3. ആവശ്യമുള്ള എല്ലാ എൻവയോൺമെന്റ് വേരിയബിളുകളും സജ്ജമാക്കുക

**ഡിപ്പ്ലോയ്മെന്റ് പ്രശ്നങ്ങൾ ഉണ്ടോ?** [Infrastructure README](infra/README.md) നോക്കുക, സബ്ഡോമെയ്ൻ നാമങ്ങളിലെ സംഗതികൾ, Azure പോർട്ടൽ മാൻവൽ ഡിപ്പ്ലോയ്മെന്റ് പ്രക്രിയകൾ, മോഡൽ കോൺഫിഗറേഷൻ മാർഗ്ഗനിർദ്ദേശങ്ങൾ ഉൾപ്പെടെ.

**ഡിപ്പ്ലോയ്മെന്റ് വിജയിച്ചിട്ടുണ്ടെന്ന് ഉറപ്പാക്കുക:**

### സ്റ്റേറ്റ്ലസ് ചാറ്റ് (ഇടത് പ്രാന്ത്)

ഇതിനു ആദ്യമൊഴിയാമാകും. "എന്റെ പേര് ജോൺ ആണ്" എന്നു ചോദിച്ച് ഉടൻപിന്നീട് "എന്റെ പേര് എന്ത്?" എന്ന് ചോദിക്കുക. മോഡൽ ഓർക്കില്ല, കാരണം ഓരോ സന്ദേശവും സ്വതന്ത്രമാണ്. ഇതാണ് ആശയ പടര്ത്തല അടിസ്ഥാന പ്രശ്നം - സംഭാഷണം context ഇല്ല.

<img src="../../../translated_images/ml/simple-chat-stateless-demo.13aeb3978eab3234.webp" alt="Stateless Chat Demo" width="800"/>

*AI കഴിഞ്ഞ സന്ദേശത്തിൽ നിന്നുള്ള നിങ്ങളുടെ പേര് ഓർക്കുന്നില്ല*

### സ്റ്റേറ്റ്‌ഫുൾ ചാറ്റ് (വലത് പ്രാന്ത്)

ഇപ്പോൾ ഇവിടെ അതേ തകർച്ച പരീക്ഷിച്ചു നോക്കൂ. "എന്റെ പേര് ജോൺ ആണ്" എന്ന് ചോദിച്ച് "എന്റെ പേര് എന്ത്?" എന്ന് ചോദിക്കുമ്പോൾ അത് ഓർക്കുന്നു. വ്യത്യാസം MessageWindowChatMemory ആണ് - ഇത് സംഭാഷണ ചരിത്രം നിലനിർത്തുന്നു, ഓരോ അഭ്യർത്ഥനയോടും ഈ ചരിത്രം ഉൾക്കൊള്ളുന്നു. ഇതാണ് ഉത്പാദന സംഭാഷണ AI പ്രവർത്തിക്കുന്ന വിധം.

<img src="../../../translated_images/ml/conversational-chat-stateful-demo.e5be9822eb23ff59.webp" alt="Stateful Chat Demo" width="800"/>

*AI മുൻപ് പറഞ്ഞ നിങ്ങളുടെ പേര് ഓർക്കുന്നു*

രണ്ട് പാനലുകളും ഒരേ GPT-5.2 മോഡൽ ഉപയോഗിക്കുന്നു. memory മാത്രം വ്യത്യാസമാണ്. ഇത് നിങ്ങൾക്ക് മെമ്മറി നിങ്ങളുടെ പ്രയോഗത്തിൽ എന്ത് തരുന്നു എന്ന് വ്യക്തമായി കാണിക്കുന്നു, യഥാർത്ഥ ഉപയോഗത്തിന് അത്യാവശ്യമാണ്.
