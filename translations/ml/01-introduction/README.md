## നിങ്ങൾക്കു പഠിക്കാനിരിക്കുന്ന കാര്യങ്ങൾ

നിങ്ങൾ ക്വിക് സ്റ്റാർട്ട് പൂര്‍ത്തിയാക്കിയിട്ടുണ്ടെങ്കിൽ, പ്രോംപ്റ്റുകൾ അയച്ച് പ്രതികരണങ്ങൾ എങ്ങനെ ലഭിക്കാമെന്നു കണ്ടു. ഇതാണ് അടിസ്ഥാനമായത്, പക്ഷെ യാഥാർത്ഥ്യ അപ്ലിക്കേഷനുകൾക്ക് കൂടുതല്‍ ആവശ്യമാണ്. ഈ മോഡ്യൂളിൽ, കാരണ സന്ദർഭം ഓർക്കുകയും സ്റ്റേറ്റ് നിലനിർത്തുകയും ചെയ്യുന്ന സംഭാഷണ എഐ എങ്ങനെ നിർമ്മിക്കാമെന്ന് നമുക്ക് പഠിക്കാം - ഒരിക്കൽ മാത്രം പ്രദർശിപ്പിക്കുന്ന ഡെമോയും പ്രൊഡക്ഷൻ റെഡിയായ അപ്ലിക്കേഷനും തമ്മിലുള്ള വ്യത്യാസം.

ഈ മാർഗ്ഗനിർദ്ദേശത്തിൽ ആദ്യം മുതൽ അവസാനം വരെ Azure OpenAI-യുടെ GPT-5.2 ഉപയോഗിക്കുന്നതാണ്, കാരണം അതിന്റെ വികസിത ന്യായീകരണ ശേഷികൾ വിവിധ പാറ്റേണുകളുടെ പെരുമാറ്റം വ്യക്തമായി കാണാൻ സഹായിക്കും. നിങ്ങൾ മെമ്മറി ചേർക്കുമ്പോൾ വ്യത്യാസം വ്യക്തമാകും. ഇതായിരിക്കും ഓരോ ഘടകവും നിങ്ങളുടെ അപ്ലിക്കേഷനിലേക്ക് എന്ത് കൊണ്ടുവരുന്നു എന്ന് മനസ്സിലാക്കാൻ എളുപ്പം.

നൽകുന്ന ഒരു അപ്ലിക്കേഷൻ നിർമ്മിക്കും, ഇത് രണ്ട് പാറ്റേണുകളും കാണിക്കുന്നു:

**സ്റ്റേറ്റ്ലെസ് ചാറ്റ്** - ഓരോ അഭ്യർത്ഥനയും സ്വതന്ത്രമാണ്. മോഡലിന് പൂർവ സന്ദേശങ്ങളുടെ ഓർമ്മയില്ല. ഇത് ക്വിക് സ്റ്റാർട്ടിൽ നിങ്ങൾ ഉപയോഗിച്ച പാറ്റേണാണു്.

**സ്റ്റേറ്റ്‌ഫുൾ സംഭാഷണം** - ഓരോ അഭ്യർത്ഥനയിലും സംഭാഷണ ചരിത്രം ഉൾപ്പെടുത്തി. മോഡൽ ഒന്നിലധികം ടേണുകൾക്ക് ഇടയിലുള്ള സമ്ബ്രേഷണവും സംരക്ഷിക്കുന്നു. ഇതാണ് പ്രൊഡക്ഷൻ അപ്ലിക്കേഷനുകൾക്ക് ആവശ്യമുള്ളത്.

## മുൻകൂർ ആവശ്യങ്ങൾ

- Azure സബ്‌സ്‌ക്രിപ്ഷൻ, Azure OpenAI ആക്സസ് সহ
- Java 21, Maven 3.9+ 
- Azure CLI (https://learn.microsoft.com/en-us/cli/azure/install-azure-cli)
- Azure Developer CLI (azd) (https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/install-azd)

> **കുറിപ്പ്:** ദയവായി ദയവായി ഓർക്കുക, നിങ്ങൾക്ക് നൽകിയ ഡെവ്‌ കന്റൈനറിലെ ജാവ, മേവൻ, Azure CLI, Azure Developer CLI (azd) മുൻകൂട്ടി ഇൻസ്റ്റാൾ ചെയ്‌തിരിക്കുന്നു.

> **കുറിപ്പ്:** ഈ മോഡ്യൂൾ Azure OpenAI-യിൽ GPT-5.2 ഉപയോഗിക്കുന്നു. ഡിപ്ലോയ്‌മെന്റ് നിങ്ങൾ `azd up` വഴി ഓട്ടോമാറ്റിക് ആയി ക്രമീകരിച്ചിരിക്കുന്നു - കോഡിൽ മോഡൽ നാമം മാറ്റരുത്.

## മൂല പ്രശ്നം മനസ്സിലാക്കൽ

ഭാഷാ മോഡലുകൾ സ്റ്റേറ്റ്ലെസ് ആണ്. ഓരോ API കോൾവുമും സ്വതന്ത്രമാണ്. നിങ്ങൾ “എന്റെ പേര് ജോൺ ആണ്” എന്ന് അയച്ച് ശേഷം “എന്റെ പേര് എന്താണ്?” എന്ന് ചോദിച്ചാൽ, മോഡലിന് അതിന്റെ ആവിഗതിനില്ല - നിങ്ങൾ സ്വയം പരിചയപ്പെടുത്തിയിരിക്കുന്നത് അറിയില്ല. അത് ഓരോ അഭ്യർത്ഥനയെയും ആദ്യ സംഭാഷണമായിരിക്കുകയാണ് എന്നു കരുതുന്നു.

ഇത് ലളിതമായ Q&A-ക്കു ശരിയാകാം, എന്നാൽ യഥാർത്ഥ അപ്ലിക്കേഷനുകളിൽ കാര്യക്ഷമമായിട്ടില്ല. കസ്റ്റമർ സർവീസ് ബോട്ടുകൾ നിങ്ങൾ പറഞ്ഞ കാര്യം ഓർക്കണം. വ്യക്തിഗത അസിസ്റ്റന്റുകൾക്ക് സംദർഭം വേണം. ഏതെങ്കിലും ഒട്ടേറെ ടേൺ സംഭാഷണം മെമ്മറി ആവശ്യപ്പെടുന്നു.

<img src="../../../translated_images/ml/stateless-vs-stateful.cc4a4765e649c41a.webp" alt="Stateless vs Stateful Conversations" width="800"/>

*സ്റ്റേറ്റ്ലെസ് (സ്വതന്ത്ര കോൾസ്) vs സ്റ്റേറ്റ്‌ഫുൾ (സന്ദർഭ-ജ്ഞാനമുള്ള) സംഭാഷണങ്ങൾ തമ്മിലുള്ള വ്യത്യാസം*

## ടോക്കണുകൾ മനസ്സിലാക്കൽ

സംഭാഷണത്തിലേക്ക് കയറുന്നതിനുമുമ്പ്, ടോക്കണുകൾ മനസ്സിലാക്കുന്നത് പ്രധാനമാണ് - ഭാഷാ മോഡലുകൾ പ്രോസസ്സ് ചെയ്യുന്ന ടെക്സ്റ്റിന്റെ അടിസ്ഥാന ഘടകങ്ങൾ:

<img src="../../../translated_images/ml/token-explanation.c39760d8ec650181.webp" alt="Token Explanation" width="800"/>

*ടെക്‌സ്‌റ്റ് എങ്ങനെ ടോക്കണുകളാക്കി വിഭജിക്കപ്പെടുന്നു എന്ന ഉദാഹരണം - "I love AI!" 4 പ്രോസസ്സിംഗ് യൂണിറ്റുകളായി*

ടോക്കണുകൾ എങ്ങനെ എഐ മോഡലുകൾ ടെക്സ്റ്റ് അളക്കും പ്രോസസ്സ് ചെയ്യുമെന്നതാണ്. വാക്കുകൾ, വിശേഷണങ്ങൾ, ഇടങ്ങളും ടോക്കണുകൾ ആകാം. നിങ്ങളുടെ മോഡലിന് ഒരേസമയം എത്ര ടോക്കണുകൾ പ്രോസസ്സ് ചെയ്യാമെന്നുള്ള പരിധിയുണ്ട് (GPT-5.2-ന് 400,000, അതിൽ 272,000 ഇൻപുട്ട് ടോക്കണുകളും 128,000 ഔട്ട്പുട്ട് ടോക്കണുകളുമാണ്). ടോക്കണുകൾ മനസ്സിലാക്കുന്നത് സംഭാഷണ ദൈർഘ്യവും ചെലവുകളും നിയന്ത്രിക്കാൻ സഹായിക്കുന്നു.

## മെമ്മറി എങ്ങനെ പ്രവർത്തിക്കുന്നു

ചാറ്റ് മെമ്മറി സ്റ്റേറ്റ്ലെസ് പ്രശ്നം പരിഹരിക്കുന്നു സംഭാഷണ ചരിത്രം നിലനിർത്തി. നിങ്ങളുടെ അഭ്യർത്ഥന മോഡലിലേക്ക് അയയ്‌ക്കുന്നതിനു മുൻപ്, ഫ്രെയിംവർക്ക് ബന്ധപ്പെട്ട മുൻസന്ദേശങ്ങൾ ഇടുന്നു. നിങ്ങൾ “എന്റെ പേര് എന്താണ്?” എന്ന് ചോദിച്ചാൽ, സിസ്റ്റം മുഴുവൻ സംഭാഷണ ചരിത്രവും അയക്കുന്നു, അതിനാൽ മോഡൽ “എന്റെ പേര് ജോൺ ആണ്” എന്ന നിങ്ങളടെ മുൻപ് പറഞ്ഞത് കാണാം.

LangChain4j മെമ്മറി നടപ്പിലാക്കലുകൾ നൽകുന്നു, ഇത് സ്വയം കൈകാര്യം ചെയ്യും. നിങ്ങൾ എത്ര സന്ദേശങ്ങൾ സൂക്ഷിക്കണം എന്ന് തെരഞ്ഞെടുക്കാം, ഫ്രെയിംവർക്ക് കോൺടെക്സ്റ്റ് വിൻഡോ മാനേജുചെയ്യും.

<img src="../../../translated_images/ml/memory-window.bbe67f597eadabb3.webp" alt="Memory Window Concept" width="800"/>

*MessageWindowChatMemory അടുത്തിടെ വന്ന സന്ദേശങ്ങളായി സ്ലൈഡിംഗ് വിൻഡോ നിലനിർത്തുന്നു, പഴയ സന്ദേശങ്ങൾ സ്വയം ഒഴിവാക്കുന്നു*

## ഇത് LangChain4j എങ്ങനെ ഉപയോഗിക്കുന്നു

ഈ മോഡ്യൂൾ ക്വിക് സ്റ്റാർട്ടിനെ വിപുലീകരിച്ച് Spring Boot ചേർത്ത് സംഭാഷണ മെമ്മറി കൂട്ടിച്ചേർക്കുന്നു. ഘടകങ്ങൾ ഇങ്ങനെ ബന്ധപ്പെടുന്നു:

**ഡിപ്പൻഡൻസീസുകൾ** - രണ്ട് LangChain4j ലൈബ്രറികൾ ചേർക്കുക:

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```

**ചാറ്റ് മോഡൽ** - Azure OpenAI-യെ Spring ബീൻ ആയി ക്രമീകരിക്കൂ ([LangChainConfig.java](../../../01-introduction/src/main/java/com/example/langchain4j/config/LangChainConfig.java)):

```java
@Bean
public OpenAiOfficialChatModel openAiOfficialChatModel() {
    return OpenAiOfficialChatModel.builder()
            .baseUrl(azureEndpoint)
            .apiKey(azureApiKey)
            .modelName(deploymentName)
            .timeout(Duration.ofMinutes(5))
            .maxRetries(3)
            .build();
}
```

ബിൽഡർ `azd up` നിർദ്ദേശിച്ച എൻവയോൺമെന്റ് വേരിയബിളുകളിൽ നിന്നു ക്രെഡൻഷ്യൽസ് വായിക്കുന്നു. `baseUrl` നിങ്ങളുടെ Azure എൻഡ്‌പോയിന്റ് ആയിട്ട് സജ്ജമാക്കുമ്പോൾ, ഒരു OpenAI ക്ലയന്റ് Azure OpenAI-യുമായി പ്രവർത്തിക്കും.

**സംഭാഷണ മെമ്മറി** - MessageWindowChatMemory ഉപയോഗിച്ച് ചാറ്റ് ചരിത്രം ട്രാക്ക് ചെയ്യൂ ([ConversationService.java](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java)):

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("My name is John"));
memory.add(AiMessage.from("Nice to meet you, John!"));

memory.add(UserMessage.from("What's my name?"));
AiMessage aiMessage = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage);
```

`withMaxMessages(10)` ഉപയോഗിച്ച് കഴിഞ്ഞ 10 സന്ദേശങ്ങൾ സൂക്ഷിക്കുന്നു. ഉപയോക്തൃവും എഐ സന്ദേശങ്ങളും ടൈപ്പ് ചെയ്ത റാപ്പറുകളിലൂടെ ചേർക്കുക: `UserMessage.from(text)`, `AiMessage.from(text)`. ചരിത്രം `memory.messages()` വഴിയിൽ തിരിച്ചെടുക്കുകയും മോഡലിന് അയയ്ക്കുക. സർവീസ് ഓരോ സംഭാഷണ IDന് പ്രത്യേകം മെമ്മറി ഇൻസ്റ്റൻസുകൾ സൂക്ഷിക്കുന്നു, ഇതുവഴി ഒരേ സമയം പല ഉപയോക്താക്കളും ചാറ്റ് ചെയ്യാൻ കഴിയും.

> **🤖 GitHub Copilot ചാറ്റ് ഉപയോഗിച്ച് പരീക്ഷിക്കൂ:**  
> [`ConversationService.java`](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java) തുറന്ന് ചോദിക്കൂ:  
> - "MessageWindowChatMemory എപ്പോൾ വിൻഡോ നിറഞ്ഞാൽ ഏതെല്ലാം സന്ദേശങ്ങൾ ഒഴിവാക്കണമെന്ന് എങ്ങനെ തീരുമാനിക്കുന്നു?"  
> - "ഇൻ-മെമ്മറിയുടെ പകരം ഒരു ഡാറ്റാ ബേസിൽ കസ്റ്റം മെമ്മറി സ്റ്റോറേജ് എങ്ങനെ നടപ്പിലാക്കാം?"  
> - "പഴയ സംഭാഷണ ചരിത്രം സംക്ഷേപിക്കാൻ സಮ್ಮമായി എങ്ങനെ അല്പം ചേർക്കാം?"

സ്റ്റേറ്റ്ലെസ് ചാറ്റ് എൻഡ്‌പോയിന്റിൽ മെമ്മറി സമ്പൂർണം ഒഴിവാക്കുന്നു - ക്വിക് സ്റ്റാർട്ടിലെപോലെ `chatModel.chat(prompt)` മാത്രം. സ്റ്റേറ്റ്‌ഫുൾ എൻഡ്‌പോയിന്റ് സന്ദേശങ്ങൾ മെമ്മറിയിൽ ചേർക്കുന്നു, ചരിത്രം തിരിച്ച് എടുക്കുന്നു, പ്ര konte\[qnst\\\] അളവിൽ ഉൾപ്പെടുത്തുന്നു. മോഡൽ ക്രമീകരണം സമാനമാണ്, പാറ്റേണുകൾ വ്യത്യസ്തം.

**ഡിപ്ലോയ്‌മെന്റ് പ്രശ്നങ്ങളുണ്ടോ?** സബ്‌ഡൊമെയ്ൻ നാമ സംഘട്ടനങ്ങൾ, മാനുവൽ Azure പോർട്ടൽ ഡിപ്ലോയ്മെന്റ് ഘട്ടങ്ങൾ, മോഡൽ ക്രമീകരണ മാനദണ്ഡങ്ങൾ ഉൾപ്പെടെ കൂടുതൽ വിവരങ്ങൾക്കായി [Infrastructure README](infra/README.md) നോക്കുക.

**ഡിപ്ലോയ്‌മെന്റ് വിജയിച്ചുവെന്ന് ഉറപ്പാക്കാം:**

### സ്റ്റേറ്റ്ലെസ് ചാറ്റ് (ഇടതുകൈ പാനൽ)

ഇത് ആദ്യം പരീക്ഷിക്കുക. "എന്റെ പേര് ജോൺ ആണ്" എന്നു ചോദിച്ച് തുടർന്ന് ഉടൻ "എന്റെ പേര് എന്താണ്?" എന്ന് ചോദിക്കുക. മോഡൽ ഓർക്കുന്നില്ല, കാരണം ഓരോ സന്ദേശവും സ്വതന്ത്രമാണ്. അടിസ്ഥാന ഭാഷാ മോഡൽ ഇന്റഗ്രേഷനിലെ പ്രധാന പ്രശ്നം ഇത് - സംഭാഷണ സംദർഭം ഇല്ല.

### സ്റ്റേറ്റ്‌ഫുൾ ചാറ്റ് (വലതുകൈ പാനൽ)

ഇപ്പോൾ ഇവിടെ സമാനമായി ശ്രമിക്കുക. "എന്റെ പേര് ജോൺ ആണ്" എന്ന ശേഷം "എന്റെ പേര് എന്താണ്?" ചോദിച്ചാൽ, മോഡൽ ഓർക്കുന്നു. വ്യത്യാസം MessageWindowChatMemory ആണ് - അത് സംഭാഷണ ചരിത്രം നിലനിർത്തുകയും ഓരോ അഭ്യർത്ഥനയിലും അതു ഉൾപ്പെടുകയും ചെയ്യുന്നു. ഇതാ പ്രൊഡക്ഷൻ conversational AI പ്രവർത്തിക്കുന്നത്.

രണ്ടും സമാനമായ GPT-5.2 മോഡൽ ഉപയോഗിക്കുന്നു. വ്യത്യാസം മെമ്മറിയിലാണ്. ഇത് മെമ്മറി നിങ്ങളുടെ അപ്ലിക്കേഷനിലേക്ക് എന്ത് കൊണ്ടുവരുന്നു എന്നു വ്യക്തമായി കാണിക്കുന്നതും യഥാർത്ഥ ഉപയോഗത്തിനായിരിക്കും പ്രധാനമാണെന്ന് കാണിക്കുന്നതും ആണ്.

**അസൂയാപത്രം**:  
ഈ രേഖ AI വിവർത്തനം സേവനമായ [Co-op Translator](https://github.com/Azure/co-op-translator) ഉപയോഗിച്ച് വിവർത്തനം ചെയ്തതാണ്. ഞങ്ങൾ കൃത്യതയ്ക്കായി ശ്രമിച്ചെങ്കിലും, സ്വയംചാലിത വിവർത്തനങ്ങൾക്ക് പിശകുകൾ അല്ലെങ്കിൽ അകൃതികൾ ഉണ്ടാകാമെന്ന് ദയവായി ശ്രദ്ധിക്കുക. സ്വതന്ത്ര ഭാഷയിലെ പ്രഥമ രേഖയാണ് അതിന്റെ ശാസ്ത്രീയ ഉറവിടമായി കരുതേണ്ടത്. മുഖ്യമായ വിവരങ്ങൾക്ക് പ്രൊഫഷണൽ മനുഷ്യ വിവർത്തനം നിർദ്ദേശிக்கപ്പെടുന്നു. ഈ വിവർത്തനത്തിന്റെ ഉപയോഗത്തിൽ നിന്നുണ്ടാകുന്ന ഏതെങ്കിലും മനസ്സിലാക്കലിൽ or തെറ്റിദ്ധാരണയിൽ ഞങ്ങൾ ഉത്തരവാദിത്വം ഏറ്റെടുക്കുന്നില്ല.
