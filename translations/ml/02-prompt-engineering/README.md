## നിങ്ങൾ പഠിക്കേണ്ടത്

മുമ്പത്തെ മോഡ്യൂളിൽ, ഓർമ്മ കൂടലിലൂടെ സംഭാഷണ മ ഷീന AI എങ്ങനെ സാധ്യമാക്കുന്നു എന്നത് കാണുകയും അടിസ്ഥാന ഇടപാടുകൾക്കായി GitHub മോഡലുകൾ ഉപയോഗിക്കുകയും ചെയ്തു. ഇപ്പോൾ നാം Azure OpenAIയുടെ GPT-5.2 ഉപയോഗിച്ച് നിങ്ങൾ ചോദിക്കുന്ന ചോദ്യങ്ങൾ - പ്രോംപ്റ്റുകൾ - എങ്ങനെ രൂപകൽപ്പന ചെയ്യാമെന്ന് ശ്രദ്ധിക്കാം. നിങ്ങൾ പ്രോംപ്റ്റുകൾ ഘടിപ്പിക്കുന്ന വിധം ലഭിക്കുന്ന മറുപടി ഗുണനിലവാരത്തെയ് പ്രകൃതമായും പ്രഭാവിതമാക്കും.

Reasoning control പരിചയപ്പെടുത്തുന്ന GPT-5.2 ആണ് ഉപയോഗിക്കുന്നത് - നിങ്ങൾ മോഡലിനെ എത്ര ആകുന്നതുവരെ ചിന്തിക്കണമെന്ന് പറയാം. ഇത് വ്യത്യസ്ത prompting തന്ത്രങ്ങൾ കൂടുതൽ വ്യക്തമായി കാണിക്കുകയും ഓരോ സമീപനം ഏപ്പോൾ ഉപയോഗിക്കണമെന്നും മനസ്സിലാക്കാൻ സഹായിക്കുകയും ചെയ്യും. GitHub മോഡലുകളുമായി താരതമ്യപ്പെടുത്തുമ്പോൾ Azure-ന് കുറവ് റേറ്റ് പരിധികളുണ്ടായിരിക്കും GPT-5.2-ന്.

## ആവശ്യമായ മുൻവശിപ്പ്

- മോഡ്യൂൾ 01 പൂർത്തിയാക്കിയിരിക്കുന്നതു (Azure OpenAI വിഭവങ്ങൾ വിന്യസിച്ചിരിക്കണം)
- അറ്റം ഡയറക്ടറിയിൽ `.env` ഫയൽ Azure ക്രെഡൻഷ്യൽസോടെ (മോഡ്യൂൾ 01-ൽ `azd up` ഉപയോഗിച്ച് സൃഷ്ടിച്ചത്)

> **നോട്ടീസ്:** മോഡ്യൂൾ 01 പൂർത്തിയാക്കിയിട്ടില്ലെങ്കിൽ, ആദ്യം അവിടെ കൊടുത്ത വിന്യാസ നിർദ്ദേശങ്ങൾ अनुसരിക്കുക.

## Prompt Engineering മനസ്സിലാക്കുക

Prompt engineering എന്നത് നിങ്ങൾ ആഗ്രഹിക്കുന്ന ഫലങ്ങൾ സ്ഥിരമായി സമ്മാനിക്കുന്ന ഇൻപുട്ട് ടെക്സ്റ്റ് രൂപകൽപ്പന ക്കാണ്. അത് വെറും ചോദ്യങ്ങൾ ചോദിക്കുന്നതല്ല - മോഡൽ നിങ്ങൾക്ക് വേണ്ടത് തീർച്ചയായും മനസ്സിലാക്കാനും അതിൻറെ നൽകേണ്ട രീതിയും നിർദ്ദേശിക്കുന്നതിനുമുള്ള അഭ്യർത്ഥനകളുടെ ഘടനയുമായി ബന്ധപ്പെട്ടതാണ്.

ഒരു സഹപ്രവർത്തകനെ നിർദ്ദേശം നൽകുന്നതുപോലെയാണ് ഇതിനെ കാണുക. "ബഗ് പരിഹരിക്കുക" എന്നത് സംശയാസ്പദമാണ്. "UserService.java ലെ 45-ആം ലൈനിലെ നൾ പോയിന്റർ എക്സെപ്ഷൻ നൾ ചെക്ക് ചേർത്ത് പരിഹരിക്കുക" എന്നത് വ്യക്തമാണ്. ഭാഷ മോഡലുകൾക്കും ഇതുപോലെ - സ്പെസിഫിസിറ്റി, ഘടന എന്നിവ പ്രധാനമാണ്.

## ഇത് LangChain4j എങ്ങനെ ഉപയോഗിക്കുന്നു

മുൻ മോഡ്യൂളുകളിൽ നിന്ന് എടുത്ത LangChain4j അടിസ്ഥാനത്തിൽ ഉയർന്ന നില prompting പാറ്റേണുകൾ ഈ മോഡ്യൂൾ പ്രദർശിപ്പിക്കുന്നു, പ്രോംപ്റ്റുകളുടെ ഘടനയ്ക്കും reasoning control-നും പ്രത്യേക ശ്രദ്ധ കൊടുക്കുന്ന.

<img src="../../../translated_images/ml/langchain4j-flow.48e534666213010b.webp" alt="LangChain4j Flow" width="800"/>

*LangChain4j നിങ്ങളുടെ പ്രോംപ്റ്റുകൾ Azure OpenAI GPT-5.2-യുമായി കണെക്ട് ചെയ്യുന്നത് എങ്ങനെ*

**ഡിപ്പൻഡൻസികൾ** - `pom.xml`-ൽ നിർവ്വചിച്ചിരിക്കുന്ന താഴെ പറയുന്ന LangChain4j ഡിപ്പൻഡൻസികൾ മോഡ്യൂൾ 02 ഉപയോഗിക്കുന്നു:  
```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```
  
**OpenAiOfficialChatModel കോൺഫിഗറേഷൻ** - [LangChainConfig.java](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/config/LangChainConfig.java)

ചാറ്റ് മോഡൽ OpenAI ഔദ്യോഗിക ക്ലയന്റ് ഉപയോഗിച്ച് Spring ബീൻ ആയി മാന്വൽ കോൺഫിഗർ ചെയ്തിട്ടുണ്ട്, ഇത് Azure OpenAI എന്റ്പോയിൻ്റുകൾ പിന്തുണയ്ക്കുന്നു. മോഡ്യൂൾ 01-ൽ നിന്ന് വലിയ വ്യത്യാസം ഈ മോഡലിന് അയക്കുന്ന `chatModel.chat()` എന്ന പ്രോംപ്റ്റുകളുടെ ഘടനയാണ്, മോഡൽ സജ്ജീകരണമല്ല.

**സിസ്റ്റംയും ഉപയോക്തൃ സന്ദേശങ്ങളും** - [Gpt5PromptService.java](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java)

LangChain4j വ്യക്തമാക്കുന്നതിനായി സന്ദേശ തരം വേർതിരിക്കുന്നു. `SystemMessage` AI-യുടെ പെരുമാറ്റവും സന്ധർഭവും സജ്ജമാക്കുന്നു (“നിവർത്തന കൃതി വിശകലനകന് നീ ആണ്” പോലുള്ള), `UserMessage` യഥാർത്ഥ അഭ്യർത്ഥന ഉൾക്കൊള്ളുന്നു. ഇത് വ്യത്യസ്ത ഉപയോക്തൃ ചോദ്യങ്ങൾക്കിടയിൽ സ്ഥിരതയുള്ള AI പെരുമാറ്റം നിലനിർത്താൻ സഹായിക്കുന്നു.

```java
SystemMessage systemMsg = SystemMessage.from(
    "You are a helpful Java programming expert."
);

UserMessage userMsg = UserMessage.from(
    "Explain what a List is in Java"
);

String response = chatModel.chat(systemMsg, userMsg);
```
  
<img src="../../../translated_images/ml/message-types.93e0779798a17c9d.webp" alt="Message Types Architecture" width="800"/>

*SystemMessage തുടർച്ചയായ സന്ധർഭം നൽകുന്നു, UserMessages വ്യക്തിഗത അഭ്യർത്ഥനകൾ*

**Multi-Turn ന് MessageWindowChatMemory** - മൾട്ടി-ടേൺ സംഭാഷണ പാറ്റേണിനായി, മോഡ്യൂൾ 01-ൽ നിന്നുള്ള `MessageWindowChatMemory` വീണ്ടും ഉപയോഗിക്കുന്നു. ഓരോ സെഷനും സ്വന്തം മെമ്മറി ഇൻസ്റ്റൻസ് `Map<String, ChatMemory>`-ലേക്ക് സൂക്ഷിക്കപ്പെടുന്നു, ഇതോടെ പല സമകാലീന സംഭാഷണങ്ങൾ പ്രാസംഗികമായി വേർതിരിക്കുന്നത് സാധ്യമാകും.

**Prompt Templates** - യഥാർത്ഥ ശ്രദ്ധ prompting ഡിസൈനിലാണ്, പുതിയ LangChain4j API-കൾക്കല്ല. ഓരോ പാറ്റേണും (ലോ എഗർനസ്, ഹൈ എഗർനസ്, ടാസ്‌ക് എക്സിക്യൂഷൻ തുടങ്ങിയവ) ഒരേ `chatModel.chat(prompt)` നടപടിചെയ്യും, ജാഗ്രതയോടെ ഘടിപ്പിച്ച പ്രോംപ്റ്റ് സ്ട്രിംഗുകൾ ഉപയോഗിച്ച്. XML ടാഗുകൾ, നിർദ്ദേശങ്ങൾ, ഫോർമാറ്റിംഗ് എല്ലാം പ്രോംപ്റ്റ് ടെക്സ്റ്റിൻ്റെ ഭാഗമാണ്; LangChain4j പ്രത്യേകതകൾ അല്ല.

**Reasoning Control** - GPT-5.2-ന്റെ reasoning ശ്രമം “maximum 2 reasoning steps” അല്ലെങ്കിൽ “explore thoroughly” പോലെയുള്ള പ്രോംപ്റ്റ് നിർദ്ദേശങ്ങളാൽ നിയന്ത്രിക്കുന്നു. ഇവ prompting തന്ത്രങ്ങളാണ്, LangChain4j കോൺഫിഗറേഷനല്ല. ലൈബ്രറി നിങ്ങളുടെ പ്രോംപ്റ്റുകൾ മോഡലിലേക്ക് വീണ്ടും പ്രദാനം ചെയ്യുന്നു.

മુખ്യ ഓർമ്മപ്പെടുത്തൽ: LangChain4j അടിസ്ഥാന സൗകര്യങ്ങൾ (model connection via [LangChainConfig.java](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/config/LangChainConfig.java), memory, message handling via [Gpt5PromptService.java](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java)) നൽകുകയും, ഈ മോഡ്യൂൾ ആ അടിസ്ഥാനത്തിൽ ഫലപ്രദമായ പ്രോംപ്റ്റുകൾ എങ്ങനെ നിർമ്മിക്കാമെന്ന് പഠിപ്പിക്കുകയും ചെയ്യുന്നു.

## പ്രധാന പാറ്റേണുകൾ

എല്ലാ പ്രശ്നങ്ങൾക്കും ഒരേയൊരു സമീപനം വേണ്ടതല്ല. ചില ചോദ്യങ്ങൾ വേഗം മറുപടി ആവശ്യപ്പെടുന്നു, മറ്റുള്ളവക്ക് ആഴത്തിലുള്ള ചിന്ത നിർബന്ധമാണ്. ചിലപ്പോൾ reasoning ദൃശ്യമായിരിക്കണം, ചിലപ്പോൾ ഫലങ്ങൾ മാത്രം. ഈ മോഡ്യൂളിൽ എട്ട് prompting പാറ്റേണുകൾ ഉൾപ്പെടുന്നു - ഓരോന്നും വ്യത്യസ്ത സാഹചര്യങ്ങൾക്ക് അനുയോജ്യമാണ്. അവയെല്ലാം പരീക്ഷിച്ച് ഓരോ സമീപനവും ഏപ്പോൾ ഫലപ്രദമാണെന്ന് താങ്കൾ മനസ്സിലാക്കും.

<img src="../../../translated_images/ml/reasoning-effort.db4a3ba5b8e392c1.webp" alt="Reasoning Effort Comparison" width="800"/>

*ലോ എഗർനസ് (വേഗമേറിയ, നേരിട്ടുള്ള) vs ഹൈ എഗർനസ് (വിശദമായ, അന്വേഷണപരമായ)* reasoning സമീപനങ്ങൾ*

**ലോ എഗർനസ് (തെരെഞ്ഞെടുത്ത, വേഗത്തിൽ)** - ലളിതമായ ചോദ്യങ്ങൾക്ക് വേഗവും നേരിട്ടുള്ള മറുപടികളും വേണ്ടതായപ്പോൾ. മോഡൽ ഏറ്റവും കൂടുതലായി 2 reasoning ഘട്ടങ്ങൾ മാത്രം നടത്തും. കണക്കുകൂട്ടലുകൾ, അന്വേഷങ്ങൾ, നേരിട്ടുള്ള ചോദ്യങ്ങൾക്കായി ഉപയോഗിക്കുക.

```java
String prompt = """
    <reasoning_effort>low</reasoning_effort>
    <instruction>maximum 2 reasoning steps</instruction>
    
    What is 15% of 200?
    """;

String response = chatModel.chat(prompt);
```
  
> 💡 **GitHub Copilot-നോട് അന്വേഷിക്കുക:** തുറക്കുക [`Gpt5PromptService.java`](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java) ചോദിക്കുക:  
> - "ലോ എഗർനസ് ആയും ഹൈ എഗർനസ് ആയും prompting പാറ്റേണുകൾ തമ്മിലുള്ള വ്യത്യാസം എന്താണ്?"  
> - "XML ടാഗുകൾ പ്രോംപ്റ്റുകളിൽ എങ്ങനെ AI-ന്റെ മറുപടി ഘടിപ്പിക്കാൻ സഹായിക്കുന്നു?"  
> - "സ്വയം-പ്രതിഫലന പാറ്റേണുകളും നേരിട്ടുള്ള നിർദ്ദേശവും ഏപ്പോൾ ഉപയോഗിക്കണം?"

**ഹൈ എഗർനസ് (ആഴത്തിലുള്ള, വിശദമായ)** - സങ്കീർണ്ണമായ പ്രശ്നങ്ങൾക്ക് സമഗ്ര വിശകലനം വേണമെങ്കിൽ. മോഡൽ വിശദമായി പരിശോധിച്ച് വിശദമായ reasoning കാണിക്കുന്നു. സിസ്റ്റം ഡിസൈൻ, ആർക്കിടെക്ചർ തീരുമാനങ്ങൾ, സങ്കീർണ്ണമായ ഗവേഷണത്തിനായി ഉപയോഗിക്കുക.

```java
String prompt = """
    <reasoning_effort>high</reasoning_effort>
    <instruction>explore thoroughly, show detailed reasoning</instruction>
    
    Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```
  
**ടാസ്‌ക് എക്സിക്യൂഷൻ (പടി പടിയായി പുരോഗതി)** - പലയിടത്ത് പ്രവർത്തിക്കുന്ന പ്രവൃത്തികൾക്കായി. മോഡൽ മുൻപൊരു പദ്ധതി നൽകുകയും, ഓരോ ഘട്ടവും വിശദമായി വിവരിക്കുകയും, അവസാനം ചുരുക്കം നൽകുകയും ചെയ്യും. മൈഗ്രേഷൻസ്, നടപ്പാക്കലുകൾ, ഒന്നിലധികം ഘട്ടങ്ങളുള്ള പ്രക്രിയകൾക്കായി ഉപയോഗിക്കുക.

```java
String prompt = """
    <task>Create a REST endpoint for user registration</task>
    <preamble>Provide an upfront plan</preamble>
    <narration>Narrate each step as you work</narration>
    <summary>Summarize what was accomplished</summary>
    """;

String response = chatModel.chat(prompt);
```
  
Chain-of-Thought prompting മോഡലിനെ reasoning പ്രക്രിയ കാണിക്കാൻ അനുവദിക്കുന്നു, സങ്കീർണ്ണമായ പ്രവർത്തനങ്ങളിൽ കൃത്യത കൂട്ടുന്നു. പടി പടിയായി നിർവചനം മനുഷ്യനും AI-ഉം ലജിക് മനസ്സിലാക്കാൻ സഹായിക്കുന്നു.

> **🤖 GitHub Copilot Chat ഉപയോഗിച്ച് പരീക്ഷിക്കുക:** ഈ പാറ്റേണിനെക്കുറിച്ച് ചോദിക്കുക:  
> - "ദീർഘകാല പ്രവർത്തനങ്ങൾക്കായി ടാസ്‌ക് എക്സിക്യൂഷൻ പാറ്റേൺ എങ്ങനെ അനുയോജ്യമായി മാറ്റാം?"  
> - "ഉത്പാദന അപ്ലിക്കേഷനിൽ ടൂൾ പ്രീഅംബിൾസിന്റെ ഘടന എങ്ങനെ മികച്ച രീതിയിലാക്കാം?"  
> - "UI-യിൽ ഇടനില പുരോഗതി അപ്ഡേറ്റുകൾ എങ്ങനെ അടങ്ങിയെടുക്കും, പ്രദർശിപ്പിക്കും?"

<img src="../../../translated_images/ml/task-execution-pattern.9da3967750ab5c1e.webp" alt="Task Execution Pattern" width="800"/>

*പദ്ധതി → നടപ്പാക്കൽ → ചുരുക്കം, മൾട്ടി-സ്‌റ്റെപ്പ് ടാസ്‌കുകൾക്ക്*

**സ്വയം പ്രതിഫലിക്കുന്ന കോഡ്** - ഉത്പാദന നിലവാരമുള്ള കോഡ് സൃഷ്ടിക്കാനുള്ളത്. മോഡൽ ആദ്യം കോഡ് സൃഷ്ടിച്ച്, ഗുണനിലവാര മാനദണ്ഡത്തിൽ പരിശോധിച്ച്, ശേഷം മെച്ചപ്പെടുത്തുന്നു. പുതിയ ഫീച്ചറുകൾ അല്ലെങ്കിൽ സർവീസുകൾ നിർമ്മിക്കുമ്പോൾ ഇത് ഉപയോഗിക്കുക.

```java
String prompt = """
    <task>Create an email validation service</task>
    <quality_criteria>
    - Correct logic and error handling
    - Best practices (clean code, proper naming)
    - Performance optimization
    - Security considerations
    </quality_criteria>
    <instruction>Generate code, evaluate against criteria, improve iteratively</instruction>
    """;

String response = chatModel.chat(prompt);
```
  
<img src="../../../translated_images/ml/self-reflection-cycle.6f71101ca0bd28cc.webp" alt="Self-Reflection Cycle" width="800"/>

*പര്യവേക്ഷണ ചക്രം - സൃഷ്ടിക്കുക, മൂല്യനിർണ്ണയം നടത്തുക, പ്രശ്നങ്ങൾ തിരിച്ചറിയുക, മെച്ചപ്പെടുത്തുക, ആവർത്തിക്കുക*

**സ്ട്രക്ചർഡ് അനാലിസിസ്** - സ്ഥിര രീതി നോക്കിപ്പ്. കോഡ് ശരിതിരുത്തൽ, പോരായ്മകൾ, പ്രകടനം, സുരക്ഷ എന്നീ മൂലങ്ങൾ ഉപയോഗിച്ച് മോഡൽ വിലയിരുത്തുന്നു. കോഡ് റിവ്യൂ അല്ലെങ്കിൽ ഗുണനിലവാര പരിശോധനയ്ക്ക് ഉപയോഗിക്കുക.

```java
String prompt = """
    <code>
    public List getUsers() {
        return database.query("SELECT * FROM users");
    }
    </code>
    
    <framework>
    Evaluate using these categories:
    1. Correctness - Logic and functionality
    2. Best Practices - Code quality
    3. Performance - Efficiency concerns
    4. Security - Vulnerabilities
    </framework>
    """;

String response = chatModel.chat(prompt);
```
  
> **🤖 GitHub Copilot Chat ഉപയോഗിച്ച് പരീക്ഷിക്കുക:** സ്ട്രക്ചർഡ് അനാലിസിസ് സംബന്ധിച്ച് ചോദിക്കുക:  
> - "വ്യത്യസ്ത കോഡ് റിവ്യൂകൾക്കായി വിശകലന ഫ്രെയിംവർക്ക് എങ്ങനെ കസ്റ്റമൈസ് ചെയ്യാം?"  
> - "സ്റ്റ്രക്ചർഡ് ഔട്ട്പുട്ട് പ്രോഗ്രാമാറ്റിക്കായി പാർസ് ചെയ്ത് പ്രവർത്തിപ്പിക്കാൻ മികച്ച മാർഗം എന്താണ്?"  
> - "വിവിധ റിവ്യൂ സെഷനുകളിൽ സ്ഥിരമായ ഗൗരവമനിക്ക് എങ്ങനെ ഉറപ്പ് നൽകാം?"

<img src="../../../translated_images/ml/structured-analysis-pattern.0af3b690b60cf2d6.webp" alt="Structured Analysis Pattern" width="800"/>

*നിത്യസ്ഥിരതയുള്ള കോഡ് റിവ്യൂകൾക്ക് നാലു വിഭാഗം ഫ്രെയിംവർക്ക്, ഗൗരവ തോതുകൾ ഉൾപ്പെടെ*

**മൾട്ടി-ടേൺ ചാറ്റ്** - പ്രാധാന്യമുള്ള സന്ധർഭം ആവശ്യമുള്ള സംഭാഷണങ്ങൾക്ക്. മോഡൽ മുമ്പത്തെ സന്ദേശങ്ങൾ ഓർത്ത് പുതിയവയുടെ അടിസ്ഥാനത്തിൽ വികസിപ്പിക്കുന്നു. ഇന്ററാക്ടീവ് സഹായ സെഷനുകൾക്കും സങ്കീർണ്ണ QA-യ്ക്കും ഉപയോഗിക്കുക.

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("What is Spring Boot?"));
AiMessage aiMessage1 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage1);

memory.add(UserMessage.from("Show me an example"));
AiMessage aiMessage2 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage2);
```
  
<img src="../../../translated_images/ml/context-memory.dff30ad9fa78832a.webp" alt="Context Memory" width="800"/>

*ചർച്ചാ സന്ധർഭം എങ്ങനെ ടോക്കൺ പരിധി വരെ നിലനിർത്തുന്നു*

**പടി പടിയായി വിന്യാസം** - ദൃശ്യമായ ലജിക് ആവശ്യമുള്ള പ്രശ്നങ്ങൾക്ക്. ഓരോ ഘട്ടത്തിനും സ്പষ্ট reasoning കാണിക്കുന്നു. ഗണിത പ്രശ്നങ്ങൾ, ലജിക് പസിലുകൾ അല്ലെങ്കിൽ ചിന്താ പ്രക്രിയ മനസ്സിലാക്കേണ്ടപ്പോൾ ഉപയോഗിക്കുക.

```java
String prompt = """
    <instruction>Show your reasoning step-by-step</instruction>
    
    If a train travels 120 km in 2 hours, then stops for 30 minutes,
    then travels another 90 km in 1.5 hours, what is the average speed
    for the entire journey including the stop?
    """;

String response = chatModel.chat(prompt);
```
  
<img src="../../../translated_images/ml/step-by-step-pattern.a99ea4ca1c48578c.webp" alt="Step-by-Step Pattern" width="800"/>

*പ്രശ്നങ്ങളെ ബദ്ധപ്പെട്ട ലജിക്കൽ ഘട്ടങ്ങളായി വിഭജിക്കൽ*

**നിയന്ത്രിത ഔട്ട്പുട്** - പ്രത്യേക ഫോർമാറ്റ് ആവശ്യങ്ങളുള്ള മറുപടികൾക്കായി. മോഡൽ കർശനമായ ഫോർമാറ്റ്, നീളം നിയമങ്ങൾ പിന്തുടരുന്നു. ചുരുക്കങ്ങൾക്കോ, കൃത്യമായ ഘടനകളുള്ള ഔട്ട്പുട്ടിനോ വേണ്ട സമയം.

```java
String prompt = """
    <constraints>
    - Exactly 100 words
    - Bullet point format
    - Technical terms only
    </constraints>
    
    Summarize the key concepts of machine learning.
    """;

String response = chatModel.chat(prompt);
```

*പ്രത്യേക ഫോർമാറ്റ്, നീളം, ഘടന ആവശ്യകതകൾ പാലിക്കൽ*

## ഉള്ളിരുന്ന Azure വിഭവങ്ങൾ ഉപയോഗിക്കൽ

**വിന്യാസം പരിശോധിക്കുക:**

മൂല ഡയറക്ടറിയിൽ `.env` ഫയൽ Azure ക്രെഡൻഷ്യൽസോടെ ഉണ്ടെന്ന് ഉറപ്പാക്കുക (മോഡ്യൂൾ 01 ന്റെ സമയത്ത് സൃഷ്ടിച്ചത്):  
```bash
cat ../.env  # AZURE_OPENAI_ENDPOINT, API_KEY, DEPLOYMENT കാണിക്കണം
```
  
**ആപ്ലിക്കേഷൻ ആരംഭിക്കുക:**

> **നോട്ടീസ്:** മോഡ്യൂൾ 01-ൽ നിന്നുള്ള `./start-all.sh` ഉപയോഗിച്ച് ഇതിനകം എല്ലാ ആപ്ലിക്കേഷനുകളും ആരംഭിച്ചിരിക്കും എങ്കിൽ, ഈ മോഡ്യൂൾ പോർട്ട് 8083-ൽ പ്രവർത്തിക്കുക. താഴെയുള്ള തുടങ്ങൽ കമാൻഡുകൾ ഒഴിവാക്കാം, നേരിട്ട് http://localhost:8083 സന്ദർശിക്കാം.

**ഓപ്ഷൻ 1: Spring Boot ഡാഷ്ബോർഡ് (VS Code ഉപയോക്താക്കൾക്കു ശുപാർത്ഥി)**

ഡെവ് കണ്ടെയ്‌നറിൽ Spring Boot ഡാഷ്ബോർഡ് എക്സ്റ്റൻഷൻ ഉൾപ്പെട്ടിരിക്കുന്നു, എല്ലായ്പ്പോഴും Spring Boot ആപ്ലിക്കേഷനുകൾ മാനേജ് ചെയ്യാനുള്ള ഒരു ദൃശ്യ ഇന്റർഫേസ് നൽകുന്നു. ഇത് VS Code-ന്റെ ഇടത് Activity Bar-ൽ (Spring Boot ഐക്കൺ കാണുക) കാണാവുന്നതാണ്.

ഡാഷ്ബോർഡിൽനിന്ന് നിങ്ങൾക്ക്:  
- വർക്ക്സ്പേസിലുള്ള എല്ലാ Spring Boot ആപ്ലിക്കേഷനുകളും കാണാം  
- single click-ൽ ആപ്ലിക്കേഷനുകൾ ആരംഭിക്കുക/അവസാനിപ്പിക്കുക  
- real-time ലോഗുകൾ കാണുക

## പാറ്റേണുകൾ പരിശോധിക്കുക

വെബ് ഇന്റർഫേസ് prompting തന്ത്രങ്ങൾ വ്യത്യസ്തീകൃതമായി പരീക്ഷിക്കാനാണ് അനുവദിക്കുന്നത്. ഓരോ പാറ്റേണും വ്യത്യസ്ത പ്രശ്നങ്ങൾ പരിഹരിക്കുന്നു - ഏപ്പോൾ ഓരോ സമീപനം ഫലപ്രദമാണെന്ന് കണ്ടെത്താൻ ഇവ പരീക്ഷിക്കുക.

### ലോ vs ഹൈ എഗർനസ്

ലളിതമായ ചോദ്യമായി "200-ൽ 15% എന്താണ്?" ലോ എഗർനസ് ഉപയോഗിച്ച് ചോദിക്കുക. ഉടൻ നേരിട്ടുള്ള മറുപടി ലഭിക്കും. ഇപ്പോൾ "ഉയർന്ന ട്രാഫിക് API ക്ക് caching 전략 രൂപകൽപ്പന ചെയ്‌യുക" എന്ന് ഹൈ എഗർനസ് ഉപയോഗിച്ച് ചോദിക്കുക. മോഡൽ വൈകിയും വിശദമായ reasoning നൽകുന്നതും കാണുക. മോഡൽ തനിക്കു തനിക കൂടാതെ, പ്രോംപ്റ്റിന്റെ ഘടന എങ്ങനെ വേണമെന്ന് പറഞ്ഞ് ചില ചിന്തനത്തിന്റെ വ്യാപ്തി വ്യക്തമാക്കുന്നു.

*അല്പം വിചാരിച്ച് വേഗത്തിലുള്ള കണക്കുകൂട്ടല്‍*

*സമ്പൂര്‍ണ്ണ കാഷിങ്ങ് നയമൊഴുക്കല്‍ (2.8MB)*

### പ്രവൃത്തി നടപ്പാക്കല്‍ (ടൂൾ പ്രീാംബിളുകൾ)

หลายപടിയുള്ള വർക്ക്‌ഫ്ലോകൾ മുമ്പ് പദ്ധതി തയ്യാറാക്കലിനും പുരോഗതി വിവർത്തനത്തിനും സഹായിക്കുന്നു. മോഡല്‍ എന്ത് ചെയ്യും എന്ന് വിശദീകരിക്കുന്നു, ഓരോ ഘട്ടവും വിശദീകരിച്ച്, അവസാനം ഫലങ്ങൾ സംഗ്രഹിക്കുന്നു.

### സ്വയം-പരിശോധന ചെയ്യുന്നു കോഡ്

"ഇമെയിൽ സാധുതാ സേവനം സൃഷ്ടിക്കുക" പരീക്ഷിക്കുക. വെറും കോഡ് സൃഷ്ടിച്ച് നിര്‍ത്തുന്നതിനു പകരം, മോഡൽ നിർമാണം ചെയ്തു, ഗുണനിലവാര മാനദണ്ഡങ്ങളെതിരെ വിലയിരുത്തി, ദുർബലതകൾ കണ്ടെത്തി, മെച്ചപ്പെടുത്തുന്നു. നിങ്ങൾക്ക് കോഡ് നിർമ്മാണം നിർമ്മാണ നിലവാരം വരെ പുനരാവൃതം ചെയ്യുന്നുവെന്ന് കാണാം.

*പൂര്‍ണ ഇമെയിൽ സാധുതാ സേവനം (5.2MB)*

കോഡ് അവലോകനങ്ങൾക്ക് സ്ഥിരമായ വിലയിരുത്തൽ ഘടനകൾ ആവശ്യമാണ്. മോഡൽ നിലനിൽക്കുന്ന വർഗ്ഗങ്ങളിൽ (ശരിയാണോ, രീതികൾ, പ്രവർത്തനം, സുരക്ഷ) ഗൗരവ നിലവാരങ്ങളോടെ കോഡ് വിശകലനം ചെയ്യുന്നു.

### ഘട്ടം-ഘട്ടമായ വിധാനം

ഒരു ഗണിത പ്രശ്‌നം എടുത്ത് ഔദ്യോഗിക ഘട്ടം-ഘട്ടമായ വിധാനത്തിലും, കുറഞ്ഞ ഉത്സാഹത്തിലും പരീക്ഷിക്കുക. കുറഞ്ഞ ഉത്സാഹം വേഗത്തിൽ ഉത്തരം നൽകുന്നു - വേഗം എന്നാൽ വ്യക്തമല്ല. ഘട്ടം-ഘട്ടം നിങ്ങൾക്ക് എല്ലാ കണക്കുകളും തീരുമാനങ്ങളും കാണിക്കുന്നു.

## നിങ്ങൾ യഥാർത്ഥത്തിൽ പഠിക്കുന്നത്

**വിവേക ശ്രമം എല്ലാം മാറ്റിവിടുന്നു**

GPT-5.2 നിങ്ങളുടെ പ്രോപ്പംപ്റ്റുകളിലൂടെ കണക്കായുന്ന ശ്രമം നിയന്ത്രിക്കാന്‍ അനുമതിക്കുന്നു. കുറഞ്ഞ ശ്രമം അൽപമായ അന്വേഷങ്ങളോടുകൂടി വേഗത്തിലുള്ള മറുപടി നൽകുന്നു. ഉയർന്ന ശ്രമം മോഡല്‍ ആഴത്തിൽ ആലോചിക്കാൻ സമയംെടുക്കുന്നു. നിങ്ങൾ പഠിക്കുന്നത് പ്രവൃത്തി സങ്കീർണതയ്ക്കനുസരിച്ച് ശ്രമം പൊരുത്തപ്പെടുത്തുന്നതിനാണ് - സുഗമമായ ചോദ്യങ്ങൾക്ക് സമയം പാഴാക്കരുത്, പക്ഷേ സങ്കീർണ തീരുമാനങ്ങൾ വേഗത്തിൽ തീർക്കരുത്.

**ഘടന പെരുമാറ്റം നയിക്കുന്നു**

പ്രോപ്പംപ്റ്റുകളിൽ XML ടാഗുകൾ കാണുന്നുണ്ടോ? അവ ഭീകരമല്ല. മോഡലുകൾ ഘടനാപരമായ നിർദ്ദേശങ്ങൾ സ്വതന്ത്ര വാചകങ്ങളേക്കാൾ വിശ്വാസ്യതയോടെ പാലിക്കുന്നു. നിങ്ങൾക്ക് ബഹുവിധ ഘട്ട നടപടിക്രമങ്ങൾ അല്ലെങ്കിൽ സങ്കീർണ തർകരണങ്ങൾ ആവശ്യമുള്ളപ്പോള്‍, ഘടന മോഡൽ എന്ത് സ്ഥലത്താണും എന്ത് ചെയ്യാൻ പോകുന്നതും പിന്തുടരുവാൻ സഹായിക്കുന്നു.

<img src="../../../translated_images/ml/prompt-structure.a77763d63f4e2f89.webp" alt="പ്രോപ്പംപ്റ്റ് ഘടന" width="800"/>

*स्पष्ट വിഭാഗങ്ങളോടെ XML-ശൈലിയിൽ സങ്കീർണ്ണമായി ഘടനയ Discipline-ലുള്ള ഒരു പ്രോപ്പംപ്റ്റിന്റെ ഘടന*

**സ്വയം-മൂല്യനിർണയത്തിലൂടെ ഗുണമേൻമ**

സ്വയം-പരിശോധന മാതൃകകൾ ഗുണനിലവാര മാനദണ്ഡങ്ങൾ വ്യക്തമായി ചെയ്തുകൊണ്ട് പ്രവർത്തിക്കുന്നു. മോഡൽ "ശരിയായി ചെയ്യുമെന്ന്" പ്രതീക്ഷിക്കുന്നതിന് പകരം, നിങ്ങൾ "ശരി" എന്നതിന്റെ അർത്ഥം: ശരിയായ ലജിക്ക്, പിശക് കൈകാര്യം, പ്രകടനം, സുരക്ഷ എന്നിവ വ്യക്തമാക്കുന്നു. മോഡൽ അപ്പോള്‍ തന്നെ തന്റെ ഔട്ട്പുട്ട് അനലിസിസ് ചെയ്ത് മെച്ചപ്പെടുത്താൻ കഴിയും. ഇത് കോഡ് നിര്‍മ്മാണത്തെ ലോട്ടറിയിൽ നിന്ന് ഒരു പ്രക്രിയയാക്കി മാറ്റുന്നു.

**പ്രസംഗം പരിധിയുണ്ട്**

ബഹു-ടേൺ സംഭാഷണങ്ങൾ ഓരോ അഭ്യർത്ഥനയോടും സന്ദേശ ചരിത്രം ഉൾപ്പെടുത്തുക വഴി പ്രവർത്തിക്കുന്നു. പക്ഷേ പരിധി ഉണ്ട് - ഓരോ മോഡലിനും പരമാവധി ടോക്കൺ എണ്ണമാണുള്ളത്. സംഭാഷണങ്ങൾ അരികിലെത്തുമ്പോൾ, ആ പരിധി കടത്താതെ പ്രസംഗം പ്രാസക്തമാക്കാനുള്ള തന്ത്രങ്ങൾ ആവശ്യപ്പെടാം. ഈ മോഡ്യൂൾ നിങ്ങളെ ഓർമ്മയുടെ പ്രവർത്തനം അറിയിക്കും; പിന്നീട് സംഗ്രഹിക്കേണ്ടത്, മറക്കേണ്ടത്, തിരികെ കൊണ്ടുവരേണ്ടത് ഏത് സമയമായാണെന്നു നിങ്ങളറിയും.
