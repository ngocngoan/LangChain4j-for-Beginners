## നിങ്ങള്‍ പഠിക്കേണ്ടത്

<img src="../../../translated_images/ml/what-youll-learn.c68269ac048503b2.webp" alt="നിങ്ങള്‍ പഠിക്കേണ്ടത്" width="800"/>

മുന്‍പത്തെ മോഡ്യൂളില്‍, മെമ്മറി എങ്ങനെ സംസാരിക്കുന്ന AIന് സഹായകരമാണെന്ന് കാണുകയും GitHub മോഡലുകള്‍ അടിസ്ഥാന ആശയവിനിമയത്തിനായി ഉപയോഗിക്കുകയും ചെയ്തു. ഇപ്പോള്‍, Azure OpenAIയുടെ GPT-5.2 ഉപയോഗിച്ച് നിങ്ങള്‍ കിടക്കുന്ന ചോദ്യം - പ്രംപ്റ്റുകള്‍ സ്വയം - എങ്ങനെ ചോദിക്കാമെന്നിലേയ്ക്ക് ശ്രദ്ധ കേന്ദ്രീകരിക്കാം. നിങ്ങളുടെ പ്രോമ്പ്റ്റുകള്‍ ഘടിപ്പിക്കുന്ന വിധം ലഭിക്കുന്ന മറുപടികളുടെ ഗുണമേന്മയെ വലിയ തോതില്‍ സ്വാധീനിക്കും. ആദ്യം അടിസ്ഥാന പ്രോമ്പ്റ്റിംഗ് സാങ്കേതികതകള്‍ അവലോകനം ചെയ്യുകയും, ശേഷം GPT-5.2 യുടെ കഴിവുകള്‍ പുളിപ്പിക്കുന്ന എട്ട് അത്യാധുനിക പാറ്റേണുകളിലേക്ക് പ്രവേശിക്കുകയും ചെയ്യും.

നാം GPT-5.2 ഉപയോഗിക്കുകയാണ് കാരണം ഇത് കാരണമാക്കല്‍ നിയന്ത്രണം അവതരിപ്പിക്കുന്നു - ഒരുക്കം നല്‍കുന്നതിനു മുമ്പായി എത്ര ചിന്തിക്കണമെന്ന് മോഡലിന് പറയാവുന്നതാണ്. ഇത് വ്യത്യസ്ത പ്രോമ്പ്റ്റിംഗ് തന്ത്രങ്ങള്‍ വ്യക്തം ആക്കും, ഓരോ സമീപനവും 언제 ഉചിതമാണ് എന്നും മനസിലാക്കാന്‍ സഹായിക്കും. GitHub മോഡല്‍സിനേക്കാള്‍ GPT-5.2 ല്‍ Azure ന്റെ കുറവ് നിരക്ക് പരിധികള്‍ ഉപയോഗിച്ച് നാം ലാഭം ലഭിക്കും.

## പ്രോമ്പ്റ്റ് എഞ്ചിനീയറിങ് മനസിലാക്കല്‍

<img src="../../../translated_images/ml/what-is-prompt-engineering.5c392a228a1f5823.webp" alt="പ്രോമ്പ്റ്റ് എഞ്ചിനീയറിങ്‌ چیست?" width="800"/>

പ്രോമ്പ്റ്റ് എഞ്ചിനീയറിങ് എന്നത് ആവര്‍ത്തിച്ച് നിങ്ങള്‍ക്കു ആവശ്യമായ ഫലങ്ങള്‍ ലഭിക്കാനുള്ള ഇന്‍പുട്ട് ടെക്സ്റ്റ് രൂപകല്‍പ്പന ചെയ്യുന്നതാണ്. അത് വെറുതെ ചോദിക്കുന്നത് മാത്രമല്ല - മോഡല്‍ നിങ്ങള്‍ എന്ത് ആവശ്യമുള്ളതും എങ്ങനെ നല്‍കണമെന്ന് പരിപൂര്‍ണമായി മനസ്സിലാക്കുന്നതിന് ആവശ്യമായ কাঠമരണ конструк उनശി ണ്.

അതെ പോലെ സുഹൃത്ത്‌ക്ക് നിര്‍ദേശം നല്‍കുന്നതായി കരുതുക. "ബഗ് ശരിയാക്കുക" എന്നത് പ്രത്യേകം വ്യക്തമല്ല. "UserService.java ലെ 45-ാം ലൈനിലുള്ള നാള്‌പോയിന്റര്‍ എക്‌സെപ്ഷന്‍ null പരിശോധനം ചേര്‍ത്തുകൊണ്ട് ശരിയാക്കുക" അഥവാ പ്രത്യേകവും വ്യക്തമുമാണ്. ഭാഷ മോഡലുകളും ഇതുപോലെ തന്നെ പ്രവര്‍ത്തിക്കുന്നു - വ്യത്യാസവും ഘടനയും നിര്‍ണായകം.

<img src="../../../translated_images/ml/how-langchain4j-fits.dfff4b0aa5f7812d.webp" alt="LangChain4j എങ്ങനെ ഇന്‍റഗ്രേറ്റ് ചെയ്യുന്നു" width="800"/>

LangChain4j മോഡല്‍ കണക്ഷന്‍സ്, മെമ്മറി, സന്ദേശ തരം എന്നിവയുമായി ബന്ധമുള്ള സംരംഭം നല്‍കുന്നു - പ്രോമ്പ്റ്റ് പാറ്റേണുകള്‍ക്ക് അത് വഴി അയക്കുന്ന സൂക്ഷ്മമായി ഘടിപ്പിച്ച ടെക്സ്റ്റ് മാത്രം ആണ്. പ്രധാന ഘടകങ്ങള്‍ `SystemMessage` (AIയുടെ പെരുമാറ്റവും വേഷവുമെല്ലാം ക്രമീകരിക്കുന്നു)യും `UserMessage` (നിങ്ങളുടെ യഥാര്‍ത്ഥ അഭ്യര്‍ത്ഥന കൈവരിക്കുന്നതു)യുമാണ്.

## പ്രോമ്പ്റ്റ് എഞ്ചിനീയറിങ് അടിസ്ഥാനങ്ങള്‍

<img src="../../../translated_images/ml/five-patterns-overview.160f35045ffd2a94.webp" alt="അഞ്ചു പ്രോമ്പ്റ്റ് എഞ്ചിനീയറിങ് പാറ്റേണുകളുടെ അവലോകനം" width="800"/>

ഈ മോഡ്യൂളില്‍ അത്യാധുനിക പാറ്റേണുകള്‍ ഏറ്റെടുക്കുന്നതിനു മുമ്പ്, അഞ്ചു അടിസ്ഥാന പ്രോമ്പ്റ്റിംഗ് സാങ്കേതികങ്ങളിലേക്ക് ഒരു അവലോകനം നടത്താം. ഓരോ പ്രോമ്പ്റ്റ് എഞ്ചിനീയറിന്റെയും അറിയേണ്ട പ്രധാന ഗണണമാണിവ. നിങ്ങൾ ഇതിനകം [ക്വിക്ക് സ്റ്റാർട്ട് മോഡ്യൂള്‍](../00-quick-start/README.md#2-prompt-patterns) കാണിച്ചുള്ളതാണ്, അവയുടെ ആശയരൂപത്തിലാണ് ഇവിടെ വിശദീകരണം.

### സീറോ-ഷോട്ട് പ്രോമ്പ്റ്റിംഗ്

ഏറ്റവും ലളിതമായ സമീപനം: ഉദാഹരണങ്ങളില്ലാതെ നേരിട്ട് മോഡലിന് നിര്‍ദേശം നല്‍കുക. മോഡല്‍ അതിന്റെ പരിശീലന ലളിതമായ ശീലങ്ങള്‍ ആശ്രയിച്ചു ടാസ്‌ക് മനസ്സിലാക്കി നടപ്പിലാക്കും. പൊതുവെ പ്രതീക്ഷിക്കുന്ന പ്രവർത്തി വ്യക്തമായ സെട്ടിങുകളില്‍ ഇത് നന്നായി പ്രവര്‍ത്തിക്കും.

<img src="../../../translated_images/ml/zero-shot-prompting.7abc24228be84e6c.webp" alt="സീറോ-ഷോട്ട് പ്രോമ്പ്റ്റിംഗ്" width="800"/>

*ഉദാഹരണങ്ങളില്ലാത്ത നേരിട്ടുള്ള നിര്‍ദേശം — മോഡല്‍ നിര്‍ദേശത്തിലെ മൂടിപ്പ് നിന്ന് ടാസ്‌ക് കണ്ടെത്തുന്നു*

```java
String prompt = "Classify this sentiment: 'I absolutely loved the movie!'";
String response = model.chat(prompt);
// പ്രതികരണം: "സकारാത്മകം"
```
  
**എപ്പോൾ ഉപയോഗിക്കാം:** ലളിതമായ ക്ലാസിഫിക്കേഷനുകള്‍, നേരിട്ട് ചോദ്യങ്ങള്‍, വിവര്‍ത്തനം, അല്ലെങ്കിൽ അധിക മാര്‍ഗ്ഗനിര്‍ദേശം ആവശ്യമില്ലാത്ത ഏത് ടാസ്‌കും.

### ഫ്യു-ഷോട്ട് പ്രോമ്പ്റ്റിംഗ്

നിങ്ങള്‍ക്ക് ആവശ്യമായ മാതൃക മോഡല്‍ പിന്തുടരാന്‍ സേവകം ചെയ്യുന്ന ഉദാഹരണങ്ങള്‍ കൊടുക്കുക. മോഡല്‍ അതില്‍ നിന്നു എന്‍റ് പോട്ടേ‍ർന്‍ മനസ്സിലാക്കി പുതിയ ഇന്‍പുട്ടുകളിലേക്ക് ഉപയോഗിക്കും. ഇത് ആവശ്യമായ ഫോം അല്ലെങ്കില്‍ പെരുമാറ്റം കണ്ടെത്തികൂടാത്ത ടാസ്‍ക്കുകളില്‍ സ്ഥിരത വളരെ മെച്ചപ്പെടുത്തും.

<img src="../../../translated_images/ml/few-shot-prompting.9d9eace1da88989a.webp" alt="ഫ്യു-ഷോട്ട് പ്രോമ്പ്റ്റിംഗ്" width="800"/>

*ഉദാഹരണങ്ങളിലൂടെ പഠിക്കല്‍ — മോഡല്‍ മാതൃക തിരിച്ചറിയുകയും പുതിയ ഇന്‍പുട്ടിലാണ് ബാധകമാക്കുകയും ചെയ്യുന്നു*

```java
String prompt = """
    Classify the sentiment as positive, negative, or neutral.
    
    Examples:
    Text: "This product exceeded my expectations!" → Positive
    Text: "It's okay, nothing special." → Neutral
    Text: "Waste of money, very disappointed." → Negative
    
    Now classify this:
    Text: "Best purchase I've made all year!"
    """;
String response = model.chat(prompt);
```
  
**എപ്പോള്‍ ഉപയോഗിക്കാം:** കസ്റ്റം ക്ക്ലാസിഫിക്കേഷനുകൾ, സ്ഥിരമായ ഫോര്മാറ്റിംഗ്, ഡൊമെയ്ൻ-പ്രത്യേക ടാസ്‌കുകള്‍, അല്ലെങ്കിൽ സീറോ-ഷോട്ട് ഫലം സുസ്ഥിരമല്ലാത്ത അവസ്ഥകള്‍.

### ചെയിൻ ഓഫ് തോത്ത്

മോഡലില്‍ തമുഖ്യം അവിയുടെ കാര്യങ്ങള്‍ ചുവടു ചുവടായി കാണിച്ച് നല്‍കാന്‍ ചോദിക്കുക. നേരിട്ട് ഉത്തരം പറയാതെ മോഡല്‍ പ്രശ്നം വിഭജിച്ചു അവ്യക്തമായി ഓരോ ഘട്ടവും വിശദീകരിക്കുന്നു. ഇത് ഗണിതം, തര്‍ക്കം, ബഹു ഘട്ട കാരണമാക്കല്‍ ടാസ്‌കുകള്‍ക്കായുള്ള കൃത്യത കൂട്ടുന്നു.

<img src="../../../translated_images/ml/chain-of-thought.5cff6630e2657e2a.webp" alt="ചെയിൻ ഓഫ് തോത്ത് പ്രോമ്പ്റ്റിംഗ്" width="800"/>

*ചുവടു ചുവടായുള്ള കാരണമാക്കല്‍ — സങ്കീര്‍ണ പ്രശ്നങ്ങള്‍ വ്യക്തമായ ലൊജിക് ഘട്ടങ്ങളില്‍ വിഭജിക്കുന്നതു*

```java
String prompt = """
    Problem: A store has 15 apples. They sell 8 apples and then 
    receive a shipment of 12 more apples. How many apples do they have now?
    
    Let's solve this step-by-step:
    """;
String response = model.chat(prompt);
// മോഡല്‍ കാണിക്കുന്നത്: 15 - 8 = 7, പിന്നെ 7 + 12 = 19 ആപ്പിളുകള്‍
```
  
**എപ്പോള്‍ ഉപയോഗിക്കാം:** ഗണിത പ്രശ്നങ്ങള്‍, തര്‍ക്ക പസిలുകള്‍, ഡീബഗ്ഗിംഗ്, അഥവാ കാരണമാക്കല്‍ പ്രക്രിയ കാണിക്കുന്നത് വിശ്വാസ്യതയും കൃത്യതയും കൂട്ടിയിടಿಸುವ ടാസ്‌കുകള്‍.

### റോള്‍ അടിസ്ഥാന പ്രോമ്പ്റ്റിംഗ്

നിങ്ങളുടെ ചോദ്യത്തിന് മുമ്പ് AI ന് ഒരു വ്യക്തിത്വം അല്ലെങ്കില്‍ വേഷം നിശ്ചയിക്കുക. ഇത് മറുപടി രീതി, ആഴം, കേന്ദ്രം എന്നിവ നിര്‍ണ്ണയിക്കുന്ന കേസ് നല്‍കുന്നു. "സോഫ്റ്റ്‍വെയര്‍ ആര്‍ക്കിടെക്ട്" ഒരു "ജൂനിയര്‍ ഡെവലപ്പര്‍" അല്ലെങ്കില്‍ "സുരക്ഷാ ഓഡിറ്റര്‍" ക്കായി വ്യത്യസ്ത ഉപദേശങ്ങള്‍ നല്‍കും.

<img src="../../../translated_images/ml/role-based-prompting.a806e1a73de6e3a4.webp" alt="റോൾ അടിസ്ഥാന പ്രോമ്പ്റ്റിംഗ്" width="800"/>

*പ്രസ്‌താവനയും വ്യക്തിത്വവും ക്രമീകരിച്ച് — ഒരേ ചോദ്യം വ്യത്യസ്ത പരിഹാരം കൈവരിക്കുന്നു വ്യത്യസ്ഥ വേഷമേല്‍ക്കുന്നതിന്*

```java
String prompt = """
    You are an experienced software architect reviewing code.
    Provide a brief code review for this function:
    
    def calculate_total(items):
        total = 0
        for item in items:
            total = total + item['price']
        return total
    """;
String response = model.chat(prompt);
```
  
**എപ്പോള്‍ ഉപയോഗിക്കാം:** കോഡ് റിവ്യൂസ്‌, ട്യൂട്ടറിംഗ്, ഡൊമെയ്ൻ-സ്പെസിфик അനാലിസിസ്, അല്ലെങ്കില്‍ പ്രത്യേക പരിചയ തലമുറപ്പാട് ആവശ്യമായ മറുപടികള്‍ക്ക്.

### പ്രോമ്പ്റ്റ് ടെംപ്ലേറ്റുകൾ

വ്യത്യസ്ത മൂല്യങ്ങള്‍ പ്ലേസ്‌ഹോള്‍ഡറുകളുമായി പുനരുപയോഗിക്കാവുന്ന പ്രോമ്പ്റ്റുകള്‍ സൃഷ്ടിക്കുക. ഓരോ തവണ പുതിയ പ്രോമ്പ്റ്റ് എഴുതാതെ ഒരു ടെംപ്ലേറ്റ് ഒരു തവണ നിര്‍മ്മിച്ച് വ്യത്യസ്ത മൂല്യങ്ങള്‍ പൂരിപ്പിക്കുക. LangChain4j ന്റെ `PromptTemplate` ക്ലാസ്സ് `{{variable}}` വിഭവം ഉപയോഗിച്ച് ഇത് എളുപ്പമാണ്.

<img src="../../../translated_images/ml/prompt-templates.14bfc37d45f1a933.webp" alt="പ്രോമ്പ്റ്റ് ടെംപ്ലേറ്റുകൾ" width="800"/>

*വ്യത്യസ്ത മൂല്യങ്ങളുമായുള്ള പുനരുപയോഗ പ്രോമ്പ്റ്റുകള്‍ — ഒരു ടെംപ്ലേറ്റ്, പല ഉപയോഗങ്ങള്‍*

```java
PromptTemplate template = PromptTemplate.from(
    "What's the best time to visit {{destination}} for {{activity}}?"
);

Prompt prompt = template.apply(Map.of(
    "destination", "Paris",
    "activity", "sightseeing"
));

String response = model.chat(prompt.text());
```
  
**എപ്പോള്‍ ഉപയോഗിക്കാം:** വ്യത്യസ്ത ഇന്‍പുട്ടുകളോടെ ആവര്‍ത്തിച്ച് ചോദിക്കുമ്പോള്‍, ബാച്ച് പ്രോസസ്സിങ്ങ്, പുനരുപയോഗ AI പ്രവൃത്തി പ്രക്രിയകള്‍ നിർമ്മിക്കുമ്പോള്‍, അല്ലെങ്കില്‍ പ്രോമ്പ്റ്റ് ഘടന മാറ്റമില്ലാതെ ഡാറ്റ മാറുന്ന ഘട്ടങ്ങളില്‍.

---

ഈ അഞ്ചു അടിസ്ഥാനങ്ങള്‍ അധികഭാഗം പ്രോമ്പ്റ്റിംഗ് ടാസ്‌കുകള്‍ക്കായി ഒരു ഉറച്ച ഉപകരണപ്പെട്ടി നല്‍കുന്നു. ഈ മോഡ്യൂളിലെ ശേഷവും **എട്ട് അത്യാധുനിക പാറ്റേണുകള്‍** ആണ് GPT-5.2 വിന്റെ കാരണമാക്കല്‍, സ്വയം വിലയിരുത്തല്‍, ഘടിത ഔട്ട്പുട്ട് കഴിവുകള്‍ ഉപയോഗിക്കുന്നത്.

## അത്യാധുനിക പാറ്റേണുകള്‍

അടിസ്ഥാനങ്ങള്‍ കവർച്ച ചെയ്ത ശേഷം, ഈ മോഡ്യൂള്‍ വ്യത്യസ്തമാക്കുന്ന എട്ട് അത്യാധുനിക പാറ്റേണുകളിലേക്ക് നമുക്ക് മുന്നോട്ട് പോകാം. എല്ലാപ്രശ്നങ്ങള്‍ക്കും ഒരേ സമീപനം ആവശ്യമായിരിക്കുന്നുവെന്നും അല്ല. ചില ചോദ്യങ്ങള്‍ക്ക് വേഗത്തിലുള്ള മറുപടി, മറ്റുള്ളവര്‍ക്ക് ആഴത്തിലുള്ള ചിന്ത ആവശ്യമുണ്ട്. ചിലത് ദൃശ്യമായ കാരണമാക്കലും ചിലത് ഫലം മാത്രം ആവശ്യമാണ്. ഓരോ പാറ്റേണും വ്യത്യസ്ത സാഹചര്യത്തിന് വേണ്ടിയാണ് അനുയോജ്യം — GPT-5.2 യുടെ കാരണമാക്കല്‍ നിയന്ത്രണം വ്യത്യാസങ്ങള്‍ കൂടുതല്‍ തെളിയിക്കുന്നു.

<img src="../../../translated_images/ml/eight-patterns.fa1ebfdf16f71e9a.webp" alt="എട്ട് പ്രോമ്പ്റ്റിംഗ് പാറ്റേണുകള്‍" width="800"/>

*എട്ട് പ്രോമ്പ്റ്റ് എഞ്ചിനീയറിങ് പാറ്റേണുകളും അവയുടെ ഉപയോഗ സന്ധര്‍ഭങ്ങളും അവലോകനം*

<img src="../../../translated_images/ml/reasoning-control.5cf85f0fc1d0c1f3.webp" alt="GPT-5.2 ഉപയോഗിച്ച് കാരണമാക്കല്‍ നിയന്ത്രണം" width="800"/>

*GPT-5.2 കാരണമാക്കല്‍ നിയന്ത്രണം മോഡല്‍ എത്ര ചിന്തിക്കണം എന്ന് നിശ്ചയിക്കാമാകും — വേഗം ഉള്ള നേരിട്ടുള്ള മറുപടിയ്ക്ക് നിന്നും ആഴത്തിലുള്ള അന്വേഷത്തിനും*

**കുറഞ്ഞ മനസാര്ഹത (വേഗവും കേന്ദ്രീകൃതവുമുള്ളത്)** - എളുപ്പത്തിലുള്ള ചോദ്യങ്ങള്‍ക്ക് വേഗത്തിലുള്ള, നേരിട്ട് ഉത്തരം വേണ്ടതിന്. മോഡല്‍ ശരിയായ തരണം ചിന്തിക്കാറില്ല - പരമാവധി 2 ഘട്ടം. കണക്കുകള്‍, തിരയല്‍, അല്ലെങ്കില്‍ ലളിത ചോദ്യങ്ങള്‍ക്ക് ഉപയോഗിക്കുക.

```java
String prompt = """
    <context_gathering>
    - Search depth: very low
    - Bias strongly towards providing a correct answer as quickly as possible
    - Usually, this means an absolute maximum of 2 reasoning steps
    - If you think you need more time, state what you know and what's uncertain
    </context_gathering>
    
    Problem: What is 15% of 200?
    
    Provide your answer:
    """;

String response = chatModel.chat(prompt);
```

> - "പ്രോമ്പ്റ്റുകളില്‍ XML ടാഗുകള്‍ എങ്ങനെ AIയുടെ മറുപടിയുടെ ഘടന സഹായിക്കുന്നു?"  
> - "സ്വയം-പരിശോധന പാറ്റേണുകളും നേരിട്ടുള്ള നിര്‍ദേശവും എപ്പോള്‍ ഉപയോഗിക്കണം?"

**ഉയർന്ന മനസാര്ഹത (ആഴവും സമഗ്രവും)** - സങ്കീര്‍ണ്ണ പ്രശ്നങ്ങള്‍ക്കായി സമഗ്ര വിശകലനം വേണ്ടെങ്കില്‍. മോഡല്‍ ആഴത്തില്‍ അന്വേഷിച്ച് വിശദമായ കാരണമാക്കല്‍ കാണിക്കുന്നു. സിസ്റ്റം ഡിസൈന്‍, ആര്‍ക്കിടെക്ചര്‍ തീരുമാനങ്ങള്‍, അല്ലെങ്കില്‍ സങ്കീര്‍ണ ഗവേഷണത്തിലാണെങ്കിൽ ഇത് ഉപയോഗിക്കുക.

```java
String prompt = """
    Analyze this problem thoroughly and provide a comprehensive solution.
    Consider multiple approaches, trade-offs, and important details.
    Show your analysis and reasoning in your response.
    
    Problem: Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```
  
**ടാസ്‌ക് എക്സിക്യൂഷന്‍ (ചുവടു ചുവടായി പുരോഗതി)** - ബഹു ഘട്ട പ്രവര്‍ത്തനങ്ങള്‍ക്കായി. മോഡല്‍ ആദ്യത്തെ പദ്ധതി നല്‍കുകയും, ഓരോ ഘട്ടവും നാരേറ്റു ചെയ്യുകയും, തുടര്‍ന്ന് ഒരു സംക്ഷേപം കൊടുക്കുകയും ചെയ്യും. മൈഗ്രേഷനുകള്‍, നടപ്പിലാക്കലുകള്‍, അല്ലെങ്കില്‍ ഏത് বহু ഘട്ട പ്രക്രിയയുണ്ടെങ്കില്‍ ഇത് ഉപയോഗിക്കുക.

```java
String prompt = """
    <task_execution>
    1. First, briefly restate the user's goal in a friendly way
    
    2. Create a step-by-step plan:
       - List all steps needed
       - Identify potential challenges
       - Outline success criteria
    
    3. Execute each step:
       - Narrate what you're doing
       - Show progress clearly
       - Handle any issues that arise
    
    4. Summarize:
       - What was completed
       - Any important notes
       - Next steps if applicable
    </task_execution>
    
    <tool_preambles>
    - Always begin by rephrasing the user's goal clearly
    - Outline your plan before executing
    - Narrate each step as you go
    - Finish with a distinct summary
    </tool_preambles>
    
    Task: Create a REST endpoint for user registration
    
    Begin execution:
    """;

String response = chatModel.chat(prompt);
```

## പാറ്റേണുകൾ പരിശോധിക്കൽ

വെബ് ഇന്റർഫേസിലൂടെ വിവിധ പ്രോമ്പ്റ്റിംഗ് തന്ത്രങ്ങളെ പരീക്ഷിക്കാമെന്ന് കഴിയും. ഓരോ പാറ്റേൺനും വ്യത്യസ്ത പ്രശ്‌നങ്ങൾ പരിഹരിക്കുന്നു - ഓരോ സമീപനവും എപ്പോൾ മികച്ചതായി പ്രവർത്തിക്കുന്നു എന്ന് നോക്കുക.

> **കുറിപ്പ്: Streaming vs Non-Streaming** — ഓരോ പാറ്റേൺ പേജിലും രണ്ടു ബട്ടണുകൾ ഉണ്ട്: **🔴 Stream Response (Live)** കൂടാതെ ഒരു **Non-streaming** ഓപ്ഷൻ. Streaming Server-Sent Events (SSE) ഉപയോഗിച്ച് മോഡൽ ടോക്കൺ തയ്യാറാക്കുന്നതിനിടെ തൽക്ഷണമായി കാണിക്കുന്നു, അതിനാൽ താൽക്ഷണം പുരോഗതി കാണാം. Non-streaming ഓപ്ഷൻ മുഴുവൻ പ്രതികരണം വരെയായി കാത്തിരിക്കും. (ഉദാ: High Eagerness, Self-Reflecting Code പോലുള്ള ദീർഘമായ ചിന്ത ആവശ്യമായ പ്രോമ്പ്റ്റുകളിൽ) Non-streaming വിളി വളരെ നീണ്ട സമയം (മിനിറ്റുകൾ) ആരംഭിക്കാൻ എങ്ങനെ പരിശ്രാമം ഇല്ലാതെ കാണാം. **സങ്കീർണ്ണ പ്രോംപ്റ്റുകളുമായി പരീക്ഷിക്കുമ്പോൾ streaming ഉപയോഗിക്കുക** അതിനാൽ മോഡൽ പ്രവർത്തിക്കുന്നത് നിങ്ങളക്ഷണികമായി കാണുകയും അഭ്യർത്ഥന സമയംവിട്ടെന്ന തെറ്റിദ്ധാരണ ഒഴിവാക്കുകയും ചെയ്യാം.
>
> **കുറിപ്പ്: ബ്രൗസർ ആവശ്യകത** — Streaming സവിശേഷത Fetch Streams API (`response.body.getReader()`) ഉപയോഗിക്കുന്നു, ഇത് പൂർണ ബ്രൗസർ (Chrome, Edge, Firefox, Safari) ആവശ്യമാണ്. ഇത് VS Code ന്റെ ഉള്ളിൽ ഉള്ള Simple Browser-ൽ പ്രവർത്തിക്കില്ല, കാരണം അതിന്റെ webview ReadableStream API പിന്തുണയ്ക്കുന്നില്ല. Simple Browser ഉപയോഗിച്ചാൽ non-streaming ബട്ടണുകൾ സാധാരണ പ്രവർത്തിക്കും — streaming ബട്ടണുകൾ മാത്രം ബാധിക്കപ്പെടും. പൂർണാനുഭവത്തിനായി `http://localhost:8083` വർഷം ബാഹ്യ ബ്രൗസറിൽ തുറക്കുക.

### താഴ്ന്നതും ഉയർന്ന Eagerness

സാധാരണ ചോദ്യം പോലെ "200-ന്റെ 15% എന്താണ്?" എന്നത് താഴ്ന്ന Eagerness ഉപയോഗിച്ച് ചോദിക്കുക. നിങ്ങൾക്ക് ഉടൻ നേരിട്ട് ഉത്തരം ലഭിക്കും. ഇപ്പോൾ "ഉയർന്ന-യാത്രയുള്ള APIയ്ക്ക് caching തന്ത്രം രൂപകൽപ്പന ചെയ്യുക" എന്നക്കുറിച്ച് ഉയർന്ന Eagerness ഉപയോഗിച്ച് ചോദിക്കുക. **🔴 Stream Response (Live)** ക്ലിക്ക് ചെയ്ത് മോഡലിന്റെ വിശദമായ ലജിക്സ് ടോക്കൺ-മട്ടായി കാണുക. ഒരേ മോഡൽ, ഒരേ ചോദ്യ ഘടന - പക്ഷേ പ്രോംപ്റ്റ് എത്ര ചിന്തിക്കാമെന്ന് പറയുന്നു.

### ടാസ്ക് നിർവഹണം (ടൂൾ പ്രീambles)

മെൾക്കൊണ്ട പ്രവൃത്തി流ുകള് ഉന്നതമായ പരിപാടിയും പുരോഗതി ശബ്ദവുമുണ്ട്. മോഡൽ എന്ത് ചെയ്യുമെന്നു വിശദീകരിക്കുന്നു, ഓരോ ഘട്ടവും വിശദീകരിക്കുന്നു, പിന്നീട് ഫലം സംഗ്രഹിക്കുന്നു.

### സ്വയം-പരിശോധന കോഡ്

"ഒരു ഇമെയിൽ സാധൂകരിക്കൽ സേവനം സൃഷ്ടിക്കുക" പരീക്ഷിക്കുക. പതിവായി കോഡ് മാത്രം നിർമ്മിച്ച് നിർത്തുന്നതിന് പകരം, മോഡൽ കോഡ് നിർമ്മിക്കുന്നു, ഗുണനിലവാര മാനദണ്ഡങ്ങൾക്കെതിരെ വിലയിരുത്തുന്നു, ദുർബലതകൾ തിരിച്ചറക്കുന്നു, മെച്ചപ്പെടുത്തുന്നു. കോഡ് ഉൽപ്പാദന നിലവാരത്തിലെത്തിത്തുടങ്ങിയാൽ നിങ്ങൾ അതിന്റെ ആവർത്തനം കാണും.

### ഘടനാപരമായ വിശകലനം

കോഡ് അവലോകനത്തിന് സ്ഥിരമായ മൂല്യനിർണയ ഘടന ആവശ്യമാണ്. മോഡൽ കോഡ് വിശകലനം ചെയ്യുന്നു സ്ഥിരം തരങ്ങൾ (തെറ്റില്ലായ്മ, പ്രാധാന്യങ്ങൾ, പ്രകടനം, സുരക്ഷ)PLICATION = "" പ്രകടന നിരക്കുകൾ ഉപയോഗിച്ച്.

### മൾട്ടി-ടേൺ ചാറ്റ്

"Spring Boot എന്താണ്?" എന്ന് ചോദിച്ച് ഉടൻ "ഒരു ഉദാഹരണം കാണിക്കുക" എന്നും ചോദിക്കുക. മോഡൽ നിങ്ങളുടെ ആദ്യത്തെ ചോദ്യമായ്ക് ഓർമ്മിക്കുന്നു, പ്രത്യേകിച്ച് Spring Boot ഉദാഹരണം നൽകുന്നു. ഓർമ്മശേഷിയില്ലാതെ, രണ്ടാം ചോദ്യമ വളരെ അസ്പഷ്ടമായിരിക്കും.

### പടി-പടി കരുതൽ

ഒരു ഗണിത പ്രശ്നം തിരഞ്ഞെടുക്കൂ, അതിനെ Step-by-Step Reasoning ഉം Low Eagerness ഉം ഉപയോഗിച്ച് പരീക്ഷിച്ചു നോക്കൂ. താഴ്ന്ന Eagerness യഥാർത്ഥ ഉത്തരം വേഗം നൽകും, പക്ഷേ അനാകൃതമായി. പടി-പടി കരുതൽ ഓരോ കണക്കും തീരുമാനം കാണിക്കും.

### നിയന്ത്രിത ഔട്ട്പുട്ട്

നിർദ്ദിഷ്ട ഫോർമാറ്റുകൾ അല്ലെങ്കിൽ വാക്ക് എണ്ണങ്ങൾ ആവശ്യമായപ്പോഴുള്ള പാറ്റേൺ കർശനമായ അനുസരണ ഉറപ്പാക്കുന്നു. 100 വാക്കുകളിൽ നിന്ന് അടയാളപ്പെടുത്തൽ രൂപത്തിൽ ഒരു സംഗ്രഹം സൃഷ്ടിക്കാൻ ശ്രമിക്കുക.

## നിങ്ങൾ എന്താണ് ശരിയാക്കിയിട്ടുള്ളത്

**കരുതൽ ശ്രമം എല്ലാ മാറ്റങ്ങളും വരുത്തുന്നു**

GPT-5.2 നിങ്ങളുടെ പ്രോംപ്റ്റുകൾ വഴി കണക്കു ശ്രമം നിയന്ത്രിക്കുവാൻ അനുവദിക്കുന്നു. താഴ്ന്ന ശ്രമം നേർവശപ്പെട്ട ഉത്തരം വേഗത്തിൽ നൽകുന്നു. ഉയർന്ന ശ്രമം മോഡലിന് ദീർഘമായി ചിന്തിക്കാൻ സമയം സ്വീകരിക്കുന്നു. നിങ്ങൾ প্রচോദനത്തെ പ്രവർത്തി സങ്കീർണ്ണതയ്ക്ക് അനുയോജ്യമായി അളക്കുന്നത് പഠിക്കുന്നു - ലളിത ചോദ്യങ്ങളിൽ സമയം വഹിക്കരുത്, എന്നാൽ സംഘട്ടനങ്ങൾക്കും വൈകാതെ തീരുമാനിക്കേണ്ടതുമില്ല.

**ഘടന മനസ്സിൽ വെക്കുന്നു**

പ്രോംപ്റ്റുകളിൽ XML ടാഗുകൾ ശ്രദ്ധിച്ചിട്ടുണ്ടോ? അവ അലങ്കാരമല്ല. മോഡലുകൾ ഘടനാപരമായ നിർദ്ദേശങ്ങൾ സ്വതന്ത്ര പാഠത്തിൽ നിന്ന് കൂടുതൽ വിശ്വാസപ്രദമായ രീതിയിൽ പിന്തുടരുന്നു. നിങ്ങൾക്ക് മൾട്ടി-പടി പ്രക്രിയകൾ അല്ലെങ്കിൽ സങ്കീർണ്ണ തർക്കങ്ങൾ ആവശ്യമാണെങ്കിൽ, ഘടന മോഡലിന് അവിടെ എന്ത് ചെയ്യണമെന്നു പെരുമാറ്റാൻ സഹായിക്കുന്നു.

<img src="../../../translated_images/ml/prompt-structure.a77763d63f4e2f89.webp" alt="Prompt Structure" width="800"/>

*സ്പഷ്ട വിഭാഗങ്ങൾക്കും XML-ശൈലിയിലെ ഘടനയ്‌ക്കുമായുള്ള നല്ലഘടന ഉള്ള പ്രോംപ്റ്റിന്റെ ഘടന*

**സ്വയം-വിലയിരുത്തലിലൂടെ ഗുണമേന്മ**

സ്വയം-പരിശോധന പാറ്റേണുകൾ ഗുണനിലവാര മാനദണ്ഡങ്ങൾ വിശദമാക്കുക വഴി പ്രവർത്തിക്കുന്നു. മോഡൽ "നന്നായി ചെയ്തു" എന്ന ആശയിൽ ആശ്രയിക്കുന്നതിന് പകരം, നിങ്ങൾ അതിന്റെ അർഥം വ്യക്തമാക്കുന്നു: ശരിയായ ലജിക്, പിഴവ് കൈകാര്യം, പ്രകടനം, സുരക്ഷ. മോഡൽ പിന്നീട് തൻ്റെ ഔട്ട്പുട്ട് മൂല്യനിർണയം ചെയ്ത് മെച്ചപ്പെടുത്താം. ഇത് കോഡ് നിർമ്മാണത്തെ ലോട്ടറിയിൽ നിന്നൊരു പ്രക്രിയയാക്കി മാറ്റുന്നു.

**സന്ദർഭം പരിധിയുള്ളതാണ്**

മൾട്ടി-ടേൺ സംഭാഷണം ഓരോ അഭ്യർത്ഥനയോടൊപ്പം സന്ദേശ ചരിത്രം ഉൾപ്പെടുത്തിയും പ്രവർത്തിക്കുന്നു. പക്ഷേ പരിധിയുണ്ട് - ഓരോ മോഡലിനും പരമാവധി ടോക്കൺ സ്ഥലം ഉണ്ട്. സംഭാഷണങ്ങൾ വലിയതാകുമ്പോൾ, പ്രാധാന്യമുള്ള സന്ദർഭം നിലനിർത്താനും സെല്ലിംഗ് മുറുകാതെ സൂക്ഷിക്കുന്നതിനു തന്ത്രങ്ങൾ ആവശ്യമാകും. ഈ മോഡ്യൂൾ മെമ്മറി എങ്ങനെ പ്രവർത്തിക്കുന്നു എന്ന് കാണിക്കുന്നു; പിന്നീട് സംഗ്രഹിക്കുക, മറക്കുക, തിരിച്ച് പ്രാപിക്കുക ഏപ്പോൾ ചെയ്യണമെന്നു പഠിപ്പിക്കും.
