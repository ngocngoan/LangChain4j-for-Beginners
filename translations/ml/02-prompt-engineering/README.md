## നിങ്ങൾ പഠിക്കാനുള്ളത്

<img src="../../../translated_images/ml/what-youll-learn.c68269ac048503b2.webp" alt="നിങ്ങൾ പഠിക്കാനുള്ളത്" width="800"/>

മുമ്പത്തെ മോഡ്യൂളിൽ, മെമ്മറി സംവാദ എഐയെ എങ്ങനെ സജീവമാക്കുന്നുവെന്ന് നിങ്ങൾ കണ്ടു, ഗിറ്റ്ഹബ് മോഡലുകൾ ഉപയോഗിച്ച് അടിസ്ഥാന अन्तरസംബന്ധങ്ങൾ നടത്തുകയും ചെയ്തു. ഇപ്പോൾ നിങ്ങൾ ചോദ്യങ്ങൾ എങ്ങനെയാണ് ചോദിക്കാൻ സംസാരിക്കുന്നത് — പ്രോംപ്റ്റുകളാണ് — അതിൽ ശ്രദ്ധ കേന്ദ്രീകരിക്കുകയും Azure OpenAI ന്റെ GPT-5.2 ഉപയോഗിച്ച് ഈ പ്രോംപ്റ്റുകൾ എങ്ങനെയാണ് ഘടിപ്പിക്കേണ്ടത് എന്നും പഠിക്കുന്ന സമയമാണ്. നിങ്ങളുടെ പ്രോംപ്റ്റുകളെ ഘടിപ്പിക്കുന്ന രീതിയാണ് നിങ്ങൾക്ക് ലഭിക്കുന്ന പ്രതികരണങ്ങളുടെ ഗുണനിലവ് നൂറു പേരും ബാധിക്കുന്നത്. നാം പ്രാരംഭ prompting സാങ്കേതികവിദ്യകൾ ആവലോകനം ചെയ്ത് തുടങ്ങുകയും പിന്നീട് GPT-5.2 ന്റെ കഴിവുകൾ പൂർണ്ണമായി ഉപയോഗപ്പെടുത്തുന്ന എട്ടു ഉന്നത മാതൃകകളിലേക്ക് നീങ്ങുകയും ചെയ്യുന്നു.

Reasoning control എന്നത് കെട്ടിച്ചേർക്കുകയാണ് GPT-5.2-യെ ഉപയോഗിക്കാനുള്ള കാരണം — ഉത്തരം നൽകുന്നതിനു മുമ്പ് മോഡലിന 얼마나 ചിന്തിക്കണമെന്ന് നിങ്ങൾക്ക് പറഞ്ഞുതരാന് സാധിക്കും. ഇത് prompting തന്ത്രങ്ങൾ കൂടുതൽ വ്യക്തതയോടെ പിൻതിരിയുകയും, ഏത് സമീപനം ഉപയോഗിക്കേണ്ടതെന്ന് നിങ്ങൾക്കറിയുവാൻ സഹായിക്കുകയും ചെയ്യും. GitHub മോഡലിനെ അപേക്ഷിച്ച് Azure ന്റെ കുറച്ചും ഭാരഹീനമായ റേറ്റ് നിയന്ത്രണങ്ങൾക്കും നാം പ്രയോജനപ്പെടും.

## പ്രോംപ്റ്റ് എൻജിനീയറിംഗ് മനസിലാക്കുക

<img src="../../../translated_images/ml/what-is-prompt-engineering.5c392a228a1f5823.webp" alt="പ്രോംപ്റ്റ് എൻജിനീയറിംഗ് എന്താണ്?" width="800"/>

പ്രോംപ്റ്റ് എൻജിനീയറിംഗ്란 consistently നിങ്ങൾ ആവശ്യപ്പെട്ട ഫലങ്ങൾമാത്രം നൽകുന്ന ഇൻപുട്ട് ടെക്സ്റ്റ് രൂപകല്‍പന ചെയ്യുന്നതാണ്. ഇത് വെറും ചോദ്യങ്ങൾ ചോദിക്കുന്നതിലേക്കല്ല - മോഡൽ മനസ്സിലാക്കേണ്ടത് നിങ്ങൾ എന്താണെന്നും എങ്ങനെ അത് നൽകണമെന്നും പ്രാപ്യമായി രൂപകല്‍പന ചെയ്യുന്ന കാര്യമാണ്.

ഒരു സഹപ്രവർത്തകനോട് നിർദ്ദേശങ്ങൾ നൽകുന്നതുപോലെ ഇതിനെപ്പറയൂ. "തീശൂപോലെ പിഴവ് തിരുത്തുക" എന്നു പറയുന്നത് സമാനമാണ്. "UserService.java ലെ 45-ാം വരിയിൽ നിന്നുള്ള null pointer exception ന്റെ പിഴവ് പരിശോധിച്ച് ഒരു null check ചേർക്കുക" എന്നത് വേരിയാന്റ് ചെയ്യുന്നു. ഭാഷാ മോഡലുകൾ ഇതുപോലെ പ്രവർത്തിക്കുന്നു — പ്രത്യേകതയും ഘടനയും മൂല്യവത്താണ്.

<img src="../../../translated_images/ml/how-langchain4j-fits.dfff4b0aa5f7812d.webp" alt="LangChain4j എങ്ങനെ ചേരുന്നുണ്ട്" width="800"/>

LangChain4j മൂലധന സംവിധാനം നൽകുന്നു — മോഡൽ കണക്ഷനുകൾ, മെമ്മറി, മെസേജ് തരം എന്നിവ - എന്നാൽ പ്രോംപ്റ്റ് മാതൃകകൾ അതിൽ നിഷ്പക്ഷം ശ്രദ്ധിച്ചത് നന്നായി ഘടിപ്പിച്ച ടെക്സ്റ്റുകൾ മാത്രമാണ്. മുഖ്യ ഘടകങ്ങൾ SystemMessage (AIന്റെ പെരുമാറ്റവും ചുമതലയുമാണ് നിർണ്ണയിക്കുന്നത്) ഉം UserMessage (നിങ്ങളുടെ യഥാർത്ഥ അഭ്യർത്ഥന വഹിക്കുന്നതു) ഉം ആണ്.

## പ്രോംപ്റ്റ് എൻജിനീയറിംഗ് അടിസ്ഥാനങ്ങൾ

<img src="../../../translated_images/ml/five-patterns-overview.160f35045ffd2a94.webp" alt="അഞ്ചു പ്രോംപ്റ്റ് എൻജിനീയറിംഗ് മാതൃകകളുടെ അവലോകനം" width="800"/>

ഈ മോഡ്യൂളിലെ ഉന്നത മാതൃകകളിലേക്ക് പ്രവേശിക്കുന്നതിന് മുൻപായി, അഞ്ചു അടിസ്ഥാന prompting സാങ്കേതികവിദ്യകൾ നിരീക്ഷിക്കാം. പ്രോംപ്റ്റ് എൻജിനീയറിംഗ് പ്രവർത്തകർക്ക് അവ അറിയേണ്ടതാണ്. [ക്വിക്ക് സ്റ്റാർട്ട് മോഡ്യൂൾ](../00-quick-start/README.md#2-prompt-patterns) നിങ്ങൾ പ്രവർത്തിച്ചിട്ടുണ്ടെങ്കിൽ അവ നന്നായി അഭ്യാസപ്പെടുത്തിയിട്ടുണ്ട് — ഇതാണ് അവയുടെ ആശയപരമായ ഘടന.

### സീറോ-ഷോട്ട് പ്രോംപ്റ്റിംഗ്

പലപ്പോഴും ഏറ്റവും എളുപ്പം: മാതൃകക്കൊന്നും ഉദാഹരണങ്ങൾ നൽകാതെ നേരിട്ട് നിർദ്ദേശം. മാതൃകം തനത് പരിശീലനം ഉപയോഗിച്ച് മാത്രം പ്രവർത്തനക്ഷമമായ പ്രക്രിയയിൽ ഏർപ്പെടുന്നു. പ്രതീക്ഷിക്കാവുന്ന പെരുമാറ്റം വ്യക്തമാകുന്ന ലളിതമായ അഭ്യർത്ഥനകൾക്ക് ഇത് നല്ലതാണ്.

<img src="../../../translated_images/ml/zero-shot-prompting.7abc24228be84e6c.webp" alt="സീറോ-ഷോട്ട് പ്രോംപ്റ്റിംഗ്" width="800"/>

*ഉദാഹരണങ്ങൾ കൂടാതെയുള്ള നേരിയ നിർദ്ദേശം — മാതൃകം നിർദ്ദേശത്തിലൂടെ ടാസ്‌ക് മനസിലാക്കുന്നു*

```java
String prompt = "Classify this sentiment: 'I absolutely loved the movie!'";
String response = model.chat(prompt);
// പ്രതികരണം: "സPozitive"
```

**എപ്പോൾ ഉപയോഗിക്കാം:** ലളിതമായ വർഗ്ഗീകരണങ്ങൾ, നേരിട്ട് ചോദിക്കലുകൾ, വിവർത്തനങ്ങൾ, മറ്റ് മാർഗ്ഗനിർദ്ദേശം ആവശ്യമില്ലാത്ത മുപ്പരിപാടികൾ.

### ഫ്യൂ-ഷോട്ട് പ്രോംപ്റ്റിംഗ്

മാതൃകയിൽ പാലിക്കേണ്ട രീതി വ്യക്തമാക്കുന്ന ഉദാഹരണങ്ങൾ നൽകുക. മാതൃകം നിങ്ങളുടെ ഉദാഹരണങ്ങളിൽ നിന്നാണ് പ്രതീക്ഷിക്കാവുന്ന ഇൻപുട്ട്-ഔട്ട്പുട്ട് ഫോർമാറ്റ് പഠിക്കുകയും പുതിയ ഇൻപുട്ടുകൾക്ക് പ്രയോഗിക്കുകയും ചെയ്യുക. ഇതിൽ സ്ഥിരതയിൽ വലിയ പുരോഗതി കാണാം, പ്രത്യേകിച്ച് ആവശ്യമായ ഫോർമാറ്റ് അല്ലെങ്കിൽ പെരുമാറ്റം വ്യക്തമല്ലാത്ത ടാസ്‌കുകൾക്കായി.

<img src="../../../translated_images/ml/few-shot-prompting.9d9eace1da88989a.webp" alt="ഫ്യൂ-ഷോട്ട് പ്രോംപ്റ്റിംഗ്" width="800"/>

*ഉദാഹരണങ്ങളിൽ നിന്ന് പഠിക്കുക — മാതൃകം മാതൃക തിരിച്ചറിയുകയും പുതിയ ഇൻപുട്ടിന് പ്രയോഗിക്കുകയും ചെയ്യുന്നു*

```java
String prompt = """
    Classify the sentiment as positive, negative, or neutral.
    
    Examples:
    Text: "This product exceeded my expectations!" → Positive
    Text: "It's okay, nothing special." → Neutral
    Text: "Waste of money, very disappointed." → Negative
    
    Now classify this:
    Text: "Best purchase I've made all year!"
    """;
String response = model.chat(prompt);
```

**എപ്പോൾ ഉപയോഗിക്കാം:** ഇഷ്ടാനുസൃത വർഗ്ഗീകരണങ്ങൾ, സ്ഥിരമായ ഫോർമാറ്റിംഗ്, പ്രാദേശിക ടാസ്‌കുകൾ, അല്ലെങ്കിൽ സീറോ-ഷോട്ട് ഫലങ്ങൾ സ്ഥിരതയില്ലാത്തപ്പോൾ.

### ചെയിൻ ഓഫ് തോട്ട്

മാതൃക സ്ഥിരീകരണപ്രക്രിയ തുടർച്ചയായി കാണിക്കുവാൻ ചോദിക്കുക. പരിഹാരത്തിലേക്ക് നേരിട്ട് ഒടുവിൽ എത്താൻ പകരം, മാതൃകം പ്രശ്നം അസംഘടിതമായി ത当に বিচ്ഛേദിച്ച് ഓരോ ഘട്ടവും വ്യക്തമായി പ്രവർത്തനക്ഷമമാക്കുന്നു. ഗണിതം, ലജിക്ക്, മൾട്ടി-സ്റ്റെപ്പ് ചിന്തന ടാസ്‌കുകൾക്ക് ഇതു ശരിയായ ഉത്തരം നൽകുവാൻ സഹായിക്കുന്നു.

<img src="../../../translated_images/ml/chain-of-thought.5cff6630e2657e2a.webp" alt="ചെയിൻ ഓഫ് തോട്ട് പ്രോംപ്റ്റിംഗ്" width="800"/>

*പടി പടി ലജിക്കും — സങ്കീർണ്ണ പ്രശ്നങ്ങളെ വ്യക്തമായ അന്തർദേശീയ ലജിക്കും ഘട്ടങ്ങളായി വിഭജിക്കുന്നു*

```java
String prompt = """
    Problem: A store has 15 apples. They sell 8 apples and then 
    receive a shipment of 12 more apples. How many apples do they have now?
    
    Let's solve this step-by-step:
    """;
String response = model.chat(prompt);
// മോഡൽ കാണിക്കുന്നത്: 15 - 8 = 7, ശേഷം 7 + 12 = 19 ആപ്പിൾസ്
```

**എപ്പോൾ ഉപയോഗിക്കാം:** ഗണിത പ്രശ്നങ്ങൾ, ലജിക്ക് പസിലുകൾ, ഡീബഗ്ഗിംഗ് അല്ലെങ്കിൽ ചിന്തന പ്രക്രിയ പ്രകടിപ്പിക്കുമ്പോൾ കൃത്യതയും വിശ്വാസവും മെച്ചപ്പെടുത്തേണ്ട ടാസ്‌കുകൾ.

### റോൾ-ബേസ്ഡ് പ്രോംപ്റ്റിംഗ്

സംഗീത് ചോദിക്കാൻ മുമ്പായി AI-ക്ക് ഒരു വ്യക്തിത്വം അല്ലെങ്കിൽ ചുമതല നൽകുക. ഇത് പ്രതികരണത്തിന്റെ സ്വരം, ആഴം, ഫോക്കസ് എന്നിവ രൂപപ്പെടുത്തുന്നു. "സോഫ്റ്റ്വെയർ ആർക്കിടെക്ട്" "ജൂനിയർ ഡെവലപ്പർ" അല്ലെങ്കിൽ "സുരക്ഷാ ഓഡിറ്റർ" എന്നതിനോട് വ്യത്യസ്ത ഉപദേശം നൽകും.

<img src="../../../translated_images/ml/role-based-prompting.a806e1a73de6e3a4.webp" alt="റോൾ-ബേസ്ഡ് പ്രോംപ്റ്റിംഗ്" width="800"/>

*പ്രസംഗവും വ്യക്തിത്വവും സജ്ജീകരിക്കൽ — ചുമതലക്കനുസരിച്ച് ഒരേ ചോദ്യത്തിന് വ്യത്യസ്ത മറുപടി ലഭിക്കുന്നു*

```java
String prompt = """
    You are an experienced software architect reviewing code.
    Provide a brief code review for this function:
    
    def calculate_total(items):
        total = 0
        for item in items:
            total = total + item['price']
        return total
    """;
String response = model.chat(prompt);
```

**എപ്പോൾ ഉപയോഗിക്കാം:** കോഡ് റിവ്യൂകൾ, ട്യൂട്ടറിംഗ്, പ്രാദേശിക വിശകലനം, പ്രത്യേക പരിജ്ഞാന നിലയ്ക്ക് അനുയോജ്യമായ പ്രതികരണങ്ങൾ ആവശ്യമുള്ളപ്പോൾ.

### പ്രോംപ്റ്റ് ടെംപ്ലേറ്റുകൾ

മാറ്റിക്കാര്യമുള്ള പ്ലേസ്ഹോൾഡറുകൾ ഉള്ള പുനരുപയോഗയോഗ്യ പ്രോംപ്റ്റുകൾ സൃഷ്ടിക്കുക. ഓരോ പ്രയോഗത്തിനും പുതിയൊരു പ്രോംപ്റ്റ് എഴുതാനുള്ള പകരം, ഒരു ടെംപ്ലേറ്റ് നിർവ്വചിച്ച് വ്യത്യസ്ത മൂല്യങ്ങൾ നിറയ്ക്കുക. LangChain4j-യുടെ `PromptTemplate` ക്ലാസ്സ് ഇത് `{{variable}}` സിന്റക്സിലൂടെ എളുപ്പമാക്കുന്നു.

<img src="../../../translated_images/ml/prompt-templates.14bfc37d45f1a933.webp" alt="പ്രോംപ്റ്റ് ടെംപ്ലേറ്റുകൾ" width="800"/>

*വ്യത്യസ്ത ഫലങ്ങൾക്കായി പുനരുപയോഗയോഗ്യമായ പ്രോംപ്റ്റുകൾ — ഒരു ടെംപ്ലേറ്റ്, അനേകം പ്രയോഗങ്ങൾ*

```java
PromptTemplate template = PromptTemplate.from(
    "What's the best time to visit {{destination}} for {{activity}}?"
);

Prompt prompt = template.apply(Map.of(
    "destination", "Paris",
    "activity", "sightseeing"
));

String response = model.chat(prompt.text());
```

**എപ്പോൾ ഉപയോഗിക്കാം:** വ്യത്യസ്ത ഇൻപുട്ടുകളുള്ള ആവർത്തിത ചോദിക്കുന്നത്, ബാച്ച് പ്രോസസ്സിംഗ്, പുനരുപയോഗയോഗ്യ AI ജോലികൾ നിർമ്മിക്കൽ, അല്ലെങ്കിൽ പ്രോംപ്റ്റ് ഘടന ഒരുപോലെയുണ്ടെങ്കിലും ഡാറ്റ മാറുന്നതും.

---

ഈ അഞ്ചു അടിസ്ഥാനങ്ങൾ പ്രോംപ്റ്റിങ്ങിന്റെ മിക്ക ടാസ്‌കുകൾക്കായി ദൃഢമായ സൗകര്യം നൽകുന്നു. ഈ മോഡ്യൂളിലെ ബാക്കി ഭാഗം GPT-5.2-ന്റെ reasoning control, സ്വയം-അവലോകനം, ഘടനാപരമായ പുറപ്പെട്ടൽ കഴിവുകൾ ഉപയോഗിക്കുന്ന **എട്ട് ഉന്നത മാതൃകകളുമായി** അടിയുറച്ചിരിക്കുന്നു.

## ഉന്നത മാതൃകകൾ

അടിസ്ഥാനങ്ങൾ അറിഞ്ഞ ശേഷമാണ് ഈ മോഡ്യൂളിന്റെ വൈവിധ്യമാർന്ന എട്ടു ഉന്നത മാതൃകകളിലേക്ക് പോകുന്നത്. എല്ലാ പ്രശ്നങ്ങൾക്കും എപ്പോഴും ഒരേ സമീപനം വേണമെന്നില്ല. ചില ചോദ്യങ്ങൾക്ക് വേഗത്തിലുള്ള എളുപ്പം ഉത്തരം വേണം, ചിലയ്ക്ക് ലളിതമായ ആഴത്തിലുള്ള ചിന്ത്ചെയ്‌തു് വേണം. ചിലക്ക് ദൃശ്യ reasoning ആവശ്യമാണ്, ചിലയ്ക്ക് ഫലങ്ങൾ മാത്രമേ വേണ്ടുള്ളൂ. താഴെപ്പറയുന്ന ഓരോ മാതൃകയും വ്യത്യസ്ത സാഹചര്യങ്ങൾക്ക് അനുയോജ്യമായ സംഗീതമായി രൂപകൽപനയുണ്ട് — GPT-5.2-ന്റെ reasoning control വ്യത്യാസങ്ങൾ കൂടുതൽ വ്യക്തമായി കാണിക്കുന്നു.

<img src="../../../translated_images/ml/eight-patterns.fa1ebfdf16f71e9a.webp" alt="എട്ട് പ്രോംപ്റ്റിംഗ് മാതൃകകൾ" width="800"/>

*എട്ട് പ്രോംപ്റ്റ് എൻജിനീയറിംഗ് മാതൃകകളുടെ അവലോകനം, അവയുടെ ഉപയോഗ സൗകര്യങ്ങൾ*

<img src="../../../translated_images/ml/reasoning-control.5cf85f0fc1d0c1f3.webp" alt="GPT-5.2 യോടുള്ള reasoning control" width="800"/>

*GPT-5.2-യുടെ reasoning control മോഡലിന് എത്രം ചിന്തിക്കാമെന്നു നിർദേശിക്കാൻ പാടുള്ളതു് — ഫാസ്റ്റ് നേരിട്ടുള്ള ഉത്തരങ്ങളിൽ നിന്നും ആഴത്തിലുള്ള ഗവേഷണവരെ*

<img src="../../../translated_images/ml/reasoning-effort.db4a3ba5b8e392c1.webp" alt="തർക്കച്ചിന്തന പരിശ്രമത്തിന്റെ താരതമ്യം" width="800"/>

*കുറഞ്ഞ ഉത്സാഹം (വേഗം, നേരിട്ടുള്ളത്) vs ഉയർന്ന ഉത്സാഹം (ആഴത്തിലുള്ള, പരീക്ഷണാത്മകമായ) reasoning സമീപനങ്ങൾ*

**കുറഞ്ഞ ഉത്സാഹം (വേഗം & കേന്ദ്രീകൃതം)** - എളുപ്പമുള്ള ചോദ്യങ്ങൾക്ക് വേഗത്തിലുള്ള നേരിട്ടുള്ള ഉത്തരങ്ങൾ നൽകാൻ. മോഡൽ കുറഞ്ഞ reasoning ഘട്ടങ്ങൾ (അതി ശീർഷം രണ്ട് ഘട്ടം) മാത്രമേ നടത്തൂ. കണക്കുകൾ, ലഭ്യമാക്കലുകൾ, ലളിത ചോദ്യങ്ങൾക്ക് ഇത് ഉപയോഗിക്കുക.

```java
String prompt = """
    <reasoning_effort>low</reasoning_effort>
    <instruction>maximum 2 reasoning steps</instruction>
    
    What is 15% of 200?
    """;

String response = chatModel.chat(prompt);
```

> 💡 **GitHub Copilot ഉപയോഗിച്ച് പരിശോധിക്കൂ:** [`Gpt5PromptService.java`](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java) തുറന്ന് ചോദിക്കുക:
> - "കുറഞ്ഞ ഉത്സാഹവും ഉയർന്ന ഉത്സാഹവും പ്രോംപ്റ്റിംഗ് മാതൃകകളിലെ വ്യത്യാസം എന്താണ്?"
> - "XML ടാഗുകൾ പ്രോംപ്റ്റുകളിലെ AIയുടെ പ്രതികരണം ഘടിപ്പിക്കാൻ എങ്ങനെ സഹായിക്കുന്നു?"
> - "സ്വയം-സംവേദന മാതൃകകൾ നേരത്തെ നൽകിയ നിർദ്ദേശങ്ങൾക്കോട് എപ്പോൾ ഉപയോഗിക്കണം?"

**ഉയർന്ന ഉത്സാഹം (ആഴത്തിലുള്ള & വിശദമായ)** - സമഗ്ര വിശകലനം ആവശ്യമായ സങ്കീർണ്ണ പ്രശ്നങ്ങൾക്ക്. മോഡൽ പൂർണ്ണമായും അന്വേഷിക്കുകയും വിശദമായ reasoning കാണിക്കുകയും ചെയ്യും. സിസ്റ്റം ഡിസൈൻ, ആർക്കിടെക്ചർ തീരുമാനങ്ങൾ, സങ്കീർണ്ണ ഗവേഷണം എന്നിവയ്ക്ക് ഇത് ഉപയോഗിക്കുക.

```java
String prompt = """
    <reasoning_effort>high</reasoning_effort>
    <instruction>explore thoroughly, show detailed reasoning</instruction>
    
    Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```

**ടാസ്‌ക് എക്സിക്യൂഷൻ (പടിപടി പുരോഗതി)** - മൾട്ടി-സ്റ്റെപ്പ് പ്രവൃത്തി പ്രവാഹങ്ങൾക്ക്. മോഡൽ മുൻകൂർ പദ്ധതി നൽകുകയും, ഓരോ ഘട്ടവും പ്രവർത്തിക്കുമ്പോൾ വിവരിക്കുകയും, അവസാനം സംഗ്രഹം നൽകുകയും ചെയ്യുന്നു. മൈഗ്രേഷനുകൾ, നടപ്പാക്കലുകൾ, പലറ്റിപ്പാടുകളും സംയോജിപ്പിക്കാൻ ഈ രീതിയാണ്.

```java
String prompt = """
    <task>Create a REST endpoint for user registration</task>
    <preamble>Provide an upfront plan</preamble>
    <narration>Narrate each step as you work</narration>
    <summary>Summarize what was accomplished</summary>
    """;

String response = chatModel.chat(prompt);
```

Chain-of-Thought prompting മോഡലിന് reasoning പ്രക്രിയ കാണിക്കുവാൻ നിർദ്ദേശിക്കുന്നു, സങ്കീർണ്ണ ടാസ്‌കുകളിൽ കൃത്യത മെച്ചപ്പെടുത്തുന്നു. പടി പടി വിശദീകരണം മനുഷ്യരും AI ഉം ലജിക്കും മനസ്സിലാക്കാൻ സഹായിക്കുന്നു.

> **🤖 [GitHub Copilot](https://github.com/features/copilot) Chat ഉപയോഗിച്ച് പരീക്ഷിയ്ക്കൂ:** ഈ മാതൃകയെ കുറിച്ച് ചോദിക്കുക:
> - "നീണ്ട സമയമായ ഓപ്പറേഷനുകൾക്കായി ടാസ്‌ക് എക്സിക്യൂഷൻ മാതൃക എങ്ങനെ അപേക്ഷിക്കാം?"
> - "പ്രൊഡക്ഷൻ അപ്ലിക്കേഷനുകളിൽ ടൂൾ പ്രീആംബിളുകൾ സജ്ജീകരിക്കാനുളള മികച്ച രീതികൾ എന്തെല്ലാം?"
> - "ഇന്റർമീഡിയറ്റ് പ്രോഗ്രസ് അപ്‌ഡേറ്റുകൾ UIയിൽ എങ്ങനെ തടയുക, പ്രദർശിപ്പിക്കുക?"

<img src="../../../translated_images/ml/task-execution-pattern.9da3967750ab5c1e.webp" alt="ടാസ്‌ക് എക്സിക്യൂഷൻ മാതൃക" width="800"/>

*പദ്ധതി → നടപ്പാക്കൽ → സംഗ്രഹം പ്രവൃത്തി പ്രക്രിയ*

**സ്വയം-സംവേദന കോഡ്** - പ്രൊഡക്ഷൻ നിലവാരമുള്ള കോഡ് ജനറേറ്റ് ചെയ്യൽ. മോഡൽ കോഡ് സൃഷ്ടിക്കുകയും ഗുണനിലവാരത്തിൽ പരിശോധിക്കുകയും മെച്ചപ്പെടുത്തുകയും ചെയ്യും. പുതിയ സവിശേഷതകൾ, സർവീസുകൾ നിർമ്മിക്കുമ്പോൾ ഇത് ഉപയോഗിക്കുക.

```java
String prompt = """
    <task>Create an email validation service</task>
    <quality_criteria>
    - Correct logic and error handling
    - Best practices (clean code, proper naming)
    - Performance optimization
    - Security considerations
    </quality_criteria>
    <instruction>Generate code, evaluate against criteria, improve iteratively</instruction>
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/ml/self-reflection-cycle.6f71101ca0bd28cc.webp" alt="സ്വയം-സംവേദന ചക്രം" width="800"/>

*പര്യവേഷണഓപ്പറേഷൻ ഭ്രമണചക്രം - സൃഷ്ടിക്കുക, വിലയിരുത്തുക, പ്രശ്‌നങ്ങൾ കണ്ടെത്തുക, മെച്ചപ്പെടുത്തൽ, ആവർത്തിക്കുക*

**സമർദ്ദിത വിശകലനം** - സ്ഥിരം മൂല്യനിർണയം. ഫെക്സഡ് ഫ്രെയിംവർക്ക് (ശരിയാനല്ല, പ്രാക്ടീസുകൾ, പ്രകടനം, സുരക്ഷ) ഉപയോഗിച്ച് കോഡ് പരിശോധിക്കുന്നു. കോഡ് റിവ്യൂകൾക്ക് അനുയോജ്യമാണ്.

```java
String prompt = """
    <code>
    public List getUsers() {
        return database.query("SELECT * FROM users");
    }
    </code>
    
    <framework>
    Evaluate using these categories:
    1. Correctness - Logic and functionality
    2. Best Practices - Code quality
    3. Performance - Efficiency concerns
    4. Security - Vulnerabilities
    </framework>
    """;

String response = chatModel.chat(prompt);
```

> **🤖 [GitHub Copilot](https://github.com/features/copilot) Chat ഉപയോഗിച്ച് പരീക്ഷിയ്ക്കൂ:** സമർദ്ദിത വിശകലനത്തെ കുറിച്ച് ചോദിക്കുക:
> - "വ്യത്യസ്ത കോഡ് റിവ്യൂ വിഭാഗങ്ങൾക്ക് വിശകലന ഫ്രെയിംവർക്ക് എങ്ങനെ ഇഷ്ടാനുസൃതമാക്കാം?"
> - "സംരംഭം പ്രോഗ്രാമാറ്റിക്കായി സമർദ്ദിത ഔട്ട്പുട്ട് how to parse എങ്കിലും പ്രവർത്തനം?"
> - "വിശകലനങ്ങളിൽ സ്ഥിരതയുള്ള ഗുരുത്വമുളള തലങ്ങൾ എങ്ങനെ ഉറപ്പാക്കാം?"

<img src="../../../translated_images/ml/structured-analysis-pattern.0af3b690b60cf2d6.webp" alt="സമർദ്ദിത വിശകലന മാതൃക" width="800"/>

*നാലു വിഭാഗങ്ങളുള്ള സ്ഥിരമായ കോഡ് റിവ്യൂകൾ, ഗുരുത്വ നിലകളോടെ*

**മൾട്ടി-ടേൺ ചാറ്റ്** - കോൺടെക്സ്റ്റ് ആവശ്യമായ സംവാദങ്ങൾക്കായി. മോഡൽ മുമ്പത്തെ സന്ദേശങ്ങൾ ഓർക്കുകയും അവയുടെ അടിസ്ഥാനത്തിൽ പ്രതികരണങ്ങൾ വളർത്തുകയും చేస్తു. സംവേദന സഹായം സെഷനുകൾക്കോ സങ്കീർണ്ണ ചോദ്യോത്തരങ്ങൾക്കോ ഉപയോഗിക്കാം.

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("What is Spring Boot?"));
AiMessage aiMessage1 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage1);

memory.add(UserMessage.from("Show me an example"));
AiMessage aiMessage2 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage2);
```

<img src="../../../translated_images/ml/context-memory.dff30ad9fa78832a.webp" alt="കോൺടെക്സ്റ്റ് മെമ്മറി" width="800"/>

*ഒന്നിലധികംഷേണ്ഡുകളിലായി സംഭാഷണമാറ്റത്തിനു കോൺടെക്സ്റ്റ് എങ്ങനെ സമ്പാദിക്കുന്നു, ടോക്കൺ പരിധി വരെ*

**പടി പടി ലജിക്കും പരിണാമം** - ദൃശ്യമായ ലജിക്കും ആവശ്യമായ പ്രശ്നങ്ങൾക്ക്. ഓരോ ഘട്ടത്തിന്റെയും വ്യക്തമായ ലോജിക്കും കാണിക്കുന്നു. ഗണിത, ലജികൽ പസിലുകൾ, ചിന്തന പ്രക്രിയ മനസ്സിലാക്കേണ്ടുന്നപ്പോൾ ഉപകാരപ്രദം.

```java
String prompt = """
    <instruction>Show your reasoning step-by-step</instruction>
    
    If a train travels 120 km in 2 hours, then stops for 30 minutes,
    then travels another 90 km in 1.5 hours, what is the average speed
    for the entire journey including the stop?
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/ml/step-by-step-pattern.a99ea4ca1c48578c.webp" alt="പടി പടി മാതൃക" width="800"/>

*പ്രശ്നങ്ങളെ വ്യക്തമായ ലോജിക്കൽ ഘട്ടങ്ങളിൽ വിഭജിക്കൽ*

**നിയന്ത്രിത ഔട്ട്പുട്ട്** - പ്രത്യേക രൂപ രീതി ആവശ്യമായ പ്രതികരണങ്ങൾക്കായി. മാതൃകം കർശനമായി ഫോർമാറ്റും നീളവും പാലിക്കും. സംഗ്രഹങ്ങൾക്കും ശരിയായ ഔട്ട്പുട്ട് ഘടന ആവശ്യമുള്ളു എങ്കിൽ ഉപയോഗിക്കുക.

```java
String prompt = """
    <constraints>
    - Exactly 100 words
    - Bullet point format
    - Technical terms only
    </constraints>
    
    Summarize the key concepts of machine learning.
    """;

String response = chatModel.chat(prompt);
```

*നിർദ്ദിഷ്ട ഫോർമാറ്റ്, നീളം, ഘടനാ ആവശ്യങ്ങൾ പ്രാബല്യപ്പെടുത്തൽ*

ഡെവ് കണ്ടെയ്‌നറിൽ Spring Boot ഡാഷ്ബോർഡ് എക്സ്റ്റൻഷൻ ഉൾപ്പെടുത്തിയിട്ടുണ്ട്, ഇത് എല്ലാ Spring Boot ആപ്ലിക്കേഷനുകളും ദൃശ്യമായ ഇന്റർഫേസിലൂടെ നിയന്ത്രിക്കാൻ സഹായിക്കുന്നു. ഇത് VS കോഡിന്റെ ഇടത് ഭാഗത്തെ ആക്ടിവിറ്റി ബാറിൽ സ്പ്രിംഗ് ബൂട്ട് ഐക്കൺ ആയി കാണാം.
Spring Boot ഡാഷ്ബോർഡിൽ നിന്നു, നിങ്ങൾക്ക് ചെയ്യാൻ കഴിയും:
- വർക്ക്‌സ്പേസിലെ എല്ലാ ലഭ്യമായ Spring Boot ആപ്ലിക്കേഷനുകളും കാണുക
- സിംഗിൾ ക്ലിക്കിൽ ആപ്ലിക്കേഷനുകൾ ആരംഭിക്കുകയും നിർത്തുകയും ചെയ്യുക
- ആപ്ലിക്കേഷൻ ലോഗുകൾ റിയൽ-ടൈമിൽ കാണുക
- ആപ്ലിക്കേഷൻ സ്ഥിതിവിവരക്കണക്കുകൾ നിരീക്ഷിക്കുക

## മാതൃകകൾ പരിശോധിക്കൽ

വെബ് ഇന്റർഫെയ്‌സ് വ്യത്യസ്ത പ്രോംപ്റ്റിംഗ് തന്ത്രങ്ങൾ പരീക്ഷിക്കാൻ സൗകര്യമുണ്ട്. ഓരോ മാതൃകയും വ്യത്യസ്ത പ്രശ്‌നങ്ങൾ പരിഹരിക്കുന്നു - ഓരോ സമീപനം എപ്പോൾ ഉജ്ജ്വലമാകുന്നു എന്ന് കാണാൻ അവ പരീക്ഷിക്കൂ.

### കുറഞ്ഞ ഉത്സാഹം vs ഉയർന്ന ഉത്സാഹം

"200 ന്റെ 15% എത്രയാണ്?" എന്ന ലഘു ചോദ്യം കുറഞ്ഞ ഉത്സാഹം ഉപയോഗിച്ച് ചോദിക്കുക. നിങ്ങൾക്ക് ഉടൻ, നേരിട്ടുള്ള ഉത്തരം ലഭിക്കും. ഇപ്പോൾ "ഹൈ-ട്രാഫിക് API യ്ക്ക് കാഷിംഗ് തന്ത്രം രൂപകൽപ്പന ചെയ്യൂ" എന്ന സങ്കീർണ്ണമായ ചോദ്യം ഉയർന്ന ഉത്സാഹം ഉപയോഗിച്ച് ചോദിക്കുക. മോഡൽ മന്ദഗതിയിലാകും, വിശദമായ വാദങ്ങൾ നൽകുകയും ചെയ്യും. ഒരേ മോഡൽ, ഒരേ ചോദ്യം ഘടന - പക്ഷെ പ്രോംപ്റ്റ് എത്രമാത്രം ആലോചന വേണമെന്ന് പറയുന്നു.

<img src="../../../translated_images/ml/low-eagerness-demo.898894591fb23aa0.webp" alt="Low Eagerness Demo" width="800"/>

*കുറഞ്ഞ വാദത്തോടെ വേഗത്തിലുള്ള കണക്കുകൂട്ടൽ*

<img src="../../../translated_images/ml/high-eagerness-demo.4ac93e7786c5a376.webp" alt="High Eagerness Demo" width="800"/>

*വളരെയധികം വിശദമായ കാഷിംഗ് തന്ത്രം (2.8MB)*

### ടാസ്‌ക് എക്സിക്യൂഷൻ (ടൂൾ പ്രീഅംബിൾസ്)

ബഹുനിരവധിഘട്ട പ്രവാഹങ്ങൾ മുൻകൂട്ടി ആസൂത്രണം և പുരോഗതി വിവരിക്കുന്നതിൽ ലാഭം കാണുന്നു. മോഡൽ ചെയ്യേണ്ടത് അവലോകനം ചെയ്യുകയും ഓരോ ഘട്ടവും വിവരിക്കുകയും പിന്നെ ഫലങ്ങൾ സംഗ്രഹിക്കുകയും ചെയ്യുന്നു.

<img src="../../../translated_images/ml/tool-preambles-demo.3ca4881e417f2e28.webp" alt="Task Execution Demo" width="800"/>

*ഘട്ടങ്ങൾ വിവരണം ഉൾപ്പെടുന്ന REST എൻഡ്‌പോയിന്റ് നിർമ്മാണം (3.9MB)*

### സ്വയംപരിശോധനാ കോഡ്

"ഇമെയിൽ സ്ഥിരീകരണ സേവനം സൃഷ്ടിക്കുക" എന്നത് പരീക്ഷിക്കുക. കോഡ് മാത്രമല്ല സൃഷ്ടിക്കുന്നത്; മോഡൽ നിലവാര മാനദണ്ഡങ്ങളുടെ അടിസ്ഥാനത്തിൽ വിലയിരുത്തുകയും ദുർബലതകൾ കണ്ടെത്തുകയും മെച്ചപ്പെടുത്തുകയും ചെയ്യുന്നു. പ്രൊഡക്ഷൻ മാനദണ്ഡങ്ങൾ പാലിക്കുന്നതിേരെം വരെ ദൈർഘ്യമേറിയ മടക്കം കാണാൻ കഴിയുമെന്നും നിങ്ങൾ കാണും.

<img src="../../../translated_images/ml/self-reflecting-code-demo.851ee05c988e743f.webp" alt="Self-Reflecting Code Demo" width="800"/>

*പൂർണ്ണമായ ഇമെയിൽ സ്ഥിരീകരണ സേവനം (5.2MB)*

### ഘടനാപരമായ വിശകലനം

കോഡ് അവലോകനങ്ങൾ സ്ഥിരതയുള്ള മൂല്യനിർണ്ണയ ഫ്രെയിമ്വർക്കുകൾ ആവശ്യമാണ്. മോഡൽ കൃത്യത, പ്രാക്ടീസുകൾ, പ്രകടനം, സുരക്ഷ തുടങ്ങിയ സ്ഥിര നിലകളിൽ കോഡ് വിശകലനം ചെയ്യുന്നു.

*ഫ്രെയിംവർക്ക്ഭദ്ദമായ കോഡ് അവലോകനം*

### ബഹുവിഭാഗ ചാറ്റ്

"Spring Boot എന്താണ്?" എന്ന് ചോദിച്ച് പിന്നെ ഉടനെ "ഉദാഹരണം കാണിക്കൂ" എന്ന ചോദ്യം നൽകുക. ആദ്യ ചോദ്യം ഓർമിക്കുക എന്നതിനാൽ മോഡൽ തികച്ചും സംബദ്ധമായ ഒരു Spring Boot ഉദാഹരണം നൽകുന്നു. ഓർമ്മ ഇല്ലെങ്കിൽ രണ്ടാം ചോദ്യം പരാജയപ്പെടും.

### ഘട്ടം ഘട്ടം കാരണം കാണിക്കൽ

ഒരു ഗണിത പ്രശ്‌നം തിരഞ്ഞെടുക്കുക, അത് ഘട്ടം ഘട്ടം കാരണം കാണിക്കൽ സൗകര്യത്തോടെയും കുറഞ്ഞ ഉത്സാഹത്തോടെയും പരീക്ഷിക്കുക. കുറഞ്ഞ ഉത്സാഹം വെറും ഉത്തരം വേഗത്തിൽ നൽകും, പക്ഷെ പ്രകടനപരമായി മൂടിയാകും. ഘട്ടം ഘട്ടം കാരണവന്നു് നിങ്ങളെ ഓരോ കണക്കും തീരുമാനവും കാണിക്കും.

<img src="../../../translated_images/ml/step-by-step-reasoning-demo.12139513356faecd.webp" alt="Step-by-Step Reasoning Demo" width="800"/>

*സ്പഷ്ട ഘട്ടങ്ങളുകളുള്ള ഗണിത പ്രശ്‌നം*

### നിയന്ത്രിത ഔട്ട്‌പുട്ട്

നിങ്ങൾക്ക് കൃത്യമായ ഫോർമാറ്റുകളും വാക്കുകളുടെ എണ്ണം ആവശ്യമായപ്പോൾ, ഈ മാതൃക കർശനമായ പാലനത്തിലേക്ക് നിർബന്ധിക്കുന്നു. 100 വാക്കുകൾ അത്യാടിപെട്ട ബുള്ളറ്റ് പോയിന്റായി സമാരംഭം സൃഷ്ടിക്കുക.

<img src="../../../translated_images/ml/constrained-output-demo.567cc45b75da1633.webp" alt="Constrained Output Demo" width="800"/>

*ഫോർമാറ്റ് നിയന്ത്രണത്തോടെ മെഷീൻ ലേണിംഗ് സംഗ്രഹം*

## നിങ്ങൾ ശരിക്കും പഠിക്കുന്നത് എന്താണ്

**കാരണവിചാരം ചെയ്യുന്ന ശ്രമം എല്ലാം മാറ്റുന്നു**

GPT-5.2 നിങ്ങളുടെ പ്രോംപ്റ്റുകൾ വഴിയുള്ള കണക്കുകൂട്ടൽ ശ്രമം നിയന്ത്രിക്കാൻ സഹായിക്കുന്നു. കുറഞ്ഞ ശ്രമം വേഗത്തിൽ ഉത്തരം, ചെറിയ അന്വേഷണത്തോടെ നൽകുന്നു. ഉയർന്ന ശ്രമം മോഡലിന് സമയം എടുത്ത് ആഴത്തിലുള്ള ആലോചനം ചെയ്യും. നിങ്ങൾ ശീലിക്കുന്നത് പ്രവൃത്തി സംബന്ധമായ സങ്കീർണ്ണതയ്ക്ക് അനുയോജ്യമായ ശ്രമം നൽകുന്നതാണ് - എളുപ്പമുള്ള ചോദ്യങ്ങളിൽ സമയം മോഷ്ടിക്കരുത്, സങ്കീർണ്ണ തീരുമാനങ്ങളിൽ ഉപദേശം മുന്നോട്ടാക്കുക.

**ഘടന പെരുമാറ്റം നിർദ്ദേശിക്കുന്നു**

പ്രോംപ്റ്റുകളിൽ കാണുന്ന XML ടാഗുകൾ ശ്രദ്ധിക്കുക? അവ അലങ്കാരങ്ങൾ അല്ല. മോഡലുകൾ ഘടനാപരമായ നിർദ്ദേശങ്ങൾ സ്വതന്ത്ര എഴുത്തിനേക്കാൾ വിശ്വാസയോഗ്യമായി പിന്തുടരുന്നു. ബഹുനിരവധി ഘട്ട പ്രക്രിയകൾക്കും സങ്കീർണ്ണ ട്രിക്കുകൾക്കും ഘടന സഹായിക്കുന്നു മോഡൽ എവിടെ ഉണ്ട് എന്ന് ട്രാക്ക് ചെയ്യാനും അടുത്ത ഘട്ടം എന്തെന്ന് അറിയാനും.

<img src="../../../translated_images/ml/prompt-structure.a77763d63f4e2f89.webp" alt="Prompt Structure" width="800"/>

*വ്യവസ്ഥാപിത സെക്ഷനുകളും XML-സമാന സംഘടനയുമുള്ള ഒരു മികച്ച ഘടനയുള്ള പ്രോംപ്റ്റിന്റെ രചന*

**സ്വയം-മൂല്യനിർണയത്തിലൂടെ ഗുണനിലവാരം**

സ്വയംപരിശോധന മാതൃകകൾ ഗുണനിലവാര മാനദണ്ഡങ്ങൾ വ്യക്തമാക്കുന്നുവെന്ന് പരിഗണിക്കുന്നു. മോഡൽ "സരിയായി ചെയ്യും" എങ്കിൽ പ്രതീക്ഷിക്കുന്നതിന് പകരം നിങ്ങൾക്കു "സരി" എന്ന് دقیقമായി പറയാനാകും: ശരിയായ ലാജിക്, പിശക് കൈകാര്യംചെയ്യൽ, പ്രകടനം, സുരക്ഷ. മോഡൽ അത് സ്വയം വിലയിരുത്തുകയും മെച്ചപ്പെടുത്തുകയും ചെയ്യും. ഇത് കോഡ് ജനനത്തെ ലോട്ടറിയിൽ നിന്നും പ്രക്രിയയാക്കി മാറ്റുന്നു.

**സന്ദർഭം പരിമിതമാണ്**

ബഹുനിരവധി സംഭാഷണങ്ങൾ ഓരോ അഭ്യർത്ഥനയോടും മസേജ് ചരിത്രം ഉൾക്കൊള്ളുന്നു. പക്ഷേ പരിധി ഉണ്ട് - ഓരോ മോഡലിനും പരമാവധി ടോക്കൺ എണ്ണം ഉണ്ട്. സംഭാഷണങ്ങൾ വളരുകയാണെങ്കിൽ, പ്രാസംഗിക സന്ദർഭം പൂർണ്ണമായി സൂക്ഷിക്കാനുള്ള തന്ത്രങ്ങൾ ഉപയോഗിക്കണം. ഈ മോഡ്യൂൾ ഓർമ്മ എങ്ങനെ പ്രവർത്തിക്കുന്നു കാണിക്കുന്നു; മേൽകഴിഞ്ഞു എപ്പോൾ സംഗ്രഹിക്കണം, എപ്പോൾ മറക്കണം, എപ്പോൾ തിരുവനന്തപുരിക്കണം എന്നുമറിയിക്കും.
