## നിങ്ങൾ എന്ത് പഠിക്കുകയാണ്

<img src="../../../translated_images/ml/what-youll-learn.c68269ac048503b2.webp" alt="What You'll Learn" width="800"/>

മുന്‍പ് മോഡ്യൂളില്‍, നിങ്ങൾ സംസാരിച്ചു പഠിക്കുന്ന എ.ഐ.യുടെ സ്മൃതി എന്നീ ഘടകങ്ങളെ കണ്ടു, മാത്രമല്ല GitHub മോഡലുകൾ ഉപയോഗിച്ച് അടിസ്ഥാന സംവാദങ്ങൾ നടത്തിയിരുന്നു. ഇനി നിങ്ങള്‍ ചോദിക്കുന്ന തരത്തിന് -- പ്രോംപ്റ്റുകൾക്ക് -- Azure OpenAI യുടെ GPT-5.2 ഉപയോഗിച്ച് ശ്രദ്ധ കേന്ദ്രീകരിക്കാം. പ്രോംപ്റ്റുകള്‍ എങ്ങനെ ഘടിപ്പിക്കപ്പെടുന്നു എന്നത് വളരെ ഭംഗിയായി ലഭിക്കുന്ന മറുപടികളിന്റെ ഗുണമേന്‍മയെ സ്വാധീനിക്കുന്നു. നാം അടിസ്ഥാന പ്രോംപ്റ്റിംഗ് സാങ്കേതികവിദ്യകള്‍ അവലോകനം ചെയ്ത് തുടക്കംകഴിഞ്ഞ്, ശേഷം GPT-5.2 യുടെ ശേഷികള്‍ പൂർണമായി പ്രയോജനപ്പെടുത്തുന്ന എട്ട് പരിച്ഛേദിക്കുന്ന പാറ്റേണുകളില്‍ പ്രവേശിക്കും.

നാം GPT-5.2 ഉപയോഗിക്കുന്നതിന്റെ കാരണം എന്നാൽ, ഇത് കരുതൽ നിയന്ത്രണം കൊണ്ടുവരുന്നു — നിങ്ങൾ മോഡലിന് എത്രകൂടി ചിന്തിക്കണമെന്ന് പറയാൻ സാധിക്കും. ഇത് വ്യത്യസ്ത പ്രോംപ്റ്റിംഗ് രണരീതികളെ കൂടുതൽ വ്യക്തമാക്കുകയും ഓരോ രീതിയും എപ്പോൾ ഉപയോഗിക്കണമെന്ന് മനസ്സിലാക്കാന്‍ സഹായിക്കുകയും ചെയ്യും. കൂടാതെ GPT-5.2 ന് GitHub മോഡലുകളെ അപേക്ഷിച്ച് Azure യുടെ കുറവ് നിരക്കു പരിമിതികൾ ഉപകാരപ്പെടും.

## പ്രോംപ്റ്റ് എഞ്ചിനീയറിംഗ് മനസിലാക്കല്‍

<img src="../../../translated_images/ml/what-is-prompt-engineering.5c392a228a1f5823.webp" alt="What is Prompt Engineering?" width="800"/>

പ്രോംപ്റ്റ് എഞ്ചിനീയറിംഗ് എന്നത് നിങ്ങൾ ആവശ്യമായ ഫലങ്ങൾ സ്ഥിരമായി നേടാൻ കഴിയുന്ന ഇന്പുട്ട് ടെക്സ്റ്റ് രൂപകൽപ്പന ചെയ്യുകയാണ്. ഇത് വെറും ചോദ്യങ്ങൾ ചോദിക്കുന്നതല്ല — മോഡലിന് നിങ്ങൾ എന്ത് വേണമെന്ന് എങ്ങനെ കൊടുക്കാം എന്നും അതു എങ്ങനെ നൽകണം എന്നും ബോധ്യപ്പെടുന്ന വിധം അഭ്യർത്ഥനകൾ ഘടിപ്പിക്കുന്നതും ആണ്.

നിങ്ങളുടെ സഹപ്രവർത്തകനെ നിർദ്ദേശങ്ങൾ നൽകുന്നതുപോലെ ചിന്തിക്കാം. "ബഗ് പരിഹരിക്കുക" എന്നത് മോടിയുള്ളൊരു പ്രസ്താവനയാണ്. "UserService.java ലെ 45-ാം വരിയില്‍ നൾ ചെക്കു ചേർത്ത് നൾ പോയിന്റര്‍ എക്‌സെപ്ഷൻ പരിഹരിക്കുക" എന്നത് വ്യക്തമായും സവിശേഷമായും ആണ്. ഭാഷാ മോഡലുകൾക്കും ഒരുപോലെ ലക്ഷ്യവും ഘടനയും പ്രധാനമാണ്.

<img src="../../../translated_images/ml/how-langchain4j-fits.dfff4b0aa5f7812d.webp" alt="How LangChain4j Fits" width="800"/>

LangChain4j മോഡൽ കണക്ഷനുകൾ, സ്മൃതി, മെസേജ് തരങ്ങൾ തുടങ്ങിയ അടിസ്ഥാന सङഘടനകൾ നൽകുന്നു — പ്രോംപ്റ്റ് പാറ്റേണുകൾ ആ ഘടനയിലൂടെ അയക്കുന്ന സൂക്ഷ്മ രൂപകൽപ്പന ചെയ്ത ടെക്സ്റ്റ് മാത്രമാണ്. പ്രധാന ഘടകങ്ങൾ `SystemMessage` (ആർട്ടിഫിഷ്യൽ ഇന്റലിജൻസിന്റെ പെരുമാറ്റവും പദവിയും നിശ്ചയിക്കുന്നത്)യും `UserMessage` (നിങ്ങളുടെ യഥാർത്ഥ അഭ്യർത്ഥന ഉൾക്കൊള്ളുന്നത്)ഉം ആണ്.

## പ്രോംപ്റ്റ് എഞ്ചിനീയറിംഗ് അടിസ്ഥാനങ്ങള്‍

<img src="../../../translated_images/ml/five-patterns-overview.160f35045ffd2a94.webp" alt="Five Prompt Engineering Patterns Overview" width="800"/>

ഈ മോഡ്യൂളിലെ മുൻനിര പാറ്റേണുകളില്‍ പ്രവേശിക്കുന്നതിനു മുമ്പ്, അഞ്ചു അടിസ്ഥാന പ്രോംപ്റ്റിംഗ് സാങ്കേതികവിദ്യകള്‍ അവലോകനം ചെയ്യാം. ഓരോ പ്രോംപ്റ്റ് എഞ്ചിനീയറും അറിയേണ്ട അടിസ്ഥാന ഘടകങ്ങൾ ഇവയാണ്. നിങ്ങൾ ഇതിനകം [Quick Start മോഡ്യൂള്‍](../00-quick-start/README.md#2-prompt-patterns) കണ്ടിട്ടുണ്ടെങ്കിൽ ഇവയുടെ ചിന്താഗതികൾക്കു പിറകെ നേരത്തെ ഒരു ദൃശ്യഭാഗം കാണിച്ചിട്ടുണ്ട്.

### സീറോ-ഷോട്ട് പ്രോംപ്റ്റിംഗ്

ഏറ്റവും ലളിതമായ മാര്‍ഗം: ഉദാഹരണമില്ലാതെ മോഡലിന് നേരിട്ടുള്ള നിർദ്ദേശം നൽകുക. മോഡല്‍ പൂർണമായും പരിശീലനത്തിൽ നിന്നുള്ള വിവരം ആശ്രയിച്ച് ജോലി നിർവഹിക്കുന്നു. പ്രതീക്ഷിച്ച പെരുമാറ്റം വ്യക്തമായി അറിയാവുന്ന നേരിയ അഭ്യർത്ഥനകൾക്ക് ഇത് ഉത്തമമാണ്.

<img src="../../../translated_images/ml/zero-shot-prompting.7abc24228be84e6c.webp" alt="Zero-Shot Prompting" width="800"/>

*ഉദാഹരണങ്ങൾ ഇല്ലാതെ നേരിട്ടുള്ള നിർദ്ദേശം — നിർദ്ദേശം മാത്രം നോക്കി മോഡൽ ടാസ്ക്ക് തിരിച്ചറിയുന്നു*

```java
String prompt = "Classify this sentiment: 'I absolutely loved the movie!'";
String response = model.chat(prompt);
// പ്രതികരണം: "സൗകര്യം"
```

**എപ്പോള്‍ ഉപയോഗിക്കണം:** ലളിതമായ വർഗ്ഗീകരണങ്ങൾ, നേരിട്ടുള്ള ചോദ്യങ്ങൾ, പരിഭാഷകൾ, അല്ലെങ്കിൽ അധിക മാർഗ്ഗനിർദ്ദേശം ആവശ്യമില്ലാത്ത ഏതു ജോലി.

### ഫ്യൂ-ഷോട്ട് പ്രോംപ്റ്റിംഗ്

മോഡലിന് പിന്തുടരേണ്ട മാതൃക കാണിക്കുന്ന ഉദാഹരണങ്ങൾ നൽകുക. ഉദാഹരണങ്ങളിൽ നിന്നായി മോഡൽ പ്രതീക്ഷിക്കപ്പെട്ട ഇന്പുട്ട്-ഔട്ട്‌പുട്ട് ഫോർമാറ്റ് പഠിച്ച് പുതിയ ഇന്പുട്ടുകളിൽ പ്രയോഗിക്കുന്നു. ഇച്ഛිත ഫോർമാറ്റ് വ്യക്തമല്ലാത്ത ടിàsകളുടെ ഏറെ കാര്യക്ഷമത വർധിപ്പിക്കുന്നതിൻറെ മാർഗം.

<img src="../../../translated_images/ml/few-shot-prompting.9d9eace1da88989a.webp" alt="Few-Shot Prompting" width="800"/>

*ഉദാഹരണങ്ങളിലൂടെ പഠനം — മാതൃക തിരിച്ചറിയുകയും പുതുവരകളിൽ പ്രയോഗിക്കയും ചെയ്യുന്നു*

```java
String prompt = """
    Classify the sentiment as positive, negative, or neutral.
    
    Examples:
    Text: "This product exceeded my expectations!" → Positive
    Text: "It's okay, nothing special." → Neutral
    Text: "Waste of money, very disappointed." → Negative
    
    Now classify this:
    Text: "Best purchase I've made all year!"
    """;
String response = model.chat(prompt);
```

**എപ്പോള്‍ ഉപയോഗിക്കണം:** കസ്റ്റം വർഗ്ഗീകരണങ്ങൾ, സ്ഥിരം ഫോർമാറ്റിംഗ്, ഡൊമെയ്ൻ-സ്പെസിഫിക് ടാസ്‌കുകൾ, സീറോ-ഷോട്ട് ഫലങ്ങൾ അസംസ്ഥാനമായിരിക്കാൻ സാധ്യത ഉണ്ടാകുമ്പോൾ.

### ചെയിന് ഓഫ് തോട്ട്

മോഡലിന് ചിന്തിച്ചതിലുള്ള ഘട്ടങ്ങളായി വിശദീകരിക്കാൻ പറയുക. നേരിട്ട് ഉത്തരത്തിലേക്ക് ചാടി പോകാതെ മോദല്‍ പ്രശ്നം നന്നായി തിരിച്ചറിയുകയും ഓരോ ഘട്ടവും വ്യക്തമായി പ്രവർത്തിക്കുകയും ചെയ്യുന്നു. ഗണിതം, തർക്കം, ബഹുഘട്ട ചിന്ത ആവശ്യമായ ടാസ്‌കുകളിൽ കാര്യക്ഷമത വർധിപ്പിക്കുന്നു.

<img src="../../../translated_images/ml/chain-of-thought.5cff6630e2657e2a.webp" alt="Chain of Thought Prompting" width="800"/>

*ഘട്ടം-തുടർന്ന് കരുതല്‍ — സങ്കീർണമായ പ്രശ്നങ്ങൾ വിശദമായ തർക്ക ഘട്ടങ്ങളാക്കി വിഭജിക്കുന്നു*

```java
String prompt = """
    Problem: A store has 15 apples. They sell 8 apples and then 
    receive a shipment of 12 more apples. How many apples do they have now?
    
    Let's solve this step-by-step:
    """;
String response = model.chat(prompt);
// മോഡൽ കാണിക്കുന്നു: 15 - 8 = 7, പിന്നീട് 7 + 12 = 19 ആപ്പിളുകൾ
```

**എപ്പോള്‍ ഉപയോഗിക്കണം:** ഗണിത പ്രശ്നങ്ങൾ, തർക്ക ഗെയിംസ്, ഡീബഗ്ഗിംഗ്, അല്ലെങ്കിൽ ഉദ്ധാരണ പ്രശ്നങ്ങളിൽ സത്യസന്ധതയും വിശ്വാസയോഗ്യതയും വർദ്ധിപ്പിക്കാൻ.

### റോള്‍-അധിഷ്ഠിത പ്രോംപ്റ്റിംഗ്

നിങ്ങളുടെ ചോദ്യം മുൻപ് എഐയ്ക്ക് ഒരു വ്യക്തിത്വം അല്ലെങ്കിൽ പദവി നിശ്ചയിക്കുക. ഇതു മറുപടി നൽകാനുള്ള ശൈലി, ആഴം, ശ്രദ്ധ എന്നിവയെ സ്വാധീനിക്കും. "സോഫ്റ്റ്‌വെയർ ആർകിടെക്ട്" ഒരു "ജൂനിയർ ഡെവലപ്പർ" അല്ലെങ്കിൽ "സെക്യുരിറ്റി ഓഡിറ്റർ" തന്നെയുള്ള ഉപദേശങ്ങളെ അപേക്ഷിച്ച് വ്യത്യസ്തമായിരിക്കും.

<img src="../../../translated_images/ml/role-based-prompting.a806e1a73de6e3a4.webp" alt="Role-Based Prompting" width="800"/>

*സന്ദർഭവും വ്യക്തിത്വവും നിശ്ചയിക്കൽ — സമാനമായ ചോദ്യം പദവിയുടെ അടിസ്ഥാനത്തിൽ വ്യത്യസ്തമായ മറുപടി ലഭിക്കുന്നു*

```java
String prompt = """
    You are an experienced software architect reviewing code.
    Provide a brief code review for this function:
    
    def calculate_total(items):
        total = 0
        for item in items:
            total = total + item['price']
        return total
    """;
String response = model.chat(prompt);
```

**എപ്പോള്‍ ഉപയോഗിക്കണം:** കോഡ് റിവ്യൂകൾ, ട്യൂട്ടറിംഗ്, മേഖല പ്രത്യേതു വിശകലനം, അല്ലെങ്കിൽ ഏതെങ്കിലും പ്രത്യേക വിദഗ്ധ തലത്തിനോ കാഴ്ചപ്പാടിനോ അനുയോജ്യമായ മറുപടികൾ ആവശ്യമായപ്പോൾ.

### പ്രോംപ്റ്റ് ടെംപ്ലേറ്റുകള്‍

വ്യാപകമായി ഉപയോഗിക്കാവുന്ന പ്രോംപ്റ്റുകള്‍ വ്യത്യസ്തമായ ഡാറ്റയ്ക്കായി നാം ഉപയോഗിക്കാൻ കഴിയും വിധം മൊത്തത്തിലുള്ള ടെംപ്ലേറ്റുകൾ സൃഷ്ടിക്കുക. ഓരോ പ്രാവശ്യം പുതിയൊരു പ്രോംപ്റ്റ് എഴുതാതെ, ടെംപ്ലേറ്റ് ഒന്ന് നിർവചിച്ച് വ്യത്യസ്ത മൂല്യങ്ങൾ അത് നിറയ്ക്കുക. LangChain4j യുടെ `PromptTemplate` ക്ലാസ്സ് `{{variable}}` സിന്റാക്സുമായ് ഇത് എളുപ്പമാണ്.

<img src="../../../translated_images/ml/prompt-templates.14bfc37d45f1a933.webp" alt="Prompt Templates" width="800"/>

*വ്യത്യസ്ത മൂല്യങ്ങൾ കൊണ്ട് പൂരിപ്പിക്കാവുന്ന പുനരുപയോഗയോഗ്യ പ്രോംപ്റ്റുകള്‍*

```java
PromptTemplate template = PromptTemplate.from(
    "What's the best time to visit {{destination}} for {{activity}}?"
);

Prompt prompt = template.apply(Map.of(
    "destination", "Paris",
    "activity", "sightseeing"
));

String response = model.chat(prompt.text());
```

**എപ്പോള്‍ ഉപയോഗിക്കണം:** വിവിധ ഇന്പുട്ടുകളുള്ള ആവർത്തന പരാമർശങ്ങൾ, ബാച്ച് പ്രോസസ്സിംഗ്, പുനരുപയോഗാവുന്ന എ.ഐ. പ്രവൃത്തികൾ നിർമ്മിക്കൽ, അല്ലെങ്കിൽ പ്രോംപ്റ്റിന്റെ ഘടന തരം മാറ്റാതെ ഡാറ്റ മാറുമ്പോള്‍.

---

ഈ അഞ്ചു അടിസ്ഥാന സിദ്ധാന്തങ്ങൾ നിങ്ങളുടെ പ്രോംപ്റ്റിംഗ് ടാസ്‌കുകളുടെ ഭൂരിഭാഗത്തിനും ശക്തിയായ ഉപകരണങ്ങൾ നൽകുന്നു. ഈ മോഡ്യൂള്‍ അതിനുള്ളിൽ നിന്നു കടന്നുപോയി GPT-5.2 യുടെ കരുതല്‍ നിയന്ത്രണം, സ്വയം-മൂല്യനിർണ്ണയം, ഘടനാപരമായ ഔട്ട്പുട്ട് ശേഷി തുടങ്ങിയവ പ്രയോജനപ്പെടുത്തി **എട്ട് അധികമായ പാറ്റേണുകൾ** വികസിപ്പിക്കുന്നു.

## അധീനമായ പാറ്റേണുകള്‍

അടിസ്ഥാനങ്ങള്‍ പൂർത്തിയായപ്പോള്‍, ഈ മോഡ്യൂള്‍ അനന്യമായ മാറ്റനുള്ള എട്ട് പാറ്റേണുകളിൽ പ്രവേശിക്കാം. എല്ലാ പ്രശ്നങ്ങളും ഒരേ രീതിയിൽ പരിഹരിക്കേണ്ടതില്ല. ചില ചോദ്യങ്ങൾ ത്വരിതമായ മറുപടി വേണമെന്നു കാണിക്കുന്നു, മറ്റുമുണ്ട് ആഴത്തിലുള്ള ചിന്ത ആവശ്യപ്പെടുന്നത്. ചിലതിന് ദൃശ്യമായ കരുതല്‍ വേണം, മറ്റത് ഫലം മാത്രം. താഴെ കൊടുത്ത പാറ്റേണുകൾ ഓരോത് വ്യത്യസ്ത സാഹചര്യങ്ങളിലേക്കും ഏറെ അനുയോജ്യമാണ് — കൂടാതെ GPT-5.2 യുടെ കരുതൽ നിയന്ത്രണം വ്യത്യാസങ്ങൾ കൂടുതൽ വ്യക്തമാക്കുന്നു.

<img src="../../../translated_images/ml/eight-patterns.fa1ebfdf16f71e9a.webp" alt="Eight Prompting Patterns" width="800"/>

*എട്ട് പ്രോംപ്റ്റ് എഞ്ചിനീയറിംഗ് പാറ്റേണുകളുടെ പരാമർശവും ഉപയോഗ ক্ষেত്രങ്ങളും*

<img src="../../../translated_images/ml/reasoning-control.5cf85f0fc1d0c1f3.webp" alt="Reasoning Control with GPT-5.2" width="800"/>

*GPT-5.2 യുടെ കരുതൽ നിയന്ത്രണം മോഡലിന് എത്രകൂടി ചിന്തിക്കണമെന്ന് Specify ചെയ്യാൻ സഹായിക്കുന്നു — വേഗം നേരിട്ടുള്ള മറുപടികളിൽ നിന്ന് ആഴത്തിലുള്ള പരിശോധലുകൾ വരെ*

**കുറഞ്ഞ ഇച്ഛാശക്തി (ത്വരിതവും കേന്ദ്രീകൃതവും)** - എളുപ്പമുള്ള ചോദ്യങ്ങൾക്ക് വേഗം, നേരിട്ടുള്ള മറുപടികൾ വേണമെങ്കില്‍. മോഡൽ കുറഞ്ഞ കരുതൽ (പരമാവധി 2 ഘട്ടം) മാത്രമേ നടത്തൂ. കണക്കുകൂട്ടലുകൾ, നിന്നുള്ള അന്വേഷങ്ങൾ, സാദാരണ ചോദ്യങ്ങൾക്കു ഇത് ഉപയോഗിക്കുക.

```java
String prompt = """
    <context_gathering>
    - Search depth: very low
    - Bias strongly towards providing a correct answer as quickly as possible
    - Usually, this means an absolute maximum of 2 reasoning steps
    - If you think you need more time, state what you know and what's uncertain
    </context_gathering>
    
    Problem: What is 15% of 200?
    
    Provide your answer:
    """;

String response = chatModel.chat(prompt);
```

> 💡 **GitHub Copilot ഉപയോഗിച്ച് പരീക്ഷിക്കുക:** [Gpt5PromptService.java](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java) തുറന്ന് ചോദിക്കുക:
> - "കുറഞ്ഞ ഇച്ഛാശക്തിയും ഉയർന്ന ഇച്ഛാശക്തിയും പ്രോംപ്റ്റിംഗ് പാറ്റേണുകൾ തമ്മിലുള്ള വ്യത്യാസം എന്ത്?"
> - "പ്രോംപ്റ്റുകളിൽ XML ടാഗുകൾ എഐയുടെ മറുപടി ഘടിപ്പിക്കാൻ എങ്ങനെ സഹായിക്കുന്നു?"
> - "സ്വയം പരിഗണന പാറ്റേണുകൾ നേരിട്ട് നിർദ്ദേശത്തിനേക്കാൾ എപ്പോള്‍ ഉപയോഗിക്കണം?"

**ഉയർന്ന ഇച്ഛാശക്തി (ആഴമുള്ളതും സമഗ്രവുമാണ്)** - സങ്കീർണ്ണ പ്രശ്നങ്ങൾക്ക് തദനുസരിച്ച് സമഗ്രമായ വിശകലനം വേണമെന്നു. മോഡൽ ആഴത്തിൽ അന്വേഷിച്ച് വിശദമായ കരുതൽ കാണിക്കുന്നു. സിസ്റ്റം ഡിസൈൻ, ആർക്കിടെക്ചർ തീരുമാനങ്ങൾ, സങ്കീർണ്ണ ഗവേഷണങ്ങൾക്കു ഇത് ഉപയോഗിക്കുക.

```java
String prompt = """
    Analyze this problem thoroughly and provide a comprehensive solution.
    Consider multiple approaches, trade-offs, and important details.
    Show your analysis and reasoning in your response.
    
    Problem: Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```

**ടാസ്‌ക്ക് എക്സിക്യൂഷൻ (ഘട്ടം-തുടർന്ന് പുരോഗതി)** - ബഹുഘട്ട പ്രവൃത്തികള്ക്ക്. മോഡൽ മുന്നോട്ട് ഒരു പദ്ധതി നൽകുകയും, പ്രവർത്തന സമയത്ത് ഓരോ ഘട്ടവും പറയുകയും, അവസാനത്തിൽ സംഗ്രഹം നൽകുകയും ചെയ്യുന്നു. മൈഗ്രേഷൻ, നടപ്പിലാക്കൽ അല്ലെങ്കിൽ ബഹുഘട്ട പ്രവർത്തനങ്ങൾക്ക് ഉചിതം.

```java
String prompt = """
    <task_execution>
    1. First, briefly restate the user's goal in a friendly way
    
    2. Create a step-by-step plan:
       - List all steps needed
       - Identify potential challenges
       - Outline success criteria
    
    3. Execute each step:
       - Narrate what you're doing
       - Show progress clearly
       - Handle any issues that arise
    
    4. Summarize:
       - What was completed
       - Any important notes
       - Next steps if applicable
    </task_execution>
    
    <tool_preambles>
    - Always begin by rephrasing the user's goal clearly
    - Outline your plan before executing
    - Narrate each step as you go
    - Finish with a distinct summary
    </tool_preambles>
    
    Task: Create a REST endpoint for user registration
    
    Begin execution:
    """;

String response = chatModel.chat(prompt);
```

ചെയിന് ഓഫ് തോട്ട് പ്രോംപ്റ്റിംഗ് മോഡലിന് തന്റെ കരുതല്‍ ഘട്ടങ്ങൾ കാണിക്കാനാണ് നിർദ്ദേശം നൽകുന്നത്, ഇത് സങ്കീർണ്ണ ടാസ്‌ക്കുകളിൽ കൃത്യത വർദ്ധിപ്പിക്കുന്നു. ഘട്ടം-തുടർന്ന് തിരിച്ചറിവ് മനുഷ്യർക്കും എഐയ്ക്കും ലജിക് മനസ്സിലാക്കാൻ സഹായിക്കുന്നു.

> - "ദീർഘകാല വൃത്താന്തങ്ങൾക്കായി ടാസ്‌ക്ക് എക്സിക്യൂഷൻ പാറ്റേൺ എങ്ങനെ ക്രമീകരിക്കും?"
> - "പ്രൊഡക്ഷൻ അപ്ലിക്കേഷനുകളിൽ ടൂൾ പീമ്പിള്സ് ഘടിപ്പിക്കുന്നതിന് മികച്ച മാർഗ്ഗങ്ങൾ എന്തെല്ലാം?"
> - "ഇന്റർമീഡിയറ്റ് പ്രോഗ്രസ് അപ്‌ഡേറ്റുകൾ UI-യിൽ വെളിപ്പെടുത്താനും ക്യാപ്ചർ ചെയ്യാനും എങ്ങനെ കഴിയും?"

*യോജനം → നടപ്പാക്കല്‍ → സംഗ്രഹം ബഹുഘട്ട ടാസ്‌ക്കുകളിൽ*

**സ്വയം പരിഗണിക്കുന്ന കോഡ്** - പ്രൊഡക്ഷൻ-തസ്തറിയിലുള്ള കോഡ് സൃഷ്ടിക്കുവാനായി. മോഡൽ ഉത്കൃഷ്ടമായ പിഴവ് കൈകാര്യം ചെയ്യുന്നതോടുകൂടിയ പ്രൊഡക്ഷൻ നിലവാരത്തെ അനുസരിച്ച് കോഡ് സൃഷ്ടിക്കുന്നു. പുതിയ ഫീച്ചറുകൾ അല്ലെങ്കിൽ സേവനങ്ങൾ നിർമ്മിക്കുമ്പോൾ ഇതു ഉപയോഗിക്കുക.

```java
String prompt = """
```

*ഇടപെടുന്ന നവീകരണ ചക്രം - സൃഷ്ടിക്കുക, മൂല്യനിർണ്ണയം ചെയ്യുക, പ്രശ്നങ്ങൾ തിരിച്ചറിയുക, മെച്ചപ്പെടുത്തുക, ആവർത്തിക്കുക*

**സംഘടിത വിശകലനം** - സ്ഥിരം മൂല്യനിർമ്മാണത്തിനായി. മോഡൽ ഒരു സ്ഥിരമായ ഘടന (തെറ്റില്ലായ്മ, പ്രാക്ടീസുകൾ, പ്രകടനം, സുരക്ഷ, പരിപാലന ശേഷി) ഉപയോഗിച്ച് കോഡ് അവലോകനം ചെയ്യുന്നു. കോഡ് റിവ്യൂകൾക്കോ ഗുണമേന്‍മ പരിശോദനക്കോ ഇത് ഉപയോഗിക്കുക.

```java
String prompt = """
    ```

> - "വ്യത്യസ്ത തരത്തിലുള്ള കോഡ് റിവ്യൂസിനായി വിശകലന ഘടന എങ്ങനെ ഇഷ്ടാനുസൃതമാക്കാം?"
> - "ഘടിത ഔട്ട്പുട്ട് പ്രോഗ്രാമാറ്റിക് ആയി പാഴ്സും നടപടിക്രമവും എങ്ങനെ ഉറപ്പാക്കാം?"
> - "വ്യത്യസ്ത റിവ്യൂ സെഷനുകളിൽ സ്ഥിരമായ സി‌ടി‌വി നിലകൾ എങ്ങനെ ഉറപ്പാക്കാം?"

**മള്‍ട്ടി-ടേണ്‍ ചാറ്റ്** - സന്ദർഭം ആവശ്യമായ സംഭാഷണങ്ങൾക്ക്. മോഡൽ മുൻസന്ദേശങ്ങൾ ഓർത്തു തുടർന്നു വളർന്ന് സംസാരിക്കുന്നു. ഇന്ററാക്ടീവ് സഹായ സെഷനുകൾക്കോ സങ്കീർണ്ണ ചോദ്യോത്തരങ്ങള്കോ ഇത് ഉപയോഗിക്കുക.

**ഘട്ടം-തുടർന്ന് കരുതല്‍** - ദൃശ്യമായ ലജിക് ആവശ്യമുള്ള പ്രശ്നങ്ങൾക്ക്. മോഡൽ ഓരോ ഘട്ടത്തിനും വ്യക്തമായ കരുതല്‍ കാണിക്കുന്നു. ഗണിത, തർക്ക ഗെയിംസ്, അല്ലെങ്കിൽ ചിന്ത പ്രക്രിയ മനസ്സിലാക്കാനുള്ളിടത്തും ഇതു ഉപയോഗിക്കുക.

```java
String prompt = """

    then travels another 90 km in 1.5 hours, what is the average speed
    for the entire journey including the stop?
    """;

```

**പരിധി വിധേയമായ ഔട്ട്പുട്ട്** - പ്രത്യേക ഫോർമാറ്റും നീളവും ആവശ്യമായ മറുപടികൾക്കായി. മോഡൽ ഫോർമാറ്റ്, നീളം, ഘടനാ നിയമങ്ങൾ കർശനമായി പാലിക്കുന്നു. സംഗ്രഹങ്ങൾക്കോ കൃത്യമായ ഔട്ട്പുട്ട് ഘടന ആവശ്യമായ സാഹചര്യങ്ങൾക്ക്.


ഡെവ് കണ്ടെയ്നറിൽ സ്പ്രിംഗ് ബൂട്ട് ഡാഷ്ബോർഡ് എക്‌സ്റ്റൻഷൻ ഉൾപ്പെടുത്തി, ഇത് എല്ലാ സ്പ്രിംഗ് ബൂട്ട് ആപ്ലിക്കേഷനുകളെയും നിയന്ത്രിക്കാൻ ദൃശ്യമാക്കുന്ന ഇടവകയാണ്. ഇത് VS Code-യുടെ ഇടതു പക്കൽ പ്രവർത്തന ബാറിൽ (സ്പ്രിംഗ് ബൂട്ട് ഐക്കൺ നോക്കി) കാണാം.

സ്പ്രിംഗ് ബൂട്ട് ഡാഷ്ബോർഡിൽനിന്ന് നിങ്ങൾക്ക്:
- വര്‍ക്ക്‌സ്പേസിലെ എല്ലാ ലഭ്യമായ സ്പ്രിംഗ് ബൂട്ട് ആപ്ലിക്കേഷനുകൾ കാണാം
- ഒരൊറ്റ ക്ലിക്കിൽ ആപ്ലിക്കേഷനുകൾ സ്റ്റാർട്ട്/സ്റ്റോപ് ചെയ്യാം
- അപ്ലിക്കേഷൻ ലോഗുകൾ റിയൽ ടൈമിൽ കാണാം
- അപ്ലിക്കേഷൻ നില നിരീക്ഷണം നടത്താം
"prompt-engineering" നെ സമീപമുള്ള പ്ലേ ബട്ടൺ ക്ലിക്ക് ചെയ്യുക ഈ മോഡ്യൂൾ ആരംഭിക്കാൻ, അല്ലെങ്കില്‍ എല്ലാ മോഡ്യൂളുകളും ഒരുമിച്ചേ ആരംഭിക്കുക.

## പാറ്റേണുകൾ അന്വേഷിക്കൽ

വെബ് ഇന്റർഫേസ് വ്യത്യസ്ത പ്രോമ്പ്റ്റിംഗ് സ്ട്രാറ്റജികളുമായി പരീക്ഷണങ്ങൾ നടത്താൻ അനുവദിക്കുന്നു. ഓരോ പാറ്റേൺ വ്യത്യസ്ത പ്രശ്‌നങ്ങൾ പരിഹരിക്കുന്നു - ഓരോ സമീപനവും എപ്പോൾ മെച്ചമാണ് എന്ന് കാണാൻ അവ പരീക്ഷിക്കുക.

### കുറവ് vs ഉയർന്ന ആവേശം

"200-ന്റെ 15% എന്താണ്?" എന്ന ലളിതമായ ചോദ്യമേറ്റുക കുറഞ്ഞ ആവേശത്തോടെയാണ്. നിങ്ങൾക്ക് ഉടൻ, നേരിട്ടുള്ള ഉത്തരം ലഭിക്കും. ഇപ്പോൾ "ഉയർന്ന-ട്രാഫിക് API-ക്കായി കാഷ് സ്ട്രാറ്റജി രൂപകൽപ്പന ചെയ്യൂ" എന്ന് ഉയർന്ന ആവേശത്തോടെയാണ് ചോദിക്കുക. മോഡൽ കുറച്ചു സമയം ചെലവഴിക്കുകയും വിശദമായ തർക്കം നൽകുകയും ചെയ്യും. ഒരേ മോഡൽ, ഒരേ ചോദ്യ ഘടന - എന്നാൽ പ്രോമ്പ്റ്റ് എത്ര ചിന്തിക്കണെന്ന് വ്യക്തമാക്കുന്നു.

<img src="../../../translated_images/ml/low-eagerness-demo.898894591fb23aa0.webp" alt="Low Eagerness Demo" width="800"/>

*കുറഞ്ഞ തർക്കത്തോടെ വേഗത്തിലുള്ള കണക്ക്*

<img src="../../../translated_images/ml/high-eagerness-demo.4ac93e7786c5a376.webp" alt="High Eagerness Demo" width="800"/>

*സമ്പൂർണ്ണമായ കാഷ് സ്ട്രാറ്റജി (2.8MB)*

### ടാസ്ക് എക്സിക്യൂഷൻ (ടൂൾ പ്രീഅംബിളുകൾ)

മൾട്ടി-സ്റ്റെപ്പ് വേർക്‌ഫ്ലോകൾ മുൻകൂർ പദ്ധതി തയ്യാറാക്കലും പുരോഗതി വാക്കുതാളും സഹായിക്കുമെന്ന് കാണിച്ചു. മോഡൽ ചെയ്യാനിരിക്കുന്ന കാര്യങ്ങൾ വിശദീകരിക്കുകയും ഓരോ ഘട്ടവും വിവരിക്കുകയും, പിന്നെ ഫലം സംഗ്രഹിക്കുകയും ചെയ്യും.

<img src="../../../translated_images/ml/tool-preambles-demo.3ca4881e417f2e28.webp" alt="Task Execution Demo" width="800"/>

*ഓരോ ഘട്ടവും വിവരിച്ചുകൊണ്ടുള്ള REST എന്റ്പോയിന്റ് സൃഷ്ടി (3.9MB)*

### സ്വയം-പരിശോധന കോഡ്

"ഒരു ഇമെയിൽ സാധുത സേർവീസ് സൃഷ്ടിക്കുക" എന്ന് ശ്രമിക്കുക. കോഡ് സൃഷ്ടിച്ച് നിർത്താതെ മോഡൽ ഗുണമേന്മാ മാനദണ്ഡങ്ങളെ അടിസ്ഥാനമാക്കി വിലയിരുത്തുകയും ദുർബലതകൾ കണ്ടെത്തി മെച്ചപ്പെടുത്തുകയും ചെയ്യുന്നു. നിങ്ങൾ അതു പുനരാവൃത്തി ചെയ്ത് പ്രൊഡക്ഷൻ നിലയ്ക്ക് എത്തും വരെ കാണും.

<img src="../../../translated_images/ml/self-reflecting-code-demo.851ee05c988e743f.webp" alt="Self-Reflecting Code Demo" width="800"/>

*പൂർണ്ണമായ ഇമെയിൽ സാധുത സേർവീസ് (5.2MB)*

### ഘടനാപരമായ വിശകലനം

കോഡ് റിവ്യൂകൾക്കായി സ്ഥിരതയുള്ള മൂല്യനിർണയ ഫ്രെയിംവർക്ക് ആവശ്യമാണ്. മോഡൽ നിശ്ചിത വിഭാഗങ്ങൾ (ശുദ്ധി, പ്രാക്ടീസുകൾ, പ്രകടനം, സുരക്ഷ) പരിഗണിച്ച് കോഡ് വിശകലനം ചെയ്യുന്നു, ഗൗരവത്വം പ്രകാരം.

<img src="../../../translated_images/ml/structured-analysis-demo.9ef892194cd23bc8.webp" alt="Structured Analysis Demo" width="800"/>

*ഫ്രെയിംവർക്ക് അടിസ്ഥാനക്കൂടിയ കോഡ് റിവ്യൂ*

### മൾട്ടി-ടേൺ ചാറ്റ്

"Spring Boot എന്താണ്?" എന്ന് ചോദിച്ചു. ഉടനെ തന്നെ "ഒരു ഉദാഹരണം കാണിക്കൂ" എന്ന ചോദ്യം തുടരണവർഷം. മോഡൽ നിങ്ങളുടെ ആദ്യ ചോദ്യവും ഓർക്കുന്നു, പ്രത്യേകം Spring Boot ഉദാഹരണം നൽകുന്നു. ഓർമ്മയില്ലാതെയെങ്കിൽ രണ്ടാമത്തെ ചോദ്യം വായിക്കാൻ അപര്യാപ്തമായിരിക്കും.

<img src="../../../translated_images/ml/multi-turn-chat-demo.0d2d9b9a86a12b4b.webp" alt="Multi-Turn Chat Demo" width="800"/>

*ചോദ്യങ്ങൾ തമ്മിലുള്ള സാഹചര്യം ഓർക്കൽ*

### ഘട്ടം-ഘട്ടമായ ആശയവിനിമയം

ഒരു ഗണിത പ്രശ്‌നം തിരഞ്ഞെടുക്കുക, അതിനു Step-by-Step Reasoning നെ കുറവിന്റെ ആവേശത്തോടെയും പരീക്ഷിക്കുക. കുറവുള്ള ആവേശം നിങ്ങൾക്ക് ഉത്തരം മാത്രം വേഗത്തിൽ നൽകുന്നു - പക്ഷേ അപാരദർശിയാണ്. ഘട്ടം-ഘട്ടം എല്ലാം കണക്ക്, തീരുമാനം കാണിക്കുന്നു.

<img src="../../../translated_images/ml/step-by-step-reasoning-demo.12139513356faecd.webp" alt="Step-by-Step Reasoning Demo" width="800"/>

*ഘട്ടങ്ങൾ വ്യക്തമാക്കിയ ഗണിത പ്രശ്‌നം*

### നിയന്ത്രിത ഔട്ട്‌പുട്ട്

നിശ്ചിത ഫോർമാറ്റ് അല്ലെങ്കിൽ വാക്കുകളുടെ എണ്ണം ആവശ്യമായപ്പോൾ, ഈ പാടേൺ കർശനമായ അനുസരണം ഏർപ്പെടുത്തുന്നു. 100 വാക്കുകൾ ഉള്ള ബുള്ളറ്റ് പോയിന്റ് ഫോർമാറ്റിൽ സാരാംശം ഉണ്ടാക്കാൻ ശ്രമിക്കൂ.

<img src="../../../translated_images/ml/constrained-output-demo.567cc45b75da1633.webp" alt="Constrained Output Demo" width="800"/>

*ഫോർമാറ്റ് നിയന്ത്രണത്തോടെയുള്ള മെഷീൻ ലേണിംഗ് സാരാംശം*

## നിങ്ങൾ യാഥാർത്ഥ്യത്തിൽ പഠിക്കുന്നത്

**ചിന്താശേഷി എല്ലാം മാറ്റം വരുത്തുന്നു**

GPT-5.2 നിങ്ങളുടെ പ്രോമ്പ്റ്റുകൾ വഴി കണക്കുകൂട്ടൽ ശ്രമം നിയന്ത്രിക്കാൻ അനുവദിക്കുന്നു. കുറഞ്ഞ ശ്രമം അതിവേഗത്തിലായും കുറഞ്ഞപ്പൊതു പരിശോധനയോടെയും ഉത്തരം നൽകുന്നു. ഉയര്‍ന്ന ശ്രമം മോഡൽ കൂടുതൽ ആഴത്തിൽ ചിന്തിക്കാൻ സമയം ചെലവഴിക്കുന്നു. നിങ്ങൾ പഠിക്കുന്നത് ടാസ്കിന്റെ മൗലികത അനുസരിച്ച് ശ്രമം ക്രമീകരിക്കുകയാണ് - ലളിതമായ ചോദ്യങ്ങളിൽ സമയംWaste ചെയ്യരുത്, പക്ഷേ മുകൾഭാഗം തീരുമാനങ്ങൾ വേഗത്തിൽ എടുത്ത് വിടരുത്.

**ഘടന പെരുമാറ്റം നയിക്കുന്നു**

പ്രോമ്പ്റ്റുകളിലെ XML ടാഗുകൾ ശ്രദ്ധിച്ചിട്ടുണ്ടോ? അവ അലങ്കാരമല്ല. മോഡലുകൾ സുസംബന്ധമായ നിർദ്ദേശങ്ങൾ സ്വതന്ത്രരചന താൾക്കൊച്ച് ബന്ധം കഴിയും. മൾട്ടി-സ്റ്റെപ്പ് പ്രക്രിയകൾ അല്ലെങ്കിൽ സങ്കീർണ്ണമായ ലൊജിക്ക് ആവശ്യമായപ്പോൾ, ഘടന മോഡലിനുള്ള സ്ഥാനം ട്രാക്ക് ചെയ്യാനും സ്വന്തം അടുത്തതായി വേണമെന്നും അറിയാൻ സഹായിക്കുന്നു.

<img src="../../../translated_images/ml/prompt-structure.a77763d63f4e2f89.webp" alt="Prompt Structure" width="800"/>

*വിവരമാർന്ന അവയവങ്ങളോടും XML-ശൈലിയിലുള്ള സംഘാടക ഘടനയോടും കൂടി ഒരു നല്ല ഘടനയുള്ള പ്രോമ്പ്റ്റിന്റെ നിർമ്മിതം*

**സ്വയം-വിലയിരുത്തലിലൂടെ ഗുണമേന്മ**

സ്വയം-പരിശോധന പാറ്റേണുകൾ ഗുണമേന്മാ മാനദണ്ഡങ്ങൾ വ്യക്തതയിലാക്കിയാണ് പ്രവർത്തിക്കുന്നത്. മോഡൽ "ശരിയായി ചെയ്യും" എന്ന് പ്രതീക്ഷിക്കാതെ നിങ്ങള്‍ "ശരി" എന്നതുകൊണ്ടുള്ള അർത്ഥം വ്യക്തമാക്കണം: ശരിയായ ബാലൻസ്, ошибки കൈകാര്യം, പ്രകടനം, സുരക്ഷ. മോഡൽ പിന്നീട് തന്നെയായി ഔട്ട്പുട്ട് വിലയിരുത്തി മെച്ചപ്പെടുത്താം. ഇത് കോഡ് സൃഷ്ടിയെ ഭാഗ്യനിർണ്ണയത്തിൽ നിന്ന് പ്രക്രിയയാക്കി മാറ്റുന്നു.

**സന്ദർഭം പരിധിയുള്ളതാണ്**

മൾട്ടി-ടേൺ സംഭാഷണങ്ങൾ ഓരോ അഭ്യർത്ഥനയോടും സന്ദേശഹ്രസ്വത പകർന്നുകൊണ്ടാണ് പ്രവർത്തിക്കുന്നത്. എന്നാൽ പരിമിതവും ഉണ്ട് - ഓരോ മോഡലിനും പരമാവധി ടോക്കൺ പരിധിയുണ്ട്. സംഭാഷണങ്ങൾ വലുത് ആവുമ്പോൾ ബന്ധപ്പെട്ട സാന്ദർഭ്യം കൈവശം വയ്ക്കുന്ന സ്ട്രാറ്റജികൾ ആവശ്യമാകും അത്യാവശ്യമായ പരിധി തടയാതെ. ഈ മോഡ്യൂൾ നിങ്ങൾക്ക് ഓർമ്മ എങ്ങനെ പ്രവർത്തിക്കുന്നുവെന്ന് കാണിക്കുന്നു; പിന്നീട് നിങ്ങൾ തിരുമാനിക്കേണ്ട സമയങ്ങൾ, മറക്കേണ്ട സമയങ്ങൾ, തിരിച്ചെടുക്കേണ്ട സമയങ്ങൾ പഠിക്കും.
