## നിങ്ങള് പഠിക്കുന്നതെന്ത്

<img src="../../../translated_images/ml/what-youll-learn.c68269ac048503b2.webp" alt="What You'll Learn" width="800"/>

മുൻ മോഡ്യൂളിൽ, മെമ്മറി സംഭാഷണ ആർട്ടിഫിഷ്യൽ ഇന്റലിജൻസിന് എങ്ങനെ സഹായിക്കുന്നു എന്ന് നിങ്ങൾ കണ്ടു, കൂടാതെ GitHub മോഡലുകൾ ഉപയോഗിച്ച് അടിസ്ഥാന ഇടപാടുകൾ ചെയ്യാൻ പഠിച്ചു. ഇപ്പോൾ, നിങ്ങൾ എങ്ങനെ ചോദിക്കുന്നുവെന്ന് — പ്രോംപ്റ്റുകൾ — Azure OpenAI യുടെ GPT-5.2 ഉപയോഗിച്ച് നോക്കാം. നിങ്ങൾ എങ്ങനെ പ്രോംപ്റ്റുകൾ രൂപപ്പെടുത്തുന്നുവെന്ന് തന്നെ ലഭിക്കുന്ന മറുപടികളുടെ ഗുണമേന്മയെ വലിയ തോതിൽ ബാധിക്കുന്നു. അടിസ്ഥാന പ്രോംപ്റ്റിംഗ് സാങ്കേതികതകളുടെ അവലോകനത്തോടെ തുടങ്ങും, ശേഷം GPT-5.2 ന്റെ ശേഷികളെ പൂർണമായി ഉപയോഗിക്കുന്ന എട്ട് ഉന്നത മാതൃകകളിലേക്ക് പോകും.

നാം GPT-5.2 ഉപയോഗിക്കുന്നത് കാരണം ഇത് റിസണിങ്ങ് നിയന്ത്രണം അവതരിപ്പിക്കുന്നു - മറുപടി നൽകുന്നതിന് മുമ്പ് മോഡലിന് എത്ര ചിന്തനം ചെയ്യണമെന്ന് നിങ്ങൾ പറയാൻ കഴിയും. ഇത് വ്യത്യസ്ത പ്രോംപ്റ്റിംഗ് തന്ത്രങ്ങൾ കൂടുതൽ വ്യക്തമാക്കുന്നു, കൂടാതെ ഏത് സമീപനം എപ്പോഴാണ് ഉപയോഗിക്കേണ്ടെന്ന് മനസ്സിലാക്കാൻ സഹായിക്കുന്നു. GitHub മോഡലുകൾക്കൊപ്പം താരതമ്യമായി Azure യുടെ കുറവ് നിരക്കു പരിധികളും നമ്മെ സഹായിക്കും.

## ആവശ്യങ്ങൾ

- മոդ്യൂൾ 01 പൂർത്തിയാക്കി (Azure OpenAI റിസോഴ്‌സുകൾ ഉപയോഗത്തിലാക്കി)
- റൂട്ട് ഡയറക്ടറിയിൽ `.env` ഫയൽ Azure ക്രെഡൻഷ്യലുകളോടെ (മോദ്യൂൾ 01ൽ `azd up` ഉപയോഗിച്ച് സൃഷ്ടിച്ചത്)

> **നോട്ട്സ്:** മേഡ്യൂൾ 01 പൂർത്തിയാക്കിയിട്ടില്ലെങ്കിൽ, ആദ്യം അവിടെ നൽകിയ ഡിപ്പ്ലോയ്മെന്റ് നിർദേശങ്ങൾ പാലിക്കുക.

## പ്രോംപ്റ്റ് എൻജിനീയറിംഗ് മനസ്സിലാക്കൽ

<img src="../../../translated_images/ml/what-is-prompt-engineering.5c392a228a1f5823.webp" alt="What is Prompt Engineering?" width="800"/>

പ്രോംപ്റ്റ് എൻജിനീയറിംഗ് എന്നാൽ നിങ്ങൾക്ക് ആവശ്യമായ ഫലങ്ങൾ സ്ഥിരമായി ലഭിക്കുന്ന ഇൻപൂട് ടെക്സ്റ്റ് ഡിസൈൻ ചെയ്യലാണ്. ഇത് വെറും ചോദ്യങ്ങൾ ചോദിക്കുന്നതല്ല - മോഡൽ നിങ്ങൾ എന്ത് വേണമെന്ന് എങ്ങനെ ഉറ്റുകാണുന്നു എന്നും നൽകുന്നതിന് രൂപരേഖയിടുന്നതാണ്.

ഇത് ഒരു സഹപ്രവർത്തകനെ നിർദ്ദേശങ്ങൾ നൽകുന്നതുപോലെയാണ്. "ബഗ് പരിഹരിക്കുക" വളരെ അപൂർവ്വമാണ്. "UserService.java ലെ 45-ആം വരിയിൽ നൾ(pointer) എക്സെപ്ഷൻ പരിഹരിക്കൂ, ഒരു നൾ പരിശോധന ചേർത്ത്" എന്നത് കൃത്യമാണ്. ഭാഷാ മോഡലുകളും അതുപോലെ പ്രവർത്തിക്കുന്നു - കൃത്യതയും ഘടനയും അവശ്യമാണ്.

<img src="../../../translated_images/ml/how-langchain4j-fits.dfff4b0aa5f7812d.webp" alt="How LangChain4j Fits" width="800"/>

LangChain4j മോഡൽ കണക്ഷനുകൾ, മെമ്മറി, മെസേജ് തരങ്ങൾ എന്നിവ നൽകുന്ന അടിസ്ഥാനം പ്രദാനം ചെയ്യുന്നു — പ്രോംപ്റ്റ് മാതൃകകൾ ആ അടിസ്ഥാനത്തിലൂടെയാണ് കൃത്യമായ ടെക്സ്റ്റ് അയയ്ക്കുന്നത്. പ്രധാന ഘടകങ്ങൾ `SystemMessage` (AIയുടെ പെരുമാറ്റവും റോളും സജ്ജമാക്കുന്നു)യും `UserMessage` (നിങ്ങളുടെ യഥാർത്ഥ അഭ്യർത്ഥനയും എത്തിക്കുന്നു) ഉം ആണ്.

## പ്രോംപ്റ്റ് എൻജിനീയറിംഗ് അടിസ്ഥാനങ്ങൾ

<img src="../../../translated_images/ml/five-patterns-overview.160f35045ffd2a94.webp" alt="Five Prompt Engineering Patterns Overview" width="800"/>

ഈ മോദ്യൂളിലെ ഉന്നത മാതൃകകളിലേക്ക് പ്രവേശിക്കുന്നതിന് മുമ്പ്, അഞ്ചു അടിസ്ഥാന പ്രോംപ്റ്റിംഗ് സാങ്കേതികതകൾ പരിശോധിക്കാം. ഓരോ പ്രോംപ്റ്റ് എൻജിനിയർഷനും അറിയേണ്ട അടിസ്ഥാന ഘടകങ്ങൾ ഇവയാണ്. നിങ്ങൾ ഇതിനകം [ക്വിക്ക് സ്റ്റാർട്ട് മോദ്യൂൾ](../00-quick-start/README.md#2-prompt-patterns) ടൂട് കണ്ടിരിക്കുന്നു; അവയുടെ പിറകിൽ ഉള്ള ആശയഘടന ഇതാണ്.

### സീറോ-ഷോട്ട് പ്രോംപ്റ്റിംഗ്

ഏറ്റവും ലളിതമായ സമീപനം: ഉദ്ദിഷ്ട പ്രവർത്തനം തെളിയിക്കുന്ന ഉദാഹരണമില്ലാതെ സരാസായി നിർദ്ദേശിക്കുക. മോഡൽ പ്രവർത്തനം മനസിലാക്കാനും നടപ്പാക്കാനും സ്വയം പരിശീലനത്തെ ആശ്രയിക്കുന്നു. പ്രതീക്ഷിക്കുന്ന പെരുമാറ്റം രൂക്ഷമായ ഇടപെടലുകളിൽ ഇത് നല്ലത്.

<img src="../../../translated_images/ml/zero-shot-prompting.7abc24228be84e6c.webp" alt="Zero-Shot Prompting" width="800"/>

*ഉദാഹരണങ്ങളില്ലാതെ നേരെ നിർദ്ദേശം — നിർദ്ദേശത്തിൽ നിന്നുതന്നെ ടാസ്‌ക് താൻ മനസിലാക്കും*

```java
String prompt = "Classify this sentiment: 'I absolutely loved the movie!'";
String response = model.chat(prompt);
// പ്രതികരണം: "ധനാത്മകമായ"
```

**ഉപയോഗിക്കേണ്ടപ്പോൾ:** ലളിതമായ വർഗ്ഗീകരണങ്ങൾ, നേരെ ചോദ്യങ്ങൾ, വിവർത്തനങ്ങൾ, കൂടാതെ അധിക മാർഗ്ഗനിർദേശമില്ലാതെ മോഡലിന് ചുമതലയുള്ള ഏതെങ്കിലും ടാസ്‌കിനും.

### ഫ്യൂ-ഷോട്ട് പ്രോംപ്റ്റിംഗ്

മോഡലിന് പിന്തുടരാനുള്ള മാതൃക തെളിയിക്കുന്ന ഉദാഹരണങ്ങൾ നൽകുക. മോഡൽ നിങ്ങളുടെ ഉദാഹരണങ്ങളിൽ നിന്നും പ്രതീക്ഷിക്കുന്ന ഇൻപുട്ട്-ഔട്ട്‌പുട്ട് ഫോർമാറ്റ് പഠിച്ച് പുതിയ ഇൻപുട്ടുകളിൽ ഇത് പ്രയോഗിക്കും. ഇതു format അല്ലെങ്കിൽ പെരുമാറ്റം വ്യക്തമല്ലാത്ത ജോലി സ്ഥലങ്ങളിൽ സ്ഥിരത മെച്ചപ്പെടുത്തും.

<img src="../../../translated_images/ml/few-shot-prompting.9d9eace1da88989a.webp" alt="Few-Shot Prompting" width="800"/>

*ഉദാഹരണങ്ങളിൽ നിന്ന് പഠിച്ചു — മാതൃക തിരിച്ചറിഞ്ഞ് പുതിയ ഇൻപുട്ടിലേക്ക് പ്രയോഗിക്കുന്നു*

```java
String prompt = """
    Classify the sentiment as positive, negative, or neutral.
    
    Examples:
    Text: "This product exceeded my expectations!" → Positive
    Text: "It's okay, nothing special." → Neutral
    Text: "Waste of money, very disappointed." → Negative
    
    Now classify this:
    Text: "Best purchase I've made all year!"
    """;
String response = model.chat(prompt);
```

**ഉപയോഗിക്കേണ്ടപ്പോൾ:** ഇഷ്ടാനുസൃത വർഗ്ഗീകരണങ്ങൾ, സ്ഥിര ഫോംമാറ്റിംഗ്, വിഷയം-നിർദ്ധാരിത ടാസ്‌കുകൾ, അല്ലെങ്കിൽ സീറോ-ഷോട്ട് ഫലങ്ങൾ സ്ഥിരമല്ലെങ്കിൽ.

### ചെയിൻ ഓഫ് തോട്ടു

മോഡൽ തന്റെ തർക്കനം പടി പടിയായി കാണിക്കാൻ പറയുക. നേരെ ഉത്തരം നൽകാൻ പകരം, മോഡൽ പ്രശ്നം വിഭജിച്ച് ഓരോ ഘടകവും വ്യക്തമാക്കുന്നു. ഇത് ഗണിതം, തർക്ക്, മൾട്ടി-സ്റ്റെപ്പ് റിസണിങ്ങ് ടാസ്‌കുകളിൽ കൃത്യത വർദ്ധിപ്പിക്കും.

<img src="../../../translated_images/ml/chain-of-thought.5cff6630e2657e2a.webp" alt="Chain of Thought Prompting" width="800"/>

*നടത്തിയ ഘട്ടങ്ങൾ ഒറ്റ ഒറ്റ തിരിച്ചറിവിൽ — സങ്കീർണ്ണ പ്രശ്നങ്ങൾ വിശദമായ ലాజിക്കൽ ഘട്ടങ്ങളായി വിപുലീകരിക്കുന്നു*

```java
String prompt = """
    Problem: A store has 15 apples. They sell 8 apples and then 
    receive a shipment of 12 more apples. How many apples do they have now?
    
    Let's solve this step-by-step:
    """;
String response = model.chat(prompt);
// മോഡൽ കാണിക്കുന്നു: 15 - 8 = 7, തുടർന്ന് 7 + 12 = 19 ആപ്പിളുകൾ
```

**ഉപയോഗിക്കേണ്ടപ്പോൾ:** ഗണിത പ്രശ്നങ്ങൾ, തർമ, പിഴവുകൾ തിരയൽ, അല്ലെങ്കിൽ തർക്കന പ്രക്രിയ കാണിക്കുന്നത് കൃത്യതയും വിശ്വാസ്യതയും വർദ്ധിപ്പിക്കുന്ന ടാസ്കുകൾ.

### റോൾ-ബേസ്ഡ് പ്രോംപ്റ്റിംഗ്

നിങ്ങളുടെ ചോദ്യത്തിന് മുൻപ് AI-ക്ക് ഒരു വ്യക്തിത്വമോ റോളോ ഏർപ്പെടുത്തുക. ഇതു മറുപടിയുടെ ശൈലി, ആഴം, ശ്രദ്ധ മാറി തീർക്കുന്നു. "സോഫ്റ്റ്വെയർ ആർക്കിടെക്ട്" ഒരു "ജൂനിയർ ഡെവലപ്പർ" അല്ലെങ്കിൽ "സുരക്ഷാ ഓഡിറ്റർ" എന്ന തരം ആളുo പോലെ വ്യത്യസ്ത ഉപദേശം നൽകും.

<img src="../../../translated_images/ml/role-based-prompting.a806e1a73de6e3a4.webp" alt="Role-Based Prompting" width="800"/>

*കാര്യം സജ്ജീകരിക്കൽ — പകർന്ന റോളുകൾ അനുസരിച്ച് ഒരേ ചോദ്യത്തിന് വ്യത്യസ്ത മറുപടി ലഭിക്കും*

```java
String prompt = """
    You are an experienced software architect reviewing code.
    Provide a brief code review for this function:
    
    def calculate_total(items):
        total = 0
        for item in items:
            total = total + item['price']
        return total
    """;
String response = model.chat(prompt);
```

**ഉപയോഗിക്കേണ്ടപ്പോൾ:** കോഡ് റിവ്യൂകൾ, ട്യൂട്ടറിംഗ്, വിഷയാസന്നിധി വിശകലനങ്ങൾ, അല്ലെങ്കിൽ പ്രത്യേക വിദഗ്ധതാ നില അല്ലെങ്കിൽ പരാമർശം ആവശ്യമായ വെളിപ്പെടുത്തൽ.

### പ്രോംപ്റ്റ് ടെംപ്ലേറ്റുകൾ

വിവിധ മൂല്യങ്ങൾക്കായി വീണ്ടെടുക്കാവുന്ന പ്രോംപ്റ്റുകൾ സൃഷ്ടിക്കുക. ഓരോ പ്രാവശ്യവും പുതിയ പ്രോംപ്റ്റ് എഴുതിസമ്മർശിക്കാനുള്ള പകരം, ഒരിക്കൽ ടെംപ്ലേറ്റ് നിർവ്വചിച്ച് വ്യത്യസ്ത ഡാറ്റ ഉപയോഗിക്കുക. LangChain4jയുടെ `PromptTemplate` ക്ലാസ് `{{variable}}` സിന്റാക്സിനൊപ്പം ഇത് എളുപ്പമാക്കുന്നു.

<img src="../../../translated_images/ml/prompt-templates.14bfc37d45f1a933.webp" alt="Prompt Templates" width="800"/>

*വേരിയബിൾ പ്ലേസ്‌ഹോൾഡറുകളുള്ള വീണ്ടെടുക്കാവുന്ന പ്രോംപ്റ്റുകൾ — ഒരു ടെംപ്ലേറ്റ്, നിരവധി ഉപയോഗങ്ങൾ*

```java
PromptTemplate template = PromptTemplate.from(
    "What's the best time to visit {{destination}} for {{activity}}?"
);

Prompt prompt = template.apply(Map.of(
    "destination", "Paris",
    "activity", "sightseeing"
));

String response = model.chat(prompt.text());
```

**ഉപയോഗിക്കേണ്ടപ്പോൾ:** വ്യത്യസ്ത ഇൻപുട്ടുകൾ ഉള്ള ആവർത്തിത ക്വെറിയുകൾ, ബാച്ച് പ്രോസസ്സിങ്, വീണ്ടെടുക്കാവുന്ന AI പ്രവൃത്തികൾ സൃഷ്ടിക്കൽ, പ്രോംപ്റ്റ് ഘടന മാറാതെ ഡാറ്റ മാത്രം മാറുമ്പോൾ.

---

ഈ അഞ്ചു അടിസ്ഥാനങ്ങൾ ഒരു വലുത് ടൂൾകിറ്റ് നൽകുന്നു. മേശയിൽ ശേഷിച്ചിരിക്കുന്ന മോദ്യൂളിന്റെ ബാക്കിയുള്ള ഭാഗം GPT-5.2-ന്റെ റിസണിങ് നിയന്ത്രണം, സ്വയം-വിലയിരുത്തൽ, ഘടിത ഔട്ട്പുട്ട് കഴിവുകൾ ഉപയോഗിക്കുന്ന **എട്ട് ഉന്നത മാതൃകകൾ** നൽകുന്നു.

## ഉന്നത മാതൃകകൾ

അടിസ്ഥാനങ്ങൾ മറികടന്നതിനു ശേഷം, ഈ മോദ്യൂൾ പ്രത്യേകമാക്കുന്ന എട്ട് ഉന്നത മാതൃകകളിലേക്ക് പോവാം. എല്ലാകാര്യത്തിനും ഒരേ സമീപനം ആവശ്യമില്ല. ചില ചോദ്യങ്ങൾക്ക് വേഗമുള്ള ഉത്തരങ്ങൾ വേണം, ചിലത് ദീർഘചിന്ത പ്രവർത്തനം. ചിലത് ദൃശ്യമായ തർക്കനം, മറ്റേതൊക്കെയും ഫലങ്ങൾ മാത്രം. താഴെ പറയുന്ന ഓരോ മാതൃകയും വ്യത്യസ്ത സന്നിവേശത്തിനായി മെച്ചപ്പെടുത്തിയതാണ് — GPT-5.2-ന്റെ റിസണിങ് നിയന്ത്രണം വ്യത്യാസങ്ങൾ കൂടുതൽ വ്യക്തമായി തീർക്കുന്നു.

<img src="../../../translated_images/ml/eight-patterns.fa1ebfdf16f71e9a.webp" alt="Eight Prompting Patterns" width="800"/>

*എട്ട് പ്രോംപ്റ്റ് എൻജിനീയറിങ് മാതൃകകൾക്കും അവയുടെ ഉപയോഗ സാഹചര്യങ്ങളുടെ അവലോകനം*

<img src="../../../translated_images/ml/reasoning-control.5cf85f0fc1d0c1f3.webp" alt="Reasoning Control with GPT-5.2" width="800"/>

*GPT-5.2-ന്റെ റിസണിങ് നിയന്ത്രണം നിങ്ങൾക്ക് മോഡലിന് എത്ര ചിന്തനം ചെയ്യണമെന്നു തിരഞ്ഞെടുക്കാൻ അനുവദിക്കുന്നു — വേഗത്തിൽ നേരെ മറുപടികൊടുക്കൽ മുതൽ ദീർഘഗവേഷണം വരെ*

**ലോ എഗർനെസ് (വേഗം & കേന്ദ്രീകൃതം)** - എളുപ്പമേറിയ ചോദ്യങ്ങൾക്ക് വേഗത്തിലുള്ള നേരെ മറുപടികൾ വേണമെങ്കിൽ. മോഡൽ കൃത്യമായ ചിന്തനം കുറഞ്ഞുതന്നെ (ആക максимум 2 ഘട്ടം) നടത്തും. കണക്കുകൂട്ടൽ, ഡാറ്റാന്വേഷണം, ലളിത ചോദ്യങ്ങൾക്കായി ഇതുപയോഗിക്കുക.

```java
String prompt = """
    <context_gathering>
    - Search depth: very low
    - Bias strongly towards providing a correct answer as quickly as possible
    - Usually, this means an absolute maximum of 2 reasoning steps
    - If you think you need more time, state what you know and what's uncertain
    </context_gathering>
    
    Problem: What is 15% of 200?
    
    Provide your answer:
    """;

String response = chatModel.chat(prompt);
```

> 💡 **GitHub Copilot ഉപയോഗിച്ച് പരീക്ഷിയ്ക്കൂ:** [`Gpt5PromptService.java`](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java) തുറക്കുക, ചോദിക്കൂ:
> - "ലോ എഗർനെസ്-ഉം ഹൈ എഗർനെസ് പ്രോംപ്റ്റിംഗ് മാതൃകകളിലും有什么 വ്യത്യാസം?"
> - "പ്രോംപ്റ്റിലെ XML ട്യാഗുകൾ എങ്ങനെ AI-യുടെ മറുപടി ഘടിപ്പിക്കാൻ സഹായിക്കുന്നു?"
> - "സ്വയം-പരിശോധന മാതൃകകൾ vs നേരെ നിർദ്ദേശം എപ്പോൾ ഉപയോഗിക്കണം?"

**ഹൈ എഗർനെസ് (ഗഹനം & സമഗ്രം)** - സങ്കീർണ്ണ പ്രശ്നങ്ങൾക്ക് സമഗ്രമായ വിശകലനം വേണ്ടപ്പോൾ. മോഡൽ പൂർണ്ണമായി അന്വേഷിച്ച് വിശദമായ തർക്കനം കാണിക്കും. സിസ്റ്റം ഡിസൈൻ, ആർക്കിടെക്ചർ തീരുമാനം, ഗഹന ഗവേഷണം എന്നിവക്ക് ഉപയോഗിക്കൂ.

```java
String prompt = """
    Analyze this problem thoroughly and provide a comprehensive solution.
    Consider multiple approaches, trade-offs, and important details.
    Show your analysis and reasoning in your response.
    
    Problem: Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```

**ടാസ്‌ക് എക്സിക്യൂഷൻ (പടി പടിയായി പുരോഗതി)** - മൾട്ടി-സ്റ്റെപ്പ് പ്രവൃത്തികൾക്കായി. മോഡൽ ആദ്യം പദ്ധതി നൽകും, യഥാക്രമം ഓരോ ഘട്ടവും വിവരിക്കുന്നു, ശേഷം സംക്ഷേപം നൽകുന്നു. മൈഗ്രേഷൻ, നടപ്പാക്കൽ, അല്ലെങ്കിൽ ഏതു മൾട്ടി-സ്റ്റെപ്പ് പ്രക്രിയക്കും ഉപയോഗിക്കുക.

```java
String prompt = """
    <task_execution>
    1. First, briefly restate the user's goal in a friendly way
    
    2. Create a step-by-step plan:
       - List all steps needed
       - Identify potential challenges
       - Outline success criteria
    
    3. Execute each step:
       - Narrate what you're doing
       - Show progress clearly
       - Handle any issues that arise
    
    4. Summarize:
       - What was completed
       - Any important notes
       - Next steps if applicable
    </task_execution>
    
    <tool_preambles>
    - Always begin by rephrasing the user's goal clearly
    - Outline your plan before executing
    - Narrate each step as you go
    - Finish with a distinct summary
    </tool_preambles>
    
    Task: Create a REST endpoint for user registration
    
    Begin execution:
    """;

String response = chatModel.chat(prompt);
```

ചെയിൻ ഓഫ് തോട്ടു പ്രോംപ്റ്റിംഗ് മോഡലിന് തർക്കന പ്രക്രിയ പടി പടിയായി കാണിക്കാൻ നിർദേശിക്കുന്നു, ഇത് സങ്കീർണ്ണ ടാസ്‌കുകൾക്ക് കൃത്യത വർദ്ധിപ്പിക്കുന്നു. ഘട്ടങ്ങളായി വിഭജിക്കൽ മനുഷ്യർക്കും AI-ക്കും ലാജിക് മനസ്സിലാക്കാൻ സഹായിക്കുന്നു.

> **🤖 [GitHub Copilot](https://github.com/features/copilot) ചാറ്റിൽ പരീക്ഷിയ്ക്കൂ:** ഈ മാതൃകയെക്കുറിച്ച് ചോദിക്കാം:
> - "ദീർഘകാല ഓപ്പറേഷനുകൾക്കായി ടാസ്‌ക് എക്സിക്യൂഷൻ മാതൃകം എങ്ങനെ രൂപപ്പെടുത്തിയാൽ ശരിയാകും?"
> - "പ്രൊഡക്ഷൻ ആപ്ലിക്കേഷനുകളിൽ ടൂൾ പ്രീഅംബിളുകൾ ഘടിപ്പിക്കുന്ന മികച്ച രീതികൾ എന്തെന്ത്?"
> - "UIയിൽ ഇടവേളയിലുള്ള പുരോഗതി അപ്‌ഡേറ്റുകൾ എങ്ങനെ കൈകാര്യം ചെയ്ത് കാണിക്കാം?"

<img src="../../../translated_images/ml/task-execution-pattern.9da3967750ab5c1e.webp" alt="Task Execution Pattern" width="800"/>

*പദ്ധതി → നടപ്പാക്കല് → സംക്ഷേപം മൾട്ടി-സ്റ്റെപ്പ് ടാസ്‌കുകൾക്കായി*

**സ്വയം-പരിശോധന കോഡ്** - ഉൽപ്പാദന നിലവാരമുള്ള കോഡ് സൃഷ്ടിക്കാൻ. മോഡൽ ഉൽപ്പാദന മാനദണ്ഡങ്ങൾ പാലിക്കുന്ന കോഡ് ലീക്ക് തെറ്റായ നിയന്ത്രണത്തോടെ സൃഷ്ടിക്കും. പുതുമുഖ ഫീച്ചറുകൾ അല്ലെങ്കിൽ സേവനങ്ങൾ നിർമ്മിക്കുമ്പോൾ ഉപയോഗിക്കുക.

```java
String prompt = """
    Generate Java code with production-quality standards: Create an email validation service
    Keep it simple and include basic error handling.
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/ml/self-reflection-cycle.6f71101ca0bd28cc.webp" alt="Self-Reflection Cycle" width="800"/>

*മാറ്റം തുടരുന്ന മെച്ചപ്പെടുത്തൽ ചക്രം - സൃഷ്ടിക്കുക, വിലയിരുത്തുക, പ്രശ്നങ്ങൾ കണ്ടെത്തുക, മെച്ചപ്പെടുത്തുക, ആവർത്തിക്കുക*

**ഘടിത വിശകലനം** - സ്ഥിരം വിലയിരുത്തലിനായി. മോഡൽ നിര്‍ണായകമായ ഫ്രെയിംവർക്ക് അനുസരിച്ച് കോഡ് വിലയിരുത്തും (ശുദ്ധത, പ്രവൃത്തിപാടവം, പ്രകടനം, സുരക്ഷ, പരിപാലനക്ഷമത). കോഡ് റിവ്യൂകൾ അല്ലെങ്കിൽ ഗുണനിലവാര വിലയിരുത്തലിനായി ഉപയോഗിക്കുക.

```java
String prompt = """
    <analysis_framework>
    You are an expert code reviewer. Analyze the code for:
    
    1. Correctness
       - Does it work as intended?
       - Are there logical errors?
    
    2. Best Practices
       - Follows language conventions?
       - Appropriate design patterns?
    
    3. Performance
       - Any inefficiencies?
       - Scalability concerns?
    
    4. Security
       - Potential vulnerabilities?
       - Input validation?
    
    5. Maintainability
       - Code clarity?
       - Documentation?
    
    <output_format>
    Provide your analysis in this structure:
    - Summary: One-sentence overall assessment
    - Strengths: 2-3 positive points
    - Issues: List any problems found with severity (High/Medium/Low)
    - Recommendations: Specific improvements
    </output_format>
    </analysis_framework>
    
    Code to analyze:
    ```
    public List getUsers() {
        return database.query("SELECT * FROM users");
    }
    ```
    Provide your structured analysis:
    """;

String response = chatModel.chat(prompt);
```

## പാറ്റേണുകൾ കണ്ടെത്തൽ

വെബ് ഇന്റർഫേസ് വ്യത്യസ്ത പ്രൊംപ്റ്റിംഗ് തന്ത്രങ്ങൾ പരീക്ഷിക്കാൻ അനുവദിക്കുന്നു. ഓരോ പാറ്റേണും വ്യത്യസ്ത പ്രശ്നങ്ങൾ പരിഹരിക്കുന്നു - ഓരോ സമീപനവും ഏപ്പോൾ ഉജ്ജ്വലമാകുന്നുവെന്ന് കാണാൻ ശ്രമിക്കുക.

> **കുറിപ്പ്: സ്റ്റ്രീമിംഗ് vs നോൺ-സ്റ്റ്രീമിംഗ്** — ഓരോ പാറ്റേൺ പേജിലും രണ്ട് ബട്ടണുകൾ ഉണ്ട്: **🔴 Stream Response (Live)** എന്നതും **നോൺ-സ്റ്റ്രീമിംഗ്** ഓപ്ഷനും. സ്റ്റ്രീമിംഗ് സർവർ-സെന്റ് ഇവൻറ്സ് (SSE) ഉപയോഗിച്ച് മോഡൽ ടോക്കണുകൾ ജീവിച്ചിരിക്കുന്നോ പോലെ പ്രദർശിപ്പിക്കുന്നു, അതിനാൽ നിങ്ങൾക്ക് പോലെModel സൃഷ്‌ടിക്കുന്നത് ഉടനെ കാണാം. നോൺ-സ്റ്റ്രീമിംഗ് ഓപ്ഷൻ മുഴുവൻ മറുപടി ലഭിക്കുന്നതിന് ശേഷം പ്രദർശിപ്പിക്കുന്നു. গভീരമായ ചിന്ത ആവശ്യമായ പ്രൊംപ്റ്റുകൾക്കായി (ഉദാ. High Eagerness, Self-Reflecting Code), നോൺ-സ്റ്റ്രീമിംഗ് കോൾ വളരെ യുവിക്കാം - ചിലപ്പോൾ മിനിറ്റുകൾ വരെ - ദൃശ്യമായ പ്രതികരണം ഇല്ലാതെ. **കുട്ടി പ്രൊംപ്റ്റുകൾ പരീക്ഷിക്കുമ്പോൾ സ്റ്റ്രീമിംഗ് ഉപയോഗിക്കുക** മോഡൽ പ്രവർത്തിക്കുന്നത് കാണാൻ നിർബന്ധമാക്കാൻ കൂടാതെ അഭ്യർത്ഥന ടൈംഔട്ട് ആണെന്ന ആശയം ഒഴിവാക്കാൻ.
>
> **കുറിപ്പ്: ബ്രൗസർ ആവശ്യം** — സ്റ്റ്രീമിംഗ് ഫീച്ചർ Fetch Streams API (`response.body.getReader()`) ഉപയോഗിക്കുന്നു, ഇതിന് ഫുൾ ബ്രൗസർ (Chrome, Edge, Firefox, Safari) ആവശ്യമാണ്. VS Code-ലെ ഇൻബിൽറ്റ് സിമ്പിൾ ബ്രൗസറിൽ പ്രവർത്തിക്കുന്നില്ല, കാരണം അതിന്റെ വെബ്‌വ്യൂ ReadableStream API പിന്തുണയ്ക്കുന്നില്ല. സിമ്പിൾ ബ്രൗസർ ഉപയോഗിച്ചാൽ നോൺ-സ്റ്റ്രീമിംഗ് ബട്ടണുകൾ സാധാരണ പ്രവർത്തിക്കും — സ്റ്റ്രീമിംഗ് ബട്ടണുകൾ മാത്രമാണ് ബാധിക്കപ്പെടുന്നത്. പൂർണ്ണ അനുഭവത്തിനായി http://localhost:8083 ഒരു ബാഹ്യ ബ്രൗസറിൽ തുറക്കുക.

### കുറവ് vs ഉയർന്ന ആഗ്രഹം

"200ന്റെ 15% എന്താണ്?" പോലൊരു ലളിതമായ ചോദ്യത്തോട് കുറവ് ആഗ്രഹം ഉപയോഗിച്ച് ചോദിക്കുക. നിങ്ങൾക്ക് ഉടൻ, നേരിട്ടുള്ള ഉത്തരം ലഭിക്കും. ഇപ്പോൾ "ഉയർന്ന ട്രാഫിക് APIയ്ക്കുള്ള ക്യാഷിങ് തന്ത്രം രൂപകൽപ്പന ചെയ്യുക" പോലൊരു സങ്കീർണ്ണമായ ചോദ്യവുമായി ഉയർന്ന ആഗ്രഹം ഉപയോഗിച്ച് ചോദിക്കുക. **🔴 Stream Response (Live)** ക്ലിക്ക് ചെയ്ത് മോഡലിന്റെ വിശദമായ ചിന്ത പദം പദം കാണുക. ഒരു മോഡൽ, ഒരേ ചോദ്യരൂപം - എന്നാൽ പ്രൊംപ്റ്റ് മഡലിന് എത്ര ചിന്തിക്കണമെന്ന് പറയുന്നു.

### ടാസ്‌ക് എക്സിക്യൂഷൻ (ടൂൾ പ്രീഅംബിൾസ്)

ബഹു-പടി പ്രവൃത്തികൾ മുന്നോട്ടുള്ള പദ്ധതീകരണവും പുരോഗതി വിവരണവും ഉപയോഗിച്ച് ലാഭം പെടുന്നു. മോഡൽ ചെയ്യേണ്ടതും പടി പടി വിവരണം നൽകുകയും ശേഷം ഫലങ്ങൾ സംഗ്രഹിക്കുകയും ചെയ്യും.

### സെൽഫ്-റിഫ്ലെക്ടിങ്ങ് കോഡ്

"ഇമെയിൽ വാലിഡേഷൻ സർവീസ് സൃഷ്ടിക്കുക" ശ്രമിക്കുക. കോഡ് സൃഷ്ടിച്ച് നിർത്തുന്നതിന് പകരം, മോഡൽ നിർമ്മിക്കുകയും ഗുണമേൻമ മാനദണ്ഡങ്ങളുടെ അടിസ്ഥാനത്തിൽ വിലയിരുത്തുകയും, ദുർബലതകൾ കണ്ടെത്തുകയും മെച്ചപ്പെടുത്തുകയുമാണ്. കോഡ് ഉത്പാദന നിലവാരത്തെ എത്തുന്നതുവരെ ഇത് ആവർത്തിക്കാൻ നിങ്ങൾ കാണും.

### ഘടനാത്മക വിശകലനം

കോഡ് അവലോകനങ്ങൾക്ക് ഏകീകൃത മൂല്യനിർണയ ചട്ടങ്ങൾ ആവശ്യമുണ്ട്. മോഡൽ കോഡ് വിശകലനം നിശ്ചിത വിഭാഗങ്ങൾ (ശുദ്ധത, പ്രാധാന്യങ്ങൾ, പ്രകടനം, സുരക്ഷ) ഉപയോഗിച്ച് ഗുരുതരത്വ തരം നിർണ്ണയിച്ച് നടത്തുന്നു.

### മൾട്ടി-ടേൺ ചാറ്റ്

"Spring Boot എന്താണ്?" ചോദിച്ച് ഉടനെ "ഒരു ഉദാഹരണം കാണിക്കൂ" എന്നെ پرسിക്കുക. മോഡൽ നിങ്ങളുടെ പ്രഥമ ചോദ്യമനസ്റാക്കുന്നു, പ്രത്യേകിച്ച് Spring Boot ഉദാഹരണം നൽകുന്നു. ഓർമ്മശക്തിയില്ലാതെ, രണ്ടാമത്തെ ചോദ്യവും വാഗ്ദാനം നൽകാത്തതായിരിക്കും.

### പടി-പടിയായി ചിന്തിക്കുക

ഒരു ഗണിത പ്രശ്നം തിരഞ്ഞെടുക്കുക, Step-by-Step Reasoning ഫലവും കുറവായ ആഗ്രഹത്തോടെയുള്ള ഫലവും പരീക്ഷിക്കുക. കുറവായ ആഗ്രഹം ഈജിയായി മറുപടി നൽകുന്നു - വേഗത്തിൽ പക്ഷേ പുഞ്ചിരിയായ مقرപ്പിലും. പടി-പടി ഓരോ കണക്കുകളും തീരുമാനപ്പെടലും കാണിച്ചുകൊടുക്കും.

### നിയന്ത്രിച്ച ഔട്ട്പുട്ട്

നിർദിഷ്ട ഫോർമാറ്റുകളും വാക്കുകളുടെ അളവുകളുള്ളപ്പോൾ, ഈ പാറ്റേൺ കടുത്ത വിധേയത്വം നിർബന്ധിക്കുന്നു. കൃത്യമായി 100 വാക്കുകളിൽ ബുള്ളറ്റ് പോയിന്റ് ഫോർമാറ്റിൽ സംഗ്രഹം സൃഷ്ടിക്കാൻ ശ്രമിക്കുക.

## നിങ്ങൾ യഥാർത്ഥത്തിൽ പഠിക്കുന്നത്

**ചിന്താശേഷി എല്ലാം മാറ്റുന്നു**

GPT-5.2 നിങ്ങളുടെ പ്രൊംപ്റ്റുകൾ വഴി കണക്കുകൂട്ടൽ ശ്രമം നിയന്ത്രിക്കാൻ അനുവദിക്കുന്നുണ്ട്. കുറവ് ശ്രമം വേഗത്തിലുള്ള മറുപടികൾ കുറഞ്ഞ ഗവേഷണത്തോടെയാണ്. ഉയർന്ന ശ്രമം മോഡലിനെ ആഴത്തിൽ ചിന്തിക്കാൻ സമയം നൽകുന്നു. നിങ്ങൾ പരിശ്രമം ടാസ്‌ക് സങ്കീർണ്ണതയുമായി ഒത്തുചേരുന്നതാണ് പഠിക്കുന്നത് - ലളിതമായ ചോദ്യങ്ങളിൽ സമയം வீഴ്ച്ചയാക്കരുത്, പക്ഷേ ജടിലമായ തീരുമാനങ്ങളിൽപോലും വേഗം നടത്തരുത്.

**ഘടന പെരുമാറ്റം നയിക്കുന്നു**

പ്രൊംപ്റ്റുകളിലെ XML ടാഗുകൾ കാണുന്നുണ്ടോ? അവ അലങ്കാരമാകുന്നില്ല. മോഡലുകൾ സ്വതന്ത്ര ടെക്സ്റ്റിനേക്കാൾ ഘടനയുള്ള നിർദ്ദേശങ്ങൾ കൂടുതൽ വിശ്വസനീയമായി പിന്തുടരുന്നു. ബഹുപടിമുറിവുകൾ അല്ലെങ്കിൽ സങ്കീർണ്ണ ലജിക് ആവശ്യപ്പെട്ടാൽ, ഘടന മോഡലിന് എവിടെയാണ് എന്ന് ട്രാക്ക് ചെയ്യാനും അടുത്തത് എന്താണെന്ന് അറിയാനും സഹായിക്കുന്നു.

<img src="../../../translated_images/ml/prompt-structure.a77763d63f4e2f89.webp" alt="Prompt Structure" width="800"/>

*സ്പഷ്ട വിഭാഗങ്ങളും XML-ശൈലി സംഘടനയുള്ള നന്നായി ഘടനപ്പെടുത്തിയ പ്രൊംപ്റ്റിന്റെ ഘടനാവിവരം*

**സ്വയം മൂല്യനിർണ്ണയത്തിലൂടെ ഗുണനിലവാരം**

സ്വയം-റിഫ്ലെക്ടിങ്ങ് പാറ്റേണുകൾ ഗുണനിലവാരം വ്യക്തമായി നിർവചിക്കുന്നതിലൂടെ പ്രവർത്തിക്കുന്നു. മോഡൽ "ശരി ചെയ്യുമെന്ന്" പ്രതീക്ഷിക്കുന്നതിനു പകരം, നിങ്ങൾ അതിനർത്ഥം എന്താണെന്ന് നിർഘോഷിക്കുന്നു: ശരിയായ പാത, പിശക് കൈകാര്യം ചെയ്യൽ, പ്രകടനം, സുരക്ഷ. മോഡൽ താൻ നിർമ്മിച്ച ഔട്ട്പുട്ട് വിലയിരുത്തിയും മെച്ചപ്പെടുത്തുകയും കഴിയും. ഇത് കോഡ് സൃഷ്ടിയെ ലോട്ടറിയിൽ നിന്ന് പ്രക്രിയയാക്കിയിരിക്കുന്നു.

**പ്രസംഗം പരിമിതമാണ്**

ബഹു-ടേൺ സംഭാഷണങ്ങൾ ഓരോ അഭ്യർത്ഥനയുമായി സന്ദേശം ചരിത്രം ഉൾപ്പെടുന്നുണ്ടാകുന്നു. പക്ഷേ പരിധിയുണ്ട് - ഓരോ മോഡലിനും പരമാവധി ടോക്കൺ കണക്ക് ഉണ്ട്. സംഭാഷണങ്ങൾ വളരുമ്പോൾ, അതേ അതിയായ പ്രസംഗം ബ്ലോക്ക് ചെയ്യാതെ ബന്ധപ്പെട്ട സാഹചര്യങ്ങൾ സൂക്ഷിക്കാൻ തന്ത്രങ്ങൾ ആവശ്യമാണ്. ഈ മോഡ്യൂൾ ഓർമ എങ്ങനെ പ്രവർത്തിക്കുന്നുവെന്ന് കാണിക്കുന്നു; പിന്നീട് നിങ്ങൾക്ക് സംഗ്രഹിക്കേണ്ടപ്പോഴെങ്കിലും, മറക്കേണ്ടപ്പോഴേയും, തിരയേണ്ടപ്പോഴേയും പഠിക്കും.
