## മുൻ കാര്യങ്ങൾ

**AI Agent** - സ്വയം പരിഗണിക്കുകയും പ്രവർത്തിക്കുകയും ചെയ്യുന്ന AI-ഉപയോഗിക്കുന്ന സംവിധാനം. [മോഡ്യൂൾ 04](../04-tools/README.md)

**Chain** - ഔട്ട്പുട്ട് അടുത്ത ഘട്ടത്തിലേക്ക് നൽകുന്ന നടപടികളുടെ ഘട്ടം.

**Chunking** - രേഖകൾ ചെറുതായി വിഭജിക്കൽ. സാധാരണ: 300-500 ടോകൺസ് ഓവർലാപ്പോടെ. [മോഡ്യൂൾ 03](../03-rag/README.md)

**Context Window** - ഒരു മോഡൽ പ്രോസസ് ചെയ്യാവുന്ന പരമാവധി ടോകൺസ്. GPT-5.2: 400K ടോകൺസ് (272K ഇൻപുട്ട് വരെ, 128K ഔട്ട്പുട്ട്).

**Embeddings** - ടെക്സ്റ്റിന്റെ അർത്ഥം പ്രതിനിധാനം ചെയ്യുന്ന അക്കസംഖ്യാ വെക്ടറുകൾ. [മോഡ്യൂൾ 03](../03-rag/README.md)

**Function Calling** - മോഡൽ ഘടിതമായ അഭ്യർത്ഥനകൾ സൃഷ്ടിച്ച് പുറംഫംഗ്ഷനുകൾ വിളിക്കുന്നു. [മോഡ്യൂൾ 04](../04-tools/README.md)

**Hallucination** - മോഡലുകൾ തെറ്റായെങ്കിലും സാധാരണമെന്നു തോന്നുന്ന വിവരങ്ങൾ സൃഷ്ടിക്കുമ്പോൾ.

**Prompt** - ഭാഷാ മോഡലിലേക്ക് പാരായണാനുബന്ധമായ ടെക്സ്റ്റ്. [മോഡ്യൂൾ 02](../02-prompt-engineering/README.md)

**Semantic Search** - കീവർഡുകൾക്കുപകരം embeddings ഉപയോഗിച്ച് അർത്ഥം അടിസ്ഥാനമാക്കി തിരയൽ. [മോഡ്യൂൾ 03](../03-rag/README.md)

**Stateful vs Stateless** - Stateless: ഓർമയില്ല. Stateful: സംഭാഷണ ചരിത്രം നിലനിൽക്കുന്നു. [മോഡ്യൂൾ 01](../01-introduction/README.md)

**Tokens** - മോഡലുകൾ പ്രോസസ് ചെയ്യുന്ന അടിസ്ഥാന ടെക്സ്റ് യൂണിറ്റുകൾ. ചിലവുകളും പരിധികളും ബാധിക്കുന്നു. [മോഡ്യൂൾ 01](../01-introduction/README.md)

**Tool Chaining** - ഔട്ട്പുട്ട് അടുത്ത ടൂളിന്റെ ഉപയോഗം നിർണയിക്കുന്ന അനുക്രമ ടൂൾ പ്രവർത്തനം. [മോഡ്യൂൾ 04](../04-tools/README.md)

## LangChain4j ഘടകങ്ങൾ

**AiServices** - ടൈപ്പിന്റെ സുരക്ഷയുള്ള AI സേർവീസ് ഇന്റർഫേസുകൾ സൃഷ്ടിക്കുന്നു.

**OpenAiOfficialChatModel** - OpenAIയും Azure OpenAIയും മോഡലുകൾക്കുള്ള ഐക്യക്ലയന്റ്.

**OpenAiOfficialEmbeddingModel** - OpenAI ഔദ്യോഗിക ക്ലയന്റ് ഉപയോഗിച്ച് embeddings സൃഷ്ടിക്കുന്നു (OpenAIയും Azure OpenAIയും പിന്തുണയുള്ളത്).

**ChatModel** - ഭാഷാ മോഡലുകളുടെ പ്രധാനം ഇന്റർഫേസ്.

**ChatMemory** - സംഭാഷണ ചരിത്രം സൂക്ഷിക്കുന്നു.

**ContentRetriever** - RAG-ക്കായി അനുയോജ്യമായ രേഖചങ്കുകൾ കണ്ടെത്തുന്നു.

**DocumentSplitter** - രേഖകൾ ചങ്കുകളായി വിഭജിക്കുന്നു.

**EmbeddingModel** - ടെക്സ്റ്റ് അക്കസംഖ്യാ വെക്ടറുകളാക്കി മാറ്റുന്നു.

**EmbeddingStore** - embeddings സൂക്ഷിക്കുകയും പുനരുപയോഗിക്കുകയും ചെയ്യുന്നു.

**MessageWindowChatMemory** - ഒടുവിൽ വന്ന സന്ദേശങ്ങളുടെ സ്ലൈഡിംഗ് വിൻഡോ പാലിക്കുന്നു.

**PromptTemplate** - `{{variable}}` പ്ലേസ്‌ഹോൾഡറുകൾ ഉപയോഗിച്ച് പുനരുപയോഗയോഗ്യമായ പ്രോംപ്റ്റുകൾ സൃഷ്ടിക്കുന്നു.

**TextSegment** - മെറ്റാഡേറ്റാ ഉൾപ്പെട്ട ടെക്സ്റ്റ് ഘടകം. RAG-യിൽ ഉപയോഗിക്കുന്നു.

**ToolExecutionRequest** - ടൂൾ പ്രവർത്തന അഭ്യർത്ഥന പ്രതിനിധാനം ചെയ്യുന്നു.

**UserMessage / AiMessage / SystemMessage** - സംഭാഷണ സന്ദേശ തരംകൾ.

## AI/ML ആശയങ്ങൾ

**Few-Shot Learning** - പ്രോംപ്റ്റുകളിൽ ഉദാഹരണങ്ങൾ നൽകൽ. [മോഡ്യൂൾ 02](../02-prompt-engineering/README.md)

**Large Language Model (LLM)** - വൻ ടെക്സ്റ്റ് ഡാറ്റയിൽ പരിശീലിച്ച AI മോഡലുകൾ.

**Reasoning Effort** - ചിന്താ ആഴം നിയന്ത്രിക്കുന്ന GPT-5.2 പാരാമീറ്റർ. [മോഡ്യൂൾ 02](../02-prompt-engineering/README.md)

**Temperature** - ഔട്ട്പുട്ടിന്റെ അസാധാരണത്വം നിയന്ത്രിക്കുന്നു. കുറവ്=നിശ്ചിതം, ഉയരം=സൃഷ്ടിപരമായത്.

**Vector Database** - embeddings-ക്കായി പ്രത്യേകിച്ചുള്ള ഡാറ്റാബേസ്. [മോഡ്യൂൾ 03](../03-rag/README.md)

**Zero-Shot Learning** - ഉദാഹരണങ്ങൾ ഇല്ലാതെയൊരു ജോലികൾ പ്രാവർത്തികപ്പെടുത്തൽ. [മോഡ്യൂൾ 02](../02-prompt-engineering/README.md)

## ഗാർഡ്റെയിൽസ് - [മോഡ്യൂൾ 00](../00-quick-start/README.md)

**Defense in Depth** - അപ്ലിക്കേഷൻ-തല ഗാർഡ്റെയിൽസും പ്രൊവൈഡർ സുരക്ഷാ ഫിൽറ്ററുകളും ചേർന്ന ബഹുഭാഗ സുസ്ഥിര സുരക്ഷ.

**Hard Block** - ഗൗരവമായ ഉള്ളടക്ക ലംഘനങ്ങൾക്ക് പ്രൊവൈഡർ HTTP 400 പിശക് തള്ളി മറുപടി നൽകുന്നു.

**InputGuardrail** - LLM-നു മുന്നോടിയായി ഉപയോക്തൃ ഇൻപുട്ട് പരിശോദിക്കുവാൻ LangChain4j ഇന്റർഫേസ്. അപകടകാരി പ്രോംപ്റ്റുകൾ ഉണ്ടാകുന്നതും ചെലവും വൈകല്യവും കളയുന്നു.

**InputGuardrailResult** - ഗാർഡ്റെയിൽ പരിശോധനയ്ക്ക് `success()` അല്ലെങ്കിൽ `fatal("reason")` തരം.

**OutputGuardrail** - ഉപയോക്താക്കൾക്ക് മറുപടി നൽകുന്നതിന് മുൻപായി AI ഉത്തരം പരിശോധന ചെയ്യാനുള്ള ഇന്റർഫേസ്.

**Provider Safety Filters** - AI പ്രൊവൈഡർമാരുടെ സംയോജിത ഉള്ളടക്ക പരിശോധന ഫിൽറ്ററുകൾ (ഉദാ: GitHub മോഡലുകൾ), API തലത്തിൽ ലംഘനങ്ങൾ പിടികൂടുന്നു.

**Soft Refusal** - പിശക് തരാതെ മോഡൽ മിനിമം ആദരത്തോടെ മറുപടി നൽകാൻ നിർവൃത്തിയാക്കുന്നു.

## പ്രോംപ്റ്റ് എഞ്ചിനിയറിംഗ് - [മോഡ്യൂൾ 02](../02-prompt-engineering/README.md)

**Chain-of-Thought** - ഘട്ടം-പ്രതി നടപടി വഴി തെറ്റില്ലാത്ത നിർവ്വചനത്തിനായി.

**Constrained Output** - പ്രത്യേക ഫോർമാറ്റ് അല്ലെങ്കിൽ ഘടന നിർബന്ധിക്കുക.

**High Eagerness** - സമഗ്ര ചിന്തയ്ക്കായി GPT-5.2 മാതൃക.

**Low Eagerness** - വേഗതയുള്ള മറുപടിയ്ക്കായി GPT-5.2 മാതൃക.

**Multi-Turn Conversation** - സംവാദങ്ങളിൽ തെളിവുള്ള പാരമ്പര്യം നിലനിർത്തൽ.

**Role-Based Prompting** - സിസ്റ്റം സന്ദേശങ്ങളിലൂടെ മോഡലിന്റെ വ്യക്തിത്വം ക്രമീകരിക്കൽ.

**Self-Reflection** - മോഡൽ തങ്ങളുടെ ഔട്ട്പുട്ട് വിലയിരുത്തുകയും മെച്ചപ്പെടുത്തുകയും ചെയ്യുന്നു.

**Structured Analysis** - നിയന്ത്രിത വിലയിരുത്തൽഘടകം.

**Task Execution Pattern** - പദ്ധതി → നടപ്പാക്കുക → സംഗ്രഹിക്കുക.

## RAG (റിട്ട്രീവൽ-ഓഗ്മെന്റഡ് ജനറേഷൻ) - [മോഡ്യൂൾ 03](../03-rag/README.md)

**Document Processing Pipeline** - ലോഡ് → ചങ്ക് ചെയ്യുക → എംബഡ് ചെയ്യുക → സൂക്ഷിക്കുക.

**In-Memory Embedding Store** - ടെസ്റ്റിങ്ങിനായി നിലവറിയാത്ത സ്റ്റോറേജ്.

**RAG** - ഉത്തരം അടിസ്ഥാർത്ഥമാക്കാൻ റിട്ട്രീവലും ജനറേഷനും സംയോജിപ്പിക്കുന്നു.

**Similarity Score** - സിമാന്റിക് സാമ്യമുള്ളതിനുള്ള അളവ് (0-1).

**Source Reference** - റിട്ട്രീവുചെയ്‌ത ഉള്ളടക്കത്തിന്റെ മെറ്റാഡേറ്റ.

## എജന്റുകളും ടൂളുകളും - [മോഡ്യൂൾ 04](../04-tools/README.md)

**@Tool Annotation** - ജാവ മെതഡുകൾ AI-കോളബിൾ ടൂളുകളായി അടയാളപ്പെടുത്തുന്നു.

**ReAct Pattern** - തർക്കിക്കുക → പ്രവർത്തിക്കുക → നിരീക്ഷിക്കുക → ആവർത്തിക്കുക.

**Session Management** - വ്യത്യസ്ത ഉപയോക്താക്കൾക്ക് പ്രത്യേകം കോൺടെക്സ്റ്റുകൾ.

**Tool** - AI എജന്റ് വിളിക്കാമെന്ന് ഫംഗ്ഷൻ.

**Tool Description** - ടൂളിന്റെ ഉദ്ദേശവും പാരാമീറ്ററുകളും രേഖപ്പെടുത്തൽ.

## എജന്റിക് മോഡ്യൂൾ - [മോഡ്യൂൾ 05](../05-mcp/README.md)

**@Agent Annotation** - AI എജന്റുകളായി ഇന്റർഫേസുകളെ മാർക്ക് ചെയ്യുന്നതും ഡിസ്ക്ലേരറ്റീവ് പെരുമാറ്റ നിർവചനവും.

**Agent Listener** - `beforeAgentInvocation()` ഉം `afterAgentInvocation()` ഉം ഉപയോഗിച്ച് എജന്റ് പ്രവർത്തനം നിരീക്ഷിക്കാൻ ഹുക്ക്.

**Agentic Scope** - എജന്റുകൾ ഔട്ട്പുട് `outputKey` ഉപയോഗിച്ച് സംഭരിക്കുകയും അനന്തര എജന്റുകൾ ഉപയോഗിക്കാൻ പങ്കുവെക്കുകയും ചെയ്യുന്നതിന് പങ്കിട്ട ഓർമ്മ.

**AgenticServices** - `agentBuilder()` ഉം `supervisorBuilder()` ഉം ഉപയോഗിച്ച് എജന്റുകൾ സൃഷ്ടിക്കുന്ന ഫാക്ടറി.

**Conditional Workflow** - വ്യത്യസ്ത വിദഗ്ധ എജന്റുകളിലേക്ക് വ്യത്യസ്ത ചടുവടികൾ അടിസ്ഥാനമാക്കി പാത കണ്ടെത്തൽ.

**Human-in-the-Loop** - അംഗീകാരം അല്ലെങ്കിൽ ഉള്ളടക്ക പരിശോധനയ്ക്കായി മനുഷ്യ checkpoints ചേർക്കുന്ന പ്രവൃത്തി മാതൃക.

**langchain4j-agentic** - ഡിസ്ക്ലയറേറ്റീവ് എജന്റ് നിർമാണത്തിന് വേണ്ടി മേവൻ ഡിപ്പൻഡൻസി (പരീക്ഷണത്തിലുള്ളത്).

**Loop Workflow** - ഒരു നിബന്ധന തികയ്ക്കുന്ന വരെ (ഉദാ: ക്വാളിറ്റി സ്കോർ ≥ 0.8) എജന്റ് പ്രവർത്തനം ആവർത്തിക്കൽ.

**outputKey** - എജന്റ് അന്നോട്ടേഷൻ പാരാമീറ്റർ, എജന്റിക് സ്‌കോപ്പിൽ ഫലങ്ങൾ സൂക്ഷിക്കുന്ന സ്ഥലം.

**Parallel Workflow** - സ്വതന്ത്ര ജോലികൾക്കായി ഒരേസമയം നിരവധി എജന്റുകൾ ഓടിക്കുക.

**Response Strategy** - സൂപ്പർവൈസർ അന്തിമ ഉത്തരം രൂപപ്പെടുത്തുന്ന വിധം: അവസാനത്തെ(LAST), സംഗ്രഹം(SUMMARY), അല്ലെങ്കിൽ സ്കോർ ചെയ്തു(SCORED).

**Sequential Workflow** - പരമ്പരാഗതമായി എജന്റുകൾ പ്രവർത്തിക്കുന്ന ഘടകം, ഔട്ട്പുട്ട് അടുത്ത ഘട്ടത്തിലേക്ക്.

**Supervisor Agent Pattern** - ഒരു സൂപ്പർവൈസർ LLM ഡൈനാമിക് ആയി ഏത് സബ്-എജന്റുകളെ വിളിക്കണമെന്നു തീരുമാനിക്കുന്ന മുന്നേറ്റ എജന്റിക് മാതൃക.

## മോഡൽ_Context_പ്രോട്ടോക്കോൾ (MCP) - [മോഡ്യൂൾ 05](../05-mcp/README.md)

**langchain4j-mcp** - LangChain4j-യിൽ MCP സംയോജനത്തിന് മേവൻ ഡിപ്പൻഡൻസി.

**MCP** - മോഡൽ_Context_പ്രോട്ടോക്കോൾ: AI ആപ്പുകളെ പുറം ടൂളുകളുമായി ബന്ധിപ്പിക്കുന്ന സ്റ്റാൻഡേർഡ്. ഒരിക്കൽ രൂപീകരിക്കുക, എവിടെയും ഉപയോഗിക്കുക.

**MCP Client** - MCP സെർവറുകളെ കണക്ട് ചെയ്ത് ടൂളുകൾ കണ്ടെത്താനും ഉപയോഗിക്കാനും ഉള്ള അപ്ലിക്കേഷൻ.

**MCP Server** - ടൂളുകൾ വ്യക്തമാക്കിയ വിവരണവും പാരാമീറ്റർ സ്കീമകളും സഹിതം MCP വഴി പ്രദർശിപ്പിക്കുന്ന സേവനം.

**McpToolProvider** - MCP ടൂളുകൾ AI സേവനങ്ങളിലും എജന്റുകളിലും ഉപയോഗിക്കാൻ LangChain4j ഘടകം.

**McpTransport** - MCP കമ്മ്യൂണിക്കേഷൻ ഇന്റർഫേസ്.  നടപ്പിലാക്കലുകൾ: Stdio, HTTP.

**Stdio Transport** - stdin/stdout വഴി ലൊക്കൽ പ്രോസസ് ട്രാൻസ്പോർട്ട്. ഫയൽസിസ്റ്റം ആക്‌സസിനു അല്ലെങ്കിൽ കമാൻഡ്-ലൈൻ ടൂളുകൾക്കായി പ്രയോജനപ്പെടുന്നു.

**StdioMcpTransport** - MCP സെർവർ subprocess ആയി സ്രഷ്ടിക്കുന്ന LangChain4j നടപ്പാക്കൽ.

**Tool Discovery** - ക്ലയന്റ് ലഭ്യമായ ടൂളുകൾ വിവരണങ്ങളും സ്കീമകളും ചോദിക്കുന്നു.
