## പ്രധാന ആശയങ്ങൾ

**AI ഏജന്റ്** - സ്വയംയന്ത്രമായി ആർക്ക്യമാക്കാനും പ്രവർത്തിക്കാനും AI ഉപയോഗിക്കുന്ന സംവിധാനം. [Module 04](../04-tools/README.md)

**ചൈൻ** - ഔട്ട്പുട്ട് അടുത്ത ഘട്ടത്തിലേക്ക് എത്തുന്നതോടെ സജ്ജമാകുന്ന പ്രവർത്തനങ്ങളുടെ ശ്രേണി.

**ചങ്കിംഗ്** - ദസ്സ്യമെന്ന് ചെറിയ ഭാഗങ്ങളായി വിഭജിക്കൽ. സാധാരണ: 300-500 ടോക്കണുകൾ ഓവർലാപ് ഉള്ളവ. [Module 03](../03-rag/README.md)

**കോൺടക്സ്റ്റ് വിൻഡോ** - ഒരു മോഡൽ പ്രോസസ്സ് ചെയ്യാവുന്ന പരമാവധി ടോക്കൺ. GPT-5.2: 400K ടോക്കൺ.

**എംബെഡിങ്ങ്സ്** - ടെക്സ്റ്റിന്റെ അർത്ഥം പ്രതിനിധാനം ചെയ്യുന്ന സംഖ്യാത്മക വെക്റ്ററുകൾ. [Module 03](../03-rag/README.md)

**ഫംഗ്ഷൻ കോല്ലിംഗ്** - മോഡൽ എക്സ്റ്റേണൽ ഫംഗ്ഷനുകൾ വിളിക്കാൻ ഘടനാപരമായ അഭ്യർത്ഥനകൾ സൃഷ്ടിക്കുന്നു. [Module 04](../04-tools/README.md)

**ഹല്ല്യൂസിനേഷൻ** - മോഡലുകൾ തെറ്റായെങ്കിലും സാധ്യമായ വിവരങ്ങൾ സൃഷ്ടിക്കുന്നത്.

**പ്രോംപ്റ്റ്** - ഭാഷാ മോഡലിലേക്ക് നൽകുന്ന ടെക്സ്റ്റ് ഇൻപുട്ട്. [Module 02](../02-prompt-engineering/README.md)

**സെമാന്റിക് സർച്ച്** - എമ്പെഡിങ്ങുകൾ ഉപയോഗിച്ച് അർത്ഥം അടിസ്ഥാനമാക്കിയുള്ള തിരച്ചിൽ, കീവേഡുകൾ അല്ല. [Module 03](../03-rag/README.md)

**സ്റ്റേറ്റ്‌ഫുൾ vs സ്റ്റേറ്റ്‌ലെസ്** - സ്റ്റേറ്റ്‌ലെസ്: യാദൃച്ഛികം ഓർമ്മ ഇല്ല. സ്റ്റേറ്റ്‌ഫുൾ: സംഭാഷണ ചരിത്രം നിലനിർത്തുന്നു. [Module 01](../01-introduction/README.md)

**ടോക്കണുകൾ** - മോഡലുകൾ പ്രോസസ്സ് ചെയ്യുന്ന അടിസ്ഥാന ടെക്സ്റ്റ് യൂണിറ്റുകൾ. ചെലവുകളിലും പരിധികളിലും സ്വാധീനിക്കുന്നു. [Module 01](../01-introduction/README.md)

**ടൂൾ ചെയിനിംഗ്** - ഔട്ട്പുട്ട് അടുത്ത കോളിൽ വിവരമുന്നാട്ട് തയ്യാറാക്കി ടൂൾ ക്രമത്തിൽ പ്രവർത്തനം. [Module 04](../04-tools/README.md)

## LangChain4j ഘടകങ്ങൾ

**AiServices** - തരം-സുരക്ഷിത AI സർവീസ് ഇന്റർഫെയിസുകൾ ഉണ്ടാക്കുന്നു.

**OpenAiOfficialChatModel** - OpenAIയും Azure OpenAI മോഡലുകളും സംയോജിപ്പിച്ച ക്ലയന്റ്.

**OpenAiOfficialEmbeddingModel** - OpenAI ഔദ്യോഗിക ക്ലയന്റ് ഉപയോഗിച്ച് എമ്പെഡിങ്ങുകൾ സൃഷ്ടിക്കുന്നു (OpenAIയും Azure OpenAIയും പിന്തുണയ്‌ക്കുന്നു).

**ChatModel** - ഭാഷാ മോഡലുകളുടെ പ്രധാന ഇന്റർഫെയ്സ്.

**ChatMemory** - സംഭാഷണ ചരിത്രം സൂക്ഷിക്കുന്നു.

**ContentRetriever** - RAG ന് അനുയോജ്യമായ രേഖാഖണ്ഡങ്ങൾ കണ്ടെത്തുന്നു.

**DocumentSplitter** - രേഖകളെ കഷണങ്ങളായി വിഭജിക്കുന്നു.

**EmbeddingModel** - ടെക്സ്റ്റ് സംഖ്യാത്മക വെക്റ്ററുകളായി പരിവർത്തനം ചെയ്യുന്നു.

**EmbeddingStore** - എമ്പെഡിങ്ങുകൾ സൂക്ഷിക്കുകയും പുനരുപയോഗം ചെയ്യുകയും ചെയ്യുന്നു.

**MessageWindowChatMemory** - ഏറ്റവും സമീപകാല സന്ദേശങ്ങളുടെ സ്ലൈഡിംഗ് വിൻഡോ നിലനിർത്തുന്നു.

**PromptTemplate** - `{{variable}}` പ്ലേസ്ഹോൾഡറുകൾ ഉപയോഗിച്ച് പുനരുപയോഗിക്കാവുന്ന പ്രോംപ്റ്റുകൾ സൃഷ്ടിക്കുന്നു.

**TextSegment** - മെറ്റാഡാറ്റയുള്ള ടെക്സ്റ്റ് ചങ്ക്. RAG-ൽ ഉപയോഗിക്കുന്നു.

**ToolExecutionRequest** - ടൂൾ പ്രവർത്തന അഭ്യർത്ഥന നിഗമനം.

**UserMessage / AiMessage / SystemMessage** - സംഭാഷണം സന്ദേശ വിഭാഗങ്ങൾ.

## AI/ML ആശയങ്ങൾ

**Few-Shot Learning** - പ്രോംപ്റ്റുകളിൽ ഉദാഹരണങ്ങൾ നൽകുന്നത്. [Module 02](../02-prompt-engineering/README.md)

**ലാർജ് ലാംഗ്വേജ് മോഡൽ (LLM)** - വലുതായ ടെക്സ്റ്റ് ഡാറ്റയിൽ പരിശീലിക്കപ്പെട്ട AI മോഡലുകൾ.

**റീസണിംഗ് എഫർട്ട്** - ചിന്താവ്യവസ്ഥ നിയന്ത്രിക്കുന്ന GPT-5.2 പാരാമീറ്റർ. [Module 02](../02-prompt-engineering/README.md)

**ടെംപറേച്ചർ** - ഔട്ട്പുട്ടിന്റെ യാദൃച്ഛികത നിയന്ത്രിക്കുന്നു. താഴ്ന്നത്=നിയമിതം, ഉയർന്നത്=സൃഷ്ടിപരമായ.

**വെക്റ്റർ ഡേറ്റാബേസ്** - എമ്പെഡിങ്ങുകൾക്കായി പ്രത്യേകിച്ച് രൂപകൽപ്പന ചെയ്ത ഡേറ്റാബേസ്. [Module 03](../03-rag/README.md)

**സീറോ-ഷോട്ട് ലേണിംഗ്** - ഉദാഹരണങ്ങൾ ഇല്ലാതെ പ്രവർത്തനങ്ങൾ ചെയ്യുക. [Module 02](../02-prompt-engineering/README.md)

## Guardrails - [Module 00](../00-quick-start/README.md)

**ഡിഫൻസ് ഇന്‍ ഡെപ്ത്** - ആപ്ലിക്കേഷൻ ലെവൽ ഗാർഡ്‌റെയിൽസും പ്രൊവൈഡർ സുരക്ഷാ ഫിൽട്ടറുകളും സംയോജിപ്പിച്ച ഭദ്രതാ സമീപനം.

**ഹാർഡ് ബ്ലോക്ക്** - ഗുരുതര ഉള്ളടക്ക ലംഘനങ്ങളിലേക്ക് പ്രൊവൈഡർ HTTP 400 പിശക് എററും.

**InputGuardrail** - LLM-നെ എത്തുന്നതിനു മുമ്പ് ഉപയോക്തൃ ഇൻപുട്ട് സാധുതയാക്കാനുള്ള LangChain4j ഇന്റർഫെയ്സ്. കേടുതല്പരിക്കുന്ന പ്രോംപ്റ്റുകൾ നേരത്തേ തടയുന്നതിലൂടെ ചെലവും നീളവും ലാഭിക്കുന്നു.

**InputGuardrailResult** - ഗാർഡ്‌റെയിൽ സാധുതയാക്കൽ റിടേൺ തരം: `success()` അല്ലെങ്കിൽ `fatal("reason")`.

**OutputGuardrail** - ഉപയോക്താക്കൾക്ക് തിരിച്ചുവരുന്നതിനു മുമ്പ് AI വീാതരണങ്ങൾ സാധുതയാക്കാൻ ഇന്റർഫെയ്സ്.

**Provider Safety Filters** - AI പ്രൊവൈഡർമാരിൽ നിന്ന് നാടോടി ഉൾപ്പടെയുള്ള ഉള്ളടക്ക ഫിൽട്ടറുകൾ (ഉദാ., GitHub മോഡലുകൾ) API തലത്തിൽ ലംഘനങ്ങൾ പിടികൂടുന്നു.

**Soft Refusal** - മോഡൽ പിശക് എറർ എററില്ലാതെ വിനീതമായി മറുപടി നൽകാൻ തള്ളുന്നു.

## പ്രോംപ്റ്റ് എഞ്ചിനീയറിംഗ് - [Module 02](../02-prompt-engineering/README.md)

**ചൈൻ-ഓഫ്-തോട്ട്** - മെച്ചപ്പെട്ട കൃത്യതയ്ക്കായി ഘട്ടം ഘട്ടമായ റീസണിംഗ്.

**ബന്ധിത ഔട്ട്പുട്ട്** - സഹജമായ ഒരു ഫോർമാറ്റ് അല്ലെങ്കിൽ ഘടന നിർബന്ധിക്കുക.

**ഉയർന്ന താൽപര്യം** - മുഴുവൻ റീസണിംഗ് ഉറപ്പാക്കാൻ GPT-5.2 പാറ്റേൺ.

**താഴ്ന്ന താൽപര്യം** - വേഗത്തിൽ ഉത്തരങ്ങൾ നൽകാൻ GPT-5.2 പാറ്റേൺ.

**മൾട്ടി-ടർൺ സംഭാഷണം** - പരസ്പരം ഉള്ള സംവിധാനത്തിൽ കോൺടക്‌സ്‌റ്റ് നിലനിർത്തൽ.

**റോൾ-ബേസ് പ്രോംപ്റ്റിംഗ്** - സിസ്റ്റം സന്ദേശങ്ങളിലൂടെ മോഡൽ വ്യക്തിത്വം ക്രമീകരിക്കൽ.

**സ്വയം-പരിശോധന** - മോഡൽ തങ്ങളുടെ ഔട്ട്പുട്ട് വിലയിരുത്തുന്നും മെച്ചപ്പെടുത്തുന്നതും.

**ഘടനാപരമായ വിശകലനം** - സ്ഥിരമായ വിലയിരുത്തൽ രീതി.

**ടാസ്‌ക് എക്സിക്യൂഷൻ പാറ്റേൺ** - പ്ലാൻ → നടപ്പിലാക്കുക → സംഗ്രഹിക്കുക.

## RAG (Retrieval-Augmented Generation) - [Module 03](../03-rag/README.md)

**ഡോക്യുമെന്റ് പ്രോസസ്സിംഗ് പൈപ്പ്‌ലൈൻ** - ലോഡ് → ചങ്ക് → എംബെഡ് → സംഭരണം.

**ഇൻ-മെമ്മറി എംബെഡിംഗ് സ്റ്റോർ** - ടെസ്റ്റിനായി നിഷ്പ്രഭമായ സംഭരണം.

**RAG** - തിരയുന്നും സൃഷ്ടിക്കുകയും ചേർത്ത് മറുപടികൾ അടിസ്ഥാനമാക്കുന്നു.

**സിമിലാരിറ്റി സ്‌കോർ** - സെമാന്റിക് സമാനതയുടെ അളവ് (0-1).

**സോഴ്‌സ് റഫറൻസ്** - തിരഞ്ഞെടുത്ത ഉള്ളടക്കത്തെക്കുറിച്ചുള്ള മെറ്റാഡാറ്റ.

## ഏജന്റുകളും ടൂൾസും - [Module 04](../04-tools/README.md)

**@Tool അനോട്ടേഷൻ** - ജാവാ മെത്തഡുകൾ AI വിളിക്കാൻ കഴിയുന്ന ടൂളുകളായി അടയാളപ്പെടുത്തുന്നു.

**ReAct പാറ്റേൺ** - റീസണിംഗ് → പ്രവർത്തനം → നിരീക്ഷണം → ആവർത്തിക്കൽ.

**സെഷൻ മാനേജ്മെന്റ്** - വ്യത്യസ്ത ഉപയോക്താക്കൾക്കായി വേറിട്ട കോൺടക്സ്റ്റുകൾ.

**ടൂൾ** - AI ഏജന്റ് വിളിക്കാൻ കഴിയുന്ന ഫംഗ്ഷൻ.

**ടൂൾ വിവരണം** - ടൂളിന്റെ ഉദ്ദേശ്യവും പാരാമീറ്ററുകളുടെയും രേഖപ്പെടുത്തൽ.

## ഏജന്റിക് മോഡ്യൂൾ - [Module 05](../05-mcp/README.md)

**@Agent അനോട്ടേഷൻ** - ഡിക്ലറേറ്റീവ് പെരുമാറ്റ നിർവ്വചനത്തോടെ ഇന്റർഫേസുകളെ AI ഏജന്റുകളായി അടയാളപ്പെടുത്തുന്നു.

**ഏജന്റ് ലിസണർ** - `beforeAgentInvocation()` மற்றும் `afterAgentInvocation()` വഴി ഏജന്റ് നിര്‍വഹണ നിരീക്ഷണ ഹുക്ക്.

**ഏജന്റിക് സ്കോപ്പ്** - ഏജന്റുകൾ തങ്ങളുടെ ഔട്ട്പുട്ട് `outputKey` ഉപയോഗിച്ച് സംഭരിക്കുന്ന പങ്കുവെക്കുന്ന മെമ്മറി, പിന്നോടുള്ള ഏജന്റുകൾ ഉപയോഗിക്കാൻ.

**AgenticServices** - `agentBuilder()` and `supervisorBuilder()` ഉപയോഗിച്ച് ഏജന്റുകൾ സൃഷ്ടിക്കുന്ന ഫാക്ടറി.

**പുരോഗമന പ്രകടനം** - വ്യത്യസ്ത വിദഗ്ധ ഏജന്റുകളിലേക്ക് നിബന്ധനകൾ അടിസ്ഥാനമാക്കിയുള്ള മാർഗനിർദ്ദേശം.

**ഹ്യൂമൻ-ഇൻ-ദ-ലൂപ്പ്** - അനുമതി അല്ലെങ്കിൽ ഉള്ളടക്ക പരിശോധനയ്ക്കായി മനുഷ്യനെ ഇടപെടുന്നു.

**langchain4j-agentic** - ഡിക്ലറേറ്റീവ് ഏജന്റ് നിർമ്മാണത്തിനായുള്ള മേവൻ ഡിപ്പെൻഡൻസി (പരീക്ഷണ ഘട്ടം).

**ലൂപ് വർക്‌ഫ്ലോ** - നിബന്ധന പൂരിപ്പിക്കുന്നവരെ വരെ ഏജന്റ് പ്രവർത്തനം നിരന്തരമാക്കൽ (ഉദാ., ഗുണനിലവാര സ്കോർ ≥ 0.8).

**outputKey** - ഏജന്റ് അനോട്ടേഷൻ പാരാമീറ്റർ, ഏജന്റിക് സ്കോപ്പിൽ ഫലങ്ങൾ സൂക്ഷിക്കാനുള്ള ഇടം നിർദ്ദേശിക്കുന്നു.

**പരല്ലൽ വർക്‌ഫ്ലോ** - സ്വതന്ത്ര ടാസ്കുകൾക്കായി ഒരേസമയം നിരവധി ഏജന്റുകൾ പ്രവർത്തിക്കുക.

**റസ്പോൺസ് സ്റ്റ്രാറ്റജറി** - സൂപ്പർവൈസർ അവസാന ഉത്തരമെങ്ങനെ രൂപകല്‍പ്പന ചെയ്യുന്നു: LAST, SUMMARY, SCORED.

**ക്രമാത്മക വർക്‌ഫ്ലോ** - ഔട്ട്പുട്ട് അടുത്ത ഘട്ടത്തിലേക്ക് പ്രവഹിക്കുന്നനിലയിൽ ഏജന്റുകൾക്ക് ഒരിൽ ക്രമത്തിൽ നിർവഹണം.

**സൂപ്പർവൈസർ ഏജന്റ് പാറ്റേൺ** - ഒരു സൂപ്പർവൈസർ LLM ഡൈനാമികായി ഏത് ഉപഏജന്റുകൾ വിളിക്കണമെന്ന് തീരുമാനിക്കുന്ന പുരോഗമന ഏജന്റിറ്റ് പാറ്റേൺ.

## മോഡൽ കോൺടക്സ്റ്റ് പ്രോട്ടോകോൾ (MCP) - [Module 05](../05-mcp/README.md)

**langchain4j-mcp** - LangChain4j-ൽ MCP സംയോജനത്തിനുള്ള മേവൻ ഡിപ്പെൻഡൻസി.

**MCP** - Model Context Protocol: AI അപ്ലിക്കേഷനുകൾ എക്സ്റ്റേണൽ ടൂളുകളുമായി കണക്ട് ചെയ്യുന്നതിനുള്ള സ്റ്റാൻഡേർഡ്. ഒറ്റമാറ്റിൽ നിർമ്മിച്ച് എല്ലായിടത്തും ഉപയോഗിക്കുക.

**MCP ക്ലയന്റ്** - MCP സർവറുകളിലേക്ക് കണക് ചെയ്ത് ടൂളുകൾ കണ്ടെത്തുകയും ഉപയോഗപ്പെടുത്തുകയും ചെയ്യുന്ന അപ്ലിക്കേഷൻ.

**MCP സർവർ** - സ്പഷ്ട വിവരണങ്ങളും പാരാമീറ്റർ സ്കീമകളും ഉള്ള ടൂൾസ് MCP വഴി എക്സ്പോസ് ചെയ്യുന്ന സർവീസ്.

**McpToolProvider** - AI സർവീസുകളിലും ഏജന്റുകളിലും MCP ഉപകരണങ്ങൾ ഉപയോഗിക്കുന്നതിനുള്ള LangChain4j ഘടകം.

**McpTransport** - MCP കമ്മ്യൂണിക്കേഷനുകൾക്കുള്ള ഇന്റർഫെയ്സ്. നടപ്പാക്കലുകളിൽ Stdio, HTTP ഉണ്ട്.

**Stdio Transport** - stdin/stdout വഴി ലോക്കൽ പ്രോസസ് ട്രാൻസ്പോർട്ട്. ഫയൽസിസ്റ്റം ആക്‌സസ് അല്ലെങ്കിൽ കമാൻഡ്-ലൈൻ ടൂളുകൾക്ക് ഉത്തമം.

**StdioMcpTransport** - MCP സർവർ subprocess ആയി ആരംഭിക്കുന്ന LangChain4j നടപ്പാക്കൽ.

**ടൂൾ കണ്ടെത്തൽ** - ക്ലയന്റ് സേർവർക്ക് ലഭ്യമായ ടൂളുകൾ, വിവരണങ്ങളും സ്കീമാനുംൾ നേരിട്ട് ചോദിക്കുന്നു.
