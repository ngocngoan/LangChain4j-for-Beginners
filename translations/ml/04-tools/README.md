## നിങ്ങൾ പഠിക്കുന്നതെന്താണ്

ഇവിടെ വരെ, നിങ്ങൾയ്ക്ക് AI ഒപ്പം സംവാദങ്ങൾ നടത്തുന്നത്, പ്രാമ്പ്റ്റുകൾ അനുയോജ്യമായി ക്രമീകരിക്കുന്നത്, മറുപടികളെ നിങ്ങളുടെ ഡോക്കുമെന്റുകളിൽ അടിസ്ഥാനമാക്കുന്നത് എന്നിവ പഠിച്ചിരിക്കുന്നു. പക്ഷേ ഇതിൽ ഒരു അടിസ്ഥാനപരമായ പരിമിതി ഉണ്ട്: ഭാഷ മോഡലുകൾ വെറും ടക്സ്റ്റ് മാത്രം സൃഷ്ടിക്കാനാണ് കഴിയുന്നത്. അവ കാലാവസ്ഥ പരിശോധിക്കാനോ, കണക്കുകൾ ചെയ്യാനോ, ഡാറ്റാബേസുകൾ ചോദിക്കാനോ, ബാഹ്യ സിസ്റ്റങ്ങളുമായി ബന്ധപ്പെടാനോ കഴിയില്ല.

ടൂളുകൾ ഇതു മാറ്റുന്നു. മോഡലിന് ഫംഗ്ഷനുകൾ കോളുചെയ്യാനുള്ള ആക്സസ് നൽകുമ്പോൾ അത് ഒരു ടെക്സ്റ്റ് ജനറേറ്ററിൽ നിന്നു് പ്രവർത്തി എടുക്കുന്ന ഏജൻറായി മാറുന്നു. മോഡൽക്ക് എപ്പോള് ടൂൾ വേണ്ടതാണെന്ന്, ഏത് ടൂൾ ഉപയോഗിക്കണമെന്നു, എന്ത് പാരാമീറ്ററുകൾ അയക്കണമെന്നു തീരുമാനിക്കും. നിങ്ങൾക്ക് കോഡ് ഫംഗ്ഷൻ നിർവഹിച്ചു ഫലം മടങ്ങിപ്പറയും. മോഡൽ ആ ഫലം മറുപടിയിൽ ഉൾപ്പെടുത്തും.

## ടൂളുകളുള്ള AI ഏജന്റുകൾ മനസിലാക്കൽ

> **📝 ശ്രദ്ധിക്കുക:** ഈ മോഡ്യൂളിലെ "ഏജന്റുകൾ" എന്ന പദം ടൂൾ-കോളിംഗ് സജ്ജമാക്കിയ AI സഹായികൾക്കാണ്. ഇത് ഞങ്ങൾ [മോഡ്യൂൾ 05: MCP](../05-mcp/README.md) ൽ വിലയിരുത്തുന്ന **Agentic AI** പാറ്റേണുകൾ (സ്വയംപരിപാലന, നിജപേക്ഷണം, നിരവധി ഘട്ട ചിന്തനങ്ങൾ എന്നിവ) ന് വ്യത്യസ്തമാണു്.

ടൂളുകൾ ഇല്ലാതെ, ഒരു ഭാഷ മോഡൽ തന്റെ പരിശീലന ഡേറ്റയിൽ നിന്ന് മാത്രമേ ടെക്സ്റ്റ് സൃഷ്ടിക്കുകയുള്ളൂ. ഇപ്പോഴത്തെ കാലാവസ്ഥ ചോദിച്ചാൽ അത് അനുമാനിച്ചതു മാത്രം നൽകും. ടൂളുകൾ നൽകിയാൽ അത് കാലാവസ്ഥ API കോളുചെയ്യാനും കണക്കുകൂട്ടലുകൾ നടത്താനും ഡാറ്റാബേസുകൾ ചോദിക്കാനും കഴിയും — പിന്നെ ആ യഥാർത്ഥ ഫലങ്ങൾ മറുപടിയിൽ സഹിതം ചേർക്കും.

<img src="../../../translated_images/ml/what-are-tools.724e468fc4de64da.webp" alt="ടൂളുകൾ ഇല്ലാതെ vs ടൂളുകളോടെ" width="800"/>

*ടൂളുകൾ ഇല്ലാതെ മോഡൽ അനുമാനിച്ചുതന്നെയാണ്— ടൂളുകളോടെ API കോളുചെയ്ത് കണക്ക് നടത്തുകയും യഥാർത്ഥ ഡാറ്റ മടങ്ങിപ്പറയും.*

ടൂളുകളുള്ള AI ഏജന്റ് **ReAct (Reasoning and Acting)** പാറ്റേൺ പിന്തുടരുന്നു. മോഡൽ വെറും മറുപടി നൽകുന്നില്ല — അതിന് എന്ത് വേണമെന്ന് ചിന്തിക്കുകയും, ടൂൾ വഴിയുള്ള പ്രവർത്തനം നടത്തുകയും, ഫലം നിരീക്ഷിക്കുകയും, പിന്നെ വീണ്ടും പ്രവർത്തിക്കാമോ അവസാന മറുപടി നല്‍കാമോ എന്ന് തീരുമാനിക്കുകയാണ്:

1. **ചിന്തിക്കുക** — ഉപയോക്താവിന്റെ ചോദ്യം വിശകലനം ചെയ്ത് ആവശ്യമായ വിവരങ്ങൾ കണ്ടെത്തുക
2. **പ്രവർത്തിക്കുക** — ശരിയായ ടൂൾ തിരഞ്ഞെടുക്കുക, ശരിയായ പാരാമീറ്ററുകൾ സൃഷ്ടിക്കുക, അക്കാൾ നടത്തുക
3. **നിരീക്ഷിക്കുക** — ടൂളിന്റെ ഫലം സ്വീകരിച്ച് വിലയിരുത്തുക
4. **പുനരാഖ്യാനം അല്ലെങ്കിൽ മറുപടി** — കൂടുതൽ ഡേറ്റ ആവശ്യമെങ്കിൽ നടപടിക്രമം ആവർത്തിക്കുക; അല്ലെങ്കിൽ സ്വാഭാവിക ഭാഷയിൽ മറുപടി രൂപീകരിക്കുക

<img src="../../../translated_images/ml/react-pattern-detail.96a5efeeb6dd2f61.webp" alt="ReAct പാറ്റേൺ" width="800"/>

*ReAct സൈക്കിൾ - ഏജന്റ് ചെയ്യേണ്ടത് ചിന്തിച്ചു, ടൂൾ കോളും ചെയ്തു, ഫലം നിരീക്ഷിച്ച് അവസാന മറുപടി നല്‍കും വരെ ആവർത്തിക്കുന്നു.*

ഇതു ഓട്ടോമാറ്റിക്കായാണ് സംഭവിക്കുന്നത്. നിങ്ങൾ ടൂളുകളും അവയുടെ വിവരണങ്ങളും നിർവചിക്കുന്നു. മോഡൽ എപ്പോൾ എങ്ങനെ അവ ഉപയോഗിക്കണമെന്ന് തീരുമാനിക്കുന്നു.

## ടൂൾ കോൾ ചെയ്യുന്നത് എങ്ങനെയാണ്

### ടൂൾ നിർവചനങ്ങൾ

[WeatherTool.java](../../../04-tools/src/main/java/com/example/langchain4j/agents/tools/WeatherTool.java) | [TemperatureTool.java](../../../04-tools/src/main/java/com/example/langchain4j/agents/tools/TemperatureTool.java)

നിങ്ങൾ വ്യക്തമായ വിവരണങ്ങളോടുകൂടിയത് വഴികാട്ടുകയും പാരാമീറ്റർ നിർദ്ദേശങ്ങൾ ഉൾപ്പെടുത്തുകയും ചെയ്ത് ഫംഗ്ഷനുകൾ നിർവചിക്കുന്നു. മോഡലിന് സിസ്റ്റം പ്രാമ്പ്റ്റ് വഴി അവു കാണിച്ച് ഓരോ ടൂൾ ചെയ്യുക എന്നത് മനസ്സിലാക്കും.

```java
@Component
public class WeatherTool {
    
    @Tool("Get the current weather for a location")
    public String getCurrentWeather(@P("Location name") String location) {
        // നിങ്ങളുടെ കാലാവസ്ഥാ തിരയൽ തന്ത്രം
        return "Weather in " + location + ": 22°C, cloudy";
    }
}

@AiService
public interface Assistant {
    String chat(@MemoryId String sessionId, @UserMessage String message);
}

// അസിസ്റ്റന്റ് സ്വയമേവ സ്രിംഗ് ബൂട്ട് ഉപയോഗിച്ച് ബന്ധിപ്പിച്ചിരിക്കുന്നു:
// - ChatModel ബീൻ
// - @Component ക്ലാസുകളിൽ നിന്നും എല്ലാ @Tool മുറ്റ്്‌ഡുകൾ
// - സഷൻ കൈകാര്യം ചെയ്യുന്നതിനുള്ള ChatMemoryProvider
```
  
ഇനിമനായി കാർഡ് പ്രതീകം ഓരോ അനോട്ടേഷൻ വീക്ഷിച്ച് എങ്ങനെ AI ടൂൾ കോളും പാരാമീറ്ററുകൾ നൽകുമെന്നു മനസ്സിലാക്കുന്നതിന് സഹായിക്കുന്നു:

<img src="../../../translated_images/ml/tool-definitions-anatomy.f6468546037cf28b.webp" alt="ടൂൾ നിർവചനങ്ങളുടെ ഘടന" width="800"/>

*ടൂൾ നിർവചനങ്ങളുടെ ഘടന - @Tool AI ഉപയോഗിക്കുന്നത് എപ്പോഴെന്ന് പറയും, @P ഓരോ പാരാമീറ്ററും വിശദീകരിക്കും, @AiService സ്റ്റാർട്ടപ്പിൽ ഇനങ്ങളെല്ലാം യോജിപ്പിക്കും.*

> **🤖 [GitHub Copilot](https://github.com/features/copilot) ചാറ്റിൽ പരീക്ഷിക്കുക:**  [`WeatherTool.java`](../../../04-tools/src/main/java/com/example/langchain4j/agents/tools/WeatherTool.java) തുറന്ന് ചോദിക്കൂ:
> - "സത്യമായ കാലാവസ്ഥ API OpenWeatherMap ഐ mock ഡാറ്റയ്ക്കുള്ള പകരം എങ്ങനെ സംയോജിപ്പിക്കാം?"
> - "AI യെ ടൂൾ ശരിയായി ഉപയോഗിക്കാൻ സഹായിക്കുന്ന നല്ല ടൂൾ വിവരണം എന്തെല്ലാം ഉള്ളത്?"
> - "ടൂൾ നടപ്പാക്കലുകളിൽ API പിഴവ് കുറയും ലോകരം എങ്ങനെ കൈകാര്യം ചെയ്യാം?"

### തീരുമാനം നടത്തൽ

"സിയാറ്റിലിൽ കാലാവസ്ഥ എങ്ങനെയാണ്?" എന്ന് ഉപയോക്താവ് ചോദിച്ചാൽ, മോഡൽ അറിവില്ലാതെ ഏതെങ്കിലും ടൂൾ തിരഞ്ഞെടുക്കുന്നു എന്നില്ല. ഉപയോക്താവിന്റെ ഉദ്ദേശം ഓരോ ടൂൾ വിവരണങ്ങളോടും താരതമ്യം ചെയ്ത് അവയിൽ ഏറ്റവും ബന്ധമുള്ളത് തിരഞ്ഞെടുത്ത് അത് ഉപയോഗിക്കുന്നു. തുടർന്ന് ശരിയായ പാരാമീറ്ററുകളുമായി ക്രമബദ്ധമായ ഫംഗ്ഷൻ കോളും നിർമ്മിക്കുന്നു — ഈ ഉദാഹരണത്തിൽ `location`എന്ന് `"Seattle"` ആയി സജ്ജമാക്കുന്നു.

ഉപയോക്താവിന്റെ അഭ്യർത്ഥനക്ക് യാതൊരു ടൂൾയും ചേരാത്ത പക്ഷം മോഡൽ തന്റെ അറിവിൽ നിന്നു മറുപടി നൽകും. ഒരേ സമയം ഒരുപാട് ടൂളുകൾക്ക് അനുയോജ്യമായാൽ ഏറ്റവും പ്രത്യേകമായൊരു ടൂൾ തെരഞ്ഞെടുക്കും.

<img src="../../../translated_images/ml/decision-making.409cd562e5cecc49.webp" alt="AI ടൂൾ തിരഞ്ഞെടുക്കുന്നത് എങ്ങനെ" width="800"/>

*മോഡൽ ഉപയോക്താവിന്റെ ഉദ്ദേശം ഓരോ ടൂളുമായി താരതമ്യം ചെയ്ത് ഏറ്റവും അനുയോജ്യമായ ഒന്നു തിരഞ്ഞെടുത്തുകൊണ്ടിരിക്കുകയാണ്- അതിനാൽ വ്യക്തമായ ടൂൾ വിവരണങ്ങൾ എഴുതുന്നത് പ്രധാനമാണ്.*

### നിർവഹണം

[AgentService.java](../../../04-tools/src/main/java/com/example/langchain4j/agents/service/AgentService.java)

സ്പ്രിംഗ് ബൂട്ട് സ്വയമേവ `@AiService` ഇന്റർഫേസ് എല്ലാ രജിസ്റ്റർ ചെയ്ത ടൂളുകളോടും യോജിപ്പിക്കുന്നു, LangChain4j ടൂൾ കോളുകൾ ഓട്ടോമാറ്റിക്‌ ആയി നടപ്പാക്കുന്നു. ഉപയോക്തൃഭാഷാപ്രശ്നത്തിൽ നിന്നാരംഭിച്ച് സ്വാഭാവികഭാഷ മറുപടി എടുക്കുന്നത് ആറു ഘട്ടങ്ങളിലൂടെയാണ് ടൂൾ കോളിന്റെ പ്രവാഹം നടക്കുന്നത്:

<img src="../../../translated_images/ml/tool-calling-flow.8601941b0ca041e6.webp" alt="ടൂൾ കോളിംഗ് പ്രവാഹം" width="800"/>

*ആരംഭത്തിൽ ഉപയോക്താവ് ചോദിച്ചാൽ മോഡൽ ടൂൾ തിരഞ്ഞെടുക്കും, LangChain4j ടൂൾ പ്രവർത്തിപ്പിക്കും, മോഡൽ ഫലം സ്വാഭാവിക മറുപടിയിൽ ചേർക്കുന്നു.*

> **🤖 [GitHub Copilot](https://github.com/features/copilot) ചാറ്റിൽ പരീക്ഷിക്കുക:**  [`AgentService.java`](../../../04-tools/src/main/java/com/example/langchain4j/agents/service/AgentService.java) തുറന്ന് ചോദിക്കൂ:
> - "ReAct പാറ്റേൺ എങ്ങനെ പ്രവർത്തിക്കുന്നു, AI ഏജന്റുകൾക്ക് ഇത് എങ്ങനെയും ഫലപ്രദമാണോ?"
> - "ഏജന്റ് ടൂൾ തിരഞ്ഞെടുക്കുന്ന പ്രക്രിയ എങ്ങനെ, ഏത് ഓർഡറിൽ ടൂളുകൾ ഉപയോഗിക്കണമെന്നുസ 결정ിക്കുന്നു?"
> - "ടൂൾ പ്രവർത്തനം പരാജയപ്പെട്ടാൽ എന്ത് സംഭവിക്കും, പിഴവുകൾ സുതാര്യമായി കൈകാര്യം ചെയ്യുന്നതിനുള്ള മാർഗങ്ങൾ എന്തെല്ലാം?"

### പ്രതികരണം നിർമ്മാണം

മോഡൽ കാലാവസ്ഥ ഡാറ്റ സ്വീകരിച്ച് ഉപയോക്താവിന് സ്വാഭാവിക ഭാഷയിൽ മറുപടി രൂപീകരിക്കും.

### ആർക്കിടെക്ചർ: സ്പ്രിംഗ് ബൂട്ട് ഓട്ടോ-വൈറിംഗ്

ഈ മോഡ്യൂൾ LangChain4j-ന്റെ സ്പ്രിംഗ് ബൂട്ട് ഇന്റഗ്രേഷൻ ഉപയോഗിച്ച് ഡിക്ലറേറ്റീവ് `@AiService` ഇന്റർഫേസുകളാണു്. സ്റ്റാർട്ടപ്പിൽ സ്പ്രിംഗ് ബൂട്ട് എല്ലാ `@Component`കൾ കണ്ടുപിടിക്കുകയും അവയിലെ `@Tool` മെത്തഡുകളും, നിങ്ങളുടെ `ChatModel` ബീനും, `ChatMemoryProvider`-ഉം ഒന്നായി ഒരു `Assistant` ഇന്റർഫേസിലേക്ക് യോജിപ്പിക്കുകയും ചെയ്യുന്നു. ഇതിനായുള്ള ബോയ്ലർപ്ലേറ്റ് ഒന്നും വേണ്ട.

<img src="../../../translated_images/ml/spring-boot-wiring.151321795988b04e.webp" alt="സ്പ്രിംഗ് ബൂട്ട് ഓട്ടോ-വൈറിംഗ് ആർക്കിടെക്ചർ" width="800"/>

*@AiService ഇന്റർഫേസ് ChatModel, ടൂൾ കോമ്പോണന്റുകൾ, മെമ്മറി പ്രൊവൈഡർ എന്നിവയെപ്പൊതിച്ചു ബന്ധിപ്പിക്കുന്നു — സ്പ്രിംഗ് ബൂട്ട് എല്ലാ വൈറിംഗും ഓട്ടോമാറ്റിക്ക് കൈകാര്യം ചെയ്യും.*

ഈ രീതിയുടെ പ്രധാന ഗുണങ്ങൾ:

- **സ്പ്രിംഗ് ബൂട്ട് ഓട്ടോ-വൈറിംഗ്** — ChatModel, ടൂളുകൾ താനായി ഇൻജെക്ടുചെയ്യുന്നു
- **@MemoryId പാറ്റേൺ** — സെഷൻ അടിസ്ഥാനത്തിൽ മെമ്മറി മാനേജ്മെന്റ് സ്വയം
- **ഒറ്റ ഇൻസ്റ്റൻസ്** — അസിസ്റ്റന്റ് ഒരിക്കൽ സൃഷ്ടിച്ച് വേണ്ടത്ര ഫലപ്രദതയ്ക്ക് ഉപയോഗിക്കുന്നു
- **ടൈപ്പ്-സേഫ് എക്സിക്യൂഷൻ** — ജാവ മെത്തഡുകൾ നേരിട്ട് ടൈപ്പ് പരിവർത്തനത്തോടെ കോളുചെയ്യുന്നു
- **മൾട്ടി-ടേൺ ഓർക്കസ്ട്രേഷൻ** — ടൂൾ ചേയിനിംഗ് സ്വയം കൈകാര്യം ചെയ്യുന്നു
- **സീറോ ബോയ്ലർപ്ലേറ്റ്** — മാനുവൽ `AiServices.builder()` കോളുകൾ അല്ല, മെമ്മറി ഹാഷ്‌മാപ്പ് ഒന്നും വേണ്ട

ഇനിയും ചില സമാന രീതികൾ (മാനുവൽ `AiServices.builder()`) കൂടുതൽ കോഡ് ആവശ്യപ്പെടുകയും സ്പ്രിംഗ് ബൂട്ട് ഇന്റഗ്രേഷൻ ഗുണങ്ങൾ നഷ്ടപ്പെടുകയും ചെയ്യും.

## ടൂൾ ചേയിനിംഗ്

**ടൂൾ ചേയിനിംഗ്** — ഒരു ചോദ്യം ഒന്നിലധികം ടൂളുകൾ ആവശ്യപ്പെടുന്നപ്പോൾ ടൂൾ അടിസ്ഥാനമായ ഏജന്റുകളുടെ യാഥാർത്ഥ ശക്തി പ്രകടമാകുന്നു. "സിയാറ്റിലിൽ ഫാരൻഹൈಟ್ കാലാവസ്ഥ എന്ത്?" എന്ന് ചോദിക്കുമ്പോൾ ഏജന്റ് രണ്ട് ടൂളുകൾ കീഴടക്കുന്നു: ആദ്യം `getCurrentWeather` കോളുചെയ്ത് സെൽഷ്യസ് താപനില വീണ്ടെടുക്കുന്നു, പിന്നെ അത് `celsiusToFahrenheit` കോളിൽ നൽകി മാറ്റുന്നു — എല്ലാം ഒരേ സംവാദത്തിൽ.

<img src="../../../translated_images/ml/tool-chaining-example.538203e73d09dd82.webp" alt="ടൂൾ ചേയിനിംഗ് ഉദാഹരണം" width="800"/>

*ടൂൾ ചേയിനിംഗ് പ്രവർത്തനത്തിൽ — ഏജന്റ് ആദ്യം getCurrentWeather കോളും ചെയ്യും, തുടർന്ന് സെൽഷ്യസ് ഫലത്തെ celsiusToFahrenheit കോളിലേക്ക് നൽകുകയും സംയോജിത മറുപടി നൽകുകയും ചെയ്യുന്നു.*

**സൗമ്യമായി പരാജയപ്പെടൽ** — മോക്ക് ഡാറ്റയിൽ ഇല്ലാത്ത നഗരത്തിൽ കാലാവസ്ഥ ആവശ്യപ്പെടുക. ടൂൾ പിശക് സന്ദേശം മടങ്ങിച്ച് AI സഹായം നൽകാൻ കഴിയില്ല എന്ന് വിശദീകരിക്കും, അപ്പോഴുമെന്നു കാണിക്കും. ടൂളുകൾ സുരക്ഷിതമായി പരാജയപ്പെടുന്നു. താഴെ കാണുന്ന ഭര്യമായ രണ്ടു സമീപനങ്ങൾ തമ്മിലുള്ള വ്യത്യാസമാണ് — ശരിയായ പിശക് കൈകാര്യം ചെയ്‌താൽ ഏജന്റ് പിശക് പിടിച്ച് സഹായകമായ മറുപടി നൽകുന്നു; മറ്റുള്ളപോലെ അത് ഇല്ലാത്ത പക്ഷം മുഴുവൻ ആപ്ലിക്കേഷൻ ക്രാഷ് ചെയ്യും:

<img src="../../../translated_images/ml/error-handling-flow.9a330ffc8ee0475c.webp" alt="പിശക് കൈകാര്യം പ്രവാഹം" width="800"/>

*ടൂൾ പരാജയപ്പെടുമ്പോൾ ഏജന്റ് പിശക് പിടിച്ചു ക്രാഷ് కాకാതെ സഹായകമായ വിശദീകരണം നൽകുന്നു.*

ഇത് ഒരേ സംവാദ ടേണിൽ നടക്കുന്നു. ഏജന്റ് സ്വയം നിരവധി ടൂൾ കോളുകൾ ഓർക്കസ്ട്രേറ്റ് ചെയ്യുന്നു.

## ആപ്ലിക്കേഷൻ ചലിപ്പിക്കുക

**വിന്യാസം സ്ഥിരീകരിക്കുക:**

Azure അറ്റവിടുന്ന സമയത്ത് സൃഷ്ടിച്ച `.env` ഫയൽ റൂട്ടിലെ ഉള്ളത് ഉണ്ടെന്ന് ഉറപ്പാക്കുക (Module 01 ൽ). ഈ മോഡ്യൂൾ ഡയറക്ടറിയിൽ നിന്ന് ഓടിക്കുക (`04-tools/`):

**Bash:**
```bash
cat ../.env  # AZURE_OPENAI_ENDPOINT, API_KEY, DEPLOYMENT കാണിക്കണം
```

**PowerShell:**
```powershell
Get-Content ..\.env  # AZURE_OPENAI_ENDPOINT, API_KEY, DEPLOYMENT കാണിക്കണം
```

**ആപ്ലിക്കേഷൻ ആരംഭിക്കുക:**

> **ശ്രദ്ധിക്കുക:** മൂല ഡയറക്ടറിയിൽ നിന്ന് `./start-all.sh` ഉപയോഗിച്ച് പൂർണ്ണ ആപ്ലിക്കേഷനുകൾ ആദ്യം തന്നെ ആരംഭിച്ചിട്ടുണ്ടെങ്കിൽ (Module 01 ലെ വിശദീകരണം പോലെ), ഈ മോഡ്യൂൾ ഇതിനകം 8084 പോർട്ടിൽ പ്രവർത്തിക്കുന്നുണ്ട്. താഴെ കൊടുത്ത സ്റ്റാർട്ട് കമാൻഡുകൾ ഒഴിവാക്കി http://localhost:8084 ൽ നേരിട്ട് പോയാല് മതിയാകും.

**ഓപ്ഷൻ 1: സ്പ്രിംഗ് ബൂട്ട് ഡാഷ്ബോർഡ് ഉപയോഗിക്കുക (VS Code ഉപയോക്താക്കൾക്ക് ശുപാർശ)**

ഡെവ് കണ്ടെയ്നറിൽ സ്പ്രിംഗ് ബൂട്ട് ഡാഷ്ബോർഡ് എക്സ്റ്റൻഷൻ ഉൾപ്പെടുന്നു, ഇത് സ്പ്രിംഗ് ബൂട്ട് ആപ്ലിക്കേഷനുകൾ കൈകാര്യം ചെയ്യുന്നതിനുള്ള ദൃശ്യ അന്തർഫേസ് നൽകുന്നു. ഇത് VS Code Activity ബാറിൽ ഇടതു ഭാഗത്തു സ്പ്രിംഗ് ബൂട്ട് ചിഹ്നം കാണിച്ചിരിക്കും.

സ്പ്രിംഗ് ബൂട്ട് ഡാഷ്ബോർഡിൽ നിങ്ങൾക്ക്:  
- പ്രവർത്തന മേഖലയിലുള്ള എല്ലാ സ്പ്രിംഗ് ബൂട്ട് ആപ്ലിക്കേഷനുകളും കാണാം  
- ഒറ്റ ക്ലിക്കിൽ ആപ്ലിക്കേഷനുകൾ ആരംഭിക്കാനും നിർത്താനും കഴിയും  
- ആപ്ലിക്കേഷൻ ലോഗുകൾ യഥാസമയം കാണാം  
- ആപ്ലിക്കേഷൻ നില നിരീക്ഷിക്കാം  

"tools" എന്ന മോഡ്യൂൾക്കൊപ്പം ഉള്ള പ്ലേ ബട്ടൺ ക്ലിക് ചെയ്ത് അല്ലെങ്കിൽ എല്ലാ മോഡ്യൂളുകളും ഒന്നിച്ച് ആരംഭിക്കാം.

VS Code ലെ സ്പ്രിംഗ് ബൂട്ട് ഡാഷ്ബോർഡിന്റെ കാണപ്പ്:

## ആപ്ലിക്കേഷൻ ഉപയോഗിക്കുന്നത്

അപ്ലിക്കേഷൻ ഒരു വെബ് ഇന്റർഫേസ് നൽകുന്നു, അവിടെ നിങ്ങൾക്ക് കാലാവസ്ഥയും താപനില പരിവർത്തനവും ടൂളുകൾ ഉപയോഗിക്കുന്ന AI ഏജന്റുമായുള്ള സംവാദം നടത്താനാകും. ഇന്റർഫേസ് ഇങ്ങിനെയാണ് — ഇത് ക്വിക്ക്-സ്റ്റാർട്ട് ഉദാഹരണങ്ങളും അഭ്യർത്ഥനകൾ അയയ്ക്കാനുള്ള ചാറ്റ് പാനലും ഉൾക്കൊള്ളുന്നു:
<a href="images/tools-homepage.png"><img src="../../../translated_images/ml/tools-homepage.4b4cd8b2717f9621.webp" alt="AI ഏജന്റ് ടൂളുകൾ ഇന്റർഫേസ്" width="800" style="border: 1px solid #ddd; box-shadow: 0 2px 8px rgba(0,0,0,0.1);"/></a>

*AI ഏജന്റ് ടൂളുകൾ ഇന്റർഫേസ് - ടൂളുകളുമായി ഇടപഴകുന്നതിനുള്ള ദ്രുത ഉദാഹരണങ്ങളും ചാറ്റ് ഇന്റർഫേസും*

### ലളിതമായ ടൂൾ ഉപയോഗം പരീക്ഷിക്കുക

ഒരു നേരിട്ട് അഭ്യർത്ഥന നേടുക: "100 ഡിഗ്രി ഫാരൻഹീറ്റിനെ സെൽഷ്യസിലേക്ക് മാറ്റുക". ഏജന്റ് താപനില പരിവർത്തന ടൂൾ ആവശ്യ olduğunu തിരിച്ചറിഞ്ഞ്, ശരിയായ പാർാമീറ്ററുകളോടെ അതിനെ കോൾ ചെയ്ത് ഫലം തിരിച്ചുനൽകുന്നു. ഇത് എത്ര സ്വാഭാവികമാണെന്ന് ശ്രദ്ധിക്കുക - നിങ്ങൾ ഏതുകാര്യത്തിനയറത്ത് ടൂൾ ഉപയോഗിക്കുമെന്ന് അല്ലെങ്കിൽ അത് എങ്ങനെ കോൾ ചെയ്യുമെന്ന് വ്യക്തമാക്കിയിരുന്നില്ല.

### ടൂൾ ചേയിനിംഗ് പരീക്ഷിക്കുക

ഇപ്പോൾ ഒരു കൂടുതൽ സങ്കീർണ്ണമായ കാര്യം ശ്രമിക്കുക: "സീയാറ്റിലും കാലവസ്ഥ എന്ത് ആണ്, അത് ഫാരൻഹീറ്റിലേക്ക് മാറ്റൂ?" എന്ന ആജന്റ് ഇത് ഘട്ടങ്ങളായി പ്രവർത്തിക്കുന്നതും കാണുക. ആദ്യം കാലാവസ്ഥ ലഭിക്കുന്നു (സെൽഷ്യസ് തരികയാകും), ഫാരൻഹീറ്റിലേക്ക് മാറ്റേണ്ടത് തിരിച്ചറിയുന്നു, പരിവർത്തന ടൂൾ കോൾ ചെയ്യുന്നു, പിന്നെ രണ്ട് ഫലങ്ങളും ഒന്നായി ചേർത്ത് മറുപടി നൽകുന്നു.

### സംവാദ ഒഴുക്ക് കാണുക

ചാറ്റ് ഇന്റർഫേസ് സംവാദ ചരിത്രം സൂക്ഷിക്കുകയും ബഹുവട്ടിക ഇടപെടലുകൾ കൈകാര്യം ചെയ്യുകയും ചെയ്യുന്നു. എല്ലാ മുൻവറിപ്പിക്കുന്ന ചോദ്യങ്ങളും മറുപടികളും കാണാൻ കഴിയും, സംവാദം എങ്ങനെ പ്രീസ്ലിപ്പിക്കുന്നു എന്നതും ഏജന്റ് എങ്ങനെ നിരവധി പരസ്പരം സംബന്ധപെട്ട സംഭാഷണങ്ങളിലും സന്ദർഭം നിർമ്മിക്കുന്നു എന്നും ധാരാളം മനസിലാക്കാൻ സുലഭമാകും.

<a href="images/tools-conversation-demo.png"><img src="../../../translated_images/ml/tools-conversation-demo.89f2ce9676080f59.webp" alt="ബഹു ടൂൾ കോൾസോടുകൂടിയ സംവാദം" width="800" style="border: 1px solid #ddd; box-shadow: 0 2px 8px rgba(0,0,0,0.1);"/></a>

*ലളിതമായ പരിവർത്തനങ്ങൾ, കാലാവസ്ഥ അന്വേഷണങ്ങൾ, ടൂൾ ചേയിനിംഗ് എന്നിവയെ göstərിക്കുന്ന ബഹുവട്ടിക സംവാദം*

### വ്യത്യസ്ത അഭ്യർത്ഥനകളോട് പരീക്ഷണം നടത്തുക

വിവിധ സംയോജനങ്ങൾ പരീക്ഷിക്കുക:  
- കാലാവസ്ഥ അന്വേഷണം: "ടോക്കിയോയിൽ കാലാവസ്ഥ എന്ത് ആണ്?"  
- താപനില പരിവർത്തനങ്ങൾ: "25°C എത്ര കെൽവിൻ ആണ്?"  
- സംയുക്ത ചോദ്യങ്ങൾ: "പാരിസിൽ കാലാവസ്ഥ പരിശോധിച്ച് 20°C ആകുന്നതിലും മേലാണോയെന്ന് പറയൂ"  

ബ്രഹ്മാണ്ഡ ഭാഷയെ ഏജന്റ് എങ്ങനെ വ്യാഖ്യാനിക്കുകയും അനുയോജ്യമായ ടൂൾ കോൾസിലേക്ക് മാപ്പുചെയ്‌തിനും ശ്രദ്ധിക്കുക.

## പ്രധാന ആശയങ്ങൾ

### ReAct പാറ്റേൺ (സമാധാനവും പ്രവൃത്തിയും)

എജന്റ് നിർദ്യോഷങ്ങളും പ്രവർത്തനങ്ങളും (ടൂൾ ഉപയോഗവും) ഇടക്കാലം നടത്തുന്നു. ഈ മാതൃക സ്വയംപരിഹാരം സൌകര്യമാക്കുന്നു, നിർദേശങ്ങൾ പാലിക്കുന്നതിനു മാത്രമല്ല.

### ടൂൾ വിവരണങ്ങൾ പ്രധാനമാണ്

നിങ്ങളുടെ ടൂൾ വിവരണങ്ങളുടെ ഗുണമേന്മ ഈജന്റ് അവ എങ്ങനെ ഉപയോഗിക്കുന്നു എന്നതിനെ നേരിട്ട് ബാധിക്കുന്നു. വ്യക്തവും പ്രത്യേകവും ആയ വിവരണങ്ങൾ മോഡലിന് ഓരോ ടൂളും എപ്പോൾ എങ്ങനെ കോൾ ചെയ്യണമെന്ന് മനസ്സിലാക്കുന്നതിൽ സഹായിക്കുന്നു.

### സെഷൻ മാനേജ്മെന്റ്

`@MemoryId` അനൊട്ടേഷൻ സ്വയം പ്രവർത്തിക്കുന്ന സെഷൻ അടിസ്ഥാനമാക്കിയുള്ള മെമ്മറി മാനേജ്മെന്റിനെ സാധ്യമാക്കുന്നു. ഓരോ സെഷൻ ഐഡി `ChatMemory` എൻസ്റ്റൻസ് കിട്ടുന്നു, ഇത് `ChatMemoryProvider` ബീൻ കൈകാര്യം ചെയ്യുന്നു, അതുകൂടെ ഒരേ സമയം ബഹുവ്യക്തികൾ ഏജന്റുമായി ഇടപഴകുമ്പോഴും അവരുടെ സംഭാഷണങ്ങൾ മിശ്രിതമാവാതെ നിരീക്ഷിക്കപ്പെടുന്നു. താഴെ കാണിച്ചിരിക്കുന്ന ചിത്രം ബഹു വ്യക്തികളെ അവരുടെ സെഷൻ ഐഡികൾ അടിസ്ഥാനമാക്കി വേർതിരിച്ച മെമ്മറി സ്റ്റോറുകളിലേക്കു വഴി നല്കുന്നു:

<img src="../../../translated_images/ml/session-management.91ad819c6c89c400.webp" alt="@MemoryId ഉള്ള സെഷൻ മാനേജ്മെന്റ്" width="800"/>

*ഓരോ സെഷൻ ഐഡിയും വേർതിരിച്ച സംവാദ ചരിത്രത്തിലേക്ക് മാപ്പുചെയ്‌തതാണ് — ഉപയോക്താക്കൾ തമ്മിന്റേതായ സന്ദേശങ്ങൾ ഒരിക്കലും കാണാറില്ല.*

### പങ്ക് കൈകാര്യം ചെയ്യൽ

ടൂളുകൾ പരാജയപ്പെടാം — API ടൈംഔട്ട്‌, പാർാമീറ്റർ തെറ്റായിരിക്കും, പുറം സേവനങ്ങൾ പ്രവർത്തിക്കുന്നില്ല. ഉത്പാദന ഏജന്റുകൾക്ക് പിഴവുകൾ കൈകാര്യം ചെയ്യണം, അത് മോഡലിന് പ്രശ്നങ്ങൾ വിശദീകരിക്കാൻ അല്ലെങ്കിൽ പകരം മാർഗങ്ങൾ പരീക്ഷിക്കാൻ നൽകുന്നു, പൂർണ്ണ ആപ്ലിക്കേഷൻ തകരാറിലാകാതിരിക്കാൻ. ഒരു ടൂൾ എക്സെപ്ഷൻ എറിയുമ്പോൾ, LangChain4j അത് പിടിച്ച് പിഴവു സന്ദേശം മോഡലിനാക്കി നൽകുന്നു, അത് തുടർന്ന് സ്വാഭാവിക ഭാഷയിൽ പ്രശ്നം വിശദീകരിക്കും.

## ലഭ്യമാകുന്ന ടൂളുകൾ

താഴെയുള്ള ഛർമ്മം നിങ്ങൾക്കായി നിർമ്മിക്കാവുന്ന ടൂൾ പരിസ്ഥിതിയെ കാണിക്കുന്നു. ഈ മോഡ്യൂൾ കാലാവസ്ഥയും താപനില ടൂളുകളും പ്രദർശിപ്പിക്കുന്നു, പക്ഷേ സമാന `@Tool` പാറ്റേൺ ഏതെങ്കിലും ജാവ മെത്തഡിനോ പ്രയോഗിക്കാം - ഡാറ്റാബേസ് അഭ്യാസ അലങ്കാരങ്ങളോ പേയ്‌മെന്റ് പ്രോസസ്സിംഗോ ഉൾപ്പെടെ.

<img src="../../../translated_images/ml/tool-ecosystem.aad3d74eaa14a44f.webp" alt="ടൂൾ പരിസ്ഥിതി" width="800"/>

*@Tool അനോട്ടേഷൻ ഉപയോഗിച്ചുള്ള ഏത് ജാവ മെത്തഡും AI-ന് ലഭ്യമായിരിക്കും — പാറ്റേൺ ഡാറ്റാബേസുകൾ, APIകൾ, ഇമെയിൽ, ഫയൽ പ്രവർത്തനങ്ങൾ, ഇനി മറ്റു പലതും ഉൾപ്പെടുത്തുന്നു.*

## ടൂൾ അടിസ്ഥാനമായ ഏജന്റുകൾ എപ്പോൾ ഉപയോഗിക്കാം

പ്രതി എല്ലാ അഭ്യർത്ഥനകളും ടൂളുകളെ ആവശ്യമായിട്ടില്ല. തീരുമാനമുണ്ടാകുന്നത് AIയ്ക്ക് ബാഹ്യ സംവിധാനങ്ങളുമായി ഇടപഴകേണ്ടതുണ്ടോ എന്നതും അതിന്റെ സ്വന്തം അറിവിൽ നിന്ന് മറുപടി നൽകാമോ എന്നതുമായിരിക്കും. താഴെ വരുന്ന ഗൈഡു ടൂളുകൾ മൂല്യം കൂട്ടുന്ന അവസരങ്ങളും ആവശ്യമില്ലാത്ത സാഹചര്യങ്ങളും അണിനിരത്തുന്നു:

<img src="../../../translated_images/ml/when-to-use-tools.51d1592d9cbdae9c.webp" alt="ടൂൾ ഉപയോഗിക്കേണ്ടപ്പോൾ" width="800"/>

*തൽവ സ്ഥലത്തുള്ള ഡാറ്റ, കണക്കുകൂട്ടലുകൾ, പ്രവർത്തനങ്ങൾ എന്നിവയ്ക്ക് ടൂളുകൾ വേണ്ടി വരും; പൊതുവായ അറിവും സൃഷ്ടിപ്രധാന ജോലികളും ആവശ്യമില്ല.*

## ടൂളുകൾ vs RAG

മോഡ്യൂളുകൾ 03, 04 രണ്ട് ഭിന്നമായ സമീപനങ്ങളിൽ AIന് അധികാരങ്ങൾ നൽകുന്നു. RAG മോഡലിന് **അറിവ്** ലഭ്യമാക്കാൻ ഡോക്യുമെന്റുകൾ കണ്ടെത്തുന്നു. ടൂളുകൾ ഫംഗ്ഷനുകൾ കോൾ ചെയ്ത് **പ്രവർത്തനങ്ങൾ** കൈകാര്യം ചെയ്യാൻ കഴിവ് നൽകുന്നു. താഴെയുള്ള ചിത്രം രണ്ട് സമീപനങ്ങളെ സൈഡ് ബൈ സൈഡ് താരതമ്യം ചെയ്യുന്നു — ഓരോ തൊഴിലായി പ്രവർത്തിക്കുന്ന രീതി മുതൽ അവയുടെ ഗുണദോഷങ്ങൾ വരെ:

<img src="../../../translated_images/ml/tools-vs-rag.ad55ce10d7e4da87.webp" alt="ടൂളുകൾ vs RAG താരതമ്യം" width="800"/>

*RAG സ്റ്റാറ്റിക് ഡോക്യുമെന്റുകളിൽ നിന്ന് വിവരങ്ങൾ കണ്ടെത്തുന്നു — ടൂൾസ് പ്രവർത്തനങ്ങൾ നിർവഹിക്കുകയും ജീവിച്ചിരിക്കുന്ന, തത്സമയ ഡാറ്റ സ്വീകരിക്കുകയും ചെയ്യുന്നു. പല ഉത്പാദന സംവിധാനങ്ങളും ഈ രണ്ട് സമീപനങ്ങളും സംയോജിപ്പിക്കുന്നു.*

പ്രായോഗികമായി പല ഉത്പാദന സംവിധാനങ്ങളും ഈ രണ്ട് സമീപനങ്ങളും ഉദാത്തമായി ചേർത്ത് ഉപയോഗിക്കുന്നു: RAG നിങ്ങളുടെ പ്രമാണങ്ങളിൽ മുന്നോട്ടുള്ള മറുപടികൾക്കായി, ടൂളുകൾ സജീവ ഡാറ്റ ശേഖരിക്കൽക്കോ പ്രവർത്തനങ്ങൾ നടത്തലിനോ.
