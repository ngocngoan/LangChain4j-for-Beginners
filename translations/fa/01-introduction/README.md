## آنچه می‌آموزید

اگر سریع شروع کردید، دیدید چگونه پیام‌ها را ارسال کنید و پاسخ بگیرید. این پایه است، اما برنامه‌های واقعی به بیشتر نیاز دارند. این ماژول به شما آموزش می‌دهد چگونه یک هوش مصنوعی مکالمه‌ای بسازید که متن را به خاطر بسپارد و حالت را حفظ کند — تفاوت بین یک دمو یک‌باره و یک برنامه آماده تولید.

ما در این راهنما از GPT-5.2 آژور OpenAI استفاده می‌کنیم چون قابلیت‌های استدلال پیشرفته‌اش رفتار الگوهای متفاوت را واضح‌تر نشان می‌دهد. وقتی حافظه اضافه می‌کنید، تفاوت را واضح می‌بینید. این فهمیدن اینکه هر بخش چه ارزشی به برنامه شما می‌آورد را آسان‌تر می‌کند.

شما یک برنامه می‌سازید که هر دو الگو را نشان می‌دهد:

**چت بدون حالت** — هر درخواست مستقل است. مدل حافظه‌ای از پیام‌های قبلی ندارد. این همان الگویی است که در شروع سریع استفاده کردید.

**مکالمه با حالت** — هر درخواست شامل تاریخچه مکالمه است. مدل متن زمینه را در چندین نوبت حفظ می‌کند. این همان چیزی است که برنامه‌های تولیدی نیاز دارند.

## پیش‌نیازها

- اشتراک Azure با دسترسی به Azure OpenAI
- جاوا ۲۱، Maven 3.9+
- Azure CLI (https://learn.microsoft.com/en-us/cli/azure/install-azure-cli)
- Azure Developer CLI (azd) (https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/install-azd)

> **توجه:** جاوا، Maven، Azure CLI و Azure Developer CLI (azd) در devcontainer ارائه‌شده از پیش نصب شده‌اند.

> **توجه:** این ماژول از GPT-5.2 روی Azure OpenAI استفاده می‌کند. استقرار به‌صورت خودکار از طریق `azd up` تنظیم شده است — نام مدل را در کد تغییر ندهید.

## درک مسئله اصلی

مدل‌های زبان بدون حالت هستند. هر تماس API مستقل است. اگر بگویید "نام من جان است" و سپس بپرسید "نام من چیست؟"، مدل نمی‌داند که تازه خودتان را معرفی کردید. هر درخواست را انگار اولین مکالمه‌ای که تاکنون داشتید می‌داند.

این برای پرسش و پاسخ ساده کافی است اما برای برنامه‌های واقعی بی‌فایده است. ربات‌های خدمات مشتری باید آنچه گفته‌اید را به خاطر بسپارند. دستیاران شخصی به زمینه نیاز دارند. هر مکالمه چند نوبتی نیازمند حافظه است.

*تفاوت بین گفتگوهای بدون حالت (تماس‌های مستقل) و با حالت (آگاه به متن)*

## درک توکن‌ها

قبل از ورود به مکالمات، مهم است توکن‌ها را درک کنید — واحدهای پایه متن که مدل‌های زبان پردازش می‌کنند:

<img src="../../../translated_images/fa/token-explanation.c39760d8ec650181.webp" alt="توضیح توکن" width="800"/>

*نمونه‌ای از نحوه شکستن متن به توکن‌ها — "I love AI!" تبدیل به ۴ واحد مستقل پردازش می‌شود*

توکن‌ها راهی هستند که مدل‌های AI متن را اندازه‌گیری و پردازش می‌کنند. کلمات، علائم نگارشی و حتی فاصله‌ها می‌توانند توکن باشند. مدل شما محدودیتی دارد که چقدر توکن می‌تواند همزمان پردازش کند (۴۰۰,۰۰۰ برای GPT-5.2، با حداکثر ۲۷۲,۰۰۰ توکن ورودی و ۱۲۸,۰۰۰ توکن خروجی). درک توکن‌ها به شما کمک می‌کند طول مکالمه و هزینه‌ها را مدیریت کنید.

## چگونگی کار حافظه

حافظه چت مشکل عدم حالت را با حفظ تاریخچه مکالمه حل می‌کند. قبل از ارسال درخواست به مدل، فریمورک پیام‌های مرتبط قبلی را اضافه می‌کند. وقتی می‌پرسید "نام من چیست؟"، سیستم در واقع کل تاریخچه مکالمه را ارسال می‌کند تا مدل ببیند قبلاً گفتید "نام من جان است."

LangChain4j پیاده‌سازی‌های حافظه‌ای دارد که این کار را به‌صورت خودکار انجام می‌دهد. شما تعداد پیام‌هایی را که می‌خواهید نگه دارید انتخاب می‌کنید و فریمورک پنجره متن را مدیریت می‌کند.

<img src="../../../translated_images/fa/memory-window.bbe67f597eadabb3.webp" alt="مفهوم پنجره حافظه" width="800"/>

*MessageWindowChatMemory پنجره‌ای لغزنده از پیام‌های اخیر را حفظ می‌کند و به‌صورت خودکار پیام‌های قدیمی‌تر را حذف می‌کند*

## نحوه استفاده از LangChain4j در این ماژول

این ماژول شروع سریع را با ادغام Spring Boot و افزودن حافظه مکالمه گسترش می‌دهد. قطعات به این صورت بهم متصل می‌شوند:

**وابستگی‌ها** — افزودن دو کتابخانه LangChain4j:

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```
  
**مدل چت** — پیکربندی Azure OpenAI به عنوان bean اسپرینگ ([LangChainConfig.java](../../../01-introduction/src/main/java/com/example/langchain4j/config/LangChainConfig.java)):

```java
@Bean
public OpenAiOfficialChatModel openAiOfficialChatModel() {
    return OpenAiOfficialChatModel.builder()
            .baseUrl(azureEndpoint)
            .apiKey(azureApiKey)
            .modelName(deploymentName)
            .timeout(Duration.ofMinutes(5))
            .maxRetries(3)
            .build();
}
```
  
سازنده اعتبارنامه‌ها را از متغیرهای محیطی که توسط `azd up` تنظیم شده است، می‌خواند. تنظیم `baseUrl` به نقطه پایان Azure شما باعث می‌شود کلاینت OpenAI با Azure OpenAI کار کند.

**حافظه مکالمه** — پیگیری تاریخچه چت با MessageWindowChatMemory ([ConversationService.java](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java)):

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("My name is John"));
memory.add(AiMessage.from("Nice to meet you, John!"));

memory.add(UserMessage.from("What's my name?"));
AiMessage aiMessage = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage);
```
  
با `withMaxMessages(10)` حافظه ایجاد کنید تا ۱۰ پیام آخر نگه داشته شود. پیام‌های کاربر و AI را با بسته‌های تایپ‌شده اضافه کنید: `UserMessage.from(text)` و `AiMessage.from(text)`. تاریخچه را با `memory.messages()` بازیابی و برای مدل ارسال کنید. سرویس نمونه‌های حافظه جداگانه برای هر شناسه مکالمه ذخیره می‌کند و اجازه می‌دهد چندین کاربر همزمان چت کنند.

> **🤖 امتحان با [GitHub Copilot](https://github.com/features/copilot) Chat:** فایل [`ConversationService.java`](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java) را باز کنید و بپرسید:  
> - "چطور MessageWindowChatMemory تصمیم می‌گیرد وقتی پنجره پر است کدام پیام‌ها را حذف کند؟"  
> - "آیا می‌توانم پیاده‌سازی حافظه سفارشی با استفاده از پایگاه داده به جای حافظه درون‌برنامه‌ای انجام دهم؟"  
> - "چگونه می‌توانم خلاصه‌سازی برای فشرده‌سازی تاریخچه مکالمه قدیمی اضافه کنم؟"

نقطه پایانی چت بدون حالت کل حافظه را نادیده می‌گیرد — فقط `chatModel.chat(prompt)` مانند شروع سریع. نقطه پایانی با حالت پیام‌ها را به حافظه اضافه می‌کند، تاریخچه را بازیابی می‌کند و متن زمینه را با هر درخواست می‌فرستد. پیکربندی مدل همان است، الگوها متفاوت.

## استقرار زیرساخت Azure OpenAI

**بشقاب:**
```bash
cd 01-introduction
azd up  # اشتراک و مکان را انتخاب کنید (eastus2 توصیه می‌شود)
```
  
**پاورشل:**
```powershell
cd 01-introduction
azd up  # اشتراک و مکان را انتخاب کنید (توصیه‌شده eastus2)
```
  

> **توجه:** اگر با خطای زمان‌تمامی مواجه شدید (`RequestConflict: Cannot modify resource ... provisioning state is not terminal`)، به‌سادگی دوباره `azd up` را اجرا کنید. ممکن است منابع Azure هنوز در حال آماده‌سازی باشند و تلاش مجدد اجازه می‌دهد استقرار زمانی که منابع حالت نهایی رسیدند کامل شود.

این کار انجام می‌دهد:  
۱. منابع Azure OpenAI با مدل‌های GPT-5.2 و text-embedding-3-small مستقر می‌کند  
۲. فایل `.env` را خودکار در ریشه پروژه با اعتبارنامه‌ها ایجاد می‌کند  
۳. همه متغیرهای محیطی مورد نیاز را تنظیم می‌کند

**دچار مشکل در استقرار هستید؟** به [README زیرساخت](infra/README.md) مراجعه کنید برای رفع اشکال شامل تضاد نام زیردامنه، مراحل استقرار دستی در Azure Portal و راهنمای پیکربندی مدل.

**بررسی موفقیت استقرار:**

**بشقاب:**
```bash
cat ../.env  # باید AZURE_OPENAI_ENDPOINT، API_KEY و غیره نمایش داده شود.
```
  
**پاورشل:**
```powershell
Get-Content ..\.env  # باید AZURE_OPENAI_ENDPOINT، API_KEY و غیره را نشان دهد.
```
  

> **توجه:** دستور `azd up` به‌صورت خودکار فایل `.env` را ایجاد می‌کند. اگر بعداً نیاز به بروزرسانی داشتید، می‌توانید فایل `.env` را به‌صورت دستی ویرایش کنید یا با اجرای مجدد:  
>  
> **بشقاب:**
> ```bash
> cd ..
> bash .azd-env.sh
> ```
>  
> **پاورشل:**
> ```powershell
> cd ..
> .\.azd-env.ps1
> ```
  

## اجرای برنامه به صورت محلی

**بررسی استقرار:**

اطمینان حاصل کنید فایل `.env` در دایرکتوری ریشه با اعتبارنامه Azure وجود دارد:

**بشقاب:**
```bash
cat ../.env  # باید AZURE_OPENAI_ENDPOINT، API_KEY، DEPLOYMENT را نشان دهد
```
  
**پاورشل:**
```powershell
Get-Content ..\.env  # باید AZURE_OPENAI_ENDPOINT، API_KEY، DEPLOYMENT را نشان دهد
```
  

**اجرای برنامه‌ها:**

**گزینه ۱: استفاده از داشبورد Spring Boot (توصیه شده برای کاربران VS Code)**

dev container افزونه Spring Boot Dashboard را شامل می‌شود که یک رابط بصری برای مدیریت همه برنامه‌های Spring Boot فراهم می‌کند. می‌توانید آن را در نوار فعالیت سمت چپ VS Code پیدا کنید (علامت Spring Boot را جستجو کنید).

از داشبورد Spring Boot، می‌توانید:  
- همه برنامه‌های Spring Boot موجود در فضای کاری را ببینید  
- برنامه‌ها را با یک کلیک راه‌اندازی/متوقف کنید  
- لاگ‌های برنامه را به‌صورت زنده مشاهده کنید  
- وضعیت برنامه را نظارت کنید

به سادگی روی دکمه پخش کنار "introduction" کلیک کنید تا این ماژول را اجرا کنید، یا همه ماژول‌ها را همزمان اجرا کنید.

<img src="../../../translated_images/fa/dashboard.69c7479aef09ff6b.webp" alt="داشبورد Spring Boot" width="400"/>

**گزینه ۲: استفاده از اسکریپت‌های shell**

برای اجرای همه برنامه‌های وب (ماژول‌های ۰۱ تا ۰۴):

**بشقاب:**
```bash
cd ..  # از دایرکتوری ریشه
./start-all.sh
```
  
**پاورشل:**
```powershell
cd ..  # از دایرکتوری ریشه
.\start-all.ps1
```
  

یا فقط این ماژول را اجرا کنید:

**بشقاب:**
```bash
cd 01-introduction
./start.sh
```
  
**پاورشل:**
```powershell
cd 01-introduction
.\start.ps1
```
  

هر دو اسکریپت متغیرهای محیطی را از فایل `.env` ریشه بارگذاری می‌کنند و در صورتی که فایل‌های JAR وجود نداشته باشند، آنها را می‌سازند.

> **توجه:** اگر ترجیح می‌دهید همه ماژول‌ها را به‌صورت دستی قبل از اجرا بسازید:  
>  
> **بشقاب:**
> ```bash
> cd ..  # Go to root directory
> mvn clean package -DskipTests
> ```
>  
> **پاورشل:**
> ```powershell
> cd ..  # Go to root directory
> mvn clean package -DskipTests
> ```
  

آدرس http://localhost:8080 را در مرورگر خود باز کنید.

**برای توقف:**

**بشقاب:**
```bash
./stop.sh  # فقط این ماژول
# یا
cd .. && ./stop-all.sh  # همه ماژول‌ها
```
  
**پاورشل:**
```powershell
.\stop.ps1  # فقط این ماژول
# یا
cd ..; .\stop-all.ps1  # همه ماژول‌ها
```
