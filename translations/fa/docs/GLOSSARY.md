## مفاهیم اصلی

**عامل AI** - سامانه‌ای که از هوش مصنوعی برای استدلال و عمل خودکار استفاده می‌کند. [ماژول 04](../04-tools/README.md)

**زنجیره** - دنباله‌ای از عملیات که خروجی به مرحله بعدی داده می‌شود.

**تقسیم‌بندی** - شکستن اسناد به قطعات کوچکتر. معمولاً: ۳۰۰-۵۰۰ توکن با هم‌پوشانی. [ماژول 03](../03-rag/README.md)

**پنجره زمینه** - بیشینه توکن‌هایی که یک مدل می‌تواند پردازش کند. GPT-5.2: ۴۰۰ هزار توکن.

**بردارهای جاسازی** - بردارهای عددی که نمایانگر معنای متن هستند. [ماژول 03](../03-rag/README.md)

**فراخوانی تابع** - مدلی که درخواست‌هایی ساختاریافته برای فراخوانی توابع خارجی تولید می‌کند. [ماژول 04](../04-tools/README.md)

**توهم** - زمانی که مدل‌ها اطلاعات نادرست اما قابل قبول تولید می‌کنند.

**درخواست** - ورودی متنی به مدل زبان. [ماژول 02](../02-prompt-engineering/README.md)

**جستجوی معنایی** - جستجو بر اساس معنا با استفاده از بردارهای جاسازی، نه کلمات کلیدی. [ماژول 03](../03-rag/README.md)

**حالت‌دار در برابر بدون‌حالت** - بدون‌حالت: بدون حافظه. حالت‌دار: تاریخچه گفتگو را نگه می‌دارد. [ماژول 01](../01-introduction/README.md)

**توکن‌ها** - واحدهای اصلی متن که مدل‌ها پردازش می‌کنند. بر هزینه‌ها و محدودیت‌ها تأثیر می‌گذارد. [ماژول 01](../01-introduction/README.md)

**زنجیره ابزار** - اجرای متوالی ابزارها که خروجی برای فراخوانی بعدی اطلاع می‌دهد. [ماژول 04](../04-tools/README.md)

## اجزای LangChain4j

**AiServices** - ایجاد رابط‌های سرویس AI با نوع امن.

**OpenAiOfficialChatModel** - مشتری متحد برای مدل‌های OpenAI و Azure OpenAI.

**OpenAiOfficialEmbeddingModel** - ایجاد جاسازی‌ها با استفاده از مشتری رسمی OpenAI (پشتیبانی از هر دو OpenAI و Azure OpenAI).

**ChatModel** - رابط اصلی برای مدل‌های زبان.

**ChatMemory** - نگهداری تاریخچه گفتگو.

**ContentRetriever** - یافتن قطعات مستند مرتبط برای RAG.

**DocumentSplitter** - شکستن اسناد به قطعات.

**EmbeddingModel** - تبدیل متن به بردارهای عددی.

**EmbeddingStore** - ذخیره و بازیابی بردارهای جاسازی.

**MessageWindowChatMemory** - حفظ پنجره لغزان پیام‌های اخیر.

**PromptTemplate** - ایجاد درخواست‌های قابل استفاده مجدد با جایگزین‌های `{{variable}}`.

**TextSegment** - قطعه‌ای از متن با فراداده. در RAG استفاده می‌شود.

**ToolExecutionRequest** - نمایانگر درخواست اجرای ابزار.

**UserMessage / AiMessage / SystemMessage** - انواع پیام‌های گفتگو.

## مفاهیم AI/ML

**آموزش چندتایی** - ارائه نمونه‌ها در درخواست‌ها. [ماژول 02](../02-prompt-engineering/README.md)

**مدل زبان بزرگ (LLM)** - مدل‌های هوش مصنوعی که بر روی داده متن بسیار زیاد آموزش دیده‌اند.

**تلاش استدلالی** - پارامتر GPT-5.2 برای کنترل عمق تفکر. [ماژول 02](../02-prompt-engineering/README.md)

**دما** - کنترل تصادفی بودن خروجی. پایین = قطعی، بالا = خلاقانه.

**پایگاه داده برداری** - پایگاه داده تخصصی برای بردارهای جاسازی. [ماژول 03](../03-rag/README.md)

**آموزش بدون نمونه** - انجام کارها بدون نمونه. [ماژول 02](../02-prompt-engineering/README.md)

## محدودیت‌ها - [ماژول 00](../00-quick-start/README.md)

**دفاع در عمق** - رویکرد امنیت چندلایه که محدودیت‌های سطح برنامه را با فیلترهای ایمنی ارائه‌دهنده ترکیب می‌کند.

**مسدودسازی سخت** - ارائه‌دهنده خطای HTTP ۴۰۰ برای تخلفات شدید محتوا صادر می‌کند.

**InputGuardrail** - رابط LangChain4j برای اعتبارسنجی ورودی کاربر پیش از رسیدن به LLM. با مسدود کردن زودهنگام درخواست‌های مضر، هزینه و تأخیر را کاهش می‌دهد.

**InputGuardrailResult** - نوع بازگشتی برای اعتبارسنجی محدودیت: `success()` یا `fatal("دلیل")`.

**OutputGuardrail** - رابطی برای اعتبارسنجی پاسخ‌های AI پیش از بازگرداندن به کاربران.

**فیلترهای ایمنی ارائه‌دهنده** - فیلترهای داخلی محتوا از ارائه‌دهندگان AI (مانند GitHub Models) که تخلفات را در سطح API شناسایی می‌کنند.

**رد نرم** - مدل با احترام از پاسخ دادن امتناع می‌کند بدون صادر کردن خطا.

## ماژول عامل‌دار - [ماژول 05](../05-mcp/README.md)

**@Agent Annotation** - علامت‌گذاری رابط‌ها به‌عنوان عوامل AI با تعریف رفتار اعلانی.

**شنونده عامل** - قلابی برای نظارت بر اجرای عامل از طریق `beforeAgentInvocation()` و `afterAgentInvocation()`.

**حوزه عامل‌دار** - حافظه مشترکی که عوامل نتایج را با استفاده از `outputKey` ذخیره می‌کنند تا عوامل پایین‌دستی مصرف کنند.

**AgenticServices** - کارخانه‌ای برای ایجاد عوامل با استفاده از `agentBuilder()` و `supervisorBuilder()`.

**گردش کار شرطی** - مسیر دهی بر اساس شرایط به عوامل متخصص مختلف.

**انسان در حلقه** - الگوی کاری افزودن نقاط بررسی انسانی برای تأیید یا بازبینی محتوا.

**langchain4j-agentic** - وابستگی Maven برای ساخت عامل اعلانی (آزمایشی).

**گردش کار حلقه‌ای** - تکرار اجرای عامل تا برآورده شدن شرطی (مثلاً نمره کیفیت ≥ ۰.۸).

**outputKey** - پارامتر علامت‌گذاری عامل که مشخص می‌کند نتایج کجا در حوزه عامل‌دار ذخیره شود.

**گردش کار موازی** - اجرای چند عامل به طور همزمان برای وظایف مستقل.

**استراتژی پاسخ** - نحوه فرموله کردن پاسخ نهایی توسط سرپرست: آخرین، خلاصه، یا امتیازدهی شده.

**گردش کار متوالی** - اجرای عوامل به ترتیب که خروجی به مرحله بعدی جریان می‌یابد.

**الگوی عامل سرپرست** - الگوی پیشرفته عامل‌دار که در آن LLM سرپرست به طور پویا تصمیم می‌گیرد کدام زیرعامل‌ها فراخوانی شوند.

## پروتکل زمینه مدل (MCP) - [ماژول 05](../05-mcp/README.md)

**langchain4j-mcp** - وابستگی Maven برای ادغام MCP در LangChain4j.

**MCP** - پروتکل زمینه مدل: استانداردی برای اتصال اپلیکیشن‌های AI به ابزارهای خارجی. یک‌بار بسازید، همه‌جا استفاده کنید.

**کلاینت MCP** - برنامه‌ای که به سرورهای MCP متصل می‌شود تا ابزارها را کشف و استفاده کند.

**سرور MCP** - سرویس ارائه‌دهنده ابزارها از طریق MCP با توضیحات و ساختار پارامترهای واضح.

**McpToolProvider** - جزء LangChain4j که ابزارهای MCP را برای استفاده در سرویس‌ها و عامل‌های AI بسته‌بندی می‌کند.

**McpTransport** - رابطی برای ارتباط MCP. پیاده‌سازی‌ها شامل Stdio و HTTP است.

**انتقال Stdio** - انتقال فرآیند محلی از طریق stdin/stdout. برای دسترسی به سیستم فایل یا ابزارهای خط فرمان مفید است.

**StdioMcpTransport** - پیاده‌سازی LangChain4j که سرور MCP را به عنوان زیرفرآیند راه‌اندازی می‌کند.

**کشف ابزار** - کلاینت از سرور درباره ابزارهای موجود همراه توضیحات و ساختار پرس‌وجو می‌کند.
