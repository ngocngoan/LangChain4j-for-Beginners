## מושגי יסוד

**סוכן AI** - מערכת המשתמשת בבינה מלאכותית כדי להסיק מסקנות ולפעול בצורה אוטונומית. [מודול 04](../04-tools/README.md)

**שרשרת** - רצף פעולות שבו הפלט מזין את השלב הבא.

**פרגמנטציה** - פיצול מסמכים לחלקים קטנים יותר. אופייני: 300-500 תווים עם חפיפה. [מודול 03](../03-rag/README.md)

**חלון הקשר** - מספר התווים המקסימלי שמודל יכול לעבד. GPT-5.2: 400K תווים.

**הטמעות** - וקטורים מספריים המייצגים את משמעות הטקסט. [מודול 03](../03-rag/README.md)

**קריאת פונקציות** - המודל יוצר בקשות מובנות לקריאה לפונקציות חיצוניות. [מודול 04](../04-tools/README.md)

**הלוצינציה** - כאשר מודלים מייצרים מידע שגוי אך הגיוני.

**פרומפט** - קלט טקסטואלי למודל שפה. [מודול 02](../02-prompt-engineering/README.md)

**חיפוש סמנטי** - חיפוש לפי משמעות באמצעות הטמעות, לא לפי מילות מפתח. [מודול 03](../03-rag/README.md)

**מצב מבוסס מצב לעומת ללא מצב** - ללא מצב: ללא זיכרון. מבוסס מצב: שומר היסטוריית שיחה. [מודול 01](../01-introduction/README.md)

**תווים** - יחידות טקסט בסיסיות שהמודל מעבד. משפיע על עלויות ומגבלות. [מודול 01](../01-introduction/README.md)

**שרשור כלים** - ביצוע כלי רציף שבו הפלט מספק מידע לקריאה הבאה. [מודול 04](../04-tools/README.md)

## מרכיבי LangChain4j

**AiServices** - יוצר ממשקי שירות בינה מלאכותית מסוג בטוח.

**OpenAiOfficialChatModel** - לקוח מאוחד עבור מודלים של OpenAI ו-Azure OpenAI.

**OpenAiOfficialEmbeddingModel** - יוצר הטמעות באמצעות הלקוח הרשמי של OpenAI (תומך גם ב-OpenAI וגם ב-Azure OpenAI).

**ChatModel** - ממשק ליבה למודלי שפה.

**ChatMemory** - שומר היסטוריית שיחה.

**ContentRetriever** - מוצא חתיכות מסמכים רלוונטיות עבור RAG.

**DocumentSplitter** - מפרק מסמכים לחתיכות.

**EmbeddingModel** - ממיר טקסט לוקטורים מספריים.

**EmbeddingStore** - מאחסן ומשחזר הטמעות.

**MessageWindowChatMemory** - שומר חלון ריצפתי של הודעות אחרונות.

**PromptTemplate** - יוצר פרומפטים לשימוש חוזר עם תבניות `{{variable}}`.

**TextSegment** - חתיכת טקסט עם מטה-נתונים. משומש ב-RAG.

**ToolExecutionRequest** - מייצג בקשת הפעלת כלי.

**UserMessage / AiMessage / SystemMessage** - סוגי הודעות שיחה.

## מושגי AI/ML

**למידה עם דוגמאות מועטות (Few-Shot Learning)** - מתן דוגמאות בפרומפטים. [מודול 02](../02-prompt-engineering/README.md)

**מודל שפה גדול (LLM)** - מודלי AI המאומנים על נתוני טקסט עצומים.

**מאמץ הסקה** - פרמטר GPT-5.2 השולט בעומק החשיבה. [מודול 02](../02-prompt-engineering/README.md)

**טמפרטורה** - שולט ברנדומליות הפלט. נמוך=דטרמיניסטי, גבוה=יצירתי.

**מסד נתונים של וקטורים** - מסד נתונים ייעודי להטמעות. [מודול 03](../03-rag/README.md)

**למידה ללא דוגמאות (Zero-Shot Learning)** - ביצוע משימות ללא דוגמאות. [מודול 02](../02-prompt-engineering/README.md)

## שמירות - [מודול 00](../00-quick-start/README.md)

**הגנה בשכבות** - גישת אבטחה רב-שכבתית המשלבת שמירות אפליקטיביות עם מסנני בטיחות של ספקים.

**חסימה קשיחה** - ספק מחזיר שגיאת HTTP 400 עבור הפרות תוכן חמורות.

**InputGuardrail** - ממשק LangChain4j לאימות קלט משתמש לפני הגעתו ל-LLM. חוסך עלויות ועיכובים על ידי חסימת פרומפטים מזיקים מוקדם.

**InputGuardrailResult** - סוג החזרת תוצאה לאימות שמירה: `success()` או `fatal("reason")`.

**OutputGuardrail** - ממשק לאימות תגובות AI לפני החזרתן למשתמשים.

**מסנני בטיחות ספקים** - מסנני תוכן מובנים מספקי AI (כגון GitHub Models) שנתפסים ברמת ה-API.

**סרבנות רכה** - המודל מסרב בנימוס להשיב מבלי להחזיר שגיאה.

**פרומפט מבוסס תפקיד** - קביעת דמות המודל באמצעות הודעות מערכת.

**השתקפות עצמית** - המודל מעריך ומשפר את הפלט שלו.

**ניתוח מובנה** - מסגרת הערכה קבועה.

**תבנית ביצוע משימה** - תכנון → ביצוע → סיכום.

## RAG (Retrieval-Augmented Generation) - [מודול 03](../03-rag/README.md)

**צינור עיבוד מסמכים** - טעינה → פרגמנטציה → הטמעה → אחסון.

**מחסן הטמעות בזיכרון** - אחסון לא מתמיד למטרות בדיקה.

**RAG** - משלב שליפה עם יצירה להענקת מענה מבוסס.

**ציון דמיון** - מדד (0-1) לדמיון סמנטי.

**הפנייה למקור** - מטה-נתונים על התוכן שנשלף.

## סוכנים וכלים - [מודול 04](../04-tools/README.md)

**@Tool Annotation** - מסמן שיטות ב-Java ככלים הניתנים לקריאה על ידי AI.

**תבנית ReAct** - הסקה → פעולה → תצפית → חזרה.

**ניהול מושבים** - הקשרים נפרדים למשתמשים שונים.

**כלי** - פונקציה ש-Agent AI יכול לקרוא לה.

**תיאור כלי** - תיעוד מטרת הכלי ופרמטרים.

## מודול סוכני - [מודול 05](../05-mcp/README.md)

**@Agent Annotation** - מסמן ממשקים כסוכני AI עם הגדרת התנהגות הכרזתית.

**Agent Listener** - קרס למעקב אחר ביצוע סוכן באמצעות `beforeAgentInvocation()` ו-`afterAgentInvocation()`.

**Agentic Scope** - זיכרון משותף שבו סוכנים מאחסנים תוצרים באמצעות `outputKey` לצריכה על ידי סוכנים מפלסים.

**AgenticServices** - מפעל ליצירת סוכנים באמצעות `agentBuilder()` ו-`supervisorBuilder()`.

**זרימת עבודה מותנית** - מסלול בהתאם לתנאים לסוכנים מתמחים שונים.

**אדם בלולאה** - דפוס זרימת עבודה המוסיף נקודות ביקורת אנושיות לאישור או סקירת תוכן.

**langchain4j-agentic** - תלות Maven לבניית סוכנים באופן הכרזתי (ניסיוני).

**לולאת זרימת עבודה** - איטרציה על ביצוע סוכן עד שהתקיים תנאי (למשל ציון איכות ≥ 0.8).

**outputKey** - פרמטר אנוטציית סוכן המציין היכן מאוחסנים התוצאות ב-Agentic Scope.

**זרימת עבודה מקבילה** - הפעלת סוכנים מרובים במקביל למשימות בלתי תלויות.

**אסטרטגיית תגובה** - איך המפקח מנוסח את התשובה הסופית: אחרון, סיכום, או מדורג.

**זרימת עבודה סדרתית** - ביצוע סוכנים לפי סדר שבו הפלט זורם לשלב הבא.

**תבנית סוכן מפקח** - תבנית סוכנות מתקדמת שבה מפקח LLM מחליט בדינמיות אילו סוכנים משניים לקרוא.

## פרוטוקול הקשר מודל (MCP) - [מודול 05](../05-mcp/README.md)

**langchain4j-mcp** - תלות Maven לאינטגרציה של MCP ב-LangChain4j.

**MCP** - Model Context Protocol: תקן לחיבור אפליקציות AI לכלים חיצוניים. בנה פעם, השתמש בכל מקום.

**לקוח MCP** - אפליקציה המחברת לשרתי MCP כדי לגלות ולהשתמש בכלים.

**שרת MCP** - שירות החושף כלים באמצעות MCP עם תיאורים ברורים וסכימות פרמטרים.

**McpToolProvider** - מרכיב LangChain4j העוטף כלים MCP לשימוש בשירותי AI וסוכנים.

**McpTransport** - ממשק תקשורת MCP. מימושים כוללים Stdio ו-HTTP.

**טראנספורט Stdio** - טראנספורט תהליך מקומי באמצעות stdin/stdout. שימושי לגישה למערכת הקבצים או כלים שורת פקודה.

**StdioMcpTransport** - מימוש LangChain4j שמפעיל שרת MCP כתהליך משני.

**גילוי כלים** - הלקוח שואל את השרת לגבי כלים זמינים עם תיאורים וסכימות.
