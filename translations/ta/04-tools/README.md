## நீங்கள் கற்றுக்கொள்வது என்ன

இப்போது வரை, நீங்கள் AI உடன் உரையாடல்களைச் செய்ய முடியுமென, விளக்கங்களை திறமையாக அமைக்க மற்றும் உங்கள் ஆவணங்களில் பதில்களை அடிப்படையாக்க முடியும். ஆனால் இன்னும் ஒரு அடிப்படையான வரையறை உள்ளது: மொழி மாதிரிகள் உரையை மட்டுமே உருவாக்க முடியும். அவை வானிலை அறிய முடியாது, கணக்கிட முடியாது, தரவுத்தளங்களை விசாரிக்க முடியாது அல்லது வெளிப்புற அமைப்புகளுடன் தொடர்பு கொள்ள முடியாது.

கருவிகள் இதை மாற்றுகின்றன. மாதிரிக்கு அழைக்கக்கூடிய செயல்பாடுகளை வழங்கி, அதை உரை உருவாக்குநரையிலிருந்து செயலாற்றும் முகவராக மாற்றுகிறீர்கள். மாதிரி எப்போது கருவி தேவைப்படும், எந்த கருவியை பயன்படுத்தவேண்டும் மற்றும் எந்த அளவுருக்களை வழங்க வேண்டும் என்று முடிவு செய்கிறது. உங்கள் குறியீடு செயல்பாட்டை இயக்கி முடிவை திரும்ப வழங்குகிறது. மாதிரி அந்த முடிவை தனது பதிலில் ஒருங்கிணைக்கிறது.

## முன் அறிவிப்பு

- Module 01 முடிந்திருக்க வேண்டும் (ஆஸ்யூர் OpenAI வளங்கள் நடாத்தப்பட்டவை)
- ரூட் அடைவில் `.env` கோப்பு ஆஸ்யூர் அடையாளங்களுடன் இருக்க வேண்டும் (`azd up` மூலம் Module 01 இல் உருவாக்கப்பட்டது)

> **குறிப்பு:** நீங்கள் Module 01 முடிக்கவில்லை என்றால், முதலில் அங்கு வழங்கப்பட்ட நடாத்தல் வழிகாட்டுதல்களை பின்பற்றவும்.

## கருவிகளுடன் AI முகவர்களை புரிந்து கொள்வது

> **📝 குறிப்பு:** இந்த தொகுதியில் "முகவர்கள்" என்பது கருவி அழைப்பு திறன்களுடன் மேம்படுத்தப்பட்ட AI உதவியாளர்களை குறிக்கிறது. இது [Module 05: MCP](../05-mcp/README.md) இல் காணப்படும் **Agentic AI** மாதிரிகளிலிருந்து வேறுபடுகிறது (தன்னிச்சையான முகவர்கள், திட்டமிடல், நினைவகம் மற்றும் பல-படி கருத்துக்களிப்பு).

கருவிகள் இல்லாமல், ஒரு மொழி மாதிரி தனது பயிற்சி தரவிலிருந்து மட்டும் உரையை உருவாக்க முடியும். தற்போதைய வானிலை கேட்கும்போது, அது ஊகிக்க வேண்டும். கருவிகள் வழங்கினால், அது வானிலை API-ஐ அழைக்க, கணக்கிட, அல்லது தரவுத்தளத்தை விசாரிக்க முடியும் – பின்னர் அந்த உண்மை முடிவுகளை தனது பதிலில் கலக்கிறது.

<img src="../../../translated_images/ta/what-are-tools.724e468fc4de64da.webp" alt="கருவிகள் இல்லாமல் மற்றும் கருவிகளுடன்" width="800"/>

*கருவிகள் இல்லாமல் மாதிரி ஊகிக்கிறது — கருவிகளுடன் API-ஐ அழைக்க, கணக்கிடவேண்டும் மற்றும் நேரடி தரவுகளை வழங்க முடியும்.*

கருவிகள் கொண்ட AI முகவர் **கருத்துக்களித்தல் மற்றும் செயலாற்றல் (ReAct)** மாதிரியை பின்பற்றுகிறது. மாதிரி பதிலளிப்பதல்லாமல் – அது தேவையானதை யோசித்து, கருவியை அழைத்து, முடிவை காண்கிறது, பின்னர் மீண்டும் செயல்பட வேண்டியோ அல்லது இறுதி பதிலை தர வேண்டியோ என்று முடிவு செய்கிறது:

1. **கருத்துக்களித்தல்** — முகவர் பயனர் கேள்வியை பகுப்பாய்வு செய்து தேவையான தகவலை நிர்ணயிக்கிறது
2. **செயல்** — முகவர் சரியான கருவியை தேர்வு செய்து சரியான அளவுருக்களை உருவாக்கி அதை அழைக்கிறது
3. **பார்வை** — முகவர் கருவியின் வெளியீட்டை பெற்று முடிவைக் கணிக்கிறது
4. **மீண்டும் செயல் அல்லது பதில்** — கூடுதல் தரவு தேவைப்பட்டால் சுற்றுக; இல்லையெனில் இயல்பான மொழி பதிலை உருவாக்கு

<img src="../../../translated_images/ta/react-pattern-detail.96a5efeeb6dd2f61.webp" alt="ReAct மாதிரி" width="800"/>

*ReAct சுழற்சி — முகவர் என்ன செய்ய வேண்டும் என யோசித்து, கருவி அழைக்க செயல்பட்டு, முடிவைக் கணித்து இறுதி பதிலை வழங்கும் வரை மீண்டும் செயல் படுகிறது.*

இது தானாகவே நடக்கிறது. நீங்கள் கருவிகள் மற்றும் அவற்றின் விளக்கங்களை வரையறுக்கிறீர்கள். மாதிரி அவற்றை எப்போது மற்றும் எப்படி பயன்படுத்துவது என்பதில் முடிவெடுக்கிறது.

## கருவி அழைப்பது எப்படி வேலை செய்கிறது

### கருவி வரையறைகள்

[WeatherTool.java](../../../04-tools/src/main/java/com/example/langchain4j/agents/tools/WeatherTool.java) | [TemperatureTool.java](../../../04-tools/src/main/java/com/example/langchain4j/agents/tools/TemperatureTool.java)

நீங்கள் விளக்கங்கள் மற்றும் அளவுரு விவரணங்களுடன் செயல்பாடுகளை வரையறுப்பீர்கள். மாதிரி இந்த விளக்கங்களை தனது அமைப்பு உத்தரவியில் பார்க்கிறது மற்றும் ஒவ்வொரு கருவியும் என்ன செய்கிறது என்று புரிகிறது.

```java
@Component
public class WeatherTool {
    
    @Tool("Get the current weather for a location")
    public String getCurrentWeather(@P("Location name") String location) {
        // உங்கள் வெப்ப நிலை தேடல் தர்க்கம்
        return "Weather in " + location + ": 22°C, cloudy";
    }
}

@AiService
public interface Assistant {
    String chat(@MemoryId String sessionId, @UserMessage String message);
}

// உதவியாளர் Spring Boot மூலம் தானாக இணைக்கப்படுகிறது:
// - ChatModel பீன்
// - @Component வகுப்புகளில் உள்ள அனைத்து @Tool முறைகள்
// - அமர்வுக் கட்டுப்பாட்டுக்கான ChatMemoryProvider
```

டைகரம் ஒவ்வொரு குறும்புள்ளியையும் உடைக்கிறது மற்றும் ஒவ்வொரு பகுதியும் AI-க்கு எப்போது கருவியை அழைக்க வேண்டும் மற்றும் எந்த வாதங்களை அனுப்ப வேண்டும் என்பதைக் எப்படி உதவுகிறது என்று காட்டுகிறது:

<img src="../../../translated_images/ta/tool-definitions-anatomy.f6468546037cf28b.webp" alt="கருவி வரையறைகளின் அமைப்பு" width="800"/>

*கருவி வரையறையின் அமைப்பு — @Tool AI-க்கு எப்போது பயன்படுத்துவது என்பதைக் குறிப்பது, @P ஒவ்வொரு அளவுருவையும் விவரிக்கும், மற்றும் @AiService துவக்கம் முன்னர் அனைத்தையும் இணைக்கிறது.*

> - "ஒரு நல்ல கருவி விளக்கம் AI-க்குப் பயன்படுத்த உதவும் பயனுள்ளது எப்படி?"
> - "API பிழைகள் மற்றும் விகித வரம்புகளை கருவி இடையில் எப்படி கையாளலாம்?"

### முடிவு எடுப்பு

பயனர் "சீயாட்டிலில் வானிலை என்ன?" என்று கேட்டால், மாதிரி சீரிய தவறுதலாக கருவியை தேர்வு செய்யாது. அது பயனர் நோக்கத்தை உள்ள இடைமுக விளக்கங்கள் ஒவ்வொன்றுடனும் ஒப்பிட்டு, தொடர்பு வளர்ச்சியை மதிப்பிட்டு, சிறந்த பொருத்தமானதையே தேர்ந்தெடுக்கிறது. அதன் பிறகு சரியான அளவுருக்களுடன் கட்டுமான செய்யப்பட்ட செயல்பாடு அழைப்பை உருவாக்குகிறது – இந்தச் சூழலில் `location`-ஐ `"Seattle"` ஆக அமைக்கிறது.

பயனர் கோரிக்கைக்கு எதுவும் பொருந்தவில்லை என்றால், மாதிரி அதற்கு பதிலாக தனது அறிவுத்தொகுப்பிலிருந்து பதிலளிக்கும். பல கருவிகள் பொருந்தினால், மிக குறிப்பிட்ட ஒன்றை தேர்ந்தெடுக்கிறது.

<img src="../../../translated_images/ta/decision-making.409cd562e5cecc49.webp" alt="AI எது கருவி பயன்படுத்துவது என்பதை எப்படி முடிவு செய்கிறது" width="800"/>

*மாதிரி ஒவ்வொரு கருவியையும் பயனர் நோக்கத்துடன் மதிப்பிடுகிறது மேலும் சிறந்த பொருத்தத்தைத் தேர்வு செய்கிறது — இத்துக்காக தெளிவான, குறிப்பிட்ட கருவி விளக்கங்களை எழுதுவது முக்கியம்.*

### பதில் உருவாக்கல்

மாதிரி வானிலை தரவை பெற்று, பயனருக்கு இயல்பான மொழி பதிலாக வடிவமைக்கிறது.

### கட்டமைப்பு: ஸ்பிரிங் பூட் ஆட்டோ-வையரிங்

இந்த தொகுதி LangChain4j இன் ஸ்பிரிங் பூட் ஒருங்கிணைப்பை `@AiService` இடைமுகங்களுடன் பயன்படுத்துகிறது. துவக்கம் நேரத்தில் ஸ்பிரிங் பூட் `@Tool` முறைமைகள் உள்ள அனைத்து `@Component` களை, உங்கள் `ChatModel` பீனை மற்றும் `ChatMemoryProvider` ஐ கண்டறிந்து, அவற்றை எல்லாம் ஒரு அலகான `Assistant` இடைமுகத்தில் இணைக்கிறது, எதுவும் கையொப்பம் இல்லாமல்.

<img src="../../../translated_images/ta/spring-boot-wiring.151321795988b04e.webp" alt="ஸ்பிரிங் பூட் ஆட்டோ-வையரிங் கட்டமைப்பு" width="800"/>

*@AiService இடைமுகம் ChatModel, கருவி கூறுகள் மற்றும் நினைவக வழங்குநரை ஒன்றாக இணைக்கிறது — ஸ்பிரிங் பூட் அனைத்தையும் தானாக கையாள்கிறது.*

இந்த முறையின் முக்கிய நன்மைகள்:

- **ஸ்பிரிங் பூட் ஆட்டோ-வையரிங்** — ChatModel மற்றும் கருவிகள் தானாக இணைக்கப்படுகின்றன
- **@MemoryId மாதிரி** — தானாக அமர்வு அடிப்படையிலான நினைவக மேலாண்மை
- **ஒரே எடுத்துக்காட்டு** — Assistant ஒருமுறை உருவாக்கப்பட்டு மீண்டும் பயன்படுத்தப்படுகிறது அதிக செயல்திறனுக்காக
- **வகை-பாதுகாப்பான நிறைவேற்றல்** — Java முறைகள் நேரடியாக வகை மாற்றத்துடன் அழைக்கப்படுகின்றன
- **பல-மடங்கு ஒருங்கிணைப்பு** — கருவி சங்கிலி தானாக கையாளப்படுகின்றது
- **பூலோவெர் இல்லாதது** — கைமுறை `AiServices.builder()` அழைப்புகள் அல்லது நினைவக ஹாஷ்மேப் இல்லாது

மாற்று முறைகள் (கைமுறை `AiServices.builder()`) அதிக குறியீடு தேவைப்படுவதாகவும், ஸ்பிரிங் பூட் ஒருங்கிணைப்பு நன்மைகளை இழந்து விடும்.

## கருவி சங்கிலி

**கருவி சங்கிலி** — கருவி அடிப்படையிலான முகவர்களின் உண்மையான சக்தி, ஒரு கேள்விக்கு பல கருவிகள் தேவைப்படும் போது வெளிப்படுகிறது. "சீயாட்டிலின் வானிலை ஃபாரன்ஹீட்டில் என்ன?" என்று கேள், முகவர் தானாக இரண்டு கருவி அழைப்புகளை சங்கிலி செய்கிறது: முதலில் `getCurrentWeather` ஐ அழைத்து செல்சியஸ் வெப்பநிலையை பெறுகிறது, பின்னர் அந்த மதிப்பை `celsiusToFahrenheit` க்கு மாற்ற அனுப்புகிறது – இவை எல்லாம் ஒரு உரையாடல் சுற்றில்.

<img src="../../../translated_images/ta/tool-chaining-example.538203e73d09dd82.webp" alt="கருவி சங்கிலி எடுத்துக்காடு" width="800"/>

*கருவி சங்கிலி செயலில் — முகவர் முதலில் getCurrentWeather ஐ அழைக்கிறான், பின்னர் செல்சியஸ் முடிவை celsiusToFahrenheit இல் அனுப்பி மொத்த பதிலை வழங்குகிறான்.*

இது இயங்கி வரும் செயலியில் எப்படி இருக்கும் என்றோ இதோ — முகவர் ஒரே உரையாடல் சுற்றில் இரண்டு கருவி அழைப்புகளை சங்கிலி செய்கிறது:

<a href="images/tool-chaining.png"><img src="../../../translated_images/ta/tool-chaining.3b25af01967d6f7b.webp" alt="கருவி சங்கிலி" width="800" style="border: 1px solid #ddd; box-shadow: 0 2px 8px rgba(0,0,0,0.1);"/></a>

*வாசுவல அளிப்பு வெளியீடு — முகவர் தானாக getCurrentWeather → celsiusToFahrenheit ஐ ஒரே சுற்றில் சங்கிலி செய்கிறது.*

**மென்மையான தோல்விகள்** — மோக_DATA-வில் இல்லாத நகரத்தில் வானிலை கேளுங்கள். கருவி பிழை செய்தியுடன் திரும்புகிறது, மற்றும் AI உதவ முடியாது என்று விளக்குகிறது இது இடையூறு ஏற்படுத்தாமல். கருவிகள் பாதுகாப்பாக தோல்வியடைகின்றன.

<img src="../../../translated_images/ta/error-handling-flow.9a330ffc8ee0475c.webp" alt="பிழை கையாளும் ஓட்டம்" width="800"/>

*ஒரு கருவி தோல்வியடைந்தால், முகவர் பிழையைப் பிடித்து இடையூறு ஏற்படுத்தாமல் உதவிக்குரிய விளக்கத்துடன் பதிலளிக்கிறது.*

இது ஒரே உரையாடல் சுற்றில் நடக்கிறது. முகவர் பல கருவி அழைப்புகளை தன்னிச்சையாக ஒருங்கிணைக்கிறான்.

## ஆப்ளிகேஷனை இயக்குக

**நடாத்தல் உறுதிப்படுத்துக:**

ஆஸ்யூர் அடையாளங்களுடன் ரூட் கோப்புறையில் `.env` கோப்பு உள்ளது என்பதை உறுதிசெய்க (Module 01ல் உருவாக்கப்பட்டது):
```bash
cat ../.env  # AZURE_OPENAI_ENDPOINT, API_KEY, DEPLOYMENT ஐ காட்ட வேண்டும்
```

**ஆப்ளிகேஷனை துவங்குக:**

> **குறிப்பு:** நீங்கள் Module 01 இலிருந்து `./start-all.sh` கொண்டு அனைத்து ஆப்ளிகேஷன்களையும் ஏற்கனவே துவக்கியிருந்தால், இந்த தொகுதி 8084 போர்ட்டில் இயங்கிக் கொண்டிருக்கிறது. கீழே உள்ள துவக்கும் கட்டளைகளை தவிர்த்து நேரடியாக http://localhost:8084 சென்று பயன்படுத்தலாம்.

**விருப்பம் 1: ஸ்பிரிங் பூட் டாஷ்போர்டு மூலம் (VS Code பயனர்களுக்கு பரிந்துரை)**

டெவ் கொண்டெய்னர் ஸ்பிரிங் பூட் டாஷ்போர்டு நீட்டிப்பை கொண்டுள்ளது, இது அனைத்து ஸ்பிரிங் பூட் ஆப்ளிகேஷன்களையும் வேகமாக நிர்வகிக்கும் ஒரு கண்ணோட்ட இடைமுகத்தை வழங்கும். அதை VS Code இடது பக்க Activity Bar இல் (ஸ்பிரிங் பூட் ஐகான் பார்க்கவும்) காணலாம்.

ஸ்பிரிங் பூட் டாஷ்போர்டிலிருந்து நீங்கள்:
- பணிமுழுவதும் கிடைக்கும் ஸ்பிரிங் பூட் ஆப்ளிகேஷன்களை காணலாம்
- ஒரு கிளிக் மூலம் ஆப்ளிகேஷன்களை துவக்க / நிறுத்தலாம்
- நிகழ்கால செயல்பாட்டு பதிவுகளை பார்க்கலாம்
- ஆப்ளிகேஷன் நிலையை கண்காணிக்கலாம்

"tools" அதிரடிப்படியில் துவக்க புள்ளிக்கு பக்கத்தில் உள்ள பிளே பொத்தானை அழுத்தவும் அல்லது அனைத்து தொகுதிகளையும் ஒரே நேரத்தில் துவங்கவும்.

**விருப்பம் 2: ஷெல் ஸ்கிரிப்ட் பயன்பாடு**

**PowerShell:**

> **PowerShell:**
> ```powershell
> cd ..  # Go to root directory

உங்கள் உலாவியில் http://localhost:8084 ஐ திறக்கவும்.

**PowerShell:**

*AI முகவர் கருவிகள் இடைமுகம் - கருவிகளுடன் தொடர்பு கொள்ள விரைவான உதாரணங்கள் மற்றும் உரையாடல் இடைமுகம்*

### எளிய கருவி பயன்பாட்டை முயற்சி செய்
ஒரு நேரடியாகக் கோரிக்கையுடன் துவங்குங்கள்: "100 டிகிரி பார்ன்ஹீட் ஐ செல்சியஸ் ஆக மாற்றவும்". முகவர் அது வெப்பநிலை மாற்றும் கருவியை தேவைப்படுகிறதா என அறிந்து, சரியான அளவுருக்களுடன் அதை அழைக்கிறது மற்றும் முடிவை வழங்குகிறது. இது எவ்வாறு இயற்கையாகத் தோன்றுகிறது என்பதை கவனியுங்கள் - நீங்கள் எந்த கருவியை பயன்படுத்துவது அல்லது அதை எப்படி அழைப்பது என்பதை குறிப்பிடவில்லை.

### கருவி சங்கிலிப்படுத்தலைத் சோதிக்கவும்

இப்போது கொஞ்சம் சிக்கலானதையா முயற்சிக்கவும்: "சியாட்டிலில் நிலவுள்ள வானிலை என்ன மற்றும் அதை பார்ன்ஹீட்டுக்கு மாற்றவும்?" முகவர் இதனை படிகள் மூலம் செய்கிறது. முதலில் அது வானிலை பெறுகிறது (இது செல்சியஸ் ஆக வருகிறது), பிறகு பார்ன்ஹீட்டுக்கு மாற்ற வேண்டிய தேவையை உணர்கிறது, மாற்றும் கருவியை அழைக்கிறது, மற்றும் இரு முடிவுகளையும் ஒரே பதிலில் சேர்க்கிறது.

### உரையாடல் ஓட்டத்தை காண்க

அரட்டை இடைமுகம் உரையாடல் வரலாறை பேணுகிறது, இது உங்களுக்கு பல-முறை பரிமாற்றங்களை ஏற்படுத்த அனுமதிக்கிறது. நீங்கள் அனைத்து முன் கேள்விகளையும் மற்றும் பதில்களையும் பார்க்கலாம், இது உரையாடலை பின்தொடர்ந்து முகவர் பல முறை பரிமாற்றங்களில் உள்ள சூழலை எவ்வாறு உருவாக்குகிறான் என்பதை புரிந்து கொள்ள எளிதாகிறது.

<a href="images/tools-conversation-demo.png"><img src="../../../translated_images/ta/tools-conversation-demo.89f2ce9676080f59.webp" alt="பல கருவி அழைப்புகளுடன் உரையாடல்" width="800" style="border: 1px solid #ddd; box-shadow: 0 2px 8px rgba(0,0,0,0.1);"/></a>

*எளிய மாற்றங்கள், வானிலை தேடல்கள் மற்றும் கருவி சங்கிலிப்படுத்தலை பின்பற்றும் பல முறை உரையாடல்*

### வெவ்வேறு கோரிக்கைகள் உடன் முயற்சி செய்யவும்

பல இணைப்புகளை முயற்சிக்கவும்:
- வானிலை தேடல்கள்: "டோக்கியோவில் வானிலை என்ன?"
- வெப்பநிலை மாற்றங்கள்: "25°C என்பது கெல்வினில் என்ன?"
- இணைந்த கேள்விகள்: "பாரிஸில் வானிலை பார்க்கவும், அது 20°C க்கும் மேல் உள்ளதா என்று சொல்லவும்"

முகவர் இயற்கை மொழியை எவ்வாறு புரிந்து கொண்டு அதை சரியான கருவி அழைப்புகளுடன் இணைக்கிறதா என்பதை கவனியுங்கள்.

## முக்கிய கருத்துக்கள்

### ReAct மாதிரி (கருத்தாய்வு மற்றும் செயல்பாடு)

முகவர் கருத்தாய்வு (எதைச் செய்ய வேண்டும் என முடிவெற்பது) மற்றும் செயல்பாடு (கருவிகளை பயன்படுத்துவது) என்பவற்றை மாறி செயல்படுகிறான். இந்த மாதிரி கட்டளைகளுக்கு பதிலளிப்பதற்குப் பதிலாக சுயாதீனமான பிரச்சனை தீர்க்க உதவுகிறது.

### கருவி விவரணங்கள் முக்கியம்

உங்கள் கருவி விவரணங்களின் தரம் முகவர் அவற்றை எவ்வாறு பயன்படுத்துகிறான் என்பது மீது நேரடியான பாதிப்பை ஏற்படுத்துகிறது. தெளிவான, குறிப்பிட்ட விவர அணுகுமுறை மாடலை எப்போது மற்றும் எப்படி ஒவ்வொரு கருவியையும் அழைக்க வேண்டும் என்று புரிந்துகொள்ள உதவுகிறது.

### அமர்வு மேலாண்மை

`@MemoryId` குறியீடு தானாக அமர்வு அடிப்படையிலான நினைவூட்டலை நிர்வகிக்க உதவுகிறது. ஒவ்வொரு அமர்வு ஐடியும் தனித்த `ChatMemory` அமைப்பை பெறுகிறது, இது `ChatMemoryProvider` பீனால் நிர்வகிக்கப்படுகிறது, ஆகவே பல பயனர்களும் ஒரே நேரத்தில் முகவருடன் உரையாடலாம் ஆனால் அவர்களது உரையாடல்கள் கலந்துபோகாது.

<img src="../../../translated_images/ta/session-management.91ad819c6c89c400.webp" alt="@MemoryId உடன் அமர்வு மேலாண்மை" width="800"/>

*ஒவ்வொரு அமர்வு ஐடியும் தனி உரையாடல் வரலாறை அமைக்கிறது — பயனர்கள் ஒருவருக்கொருவர் செய்திகளை காண மாட்டார்கள்.*

### பிழை கையாளுதல்

கருவிகள் தப்பிக்கலாம் — API கள் நேரம் முடிவடையலாம், அளவுருக்கள் தவறாக இருக்கலாம், வெளிப்புற சேவைகள் செயலிழக்கலாம். உற்பத்தி முகவர்கள் பிழை கையாளல்களை தேவைப்படுத்துகின்றனர், இதனால் மாடல் பிரச்சனைகளை விளக்க அல்லது மாற்று முயற்சிகளை செய்ய முடியும். ஒரு கருவி தவறை ஏற்படுத்தும்போது, LangChain4j அதனை பிடித்து பிழை செய்தியை மாடலுக்கு அனுப்புகிறது, பின்னர் அது இயற்கை மொழியில் பிரச்சனையை விளக்க முடியும்.

## கிடைக்கும் கருவிகள்

கீழே உள்ள வரைபடம் நீங்கள் உருவாக்கக்கூடிய கருவிகளின் பரந்து விரிந்த சுற்றுச்சூழலைக் காட்டுகிறது. இந்த பகுதி வானிலை மற்றும் வெப்பநிலை கருவிகளைப் பயன்படுத்தும் உதாரணமாக காட்டுகிறது, ஆனால் அதே `@Tool` மாதிரி எந்த Java முறைக்கும் வேலை செய்யும் — தரவுத்தள கேள்விகளிலிருந்து கட்டணம் செயலாக்கம் வரை.

<img src="../../../translated_images/ta/tool-ecosystem.aad3d74eaa14a44f.webp" alt="கருவி சுற்றுச்சூழல்" width="800"/>

*@Tool குறிச்சொற்றுடன் குறிக்கப்பட்ட எந்த Java முறைவும் AI க்கு கிடைக்கும் — இந்த மாதிரி தரவுத்தளங்கள், API கள், மின்னஞ்சல், கோப்பு செயல்பாடுகள் மற்றும் மேலும் பலவற்றிற்கு நீட்டிக்கிறது.*

## கருவி அடிப்படையிலான முகவர்களை எப்போது பயன்படுத்த வேண்டும்

<img src="../../../translated_images/ta/when-to-use-tools.51d1592d9cbdae9c.webp" alt="கருவிகளை எப்போது பயன்படுத்துவது" width="800"/>

*ஒரு விரைவு முடிவு கையேடு — கருவிகள் நேரடி தரவு, கணக்கீடுகள், மற்றும் செயல்களுக்கு; பொதுவான அறிவும் படைப்பாற்றல் பணிகளுக்கு அவை தேவையில்லை.*

**கருவிகள் பின்வருமான போது பயன்படுத்தவும்:**
- பதிலளிக்க நேரடி தரவு தேவைப்படும் போது (வானிலை, பங்கு விலை, சரக்கு)
- எளிய கணக்குகள் விட்டு மேலாக கணக்கீடுகள் செய்ய வேண்டிய போது
- தரவுத்தளங்கள் அல்லது API களை அணுகும் போது
- செயல்களை மேற்கொள்ளும் போது (மின்னஞ்சல் அனுப்பு, டிக்கெட் உருவாக்கு, பதிவுகளை புதுப்பி)
- பல தரவு மூலங்களை ஒன்றிணைக்கும் போது

**கருவிகளை பயன்படுத்த வேண்டாம்:**
- கேள்விகள் பொதுவான அறிவிலிருந்து பதிலளிக்கக்கூடியவை
- பதில் வெறும் உரையாடல் சார்ந்தது
- கருவி தாமதம் அனுபவத்தை மோசமாக்கும் போது

## கருவிகள் vs RAG

ஜூலை 03 மற்றும் 04 இடையே AI இன் திறனை விரிவாக்குகிறன, ஆனால் அடிப்படையாக வெவ்வேறு முறைகளில். RAG மாறிகள் மூலம் **அறிவைப்** பெறுகிறது; கருவிகள் செயல்பாடுகளை அழைத்து **செயல்களை** எடுக்க உதவுகிறது.

<img src="../../../translated_images/ta/tools-vs-rag.ad55ce10d7e4da87.webp" alt="கருவிகள் மற்றும் RAG ஒப்பீடு" width="800"/>

*RAG நிலையான ஆவணங்களில் இருந்து தகவலை மீட்டெடுக்கிறது — கருவிகள் செயல்களை இயக்கி, உயிர்வாழும் நேரடி தரவை பெறுகின்றன. பல உற்பத்தி அமைப்புகள் இரண்டையும் இணைத்து பயன்படுத்துகின்றன.*

நடைமுறையில், பல உற்பத்தி அமைப்புகள் இரண்டு அணுகுமுறைகளையும் சேர்த்து பயன்படுத்துகின்றன: உங்கள் ஆவணங்களில் பதில்களை அடிப்படையாக்க RAG மற்றும் உயிர்வாழும் தரவோ அல்லது நடவடிக்கைகளுக்காக கருவிகள்.
