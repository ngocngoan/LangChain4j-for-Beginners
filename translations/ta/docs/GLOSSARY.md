## முக்கியக் கான்செப்ட்கள்

**AI ஓஜனையாளர்** - தானாக சிந்தித்து செயல்படும் AI ஐ பயன்படுத்தும் அமைப்பு. [தொகுதி 04](../04-tools/README.md)

**சிங்காரம்** - வெளியீடு அடுத்து வரும் படிக்கு ஊட்டமாக வழங்கப்படும் செயல்பாடுகளின் தொடர்ச்சி.

**வெட்டு செய்தல்** - ஆவணங்களை சிறிய துண்டுகளாக பிரித்தல். பொதுவானது: 300-500 குறியிடிகளுடன் 겹ிமையுடன். [தொகுதி 03](../03-rag/README.md)

**சூழல் சாளரம்** - ஒரு மாதிரிக்கு maximum tokens ப்ராசஸ் செய்யலாம். GPT-5.2: 400K tokens (272K உள்ளீடு வரை, 128K வெளியீடு வரை).

**எம்பெட்டிங்** - உரையின் பொருளை ஒப்புள்ளி எண் வடிவத்தில் காட்டுதல். [தொகுதி 03](../03-rag/README.md)

**சேவை அழைப்பு** - மாதிரி வெளிப்புற செயல்பாடுகளை அழைக்க வடிவமைக்கப்பட்ட கோரிக்கைகளை உருவாக்குதல். [தொகுதி 04](../04-tools/README.md)

**தூண்டுதல்** - மாதிரிகள் தவறான ஆனால் நம்பத்தக்க தகவலை உருவாக்கும் போது.

**ப்ராம்ட்** - மொழி மாதிரிக்கு உள்ளீட்டு உரை. [தொகுதி 02](../02-prompt-engineering/README.md)

**அர்த்தவாய்ந்த தேடல்** - முக்கிய வார்த்தைகள் இல்லை; பொருள் மற்றும் எம்பெட்டிங்குகளை பயன்படுத்தி தேடல். [தொகுதி 03](../03-rag/README.md)

**நிலைத்தன்மை மற்றும் நிலைத்தன்மையற்றவை** - நிலைத்தன்மையற்றவை: நினைவில்லாமல் செயல்படுதல். நிலைத்தன்மை: உரையாடல் வரலாறை பாதுகாத்தல். [தொகுதி 01](../01-introduction/README.md)

**குறியிடுகள்** - மாதிரிகள் செயல்படுத்தும் அடிப்படை உரை அலகுகள். செலவு மற்றும் வரம்புகளை பாதிக்கும். [தொகுதி 01](../01-introduction/README.md)

**கருவி சங்கிலி** - ஒரு கருவியின் வெளியீடு அடுத்த கருவி அழைப்பிற்கு தகவலளிக்கும் தொடர் செயல்பாடு. [தொகுதி 04](../04-tools/README.md)

## LangChain4j கூறுகள்

**AiServices** - வகை பாதுகாப்பான AI சேவை இடைமுகங்களை உருவாக்குகிறது.

**OpenAiOfficialChatModel** - OpenAI மற்றும் Azure OpenAI மாதிரிகளுக்கான ஒருமித்த கிளையண்ட்.

**OpenAiOfficialEmbeddingModel** - OpenAI அதிகாரப்பூர்வ கிளையண்ட் பயன்படுத்தி எம்பெட்டிங்குகளை உருவாக்குகிறது (OpenAI மற்றும் Azure OpenAI இரண்டும் ஆதரிக்கின்றன).

**ChatModel** - மொழி மாதிரிகளுக்கான முக்கிய இடைமுகம்.

**ChatMemory** - உரையாடல் வரலாறை நிர்வகிக்கும்.

**ContentRetriever** - RAG க்காக பொருத்தமான ஆவண துண்டுகளை கண்டுபிடிக்கும்.

**DocumentSplitter** - ஆவணங்களை துண்டுகளாக பிரிக்கும்.

**EmbeddingModel** - உரையை எண் வடிவ எம்பெட்டிங்குகளாக மாற்றும்.

**EmbeddingStore** - எம்பெட்டிங்குகளை சேமித்து மீட்டெடுக்கும்.

**MessageWindowChatMemory** - சமீபத்திய செய்திகள் கொண்ட ஸ்லைடிங் சாளரத்தை பராமரிக்கும்.

**PromptTemplate** - `{{variable}}` இடம் பிடிப்பாளர்களுடன் மறுபயன்பாடான ப்ராம்ட்களை உருவாக்குகிறது.

**TextSegment** - RAG இல் பயன்படுத்தப்படும் மெட்டாடேட்டாவுடன் உரை துண்டு.

**ToolExecutionRequest** - கருவி செயல்பாடு கோரிக்கை பிரதிநிதித்துவம்.

**UserMessage / AiMessage / SystemMessage** - உரையாடல் செய்தி வகைகள்.

## AI/ML கான்செப்ட்கள்

**சிறிய எண் எடுத்துக்காட்டுகளுடன் கற்றல்** - ப்ராம்ட்களில் எடுத்துக்காட்டுகளை வழங்குதல். [தொகுதி 02](../02-prompt-engineering/README.md)

**பெரிய மொழி மாதிரி (LLM)** - பரந்த உரை தரவிலான AI மாதிரிகள்.

**கருத்து முயற்சி** - GPT-5.2 இல் சிந்தனைக் கோட்பாட்டை கட்டுப்படுத்தும் அளவுகோல். [தொகுதி 02](../02-prompt-engineering/README.md)

**வெப்பநிலை** - வெளியீடு சிதறலை கட்டுப்படுத்து. குறைந்தது=முடிவானது, அதிகம்=புதுமையானது.

**வெக்டர் தரவுத்தளம்** - எம்பெட்டிங்குகளுக்கான சிறப்பு தரவுத்தளம். [தொகுதி 03](../03-rag/README.md)

**பூஜ்ய-எடுத்துக்காட்டில்லா கற்றல்** - எடுத்துக்காட்டுகள் இல்லாமல் பணிகளை செய்யல். [தொகுதி 02](../02-prompt-engineering/README.md)

## காப்புரைகள் - [தொகுதி 00](../00-quick-start/README.md)

**ஆழமான பாதுகாப்பு** - பயன்பாட்டு நிலை காப்புறைகள் மற்றும் வழங்குனர் பாதுகாப்பு கட்டுப்பாடுகளை இணைக்கும் பல அடுக்கு பாதுகாப்பு அணுகுமுறை.

**முக்கிய தடை** - கடுமையான உள்ளடக்க மீறல்களுக்கு வழங்குனர் HTTP 400 தவறு காட்டுகிறான்.

**InputGuardrail** - LLM அடைவதற்கு முன்னர் பயனர் உள்ளீட்டை சரிபார்க்க LangChain4j இடைமுகம். தீங்கான ப்ராம்ட்களை ஆரம்பத்தில் தடுக்கும் மூலம் செலவு மற்றும் தாமதத்தை காப்பாற்றும்.

**InputGuardrailResult** - காப்புரை சரிபார்ப்புக்கான திரும்பும் வகை: `success()` அல்லது `fatal("காரணம்")`.

**OutputGuardrail** - பயனர்களுக்கு திருப்புவதற்கு முன்னர் AI பதில்களை சரிபார்க்க இடைமுகம்.

**வழங்குநர் பாதுகாப்பு வடிகட்டிகள்** - AI வழங்குநர்களிடமிருந்து உள்ளடக்கத்தை API நிலை에서 பறிப்பது போன்ற கட்டுப்பாடுகளை கொண்டுள்ளு வடிகட்டிகள் (எ.கா., GitHub மாதிரிகள்).

**மென்மையான மறுப்பு** - மாதிரி தவறு காட்டாமல் மரியாதையாக பதில் அளிக்க மறுப்பது.

## ப்ராம்ட் பொறியியல் - [தொகுதி 02](../02-prompt-engineering/README.md)

**சிந்தனை சங்கிலி** - சிறந்த துல்லியத்துக்கு படி படியாக நியாயம் செய்யல்.

**மூடிய வெளியீடு** - குறிப்பிட்ட வடிவம் அல்லது அமைப்பை கட்டாயப்படுத்தல்.

**மிகவும் ஆர்வம்** - GPT-5.2 இல் விரிவான சிந்தனைக்கான முறை.

**குறைந்த ஆர்வம்** - GPT-5.2 இல் விரைவான பதில்களுக்கான முறை.

**பலபடியான உரையாடல்** - மாற்றங்களுக்கிடையில் சூழலை பராமரித்தல்.

**பாத்திர அடிப்படையிலான ப்ராம்டிங்** - கணினி தன்மையை system செய்திகளின் மூலம் அமைத்தல்.

**தனிமனிதம்** - மாதிரி தனது வெளியீட்டை மதிப்பீடு செய்து மேம்படுத்துதல்.

**ஒழுங்கமைக்கப்பட்ட பகுப்பாய்வு** - நிர்ணயிக்கப்பட்ட மதிப்பீட்டுக் கட்டமைப்பு.

**பணி செயலாக்க முறை** - திட்டமிடு → செயல்படுத்து → சுருக்கம்.

## RAG (மீட்டெடுப்பு-சேர்க்கப்பட்ட உருவாக்கம்) - [தொகுதி 03](../03-rag/README.md)

**ஆவண செயலாக்க குழாய்** - ஏற்று → துண்டாக்கு → எம்பெடு → சேமி.

**உள்ளமைவாக இருக்கும் எம்பெட்டிங் ஸ்டோர்** - சோதனைகளுக்கான நிரந்தரமற்ற சேமிப்பகம்.

**RAG** - பதில்களை நிலைத்துள்ள தகவலுடன் இணைக்கும் மீட்டெடுப்பு மற்றும் உருவாக்கக் கலவையுடன்.

**சமத்துவ மதிப்பெண்** - அர்த்தவாய்ந்த ஒத்திசைவு அளவு (0-1).

**மூல மேற்கோள்** - மீட்டெடுக்கப்பட்ட உள்ளடக்கத்தைச் பற்றிய மெட்டாடேட்டா.

## ஓஜனையாளர்கள் மற்றும் கருவிகள் - [தொகுதி 04](../04-tools/README.md)

**@Tool குறிச்சொல்** - ஜாவா முறைகளுக்கு AI அழைக்கக்கூடிய கருவிகள் என்று குறிகள்.

**ReAct மாதிரி** - நியாயம் → செயல் → கவனிப்பு → மீண்டும் செயல்.

**அமர்வு மேலாண்மை** - வேறுபட்ட பயனர்களுக்கான தனி சூழல்கள்.

**கருவி** - AI ஓஜனையாளர் அழைக்கக்கூடிய செயல்பாடு.

**கருவி விளக்கம்** - கருவியின் நோக்கம் மற்றும் அளவுருக்களின் ஆவணம்.

## ஓஜனையியல் தொகுதி - [தொகுதி 05](../05-mcp/README.md)

**@Agent குறிச்சொல்** - இடைமுகங்களை பேசும் குணாதிசயங்களுடன் AI ஓஜனையாளர்களாக குறிக்கிறது.

**ஓஜனையாளர் விசைப்பெயர்** - `beforeAgentInvocation()` மற்றும் `afterAgentInvocation()` மூலம் ஓஜனையாளர் செயல்பாட்டைக் கண்காணிக்க ஹுக்.

**ஓஜனையியல் பரியலம்** - ஓஜனையாளர்கள் வெளியீடுகளை `outputKey` பயன்படுத்தி பகிரும் பகுதி, பின்னர் பிற ஓஜனையாளர்கள் 이를 பயன்படுத்துவார்கள்.

**AgenticServices** - `agentBuilder()` மற்றும் `supervisorBuilder()` மூலம் ஓஜனையாளர்களை உருவாக்கும் தொழிற்சாலை.

**நிபந்தனை அடிப்படையிலான பணியாளர் வழி** - விதிகளின் அடிப்படையில் வேறு நிபுண ஓஜனையாளர்களிடம் வழிமாற்றல்.

**மனிதன்-நுழைந்த செயல்முறை** - ஒப்புதல் அல்லது உள்ளடக்க பரிசீலனைக்கு மனிதரைச் சேர்க்கும் பணிச்செழு.

**langchain4j-agentic** - உரையாடல் ஓஜனையாளர் கட்டுமானத்திற்கான மெய்வரி நிச்சயமற்ற தொகுப்பு (அனுபவ்ச்சி).

**மடக்கு பணிச்செழு** - ஒரு நிபந்தனையை பூர்த்தி செய்யும் வரை ஓஜனையாளர் செயல்பாட்டினை மீண்டும் மீண்டும் செய்யும் வகை (எ.கா., தர மதிப்பெண் ≥ 0.8).

**outputKey** - ஓஜனையாளர் குறிச்சொல் அளவுரு, எங்கே முடிவுகள் ஓஜனையியல் பரியலத்தில் சேமிக்கப்படுகின்றன என்பதைக் குறிக்கிறது.

**இணை செயல் பணிச்செழு** - பல ஓஜனையாளர்களை சுதந்திர பணிகளுக்காக ஒரே நேரத்தில் இயக்குதல்.

**பதில் நெறிமுறை** - மேற்பார்வையாளர் கடைசி பதில் உருவாக்கும் முறை: கடைசி, சுருக்கம், அல்லது மதிப்பிடப்பட்டது.

**எதிரொலி பணிச்செழு** - ஒன்றுக்கு பிறகு ஓஜனையாளர்களை இயக்கி, வெளியீடு அடுத்த படிக்கு செல்லும்.

**மேற்பார்வையாளர் ஓஜனையாளர் மாதிரி** - மேற்பார்வை LLM இயக்கத்தான் எந்த துணை ஓஜனையாளர்களை அழைக்க வேண்டும் என்பதை உள்ளடக்கிய மேம்பட்ட ஓஜனையியல் மாதிரி.

## மாதிரி சூழல் நெறிமுறை (MCP) - [தொகுதி 05](../05-mcp/README.md)

**langchain4j-mcp** - LangChain4j இல் MCP ஒருங்கிணைப்பிற்கு மெய்வரி.

**MCP** - மாதிரி சூழல் நெறிமுறை: வெளிப்புற கருவிகளுடன் AI பயன்பாடுகளை இணைக்க ஒரு தரநிலை. ஒருமுறை உருவாக்கி எங்கும் பயன்படுத்து.

**MCP கிளையண்ட்** - MCP சேவைகளுடன் இணைத்து கருவிகளை கண்டறிந்து பயன்படுத்தும் பயன்பாடு.

**MCP சேவையகம்** - MCP வழியாக கருவிகளை வெளிப்படுத்தும் சேவை, தெளிவான விளக்கங்கள் மற்றும் அளவுரு திட்டங்கள் உடன்.

**McpToolProvider** - LangChain4j கூறு, MCP கருவிகளை AI சேவைகள் மற்றும் ஓஜனையாளர்களுக்காக பயன்படுத்துவதற்கான சுருள்.

**McpTransport** - MCP தொடர்புக்கும் இடைமுகம். செயல்முறைகள் Stdio மற்றும் HTTP உட்பட.

**Stdio தொடர்பு** - உள்ளூர் செயல்முறை தொடர்பு stdin/stdout மூலம். கோப்பு அணுகல் அல்லது கட்டளை வரிசை கருவிகளுக்கு பயன்படும்.

**StdioMcpTransport** - LangChain4j செயலாக்கம் MCP சேவையகத்தை துணைசெயல்பாடுகளாக உருவாக்கும்.

**கருவி கண்டறிதல்** - கிளையண்ட் விளக்கங்கள் மற்றும் திட்டங்களுடன் கிடைக்கும் கருவிகளை சேவையகத்திலிருந்து கேட்கும்.
