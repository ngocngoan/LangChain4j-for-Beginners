## வீடியோ நடைமேட்பு

இந்த மொட்யூலைத் துவங்குவது எப்படி என்பதை விளக்கும் நேரடி நிகழ்ச்சியை காணவும்: [LangChain4j உடன் துவங்குதல் - நேரடி நிகழ்ச்சி](https://www.youtube.com/live/nl_troDm8rQ?si=6b85S8xGjWnT2fX9)

## நீங்கள் கற்றுக்கொள்ளவிருப்பது

நீங்கள் வேக ஆரம்பத்தைக் முடித்திருந்தால், எப்படி முன்மொழிவுகளை அனுப்பி பதில்களைப் பெறுவது என்பதை பார்ந்தீர்கள். இது அடித்தளம், ஆனால் உண்மையான பயன்பாடுகள் இன்னும் அதிகத்தை தேவையாக்குகின்றன. இந்த மொட்யூல், உரையாடலின் சாராம்சத்தை நினைவில் வைத்து நிலையை பராமரிக்கும் உரையாடலாக்க செயற்கை நுண்ணறிவை உருவாக்குவது எப்படி என்பதைக் கற்பிக்கும் — ஒரு ஒரே முறை டெமோவும் தயாரிப்பு பயன்பாட்டுக்கும் இடையேயான வித்தியாசம்.

இந்த கையேட்டின் முழுவதும் Azure OpenAI-வின் GPT-5.2 ஐப் பயன்படுத்துவோம், ஏனெனில் அதனுடைய மேம்பட்ட கருதிச் செயல்திறன்கள் வெவ்வேறு வடிவமைப்புகளின் செயற்பாட்டைத் தெளிவாக காட்டுகின்றன. நீங்கள் நினைவுவழக்கத்தைச் சேர்க்கையில், அந்த வித்தியாசத்தை தெளிவாகப் பார்ப்பீர்கள். இதனால் உங்கள் பயன்பாட்டுக்கு ஒவ்வொரு கூறும் என்ன கொடுப்பதைப் புரிந்து கொள்வது எளிதாகிறது.

நீங்கள் இரண்டு வடிவமைப்புகளையும் விளக்குவதற்கான ஒரு பயன்பாட்டை உருவாக்குவீர்கள்:

**நிலைமையில்லா உரையாடல்** - ஒவ்வொரு கோருமானும் தனித்துவமானது. மாடல் முன்பு வரும் தகவலை நினைவில் வைத்திருக்கவில்லை. இது உங்கள் வேக ஆரம்பத்தில் பயன்படுத்திய வடிவமைப்பே.

**நிலைமை கொண்ட உரையாடல்** - ஒவ்வொரு கோருமானும் உரையாடல் வரலாற்றையும் அடங்கும். மாடல் பல முறை இடைமுகத்தின்போது சாராம்சத்தை பராமரிக்கிறது. இது தயாரிப்பு பயன்பாடுகளுக்கு தேவையானது.

## முன் நிபந்தனைகள்

- Azure சந்தா மற்றும் Azure OpenAI அணுகல்
- Java 21, Maven 3.9+
- Azure CLI (https://learn.microsoft.com/en-us/cli/azure/install-azure-cli)
- Azure Developer CLI (azd) (https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/install-azd)

> **குறிப்பு:** Java, Maven, Azure CLI மற்றும் Azure Developer CLI (azd) க்கான தேவைகள் கொடுக்கப்பட்ட devcontainer-ல் முன்கூட்டி நிறுவப்பட்டுள்ளன.

> **குறிப்பு:** இந்த மொட்யூல் Azure OpenAI-ல் GPT-5.2 ஐப் பயன்படுத்துகிறது. `azd up` மூலம் உற்பத்தி நடவடிக்கை தானாகவே நடைபெறுவதால், குறியீட்டில் மாடல் பெயரை மாற்ற வேண்டாம்.

## முக்கிய சிக்கலைப் புரிந்துகொள்ளுதல்

மொழி மாதிரிகள் நிலைமையில்லாதவை. ஒவ்வொரு API அழைப்பும் தனித்துவமானது. நீங்கள் "என் பெயர் ஜான்" என்று அனுப்பி பிறகு "என் பெயர் என்ன?" என்று கேட்டால், மாடல் உங்கள் அறிமுகத்தை அறியாது. ஒவ்வொரு கோருமானையும் இது முதல் உரையாடல் போலவே கருதுகிறது.

இது எளிய கேள்வி பதில் சந்திப்பிற்கு சரியானது ஆனாலும் உண்மையான பயன்பாடுகளுக்கு பயனற்றது. வாடிக்கையாளர் சேவை போட்கள் நீங்கள் சொன்னதை நினைவில் வைக்க வேண்டும். தனிப்பட்ட உதவிக்காரர்கள் குறிப்பியுள்ளதைக் காண வேண்டும். பல முறை உரையாடலுக்கு நினைவிடம் அவசியம்.

<img src="../../../translated_images/ta/stateless-vs-stateful.cc4a4765e649c41a.webp" alt="Stateless vs Stateful Conversations" width="800"/>

*நிலைமையில்லாத (தனித்துவ அழைப்புகள்) மற்றும் நிலைமை கொண்ட (சாராம்சம் அறிந்த) உரையாடலின் வித்தியாசம்*

## டோக்கன்களைப் புரிந்துகொள்ளுதல்

உரையாடல்களில் இறங்கி விடுவதற்கு முன், டோக்கன்களை புரிந்து கொள்வது முக்கியம் - மொழி மாதிரிகள் குறிக்குறியாக செயல்படும் முகப்பு உரை அலகுகள்:

<img src="../../../translated_images/ta/token-explanation.c39760d8ec650181.webp" alt="Token Explanation" width="800"/>

*உரை எவ்வாறு டோக்கன்களாக பிரிக்கப்படுகிறது என்பதற்கான உதாரணம் - "I love AI!" என்பது 4 தனித்தனியான செயலாக்க அலகுகளாக மாறும்*

டோக்கன்கள் AI மாதிரிகள் உரையை அளவிடி செயல்படுத்தும் முறை. சொற்கள், குறியீடுகள் மற்றும் இடைவெளிகள் கூட டோக்கன்கள் ஆகலாம். உங்கள் மாடல் ஒரே நேரத்தில் எவ்வளவு டோக்கன்களை செயல்படுத்த முடியும் என்று ஒரு வரம்பு உள்ளது (GPT-5.2 க்கானது 400,000, இதில் வரைவு 272,000 மற்றும் வெளியீடு 128,000). டோக்கன்களைப் புரிந்துகொள்வது உரையாடல் நீளம் மற்றும் செலவுகளை நிர்வகிக்க உதவும்.

## ச್ಮৃতি எப்படி செயல்படுகிறது

உரையாடல் நினைவு நிலைமையில்லாத பிரச்சனையை உரையாடல் வரலாற்றை பராமரித்து தீர்க்கிறது. உங்கள் கோருவை மாடலுக்கு அனுப்புவதற்கு முன், கட்டமைப்பு தொடர்புடைய கடந்த செய்திகளை முன்னிலை சேர்க்கிறது. நீங்கள் "என் பெயர் ஜான்" என்று கேட்டபோது, மொத்த உரையாடல் வரலாற்றையும் மாடல் காணலாம்.

LangChain4j சுயமாக இதை கையாளும் நினைவக செயல்பாடுகளை வழங்குகிறது. நீங்கள் எத்தனை செய்திகளை வைக்கவேண்டும் என்பதைத் தேர்ந்தெடுக்கிறீர்கள், கட்டமைப்பு சாராம்ச பெட்டியைக் கட்டுப்படுத்துகிறது.

<img src="../../../translated_images/ta/memory-window.bbe67f597eadabb3.webp" alt="Memory Window Concept" width="800"/>

*MessageWindowChatMemory கடந்த சமீபத்திய செய்திகளின் அகலம் கொண்ட ஒரு சுழற்சி நினைவகம் பராமரிக்கிறது, பழைய செய்திகளை தானாக நீக்குகிறது*

## இது LangChain4j ஐ எப்படி பயன்படுத்துகிறது

இந்த மொட்யூல் வேக ஆரம்பத்தை விரிவுபடுத்தி Spring Boot மற்றும் உரையாடல் நினைவகத்தைச் சேர்க்கிறது. கூறுகள் எப்படி பொருந்துகிறார்கள் என்பதை இங்கே பார்க்கலாம்:

**ஆதரவுகள்** - இரண்டு LangChain4j நூலகங்களைச் சேர்க்கவும்:

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```

**உரையாடல் மாதிரி** - Azure OpenAI-ஐ Spring பீன் ([LangChainConfig.java](../../../01-introduction/src/main/java/com/example/langchain4j/config/LangChainConfig.java)) ஆக விவரிக்கவும்:

```java
@Bean
public OpenAiOfficialChatModel openAiOfficialChatModel() {
    return OpenAiOfficialChatModel.builder()
            .baseUrl(azureEndpoint)
            .apiKey(azureApiKey)
            .modelName(deploymentName)
            .timeout(Duration.ofMinutes(5))
            .maxRetries(3)
            .build();
}
```

இந்த பில்டர் `azd up` மூலம் அமைக்கப்பட்ட சூழல் மாறிலிகளிலிருந்து அங்கீகாரங்களைப் படிக்கிறது. `baseUrl` ஐ உங்கள் Azure குறிக்கும் முகவரிக்கு அமைத்தால் OpenAI கிளையண்ட் Azure OpenAI உடன் வேலை செய்கிறது.

**உரையாடல் நினைவு** - MessageWindowChatMemory மூலம் உரையாடல் வரலாற்றை கண்காணிக்கவும் ([ConversationService.java](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java)):

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("My name is John"));
memory.add(AiMessage.from("Nice to meet you, John!"));

memory.add(UserMessage.from("What's my name?"));
AiMessage aiMessage = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage);
```

`withMaxMessages(10)` கொண்டு நினைவகத்தை உருவாக்கி கடைசி 10 செய்திகளை வைத்திருங்கள். பாவனையாளர் மற்றும் AI செய்திகளுக்கு `UserMessage.from(text)` மற்றும் `AiMessage.from(text)` என்னும் வகைப்பட்ட காப்பிகள் இணைக்கவும். `memory.messages()` மூலம் வரலாற்றை பெறவும் அதை மாடலுக்கு அனுப்பவும். சேவை ஒவ்வொரு உரையாடல் ஐடியும் தனித்தனியான நினைவகங்களை வைக்கிறது, இது பல பாவனையாளர்கள் அதே நேரத்தில் உரையாட கூடியதாக இருக்க உதவும்.

> **🤖 [GitHub Copilot](https://github.com/features/copilot) உரையாடலுடன் முயற்சிக்கவும்:** [`ConversationService.java`](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java) திறந்து கேட்கவும்:
> - MessageWindowChatMemory முழு சாளரமான போது எந்த செய்திகளை நீக்க வேண்டும் என்று எப்படி முடிவு செய்கிறது?
> - நினைவகத்தை தரவுத்தளத்தில் கையாளும் தனிப்பயன் நினைவக சேமிப்பை உருவாக்க முடியுமா?
> - பழைய உரையாடல் வரலாற்றை சுருக்க தயாரித்தல் (சுருக்கம்) எப்படி சேர்க்கலாம்?

நிலைமையில்லா உரையாடலில் நினைவகம் இருக்காது - வேக ஆரம்ப மாதிரியாக `chatModel.chat(prompt)` தான். நிலைமை கொண்ட இடைமுகம் நினைவகம் சேர்க்கிறது, வரலாற்றுக்களை பெறுகிறது மற்றும் ஒவ்வொரு கோருவிலும் சாராம்சத்தை இணைக்கிறது. மாதிரிகள் ஒரே அளவில், ஆனால் முறைகள் வேறுபட்டவை.

## Azure OpenAI பொறியமைவை உபயோகப்படுத்தல்

**Bash:**
```bash
cd 01-introduction
azd up  # சந்தா மற்றும் இடத்தை தேர்ந்தெடுக்கவும் (eastus2 பரிந்துரைக்கப்படுகிறது)
```

**PowerShell:**
```powershell
cd 01-introduction
azd up  # சந்தா மற்றும் இடத்தை தேர்ந்தெடுக்கவும் (eastus2 பரிந்துரைக்கப்படுகிறது)
```

> **குறிப்பு:** தடுமாறும் நேர நேரமுறை தவறு (RequestConflict: Cannot modify resource ... provisioning state is not terminal) ஏற்பட்டால், மீண்டும் `azd up` இயக்கவும். Azure வளங்கள் பின்னணியில் இன்னும் செயல்பட்டுக் கொண்டிருக்கலாம், மீண்டும் முயற்சி செய்வது வளங்கள் இறுதி நிலையில் வந்தவுடன் நிறைவடைய உதவும்.

இதனால்:
1. GPT-5.2 மற்றும் text-embedding-3-small மாதிரிகள் கொண்ட Azure OpenAI வளம் உபயோகப்படுத்தப்படும்
2. அங்கீகாரங்கள் உட்பட `.env` கோப்பு திட்ட அடிப்படையில் தானாக உருவாக்கப்படும்
3. தேவையான அனைத்து சூழல் மாறிலிகள் அமைக்கப்படும்

**பொறியமைப்பு பிரச்சினைகளா?** துணைக்குறுந்தகடு பெயர் மோதல்கள், கைமுறை Azure Portal இல் உபயோகப்படுத்தல் படிகள் તેમજ மாடல் அமைப்பிற்கான வழிகாட்டல்களுக்கான விவரம் [Infrastructure README](infra/README.md) -ல் உள்ளது.

**பொறியமைப்பு வெற்றி பெற்றதா என்பதைச் சரிபார்க்கவும்:**

**Bash:**
```bash
cat ../.env  # AZURE_OPENAI_ENDPOINT, API_KEY மற்றும் பிறவற்றை காண்பிக்க வேண்டும்.
```

**PowerShell:**
```powershell
Get-Content ..\.env  # AZURE_OPENAI_ENDPOINT, API_KEY போன்றவை காட்டப்பட வேண்டும்.
```

> **குறிப்பு:** `azd up` கட்டளை தானாக `.env` கோப்பை உருவாக்கும். அதைத் திருத்த வேண்டுமானால், `.env` கோப்பை கைமுறை திருத்தலாம் அல்லது மீண்டும் உருவாக்க:
>
> **Bash:**
> ```bash
> cd ..
> bash .azd-env.sh
> ```

> **PowerShell:**
> ```powershell
> cd ..
> .\.azd-env.ps1
> ```


## மேலேற்பாட்டை உள்ளூர் இயக்குதல்

**பொறியமைப்பு சரிபார்க்கவும்:**

`.env` கோப்பு அடிப்படையில் Azure அங்கீகாரங்களுடன் இருப்பதை உறுதிப்படுத்தவும்:

**Bash:**
```bash
cat ../.env  # AZURE_OPENAI_ENDPOINT, API_KEY, DEPLOYMENT காட்ட வேண்டும்
```

**PowerShell:**
```powershell
Get-Content ..\.env  # AZURE_OPENAI_ENDPOINT, API_KEY, DEPLOYMENT ஐ காட்ட வேண்டும்
```


**மேலேற்பாட்டை துவங்குக:**

**விருப்பம் 1: Spring Boot டேஷ்போர்டு பயன்படுத்துதல் (VS Code பயனர்களுக்கு பரிந்துரைக்கப்படுகிறது)**

devcontainer-ல் Spring Boot டேஷ்போர்டு விரிவு உள்ளது, இது அனைத்து Spring Boot பயன்பாடுகளையும் பார்வையிடவும் நிர்வகிக்க வழங்கும். அதை VS Code இடப்பக்கத்தில் உள்ள Activity Bar ல Spring Boot ஐகான் மூலம் காணலாம்.

Spring Boot டேஷ்போர்டு மூலம் நீங்கள்:
- ஊழிய இடத்தில் உள்ள அனைத்து Spring Boot பயன்பாடுகளையும் பார்க்கலாம்
- ஒரு கிளிக்கில் பயன்பாடுகளைத் துவக்க/நிறுத்தலாம்
- பயன்பாட்டு பதிவு உரைகளை நேரடியாக பார்ைலாம்
- பயன்பாட்டு நிலையை கண்காணிக்கலாம்

"introduction" பக்கத்தில் உள்ள பிளே பட்டனை அழுத்தி இந்த மொட்யூலை துவங்கவும், அல்லது அனைத்து மொட்யூல்களையும் ஒரே நேரத்தில் துவங்கலாம்.

அனைத்து வலை பயன்பாடுகளையும் துவங்க (மொட்யூல் 01-04):

## பயன்பாட்டை பயன்படுத்துதல்

பயன்பாடு இரண்டு வகையான உரையாடல் அமுல்படுத்தல்களுடன் இணைய முகப்பை வழங்குகிறது.

<img src="../../../translated_images/ta/home-screen.121a03206ab910c0.webp" alt="Application Home Screen" width="800"/>

*Dashboard-ல் எளிமையான உரையாடல் (நிலைமையில்லா) மற்றும் உரையாடல் உரையாடல் (நிலைமை கொண்ட) விருப்பங்களைக் காண்பிகிறது*

### நிலைமையில்லா உரையாடல் (இடது பக்கத் துறை)

முதலில் இதனை முயற்சிக்கவும். "என் பெயர் ஜான்" என்று கூறி உடனடியாக "என் பெயர் என்ன?" என்று கேட்கவும். மாடல் நினைவில் வைத்திருக்காது ஏனெனில் ஒவ்வொரு செய்தியும் தனித்துவமாக இருக்கிறது. இது அடிப்படை மொழி மாதிரி ஒருங்கிணைப்பில் இருக்கும் முக்கிய சிக்கலை வெளிப்படுத்துகிறது - உரையாடல் சாராம்சம் எதுவும் இல்லை.

<img src="../../../translated_images/ta/simple-chat-stateless-demo.13aeb3978eab3234.webp" alt="Stateless Chat Demo" width="800"/>

*AI உங்கள் பெயரை முந்தைய செய்தியிலிருந்து நினைவில் வைக்கவில்லை*

### நிலைமை கொண்ட உரையாடல் (வலது பக்கத் துறை)

இப்போது அதே வரிசையை இங்கே முயற்சிக்கவும். "என் பெயர் ஜான்" என்று கூறி பிறகு "என் பெயர் என்ன?" என்று கேட்கவேண்டியது. இந்த முறை அது நினைவில் வைக்கும். வித்தியாசம் MessageWindowChatMemory-ல் உள்ளது - இது உரையாடல் வரலாற்றை பராமரித்து ஒவ்வொரு கோருவுடனும் அந்த சாராம்சத்தை சேர்க்கிறது. தயாரிப்பு உரையாடல் AI இதுவே.

<img src="../../../translated_images/ta/conversational-chat-stateful-demo.e5be9822eb23ff59.webp" alt="Stateful Chat Demo" width="800"/>

*AI உரையாடலின் முன்னிலையில் உங்கள் பெயரை நினைவில் வைத்திருக்கிறது*

இரு பக்கங்களும் ஒரே GPT-5.2 மாதிரியை பயன்படுத்துகின்றன. வித்தியாசம் நினைவில் இருக்கிறது. இது நினைவகம் உங்கள் பயன்பாட்டுக்கு என்ன கொடுக்கிறது என்று தெளிவாக காட்டும் மற்றும் உண்மையான நடைமுறைகளுக்கு அது அவசியம் என்பதையும்.
