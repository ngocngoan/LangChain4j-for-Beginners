## நீங்கள் கற்றுக்கொள்ளப்போகும் விடயங்கள்

தேவைப்படும் விரைவு தொடக்கத்தைக் கடந்த நீங்கள், எப்படி முன்மொழிவுகளை அனுப்பி பதில்கள் பெறுவது என்பதை கண்டீர்கள். அது அடிப்படை மட்டுமே, ஆனால் உண்மையான செயலிகள் அதற்குப் பதிலாக அதிகமானவை தேவைப்படுகின்றன. இந்த மொடியூல், உரையாடல் செயற்கை நுண்ணறிவை உருவாக்குவது எப்படி என்பதை உருவாக்க கற்பிக்கிறது; அது உள்ளடக்கத்தை நினைவில் வைத்துக் கொண்டு நிலையை பராமரிக்கும் - ஒரு ஒரே தடவையிலான முன்னமொழியும் உற்பத்தி நிலையில் இயங்கும் செயலியும் இடையேயான வேறுபாடு.

இந்த வழிகாட்டியில் Azure OpenAI GPT-5.2ஐ பயன்படுத்துவோம், ஏனெனில் அதன் முன்னேறிய கூற்றுக் கற்றல் திறன்கள் வெவ்வேறு வடிவமைப்புகளின் நடத்தையை தெளிவாக காட்டுகின்றன. நினைவகம் சேர்க்கும்போது, அந்த வேறுபாட்டைப் பிரத்தியேகமாக பார்க்க முடியும். இது ஒவ்வொரு கூறும் உங்கள் செயலியில் என்ன கொண்டுவருகிறது என்பதை விளங்க வைக்கிறது.

இரு வடிவமைப்புகளையும் காட்ட ஒரு செயலியை நீங்கள் உருவாக்குவீர்கள்:

**நிலைமற்ற உரையாடல்** - ஒவ்வொரு கோரும் தனித்தனியாக இருக்கும். மாடல் முன்பு வந்த செய்திகளை நினைவில் வைக்காது. விரைவு தொடக்கத்தில் பயன்படுத்திய வடிவமைப்பே இது.

**நிலையான உரையாடல்** - ஒவ்வொரு கோருதலும் உரையாடல் வரலாற்றை உள்ளடக்கியது. மாடல் பல முறை உரையாடலில் உள்ளடக்கத்தை பராமரிக்கின்றது. நிஜ செயலிகள் இதையே தேவைபடும்.

## முன்னிலைத் தேவைகள்

- Azure சந்தா மற்றும் Azure OpenAI அணுகல்
- ஜாவா 21, மேவன் 3.9+
- Azure CLI (https://learn.microsoft.com/en-us/cli/azure/install-azure-cli)
- Azure Developer CLI (azd) (https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/install-azd)

> **குறிப்பு:** ஜாவா, மேவன், Azure CLI மற்றும் Azure Developer CLI (azd) வழங்கப்பட்ட devcontainer இல் முன்கூட்டியே நிறுவப்பட்டுள்ளன.

> **குறிப்பு:** இந்த மொடியூல் Azure OpenAI இல் GPT-5.2 ஐப் பயன்படுத்துகிறது. `azd up` மூலம் தானாகவே அமைக்கப்படும் - குறியீட்டில் மாடல் பெயரை மாற்ற வேண்டாம்.

## முக்கிய பிரச்சினையைப் புரிந்து கொள்ளுதல்

மொழி மாடல்கள் நிலைமற்றவை (stateless). ஒவ்வொரு API அழைப்பும் தனித்தனியாக இருக்கும். நீங்கள் "என் பெயர் ஜான்" என்று அனுப்பி, பின்னர் "என் பெயர் என்ன?" என கேட்டால், மாடல் நீண்டகால நினைவுகளை வைத்திருப்பதில்லை. ஒவ்வொரு கோருதலும் முதன்முதலான உரையாடல் போல் நடத்தப்படுகிறது.

இதுவும் எளிமையான கேள்வி மற்றும் பதில் ஏற்பாட்டுக்கு பரவாயில்லை, ஆனால் உண்மையான செயலிகளுக்கு பயனற்றது. வாடிக்கையாளர் சேவை பாட்டுக்கு நீங்கள் கூறியதை நினைவில் வைக்க வேண்டும். தனிப்பட்ட உதவியாளர்கள் உள்ளடக்கத்தைக் கொண்டு இருக்க வேண்டும். பல முறை உரையாடல்கள் நினைவகத்தை தேவைபடுகின்றன.

<img src="../../../translated_images/ta/stateless-vs-stateful.cc4a4765e649c41a.webp" alt="Stateless vs Stateful Conversations" width="800"/>

*நிலைமற்ற (தனித்தனியான அழைப்புகள்) மற்றும் நிலையான (உள்ளடக்கக்கான) உரையாடல் இடையேயான வேறுபாடு*

## டோக்கன்களைப் பற்றி புரிந்து கொள்ளுதல்

உரையாடல்களில் நுழைவதற்கு முன், டோக்கன்களை (tokens) புரிந்துகொள்ளல் முக்கியம் - மொழி மாடல்கள் செயலாக்கும் உரையின் அடிப்படைக் கூறுகள்:

<img src="../../../translated_images/ta/token-explanation.c39760d8ec650181.webp" alt="Token Explanation" width="800"/>

*உரை எப்படி டோக்கன்களாக பிரிக்கப்படுகிறது என்பதின் எடுத்துக்காட்டு - "I love AI!" என்பது 4 தனித்தனியாயான செயலாக்க அலகுகளாகும்*

டோக்கன்கள் ஆற்றல் அளவு மற்றும் உரையை செயலாக்கும் அளவைக் குறிக்கின்றன. சொற்கள், குறிச்சொற்கள் மற்றும் இடைவெளிகளும் டோக்கன்களாக இருக்கலாம். உங்கள் மாடல் ஒரே நேரத்தில் எத்தனை டோக்கன்களை செயலாக்க முடியும் என்பதில் வரம்பு உள்ளது (GPT-5.2க்கு 400,000; இதில் 272,000 உள்ளீடு மற்றும் 128,000 வெளியீடு டோக்கன்கள்). டோக்கன்களைப் புரிந்துகொள்வது உரையின் நீளம் மற்றும் செலவுகளைக் கையாள எளிதாக்கும்.

## நினைவகம் எப்படி செயல்படுகிறது

உரையாடல் நினைவகம் நிலைமற்ற பிரச்சினையை தீர்க்க உரையாடல் வரலாற்றை பராமரிக்கிறது. உங்கள் கோருதலை மாடலுக்கு அனுப்புவதற்கு முன்பு, தானாகச் சம்வேதனையுடைய முந்தைய செய்திகள் சேர்க்கப்படுகின்றன. நீங்கள் "என் பெயர் ஜான்" என்று கூறியதை மாடல் உண்மையில் தெரியும், ஏனெனில் முழு உரையாடல் வரலாறு அனுப்பப்படுகிறது.

LangChain4j நினைவகம் தானாக கையாளும் செயலாக்கங்களை வழங்குகிறது. நீங்கள் எத்தனை செய்திகளை வைத்திருக்க விரும்புகிறீர்கள் என்று தேர்வு செய்வீர்கள், அதற்கான உள்ளடக்க ஜன்னல் (context window) பிணைப்பாளர் அதை பராமரிக்கும்.

<img src="../../../translated_images/ta/memory-window.bbe67f597eadabb3.webp" alt="Memory Window Concept" width="800"/>

*MessageWindowChatMemory சமீபத்திய செய்திகளை ஒரு ஸ்லைடிங் ஜன்னல் போல பராமரித்து பழையவற்றை தானாக நீக்குகிறது*

## LangChain4j இதனை எப்படி பயன்படுத்துகிறது

இந்த மொடியூல் விரைவு தொடக்கத்தை விரிவாக்கி Spring Boot இணைத்து உரையாடல் நினைவகத்தைச் சேர்க்கிறது. கூறுகள் ஒருங்கிணைப்புப் பாணி:

**ஆதாரங்கள்** - இரண்டு LangChain4j நூலகங்களைச் சேர்க்கவும்:

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```

**உரையாடல் மாடல்** - Azure OpenAI ஐ Spring bean ஆக கட்டமைக்கவும் ([LangChainConfig.java](../../../01-introduction/src/main/java/com/example/langchain4j/config/LangChainConfig.java)):

```java
@Bean
public OpenAiOfficialChatModel openAiOfficialChatModel() {
    return OpenAiOfficialChatModel.builder()
            .baseUrl(azureEndpoint)
            .apiKey(azureApiKey)
            .modelName(deploymentName)
            .timeout(Duration.ofMinutes(5))
            .maxRetries(3)
            .build();
}
```

பில்ட்டர் `azd up` மூலம் அமைக்கப்படும் சூழல் மாறிகளைப் படிக்கிறது. `baseUrl` ஐ உங்கள் Azure முடிவிடமாக அமைத்தால் OpenAI கிளையண்ட் Azure OpenAI உடன் வேலை செய்யும்.

**உரையாடல் நினைவகம்** - MessageWindowChatMemory உடன் உரையாடல் வரலாற்றை பின்தொடரவும் ([ConversationService.java](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java)):

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("My name is John"));
memory.add(AiMessage.from("Nice to meet you, John!"));

memory.add(UserMessage.from("What's my name?"));
AiMessage aiMessage = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage);
```

`withMaxMessages(10)` மூலம் கடைசியாக 10 செய்திகளை வைத்திருக்கும் நினைவகத்தை உருவாக்கவும். பயனர் மற்றும் AI செய்திகளை `UserMessage.from(text)` மற்றும் `AiMessage.from(text)` போன்ற வகைப்பட்டு வடிவில் சேர்க்கவும். வரலாற்றை `memory.messages()` மூலம் பெற்று அதை மாடலுக்கு அனுப்பவும். இந்த சேவை ஒவ்வொரு உரையாடல் IDக்கு தனியான நினைவகங்களை வைத்திருக்கும்; இதனால் பல பயனர்கள் ஒரே நேரத்தில் உரையாடலாம்.

> **🤖 [GitHub Copilot](https://github.com/features/copilot) உரையாடலுடன் முயற்சிக்கவும்:** [`ConversationService.java`](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java) கோப்பை திறந்து கேளுங்கள்:
> - ஜன்னல் நிரம்பியபோது MessageWindowChatMemory எந்த செய்திகள் நீக்கப்பட முடிவு செய்கிறது?
> - நினைவகத்தை In-memory அல்லாமல் தரவுத்தளத்தில் சேமிப்பதற்கான தனிப்பயன் செயல்முறை உருவாக்க முடியுமா?
> - பழைய உரையாடல் வரலாற்றை சுருக்கம் செய்ய சுருக்குரை (summarization) சேர்ப்பது எப்படி?

நிலைமற்ற உரையாடல் இறுதிப்புள்ளி நினைவகத்தை விட்டு விலகுகிறது - `chatModel.chat(prompt)` ஆக மட்டுமே செயல்படுகிறது விரைவு தொடக்க போல். நிலையான இறுதிப்புள்ளி நினைவகத்தில் செய்திகள் சேகரித்து வரலாற்றை பெறும் மற்றும் ஒவ்வொரு கோருதலுடன் உள்ளடக்கமாகக் கூடியதாக சேர்க்கிறது. மாடல் கட்டமைப்பு ஒரே ஆனால் நடைமுறை வேறுபாடு.

## Azure OpenAI உட்படையை அமல்படுத்துதல்

**Bash:**
```bash
cd 01-introduction
azd up  # சந்தா மற்றும் இடத்தை தேர்ந்தெடுக்கவும் (கிழக்குus2 பரிந்துரைக்கப்படுகிறது)
```

**PowerShell:**
```powershell
cd 01-introduction
azd up  # சந்தா மற்றும் இடத்தை தேர்ந்தெடுக்கவும் (eastus2 பரிந்துரைக்கப்படுகிறது)
```

> **குறிப்பு:** நீங்கள் timeout பிழை (RequestConflict: Cannot modify resource ... provisioning state is not terminal) சந்தித்தால், `azd up`ஐ மீண்டும் இயக்கவும். பின்னணி Azure வளங்கள் இன்னும் வகைப்படுத்தப்பட்டிருக்கும்; மீண்டும் முயற்சிப்பது வளங்கள் இறுதி நிலைக்கு வந்ததும் அமல்படுத்த உதவும்.

இது செய்கிறது:
1. GPT-5.2 மற்றும் text-embedding-3-small மாடல்களுடன் Azure OpenAI வளத்தை நிரூபிக்கிறது
2. குழுவின் ஃபோட்டோடு ஒரு .env கோப்பை தானாக உருவாக்குகிறது
3. தேவையான அனைத்து சூழல் மாறிகளையும் அமைக்கிறது

**நிறுவல் சிக்கல்கள் உள்ளதா?** [உதவி README](infra/README.md) ஐப் பாருங்கள், இதில் துணை-களம் பெயர் மோதல்கள், கையேடு Azure போர்டல் நிறுவுதல் படிகள் மற்றும் மாடல் கட்டமைப்பு வழிகாட்டல் உள்ளது.

**நிறுவல் வெற்றியது என்பதை உறுதிப்படுத்துக:**

**Bash:**
```bash
cat ../.env  # AZURE_OPENAI_ENDPOINT, API_KEY மற்றும் பிறவற்றை காட்ட வேண்டும்.
```

**PowerShell:**
```powershell
Get-Content ..\.env  # AZURE_OPENAI_ENDPOINT, API_KEY, மற்றும் பிறவற்றை காண்பிக்க வேண்டும்.
```

> **குறிப்பு:** `azd up` குறியீடு தானாக `.env` கோப்பை உருவாக்குகிறது. அதை புதுப்பிக்க விரும்பினால், நீங்கள் `.env` கோப்பை உடைத்துணைமுறையில் திருத்தலாம் அல்லது கீழ்கண்ட கோட் எனவும் இயக்கலாம்:
>
> **Bash:**
> ```bash
> cd ..
> bash .azd-env.sh
> ```
>
> **PowerShell:**
> ```powershell
> cd ..
> .\.azd-env.ps1
> ```

## செயலியைத் தன்னியக்கமாக இயக்குதல்

**நிறுவல் உறுதிப்படுத்துக:**

மூல அடைவை `.env` கோப்புடன் Azure அங்கீகார விவரங்கள் உள்ளன என்பதை உறுதிப்படுத்தவும்:

**PowerShell:**

**விருப்பம் 1: Spring Boot Dashboard பயன்படுத்துதல் (VS Code பயனர்களுக்கு பரிந்துரை)**

Spring Boot Dashboard மூலம் நீங்கள்:
- பணியிடத்தில் உள்ள அனைத்து Spring Boot செயலிகளையும் பார்க்கலாம்
- ஒரே கிளிக்கில் செயலிகளை தொடங்கு/நிறுத்தலாம்

"introduction" அருகிலுள்ள விளையாட்டு பொத்தானை அழுத்தவும், அல்லது அனைத்து மொடியூல்களையும் ஒரே நேரத்தில் தொடங்கவும்.

## செயலியைப் பயன்படுத்துதல்

இந்த செயலி இரண்டு உரையாடல் செயல்பாடுகளுடன் வலை இடைமுகத்தைக் கொடுக்கிறது.

<img src="../../../translated_images/ta/home-screen.121a03206ab910c0.webp" alt="Application Home Screen" width="800"/>

*சமயமாகும் சாதா உரையாடல் (stateless) மற்றும் உரையாடலான உரையாடல் (stateful) தேர்வுகளைக் காண்பிக்கும் கட்டளைப் பலகை*

### நிலைமற்ற உரையாடல் (இடது பக்கவாட்டில்)

முதலில் இதைப் பயன்படுத்தி பார்க்கவும். "என் பெயர் ஜான்" எனக் கேட்டு பின்னர் உடனேயே "என் பெயர் என்ன?" என்று கேளுங்கள். மாடல் நினைவில் வைக்காது, ஏனெனில் ஒவ்வொரு செய்தி தனித்தனியாக இருக்கிறது. இத்துடன் அடிப்படைக் கோட்பாடு - மொழி மாடல் ஒரே உரையாடலை நினைவில் வைத்திருக்காது என்பதே காட்டப்படுகிறது.

<img src="../../../translated_images/ta/simple-chat-stateless-demo.13aeb3978eab3234.webp" alt="Stateless Chat Demo" width="800"/>

*AI உங்கள் பெயரை முந்தைய செய்தியில் நினைவில் வைக்கவில்லை*

### நிலையான உரையாடல் (வலது பக்கவாட்டில்)

இப்போதும் அதே வரிசையை இங்கே முயற்சிக்கவும். "என் பெயர் ஜான்" என்று கூறி, "என் பெயர் என்ன?" என்று கேட்டால், இப்போது அது நினைவில் வைக்கிறது. இதற்கு காரணமாக MessageWindowChatMemory - இது உரையாடல் வரலாற்றை பராமரித்து, ஒவ்வொரு கோருதலுடன் சேர்க்கிறது. இதுவே உற்பத்தியுத் தொகுதிகளில் உரையாடல் AI வேலையைச் செய்கிறது.

<img src="../../../translated_images/ta/conversational-chat-stateful-demo.e5be9822eb23ff59.webp" alt="Stateful Chat Demo" width="800"/>

*AI உரையாடலில் முன்பு கூறிய உங்கள் பெயரை நினைவில் வைக்கிறது*

இரு பக்கங்களும் அதே GPT-5.2 மாடலைப் பயன்படுத்துகின்றன. ஒரே வேறுபாடு நினைவகம். இது நினைவகம் உங்கள் செயலிக்கு என்ன தருகிறது என்பதையும் நிர்வாகச் சூழலை எதனால் பொருந்துகிறது என்பதையும் தெளிவாகக் காட்டுகிறது.
