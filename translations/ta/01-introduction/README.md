## நீங்கள் கற்றுக் கொள்வது

நீங்கள் விரைவு தொடக்கம் முடித்திருந்தால், நீங்கள் கேள்விகளை அனுப்பி பதில்களை பெறுவது எப்படி என்பதை பார்த்தீர்கள். அது அடித்தளம் தான், ஆனால் உண்மையான செயலிகள் அதிகமாக தேவைப்படுகின்றன. இந்த மாடியூல், உரையாடல் AI-யை கட்டமைப்பதை கற்பித்து, அது சந்தர்ப்பத்தை நினைவில் வைத்துக் கொண்டு நிலையைக் கடைபிடிக்கும் - ஒரு ஒருமுறை டெமோ மற்றும் உற்பத்தி தயாரான செயலியின் இடையேயான வேறுபாடு.

இந்த வழிகாட்டியில் நாம் Azure OpenAI இன் GPT-5.2 ஐ பயன்படுத்துவோம், ஏனெனில் அதன் முன்னேற்றமான காரணிய திறன்கள் வெவ்வேறு வடிவமைப்புகளின் நடத்தையை தெளிவாக வெளிப்படுத்துகின்றன. நீங்கள் நினைவாற்றலை சேர்க்கும்போது, அதில் மட்டும் வேறுபாடு தெளிவாக தெரியும். இதனால் உங்கள் செயலியில் ஒவ்வொரு கூறும் என்னை கொண்டுவருகிறது என்பதை எளிதில் புரிந்துகொள்ள முடியும்.

நீங்கள் இரண்டு வடிவமைப்புகளைக் காட்டும் ஒரு செயலியை உருவாக்கப் போகிறோம்:

**நிலையற்ற உரையாடல்** - ஒவ்வொரு கோரிக்கையும் தனித்துவமாக இருக்கும். மாடல் முன் செய்த தகவல்களை நினைவில் வைக்காது. இது நீங்கள் விரைவு தொடக்கத்தில் பயன்படுத்திய வடிவமானது.

**நிலையான உரையாடல்** - ஒவ்வொரு கோரிக்கும் உரையாடல் வரலாற்று சேர்க்கப்படுகிறது. மாடல் பலவழிகள் இடையே சந்தர்ப்பத்தை வைத்திருக்கிறது. இது உற்பத்தி செயலிகள் தேவைப்படும்.

## முன் தேவைகள்

- Azure சந்தா மற்றும் Azure OpenAI அணுகல்
- Java 21, Maven 3.9+
- Azure CLI (https://learn.microsoft.com/en-us/cli/azure/install-azure-cli)
- Azure Developer CLI (azd) (https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/install-azd)

> **குறிப்பு:** Java, Maven, Azure CLI மற்றும் Azure Developer CLI (azd) முன்னே நிறுவப்பட்டுள்ளன devcontainer இல்.

> **குறிப்பு:** இந்த மாடியூல் Azure OpenAI இல் GPT-5.2 ஐப் பயன்படுத்துகிறது. `azd up` மூலம் பட்ஜெட் தானாக அமைக்கப்படுகிறது - குறியீட்டில் மாடல் பெயரை மாற்ற வேண்டாம்.

## முக்கிய பிரச்சினையை புரிந்துகொள்வது

மொழி மாடல்கள் நிலையற்றவை. ஒவ்வொரு API அழைப்பும் தனிப்பட்டது. நீங்கள் "என் பெயர் ஜான்" என்று அனுப்பி பின்னர் "என் பெயர் என்ன?" என கேட்டால், மாடல் நீங்கள் எனவரைப் பரிச்சயப்படுத்தியதை அறியாது. அது ஒவ்வொரு கோரிக்கையையும் நீங்கள் எப்போதும் செய்த முதல் உரையாடல் போன்று கையாள்கிறது.

இது எளிய கேள்வி மற்றும் பதிலுக்கு சரி, ஆனாலும் உண்மையான செயலிகளுக்கு பயனற்றது. வாடிக்கையாளர் சேவை பாட்டுகள் நீங்கள் கூறியதை நினைவில் வைத்திருக்க வேண்டும். தனிப்பட்ட உதவியாளர்கள் சந்தர்ப்பத்தை தேவைப்படுகின். எந்த பல சுற்று உரையாடலும் நினைவாற்றல் தேவைப்படுகிறது.

<img src="../../../translated_images/ta/stateless-vs-stateful.cc4a4765e649c41a.webp" alt="Stateless vs Stateful Conversations" width="800"/>

*நிலையற்ற (தனித்துவ அழைப்புகள்) மற்றும் நிலையான (சந்தர்ப்ப அறிந்த) உரையாடல்களிடையேயான வேறுபாடு*

## டோக்கன்களைப் புரிந்துகொள்வது

உரையாடல்களில் இறங்குவதற்கு முன், டோக்கன்களைப் புரிந்துகொள்ள முக்கியம் - மொழி மாடல்கள் செயலாக்கும் அடிப்படையில் உள்ள உரை அலகுகள்:

<img src="../../../translated_images/ta/token-explanation.c39760d8ec650181.webp" alt="Token Explanation" width="800"/>

*உரை எப்படி டோக்கன்களாகப் பிரிக்கப்படுகிறது - "I love AI!" 4 தனிப்பட்ட செயலாக்க அலகுகளாக மாறுகிறது*

டோக்கன்கள் AI மாடல்களின் உரையை அளவிடும் மற்றும் செயலாக்கும் விதம். வார்த்தைகள், முழங்குறிகள் மற்றும் இடைவெளிகளும் டோக்கன்கள் ஆக இருக்கலாம். உங்கள் மாடலுக்கு ஒரே நேரத்தில் எத்தனை டோக்கன்களை செயலாக்கமுடியும் என்பதில் வரம்பு உள்ளது (GPT-5.2 க்கான 400,000, இதில் 272,000 உள்ளீட்டு டோக்கன்கள் மற்றும் 128,000 வெளியீட்டு டோக்கன்கள்). டோக்கன்களைப் புரிதல் உரையாடல் நீளம் மற்றும் செலவுகளை நிர்வகிக்க உதவும்.

## சோம்சுத்தன்மை எப்படி வேலை செய்கிறது

சாட் நினைவாற்றல் நிலையற்ற பிரச்சினையை உரையாடல் வரலாற்றை வைத்திருத்தல் மூலம் தீர்க்கிறது. உங்கள் கோரிக்கையை மாடலுக்கு அனுப்பும் முன், கட்டமைப்பு சம்பந்தப்பட்ட முன் செய்திகளைக் கூட்டிச் சேர்க்கிறது. நீங்கள் "என் பெயர் ஜான்" என்றால், மாடல் உங்கள் முந்தைய உரையாடல் வரலாற்றை முழுமையாகப் பெறுகிறது, எனவே அது நீங்கள் பேசியதை பார்க்கிறது.

LangChain4j தானாக இதை பராமரிக்கும் நினைவாற்றல் செயல்திறன்களை வழங்குகிறது. நீங்கள் எத்தனை செய்திகளை வைத்திருக்க வேண்டும் என்பதைத் தேர்ந்தெடுக்கின்றீர்கள் மற்றும் கட்டமைப்பு சந்தர்ப்ப சாளரத்தை பராமரிக்கிறது.

<img src="../../../translated_images/ta/memory-window.bbe67f597eadabb3.webp" alt="Memory Window Concept" width="800"/>

*MessageWindowChatMemory சமீபத்திய செய்திகளின் ஒரு ஸ்லைடிங் சாளரத்தை பராமரித்து, பழையவற்றை தானாக அகற்றும்*

## இது LangChain4j ஐ எப்படி பயன்படுத்துகிறது

இந்த மாடியூல் விரைவு தொடக்கத்தை விரிவாக்கி Spring Boot ஐ இணைத்து உரையாடல் நினைவாற்றலைச் சேர்க்கிறது. கூறுகள் எப்படி இணைக்கப்படுகின்றன என்பதைக் கீழே பார்க்கலாம்:

**இணைப்புகள்** - இரண்டு LangChain4j நூலகங்களைச் சேர்க்கவும்:

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```

**சாட் மாடல்** - Azure OpenAI ஐ Spring கேனை ([LangChainConfig.java](../../../01-introduction/src/main/java/com/example/langchain4j/config/LangChainConfig.java)) ஆக்கமாக கட்டமைக்கவும்:

```java
@Bean
public OpenAiOfficialChatModel openAiOfficialChatModel() {
    return OpenAiOfficialChatModel.builder()
            .baseUrl(azureEndpoint)
            .apiKey(azureApiKey)
            .modelName(deploymentName)
            .timeout(Duration.ofMinutes(5))
            .maxRetries(3)
            .build();
}
```

பில்டர்  `azd up` மூலம் அமைக்கப்பட்ட சுற்றுச்சூழல் மாறிகளில் இருந்து கடவுச்சொற்களை வாசிக்கிறது. உங்கள் Azure முடிவுத் தளத்துக்கு `baseUrl` அமைத்தல் OpenAI கிளையன்டை Azure OpenAI உடன் வேலை செய்யச் செய்கிறது.

**உரையாடல் நினைவாற்றல்** - MessageWindowChatMemory உடன் உரையாடல் வரலாற்றை பின்தொடரவும் ([ConversationService.java](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java)):

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("My name is John"));
memory.add(AiMessage.from("Nice to meet you, John!"));

memory.add(UserMessage.from("What's my name?"));
AiMessage aiMessage = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage);
```

`withMaxMessages(10)` ஐப் பயன்படுத்தி கடைசியாக 10 செய்திகள் மட்டுமே வைத்திருத்தல் செய்யவும். பயனர் மற்றும் AI செய்திகளை `UserMessage.from(text)` மற்றும் `AiMessage.from(text)` typed wrapper களுடன் சேர்க்கவும். வரலாற்றை `memory.messages()` மூலம் எடுத்துக் கொண்டு மாடலுக்கு அனுப்பவும். இந்த சேவை உரையாடல் ஐடியின் அடிப்படையில் தனித்த நினைவுகளைச் சேமிப்பதால் பல பயனர்கள் ஒரே நேரத்தில் உரையாடலாம்.

> **🤖 [GitHub Copilot](https://github.com/features/copilot) Chats உடன் முயற்சிக்கவும்:** [`ConversationService.java`](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java) ஐ திறந்து கேளுங்கள்:
> - "MessageWindowChatMemory எந்த செய்திகள் திருத்தப்பட வேண்டும் என்று எப்படி தீர்மானிக்கிறது?"
> - "நான் நினைவாற்றலை நேரடியாக ஒரு தரவுத்தளத்தைப் பயன்படுத்தி மறுபடியும் உருவாக்க முடியுமா?"
> - "பழைய உரையாடலை சுருக்குவதற்கான சுருக்கமென்பதை நான் எப்படி சேர்க்க முடியும்?"

நிலையற்ற சாட் எண்ட்போயிண்ட் நினைவாற்றலை முழுமையாக தவிர்க்கிறது - விரைவு தொடக்கம் போலவே `chatModel.chat(prompt)` பயன்படுத்துகிறது. நிலையான எண்ட்போயிண்ட் நினைவுகளைச் சேர்க்கிறது, வரலாற்றை மீட்டெடுக்கிறது மற்றும் அதை ஒவ்வொரு கோரிக்கும் சேர்க்கிறது. ஒன்றே மாடல் கட்டமைப்பு, வெவ்வேறு வடிவமைப்புகள்.

```powershell
cd 01-introduction
azd up  # சந்தாவையும் இடத்தையும் தேர்ந்தெடுக்கவும் (கிழக்குஎஸ்2 பரிந்துரைக்கப்படுகிறது)
```

> **குறிப்பு:** நேரம் முடிந்த பிழை வந்தால் (`RequestConflict: Cannot modify resource ... provisioning state is not terminal`), `azd up` முடுக்கி மீண்டும் இயக்கவும். Azure வளங்கள் பின்னணி முறையில் இன்னும் சேவையாக இருக்கலாம், மீண்டும் முயற்சித்தால் வளங்கள் இறுதி நிலையில் வந்ததும் பட்ஜெட் முடியும்.

இது பின்வரும் செயலைச் செய்யும்:
1. GPT-5.2 மற்றும் text-embedding-3-small மாடல்கள் உடன் Azure OpenAI வளத்தை நிறுவல்
2. சொந்த பயனர் அடையாள விவரங்களுடன் `.env` கோப்பை திட்ட வேர் அடியிலேயே தானாக உருவாக்கல்
3. தேவையான அனைத்து சுற்றுச்சூழல் மாறிகளையும் அமைத்தல்

**பட்ஜெட் பிரச்சினைகள் உள்ளதா?** [படைப்பமைப்பு README](infra/README.md) இல் விரிவான பிரச்சினைகள், துணைக்கோப்பு பெயர் மோதல்கள், கையால் Azure போர்டல் மூலம் பட்ஜெட் எடுக்கும் படிகள் மற்றும் மாடல் கட்டமைப்பு வழிகாட்டிகள் உள்ளன.

**பட்ஜெட் வெற்றிகரமாக முடிந்ததா என்பதை சரிபார்க்கவும்:**

**Bash:**
```bash
cat ../.env  # AZURE_OPENAI_ENDPOINT, API_KEY மற்றும் பிறவற்றை காட்டு வேண்டும்.
```

**PowerShell:**
```powershell
Get-Content ..\.env  # AZURE_OPENAI_ENDPOINT, API_KEY போன்றவை காட்டப்பட வேண்டும்.
```

> **குறிப்பு:** `azd up` கட்டளை `.env` கோப்பை தானாக உருவாக்கும். பின்னர் மாற்ற வேண்டும் என்றால், வீட்டில் `.env` கோப்பை நேரடியாக திருத்தவும் அல்லது இதை மீண்டும் உருவாக்க:
>
> **Bash:**
> ```bash
> cd ..
> bash .azd-env.sh
> ```
>
> **PowerShell:**
> ```powershell
> cd ..
> .\.azd-env.ps1
> ```

## செயலியை சொந்த கட்டணத்தில் இயக்குக

**பட்ஜெட் சரிபார்த்தல்:**

Azure அடையாள விவரங்களுடன் `root` அடைவில் `.env` கோப்பு உள்ளது என்று உறுதிசெய்யவும்:

**Bash:**
```bash
cat ../.env  # AZURE_OPENAI_ENDPOINT, API_KEY, DEPLOYMENT காட்டு வேண்டும்
```

**PowerShell:**
```powershell
Get-Content ..\.env  # AZURE_OPENAI_ENDPOINT, API_KEY, DEPLOYMENT ஐ காட்ட வேண்டும்
```

**செயலிகளைக் தொடங்கு:**

**விருப்பம் 1: Spring Boot டேஷ்போர்டை பயன்படுத்துதல் (VS Code பயனர்களுக்கு பரிந்துரைக்கப்படுகிறது)**

devcontainer இல் Spring Boot டேஷ்போர்டு நீளாக்கம் உள்ளது, இது அனைத்து Spring Boot செயலிகளையும்視ிடுவதற்கான விசுவல் முகப்பை வழங்குகிறது. VS Code இன் இடது Activity Bar இல் Spring Boot ஐகானை காணலாம்.

Spring Boot டேஷ்போர்டிலிருந்து நீங்கள்:
- பணியிடத்தில் உள்ள அனைத்து Spring Boot செயலிகளையும் பார்க்கலாம்
- ஒரு கிளிக்கில் செயலிகளை தொடங்கு/நிறுத்தலாம்
- செயலி பதிவு அடையாளங்களை நேரடியாக பார்க்கலாம்
- செயலி நிலையை கண்காணிக்கலாம்

"introduction" உடன் இணைக்கப்பட்ட பிளே பொத்தானை கிளிக் செய்து இம்மாடியூலை தொடங்கு அல்லது அனைத்து மாடியூல்களையும் ஒரே நேரத்தில் தொடங்கலாம்.

<img src="../../../translated_images/ta/dashboard.69c7479aef09ff6b.webp" alt="Spring Boot Dashboard" width="400"/>

**விருப்பம் 2: ஷெல் ஸ்கிரிப்ட்கள் பயன்படுத்துதல்**

அனைத்து வலை செயலிகள் (மாடியூல்கள் 01-04) துவக்கம்:

**Bash:**
```bash
cd ..  # மூல அடைவைச் சார்ந்து

```

**PowerShell:**

```powershell
.\start-all.ps1
```

அல்லது இம்மாடியூலை மட்டும் துவங்க:

**Bash:**
```bash

./start.sh
```

**PowerShell:**
```powershell

.\start.ps1
```

இரு ஸ்கிரிப்ட்களும் தானாக `.env` கோப்பிலிருந்து சுற்றுச்சூழல் மாறிகளை ஏற்றும் மற்றும் JARகளை உருவாக்கும்.

> **குறிப்பு:** செயலிகளை துவங்கும்முன் அனைத்து மாடியூல்களையும் கையேடு படி கட்டமைக்க விரும்பினால்:

## செயலியைப் பயன்படுத்துதல்

செயலி இரு சாட் அமலாக்கங்களைக் கொண்ட வலை முகம்மை வழங்குகிறது.

<img src="../../../translated_images/ta/home-screen.121a03206ab910c0.webp" alt="Application Home Screen" width="800"/>

*இரு தேர்வுகளைக் கொண்ட டேஷ்போர்டு - எளிமையான சாட் (நிலையற்ற) மற்றும் உரையாடல் சாட் (நிலையான)*

### நிலையற்ற உரையாடல் (இடது பக்கம்)

இதனால் தொடங்குக. "என் பெயர் ஜான்" என்று கேளுங்கள், பின்னர் உடனடியாக "என் பெயர் என்ன?" என்று கேளுங்கள். மாடல் நினைவில் வைக்காது ஏனெனில் ஒவ்வொரு செய்தியும் தனித்துவமானது. இது அடிப்படையிலான மொழி மாடல் ஒருங்கிணைப்பின் முக்கிய பிரச்சினையை காட்டுகிறது - உரையாடல் சந்தர்ப்பம் இல்லை.

<img src="../../../translated_images/ta/simple-chat-stateless-demo.13aeb3978eab3234.webp" alt="Stateless Chat Demo" width="800"/>

*AI முன் செய்தியில் உங்கள் பெயரை நினைவில் வைக்காது*

### நிலையான உரையாடல் (வலது பக்கம்)

இப்போது அதே தொடர் முறையை இங்கே முயற்சிக்கவும். "என் பெயர் ஜான்" என்று கேளுங்கள், பின்னர் "என் பெயர் என்ன?" கேளுங்கள். இம்முறை மாடல் நினைவில் வைத்திருக்கும். வேறுபாடு MessageWindowChatMemory ஆகும் - இது உரையாடல் வரலாற்றை பராமரித்து ஒவ்வொரு கோரிக்கைக்கும் சேர்க்கிறது. இதுவே உற்பத்தி உரையாடல் AI வேலை செய்கிறது.

<img src="../../../translated_images/ta/conversational-chat-stateful-demo.e5be9822eb23ff59.webp" alt="Stateful Chat Demo" width="800"/>

*AI உரையாடலில் முன்பு கூறிய உங்கள் பெயர் நினைவில் உள்ளது*

இரு பக்கங்களும் ஒரே GPT-5.2 மாடலைப் பயன்படுத்துகின்றன. ஒரே வேறுபாடு நினைவாற்றல். இது நினைவாற்றல் உங்கள் செயலிக்கு என்ன கூட்டும் மற்றும் உண்மையான பயன்பாடுகளுக்கு அது ஏன் அவசியம் என்பதை தெளிவாக தருகிறது.
