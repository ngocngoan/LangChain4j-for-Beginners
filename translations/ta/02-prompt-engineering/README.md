## நீங்கள் கற்றுக்கொள்ளவிருக்கும் விஷயங்கள்

<img src="../../../translated_images/ta/what-youll-learn.c68269ac048503b2.webp" alt="What You'll Learn" width="800"/>

முந்தைய மாட்யூலில், நினைவகத்தின் மூலம் உரையாடல் AI எவ்வாறு இயங்குகிறது என்பதைக் காண்பீர்கள் மற்றும் அடிப்படை தொடர்புகளுக்காக GitHub மாடல்களை பயன்படுத்தினீர்கள். இப்பொழுது நாம் கேள்விகளை எவ்வாறு கேட்கிறோமோ அதற்கான ப்ராம்ட்கள் குறித்தே கவனம் செலுத்தப்போகிறோம் — Azure OpenAI இன் GPT-5.2 ஐப் பயன்படுத்தி. நீங்கள் உங்கள் ப்ராம்ட்களை கட்டமைக்கும் முறை உங்கள் பெறும் பதில்களின் தரத்தை வெகுவாக பாதிக்கிறது. நாம் அடிப்படை ப்ராம்டிங் தொழில்நுட்பங்களை மீண்டும் பார்க்கிறோம், பின்னர் GPT-5.2 இன் திறன்களை முழுமையாக பயன்படுத்தும் எட்டு மேம்பட்ட வடிவிகளுக்கு வெளியே செல்கிறோம்.

Reasoning control (காரணமாக்கல் கட்டுப்பாடு) அறிமுகப்படுத்தப்பட்டதால் GPT-5.2 ஐப் பயன்படுத்துகிறோம் - பதிலை தருவதற்கு முன் கானதை எவ்வளவு செய்ய வேண்டும் என்று நீங்கள் மாடல் க்கு சொல்கிறீர்கள். இது வெவ்வேறு ப்ராம்டிங் முறைகளை தெளிவாக காட்டுகிறது மற்றும் எந்த முறையை எப்போது பயன்படுத்துவது என்பதை புரிந்து கொள்வதற்கு உதவுகிறது. மேலும் GPT-5.2 க்கான Azure இன் குறைந்த விகித வரம்புகளின் நன்மையை நாம் அனుభவிக்கப் போகிறோம், ஒப்பிடும்போது GitHub மாடல்கள்.

## ப்ராம்ட் பொறியியலைப் புரிந்துகொள்வது

<img src="../../../translated_images/ta/what-is-prompt-engineering.5c392a228a1f5823.webp" alt="What is Prompt Engineering?" width="800"/>

ப்ராம்ட் பொறியியல் என்பது நீங்கள் தேடிக் கொண்டிருக்கும் முடிவுகளை நன்கு பெற consistently input உரை வடிவமைப்பது குறித்தது. இது கேள்விகள் கேட்குவது மட்டுமல்ல - மாடல் உங்கள் வேண்டுகோளை முழுமையாக புரிந்து கொண்டு அதை எவ்வாறு வழங்க வேண்டும் என்பதற்கான கதவுகளை திறக்கின்றது.

ஏதேனும் சக ஊழியருக்கு வழிமுறைகளை கொடுப்பதைப் போல நினைத்துக் கொள்ளுங்கள். "பிழையை சரி செய்" என்பது தெளிவற்றது. "UserService.java வரி 45 லுள்ள null pointer exception ஐ null check சேர்த்து சரி செய்" என்பது குறிப்பிட்டது. மொழி மாதிரிகள் அதே விதத்தில் செயல்படுகின்றன - தெளிவுத்தன்மையும் அமைப்பும் முக்கியம்.

<img src="../../../translated_images/ta/how-langchain4j-fits.dfff4b0aa5f7812d.webp" alt="How LangChain4j Fits" width="800"/>

LangChain4j உடைய நிலைமை - மாடல் இணைப்புகள், நினைவகமும், செய்தி வகைகளும் ஆகியவற்றை வழங்குகிறது, ப்ராம்ட் வடிவங்கள் கவனமாக கட்டமைக்கப்பட்ட உரையாகும், அது அந்த அடித்தளத்தில் அனுப்பப்படுகின்றன. முக்கிய கட்டமைப்புகள் `SystemMessage` (AI இன் நடத்தை மற்றும் பங்கு அமைக்கும்) மற்றும் `UserMessage` (உங்கள் உண்மையான கோரிக்கையை கொண்டிருக்கும்).

## ப்ராம்ட் பொறியியல் அடிப்படைகள்

<img src="../../../translated_images/ta/five-patterns-overview.160f35045ffd2a94.webp" alt="Five Prompt Engineering Patterns Overview" width="800"/>

இந்த மாட்யூலில் உள்ள உண்மைச் சிக்கலான வடிவிகளுக்குள் நுழைய முன்பாக, ஐந்து அடிப்படை ப்ராம்டிங் தொழில்நுட்பங்களை மீண்டும் பார்ப்போம். இவை ப்ராம்ட் பொறியியலாளர் அனைவரும் அறிந்திருக்க வேண்டிய அடித்தளக் கட்டுமானங்கள். நீங்கள் ஏற்கனவே [Quick Start module](../00-quick-start/README.md#2-prompt-patterns) வழியாக இதைப் பார்த்திருக்கலாம் — இதன் பின்னணி கருத்தாக்கங்கள் இப்போது.

### சூடாக இல்லாத ப்ராம்டிங்

எளிய அணுகுமுறை: எந்த உதாரணங்களும் இல்லாமல் நேரடி வழிமுறையை மாடலுக்கு கொடுக்கவும். மாடல் முழுமையாக தனது பயிற்சியிலேயே பின்னர் அந்தபணி செய்கிறது. இது தெளிவான கோரிக்கைகளுக்கு சிறப்பாக வேலை செய்கிறது.

<img src="../../../translated_images/ta/zero-shot-prompting.7abc24228be84e6c.webp" alt="Zero-Shot Prompting" width="800"/>

*உதாரணங்கள் இல்லாமல் நேரடி வழிமுறை — மாடல் வழிமுறை மட்டுமே பயன்படுத்தி பணி கற்றுக்கொள்கிறது*

```java
String prompt = "Classify this sentiment: 'I absolutely loved the movie!'";
String response = model.chat(prompt);
// பதில்: "நண்பர்மிகு"
```

**எப்போது பயன்படுத்த வேண்டும்:** எளிய வகைப்பாடு, நேரடி கேள்விகள், மொழிபெயர்ப்புகள் அல்லது எந்தவொரு கூடுதல் வழிகாட்டல் இல்லா பணிகள்.

### குறைந்த உதாரண ப்ராம்டிங்

கோர்க்கும் மாதிரியை காட்டும் உதாரணங்களை வழங்கவும். மாடல் உங்கள் உதாரணங்களில் இருந்து உள்ளீடு-வெளியீடு வடிவத்தை கற்றுக்கொண்டு புதிய உள்ளீடுகளில் அதைப் பயன்படுத்துகிறது. இது வேண்டப்படும் வடிவம் அல்லது நடத்தைகள் தெளிவாக இல்லாத பணிகளிற்கு தரம் மேம்பட உதவும்.

<img src="../../../translated_images/ta/few-shot-prompting.9d9eace1da88989a.webp" alt="Few-Shot Prompting" width="800"/>

*உதாரணங்களிலிருந்து கற்றுக்கொள்கிறது — மாடல் மாதிரியை கண்டறிந்து புதிய உள்ளீடுகளில் பயன்படுத்துகிறது*

```java
String prompt = """
    Classify the sentiment as positive, negative, or neutral.
    
    Examples:
    Text: "This product exceeded my expectations!" → Positive
    Text: "It's okay, nothing special." → Neutral
    Text: "Waste of money, very disappointed." → Negative
    
    Now classify this:
    Text: "Best purchase I've made all year!"
    """;
String response = model.chat(prompt);
```

**எப்போது பயன்படுத்த வேண்டும்:** தனிப்பயன் வகைப்பாடு, ஒருங்கிணைந்த வடிவமைப்பு, துறை சார்ந்த பணிகள் அல்லது சூடாக இல்லாத ப்ராம்ட் முடிவுகள் ஒருமைப் படாத போது.

### மாக்கையின் சங்கிலி

மாடலைத் தனிப்பட்ட படி படியாக அதன் காரணமறிதலை காட்ட கேட்கின்றீர்கள். பதிலை நேரடியாகப் பெறுவதற்கு பதில், மாடல் பிரச்சனையைக் கட்டவிழ்க்கிறது மற்றும் ஒவ்வொரு பகுதியையும் தெளிவாக செயல்படுத்துகிறது. இது கணக்கு, தர்க், மற்றும் பல படி காரணமறிதல் பணிகளில் துல்லியத்தைக் மேம்படுத்துகிறது.

<img src="../../../translated_images/ta/chain-of-thought.5cff6630e2657e2a.webp" alt="Chain of Thought Prompting" width="800"/>

*படி படியாக காரணமறிதல் — சிக்கலான பிரச்சனைகளை தெளிவான தர்க் படிகளாக உடைத்தல்*

```java
String prompt = """
    Problem: A store has 15 apples. They sell 8 apples and then 
    receive a shipment of 12 more apples. How many apples do they have now?
    
    Let's solve this step-by-step:
    """;
String response = model.chat(prompt);
// மாதிரி காட்டுகிறது: 15 - 8 = 7, அப்புறம் 7 + 12 = 19 ஆப்பிள்கள்
```

**எப்போது பயன்படுத்த வேண்டும்:** கணிதம், தர்க்க புதிர்கள், பிழைத்திருத்துதல், அல்லது காரணமறிதல் செயல்முறை துல்லியத்தையும் நம்பிக்கையையும் மேம்படுத்தும் பணிகள்.

### பங்கு அடிப்படையிலான ப்ராம்டிங்

அந்த AIக்கு முன்பே ஒரு பங்கு அல்லது தனிமையை அமைக்கவும். இது பதிலின் குரல், ஆழம் மற்றும் கவனத்தை வடிவமைக்கும். ஒரு "சாப்ட்வேர் வடிவமைப்பாளர்" ஒரு "குழந்தை டெவலப்பர்" அல்லது "பாதுகாப்பு ஆய்வாளர்"இல் இருந்து பிறந்த வேறுபாடாக அறிவுரைகள் தருவார்.

<img src="../../../translated_images/ta/role-based-prompting.a806e1a73de6e3a4.webp" alt="Role-Based Prompting" width="800"/>

*சார்ப்ப_CONTEXT__ாளும் பங்கு அமைத்தல் — அதே கேள்விக்கு அத்தகைய பாத்திரத்தை ஒதுக்குதல் மூலம் மாறுபட்ட பதில்*

```java
String prompt = """
    You are an experienced software architect reviewing code.
    Provide a brief code review for this function:
    
    def calculate_total(items):
        total = 0
        for item in items:
            total = total + item['price']
        return total
    """;
String response = model.chat(prompt);
```

**எப்போது பயன்படுத்த வேண்டும்:** குறியீடு மதிப்பீடு, பயிற்சி, துறை சார்ந்த பகுப்பாய்வு, அல்லது குறிப்பிட்ட திறன் மட்டத்தில் அல்லது பார்வையில் பதில்களை விரும்பும்போது.

### ப்ராம்ட் மாதிரிகள்

மாறிலி இடங்கள் கொண்ட மீண்டும் பயன்படுத்தக்கூடிய ப்ராம்ட்களை உருவாக்கவும். ஒவ்வொரு முறை புதிய ப்ராம்ட் எழுதும் பதிலாக, ஒரே மாதிரியை ஒருமுறை வரையறுத்து வெவ்வேறு மதிப்புகளை நிரப்பவும். LangChain4j இன் `PromptTemplate` வகுப்பு `{{variable}}` என்பது பெரிதும் உதவுகிறது.

<img src="../../../translated_images/ta/prompt-templates.14bfc37d45f1a933.webp" alt="Prompt Templates" width="800"/>

*மாறிலி இடங்கள் கொண்ட மீண்டும் பயன்படுத்தக்கூடிய ப்ராம்ட்கள் — ஒரு மாதிரி, பல பயன்பாடுகள்*

```java
PromptTemplate template = PromptTemplate.from(
    "What's the best time to visit {{destination}} for {{activity}}?"
);

Prompt prompt = template.apply(Map.of(
    "destination", "Paris",
    "activity", "sightseeing"
));

String response = model.chat(prompt.text());
```

**எப்போது பயன்படுத்த வேண்டும்:** வெவ்வேறு உள்ளீடுகளுக்கான மீண்டும் கேள்விகள், தொகுப்பு செயலாக்கம், மீண்டும் பயன்படுத்தக்கூடிய AI வேலைபாடுகள், அல்லது ப்ராம்ட் வடிவமைப்பு சீராக இருக்கும் ஆனால் தரவு மாறும் நிலைகளுக்கு.

---

இந்த ஐந்து அடிப்படைகள் பெரும்பாலும் ப்ராம்டிங் பணிகளுக்கான வலுவான கருவித்தொகுப்பை வழங்குகின்றன. இது மாட்யூலை மறுசுழற்சி செய்யும் மறு பகுதியான **எட்டு மேம்பட்ட வடிவுகள்** GPT-5.2 இன் காரணமாக்கல் கட்டுப்பாடு, சுய மதிப்பிடுதல் மற்றும் கட்டமைக்கப்பட்ட வெளியீடு திறன்களை முழுமையாகப் பயன்படுத்துகின்றன.

## உண்மைச் சிக்கலான வடிவிகள்

அடிப்படைகளை முடித்து, இந்த மாட்யூலை தனிப்பட்டது ஆக்கும் எட்டு மேம்பட்ட வடிவுகளுக்கு செல்லலாம். அனைத்து பிரச்சனைகளுக்கும் ஒரே அணுகுமுறை தேவையில்லை. சில கேள்விகள் விரைவு பதில்களை தேவைப்படுத்துகின்றன, சில ஆழமான சிந்தனையை. சில வெளிப்படையான காரணமறிதலை, சில பழுதற்ற முடிவுகளை மட்டுமே. கீழ்க்காணும் ஒவ்வொரு வடிவும் வேறு ஒரு சூழலுக்கானதாக தகுதி பெற்றது — மேலும் GPT-5.2 இன் காரணமாக்கல் கட்டுப்பாடு வேறுபாடுகளை இன்னும் தெளிவாக்குகிறது.

<img src="../../../translated_images/ta/eight-patterns.fa1ebfdf16f71e9a.webp" alt="Eight Prompting Patterns" width="800"/>

*எட்டு ப்ராம்ட் பொறியியல் வடிவிகளின் கண்ணோட்டம் மற்றும் அவற்றின் பயன்பாடுகள்*

<img src="../../../translated_images/ta/reasoning-control.5cf85f0fc1d0c1f3.webp" alt="Reasoning Control with GPT-5.2" width="800"/>

*GPT-5.2 இன் காரணமாக்கல் கட்டுப்பாடு நீங்கள் எவ்வளவு சிந்தனை செய்ய வேண்டும் என்று குறிப்பிட அனுமதிக்கிறது — விரைவு நேரடி பதில்களிலிருந்து ஆழமான ஆராய்ச்சிவரை*

**குறைந்த உற்சாகம் (விரைவு மற்றும் மையமாக்கப்பட்ட)** - எளிய கேள்விகளுக்கு விரைவான, நேரடி பதில்களுக்காக. மாடல் குறைந்த காரணமாக்கலை செய்கிறது - அதிகபட்சம் 2 படிகள். கணக்கீடுகள், தேடல்கள் அல்லது நேரடியான கேள்விகளுக்குப் பயன்படுத்தவும்.

```java
String prompt = """
    <context_gathering>
    - Search depth: very low
    - Bias strongly towards providing a correct answer as quickly as possible
    - Usually, this means an absolute maximum of 2 reasoning steps
    - If you think you need more time, state what you know and what's uncertain
    </context_gathering>
    
    Problem: What is 15% of 200?
    
    Provide your answer:
    """;

String response = chatModel.chat(prompt);
```

> 💡 **GitHub Copilot உடன் ஆராய்வு செய்யவும்:** [`Gpt5PromptService.java`](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java) திறந்து கேளுங்கள்:
> - "குறைந்த உற்சாகம் மற்றும் அதிக உற்சாகம் ப்ராம்ட் வடிவுகளுக்கு என்ன வேறுபாடு உள்ளது?"
> - "ப்ராம்ட்களில் XML குறிச்சொற்கள் AI பதிலை எப்படி கட்டமைக்க உதவி செய்கின்றன?"
> - "சுய-பாங்காற்றும் வடிவுகளை நேரடி வழிமுறைகளுடன் எப்போது பயன்படுத்த வேண்டும்?"

**அதிக உற்சாகம் (ஆழமான மற்றும் விரிவான)** - சிக்கலான பிரச்சனைகளை விரிவான பகுப்பாய்வுடன் விரும்பும்போது. மாடல் தெளிவாக காரணமாக்கல் செய்தி வழங்குகிறது. அமைப்பு வடிவமைப்பு, கட்டமைப்பு முடிவுகள் அல்லது சிக்கலான ஆராய்ச்சி பணிகளுக்கு இதைப் பயன்படுத்தவும்.

```java
String prompt = """
    Analyze this problem thoroughly and provide a comprehensive solution.
    Consider multiple approaches, trade-offs, and important details.
    Show your analysis and reasoning in your response.
    
    Problem: Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```

**பணி நிறைவேற்றம் (படி படிப்படி முன்னேற்றம்)** - பல படி வேலைநெறிகளுக்காக. மாடல் முன்னணி திட்டம் வழங்கி, ஒவ்வொரு படியிலும் செயலில் கதை அமைத்து, பின்னர் சுருக்கத்தை அளிக்கிறது. இடமாற்றம், செயல்படுத்தல் அல்லது ஏதேனும் பல படி செயல்முறைகளுக்குப் பயன்படுத்தவும்.

```java
String prompt = """
    <task_execution>
    1. First, briefly restate the user's goal in a friendly way
    
    2. Create a step-by-step plan:
       - List all steps needed
       - Identify potential challenges
       - Outline success criteria
    
    3. Execute each step:
       - Narrate what you're doing
       - Show progress clearly
       - Handle any issues that arise
    
    4. Summarize:
       - What was completed
       - Any important notes
       - Next steps if applicable
    </task_execution>
    
    <tool_preambles>
    - Always begin by rephrasing the user's goal clearly
    - Outline your plan before executing
    - Narrate each step as you go
    - Finish with a distinct summary
    </tool_preambles>
    
    Task: Create a REST endpoint for user registration
    
    Begin execution:
    """;

String response = chatModel.chat(prompt);
```

சங்கிலி-ஆஃப்-தவுண்ட் ப்ராம்டிங் சுயேட்சி முயற்சி செயல்முறையை தெளிவாக காட்ட வேண்டும் என மாடலை கேட்கிறது, இது சிக்கலான பணிகளுக்கான துல்லியத்தைக் கூட்டுகிறது. படி படியாக உடைப்பது மனிதர்களுக்கும் AI க்கும் தர்க்கத்தை புரிந்து கொள்ள உதவுகிறது.

> **🤖 [GitHub Copilot](https://github.com/features/copilot) உரையாடலுடன் முயற்சி செய்யவும்:** இந்த வடிவம் பற்றி கேளுங்கள்:
> - "நீண்டநேரம் இயங்கும் செயலிகளுக்கான பணி நிறைவேற்ற வடிவத்தை எப்படி பயன்படுத்துவேன்?"
> - "தயாரிப்பு பயன்பாடுகளில் கருவி முன்னுரைகள் எப்படி சிறந்த முறையில் அமைக்கப்பட வேண்டும்?"
> - "மூன்றாம் நிலை முன்னேற்ற புதுப்பிப்புகளை UI இல் எப்படி பதிவு செய்து காட்டுவது?"

<img src="../../../translated_images/ta/task-execution-pattern.9da3967750ab5c1e.webp" alt="Task Execution Pattern" width="800"/>

*பதிக → நிறைவேற்ற → சுருக்க நண்பருக்கு பல படி பணிகளுக்கான வேலைநெறி*

**சுய பாங்காற்றும் குறியீடு** - தயாரிப்பு தர குறியீட்டை உருவாக்க. மாடல் பிழை கையாளல் உட்பட தயாரிப்பு நிலையை பின்பற்றும் குறியீட்டை உருவாக்குகிறது. புதிய அம்சங்கள் அல்லது சேவைகளை கட்டுவதற்கு இதைப் பயன்படுத்தவும்.

```java
String prompt = """
    Generate Java code with production-quality standards: Create an email validation service
    Keep it simple and include basic error handling.
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/ta/self-reflection-cycle.6f71101ca0bd28cc.webp" alt="Self-Reflection Cycle" width="800"/>

*மீண்டும் மேம்படுத்தும் செயற்பாட்டுக் கட்டம் - உருவாக்கு, மதிப்பாய்வு செய், பிரச்சனைகளை கண்டறி, மேம்படு, மீண்டும் செய்*

**கட்டமைக்கப்பட்ட பகுப்பாய்வு** - ஒழுங்கான மதிப்பீடு. மாடல் ஒரு நிர்ணய செய்யப்பட்ட கட்டமைப்பில் (துல்லியம், நடைமுறைகள், செயல்திறன், பாதுகாப்பு, பராமரிப்ப்திறன்) குறியீட்டை மதிப்பாய்வு செய்கிறது. குறியீடு மதிப்பீடுகளுக்கோ தரம் ஆய்வுகளுக்கோ இதைப் பயன்படுத்தவும்.

```java
String prompt = """
    <analysis_framework>
    You are an expert code reviewer. Analyze the code for:
    
    1. Correctness
       - Does it work as intended?
       - Are there logical errors?
    
    2. Best Practices
       - Follows language conventions?
       - Appropriate design patterns?
    
    3. Performance
       - Any inefficiencies?
       - Scalability concerns?
    
    4. Security
       - Potential vulnerabilities?
       - Input validation?
    
    5. Maintainability
       - Code clarity?
       - Documentation?
    
    <output_format>
    Provide your analysis in this structure:
    - Summary: One-sentence overall assessment
    - Strengths: 2-3 positive points
    - Issues: List any problems found with severity (High/Medium/Low)
    - Recommendations: Specific improvements
    </output_format>
    </analysis_framework>
    
    Code to analyze:
    ```
    public List getUsers() {
        return database.query("SELECT * FROM users");
    }
    ```
    Provide your structured analysis:
    """;

String response = chatModel.chat(prompt);
```

> **🤖 [GitHub Copilot](https://github.com/features/copilot) உரையாடலுடன் முயற்சி செய்யவும்:** கட்டமைக்கப்பட்ட பகுப்பாய்வில் கேளுங்கள்:
> - "வேறு வகையான குறியீடு மதிப்பீடுகளுக்கான பகுப்பாய்வு கட்டமைப்பை எப்படி தனிப்பயனாக்கலாம்?"
> - "கட்டமைக்கப்பட்ட வெளியீட்டை நிரலை பெருக்கி செயலாற்றும் சிறந்த வழி என்ன?"
> - "வேறு மதிப்பீட்டு அமர்வுகளில் ஒரே தீவிரத்துவ நிலைகளை எப்படி உறுதி செய்வது?"

<img src="../../../translated_images/ta/structured-analysis-pattern.0af3b690b60cf2d6.webp" alt="Structured Analysis Pattern" width="800"/>

*தீவிரம் நிலைகளுடன் ஒழுங்கான குறியீடு மதிப்பீட்டுக் கட்டமைப்பு*

**பன்முறை உரையாடல்** - சூழலை தேவைப்படுத்தும் உரையாடல்கள். மாடல் முன் செய்திகளை நினைவில் வைக்கிறது மற்றும் அவற்றை மேம்படுத்தி கட்டமைக்கிறது. இடைமுக உதவித்_sessions_க்கோ சிக்கலான கேள்வி-பதில்களுக்கு இதைப் பயன்படுத்தவும்.

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("What is Spring Boot?"));
AiMessage aiMessage1 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage1);

memory.add(UserMessage.from("Show me an example"));
AiMessage aiMessage2 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage2);
```

<img src="../../../translated_images/ta/context-memory.dff30ad9fa78832a.webp" alt="Context Memory" width="800"/>

*பன்முறை உரையாடல் சூழல் தொகுக்கப்படுவது பல முறைகள் வரை, டோக்கன் வரம்புக்கு வரும் வரை*

**படி படியாகக் காரணமறிதல்** - வெளிப்படையான தர்க்கம் தேவைப்படும் பிரச்சனைகள். மாடல் ஒவ்வொரு படிக்கும் தெளிவான காரணித்தலை காட்டுகிறது. கணிதப் பிரச்சனைகள், தர்க் புதிர்கள் அல்லது சிந்தனைக் கட்டமைப்பை புரிந்துகொள்ள வேண்டியபோது இதைப் பயன்படுத்தவும்.

```java
String prompt = """
    <instruction>Show your reasoning step-by-step</instruction>
    
    If a train travels 120 km in 2 hours, then stops for 30 minutes,
    then travels another 90 km in 1.5 hours, what is the average speed
    for the entire journey including the stop?
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/ta/step-by-step-pattern.a99ea4ca1c48578c.webp" alt="Step-by-Step Pattern" width="800"/>

*பிரச்சனைகளை தெளிவான தர்க்க படிகளாக உடைத்தல்*

**கட்டுப்பட்ட வெளியீடு** - குறிப்பிட்ட வடிவ வேண்டுகோள்களுக்கான பதில்கள். மாடல் வடிவமும் நீளமும் கொள்ளும் விதிகள் கடைபிடிக்கிறது. சுருக்கங்களுக்கோ அல்லது துல்லியமான வெளியீடு கட்டமைப்பு தேவைப்படும் சூழலில் இதைப் பயன்படுத்தவும்.

```java
String prompt = """
    <constraints>
    - Exactly 100 words
    - Bullet point format
    - Technical terms only
    </constraints>
    
    Summarize the key concepts of machine learning.
    """;

String response = chatModel.chat(prompt);
```

## முறைகளை ஆராய்தல்

இணைய இடைமுகம் உங்களுக்கு வேறுபட்ட prompting நிபுணத்துவங்களை முயற்சிக்க அனுமதிக்கிறது. ஒவ்வொரு உள்ளமைவும் வேறுபட்ட சிக்கல்களைத் தீர்க்கிறது - அவற்றை முயற்சித்து ஒவ்வொரு முறையிலும் எப்போது சிறந்தது என்பதை காணவும்.

> **குறிப்பு: ஸ்ட்ரீமிங் மற்றும் இல்லாத ஸ்ட்ரீமிங்** — ஒவ்வொரு முறை பக்கமும் இரண்டு பொத்தான்களை வழங்குகிறது: **🔴 Stream Response (Live)** மற்றும் ஒரு **Non-streaming** விருப்பம். ஸ்ட்ரீமிங் Server-Sent Events (SSE) பயன்படுத்தி, மாதிரி உருவாக்கும் போது டோக்கன்களை நேரடியாகக் காண்பிக்க உதவுகிறது, எனவே நீங்கள் முன்னேற்றத்தை உடனடியாக காணலாம். இல்லாத ஸ்ட்ரீமிங் விருப்பம் முழு பதிலை காத்திருந்து பார்வையிடும். குழப்பமான புரிந்துணர்வு தேவைப்படும் prompting களுக்கான (எ.கா., High Eagerness, Self-Reflecting Code), இல்லாத ஸ்ட்ரீமிங் அழைப்பு கடினமாகவும் நீண்ட நேரம் (சில நேரங்களில் நிமிடங்கள்) எடுத்து, எந்தவொரு விசுவல் பின்னூட்டமும் இல்லாமல் இருக்கும். **பயிர்ச்சியில் ஸ்ட்ரீமிங் உள்ளடக்கம் பயன்படுத்தவும்** இவ்வாறு, மாதிரியின் இயக்கத்தைக் காணலாம் மற்றும் வேண்டாமென தோன்றும் நேரம் முடிந்ததாக உள்ளார்ந்த பொத்தானைக் தவிர்க்கலாம்.
>
> **குறிப்பு: பிரௌசர் தேவையானது** — ஸ்ட்ரீமிங் செயல்பாடு Fetch Streams API (`response.body.getReader()`) பயன்படுத்துகிறது, இது முழுமையான பிரௌசரை (Chrome, Edge, Firefox, Safari) தேவைப்படுகிறது. இது VS Code இன் உட்பொதிக்கப்பட்ட Simple Browser-இல் வேலை செய்யாது, ஏனெனில் அதன் webview ReadableStream API-ஐ ஆதரிக்காது. நீங்கள் Simple Browser பயன்பாட்டை பயன்படுத்தினால், இல்லாத ஸ்ட்ரீமிங் பொத்தான்கள் வழக்கமாய் வேலை செய்யும் — ஸ்ட்ரீமிங் பொத்தான்கள் மட்டுமே பாதிக்கப்படும். முழு அனுபவத்துக்கு `http://localhost:8083` ஐ வெளிப்புற பிரௌசரில் திறக்கவும்.

### குறைந்த Eagerness மற்றும் அதிக Eagerness

"200 இன் 15% என்ன?" போன்ற ஒரு எளிய கேள்வியை குறைந்த Eagerness உடன் கேளுங்கள். நீங்கள் உடனடி, நேரடி பதிலை பெறுவீர்கள். இப்போது "ஒரு உயர்-பொழுது API க்கான காட்சிங் யோசனையை உருவாக்கு" போன்ற சிக்கலான கேள்வியை அதிக Eagerness உடன் கேளுங்கள். **🔴 Stream Response (Live)** ஐ கிளிக் செய்து மாதிரியின் விரிவான காரணீசியை டோக்கன் மூலம் தோற்றமளிப்பதை பாருங்கள். அதே மாதிரி, அதே கேள்வி அமைப்பு - ஆனால் prompting அதற்கு எவ்வளவு எண்ணம் செய்ய வேண்டும் என்பதைக் கூறுகிறது.

### பணிமுறை நடைமுறை (Tool Preambles)

பன்மடங்கான வேலைநிலைகள் முன்னோட்டு திட்டமிடலும் முன்னேற்றக் கதைக்களங்களும் மூலம் பயனுள்ளதாக இருக்கும். மாதிரி செய்வதைக் கொடுத்தல், ஒவ்வொரு படியும் கூறுதல், பிறகு முடிவுகளை சுருக்கி கூறுதல்.

### சுயபரிசீலனை குறியீடு

"ஒரு மின்னஞ்சல் சேவை சரிபார்ப்பு உருவாக்கவும்" என்று முயற்சியுங்கள். மட்டும் குறியீடு உருவாக்குவதற்கும் நிறுத்துவதற்கும் பதிலாக, மாதிரி உருவாக்கி, தரத்துடன் ஒப்பிடும், குறைகள் கண்டுபிடித்து மேம்படுத்தும். குறியீடு உற்பத்தி தரத்துடன் பொருந்தும் வரை அதை பரிசீலனை செய்து மேம்படுத்தும்.

### கட்டமைக்கப்பட்ட பகுப்பாய்வு

குறியீடு விமர்சனங்களுக்கு நிலையான மதிப்பீட்டு வடிவமைப்புகள் தேவை. மாதிரி குறியீட்டை நிலையான பிரிவுகளுக்கு (தகவல் சரியானது, நடைமுறைகள், செயல்திறன், பாதுகாப்பு) மற்றும் தீவிர முறைபாடுகளுடன் பகுப்பாய்வு செய்கிறது.

### பல தடவை உரையாடல்

"Spring Boot என்ன?" என்று கேட்கவும், பின்னர் உடனடியாக "ஒரு உதாரணத்தை காண்பி" என்று தொடரவும். மாதிரி உங்கள் முதல் கேள்வியை நினைவில் வைத்துக் கொண்டு Spring Boot உதாரணத்தை வழங்கும். நினைவில் இல்லாவிட்டால், இரண்டாவது கேள்வி மிக தெளிவற்றதாக இருக்கும்.

### படி படியாக காரணித்தல்

ஒரு கணிதப் பிரச்சினையை எடுத்துக்கொண்டு அதை Step-by-Step Reasoning மற்றும் குறைந்த Eagerness உடன் முயற்சிக்கவும். குறைந்த Eagerness விரைவில் பதிலை தரும் ஆனால் புரிதலில் தாடவில்லை. படி படியாக காரணித்தல் ஒவ்வொரு கணக்கையும் முடிவையும் காட்டும்.

### கட்டுப்படுத்தப்பட்ட வெளியீடு

நீங்கள் குறிப்பிட்ட வடிவங்களோ அல்லது சொல் எண்ணிக்கையோ தேவைப்படும்போது, இந்த முறை கடுமையான ஒத்துழைப்பை நடைமுறைப்படுத்துகிறது. 100 சொற்கள் கொண்ட சுருக்கத்தை புள்ளிவிவர வடிவத்தில் உருவாக்க முயற்சிக்கவும்.

## நீங்கள் உண்மையில் கற்றுக் கொண்டிருக்கும் விஷயம்

**காரணித்தல் முயற்சி அனைத்தையும் மாற்றுகிறது**

GPT-5.2 உங்கள் prompting மூலம் கணினி முயற்சியை ஒழுங்குப்படுத்த அனுமதிக்கிறது. குறைந்த முயற்சி விரைவான பதில்கள் மற்றும் குறைந்த ஆராய்ச்சியைக் குறிக்கிறது. அதிக முயற்சி என்பது மாதிரி உள்ளடக்கமாக ஆழமாக யோசிக்க நேரம் தருகின்றது. நீங்கள் முயற்சியை பணி சிக்கலுக்கு ஏற்ப ஒத்துப்போக கற்றுக்கொள்கிறீர்கள் - எளிய கேள்விகளுக்கு நேரத்தை வீணாக்க வேண்டாம், ஆனால் சிக்கலான முடிவுகளுக்கு விரைவாக சென்று விட வேண்டாம்.

**கட்டமைப்பு நடத்தையை வழிநடத்துகிறது**

prompt களில் XML குறிச்சொற்களை கவனித்தீர்களா? அவை அலங்காரமல்ல. மாதிரிகள் கட்டமைக்கப்பட்ட வழிகாட்டுதல்களை சுதந்திர உரை விட நம்பகமாக பின்பற்றுகின்றன. பன்மடங்கு செயல்பாடுகள் அல்லது சிக்கலான தார்க்கங்களை நீங்கள் தேவையான போது, கட்டமைப்பு மாதிரிக்கு எங்கே இருக்கிறது மற்றும் அடுத்து என்ன என்பதை கண்காணிக்க உதவுகிறது.

<img src="../../../translated_images/ta/prompt-structure.a77763d63f4e2f89.webp" alt="Prompt Structure" width="800"/>

*துண்டு பகுதி மற்றும் XML-போன்ற அமைப்புடன் நன்றாக கட்டமைக்கப்பட்ட promptன் அமைப்பு*

**தரத்தை சுய மதிப்பீட்டின் மூலம் உயர்த்துதல்**

சுயபரிசீலனை முறைகள் தரத்திற்கான கொள்கைகளை தெளிவாகக் குறிப்பிடுவதன் மூலம் இயங்கு படுத்தப்படுகின்றன. மாதிரிக்கு "சரி செய்வதற்கு" நம்பிக்கை வைக்காமல், நீங்கள் அதற்கான சரியான பொருளைக் கூறுகிறீர்கள்: சரியான தர்க்கம், பிழை கையாளல், செயல்திறன், பாதுகாப்பு. மாதிரி தனது வெளியீட்டை மதிப்பீடு செய்து மேம்படுத்த முடியும். இது குறியீடு உருவாக்கத்தை ஒரு சூட்டையிலிருந்து செயல்முறையாக மாற்றுகிறது.

**சூழல் வரம்பு உள்ளது**

பல தடவைகள் உரையாடல்கள் ஒவ்வொரு கோரிக்கையுடனும் செய்தி வரலாற்றை சேர்க்கின்றன. ஆனால் ஒரு வரம்பு உள்ளது - ஒவ்வொரு மாதிரியும் அதிகபட்ச டோக்கன் எண்ணிக்கையைக் கொண்டுள்ளது. உரையாடல்கள் விரிவடைவதில், நீங்கள் தொடர்புடைய சூழலைத் தக்கவைத்து அந்த வரம்பை கடக்காமல் வைத்திருக்க முறைகளைப் பயன்படுத்த வேண்டும். இந்த மொடியூல் நினைவகம் எப்படி செயல்படுகின்றது என்பதைக் காட்டுகிறது; பிறகு எப்போது சுருக்குவது, எப்போது மறக்கவேண்டும், எப்போது மீட்டெடுக்கவேண்டும் என்று கற்றுக்கொள்ளலாம்.
