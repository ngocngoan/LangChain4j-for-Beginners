## What You'll Learn

<img src="../../../translated_images/ta/what-youll-learn.c68269ac048503b2.webp" alt="நீங்கள் கற்றுக்கொள்ளப்போகும்" width="800"/>

முந்தைய மாட்யூலில், நினைவகம் எப்படி உரையாடல் ஏ.ஐ க்கு உதவுகிறது என்பதைக் காண்போம் மற்றும் அடிவரிசைப் பணிகளுக்கு GitHub மாதிரிகளை பயன்படுத்தினோம். இப்போது நீங்கள் கேட்கும் கேள்விகள் — ப்ராம்ட்கள் என்றதே — எப்படி வடிவமைக்கப்படுகின்றன என்பதை கவனம் செலுத்துகிறோம், Azure OpenAI இன் GPT-5.2 ஐ பயன்படுத்தி. உங்கள் ப்ராம்ட்களை வடிவமைக்கும் முறை உங்கள் பெறும் பதில்களின் தரத்தை மிகுந்த வகையில் பாதிக்கும். நாங்கள் முதலில் அடிப்படை ப்ராம்டிங் முறைகளை விமர்சனம் செய்து, பின்னர் GPT-5.2 இன் முழு திறன்களை பயன்படுத்தும் எட்டு மேம்பட்ட வடிவங்களுக்கு செல்லப்போகின்றோம்.

GPT-5.2 ஐ பயன்படுத்துவோம் ஏனெனில் அது அறிந்துறையை கட்டுப்படுத்தும் திறனை அறிமுகப்படுத்துகிறது - பதில் அளிப்பதற்கு முன்னர் எவ்வளவு சிந்திக்க வேண்டும் என்பதை நீங்கள் மாதிரிக்குத் தெரிவிக்கலாம். இது ப்ராம்டிங் ռազմவீதிகளைப் பயனுள்ளதாக மாற்றி, எந்த அணுகுமுறையை எப்போது பயன்படுத்த வேண்டும் என்பதை புரிந்து கொள்ள உதவும். GitHub மாதிரிகளுடன் ஒப்பிடுகையில் Azure இல் GPT-5.2 க்கான குறைந்த கட்டுப்பாடுகள் நமக்கு கூடுதல் நன்மையைக் கொடுக்கும்.

## Understanding Prompt Engineering

<img src="../../../translated_images/ta/what-is-prompt-engineering.5c392a228a1f5823.webp" alt="ப்ராம்ட் பொறியியல்란 என்ன?" width="800"/>

ப்ராம்ட் பொறியியல் என்பது எப்போதும் உங்கள் தேவையான முடிவுகளை தரும் நுழைவுச் சொற்களை வடிவமைக்கிறது. இது வெறும் கேள்விகள் கேட்குதல்ல - மாதிரி உங்கள் கேள்விகளே என்ன மற்றும் எப்படி வழங்கப்பட வேண்டும் என்பதை தெளிவாக புரிந்து, உரிய வடிவமைப்பில் கேட்கும் பணி.

இதைக் கொள்ளுங்கள் ஒரு சக ஊழியருக்கு அடி எடுக்கும் வழிகாட்டல் போல். "பிழையை சரி செய்" என்பது பொதுவானது. "UserService.java மொழியில் 45வது வரியில் உள்ள null pointer exception ஐ null சரிபார்ப்புடன் சரி செய்" என்றால் அது தெளிவானது. மொழி மாதிரிகள் அதே விதத்தில் செயல்படுகின்றன - தெளிவுத்தன்மை மற்றும் கட்டமைப்பு முக்கியம்.

<img src="../../../translated_images/ta/how-langchain4j-fits.dfff4b0aa5f7812d.webp" alt="LangChain4j எப்படி பொருந்துகிறது" width="800"/>

LangChain4j அமைப்புகளையும் — மாதிரி இணைப்புகள், நினைவு, செய்தி வகைகள் — வழங்குகிறது, ஆனால் ப்ராம்ட் மாதிரிகள் அந்த அமைப்புகளால் அனுப்பப்படும் மிக கவனமாக வடிவமைக்கப்பட்ட உரைகளே ஆகும். முக்கிய கட்டுமானமானவை `SystemMessage` (இது ஏ.ஐ நடத்தை மற்றும் பாத்திரத்தை அமைக்கும்) மற்றும் `UserMessage` (உங்கள் உண்மையான கோரிக்கையை கொண்டு வரும்).

## Prompt Engineering Fundamentals

<img src="../../../translated_images/ta/five-patterns-overview.160f35045ffd2a94.webp" alt="ஐந்து ப்ராம்ட் பொறியியல் வடிவங்களை விரிவாக்கம்" width="800"/>

இந்த மாட்யூலில் மேம்பட்ட வடிவங்களுக்கு முன், ஐந்து அடிப்படை ப்ராம்டிங் முறைகளை பார்ப்போம். இவை அனைத்து ப்ராம்ட் பொறியாளர் அறிந்திருக்க வேண்டிய கட்டுமானங்களாகும். நீங்கள் [ஆரம்பிக் கையேடு](../00-quick-start/README.md#2-prompt-patterns) மூலம் ஏற்கனவே பார்த்திருப்பீர்கள் — இதோ அவற்றின் கருத்துருவாக்க அமைப்பு.

### Zero-Shot Prompting

எளிமையான முறையை பயன்படுத்துங்கள்: மாதிரிக்கு எடுத்துக்காட்டுகள் இல்லாமல் நேரடியாக அடி கொடுங்கள். மாதிரி முழுமையாக பயிற்சியின் அடிப்படையில் எந்த உதவியும் இல்லாமல் பணியை புரிந்து செயற்கொள்கிறது. இது எளிமையான கோரிக்கைகளுக்கு சிறப்பானது.

<img src="../../../translated_images/ta/zero-shot-prompting.7abc24228be84e6c.webp" alt="சுழியில்லா ப்ராம்டிங்" width="800"/>

*எடுத்துக்காட்டுகள் இல்லாமல் நேரடி அறிவுறுத்தல் — மாதிரி அறிவுறுத்தல் மட்டுமே கொண்டு பணியை கற்பது*

```java
String prompt = "Classify this sentiment: 'I absolutely loved the movie!'";
String response = model.chat(prompt);
// பதில்: "நேர்மறை"
```

**எப்போது பயன்படுத்துவது:** எளிய வகைப்படுத்தல்கள், நேரடி கேள்விகள், மொழிபெயர்ப்புகள் அல்லது கூடுதல் வழிகாட்டல் தேவையில்லாத பணிகள்.

### Few-Shot Prompting

மாதிரிக்க நீங்கள் விரும்பும் வடிவமைப்பை காட்டும் எடுத்துக்காட்டுகளை கொடுங்கள். மாதிரி உங்கள் எடுத்துக்காட்டுகளிலிருந்து நுழைவு மற்றும் வெளியீடு வடிவத்தை கற்றுக்கொண்டு புதிய உள்ளீடுகளுக்கு அதை பொருத்துகிறது. இது அழுத்தமான மற்றும் தொடர்ச்சியான பணிகளுக்கு சிறப்பானது.

<img src="../../../translated_images/ta/few-shot-prompting.9d9eace1da88989a.webp" alt="சிறிய உதாரண ப்ராம்டிங்" width="800"/>

*எடுத்துக்காட்டுகளிலிருந்து கற்றல் — மாதிரி வடிவத்தை கண்டறிந்து புதிய உள்ளீடுகளுக்கு பயன்படுத்துகிறது*

```java
String prompt = """
    Classify the sentiment as positive, negative, or neutral.
    
    Examples:
    Text: "This product exceeded my expectations!" → Positive
    Text: "It's okay, nothing special." → Neutral
    Text: "Waste of money, very disappointed." → Negative
    
    Now classify this:
    Text: "Best purchase I've made all year!"
    """;
String response = model.chat(prompt);
```

**எப்போது பயன்படுத்துவது:** தனிப்பயன் வகைப்படுத்தல்கள், தொடர் வடிவமைப்பு, துறையின் பணிகள் அல்லது சுழி இல்லாத முடிவுகள் கடுமையாக இருக்கும்போது.

### Chain of Thought

மாதிரியிடம் அதன் சிந்தனை செயல்முறையை படிநிலைபடுத்திக் காட்டும்படி கேளுங்கள். பதில் நேரடியாக வழங்காமல், மாதிரி பிரச்சனையை விரிவாக உடைத்துக் கொண்டு, ஒவ்வொரு பகுதியையும் தெளிவாக செயல்படுத்துகிறது. இது கணக்கு, தர்க்கம், மற்றும் பல அடுக்கு சிந்தனை பணிகளுக்கு மெய்யான முடிவை தரும்.

<img src="../../../translated_images/ta/chain-of-thought.5cff6630e2657e2a.webp" alt="சிந்தனை தொடர் ப்ராம்டிங்" width="800"/>

*அடியடியாக சிந்தனை – சிக்கலான பிரச்சனைகளை தெளிவான தர்க்கமான படிகள் ஆக உடைத்தல்*

```java
String prompt = """
    Problem: A store has 15 apples. They sell 8 apples and then 
    receive a shipment of 12 more apples. How many apples do they have now?
    
    Let's solve this step-by-step:
    """;
String response = model.chat(prompt);
// மாடல் காட்டுகிறது: 15 - 8 = 7, பின்னர் 7 + 12 = 19 ஆப்பிள்கள்
```

**எப்போது பயன்படுத்துவது:** கணக்கு பிரச்சனைகள், தர்க்க புதிர்கள், பிழைத்திருத்தம், அல்லது சிந்தனை செயல்முறை காட்டுதல் மூலம் நம்பகத்தன்மை உயர்த்தும் பணிகள்.

### Role-Based Prompting

உங்கள் கேள்விக்கு முன் ஏ.ஐக்குச் பாத்திரம் அல்லது வேடம் அமைக்கவும். இது பதிலின் தடம், ஆழம், மற்றும் கவனத்தை வடிவமைக்க உதவும். "மென்பொருள் சுபிட்ச நிபுணர்" மற்றும் "சிறுதொழிலாளர்" அல்லது "பாதுகாப்பு ஆய்வாளர்" என்பவர்கள் வழங்கும் ஆலோசனைகள் வேறுபடும்.

<img src="../../../translated_images/ta/role-based-prompting.a806e1a73de6e3a4.webp" alt="பாத்திரம் அடிப்படையிலான ப்ராம்டிங்" width="800"/>

*கட்டமைப்பும் பாத்திரமும் அமைத்தல் — பயன்பாட்டின் அடிப்படையில் பதில் மாறுபடும்*

```java
String prompt = """
    You are an experienced software architect reviewing code.
    Provide a brief code review for this function:
    
    def calculate_total(items):
        total = 0
        for item in items:
            total = total + item['price']
        return total
    """;
String response = model.chat(prompt);
```

**எப்போது பயன்படுத்துவது:** குறியீடு ஆய்வு, பயிற்சி, துறையை சார்ந்த பகுப்பாய்வு அல்லது ஒரு குறிப்பிட்ட திறன் மற்றும் பார்வைக்கான பதில்கள் தேவையான போது.

### Prompt Templates

மாறிலிகள் உள்ள மறுபயன்பாட்டுக்கூடிய ப்ராம்ட்களை உருவாக்குங்கள். ஒவ்வொரு முறையும் புதிய ப்ராம்ட் எழுதாமல், ஒரே மாதிரியை ஒருமுறை நிரூபித்து பல மாறில்களைப் பயன்படுத்தவும். LangChain4j இன் `PromptTemplate` வகுப்பு `{{variable}}` சிந்தர விவரத்துடன் இதைப் பதிவு செய்கிறது.

<img src="../../../translated_images/ta/prompt-templates.14bfc37d45f1a933.webp" alt="ப்ராம்ட் வடிவங்கள்" width="800"/>

*மாறிலிகளுடன் மறுபயன்பாடான ப்ராம்ட்கள் — ஒரு மாதிரி, பல பயன்பாடுகள்*

```java
PromptTemplate template = PromptTemplate.from(
    "What's the best time to visit {{destination}} for {{activity}}?"
);

Prompt prompt = template.apply(Map.of(
    "destination", "Paris",
    "activity", "sightseeing"
));

String response = model.chat(prompt.text());
```

**எப்போது பயன்படுத்துவது:** பல வேறு உள்ளீடுகளுடன் மீண்டும் மீண்டும் கேள்விகள், தொகுப்பு செயல்பாடு, மறுபயன்பாடான ஏஐ பணிகள் அல்லது ப்ராம்ட் அமைப்பு ஒரே மாதிரியாக இருந்தாலும் தரவு மாறும் சந்தர்ப்பங்கள்.

---

இந்த ஐந்து அடிப்படைகள் பெரும்பாலான ப்ராம்டிங் பணிகளுக்கு உறுதியான கருவி தொகுப்பு தரும். இந்த மாட்யூல் அவற்றின் அடிப்படையில் **எட்டு மேம்பட்ட வடிவங்களை** உருவாக்குகிறது, அவை GPT-5.2 இன் சிந்தனை கட்டுப்பாடு, சுய மதிப்பீடு மற்றும் கட்டமைக்கப்பட்ட வெளியீடு திறன்களைப் பயன்படுத்துகின்றன.

## Advanced Patterns

அடிப்படைகள் முடிந்ததும், இந்த மாட்யூலை தனிப்பட்டதாகும் எட்டு மேம்பட்ட வடிவங்களுக்கு நுட்பமாக செல்லுங்கள். அனைத்து பிரச்சனைகளுக்கும் ஒரே அணுகுமுறை தேவையில்லை. சில கேள்விகள் விரைவான பதில்களைத் வேண்டும், சில ஆழமான சிந்தனையை. சில கேள்விகளுக்கு சிந்தனை காட்சியளிக்க வேண்டும், சில பதில்களை மட்டும். கீழ்காணும் ஒவ்வொரு வடிவமும் வேறுபட்ட சூழலுக்கு ஏற்ப பொருந்தும் — GPT-5.2 இன் சிந்தனை கட்டுப்பாடு வித்தியாசங்களை மேலும் தெளிவாக்குகிறது.

<img src="../../../translated_images/ta/eight-patterns.fa1ebfdf16f71e9a.webp" alt="எட்டு ப்ராம்ட் பொறியியல் வடிவங்கள்" width="800"/>

*எட்டு ப்ராம்ட் பொறியியல் வடிவங்களும் அவற்றின் பயன்பாடுகளும்*

<img src="../../../translated_images/ta/reasoning-control.5cf85f0fc1d0c1f3.webp" alt="GPT-5.2 உடன் சிந்தனை கட்டுப்பாடு" width="800"/>

*GPT-5.2 இன் சிந்தனை கட்டுப்பாடு மாதிரிக்கு எவ்வளவு சிந்திக்கவேண்டும் என்று குறிப்பது — விரைவான நேரடி பதில்களிலிருந்து ஆழமான ஆராய்ச்சிவரை*

**குறைந்த ஆர்வம் (விரைவு மற்றும் கவனம்செய்தல்)** - எளிய கேள்விகளுக்கு விரைவான நேரடி பதில்களுக்காக. மாதிரி அதிக சிந்தனை செய்யாது - ஆக максимம் 2 படிகள். கணக்கீடுகள், தேடல்கள், அல்லது எளிய கேள்விகளுக்கு இது உகந்தது.

```java
String prompt = """
    <context_gathering>
    - Search depth: very low
    - Bias strongly towards providing a correct answer as quickly as possible
    - Usually, this means an absolute maximum of 2 reasoning steps
    - If you think you need more time, state what you know and what's uncertain
    </context_gathering>
    
    Problem: What is 15% of 200?
    
    Provide your answer:
    """;

String response = chatModel.chat(prompt);
```

> 💡 **GitHub Copilot உடன் ஆராயவும்:** [`Gpt5PromptService.java`](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java) திறந்துவைத்து கேளுங்கள்:
> - "குறைந்த ஆர்வம் மற்றும் அதிக ஆர்வம் ப்ராம்ட் வடிவ மாறுபாடுகளில் என்ன வேறுபாடு?"
> - "ப்ராம்ட்களில் XML குறிச்சொற்கள் ஏன் AI பதிலை கட்டமைக்க உதவுகிறது?"
> - "சுய மதிப்பீடு வடிவங்களை நேரடி அறிவுறுத்தல்களுடன் எப்போது பயன்படுத்த வேண்டும்?"

**அதிக ஆர்வம் (ஆழமான மற்றும் விரிவான)** - சிக்கலான பிரச்சனைகளுக்கு முழுமையான பகுப்பாய்வை நீங்கள் விரும்பும்போது. மாதிரி விரிவாக ஆராய்ச்சி மேற்கொண்டு தெளிவான சிந்தனையை காட்டு. சிஸ்டம் வடிவமைப்பு, கட்டமைப்பு முடிவுகள், அல்லது சிக்கலான ஆய்வுக்கு இது சரியானது.

```java
String prompt = """
    Analyze this problem thoroughly and provide a comprehensive solution.
    Consider multiple approaches, trade-offs, and important details.
    Show your analysis and reasoning in your response.
    
    Problem: Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```

**பணி அமல்படுத்தல் (படியடியாக முன்னேற்றம்)** - பல படி பணிகளுக்கு. மாதிரி முன்கூட்டிய திட்டம் தரும், ஒவ்வொரு படியையும் செயல்படுத்தும் போது விளக்குகிறது, பின்னர் சுருக்கம் அளிக்கிறது. மைக்ரேஷன்கள், செயலாக்கம் அல்லது ஏதும் பல படி செயல்முறைக்கு இது உபயோகமாக இருக்கும்.

```java
String prompt = """
    <task_execution>
    1. First, briefly restate the user's goal in a friendly way
    
    2. Create a step-by-step plan:
       - List all steps needed
       - Identify potential challenges
       - Outline success criteria
    
    3. Execute each step:
       - Narrate what you're doing
       - Show progress clearly
       - Handle any issues that arise
    
    4. Summarize:
       - What was completed
       - Any important notes
       - Next steps if applicable
    </task_execution>
    
    <tool_preambles>
    - Always begin by rephrasing the user's goal clearly
    - Outline your plan before executing
    - Narrate each step as you go
    - Finish with a distinct summary
    </tool_preambles>
    
    Task: Create a REST endpoint for user registration
    
    Begin execution:
    """;

String response = chatModel.chat(prompt);
```

சிந்தனை தொடர் ப்ராம்டிங் மாதிரியை அதன் சிந்தனை செயல்முறையை தெளிவாக காட்டச் செய்வதாகும், இது சிக்கலான பணிகளுக்கு துல்லியத்தை மேம்படுத்தும். படி படியாக உடைத்தல் மனிதர்களுக்கும் ஏ.ஐக்கும் தர்க்கத்தை புரிந்துகொள்ள உதவும்.

> **🤖 [GitHub Copilot](https://github.com/features/copilot) அரட்டை மூலம் முயற்சி செய்க:** இந்த வடிவத்தை பற்றி கேளுங்கள்:
> - "நீண்டகால செயல்பாடுகளுக்கு பணி அமல்படுத்தல் வடிவத்தை எப்படி மாற்றுவது?"
> - "உற்பத்தி பயன்பாடுகளில் கருவி முன்மொழிவுகளை எப்படி அமைக்க வேண்டும்?"
> - "UI ல் இடைநிலை முன்னேற்றங்களைக் எப்படி பதிவு செய்து காட்டுவது?"

<img src="../../../translated_images/ta/task-execution-pattern.9da3967750ab5c1e.webp" alt="பணி அமல்படுத்தல் வடிவம்" width="800"/>

*திட்டமிடு → செயல்படு → சுருக்கம் போன்று பல படி பணிக்கான வேலைநடை*

**சுய மதிப்பீடு குறியீடு** - உற்பத்தி தரத்துடன் குறியீடு உருவாக்க. மாதிரி உற்பத்தி தரநிலைகளுக்கு ஏற்ப குறியீட்டைப் பதிந்துசெய்கிறது, தவறுகளுக்கு சரியான கையாளல் உடன். புதிய அம்சங்கள் அல்லது சேவைகளுக்காக இது பயன்படும்.

```java
String prompt = """
    Generate Java code with production-quality standards: Create an email validation service
    Keep it simple and include basic error handling.
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/ta/self-reflection-cycle.6f71101ca0bd28cc.webp" alt="சுய-மதிப்பீட்டுச் சுற்று" width="800"/>

*தொடர்ந்து மேம்படுத்தும் சுற்று - உருவாக்கு, மதிப்பாய்வு செய், பிரச்சனைகளை கண்டறி, மேம்படுத்து, மீண்டும் செய்*

**விருத்தமான பகுப்பாய்வு** - தொடர்ச்சியான மதிப்பீட்டிற்கு. மாதிரி குறியீட்டை ஒரு நிலையான வடிவமைப்பில் (தீர்மானித்தல், நடைமுறைகள், செயல்திறன், பாதுகாப்பு, பராமரிப்பு) மதிப்பீடு செய்கிறது. குறியீடு ஆய்வுகள் அல்லது தர விசாரணைகளுக்கு இது பொருந்தும்.

```java
String prompt = """
    <analysis_framework>
    You are an expert code reviewer. Analyze the code for:
    
    1. Correctness
       - Does it work as intended?
       - Are there logical errors?
    
    2. Best Practices
       - Follows language conventions?
       - Appropriate design patterns?
    
    3. Performance
       - Any inefficiencies?
       - Scalability concerns?
    
    4. Security
       - Potential vulnerabilities?
       - Input validation?
    
    5. Maintainability
       - Code clarity?
       - Documentation?
    
    <output_format>
    Provide your analysis in this structure:
    - Summary: One-sentence overall assessment
    - Strengths: 2-3 positive points
    - Issues: List any problems found with severity (High/Medium/Low)
    - Recommendations: Specific improvements
    </output_format>
    </analysis_framework>
    
    Code to analyze:
    ```
    public List getUsers() {
        return database.query("SELECT * FROM users");
    }
    ```
    Provide your structured analysis:
    """;

String response = chatModel.chat(prompt);
```

> **🤖 [GitHub Copilot](https://github.com/features/copilot) அரட்டை மூலம் முயற்சி செய்க:** விருத்தமான பகுப்பாய்வு பற்றி கேளுங்கள்:
> - "விதவிதமான குறியீடு ஆய்வுகளுக்கு பகுப்பாய்வு வடிவத்தை எப்படி தனிப்பயனாக்குவது?"
> - "கட்டமைக்கப்பட்ட வெளியீட்டை நிரல் மூலம் எவ்வாறு பகுப் பத்திரிகைப்படுத்தி செயல்படுத்துவது?"
> - "வேறுபட்ட ரிவியூ அமர்வுகளில் ஒரே மாதிரியான கடுமை நிலைகளை எப்படிச் சரிபார்ப்பது?"

<img src="../../../translated_images/ta/structured-analysis-pattern.0af3b690b60cf2d6.webp" alt="விருத்தமான பகுப்பாய்வு வடிவம்" width="800"/>

*ஒருங்கிணைந்த நிலையான கடுமை மட்டங்களுடன் குறியீடு ஆய்வுகளுக்கான கட்டமைப்பு*

**பல சுற்றுப் பேச்சு** - சூழல் தேவைப்படும் உரையாடல்களுக்கு. மாதிரி முந்தைய செய்திகளை நினைவில் வைத்து அதில் அடிப்படையாய் பதிற்குழு அமைக்கிறது. இடையூறு உதவி அமர்வுகள் அல்லது சிக்கலான கேள்வி பதிலுக்கு இது பொருந்தும்.

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("What is Spring Boot?"));
AiMessage aiMessage1 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage1);

memory.add(UserMessage.from("Show me an example"));
AiMessage aiMessage2 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage2);
```

<img src="../../../translated_images/ta/context-memory.dff30ad9fa78832a.webp" alt="சூழல் நினைவகம்" width="800"/>

*பல சுற்றுகளுக்குள் பேச்சு சூழல் எவ்வாறு சேர்க்கப்படுகிறது, டோக்கன் வரம்பை அடையும் வரை*

**அடியடியாக சிந்தனை** - தெளிவாக உள்ள தர்க்கம் தேவையான பிரச்சனைகள். மாதிரிக்கு ஒவ்வொரு படியையும் தெளிவாக நிரூபிக்கச் சொல். கணக்கு, தர்க்க புதிர்கள், அல்லது சிந்தனை செயல்முறையைப் புரிந்துகொள்ள வேண்டியபோது இது பயன்படும்.

```java
String prompt = """
```

## மாதிரிகளைக் ஆராய்ச்சி செய்தல்

வலை இடைமுகம் உங்களுக்கு வேறு வேறு prompting கைத்தொழில்களை முயற்சிக்க அனுமதிக்கிறது. ஒவ்வொரு மாதிரியும் வேறு பிரச்சினைகளை தீர்க்கிறது - எப்போது எந்த அணுகுமுறை சிறந்தது என்று பார்க்க முயற்சியுங்கள்.

> **குறிப்பு: ஸ்ட்ரீமிங் மற்றும் ஸ்ட்ரீமிங் அல்லாதவை** — ஒவ்வொரு மாதிரி பக்கமும் இரண்டு பொத்தான்களை வழங்குகிறது: **🔴 Stream Response (Live)** மற்றும் **Non-streaming** விருப்பம். ஸ்ட்ரீமிங் Server-Sent Events (SSE) ஐ பயன்படுத்தி மாடல் உருவாக்கும் முறையில் டோக்கன்களை நேரடியாக காட்டுகிறது, எனவே நீங்கள் முறையாக முன்னேற்றத்தைக் காணலாம். ஸ்ட்ரீமிங் அல்லாத விருப்பம் முழு பதிலை கிடைக்கும் வரை காத்திருக்கும். ஆழமான காரணமளிப்பதற்கு பயனுள்ள prompts (எ.கா., High Eagerness, Self-Reflecting Code) க்கான ஸ்ட்ரீமிங் அல்லாத அழைப்பு மிக நீண்ட நேரம் எடுக்கும் — சில நேரங்களில் நிமிடங்கள் — எந்தத் திடுக்கிடும் பதிலையும் காண்பிக்காமல். **கடினமான prompts-ஐ முயற்சிக்கும்போது ஸ்ட்ரீமிங் பயன்படுத்தவும்** அதன் மூலம் மாடல் எப்படிச் செயல்படுகிறது என்பதை நீங்கள் பார்க்க முடியும், மற்றும் கோரிக்கை காலாவதியாகிவிட்டது என்ற தவறான அலைமோதலை தவிர்க்கலாம்.
>
> **குறிப்பு: உலாவி தேவைகள்** — ஸ்ட்ரீமிங் அம்சம் Fetch Streams API (`response.body.getReader()`) ஐ பயன்படுத்துகிறது, இது முழு உலாவியாகச் செயல்படும் (Chrome, Edge, Firefox, Safari) தேவைப்படுகிறது. VS Code-இன் உள்ளமைவு Simple Browser-ல் இது வேலை செய்யாது, ஏனெனில் அதன் webview ReadableStream API-ஐ ஆதரிக்காது. நீங்கள் Simple Browser பயன்படுத்தினால், ஸ்ட்ரீமிங் அல்லாத பொத்தான்கள் சரியான முறையில் இயங்குகின்றன — ஸ்ட்ரீமிங் பொத்தான்கள் மட்டுமே பாதிக்கப்படுகின்றன. முழு அனுபவத்திற்கு `http://localhost:8083` ஐ வெளி உலாவியில் திறக்கவும்.

### குறைந்த Eagerness மற்றும் அதிக Eagerness

"15% of 200 என்ன?" என ஒரு எளிய கேள்வி குறைந்த Eagerness மேலானது. உடனடி, நேரடி பதிலை பெறுவீர்கள். இப்போது "உயர்-பயனர் API க்கான கேச்சிங் திட்டத்தை வடிவமைக்கவும்" போன்ற கடினமான கேள்வியை அதிக Eagerness கொண்டு கேளுங்கள். **🔴 Stream Response (Live)** கிளிக் செய்து, மாடலின் விரிவான காரணத்தைக்கான டோக்கன்-தோறும் தோற்றம் காணுங்கள். அதே மாடல், அதே கேள்வி ஆனால் prompt எவ்வளவு ஆழமாக சிந்திக்க வேண்டும் என்பதனை கூறுகிறது.

### பணிகள் நிறைவேற்றல் (கருவி பேராம்பிள்கள்)

பன்மடங்கான வேலைச் செயற்பாடுகள் முன்கூட்டியே திட்டமிடல் மற்றும் முன்னேற்ற உரையாடலை ஆதரிக்கின்றன. மாடல் என்ன செய்யப்போகிறது என்பதை விளக்குகிறது, ஒவ்வொரு படியையும் சொல்கிறது, பின்னர் முடிவுகளை சுருக்குகிறது.

### தானாக தன்னை விமர்சிக்கும் கோடு

"ஒரு மின்னஞ்சல் சரிபார்ப்பு சேவையை உருவாக்கவும்" என முயற்சிக்கவும். வெறும் கோடுகளை உருவாக்கி நிறுத்துவதற்கு பதிலாக, மாடல் உருவாக்குகிறது, தரத்துக்கேற்ப மதிப்பீடு செய்கிறது, பலவீனங்களை கண்டறிகிறது மற்றும் மேம்படுத்துகிறது. உற்பத்தி தரங்கள் கிடைக்கும் வரை அது மீண்டும் மீண்டும் முயற்சிப்பதை கூட காணலாம்.

### கட்டமைக்கப்பட்ட பகுப்பாய்வு

கோடு மதிப்பாய்வுகள் ஒத்த கட்டமைப்பு மதிப்பீடு முறைகளை தேவைப்படுத்துகின்றன. மாடல் குறியீட்டை நிலைத்த வகைகளில் (சரியானது, நடைமுறைகள், செயல்திறன், பாதுகாப்பு) மற்றும் தீவிரத்திற்கேற்ப ஆய்வு செய்கிறது.

### பன்மடங்கு உரையாடல்

"Spring Boot என்பது என்ன?" என்று கேட்கவும், உடனடியாக "எனக்கு ஒரு உதாரணம் காண்பி" என்ற கேள்வி கேட்கவும். மாடல் உங்கள் முதல் கேள்வியை நினைவில் வைத்திருக்கும் மேலும் Spring Boot உதாரணத்தை மட்டும் வழங்கும். நினைவில்லாமல் அந்த இரண்டாவது கேள்வி மிகவும் மங்கியதாக இருக்கும்.

### படி படியாக காரணமளிப்பு

ஒரு கணிதப் பிரச்சினையை எடுத்து Step-by-Step Reasoning மற்றும் குறைந்த Eagerness உடன் முயற்சியுங்கள். குறைந்த Eagerness கெட்டுப்படாத பதிலை விரைவில் அளிக்கும். படி படியாக காரணமளிப்பு ஒவ்வொரு கணக்கீடும் முடிவும் காட்டும்.

### கட்டுப்படுத்தப்பட்ட வெளியீடு

குறிப்பிட்ட வடிவங்கள் அல்லது சொல் எண்ணிக்கைகள் தேவைப்பட்டால், இந்த மாதிரி கடுமையான கடமைகளை பின்பற்ற வைக்கிறது. 100 சொற்கள் துல்லியமாக இருக்கும்படி புள்ளிவிவர வடிவத்தில் தகவல் உண்டாக்க முயற்சிக்கவும்.

## நீங்கள் உண்மையில் கற்றுக்கொள்வது என்ன

**காரணமளிப்பில் முயற்சி அனைத்தையும் மாற்றும்**

GPT-5.2 உங்கள் prompting வாயிலாக கணினி முயற்சியை கட்டுப்படுத்த உதவுகிறது. குறைந்த முயற்சி என்பது குறுகிய ஆராய்ச்சியுடன் விரைவான பதில்களை குறிக்கிறது. அதிக முயற்சி என்பது மாடல் ஆழமாக சிந்திப்பதற்கு நேரம் செலவிடுவதை குறிக்கிறது. நீங்கள் பணியின் கடினத்தன்மைக்கு ஏற்ப முயற்சியை பொருத்த கற்றுக்கொள்கிறீர்கள் - எளிய கேள்விகளுக்கு நேரத்தை வீணாக்க வேண்டாம், ஆனால் கடின முடிவுகளை விரைவாக எடுக்கவும் வேண்டாம்.

**உருவமைப்பு நடத்தை வழிநடத்துகிறது**

prompt-களில் உள்ள XML குறிச்சொற்களை கவனிக்கிறீர்களா? அவை அலங்காரமாக இல்லை. மாடல்கள் கட்டமைக்கப்பட்ட வழிமுறைகளை அதிக நம்பகத்தன்மையுடன் பின்பற்றுகின்றன. பன்மடங்கு செயல்முறைகள் அல்லது சிக்கலான லாஜிக்குக்கு, உருவமைப்பு மாடல் எங்கு இருக்கிறது மற்றும் அடுத்து என்ன வருகிறது என்பதைக் கண்காணிக்க உதவுகிறது.

<img src="../../../translated_images/ta/prompt-structure.a77763d63f4e2f89.webp" alt="Prompt Structure" width="800"/>

*தெளிவான பிரிவுகள் மற்றும் XML-பாணி அமைப்புடன் ஒரு நன்கு கட்டமைக்கப்பட்ட prompt-இன் உட்காத்து*

**தனித்தன்மையான மதிப்பீட்டின் மூலம் தரம்**

தன்-தன்னை விமர்சிக்கும் மாதிரிகள் தரத்துக்கான அளவுகோல்களை தெளிவாகச் செய்கின்றன. "சரி செய்கிறார்" என்று நம்புவதை விட, நீங்கள் "சரி" என்பது என்ன என்று மாடலுக்கு துல்லியமாக கூறுகிறீர்கள்: சரியான லாஜிக், பிழை கையாளல், செயல்திறன், பாதுகாப்பு. அதன்பின்பு மாடல் தன்னுடைய வெளியீட்டை மதிப்பீடு செய்து மேம்படுத்த முடியும். இது கோடு உருவாக்கத்தை ஒரு லாட்டரி இயங்குதல் இலாமல் செயல்முறை ஆக மாற்றுகிறது.

**சூழல் வரம்பாகிறது**

பன்மடங்கு உரையாடல்கள் ஒவ்வொரு கோரிக்கையுடனும் செய்தி வரலாற்றைச் சேர்க்கின்றன. ஆனால் ஒரு எல்லை உள்ளது - ஒவ்வொரு மாடல்வும் அதிகபட்ச டோக்கன் எண்ணிக்கையை கொண்டுள்ளது. உரையாடல்கள் விரிவடையும் போது, சம்பந்தமான சூழலை அந்த வரம்பைத் தாண்டாமை காக்கும் திட்டங்களை நீங்கள் அறிய வேண்டியிருக்கும். இந்த மாடியூல் நினைவின் செயற்பாடு எப்படி இருக்கிறது என்பதை வெளிப்படுத்துகிறது; பின்னர் நீங்கள் எப்போது சுருக்குவது, எப்போது மறந்துவிடுவது, எப்போது மீட்டெடுக்க வேண்டியது என்பதைக் கற்றுக்கொள்வீர்கள்.
