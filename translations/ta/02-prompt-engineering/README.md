## நீங்கள் என்ன கற்கப்போகிறீர்கள்

<img src="../../../translated_images/ta/what-youll-learn.c68269ac048503b2.webp" alt="What You'll Learn" width="800"/>

முன்கூட்டியுள்ள தொகுதியில், நினைவகத்தின் மூலம் உரையாடல் AI எப்படி இயங்குகிறது என்பதைப் பார்த்தீர்கள் மற்றும் GitHub மாதிரிகளைக் கொண்டு அடிப்படையான தொடர்புகளைப் பயன்படுத்தினீர்கள். இப்போது நாம் கேள்விகள் கேட்கும் முறையைக் கவனம் செலுத்துவோம் — அதாவது prompt-கள் — Azure OpenAIயின் GPT-5.2 ஐப் பயன்படுத்தி. உங்கள் prompt-களை அமைக்கும் விதம் பெறும் பதில்களின் தரத்தை முற்றிலும் பாதிக்கிறது. அடிப்படையான prompting தொழில்நுட்பங்களை நாம் மீளாய்வு செய்வோம், அதன் பிறகு GPT-5.2-ன் முழு திறமையை பயன்படுத்தக்கூடிய எட்டு மேம்பட்ட வடிவங்களுக்கு செல்லப்போகிறோம்.

நாம் GPT-5.2-ஐ பயன்படுத்துவோம் ஏனெனில் இது காரணிப்பயணம் கட்டுப்பாட்டை அறிமுகப்படுத்துகிறது - நீங்கள் மாதிரிக்கு பதில் அளிக்க முன்னால் எவ்வளவு சிந்தனையைச் செய்ய வேண்டும் என்று சொல்ல முடியும். இது வித்தியாசமான prompting யுக்திகளை தெளிவாக்குகிறது மற்றும் எந்த போது எந்த அணுகுமுறையை பயன்படுத்த வேண்டும் என்பதை புரிந்துகொள்ள உதவுகிறது. மேலும் Azure இன் நுண்ணிய rate களவுகளில் GPT-5.2 யை GitHub மாதிரிகளுடன் ஒப்பிடுகையில் அதிக அனுமதிகளை அனுமதிக்கும்.

## Prompt Engineering ஐப் புரிந்துகொள்ளுதல்

<img src="../../../translated_images/ta/what-is-prompt-engineering.5c392a228a1f5823.webp" alt="What is Prompt Engineering?" width="800"/>

Prompt Engineering என்பது நீங்கள் தேவைப்படும் முடிவுகளை நிலைத்தியமாக பெறும் வகையில் உள்ளீட்டு உரையை வடிவமைப்பதற்கான செயல். இது கேள்விகள் கேட்குவதற்கு மட்டும் அல்ல - மாதிரி என்ன வேண்டும் மற்றும் எவ்வாறு வழங்க வேண்டும் என்பதை முறையாக புரிந்துகொள்ளப்படக்கூடிய வகையில் கோரிக்கைகளை அமைக்கிறது.

இதனை ஒரு கூட்டு ஊழியருக்கு பணிகளை குறிக்கும் வழியில் நேற்று நினைத்துக்கொள்ளுங்கள். "பிழையை சரி செய்" என்பது தெளிவற்றது. "UserService.java கோப்பின் வரி 45-ல் null pointer exceptionஐ கண்காணிப்பதற்காக null சரிபார்ப்பை சேர்த்து பிழையை சரி செய்" என்பது தெளிவானது. மொழி மாதிரிகள் அதே விதமாக செயல்படுகின்றன – சரியான தனித்துவமும் கட்டமைப்பும் முக்கியம்.

<img src="../../../translated_images/ta/how-langchain4j-fits.dfff4b0aa5f7812d.webp" alt="How LangChain4j Fits" width="800"/>

LangChain4j அடிப்படை கட்டமைப்புகளை வழங்குகிறது — மாதிரி இணைப்புகள், நினைவு, மற்றும் செய்தி வகைகள் — prompt வடிவங்கள் அவ்வாறு படைத்த கட்டமைப்பில் நீங்கள் அனுப்பும் கவனமாக வடிவமைக்கப்பட்ட உரை மட்டுமே. முக்கிய கட்டமைப்பு கூறுகள் `SystemMessage` (AI ஐ நடத்தும் நிர்ணயங்கள் மற்றும் பணி வைக்கும்) மற்றும் `UserMessage` (உங்கள் உண்மையான கோரிக்கையை கொண்டவை) ஆகும்.

## Prompt Engineering அடிப்படைகள்

<img src="../../../translated_images/ta/five-patterns-overview.160f35045ffd2a94.webp" alt="Five Prompt Engineering Patterns Overview" width="800"/>

இந்த தொகுதியில் உள்ள மேம்பட்ட வடிவத்திற்குள் நுழைய முன், ஐந்து அடிப்படை prompting தொழில்நுட்பங்களை பார்ப்போம். இவை ஒவ்வொரு prompt பொறியாளருக்கும் தெரிந்து கொள்ள வேண்டிய கட்டுமானத் துணைக்கற்களாகும். நீங்கள் ஏற்கனவே [Quick Start தொகுதி](../00-quick-start/README.md#2-prompt-patterns) மூலம் இதைப் பார்த்திருப்பீர்கள் — இங்கு அவற்றின் கருத்து வடிவத்தை காணலாம்.

### Zero-Shot Prompting

எளிதாக: மாதிரிக்கு எந்த எடுத்துக்காட்டு இல்லாமல் நேரடி அறிவுரை கொடுக்கவும். மாதிரி தன் பயிற்சியை முழுமையாக பயன்படுத்தி காரியத்தை புரிந்து செயல்படுகிறது. இது மேற்கொள்ளும் பணிக்கே தெளிவான வழிகளுக்கு சிறந்தது.

<img src="../../../translated_images/ta/zero-shot-prompting.7abc24228be84e6c.webp" alt="Zero-Shot Prompting" width="800"/>

*எடுத்துக்காட்டுகள் இல்லாமல் நேரடி அறிவுரை — மாதிரி அறிவுரையிலிருந்து பணியை ஊகிக்கிறது*

```java
String prompt = "Classify this sentiment: 'I absolutely loved the movie!'";
String response = model.chat(prompt);
// பதில்: "நன்மை"
```

**எப்பொழுது பயன்படுத்துவது:** எளிய வகைப்படுத்தல்கள், நேரடி கேள்விகள், மொழிபெயர்ப்புகள் அல்லது கூடுதல் வழிகாட்டு தேவையில்லாத பணிகளுக்கு.

### Few-Shot Prompting

மாதிரிக்கு நீங்கள் கோரவேண்டிய ஒழுங்கினைக் காட்டும் எடுத்துக்காட்டுகளை கொடுங்கள். மாதிரி உங்கள் எடுத்துக்காட்டுகளிலிருந்து உள்ளீடு-வெளியீட்டு வடிவத்தை கற்பது மற்றும் அதனை புதிய உள்ளீடுகளுக்கு பயன்படுத்துகிறது. இது கோரிக்கை வடிவம் தெளிவாக இல்லாத பணிகளில் ஒரே மாதிரியான முடிவுகளை வழங்கக் கூட உதவுகிறது.

<img src="../../../translated_images/ta/few-shot-prompting.9d9eace1da88989a.webp" alt="Few-Shot Prompting" width="800"/>

*எடுத்துக்காட்டுகளில் இருந்து கற்றுக் கொண்டல் — மாதிரி வடிவத்தை கண்டறிந்து புதிய உள்ளீடுகளுக்கு பயன்படுத்துகிறது*

```java
String prompt = """
    Classify the sentiment as positive, negative, or neutral.
    
    Examples:
    Text: "This product exceeded my expectations!" → Positive
    Text: "It's okay, nothing special." → Neutral
    Text: "Waste of money, very disappointed." → Negative
    
    Now classify this:
    Text: "Best purchase I've made all year!"
    """;
String response = model.chat(prompt);
```

**எப்பொழுது பயன்படுத்துவது:** தனிப்பட்ட வகைப்படுத்தல்கள், ஒரே மாதிரி வடிவமைப்பு, துறை-பொருந்தும் பணிகள், அல்லது zero-shot முடிவுகள் திருப்திகரமல்ல எனில்.

### Chain of Thought

மாதிரியை அதன் காரணிப்பயன்முறையை படி படியாக காட்டச் சொல்லுங்கள். நேரடியாக பதிலுக்கு செல்லாமல், மாதிரி பிரச்சனையை உடைக்கிறது மற்றும் ஒவ்வொரு படியையும் தெளிவாக செயல்படுத்துகிறது. இது கணிதம், தர்க்கம் மற்றும் பல படி காரணிப்பணி செயல்களில் துல்லியத்தை மேம்படுத்துகிறது.

<img src="../../../translated_images/ta/chain-of-thought.5cff6630e2657e2a.webp" alt="Chain of Thought Prompting" width="800"/>

*படி படி காரணிப்பயம் — சிக்கலான பிரச்சனைகளை தெளிவான தர்க்க படிகளாக உடைக்கிறது*

```java
String prompt = """
    Problem: A store has 15 apples. They sell 8 apples and then 
    receive a shipment of 12 more apples. How many apples do they have now?
    
    Let's solve this step-by-step:
    """;
String response = model.chat(prompt);
// மாடல் காட்டுகிறது: 15 - 8 = 7, பின்னர் 7 + 12 = 19 ஆப்பிள்கள்
```

**எப்பொழுது பயன்படுத்துவது:** கணிதப் பிரச்சனைகள், தர்க்க புதிர்கள், பிழைத்திருத்தல், அல்லது காரணிப்பயன் வெளிப்படுத்தி நம்பகத்தன்மை மேம்படும்போது.

### Role-Based Prompting

உங்கள் கேள்விக்கு முன் AI க்கு ஒரு தனிப்பட்ட வேடம் அல்லது பண்பை அமைக்கவும். இது பதிலின் பார்வை, ஆழம் மற்றும் கவனத்தை அமைக்க உதவுகிறது. "மென்பொருள் கட்டமைப்பாளர்" என்பது "இளங்கலை நிரலாளர்" அல்லது "பாதுகாப்பு ஆய்வாளர்" என்பவர்களோடு வேறுபட்ட ஆலோசனைகள் தரும்.

<img src="../../../translated_images/ta/role-based-prompting.a806e1a73de6e3a4.webp" alt="Role-Based Prompting" width="800"/>

*சந்தர்ப்பம் மற்றும் வேடம் அமைத்தல் — அதே கேள்விக்கு வேறு வேறு பதில்கள் வழங்கப்படும்.*

```java
String prompt = """
    You are an experienced software architect reviewing code.
    Provide a brief code review for this function:
    
    def calculate_total(items):
        total = 0
        for item in items:
            total = total + item['price']
        return total
    """;
String response = model.chat(prompt);
```

**எப்பொழுது பயன்படுத்துவது:** குறியீடு மதிப்பாய்வு, வழிகாட்டல், துறை சார்ந்த பகுப்பாய்வு, அல்லது ஒருசில திறன்முறைகளுக்கு அல்லது பார்வை நிலைமைகளுக்கு தகுந்த பதில்கள் வேண்டுமானால்.

### Prompt Templates

இயல்புகூறுகளுடன் மறுபயன்பாட்டுக்கு கிடையாத prompts உருவாக்கவும். ஒவ்வொரு முறையும் புதிய prompt எழுதாமல், ஒரு வார்ப்புருவை ஒரே முறை வரையறுத்து அதில் வெவ்வேறு மதிப்புகளை நிரப்புங்கள். LangChain4j இன் `PromptTemplate` வகுப்பு இதைக் `{{variable}}` சில்லறை வடிவிலான எழுத்தியம் கொண்டு எளிதாக்குகிறது.

<img src="../../../translated_images/ta/prompt-templates.14bfc37d45f1a933.webp" alt="Prompt Templates" width="800"/>

*மாற்றிடங்களுடன் மறுபயன்பாட்டுக்கு ஏற்ப prompt கள் — ஒரே வார்ப்புரு, பல பயன்பாடுகள்*

```java
PromptTemplate template = PromptTemplate.from(
    "What's the best time to visit {{destination}} for {{activity}}?"
);

Prompt prompt = template.apply(Map.of(
    "destination", "Paris",
    "activity", "sightseeing"
));

String response = model.chat(prompt.text());
```

**எப்பொழுது பயன்படுத்துவது:** வெவ்வேறு உள்ளீடுகளுடன் மீண்டும் மீண்டும் கேள்விகள், தொகுதி செயலாக்கம், மறுபயன்பாடு செய்யக்கூடிய AI வேலைப்பாட்டுக் கட்டமைப்புகள், அல்லது prompt கட்டமைப்பு ஒரே மாதிரியானும் தரவு மாறும் தேவைகளுக்கு.

---

இந்த ஐந்து அடிப்படைகள் பெரும்பாலான prompting பணிகளுக்கு முக்கியமான கருவிஸ்தானங்களை வழங்குகிறது. இதைப் பின்பு GPT-5.2-ன் காரணிப்பயன் கட்டுப்பாடு, சுய மதிப்பீடு மற்றும் கட்டமைக்கப்பட்ட வெளியீடு திறன்களை பயன்படுத்தும் **எட்டு மேம்பட்ட வடிவங்களுடன்** விரிவாக்குவது.

## மேம்பட்ட வடிவங்கள்

அடிப்படைக் கருத்துக்களை கவனித்த பிறகு, இந்த தொகுதியை தனித்துவமாக்கும் எட்டு மேம்பட்ட வடிவங்களுக்கு செல்லலாம். எல்லா பிரச்சனைகளுக்கும் ஒரே அணுகுமுறை தேவையில்லை. சில கேள்விகள் விரைவான பதில்களை வேண்டுமென்றால், சில ஆழமான சிந்தனையை. சில பார்வை காரணிப்பயன்முறையை, சில நேர்த்தியான முடிவுகளுக்கு. தக்கப்படியான ஒவ்வொரு வடிவமும் தனித்தனி சூழ்நிலைக்கு 최적화 செய்யப்பட்டவை — GPT-5.2-ன் காரணிப்பயனர் கட்டுப்பாடு வித்தியாசங்களை மேலும் தெளிவாக்குகிறது.

<img src="../../../translated_images/ta/eight-patterns.fa1ebfdf16f71e9a.webp" alt="Eight Prompting Patterns" width="800"/>

*எட்டு prompting வடிவங்களின் ஒரு பார்வை மற்றும் அவற்றின் பயன்பாடுகள்*

<img src="../../../translated_images/ta/reasoning-control.5cf85f0fc1d0c1f3.webp" alt="Reasoning Control with GPT-5.2" width="800"/>

*GPT-5.2 வாயிலாக நீங்கள் மாதிரி எவ்வளவு சிந்திக்க வேண்டும் என்பதை நிர்ணயிக்க முடியும் — விரைவு நேரடி பதில்களில் இருந்து ஆழமான ஆராய்ச்சிவரை*

<img src="../../../translated_images/ta/reasoning-effort.db4a3ba5b8e392c1.webp" alt="Reasoning Effort Comparison" width="800"/>

*குறைந்த எதிர்பார்ப்பு (விரைவு, நேரடி) மற்றும் உயர் எதிர்பார்ப்பு (ஆழமான, ஆராய்ச்சிசெய்யும்) காரணிப்பயன் அணுகுமுறைகள்*

**குறைந்த எதிர்பார்ப்பு (விரைவு மற்றும் கவனமுடிவுற்றது)** - எளிய கேள்விகளுக்கு விரைவான, நேரடி பதில்களை நீங்கள் விரும்பும் போது. மாதிரி 2 படிகளுக்கு மேல் என்ன செயல் பிற்படுத்தாது. கணக்குகள், தேடல்கள் அல்லது நேரடி கேள்விகளுக்கு இதைப் பயன்படுத்தவும்.

```java
String prompt = """
    <reasoning_effort>low</reasoning_effort>
    <instruction>maximum 2 reasoning steps</instruction>
    
    What is 15% of 200?
    """;

String response = chatModel.chat(prompt);
```

> 💡 **GitHub Copilot உடன் ஆராயவும்:** [`Gpt5PromptService.java`](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java) திறந்து கேளுங்கள்:
> - "குறைந்த எதிர்பார்ப்பு மற்றும் உயர் எதிர்பார்ப்பு prompting வடிவங்களின் வித்தியாசம் என்ன?"
> - "Prompt-களில் உள்ள XML குறிச்சொற்கள் AI பதிலை எப்படி கட்டமைக்க உதவுகின்றன?"
> - "சுய அதிபர்கூறு வடிவங்களையும் நேரடி அறிவுரையையும் எந்த நேரத்தில் பயன்படுத்த வேண்டும்?"

**உயர் எதிர்பார்ப்பு (ஆழமான மற்றும் விரிவானது)** - சிக்கலான பிரச்சனைகளுக்கு முழுமையான பகுப்பாய்வை நீங்கள் எதிர்பார்க்கிறீர்கள். மாதிரி ஆழமாக ஆராய்கிறது மற்றும் விரிவான காரணிப்பயன் காட்டுகிறது. அமைப்பு வடிவமைப்பு, கட்டமைப்பு முடிவுகள், அல்லது சிக்கலான ஆராய்ச்சி வேலைக்கு இதைப் பயன்படுத்தவும்.

```java
String prompt = """
    <reasoning_effort>high</reasoning_effort>
    <instruction>explore thoroughly, show detailed reasoning</instruction>
    
    Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```

**பணி நடைமுறை (படி படியான முன்னேற்றம்)** - பல படி வேலைப்பாடுகளுக்கு. மாதிரி முன்கூட்டியே ஒரு திட்டத்தை வழங்கி, ஒவ்வொரு படியையும் விளக்கமாகச் சொல்லி தொகுப்பைத் தருகிறது. இடமாற்றங்கள், நடைமுறைப்படுத்தல்கள் அல்லது வேறு எந்த பல படி செயலுக்காக இதைப் பயன்படுத்துங்கள்.

```java
String prompt = """
    <task>Create a REST endpoint for user registration</task>
    <preamble>Provide an upfront plan</preamble>
    <narration>Narrate each step as you work</narration>
    <summary>Summarize what was accomplished</summary>
    """;

String response = chatModel.chat(prompt);
```

Chain-of-Thought prompting மாதிரியை அதன் காரணிப்பு முறையை தெளிவாகக் காட்டும்படி கேட்கிறது, இது சிக்கலான பணிகளுக்கான துல்லியத்தைக் கூட்டுகிறது. படி படியான துண்டாக்கல் மனிதருக்கும் AI க்கும் தர்க்கத்தை புரிந்து கொள்கையாக்க உதவுகிறது.

> **🤖 [GitHub Copilot](https://github.com/features/copilot) வினாடி உரையாடலுடன் முயற்சி செய்யவும்:** இந்த வடிவத்தைப் பற்றிய கேள்விகள்:  
> - "நீண்ட நேர இயங்குதிறன் பணிகளுக்கு பணி நடைமுறை வடிவத்தை எப்படி மாற்றுவது?"  
> - "உற்பத்தி பயன்பாடுகளில் கருவி முன்னுரைகளை அமைப்பதில் சிறந்த நடைமுறைகள் என்ன?"  
> - "UIல் இடைநிலைக் முன்னேற்றங்களை எப்படிக் கையாள்பது மற்றும் காண்பிப்பது?"

<img src="../../../translated_images/ta/task-execution-pattern.9da3967750ab5c1e.webp" alt="Task Execution Pattern" width="800"/>

*பல படி பணிகளுக்கான திட்டம் → நடைமுறை → தொகுப்பு வேலைபாடு*

**சுயபார்வையிடும் குறியீடு** - உற்பத்தி தரமான குறியீட்டை உருவாக்க. மாதிரி குறியீடு உருவாக்கி, தரம் மதிப்பீடு செய்து, மேம்படுத்தும்படி செயல் பெற்றுக் கொள்கிறது. புதிய அம்சங்கள் அல்லது சேவைகளை உருவாக்கும் போது இதைப் பயன்படுத்துங்கள்.

```java
String prompt = """
    <task>Create an email validation service</task>
    <quality_criteria>
    - Correct logic and error handling
    - Best practices (clean code, proper naming)
    - Performance optimization
    - Security considerations
    </quality_criteria>
    <instruction>Generate code, evaluate against criteria, improve iteratively</instruction>
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/ta/self-reflection-cycle.6f71101ca0bd28cc.webp" alt="Self-Reflection Cycle" width="800"/>

*மீண்டும் மீண்டும் மேம்படுத்தும் விருப்பச் சுற்று - உருவாக்கு, மதிப்பாய்வு செய், சிக்கல்களை கண்டுபிடி, மேம்படுத்து, மீண்டும் செய்*

**விடைமுறை பகுப்பு** - ஒரே மாதிருடைய மதிப்பீட்டிற்காக மாதிரி குறியீட்டை நிலையான கட்டமைப்புடன் மதிப்பாய்வு செய்கிறது (சரியானது, நடைமுறைகள், செயல்திறன், பாதுகாப்பு). குறியீடு மதிப்பாய்வு அல்லது தர நிலை மதிப்பீட்டிற்குப் பயன்படுத்தவும்.

```java
String prompt = """
    <code>
    public List getUsers() {
        return database.query("SELECT * FROM users");
    }
    </code>
    
    <framework>
    Evaluate using these categories:
    1. Correctness - Logic and functionality
    2. Best Practices - Code quality
    3. Performance - Efficiency concerns
    4. Security - Vulnerabilities
    </framework>
    """;

String response = chatModel.chat(prompt);
```

> **🤖 [GitHub Copilot](https://github.com/features/copilot) உபயோகித்து முயற்சி செய்யவும்:** அமைப்பு பகுப்பாய்வு குறித்து கேளுங்கள்:  
> - "வேறுபட்ட குறியீடு மதிப்பாய்வுகளுக்கான பகுப்பாய்வு வடிவத்தை எப்படி தனிப்பயனாக்குவது?"  
> - "கட்டமைக்கப்பட்ட வெளியீட்டை நிரல்நிரல் ரீதியாக எப்படி பரிசீலித்து இயக்குவது?"  
> - "வேறுபட்ட மதிப்பாய்வு அமர்வுகளில் ஒரே மாதிரியான தீவிரத்தன்மையை எப்படி உறுதிபடுத்துவது?"

<img src="../../../translated_images/ta/structured-analysis-pattern.0af3b690b60cf2d6.webp" alt="Structured Analysis Pattern" width="800"/>

*மேடை உட்பட நான்கு வகை அமைப்புடன் ஒரே மாதிரின் குறியீடு மதிப்பாய்வு மற்றும் தீவிர தன்மை*

**மல்டி-தெட் உரையாடல்** - காலண்ட் சார்ந்த உரையாடல்களுக்குப். மாதிரி முந்தைய செய்திகளை நினைவில் வைத்துக் கொண்டு அதனை மேலும் விரிவாக்குகிறது. தொடர்பு உதவி அமர்வுகள் அல்லது சிக்கலான கேள்வி & பதில்களுக்கு பயன்படுத்தவும்.

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("What is Spring Boot?"));
AiMessage aiMessage1 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage1);

memory.add(UserMessage.from("Show me an example"));
AiMessage aiMessage2 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage2);
```

<img src="../../../translated_images/ta/context-memory.dff30ad9fa78832a.webp" alt="Context Memory" width="800"/>

*ஒரிட்ட அரைபாடிகளில் உரையாடல் உள்ளடக்கம் மொத்தமாகச் சேரும் வழி*

**படி-படி காரணம் கூறுதல்** - பகுத்தறிவுக்கு தேவைப்படுகிற சிக்கலான பிரச்சனைகளுக்கு. மாதிரி ஒவ்வொரு படி காரணிப்பயனை தெளிவாகக் காட்டுகிறது. கணிதப் பிரச்சனைகள், தர்க்க புதிர்கள் அல்லது சிந்தனை முறையை புரிந்துகொள்ள வேண்டிய போது இதைப் பயன்படுத்தவும்.

```java
String prompt = """
    <instruction>Show your reasoning step-by-step</instruction>
    
    If a train travels 120 km in 2 hours, then stops for 30 minutes,
    then travels another 90 km in 1.5 hours, what is the average speed
    for the entire journey including the stop?
    """;

String response = chatModel.chat(prompt);
```

**கட்டுப்படுத்தப்பட்ட வெளியீடு** - குறிப்பிட்ட வடிவ மற்றும் நீளம் அடிப்படையிலான பதில்களுக்கு. மாதிரி வடிவம் மற்றும் நீளம் விதிகளை கடுமையாக பின்பற்றுகிறது. சுருக்கமான நிருபங்கள் அல்லது துல்லியமான வெளியீடு கட்டமைப்புக்குப் பயன்படும்.

    - Technical terms only

    """;

இணைய இடைமுகம் பலவிதமான prompting முறைகளை பரிசோதிக்க அனுமதிக்கும். ஒவ்வொரு மாதிரியும் வேறுபட்ட பிரச்சனைகளைத் தீர்க்கிறது - ஒவ்வொன்றும் எப்போது சிறப்பாக செயல்படுகிறது என்பதை பார்க்க முயற்சிக்கவும்.

GPT-5.2 உங்கள் prompting மூலம் கணக்கீட்டு முயற்சியை கட்டுப்படுத்த உதவுகிறது. குறைந்த முயற்சி விரைவு பதில்களுடன் குறைந்த ஆராய்ச்சியை குறிக்கிறது. அதிக முயற்சி மாதிரி ஆழம் காட்டி எண்ணிக்கலாம். நீங்கள் பணியின் சிக்கலைத்தோட்ட முயற்சியைக் குறிப்பதற்காக கற்றுக் கொள்கின்றீர்கள் - எளிய கேள்விகளுக்கு நேரம் வீண் செய்யாமல்உயர்ந்த முடிவுகளை விரைவில் எடுத்துக் கொள்ளவும்.

**கட்டமைப்பு நடத்தை வழிநடத்துகிறது**

promptகளில் XML குறிச்சுட்கள் இருக்கின்றன என்பதை கவனியுங்கள்? அவை அலங்காரத் தொகுப்புகள் அல்ல. மாதிரிகள் கட்டமைக்கப்பட்ட அறிவுறுப்புகளை விட அதிக நம்பகத்தன்மையுடன் பின்பற்றுகின்றன. பல படி செயல்முறைகள் அல்லது சிக்கலான கொள்கைகள் தேவைப்பட்டால், கட்டமைப்பு மாதிரியின் நிலையை மற்றும் அடுத்ததை எங்கே என்று கண்காணிக்க உதவுகிறது.

<img src="../../../translated_images/ta/prompt-structure.a77763d63f4e2f89.webp" alt="Prompt Structure" width="800"/>

*சுருக்கமான பிரிவுகளும் XML-பாணி அமைப்பும் கொண்ட நன்றாக கட்டமைக்கப்பட்ட prompt ஆனாட்டமி*

**தரத்தை சுயமதிப்பீட்டின் மூலம்**

சுயவிமர்சன மாதிரிகள் தரமான அளவுகளை தெளிவாகச் சொல்கின்றன. மாதிரி "சரி செய்கிறது" என்று நம்புவதை விட, நீங்கள் "சரி" என்பது என்ன என்பதை துல்லியமாகக் கூறுகிறீர்கள்: சரியான வழிகாட்டல், பிழை கையாளுதல், செயல்திறன், பாதுகாப்பு. அதன் பிறகு மாதிரி தனது வெளிப்பாட்டை மதிப்பாய்வு செய்து மேம்படுத்த முடியும். இது குறியீடு உருவாக்கத்தை வாய்க்குப்போன்று இல்லாமல் ஒரு செயல்முறையாக மாற்றுகிறது.

**சூழல் வரம்பு உள்ளது**

பல-முறை உரையாடல்கள் ஒவ்வொரு கோரிக்கையுடனும் செய்தி வரலாற்றை உள்ளடக்குவதால் செயல்படுகிறது. ஆனால் எல்லை உள்ளது - ஒவ்வொரு மாதிரிக்கும் அதிகபட்ச டோக்கன் எண்ணிக்கை உள்ளது. உரையாடல்கள் பெருகும் போது, தொடர்புடைய சூழலைச் சீர் செய்யும் முறைகளை பயன்படுத்த வேண்டும். இந்த முறை நீங்கள் நினைவூட்டல் எப்படி செயல்படுகிறது என்பதை கற்றுக்கொடுக்கிறது; பின்னர் எப்போது சுருக்க வேண்டும், எப்போது மறக்க வேண்டும் மற்றும் எப்போது மீட்டெடுக்க வேண்டும் என்பதையும் கற்றுக்கொள்வீர்கள்.
