## நீங்கள் கற்றுக்கொள்ளப்போகிறீர்கள்

<img src="../../../translated_images/ta/what-youll-learn.c68269ac048503b2.webp" alt="நீங்கள் கற்றுக்கொள்ளப்போகும் விஷயங்கள்" width="800"/>

முன்னாள் தொகுதியில், நினைவகம் உரையாடல் AI-யை எவ்வாறு இயங்கச் செய்கிறது என்பதை புரிந்து GitHub மாதிரிகள் மூலம் அடிப்படைக் கருத்துகளை கற்றீர்கள். இப்போது நாம் கேள்விகள் கேட்டு — தூண்டல்கள் பற்றிய — Azure OpenAI இன் GPT-5.2 பயன்படுத்துவது எப்படி என்பதை கவனம் செலுத்துகிறோம். உங்கள் தூண்டல்கள் அமைக்கும் முறையே நீங்கள் பெறும் பதில்களின் தரத்தை பெரிதும் பாதிக்கிறது. முதலில் அடிப்படை தூண்டுதல் தொழில்நுட்பங்களை பார்வையிட்டு பின்னர் GPT-5.2 இன் முழு திறனை பயன்படுத்தும் எட்டு மேம்பட்ட முறைமைகளை ஆராய்வோம்.

GPT-5.2யைப் பயன்படுத்துவோம் ஏனெனில் இது சிந்தனை கட்டுப்பாட்டை அறிமுகப்படுத்துகிறது - பதிலளிக்க முன்னர் எவ்வளவு சிந்திக்க வேண்டும் என்பதைக் கடைத்தெரிவிக்க முடியும். இது வேறுபட்ட தூண்டுதல் முறைகளை தெளிவாக காட்டுகிறது, எந்த அணுகுமுறையை எப்போது பயன்படுத்த வேண்டுமென்றாக புரிய உதவுகிறது. மேலும், GPT-5.2க்கு Azure உடைய குறைந்த வேக வரிசை எல்லைகள் GitHub மாதிரிகளுடன் ஒப்பிடும்போது நன்மை தருகிறது.

## முன்னேற்பாடு

- தொகுதி 01 நிறைவேற்றப்பட்டது (Azure OpenAI வளங்கள் நிறுவப்பட்டுள்ளன)
- ரூட் அடைவின் `.env` கோப்பில் Azure அங்கீகாரங்கள் உள்ளன (தொகுதி 01 இல் `azd up` மூலம் உருவாக்கப்பட்டது)

> **குறிப்பு:** தொகுதி 01 நிறைவேற்றப்படாவிட்டால், முதலில் அங்கு வழங்கிய நிறுவல் அறிவுரைகளை பின்பற்றவும்.

## தூண்டுதல் பொறியியல் புரிதல்

<img src="../../../translated_images/ta/what-is-prompt-engineering.5c392a228a1f5823.webp" alt="தூண்டுதல் பொறியியல் என்றால் என்ன?" width="800"/>

தூண்டுதல் பொறியியல் என்பது எப்போதும் தேவையான முடிவுகளை தரும் உள்ளீட்டு உரையை வடிவமைப்பதற்கான செயல்பாடு. கேள்விகள் கேட்க மட்டும் அல்ல - மாதிரிக்கு நீங்கள் என்ன வேண்டும் மற்றும் எப்படி வழங்க வேண்டும் என்பதை தெளிவாக புரிய வைக்கும் கோரிக்கைகளை அமைப்பது.

இதை உத்தியோகத்தருக்கு வழிகாட்டி அளிப்பது போல நினைத்துக் கொள்ளுங்கள். "பிழையை சரிசெய்" என்றால் அசிடுப்பு. "UserService.java 45-வது வரியில் null pointer exception-ஐ null சோதனை சேர்த்து சரி செய்" என்று கூறுவது தெளிவானது. மொழிநூல்கள் அதேபோல் செயல்படுகின்றன - தெளிவும் அமைப்பும் முக்கியம்.

<img src="../../../translated_images/ta/how-langchain4j-fits.dfff4b0aa5f7812d.webp" alt="LangChain4j எப்படி பொருந்துகிறது" width="800"/>

LangChain4j உடைய உருவமைப்பு — மாதிரி தொடர்புகள், நினைவகம் மற்றும் செய்தி வகைகள் — மற்றும் தூண்டுதல் முறைமைகள் என்பது அந்த அமைப்பின் வழியாக அனுப்பப்படும் கவனமாக அமைக்கப்பட்ட உரை. முக்கிய கட்டமைப்புகள் `SystemMessage` (AI நடத்தை மற்றும் பங்கினை அமைக்கும்) மற்றும் `UserMessage` (உங்கள் கோரிக்கையை கொண்டிருக்கும்) ஆகும்.

## தூண்டுதல் பொறியியல் அடிப்படைகள்

<img src="../../../translated_images/ta/five-patterns-overview.160f35045ffd2a94.webp" alt="ஐந்து தூண்டுதல் பொறியியல் முறைமைகள் மேம்புலம்" width="800"/>

இந்த தொகுதியில் உள்ள மேம்பட்ட முறைமைகளின் முன், ஐந்து அடிப்படை தூண்டுதல் தொழில்நுட்பங்களை பார்ப்போம். இவை அனைத்து தூண்டல் பொறியாளர்களுக்கும் தெரிந்திருக்க வேண்டிய கட்டுப்பாடுகள். நீங்கள் ஏற்கனவே [விரைவு தொடக்க தொகுதியில்](../00-quick-start/README.md#2-prompt-patterns) இவற்றை பார்த்திருப்பீர்கள் — இவை அடிப்படையான கோட்பாடுகளாய் உள்ளன.

### சூன்ய-ஷாட் தூண்டுதல்

எளிமையான அணுகுமுறை: மாதிரிக்கு எடுத்துக்காட்டுகள் இல்லாமல் நேரடி கட்டளை கொடுங்கள். மாதிரி முழுமையாக தனது பயிற்சியை பொருந்தித்து பணியை புரிந்து செயல்படுகிறது. எதிர்பார்க்கப்படும் நடத்தை தெளிவான நேர்மையான கோரிக்கைகளுக்கு இது சாத்தியமாகும்.

<img src="../../../translated_images/ta/zero-shot-prompting.7abc24228be84e6c.webp" alt="சூன்ய-ஷாட் தூண்டுதல்" width="800"/>

*எடுத்துக்காட்டுகள் இல்லாமல் நேரடி கட்டளை — மாதிரி கட்டளையிலிருந்தே பணியை ஊகிக்கிறது*

```java
String prompt = "Classify this sentiment: 'I absolutely loved the movie!'";
String response = model.chat(prompt);
// பதில்: "நன்மை"
```

**எப்போது பயன்படுத்தலாம்:** எளிய வகைப்பாடு, நேரடி கேள்விகள், மொழிபெயர்ப்பு, அல்லது கூடுதல் வழிகாட்டல் தேவையில்லாத பணிகள்.

### சில-ஷாட் தூண்டுதல்

மாதிரியை பின்பற்றும் முறைமையை காட்டும் எடுத்துக்காட்டுகளை கொடுங்கள். மாதிரி உங்கள் எடுத்துக்காட்டுகளிலிருந்து உள்ளீடு-வெளியீட்டு வடிவத்தைக் கற்றுக்கொண்டு புதிய உள்ளீடுகளுக்கு அதனைப் பயன்படுத்துகிறது. இது எதிர்பார்க்கப்பட்ட வடிவம் அல்லது நடத்தை தெளிவில்லாத பணிகளுக்கு ஒத்திசைவைக் பெருக்குகிறது.

<img src="../../../translated_images/ta/few-shot-prompting.9d9eace1da88989a.webp" alt="சில-ஷாட் தூண்டுதல்" width="800"/>

*எடுத்துக்காட்டுகளிலிருந்து கற்றுக்கொள்வது — மாதிரி முறைமையை கண்டறிந்து புதிய உள்ளீடுகளில் பயன்படுத்துகிறது*

```java
String prompt = """
    Classify the sentiment as positive, negative, or neutral.
    
    Examples:
    Text: "This product exceeded my expectations!" → Positive
    Text: "It's okay, nothing special." → Neutral
    Text: "Waste of money, very disappointed." → Negative
    
    Now classify this:
    Text: "Best purchase I've made all year!"
    """;
String response = model.chat(prompt);
```

**எப்போது பயன்படுத்தலாம்:** தனிப்பயன் வகைப்படுத்தல், ஒத்திசைக்கக்கூடிய வடிவமைப்பு, துறை சார்ந்த பணிகள் அல்லது சூன்ய-ஷாட் விளைவுகள் ஒத்திசைவற்றவை ஆகியவை.

### சிந்தனை சங்கிலி

மாதிரியை தனது காரணத்தை படித்தடி காட்டச் சொல்லுங்கள். நேரடி பதில் சொல்லாமல் பதில் கூறும் முன் பிரச்சினையை தவிர்த்து ஒவ்வொரு பகுதியையும் தெளிவாக முன்னேறுகிறது. இது கணிதம், தர்க்கம் மற்றும் பல கட்ட அடுக்கு சிந்தனை பணிகளுக்கு துல்லியம் மேம்படுத்துகிறது.

<img src="../../../translated_images/ta/chain-of-thought.5cff6630e2657e2a.webp" alt="சிந்தனை சங்கிலி தூண்டுதல்" width="800"/>

*படித்தடியான காரணவியல் — கூடிய பிரச்சினைகளை தெளிவாக படி படியாக உடைத்தல்*

```java
String prompt = """
    Problem: A store has 15 apples. They sell 8 apples and then 
    receive a shipment of 12 more apples. How many apples do they have now?
    
    Let's solve this step-by-step:
    """;
String response = model.chat(prompt);
// மாதிரி காட்டுகிறது: 15 - 8 = 7, பின்னர் 7 + 12 = 19 ஆப்பிள்கள்
```

**எப்போது பயன்படுத்தலாம்:** கணிதப் பிரச்சினைகள், தர்க்கக் புதிர்கள், பிழை கண்டறிதல் அல்லது காரணவியலை காட்டுதல் நம்பகத்தன்மை மேம்படுகிற பணிகள்.

### பங்கு அடிப்படையிலான தூண்டுதல்

கேள்விக்குப் முன்னர் AI-க்கு ஒரு பங்கு அல்லது தன்மை அமைக்கவும். இது பதிலின் தொனியை, ஆழத்தையும், கவனத்தையும் வடிவமைக்க உதவுகிறது. "மென்பொருள் வடிவமைப்பாளர்" "கீழ்ப்படிகள் உருவாக்குநர்" அல்லது "பாதுகாப்பு ஆய்வாளர்" என்பவர்கள் வழங்கும் ஆலோசனை வெவ்வேறு.

<img src="../../../translated_images/ta/role-based-prompting.a806e1a73de6e3a4.webp" alt="பங்கு அடிப்படையிலான தூண்டுதல்" width="800"/>

*பார்வை மற்றும் தன்மையை அமைத்தல் — அதே கேள்விக்கு மாறுபட்ட பங்கு அமையும்போது வேறுபட்ட பதில்கள்*

```java
String prompt = """
    You are an experienced software architect reviewing code.
    Provide a brief code review for this function:
    
    def calculate_total(items):
        total = 0
        for item in items:
            total = total + item['price']
        return total
    """;
String response = model.chat(prompt);
```

**எப்போது பயன்படுத்தலாம்:** குறியீடு பரிசீலனைகள், கற்பித்தல், துறை சார்ந்த பகுப்பாய்வுகள் அல்லது குறிப்பிட்ட திறன் நிலைக்கு ஏற்ப பதில்கள் வேண்டும்.

### தூண்டுதல் வார்ப்புருக்கள்

மாறிலி இடைவெளியைக் கொண்ட மீண்டும் பயன்படுத்தக்கூடிய தூண்டல்களை உருவாக்கவும். ஒவ்வொரு முறையும் புதிய தூண்டலை எழுதாமல் ஒருமுறை வார்ப்புருவை வரையறுத்து வெவ்வேறு மதிப்புகளைக் கேளுங்கள். LangChain4j இன் `PromptTemplate` வகுப்பு `{{variable}}` யை எளிதாக்குகிறது.

<img src="../../../translated_images/ta/prompt-templates.14bfc37d45f1a933.webp" alt="தூண்டுதல் வார்ப்புருக்கள்" width="800"/>

*மாறிலி இடைவெளிகளுடன் மீண்டும் பயன்படுத்தக்கூடிய தூண்டல்கள் — ஒரு வார்ப்புரு, பல பயன்கள்*

```java
PromptTemplate template = PromptTemplate.from(
    "What's the best time to visit {{destination}} for {{activity}}?"
);

Prompt prompt = template.apply(Map.of(
    "destination", "Paris",
    "activity", "sightseeing"
));

String response = model.chat(prompt.text());
```

**எப்போது பயன்படுத்தலாம்:** வித்தியாசமான உள்ளீடுகள் கொண்ட மீண்டும் கேள்விகள், தொகுதி செயலாக்கம், மீண்டும் பயன்படுத்தக்கூடிய AI வேலைநடவடிக்கைகள் அல்லது தூண்டல் அமைப்பு ஒரே மாதிரிதான் ஆனால் தரவு மாறினால்.

---

இந்த ஐந்து அடிப்படைகள் பெரும்பாலான தூண்டல் பணிகளைச் செய்துக்கொள்ள ஒரு வலுவான கருவிமிக்குறி அளிக்கின்றன. இந்த தொகுதியின் மற்ற பகுதிகள் GPT-5.2 இன் சிந்தனை கட்டுப்பாடு, சுய மதிப்பீடு மற்றும் கட்டமைக்கப்பட்ட வெளியீடு திறன்களை பயன்படுத்தும் **எட்டு மேம்பட்ட முறைமைகளைக்** கொண்டவை.

## அதிக முன்னேற்ற முறைமைகள்

அடிப்படைகள் முழுமையாக கற்றுவிட்டதுடன், இப்போது இந்த தொகுதியின் தனித்துவத்தை உருவாக்கும் எட்டு மேம்பட்ட முறைமைகளுக்கு மாறுவோம். எல்லா பிரச்சினைகளும் ஒரே அணுகுமுறையைப் பெற முடியாது. சில கேள்விகள் விரைவான பதில்களை தேடுகின்றன, சில ஆழ்ந்த சிந்தனையை. சில வெளிப்படையான காரணியலை நாடுகின்றன, அதற்கு பதிலாக சில நீதிமூலம் பெறுவதற்கு. கீழ்காணும் ஒவ்வொரு முறைமையும் வேறுபட்ட சூழலைக் கருத்தில் கொண்டு அமைக்கப்பட்டுள்ளது — மேலும் GPT-5.2 யின் சிந்தனை கட்டுப்பாடு வேறுபாடுகளை இன்னும் தெளிவாக்க காட்டுகிறது.

<img src="../../../translated_images/ta/eight-patterns.fa1ebfdf16f71e9a.webp" alt="எட்டு தூண்டுதலை முறைமைகள்" width="800"/>

*எட்டு தூண்டுதல் பொறியியல் முறைமைகள் மற்றும் அவற்றின் பயன்பாடுகள் மேம்புலம்*

<img src="../../../translated_images/ta/reasoning-control.5cf85f0fc1d0c1f3.webp" alt="GPT-5.2 உடன் சிந்தனை கட்டுப்பாடு" width="800"/>

*GPT-5.2 இன் சிந்தனை கட்டுப்பாடு — மாதிரி எவ்வளவு சிந்திக்க வேண்டும் என்பதைக் குறிப்பது — விரைவு நேரடி பதில்களிலிருந்து ஆழ்ந்த ஆராய்ச்சியுடனான வரை*

<img src="../../../translated_images/ta/reasoning-effort.db4a3ba5b8e392c1.webp" alt="சிந்தனை முயற்சி ஒப்பீடு" width="800"/>

*குறைந்த ஆர்வம் (விரைவு, நேரடி) vs அதிக ஆர்வம் (ஆழ்ந்த, ஆராய்ச்சியுடன்) சிந்தனை அணுகுமுறைகள்*

**குறைந்த ஆர்வம் (விரைவான மற்றும் கவனமாக)** - விரைவான, நேமறடி பதில்களை வேண்டிய எளிய கேள்விகளுக்கு. மாதிரி சிந்தனையை மிகுந்தபட்சம் இரண்டு படிகள் மட்டுமே செய்கிறது. கணக்குகள், தேடல்கள் அல்லது பொது கேள்விகளுக்குப் பயன்படுத்தவும்.

```java
String prompt = """
    <context_gathering>
    - Search depth: very low
    - Bias strongly towards providing a correct answer as quickly as possible
    - Usually, this means an absolute maximum of 2 reasoning steps
    - If you think you need more time, state what you know and what's uncertain
    </context_gathering>
    
    Problem: What is 15% of 200?
    
    Provide your answer:
    """;

String response = chatModel.chat(prompt);
```

> 💡 **GitHub Copilot உடன் ஆராய்ந்து பார்க்கவும்:** [`Gpt5PromptService.java`](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java) திறந்து கேளுங்கள்:
> - "குறைந்த ஆர்வம் மற்றும் அதிக ஆர்வம் தூண்டுதல் முறைமைகளுக்கு இடையேயான வேறுபாடு என்ன?"
> - "தூண்டலில் உள்ள XML குறிசொற்கள் AI பதிலின் அமைப்புக்கு எப்படி உதவுகின்றன?"
> - "நான் எப்போது சுய-பரிசீலனை முறைமைகள் மற்றும் நேரடி கட்டளைகளைப் பயன்படுத்த வேண்டும்?"

**அதிக ஆர்வம் (ஆழ்ந்த மற்றும் முழுமையான)** - விவரமான பகுப்பாய்வுக்கு தேவையான சிக்கலான பிரச்சினைகளுக்கு. மாதிரி பல்வேறு வதிவுகளுடன் ஆராய்ச்சி செய்து சிந்தனை முறைகளை காட்டுகிறது. சிஸ்டம் வடிவமைப்பு, கட்டமைப்பு தீர்மானங்கள் அல்லது சிக்கலான ஆராய்ச்சிக்குப் பயன்படுத்தவும்.

```java
String prompt = """
    Analyze this problem thoroughly and provide a comprehensive solution.
    Consider multiple approaches, trade-offs, and important details.
    Show your analysis and reasoning in your response.
    
    Problem: Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```

**பணியினை நடத்தியல் (அடிச்சடி முன்னேற்றம்)** - பல படி பணிகளுக்கு. மாதிரி முன் திட்டத்தை கொடுத்து வேலை செய்யும் ஒவ்வொரு படியையும் விவரிக்கிறது, பின்னர் சுருக்கமாக விளக்குகிறது. இடமாற்றல்கள், அமலாக்கங்கள் அல்லது எந்தவொரு பல படி செயல்முறைக்கும் இதைக் பயன்படுத்த முடியும்.

```java
String prompt = """
    <task_execution>
    1. First, briefly restate the user's goal in a friendly way
    
    2. Create a step-by-step plan:
       - List all steps needed
       - Identify potential challenges
       - Outline success criteria
    
    3. Execute each step:
       - Narrate what you're doing
       - Show progress clearly
       - Handle any issues that arise
    
    4. Summarize:
       - What was completed
       - Any important notes
       - Next steps if applicable
    </task_execution>
    
    <tool_preambles>
    - Always begin by rephrasing the user's goal clearly
    - Outline your plan before executing
    - Narrate each step as you go
    - Finish with a distinct summary
    </tool_preambles>
    
    Task: Create a REST endpoint for user registration
    
    Begin execution:
    """;

String response = chatModel.chat(prompt);
```

சிந்தனை சங்கிலி தூண்டுதல் மாதிரியை காரணவியல் செயல்முறையினை வெளிப்படையாகக் காட்டவும்கூட கூர்ந்து அறிவை மேம்படுத்துகிறது, கடினமான பணிகளில் துல்லியத்தை அதிகரிக்க உதவுகிறது. படித்தடியான முறையில் பிரித்து தூண்டல் மனிதர்களும் AI-உம் தர்க்கத்தை புரிந்துகொள்ள உதவும்.

> **🤖 [GitHub Copilot](https://github.com/features/copilot) அரட்டை மூலம் முயற்சி செய்யவும்:** இந்த முறைமையைப் பற்றி கேளுங்கள்:
> - "நீண்ட நேரம் நடைபெறும் செயல்களின் பணிநடத்தல் முறைமையை நான் எவ்வாறு ஏற்க வேண்டும்?"
> - "உற்பத்தி பயன்பாடுகளில் கருவி முன்னுரைகளை அமைக்கும் சிறந்த நடைமுறைகள் என்ன?"
> - "UI-வில் இடைநிலை முன்னேற்றத்தைப் பதிவு செய்து காட்ட நான் எப்படி செயல்பட வேண்டும்?"

<img src="../../../translated_images/ta/task-execution-pattern.9da3967750ab5c1e.webp" alt="பணியினை நடத்தியல் முறைமை" width="800"/>

*திட்டமிடு → செயல்படு → சுருக்கம் உருவாக்கு வேலைநடவடிக்கை*

**சுய-பரிசீலனை குறியீடு** - உற்பத்தி தரமான குறியீடு உருவாக்க. மாதிரி உற்பத்தி தரநிலைகளை பின்பற்றி பிழை கையாளுதலைச் சேர்த்து குறியீடை தயாரிக்கிறது. புதிய அம்சங்கள் அல்லது சேவைகள் உருவாக்கும் போது இதைப் பயன்படுத்தவும்.

```java
String prompt = """
    Generate Java code with production-quality standards: Create an email validation service
    Keep it simple and include basic error handling.
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/ta/self-reflection-cycle.6f71101ca0bd28cc.webp" alt="சுய-பரிசீலனை சுழற்சி" width="800"/>

*படியாடல் மேம்படுத்தும் வட்டம் - உருவாக்கு, மதிப்பிடு, பிரச்சினைகளை கண்டறி, மேம்படுத்து, மீண்டும் செய்*

**சட்டப்படுத்தப்பட்ட பகுப்பு** - ஒரே மாதிரியில் மதிப்பீடு செய்ய. மாதிரி குறியீட்டை ஒரு நிரந்தர வடிமுறையில் (சரியான தன்மை, நடைமுறைகள், செயல்திறன், பாதுகாப்பு, பராமரிக்க தகுதி) மதிப்பிடுகிறது. குறியீடு பரிசீலனைகள் அல்லது தரம்தரிசனைகளுக்கு இதைப் பயன்படுத்தவும்.

```java
String prompt = """
    <analysis_framework>
    You are an expert code reviewer. Analyze the code for:
    
    1. Correctness
       - Does it work as intended?
       - Are there logical errors?
    
    2. Best Practices
       - Follows language conventions?
       - Appropriate design patterns?
    
    3. Performance
       - Any inefficiencies?
       - Scalability concerns?
    
    4. Security
       - Potential vulnerabilities?
       - Input validation?
    
    5. Maintainability
       - Code clarity?
       - Documentation?
    
    <output_format>
    Provide your analysis in this structure:
    - Summary: One-sentence overall assessment
    - Strengths: 2-3 positive points
    - Issues: List any problems found with severity (High/Medium/Low)
    - Recommendations: Specific improvements
    </output_format>
    </analysis_framework>
    
    Code to analyze:
    ```
    public List getUsers() {
        return database.query("SELECT * FROM users");
    }
    ```
    Provide your structured analysis:
    """;

String response = chatModel.chat(prompt);
```

> - "பல வகைப்பதிகள் உள்ள குறியீடு பரிசீலனைக்கு பகுப்பாய்வு வடிமுறையை நான் எவ்வாறு தனிப்பயனாக்க வேண்டும்?"
> - "கட்டமைக்கப்பட்ட வெளியீட்டை நிரல்பூர்வமாக எவ்வாறு பிரித்தெடுக்க மற்றும் செயல்படுத்துவது சிறந்த முறையா?"
> - "பல் பரிசீலனை அமர்வுகளுக்கு இடையேயான ஒரே மாதிரிப் பருண்மிதான நிலைகளை நான் எப்படி உறுதி செய்கிறேன்?"

*கூற்று நிலைகளுடன் ஒரே மாதிரிப் குறியீடு பரிசீலனைக்கான வடிவமைப்பு*

**பல திருப்புக் காட்சி** - உரையாடலுக்கு முன்னகமான தகவலோடு. மாதிரி முன்கூட்டியே செய்தியைக் நினைவில் வைத்துக் கொண்டு அதை விரிவுபடுத்தும். இடைநிலை உதவி அமர்வுகள் அல்லது சிக்கலான கேள்வி பதில் கூடைகள் இதற்கு பொருந்தும்.

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("What is Spring Boot?"));
AiMessage aiMessage1 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage1);

memory.add(UserMessage.from("Show me an example"));
AiMessage aiMessage2 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage2);
```

*மிகவும் திருப்புகளுக்கிடையில் உரையாடல் முக்கியத்துவம் சேரும் வரை - டோக்கன் வரம்புக்கு வரைக்கும்*

**அடிச்சடி விளக்கமுருகல்** - வெளிப்படையான தர்க்கம் தேவைப்படும் பிரச்சினைகளுக்கு. மாதிரி ஒவ்வொரு படியையும் தெளிவான காரணவியலுடன் காட்டுகிறது. கணிதம், தர்க்க புதிர்கள் அல்லது சிந்தனை முறையைப் புரிந்துகொள்ள வேண்டிய பொழுதுகளில் இதைக் பயன்படுத்தவும்.

```java
String prompt = """
    <instruction>Show your reasoning step-by-step</instruction>
    
    If a train travels 120 km in 2 hours, then stops for 30 minutes,
    then travels another 90 km in 1.5 hours, what is the average speed
    for the entire journey including the stop?
    """;

String response = chatModel.chat(prompt);
```

*பிரச்சினைகளை தெளிவான தர்க்க படிகளாக இடித்தல்*

**கட்டுப்படுத்தப்பட்ட வெளியீடு** - குறிப்பிட்ட வடிவ முறைகள் தேவையான பதில்களுக்கு. மாதிரி கட்டமைப்பு மற்றும் நீளம் விதிகளை கடுமையாக பின்பற்றுகிறது. சுருக்கங்கள் அல்லது துல்லியமான வெளியீடு அமைப்பு வேண்டும் என்றால் இதில் பயன்படுத்தவும்.

```java
String prompt = """
    <constraints>
    - Exactly 100 words
    - Bullet point format
    - Technical terms only
    </constraints>
    
    Summarize the key concepts of machine learning.
    """;

String response = chatModel.chat(prompt);
```

### படி படி காரணம்

ஒரு கணித பிரச்சினையை எடுத்துக் கொண்டு அதனை படி படி காரணிக்கவும் மற்றும் குறைந்த ஆர்வத்துடனும் முயற்சிக்கவும். குறைந்த ஆர்வம் விரைவான பதிலை அளிக்கும் - ஆனால் மறைவு அதிகம். படி படி காரணிப்பு நடவடிக்கை மற்றும் முடிவுகள் அனைத்தையும் காட்டும்.

<img src="../../../translated_images/ta/step-by-step-reasoning-demo.12139513356faecd.webp" alt="Step-by-Step Reasoning Demo" width="800"/>

*வெளிப்படையான படிகள் கொண்ட கணித பிரச்சனை*

### கட்டுப்படுத்தப்பட்ட வெளியீடு

தெரிந்த கட்டமைப்புகள் அல்லது சொல் எண்ணிக்கைகள் தேவைப்பட்டால், இந்த மாதிரி கடுமையான ஒட்டுமொத்தப்படுத்தலை அமல்படுத்துகிறது. 100 சொற்கள் உள்ள புள்ளி வடிவ சுருக்கத்தை உருவாக்க முயற்சிக்கவும்.

<img src="../../../translated_images/ta/constrained-output-demo.567cc45b75da1633.webp" alt="Constrained Output Demo" width="800"/>

*வடிவுப்படுத்தல் கட்டுப்பாடுடன் இயந்திரக் கற்றல் சுருக்கம்*

## நீங்கள் உண்மையில் கற்றுக்கொள்ளும் বিষয়ங்கள்

**காரண முனைப்பின் முயற்சி அனைத்தையும் மாற்றும்**

GPT-5.2 உங்களுக்குத் திறனுக்கு ஏற்ப கணக்கீட்டு முயற்சியைக் கட்டுப்படுத்த அனுமதிக்கும். குறைந்த முயற்சி விரைவான பதில்கள் மற்றும் குறைந்த ஆராய்ச்சியுடன் வருகிறது. அதிக முயற்சி மாடல் ஆழமான சிந்தனைக்குப் பெரும்பேசுவீர்கள். நீங்கள் பணி சிக்கலுக்கு முயற்சியை பொருத்த கற்றுக்கொள்கிறீர்கள் - எளிய கேள்விகளுக்கு நேரம் வீணாக்க வேண்டாம், ஆனால் சிக்கலான முடிவுகளை விரைவாக முடிக்க கூடாது.

**கட்டமைப்பு நடத்தை வழிநடத்துகிறது**

பிராஂப்ட்களில் XML குறிச்சொற்களை கவனித்தீர்களா? அவை அலங்காரப்படுத்தப்படவில்லை. மாடல்கள் கட்டமைக்கப்பட்ட வழிமுறைகளை சுதந்திர உரையை விட நம்பகமாக பின்பற்றுகின்றன. பல படி செயல்முறைகள் அல்லது சிக்கலான தர்க்கங்கள் தேவைப்பட்டால், கட்டமைப்பு மாடலுக்கு அது எங்கே உள்ளதோ, அடுத்தது என்ன என்று கண்காணிக்க உதவுகிறது.

<img src="../../../translated_images/ta/prompt-structure.a77763d63f4e2f89.webp" alt="Prompt Structure" width="800"/>

*தெளிவான பிரிவுகள் மற்றும் XML-பாணி ஒழுங்கமைப்புடன் ஒரு நன்கு கட்டமைக்கப்பட்ட பிராஂப்ட் அமைப்பு*

**சுய-மதிப்பீட்டின் மூலம் தரம்**

சுய-பரிசீலனை மாதிரிகள் தரத் தரக்குறியீடுகளை தெளிவுபடுத்துவதன் மூலம் செயல்படுகின்றன. மாடல் "சரி செய்கிறது" என்று நம்புவதற்குப் பதிலாக, நீங்கள் சரியானது என்ன என்பது: சரியான தர்க்கம், பிழை கையாளுதல், செயல்திறன், மற்றும் பாதுகாப்பு என்று தெளிவுபடுத்துகிறீர்கள். அதன் பிறகு மாடல் தன்னுடைய கீழமைப்பை மதிப்பாய்வு செய்து மேம்படுத்த முடியும். இதனால் குறியீடு உருவாக்கம் சாதனையுடைய செயல்முறையாக மாறுகிறது.

**சரதிகட்டம் எல்லை உள்ளது**

பல-முறை உரையாடல்கள் ஒவ்வொரு கேட்டலும் செய்தி வரலாற்றை உள்ளடக்கியிருப்பதால் செயல்படுகின்றன. ஆனால் எல்லை உள்ளது - ஒவ்வொரு மாடலும் அதிகபட்ச டோக்கன் எண்ணிக்கை கொண்டது. உரையாடல்கள் பெருகும்போது, சார் தொடர்புடைய சூழலை தக்கவைத்துக் கொள்ள முயற்சிகள் தேவைப்படும். இந்த மொடைல் நினைவோடு எப்படி வேலை செய்கிறது என்பதை காட்டுகிறது; பிறகு எப்போது சுருக்க வேண்டும், எப்போது மறக்க வேண்டும், மற்றும் எப்போது மீட்டெடுக்க வேண்டும் என்பதைக் கற்கப்போகிறீர்கள்.
