## நீங்கள் கற்பீர்கள்

<img src="../../../translated_images/ta/what-youll-learn.c68269ac048503b2.webp" alt="What You'll Learn" width="800"/>

கடந்த மோட்யூலில், நினைவகத்தின் மூலம் உரையாடல் AI எப்படி இயங்குகிறது என்பதைக் காண்பீர்கள் மற்றும் அடிப்படைக் கருத்துக்களுக்காக GitHub மாடல்கள் பயன்படுத்தினீர்கள். இப்போது, கேள்விகள் கேட்கும் வழி — புரம்ப்கள் தக்கவாறு அமைக்கும் முறை — Azure OpenAI இன் GPT-5.2 ஐப் பயன்படுத்தி கவனம் செலுத்துவோம். உங்கள் புரம்ப்களை அமைக்கும் முறை, நீங்கள் பெறும் பதில்களின் தரத்தை பதிலளிக்கும் விதத்தை பெரிதும் பாதிக்கிறது. முதலில் அடிப்படையான புரம்ப் தொழில்நுட்பங்களை மதிப்பீடு செய்வோம், பின்னர் GPT-5.2இன் முழு திறன்களை பயன்படுத்தும்  எட்டு மேம்பட்ட வடிவங்களை நோக்கிச் செல்லுவோம்.

GPT-5.2ஐப் பயன்படுத்துவோம், ஏனெனில் இது காரணித்தலை கட்டுப்படுத்துவதைக் கொண்டுவருகிறது - பதில் அளிப்பதற்கு முன் எவ்வளவு எண்ணம் செய்ய வேண்டும் என்று நீங்கள் கூறலாம். இது வித்தியாசமான புரம்பிங் வணிக நுட்பங்களைக் காட்டும் மற்றும் ஒவ்வொன்றும் எப்போது பயன்படுத்தப்பட வேண்டும் என்பதனைக் விளக்குகிறது. GitHub மாடல்களைவிட Azure இன் GPT-5.2க்கான குறைந்த வீதக் கட்டுப்பாடுகளிலும் நாம் நன்மை பெறுவோம்.

## புரம்ப் பொறியியலைப் புரிதல்

<img src="../../../translated_images/ta/what-is-prompt-engineering.5c392a228a1f5823.webp" alt="What is Prompt Engineering?" width="800"/>

புரம்ப் பொறியியல் என்பது நீங்கள் தேவையான முடிவுகளை நிலைத்தன்மையுடன் பெற, உள்ளீடான உரையை வடிவமைப்பதாகும். கேள்விகள் கேட்கும் பொது மட்டும் அல்ல - கோரிக்கைகள் அமைப்பது என்பதுடன், மாடல் நீங்கள் எதை வேண்டுகிறீர்கள் மற்றும் எப்படி வழங்க வேண்டும் என்பதை சரியாகப் புரிந்து கொள்ள உதவுவதாகும்.

இதை ஒரே கால நிறுவனத்துடன் வழிமுறை கொடுக்கும் போல நினைத்துக் கொள்ளுங்கள். "பிழையை சரிசெய்" என்பது अस्पष्टம். "UserService.java பகுதியில் 45-வது வரியில் நல் சரிபார்ப்பைச் சேர்த்து நல் பாயிண்டர் தவறை சரி செய்" என்றால் விரிவாகவும் தெளிவாகவும் இருக்கும். மொழி மாடல்களும் இதே மாதிரியே இயங்கும் - தெளிவும் அமைப்பும் தான் முக்கியம்.

<img src="../../../translated_images/ta/how-langchain4j-fits.dfff4b0aa5f7812d.webp" alt="How LangChain4j Fits" width="800"/>

LangChain4j கட்டமைப்பை வழங்குகிறது — மாடல் தொடர்புகள், நினைவு மற்றும் செய்தி வகைகள் — மற்றும் புரம்பின் வடிவங்கள், அந்த கட்டமைப்பின் வாயிலாக அனுப்பப்படும் கவனித்துக் கட்டமைக்கப்பட்ட உரை மட்டும். முக்கிய கட்டுமானக் கட்டிகள் `SystemMessage` (AI நடத்தை மற்றும் பங்கு அமைக்கிறது) மற்றும் `UserMessage` (உங்கள் உண்மை கோரிக்கையை எடுத்துச் செல்கிறது).

## புரம்ப் பொறியியல் அடிப்படைகள்

<img src="../../../translated_images/ta/five-patterns-overview.160f35045ffd2a94.webp" alt="Five Prompt Engineering Patterns Overview" width="800"/>

இந்த மோடியூலில் மேம்பட்ட வடிவங்களுக்குள் முன்கூட்டியே நுழைவதற்கு முன், ஐந்து அடிப்படையான புரம்ப் முறைகளை மறுபரிட்சை செய்யலாம். இவை ஒவ்வொரு புரம்ப் பொறியாளரும் அறிந்திருக்கும் கட்டுமானக் கட்டalai. நீங்கள் ஏற்கனவே [குவிக் ஸ்டார்ட் மோடியூல்](../00-quick-start/README.md#2-prompt-patterns) மூலம் இதை பார்த்திருக்கலாம் — இவை எவ்வாறு இயங்குகின்றன என்பதன் கருதுகோள் அட்டவணை இதுவே.

### ஜீரோ-ஷாட் புரம்பிங்

எளிய முறையாக: எந்த உதாரணங்களும் இல்லாமல் நேரடி கட்டளை மாடலுக்கு கொடுங்கள். மாடல் உதவி இல்லாமல் மட்டும் பயிற்சியில் இருந்து நுட்பத்தைப் புரிந்து செயல்படுத்துகிறது. இது தெளிவான கோரிக்கைகளுக்கு சிறந்தது.

<img src="../../../translated_images/ta/zero-shot-prompting.7abc24228be84e6c.webp" alt="Zero-Shot Prompting" width="800"/>

*எடுத்துக்காட்டுகள் இல்லாமல் நேரடி கட்டளை — மாடல் கட்டளையை தனக்கே புரிந்து செயல் படுத்துக*

```java
String prompt = "Classify this sentiment: 'I absolutely loved the movie!'";
String response = model.chat(prompt);
// பதில்: "நின்மையான"
```

**பயன்படுத்தும் நேரம்:** எளிய வகைப்படுத்தல்கள், நேரடி கேள்விகள், மொழிபெயர்ப்புகள் அல்லது கூடுதல் வழிகாட்டி இல்லாத எந்த பணிகளுக்கும்.

### ஃபியூ-ஷாட் புரம்பிங்

நீங்கள் வேண்டிய வடிவத்தை மாடல் பின்பற்ற உதாரணங்களை கொடுங்கள். மாடல் உங்கள் எடுத்துக்காட்டுகளில் இருந்து எதிர்பார்க்கப்படும் உள்ளீடு-வெளியீடு வடிவத்தைக் கற்றுக்கொண்டு புதிய உள்ளீடுகளுக்கு அதை பயன்படுத்தும். இது குறிப்பிட்ட வடிவம் அதிக சீரமைப்புக்கு உதவுகிறது.

<img src="../../../translated_images/ta/few-shot-prompting.9d9eace1da88989a.webp" alt="Few-Shot Prompting" width="800"/>

*எடுத்துக்காட்டுகளிலிருந்து கற்றல் — மாடல் இருக்குள் வடிவத்தை அடையாளம் கண்டு புதிய உள்ளீடுகளுக்கு அது பயன்படுத்தும்*

```java
String prompt = """
    Classify the sentiment as positive, negative, or neutral.
    
    Examples:
    Text: "This product exceeded my expectations!" → Positive
    Text: "It's okay, nothing special." → Neutral
    Text: "Waste of money, very disappointed." → Negative
    
    Now classify this:
    Text: "Best purchase I've made all year!"
    """;
String response = model.chat(prompt);
```

**பயன்படுத்தும் நேரம்:** தனிப்பயன் வகைப்படுத்தல்கள், நிலையான வடிவமைப்பு, பகுதி-துறை பணிகள், அல்லது ஜீரோ-ஷாட் முடிவுகள் ஒரே மாதிரியாக இல்லாத போது.

### தொடரியல் எண்ணம்

மாடலை படிநிலை படியான காரணத்தை காட்டச் சொல்க. பதிலை நேரடியாகச் சேர்க்காமல், மாடல் பிரச்சனைவைப் பிரித்து ஒவ்வொரு பகுதிக்கும் விதமாக வேலை செய்கிறது. இது கணிதம், தார்க்கிகம் மற்றும் பல படி காரண задачல் என்பதை சரியாக செய்ய உதவும்.

<img src="../../../translated_images/ta/chain-of-thought.5cff6630e2657e2a.webp" alt="Chain of Thought Prompting" width="800"/>

*படி படியாக கூறுதல் — சிக்கலான பிரச்சனைகளை தார்க்கிகமான வழிகளில் உடைத்தல்*

```java
String prompt = """
    Problem: A store has 15 apples. They sell 8 apples and then 
    receive a shipment of 12 more apples. How many apples do they have now?
    
    Let's solve this step-by-step:
    """;
String response = model.chat(prompt);
// மாதிரி காட்டுகிறது: 15 - 8 = 7, பின்னர் 7 + 12 = 19 ஆப்பிள்கள்
```

**பயன்படுத்தும் நேரம்:** கணிதப் பிரச்சனைகள், தர்க்கத் துருவங்கள், பிழைத்திருத்தம் அல்லது காரணித் திட்டத்தின் காட்டுதல் தேவைப்படும் பணிகளில்.

### பங்கு அடிப்படையிலான புரம்பிங்

உங்கள் கேள்விக்கு முன் AIக்கு ஒரு பெர்சோனா அல்லது பங்கு கொடுக்கவும். இது பதிலின் சுருள், ஆழம் மற்றும் கவனத்தை வடிவமைக்கும். "சாஃப்ட்வேர்அர்கிடெக்ட்" ஒரு "ஜூனியர் டெவலப்பர்" அல்லது "பாதுகாப்பு ஆய்வாளர்" என்றவர்களிலிருந்து வேறுபட்ட ஆலோசனையை தரும்.

<img src="../../../translated_images/ta/role-based-prompting.a806e1a73de6e3a4.webp" alt="Role-Based Prompting" width="800"/>

*சூழல் மற்றும் பெர்சோனா அமைத்தல் — அதே கேள்விக்கு, பிற பங்குகளில் வேறுபட்ட பதில்கள்*

```java
String prompt = """
    You are an experienced software architect reviewing code.
    Provide a brief code review for this function:
    
    def calculate_total(items):
        total = 0
        for item in items:
            total = total + item['price']
        return total
    """;
String response = model.chat(prompt);
```

**பயன்படுத்தும் நேரம்:** கோடு மதிப்பீடுகள், பயிற்சி, பகுதி-துறை பகுப்பாய்வு, அல்லது குறிப்பிட்ட திறனின் படி பதில்கள் தேவையான பொழுது.

### புரம்ப் மாதிரிகள்

மீண்டும் பயன்படுத்தக்கூடிய புரம்ப்களை மாற்றங்களுடன் உருவாக்கு. ஒவ்வொரு முறையும் புதிய புரம்ப் எழுதாமல், ஒரு மாதிரியை ஒருமுறை உருவாக்கி, வெவ்வேறு மதிப்புகளை நிரப்பவும். LangChain4j இன் `PromptTemplate` வகுப்பு இதை எளிதாக்குகிறது `{{variable}}` சின்டாக்ஸ் மூலம்.

<img src="../../../translated_images/ta/prompt-templates.14bfc37d45f1a933.webp" alt="Prompt Templates" width="800"/>

*மாற்றங்களுடன் மீண்டும் பயன்படுத்தக்கூடிய புரம்ப்கள் — ஒரு மாதிரி, பல பயன்பாடுகள்*

```java
PromptTemplate template = PromptTemplate.from(
    "What's the best time to visit {{destination}} for {{activity}}?"
);

Prompt prompt = template.apply(Map.of(
    "destination", "Paris",
    "activity", "sightseeing"
));

String response = model.chat(prompt.text());
```

**பயன்படுத்தும் நேரம்:** வெவ்வேறு உள்ளீடுகளுடன் மீண்டும் கேள்விகள், தொகுதி செயலாக்கம், மீண்டும் பயன்படுத்தக்கூடிய AI வேலைவளங்கள், அல்லது புரம்ப் அமைப்பு ஒரே மாதிரியாக இருக்கும் ஆனால் தரவு மாறுகிறது என்றால்.

---

இந்த ஐந்து அடிப்படைகள் பெரும்பான்மையிலும் புரம்ப் பணிகளுக்கு நல்ல கருவி தொகுப்பாக இருக்கும். இந்த மோடியூல் மீதமுள்ளவை, GPT-5.2 இன் காரணித்தலை கட்டுப்பாடு, சுய மதிப்பீடு மற்றும் கட்டமைக்கப்பட்ட வெளியீடு திறன்களை பயன்படுத்தும் **எட்டு மேம்பட்ட வடிவங்களில்** விரிவடைகிறது.

## மேம்பட்ட வடிவங்கள்

அடிப்படைகள் முடிந்ததும், இந்த மோடியூலை தனி வகையாக்கும் எட்டு மேம்பட்ட வடிவங்களுக்கு செல்லலாம். அனைத்து பிரச்சனைகளுக்கும் ஒரே முறையையோ தேவையில்லை. சில கேள்விகள் விரைவான பதில்களைக் கேட்கும், மற்றவை ஆழமான சிந்தனையைத் தேவைப்படுகிறது. சிலவை காரணத்தை காண்பிக்க வேண்டும், சிலக்கு மட்டும் முடிவுகள் வேண்டும். கீழ்காணும் ஒவ்வொரு வடிவமும் தனித்த சூழலுக்கே ஏற்ப சிறப்படுத்தப்பட்டவை — GPT-5.2 இன் காரணித்தலை கட்டுப்பாடு வித்தியாசங்களை இன்னும் தெளிவாகக் காண உதவுகிறது.

<img src="../../../translated_images/ta/eight-patterns.fa1ebfdf16f71e9a.webp" alt="Eight Prompting Patterns" width="800"/>

*எட்டு புரம்ப் பொறியியல் வடிவங்களின் நேர்முகம் மற்றும் அவற்றின் பயன்பாடுகள்*

<img src="../../../translated_images/ta/reasoning-control.5cf85f0fc1d0c1f3.webp" alt="Reasoning Control with GPT-5.2" width="800"/>

*GPT-5.2 இன் காரணித்தலை கட்டுப்பாடு, நீங்கள் மாடல் எவ்வளவு எண்ணம் செய்ய வேண்டும் என்பதை குறிப்பிட உதவுகிறது — விரைவு நேரடி பதில்களிடமிருந்து ஆழமான ஆராய்ச்சிவரை*

**குறைந்த ஆவல் (விரைவு மற்றும் கவனமானது)** - எளிய கேள்விகளுக்குத் துரிதமான, நேரடிப் பதில்கள் வேண்டும் என்றால். மாடல் குறைந்த காரணித்தலை (அதிகபட்சம் 2 படிகளுக்கு) செயல்படுத்தும். கணக்கீடுகள், தேடல்கள் அல்லது தெளிவான கேள்விகளுக்கு இது பயன்படுத்தவும்.

```java
String prompt = """
    <context_gathering>
    - Search depth: very low
    - Bias strongly towards providing a correct answer as quickly as possible
    - Usually, this means an absolute maximum of 2 reasoning steps
    - If you think you need more time, state what you know and what's uncertain
    </context_gathering>
    
    Problem: What is 15% of 200?
    
    Provide your answer:
    """;

String response = chatModel.chat(prompt);
```

> 💡 **GitHub Copilot உடன் ஆராயவும்:** திறந்துகொள்ளவும் [`Gpt5PromptService.java`](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java) மற்றும் கேளுங்கள்:
> - "குறைந்த ஆவல் மற்றும் அதிக ஆவல் புரம்ப் வடிவங்கள் வேறுபாடு என்ன?"
> - "புரம்ப்களில் XML குறிச்சொற்கள் AI பதிலை அமைக்க எப்படி உதவுகின்றன?"
> - "நான் எப்போது சுய-பரிசீலனை வடிவங்களை நேரடி கட்டளைகளுடன் ஒப்பிட வேண்டும்?"

**உயர் ஆவல் (ஆழமான மற்றும் விரிவானது)** - சிக்கலான பிரச்சனைகளுக்கு முழுமையான பகுப்பாய்வு வேண்டுமானால். மாடல் முழுமையாக ஆராய்ந்து விரிவான காரணித்தலை காட்டும். அமைப்பு வடிவமைப்பு, கட்டமைப்பு முடிவுகள் அல்லது சிக்கலான ஆராய்ச்சிக்கு பயன்படுத்தவும்.

```java
String prompt = """
    Analyze this problem thoroughly and provide a comprehensive solution.
    Consider multiple approaches, trade-offs, and important details.
    Show your analysis and reasoning in your response.
    
    Problem: Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```

**பணிப் பின்பற்றல் (படி படி முன்னேற்றம்)** - பல படி வேலைவிளைவுகளுக்கு. மாடல் முன்னரே திட்டமிடி, ஒவ்வொரு படியையும் கூறி வேலை செய்யும், பின்னர் சுருக்கத்தை அளிக்கும். இடமாற்றங்கள், செயல்படுத்தல்கள் அல்லது எந்த பல படி செயலčné process உகந்தது.

```java
String prompt = """
    <task_execution>
    1. First, briefly restate the user's goal in a friendly way
    
    2. Create a step-by-step plan:
       - List all steps needed
       - Identify potential challenges
       - Outline success criteria
    
    3. Execute each step:
       - Narrate what you're doing
       - Show progress clearly
       - Handle any issues that arise
    
    4. Summarize:
       - What was completed
       - Any important notes
       - Next steps if applicable
    </task_execution>
    
    <tool_preambles>
    - Always begin by rephrasing the user's goal clearly
    - Outline your plan before executing
    - Narrate each step as you go
    - Finish with a distinct summary
    </tool_preambles>
    
    Task: Create a REST endpoint for user registration
    
    Begin execution:
    """;

String response = chatModel.chat(prompt);
```

தொடரியல் எண்ணம் புரம்பிங் மாடல் காரணிப்பு செயல்முறையை வெளிப்படையாகக் காட்டச் சொல்கிறது, இது சிக்கலான பணிகளுக்குத் துல்லியத்தைக் கூட்டுகிறது. படிநிலை உடைப்பு மனிதர்களுக்கும் AIக்கும் தர்க்கத்தைப் புரிதலில் உதவும்.

> **🤖 [GitHub Copilot](https://github.com/features/copilot) உரையாடலுடன் முயற்சி செய்யவும்:** இந்த வடிவத்தைப் பற்றி கேளுங்கள்:
> - "நீண்ட கால செயல்பாடுகளுக்கு பணிப் பின்பற்றல் வடிவத்தை எப்படி பொருந்தச் செய்யலாம்?"
> - "உற்பத்தி பயன்பாடுகளில் கருவி முன்னுரைகள் அமைப்பு சிறந்த நடைமுறைகள் என்ன?"
> - "UIவில் இடைநிலை முன்னேற்றத்தை பிடித்து காட்சிப்படுத்துவது எப்படி?"

<img src="../../../translated_images/ta/task-execution-pattern.9da3967750ab5c1e.webp" alt="Task Execution Pattern" width="800"/>

*திட்டமிடு → செயல்படு → சுருக்கம் வகைமுறை பல படி பணிகளுக்கு*

**சுய-பரிசீலனை கோடு** - உற்பத்தி தரம் கூடிய கோடுகளை உருவாக்க. மாடல் உற்பத்தி தர நெருங்கியுடன் கோடுகளை உருவாக்கும், பொருத்தமான பிழை கையாளுதலும் உடன். புதிய அம்சங்கள் அல்லது சேவைகள் கட்டுமானத்தில் பயன்படுத்தவும்.

```java
String prompt = """
    Generate Java code with production-quality standards: Create an email validation service
    Keep it simple and include basic error handling.
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/ta/self-reflection-cycle.6f71101ca0bd28cc.webp" alt="Self-Reflection Cycle" width="800"/>

*முறைசார்ந்த மேம்பாடு சுற்று - உருவாக்கு, மதிப்பாய்வு செய், பிரச்சனைகள் கண்டறி, மேம்படு, மீண்டும் செய்*

**கட்டமைக்கப்பட்ட பகுப்பாய்வு** - நிலையான மதிப்பீடு. மாடல் கோடுகளை ஒரு சரியான கட்டமைப்பில் மதிப்பீடு செய்கிறது (தவறுதலியல், நடைமுறைகள், செயல்திறன், பாதுகாப்பு, பராமரிப்பு). கோடு மதிப்பீடுகள் அல்லது தரம் மதிப்பீடுகளுக்கு பயன்படுத்தவும்.

```java
String prompt = """
    <analysis_framework>
    You are an expert code reviewer. Analyze the code for:
    
    1. Correctness
       - Does it work as intended?
       - Are there logical errors?
    
    2. Best Practices
       - Follows language conventions?
       - Appropriate design patterns?
    
    3. Performance
       - Any inefficiencies?
       - Scalability concerns?
    
    4. Security
       - Potential vulnerabilities?
       - Input validation?
    
    5. Maintainability
       - Code clarity?
       - Documentation?
    
    <output_format>
    Provide your analysis in this structure:
    - Summary: One-sentence overall assessment
    - Strengths: 2-3 positive points
    - Issues: List any problems found with severity (High/Medium/Low)
    - Recommendations: Specific improvements
    </output_format>
    </analysis_framework>
    
    Code to analyze:
    ```
    public List getUsers() {
        return database.query("SELECT * FROM users");
    }
    ```
    Provide your structured analysis:
    """;

String response = chatModel.chat(prompt);
```

> **🤖 [GitHub Copilot](https://github.com/features/copilot) உரையாடலுடன் முயற்சி செய்யவும்:** கட்டமைக்கப்பட்ட பகுப்பாய்வைப் பற்றி கேளுங்கள்:
> - "விவித கோடு மதிப்பீடு வகைகளுக்கு பகுப்பாய்வு கட்டமைப்பை எப்படி தனிப்பயனாக்கலாம்?"
> - "கட்டமைக்கப்பட்ட வெளியீட்டை நிரல் மூலம் எவ்வாறு பாகுபடுத்தி செயல்படுத்துவது?"
> - "பல்வேறு மதிப்பீடு அமர்வுகளில் ஒரே மாதிரி தீவிரத்தன்மையை எப்படிப் பராமரிப்பது?"

<img src="../../../translated_images/ta/structured-analysis-pattern.0af3b690b60cf2d6.webp" alt="Structured Analysis Pattern" width="800"/>

*தீவிரம் மட்டங்களை உள்ளடக்கிய நிலையான கோடு மதிப்பீடு கட்டமைப்பு*

**பல்வேறு சுற்றுச்சட்ட உரையாடல்** - சூழலை தேவைப்படுத்தும் உரையாடல்கள். மாடல் முந்தைய செய்திகள் நினைவில் வைக்கும் மற்றும் அவற்றில் கட்டமைக்கும். தொடர்புடைய உதவி அமர்வுகளுக்கும் சிக்கலான கேள்வி & பதில்களுக்கும் பயன்படுத்தவும்.

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("What is Spring Boot?"));
AiMessage aiMessage1 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage1);

memory.add(UserMessage.from("Show me an example"));
AiMessage aiMessage2 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage2);
```

<img src="../../../translated_images/ta/context-memory.dff30ad9fa78832a.webp" alt="Context Memory" width="800"/>

*பல சுற்றுகளில் உரையாடல் சூழல் தொகுப்பது, டோக்கன் வரம்புக்கு வரைக்கும்*

**படி படியான காரணித்தல்** - காணக்கூடிய தர்க்கம் தேவைப்படும் பிரச்சனைகள். மாடல் ஒவ்வொரு படிக்கும் தெளிவான காரணத்தை காட்டும். கணிதப் பிரச்சனைகள், தர்க்கத் தொடர் வேட்டை அல்லது எண்ணத் திட்டத்தைப் புரிந்துகொள்ள வேண்டும் என்பதற்கு பயன்படுத்தவும்.

```java
String prompt = """
    <instruction>Show your reasoning step-by-step</instruction>
    
    If a train travels 120 km in 2 hours, then stops for 30 minutes,
    then travels another 90 km in 1.5 hours, what is the average speed
    for the entire journey including the stop?
    """;

String response = chatModel.chat(prompt);
```

**கட்டுப்படுத்தப்பட்ட வெளியீடு** - குறிப்பிட்ட வடிவமைப்புகளுடன் பதிலளிக்கும். மாடல் முடுக்கப்பட்ட வடிவமைப்பு மற்றும் நீளம் விதிமுறைகளை கடைப்பிடிக்கும். சுருக்கங்கள் அல்லது துல்லியமான வெளியீடு கட்டமைப்பு தேவைப்படும் போது பயன்படுத்து.

## முறைகளை ஆராய்தல்

வலை இடைமுகம் நீங்கள் வெவ்வேறு ப்ராம்ப்டிங் உத்திகளுடன் பயிற்சி செய்ய அனுமதிக்கிறது. ஒவ்வொரு முறைமும் வேறு பிரச்சனைகளை தீர்க்கிறது - எந்த அணுகுமுறை சிறந்தது என்பதை பாருங்கள்.

> **குறிப்பு: ஸ்ட்ரீமிங் மற்றும் ஸ்ட்ரீமிங் அல்லாதவை** — ஒவ்வொரு முறையும் இரண்டு பொத்தான்கள் வழங்கப்படுகின்றன: **🔴 ஸ்ட்ரீம் பதில் ( நேரடி)** மற்றும் **ஸ்ட்ரீம் அல்லாத** விருப்பம். ஸ்ட்ரீமிங் Server-Sent Events (SSE) பயன்படுத்தி மாடல் உருவாக்கும் டோக்கன்களை நேரடியாக காட்டும், ஆகவே முன்னிலையை உடனுக்குடன் காணலாம். ஸ்ட்ரீம் அல்லாத விருப்பம் முழுமையான பதிலை காத்திருக்கிறது. ஆழ்ந்த சிந்தனையைத் தூண்டும் ப்ராம்ப்ட்களுக்கு (எ.கா., உயர் ஆர்வம், சுய-தன்னிருப்பு குறியீடு), ஸ்ட்ரீம் அல்லாத அழைப்பு மிகவும் நீண்ட நேரம் ஆகலாம் — சில நேரங்களில் நிமிடங்கள் — எந்தவொரு பார்வைக்கு உடனடி பதிலும் கிடையாது. **கடினமான ப்ராம்ப்ட்களுடன் பரிசோதனை செய்யும்போது ஸ்ட்ரீமிங் பயன்படுத்தவும்** அது மாடல் பணியாற்றுவதை காண உதவும் மற்றும் கோரிக்கைகள் காலாவதியாகிவிட்டது என்பது போன்ற ஒரு தவறான கண்ணோட்டத்தை தவிர்க்கும்.
>
> **குறிப்பு: உலாவி தேவைகள்** — ஸ்ட்ரீமிங் அம்சம் Fetch Streams API (`response.body.getReader()`) பயன்படுத்துகிறது, இது முழு உலாவியை (Chrome, Edge, Firefox, Safari) வேண்டுகிறது. VS Code இல் உள்ள Simple Browser இல் இது வேலை செய்யாது, அதன் வலைவிழா ReadableStream API ஐ ஆதரிக்காது. Simple Browser பயன்படுத்தினால், ஸ்ட்ரீம் அல்லாத பொத்தான்கள் சாதாரணமாக செயல்படும் — ஸ்ட்ரீமிங் பொத்தான்கள் மட்டுமே பாதிக்கப்படும். முழு அனுபவத்திற்கு `http://localhost:8083` ஐ வெளிப்புற உலாவியில் திறக்கவும்.

### குறைந்த ஆர்வம் vs உயர் ஆர்வம்

"200 இன் 15%" என்ற எளிய கேள்வியை குறைந்த ஆர்வம் கொண்டு கேளுங்கள். நீங்கள் உடனடி நேரடி பதிலை பெறுவீர்கள். இப்போது "உயர் போக்குவரத்து APIக்கான கேஷிங் திட்டம் வடிவமைக்கவும்" போன்ற சிக்கலான கேள்வியை உயர் ஆர்வம் கொண்டு கேளுங்கள். **🔴 ஸ்ட்ரீம் பதில் (நேரடி)** என்பதைக் கிளிக் செய்து மாடலின் விரிவான சிந்தனையை டோக்கன்தோடடோக்கனில் காணுங்கள். ஒரே மாடல், ஒரே கேள்வி வடிவமைப்பு - ஆனால் ப்ராம்ப்ட் அதற்கு எவ்வளவு சிந்திக்க வேண்டும் என்று சொல்கிறது.

### பணிச்செயல் செயல்பாடு (கருவி முன்னோட்டங்கள்)

பல படி கூடிய பணிகள் முன்னாள் திட்டமிடலை மற்றும் முன்னேற்றக்கதை சொல்லுதலைப் பெறுகின்றன. மாடல் என்ன செய்யப்போகிறது, ஒவ்வொரு படியும் செய்யும் செயல்களை துண்டுக்கின்றது, பின்னர் முடிவுகளை சுருக்குகின்றது.

### சுய-தன்னிருப்பு குறியீடு

"மின்னஞ்சல் சரிபார்ப்பு சேவை உருவாக்கவும்" ஐ முயற்சி செய்யவும். குறியீடு உருவாக்கி நிறுத்தாமல், மாடல், தரம் அளவுருக்களின் அடிப்படையில் மதிப்பீடு செய்கிறது, பலவீனங்களை கண்டறிந்து, மேம்படுத்துகிறது. குறியீடு தயாராகும் வரை மீண்டும் மீண்டும் செயல் செய்யும்.

### கட்டமைக்கப்பட்ட பகுப்பாய்வு

குறியீடு வியூஹங்கள் சீரான மதிப்பீடு தளவமைப்புகளைத் தேவைப்படுத்துகின்றன. மாடல் குறியீட்டை நிரந்தர வகுப்புக்களுடன் (தரமாக்கல், நடைமுறைபடி, செயல்திறன், பாதுகாப்பு) மலிவான அளவுகளுடன் பகுப்பாய்வு செய்கிறது.

### பல-தகராறு உரையாடல்

"Spring Boot என்றால் என்ன?" என்று கேளுங்கள், பின்னர் உடனடியாய் "ஒரு உதாரணம் காட்டவும்" என்ற கேள்வியையும் கேளுங்கள். மாடல் உங்கள் முதல் கேள்வியை மறக்கவில்லை மற்றும் குறிப்பாக Spring Boot உதாரணம் தருகிறது. நினைவில்லை என்றால் இரண்டாம் கேள்வி மிகவும் பொதுவானதாக இருக்கும்.

### படி படியாக சிந்தித்தல்

ஒரு கணிதவினையை தேர்ந்தெடுத்து அதனை படி படியாக சிந்தித்தலும் குறைந்த ஆர்வமும் கொண்டு முயற்சிக்கவும். குறைந்த ஆர்வம் விரைந்து பதிலை தரும் ஆனால் தெளிவற்றது. படி படியானது ஒவ்வொரு கணக்கீடையும் முடிவையும் காட்டும்.

### கட்டுப்படுத்தப்பட்ட வெளியீடு

குறிப்பிட்ட வடிவங்கள் அல்லது சொற்படிகளின் எண்ணிக்கை தேவைப்பட்டால் இந்த முறை மிகக் கடுமையாக ஒத்துப்போகிறது. சரியாக 100 சொற்களுடன் புள்ளி வடிவ வடிவமைப்பில் சுருக்கத்தை உருவாக்க முயற்சி செய்யுங்கள்.

## நீங்கள் உண்மையில் கற்றுக் கொண்டிருக்கும் விஷயங்கள்

**சிந்தனை முயற்சி அனைத்தையும் மாற்றுகிறது**

GPT-5.2 நீங்கள் உங்கள் ப்ராம்ப்ட் வழியாக கணினிப் பணி முயற்சியைக் கட்டுப்படுத்த அனுமதிக்கிறது. குறைந்த முயற்சி விரைவான பதில்களைக் குறைந்த ஆராய்ச்சியுடன் வழங்குகிறது. அதிக முயற்சி ஆழ்ந்த சிந்தனை நேரத்தை எடுத்துச் செல்கிறது. நீங்கள் முயற்சியை பணிக்கேற்றவாறு பொருந்துகிறீர்கள் - எளிய கேள்விகளுக்கு நேரத்தை வீணாக்க வேண்டாம், அதே நேரம் கடின முடிவுகளை வேகமாக செய்யவும் வேண்டாம்.

**வடிவமைப்பு நடத்தை வழிநடத்துகிறது**

ப்ராம்ப்ட்களில் உள்ள XML குறிச்சொற்களைக் கவனித்தீர்களா? அவை அழகுக்காக அல்ல. மாடல்கள் அமைப்புக்குரிய விளக்கங்களைக் கடைப்பிடிப்பதன் மூலம் கூர்மையாக செயல்படுகின்றன. பல படி செயல்முறைகள் அல்லது கட்டிய தர்க்கங்கள் தேவைப்படும்போது வடிவமைப்பு மாடல் எங்கே இருக்கிறது மற்றும் அடுத்ததாக என்ன வருகிறது என்பதை கண்காணிக்க உதவுகிறது.

<img src="../../../translated_images/ta/prompt-structure.a77763d63f4e2f89.webp" alt="Prompt Structure" width="800"/>

*தெளிவான பிரிவுகளும் XML-போன்ற அமைப்பும் கொண்ட நன்கு கட்டமைக்கப்பட்ட ப்ராம்ப்டின் அமைப்பு*

**தானே மதிப்பீடு மூலம் தரம்**

சுய-தன்னிருப்பு முறைமைகள் தரமான அளவுருக்களை தெளிவாக்குவதன் மூலம் செயல்படுகின்றன. மாடல் "சரி" என்று நம்பாமல், நீங்கள் "சரி" என்பது என்ன என்பதை அதற்குக் கூறுகிறீர்கள்: சரியான தர்க்கம், பிழை கையாளுதல், செயல்திறன், பாதுகாப்பு. பின்னர் மாடல் தன்னுடைய வெளியீட்டை மதிப்பீடு செய்து மேம்படுத்தலாம். இது குறியீடு உருவாக்கத்தை ஒரு வெற்றிகரமான செயல்முறையாக மாற்றுகிறது.

**சூழல் எல்லை கிட்டத்தட்ட**

பல தடவை உரையாடல்கள் ஒவ்வொரு கோரிக்கையுடன் செய்தி வரலாறைச் சேர்க்கிறது. ஆனால் ஒரு வரம்பு உண்டு - ஒவ்வொரு மாடலுக்கும் அதிகபட்ச டோக்கன் எண்ணிக்கை உள்ளது. உரையாடல்கள் பெருகி வருவதால், சம்பந்தப்பட்ட சூழலை அக்கறையோடு வைத்திருப்பதற்கான உத்திகள் தேவைப்படும், அந்த வரம்பைத் தாண்டாமல். இந்த மொடியுல் நினைவாற்றல் எப்படி செயல்படுகிறது என்பதைக் கற்பிக்கிறது; பின்னர் எப்போது சுருக்கம் செய்ய வேண்டும், எப்போது மறக்க வேண்டும், எப்போது மீட்டெடுக்க வேண்டும் என்பதை கற்றுக்கொள்ளுவீர்கள்.
