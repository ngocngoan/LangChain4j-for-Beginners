## แนวคิดหลัก

**เอเยนต์ AI** - ระบบที่ใช้ AI ในการให้เหตุผลและทำงานโดยอิสระ [โมดูล 04](../04-tools/README.md)

**โซ่** - ลำดับของขั้นตอนที่ผลลัพธ์จากขั้นตอนหนึ่งถูกป้อนเข้าสู่ขั้นตอนถัดไป

**การแบ่งส่วน (Chunking)** - การแบ่งเอกสารเป็นชิ้นเล็ก ๆ ขนาดปกติ: 300-500 โทเคน พร้อมทับซ้อน [โมดูล 03](../03-rag/README.md)

**หน้าต่างบริบท** - จำนวนโทเคนสูงสุดที่โมเดลสามารถประมวลผล GPT-5.2: 400K โทเคน (สูงสุด 272K อินพุต, 128K เอาต์พุต)

**Embeddings** - เวกเตอร์เชิงตัวเลขที่แสดงความหมายของข้อความ [โมดูล 03](../03-rag/README.md)

**การเรียกฟังก์ชัน (Function Calling)** - โมเดลสร้างคำร้องแบบมีโครงสร้างเพื่อเรียกฟังก์ชันภายนอก [โมดูล 04](../04-tools/README.md)

**การหลอกลวง (Hallucination)** - เมื่โมเดลสร้างข้อมูลที่ไม่ถูกต้องแต่ดูสมเหตุสมผล

**Prompt** - ข้อความนำเข้าให้กับโมเดลภาษา [โมดูล 02](../02-prompt-engineering/README.md)

**การค้นหาทางความหมาย (Semantic Search)** - การค้นหาด้วยความหมายโดยใช้ embeddings แทนคีย์เวิร์ด [โมดูล 03](../03-rag/README.md)

**สถานะมีและไม่มีสถานะ (Stateful vs Stateless)** - ไม่มีสถานะ: ไม่มีหน่วยความจำ มีสถานะ: เก็บประวัติการสนทนา [โมดูล 01](../01-introduction/README.md)

**โทเคน** - หน่วยข้อความพื้นฐานที่โมเดลประมวลผล มีผลต่อค่าใช้จ่ายและข้อจำกัด [โมดูล 01](../01-introduction/README.md)

**การเชื่อมโยงเครื่องมือ (Tool Chaining)** - การเรียกใช้งานเครื่องมือเป็นลำดับโดยผลลัพธ์ใช้ในการเรียกครั้งถัดไป [โมดูล 04](../04-tools/README.md)

## ส่วนประกอบของ LangChain4j

**AiServices** - สร้างอินเทอร์เฟซบริการ AI ที่ปลอดภัยในประเภทข้อมูล

**OpenAiOfficialChatModel** - ไคลเอนต์รวมสำหรับโมเดล OpenAI และ Azure OpenAI

**OpenAiOfficialEmbeddingModel** - สร้าง embeddings โดยใช้ไคลเอนต์ OpenAI Official (รองรับทั้ง OpenAI และ Azure OpenAI)

**ChatModel** - อินเทอร์เฟซหลักสำหรับโมเดลภาษา

**ChatMemory** - เก็บประวัติการสนทนา

**ContentRetriever** - ค้นหาชิ้นส่วนเอกสารที่เกี่ยวข้องสำหรับ RAG

**DocumentSplitter** - แบ่งเอกสารเป็นชิ้น

**EmbeddingModel** - แปลงข้อความเป็นเวกเตอร์ตัวเลข

**EmbeddingStore** - เก็บและเรียกใช้ embeddings

**MessageWindowChatMemory** - ดูแลกลุ่มข้อความล่าสุดในหน้าต่างเลื่อน

**PromptTemplate** - สร้าง prompt ที่ใช้ซ้ำได้โดยมีที่ว่างสำหรับ `{{variable}}`

**TextSegment** - ชิ้นข้อความที่มีข้อมูลเมตา ใช้ใน RAG

**ToolExecutionRequest** - แทนคำขอการเรียกใช้งานเครื่องมือ

**UserMessage / AiMessage / SystemMessage** - ชนิดข้อความในการสนทนา

## แนวคิด AI/ML

**การเรียนรู้แบบ Few-Shot** - การให้ตัวอย่างใน prompt [โมดูล 02](../02-prompt-engineering/README.md)

**โมเดลภาษาใหญ่ (LLM)** - โมเดล AI ที่ผ่านการฝึกด้วยข้อมูลข้อความจำนวนมาก

**ความพยายามให้เหตุผล (Reasoning Effort)** - พารามิเตอร์ GPT-5.2 ควบคุมความลึกของการคิด [โมดูล 02](../02-prompt-engineering/README.md)

**อุณหภูมิ (Temperature)** - ควบคุมความสุ่มของผลลัพธ์ ต่ำ=กำหนด, สูง=สร้างสรรค์

**ฐานข้อมูลเวกเตอร์** - ฐานข้อมูลเฉพาะทางสำหรับ embeddings [โมดูล 03](../03-rag/README.md)

**การเรียนรู้แบบ Zero-Shot** - ทำงานโดยไม่ต้องมีตัวอย่าง [โมดูล 02](../02-prompt-engineering/README.md)

## เกราะป้องกัน - [โมดูล 00](../00-quick-start/README.md)

**Defense in Depth** - แนวทางความปลอดภัยหลายชั้นรวมเกราะโปรแกรมกับตัวกรองความปลอดภัยของผู้ให้บริการ

**Hard Block** - ผู้ให้บริการตอบข้อผิดพลาด HTTP 400 สำหรับการละเมิดเนื้อหารุนแรง

**InputGuardrail** - อินเทอร์เฟซ LangChain4j สำหรับตรวจสอบอินพุตผู้ใช้ก่อนถึง LLM ประหยัดค่าใช้จ่ายและเวลาตอบสนองโดยบล็อก prompt อันตรายตั้งแต่ต้น

**InputGuardrailResult** - ประเภทคืนสำหรับการตรวจสอบ guardrail: `success()` หรือ `fatal("reason")`

**OutputGuardrail** - อินเทอร์เฟซตรวจสอบคำตอบ AI ก่อนส่งกลับผู้ใช้

**ตัวกรองความปลอดภัยของผู้ให้บริการ** - ตัวกรองเนื้อหาติดตั้งจากผู้ให้บริการ AI (เช่น GitHub Models) ที่จับการละเมิดระดับ API

**Soft Refusal** - โมเดลปฏิเสธตอบอย่างสุภาพโดยไม่เกิดข้อผิดพลาด

## การออกแบบ Prompt - [โมดูล 02](../02-prompt-engineering/README.md)

**Chain-of-Thought** - การให้เหตุผลเป็นขั้นตอนเพื่อความแม่นยำที่ดีกว่า

**ผลลัพธ์ที่จำกัดรูปแบบ (Constrained Output)** - บังคับให้ผลลัพธ์มีรูปแบบหรือโครงสร้างเฉพาะ

**High Eagerness** - รูปแบบ GPT-5.2 สำหรับการคิดวิเคราะห์ลึกซึ้ง

**Low Eagerness** - รูปแบบ GPT-5.2 สำหรับคำตอบรวดเร็ว

**การสนทนาแบบหลายรอบ (Multi-Turn Conversation)** - การรักษาบริบทระหว่างการแลกเปลี่ยนหลายครั้ง

**Prompt ตามบทบาท (Role-Based Prompting)** - ตั้งบุคลิกของโมเดลผ่านข้อความระบบ

**การสะท้อนตนเอง (Self-Reflection)** - โมเดลประเมินและปรับปรุงผลลัพธ์ของตัวเอง

**การวิเคราะห์แบบมีโครงสร้าง (Structured Analysis)** - กรอบการประเมินที่ตั้งค่าไว้ล่วงหน้า

**รูปแบบการปฏิบัติงาน (Task Execution Pattern)** - วางแผน → ดำเนินการ → สรุปผล

## RAG (Retrieval-Augmented Generation) - [โมดูล 03](../03-rag/README.md)

**สายงานประมวลผลเอกสาร** - โหลด → แบ่งชิ้น → สร้าง embeddings → เก็บ

**ที่เก็บ embeddings ในหน่วยความจำ** - ที่เก็บไม่ถาวรสำหรับทดสอบ

**RAG** - รวมการดึงข้อมูลกับการสร้างคำตอบเพื่อยึดโยงคำตอบ

**คะแนนความคล้าย (Similarity Score)** - ตัวชี้วัดความคล้ายเชิงความหมาย (0-1)

**ข้อมูลอ้างอิงแหล่งที่มา** - ข้อมูลเมตาเกี่ยวกับเนื้อหาที่ดึงมา

## เอเยนต์และเครื่องมือ - [โมดูล 04](../04-tools/README.md)

**@Tool Annotation** - แท็กเมธอด Java เป็นเครื่องมือที่ AI สามารถเรียกได้

**ReAct Pattern** - ให้เหตุผล → ปฏิบัติ → สังเกต → ทำซ้ำ

**การจัดการเซสชัน** - บริบทแยกสำหรับผู้ใช้แต่ละคน

**เครื่องมือ (Tool)** - ฟังก์ชันที่เอเยนต์ AI สามารถเรียกใช้

**คำอธิบายเครื่องมือ** - เอกสารอธิบายวัตถุประสงค์และพารามิเตอร์ของเครื่องมือ

## โมดูล Agentic - [โมดูล 05](../05-mcp/README.md)

**@Agent Annotation** - แท็กอินเทอร์เฟซเป็นเอเยนต์ AI พร้อมนิยามพฤติกรรมแบบประกาศ

**Agent Listener** - ตะขอสำหรับติดตามการทำงานเอเยนต์ผ่าน `beforeAgentInvocation()` และ `afterAgentInvocation()`

**ขอบเขตแบบ Agentic (Agentic Scope)** - หน่วยความจำแชร์ที่เอเยนต์เก็บผลลัพธ์โดยใช้ `outputKey` เพื่อให้เอเยนต์อื่นใช้ต่อ

**AgenticServices** - โรงงานสร้างเอเยนต์โดยใช้ `agentBuilder()` และ `supervisorBuilder()`

**เวิร์กโฟลว์ตามเงื่อนไข (Conditional Workflow)** - เส้นทางตามเงื่อนไขไปยังเอเยนต์เฉพาะทางต่าง ๆ

**มนุษย์ในวงจร (Human-in-the-Loop)** - รูปแบบการทำงานเพิ่มจุดตรวจสอบโดยมนุษย์สำหรับอนุมัติหรือตรวจเนื้อหา

**langchain4j-agentic** - พึ่งพา Maven สำหรับสร้างเอเยนต์แบบประกาศ (ทดสอบ)

**เวิร์กโฟลว์ลูป (Loop Workflow)** - ทำซ้ำการทำงานของเอเยนต์จนกว่าจะครบเงื่อนไข (เช่น คะแนนคุณภาพ ≥ 0.8)

**outputKey** - พารามิเตอร์แท็กเอเยนต์ระบุที่เก็บผลลัพธ์ใน Agentic Scope

**เวิร์กโฟลว์ขนาน (Parallel Workflow)** - รันเอเยนต์หลายตัวพร้อมกันสำหรับงานอิสระ

**กลยุทธ์การตอบกลับ (Response Strategy)** - วิธีผู้ดูแลกำหนดคำตอบสุดท้าย: LAST, SUMMARY, หรือ SCORED

**เวิร์กโฟลว์เรียงลำดับ (Sequential Workflow)** - รันเอเยนต์ตามลำดับโดยผลลัพธ์ใช้ในขั้นตอนถัดไป

**รูปแบบเอเยนต์ผู้ดูแล (Supervisor Agent Pattern)** - รูปแบบขั้นสูงที่ผู้ดูแล LLM ตัดสินใจเรียกเอเยนต์ย่อยแบบไดนามิก

## โปรโตคอลบริบทของโมเดล (MCP) - [โมดูล 05](../05-mcp/README.md)

**langchain4j-mcp** - พึ่งพา Maven สำหรับรวม MCP ใน LangChain4j

**MCP** - โปรโตคอลบริบทของโมเดล: มาตรฐานเชื่อมต่อแอป AI กับเครื่องมือภายนอก สร้างครั้งเดียว ใช้ได้ทุกที่

**MCP Client** - แอปที่เชื่อมต่อกับเซิร์ฟเวอร์ MCP เพื่อค้นหาและใช้เครื่องมือ

**MCP Server** - บริการเผยแพร่เครื่องมือผ่าน MCP พร้อมคำอธิบายและสคีมาพารามิเตอร์ชัดเจน

**McpToolProvider** - ส่วนประกอบ LangChain4j ห่อหุ้มเครื่องมือ MCP สำหรับใช้ในบริการ AI และเอเยนต์

**McpTransport** - อินเทอร์เฟซสำหรับสื่อสาร MCP มีการใช้งานเช่น Stdio และ HTTP

**Stdio Transport** - ขนส่งผ่านกระบวนการภายในเครื่องโดย stdin/stdout ใช้งานสำหรับเข้าถึงไฟล์หรือเครื่องมือบรรทัดคำสั่ง

**StdioMcpTransport** - การใช้งาน LangChain4j ที่เริ่มเซิร์ฟเวอร์ MCP เป็นซับโปรเซส

**การค้นพบเครื่องมือ (Tool Discovery)** - ไคลเอนต์สอบถามเซิร์ฟเวอร์สำหรับเครื่องมือที่มีพร้อมคำอธิบายและสคีมา
