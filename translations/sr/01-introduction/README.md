## Шта ћете научити

Ако сте завршили брзи почетак, видели сте како да шаљете упите и добијете одговоре. То је основа, али праве апликације захтевају више. Овај модул вас учи како да направите конверзацијски АИ који памти контекст и одржава стање - разлика између једнократне демонстрације и апликације спремне за продукцију.

Користићемо Azure OpenAI GPT-5.2 током овог водича зато што његове напредне могућности закључивања чине понашање различитих образаца јаснијим. Када додате меморију, јасно ћете видети разлику. Ово олакшава разумевање шта свака компонента доноси вашој апликацији.

Направићете једну апликацију која демонстрира оба образца:

**Бездржавни чет** - Свакa порука је независна. Модел нема меморију претходних порука. Ово је образац који сте користили у брзом почетку.

**Државни разговор** - Свакa порука укључује историју разговора. Модел одржава контекст кроз више корака. Ово је оно што захтевају продукционе апликације.

## Захтеви

- Azure претплата са приступом Azure OpenAI
- Java 21, Maven 3.9+
- Azure CLI (https://learn.microsoft.com/en-us/cli/azure/install-azure-cli)
- Azure Developer CLI (azd) (https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/install-azd)

> **Напомена:** Java, Maven, Azure CLI и Azure Developer CLI (azd) су претходно инсталирани у обезбеђеном devcontainer-у.

> **Напомена:** Овај модул користи GPT-5.2 на Azure OpenAI. Деплој је аутоматски конфигурисан преко `azd up` - не мењајте назив модела у коду.

## Разумијевање основног проблема

Језички модели су бездржавни. Сваки API позив је независан. Ако пошаљете "Моје име је Џон" и потом питате "Како се зовем?", модел нема појма да сте се управо представили. Сваки захтев третира као да је први разговор који водите.

Ово је у реду за једноставна питања и одговоре али није корисно за праве апликације. Ботови за корисничку подршку морају да памте шта сте им рекли. Лични асистенти захтевају контекст. Свaки разговор с више корака захтева меморију.

<img src="../../../translated_images/sr/stateless-vs-stateful.cc4a4765e649c41a.webp" alt="Бездржавни и државни разговори" width="800"/>

*Разлика између бездржавних (независних позива) и државних (са свешћу о контексту) разговора*

## Разумијевање токена

Пре уласка у разговоре, важно је разумети токене - основне јединице текста које језички модели обрађују:

<img src="../../../translated_images/sr/token-explanation.c39760d8ec650181.webp" alt="Објашњење токена" width="800"/>

*Пример како се текст раздваја у токене - "I love AI!" постаје 4 одвојене јединице за обраду*

Токени су начин на који АИ модели мере и обрађују текст. Речи, интерпункција, па чак и размаке могу бити токени. Ваш модел има ограничење колико токена може обрадити одједном (400,000 за GPT-5.2, до 272,000 улазних и 128,000 излазних токена). Разумевање токена помаже вам да контролишете дужину разговора и трошкове.

## Како ради меморија

Меморија чата решава проблем бездржавности тако што одржава историју разговора. Пре слања захтева моделу, оквир додаје релевантне претходне поруке. Када питате "Како се зовем?", систем заправо шаље целу историју, дозвољавајући моделу да види да сте раније рекли "Моје име је Џон."

LangChain4j обезбеђује имплементације меморије које аутоматски управљају овиме. Ви бирајте колико порука да задржите, а оквир управља контекстним прозором.

<img src="../../../translated_images/sr/memory-window.bbe67f597eadabb3.webp" alt="Концепт меморијског прозора" width="800"/>

*MessageWindowChatMemory одржава клизни прозор недавних порука, аутоматски одбацујући старе*

## Како се користи LangChain4j

Овај модул проширује брзи почетак интеграцијом Spring Boot-а и додавањем меморије разговора. Ево како се деле:

**Зависности** - Додајте две LangChain4j библиотеке:

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```

**Модел за чет** - Конфигуришите Azure OpenAI као Spring bean ([LangChainConfig.java](../../../01-introduction/src/main/java/com/example/langchain4j/config/LangChainConfig.java)):

```java
@Bean
public OpenAiOfficialChatModel openAiOfficialChatModel() {
    return OpenAiOfficialChatModel.builder()
            .baseUrl(azureEndpoint)
            .apiKey(azureApiKey)
            .modelName(deploymentName)
            .timeout(Duration.ofMinutes(5))
            .maxRetries(3)
            .build();
}
```

Билдер чита креденцијале из променљивих окружења које поставља `azd up`. Подешавање `baseUrl` на ваш Azure крајњи параметар омогућава OpenAI клиенту да ради са Azure OpenAI.

**Меморија разговора** - Пратите историју чата помоћу MessageWindowChatMemory ([ConversationService.java](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java)):

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("My name is John"));
memory.add(AiMessage.from("Nice to meet you, John!"));

memory.add(UserMessage.from("What's my name?"));
AiMessage aiMessage = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage);
```

Креирајте меморију са `withMaxMessages(10)` да задржите последњих 10 порука. Додајте поруке корисника и АИ са типизованим омотачима: `UserMessage.from(text)` и `AiMessage.from(text)`. Добите историју помоћу `memory.messages()` и пошаљите je моделу. Сервис складишти посебне инстанце меморије по ИД-ју конверзације, омогућавајући више корисника да четују истовремено.

> **🤖 Покушајте са [GitHub Copilot](https://github.com/features/copilot) Чатом:** Отворите [`ConversationService.java`](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java) и питајте:
> - "Како MessageWindowChatMemory одлучује које поруке да одбаци када је прозор пун?"
> - "Могу ли да имплементирам прилагођено складиштење меморије користећи базу података уместо у меморији?"
> - "Како бих додао сажимање да компресујем стару историју разговора?"

Бездржавни чет крајња тачка потпуно прескаче меморију - само `chatModel.chat(prompt)` као у брзом почетку. Државна крајња тачка додаје поруке у меморију, преузима историју и укључује тај контекст сваки пут. Иста конфигурација модела, различити образци.

## Деплој Azure OpenAI инфраструктуре

**Bash:**
```bash
cd 01-introduction
azd up  # Изаберите претплату и локацију (препоручује се eastus2)
```

**PowerShell:**
```powershell
cd 01-introduction
azd up  # Изаберите претплату и локацију (препоручује се eastus2)
```

> **Напомена:** Ако добијете грешку одређеног времена (`RequestConflict: Cannot modify resource ... provisioning state is not terminal`), једноставно покрените `azd up` поново. Azure ресурси можда још увек се покрећу у позадини, а поновни покушај омогућава завршетак деплоја када ресурси достигну завршно стање.

Ово ће:
1. Деплојовати Azure OpenAI ресурс са GPT-5.2 и text-embedding-3-small моделима
2. Аутоматски генерирати `.env` фајл у корену пројекта са креденцијалима
3. Подесити све потребне променљиве окружења

**Имате проблема са деплојем?** Погледајте [Infrastructure README](infra/README.md) за детаљна упутства о решавању проблема укључујући сукобе имена поддомена, ручне кораке деплоја у Azure порталу и смернице за конфигурацију модела.

**Проверите да ли је деплој успео:**

**Bash:**
```bash
cat ../.env  # Требало би да приказује AZURE_OPENAI_ENDPOINT, API_KEY, итд.
```

**PowerShell:**
```powershell
Get-Content ..\.env  # Треба да приказује AZURE_OPENAI_ENDPOINT, API_KEY итд.
```

> **Напомена:** Команда `azd up` аутоматски генерише `.env` фајл. Ако га морате касније ажурирати, можете га или ручно изменити или га регенерисати покретањем:
>
> **Bash:**
> ```bash
> cd ..
> bash .azd-env.sh
> ```
>
> **PowerShell:**
> ```powershell
> cd ..
> .\.azd-env.ps1
> ```

- Покренути/зауставити апликације једним кликом
- Гледати логове апликација у реалном времену

Једноставно кликните на дугме за репродукцију поред "introduction" да започнете овај модул, или покрените све модуле одједном.

## Коришћење апликације

Апликација пружа веб интерфејс са две имплементације чета једна поред друге.

<img src="../../../translated_images/sr/home-screen.121a03206ab910c0.webp" alt="Почетни екран апликације" width="800"/>

*Контролна табла која приказује обе опције: Једноставни чет (бездржавни) и Конверзацијски чет (државни)*

### Бездржавни чет (леви панел)

Пробајте ово прво. Реците "Моје име је Џон" па одмах питајте "Како се зовем?" Модел неће памтити јер је свака порука независна. Ово демонстрира основни проблем интеграције језичког модела - нема контекст разговора.

<img src="../../../translated_images/sr/simple-chat-stateless-demo.13aeb3978eab3234.webp" alt="Демонстрација бездржавног чета" width="800"/>

*АИ не памти ваше име из претходне поруке*

### Државни чет (десни панел)

Сада пробајте исти низ овде. Реците "Моје име је Џон" па онда "Како се зовем?" Овај пут памти. Разлика је у MessageWindowChatMemory - он одржава историју разговора и укључује је у сваки захтев. Тако ради производни конверзацијски АИ.

<img src="../../../translated_images/sr/conversational-chat-stateful-demo.e5be9822eb23ff59.webp" alt="Демонстрација државног чета" width="800"/>

*АИ памти ваше име из претходног дела разговора*

Обa панела користе исти GPT-5.2 модел. Једина разлика је меморија. То јасно показује шта меморија доноси вашој апликацији и зашто је суштинска за стварне случајеве коришћења.
