## Шта ћете научити

Ако сте завршили брзи почетак, видели сте како се шаљу промптови и добијају одговори. То је основа, али праве апликације захтевају више. Овај модул вас учи како да направите разговорни АИ који памти контекст и одржава стање - разлика између једнократне демонстрације и апликације спремне за производњу.

У овом упутству ћемо користити Azure OpenAI GPT-5.2 јер његове напредне могућности резоновања јасније показују понашање различитих образаца. Када додате памћење, јасно ћете видети разлику. Ово олакшава разумевање шта сваки део доноси вашој апликацији.

Направићете једну апликацију која демонстрира оба образца:

**Статлес чет** - сваки захтев је независан. Модел нема памћење претходних порука. Ово је образац који сте користили у брзом почетку.

**Стейтфул разговор** - сваки захтев укључује историју разговора. Модел одржава контекст кроз више кругова. Ово је оно што захтевају производне апликације.

## Захтеви

- Azure претплата са приступом Azure OpenAI
- Java 21, Maven 3.9+
- Azure CLI (https://learn.microsoft.com/en-us/cli/azure/install-azure-cli)
- Azure Developer CLI (azd) (https://learn.microsoft.com/en-us/azure/developer/azure-developer-cli/install-azd)

> **Напомена:** Java, Maven, Azure CLI и Azure Developer CLI (azd) су већ инсталирани у пруженом девконтейнеру.

> **Напомена:** Овај модул користи GPT-5.2 на Azure OpenAI. Деплој је аутоматски конфигурисан преко `azd up` - немојте мењати назив модела у коду.

## Разумевање основног проблема

Језички модели су без стања. Сваки API позив је независан. Ако пошаљете "Зовем се Џон" а онда питате "Како се зовем?", модел нема појма да сте се управо представили. Он третира сваки захтев као да је први разговор који сте икада имали.

Ово је довољно за једноставна питања и одговоре, али непогодно за праве апликације. Ботови за корисничку подршку морају да памте шта сте им рекли. Лични асистенти требају контекст. Сваки вишекружни разговор захтева памћење.

<img src="../../../translated_images/sr/stateless-vs-stateful.cc4a4765e649c41a.webp" alt="Статлес вс Стейтфул разговори" width="800"/>

*Разлика између статлес (независних позива) и стейтфул (са контекстом) разговора*

## Разумевање токена

Пре него што задирнете у разговоре, важно је разумети токене - основне јединице текста које језички модели обрађују:

<img src="../../../translated_images/sr/token-explanation.c39760d8ec650181.webp" alt="Објашњење токена" width="800"/>

*Пример како се текст раздваја на токене - „Волим АИ!“ постаје 4 посебне јединице за обраду*

Токени су начин на који АИ модели мере и обрађују текст. Речи, интерпункција и чак размаке могу бити токени. Ваш модел има ограничење колико токена може да обради одједном (400,000 за GPT-5.2, са до 272,000 улазних и 128,000 излазних токена). Разумевање токена помаже вам да управљате дужином разговора и трошковима.

## Како памћење функционише

Памћење разговора решава проблем без стања тако што одржава историју разговора. Пре слања захтева моделу, оквир додаје релевантне претходне поруке. Када питате "Како се зовем?", систем заправо шаље целу историју разговора, што омогућава моделу да види да сте раније рекли "Зовем се Џон".

LangChain4j пружа имплементације памћења које ово аутоматски рукују. Ви изаберете колико порука да задржите, а оквир управља прозором контекста.

<img src="../../../translated_images/sr/memory-window.bbe67f597eadabb3.webp" alt="Концепт прозора памћења" width="800"/>

*MessageWindowChatMemory одржава клизни прозор недавних порука, аутоматски бацајући старе*

## Како ово користи LangChain4j

Овај модул проширује брзи почетак интеграцијом Spring Boot-а и додавањем памћења разговора. Ево како се делови уклапају:

**Зависности** - Додајте две LangChain4j библиотеке:

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-open-ai-official</artifactId> <!-- Inherited from BOM in root pom.xml -->
</dependency>
```

**Модел за чет** - Конфигуришите Azure OpenAI као Spring bean ([LangChainConfig.java](../../../01-introduction/src/main/java/com/example/langchain4j/config/LangChainConfig.java)):

```java
@Bean
public OpenAiOfficialChatModel openAiOfficialChatModel() {
    return OpenAiOfficialChatModel.builder()
            .baseUrl(azureEndpoint)
            .apiKey(azureApiKey)
            .modelName(deploymentName)
            .timeout(Duration.ofMinutes(5))
            .maxRetries(3)
            .build();
}
```

Builder чита акредитиве из променљивих окружења постављених командом `azd up`. Постављање `baseUrl` на вашу Azure тачку омогућава OpenAI клијенту да ради са Azure OpenAI.

**Памћење разговора** - Пратите историју ћаскања са MessageWindowChatMemory ([ConversationService.java](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java)):

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("My name is John"));
memory.add(AiMessage.from("Nice to meet you, John!"));

memory.add(UserMessage.from("What's my name?"));
AiMessage aiMessage = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage);
```

Креирајте памћење са `withMaxMessages(10)` да бисте задржали последњих 10 порука. Додајте корисничке и АИ поруке уз типизоване омотаје: `UserMessage.from(text)` и `AiMessage.from(text)`. Преузмите историју помоћу `memory.messages()` и пошаљите је моделу. Сервис складишти појединачна памћења по ИД-ју разговора, што омогућава вишеструким корисницима да истовремено ћаскају.

> **🤖 Испробајте са [GitHub Copilot](https://github.com/features/copilot) Четом:** Отворите [`ConversationService.java`](../../../01-introduction/src/main/java/com/example/langchain4j/service/ConversationService.java) и питајте:
> - "Како MessageWindowChatMemory одлучује које поруке да одбаци када је прозор пун?"
> - "Могу ли да имплементирам прилагођено складиште памћења користећи базу података уместо меморије?"
> - "Како да додам резимирање да смањим стару историју разговора?"

Статлес чет крајња тачка прескаче памћење - само `chatModel.chat(prompt)` као у брзом почетку. Стейтфул крајња тачка додаје поруке у памћење, преузима историју и укључује тај контекст у сваки захтев. Иста конфигурација модела, различити образци.

## Деплој Azure OpenAI инфраструктуре

**Bash:**
```bash
cd 01-introduction
azd up  # Изаберите претплату и локацију (препоручује се eastus2)
```

**PowerShell:**
```powershell
cd 01-introduction
azd up  # Изаберите претплату и локацију (препоручује се eastus2)
```

> **Напомена:** Ако наиђете на грешку о временском ограничењу (`RequestConflict: Cannot modify resource ... provisioning state is not terminal`), једноставно поново покрените `azd up`. Azure ресурси могу и даље бити у процесу провизије у позадини, и поновни покушај омогућава да се деплој заврши када ресурси достигну терминално стање.

Ово ће:
1. Деплојовати Azure OpenAI ресурс са GPT-5.2 и моделима text-embedding-3-small
2. Аутоматски генерисати `.env` фајл у корену пројекта са акредитивима
3. Поставити све потребне променљиве окружења

**Имате проблема са деплојем?** Погледајте [Infrastructure README](infra/README.md) за детаљне упутства укључујући конфликте имена поддомена, ручне кораке деплоја преко Azure портала и смернице за конфигурацију модела.

**Потврдите да је деплој успешан:**

**Bash:**
```bash
cat ../.env  # Требало би да прикаже AZURE_OPENAI_ENDPOINT, API_KEY, итд.
```

**PowerShell:**
```powershell
Get-Content ..\.env  # Треба да прикаже AZURE_OPENAI_ENDPOINT, API_KEY, итд.
```

> **Напомена:** Команда `azd up` аутоматски генерише `.env` фајл. Ако га касније треба ажурирати, можете га или ручно уредити или поново генерисати покретањем:
>
> **Bash:**
> ```bash
> cd ..
> bash .azd-env.sh
> ```
>
> **PowerShell:**
> ```powershell
> cd ..
> .\.azd-env.ps1
> ```

## Покрени апликацију локално

**Потврдите деплој:**

Обезбедите да `.env` фајл постоји у коренском директоријуму са Azure акредитивима:

**Bash:**
```bash
cat ../.env  # Требало би да прикаже AZURE_OPENAI_ENDPOINT, API_KEY, DEPLOYMENT
```

**PowerShell:**
```powershell
Get-Content ..\.env  # Треба да прикаже AZURE_OPENAI_ENDPOINT, API_KEY, DEPLOYMENT
```

**Покрените апликације:**

**Опција 1: Користећи Spring Boot Dashboard (препоручено за кориснике VS Code-а)**

Дев контейнер укључује додатак Spring Boot Dashboard који пружа визуелни интерфејс за управљање свим Spring Boot апликацијама. Можете га пронаћи у Activity Baru на левој страни VS Code-а (потражите иконицу за Spring Boot).

Из Spring Boot Dashboard-а можете:
- Видети све доступне Spring Boot апликације у радном простору
- Покретати/заустављати апликације једним кликом
- Пратити логове апликације у реалном времену
- Надгледати статус апликација

Једноставно кликните на дугме за покретање поред "introduction" да покренете овај модул или покрените све модуле одједном.

<img src="../../../translated_images/sr/dashboard.69c7479aef09ff6b.webp" alt="Spring Boot Dashboard" width="400"/>

**Опција 2: Користећи shell скрипте**

Покрените све веб апликације (модул 01-04):

**Bash:**

**PowerShell:**

Обa скрипта аутоматски учитавају променљиве окружења из коренског `.env` фајла и изградиће JAR фајлове ако већ не постоје.

> **Напомена:** Ако желите ручно да изградите све модуле пре покретања:
>

> cd ..  # Go to root directory
> mvn clean package -DskipTests
> ```

Отворите http://localhost:8080 у вашем претраживачу.

**Bash:**
```bash

```

**PowerShell:**
```powershell

# Или

```

## Коришћење апликације

Апликација пружа веб интерфејс са две имплементације ћаскања распореде једна поред друге.

<img src="../../../translated_images/sr/home-screen.121a03206ab910c0.webp" alt="Почетни екран апликације" width="800"/>

*Контролна табла која приказује и Општи чет (без стања) и Разговорни чет (са стањем)*

### Статлес чет (леви панел)

Пробајте прво ово. Питајте „Зовем се Џон“ а затим одмах „Како се зовем?“ Модел неће памтити јер је свака порука независна. Ово демонстрира основни проблем интеграције језичког модела - нема контекста разговора.

<img src="../../../translated_images/sr/simple-chat-stateless-demo.13aeb3978eab3234.webp" alt="Демо статлес чета" width="800"/>

*АИ не памти ваше име из претходне поруке*

### Стейтфул чет (десни панел)

Сада покушајте исти низ овде. Питајте „Зовем се Џон“ а затим „Како се зовем?“ Овог пута памти. Разлика је MessageWindowChatMemory - одржава историју разговора и укључује је у сваки захтев. Тако функционишу продукциони разговорни АИ.

<img src="../../../translated_images/sr/conversational-chat-stateful-demo.e5be9822eb23ff59.webp" alt="Демо стейтфул чета" width="800"/>

*АИ памти ваше име из раније у разговору*

Обa панела користе исти GPT-5.2 модел. Једина разлика је памћење. Ово јасно показује шта памћење доноси вашој апликацији и зашто је неопходно за реалне случајеве употребе.
