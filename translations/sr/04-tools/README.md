## Шта ћете научити

До сада сте научили како водити разговоре са AI, ефективно структуирати упите и осигурати да одговори буду засновани на вашим документима. Али постоји основно ограничење: језички модели могу само генерисати текст. Не могу проверити време, извршити прорачуне, упитати базе података или комуницирати са спољним системима.

Алатке то мењају. Дајући моделу приступ функцијама које може да позове, претварате га из генератора текста у агента који може да предузима акције. Модел одлучује када му треба алат, који алат да користи и које параметре да проследи. Ваш код извршава функцију и враћа резултат. Модел укључује тај резултат у свој одговор.

## Разумевање AI агената са алатима

> **📝 Напомена:** Термин "агенти" у овом модулу се односи на AI помагаче побољшане могућношћу позивања алата. Ово је другачије од **Agentic AI** образаца (аутононими агенти са планирањем, меморијом и више корака размишљања) које ћемо обрадити у [Модул 05: MCP](../05-mcp/README.md).

Без алата, језички модел може само генерисати текст из свог обученог података. Питајте га за тренутно време и онда мора да нагађа. Дајте му алате и он може да позове API за време, изврши прорачуне или упита базу података — а затим те стварне резултате исплете у свој одговор.

<img src="../../../translated_images/sr/what-are-tools.724e468fc4de64da.webp" alt="Without Tools vs With Tools" width="800"/>

*Без алата модел може само да нагађа — са алатима може позивати API, извршавати прорачуне и враћати податке у реалном времену.*

AI агент са алатима прати **ReAct** образац (разумевање и деловање). Модел не само што одговара — он размишља о томе шта му треба, дели позивањем алата, посматра резултат, а затим одлучује да ли ће поново деловати или доставити коначни одговор:

1. **Размишљање** — Агент анализира корисниково питање и одређује које информације су му потребне
2. **Деловање** — Агент одабира прави алат, генерише исправне параметре и позива га
3. **Посматрање** — Агент прими резултат алата и процењује га
4. **Понављање или одговор** — Ако је потребно више података, агент понавља циклус; у супротном саставља одговор природним језиком

<img src="../../../translated_images/sr/react-pattern-detail.96a5efeeb6dd2f61.webp" alt="ReAct Pattern" width="800"/>

*ReAct циклус — агент размишља шта треба урадити, дели позивом алата, посматра резултат и понавља док не може да да коначни одговор.*

Ово се дешава аутоматски. Ви дефинишете алате и њихове описе. Модел управља одлуком када и како их користити.

## Како функционише позивање алата

### Дефиниције алата

[WeatherTool.java](../../../04-tools/src/main/java/com/example/langchain4j/agents/tools/WeatherTool.java) | [TemperatureTool.java](../../../04-tools/src/main/java/com/example/langchain4j/agents/tools/TemperatureTool.java)

Дефинишете функције са јасним описима и спецификацијама параметара. Модел види те описе у свом системском промпту и разуме шта сваки алат ради.

```java
@Component
public class WeatherTool {
    
    @Tool("Get the current weather for a location")
    public String getCurrentWeather(@P("Location name") String location) {
        // Логика претраге временских услова
        return "Weather in " + location + ": 22°C, cloudy";
    }
}

@AiService
public interface Assistant {
    String chat(@MemoryId String sessionId, @UserMessage String message);
}

// Асистент је аутоматски повезан од стране Spring Boot-а са:
// - ChatModel беаном
// - Све @Tool методе из @Component класа
// - ChatMemoryProvider за управљање сесијама
```

Дијаграм испод разобличава сваку анотацију и показује како сваки део помаже AI-у да разуме када да позове алат и које аргументе да проследи:

<img src="../../../translated_images/sr/tool-definitions-anatomy.f6468546037cf28b.webp" alt="Anatomy of Tool Definitions" width="800"/>

*Анатомија дефиниције алата — @Tool говори AI-у када да га користи, @P описује сваки параметар, а @AiService све повезује при покретању.*

> **🤖 Испробајте са [GitHub Copilot](https://github.com/features/copilot) Чатом:** Отворите [`WeatherTool.java`](../../../04-tools/src/main/java/com/example/langchain4j/agents/tools/WeatherTool.java) и питајте:
> - "Како бих интегрисао стварни weather API као што је OpenWeatherMap уместо лажних података?"
> - "Шта чини добар опис алата који помаже AI-у да га правилно користи?"
> - "Како да руковам API грешкама и ограничењима у имплементацији алата?"

### Доношење одлука

Када корисник пита "Какво је време у Сијетлу?", модел не бира алат насумично. Он упоређује корисникову намеру са сваким описом алата којем има приступ, бодује их према релевантности и бира најбољи одговор. Онда генерише структурисани позив функције са правим параметрима — у овом случају, подешава `location` на `"Seattle"`.

Ако ни један алат не одговара корисничком захтеву, модел одговара из сопственог знања. Ако више алата одговара, бира најспецифичнији.

<img src="../../../translated_images/sr/decision-making.409cd562e5cecc49.webp" alt="How the AI Decides Which Tool to Use" width="800"/>

*Модел процењује сваки доступни алат у односу на корисникову намеру и бира најбољи одговор — зато је важно писати јасне и специфичне описе алата.*

### Извршење

Spring Boot аутоматски повезује декларативни `@AiService` интерфејс са свим регистрованим алатима, а LangChain4j аутоматски извршава позиве алата. Испод површине, комплетан позив алата пролази кроз шест фаза — од питања корисника природним језиком до одговора природним језиком:

<img src="../../../translated_images/sr/tool-calling-flow.8601941b0ca041e6.webp" alt="Tool Calling Flow" width="800"/>

*Цео ток — корисник поставља питање, модел бира алат, LangChain4j га извршава, а модел тка резултат у природан одговор.*

> **🤖 Испробајте са [GitHub Copilot](https://github.com/features/copilot) Чатом:** Отворите [`AgentService.java`](../../../04-tools/src/main/java/com/example/langchain4j/agents/service/AgentService.java) и питајте:

> - "Шта се дешава ако позив алата не успе - како да поуздано руковам грешкама?"

### Генерисање одговора

Модел добија податке о времену и форматира их у природан одговор за корисника.

### Архитектура: Spring Boot аутоматско повезивање

Овај модул користи LangChain4j-ову интеграцију са Spring Boot-ом преко декларативних `@AiService` интерфејса. При покретању Spring Boot открива сваки `@Component` који садржи `@Tool` методе, ваш `ChatModel` bean и `ChatMemoryProvider` — затим све то повезује у један `Assistant` интерфејс без било каквог додатног кода.

<img src="../../../translated_images/sr/spring-boot-wiring.151321795988b04e.webp" alt="Spring Boot Auto-Wiring Architecture" width="800"/>

*@AiService интерфејс спаја ChatModel, компоненте алата и меморијски провајдер — Spring Boot све повезује аутоматски.*

Кључне предности овог приступа:

- **Spring Boot аутоматско повезивање** — ChatModel и алати аутоматски инјектовани
- **@MemoryId образац** — Аутоматско управљање меморијом засновано на сесији
- **Једна инстанца** — Assistant креиран једном и поново коришћен за боље перформансе
- **Извршење са типском безбедношћу** — Java методи позивани директно са претворбом типова
- **Višekратна контролa** — Аутоматски управља ланцима алата
- **Без сувишног кода** — Без ручних позива `AiServices.builder()` или меморијских HashMap-ова

Алтернативни приступи (ручни `AiServices.builder()`) захтевају више кода и не користе предности Spring Boot интеграције.

## Ланац алата

**Ланац алата** — права снага агената заснованих на алатима показује се када једно питање захтева више алата. Питајте "Какво је време у Сијетлу у Фаренхајтима?" и агент аутоматски повезује два алата: прво позива `getCurrentWeather` да добије температуру у Целзијусима, затим ту вредност прослеђује у `celsiusToFahrenheit` за конверзију — све у једном кораку разговора.

<img src="../../../translated_images/sr/tool-chaining-example.538203e73d09dd82.webp" alt="Tool Chaining Example" width="800"/>

*Ланац алата у акцији — агент прво позива getCurrentWeather, потом резултат у Целзијусима шаље у celsiusToFahrenheit и даје комбиновани одговор.*

Ево како то изгледа у покренутој апликацији — агент повезује два позива алата у једном кораку разговора:

<a href="images/tool-chaining.png"><img src="../../../translated_images/sr/tool-chaining.3b25af01967d6f7b.webp" alt="Tool Chaining" width="800" style="border: 1px solid #ddd; box-shadow: 0 2px 8px rgba(0,0,0,0.1);"/></a>

*Стварни резултат апликације — агент аутоматски ланцира getCurrentWeather → celsiusToFahrenheit у једном кораку.*

**Љубазно руковање грешкама** — Питајте за време у граду који није у лажним подацима. Алат враћа поруку о грешци, а AI објашњава да не може помоћи уместо да „падне“. Алатке безбедно подносе грешке.

<img src="../../../translated_images/sr/error-handling-flow.9a330ffc8ee0475c.webp" alt="Error Handling Flow" width="800"/>

*Када алат не успе, агент хвата грешку и одговара корисним објашњењем уместо да апликација касни.*

Ово се дешава у једном кораку разговора. Агент аутономно координише више позива алата.

## Коришћење апликације

Апликација пружа веб интерфејс где можете комуницирати са AI агентом који има приступ алатима за време и претварање температуре.

<a href="images/tools-homepage.png"><img src="../../../translated_images/sr/tools-homepage.4b4cd8b2717f9621.webp" alt="AI Agent Tools Interface" width="800" style="border: 1px solid #ddd; box-shadow: 0 2px 8px rgba(0,0,0,0.1);"/></a>

*Интерфејс AI агента са алатима - брзи примери и чат интерфејс за интеракцију са алатима*

### Испробајте једноставну употребу алата
Започните са једноставним захтевом: "Претвори 100 степени Фаренхајта у Целзијус". Агенat препознаје да му је потребан алат за конверзију температуре, позове га са одговарајућим параметрима и врати резултат. Обратите пажњу колико ово звучи природно - нисте му рекли који алат да користи или како да га позове.

### Тестирајте ланац алата

Сада покушајте нешто компликованије: "Какво је време у Сијетлу и претвори то у Фаренхајт?" Пратите како агент решава ово у корацима. Прво добија информације о времену (које су у Целзијусима), препознаје да треба да претвори у Фаренхајт, позива алат за конверзију и комбинује оба резултата у један одговор.

### Погледајте ток разговора

Интерфејс за ћаскање чува историју разговора, омогућавајући вишекратне интеракције. Можете видети сва претходна питања и одговоре, што олакшава праћење и разумевање како агент гради контекст током више размена.

<a href="images/tools-conversation-demo.png"><img src="../../../translated_images/sr/tools-conversation-demo.89f2ce9676080f59.webp" alt="Разговор са вишеструким позивима алата" width="800" style="border: 1px solid #ddd; box-shadow: 0 2px 8px rgba(0,0,0,0.1);"/></a>

*Разговор са више корака који показује једноставне конверзије, проналажење времена и ланац алата*

### Испробајте различите захтеве

Испробајте различите комбинације:
- Проналажење времена: "Какво је време у Токију?"
- Конверзије температуре: "Колико је 25°C у Келвинима?"
- Комбиновна питања: "Провери време у Паризу и реци ми да ли је преко 20°C"

Обратите пажњу како агент интерпретира природни језик и мапира га на одговарајуће позиве алата.

## Кључни појмови

### ReAct образац (Размишљање и деловање)

Агент наизменично размишља (одлучује шта да уради) и делује (користи алате). Овај образац омогућава аутономно решавање проблема уместо само одговарања на упутства.

### Опис алата је важан

Квалитет ваших описа алата директно утиче на то како ће агент да их користи. Јасни и специфични описи помажу моделу да разуме када и како да позове сваки алат.

### Управљање сесијама

Анотација `@MemoryId` омогућава аутоматско управљање меморијом по сесијама. Сваком ID сесије додељује се сопствена инстанца `ChatMemory` коју управља `ChatMemoryProvider` bean, па више корисника може истовремено да разговара са агентом без мешања порука.

<img src="../../../translated_images/sr/session-management.91ad819c6c89c400.webp" alt="Управљање сесијом са @MemoryId" width="800"/>

*Сваком ID сесије одговара изолована историја разговора — корисници никада не виде поруке једни других.*

### Обрада грешака

Алати могу да не успеју — API-ји могу да истекну, параметри могу бити неважећи, спољне услуге могу бити недоступне. Продукциони агенти захтевају обраду грешака тако да модел може објаснити проблем или пробати алтернативе уместо да читаву апликацију сруши. Када алат баци изузетак, LangChain4j га ухвати и проследи поруку о грешци назад моделу, који онда може све објаснити на природном језику.

## Доступни алати

Дијаграм испод показује широку екосистему алата које можете направити. Овај модул демонстрира алате за време и температуру, али исти образац `@Tool` ради за било који Java метод — од упита базе података до обраде уплата.

<img src="../../../translated_images/sr/tool-ecosystem.aad3d74eaa14a44f.webp" alt="Екосистем алата" width="800"/>

*Било који Java метод означен са @Tool постаје доступан за AI — образац се проширује на базе података, APIје, имејл, рад са фајловима и више.*

## Када користити агенте засноване на алатима

<img src="../../../translated_images/sr/when-to-use-tools.51d1592d9cbdae9c.webp" alt="Када користити алате" width="800"/>

*Кратак водич за одлуку — алати су за податке у реалном времену, израчунавања и акције; опште знање и креативни задаци их обично не захтевају.*

**Користите алате када:**
- Потребни су подаци у реалном времену (време, цене акција, залихе)
- Морате обавити израчунавања сложенија од основне математике
- Приступате базама података или API-јима
- Позитивне акције (слање имејлова, креирање тикета, ажурирање записа)
- Комбинујете више извора података

**Не користите алате када:**
- Одговори могу да се дају из општег знања
- Одговор је чисто разговорног типа
- Застој алата би учинио корисничко искуство превише спорим

## Алати у односу на RAG

Модули 03 и 04 оба проширују шта AI може да ради, али на темељно различите начине. RAG даје моделу приступ **знању** претражујући документе. Алати дају моделу могућност да предузима **акције** позивајући функције.

<img src="../../../translated_images/sr/tools-vs-rag.ad55ce10d7e4da87.webp" alt="Поређење алата и RAG" width="800"/>

*RAG претражује информације из статичних докумената — Алати извршавају акције и доносе динамичке, реално-временске податке. Многи продукциони системи комбинују оба приступа.*

У пракси, многи продукциони системи комбинују оба приступа: RAG за базирање одговора на вашој документацији, и Алате за прибављање живих података или извођење операција.
