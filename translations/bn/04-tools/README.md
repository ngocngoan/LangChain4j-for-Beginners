## আপনি যা শিখবেন

এপর্যন্ত, আপনি শিখেছেন কীভাবে AI এর সাথে আলাপচারিতা করবেন, প্রম্পটগুলিকে কার্যকরভাবে গঠন করবেন, এবং আপনার ডকুমেন্টগুলিতে প্রতিক্রিয়াগুলি ভিত্তি করাবেন। তবে একটি মৌলিক সীমাবদ্ধতা এখনও রয়েছে: ভাষা মডেল কেবলমাত্র টেক্সট তৈরি করতে পারে। তারা আবহাওয়া পরীক্ষা করতে পারে না, গণনা করতে পারে না, ডেটাবেস থেকে তথ্য সংগ্রহ করতে পারে না, অথবা বাইরের সিস্টেমের সাথে ইন্টারঅ্যাক্ট করতে পারে না।

টুলস এটা পরিবর্তন করে। মডেলকে এমন ফাংশনগুলো অ্যাক্সেস দেওয়ার মাধ্যমে যা এটি কল করতে পারে, আপনি এটিকে একটি টেক্সট জেনারেটর থেকে এমন একটি এজেন্টে রূপান্তর করেন যা কর্ম সম্পাদন করতে পারে। মডেল সিদ্ধান্ত নেয় কখন এটিকে একটি টুলের প্রয়োজন, কোন টুল ব্যবহার করতে হবে, এবং কী প্যারামিটার পাঠাতে হবে। আপনার কোড ফাংশনটি কার্যকর করে ফলাফল ফেরত দেয়। মডেল সেই ফলাফলটি তার প্রতিক্রিয়ায় অন্তর্ভুক্ত করে।

## টুলস সহ AI এজেন্টস বোঝা

> **📝 নোট:** এই মডিউলে "এজেন্টস" বলতে সেই AI সহকারীদের বোঝানো হয়েছে যারা টুল কলিং ক্ষমতা যুক্ত করা হয়েছে। এটি আলাদা **Agentic AI** প্যাটার্ন (স্বায়ত্তশাসিত এজেন্ট পরিকল্পনা, স্মৃতি, এবং বহু-ধাপ যুক্তি) থেকে যা আমরা [মডিউল ০৫: MCP](../05-mcp/README.md) এ আলোচনা করব।

টুল ছাড়া, একটি ভাষা মডেল কেবল তার প্রশিক্ষণ ডেটা থেকে টেক্সট তৈরি করতে পারে। বর্তমান আবহাওয়া জানতে বললে, এটি অনুমান করতেই হবে। টুল দিলে, এটি আবহওয়া API কল করতে পারে, গণনা করতে পারে, অথবা ডেটাবেস থেকে তথ্য নিতে পারে — তারপর সেই প্রকৃত ফলাফলগুলো তার প্রতিক্রিয়ায় বোনা হয়।

<img src="../../../translated_images/bn/what-are-tools.724e468fc4de64da.webp" alt="টুলস ছাড়া বনাম টুলস সহ" width="800"/>

*টুলস ছাড়া মডেল কেবল অনুমান করে — টুলস সহ এটি API কল করতে পারে, গণনা চালাতে পারে, এবং প্রাসঙ্গিক তথ্য ফেরত দিতে পারে।*

টুল সহ একটি AI এজেন্ট অনুসরণ করে **যুক্তি এবং সম্পাদন (ReAct)** প্যাটার্ন। মডেল কেবল প্রতিক্রিয়া দেয় না — এটি চিন্তা করে কী দরকার, একটি টুল কল করে কাজ করে, ফলাফল পর্যবেক্ষণ করে, এবং তারপর আবার কাজ করবে না কি চূড়ান্ত উত্তর দেবে তা সিদ্ধান্ত নেয়:

১. **যুক্তি করো** — এজেন্ট ব্যবহারকারীর প্রশ্ন বিশ্লেষণ করে এবং নির্ধারণ করে কী তথ্য প্রয়োজন
২. **সম্পাদন করো** — সঠিক টুল নির্বাচন করে, সঠিক প্যারামিটার তৈরি করে এবং কল করে
৩. **পর্যবেক্ষণ করো** — টুলের আউটপুট পায় এবং ফলাফল মূল্যায়ন করে
৪. **পুনরাবৃত্তি বা প্রতিক্রিয়া দাও** — আরও তথ্য প্রয়োজন হলে লুপ করে; নাহলে, একটি স্বাভাবিক ভাষার উত্তর তৈরি করে

<img src="../../../translated_images/bn/react-pattern-detail.96a5efeeb6dd2f61.webp" alt="ReAct প্যাটার্ন" width="800"/>

*ReAct চক্র — এজেন্ট কী করবে তা যুক্তি করে, একটি টুল কল করে কাজ করে, ফলাফল পর্যবেক্ষণ করে, এবং লুপ চালায় যতক্ষণ না এটি চূড়ান্ত উত্তর দিতে পারে।*

এটি স্বয়ংক্রিয়ভাবে ঘটে। আপনি টুল এবং তাদের বর্ণনা সংজ্ঞায়িত করেন। মডেল সিদ্ধান্ত নেয় কখন এবং কীভাবে সেগুলো ব্যবহার করতে হবে।

## কিভাবে টুল কলিং কাজ করে

### টুল সংজ্ঞা

[WeatherTool.java](../../../04-tools/src/main/java/com/example/langchain4j/agents/tools/WeatherTool.java) | [TemperatureTool.java](../../../04-tools/src/main/java/com/example/langchain4j/agents/tools/TemperatureTool.java)

আপনি স্পষ্ট বর্ণনা এবং প্যারামিটার স্পেসিফিকেশন সহ ফাংশন সংজ্ঞায়িত করেন। মডেল এই বর্ণনাগুলো তার সিস্টেম প্রম্পটে দেখে বুঝতে পারে প্রতিটি টুল কী করে।

```java
@Component
public class WeatherTool {
    
    @Tool("Get the current weather for a location")
    public String getCurrentWeather(@P("Location name") String location) {
        // আপনার আবহাওয়া অনুসন্ধান লজিক
        return "Weather in " + location + ": 22°C, cloudy";
    }
}

@AiService
public interface Assistant {
    String chat(@MemoryId String sessionId, @UserMessage String message);
}

// সহকারী স্বয়ংক্রিয়ভাবে Spring Boot দ্বারা সংযুক্ত করা হয়েছে:
// - ChatModel বিন
// - @Component ক্লাস থেকে সমস্ত @Tool পদ্ধতি
// - সেশন ব্যবস্থাপনার জন্য ChatMemoryProvider
```

নিম্নলিখিত চিত্রটি প্রতিটি অ্যানোটেশন বিশ্লেষণ করে দেখায় কীভাবে প্রতিটি অংশ AI কে বুঝতে সাহায্য করে কখন টুল কল করতে হবে এবং কী আর্গুমেন্ট পাঠাতে হবে:

<img src="../../../translated_images/bn/tool-definitions-anatomy.f6468546037cf28b.webp" alt="টুল সংজ্ঞার গঠন" width="800"/>

*টুল সংজ্ঞার গঠন — @Tool AI কে বলে কখন এটি ব্যবহার করতে হবে, @P প্রতিটি প্যারামিটার বর্ণনা করে, এবং @AiService স্টার্টআপে সবকিছু যুক্ত করে।*

### সিদ্ধান্ত গ্রহণ

যখন ব্যবহারকারী "সিয়াটলে আবহাওয়া কেমন?" জিজ্ঞাসা করে, তখন মডেল এলোমেলোভাবে কোন টুল বেছে নেয় না। এটি ব্যবহারকারীর উদ্দেশ্য প্রতিটি টুল বর্ণনার সাথে তুলনা করে, প্রাসঙ্গিকতার জন্য প্রতিটি স্কোর করে এবং সর্বোত্তম ম্যাচ নির্বাচন করে। এরপর সঠিক প্যারামিটার সহ একটি কাঠামোবদ্ধ ফাংশন কল তৈরি করে — এই ক্ষেত্রে, `location` এর মান `"Seattle"` সেট করে।

যদি কোন টুল ব্যবহারকারীর অনুরোধের সাথে মেলে না, মডেল তার নিজস্ব জ্ঞানের ওপর ভিত্তি করে উত্তর দেয়। যদি একাধিক টুল মেলে, তাহলে সবচেয়ে নির্দিষ্টটি বেছে নেয়।

<img src="../../../translated_images/bn/decision-making.409cd562e5cecc49.webp" alt="AI কোন টুল ব্যবহার করবে কিভাবে সিদ্ধান্ত নেয়" width="800"/>

*মডেল ব্যবহারকারীর উদ্দেশ্য অনুযায়ী প্রতিটি টুল মূল্যায়ন করে এবং সেরা ম্যাচ বেছে নেয় — এজন্য স্পষ্ট, সুনির্দিষ্ট টুল বর্ণনা লেখা গুরুত্বপূর্ণ।*

### প্রতিক্রিয়া তৈরি

মডেল আবহাওয়ার ডেটা পায় এবং সেটিকে ব্যবহারকারীর জন্য একটি স্বাভাবিক ভাষার প্রতিক্রিয়ায় রূপান্তর করে।

### আর্কিটেকচার: স্প্রিং বুট অটো-ওয়ারিং

এই মডিউলটি LangChain4j এর স্প্রিং বুট ইন্টিগ্রেশন ব্যবহার করে যেখানে ঘোষণামূলক `@AiService` ইন্টারফেস থাকে। স্টার্টআপের সময় স্প্রিং বুট সব `@Component` যা `@Tool` মেথড ধরে, আপনার `ChatModel` বীন, এবং `ChatMemoryProvider` আবিষ্কার করে — তারপর এগুলোকে একটি একক `Assistant` ইন্টারফেসে কোন অতিরিক্ত কাজ ছাড়াই যুক্ত করে।

<img src="../../../translated_images/bn/spring-boot-wiring.151321795988b04e.webp" alt="স্প্রিং বুট অটো-ওয়ারিং আর্কিটেকচার" width="800"/>

*@AiService ইন্টারফেস ChatModel, টুল কম্পোনেন্ট, এবং মেমোরি প্রোভাইডার একত্র করে — স্প্রিং বুট সমস্ত ওয়ারিং স্বয়ংক্রিয়ভাবে পরিচালনা করে।*

এই পদ্ধতির প্রধান সুবিধাসমূহ:

- **স্প্রিং বুট অটো-ওয়ারিং** — ChatModel এবং টুলস স্বয়ংক্রিয়ভাবে ইনজেক্ট করা হয়
- **@MemoryId প্যাটার্ন** — স্বয়ংক্রিয় সেশন ভিত্তিক মেমোরি ব্যবস্থাপনা
- **একক ইনস্ট্যান্স** — অ্যাসিস্ট্যান্ট একবার তৈরি করে আরও ভাল কর্মক্ষমতার জন্য reused
- **টাইপ-সেফ এক্সিকিউশন** — জাভা মেথড সরাসরি টাইপ কনভার্শন সহ কল করা হয়
- **মাল্টি-টার্ন অর্কেস্ট্রেশন** — টুল চেইনিং স্বয়ংক্রিয়ভাবে পরিচালনা করে
- **জিরো বয়লারপ্লেট** — ম্যানুয়াল `AiServices.builder()` কল বা মেমোরি HashMap দরকার হয় না

বিকল্প পদ্ধতি (ম্যানুয়াল `AiServices.builder()`) বেশি কোড প্রয়োজন এবং স্প্রিং বুট ইন্টিগ্রেশনের সুবিধাগুলো থেকে বঞ্চিত।

## টুল চেইনিং

**টুল চেইনিং** — যখন একটি প্রশ্নের জন্য একাধিক টুলের প্রয়োজন হয় তখন টুল-ভিত্তিক এজেন্টদের আসল শক্তি প্রকাশ পায়। প্রশ্ন করুন "সিয়াটলে ফারেনহাইটে আবহাওয়া কেমন?" এবং এজেন্ট স্বয়ংক্রিয়ভাবে দুইটি টুল চেইন করে: প্রথমে `getCurrentWeather` কল করে সেলসিয়াসে তাপমাত্রা পায়, তারপর সেই মান `celsiusToFahrenheit` এ পাঠিয়ে রূপান্তর করে — সব একটি আলাপের টার্নে।

<img src="../../../translated_images/bn/tool-chaining-example.538203e73d09dd82.webp" alt="টুল চেইনিং উদাহরণ" width="800"/>

*চেইনিং চলমান — এজেন্ট প্রথমে getCurrentWeather কল করে, তারপর সেলসিয়াস ফলাফল celsiusToFahrenheit এ পাঠায় এবং সমন্বিত উত্তর দেয়।*

রানিং অ্যাপ্লিকেশনে এটি এইরকম দেখায় — এজেন্ট একটি আলাপের টার্নে দুটি টুল কল চেইন করে:

<a href="images/tool-chaining.png"><img src="../../../translated_images/bn/tool-chaining.3b25af01967d6f7b.webp" alt="টুল চেইনিং" width="800" style="border: 1px solid #ddd; box-shadow: 0 2px 8px rgba(0,0,0,0.1);"/></a>

*বাস্তব অ্যাপ্লিকেশন আউটপুট — এজেন্ট স্বয়ংক্রিয়ভাবে getCurrentWeather → celsiusToFahrenheit একটি টার্নে চেইন করে।*

**সুন্দর ব্যর্থতা** — এমন একটি শহরের আবহাওয়া জানতে চাইুন যা মক ডেটায় নেই। টুল একটি ত্রুটি বার্তা দেয়, এবং AI বলে সাহায্য করতে পারে না তবে ক্র্যাশ করে না। টুলস সুরক্ষিতভাবে ব্যর্থ হয়।

<img src="../../../translated_images/bn/error-handling-flow.9a330ffc8ee0475c.webp" alt="ত্রুটি পরিচালনা প্রবাহ" width="800"/>

*যখন একটি টুল ব্যর্থ হয়, এজেন্ট ত্রুটি ধরতে পারে এবং ক্র্যাশের পরিবর্তে সাহায্যসূচক ব্যাখ্যা দেবে।*

এটি একটি আলাপের টার্নে ঘটে। এজেন্ট স্বতন্ত্রভাবে বহু টুল কল পরিচালনা করে।

## অ্যাপ্লিকেশন ব্যবহার

অ্যাপ্লিকেশন একটি ওয়েব ইন্টারফেস প্রদান করে যেখানে আপনি একটি AI এজেন্টের সাথে ইন্টারঅ্যাক্ট করতে পারেন যার কাছে আবহাওয়া এবং তাপমাত্রা রূপান্তর টুলস রয়েছে।

*AI Agent Tools ইন্টারফেস - টুলস ব্যবহার ও আলাপচারিতার দ্রুত উদাহরণ*

### সরল টুল ব্যবহারের চেষ্টা করুন
প্রারম্ভ করুন একটি সরল অনুরোধ দিয়ে: "১০০ ডিগ্রি ফারেনহাইটকে সেলসিয়াসে রূপান্তর করুন"। এজেন্টটি বুঝতে পারে যে এটি তাপমাত্রা রূপান্তর টুলটি ব্যবহার করতে হবে, সঠিক প্যারামিটার সহ কল করে এবং ফলাফল প্রদান করে। লক্ষ্য করুন এটি কতটা প্রাকৃতিক মনে হয় - আপনি কোনো টুল ব্যবহার করতে হবে বা কিভাবে কল করতে হবে তা নির্দিষ্ট করেননি।

### টুল চেইনিং পরীক্ষা

এখন কিছু বেশি জটিল চেষ্টা করুন: "সিয়াটেলের আবহাওয়া কেমন এবং এটিকে ফারেনহাইটে রূপান্তর করুন?" লক্ষ্য করুন এজেন্ট কিভাবে এটি ধাপে ধাপে করে। প্রথমে এটি আবহাওয়া পায় (যা সেলসিয়াসে থাকে), বুঝতে পারে ফারেনহাইটে রূপান্তর করতে হবে, রূপান্তর টুলটি কল করে, এবং দুটি ফলাফল একত্রিত করে একটি উত্তর তৈরি করে।

### কথোপকথনের প্রবাহ দেখুন

চ্যাট ইন্টারফেসটি কথোপকথন ইতিহাস সংরক্ষণ করে, যা আপনাকে বহুবারের ইন্টারঅ্যাকশন করার সুযোগ দেয়। আপনি পূর্ববর্তী সব প্রশ্ন ও উত্তর দেখতে পারেন, যা কথোপকথন ট্র্যাক করা এবং এজেন্ট কিভাবে একাধিক বিনিময়ের মধ্যে প্রসঙ্গ তৈরি করে তা বুঝতে সহজ করে।

<a href="images/tools-conversation-demo.png"><img src="../../../translated_images/bn/tools-conversation-demo.89f2ce9676080f59.webp" alt="একাধিক টুল কলসহ কথোপকথন" width="800" style="border: 1px solid #ddd; box-shadow: 0 2px 8px rgba(0,0,0,0.1);"/></a>

*সরল রূপান্তর, আবহাওয়া অনুসন্ধান এবং টুল চেইনিং দেখাচ্ছে বহুবারের কথোপকথন*

### বিভিন্ন অনুরোধের সঙ্গে পরীক্ষা করুন

বিভিন্ন সমন্বয় চেষ্টা করুন:
- আবহাওয়া অনুসন্ধান: "টোকিওর আবহাওয়া কেমন?"
- তাপমাত্রা রূপান্তর: "২৫°C কে কেলভিনে কত?"
- সমন্বিত প্রশ্ন: "পারিসের আবহাওয়া দেখুন এবং জানিয়ে দিন তা ২০°C এর উপরে কিনা"

লক্ষ্য করুন কিভাবে এজেন্ট প্রাকৃতিক ভাষা ব্যাখ্যা করে এবং সঠিক টুল কলের সাথে ম্যাচ করে।

## মূল ধারণা

### ReAct প্যাটার্ন (যুক্তি এবং কার্য)

এজেন্ট যুক্তি করার (কী করবেন সিদ্ধান্ত করা) এবং কার্যকর করার (টুল ব্যবহার) মাঝে পাল্টা চলে। এই প্যাটার্নটি স্বায়ত্তশাসিত সমস্যা সমাধান সম্ভব করে, শুধু নির্দেশ অনুসরণ নয়।

### টুল বর্ণনা গুরুত্বপূর্ণ

আপনার টুল বর্ণনার গুণগত মান সরাসরি প্রভাব ফেলে এজেন্ট সেগুলো কত ভালো ব্যবহার করে। পরিষ্কার, নির্দিষ্ট বর্ণনা মডেলকে বুঝতে সাহায্য করে কখন এবং কিভাবে প্রতিটি টুল কল করতে হবে।

### সেশন ব্যবস্থাপনা

`@MemoryId` অ্যানোটেশন স্বয়ংক্রিয় সেশন-ভিত্তিক স্মৃতি ব্যবস্থাপনা সক্ষম করে। প্রতিটি সেশন আইডি এর নিজস্ব `ChatMemory` ইনস্ট্যান্স পায় যা `ChatMemoryProvider` বিন দ্বারা পরিচালিত হয়, তাই একাধিক ব্যবহারকারী একই সাথে এজেন্টের সঙ্গে ইন্টারঅ্যাক্ট করতে পারে এবং তাদের কথোপকথন একে অপরের সাথে মেশে না।

<img src="../../../translated_images/bn/session-management.91ad819c6c89c400.webp" alt="@MemoryId সহ সেশন ব্যবস্থাপনা" width="800"/>

*প্রতিটি সেশন আইডি পৃথক কথোপকথন ইতিহাসের সাথে ম্যাপ করা থাকে — ব্যবহারকারীরা একে অপরের বার্তা দেখেন না।*

### ত্রুটি পরিচালনা

টুলগুলো ব্যর্থ হতে পারে — API সময়সীমা পেরিয়ে যেতে পারে, প্যারামিটার ভুল হতে পারে, বাহ্যিক সার্ভিস ডাউন হতে পারে। প্রোডাকশন এজেন্টের জন্য ত্রুটি পরিচালনা অপরিহার্য যাতে মডেল সমস্যা ব্যাখ্যা করতে বা বিকল্প চেষ্টা করতে পারে পুরো অ্যাপ্লিকেশন ক্র্যাশ না করে। যখন একটি টুল এক্সেপশন ছুড়ে ফেলে, LangChain4j এটিকে ধরে নেয় এবং ত্রুটির বার্তা মডেলের কাছে পাঠায়, যা স্বাভাবিক ভাষায় সমস্যা ব্যাখ্যা করতে পারে।

## উপলব্ধ টুলসমূহ

নিম্নের চিত্রে আপনি টুল তৈরির বিস্তৃত ইকোসিস্টেম দেখবেন। এই মডিউল আবহাওয়া এবং তাপমাত্রা টুল প্রদর্শন করে, কিন্তু একই `@Tool` প্যাটার্ন যেকোনো Java মেথডের ক্ষেত্রে কাজ করে — ডাটাবেস কোয়েরি থেকে পেমেন্ট প্রসেসিং পর্যন্ত।

<img src="../../../translated_images/bn/tool-ecosystem.aad3d74eaa14a44f.webp" alt="টুল ইকোসিস্টেম" width="800"/>

*`@Tool` দিয়ে অ্যানোটেট করা যেকোনো Java মেথড এআই এর জন্য পাওয়া যায় — প্যাটার্নটি ডাটাবেস, API, ইমেইল, ফাইল অপারেশন এবং আরো কিছুর ক্ষেত্রেও প্রযোজ্য।*

## কখন টুল-ভিত্তিক এজেন্ট ব্যবহার করবেন

<img src="../../../translated_images/bn/when-to-use-tools.51d1592d9cbdae9c.webp" alt="কখন টুল ব্যবহার করবেন" width="800"/>

*একটি দ্রুত সিদ্ধান্ত নির্দেশিকা — টুলগুলো রিয়েল-টাইম ডেটা, হিসাব-নিকাশ, এবং কার্যকলাপের জন্য; সাধারণ জ্ঞান ও সৃজনশীল কাজের জন্য নয়।*

**টুল ব্যবহার করুন যখন:**
- উত্তর দিতে রিয়েল-টাইম ডেটা লাগে (আবহাওয়া, শেয়ার দাম, ইনভেন্টরি)
- সরল গাণিতিক হিসাব ছাড়াও হিসাব করতে হবে
- ডাটাবেস বা API থেকে তথ্য নিতে হবে
- কার্যকলাপ নিতে হবে (ইমেইল পাঠানো, টিকেট তৈরি, রেকর্ড আপডেট)
- একাধিক তথ্য উৎস একত্রিত করতে হবে

**টুল ব্যবহার করবেন না যখন:**
- প্রশ্নের উত্তর সাধারণ জ্ঞান থেকে দেওয়া যায়
- প্রতিক্রিয়া শুধুমাত্র কথোপকথন মূলক
- টুল লেটেন্সির কারণে অভিজ্ঞতা ধীর হয়ে যাবে

## টুল এবং RAG এর তুলনা

মডিউল ০৩ এবং ০৪ উভয়ই এআই এর ক্ষমতা বাড়ায়, কিন্তু মৌলিকভাবে আলাদা উপায়ে। RAG মডেলকে **জ্ঞান** প্রবেশাধিকাংশ দেয় ডকুমেন্ট উদ্ধার করে। টুল মডেলকে **কার্যকলাপ** নেওয়ার ক্ষমতা দেয় ফাংশন কলের মাধ্যমে।

<img src="../../../translated_images/bn/tools-vs-rag.ad55ce10d7e4da87.webp" alt="টুল বনাম RAG তুলনা" width="800"/>

*RAG স্ট্যাটিক ডকুমেন্ট থেকে তথ্য উদ্ধার করে — টুল লাইভ ডেটা এবং কার্য সম্পাদন করে। অনেক প্রোডাকশন সিস্টেম উভয় একত্রে ব্যবহার করে।*

প্রয়োগে, অনেক প্রোডাকশন সিস্টেম উভয় পদ্ধতি একসঙ্গে ব্যবহার করে: RAG দিয়ে আপনার ডকুমেন্টেশনের ভিত্তিতে উত্তর ভিত্তি স্থাপন এবং টুল দিয়ে লাইভ ডেটা আনা বা অপারেশন করা।
