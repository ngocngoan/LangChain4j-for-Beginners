## আপনি যা শিখবেন

<img src="../../../translated_images/bn/what-youll-learn.c68269ac048503b2.webp" alt="আপনি যা শিখবেন" width="800"/>

পূর্ববর্তী মডিউলে, আপনি দেখেছেন কীভাবে মেমোরি কথোপকথনমূলক AI সক্ষম করে এবং গিটহাব মডেল ব্যবহার করে বেসিক ইন্টারঅ্যাকশন চালানো যায়। এখন আমরা ফোকাস করব আপনি কীভাবে প্রশ্ন করেন — প্রম্পটগুলোর উপর — Azure OpenAI এর GPT-5.2 ব্যবহার করে। আপনি যেভাবে প্রম্পট তৈরি করবেন তা আপনার পাওয়া প্রতিক্রিয়ার গুণমান ব্যাপকভাবে প্রভাবিত করে। আমরা শুরু করব মৌলিক প্রম্পটিং কৌশলগুলোর পর্যালোচনায়, এবং তারপর এগিয়ে যাব আটটি উন্নত প্যাটার্নে যা GPT-5.2 এর সক্ষমতাগুলো পুরোপুরি কাজে লাগায়।

আমরা GPT-5.2 ব্যবহার করব কারণ এটি যুক্তি পরিচালনা (reasoning control) প্রবর্তন করেছে - আপনি মডেলকে বলতে পারেন কতটা চিন্তা করতে হবে উত্তর দেওয়ার পূর্বে। এভাবে বিভিন্ন প্রম্পটিং কৌশলগুলি স্পষ্ট হয়ে উঠে এবং আপনি বুঝতে পারেন কখন কোন পদ্ধতি ব্যবহার করবেন। Azure এর GPT-5.2 এর জন্য গিটহাব মডেলের তুলনায় কম রেট সীমাবদ্ধতা থাকায় সেটিও আমাদের সুবিধা দেবে।

## পূর্বপ্রতিষ্ঠিত জ্ঞান

- মডিউল ০১ সম্পন্ন (Azure OpenAI সম্পদ মোতায়েন করা হয়েছে)
- রুট ডিরেক্টরিতে `.env` ফাইল রয়েছে Azure পরিচয়পত্র সহ (মডিউল ০১ এ `azd up` দ্বারা তৈরি)

> **নোট:** আপনি যদি মডিউল ০১ শেষ না করে থাকেন, প্রথমে সেখানে থাকা মোতায়েন নির্দেশিকা অনুসরণ করুন।

## প্রম্পট ইঞ্জিনিয়ারিং বোঝা

<img src="../../../translated_images/bn/what-is-prompt-engineering.5c392a228a1f5823.webp" alt="প্রম্পট ইঞ্জিনিয়ারিং কি?" width="800"/>

প্রম্পট ইঞ্জিনিয়ারিং হলো এমন ইনপুট টেক্সট ডিজাইন করা যা ক্রমাগতভাবে আপনার চাওয়া ফলাফল দেয়। শুধু প্রশ্ন করা নয়—কিন্তু অনুরোধ এমনভাবে গঠন করা যাতে মডেল স্পষ্টভাবে বুঝতে পারে আপনি কী চান এবং কীভাবে তা সরবরাহ করবেন।

এটিকে ভাবুন একজন সহকর্মীকে নির্দেশনা দেওয়ার মতো। "বাগ ঠিক করো" অস্পষ্ট। "UserService.java এর লাইন ৪৫-এ null pointer exception ঠিক করতে null চেক যোগ করো" স্পষ্ট নির্দেশ। ভাষার মডেলগুলোও একইভাবে কাজ করে—স্পষ্টতা ও গঠন গুরুত্বপূর্ণ।

<img src="../../../translated_images/bn/how-langchain4j-fits.dfff4b0aa5f7812d.webp" alt="LangChain4j কীভাবে ফিট করে" width="800"/>

LangChain4j কেই অবকাঠামো সরবরাহ করে — মডেল সংযোগ, মেমোরি, ও মেসেজ টাইপস — আর প্রম্পট প্যাটার্নগুলো কেবল সেই অবকাঠামোর মধ্য দিয়ে পাঠানো সুপরিকল্পিত পাঠ্য। মূল নির্মাণ ব্লক হলো `SystemMessage` (যা AI এর আচরণ ও রোল নির্ধারণ করে) এবং `UserMessage` (যা আপনার প্রকৃত অনুরোধ বহন করে)।

## প্রম্পট ইঞ্জিনিয়ারিংয়ের মূলনীতি

<img src="../../../translated_images/bn/five-patterns-overview.160f35045ffd2a94.webp" alt="প্রম্পট ইঞ্জিনিয়ারিংয়ের পাঁচটি প্যাটার্ন ওভারভিউ" width="800"/>

এই মডিউলের উন্নত প্যাটার্নে যাওয়ার আগে, আসুন পাঁচটি মৌলিক প্রম্পটিং কৌশল পর্যালোচনা করি। এগুলো হলো সেই ভিত্তি যা প্রতিটি প্রম্পট ইঞ্জিনিয়ারকে জানা থাকা উচিত। আপনি যদি ইতিমধ্যে [Quick Start মডিউল](../00-quick-start/README.md#2-prompt-patterns) অনুসরণ করে থাকেন, তাহলে এগুলো আপনি ইতিমধ্যে দেখেছেন — এরা সেই ধারণাগত কাঠামো।

### জিরো-শট প্রম্পটিং

সবচেয়ে সহজ পদ্ধতি: কোনো উদাহরণ ছাড়াই সরাসরি মডেলকে নির্দেশনা দেওয়া। মডেল তার প্রশিক্ষণের উপর পুরোপুরি নির্ভর করে কাজটি বুঝে এবং সম্পাদন করে। এটি সরল অনুরোধের জন্য ভালো কাজ করে, যেখানে প্রত্যাশিত আচরণ স্পষ্ট।

<img src="../../../translated_images/bn/zero-shot-prompting.7abc24228be84e6c.webp" alt="জিরো-শট প্রম্পটিং" width="800"/>

*উদাহরণ ছাড়া সরাসরি নির্দেশনা — মডেল নির্দেশনা থেকেই কাজটি বুঝে নেয়*

```java
String prompt = "Classify this sentiment: 'I absolutely loved the movie!'";
String response = model.chat(prompt);
// প্রতিক্রিয়া: "ইতিবাচক"
```

**কখন ব্যবহার করবেন:** সহজ শ্রেণিবিভাগ, সরাসরি প্রশ্ন, অনুবাদ, অথবা এমন কোনো কাজ যেখানে মডেল অতিরিক্ত গাইডলাইন ছাড়া পরিচালনা করতে পারে।

### ফিউ-শট প্রম্পটিং

মডেলকে অনুসরণ করতে হবে এমন প্যাটার্ন দেখাতে উদাহরণ দিন। মডেল আপনার উদাহরণ থেকে ইনপুট-আউটপুট ফরম্যাট শিখে নেয় এবং নতুন ইনপুটে প্রয়োগ করে। এটি এমন কাজের জন্য ধারাবাহিকতা উল্লেখযোগ্যভাবে বৃদ্ধি করে যেখানে কাঙ্খিত ফরম্যাট বা আচরণ স্পষ্ট নয়।

<img src="../../../translated_images/bn/few-shot-prompting.9d9eace1da88989a.webp" alt="ফিউ-শট প্রম্পটিং" width="800"/>

*উদাহরণ থেকে শেখা — মডেল প্যাটার্ন চিনে নেয় এবং নতুন ইনপুটে প্রয়োগ করে*

```java
String prompt = """
    Classify the sentiment as positive, negative, or neutral.
    
    Examples:
    Text: "This product exceeded my expectations!" → Positive
    Text: "It's okay, nothing special." → Neutral
    Text: "Waste of money, very disappointed." → Negative
    
    Now classify this:
    Text: "Best purchase I've made all year!"
    """;
String response = model.chat(prompt);
```

**কখন ব্যবহার করবেন:** কাস্টম শ্রেণিবিভাগ, ধারাবাহিক ফরম্যাটিং, ডোমেইন-নির্দিষ্ট কাজ, অথবা যখন জিরো-শট ফলাফল অসমঞ্জস হয়।

### চেইন অফ থট

মডেলকে ধাপে ধাপে তার যুক্তি দেখাতে বলুন। সরাসরি উত্তর দেওয়ার পরিবর্তে, মডেল সমস্যাটি বিভক্ত করে এবং প্রতিটি অংশ স্পষ্টভাবে কাজ করে। এটি গাণিতিক, যৌক্তিক, ও বহু-ধাপযুক্ত যুক্তি বের করার কাজের নির্ভুলতা বাড়ায়।

<img src="../../../translated_images/bn/chain-of-thought.5cff6630e2657e2a.webp" alt="চেইন অফ থট প্রম্পটিং" width="800"/>

*ধাপে ধাপে যুক্তি — জটিল সমস্যা স্পষ্ট যৌক্তিক ধাপে ভেঙে দেওয়া*

```java
String prompt = """
    Problem: A store has 15 apples. They sell 8 apples and then 
    receive a shipment of 12 more apples. How many apples do they have now?
    
    Let's solve this step-by-step:
    """;
String response = model.chat(prompt);
// মডেল দেখায়: ১৫ - ৮ = ৭, তারপর ৭ + ১২ = ১৯ টি আপেল
```

**কখন ব্যবহার করবেন:** গাণিতিক সমস্যা, যৌক্তিক ধাঁধা, ডিবাগিং, অথবা এমন কাজ যেখানে যুক্তি প্রক্রিয়া দেখানো নির্ভুলতা ও বিশ্বাসযোগ্যতা বাড়ায়।

### রোল-বেইসড প্রম্পটিং

প্রশ্ন করার আগে AI এর জন্য একটি ব্যক্তি বা রোল নির্ধারণ করুন। এটি পরিপ্রেক্ষিত দেয় যা উত্তরটির স্বর, গভীরতা, এবং ফোকাস নির্ধারণ করে। একটি "সফটওয়্যার স্থপতি" ভিন্ন পরামর্শ দেয় যেটা "জুনিয়র ডেভেলপার" বা "সিকিউরিটি অডিটর" দেবে।

<img src="../../../translated_images/bn/role-based-prompting.a806e1a73de6e3a4.webp" alt="রোল-বেইসড প্রম্পটিং" width="800"/>

*পরিপ্রেক্ষিত এবং রোল নির্ধারণ — একই প্রশ্ন ভিন্ন রোল অনুযায়ী ভিন্ন উত্তর পায়*

```java
String prompt = """
    You are an experienced software architect reviewing code.
    Provide a brief code review for this function:
    
    def calculate_total(items):
        total = 0
        for item in items:
            total = total + item['price']
        return total
    """;
String response = model.chat(prompt);
```

**কখন ব্যবহার করবেন:** কোড রিভিউ, টিউটরিং, ডোমেইন-নির্দিষ্ট বিশ্লেষণ, অথবা যখন আপনাকে নির্দিষ্ট দক্ষতা স্তর বা দৃষ্টিভঙ্গি অনুযায়ী উত্তর প্রয়োজন।

### প্রম্পট টেমপ্লেটস

ভেরিয়েবল প্লেসহোল্ডার সহ পুনরায় ব্যবহারযোগ্য প্রম্পট তৈরি করুন। প্রতিবার নতুন প্রম্পট লেখার পরিবর্তে, একবার টেমপ্লেট তৈরি করুন এবং বিভিন্ন মান পূরণ করুন। LangChain4j এর `PromptTemplate` ক্লাস `{{variable}}` সিনট্যাক্স দিয়ে এটি সহজ করে।

<img src="../../../translated_images/bn/prompt-templates.14bfc37d45f1a933.webp" alt="প্রম্পট টেমপ্লেটস" width="800"/>

*পুনরায় ব্যবহারযোগ্য প্রম্পট যার মধ্যে ভেরিয়েবল প্লেসহোল্ডার থাকে — একটি টেমপ্লেট, অনেক ব্যবহার*

```java
PromptTemplate template = PromptTemplate.from(
    "What's the best time to visit {{destination}} for {{activity}}?"
);

Prompt prompt = template.apply(Map.of(
    "destination", "Paris",
    "activity", "sightseeing"
));

String response = model.chat(prompt.text());
```

**কখন ব্যবহার করবেন:** বিভিন্ন ইনপুট সহ পুনরাবৃত্তি প্রশ্ন, ব্যাচ প্রসেসিং, পুনরায় ব্যবহারযোগ্য AI ওয়ার্কফ্লো তৈরি, অথবা যেকোনো পরিস্থিতি যেখানে প্রম্পট কাঠামো একই থাকে কিন্তু ডেটা পরিবর্তিত হয়।

এই পাঁচটি মূলনীতি আপনাকে অধিকাংশ প্রম্পটিং কাজের জন্য শক্তিশালী সরঞ্জাম দিতে পারে। এই মডিউলের বাকি অংশে আমরা এগুলো ভিত্তি করে **আটটি উন্নত প্যাটার্ন** নিয়ে কাজ করব যা GPT-5.2 এর যুক্তি নিয়ন্ত্রণ, স্ব-মূল্যায়ন, এবং গঠনিক আউটপুট সক্ষমতা কাজে লাগায়।

## উন্নত প্যাটার্নস

মূলনীতি কভার করার পর, চলুন এগিয়ে যাই এই মডিউলটিকে বিশেষ করে যেই আটটি উন্নত প্যাটার্নগুলো। সব সমস্যার জন্য একই পদ্ধতি প্রয়োজন হয় না। কিছু প্রশ্ন দ্রুত উত্তর চায়, আর কিছু গভীর চিন্তাভাবনা। কিছুতে দৃশ্যমান যুক্তি দরকার, অন্যগুলোতে শুধু ফলাফল। নিচের প্রতিটি প্যাটার্ন আলাদা প্রেক্ষাপটের জন্য অপ্টিমাইজ করা — আর GPT-5.2 এর যুক্তি নিয়ন্ত্রণ পার্থক্যগুলো স্পষ্ট করে তোলে।

<img src="../../../translated_images/bn/eight-patterns.fa1ebfdf16f71e9a.webp" alt="আটটি প্রম্পট ইঞ্জিনিয়ারিং প্যাটার্ন" width="800"/>

*আটটি প্রম্পট ইঞ্জিনিয়ারিং প্যাটার্নের ওভারভিউ এবং তাদের ব্যবহারের ক্ষেত্র*

<img src="../../../translated_images/bn/reasoning-control.5cf85f0fc1d0c1f3.webp" alt="GPT-5.2 এর সাথে যুক্তি নিয়ন্ত্রণ" width="800"/>

*GPT-5.2 এর যুক্তি নিয়ন্ত্রণ আপনাকে নির্দিষ্ট করতে দেয় মডেল কতটা চিন্তা করবে — দ্রুত সরাসরি উত্তর থেকে গভীর অনুসন্ধান পর্যন্ত*

<img src="../../../translated_images/bn/reasoning-effort.db4a3ba5b8e392c1.webp" alt="যুক্তি প্রচেষ্টার তুলনা" width="800"/>

*নিম্ন আগ্রহ (দ্রুত, সরাসরি) বনাম উচ্চ আগ্রহ (গভীর, অনুসন্ধানমূলক) যুক্তি পদ্ধতি*

**নিম্ন আগ্রহ (দ্রুত ও নিবদ্ধ)** - অতিসৌক্ষ্ম প্রশ্নের জন্য যেখানে আপনি দ্রুত এবং সরাসরি উত্তর চান। মডেল সর্বোচ্চ ২ ধাপ যুক্তি তোলে। গণনা, তথ্য অনুসন্ধান, বা সরল প্রশ্নের জন্য ব্যবহার করুন।

```java
String prompt = """
    <reasoning_effort>low</reasoning_effort>
    <instruction>maximum 2 reasoning steps</instruction>
    
    What is 15% of 200?
    """;

String response = chatModel.chat(prompt);
```

> 💡 **GitHub Copilot দিয়ে পরীক্ষা করুন:** খুলুন [`Gpt5PromptService.java`](../../../02-prompt-engineering/src/main/java/com/example/langchain4j/prompts/service/Gpt5PromptService.java) এবং জিজ্ঞাসা করুন:
> - "নিম্ন আগ্রহ এবং উচ্চ আগ্রহ প্রম্পটিং প্যাটার্নের পার্থক্য কী?"
> - "XML ট্যাগগুলি কীভাবে প্রম্পটের মাধ্যমে AI এর প্রতিক্রিয়াকে গঠন করতে সাহায্য করে?"
> - "স্ব-প্রতিফলন প্যাটার্ন কখন ব্যবহার করব এবং সরাসরি নির্দেশনা কখন?"

**উচ্চ আগ্রহ (গভীর ও বিস্তারিত)** - জটিল সমস্যার জন্য যেখানে আপনি ব্যাপক বিশ্লেষণ চান। মডেল গভীর অনুসন্ধান করে এবং বিস্তারিত যুক্তি দেখায়। সিস্টেম ডিজাইন, আর্কিট্যাকচার সিদ্ধান্ত, বা জটিল গবেষণায় ব্যবহার করুন।

```java
String prompt = """
    <reasoning_effort>high</reasoning_effort>
    <instruction>explore thoroughly, show detailed reasoning</instruction>
    
    Design a caching strategy for a high-traffic REST API.
    """;

String response = chatModel.chat(prompt);
```

**টাস্ক এক্সিকিউশন (ধাপে ধাপে অগ্রগতি)** - বহু-ধাপে কাজের জন্য। মডেল একটি পূর্ণ পরিকল্পনা দেয়, প্রতিটি ধাপ হাতে-কলমে করে বলে, শেষে সারসংক্ষেপ দেয়। মাইগ্রেশন, বাস্তবায়ন, বা যেকোনো বহু-ধাপ কর্মপ্রক্রিয়ার জন্য।

```java
String prompt = """
    <task>Create a REST endpoint for user registration</task>
    <preamble>Provide an upfront plan</preamble>
    <narration>Narrate each step as you work</narration>
    <summary>Summarize what was accomplished</summary>
    """;

String response = chatModel.chat(prompt);
```

চেইন-অফ-থট প্রম্পটিং স্পষ্টভাবে মডেলকে তার যুক্তি প্রক্রিয়া দেখাতে বলে, যা জটিল কাজের নির্ভুলতা বাড়ায়। ধাপে ধাপে বিভাজন মানুষ ও AI উভয়ের জন্য যৌক্তিকতা বুঝতে সাহায্য করে।

> **🤖 [GitHub Copilot](https://github.com/features/copilot) চ্যাট দিয়ে চেষ্টা করুন:** এই প্যাটার্ন সম্পর্কে প্রশ্ন করুন:
> - "দীর্ঘমেয়াদী অপারেশনগুলোর জন্য টাস্ক এক্সিকিউশন প্যাটার্ন কিভাবে মানিয়ে নেব?"
> - "প্রোডাকশন অ্যাপ্লিকেশনে টুল প্রিমাবলস গঠনের সেরা অভ্যাস কী?"
> - "UI তে মধ্যবর্তী অগ্রগতি আপডেট ক্যাপচার এবং প্রদর্শন কিভাবে করা যায়?"

<img src="../../../translated_images/bn/task-execution-pattern.9da3967750ab5c1e.webp" alt="টাস্ক এক্সিকিউশন প্যাটার্ন" width="800"/>

*পরিকল্পনা → সম্পাদন → সারসংক্ষেপ বহু-ধাপ কাজের ওয়ার্কফ্লো*

**স্ব-পর্যালোচনামূলক কোড** - প্রডাকশন মানের কোড তৈরি করার জন্য। মডেল কোড উৎপাদন করে, গুণগত মান যাচাই করে এবং পর্যায়ক্রমে উন্নত করে। নতুন ফিচার বা সার্ভিস নির্মাণে ব্যবহার করুন।

```java
String prompt = """
    <task>Create an email validation service</task>
    <quality_criteria>
    - Correct logic and error handling
    - Best practices (clean code, proper naming)
    - Performance optimization
    - Security considerations
    </quality_criteria>
    <instruction>Generate code, evaluate against criteria, improve iteratively</instruction>
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/bn/self-reflection-cycle.6f71101ca0bd28cc.webp" alt="স্ব-পর্যালোচনা চক্র" width="800"/>

*পর্যায়ক্রমিক উন্নতি লুপ - উৎপাদন, মূল্যায়ন, সমস্যা শনাক্তকরণ, উন্নয়ন, পুনরাবৃত্তি*

**গঠনিক বিশ্লেষণ** - ধারাবাহিক মূল্যায়নের জন্য। মডেল একটি নির্দিষ্ট ফ্রেমওয়ার্ক (সঠিকতা, অনুশীলন, কর্মদক্ষতা, নিরাপত্তা) দিয়ে কোড পর্যালোচনা করে। কোড রিভিউ বা গুণগতার মূল্যায়নে ব্যবহার করুন।

```java
String prompt = """
    <code>
    public List getUsers() {
        return database.query("SELECT * FROM users");
    }
    </code>
    
    <framework>
    Evaluate using these categories:
    1. Correctness - Logic and functionality
    2. Best Practices - Code quality
    3. Performance - Efficiency concerns
    4. Security - Vulnerabilities
    </framework>
    """;

String response = chatModel.chat(prompt);
```

> **🤖 [GitHub Copilot](https://github.com/features/copilot) চ্যাট দিয়ে চেষ্টা করুন:** গঠনিক বিশ্লেষণ সম্পর্কে প্রশ্ন করুন:
> - "ভিন্ন ধরনের কোড রিভিউয়ের জন্য বিশ্লেষণ ফ্রেমওয়ার্ক কিভাবে কাস্টমাইজ করব?"
> - "গঠনিক আউটপুট প্রোগ্রাম্যাটিক্যালি পার্স ও কার্যকর করার সেরা পথ কী?"
> - "বিভিন্ন রিভিউ সেশনে ধারাবাহিক গুরুত্ত্ব স্তর নিশ্চিত করতে কী করতে হবে?"

<img src="../../../translated_images/bn/structured-analysis-pattern.0af3b690b60cf2d6.webp" alt="গঠনিক বিশ্লেষণ প্যাটার্ন" width="800"/>

*গুরুত্ত্ব স্তর সহ ধারাবাহিক কোড রিভিউয়ের জন্য চার-শ্রেণির ফ্রেমওয়ার্ক*

**মাল্টি-টার্ন চ্যাট** - প্রাসঙ্গিক কথোপকথনের জন্য। মডেল পূর্ববর্তী মেসেজ মনে রাখে এবং তার উপর ভিত্তি করে তৈরি করে। ইন্টারেক্টিভ সাহায্য সেশন বা জটিল প্রশ্নোত্তরে ব্যবহার করুন।

```java
ChatMemory memory = MessageWindowChatMemory.withMaxMessages(10);

memory.add(UserMessage.from("What is Spring Boot?"));
AiMessage aiMessage1 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage1);

memory.add(UserMessage.from("Show me an example"));
AiMessage aiMessage2 = chatModel.chat(memory.messages()).aiMessage();
memory.add(aiMessage2);
```

<img src="../../../translated_images/bn/context-memory.dff30ad9fa78832a.webp" alt="প্রসঙ্গ স্মৃতি" width="800"/>

*একাধিক টার্নে কথোপকথন প্রসঙ্গ জমা হয় যতক্ষণ না টোকেন সীমা পৌঁছায়*

**ধাপে ধাপে যুক্তি** - দৃশ্যমান যৌক্তিকতা প্রয়োজন এমন সমস্যার জন্য। মডেল প্রতিটি ধাপের বিস্তারিত যৌক্তিকতা দেখায়। গাণিতিক সমস্যা, যৌক্তিক ধাঁধা বা চিন্তার প্রক্রিয়া বোঝার জন্য ব্যবহার করুন।

```java
String prompt = """
    <instruction>Show your reasoning step-by-step</instruction>
    
    If a train travels 120 km in 2 hours, then stops for 30 minutes,
    then travels another 90 km in 1.5 hours, what is the average speed
    for the entire journey including the stop?
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/bn/step-by-step-pattern.a99ea4ca1c48578c.webp" alt="ধাপে ধাপে প্যাটার্ন" width="800"/>

*সমস্যাগুলোকে স্পষ্ট যৌক্তিক ধাপে ভেঙে ফেলা*

**সীমাবদ্ধ আউটপুট** - নির্দিষ্ট ফরম্যাটের প্রয়োজনীয়তা ছাড়াই উত্তরগুলোর জন্য যেখানে ফরম্যাট ও দৈর্ঘ্যের নিয়মকানুন কঠোরভাবে অনুসরণ করা হয়। সারসংক্ষেপ বা নির্দিষ্ট আউটপুট কাঠামোর জন্য ব্যবহার করুন।

```java
String prompt = """
    <constraints>
    - Exactly 100 words
    - Bullet point format
    - Technical terms only
    </constraints>
    
    Summarize the key concepts of machine learning.
    """;

String response = chatModel.chat(prompt);
```

<img src="../../../translated_images/bn/constrained-output-pattern.0ce39a682a6795c2.webp" alt="সীমাবদ্ধ আউটপুট প্যাটার্ন" width="800"/>

*নির্দিষ্ট ফরম্যাট, দৈর্ঘ্য, এবং কাঠামো নিয়মগুলো প্রয়োগ*

**বিকল্প ১: স্প্রিং বুট ড্যাশবোর্ড ব্যবহার (VS Code ব্যবহারকারীদের জন্য সুপারিশকৃত)**

ডেভ কন্টেইনারে স্প্রিং বুট ড্যাশবোর্ড এক্সটেনশন অন্তর্ভুক্ত, যা সব স্প্রিং বুট অ্যাপ্লিকেশন পরিচালনার জন্য ভিজ্যুয়াল ইন্টারফেস দেয়। এটি VS Code এর Activity Bar এ বাম পাশে পাওয়া যাবে (স্প্রিং বুট আইকনে খুঁজুন)।
Spring Boot ড্যাশবোর্ড থেকে আপনি করতে পারেন:
- ওয়ার্কস্পেসে উপলব্ধ সমস্ত Spring Boot অ্যাপ্লিকেশন দেখতে পারেন
- একটি ক্লিকে অ্যাপ্লিকেশন শুরু/স্টপ করতে পারেন
- রিয়েল-টাইমে অ্যাপ্লিকেশনের লগ দেখতে পারেন
- অ্যাপ্লিকেশনের অবস্থা পর্যবেক্ষণ করতে পারেন

"prompt-engineering" এর পাশে প্লে বোতামে ক্লিক করে এই মডিউল শুরু করুন, অথবা সব মডিউল একসাথে শুরু করুন।

<img src="../../../translated_images/bn/dashboard.da2c2130c904aaf0.webp" alt="Spring Boot Dashboard" width="400"/>

**পছন্দ ২: শেল স্ক্রিপ্ট ব্যবহার**

সকল ওয়েব অ্যাপ্লিকেশন (মডিউল ০১-০৪) শুরু করুন:

**Bash:**
```bash
cd ..  # রুট ডিরেক্টরি থেকে
./start-all.sh
```

**PowerShell:**
```powershell
cd ..  # রুট ডিরেক্টরি থেকে
.\start-all.ps1
```

অথবা শুধুমাত্র এই মডিউল শুরু করুন:

**Bash:**
```bash
cd 02-prompt-engineering
./start.sh
```

**PowerShell:**
```powershell
cd 02-prompt-engineering
.\start.ps1
```

উভয় স্ক্রিপ্ট স্বয়ংক্রিয়ভাবে রুট `.env` ফাইল থেকে পরিবেশ পরিবর্তনশীলগুলো লোড করবে এবং যদি JAR ফাইল না থাকে, তাহলে সেগুলো বিল্ড করবে।

> **নোট:** আপনি যদি মডিউলগুলো শুরু করার আগে ম্যানুয়ালি সবগুলো বিল্ড করতে চান:

.\stop.ps1  # এই মডিউল শুধুমাত্র

## আপনি আসলে যা শিখছেন

**যুক্তি প্রচেষ্টা সবকিছু বদলে দেয়**

GPT-৫.২ প্রম্পটের মাধ্যমে আপনার কম্পিউটেশনাল প্রচেষ্টা নিয়ন্ত্রণ করতে দেয়। কম প্রচেষ্টা মানে দ্রুত উত্তর যেখানে কম অনুসন্ধান হয়। অধিক প্রচেষ্টা মানে মডেল গভীরভাবে ভাবতে সময় নেয়। আপনি শিখছেন কাজের জটিলতার সাথে প্রচেষ্টা মিলিয়ে ব্যবহার করতে - সহজ প্রশ্নে সময় নষ্ট করবেন না, তবে জটিল সিদ্ধান্তে দ্রুত কাজ করবেন না।

**গঠন আচরণ নিয়ন্ত্রণ করে**

প্রম্প্টের মধ্যে যে XML ট্যাগগুলো আছে লক্ষ্য করেছেন? সেগুলো সাজানোর জন্য নয়। মডেলগুলো কাঠামোবদ্ধ নির্দেশাবলী ফ্রি ফর্ম টেক্সটের তুলনায় বেশি নির্ভরযোগ্যভাবে অনুসরণ করে। যখন মাল্টি-স্টেপ প্রক্রিয়া বা জটিল যুক্তি দরকার হয়, গঠন মডেলকে বুঝতে সাহায্য করে যে কোথায় আছে এবং পরবর্তী কি।

<img src="../../../translated_images/bn/prompt-structure.a77763d63f4e2f89.webp" alt="Prompt Structure" width="800"/>

*পরিষ্কার বিভাগ এবং XML-স্টাইল সংগঠনের সঙ্গে একটি সু-গঠিত প্রম্প্টের রূপরেখা*

**স্ব-মূল্যায়নের মাধ্যমে গুণগত মান**

স্ব-প্রতিফলিত প্যাটার্নগুলো গুণগত মান স্পষ্ট করে তোলে। মডেল "ঠিক করছে" আশায় না থেকে আপনি ঠিক কী মানে "সঠিক": সঠিক যুক্তি, ত্রুটি হ্যান্ডলিং, কর্মক্ষমতা, নিরাপত্তা সেট করবেন। পরে মডেল নিজেই আউটপুট মূল্যায়ন করে উন্নতি করবে। এটি কোড তৈরি করার প্রক্রিয়াকে একটি জুয়াপট্টি থেকে পরিবর্তন করে।

**প্রসঙ্গ সীমিত**

মাল্টি-টার্ন আলাপচারিতার জন্য প্রতিটি অনুরোধের সাথে মেসেজ ইতিহাস অন্তর্ভুক্ত থাকে। কিন্তু একটি সীমা আছে - প্রতিটি মডেলের সর্বোচ্চ টোকেন সংখ্যা থাকে। আলাপ বাড়ার সাথে সাথে আপনাকে প্রাসঙ্গিক প্রসঙ্গ বজায় রাখতে কৌশল প্রয়োগ করতে হবে যাতে সীমা ছাড়িয়ে না যায়। এই মডিউল দেখায় কিভাবে মেমোরি কাজ করে; পরে আপনি শিখবেন কখন সারসংক্ষেপ করতে হবে, কখন ভুলে যেতে হবে, আর কখন পুনরুদ্ধার করতে হবে।
