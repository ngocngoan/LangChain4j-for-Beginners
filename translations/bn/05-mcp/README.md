## আপনি যা শিখবেন

আপনি কথোপকথনযোগ্য AI নির্মাণ করেছেন, প্রম্পটগুলি আয়ত্ত করেছেন, নথিপত্রে প্রতিক্রিয়াগুলি ভিত্তি করেছেন, এবং টুলস সহ এজেন্ট তৈরি করেছেন। কিন্তু সবগুলো টুল ছিল আপনার নির্দিষ্ট অ্যাপ্লিকেশনের জন্য কাস্টম তৈরি। কী হয় যদি আপনি আপনার AI-কে এমন একটি মানক ইকোসিস্টেমের টুলস অ্যাক্সেস দিতে পারেন যা কেউই তৈরি করতে ও শেয়ার করতে পারে? এই মডিউলে, আপনি শিখবেন মডেল প্রসঙ্গ প্রটোকল (MCP) এবং LangChain4j এর এজেন্টিক মডিউল ব্যবহার করে ঠিক এটা কিভাবে করা যায়। আমরা প্রথমে একটি সরল MCP ফাইল রিডার প্রদর্শন করব এবং তারপর দেখাব কিভাবে এটি সুপারভাইজার এজেন্ট প্যাটার্ন ব্যবহার করে উন্নত এজেন্টিক ওয়ার্কফ্লোতে সহজে সংহত হয়।

## MCP কী?

মডেল প্রসঙ্গ প্রটোকল (MCP) ঠিক এটাই প্রদান করে - একটি স্ট্যান্ডার্ড উপায় AI অ্যাপ্লিকেশনগুলোকে বাইরের টুলস আবিষ্কার এবং ব্যবহার করার জন্য। প্রতিটি ডেটা উৎস বা সেবার জন্য কাস্টম ইন্টিগ্রেশন লেখার পরিবর্তে, আপনি MCP সার্ভারগুলোর সাথে সংযোগ স্থাপন করেন যারা তাদের ক্ষমতাগুলো সুসংগত বিন্যাসে পরিবেশন করে। আপনার AI এজেন্ট তখন স্বয়ংক্রিয়ভাবে এই টুলস আবিষ্কার এবং ব্যবহার করতে পারে।

<img src="../../../translated_images/bn/mcp-comparison.9129a881ecf10ff5.webp" alt="MCP তুলনা" width="800"/>

*MCP এর আগে: জটিল পয়েন্ট-টু-পয়েন্ট ইন্টিগ্রেশন। MCP এর পরে: একটি প্রটোকল, অসীম সম্ভাবনা।*

MCP AI উন্নয়নে একটি মৌলিক সমস্যা সমাধান করে: প্রতিটি ইন্টিগ্রেশন কাস্টম। GitHub অ্যাক্সেস করতে চান? কাস্টম কোড। ফাইল পড়তে চান? কাস্টম কোড। ডেটাবেস কোয়েরি করতে চান? কাস্টম কোড। আর এই ইন্টিগ্রেশনগুলো অন্য AI অ্যাপ্লিকেশনগুলোর সঙ্গে কাজ করে না।

MCP এটাকে স্ট্যান্ডার্ড করে তোলে। একটি MCP সার্ভার স্পষ্ট বর্ণনা এবং স্কিমাসহ টুলস প্রকাশ করে। যেকোন MCP ক্লায়েন্ট সংযোগ করতে পারে, উপলব্ধ টুলস আবিষ্কার করতে পারে, এবং ব্যবহার করতে পারে। একবার বানান, সর্বত্র ব্যবহার করুন।

<img src="../../../translated_images/bn/mcp-architecture.b3156d787a4ceac9.webp" alt="MCP আর্কিটেকচার" width="800"/>

*মডেল প্রসঙ্গ প্রটোকল আর্কিটেকচার - স্ট্যান্ডার্ডাইজড টুল আবিষ্কার এবং কার্যনির্বাহী*

## MCP কীভাবে কাজ করে

<img src="../../../translated_images/bn/mcp-protocol-detail.01204e056f45308b.webp" alt="MCP প্রটোকল বিস্তারিত" width="800"/>

*MCP কিভাবে কাজ করে - ক্লায়েন্টগণ টুলস আবিষ্কার করে, JSON-RPC বার্তা আদান-প্রদান করে, এবং একটি ট্রান্সপোর্ট লেয়ারের মাধ্যমে অপারেশন সম্পন্ন করে।*

**সার্ভার-ক্লায়েন্ট আর্কিটেকচার**

MCP একটি ক্লায়েন্ট-সার্ভার মডেল ব্যবহার করে। সার্ভারগুলি টুল সরবরাহ করে - ফাইল পড়া, ডেটাবেস কোয়েরি, API কল। ক্লায়েন্টরা (আপনার AI অ্যাপ্লিকেশন) সার্ভারগুলোর সাথে সংযোগ করে এবং তাদের টুলস ব্যবহার করে।

LangChain4j এর সাথে MCP ব্যবহারের জন্য নিচের Maven ডিপেন্ডেন্সি যোগ করুন:

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-mcp</artifactId>
    <version>${langchain4j.version}</version>
</dependency>
```

**টুল আবিষ্কার**

যখন আপনার ক্লায়েন্ট একটি MCP সার্ভারের সাথে সংযোগ স্থাপন করে, তখন এটি জিজ্ঞাসা করে "আপনার কাছে কী কী টুল আছে?" সার্ভার উপলব্ধ টুলগুলোর একটি তালিকা প্রদান করে, প্রত্যেকটির বর্ণনা এবং প্যারামিটার স্কিমা সহ। আপনার AI এজেন্ট ব্যবহারকারীর অনুরোধের ভিত্তিতে কোন টুল ব্যবহার করা হবে তা নির্ধারণ করতে পারে।

<img src="../../../translated_images/bn/tool-discovery.07760a8a301a7832.webp" alt="MCP টুল আবিষ্কার" width="800"/>

*AI সূচনা সময় উপলব্ধ টুলগুলি আবিষ্কার করে — এখন এটি জানে কোন ক্ষমতাগুলি উপলব্ধ এবং কোনগুলো ব্যবহার করা উচিত তা সিদ্ধান্ত নিতে পারে।*

**ট্রান্সপোর্ট পদ্ধতি**

MCP বিভিন্ন ট্রান্সপোর্ট পদ্ধতি সমর্থন করে। এই মডিউল লোকাল প্রসেসের জন্য Stdio ট্রান্সপোর্ট প্রদর্শন করে:

<img src="../../../translated_images/bn/transport-mechanisms.2791ba7ee93cf020.webp" alt="ট্রান্সপোর্ট পদ্ধতি" width="800"/>

*MCP ট্রান্সপোর্ট পদ্ধতি: HTTP রিমোট সার্ভারের জন্য, Stdio লোকাল প্রসেসের জন্য*

**Stdio** - [StdioTransportDemo.java](../../../05-mcp/src/main/java/com/example/langchain4j/mcp/StdioTransportDemo.java)

লোকাল প্রসেসের জন্য। আপনার অ্যাপ্লিকেশন সাবপ্রসেস হিসেবে একটি সার্ভার তৈরি করে এবং স্ট্যান্ডার্ড ইনপুট/আউটপুট এর মাধ্যমে যোগাযোগ করে। ফাইলসিস্টেম অ্যাক্সেস বা কমান্ড-লাইন টুলসের জন্য উপযোগী।

```java
McpTransport stdioTransport = new StdioMcpTransport.Builder()
    .command(List.of(
        npmCmd, "exec",
        "@modelcontextprotocol/server-filesystem@2025.12.18",
        resourcesDir
    ))
    .logEvents(false)
    .build();
```

<img src="../../../translated_images/bn/stdio-transport-flow.45eaff4af2d81db4.webp" alt="Stdio ট্রান্সপোর্ট প্রবাহ" width="800"/>

*Stdio ট্রান্সপোর্টের কার্যপ্রণালী — আপনার অ্যাপ্লিকেশন MCP সার্ভারকে একটি চাইল্ড প্রসেস হিসেবে চালু করে এবং stdin/stdout পাইপের মাধ্যমে যোগাযোগ করে।*

## এজেন্টিক মডিউল

যেখানে MCP স্ট্যান্ডার্ডাইজড টুলস প্রদান করে, LangChain4j এর **এজেন্টিক মডিউল** এমন একটি ঘোষণামূলক উপায় প্রদান করে যা দ্বারা আপনি এজেন্ট তৈরি করতে পারেন যারা সেসব টুলস সমন্বয় করে। `@Agent` অ্যানোটেশন এবং `AgenticServices` ইন্টারফেসের মাধ্যমে এজেন্ট আচরণ নির্ধারণ করার অনুমতি দেয়, ইম্পেরেটিভ কোডের পরিবর্তে।

এই মডিউলে, আপনি **সুপারভাইজার এজেন্ট** প্যাটার্ন অন্বেষণ করবেন — একটি উন্নত এজেন্টিক AI পদ্ধতি যেখানে একটি "সুপারভাইজার" এজেন্ট ব্যবহারকারী অনুরোধের ভিত্তিতে কোন সাব-এজেন্টগুলোকে আহ্বান করা হবে তা গতিশীলভাবে সিদ্ধান্ত নেয়। আমরা উভয় ধারণাকে একত্র করব একটি সাব-এজেন্টের MCP চালিত ফাইল অ্যাক্সেস ক্ষমতা প্রদান করে।

এজেন্টিক মডিউল ব্যবহারের জন্য নিচের Maven ডিপেন্ডেন্সি যোগ করুন:

```xml
<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-agentic</artifactId>
    <version>${langchain4j.mcp.version}</version>
</dependency>
```

> **⚠️ পরীক্ষামূলক:** `langchain4j-agentic` মডিউল **পরীক্ষামূলক** এবং পরিবর্তনের অধীন। AI অ্যাসিস্ট্যান্ট তৈরির স্থায়ী পথ হলো `langchain4j-core` কাস্টম টুলস (Module 04) ব্যবহার করা।

## উদাহরণগুলি চালানো

### প্রয়োজনীয়তা

- Java 21+, Maven 3.9+
- Node.js 16+ এবং npm (MCP সার্ভারের জন্য)
- `.env` ফাইলে পরিবেশ ভেরিয়েবল কনফিগার করা (রুট ডিরেক্টরি থেকে):
  - `AZURE_OPENAI_ENDPOINT`, `AZURE_OPENAI_API_KEY`, `AZURE_OPENAI_DEPLOYMENT` (মডিউল 01-04 এর মতই)

> **নোট:** যদি আপনি এখনও পরিবেশ ভেরিয়েবল সেটআপ না করে থাকেন, দেখুন [Module 00 - Quick Start](../00-quick-start/README.md) নির্দেশিকা, অথবা `.env.example` কপি করে `.env` এ রাখুন এবং মানগুলো পূরণ করুন।

## দ্রুত শুরু

**VS Code ব্যবহার:** সরলভাবে এক্সপ্লোরারে যেকোনো ডেমো ফাইলের উপর রাইট-ক্লিক করে **"Run Java"** নির্বাচন করুন, অথবা রান এবং ডিবাগ প্যানেল থেকে লঞ্চ কনফিগারেশন ব্যবহার করুন (প্রথমে `.env` ফাইলে আপনার টোকেন যোগ করা আছে কিনা নিশ্চিত করুন)।

**Maven ব্যবহার:** বিকল্পভাবে, নিচের উদাহরণ ব্যবহার করে কমান্ড লাইন থেকে চালাতে পারেন।

### ফাইল অপারেশন (Stdio)

এটি লোকাল সাবপ্রসেস ভিত্তিক টুলস প্রদর্শন করে।

**✅ কোনও পূর্বশর্ত প্রয়োজন নেই** - MCP সার্ভার স্বয়ংক্রিয়ভাবে চালু হয়।

**স্টার্ট স্ক্রিপ্ট ব্যবহার (সর্বোত্তম):**

স্টার্ট স্ক্রিপ্ট স্বয়ংক্রিয়ভাবে রুট `.env` ফাইল থেকে পরিবেশ ভেরিয়েবল লোড করে:

**Bash:**
```bash
cd 05-mcp
chmod +x start-stdio.sh
./start-stdio.sh
```

**PowerShell:**
```powershell
cd 05-mcp
```

### সুপারভাইজার এজেন্ট

**সুপারভাইজার এজেন্ট প্যাটার্ন** একটি **নমনীয়** এজেন্টিক AI ফর্ম। একটি সুপারভাইজার একটি LLM ব্যবহার করে স্বায়ত্তশাসিতভাবে নির্ধারণ করে কোন এজেন্টগুলো ব্যবহারকারীর অনুরোধের ভিত্তিতে আহ্বান করা হবে। পরবর্তী উদাহরণে, আমরা MCP চালিত ফাইল অ্যাক্সেসকে একটি LLM এজেন্টের সাথে যুক্ত করব একটি সুপারভাইজড ফাইল রিড → রিপোর্ট ওয়ার্কফ্লো নির্মাণের জন্য।

ডেমোতে, `FileAgent` MCP ফাইলসিস্টেম টুলস ব্যবহার করে একটি ফাইল পড়ে, এবং `ReportAgent` একটি কাঠামোবদ্ধ রিপোর্ট তৈরি করে যার একটি নির্বাহী সারাংশ (১ বাক্য), ৩টি মূল পয়েন্ট এবং সুপারিশ থাকে। সুপারভাইজার এই প্রবাহ স্বয়ংক্রিয়ভাবে সমন্বয় করে:

<img src="../../../translated_images/bn/supervisor-agent-pattern.06275a41ae006ac8.webp" alt="সুপারভাইজার এজেন্ট প্যাটার্ন" width="800"/>

*সুপারভাইজার তার LLM ব্যবহার করে নির্ধারণ করে কোন এজেন্টগুলো আহ্বান করা হবে এবং কোন ক্রমে — কোনও হার্ডকোডেড রাউটিং প্রয়োজন নেই।*

আমাদের ফাইল-টু-রিপোর্ট পাইপলাইনের জন্য বাস্তব কর্মপ্রবাহটি নিম্নরূপ:

<img src="../../../translated_images/bn/file-report-workflow.649bb7a896800de9.webp" alt="ফাইল থেকে রিপোর্ট কর্মপ্রবাহ" width="800"/>

*FileAgent MCP টুলসের মাধ্যমে ফাইল পড়ে, তারপর ReportAgent কাঁচা কন্টেন্টকে কাঠামোবদ্ধ রিপোর্টে রূপান্তরিত করে।*

প্রত্যেক এজেন্ট তাদের আউটপুট **Agentic Scope** (শেয়ার করা মেমোরি) তে সংরক্ষণ করে, যা পরবর্তি এজেন্টদের পূর্বের ফলাফল অ্যাক্সেস করার সুযোগ দেয়। এটি প্রদর্শন করে কিভাবে MCP টুলস মসৃণভাবে এজেন্টিক ওয়ার্কফ্লোতে সংযুক্ত হয় — সুপারভাইজারকে জানার দরকার নেই *কিভাবে* ফাইল পড়া হয়, শুধু জানতে হবে `FileAgent` এটা করতে পারে।

#### সুপারভাইজার কীভাবে কাজ করে

```java
// ধাপ ১: FileAgent MCP টুল ব্যবহার করে ফাইল পড়ে
FileAgent fileAgent = AgenticServices.agentBuilder(FileAgent.class)
        .chatModel(model)
        .toolProvider(mcpToolProvider)  // ফাইল অপারেশনের জন্য MCP টুল আছে
        .build();

// ধাপ ২: ReportAgent গঠিত রিপোর্ট তৈরি করে
ReportAgent reportAgent = AgenticServices.agentBuilder(ReportAgent.class)
        .chatModel(model)
        .build();

// Supervisor ফাইল → রিপোর্ট কাজের প্রবাহ পরিচালনা করে
SupervisorAgent supervisor = AgenticServices.supervisorBuilder()
        .chatModel(model)
        .subAgents(fileAgent, reportAgent)
        .responseStrategy(SupervisorResponseStrategy.LAST)  // চূড়ান্ত রিপোর্ট ফেরত দেয়
        .build();

// Supervisor অনুরোধের ভিত্তিতে কোন এজেন্টগুলো সক্রিয় করতে হবে তা নির্ধারণ করে
String response = supervisor.invoke("Read the file at /path/file.txt and generate a report");
```

<img src="../../../translated_images/bn/response-strategies.3d0cea19d096bdf9.webp" alt="প্রতিক্রিয়া কৌশল" width="800"/>

*সুপারভাইজার চূড়ান্ত প্রতিক্রিয়া তৈরির জন্য তিনটি কৌশল ব্যবহার করে — আপনি নির্বাচিত করবেন শেষ এজেন্টের আউটপুট, সংশ্লেষিত সারাংশ, অথবা সর্বোচ্চ স্কোর পাওয়া বিকল্পের ভিত্তিতে।*

| কৌশল | বর্ণনা |
|----------|-------------|
| **LAST** | সুপারভাইজার শেষ সাব-এজেন্ট বা কল করা টুলের আউটপুট ফেরত দেয়। এটি কাজে লাগে যখন ওয়ার্কফ্লোর শেষ এজেন্টকে সম্পূর্ণ চুড়ান্ত উত্তর তৈরি করার জন্য ডিজাইন করা হয় (যেমন গবেষণা পাইপলাইনে "Summary Agent")। |
| **SUMMARY** | সুপারভাইজার নিজের অভ্যন্তরীণ ল্যাংগুয়েজ মডেল (LLM) ব্যবহার করে সমগ্র ইন্টারঅ্যাকশন এবং সকল সাব-এজেন্ট আউটপুটের সংক্ষিপ্ত সারাংশ তৈরী করে এবং তা চূড়ান্ত উত্তর হিসেবে প্রদান করে। এটি ব্যবহারকারীকে স্পষ্ট, সম্মিলিত উত্তর দেয়। |
| **SCORED** | সিস্টেম অভ্যন্তরীণ LLM ব্যবহার করে LAST প্রতিক্রিয়া এবং SUMMARY উভয়ের মুল্যায়ন করে, এবং ব্যবহারকারীর মূল অনুরোধের সাথে সর্বোত্তম স্কোর প্রাপ্ত আউটপুট ফেরত দেয়। |

#### এজেন্টিক মডিউলের বৈশিষ্ট্যের ব্যাখ্যা

উদাহরণে এজেন্টিক মডিউলের বেশ কিছু উন্নত বৈশিষ্ট্য প্রদর্শিত হয়েছে। আসুন Agentic Scope এবং Agent Listeners-এ একটু ঘনিষ্ঠভাবে নজর দিই।

**Agentic Scope** দেখায় শেয়ার করা মেমোরি যেখানে এজেন্টরা তাদের ফলাফল `@Agent(outputKey="...")` ব্যবহার করে সংরক্ষণ করে। এর সুবিধা:
- পরবর্তী এজেন্টরা পূর্ববর্তী এজেন্টের আউটপুট অ্যাক্সেস করতে পারে
- সুপারভাইজার চূড়ান্ত প্রতিক্রিয়া সংশ্লেষ করতে পারে
- আপনি পরিদর্শন করতে পারেন প্রতিটি এজেন্ট কী তৈরি করেছে

<img src="../../../translated_images/bn/agentic-scope.95ef488b6c1d02ef.webp" alt="Agentic Scope Shared Memory" width="800"/>

*Agentic Scope শেয়ার করা মেমোরি হিসেবে কাজ করে — FileAgent `fileContent` লিখে, ReportAgent পড়ে এবং `report` লেখে, আর আপনার কোড চূড়ান্ত ফলাফল পড়ে।*

```java
ResultWithAgenticScope<String> result = supervisor.invokeWithAgenticScope(request);
AgenticScope scope = result.agenticScope();
String fileContent = scope.readState("fileContent");  // FileAgent থেকে কাঁচা ফাইল ডেটা
String report = scope.readState("report");            // ReportAgent থেকে গঠিত প্রতিবেদন
```

**Agent Listeners** এজেন্টের কার্যনির্বাহী পর্যবেক্ষণ এবং ডিবাগিং সক্ষম করে। ডেমোতে আপনি যে ধাপে ধাপে আউটপুট দেখেন তা একটি AgentListener থেকে আসে যা প্রতিটি এজেন্ট আহ্বানে হুক করে:
- **beforeAgentInvocation** - যখন supervisor একজন এজেন্ট নির্বাচন করে তখন কল হয়, এর মাধ্যমে আপনি দেখতে পারেন কোন এজেন্টটি নির্বাচিত হয়েছে এবং কেন
- **afterAgentInvocation** - যখন একটি এজেন্ট সম্পন্ন হয়, তখন কল হয় এবং এর ফলাফল প্রদর্শন করে
- **inheritedBySubagents** - যখন সত্য হয়, তখন শ্রোতা হায়ারার্কির সকল এজেন্টকে মনিটর করে

<img src="../../../translated_images/bn/agent-listeners.784bfc403c80ea13.webp" alt="Agent Listeners Lifecycle" width="800"/>

*Agent Listeners এক্সিকিউশনের লাইফসাইকেলে হুক করে — মনিটর করে কখন এজেন্ট শুরু হয়, সম্পন্ন হয়, অথবা ত্রুটি ঘটে।*

```java
AgentListener monitor = new AgentListener() {
    private int step = 0;
    
    @Override
    public void beforeAgentInvocation(AgentRequest request) {
        step++;
        System.out.println("  +-- STEP " + step + ": " + request.agentName());
    }
    
    @Override
    public void afterAgentInvocation(AgentResponse response) {
        System.out.println("  +-- [OK] " + response.agentName() + " completed");
    }
    
    @Override
    public boolean inheritedBySubagents() {
        return true; // সমস্ত সাব-এজেন্টদের কাছে প্রেরণ করুন
    }
};
```

Supervisor প্যাটার্ন ছাড়াও, `langchain4j-agentic` মডিউল কয়েকটি শক্তিশালী ওয়ার্কফ্লো প্যাটার্ন এবং ফিচার সরবরাহ করে:

<img src="../../../translated_images/bn/workflow-patterns.82b2cc5b0c5edb22.webp" alt="Agent Workflow Patterns" width="800"/>

*এজেন্টস পরিচালনার জন্য পাঁচটি ওয়ার্কফ্লো প্যাটার্ন — সাধারণ ধারাবাহিক পাইপলাইন থেকে শুরু করে হিউম্যান-ইন-দ্য-লুপ অনুমোদন ওয়ার্কফ্লো পর্যন্ত।*

| প্যাটার্ন | বর্ণনা | ব্যবহার ক্ষেত্র |
|---------|-------------|----------|
| **Sequential** | এজেন্টস ধারাবাহিকভাবে চালান, আউটপুট পরবর্তী এজেন্টে প্রবাহিত হয় | পাইপলাইন: গবেষণা → বিশ্লেষণ → রিপোর্ট |
| **Parallel** | একসাথে একাধিক এজেন্ট চালান | স্বাধীন কাজ: আবহাওয়া + খবর + শেয়ার বাজার |
| **Loop** | শর্ত পূরণ না হওয়া পর্যন্ত পুনরাবৃত্তি করুন | গুণমান স্কোরিং: স্কোর ≥ 0.8 না হওয়া পর্যন্ত উন্নয়ন |
| **Conditional** | শর্ত অনুযায়ী রুটিং করুন | শ্রেণীবিভাগ → বিশেষজ্ঞ এজেন্টকে রুট করুন |
| **Human-in-the-Loop** | মানব পর্যবেক্ষণ যোগ করুন | অনুমোদন ওয়ার্কফ্লো, কনটেন্ট পর্যালোচনা |

## মূল ধারণা

এখন আপনি MCP এবং agentic মডিউল বাস্তবে কীভাবে কাজ করে তা অন্বেষণ করেছেন, চলুন প্রতিটি পদ্ধতি কখন ব্যবহার করবেন তার সারাংশ করি।

<img src="../../../translated_images/bn/mcp-ecosystem.2783c9cc5cfa07d2.webp" alt="MCP Ecosystem" width="800"/>

*MCP একটি সার্বজনীন প্রোটোকল ইকোসিস্টেম তৈরি করে — যেকোন MCP-সঙ্গত সার্ভার যেকোন MCP-সঙ্গত ক্লায়েন্টের সঙ্গে কাজ করে, অ্যাপ্লিকেশন জুড়ে টুল শেয়ারিং সক্ষম করে।*

**MCP** আদর্শ যখন আপনি বিদ্যমান টুল ইকোসিস্টেম ব্যবহার করতে চান, এমন টুল তৈরি করতে চান যেগুলো একাধিক অ্যাপ্লিকেশন শেয়ার করতে পারে, স্ট্যান্ডার্ড প্রোটোকল দিয়ে তৃতীয় পক্ষের সেবা একত্রিত করতে চান, অথবা কোড পরিবর্তন না করে টুল ইমপ্লিমেন্টেশন বদলাতে চান।

**Agentic Module** সেরা যখন আপনি ডিক্লারেটিভ এজেন্ট সংজ্ঞা চান `@Agent` অ্যানোটেশন সহ, ওয়ার্কফ্লো অর্কেস্ট্রেশন (ধারাবাহিক, লুপ, প্যারালাল) দরকার, আদেশ-ভিত্তিক কোডের পরিবর্তে ইন্টারফেজ-ভিত্তিক এজেন্ট ডিজাইন পছন্দ করেন, অথবা একাধিক এজেন্টের আউটপুট `outputKey` এর মাধ্যমে শেয়ার করতে চান।

**Supervisor Agent প্যাটার্ন** বিশেষভাবে কাজে লাগে যখন পূর্বানুমানযোগ্য ওয়ার্কফ্লো না থাকে এবং আপনি LLM-কে সিদ্ধান্ত নিতে দিতে চান, যখন আপনার একাধিক বিশেষায়িত এজেন্ট থাকে যাদের ডায়নামিক অর্কেস্ট্রেশন দরকার, কথোপকথন ব্যবস্থা তৈরি করছেন যা বিভিন্ন ক্ষমতায় রুট করে, অথবা সবচেয়ে নমনীয়, অভিযোজিত এজেন্ট আচরণ চান।

<img src="../../../translated_images/bn/custom-vs-mcp-tools.c4f9b6b1cb65d8a1.webp" alt="Custom Tools vs MCP Tools" width="800"/>

*কখন কাস্টম @Tool মেথড ব্যবহার করবেন বনাম MCP টুলস — অ্যাপ-নির্দিষ্ট লজিকের জন্য কাস্টম টুল যে পুরো টাইপ সেফটি দেয়, MCP টুলস স্ট্যান্ডার্ডাইজড ইন্টিগ্রেশন যা অ্যাপ্লিকেশন জুড়ে কাজ করে।*
