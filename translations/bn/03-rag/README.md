## আপনি কী শিখবেন

পূর্ববর্তী মডিউলগুলোতে, আপনি শিখেছেন কীভাবে AI এর সাথে আলাপচারিতা করবেন এবং কীভাবে আপনার প্রম্পটগুলো কার্যকরভাবে গঠন করবেন। কিন্তু একটি মৌলিক সীমাবদ্ধতা রয়েছে: ভাষা মডেলগুলি কেবলমাত্র শেখানো তথ্য জানে। তারা আপনার কোম্পানির নীতি, প্রকল্পের ডকুমেন্টেশন বা যেকোনো তথ্যের প্রশ্নের উত্তর দিতে পারে না যা তাদের প্রশিক্ষণে নেই।

RAG (রিট্রিভাল-অগমেন্টেড জেনারেশন) এই সমস্যা সমাধান করে। মডেলকে আপনার তথ্য শেখানোর পরিবর্তে (যা ব্যয়বহুল এবং অকার্যকর), আপনি এটিকে আপনার ডকুমেন্টগুলোর মাধ্যমে সার্চ করার ক্ষমতা দেন। কেউ যখন প্রশ্ন করে, সিস্টেম প্রাসঙ্গিক তথ্য খুঁজে বের করে এবং প্রম্পটে অন্তর্ভুক্ত করে। মডেল তখন ঐ উদ্ধারকৃত প্রসঙ্গের ভিত্তিতে উত্তর দেয়।

RAG কে ভাবুন যেন মডেলকে একটি রেফারেন্স লাইব্রেরি দেয়া হয়েছে। যখন আপনি প্রশ্ন করেন, সিস্টেম:

1. **ব্যবহারকারীর প্রশ্ন** - আপনি একটি প্রশ্ন করেন
2. **এম্বেডিং** - আপনার প্রশ্নকে ভেক্টরে রূপান্তর করে
3. **ভেক্টর সার্চ** - মিলযুক্ত ডকুমেন্ট চাঙ্ক খুঁজে বের করে
4. **কন্টেক্সট অ্যাসেম্বলি** - প্রম্পটে প্রাসঙ্গিক চাঙ্কগুলো যোগ করে
5. **প্রতিক্রিয়া** - এলএলএম প্রাপ্ত প্রসঙ্গের ভিত্তিতে উত্তর তৈরি করে

এটি মডেলের উত্তরকে তার প্রশিক্ষণগত জ্ঞানের পরিবর্তে আপনার প্রকৃত তথ্যের উপরে ভিত্তি করে।

## RAG বোঝা

নীচের ডায়াগ্রামটি মূল ধারণা ব্যাখ্যা করে: মডেলের প্রশিক্ষণ ডেটার উপর নির্ভর করার পরিবর্তে, RAG প্রতিটি উত্তরের আগে আপনার ডকুমেন্টগুলোর একটি রেফারেন্স লাইব্রেরি প্রদান করে।

এই মডিউলের বাকি অংশ প্রতিটি ধাপে বিস্তারিতভাবে আলোচনা করে, কোডসহ যা আপনি চালাতে ও পরিবর্তন করতে পারেন।

```java
// আপলোড করা ফাইলটি পার্স করুন এবং এটিকে একটি LangChain4j ডকুমেন্টে আবৃত করুন
Document document = Document.from(content, metadata);

// ৩০০-টোকেন অংশে ভাগ করুন ৩০-টোকেন ওভারল্যাপ সহ
DocumentSplitter splitter = DocumentSplitters
    .recursive(300, 30);

List<TextSegment> segments = splitter.split(document);
```

### এম্বেডিং তৈরি

[LangChainRagConfig.java](../../../03-rag/src/main/java/com/example/langchain4j/rag/config/LangChainRagConfig.java)

প্রত্যেক চাঙ্ককে একটি সংখ্যাসূচক প্রতিনিধিত্বে বা এম্বেডিংয়ে রূপান্তরিত করা হয় — যা মৌলিকভাবে একটি গাণিতিক ফিঙ্গারপ্রিন্ট যা টেক্সটের অর্থ ধারণ করে। সদৃশ টেক্সট সমজাতীয় এম্বেডিং তৈরি করে।

```java
@Bean
public EmbeddingModel embeddingModel() {
    return OpenAiOfficialEmbeddingModel.builder()
        .baseUrl(azureOpenAiEndpoint)
        .apiKey(azureOpenAiKey)
        .modelName(azureEmbeddingDeploymentName)
        .build();
}

EmbeddingStore<TextSegment> embeddingStore = 
    new InMemoryEmbeddingStore<>();
```

নীচের ক্লাস ডায়াগ্রামে দেখানো হয়েছে কিভাবে এই LangChain4j কম্পোনেন্টগুলো সংযুক্ত। `OpenAiOfficialEmbeddingModel` টেক্সটকে ভেক্টরে পরিণত করে, `InMemoryEmbeddingStore` ভেক্টরগুলো তাদের মূল `TextSegment` ডেটার সাথে ধরে রাখে, এবং `EmbeddingSearchRequest` পুনঃপ্রাপ্তির প্যারামিটার যেমন `maxResults` ও `minScore` নিয়ন্ত্রণ করে:

### সেম্যান্টিক সার্চ

[RagService.java](../../../03-rag/src/main/java/com/example/langchain4j/rag/service/RagService.java)

আপনি যখন প্রশ্ন করেন, আপনার প্রশ্নও একটি এম্বেডিংয়ে রূপান্তরিত হয়। সিস্টেম আপনার প্রশ্নের এম্বেডিং তুলনা করে সমস্ত ডকুমেন্ট চাঙ্কের এম্বেডিংয়ের সাথে। এটি খুঁজে বের করে সবচেয়ে সদৃশ অর্থবোধক চাঙ্কগুলো — কেবল মিলিত কীওয়ার্ড নয়, বরং প্রকৃত সেম্যান্টিক সদৃশ্যতা।

```java
Embedding queryEmbedding = embeddingModel.embed(question).content();

EmbeddingSearchRequest searchRequest = EmbeddingSearchRequest.builder()
    .queryEmbedding(queryEmbedding)
    .maxResults(5)
    .minScore(0.5)
    .build();

EmbeddingSearchResult<TextSegment> searchResult = embeddingStore.search(searchRequest);
List<EmbeddingMatch<TextSegment>> matches = searchResult.matches();

for (EmbeddingMatch<TextSegment> match : matches) {
    String relevantText = match.embedded().text();
    double score = match.score();
}
```

### উত্তর জেনারেশন

[RagService.java](../../../03-rag/src/main/java/com/example/langchain4j/rag/service/RagService.java)

সবচেয়ে প্রাসঙ্গিক চাঙ্কগুলো একত্রিত করে একটি গঠিত প্রম্পট তৈরী করা হয় যাতে স্পষ্ট নির্দেশনা, উদ্ধারকৃত প্রসঙ্গ এবং ব্যবহারকারীর প্রশ্ন থাকে। মডেল ঐ নির্দিষ্ট চাঙ্কগুলো পড়ে এবং সেই তথ্যের ভিত্তিতে উত্তর দেয় — এটি শুধুমাত্র সামনে রাখা তথ্য ব্যবহার করতে পারে, যা ভুল উত্তরের সম্ভাবনা কমায়।

```java
String context = matches.stream()
    .map(match -> match.embedded().text())
    .collect(Collectors.joining("\n\n"));

String prompt = String.format("""
    Answer the question based on the following context.
    If the answer cannot be found in the context, say so.

    Context:
    %s

    Question: %s

    Answer:""", context, request.question());

String answer = chatModel.chat(prompt);
```

**বিকল্প ১: Spring Boot ড্যাশবোর্ড ব্যবহার করুন (VS Code ব্যবহারকারীদের জন্য সুপারিশকৃত)**

ডেভ কন্টেইনারে Spring Boot ড্যাশবোর্ড এক্সটেনশন অন্তর্ভুক্ত রয়েছে, যা সকল Spring Boot অ্যাপ্লিকেশন পরিচালনার জন্য ভিজ্যুয়াল ইন্টারফেস দেয়। VS Code এর বাম পাশে অ্যাক্টিভিটি বারে (Spring Boot আইকন দেখুন) এটির অবস্থান।

Spring Boot ড্যাশবোর্ড থেকে আপনি:
- ওয়ার্কস্পেসে সমস্ত Spring Boot অ্যাপ্লিকেশন দেখতে পারবেন
- এক ক্লিকে অ্যাপ্লিকেশন শুরু/বন্ধ করতে পারবেন
- রিয়েল-টাইম লগ দেখতে পারবেন
- অ্যাপ্লিকেশন স্ট্যাটাস মনিটর করতে পারবেন

শুধু "rag" এর পাশে প্লে বোতামে ক্লিক করে এই মডিউল শুরু করুন, অথবা সকল মডিউল একসাথে চালু করুন।

## অ্যাপ্লিকেশন ব্যবহার

অ্যাপ্লিকেশনটি ডকুমেন্ট আপলোড ও প্রশ্ন করার জন্য একটি ওয়েব ইন্টারফেস প্রদান করে।

<a href="images/rag-homepage.png"><img src="../../../translated_images/bn/rag-homepage.d90eb5ce1b3caa94.webp" alt="RAG Application Interface" width="800" style="border: 1px solid #ddd; box-shadow: 0 2px 8px rgba(0,0,0,0.1);"/></a>

*RAG অ্যাপ্লিকেশন ইন্টারফেস - ডকুমেন্ট আপলোড করুন এবং প্রশ্ন করুন*

### একটি ডকুমেন্ট আপলোড করুন

প্রথমে একটি ডকুমেন্ট আপলোড করুন - টেক্সট ফাইল পরীক্ষা করার জন্য সবচেয়ে ভালো। এই ডিরেক্টরিতে একটি `sample-document.txt` দেওয়া আছে যাতে LangChain4j ফিচার, RAG ইমপ্লিমেন্টেশন এবং সেরা অভ্যাসসমূহ সম্পর্কে তথ্য রয়েছে - সিস্টেম পরীক্ষা করার জন্য আদর্শ।

সিস্টেম আপনার ডকুমেন্ট প্রক্রিয়া করে, চাঙ্কে ভাগ করে এবং প্রতিটি চাঙ্কের জন্য এম্বেডিং তৈরি করে। এটি স্বয়ংক্রিয়ভাবে ঘটে যখন আপনি আপলোড করেন।

### প্রশ্ন জিজ্ঞাসা করুন

এখন ডকুমেন্টের বিষয়বস্তু সম্পর্কে সুনির্দিষ্ট প্রশ্ন করুন। এমন কিছু চেষ্টা করুন যা স্পষ্টভাবে ডকুমেন্টে উল্লেখ আছে। সিস্টেম প্রাসঙ্গিক চাঙ্ক খুঁজে বের করে, প্রম্পটে অন্তর্ভুক্ত করে এবং উত্তর তৈরি করে।

### সূত্র পরীক্ষা করুন

প্রত্যেক উত্তরের সাথে সূত্রের রেফারেন্স এবং সদৃশ্যতা স্কোর থাকে। এই স্কোরগুলো (০ থেকে ১) দেখায় প্রশ্নের সাথে প্রতিটি চাঙ্ক কতটা প্রাসঙ্গিক ছিল। উচ্চ স্কোর মানে ভালো মিল। এটি আপনাকে উত্তর যাচাই করার সুযোগ দেয়।

<a href="images/rag-query-results.png"><img src="../../../translated_images/bn/rag-query-results.6d69fcec5397f355.webp" alt="RAG Query Results" width="800" style="border: 1px solid #ddd; box-shadow: 0 2px 8px rgba(0,0,0,0.1);"/></a>

*প্রশ্নের ফলাফল দেখাচ্ছে উত্তর, সূত্র এবং প্রাসঙ্গিকতা স্কোরের সাথে*

### প্রশ্ন নিয়ে পরীক্ষা করুন

বিভিন্ন ধরনের প্রশ্ন চেষ্টা করুন:
- সুনির্দিষ্ট তথ্য: "মূল বিষয় কি?"
- তুলনা: "X এবং Y এর মধ্যে পার্থক্য কী?"
- সারসংক্ষেপ: "Z সম্পর্কে মূল পয়েন্টগুলো সংক্ষেপ করুন"

দেখুন কিভাবে প্রাসঙ্গিকতা স্কোর প্রশ্নের ডকুমেন্ট সামগ্রীর সঙ্গে মিলের ওপর ভিত্তি করে পরিবর্তিত হয়।

## মূল ধারণাসমূহ

### চাঙ্কিং স্ট্র্যাটেজি

ডকুমেন্টগুলো ৩০০ টোকেনের চাঙ্কে ভাগ করা হয়, যার মধ্যে ৩০ টোকেন ওভারল্যাপ থাকে। এই সমতার সাহায্যে প্রতিটি চাঙ্ক যথেষ্ট প্রসঙ্গ ধরে রাখতে পারে এবং ছোট আকারে থাকার কারণে একাধিক চাঙ্ক প্রম্পটে অন্তর্ভুক্ত করা যায়।

### সদৃশ্যতা স্কোর

প্রত্যেক উদ্ধারকৃত চাঙ্কের সাথে ০ থেকে ১ এর মধ্যে একটি সদৃশ্যতা স্কোর থাকে যা ব্যবহারকারীর প্রশ্নের সাথে তার মিলের মাত্রা নির্দেশ করে। নিচের ডায়াগ্রাম স্কোরের রেঞ্জগুলো এবং সিস্টেম কীভাবে সেগুলো ব্যবহার করে ফলাফলগুলো ফিল্টার করে তা দেখায়:

<img src="../../../translated_images/bn/similarity-scores.b0716aa911abf7f0.webp" alt="Similarity Scores" width="800"/>

স্কোর রেঞ্জ:
- ০.৭-১.০: অতি প্রাসঙ্গিক, সঠিক মিল
- ০.৫-০.৭: প্রাসঙ্গিক, ভালো প্রসঙ্গ
- ০.৫ এর নিচে: ফিল্টার করা হয়েছে, খুব অপ্রাসঙ্গিক

গুণগতমান নিশ্চিত করতে সিস্টেম শুধুমাত্র মিনিমাম থ্রেশহোল্ডের উপরে থাকা চাঙ্কই পুনরুদ্ধার করে।

### ইন-মেমোরি স্টোরেজ

সহজতার জন্য এই মডিউল ইন-মেমোরি স্টোরেজ ব্যবহার করে। অ্যাপ্লিকেশন রিস্টার্ট করলে আপলোড করা ডকুমেন্ট হারিয়ে যাবে। প্রোডাকশন সিস্টেম পূর্ণ-টেকসই ভেক্টর ডাটাবেস যেমন Qdrant বা Azure AI সার্চ ব্যবহার করে।

### কন্টেক্সট উইন্ডো ম্যানেজমেন্ট

প্রত্যেক মডেলের একটি সর্বোচ্চ কন্টেক্সট উইন্ডো থাকে। বড় ডকুমেন্ট থেকে সব চাঙ্ক প্রম্পটে রাখা সম্ভব নয়। সিস্টেম শীর্ষ N (ডিফল্ট ৫) সবচেয়ে প্রাসঙ্গিক চাঙ্ক পুনরুদ্ধার করে যাতে সীমা বজায় থাকে এবং যথেষ্ট প্রসঙ্গ দেয় সঠিক উত্তর দিতে।

## কখন RAG প্রযোজ্য

RAG সবসময় সঠিক পদ্ধতি নয়। নিচের সিদ্ধান্ত গাইড আপনাকে সাহায্য করবে কখন RAG ব্যবহার করবেন এবং কখন সরাসরি প্রম্পটে বিষয়বস্তু অন্তর্ভুক্ত রাখা বা মডেলের অন্তর্নির্মিত জ্ঞানের উপর নির্ভর করা যথেষ্ট হবে:

<img src="../../../translated_images/bn/when-to-use-rag.1016223f6fea26bc.webp" alt="When to Use RAG" width="800"/>

**RAG ব্যবহার করুন যখন:**
- মালিকানা নথি সম্পর্কিত প্রশ্নের উত্তর দেওয়া
- তথ্য প্রায়ই পরিবর্তিত হয় (নীতিমালা, মূল্য, স্পেসিফিকেশন)
- সঠিকতার জন্য উৎস উল্লেখ প্রয়োজন
- বিষয়বস্তু একক প্রম্পটে ফিট করার জন্য খুব বড়
- আপনার প্রমাণযোগ্য, ভিত্তিহীন উত্তর প্রয়োজন

**যখন RAG ব্যবহার করবেন না:**
- প্রশ্নের জন্য মডেলের কাছে ইতিমধ্যেই সাধারণ জ্ঞানের প্রয়োজন
- বাস্তব-সময়ের ডেটা প্রয়োজন (RAG আপলোড করা নথির উপর কাজ করে)
- বিষয়বস্তু পর্যাপ্ত ছোট যা সরাসরি প্রম্পটে অন্তর্ভুক্ত করা যায়
