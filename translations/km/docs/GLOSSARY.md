## ចំណុចមូលដ្ឋាន

**ភ្នាក់ងារ AI** - ប្រព័ន្ធដែលប្រើ AI ដើម្បីគិតយល់ និងប្រព្រឹត្តិអំពើដោយផ្ទាល់ខ្លួន។ [Module 04](../04-tools/README.md)

**ខ្សែភាព** - ចំណាត់តាងនៃប្រតិបត្តិការដែលលទ្ធផលដឹកនាំទៅជំហានបន្ទាប់។

**ការបំបែក (Chunking)** - បំបែកឯកសារជាទំព័រតូចៗ។ ជារឿយៗ៖ ៣០០-៥០០ តូកិន ជាមួយការជាប់គ្នា។ [Module 03](../03-rag/README.md)

**បង្អួចបរិបទ** - ចំនួនតូកិនអតិបរមាដែលម៉ូដែលអាចដំណើរការ។ GPT-5.2៖ ៤០០K តូកិន (រហូតដល់ ២៧២K ទិន្នន័យចូល, ១២៨K លទ្ធផលចេញ).

**កំណត់តាង (Embeddings)** - វ៉ិកទ័រសង្គ្រោះចំនួនដែលតំណាងអត្ថន័យអត្ថបទ។ [Module 03](../03-rag/README.md)

**ការហៅមុខងារ (Function Calling)** - ម៉ូដែលបង្កើតសំណើរ ទ្រង់ទ្រាយរចនាសម្ព័ន្ធដើម្បីហៅមុខងារបន្ទាប់ពីក្រៅ។ [Module 04](../04-tools/README.md)

**ការមើលឃើញមិនត្រឹមត្រូវ (Hallucination)** - នៅពេលម៉ូដែលបង្កើតព័ត៌មានខុសប្លែកប៉ុន្តែសមស្រប។

**Prompt** - អត្ថបទបញ្ចូលទៅម៉ូដែលភាសា។ [Module 02](../02-prompt-engineering/README.md)

**ស្វែងរក តាមអត្ថន័យ Semantic Search** - ស្វែងរកតាមអត្ថន័យដោយកំណត់តាង មិនមែនតាមពាក្យគន្លឹះ។ [Module 03](../03-rag/README.md)

**មានស្ថានភាព (Stateful) ទល់នឹង គ្មានស្ថានភាព (Stateless)** - Stateless៖ គ្មានចងចាំ។ Stateful៖ រក្សាបទសម្ភាសន៍ជាចំណងចាំ។ [Module 01](../01-introduction/README.md)

**តូកិន (Tokens)** - ឯកតាអត្ថបទមូលដ្ឋានដែលម៉ូដែលដំណើរការ។ មានឥទ្ធិពលដល់កម្រៃនិងកំណត់។ [Module 01](../01-introduction/README.md)

**ការបង្រួបបង្រួមឧបករណ៍ (Tool Chaining)** - ដំណើរការឧបករណ៍ជាប់ៗដែលលទ្ធផលជួយដឹកនាំហៅជំហានបន្ទាប់។ [Module 04](../04-tools/README.md)

## សមាសភាគ LangChain4j

**AiServices** - បង្កើតចំណុចប្រទាក់សេវាកម្ម AI ប្រភេទសុវត្ថិភាព។

**OpenAiOfficialChatModel** - អតិថិជនរួម​សម្រាប់ម៉ូដែល OpenAI និង Azure OpenAI។

**OpenAiOfficialEmbeddingModel** - បង្កើតកំណត់តាងដោយប្រើអតិថិជន OpenAI Official (គាំទ្រ OpenAI និង Azure OpenAI ទាំងពីរ)។

**ChatModel** - ចំណុចប្រទាក់សំខាន់សម្រាប់ម៉ូដែលភាសា។

**ChatMemory** - រក្សាបទសម្ភាសន៍ជាចំណងចាំ។

**ContentRetriever** - រកយកចំណុចឯកសារសម្រាប់ RAG។

**DocumentSplitter** - បំបែកឯកសារ​ឲ្យក្លាយជាផ្នែក។

**EmbeddingModel** - បម្លែងអត្ថបទទៅជាវ៉ិកទ័រលេខ។

**EmbeddingStore** - រក្សាទុកនិងយកកំណត់តាងវិញ។

**MessageWindowChatMemory** - រក្សាបង្អួចសារបច្ចុប្បន្នជាមួយ ប្លក់បិទ។

**PromptTemplate** - បង្កើត prompt អាចប្រើបន្តជាមួយភាគតំណាង `{{variable}}`។

**TextSegment** - ផ្នែកអត្ថបទជាមួយទិន្នន័យមេតាដាតា។ ប្រើក្នុង RAG។

**ToolExecutionRequest** - តំណាងសំណើរអនុវត្តឧបករណ៍។

**UserMessage / AiMessage / SystemMessage** - ប្រភេទសារបទសម្ភាសន៍។

## គំនិត AI/ML

**ការសិក្សា Few-Shot** - ផ្តល់ឧទាហរណ៍ក្នុង prompt។ [Module 02](../02-prompt-engineering/README.md)

**ម៉ូដែលភាសាធំ (LLM)** - ម៉ូដែល AI ដែលបានហ្វឹកហាត់លើទិន្នន័យអត្ថបទធំទូលាយ។

**កំរិតការគិត (Reasoning Effort)** - ព៉ារ៉ាម៉ែត្រ GPT-5.2 ដែលគ្រប់គ្រងជម្រៅនៃការគិត។ [Module 02](../02-prompt-engineering/README.md)

**សីតុណ្ហភាព (Temperature)** - គ្រប់គ្រងភាពចៃឆៃនៃលទ្ធផល។ ទាប=កំណត់ត្រា ឡើងខ្ពស់=ច្នៃប្រឌិត។

**មូលដ្ឋានទិន្នន័យ វ៉ិកទ័រ (Vector Database)** - មូលដ្ឋានទិន្នន័យឯកទេសសម្រាប់កំណត់តាង។ [Module 03](../03-rag/README.md)

**ការសិក្សា Zero-Shot** - អនុវត្តន៍ភារកិច្ចដោយគ្មានឧទាហរណ៍។ [Module 02](../02-prompt-engineering/README.md)

## ការក្សារព្រៀង - [Module 00](../00-quick-start/README.md)

**ការពារ ជ្រៅក្នុងជម្រៅ (Defense in Depth)** - វិធីសាស្រ្តសន្តិសុខប៉ាន់ប្រមាណជាច្រើនជាតិដែលបញ្ចូលការក្សារព្រៀងកម្រិតកម្មវិធីជាមួយជម្រៅត្រួតពិនិត្យសុវត្ថិភាពអ្នកផ្គត់ផ្គង់។

**ការបង្ហាញខ្លាំង (Hard Block)** - អ្នកផ្គត់ផ្គង់បោះបង់កំហុស HTTP 400 សម្រាប់ការបំពានខ្លាំងៗរបស់មាតិកា។

**InputGuardrail** - ចំណុចប្រទាក់ LangChain4j សម្រាប់ផ្ទៀងផ្ទាត់ការបញ្ចូលអ្នកប្រើមុនវាចូលដល់ LLM។ រក្សាទុកថវិកា និងពេលវេលាដោយការបណ្ដេញ prompt ដែលមានគ្រោះថ្នាក់ឆាប់។

**InputGuardrailResult** - ប្រភេទត្រឡប់សម្រាប់ការផ្ទៀងផ្ទាត់ guardrail៖ `success()` ឬ `fatal("reason")`។

**OutputGuardrail** - ចំណុចប្រទាក់សម្រាប់ផ្ទៀងផ្ទាត់ចម្លើយ AI មុនត្រឡប់ទៅអ្នកប្រើប្រាស់។

**ជម្រៅត្រួតពិនិត្យសុវត្ថិភាពអ្នកផ្គត់ផ្គង់** - ត្រួតពិនិត្យមាតិកាដែលសាងសង់ដោយអ្នកផ្គត់ផ្គង់ AI (ឧ. GitHub Models) ដែលចាប់ការបំពាននៅកម្រិត API។

**ការបដិសេធទន់ភ្លន់ (Soft Refusal)** - ម៉ូដែលបដិសេធយ៉ាងល្អប្រសើរដោយមិនបង្ហាញកំហុស។

## ការដឹកនាំ Prompt - [Module 02](../02-prompt-engineering/README.md)

**ចងប្រសួត Chain-of-Thought** - គិតជាជំហានទាំងមូលសម្រាប់ភាពត្រឹមត្រូវល្អប្រសើរ។

**លទ្ធផលដាក់កំណត់ (Constrained Output)** - អនុវត្តទ្រង់ទ្រាយឬរចនាសម្ព័ន្ធជាក់លាក់។

**ការត្រៀមខ្ពស់ (High Eagerness)** - លំនាំ GPT-5.2 សម្រាប់ការគិតជ្រៅល្អ។

**ការត្រៀមទាប (Low Eagerness)** - លំនាំ GPT-5.2 សម្រាប់ចម្លើយរហ័ស។

**ការសន្ទនាច្រើនជុំ (Multi-Turn Conversation)** - រក្សាបរិបទទាំងមូលនៅពេលចរចា។

**Prompting ជាប្រភេទតួនាទី (Role-Based Prompting)** - កំណត់ផ្ទាល់ខ្លួនម៉ូដែលតាមសាររបៀបប្រព័ន្ធ។

**ការបញ្ចេញចំណាស់ខ្លួន (Self-Reflection)** - ម៉ូដែលវាយតម្លៃ និងកែលម្អលទ្ធផល។

**វិភាគរចនាសម្ព័ន្ធ (Structured Analysis)** - ស៊ើបការវាយតម្លៃកំណត់។

**លំនាំអនុវត្តភារកិច្ច (Task Execution Pattern)** - គំរោង → អនុវត្ត → សង្ខេប។

## RAG (ការបង្កើតជួយដោយការទាញយក) - [Module 03](../03-rag/README.md)

**បណ្ដាញដំណើរការឯកសារ (Document Processing Pipeline)** - ផ្ទុក → បំបែក → បង្រួប → រក្សាទុក។

**ឃ្លាំងបង្រួបក្នុងនឹកស្មារតី (In-Memory Embedding Store)** - ការផ្ទុកមិនថ្នាក់ថ្នងសម្រាប់ធ្វើតេស្ត។

**RAG** - បញ្ចូលការទាញយកជាមួយការបង្កើត ដើម្បីធានាចម្លើយមានគោលដៅ។

**ពិន្ទុស្រដៀងគ្នា (Similarity Score)** - វិមាត្រពិន្ទុ (0-1) នៃអត្ថន័យស្រដៀងគ្នា។

**យោងប្រភព (Source Reference)** - ទិន្នន័យពីមាតិកាដែលបានទាញយក។

## ភ្នាក់ងារ និងឧបករណ៍ - [Module 04](../04-tools/README.md)

**@Tool Annotation** - សម្គាល់វិធីសាស្រ្ត Java ជាឧបករណ៍អាចហៅដោយ AI។

**លំនាំ ReAct** - គិត → ប្រព្រឹត្ត → ស្ទង់មើល → ជម្រុញទៀត។

**គ្រប់គ្រងសម័យ (Session Management)** - បរិបទបំបែកសម្រាប់អ្នកប្រើផ្សេងៗ។

**ឧបករណ៍ (Tool)** - មុខងារភ្នាក់ងារ AI អាចហៅបាន។

**ការពិពណ៌នាឧបករណ៍ (Tool Description)** - ឯកសារពីគោលបំណង និងប៉ារ៉ាម៉ែត្រឧបករណ៍។

## ម៉ូឌុល Agentic - [Module 05](../05-mcp/README.md)

**@Agent Annotation** - សម្គាល់ចំណុចប្រទាក់ជាភ្នាក់ងារ AI ជាមួយនឹងការកំណត់ទំរង់អនុវត្តន៍មិនប្រែប្រួល។

**Agent Listener** - Hook សម្រាប់ត្រួតពិនិត្យអនុវត្តភ្នាក់ងារ តាម `beforeAgentInvocation()` និង `afterAgentInvocation()`។

**Agentic Scope** - ចំណងចាំចែករំលែកដែលភ្នាក់ងាររក្សាទុកលទ្ធផលប្រើ `outputKey` សម្រាប់ភ្នាក់ងារដទៃទៀត។

**AgenticServices** - រោងចក្រ​ក្នុងការបង្កើតភ្នាក់ងារដោយប្រើ `agentBuilder()` និង `supervisorBuilder()`។

**លំនាំ Workflow មានលក្ខខណ្ឌ (Conditional Workflow)** - ផ្លូវទៅភ្នាក់ងារពិសេសនានាតាមលក្ខខណ្ឌ។

**មនុស្សក្នុងបន្ទាត់ (Human-in-the-Loop)** - លំនាំធ្វើការ បញ្ចូលចំណុចពិនិត្យមនុស្សសម្រាប់អនុម័ត ឬពិនិត្យមាតិកា។

**langchain4j-agentic** - Maven dependency សម្រាប់ការសង់ភ្នាក់ងារដោយប្រើការបញ្ចេញសេចក្តីថ្លែង (experimental)។

**លំនាំជុំកំណត់ (Loop Workflow)** - ច្រៀងអនុវត្តភ្នាក់ងារឡើងវិញរហូតបានលក្ខខណ្ឌ (ឧ. ពិន្ទុគុណភាព ≥ 0.8)។

**outputKey** - ប៉ារ៉ាម៉ែត្រសម្គាល់ភ្នាក់ងារដែលកំណត់ទីតាំងរក្សាលទ្ធផលក្នុង Agentic Scope។

**លំនាំដំណើរការពហុបរិច្ឆេទ (Parallel Workflow)** - ប្រតិបត្តិភ្នាក់ងារច្រើនជាលំនាំសម័យសម្រាប់ការងារឯករាជ្យ។

**យុទ្ធសាស្រ្តចម្លើយ (Response Strategy)** - របៀបដែលអ្នកដឹកនាំសម្រេចចម្លើយចុងក្រោយ៖ LAST, SUMMARY, ឬ SCORED។

**លំនាំដំណើរការតាមលំដាប់ (Sequential Workflow)** - អនុវត្តភ្នាក់ងារតាមលំដាប់ដែលលទ្ធផលចេញទៅជំហានបន្ទាប់។

**លំនាំ Supervisor Agent** - លំនាំ agentic 진보ដែលអ្នកដឹកនាំ LLM សម្រេចចិត្ត δυναμικά អំពី sub-agents ដែលត្រូវហៅ។

## ពProtocol សម្រាប់បរិបទម៉ូដែល (MCP) - [Module 05](../05-mcp/README.md)

**langchain4j-mcp** - Maven dependency សម្រាប់ការរួមបញ្ចូល MCP ក្នុង LangChain4j។

**MCP** - ប្រព័ន្ធបរិបទម៉ូដែល: ស្តង់ដារសម្រាប់ភ្ជាប់កម្មវិធី AI ទៅឧបករណ៍ក្រៅ។ សង់ម្តង ប្រើឲ្យទូទៅ។

**MCP Client** - កម្មវិធីភ្ជាប់ទៅម៉ាស៊ីនបម្រើ MCP ដើម្បីស្វែងរកនិងប្រើឧបករណ៍។

**MCP Server** - សេវាកម្មបង្ហាញឧបករណ៍តាម MCP ជាមួយពិពណ៌នាច្បាស់លាស់ និងស្គីម៉ា៉ប៉ារ៉ាម៉ែត្រ។

**McpToolProvider** - សមាសភាគ LangChain4j ដែលបាយឧបករណ៍ MCP សម្រាប់ប្រើក្នុងសេវាកម្ម AI និងភ្នាក់ងារ។

**McpTransport** - ចំណុចប្រទាក់សម្រាប់ការទំនាក់ទំនង MCP។ ការអនុវត្តមាន Stdio និង HTTP។

**Stdio Transport** - ដំណើរការបញ្ជូនក្នុងស្រុកតាម stdin/stdout។ ប្រយោជន៍សម្រាប់ចូលប្រើប្រព័ន្ធឯកសារ ឬឧបករណ៍ command-line។

**StdioMcpTransport** - អនុវត្ត LangChain4j ដែលបើកម៉ាស៊ីនបម្រើ MCP ជាកម្មវិធីរង។

**ការស្វែងរកឧបករណ៍ (Tool Discovery)** - អតិថិជនសួរសំណួរម៉ាស៊ីនបម្រើ ស្ដីពីឧបករណ៍ដែលមានជាមួយពិពណ៌នា និងស្គីម៉ា។
